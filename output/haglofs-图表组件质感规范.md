# haglofs 图表组件质感规范

> 确保新图表组件与现有 56 个组件质感一致

---

## 1. 设计宪法检查清单

每个新组件必须通过 4 条铁律检查：

### 法则 1：信息本质驱动形态

| 图表类型 | 信息本质 | 形态要求 |
|----------|----------|----------|
| Line Chart | 趋势 | 线条清晰，网格克制，重点在数据点 |
| Area Chart | 累积 | 填充区域表达累积量，不喧宾夺主 |
| Bar Chart | 对比 | 柱子高度表达对比，间距大于柱宽 |
| Donut Chart | 占比 | 中心留空放总计数字，弧段表达占比 |

**检查**：删掉装饰（渐变、阴影、3D），信息传达损失 ≤5%？如果是，合格。

### 法则 2：每个元素必须挣到自己的像素

| 元素 | 必要性 | 判断标准 |
|------|--------|----------|
| 坐标轴 | ✅ 必须 | 没有则无法读数 |
| 轴标签 | ✅ 必须 | 没有则无法理解含义 |
| 网格线 | ⚠️ 可选 | 1px 发丝线，帮助读数；删除后损失 >5% 则保留 |
| 图例 | ⚠️ 可选 | 单数据集不需要，多数据集必须 |
| 数据标签 | ⚠️ 可选 | 仅在关键数据点显示，不堆砌 |
| 渐变/阴影 | ❌ 禁止 | 纯装饰，删除无损失 |

**检查**：每个元素都删过一遍了吗？删除后损失 >5% 才保留。

### 法则 3：字体角色是信息性质的翻译器

| 元素 | 字体 | 字号 | 样式 |
|------|------|------|------|
| 图表标题 | Playfair | 24-36px | 正常 |
| 轴标题 | Inter | 12-14px | 正常 |
| 轴标签 | JetBrains Mono | 10-12px | uppercase |
| 数据标签 | JetBrains Mono | 10-12px | 正常 |
| 图例文字 | Inter | 12px | 正常 |
| 注释文字 | Inter | 11px | 正常 |

**检查**：有没有混用字体？Mono 是否只用于精确数据？

### 法则 4：层次靠留白+色调+1px 线

| 层次关系 | 处理方式 |
|----------|----------|
| 图表与图表之间 | 24-32px 间距（--s-4xl） |
| 图表与文字之间 | 16-24px 间距（--s-3xl） |
| 数据点之间 | 1px 发丝线分隔 |
| 背景层次 | cream 差异 + 间距，不用阴影 |

**检查**：有没有用阴影造层次？有没有用渐变造层次？

## 2. 质感对标

### 2.1 对标现有组件

| 新组件 | 对标现有组件 | 质感要点 |
|--------|-------------|----------|
| Line Chart | Tension Grid | 线条清晰，网格克制 |
| Area Chart | Pipeline | 区域填充表达流向/累积 |
| Bar Chart | Facet Bars | 柱子高度表达对比 |
| Donut Chart | Ring | 中心留空，弧段表达占比 |

### 2.2 质感特征清单

**必须有的**：
- [ ] 哑光质感（无渐变、无高光）
- [ ] 1px 发丝线（网格、边框）
- [ ] Mono 字体用于数据标签
- [ ] Playfair 用于标题
- [ ] 96-120px section 留白
- [ ] 深底反相走 `.section--dark` 通用机制

**必须没有的**：
- [ ] 渐变填充
- [ ] 阴影效果
- [ ] 3D 效果
- [ ] 圆角（结构件 0px，内容卡 4px）
- [ ] 装饰性元素

## 3. 深底反相规范

### 3.1 统一机制

```css
/* 深底通用反相（父级作用域） */
.section--dark { color: var(--text-inverse); }
.section--dark .text-secondary { color: var(--text-inverse-2); }
.section--dark .text-tertiary { color: var(--text-inverse-3); }
```

### 3.2 图表组件专属规则（仅 1-2 条）

```css
/* Line Chart 深底专属 */
.section--dark .haglofs-line-chart .grid-line { stroke: var(--border-rest); }
.section--dark .haglofs-line-chart .data-line { stroke: var(--color-moss); }

/* Donut Chart 深底专属 */
.section--dark .haglofs-donut-chart .center-text { fill: var(--text-inverse); }
```

**检查**：组件是否只写了 1-2 条专属规则？其他是否走通用机制？

## 4. Radius 公约

| 组件 | Radius | 类别 | 理由 |
|------|--------|------|------|
| Line Chart | 0px | 结构件 | 结构靠对齐，圆角破坏秩序 |
| Area Chart | 0px | 结构件 | 同上 |
| Bar Chart | 0px | 结构件 | 同上 |
| Donut Chart | 4px | 内容卡 | 极轻微圆角暗示"这是一个物体" |
| Histogram | 0px | 结构件 | 同 Line Chart |
| Sankey | 0px | 结构件 | 同上 |
| Radar | 4px | 内容卡 | 同 Donut |

**检查**：Radius 是否按类别选取？有没有混用？

## 5. 间距规范

### 5.1 Token 真源间距

```
--s-2xs: 2px
--s-xs: 4px
--s-sm: 8px
--s-md: 16px
--s-lg: 24px
--s-xl: 32px
--s-2xl: 48px
--s-3xl: 64px
--s-4xl: 96px
--s-5xl: 120px
```

### 5.2 图表内部间距

| 元素 | 间距 | Token |
|------|------|-------|
| 图表与标题 | 16px | --s-md |
| 图表与轴标签 | 8px | --s-sm |
| 轴标签与轴线 | 4px | --s-xs |
| 图例与图表 | 16px | --s-md |
| 数据标签与数据点 | 4px | --s-xs |

### 5.3 图表与外部间距

| 元素 | 间距 | Token |
|------|------|-------|
| 图表与图表 | 24-32px | --s-lg ~ --s-xl |
| 图表与段落文字 | 16-24px | --s-md ~ --s-lg |
| 图表与 section 边界 | 48-96px | --s-2xl ~ --s-4xl |

---

## 6. 验证清单

每个新组件完成后，逐项检查：

### 质感检查
- [ ] 哑光质感（无渐变、无高光、无阴影）
- [ ] 1px 发丝线（网格、边框）
- [ ] 字体角色正确（Playfair 标题 / Mono 数据 / Inter 正文）
- [ ] 间距使用 token（不硬编码）

### 结构检查
- [ ] Radius 按类别选取（0px 或 4px）
- [ ] 深底走 `.section--dark` 通用机制
- [ ] 专属规则 ≤2 条

### 功能检查
- [ ] 信息传达清晰
- [ ] 响应式表现正常
- [ ] 与现有组件风格一致

### 组合检查
- [ ] 与 Hero 组合时视觉连贯
- [ ] 与 Editorial 组件混用时节奏自然
- [ ] 禁止组合检查通过
