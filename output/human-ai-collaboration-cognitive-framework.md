# 人机协同认知框架：判断力的四层传递模型

> 2026-07-13 整理。源自HRBP Codex应用指南的多轮推演，综合getnote笔记库（40万次Claude Code对话分析、判断档案方法论、Agent群聊协作方案、数字分身知识体系等）提炼而成。
>
> 这份文档的定位：不是外部资料汇总（见《人机协同落地：系统深度研究报告》），也不是实操工具手册（见《人机协同Agent使用方法论》），而是**认知框架层**——回答"人机协同的本质是什么，边界怎么判断，怎么随时间演进"。

---

## 核心命题

> **人机协同的本质，不是静态地划"AI干什么、人干什么"，而是动态地把人的判断力，从脑子里传递到AI可执行的流程里。**

判断说不清楚，AI 就只能做最浅层的事（写JD、润色邮件）。
判断说得清楚，AI 就能做深层的事（招聘漏斗分析、绩效质检、组织诊断）。
**判断说得越清楚，能交出去的越多，自己就能往越高的层次走。**

这个命题有五条交叉验证：

| 来源 | 验证内容 |
|------|---------|
| 40万次Claude Code对话分析 | 专业度 = 指令精准度 + 验证要求 + 纠偏能力（全是"把判断说清楚"的不同侧面） |
| 判断档案方法论 | 直觉→显式化→可复用，整个过程就是"把判断说清楚" |
| HRBP应用指南 | 核心竞争力 = "能不能把业务问题拆成清楚的数据、流程和动作" |
| Agent群聊协作方案 | Agent的价值 = "你配置的判断框架有多清晰，它就有多有用" |
| Cursor Team Kit | 17个skill的本质 = 把团队的隐性判断固化成可复用工具 |

## 四层分工模型

分工不是一张静态表，是四层递进结构。越往上越核心，越往下越容易外包。

```
第四层 · 进化分工（谁在变好）
  人：把隐性判断显式化（判断档案）→ 不断往上游走
  AI：按显式化判断执行 → 不断往下游接
       │
第三层 · 知识分工（谁记住什么）
  人：记住"意味着什么"（判断、风格、直觉）
  AI：记住"发生了什么"（事实、过程、上下文）
       │
第二层 · 判断分工（谁定标准）
  人：判断决策 + 结果验收（只握两个环节）
  AI：信息收集 + 信息整理 + 执行落地（四个环节）
       │
第一层 · 执行分工（谁做）
  人：低频高价 + 不可逆 + 创造性跳跃
  AI：高频重复 + 可验证 + 模式匹配
```

### 第一层：执行分工（谁做）

最表层。40万次真实对话已验证：**人管"规划决策"(What) ≈ 70%，AI管"执行决策"(How) ≈ 80%。**

| 维度 | AI做 | 人做 |
|------|------|------|
| 可验证性 | 即时可验证（编程、格式转换、数据汇总） | 长周期难验证（战略、文化、关系） |
| 频率 | 高频重复（周报、质检、分类） | 低频高价（关键决策、危机处理） |
| 可逆性 | 可撤销（草稿、分析初稿） | 不可逆（发出去的邮件、签的合同） |
| 认知类型 | 模式匹配（归类、对比、检索） | 创造性跳跃（从0到1的框架） |

**HRBP场景验证**：招聘漏斗分析（高频+可验证+模式匹配）→ AI；否掉一个候选人（低频+不可逆+判断）→ 人。

**外部交叉验证**：与3A分诊法（Automation/Augmentation/Agent）、Parasuraman四阶段模型（信息获取→分析→决策→执行，前端可自动化后端留人）一致。

### 第二层：判断分工（谁定标准）

多数人只看到第一层，忽略了一个关键判断：**判断本身不外包，但"判断之前的整理"和"判断之后的执行"都可以外包。**

一个完整的判断过程：

```
信息收集 → 信息整理 → 模式识别 → 判断决策 → 执行落地 → 结果验收
   AI做       AI做      人+AI       人做       AI做      人做
```

**人只握两个环节：判断决策 + 结果验收。** 其余4个环节AI都能承接。

**陷阱**：判断决策的质量，取决于"信息整理"和"模式识别"做得好不好。如果AI整理得不对，人的判断就建立在错误信息上。

→ 这就是为什么HRBP指南强调"不能把AI当裁判"——员工关系场景，AI只帮你整理线索，最终判断必须HR+法务+业务负责人确认。**AI做的是"减少遗漏"，不是"做出裁决"。**

**外部交叉验证**：与HCAI框架（Shneiderman）的"有意义的人类控制"一致——不是消灭自动化，而是即便高自动化也保留人的判断权。

### 第三层：知识分工（谁记住什么）

传统工作里，知识存在三个互不互通的地方：
- 人的脑子里（经验、直觉、判断风格）
- 文档里（制度、流程、历史记录）
- 聊天记录里（决策过程、反馈、协作上下文）

数字分身/Agent群聊的突破在于：让AI成为"统一知识层"，同时接住三个来源：
- **判断档案** → 把脑子的隐性判断显式化
- **知识库** → 把文档结构化可调用
- **群聊记忆** → 把协作上下文自动沉淀

**知识分工的边界**：AI记住"事实和过程"（什么文件在哪、上次怎么定的、谁说了什么），人记住"判断和风格"（为什么这么定、下次遇到类似的怎么判断）。

**一句话：AI记住"发生了什么"，人记住"意味着什么"。**

### 第四层：进化分工（谁在变好）

最深层。40万次对话报告有一个关键趋势：7个月里，用户交给AI的任务价值层级持续上移（修bug 33%→19%，部署运维 14%→21%，写作分析 10%→20%）。**人在不断把低层任务"交出去"，自己往更高层走。**

这个"交出去"的过程，就是判断力传递的过程：

| 阶段 | 人做什么 | AI做什么 |
|------|---------|---------|
| 第一次做 | 人全程参与 | AI辅助 → 人提取判断样本（5问） |
| 第二次做 | 人给判断框架 | AI按框架执行 → 人验收 |
| 第三次做 | 人只验收异常 | AI自己按框架执行，判断被"外化"了 |
| 第N次 | 人去做更高层的新任务 | 这个任务完全由AI承担 |

**进化分工的边界**：人负责"把判断显式化"（让它可传递），AI负责"按显式化的判断执行"（让它可复现）。**人不断往上游走，AI不断往下游接。**

**外部交叉验证**：与五阶段自主性光谱（Autocomplete→Copilot→委托执行→监督自主→完全自主）对应——同一个任务从Stage 2逐步演进到Stage 4的过程，就是判断力逐步传递的过程。

## 一个反直觉结论

> **人机协同的核心能力，不是"会用AI"，是"能把判断说清楚"。**

所有素材最终交汇到这一点。判断说不清楚 → AI只能做最浅层的事；判断说得清楚 → AI能做深层的事。

这就是为什么**"判断档案"是所有这些思考的交汇点**——它是判断力从人的脑子里传递到AI的流程里的那个"桥梁"。

## HRBP场景验证：从案例到框架

HRBP的Codex应用指南（6大场景、8个工具）不是孤立的"AI用法清单"，而是四层模型的完整验证场：

| HRBP场景 | 第一层（谁做） | 第二层（谁定标准） | 第三层（谁记住） | 第四层（谁变好） |
|---------|--------------|-----------------|---------------|---------------|
| 招聘漏斗分析 | AI做分析+看板 | 人定"卡点怎么算异常" | AI记住历史数据 | 判断框架→自动诊断 |
| 绩效质检 | AI批量扫描 | 人定"什么算合格评价" | AI记住标准 | 质检规则→自动执行 |
| 组织诊断 | AI分类纪要 | 人定"问题怎么分类" | AI记住访谈历史 | 诊断框架→自动归类 |
| 员工关系 | AI整理时间线 | 人做最终判断（不可外包） | AI记住事件链 | 风险标记→自动预警 |
| 管理者赋能 | AI生成工具包 | 人定"什么场景需要什么话术" | AI记住沟通历史 | 话术库→持续迭代 |

**关键发现**：8个可工具化的工具中，价值最高的不是任何一个单独工具，而是**一个配置了HRBP Skill+知识库+判断档案的数字同事**——它等于8个工具的集合+持续记忆+可进化。

### ✅ 已知的（有证据支撑）

| # | 已知结论 | 来源证据 | 置信度 |
|---|---------|---------|--------|
| 1 | 人管判断，AI管执行——70/80分工有40万次数据支撑 | Anthropic报告 | 高 |
| 2 | 判断力可以显式化——5问能把"感觉不对"变成可复用样本 | 判断档案方法论+haglofs实践 | 高 |
| 3 | 显式化的判断能传递给AI——design-judgment+bridge在设计场景验证过 | haglofs-paradigm | 中（n=1） |
| 4 | AI能成为"统一知识层"——群聊记忆+知识库+技能=数字同事 | Every公司实验+Agent群聊方案 | 中 |
| 5 | 判断说不清楚=AI只能做最浅层的事——所有笔记交汇于此 | 5篇笔记交叉验证 | 高 |
| 6 | 任务价值层级会持续上移——7个月趋势数据证明 | Anthropic报告 | 高 |

### ❓ 已知未知的（知道问题，没答案）

**1. 判断显式化的"损耗率"是多少？**
判断从脑子传到AI，丢了多少？哪些类型的判断能无损传递（如"这个候选人不合适的数据支撑"），哪些必然有损耗（如"这个设计感觉不对"）？

**2. 判断档案的"保质期"是多久？**
设计判断可能3个月就过时（工具变了）。HR判断可能更长但也受组织变化影响。有没有"失效信号"？

**3. "判断说不清楚"的深层原因是什么？**
是(a)太复杂语言不够？(b)自己没想清楚？(c)依赖隐性知识？(d)是模式识别不是逻辑推导？不同原因需要不同解法。

**4. AI承担的边界到底在哪，会怎么移动？**
3年后AI能承担"组织诊断"判断吗？5年后能承担"否掉候选人"吗？哪些判断是**结构性不可外包**的（不是"现在不能"，是"本质上不能"）？

**5. 多人场景下判断冲突怎么办？**
HR判断 vs 业务负责人判断 vs 法务判断——3方对同一事件有不同判断。Agent群聊里多个人的多个Agent，判断框架可能冲突。**AI听谁的？**

### ⚠️ 完全未知的（连问题都还没问对的）

**1. 判断力本身能被"训练"吗？**
我们讨论的是"把已有判断显式化"。但判断力本身能不能通过这套方法提升？如果一个人判断力本来就弱，5问提取出来的也是弱判断。**这是在"搬运"判断力，还是在"生长"判断力？**

**2. 判断档案的"临界质量"是多少？**
10条能覆盖几个场景？50条覆盖主要工作类型？100条开始有"模式涌现"？有没有"复利拐点"？

**3. 这套框架的"失败模式"是什么？**
- 过度显式化 → 僵化checklist → 失去灵活性
- 判断污染 → 早期错误判断被AI强化 → 正反馈循环
- 能力退化 → 人把判断交给AI → 自己判断力退化（计算器效应）

**4. "判断"和"偏好"的边界在哪？**
"这个设计感觉不对"——是判断还是偏好？"这个候选人跟我气场不合"——是判断还是偏见？如果AI把"偏好"当"判断"学习了，会出什么问题？

**5. 不同人的判断档案能"合并"吗？**
团队5个HRBP各有判断档案，能合并成"团队判断档案"吗？判断冲突时谁的对？是"平均"还是"加权"还是"择优"？

## 后续可深挖的方向

1. **"判断力是搬运还是生长"** — 最根本的未知，值得做一个实验设计（比如对比haglofs-paradigm前后几十轮的判断质量变化）

4. **判断档案的"临界质量"实验** — 积累到一定数量后，是否有复利拐点
