## 必读文章

### 📖 1. ChatGPT Work：OpenAI 的 Agent 入口宣言

- **来源**: [OpenAI — ChatGPT is now a partner for your most ambitious work](https://openai.com/index/chatgpt-for-your-most-ambitious-work) · 2026-07-09
- **类型**: 产品发布 / 官方博客 | ⭐20 + 时效加成 = **高优先级**
- **摘要**: ChatGPT Work 是 OpenAI 最新 Agent 产品，基于 GPT-5.6 驱动，可在用户的文件、应用和 Web 工具之间自主执行任务——从一句话到完成幻灯片、仪表盘、Web 应用。NVIDIA、Zapier、RingCentral、维珍航空已内测数月，用例包括审查数千条销售线索、跨 Jira 检查产品发布准备、为 GTC 2026 自动化筹备工作。Desktop Work 支持本地文件和桌面应用操作，还有 Schedule Tasks 支持无人值守持续运行。
- **🎯 对你的意义**: OpenAI 把 Agent 从"编程工具"升级到"通用工作助手"，直接对标你的报表/PPT/文档工作流。GPT-5.6 + Work 的组合意味着"用对话完成月度经营分析"从愿景变为现实——值得立即上手体验。

### 📖 2. Andrew Ambrosino：产品品味是 AI 时代的新护照

- **来源**: [Lenny's Newsletter — OpenAI Codex lead on the new shape of product work](https://www.lennysnewsletter.com/p/openai-codex-lead-on-the-new-shape) · 2026-06-28
- **类型**: 深度访谈（Lenny's Podcast）| ⭐40
- **摘要**: OpenAI Codex 负责人 Andrew Ambrosino（工程/设计/PM/创业背景）深度拆解：1）AI 彻底颠倒了产品开发流程——不再是 PRD→设计→开发→测试，而是"先用原型测试想法是否值得做"；2）"品味"(taste) 是最稀缺职业能力——不只是审美，是"能否判断某个功能是否值得存在"；3）采用"区域联防"模式——PM 不再按职能切割，而是像球队一样覆盖整个产品区域；4）100% OpenAI 员工（不仅是工程师）每周使用 Codex。他对团队的愿景是"做成有史以来最好的桌面应用"。
- **🎯 对你的意义**: "品味"这个概念直接命中你的核心关切——当 AI 能写代码，PM/管理者的判断力从"功能定义"转向"价值判断"。区域联防模型对供应商管理团队的角色设计有直接参考。

### 📖 3. Cat Wu：Anthropic 如何用一周迭代周期交付世界级产品

- **来源**: [Lenny's Podcast — How Anthropic's product team moves faster than anyone](https://podscan.fm/podcasts/lennys-podcast-product-career-growth/episodes/how-anthropics-product-team-moves-faster-than-anyone-else-cat-wu-head-of-product-claude-code) · 2026-05
- **类型**: 深度访谈 | ⭐40
- **摘要**: Cat Wu（Claude Code & Cowork 产品负责人）揭秘 Anthropic 的产品节奏：1）"我们没有总路线图"——模型改进太快，任何长期计划都会被推翻；2）从"发原型而非文档"开始——代码就是 spec，内部用户和 beta 客户自然使用等于验证；3）每 10 分钟收到一条反馈——1000+ 内测用户 + 内部全员 dogfood；4）工程师端到端负责功能——从在 Twitter 看到用户反馈，到这周末功能上线；5）招聘核心是"产品品味"——设计师也要能交付前端代码，PM 也要写过代码。
- **🎯 对你的意义**: "Demo > Docs" 的方法论可以直接用在供应商管理场景——与其写长长的操作手册，不如做个原型让供应商试用、看哪里卡住。"品味"这个关键词再次出现，是本周最强信号。

### 📖 4. Boris Cherny：编程已终结，接下来是什么

- **来源**: [GLN-7.5 — Coding Is Solved. What Comes Next?](https://gln75.com/en/blog/anthropics-boris-cherny-coding-solved) + [Pragmatic Engineer — Building Claude Code with Boris Cherny](https://newsletter.pragmaticengineer.com/p/building-claude-code-with-boris-cherny) · 2026-05/03
- **类型**: 会议演讲 / 播客访谈 | ⭐40
- **摘要**: Claude Code 创始人 Boris 在 Sequoia AI Ascent 2026 的演讲核心观点：1）Claude Code 团队的所有人（PM、设计师、数据科学家、财务）都写代码；2）Anthropic 内部 Claudes 在循环中写代码，通过 Slack 互相通信；3）"编程已解决"——Opus 4 发布后能力随每个模型版本跃迁；4）Boris 回归工程后工作从"深度专注编码"变为"管理多 Agent 并行 + 快速上下文切换"；5）曾短暂去 Cursor，两周后觉得"还是在 Anthropic 做 Agent 更有意义"就回来了。Pragmatic Engineer 访谈补充了"三重法则"和无限 Token 预算策略。
- **🎯 对你的意义**: "编程已终结"的激进论断来自 Claude Code 创造者本人——这不是外部观察，是内部实践。对供应商管理的启示：当 BPO 供应商的 AI 替代大部分人力操作时，你的角色从"管理操作"转向"用品味定义什么值得做"。

### 📖 5. DeepSeek 放弃"精益AI"：融资74亿美元，全面转向产品公司

- **来源**: [TMTPOST — DeepSeek Abandons 'Lean AI' Strategy as Funding Round Vaults Valuation Past $50 Billion](https://en.tmtpost.com/post/8043250) · 2026-06-26
- **类型**: 融资/战略报道 | ⭐20 + 时效 = **高优先级**
- **摘要**: DeepSeek 完成超 500 亿元人民币（约 74 亿美元）外部融资——创始人梁文锋个人注资 30 亿美元，腾讯、CATL 跟投，估值超 500 亿美元，成为中国最高估值 AI 公司。更关键的是战略转向：从"算法效率驱动的精益研究团队"转为"全部门扩招一倍的企业级产品公司"。招聘 JD 透露方向：Agent Harness（智能体架构）、Agent 评测体系、面向个人助理/Deep Research/自动化工作流的产品。
- **🎯 对你的意义**: DeepSeek 的转型标志着"开源模型"作为独立商业模式的局限已显现——要做产品、要做消费端。对供应商管理的启示：国产AI供应商的产品能力可能快速逼近；作为甲方，评估框架需要从"模型能力"扩展到"产品成熟度和持续交付能力"。

### 📖 6. Kimi K2.7 Code：为中国编程 Agent 提供心脏

- **来源**: [Kimi 官方 — Kimi K2.7 Code：开源智能体式编程模型](https://www.kimi.com/zh-cn/resources/kimi-k2-7-code) · 2026-07
- **类型**: 产品发布 | ⭐20
- **摘要**: 月之暗面发布 Kimi K2.7 Code —— 1T 总参数（32B 激活）MoE 架构，256K 上下文，专为长程 Agent 编程优化。相比 K2.6：长上下文指令遵循更可靠、端到端任务成功率更高、思考 Token 用量减少约 30%。原生多模态（文本/图像/视频），全模型权重 MIT 开源。Kimi K2.7 Code 的 API 定价约 ¥1.30/¥6.50/¥27（输入缓存/输入非缓存/输出 百万token），远低于闭源竞品。
- **🎯 对你的意义**: K2.7 Code 是 DeepSeek V4 之后的又一强力竞争者，且直接对标编程 Agent 场景。月之暗面同步推出 Kimi Code（终端+IDE 插件+API），暗示产品化加速。Kimi 招聘 JD 中"模型策略产品经理-Agent 方向"也验证了向 Agent 产品转型的信号。

### 📖 7. NVIDIA Nemotron：企业级开放模型的精确卡位

- **来源**: [NVIDIA Developer — Advancing Agentic AI with NVIDIA Nemotron Open Reasoning Models](https://developer.nvidia.com/blog/advancing-agentic-ai-with-nvidia-nemotron-open-reasoning-models/) · 2026
- **类型**: 技术/产品博客 | ⭐30
- **摘要**: NVIDIA 正式发布 Nemotron 系列开放推理模型家族，主打：1）在推理和 Agent 任务上达到开放模型最高精度；2）通过神经架构搜索（NAS）、知识蒸馏、SFT、RL 综合优化；3）Mistral-Nemotron 子型号专为 Agent 设计，擅长代码生成和工具调用；4）以 NIM 微服务形式交付，支持从边缘到数据中心的任意部署。关键数据：吞吐量最高、TCO 最低的开放推理模型。
- **🎯 对你的意义**: NVIDIA 做开放模型不是做慈善，是卖更多 GPU。Nemotron 的"代码+工具调用"定位意味着企业可以在本地部署能跑 Agent 的模型——对于数据敏感的金融场景（声纹、支付等）是重要信号。
