## 跨项目复用原则

**"答案"项目相关,"提问方式"跨项目复用。**

同一个场景在不同项目(汽配 / 教育 / 金融 / 医疗 / 消费品)里,问的是同一组问题,但答案千差万别。

本技能的价值在于:
1. 保证关键节点不会漏问(探测类)
2. 保证硬判断不被软化(专业类)
3. 保证规范不被稀释(规范类)
4. 保证不重复踩坑(工程类)

## 场景增补流程

如果在未来项目中发现新的关键场景:

1. 项目结束时单独写一份 `SKILL_UPGRADE_NOTES_YYYYMMDD.md`
2. 提炼 Trigger Point / Open Questions / Usage / Cost of Skipping
3. 在下一次技能升级时整合进对应的 `references/*.md`
4. 更新本 `LESSONS.md` 索引

**不要边做项目边改技能**:项目当下的修改容易带入项目特殊性,沉淀要等项目结束后的冷思考。

## 工具链技术细节（供调试参考）

ppt-master 工具链的核心创新(在 `assets/svg_to_pptx/drawingml_elements.py`):

```python
# baseline 偏移系数(经验值)
box_y = svg_y - font_size * 0.85

# 字符级精确宽度估算
def estimate_text_width(text, size, weight):
    width = 0
    for ch in text:
        if is_cjk_char(ch):
            width += size          # 中文 = 1em
        elif ch == ' ':
            width += size * 0.3
        elif ch in 'mMwWOQ':
            width += size * 0.75
        elif ch in 'iIlj1!|':
            width += size * 0.3
        else:
            width += size * 0.55
    if weight in ('bold', '600', '700', '800', '900'):
        width *= 1.05
    return width
```
