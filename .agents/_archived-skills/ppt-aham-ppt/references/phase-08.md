## 执行步骤

### 步骤1：五层质检

按以下五层规则执行检查：

**L1 · 叙事质量**
- 灵魂句贯穿全文：所有内容页标题串读仍形成逻辑链，且指向灵魂句
- 执行摘要完整性：第2页包含SCQA的核心答案
- 受众关注点已回答：Phase 1受众卡中的核心关切在对应页面有明确回答
- Action Title字数：所有内容页标题≤20字
- 一页一消息：每页核心信息唯一，无「和/与/并且/同时」连接的双消息
- narrative_purpose（叙事目的）完整性：所有页面有标注

**L2 · 格式一致性**
- 全局页脚存在（页码/版本/日期）
- Action Title字体正确（宋体-简 Bold / 微软雅黑 Bold）
- 顶部3pt品牌色横线存在
- **品牌色精确比对**：扫描所有SVG中的 `fill` / `stroke` 色值，与 BRAND_RULES 精确匹配（如 `var(--brand-primary)` 不得写成 `#003D77` 或 `#003d7a`，大小写不一致也需修正为大写）
- **字体一致性逐页扫描**：检查每页 Action Title 字体是否统一，正文字体是否统一，不得在同一类别文字里混用字体
- **图文重叠检测**：检查文字元素的 `y` 坐标是否落在其他 `<rect>` 或图形区域内（坐标重叠判定：文字 y 值位于某个 rect 的 y 到 y+h 范围内，且 x 值位于 rect 的 x 到 x+w 范围内，则标记为疑似重叠，请人工确认）
- 图表颜色语义（正向/负向数据颜色正确，正向 `#1A7A42`，负向 `#B01C1C`）
- 术语一致性（同一概念用词统一，扫描全文找同义词）

**L3 · 专业交付**
- 数据来源标注（所有图表和数据有来源）
- 禁用图表类型检查（无饼图）
- 柱状图Y轴从0开始
- 封面元数据完整（委托方/版本/日期/保密标注）
- 文件数量验证（SVG文件数=规划页数）

**L4 · 视觉节奏**
- 单页信息密度不超过deck_mode上限
- 10页滑动窗口版式多样性≥6种
- 每10页至少1页impact类
- 每8-10页至少1页transition类
- 连续3页无相同版式类别

**L5 · 视觉呈现**
- 坐标重叠检测（文字/图形无非预期重叠）
- 元素遮挡检测（内容区无被遮挡的重要信息）
- SVG与PPTX版本一致（如两者都有）

**处理规则：**
- ERROR级：必须修复，阻断交付
- WARNING级：展示给用户，用户选择：修复 / 带注释交付 / 忽略
- INFO级：仅提示，不阻断

---

### 步骤2：QC报告生成

将质检结果整理为用户友好的报告格式（不展示技术细节，只展示问题和建议）：

```
QC报告 · [项目名称]

ERROR（必须修复）：
  · [页码] [问题描述] → [修复建议]

WARNING（建议修复）：
  · [页码] [问题描述] → [修复建议]
  你的选择：A.修复  B.带注释交付  C.忽略

INFO（供参考）：
  · [描述]

技术质检已通过项：
  ✓ 格式一致性
  ✓ 数据来源标注
  ✓ 视觉节奏
  （列出通过的项）
```

---

### 步骤3：问题处理

对ERROR级问题：
- AI给出具体修改建议
- 修改后重新执行相应的QC层
- 全部ERROR修复后才能继续

对WARNING级问题：
- 用户选择A（修复）→ 执行修改
- 用户选择B（带注释）→ 在交付报告中标注"已知问题"
- 用户选择C（忽略）→ 记录在final_report.md，继续

---

### 步骤4：交付物打包

所有QC问题处理完成后，打包交付物：

```
📦 交付物：
- 方案.pptx      （原生可编辑PPT，客户双击即改）
- 方案.pdf       （打印分发版，从PPTX导出）
- 方案.html      （浏览器预览版，可分享链接）
- svg_final.zip  （SVG真源文件，单一可信源）
- final_report.md（项目报告，含决策记录和已知问题）
```

根据output_format选择性输出：
- svg → 只输出svg_final.zip + preview.html
- pptx-native → 只输出方案.pptx + final_report.md
- both（默认）→ 全部输出

---

### 步骤5：最终交付说明

交付时输出：

```
技术质检已通过：
  ✓ 格式/规范/一致性
  ✓ 视觉节奏
  ✓ 视觉呈现

以下需要你做最终判断，QC无法替代：
  · 每页内容的说服力是否足够
    （格式合规不等于说服力足够）
  · 叙事角度是否真正契合这个受众的决策逻辑
  · 有没有哪页感觉"对但没用"，需要换角度

[已知问题说明，如有]
```
