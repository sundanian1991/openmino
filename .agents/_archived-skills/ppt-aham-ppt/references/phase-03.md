# Phase 3 · 核心论点提炼 ★

> 可插拔模块。独立升级不影响其他Phase。
> 本Phase是整个流程唯一的"思想飞跃"环节。
> 前面的Phase整理事实，后面的Phase搭结构，只有这里要求从材料里归纳出一个有立场的判断。
> AI能辅助引导，但最终灵魂句必须由用户确认——它是价值判断，不是逻辑推导。

## 强制等待锚点

**Phase 3 是整个流程的核心判断点。在开始之前必须先问：**

```
进入 Phase 3 · 核心论点提炼。这是最重要的一步。

在我开始之前，想先问你一个问题：
关于这份材料，你心里有没有一个初步的结论或判断？
（不需要很完整，哪怕一句话也好）

A. 有，我知道大概要说什么
B. 有一些想法，但说不太清楚
C. 还没有，需要你帮我梳理

请选一个或直接描述你的想法。
```

等待用户回答后，根据回答判断能力档位，才进入执行步骤。

## 执行步骤

### 步骤1：SCQA四元组整理

根据档位选择执行方式：

**专家模式：**
AI根据信息清单和用户输入，直接整理SCQA四元组，展示给用户确认。

**引导/教练模式：**
按 coach-engine.md Phase 3 引导序列逐步执行，逐步构建SCQA四元组。

**SCQA四元组输出标准：**

| 元素 | 内容要求 |
|------|---------|
| S 情境 | 受众已知且认可的现状背景，1-2句，客观陈述，不含判断 |
| C 矛盾 | 打破现状的触发事件，有紧迫感，最好含数字或具体事件 |
| Q 核心问题 | 受众面临的决策问题（不是项目组内部的执行问题） |
| A 答案 | 你的明确立场，有依据，指向具体行动，≤30字 |

**常见错误及处理：**

C写成了背景描述（没有打破平衡的张力）：
→ 追问「有没有一个具体事件或数字，让这件事从'有问题'变成'必须现在处理'？」

A是摘要而非立场（"本报告分析了三种方案，各有优劣"）：
→ 追问「如果受众只能记住一件事，你希望他记住什么？」

Q是执行层问题而非决策层问题：
→ 提示「这个问题更像是项目组内部要解决的，受众（[职位]）真正要做的决定是什么？」

---

### 步骤2：灵魂句生成

A的内容即为灵魂句的核心。AI帮助将A整理成符合格式要求的灵魂句。

**灵魂句格式要求：**
- ≤30字
- 一个句子，一个核心结论
- 结论性陈述（不是问题、不是描述、不是菜单）
- 包含判断依据（至少一个）
- 指向受众的预期行动

**灵魂句内部评分（不展示给用户）：**
- 有立场（30%）：是结论性陈述，明确表达推荐/不推荐/选哪个
- 有依据（25%）：包含至少一个判断理由
- 指向行动（25%）：受众看完知道该做什么
- 简洁可记（20%）：≤30字，一个核心结论

评分≥70分放行；50-69分给出修改建议后放行；<50分重新执行引导序列。

---

### 步骤3：受众预期行动确认

在灵魂句确认后，明确受众看完PPT后应该做的具体行动：

「基于这个结论，你希望受众（[职位]）做什么具体的事？」

例：
- CEO批准立项，授权IT部门启动供应商选型
- 董事会通过预算申请，拨付Q3启动资金
- 客户签署合作意向书

这个预期行动将成为Phase 4叙事骨架的终点——整个论证逻辑要能引导受众走向这个行动。

```

风险说明：

  Ghost Deck（骨架PPT）的标题序列将变成流水账，
  而不是论证路径。
  整份PPT最终会呈现为"信息展示"而非"说服工具"。
```
