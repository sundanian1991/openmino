# viz-design — 可视化设计器

> 核心定位：不是流程图生成器，是**可视化设计师**。
> 输入：抽象概念、非结构化数据、文字描述、复杂关系
> 输出：完整的可视化图表（ECharts HTML / SVG），经过减法迭代，每元素都在为"核心判断"服务
> 方法：四阶段工作流（意图决策 → 故事板 → 技术规范 → 渲染验收），阶段间有闸门确认

---

## 技能边界

| 本技能负责 | 不负责 |
|-----------|--------|
| DNA 意图匹配（选什么图） | 数据清洗（交给上游数据技能） |
| 图表设计与渲染（ECharts / SVG） | 多图 HTML 组装/容器（复杂容器场景可委派 html-interactive） |
| SPEC 记录（标题/标注/布局/叙事） | |
| 减法迭代（逐元素追问价值） | |
| 出图后质量验收 | |

**渲染目标选择**：
- 数据图表 → **ECharts HTML**（自包含，双击可开）
- 概念图 → **SVG**
- 印刷级/学术论文 → 可调用 nature-figure 技能

**Birchline 色系**：全阶段统一使用 html-interactive 中的 Birchline Design System 色彩，不新增任何颜色。

## 入口路由

| 年老师说的 | 模式 | 做法 |
|-----------|------|------|
| "先看看数据能讲什么" | A 逐步引导 | 列出分析角度（2-5 个）→ 确认 → 再选图表 |
| "你帮我决定"/"最佳方案" | B AI 推荐 | AI 自主决策 → 直接出图 |
| 直接指定图表或角度 | C 自定义 | 按指定执行 |
| 单图且意图明确 | C 快速路径 | 直接执行，中间文件可省略 |

### 快速通道

| 需求 | 走哪条路 | 阶段执行 |
|------|---------|---------|
| "帮我画个 X 图"（单图、意图明确） | 轻量模式 | Phase 1→2→3→4 连续跑，中间文件可省略 |
| "做一份 XX 报告，需要 N 张图" | 完整流程 | Phase 1 闸门 → 确认 → Phase 2 闸门 → 确认 → Phase 3→4 |
| "只有概念/文字，没有数据" | 概念路径 | Phase 1 S2 改读 concept-pattern-map.md |
| "先看看数据能讲什么" | 引导模式 | 先列角度 → 确认 → Phase 1 → 闸门 → 后续 |

**闸门规则**：
- 单图轻量：闸门自动通过，不额外打断
- 多图/复杂：闸门必须显式确认，年老师批准后才进下一阶段

## 四阶段工作流

```
收到需求 → 选择入口模式 → R0 必要性门禁
  ├── 一句话能说清？→ 文字
  ├── 精确数值/清单？→ 表格
  └── 分布/趋势/关联/复杂比较？→ 继续
  ↓
Phase 1：意图决策 → intent.md（闸门 1：年老师确认方向）
  S1. 读 fields → 了解数据/概念
  S2. 关系识别（趋势/对比/构成/因果/分布/关联）
      纯概念 → 读 concept-pattern-map.md
  S3. 备选图表 — 列出 2-4 种候选
  S4. DNA 匹配 — 三组必搜，全部搜完才能判"未命中"
      ① 按意图搜 $5（narrative_intent）
      ② 按类型搜 $3（chart_type）
      ③ 按策略搜 $8-$10（highlight/annotation/composition）
      → 合并 → 取最匹配 1-3 条 → 锁定 → 查 TSV 继承全策略
  S4b. 单通道检验（选图后必须执行）
      颜色只做一件事，冲突→拆图或降级
  S5. Q1 叙事方向 → 年老师选（2-3 选项）
  → 产出 intent.md → **闸门 1：年老师确认"用什么图、为什么"**
  → 不确认 → 退回 S3 重新选
  ↓
Phase 2：故事板 → storyboard.md（闸门 2：年老师确认布局）
  S6. 叙事弧线 → 从 narrative-arcs.md 32 条中选 1 条
  S7. 场景论文 → 每张图一句话灵魂 + 签名视觉 + 克制声明
  S8. 视觉重量表 → hero / medium / light 标注
  S9. Q2 方向+风格 → 年老师选（合并方向+风格为一次 AskUserQuestion）
  S10. 多图先锁全局风格（style-schools.md / color-themes.md）
  → 产出 storyboard.md → **闸门 2：年老师确认"讲什么故事、怎么标注、什么风格"**
  → 不确认 → 退回 S6 重新选
  ↓
Phase 3：技术规范 → spec.md（纯技术翻译，无需确认）
  S11. 从批准的 storyboard 提取精确渲染参数
  S12. 标注清单（逐项）/ 视觉权重 / 颜色映射 / 布局约束
  S13. WIIFT + TOP-T + 上下文四问
  → 产出 spec.md → **不中断，直接进 Phase 4**
  ↓
Phase 4：渲染+验收（读 references/design-principles.md 获取判断标准）
  Part A — 减法：删明显多余元素 → 8 元素追问 → 5 秒可读性检验
  Part B — 渲染：出第一版图
  Part C — 验收：
    传达检验（"读者看完应该得出什么判断"5 秒验证）
    通用检查（Birchline 色系 / 语法 / 标题 / 重叠 / 标注 ≤3）
    图表专项检查
  Part D — 修复：
    首次失败 → 调参重试
    同一策略失败 2 次 → 升级架构（换图/拆图）
    全部通过 → 输出终稿 + checklist.md
  ↓
交付终图
```

**渐进加载**：先读 `references/library-index.md` 决定下一步加载什么。

**资产分层**：内容层（数据清单、分析结论、叙事逻辑）写一次全程复用；表现层（风格、配色、SPEC 参数）每次迭代可调。

**多图场景**：先锁全局风格（style-schools.md / color-themes.md / typography-moods.md），后续每张图从全局继承。多图衔接：编号连续、视觉引导、配色统一。

## SPEC 编写规范

> SPEC 是**设计记录**，用于减法迭代和验收时对照。

```
【可视化目的】认知缺口是什么？
【信息类型】描述 / 差异 / 关联
【想传达什么】一句话
【结论】读者应得出什么判断？
【叙事弧线】编号 + 名称（从 narrative-arcs.md 32 条中选）
【模式选择】编号 + 名称 + 匹配理由
【标题】主标题结论性 + 副标题（时间范围 + 对象）
【视觉编码】X/Y/颜色/大小各编码什么
【数据组织】字段 / 排序 / 聚合
【标注策略】高亮 ≤10% / 写原因+幅度
【布局】画布尺寸 + 版式编号
```

## 交互约束（强制）

> viz-design 必须经过两个 AskUserQuestion 节点才能进入渲染阶段。
> B 模式（AI 推荐）≠ 全自动，只是图表由 AI 决定。

### 闸门 1：Phase 1 结束（intent.md 产出后）

**意图确认** — 给年老师看 intent.md 核心决策：
- 用什么图（含备选对比）
- 叙事方向（2-3 选项，来自 Q1）
- 为什么不是表格/文字

轻量模式（单图意图明确）：闸门自动通过，不额外打断。

### 闸门 2：Phase 2 结束（storyboard.md 产出后）

**故事板确认** — 给年老师看 storyboard.md 核心决策：
- 叙事弧线编号 + 名称
- 签名视觉元素描述
- 方向+风格组合（2-3 选项，来自 Q2）

轻量模式：闸门自动通过，不额外打断。

### 两个闸门之后

闸门确认后，Phase 3→4 一路跑完，不再中断。

### AskUserQuestion 格式

**Q1：叙事方向（Phase 1 S5，读完数据后必问）**

给 2-3 个叙事方向选项，不要列图表类型。格式：

```
AskUserQuestion(header: "叙事方向"):
  - 选项A: [一句话描述] — 解释为什么
  - 选项B: [一句话描述] — 解释为什么
  - 选项C: [一句话描述] — 解释为什么
```

**Q2：方向 + 风格（Phase 2 S9，用户选完方向后必问）**

合并为一次 AskUserQuestion，包含方向和风格：

```
AskUserQuestion(header: "方向+风格", multiSelect: false):
  - 方向A + 风格X: [描述]
  - 方向B + 风格Y: [描述]
  - 方向C + 风格Z: [描述]
```

风格从 color-themes.md 选 2-3 个匹配项，不要只给一个。

## 铁律（全局约束，任何阶段都生效）

1. **自己画图。** 直接写 ECharts HTML / SVG。不委派（除非 recipe 没有）。
2. **中间文件必须写。** intent.md / storyboard.md / spec.md 是给自己看的。
3. **减法必须做。** 不减到"每个元素都在为核心判断服务"不交付。
4. **多图先锁全局风格。** 不得各自选色。
5. **出图后必须验收。** 4 项以上不通过回退 Phase 4 Part A。
6. **DNA 三组必搜。** 意图($5) + 类型($3) + 策略($8-$10)，搜完才能判未命中。
7. **颜色只做一件事。** 单通道检验必须通过才能渲染。
8. **闸门必须先过。** 多图/复杂场景 Phase 1→2→3 之间有闸门，年老师确认后才进下一阶段。单图轻量闸门自动通过。
9. **拒绝必须写。** 每个中间文件必须包含"为什么不是别的"段落。写不出来 → 说明没比较过 → 退回选选项，不往下走。
   - intent.md：为什么选这个叙事而不是另一个？为什么这张图不是表格/文字？
   - storyboard.md：为什么选折线不选柱状/面积/散点？为什么这个颜色/风格？
   - spec.md：为什么 X 轴选这个不选那个？为什么高亮点选这个不选那个？
10. **Phase 3 不重新设计。** spec.md 只从批准的 storyboard 提取技术参数，不引入新决策。

## 反模式（禁止）

- 为了走流程而走流程
- 第一版就交付（不做减法迭代）
- 多个元素同等突出互相抢夺注意力
- 多图各自选色，不锁全局
- 出图后不验收即交付
- 颜色同时编码两个维度（如排名+分组），信息打架
- DNA 搜索只跑一组就判"未命中"（必须三组：意图+类型+策略）
- 验收只检查规范合规，不检验信息传达（5 秒测试不通过 = 回炉）
- **跳过闸门直接出图**（多图/复杂场景必须经 Phase 1 和 Phase 2 闸门确认）
- **跳过 Q1/Q2 直接出图**（B 模式不是全自动，叙事方向和风格必须让用户选）
- **一次性问太多问题**（只问 Q1 和 Q2，不要问 5 个以上）
- **中间文件只记录不比较**（没有"为什么不是别的" = 没有认真比较过 = 退回重选）
- **用户否定后在同一层修三次**（否 1 次改视觉，否 2 次退回叙事层，否 3 次退回数据层）
- **Phase 3 重新做设计决策**（spec.md 只从批准的 storyboard 提取参数，不引入新方向）
