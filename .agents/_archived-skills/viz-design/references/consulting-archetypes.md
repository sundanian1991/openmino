# 咨询图表原型（C01-C16）

> 麦肯锡/贝恩级商业分析图表原型库。
> 适用于供应商管理、业绩评估、战略建议、根因分析等场景。
> 每个原型包含触发条件、结构说明、标题模板、标注策略、反模式。

---

## C01 — MECE 拆解

**逻辑结构**：互斥且穷尽的分解（Mutually Exclusive, Collectively Exhaustive）

**When to use**：
- 输入问"由哪些部分组成"、"拆解一下"、"分类看一下"
- 需要证明分类完整、无重叠

**Structure**：
- 水平树状或层级卡片
- 第一层 2-5 个主分类（互斥）
- 每类下可继续拆分子类（可选）
- 总和 = 100% 或总量

**Title pattern**："[总量] 拆解：[最大类] 占 [X]%，[最小类] 仅 [Y]%"

**Annotation rules**：
- 高亮最大或异常类别
- 标注写"为什么这类最大/最小"
- 底部标注总计确认"合计 = 100%"

**Anti-pattern**：
- 分类有重叠（不互斥）→ 重写分类
- 分类未穷尽（漏了"其他"）→ 加"其他"类
- 拆了 6 个以上一级分类 → 合并为 3-5 个

**Delegate to**：viz-svg-flow

## C02 — 瀑布分析

**逻辑结构**：从 A 到 B 的驱动因素分解

**When to use**：
- 输入问"什么导致了变化"、"同比/环比差了多少"、"拆解增量"
- 有明确的起点值和终点值，中间有多个正负贡献因素

**Structure**：
- 起点柱（去年值）→ 多个正负浮动柱 → 终点柱（今年值）
- 正值向上（Warm/Teal），负值向下（Coral）
- 累计线连接各柱顶部

**Title pattern**："[指标] 同比 [±X]，主要受 [最大驱动因素] 影响（[±Y]）"

**Annotation rules**：
- 高亮最大正向和最大负向驱动因素
- 标注写因素名称 + 贡献幅度
- 底部标注净变化 = 各因素之和

**Anti-pattern**：
- 因素之和 ≠ 净变化 → 检查数据
- 超过 8 个因素 → 合并为"其他"
- 用饼图代替瀑布 → 瀑布才能表达"累积变化"

**Delegate to**：viz-echarts（瀑布图）/ viz-data-viz

## C03 — 2x2 矩阵

**逻辑结构**：两维度四象限定位

**When to use**：
- 输入问"优先级怎么排"、"战略定位"、"哪些重要哪些紧急"
- 有两个独立维度，需要四象限分类

**Structure**：
- 横轴：维度 A（低→高）
- 纵轴：维度 B（低→高）
- 四象限各有标签（如：高价值高紧迫、高价值低紧迫...）
- 数据点落入对应象限

**Title pattern**："[对象] 四象限：[X] 家位于高价值区，[Y] 家需重点关注"

**Annotation rules**：
- 高亮异常象限（过多或过少数据点）
- 标注写"为什么这些对象在此象限"
- 象限标签要行动导向（如"优先投资"、"观察"）

**Anti-pattern**：
- 维度不独立（X 和 Y 强相关）→ 换维度
- 象限标签太抽象（如"第一象限"）→ 用业务语言
- 数据点全挤在一个象限 → 维度选得不好

**Delegate to**：viz-svg-flow

---

## C04 — 时间线 + 里程碑

**逻辑结构**：带关键节点的时间序列

**When to use**：
- 输入问"何时发生了什么"、"关键节点"、"发展历程"
- 需要展示时间轴上的事件/里程碑

**Structure**：
- 水平时间轴（左→右 或 上→下）
- 关键事件用圆点/菱形标注
- 重要里程碑加粗或变色
- 可加注释说明影响

**Title pattern**："[主题] [时间范围]：[X] 个关键节点，[最大事件] 是转折点"

**Annotation rules**：
- 高亮最关键 1-2 个里程碑
- 标注写事件名 + 影响（不写日期，日期在轴上）
- 可加趋势箭头说明整体走向

**Anti-pattern**：
- 事件太多（>10 个）→ 只保留关键节点
- 只列日期不写影响 → 每个事件必须有"所以呢"
- 时间轴不线性（跳月跳年）→ 保持比例

**Delegate to**：viz-echarts / viz-svg-flow

---

## C05 — 对比表 + 结论

**逻辑结构**：多选项多维度评估

**When to use**：
- 输入问"哪个最好"、"对比一下"、"排名怎么样"
- 需要对多个选项在多个维度上比较

**Structure**：
- 行 = 选项（供应商/产品/方案）
- 列 = 评估维度
- 单元格 = 数值/评分/图标
- 最后一列 = 综合结论或排名

**Title pattern**："[X] 家 [对象] 对比：[第一名] 综合领先，[最后一名] 在 [维度] 明显落后"

**Annotation rules**：
- 高亮最佳和最差选项
- 标注写"为什么领先/落后"
- 可用 Harvey Ball 或色带辅助阅读

**Anti-pattern**：
- 维度超过 8 个 → 合并为核心维度
- 选项超过 10 个 → 只保留关键选项 + "其他"
- 没有结论列 → 对比必须有 verdict

**Delegate to**：viz-echarts / viz-data-viz

---

## C06 — 漏斗 + 瓶颈标注

**逻辑结构**：转化衰减中的瓶颈识别

**When to use**：
- 输入问"哪里卡住了"、"转化率"、"流失在哪"
- 有明确的阶段数据和逐级转化率

**Structure**：
- 漏斗从上到下逐级收窄
- 每层标注阶段名 + 数量 + 转化率
- 瓶颈层用信号色高亮

**Title pattern**："[流程] 转化率仅 [X]%，瓶颈在 [环节]（转化率 [Y]%，低于均值 [Z]%）"

**Annotation rules**：
- 唯一高亮瓶颈层（信号色）
- 标注写"为什么这里是瓶颈"
- 底部标注总转化率

**Anti-pattern**：
- 高亮多层 → 只高亮最严重的瓶颈
- 漏斗不是递减 → 数据有问题
- 只标数字不写原因 → 标注必须解释 WHY

**Delegate to**：viz-echarts

---

## C07 — 趋势 + 拐点叙事

**逻辑结构**：带注释的时间序列

**When to use**：
- 输入问"趋势怎么样"、"什么改变了"、"为什么突然涨/跌"
- 有时间序列数据，且存在明显拐点

**Structure**：
- 折线或面积图
- 拐点处加垂直参考线 + 注释
- 注释写"什么事件导致了拐点"

**Title pattern**："[指标] [时间范围]：[方向] 趋势，[拐点时间] 因 [原因] 加速/反转"

**Annotation rules**：
- 高亮拐点及原因注释
- 标注写事件名 + 影响幅度
- 可加预测虚线（标明"预测"）

**Anti-pattern**：
- 注释只写日期不写原因 → 必须写"因为什么"
- 拐点太多（>3 个）→ 只标注最关键的
- 在平稳线上硬找拐点 → 拐点必须有明确事件支撑

**Delegate to**：viz-echarts

---

## C08 — 堆叠构成

**逻辑结构**：总量的组成及变化

**When to use**：
- 输入问"总量由什么构成"、"各部分占比变化"
- 需要展示多个组成部分在总量中的占比及演变

**Structure**：
- 堆叠柱状图（时间点间对比）或堆叠面积图（连续趋势）
- 每层 = 一个组成部分
- 总高度 = 总量

**Title pattern**："[总量] 构成：[最大类] 占比 [X]%，[变化最大类] 同比 [±Y]pp"

**Annotation rules**：
- 高亮占比最大或变化最大的层
- 标注写"为什么这类占比最大/变化最大"
- 图例放右侧或底部

**Anti-pattern**：
- 层数超过 6 → 合并为"其他"
- 用普通柱状图代替堆叠 → 堆叠才能表达"部分之和 = 整体"
- 各层顺序随意 → 按大小或逻辑排序

**Delegate to**：viz-echarts / viz-data-viz

---

## C09 — 差距到目标

**逻辑结构**：现状 vs 目标 vs 路径

**When to use**：
- 输入问"差多少"、"目标达成情况"、"怎么补足差距"
- 有当前值和目标值，需要展示差距和路径

**Structure**：
- 子弹图或温度计图
- 底色 = 当前值
- 标记线/区域 = 目标值
- 缺口部分用不同色标注

**Title pattern**："[指标] 当前 [X]，距目标差 [Y]（[Z]%），需 [行动] 才能达成"

**Annotation rules**：
- 高亮差距部分
- 标注写"还需要做什么"
- 可加里程碑标记（Q1/Q2/Q3 目标）

**Anti-pattern**：
- 只标当前不标目标 → 必须有目标线
- 差距用负面色 → 差距是"还没到"不是"坏了"
- 没有行动暗示 → 差距图必须暗示"怎么做"

**Delegate to**：viz-echarts

---

## C10 — 散点 + 象限洞察

**逻辑结构**：两变量相关性 + 分区洞察

**When to use**：
- 输入问"有没有规律"、"两个指标的关系"、"哪些异常"
- 需要展示两个连续变量之间的关系

**Structure**：
- 散点图（X = 变量 A，Y = 变量 B）
- 可加趋势线
- 四象限分区（均值或中位数为界）

**Title pattern**："[变量 A] 与 [变量 B] 呈 [正/负] 相关，[异常点名] 偏离趋势"

**Annotation rules**：
- 高亮异常点或关键象限
- 标注写"为什么这个点异常"
- 加趋势线 + R²（如有）

**Anti-pattern**：
- 散点超过 50 个不加聚合 → 考虑分箱或热力图
- 象限线设在 0 而不是均值 → 用数据中位数分区
- 相关性说成因果 → 标注"相关"不写"导致"

**Delegate to**：viz-echarts

---

## C11 — 层级树

**逻辑结构**：组织/分类分解

**When to use**：
- 输入问"汇报关系"、"组织架构"、"分类体系"
- 需要展示上下级层级关系

**Structure**：
- 自上而下的树状图
- 根节点 → 一级分类 → 二级分类（可选）
- 每层用连接线，标注类别名

**Title pattern**："[主题] 层级：[X] 个一级分类，[最大类] 下设 [Y] 个子类"

**Annotation rules**：
- 高亮最深或最宽的分支
- 标注写"为什么这个分支特殊"
- 层级不超过 4 层

**Anti-pattern**：
- 层级超过 4 层 → 合并或展开为独立图
- 同级节点数量差异过大 → 检查分类是否合理
- 用饼图代替树 → 饼图不表达层级

**Delegate to**：viz-svg-flow

---

## C12 — 桥接图

**逻辑结构**：去年同期到今年的桥

**When to use**：
- 输入问"年同比变化拆解"、"从去年到今年怎么变的"
- 类似瀑布但专门用于年同比场景

**Structure**：
- 左柱 = 去年值
- 右柱 = 今年值
- 中间桥接线连接各驱动因素
- 因素按贡献大小排序

**Title pattern**："[指标] 同比 [±X]：[最大正向] 贡献 [Y]，[最大负向] 拖累 [Z]"

**Annotation rules**：
- 高亮贡献最大和拖累最大的因素
- 标注写因素名 + 贡献值
- 底部标注净变化

**Anti-pattern**：
- 和 C02 瀑布重复 → C12 专用于年同比，C02 用于一般变化
- 因素排序随意 → 按贡献绝对值排序
- 桥接线交叉 → 重排因素避免交叉

**Delegate to**：viz-data-viz

---

## C13 — Harvey Ball 矩阵

**逻辑结构**：定性评估网格

**When to use**：
- 输入问"各选项评分"、"能力对比"、"成熟度评估"
- 需要定性而非定量的多维度对比

**Structure**：
- 行 = 选项
- 列 = 评估维度
- 单元格 = Harvey Ball（空/1/4 满/半满/3/4 满/全满）
- 最后一行 = 综合评分

**Title pattern**："[X] 个 [对象] 能力矩阵：[最佳] 在 [维度] 领先，[最差] 多项待提升"

**Annotation rules**：
- 高亮最佳和最差选项行
- 标注写"为什么评分如此"
- 底部加综合评级

**Anti-pattern**：
- 用数字代替 Harvey Ball → 定性场景用球更直观
- 维度超过 8 个 → 合并核心维度
- 评分标准不统一 → 先定义每档标准

**Delegate to**：viz-svg-flow

---

## C14 — 子弹图/温度计

**逻辑结构**：单 KPI vs 目标

**When to use**：
- 输入问"是否达标"、"进度怎么样"、"KPI 完成情况"
- 只需要展示一个核心指标与目标的差距

**Structure**：
- 水平条 = 实际值
- 标记线 = 目标值
- 背景色带 = 差/中/好区间
- 标签 = 当前值 + 目标值

**Title pattern**："[指标] 完成 [X]%，距目标差 [Y]，[状态判断]"

**Annotation rules**：
- 高亮差距段
- 标注写"还需要多少"
- 状态色：达标=Teal，接近=Warm，落后=Coral

**Anti-pattern**：
- 用普通柱状图 → 子弹图才能表达"区间+目标"
- 目标线不清晰 → 必须醒目
- 没有状态判断 → 必须有"达标/接近/落后"判断

**Delegate to**：viz-echarts

---

## C15 — 桑基流

**逻辑结构**：资源/客户/资金流向

**When to use**：
- 输入问"X 去了哪里"、"资金流向"、"客户从哪里来到哪里去"
- 需要展示从一个集合到另一个集合的流量

**Structure**：
- 左列 = 来源节点
- 右列 = 去向节点
- 中间流量带 = 连接关系，宽度 = 流量大小

**Title pattern**："[来源] 流向 [去向]：[最大流] 占 [X]%，[异常流] 需关注"

**Annotation rules**：
- 高亮最大流量带
- 标注写"为什么这条流最大"
- 节点按流量大小排序

**Anti-pattern**：
- 节点超过 10 个 → 合并为大类
- 流量带交叉混乱 → 调整节点顺序减少交叉
- 用饼图代替桑基 → 桑基才能表达"从哪到哪"

**Delegate to**：viz-echarts

---

## C16 — Issue Tree

**逻辑结构**：问题分解到假设

**When to use**：
- 输入问"根因是什么"、"问题出在哪"、"怎么拆解这个问题"
- 需要将一个大问题分解为可验证的假设

**Structure**：
- 根节点 = 核心问题
- 第一层 = 2-3 个主假设（MECE）
- 第二层 = 每个假设的子假设或验证路径
- 叶子节点 = 可验证的具体问题

**Title pattern**："[核心问题] 根因分析：最可能在 [假设]，需验证 [证据]"

**Annotation rules**：
- 高亮最可能的假设分支
- 标注写"支持/反驳这个假设的证据"
- 叶子节点标注"已验证"或"待验证"

**Anti-pattern**：
- 假设不 MECE → 重写确保互斥穷尽
- 超过 3 层 → 太深，合并
- 叶子节点不可验证 → 必须可验证

**Delegate to**：viz-svg-flow
