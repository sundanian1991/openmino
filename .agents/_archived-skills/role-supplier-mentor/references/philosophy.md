# 哲学层 — 供应商管理的底层智慧

> 从 gstack 的"判断力编码"迁移，形成供应商管理的哲学基础

---

## 一、完整性优先（Boil the Lake）

### 本质

**供应商管理中没有"差不多"**——每个决策都应该完整，而不是留下隐患。

### 完整性定义

| 场景 | 不完整 | 完整 |
|------|--------|------|
| **准入评估** | 看了资质就通过 | 资质 + 产能 + 意愿 + 团队 + 背调 + 试单 |
| **业绩诊断** | 看了KPI就下结论 | KPI + 趋势 + 原因 + 对比 + 预测 + 建议 |
| **风险判断** | 看了表面问题 | 根因 + 影响 + 概率 + 预案 + 信号 |
| **沟通准备** | 想到什么说什么 | 状态 + 数据 + 目的 + 问题 + 预警 |

### 应用原则

1. **不跳步**：诊断管线完整走完，不因为"感觉差不多"就停
2. **不满足模糊**：追问到具体、可验证、可量化
3. **不只看表面**：每层都往下挖一层

## 二、先了解再决策（Search Before Building）

### 三层知识框架

| 层级 | 内容 | 供应商管理应用 |
|------|------|----------------|
| **Layer 1** | 已验证模式 | 行业成熟做法：资质审核、合同模板、KPI体系 |
| **Layer 2** | 新流行趋势 | 新供应商类型、新管理模式、数字化工具 |
| **Layer 3** | 第一性原理 | 这家供应商为什么存在？核心竞争力是什么？ |

### 应用原则

1. **先看历史**：这家供应商之前表现如何？类似供应商什么情况？
2. **再看行业**：行业怎么做的？有什么新动态？
3. **最后问本质**：为什么选这家？本质原因是什么？

### Eureka 时刻

当第一性原理推理发现"常规做法是错的"：

```
EUREKA: 大家都认为[某做法]是对的，因为[假设]。
但[证据]显示这是错的。
[更好的做法]更好因为[推理]。
```

## 三、信号优于观点

### 本质

**观点是廉价的，信号是宝贵的。**

- "这家供应商不错" → 观点（廉价）
- "这家供应商连续3个月KPI上升，上月92分，排名第5" → 信号（宝贵）

### 信号类型

| 类型 | 示例 | 可靠性 |
|------|------|--------|
| **行为信号** | 主动配合、快速响应、按时交付 | 高 |
| **数据信号** | KPI、排名、趋势、对比 | 高 |
| **态度信号** | 推诿、承诺、抱怨 | 中 |
| **传闻信号** | 行业口碑、别人评价 | 低 |

### 应用原则

1. **优先采集行为和数据信号**
2. **态度信号需要验证**
3. **传闻信号仅供参考**

## 四、多层视角

### 本质

**同一个问题，站在不同高度看，答案不同。**

### 视角层级

| 层级 | 视角 | 问题 |
|------|------|------|
| **P7执行层** | 把事做完 | 具体怎么做？什么时间？谁负责？ |
| **P8管理层** | 把事做好 | 是个案还是共性？怎么避免重复？ |
| **P9体系层** | 把机制建好 | 梯队健康吗？有B计划吗？ |
| **P10+战略层** | 把方向定对 | 这在整体战略中是什么位置？ |

### 应用原则

1. **从P7开始，逐步向上看**
2. **不是每个问题都要到P10+**
3. **让年老师练习"向上看"**

---

## 五、时间深度

### 本质

**不要只看当下，要看趋势和未来。**

### 时间维度

| 维度 | 问题 |
|------|------|
| **过去** | 之前什么情况？有什么历史？ |
| **现在** | 当前状态如何？有什么信号？ |
| **近期未来** | 1-3个月会怎样？有什么风险？ |
| **远期未来** | 1-3年还需要吗？战略匹配吗？ |

### 应用原则

1. **看趋势，不只看时点**
2. **预测未来，验证假设**
3. **为长期积累，不只是解决当下**
