# suilu-notes

> 三文档隔离式口述记录工作流。原始输入 / 越缦堂整理 / 储备沉淀，三层独立存储。

## 目录结构

```
suilu-notes/
├── README.md           ← 本文件（规范）
├── raw/                ← 原始输入：口语全保留，按年归档
│   └── 2026.md
├── log/                ← 越缦堂整理：按年归档
│   └── 2026.md
├── reserve/            ← 储备沉淀：AI 基于 raw 独立生成
│   └── 2026.md
└── scripts/
    └── analyze.py      ← 储备沉淀辅助分析脚本
```

**归档粒度**：三个文档均按**年一个文件**，一年最多 3 个文件。文件内部按日期（或批次）分块，靠搜索定位，不靠文件夹层级拆分。年底时续写新年度文件。

## 三文档工作流标准

### 1. raw/（原始输入）

- **来源**：用户口述，AI 逐字保留
- **保留原则**：口癖、停顿、重复、语气词、碎片化表达全部保留
- **格式**：文件内部按日期分块，每个日期以 `## 6月13日` 开头。同一天多条时，块内用时间戳 `> 14:30 ...` 区分先后，**不加连续序号**
- **修改规则**：**禁止修改旧条目**。如需补充，新增日期块或新条目
- **归档方式**：按年一个文件，文件名 `YYYY.md`。年底直接创建新年度文件

### 2. log/（越缦堂整理）

- **来源**：AI 基于原始输入做结构化事实复述
- **整理标准**：
  - 不是一句话标签，而是有篇幅的事实复述（50-200字/条）
  - 保留场景、时间、人物、数据、对话细节、过程描述
  - 按五字头分类：【事】【人】【读】【思】【知】
  - 不抒情、不修饰、不扩写、不包装美文
- **格式**：文件内部按日期分块，每个日期以 `## 6月13日` 开头
- **归档方式**：按年一个文件，文件名 `YYYY.md`。年底直接创建新年度文件

### 3. reserve/（储备沉淀）

- **来源**：AI **仅基于 raw/** 的内容生成，**看不到 log/ 的任何内容**
- **生成触发**：用户主动将 raw/ 内容贴给 AI，说「沉淀」「挖」「储备」
- **内容**：主题密度、反复出现的模式、交叉深挖点、待验证假设、行动线索
- **格式**：按沉淀批次组织，每批次以 `## YYYY-MM 月度沉淀` 或 `## YYYY-MM-DD 批次沉淀` 开头
- **归档方式**：按年一个文件，文件名 `YYYY.md`

## 使用流程

### 日常记录（每天）

1. 打开对话，直接口述
2. AI 输出【原始输入】+【越缦堂整理】两层
3. 将【原始输入】粘贴到 `raw/2026.md`
4. 将【越缦堂整理】粘贴到 `log/2026.md`

**注意**：日常模式下 AI **不输出**储备沉淀层

### 批量输入处理（用户直接粘贴大段内容或上传文件）

1. 将大段文字直接粘贴，或上传 Markdown 文件
2. AI 自动按日期/语义拆分为独立条目
3. AI 逐条输出【原始输入】+【越缦堂整理】两层
4. 用户分别保存到 `raw/2026.md` 和 `log/2026.md`
5. 用户说「现在沉淀」，AI 切换到储备沉淀模式，基于全部原始文本输出

**拆分规则**：
- 优先识别明确的日期标记（如 `2026-06-13`、`6月13日`、`今天`）
- 无日期标记时，按语义主题或场景转换拆分
- 同一天的多条内容保留在同一块内，用时间戳区分
- **不添加连续序号**，保持 Markdown 天然顺序

### 储备沉淀（每周/每月）

1. 打开 `raw/2026.md`，复制一段时间的内容
2. 贴给 AI，说「沉淀」「挖」「储备」
3. AI 基于原始文本生成储备沉淀（看不到 log/ 的内容）
4. 将输出粘贴到 `reserve/2026.md`

### 体系迭代（每季度/数据充足时）

1. 将 `raw/` 全部或大量内容贴给 AI
2. 说「体系」「迭代」「整理」「归纳」
3. AI 自行归纳，不参考任何固定模板
4. 将输出粘贴到 `reserve/2026.md`（作为体系迭代批次）

### 辅助分析（可选）

运行脚本做基础统计：

```bash
cd suilu-notes
python3 scripts/analyze.py raw/2026.md
```

## 核心规则

1. **零摩擦输入**：口述即记录，无需准备格式
2. **双轨保存**：每次必须同时保存 raw/ 和 log/
3. **原始优先**：reserve/ 必须只基于 raw/，不能引用 log/ 的分类
4. **不修饰**：raw/ 逐字保留，log/ 只做冷分类和事实复述
5. **跨层隔离**：AI 在沉淀模式看不到 log/ 的内容
6. **自行生长**：体系迭代不受五字头约束

## 分类字头说明（供 AI 整理参考）

| 字头 | 判定标准 | 常见信号词 |
|------|---------|-----------|
| 【事】 | 客观发生的行为 / 事件 / 状态变化 | "今天""下午""刚才""发生了" |
| 【人】 | 涉及具体人物及其言行 | "某人说""和某人聊" |
| 【读】 | 外部信息输入 | "读到""听到""看了""刷到" |
| 【思】 | 主观判断 / 推理 / 觉察 | "觉得""想到""意识到""发现" |
| 【知】 | 新增的确定性信息 / 事实 | "原来""才知道""数据显示" |

## 边界

- raw/ 禁止修改旧条目（保持原始信息的不可篡改性）
- log/ 只做整理不做扩写
- reserve/ 不做情绪分析 / 心理建议
- AI 不主动追问"还有什么要补充的"
