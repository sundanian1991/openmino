# 可视化质检

> 图表渲染后的强制质检关卡。不通过不出货。

## 触发时机

1. **任何图表/HTML渲染完成后**（viz-design / mck-chart-render / viz-echarts / html-interactive / nothing-design）
2. **年老师反馈"有问题"后**（修复后二次质检）
3. **批量出图时**（每张图独立质检）

## 质检清单

### Tier 1：致命错误（不通过不可交付）

1. **数据替换验证**
   - [ ] 图表中的数据点与输入数据一致（非模拟数据残留）
   - [ ] 标题中的核心数字与数据一致
   - [ ] 数据来源标注正确（如"来源：XX系统"）

2. **Chart.js/ECharts语法**
   - [ ] options 对象无 `&&` / `||` 逻辑运算符
   - [ ] 所有 `{}` `[]` `()` 配对闭合
   - [ ] 所有 HTML 标签配对闭合
   - [ ] canvas 元素 id 唯一且与 JS 一致

3. **HTML完整性**
   - [ ] 文件可正常打开无白屏
   - [ ] 控制台无报错（`Uncaught SyntaxError` / `ReferenceError`）
   - [ ] 无 `file://` 跨域导致的资源加载失败

### Tier 2：质量标准（修复后交付）

4. **标签与图例**
   - [ ] 数据标签不重叠（关键数据点优先标注）
   - [ ] 图例存在且与数据系列对应
   - [ ] 标题完整（主标题+副标题+时间范围）
   - [ ] 坐标轴标签清晰可读

5. **预注意属性**
   - [ ] 最大值/最小值/异常点有视觉锚点（颜色/大小/标注）
   - [ ] 背景线与数据层级分明（灰色背景 vs 彩色数据）
   - [ ] 非重点元素不过度装饰

6. **容器与截断**
   - [ ] 图表底部不被截断（grid.bottom ≥ 40px 或容器 height 充足）
   - [ ] 右侧图例/标签不被挤压
   - [ ] 移动端基本可读（如适用）

### Tier 3：精细度（加分项）

7. **信息层次**
   - [ ] 3 秒内可捕获核心结论
   - [ ] 辅助信息不喧宾夺主
   - [ ] 色彩使用有语义（非随机彩虹）

## 执行方式

### 方式一：自检（推荐）

渲染完成后，AI 按本清单逐条自查。不通过项明确列出：

> **质检结果：2项未通过**
> 1. [致命] 图表第3个数据点使用的是模拟数据（123），而非实际值（456）
> 2. [标准] 最大值缺少红色高亮标注

### 方式二：委托子代理

复杂 HTML 或多图报告可委托子代理执行质检：
- 输入：HTML文件路径或代码
- 输出：质检报告（通过/未通过 + 问题列表）

## 根因速查表

| 常见问题 | 根因 | 修复方案 |
|---------|------|---------|
| 底部截断 | grid.bottom太小 + overflow:hidden | 增大grid.bottom≥40px或容器height |
| 数据未替换 | 模板变量名不匹配或替换逻辑遗漏 | 全局搜索模拟数据关键词"sample"/"demo"/"123" |
| JS语法错误 | 逻辑运算符/括号不匹配 | 用JSHint或逐字符检查 |
| 标签重叠 | 数据点过多且无避让算法 | 减少标注点或启用labelLayout |
| 图例缺失 | 系列名未定义或legend配置遗漏 | 检查series.name和legend.data |
