# Aham 设计系统 — 轨道分流与统一规则

> **核心结论**：四份文档**不是相互矛盾**，而是**面向不同载体的"分册"**。冲突来自混用。本文件把冲突点显性化，并给出"用哪套"的判断树。

---

## 0. 三条根本原则（跨所有轨道）

无论哪个轨道，下列三条永不动摇：

1. **单一彩色**：每件作品只有一个强调色，其余皆中性灰 + 语义色。
2. **状态双通道**：颜色 + 符号/文字，永远成对出现。
3. **禁用清单共享**：渐变 / 投影 / 3D / emoji / 饼图 / 第二装饰彩色 / 纯黑 `#000` —— 所有轨道一致禁用。

---

## 1. 四个轨道（Track）的边界

四份原始文档对应四个**互不重叠**的载体场景。**选错轨道，所有数值跟着错。**

| 轨道 | 适用载体 | 主色 | 圆角 | 标准按钮高 | 顶栏 |
|---|---|---|---|---|---|
| **A · Brand** | Word / PPT / Excel / 名片 / 合同 / 官网 / 公众号 | `#1677FF` | `0 / 2 / 4 px` | — | — |
| **B · Brand-Prototype** | 独立品牌 Demo / 方案演示原型 / Web 后台 Mock | `#1677FF` | `0 / 2 / 4 px` | `32 px` | 白底 + 3px 蓝顶线 |
| **C · Platform** | 客户项目跑在 Aham 平台（Ant 5）上的真实代码 | `#1677FF` | `2 / 4 / 6 / 8 px` | `32 px`（Ant 默认）| `#001529` 深色 |
| **D · Field** | PDA 手持终端 / 工业 Pad（现场/工业环境）| `#1677FF` | `2 / 4 / 6 px` | **`48 px`**（戴手套）| `#1677FF` 品牌蓝底 |

### 决策树

```
要做什么？
├─ 文档 / PPT / 名片 / 合同 / 官网 → 轨道 A（Brand）
├─ 给客户演示的"独立 Demo"（不在平台上跑） → 轨道 B（Brand-Prototype）
├─ 在 Aham 平台二次开发的客户项目 → 轨道 C（Platform）
└─ 现场使用的 PDA / Pad 软件（如工业环境、仓储、一线作业） → 轨道 D（Field）
```

> **绝不混用**：客户的真实平台项目里冒出 `#1677FF` = 错；品牌物料里冒出 `#1677FF` = 错；Web 后台用 `48px` 大按钮 = 错；现场 PDA 用 Serif = 错。

## 2. 冲突项的统一裁定

下表把四份原文档的冲突点逐项裁定：**保留分轨道的差异**，但用共同 token 表达。

| 维度 | A Brand | B Brand-Prototype | C Platform | D Field | 统一标准 / 备注 |
|---|---|---|---|---|---|
| **主色** | `#1677FF` | `#1677FF` | `#1677FF` | `#1677FF` | A/B/D 共用 `--blue-500`；C 独立 `--platform-primary` |
| **圆角** | `0/2/4` | `0/2/4` | `2/4/6/8` | `2/4/6` | C 上限 8px，其余 ≤ 6px；**任何场景禁止 > 8px** |
| **Sans 字体** | 思源黑体 | Noto Sans SC | Noto Sans SC | 思源黑体 | **统一字体栈**：`"Source Han Sans SC", "Noto Sans SC", "PingFang SC", "Microsoft YaHei", sans-serif` —— 思源/Noto 是同一字符集，不冲突 |
| **数字字体** | Roboto Mono | JetBrains Mono | JetBrains Mono | Roboto Mono | **统一栈**：`"Roboto Mono", "JetBrains Mono", "SF Mono", monospace` —— 两者均开源、字宽接近，按可用性回退 |
| **Serif 字体** | 思源宋体 | — | — | — | **仅 Brand 文档使用**；B/C/D 一律不使用 Serif |
| **正文字号** | 10.5pt（文档）| 14px | 14px | PDA 14px / Pad 16px | 单位不同源，按载体取值 |
| **标准按钮高** | — | 32px | 32px | **48px** | 现场戴手套必须 48px；Web/平台用 32px |
| **次按钮高** | — | 24px | 24px | 38px | — |
| **表头底色** | `#1A1A1A`（黑底白字）| `#F5F5F5` | `#FAFAFA` | `#E6F4FF`（浅蓝）| **不统一** —— A 用咨询级黑底，B/C/D 用浅底；这是有意的"载体差异" |
| **状态徽标** | 淡底 + 描边 + 符号 + 文字 | 淡底 + 圆点 + 文字 | **实色填充药丸** | 淡底 + 描边 + 符号 + 文字 | **C 独立**（平台标志）；A/B/D 统一为"淡底+符号+文字" |

| **侧栏激活态** | — | 蓝字 + 浅蓝底 | `#E6F4FF` 药丸 | — | B/C 接近，但 C 必须用 Ant 药丸圆角 6px |

## 3. 共用 Token 命名约定

为避免冲突，CSS 变量**按"角色"而非"颜色名"取名**：

```css
/* 主色 —— 按轨道选其一 */
--blue-500           /* 轨道 A/B/D：品牌蓝 #1677FF */
--platform-primary   /* 轨道 C：平台蓝 #1677FF */

/* 字体 —— 全部轨道共享同一栈 */
--ff-sans            /* 思源黑体回退到 Noto Sans SC、PingFang SC */
--ff-mono            /* Roboto Mono 回退到 JetBrains Mono */
--ff-serif           /* 仅轨道 A 使用 */

/* 间距 —— 共享 */
--sp-1 ... --sp-5    /* 4/8/12/16/24 全员共享 */
--sp-6 ... --sp-8    /* 32/48/64 仅 A/B 用 */

/* 圆角 —— 各轨道按上限取值 */
--radius-xs: 2px;
--radius-sm: 4px;
--radius-md: 6px;    /* C/D 上限 */
--radius-lg: 8px;    /* 仅 C 用 */
```

→ 已在 `colors_and_type.css` 中实现。

## 4. 跨轨道一致的"绝对规则"

下列规则不分轨道，**100% 强制**：

✅ **必须做**
- 任何文字 vs 背景对比度 ≥ 4.5:1（WCAG AA）
- 数字一律用等宽（Roboto Mono / JetBrains Mono）
- 中英文之间留半角空格；数字与单位之间不留空格
- 状态用颜色 + 符号/文字双通道
- 1 px 实线分隔优先于阴影

❌ **绝不做**
- 渐变（linear/radial gradient）
- 投影（box-shadow，除 Modal/BottomSheet/Toast 极少数场景）
- 3D / 拟物 / 毛玻璃 / 光晕
- emoji（任何 UI 内）
- 饼图 / 环形图 / 3D 图表
- 纯黑 `#000000`（统一用 `#1A1A1A`）
- 第二装饰彩色（除语义色 + 极少数品牌点缀）
- 表格竖线
- 标签上浮（Material 风）
- 圆角 > 8px
- 50% 圆形头像（用 4px 方圆角代替）

## 5. 给团队的实操建议

1. **每件作品开工前先选轨道**：在文件顶部注释 `<!-- track: B (Brand-Prototype) -->`，下游能立刻判断。
2. **CSS 变量从不写死颜色**：永远用 `var(--blue-500)` / `var(--platform-primary)`。
3. **跨轨道复用组件时**：复制后必须替换 token，不允许"两个主色都出现"。
4. **AI Prompt 起手**：明确说出轨道编号 —— "请按轨道 C（Platform，#1677FF，圆角 6px，Ant 5）生成"。

## 6. 仍需用户确认的开放项

下列分歧来自原始文档本身，建议你拍板：

1. **品牌版原型 vs Field 端的圆角**：品牌版用 `0/2/4`，Field 用 `2/4/6`。是否统一为 Field 的 `2/4/6`？（目前保留差异）
2. **Roboto Mono vs JetBrains Mono**：原始文档两者并存。当前回退栈两者都包含；是否要钦定首选？
3. **Field 端 Pad 操作区背景**：原文档说 Pad 主区是 `#FAFAFA`，PDA 是 `#FFFFFF` —— 是有意区分还是错漏？
4. **平台版状态徽标**：实色填充 vs 淡底符号，平台版用前者，其余用后者。是否考虑全系统统一？（**不推荐**统一，平台版是 Ant 5 默认行为，统一会破坏客户对 Ant 的认知。）
