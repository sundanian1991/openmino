# Phase 6 · 视觉样稿确认

> 可插拔模块。独立升级不影响其他Phase。
> 本Phase用3页代表性样稿确认视觉基调，在全量设计前锁定风格方向。
> 修改成本在此处最低：3页改比50页改要快得多。

## 强制等待锚点

**在制作样稿前，先确认视觉方向：**

```
进入 Phase 6 · 视觉样稿确认。

我会先做3页样稿（封面 + 最复杂的内容页 + 图表页），
让你在全量设计前确认视觉基调。

在开始之前，有没有视觉上特别希望避免的东西？
比如：不要太花哨、配色要低调、文字要大一点、图形要简洁等等。

（没有特别要求也可以直接说"开始"）
```

等待用户回答后，才进入执行步骤。

---

## 执行步骤

### 步骤1：选取3页样稿

按以下原则选取最具代表性的3页：

1. **封面页**：代表整体视觉调性和品牌感
2. **最复杂的内容页**：包含最多信息层级的页面（通常是有数据+文字+图表的页面）
3. **图表页**：代表数据可视化的风格（若无图表页，选一页有视觉元素的内容页）

---

### 步骤2：SVG样稿设计

读取 `designer-rules.md` 和 `layout-impl-[系].md`，按布局规划卡设计3页SVG。

设计时严格执行：
- 颜色决策全部查BRAND_RULES，不自定义任何色值
- 字体严格按BRAND_RULES（品牌模式：宋体-简/微软雅黑；通用模式：Arial/微软雅黑）
- Action Title（结论式标题）≤20字，宋体-简 Bold，左侧3pt品牌色竖线
- 页脚包含：页码/版本/日期/保密标注
- 图表页读 `chart-impl.md`，禁用饼图、不用3D效果、不用渐变

---

### 步骤3：样稿展示与确认

展示3页样稿后，引导用户从以下维度反馈：

**颜色调性：** 这个配色是你想要的那种感觉吗？
**信息密度：** 文字太多了、刚刚好，还是太少了？
**整体专业感：** 给你的客户/受众看，你有信心吗？

如果反馈模糊，执行 coach-engine.md 中「Phase 6 · 样稿确认」引导序列。

**修改处理规则：**
- 用户指出具体修改项 → 修改后重新展示，再次确认
- 用户全面否定（"整体感觉不对"）→ 确认BRAND_RULES是否正确加载，重新理解用户期望的风格，重做样稿
- 用户确认满意 → 记录视觉基调，进入完成卡

## 跳过风险

```
风险等级：中
风险说明：
  跳过样稿确认，直接进入全量设计，
  如果视觉基调不满意，需要修改全部页面。
  10页PPT可能影响不大，但30页以上的PPT，
  返工成本极高。强烈建议不跳过。
```
