## 强制等待锚点

**在开始执行前，先问用户以下问题，等待回答后才进入执行步骤：**

```
进入 Phase 1 · 规范与受众定义。

在我开始之前，请告诉我：
1. 这份PPT是给谁看的？（例如：客户老板、内部管理层、投资人）

```

## 执行步骤

### 步骤1：品牌规范检测

```
检查路径：references/brand-spec/brand.md
```

**文件存在 → 品牌模式：**
读取 §2 配色体系 / §3 字体体系 / §7 PPT规范，保存为 BRAND_RULES。
告知用户：「已加载你的品牌规范，设计将严格遵循规范配色和字体。」

**文件不存在 → 通用商务模式：**
加载内置通用商务规范（见下方"通用商务规范"）。
告知用户：「未检测到品牌规范文件，将使用通用商务样式。如需使用品牌规范，可以上传brand.md文件后重新开始。」

**通用商务规范（品牌模式不可用时的降级方案）：**
- 主色：#1A3C5E（深蓝）
- 正文色：#2D2D2D（近黑，非纯黑）
- 辅助色：#4A7CAB（中蓝）
- 背景色：#FFFFFF（白）
- 标题字体：微软雅黑 Bold / Arial Bold
- 正文字体：微软雅黑 Regular / Arial Regular
- 最大字体层级：3级（标题/副标题/正文）
- 禁用：渐变、3D效果、投影、纯黑色（#000000）、饼图
- 图表、间距、版式规则与品牌模式保持一致

---

### 步骤2：需求收集

AI主动收集以下信息（可推断的不问，必须问的一次性问完）：

| 字段 | 可推断？ | 规则 |
|------|---------|------|
| PPT主题 | 否 | 必须用户给 |
| 受众是谁 | 否 | 必须用户给 |
| 汇报目标 | 可 | 客户→说服；内部→汇报；外部会议→展示 |
| 期望页数 | 可 | 未指定→默认12页 |
| 用户材料 | 否 | 无材料时质量下降，必须提醒 |
| 交付模式 | 可 | 现场讲→presenting；带走读→reading；都要→hybrid（默认） |
| 输出格式 | 可 | 未指定→both（SVG预览版+可编辑PPTX同时交付） |

**交付模式（deck_mode）推断规则：**
- 路演、发布会、演讲 → presenting（以演讲为主）
- 客户方案、顾问报告、项目汇报 → hybrid（默认，现场讲解+带走阅读）
- 分析报告、白皮书、研究文档 → reading（以阅读为主）

---

### 步骤3：受众深度定义

在获得基本信息后，针对受众进行深度定义（专家模式简化，教练模式完整执行）：

**必问（任何模式）：**
- 受众的职位或角色是什么？
- 他们在这件事上最关心的是什么？

**教练模式追问（见 coach-engine.md Phase 1引导序列）**

**输出：受众卡**
```
受众：[职位/角色]
关心：[核心关切1] / [核心关切2]
预期行动：[看完PPT后他们应该做的决定]
主要顾虑：[他们最可能提出的反对意见]
```

---

### 步骤4：能力档位检测

读取 `coach-engine.md`，执行"Phase 1末尾：初始检测（三道题）"。
记录档位，写入工作区，供后续所有Phase使用。

## 跳过风险

```
风险等级：极高
风险说明：
  受众定义是所有论点选择的基础。
  跳过此步，Phase 3的灵魂句和Phase 4的论点将无法针对
  具体受众的关切，导致整份PPT方向性偏移。
  品牌规范未加载将导致所有设计阶段颜色/字体错误，
  Phase 8质检会产生大量ERROR需要返工。
```
