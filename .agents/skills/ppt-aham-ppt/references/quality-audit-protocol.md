# Quality Audit Protocol · 质量审计协议

> 可插拔模块。独立升级不影响其他Phase。
> 适用阶段：Phase 7（逐页设计）→ Phase 8（质检与交付）。
> 目标：在调用 svg_to_pptx 转换之前，通过自动化检测拦截已知问题；
> 在交付之前，通过标准化审计流程确保输出质量。

---

## 一、三级关卡机制（Three-Gate System）

每页SVG生成后、调用转换工具前，工具链自动执行三级检查。
检查结果以 WARNING 形式输出到转换日志，**不阻断转换流程**
（因为部分WARNING是风格问题，由人工决定是否修改）。

| 关卡 | 检查对象 | 工具链行为 | 需要人工干预的情况 |
|:---:|:---|:---|:---|
| Gate 1 | 逻辑完整性 | 检测空的 `<text>` 元素（占位符未填充） | WARNING出现时，回到Phase 7修复内容 |
| Gate 2 | 版面合规性 | 检测单行CJK字符超过20个 | WARNING出现时，拆分文字或缩小字号 |
| Gate 3 | 代码合法性 | 自动修复裸露`&`；检测rgb()/hsl()颜色格式 | rgb()/hsl() WARNING出现时，改为品牌规范hex色值 |

**颜色政策（硬性底线）：** 所有颜色决策以 `brand.md` 品牌规范为唯一来源。
工具链不做颜色白名单校验，但会对 `rgb()/hsl()` 格式发出 WARNING，
因为转换器只支持 hex 格式，遇到 `rgb()` 会静默 fallback 到黑色。

## 二、转换前自动执行项（工具链内置，无需手动触发）

以下检查在 `svg_to_pptx_wrapper.py` 调用时自动执行：

```
每个SVG文件转换时：
1. [Gate 3] 自动修复裸露 & → &amp;
2. [Gate 3] 扫描 rgb()/hsl() 颜色格式，输出 WARNING
3. [Gate 2] 统计每行CJK字数，超过20字输出 WARNING
4. [Gate 1] 检测空 <text> 元素，输出 WARNING
5. [tspan] 自动将 <tspan dy=...> 拆成独立 TextBox（无需人工处理）
```

## 三、交付前人工审计流程

### 步骤1：HTML预览审查（强制）

所有SVG转换为PPTX之前，先生成 `preview.html` 预览文件。
在浏览器中逐页检查：

```
审查清单：
□ 文字是否有溢出（超出卡片或页面边界）
□ 文字是否有重叠（两个TextBox坐标重合）
□ 箭头是否正确显示（用polygon画的箭头应完整）
□ 多行文字是否正确换行（不能挤在一行）
□ 颜色是否符合品牌规范（无异常色块）
□ 转换日志中是否有 WARNING（对照警告逐项检查）
```

### 步骤2：PPTX抽样检查

转换完成后打开PPTX，抽取3页检查（选最复杂的图表页）：

```
检查项：
□ 文字是否可编辑（双击文字框应进入编辑状态）
□ 形状是否可独立选中（单击形状应选中单个元素）
□ 图片是否变形（图片宽高比应与SVG一致）
□ 中文字体是否正确（应为微软雅黑/宋体，无乱码）
```

### 步骤3：QC报告输出

Phase 8 完成卡中包含以下信息：

```
QC报告 · [项目名称] · [日期]
─────────────────────────────
总页数：[X] 页
转换模式：Native DrawingML（可编辑）
转换日志 WARNING 汇总：
  Gate 1（空内容）：[X] 处
  Gate 2（文字溢出风险）：[X] 处
  Gate 3（颜色格式）：[X] 处
颜色合规：所有颜色已按品牌规范（brand.md）使用
文件大小：[X] KB
─────────────────────────────
已知问题：[如有，列出]
```

## 四、AI生成SVG时的强制约束（Prompt附加段）

在执行逐页设计任务时，以下约束自动附加，无需重复提示：

```
[质量管控约束 · 自动执行]
1. 特殊字符：文字内容中的 & 写为 &amp;，< 写为 &lt;，> 写为 &gt;
2. 颜色格式：所有颜色使用 hex 格式（如 var(--brand-primary)），禁止 rgb() 或 hsl()
             颜色必须来自品牌规范（brand.md），这是硬性底线
3. 换行方式：多行文字用 <tspan x="..." dy="1.4em"> 或独立 <text y="...">
             禁止在文字内容中使用 \n
4. 箭头画法：禁止 <marker>，必须用 <polygon> 绘制箭头头部
5. 绝对坐标：禁止 <g transform="rotate(...)">，使用绝对坐标
6. 单行长度：CJK文字单行不超过20字（16px字号），超出必须换行
```

## 五、常见问题快速定位

| 现象 | 最可能原因 | 快速修复 |
|---|---|---|
| 整页空白 | SVG含未转义`&` 或XML结构损坏 | 检查转换日志的ParseError；工具链已自动修复`&` |
| 箭头消失 | 用了`<marker>`在`<path>`上 | 改用`<polygon>`画箭头 |
| 多行文字变一行 | 用了`\n`换行 | 改用`<tspan dy>`或独立`<text>` |
| 颜色变黑 | 用了`rgb()`格式颜色 | 改为hex格式，查brand.md对应色值 |
| 文字位置偏移 | `<g transform="rotate">`坐标偏移 | 用元素级绝对坐标替代 |
| 文字溢出容器 | 单行字数过多 | 换行，或按字数参考表缩减内容 |
| 文字无法编辑 | 非native模式转换 | 确认通过`svg_to_pptx_wrapper.py`调用 |
