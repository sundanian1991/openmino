# Visual Expression

> 数据不是孤立的，它是世界的一环。可视化是打开这个世界的方式。

---

## 核心命题

可视化不是"用什么图表"，而是"如何让数据背后的关系显性化"。

数据是冰山的一角，水下是更大的世界——原因、影响、关联、历史、未来。好的可视化让人看到冰山的全貌；差的可视化只让人看到水面的那一块。

**金条赛马案例**：不是"柱状图对比"——是"供应商之间此消彼长的权力博弈被图形化了"。数字是表象，博弈是本质。

---

## 三个层次

### 层次 1：数据呈现（最低）

**问题：** 这个数据是什么？
**答案：** 柱状图、饼图、折线图
**结果：** 数据被看见了，但没有被理解

大多数可视化停在这一层。它回答了"是什么"，但没有回答"为什么"和"意味着什么"。

### 层次 2：关系显性化（中等）

**问题：** 这个数据和什么有关？
**答案：** 数据与原因、影响、对比、趋势的关系
**结果：** 数据被理解了，但没有被连接

这一层开始看到数据背后的结构——因果、对比、演化。但仍然在数据本身的范围内。

### 层次 3：世界连接（最高）

**问题：** 这个数据在更大的世界中意味着什么？
**答案：** 数据与历史、文化、人性、自然的连接
**结果：** 数据被连接了——它不再是孤立的数字，而是世界的一部分

这是 showcase 级别的可视化。它让人看到的不是数据，而是数据背后的世界。

---

## 打开世界的方法

### 1. 追问"为什么"

每个数据点背后都有原因。追问到底：

```
数据：供应商 A 的业绩下滑了 20%
为什么？→ 市场竞争加剧
为什么？→ 新玩家进入，价格战
为什么？→ 行业门槛降低
为什么？→ 技术普及，供应链成熟
→ 可视化不是"下滑 20%"，而是"一个行业的演变"
```

### 2. 追问"意味着什么"

每个数据点都有影响。追问到底：

```
数据：供应商 A 的业绩下滑了 20%
意味着什么？→ 市场份额重新分配
意味着什么？→ 其他供应商有机会
意味着什么？→ 行业格局在变化
意味着什么？→ 我们需要重新评估合作策略
→ 可视化不是"下滑 20%"，而是"一个战略决策的时机"
```

### 3. 追问"连接什么"

每个数据点都和其他事物关联。追问到底：

```
数据：供应商 A 的业绩下滑了 20%
连接什么？→ 供应商 B 的业绩上升了 15%
连接什么？→ 整体市场增长了 5%
连接什么？→ 客户需求在变化
连接什么？→ 行业技术在迭代
→ 可视化不是"一个供应商"，而是"一个生态系统的动态平衡"
```

## 可视化策略

不是"用什么图表"，而是"用什么视角打开世界"。

### 视角 1：时间视角

**打开什么：** 数据在时间中的位置——过去、现在、未来

**可视化语言：**
- 时间轴：数据在历史中的位置
- 演化图：数据如何变化
- 预测线：数据将去向何方

**适用场景：** 品牌历史、业绩趋势、行业发展

### 视角 2：空间视角

**打开什么：** 数据在空间中的位置——地理、层级、结构

**可视化语言：**
- 地图：数据的地理分布
- 树图：数据的层级结构
- 网络图：数据的连接关系

**适用场景：** 区域业绩、组织架构、生态系统

### 视角 3：对比视角

**打开什么：** 数据与其他数据的关系——竞争、协作、消长

**可视化语言：**
- 对抗式布局：此消彼长
- 雷达图：多维度对比
- 平行坐标：多变量关系

**适用场景：** 供应商对比、竞品分析、能力矩阵

### 视角 4：因果视角

**打开什么：** 数据背后的原因和影响

**可视化语言：**
- 流程图：因果链条
- 桑基图：流量和转化
- 树形图：决策分支

**适用场景：** 问题分析、决策推导、影响评估

### 视角 5：叙事视角

**打开什么：** 数据讲述的故事

**可视化语言：**
- 信息图表：数据驱动的叙事
- 时间轴故事：数据在时间中的展开
- 渐进展开：数据的发现过程

**适用场景：** 年度报告、品牌故事、项目回顾

---

## 选择视角的方法

不是"数据适合什么图表"，而是"我想让人看到什么"。

**决策框架：**

1. **我手头的数据是什么？** → 理解数据本身
2. **我想让人看到什么？** → 确定目标
3. **数据背后更大的世界是什么？** → 打开视野
4. **哪个视角能打开这个世界？** → 选择视角
5. **用什么视觉语言表达这个视角？** → 选择语言

**例子：**

```
数据：5 个供应商的业绩排名
目标：让人看到竞争格局
更大的世界：行业生态、市场动态、合作策略
视角：对比视角 + 时间视角
语言：动态平衡 + 时间轴
→ 不是"柱状图排名"，而是"竞争格局的演变"
```

## showcase 级别的标准

**数据呈现**：数据被看见了
**关系显性化**：数据被理解了
**世界连接**：数据被连接了——它不再是孤立的数字，而是世界的一部分

达到 showcase 级别的可视化：
1. **打开视野**：让人看到数据背后更大的世界
2. **显性关系**：让关系变得可见、可理解
3. **产生共鸣**：让人感受到数据与自己的关联
4. **激发思考**：让人想要追问更多

---

## 反模式

1. **图表先行** → 数据被塞进预设的图表类型
2. **孤立呈现** → 数据被看见了，但没有被连接
3. **过度装饰** → 图表很好看，但没有打开世界
4. **忽视语境** → 数据被剥离了背景
5. **固化模式** → 每次都用同样的可视化策略
