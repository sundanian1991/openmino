# Design Process

> 从输入到输出的标准化流程——确保每次产出都符合品牌理念和质量要求。

---

## 三种输入场景

### 场景 1：链接输入

**触发：** 用户给一个 URL，要求"看看这个"或"按这个风格做"

**流程：**

```
获取链接 → 阅读页面 → 提取设计 DNA → 识别张力 → 选择可视化策略 → 产出
```

**详细步骤：**

1. **获取**：用 WebFetch 获取页面内容
2. **阅读**：理解品牌、产品、信息架构
3. **提取**：识别现有设计元素（排版、色彩、图表、交互）
4. **识别**：找到品牌的核心张力（参见 tension-philosophy.md）
5. **匹配**：根据张力选择可视化策略（参见 visual-expression.md）
6. **产出**：用 Hallmark 组件 + 品牌张力驱动的可视化

**输出检查清单：**
- [ ] 是否识别了品牌的核心张力？
- [ ] 可视化是否表达了这个张力？
- [ ] 色彩是否来自品牌色板？
- [ ] 排版是否符合品牌声音？
- [ ] 每个视觉元素是否有理由？

---

### 场景 2：内容输入

**触发：** 用户给一段文字/数据，要求"做成 HTML"

**流程：**

```
分析内容 → 识别信息类型 → 匹配可视化策略 → 选择组件 → 产出
```

**详细步骤：**

1. **分析**：理解内容结构（标题、段落、列表、数据）
2. **识别**：判断信息类型（对比、趋势、组成、关系、分布）
3. **匹配**：根据信息类型选择可视化策略（参见 visual-expression.md § 三种输入场景）
4. **选择**：从 Hallmark 组件库选择合适的组件
5. **产出**：用组件 + 可视化策略构建页面

**内容类型 → 组件映射：**

| 内容类型 | 推荐组件 | 可视化策略 |
|---------|---------|-----------|
| **品牌介绍** | Hero + Tension Grid + Color Swatch + Typography Showcase | 张力驱动 |
| **产品对比** | Feature Grid + Numeral Grid | 对抗式布局 |
| **时间线** | Step Sequence + Symbol Evolution | 演化图 |
| **数据报告** | Stat-Led + Numbered Stat Strip | 信息图表 |
| **设计理念** | Pull Quote + Design Principles | 叙事驱动 |

---

### 场景 3：文件输入

**触发：** 用户给 PDF/PPT/Excel/HTML，要求"处理这个"

**流程：**

```
转化为可读形式 → 提取数据 → 识别模式 → 匹配可视化策略 → 产出
```

**详细步骤：**

1. **转化**：
   - PDF → 提取文本 + 图表（用 Read 工具或 OCR）
   - PPT → 提取幻灯片内容（解析结构）
   - Excel/CSV → 直接读取数据
   - HTML → 解析 DOM + CSS

2. **提取**：识别数据结构（分类、数值、时间、关系）
3. **匹配**：根据数据特征选择可视化策略
4. **产出**：用 Hallmark 组件 + 数据驱动的可视化

**数据特征 → 可视化策略：**

| 数据特征 | 可视化策略 | Hallmark 组件 |
|---------|-----------|---------------|
| **分类对比** | 柱状图、雷达图 | F1 Bento, F3 Tabular |
| **占比关系** | 饼图、堆叠图 | F1 Bento |
| **趋势变化** | 折线图、面积图 | F4 Step Sequence |
| **多维关系** | 散点图、平行坐标 | F1 Bento |
| **层级结构** | 树图、旭日图 | F4 Step Sequence |

## 设计决策框架

每个产出都经过这个决策框架：

### Step 1: 识别张力

**问题：** 这个品牌/内容的核心张力是什么？

**常见张力：**
- Heritage ↔ Innovation
- Organic ↔ Engineered
- Nature ↔ Technology
- Data ↔ Narrative
- Competition ↔ Collaboration
- Simplicity ↔ Complexity

**输出：** 一句话描述核心张力

### Step 2: 选择组件

**问题：** 哪些 Hallmark 组件能表达这个张力？

**选择原则：**
- 组件的结构应该强化张力（不是削弱）
- 组件的变体应该反映张力的两端
- 组件的组合应该创造节奏感

**输出：** 组件清单 + 变体选择

### Step 3: 设计可视化

**问题：** 数据/内容如何用图形表达这个张力？

**设计原则：**
- 张力驱动（不是数据驱动）
- 约束产生风格（不是模板产生风格）
- 每个像素都要有理由

**输出：** 可视化策略 + 图表类型

### Step 4: 验证一致性

**问题：** 整体产出是否符合品牌理念？

**验证清单：**
- [ ] 色彩是否来自品牌色板？
- [ ] 排版是否符合品牌声音？
- [ ] 可视化是否表达了张力？
- [ ] 组件选择是否一致？
- [ ] 每个视觉元素是否有理由？

**输出：** 通过/不通过 + 修改建议

## 产出质量标准

### 优雅（Elegance）

- 每个元素都有理由
- 没有装饰性元素
- 信息密度合适
- 视觉节奏清晰

### 克制（Restraint）

- 不过度设计
- 不堆砌效果
- 留白合适
- 动画适度

### 专业（Professionalism）

- 符合品牌理念
- 技术细节准确
- 代码质量高
- 响应式适配

---

## 反模式

1. **跳过张力识别** — 直接选组件 → 产出像模板
2. **组件堆砌** — 用很多组件但没有逻辑 → 产出像拼贴
3. **可视化脱节** — 图表很好看但不符合品牌 → 产出像素材
4. **忽略验证** — 产出后不检查一致性 → 质量不稳定
5. **固化模式** — 每个项目都用同样的流程 → 失去表达力
