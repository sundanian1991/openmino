# Design Inspiration Index

Design philosophy, principles, and thinking patterns from world-class brand design systems. Use these as conceptual input — not templates to copy.

---

## Philosophy Files

### [The Power of Tension](tension-philosophy.md)

How the best design systems resolve dualisms — heritage vs future, organic vs engineered, nature vs tech. Tension creates identity; harmony creates wallpaper.

**Core insight:** A design system isn't a collection of assets — it's a set of tensions held in balance.

### [Typography as Voice](typography-as-voice.md)

Typeface selection isn't decoration — it's the brand's voice made visible. How to think about type pairing, weight hierarchy, and emotional register.

**Core insight:** The 2+1 rule — Display + Body + Technical companion. Weight as rhythm; bold everywhere means bold nowhere.

### [The Gap Between 小孩 and 大人](child-vs-adult.md)

Why the same components, used differently, produce completely different quality levels. The difference is process, not tools.

**Core insight:** Components are the *result*, not the *starting point*. Start with the tension, end with the components.

### [Color as System](color-as-system.md)

Color palettes aren't collections — they're systems with roles, hierarchies, and relationships. How to build palettes that work, not just look good.

**Core insight:** Every color plays one of three roles: Foundation (ground), Signal (accent), Neutral (support). Roles determine hierarchy; swatches don't.

### [Separation and Flexibility](separation-flexibility.md)

Why the best brands separate their wordmark from their symbol — and how that unlocks dynamic product applications across touchpoints.

**Core insight:** A locked logotype is the most common pattern — and the most limiting. Separation unlocks context adaptability, scale flexibility, and dynamic expression.

### [Numbers as Graphics](numbers-as-graphics.md)

When numerals become design elements — technical expression without the brand mark. The role of numeral typefaces in brand systems.

**Core insight:** The numeral typeface extends the symbol's DNA into typography — providing technical expression without overusing the symbol itself.

### [Visual Expression](visual-expression.md)

Data isn't isolated — it's a window into a larger world. Visualization is how we open that window.

**Core insight:** Three levels of visualization — data presentation (what), relationship explicit (why), world connection (meaning). Show level = level 3.

### [Design Process](design-process.md)

The standard workflow from input to output — ensuring every output meets brand philosophy and quality standards.

**Core insight:** Three input scenarios (link, content, file), one decision framework: identify tension → select components → design visualization → verify consistency.

### [Design Virtue](design-virtue.md)

做东西要有美德。克制、尊重、诚实、聚焦、呼吸——不是方法论，是设计态度。

**Core insight:** 数据有限时，聚焦是克制。留白不是没东西放，是故意留出的空间。高级感来自克制，不是来自装饰。

## The Eight Principles

These eight files share a common thread: **design systems are about constraints, tensions, and virtues** — not collections of beautiful assets.

| Principle | Core Tension | Anti-Pattern |
|-----------|--------------|--------------|
| **Tension** | Heritage ↔ Future | Picking a side |
| **Typography** | Display ↔ Body | Too many fonts |
| **Color** | Foundation ↔ Signal | Rainbow palettes |
| **Separation** | Symbol ↔ Wordmark | Forced lockup |
| **Numbers** | Graphic ↔ Functional | Overusing the symbol |
| **Visual Expression** | Data ↔ World | Chart type first |
| **Design Process** | Structure ↔ Flexibility | Skipping tension |
| **Design Virtue** | Restraint ↔ Expression | Decorative filling |

A system that holds these tensions creates identity. A system that resolves them creates wallpaper. A system that practices these virtues creates something worth keeping.

## How to Use Design Inspiration

1. **Read before building** — Let the philosophy shape your thinking before you write code
2. **Extract principles** — Don't copy solutions; extract the underlying thinking
3. **Apply to your context** — The principles are universal; the implementation is specific
4. **Test against the tensions** — Does your design hold the tension, or resolve it?

## Adding New Inspiration

To add a new design philosophy file:

1. Create a `.md` file with clear, actionable thinking patterns
2. Add an entry to this INDEX.md
3. Focus on *why* and *how to think*, not *what to build*
4. Use real examples, but extract principles, not pixels
5. Identify the core tension — every good design principle resolves a dualism
