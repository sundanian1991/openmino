# 场景论文库 — Scene Thesis Database

> 对标 fe-cinematic 的 scene thesis 机制。每种可视化场景一句话定义"灵魂 + 签名元素 + 不可简化的理由 + 克制声明"。
> 让每张图有存在的意义，而不是"为了画图而画图"。

---

## 使用说明

当你在 Phase 2 (storyboard) 中为单图写场景论文时，按以下流程：

1. **识别场景**：你要画的是什么场景？（业绩汇报？目标差距？多维评估？）
2. **查找匹配**：在下方按关键词或功能分类找到对应条目
3. **直接引用**：复制该条目的 scene thesis + 签名元素 + 不可简化理由 + 克制声明
4. **适配数据**：根据你的具体数据微调数字和描述

每个条目包含：
- **场景名称** — 简短描述
- **一句话灵魂** — 这张图存在的唯一理由
- **签名视觉元素** — 去掉它，这张图就不是这张图了
- **不可简化理由** — 为什么默认模板（柱状/饼图/折线）不行
- **克制声明** — 不加什么（比加什么更重要）

## 一、目标与差距（C09 / C14 / C12）

### 1.1 目标达标率差距

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 用一条 100% 基准线切开达标与未达标——分化不是模糊的，是精确到百分点的 |
| **签名视觉元素** | 双向横向柱状图：从 100% 线向两侧延伸，达标右伸、未达标左伸，基准线本身是视觉锚点 |
| **不可简化理由** | 默认柱状图从 0 开始，掩盖了"达标 vs 未达标"的分界线。本图以 100% 为零点，缺口和超额同时可见 |
| **克制声明** | 不展示拉新/促提分项；不加图例（柱端直接标注达标/未达标）；不用 3D/渐变/阴影 |
| **适用模式** | C09-差距到目标、C14-子弹图 |
| **典型节拍** | C-04 色彩弹出 + C-02 基准线穿越 |

### 1.2 单指标 vs 目标进度

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 距离目标还有多远——一个进度条比一张表更有紧迫感 |
| **签名视觉元素** | 子弹图：短横条 + 目标刻度线 + 背景色带（红→黄→绿）|
| **不可简化理由** | 纯柱状图显示绝对值，不显示"距离目标还有多少"。子弹图将目标、当前、区间三合一 |
| **克制声明** | 不标注具体数字在柱子上（用右侧标注）；背景色带只用 2 级（红/绿），不用 3 级 |
| **适用模式** | C14-子弹图/温度计、C09-差距到目标 |
| **典型节拍** | C-01 锚定数字 + C-09 叠加融合 |

### 1.3 同比/环比变化拆解

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 从 A 到 B 的变化不是"涨了多少"，而是"为什么涨" |
| **签名视觉元素** | 瀑布图：起点和终点加粗柱子，中间正负驱动因素柱，正绿负红 |
| **不可简化理由** | 柱状图只显示起点和终点，不显示中间的驱动因素。瀑布图把"增量从哪里来"拆解清楚 |
| **克制声明** | 驱动因素 ≤8 个，超过则聚合为"其他"；不添加百分比标签在柱子上（右侧标注） |
| **适用模式** | C02-瀑布分析、C12-桥接图 |
| **典型节拍** | C-02 基准线穿越 + C-07 标注箭头 |

## 二、对比与排名（13 / C03 / C05）

### 2.1 分类排行

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 谁行谁不行——排序本身就是一种判断 |
| **签名视觉元素** | 横向条形图严格降序排列，颜色从饱和到浅色渐变，标签在左侧可读 |
| **不可简化理由** | 纵向柱状图标签旋转 45° 不可读。横向排序让每个名字都清晰可见，排序本身传递信息 |
| **克制声明** | 不加网格线；不添加数值标签在柱内（柱端标注）；Top 1 和 Bottom 1 外其余不加粗 |
| **适用模式** | SK-13 柱状图、SK-26 并列比较 |
| **典型节拍** | C-03 排序阶梯 + C-04 色彩弹出 |

### 2.2 A/B 对比

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 两者差异不是抽象的——放在同一个坐标系里，差距会说话 |
| **签名视觉元素** | 蝴蝶图（左右对称反向柱状）或并列柱状，共享 Y 轴和刻度范围 |
| **不可简化理由** | 两张分开的图无法直接比较绝对差距。蝴蝶图让差异在同一个视觉平面内自动浮现 |
| **克制声明** | Y 轴刻度必须完全一致；不加 3 方以上对比（太复杂）；不标注所有数值（只标注差异最大的 2 个） |
| **适用模式** | SK-26 并列比较、构图模板 02 左右对比 |
| **典型节拍** | C-05 对比揭示 + C-13 近邻关联 |

### 2.3 四象限定位

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 把事物放到四个格子里——位置本身就是判断 |
| **签名视觉元素** | 2×2 网格，十字分割线清晰，每个象限有命名（如明星/瘦狗），气泡大小=第三维度 |
| **不可简化理由** | 散点图没有预设的判断框架。四象限天然表达"好/坏 × 大/小"这类二元判断 |
| **克制声明** | 象限命名必须有业务含义，不用"第一象限"等学术用语；气泡不重叠（强制错开） |
| **适用模式** | C03-2x2 矩阵、SK-05 气泡图 |
| **典型节拍** | C-04 色彩弹出 + C-10 密度变换 |

## 三、趋势与演变（07 / 08 / C07）

### 3.1 时间趋势

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 不是"数据在变化"，而是"什么事件在驱动变化" |
| **签名视觉元素** | 折线图 + 拐点标注（圆点 + 注释框），关键事件用灰色竖带标注 |
| **不可简化理由** | 纯折线图只显示波动，不解释"为什么变"。拐点标注把事件与数据变化连接起来 |
| **克制声明** | 注释 ≤25 字，写事件不写数字；线宽 1.5px，不加粗；不添加多余的趋势线 |
| **适用模式** | SK-07 折线图、C07-趋势+拐点叙事 |
| **典型节拍** | C-08 趋势拐点 + C-07 标注箭头 |

### 3.2 累积趋势

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 总量在增长，但各部分的贡献不同——面积让你看到"谁在拖后腿" |
| **签名视觉元素** | 堆叠面积图，底层用深色，顶层用浅色，总面积表达累积量 |
| **不可简化理由** | 折线图只能看到每个系列的独立走势，看不到"总量中各部分的关系" |
| **克制声明** | 系列 ≤4 个，超过用"其他"聚合；面积不用渐变填充（纯色 + 透明度） |
| **适用模式** | SK-08 面积图、C08-堆叠构成 |
| **典型节拍** | C-14 面积编码 + C-09 叠加融合 |

## 四、结构与占比（01 / 16 / C08）

### 4.1 占比构成

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 整体不是均质的——谁占大头、谁可以忽略，一眼看清 |
| **签名视觉元素** | 堆叠条形图或矩形树图，面积/长度严格∝占比，最大块用最深色 |
| **不可简化理由** | 饼图在 >5 类时无法精确比较大小。堆叠条/矩形树图支持精确读数和排序 |
| **克制声明** | ≤3 类用饼图，4-7 类用堆叠条，>7 类用矩形树图；不添加百分比在扇区/矩形内（右侧标注） |
| **适用模式** | SK-01 饼图、SK-16 矩形树图、C08-堆叠构成 |
| **典型节拍** | C-14 面积编码 + C-03 排序阶梯 |

### 4.2 多层级占比

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 从总部到部门再到个人——每一层的占比都在变化 |
| **签名视觉元素** | 旭日图或矩形树图嵌套，层级用颜色深浅或边框粗细区分 |
| **不可简化理由** | 单层饼图无法表达嵌套结构。旭日图天然支持"从内到外"的层级钻取 |
| **克制声明** | 层级 ≤3 层，超过信息过载；最内层标签不旋转（水平可读） |
| **适用模式** | SK-10 旭日图、SK-16 矩形树图 |
| **典型节拍** | C-14 面积编码 + C-19 层级折叠 |

## 五、转化与流向（20 / 32 / C06）

### 5.1 转化漏斗

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 从 100% 到最终转化——每一步都在流血，最大伤口在哪一目了然 |
| **签名视觉元素** | 上宽下窄漏斗分层，每层标注百分比，最大流失率用红色标注 + 箭头 |
| **不可简化理由** | 柱状图递减无法表达"从上一层到下一层的流失率"。漏斗的宽度变化天然表达衰减 |
| **克制声明** | 层级 ≤6 层，超过合并；不添加绝对值在漏斗内（右侧标注）；不用 3D 漏斗 |
| **适用模式** | SK-32 漏斗、C06-漏斗+瓶颈标注 |
| **典型节拍** | C-06 异常闪烁 + C-03 排序阶梯 |

### 5.2 资源流向

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 钱/人/客户从哪里来、到哪里去——路径比总量更重要 |
| **签名视觉元素** | 桑基图：左源右目标，线宽∝流量，颜色区分流向 |
| **不可简化理由** | 柱状图只显示源和目标各自的总量，不显示"谁流向了谁" |
| **克制声明** | 层级 ≤5 层，节点 ≤15 个，超过则聚合；不添加箭头（桑基的方向性已足够） |
| **适用模式** | SK-20 桑基图、C15-桑基流 |
| **典型节拍** | C-15 流动引导 + C-12 分组归类 |

## 六、多维与评估（22 / C13 / C10）

### 6.1 多维能力画像

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 不是"总分多少"，而是"哪条腿短"——短板在雷达上无处隐藏 |
| **签名视觉元素** | 雷达图：多边形覆盖 4-8 个维度，基准线虚线灰色，突出维度用饱和色描边 |
| **不可简化理由** | 柱状图把多维拆成独立的柱子，失去了"综合能力全貌"的感知。雷达的多边形形态让短板自然浮现 |
| **克制声明** | 维度 4-8 个，不在范围内不用雷达；对比对象 ≤3 个（雷达叠加）；不填充半透明色（纯色描边） |
| **适用模式** | SK-22 雷达图、SK-18 元素周期表 |
| **典型节拍** | C-04 色彩弹出 + C-05 对比揭示 |

### 6.2 定性评估矩阵

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 有些东西无法量化——但可以用"满/中/空"来表达程度差异 |
| **签名视觉元素** | Harvey Ball 矩阵：行=评估对象，列=评估维度，单元格用 ◉/◐/○ 填充 |
| **不可细化理由** | 数值评分在定性场景（成熟度、体验、能力）中制造虚假精确感。Harvey Ball 用图形直觉替代数字 |
| **克制声明** | 只设 3-4 级（满/中/空/无），不设 5 级；行 ≤15 个，超过分页；不加颜色（黑白即可） |
| **适用模式** | C13-Harvey Ball 矩阵、SK-17 网格图 |
| **典型节拍** | C-12 分组归类 + C-03 排序阶梯 |

### 6.3 散点+象限洞察

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 两个维度交叉——谁在好位置、谁跑偏了、谁是 outlier |
| **签名视觉元素** | 散点图 + 四象限背景色（10% 透明度）+ 平均线（虚线），异常值用红色标注 |
| **不可简化理由** | 单独看 X 或 Y 都无法识别 outlier。散点+象限把"相关性"和"异常值"同时暴露 |
| **克制声明** | 数据点 <15 个时不用散点（太稀疏）；不添加强制回归线（除非有明确相关性假设） |
| **适用模式** | SK-09 散点图、C10-散点+象限洞察 |
| **典型节拍** | C-10 密度变换 + C-06 异常闪烁 |

## 七、抽象类比（31-72）

### 7.1 冰山—显隐对比

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 你看到的只是表面——水面下才是真正的问题 |
| **签名视觉元素** | 水面线贯穿画面，水上小三角（可见问题），水下大三角（隐藏根因），水线标注"可量化指标" |
| **不可简化理由** | 柱状图无法表达"显性 vs 隐性"的比例关系。冰山的水线天然分隔已知与未知 |
| **克制声明** | 水下部分不用具体数字（因为本来就是未知的）；水线用蓝色虚线，不用实线 |
| **适用模式** | SK-49 冰山、SK-31 金字塔（备选） |
| **典型节拍** | C-02 基准线穿越 + C-11 隔离突出 |

### 7.2 金字塔—层级优先

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 从底部的基础到顶部的巅峰——每一层都是上一层的前提 |
| **签名视觉元素** | 自下而上收窄的梯形堆叠，底层最宽颜色最深，顶层最窄颜色最浅 |
| **不可简化理由** | 柱状图无法表达"下层是上层的基础"这种依赖关系。金字塔的收窄形态天然传递优先级 |
| **克制声明** | 层级 3-5 层，不用更多；每层文字在内部居中，不在外部标注；不加箭头 |
| **适用模式** | SK-31 金字塔、SK-03 同心图（备选） |
| **典型节拍** | C-12 分组归类 + C-03 排序阶梯 |

### 7.3 齿轮—联动依赖

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 没有一个部门是孤岛——转一个，全部跟着动 |
| **签名视觉元素** | 2-3 个咬合的齿轮，大小不同表达权重差异，箭头表示转动方向 |
| **不可简化理由** | 柱状图/饼图是静态的，无法表达"联动"和"依赖"。齿轮的咬合形态天然传递相互影响 |
| **克制声明** | 齿轮数量 ≤4 个，超过变混乱；齿轮齿数一致（表达平等咬合）；不加过多文字标签 |
| **适用模式** | SK-39 齿轮、SK-21 弦线图（备选） |
| **典型节拍** | C-13 近邻关联 + C-09 叠加融合 |

### 7.4 天平—权衡决策

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 没有完美选择——只有哪个砝码更重 |
| **签名视觉元素** | 倾斜的天平，左右托盘放置对比项，倾斜角度表达权重差异 |
| **不可简化理由** | 并列柱状图是平行的，无法表达"取舍"。天平的倾斜天然传递"偏向哪一边" |
| **克制声明** | 左右各 ≤3 个砝码，超过合并；倾斜角度不超过 15°（过大显得夸张）；不标注精确重量 |
| **适用模式** | SK-42 天平、SK-41 杠杆（备选） |
| **典型节拍** | C-05 对比揭示 + C-04 色彩弹出 |

### 7.5 拼图—互补完整

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 每一块都有位置，缺一块就不完整 |
| **签名视觉元素** | 拼图碎片咬合，每块不同颜色，缺失位置用虚线轮廓标注"空缺" |
| **不可简化理由** | 饼图表达占比，不表达"缺一不可"的互补关系。拼图的咬合形态天然传递"需要全部" |
| **克制声明** | 碎片数量 4-8 块，超过不可读；缺失块最多 2 块；不用阴影效果（纯色填充） |
| **适用模式** | SK-40 拼图、SK-01 饼图（备选） |
| **典型节拍** | C-12 分组归类 + C-06 异常闪烁 |

### 7.6 循环轮—闭环飞轮

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 没有终点，只有循环——每一步都在推动下一步 |
| **签名视觉元素** | 圆环被等分为 3-6 个弧段，箭头沿顺时针方向，中心放核心概念 |
| **不可简化理由** | 时间线是线性的，无法表达"回到起点继续循环"。循环轮的闭合形态天然传递飞轮效应 |
| **克制声明** | 弧段 3-6 个，不更多；箭头统一朝一个方向；中心不放数据（放概念词） |
| **适用模式** | SK-34 循环轮、SK-60 循环系统 |
| **典型节拍** | C-09 叠加融合 + C-15 流动引导 |

### 7.7 杠杆—以小博大

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 找到那个支点——用最小的力气撬动最大的效果 |
| **签名视觉元素** | 三角形支点 + 倾斜的杠杆条，一端小重量（输入），另一端大重量（输出），力臂长度不等 |
| **不可简化理由** | 柱状图表达大小对比，但不表达"支点"和"力臂"。杠杆天然传递"投入产出比" |
| **克制声明** | 不标注具体力臂长度；杠杆条用直线，不用曲线；支点三角形不填充（空心） |
| **适用模式** | SK-41 杠杆、SK-35 楼梯（备选） |
| **典型节拍** | C-04 色彩弹出 + C-07 标注箭头 |

### 7.8 树—生长演进

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 从根基到枝繁叶茂——成长不是一步到位的 |
| **签名视觉元素** | 树根（基础能力）→ 树干（核心能力）→ 树枝（衍生能力）→ 树冠（成果），自下而上 |
| **不可简化理由** | 时间线只表达时间维度，不表达"基础→衍生"的依赖关系。树的形态天然传递生长逻辑 |
| **克制声明** | 树枝数量 ≤8 根，超过变混乱；不用具体叶子（抽象树枝即可）；树根不写具体数字 |
| **适用模式** | SK-59 树、SK-27 抽象树 |
| **典型节拍** | C-19 层级折叠 + C-03 排序阶梯 |

## 九、逻辑关系图表（C01-C16）

### 9.1 MECE 拆解

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 把所有可能性摆出来——既不重叠也不遗漏，这就是严谨 |
| **签名视觉元素** | 树形分解或并列方块，每个分支/方块互不重叠，总和 = 100% |
| **不可简化理由** | 饼图只能做一层拆分。MECE 支持多层级拆解，每层都保持"互斥+穷尽" |
| **克制声明** | 每层分解 ≤5 个分支，超过合并为"其他"；不在同一张图超过 2 层分解 |
| **适用模式** | C01-MECE 拆解、SK-27 抽象树 |
| **典型节拍** | C-12 分组归类 + C-19 层级折叠 |

### 9.2 Issue Tree 根因分析

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 问题不是单因的——从症状到根因，每一层假设都要可验证 |
| **签名视觉元素** | 从顶到底的问题树，每个分支是一个假设，叶子节点标注验证状态（已验证/待验证/排除）|
| **不可简化理由** | 鱼骨图只列原因，不表达"假设→验证"的逻辑。Issue Tree 把分析过程透明化 |
| **克制声明** | 深度 ≤3 层；不标注具体数据（标注假设和验证状态）；排除的原因用灰色划掉 |
| **适用模式** | C16-Issue Tree、SK-29 决策树 |
| **典型节拍** | C-19 层级折叠 + C-06 异常闪烁 |

### 9.3 差距到目标（行动版）

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 缺口不是数字——是需要填补的鸿沟，每一段都有明确的行动 |
| **签名视觉元素** | 当前值和目标值之间用虚线连接，中间标注缺口值和对应的弥补举措 |
| **不可细化理由** | 子弹图只表达差距，不表达"怎么补"。差距到目标行动版把"差多少 + 怎么做"放在一起 |
| **克制声明** | 举措 ≤3 条，超过聚合；不标注具体金额（标注举措名称+预期效果） |
| **适用模式** | C09-差距到目标、C14-子弹图 |
| **典型节拍** | C-09 叠加融合 + C-07 标注箭头 |

### 9.4 桑基流—资源去向

| 字段 | 值 |
|------|------|
| **一句话灵魂** | 钱/时间/注意力去了哪里——源头和去向之间的路径揭示优先级 |
| **签名视觉元素** | 从左到右的桑基流，源头在左，去向在右，线宽∝流量，颜色区分去向类别 |
| **不可简化理由** | 饼图只表达占比，不表达"从哪来的钱去了哪"。桑基图天然追踪资源的完整路径 |
| **克制声明** | 节点 ≤12 个，超过聚合中间层；线宽最大不超过画布宽的 30%；不用渐变色填充连线 |
| **适用模式** | C15-桑基流、SK-20 桑基图 |
| **典型节拍** | C-15 流动引导 + C-12 分组归类 |
