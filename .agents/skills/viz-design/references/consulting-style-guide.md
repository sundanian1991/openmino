# 多风格指南

> 根据内容调性和场景自动选择视觉风格。
> 不限于商业——正式汇报、个人规划、学习笔记、生活记录均可覆盖。

---

## 风格自动判定规则

| 判定条件 | 风格 | 触发关键词 |
|---------|------|-----------|
| 正式分析/评估/汇报/结构化数据 | **正式**（默认，McKinsey 气质） | 分析、评估、排名、对比、业绩、健康度、汇报 |
| 战略建议/增长/创新/方向 | **前瞻**（BCG 气质） | 增长、策略、建议、方向、创新、未来、规划 |
| 观点/争议/风险/问题/立场 | **鲜明**（Economist 气质） | 观点、争议、风险、问题、立场、质疑、反驳 |
| 个人生活/兴趣/笔记/学习 | **轻松**（Warm 默认气质） | 笔记、学习、读书、旅行、生活、兴趣、计划、总结 |
| 手绘/草图/涂鸦感 | **手绘** | 手绘、草图、涂鸦、随性、手绘风 |
| 用户显式指定 | **指定风格** | 直接说风格名即可 |
| 默认 | **正式** | — |

**风格命名对照**：正式 = McKinsey，前瞻 = BCG，鲜明 = Economist，轻松 = viz-design Warm 默认，手绘 = fe-hand-drawn 气质。

## 正式风格（McKinsey）

**气质**：冷静、克制、数据驱动。不抢戏，让数据说话。

### 颜色 Token

| Token | 值 | 用途 |
|-------|-----|------|
| `MCK_DARK` | `#051C2C` | 标题、正文文字 |
| `MCK_CYAN` | `#2CBDEF` | 主数据色（高亮/正向） |
| `MCK_GRAY` | `#D4D4D4` | 灰化背景、分隔线 |
| `MCK_LIGHT` | `#F5F5F5` | 图表背景 |
| `MCK_META` | `#8C8C8C` | 脚注、Exhibit 标签 |
| `MCK_NEG` | `#C13531` | 负向信号（下跌/落后） |
| `MCK_POS` | `#2CBDEF` | 正向信号（增长/领先） |

### 排版

| 元素 | 字号 | 字重 | 颜色 |
|------|------|------|------|
| Exhibit 标签 | 10px | 400 | MCK_META |
| 分隔线 | — | 1px solid | MCK_GRAY |
| 标题 | 15-17pt | 700 | MCK_DARK |
| 副标题 | 11.5pt | 400 | MCK_DARK |
| 正文/轴标签 | 10pt | 400 | MCK_DARK |
| 脚注/来源 | 9pt | 400 | MCK_DARK |

### 布局（Exhibit 框架）

```
┌────────────────────────────────────────┐
│ Exhibit X                    [公司Logo]│
│ ────────────────────────────────────── │
│ 标题（结论性，15-17pt）                 │
│ 副标题（时间+对象，11.5pt）             │
│                                        │
│            图表区                       │
│            （占画布 60-70%）            │
│                                        │
│ 脚注/来源（9pt，底部左对齐）             │
└────────────────────────────────────────┘
```

- LEFT_X = 0.08（左边距，相对坐标）
- EXHIBIT_Y = 0.96，SEP_LINE_Y = 0.935
- TITLE_Y = 0.90，SUBTITLE_Y = 0.78
- CHART_TOP = 0.72，CHART_BOTTOM = 0.08
- SOURCE_Y = 0.04

### 标题风格

**冷静判断型**：客观陈述数据结论，不带情绪。

> "供应商 A 综合得分 92，领先第二名 15 分"
> "Q1 业绩同比+12%，主要受新客户贡献驱动"

### 标注风格

- 标注简短，8-12 个中文字符
- 只解释"为什么"，不重复数字
- 用箭头和短线连接，不用大段文字

### 适用场景

供应商分析、业绩评估、排名对比、结构化商业数据、正式汇报材料。

## BCG 风格

**气质**：前瞻、积极、增长导向。强调机会和方向。

### 颜色 Token

| Token | 值 | 用途 |
|-------|-----|------|
| `BCG_GREEN` | `#2ca02c` | 主数据色（增长/机会） |
| `BCG_DARK` | `#2d2926` | 标题、正文文字 |
| `BCG_CREAM` | `#f5f0e8` | 背景色 |
| `BCG_GOLD` | `#d4a843` | 机会/亮点色 |
| `BCG_GRAY` | `#999999` | 灰化背景 |
| `BCG_NEG` | `#c13531` | 风险/警告 |

### 排版

| 元素 | 字号 | 字重 | 颜色 |
|------|------|------|------|
| 标题 | 15-17pt | 700 | BCG_DARK |
| 副标题 | 11.5pt | 400 | BCG_DARK |
| 正文/轴标签 | 10pt | 400 | BCG_DARK |
| 脚注/来源 | 9pt | 400 | BCG_GRAY |

### 布局

- 比 McKinsey 更宽松，留白更多
- 可加浅色背景区块区分信息组
- 不强制 Exhibit 框架，更灵活

### 标题风格

**前瞻导向型**：强调机会、方向、下一步。

> "新客户贡献拉动增长 8pp，下一阶段的杠杆在留存"
> "三家供应商具备规模化潜力，建议加大资源倾斜"

### 标注风格

- 标注指向"机会"而非"问题"
- 用上升箭头、目标标记
- 可加"Next"标签标注下一步行动

### 适用场景

增长策略、创新方案、方向建议、未来规划、机会识别。

## Economist 风格

**气质**：有态度、鲜明、略带争议性。不怕站队。

### 颜色 Token

| Token | 值 | 用途 |
|-------|-----|------|
| `ECON_RED` | `#E3120B` | 主强调色（争议/问题） |
| `ECON_BLUE` | `#006BA2` | 数据主色 |
| `ECON_ORANGE` | `#ed713a` | 辅助强调 |
| `ECON_DARK` | `#1a1a1a` | 标题、正文文字 |
| `ECON_LIGHT` | `#ffffff` | 背景 |
| `ECON_GRAY` | `#cccccc` | 灰化背景 |

### 排版

| 元素 | 字号 | 字重 | 颜色 |
|------|------|------|------|
| 标题 | 15-17pt | 700 | ECON_DARK |
| 副标题 | 11.5pt | 400 | ECON_DARK |
| 正文/轴标签 | 10pt | 400 | ECON_DARK |
| 脚注/来源 | 9pt | 400 | ECON_GRAY |

### 布局

- 紧凑布局，信息密度较高
- 无 Exhibit 框架，更像杂志内页
- 标题可以跨两行

### 标题风格

**态度鲜明型**：直接表达立场，不怕争议。

> "供应商排名洗牌：旧格局已破，新秩序未立"
> "看似增长，实则分化——头部吃肉尾部挨饿"

### 标注风格

- 标注可以更口语化
- 允许带态度的评论（"这解释了为什么..."）
- 可以用问句引导思考

### 适用场景

立场表达、争议分析、风险预警、问题揭示、评论性内容。

## 轻松风格

**气质**：温暖、亲切、不端着。像朋友聊天时画给你看的东西。

### 颜色 Token

| Token | 值 | 用途 |
|-------|-----|------|
| `WARM` | `#c98a6a` | 主数据色 |
| `STONE` | `#857d74` | 灰化/辅助 |
| `TEAL` | `#4a9e8e` | 正向信号 |
| `CORAL` | `#e07a5f` | 负向/强调 |
| `PAPER` | `#faf9f7` | 背景 |
| `INK` | `#2d2926` | 文字 |

### 排版

| 元素 | 字号 | 字重 |
|------|------|------|
| 标题 | 14-16pt | 600 |
| 正文/轴标签 | 10pt | 400 |
| 脚注 | 9pt | 400 |

### 布局

- 比正式风格更宽松，留白更多
- 无 Exhibit 框架
- 可以有装饰性小元素（圆点、短线）

### 标题风格

**叙述型**：像在讲一个故事。

> "这一年读了 24 本书，历史类最多但最喜欢科幻"
> "周末自驾去了 6 个地方，最远到了莫干山"

### 标注风格

- 轻松、口语化
- 可以用 emoji（不禁止，但不滥用）
- 标注可以是感想而非分析

### 适用场景

个人总结、读书笔记、旅行记录、兴趣追踪、学习笔记、生活规划。

## 手绘风格

**气质**：随性、有温度、不像机器生成的。

### 颜色 Token

继承 fe-hand-drawn 四色系统：

| Token | 值 | 用途 |
|-------|-----|------|
| `TERRA` | `#D6654B` | 强调色 |
| `INK` | `#1A1612` | 所有描边 |
| `RICE` | `#FEFFFE` | 填充/高亮 |
| `DEEP` | `#B03A21` | 阴影/深度 |

### 排版

- 用系统默认手写体或 sans-serif
- 字号不严格分级，自然即可

### 线条

- 允许轻微抖动
- stroke-linecap="round", stroke-linejoin="round"
- 不要求完美对称

### 标题风格

**草稿型**：像是边想边画。

> "大概是这样子的…"
> "我想的逻辑是这个"

### 标注风格

- 手写标注感
- 可以用箭头、圈注、旁注
- 不拘泥于精确性

### 适用场景

灵感草图、概念解释、白板讨论、思维速写。

---

## 风格切换规则

### 同一文档多图表

- **全部统一风格**，不混用
- 以第一张图判定风格，后续沿用
- 如果用户中途切换，确认后切换

### 风格覆盖优先级

```
用户显式指定 > 内容自动判定 > 默认（正式）
```

### 风格与图表类型无关

风格只影响颜色、排版、标题语气、标注语气。不影响图表类型选择。

## 与 viz-design 默认色系的兼容

当咨询模式未触发风格自动切换时，回退到 viz-design 默认色系：

| viz-design 默认 | 对应 McKinsey Token |
|-----------------|-------------------|
| Warm `#c98a6a` | MCK_CYAN（功能替代） |
| Stone `#857d74` | MCK_GRAY |
| Coral `#e07a5f` | MCK_NEG |
| Teal `#4a9e8e` | MCK_POS |

咨询模式下，使用风格专属 Token 覆盖默认色系。
