# 多图表叙事弧线

> 32 种多图表叙事结构，定义每张图的角色和节奏

---

## 弧线列表

### 一、发现与诊断

### 01. 调查式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 抛出异常（一个反直觉的数据点或异常信号） → 图2: 追踪线索（拆解异常的维度或时间线） → 图3: 揭示真相（锁定根因或关键变量） → 结论: 行动建议 |
| 节奏 | 图1: 高冲击 → 图2: 深入 → 图3: 高密度 → 结论: 呼吸 |
| 适用场景 | 数据异常排查、问题根因分析、审计报告、安全事故复盘 |
| 代表案例 | "某供应商流失率突然飙升 → 按产线拆解发现仅金条线异常 → 定位到某班组排班变更导致离职潮 → 建议恢复原排班并增设过渡期" |
| 弧线长度 | 3-5 张图 |

调查式弧线的核心设计原则是"好奇心驱动"。第一张图必须制造认知冲突——数据与预期不符，迫使观众想追问"为什么"。第二张图承担拆解任务，将大问题切成可操作的维度，每一层都离真相更近。第三张图是"破案时刻"，所有线索汇拢到一个清晰的结论。这种弧线适合汇报场景中需要"说服上级关注某个问题"的时刻，关键在于第一张图的冲击力是否足够让听众放下手机。

### 二、对比与评估

### 02. 对比式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 建立基准（现状或对照组的全貌） → 图2: 呈现差异（实验组/对比组的差异点） → 图3: 量化影响（差异带来的实际影响） → 结论: 选择建议 |
| 节奏 | 图1: 呼吸 → 图2: 高冲击 → 图3: 高密度 → 结论: 深入 |
| 适用场景 | A/B测试汇报、方案选型、竞品分析、前后效果对比、供应商对标 |
| 代表案例 | "6家头部供应商当前人效基准 → 引入新培训体系后 vs 未引入的差异 → 差异折算为月度产能增量 → 建议全面推广" |
| 弧线长度 | 2-4 张图 |

对比式弧线的关键在于"公平感"。第一张图必须让双方站在同一起跑线——同样的指标、同样的时间段、同样的口径。如果基准不公平，后续所有对比都站不住。第二张图是最有戏剧性的一张，差异的呈现方式决定了说服力：绝对值差异适合业务决策，百分比差异适合战略判断。第三张图将差异翻译为"跟你有什么关系"——把抽象的差距变成具体的成本、收益或风险。对比式弧线不适合"微妙差异"的场景，差异不明显时用对比式反而暴露弱点。

### 三、时间与变化

### 03. 演进式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 起点（初始状态或历史基线） → 图2: 轨迹（变化趋势与关键转折点） → 图3: 终点或预测（当前状态或未来走势） → 结论: 阶段判断 |
| 节奏 | 图1: 呼吸 → 图2: 深入 → 图3: 高冲击 → 结论: 呼吸 |
| 适用场景 | 季度/年度复盘、成长曲线、市场趋势分析、项目里程碑回顾 |
| 代表案例 | "供应商准入数量12个月基线 → 每月净增减趋势及政策影响标注 → 预测Q3将达到产能瓶颈 → 建议提前启动新供应商储备" |
| 弧线长度 | 3-6 张图 |

演进式弧线的灵魂是"转折点"。一条平滑上升的曲线没有故事性，真正有力的是标注出"政策变化""人员调整""外部冲击"等关键节点。第二张图上每一个标注都是一次叙事机会，让观众理解"不是数据在变，是事件在驱动变化"。演进式弧线要警惕"后视偏差"——用历史数据讲故事很容易把随机波动解读为因果律。标注转折点时必须诚实区分"已确认的因果关系"和"时间巧合"。这种弧线在复盘会上效果最好，因为观众有亲历感，每个转折点都能引发讨论。

### 04. 诊断式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 症状呈现（表面问题的全景） → 图2: 逐层拆解（从表象到结构的原因分析） → 图3: 处方（针对根因的解决方案及预期效果） → 结论: 优先级排序 |
| 节奏 | 图1: 高冲击 → 图2: 高密度 → 图3: 呼吸 → 结论: 深入 |
| 适用场景 | 绩效诊断、流程瓶颈分析、供应商问题排查、质量事故分析 |
| 代表案例 | "3月供应商质检通过率骤降至82% → 按环节拆解发现培训环节缺失+排班过密双重因素 → 针对性方案及模拟效果 → 优先解决排班问题（影响占68%）" |
| 弧线长度 | 3-5 张图 |

诊断式弧线与调查式的区别在于"已知问题 vs 未知问题"。调查式是从异常出发找原因，诊断式是从已知症状出发做结构化分析。第一张图的任务是让所有人都对"问题有多严重"达成共识——用绝对值、同比、排名等让症状不可忽视。第二张图是技术含量最高的一张，需要建立清晰的归因框架（人/机/料/法/环，或按流程环节），让观众能跟着逻辑走而不是被数据淹没。第三张图的核心是"可操作性"，方案必须具体到"谁在什么时间做什么"，否则诊断就停留在分析层面。

### 四、决策与行动

### 05. 提案式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 机会识别（未被满足的需求或市场空白） → 图2: 方案展示（提案的核心逻辑和关键设计） → 图3: 收益预估（量化投入产出比） → 结论: 决策请求 |
| 节奏 | 图1: 深入 → 图2: 高密度 → 图3: 高冲击 → 结论: 呼吸 |
| 适用场景 | 新项目立项、预算申请、战略提案、供应商引入建议 |
| 代表案例 | "金条产线Q4产能缺口达15% → 引入新供应商A的三阶段方案 → 预估ROI及风险对冲 → 申请启动准入流程" |
| 弧线长度 | 3-4 张图 |

提案式弧线的核心矛盾是"你的热情 vs 决策者的风险厌恶"。第一张图必须证明"不做这件事的代价"，而不是"做这件事的好处"——缺口和痛点比机会更有说服力。第二张图需要平衡"足够详细以建立信心"和"不过于复杂以免引发更多质疑"。第三张图是关键的"数值锚定"时刻，用一个清晰的数字（ROI、回本周期、成本节约）让决策变成算术题。提案式弧线最忌讳的是第三张图用模糊的"预计将大幅提升"收尾——没有具体数字的提案不是提案，是愿望。

### 五、筛选与层级

### 06. 漏斗式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 全景入口（大范围、高视角的全量数据） → 图2: 层层筛选（按条件逐级过滤） → 图3: 精准聚焦（最终目标群体或关键发现） → 结论: 聚焦行动 |
| 节奏 | 图1: 呼吸 → 图2: 高密度 → 图3: 高冲击 → 结论: 深入 |
| 适用场景 | 用户分群、供应商分层、市场细分、资源分配、风险排查 |
| 代表案例 | "30+供应商全量画像 → 按产能/合规/人效三维筛选 → 锁定5家高风险供应商 → 逐家制定处置方案" |
| 弧线长度 | 3-5 张图 |

漏斗式弧线的视觉特性很强——每一张图的范围都比上一张小，数据密度却越来越高。这种"广→窄"的结构天然适合资源有限、需要聚焦的场景。设计时要注意每一层筛选的"理由"必须自洽：如果第一层按产能筛、第二层按合规筛、第三层按人效筛，三者的逻辑关系需要交代清楚（是并列维度还是递进维度）。漏斗式弧线的常见陷阱是"筛到最后只剩1-2个目标"——太少的结论让人觉得筛选条件是倒推的。保留3-5个目标最合适，既体现聚焦又保留选择空间。

### 六、结构与认知

### 07. 金字塔式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 核心结论（先亮观点，一句话结论的可视化） → 图2: 支撑论据（3-5个关键证据的并列展示） → 图3: 底层数据（原始数据或方法论） → 结论: 重申立场 |
| 节奏 | 图1: 高冲击 → 图2: 高密度 → 图3: 深入 → 结论: 呼吸 |
| 适用场景 | 高管汇报、电梯演讲配套材料、战略决策会议、董事会报告 |
| 代表案例 | "结论：应优先投资供应商培训体系 → 三大论据：人效提升27%、流失率下降15%、ROI达3.2倍 → 原始数据附录及计算方法 → 确认投资决策" |
| 弧线长度 | 3-4 张图 |

金字塔式弧线与所有其他弧线的区别是"先说结论"。这种结构来自麦肯锡的SCQA框架，核心理念是决策者的时间有限，必须在前30秒就让他知道"你到底要说什么"。第一张图是最难设计的一张——要把复杂分析浓缩为一个有力的视觉锚点（一个大数字、一个明确的趋势箭头、一个红绿灯式的判断）。第二张图是"可信度建设"，每个论据必须独立成立且相互支撑。第三张图是"备查层"，大部分人不会看，但看了就必须能还原你的分析过程。金字塔式弧线不适合学术讨论（学术需要先展示推理再得出结论），但非常适合商业决策场景。

---

### 08. 螺旋式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 初始认知（第一轮观察） → 图2: 深入后反转（新的视角推翻或修正初始认知） → 图3: 升维整合（将矛盾统一到更高框架） → 结论: 认知升级 |
| 节奏 | 图1: 呼吸 → 图2: 高冲击 → 图3: 深入 → 结论: 高密度 |
| 适用场景 | 战略复盘、认知升级型汇报、研究总结、复杂问题的多轮分析、年度反思 |
| 代表案例 | "初步观察：供应商数量与产能正相关 → 深入后发现：超过20家后管理成本吞噬规模效应 → 升维框架：最优供应商数量存在甜蜜点（12-18家） → 重新定义供应商策略" |
| 弧线长度 | 3-5 张图 |

螺旋式弧线是8种中最有"智力含量"的结构，也是风险最高的一种。它的力量来自"推翻自己"——第二张图否定第一张图的结论，制造认知张力，然后用第三张图在更高维度统一矛盾。这种弧线不适合时间紧张的汇报（观众需要时间消化反转），但非常适合深度讨论、战略研讨或内部分享。设计螺旋式弧线的核心技巧是"反转的诚实性"：第二张图的反转必须来自新的数据或新视角，而不是为了制造戏剧性而故意设一个弱的第一轮结论。如果第一轮结论本身就站不住，反转就失去了力量。

### 七、全景概览（09-14）

### 09. 全景式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: KPI 总览（核心指标一览，大字+红绿灯式判断） → 图2: 关键发现（3-5 个最重要发现的卡片式排列） → 图3: 详情下钻（某个关键发现的详细数据） → 结论: 下一步行动 |
| 节奏 | 图1: 高冲击 → 图2: 呼吸 → 图3: 深入 → 结论: 高密度 |
| 适用场景 | 月度/季度经营汇报、Dashboard 叙事化、高管快报、供应商全景概览 |
| 代表案例 | "8项核心 KPI 一览（4 达标/4 未达标） → 3 个关键发现卡片 → 下钻到未达标最严重的指标 → 下月改进计划" |
| 弧线长度 | 3-5 张图 |

全景式弧线是"给没时间的人看的"。第一张图必须让高管在 10 秒内判断"大局好不好"。红绿灯、大字、箭头是核心元素——不要让他做算术。第二张图是"为什么"的速读版，每张发现卡片不超过两行字。第三张图是给有兴趣的人准备的深读，选择最值得关注的点下钻。全景式弧线最常见的错误是第一张图太拥挤——超过 8 个 KPI 就失去了"全景"的意义，变成了"全乱"。

### 10. 拆解式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 总量（一个关键总数） → 图2: 结构分解（按维度拆解，展示构成） → 图3: 异常归因（找出偏离预期的部分并解释原因） → 结论: 关注重点 |
| 节奏 | 图1: 锚定 → 图2: 高密度 → 图3: 高冲击 → 结论: 深入 |
| 适用场景 | 收入构成分析、成本拆解、团队产出分解、预算构成、时间分配 |
| 代表案例 | "月度总产能 1200 单 → 按供应商/产线/时段三维拆解 → 发现金条产线周三产能异常偏低（排班缺人导致） → 建议固定周三替补机制" |
| 弧线长度 | 3-5 张图 |

拆解式的核心是"从粗到细的递进"。每一张图都是上一张的放大镜，而不是一张全新的图。拆解的维度选择决定了洞察力——按"时间"拆能看到周期性，按"类别"拆能看到结构性，按"人"拆能看到个体差异。拆解式弧线要防止"无限下钻"——到第 3 张图必须停，否则观众会迷失在细节中。

---

### 11. 排名式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 全量排行（排序展示，谁行谁不行一目了然） → 图2: 领先者解码（Top 做对了什么） → 图3: 落后者诊断（Bottom 的问题在哪） → 结论: 推广/改进建议 |
| 节奏 | 图1: 呼吸 → 图2: 深入 → 图3: 高冲击 → 结论: 高密度 |
| 适用场景 | 业绩排行、供应商对标、团队排名、地区对比 |
| 代表案例 | "8 个职场达成率排行（汇讯 132% vs 九通 68%） → 汇讯做对了什么（客户储备+排班优化） → 九通问题诊断（新员工占比过高+培训缺失） → Top 经验推广" |
| 弧线长度 | 2-4 张图 |

排名式弧线最容易被滥用——单纯列出排名不是分析，是通报。真正的洞察来自回答"为什么这个人在第一，那个人在最后"。第二张图和第三张图必须给出超越排名的解释。排名式弧线不适合"差距很小"的场景——如果第一名和最后一名只差 3%，排名本身就没有信息量。

### 12. 预警式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 正常范围（历史正常区间或行业基准） → 图2: 异常信号（当前数据偏离正常范围的标记） → 图3: 风险预测（如果趋势持续，未来会怎样） → 结论: 干预建议 |
| 节奏 | 图1: 呼吸 → 图2: 异常闪烁 → 图3: 高冲击 → 结论: 深入 |
| 适用场景 | 风控预警、SLA 监控、合规风险、供应商清退预警、健康度预警 |
| 代表案例 | "历史合规率正常区间 95-98% → 3 月降至 89%，连续 2 月低于下限 → 按此趋势 Q2 将触发监管红线 → 建议立即启动合规整改" |
| 弧线长度 | 3-5 张图 |

预警式弧线的力量来自"对比"。第一张图建立的"正常感"越强烈，第二张图的偏离就越刺眼。红色/橙色/黄色三色预警体系是经典选择。第三张图的"如果……那么……"是说服决策者行动的关键——不是"现在有多糟"，而是"不做会多糟"。预警式弧线要避免"狼来了"效应——频繁发预警会导致麻木，只对真正有风险的事情发预警。

---

### 13. 里程碑式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 已完成（已达成的里程碑和成果） → 图2: 进行中（当前状态和进度条） → 图3: 待达成（剩余目标、时间线和风险） → 结论: 资源请求或调整建议 |
| 节奏 | 图1: 呼吸 → 图2: 深入 → 图3: 高冲击 → 结论: 高密度 |
| 适用场景 | 项目进度汇报、OKR 中期检查、年度目标追踪、供应商准入进度 |
| 代表案例 | "Q1 完成 3/5 里程碑（新准入 2 家+清退 1 家+合规审计 2 家） → 进行中：培训体系搭建（65%） → 待达成：Q2 目标及当前风险（时间不足） → 请求增加资源" |
| 弧线长度 | 3-5 张图 |

里程碑式弧线满足人类对"进度条"的天然执念。第一张图让观众有成就感，第二张图让他们知道现在在哪，第三张图制造紧迫感。关键是第三张图必须有"如果不变会怎样"的预测——否则就是单纯的进度汇报，没有推动力。

### 14. 分配式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 资源从哪来（总预算/总人力/总时间的来源构成） → 图2: 到哪去（分配到各方向的比例和金额） → 图3: 缺口在哪（需求 vs 供给的差距） → 结论: 重新分配建议 |
| 节奏 | 图1: 呼吸 → 图2: 高密度 → 图3: 高冲击 → 结论: 深入 |
| 适用场景 | 预算分配、人力调配、时间管理、资源规划 |
| 代表案例 | "Q2 总预算 500 万来源构成 → 分配到 5 个产线的比例及金额 → 金条产线缺口最大（需 200 万，仅分配 120 万） → 建议从企金调拨 50 万" |
| 弧线长度 | 3-4 张图 |

分配式弧线回答"钱/人/时间够不够，不够怎么办"。桑基图是首选图表，因为它同时展示来源和去向。第三张图的缺口分析必须精确到"差多少"和"从哪补"，否则分配建议就没有可操作性。

### 八、能力与流程（15-18）

### 15. 能力画像式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 现状画像（多维度雷达或评分卡，综合评估当前状态） → 图2: 差距分析（vs 目标/标杆的差距热力图） → 图3: 提升路径（优先级排序的改进项及预期效果） → 结论: 行动优先级 |
| 节奏 | 图1: 深入 → 图2: 高冲击 → 图3: 呼吸 → 结论: 高密度 |

| 弧线长度 | 3-5 张图 |

### 16. 流程式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 输入（流程起点、资源投入、触发条件） → 图2: 处理（各步骤效率、瓶颈、返工率） → 图3: 输出+效率（产出量、质量、周期时间 vs 目标） → 结论: 优化建议 |
| 节奏 | 图1: 呼吸 → 图2: 高密度 → 图3: 深入 → 结论: 高冲击 |
| 适用场景 | 业务流程分析、审批链路优化、供应链效率、客户处理流程 |
| 代表案例 | "月度进件量 3000 单 → 5 步审批流程中每步耗时+返工率（第 3 步最慢） → 产出达标率 87%，周期比目标长 2 天 → 建议第 3 步自动化" |
| 弧线长度 | 3-5 张图 |

流程式弧线追踪"从 A 到 B 的过程中发生了什么"。漏斗图或桑基图适合展示转化和流失。每个步骤的效率数据（耗时、返工率、通过率）是第二张图的核心。流程式弧线最容易发现的优化点是"最慢的那一步"——但它可能不是"影响最大的那一步"，需要结合流量判断。

### 17. 关联式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 单指标趋势（核心指标的走势） → 图2: 双变量散点（X vs Y 的相关性检验） → 图3: 多因素回归（控制其他变量后，哪个因素最关键） → 结论: 关键驱动因素 |
| 节奏 | 图1: 呼吸 → 图2: 深入 → 图3: 高密度 → 结论: 高冲击 |
| 适用场景 | 根因探索、影响因素分析、相关性研究、假设验证 |
| 代表案例 | "客户满意度连续 3 月下滑 → 满意度 vs 响应时间散点（负相关） → 控制产品/价格/服务后，响应时间是唯一显著因素 → 建议缩短响应 SLA" |
| 弧线长度 | 3-4 张图 |

关联式弧线从"看到现象"走到"找到原因"。散点图和相关系数是核心工具。第三张图如果能做简单的多元分析最好——很多看似相关的因素在控制其他变量后就消失了。关联式弧线要诚实标注"相关性≠因果性"，除非有实验数据支撑。

### 18. 情景规划式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 乐观情景（最佳情况的路径和结果） → 图2: 基准情景（最可能的情况） → 图3: 悲观情景（最差情况+触发条件） → 结论: 应对策略（哪种情景下做什么） |
| 节奏 | 图1: 呼吸 → 图2: 深入 → 图3: 高冲击 → 结论: 高密度 |
| 适用场景 | 战略规划、风险预案、预算编制、产能规划 |
| 代表案例 | "乐观：Q3 产能增长 20%（新供应商到位+培训完成） → 基准：增长 10%（新供应商延迟 1 月） → 悲观：下降 5%（核心供应商流失+无替补） → 各情景触发条件及应对方案" |
| 弧线长度 | 3-4 张图 |

情景规划式弧线不是"预测未来"，而是"准备多种未来"。三种情景的设计必须基于不同的关键假设，而不是同样的假设微调数字。悲观情景特别重要——它不是为了制造恐慌，而是为了暴露"最坏情况下我们有没有准备"。

### 九、战略与生态（19-27）

### 19. 基准对标式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 我们（当前表现的自我画像） → 图2: 行业（行业平均水平或中位数） → 图3: 最佳实践（行业 Top 10% 的标准） → 结论: 差距最大的维度优先改进 |
| 节奏 | 图1: 深入 → 图2: 对比揭示 → 图3: 高冲击 → 结论: 高密度 |
| 适用场景 | 供应商对标、竞争力分析、成熟度评估、行业排名 |
| 代表案例 | "我们的人效/合规/满意度评分 → 行业 50 家供应商中位数 → Top 10% 供应商的标准 → 合规差距最大（-18 分），优先改进" |
| 弧线长度 | 3-5 张图 |

基准对标式弧线回答"我们在什么位置"。三层的对比（自己/行业/最佳）让差距有方向感——"低于行业"和"低于最佳"是不同的信号。对标数据的来源必须可靠，否则整个弧线的基础就塌了。

### 20. 故事线式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 人物/事件（一个具体的人或案例，建立情感连接） → 图2: 数据（这个人的故事代表的群体性数据） → 图3: 意义（这个数据告诉我们什么系统性问题） → 结论: 行动呼吁 |
| 节奏 | 图1: 叙事锚点 → 图2: 高密度 → 图3: 深入 → 结论: 高冲击 |
| 适用场景 | 客户故事汇报、员工体验改善、社会责任报告、用户调研呈现 |
| 代表案例 | "小李的供应商服务体验故事（入职 3 月遇到 5 个问题） → 类似小李的员工占 34%，平均问题数 4.2 → 系统性问题：新员工前 90 天是体验最低谷 → 建议启动新员工关怀计划" |
| 弧线长度 | 3-5 张图 |

故事线弧线是唯一以"人"开头的弧线。第一张图不数据，是故事——一个人的经历。第二张图把故事升级为数据，证明"这不是个案"。第三张图把数据升级为系统性洞察。故事线弧线在高管汇报中效果极好，因为"小李"比"34%"更容易被记住。但故事必须真实，否则被发现造假后信任崩塌最严重。

### 21. OKR追踪式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 目标回顾（O 是什么，为什么定这个目标） → 图2: 关键结果进度（每个 KR 的进度条/子弹图） → 图3: 风险项（可能不达标的 KR 及原因） → 结论: 调整或加码建议 |
| 节奏 | 图1: 呼吸 → 图2: 深入 → 图3: 高冲击 → 结论: 高密度 |
| 适用场景 | OKR 中期检查、季度复盘、目标追踪、KPI 进度汇报 |
| 代表案例 | "Q2 O：供应商质量提升 20% → 4 个 KR 进度（3 个达标、1 个滞后） → 滞后 KR：合规审计覆盖率仅 62%（人手不足） → 建议临时调配或延期" |
| 弧线长度 | 3-4 张图 |

OKR追踪式弧线是"最结构化"的弧线——目标/关键结果/进度的框架已经预设好了。关键是第三张图不能只说"这个 KR 落后了"，要解释"为什么落后"和"有没有救"。OKR 弧线适合定期汇报，不适合突发问题。

### 22. 瀑布解释式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 起点到终点（A 变成了 B，总变化是多少） → 图2: 正向驱动（哪些因素推动了增长） → 图3: 负向驱动（哪些因素造成了拖累） → 结论: 下一步发力点 |
| 节奏 | 图1: 高冲击 → 图2: 深入 → 图3: 深入 → 结论: 高冲击 |
| 适用场景 | 同比/环比变化解释、业绩波动分析、成本增减拆解 |
| 代表案例 | "Q1 营收从 800 万→920 万（+120 万） → 正向驱动：新客户 +80 万、老客增购 +50 万 → 负向驱动：流失客户 -30 万、价格调整 +20 万 → 下一步：降低流失率（影响最大）" |
| 弧线长度 | 2-4 张图 |

瀑布解释式弧线回答"从 A 到 B 是怎么发生的"。瀑布图本身就能同时展示正负驱动，所以有时候 1 张图就够了。弧线版的价值在于把正向和负向分开讲，每张图更深入。适合"变化很大但原因不清楚"的场景。

### 23. 生态分析式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 参与者图谱（谁在这个生态里，角色分类） → 图2: 关系网（谁依赖谁、谁制约谁、资金/信息流向） → 图3: 权力结构（谁最关键、谁最脆弱、谁是瓶颈） → 结论: 生态优化建议 |

### 29. 价值流式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 价值来源（我们创造价值的环节和方式） → 图2: 价值去向（价值被客户/中间环节获取的比例） → 图3: 浪费识别（流程中不创造价值的环节和损耗） → 结论: 消除浪费的具体举措 |

| 代表案例 | "价值创造环节：技术(40%)+服务(35%)+品牌(25%) → 利润分配：我们获取 30%，渠道 40%，客户剩余 30% → 浪费识别：审批等待时间占总周期 35%，不创造价值 → 建议并行审批替代串行" |
| 弧线长度 | 3-5 张图 |

价值流式弧线源自精益管理的价值流图。它的独特视角是"不是所有活动都创造价值"——找出那些消耗资源但不产生价值的环节（等待、返工、过度处理），消除它们比增加新投入更有效。

### 30. 数据探索式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 初始假设（我们以为是这样的） → 图2: 验证过程（数据检验的结果，确认或推翻假设） → 图3: 意外发现（数据分析中意外浮现的洞察） → 结论: 新的认知方向 |
| 节奏 | 图1: 呼吸 → 图2: 对比揭示 → 图3: 高冲击 → 结论: 深入 |
| 适用场景 | 探索性数据分析、研究总结、假设驱动的分析项目 |
| 代表案例 | "假设：供应商规模越大效率越高 → 验证：相关性 r=0.23，不显著；控制行业变量后反而负相关 → 意外发现：中等规模(50-150人)供应商效率最高 → 建议优选中等规模供应商" |
| 弧线长度 | 3-5 张图 |

数据探索式弧线最诚实——它承认"我们不一定知道答案，让数据说话"。第三张图的"意外发现"是最有价值的部分，因为它提供了之前不知道的信息。但要注意区分"真正的意外发现"和"数据挖掘的假阳性"。

### 十一、归因与复盘（31-32）

### 31. 归因分解式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 变化声明（某指标从 A 变到 B） → 图2: 维度拆解（按时间/地区/产品/人等多个维度分别看变化） → 图3: 主导因素（哪个维度贡献了最大的变化量） → 结论: 针对主导因素的干预策略 |
| 节奏 | 图1: 高冲击 → 图2: 高密度 → 图3: 深入 → 结论: 高冲击 |
| 适用场景 | 业绩波动归因、用户增长归因、成本变化归因 |
| 代表案例 | "3 月 GMV 环比下降 15% → 按产线拆解（金条 -22%、企金 +5%、信用卡 -3%、财富 -8%） → 按职场拆解（汇讯 +10%、九通 -45% 为主导因素） → 建议聚焦九通根因分析" |
| 弧线长度 | 3-5 张图 |

归因分解式弧线回答"变化到底是谁/什么造成的"。多维度拆解是核心——按一个维度看可能是 A 的问题，按另一个维度看可能是 B 的问题。瀑布图和堆叠条是最常用的图表。归因分解式要避免"归因过度"——不是每个微小变化都需要一个解释，有些就是随机波动。

### 32. 周期复盘式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 目标回顾（周期初设定的目标和期望） → 图2: 实际对比（目标 vs 实际的差距，达标/未达标分类） → 图3: 经验教训（做对了什么、做错了什么、下周期调整） → 结论: 下周期目标建议 |
| 节奏 | 图1: 呼吸 → 图2: 对比揭示 → 图3: 深入 → 结论: 高密度 |
| 适用场景 | 季度/年度复盘、项目结项、供应商周期评估、个人目标回顾 |
| 代表案例 | "Q1 设定 5 个目标 → 实际达成：3 达标（人效/合规/满意度）2 未达标（新准入/培训覆盖率） → 做对了：排班优化；做错了：准入流程太慢 → Q2 建议：简化准入流程+保持排班策略" |
| 弧线长度 | 3-5 张图 |

周期复盘式弧线是最"完整"的复盘结构——从回顾开始，到对比核心，到经验提炼结束。第三张图的"做对了什么"和"做错了什么"必须对称出现，否则复盘就变成了单纯的自我表扬或自我批评。周期复盘式弧线适合固定周期的回顾，是组织学习的核心工具。
