> viz-design 的设计知识库。SKILL.md 只装流程指令，本文件装"怎么判断"。
> 循环 2 减法时按需读取，不需要每次全读。

---

## 一、六步出图法（Cole Nussbaumer Knaflic）

> 三循环工作流的方法论基础。

| 步骤 | 做什么 | 对应循环 |
|------|--------|---------|
| **1. 理解语境** | 识别受众 + 明确他们需要知道/做什么 + 决定用什么数据 | 循环1 intent.md |
| **2. 选择图表** | 根据要展示的关系（趋势/比较/构成/分布）选最佳图表类型 | 循环1 DNA 匹配 |
| **3. 消除干扰** | 去掉颜色差异→理解数据本质→有策略地加回颜色；删除边框/网格线/冗余标签 | 循环2 减法 |
| **4. 聚焦注意力** | 利用前注意力属性，对关键数据着色+标注，引导受众看到你想让他看到的 | 循环2 注意力引导 |
| **5. 像设计师思考** | 形式服从功能，对齐元素，提升美感，让图无障碍可读 | 循环3 验收 |
| **6. 讲故事** | 用文字说明、标题写结论、注释写原因，让图自己说话 | 全程文字策略 |

**关键操作："先去掉颜色再加回来"**
循环 1 初稿时，所有数据先用统一灰色绘制。循环 2 减法时，只对核心判断必须强调的数据赋予高亮色。不是"选什么颜色好看"，而是"什么颜色能引导注意力"。

## 二、视觉层级铁律

> **视觉突出程度必须与重要性严格匹配。** 最重要的元素 = 视觉上最突出。禁止多个元素同等突出相互抢夺注意力。

**提升关键元素可视性的手段**：
- 调整大小、类型、标签和高亮方式
- 使用参考线和分割区块组织信息
- 重要文本用更大字号，节制使用加粗/倾斜/下划线
- 为关键数据加标签、提升颜色强度
- 为重要事件添加注释或说明
- 对离群值进行标注，或将同一双 Y 轴的数据拆分为两张图

**建立层级时必须做出选择**：只保留有助于受众更清晰理解的内容，隐藏不利于结论的数据。

**设计要点检查清单**：
1. **删除非数据墨水** — 删除所有不影响表意的元素，最大化数据墨水比例
2. **确保每个元素都有标签** — 文字足够大，受众能清晰理解
3. **建立视觉层级** — 最重要元素最突出、比较对象明确标示、离群值加注释

## 三、减法迭代清单（循环 2 Part A 速查）

对每个元素问这 8 个问题，按判断标准直接决定：

| 元素 | 追问 | 判断标准 |
|------|------|---------|
| **标题** | 是结论还是描述？ | 结论就留，描述就改成结论（"B 组领先 F 组 62%"） |
| **图例** | 必须吗？能用颜色+文字替代吗？ | 单系列数据不需要图例 |
| **网格线** | 帮助读图还是干扰？ | 多数情况可以淡化或去掉 |
| **边框** | 有结构价值吗？ | 几乎总是可以去掉 |
| **标注** | 写原因还是写数字？ | 写原因+幅度（"增长 62%"），不写纯数字 |
| **参考线** | 帮助理解基准吗？ | 有明确基准（目标值/平均值/前高）时保留 |
| **色块/背景** | 帮助区分阶段吗？ | 有明确阶段划分时保留，否则去掉 |
| **5 秒结论检验** | 遮住标题和解读文字，只看图——5 秒内能说出核心结论吗？ | 不能 → 回到 S4b 重新选 |

**原则**：每去掉一个元素，图都应该更清晰。减法的终点是"每个留下的元素都在为核心判断服务"。

## 四、文字使用策略

> 文字在可视化中不是装饰，是理解桥梁。不要假设两个不同的人看到同一幅图会得出相同的结论。如果你希望受众得出一个结论，用文字明确说出来。

### 文字 7 种角色

| 角色 | 位置 | 字号/颜色 | 示例 |
|------|------|----------|------|
| **标题（行动）** | 图顶部 | 17px/600, #141413 | "请批准招聘2名全职员工" |
| **副标题（背景）** | 标题下方 | 11px/400, #A3A29B | "以填补去年离职人员空缺" |
| **坐标轴标题** | 轴旁 | 10-12px, #3D3D3A | "工单数" / "月份" |
| **注释/行动** | 图中关键位置 | 10px/600, 语义色 | "两名员工在5月离职" |
| **数据标签** | 数据点旁 | 10px, #87867F | 每月处理量的数值 |
| **解读文字** | 图下方（提要） | 13px, #5F5A54 | "8月开始落后于增长，且此后一直未能赶上" |
| **数据来源** | 底部 | 11px, #A3A29B | "数据来源：XX \| 时间范围" |

### 文字分层铁律

- 标题必须是结论性的、行动性的——不是描述性的（"招聘申请"❌ → "请批准招聘2名全职员工"✅）
- 副标题提供背景，用更小字号、更浅颜色，与标题形成视觉层级
- 注释写原因+幅度（"两名员工5月离职"），不写纯数字（"5月：-2"❌）
- 利用前注意力属性让关键文字突出：更大字号、更醒目的颜色、优先位置

### 标题结构

```
[大字号，深色] 请批准招聘2名全职员工  ← 行动呼吁，第一眼看到
[小字号，灰色] 以填补去年离职人员空缺  ← 背景，第二眼理解
[图表内容] 工单数随时间变化的趋势      ← 数据支撑判断
```

### 文字7角色验证表（循环 2 Part C）

| 文字角色 | 存在？ | 字号 | 颜色 |
|----------|--------|------|------|
| 标题 | 必须有 | 17px bold | slate |
| 副标题 | 可选 | 11px | g500 |
| 轴标题 | 按需 | 10px | g500 |
| 注释 | 按需 | 9-10px | g500 |
| 数据标签 | ≤10% | 11-12px bold | 按元素 |
| 解读文字 | 有结论时 | 13px | g700 |
| 数据来源 | 必须有 | 11px | g500 |

→ 缺失必要角色 → 补上；多余 → 删除

## 五、设计原则速查

| 原则 | 如何体现 |
|------|---------|
| 标题写结论 | 标题必须是结论性的（"B 组领先 F 组 62%"） |
| 颜色是信号不是装饰 | 颜色编码重要性，不是装饰 |
| 颜色 ≤5 种 | 控制视觉复杂度 |
| 色盲友好 | 禁用红绿对比，使用色盲安全色板 |
| 高亮 ≤10% | 限定高亮范围 |
| 色系 ≤2 ramp | 注明色系选择理由 |
| 一个图一个核心洞察 | 【想传达什么】只写一句话 |

## 六、关键元素突出检查（循环 2 Part B）

- 最重要的数据/结论是否视觉上最突出？
- 有没有次要元素和核心元素抢注意力？
- 高亮范围 ≤10%？
- 先灰后亮：默认灰色，核心数据用 Birchline clay(#D97757)
- 标注是否写"原因+幅度"而非纯数字？
