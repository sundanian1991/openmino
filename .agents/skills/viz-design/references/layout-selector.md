# Layout Selector — 版式自动选择器

> 版式不是事后添加，是 Phase 1.5 的自动决策。
> 根据场景类型 + 数据特征 + 解读需求，自动选择 40 种版式之一 + 解读策略。

---

## 决策流程

```
输入（数据 + 意图）
  ↓
Step 1: 判断信息密度需求
  ├── 只有核心数据 → 单图版式组（#01/#34/#40）
  ├── 核心数据 + 需要解读 → 图表+解读版式组（#08/#14/#03）
  ├── 多个关键指标 → 看板版式组（#28/#10/#09）
  ├── 多对象对比 → 对比版式组（#02/#26/#18）
  └── 流程/阶段 → 步骤版式组（#22/#23/#25）
  ↓
Step 2: 判断解读策略
  ├── 有明确结论要传达 → 解读卡片（要点 + 原因）
  ├── 有关键数字要突出 → KPI 卡片（大数字 + 趋势）
  ├── 有排名/分级 → 高亮 + 脚注
  └── 纯数据展示 → 无解读区
  ↓
Step 3: 查映射表 → 输出版式编号 + 解读策略
```

## 映射表

### 一、单图版式组（信息密度低，纯数据展示）

| 场景特征 | 版式 | 解读策略 | 典型图表 |
|---------|------|---------|---------|
| 单一核心数据，一眼看懂 | #01 单图居中 | 无解读区 | KPI 大数、单指标趋势 |
| 强调单一结论/震撼数据 | #34 大字报 | 无解读区，大字即结论 | 年度总规模、关键指标 |
| 强视觉冲击、汇报开场 | #40 全屏沉浸式 | 叠加标题即解读 | 全屏地图、震撼时刻 |

**触发条件**：数据 ≤2 个维度、无需辅助信息、标题即可传达核心信息。

### 二、图表+解读版式组（中等信息密度，需要引导理解）

| 场景特征 | 版式 | 解读策略 | 典型图表 |
|---------|------|---------|---------|
| 图表 + 文字解读 | #08 L 型 | 右侧 3 条要点（原因+影响），不重复图表数字 | 排名、对比、分析结论 |
| 主图 + 辅助卡片 | #14 主次左右 70/30 | 右侧 KPI 卡片 + 底部解读段 | 趋势图 + 关键指标摘要 |
| 图表 + 轻量说明 | #03 左右对比 60/40 | 右栏作为注释/图例/要点 | 主图 + 数据摘要 |

**解读卡片规则**：
- 最多 3 条要点
- 每条格式：`[对象] + [判断] + [原因]`，例："毅航评分领先，交付质量双项满分"
- 不重复图表中已有的数字（图表已显示 92 分，解读区不写"92 分"）
- 用视觉标记区分层级：色点小圆点 + 加粗对象名 + 普通说明

**KPI 卡片规则**：
- 每张卡片 1 个核心数字（24-32px 大字号）
- 配趋势箭头（▲▼）+ 简短说明
- 卡片背景用 Stone 50（`#faf8f5`）区分于主图

## 解读策略决策树

```
有解读需求？
├─ 否 → 版式选单图组（#01/#34/#40），无解读区
└─ 是 → 解读要传达什么？
   ├─ 核心结论（一句话） → #08 L 型，右侧 3 条要点
   ├─ 关键数字（几个大数） → #14 主次左右，右侧 KPI 卡片
   ├─ 趋势分析 + 摘要 → #14 主次左右，KPI 卡片 + 底部解读段
   ├─ 多维度洞察（优势/短板/风险） → #01 单图居中，底部洞察卡片行
   └─ 流程解读 → #22/#23 步骤流，每步含说明
```

## 版式选择速查（按意图）

| 表达意图 | 推荐版式 | 解读策略 | 理由 |
|---------|---------|---------|------|
| 排名/谁最大 | #01 单图居中 或 #08 L 型 | 底部/右侧 2-3 条解读 | 排名一目了然，解读补充背景 |
| 趋势变化 | #14 主次左右 70/30 | KPI 卡片 + 趋势解读段 | 趋势看图，数字看卡片 |
| 多维能力 | #01 单图居中 + 底部洞察卡片 | 优势/短板/风险三卡片 | 图形看全貌，卡片看重点 |
| 占比构成 | #01 单图居中 | 无解读或脚注 | 占比本身清晰 |
| 驱动因素 | #01 单图居中 | 无解读 | 瀑布图自带叙事 |
| 对比选择 | #02 左右对比 或 #26 三栏对比 | 底部差异标注 | 对比需要同维度对齐 |
| 目标差距 | #14 主次左右 70/30 | KPI 卡片突出差距 | 数字需要突出 |
| 风险评估 | #08 L 型 | 右侧风险要点列表 | 风险需要文字解释 |
| 时间线 | #22 水平步骤流 | 每步自带说明 | 流程即解读 |
| 流程/阶段 | #23 垂直步骤流 | 每步编号 + 描述 | 步骤多时垂直更清晰 |

## 解读区避坑

| 错误 | 修正 |
|------|------|
| 解读重复图表数字 | 解读写原因+影响，不写图表已显示的数字 |
| 解读超过 3 条 | 最多 3 条，超过说明信息过载 |
| 纯文字解读无视觉标记 | 用色点/加粗/卡片背景制造层级 |
| 解读区喧宾夺主 | 解读区宽度 ≤ 主图宽度的 35% |
| 标注与数据点重叠 | 标注偏移 12px 以上，不覆盖数据标签 |

## 与 40 种版式的关系

本版式选择器是 40 种构图模板的**决策前端**。它不定义新版式，而是决定何时用哪种。

完整 40 种版式见 `composition-templates.md` + `composition-templates-extended.md`。
未在上述速查表中列出的版式（#15-#21 特殊布局、#27-#40 复杂叙事），在特定场景（大屏、地理、学术研究等）中使用。
