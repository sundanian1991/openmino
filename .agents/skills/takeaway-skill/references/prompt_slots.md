# takeaway-skill prompt slots

Notes:

- This file only contains placeholder slots.
- It intentionally avoids third-party website examples.
- Real objects should be filled in at task time, not stored here.

## 1. Take mechanism, not skin

`Study this [reference object type], but do not copy it. Only distill the [structure / mechanism / visual / rhythm] layer, and tell me what can be taken and what must be redesigned.`

## 2. Identify before judging

`First tell me whether this [reference object] is strongest in mechanism, structure, style, or rhythm, then give me takeaway advice.`

## 3. Distill only

`Turn this [page / effect / visual system] into a distillation result with fixed outputs. Do not jump straight into implementation.`

## 4. Adapt it into my version

`I want to turn this [reference object] into my own version. Tell me what must change in expression and what mechanism can stay.`

## 5. Prepare a downstream handoff

`Turn this distillation into a handoff for [workflow type], including the smallest valid result, what not to touch yet, and the stop-loss rule.`

## 6. Public release safety

`If I want to turn this result into a public template, separate what can be published from what must be deleted or converted into placeholders.`

## 7. Compare two references

`Compare these two references separately. Tell me their shared mechanisms, each one's strongest qualities, the conflicts between them, and which one should remain the primary reference.`

## 8. Build an implementation brief

`Turn this [screenshot / recording / page section] into an implementation brief without writing code. Include regions, components, states, what to lock first, what not to touch yet, and the recommended technical container.`

## 9. Archive as a pattern card

`Turn this reference into a reusable pattern card. Add object type, interaction tags, style tags, evidence confidence, originality distance, and archive value.`
