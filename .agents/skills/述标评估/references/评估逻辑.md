# 评估逻辑

> 每个维度的分析路径、判断方法、交叉验证规则。

---

## 阶段1：AnySearch 搜索逻辑

### 搜索什么

| 维度 | 搜索关键词 | 关注点 |
|------|-----------|--------|
| 工商信息 | 公司全称 | 注册资本、成立时间、法人、经营范围、股东结构 |
| 司法风险 | 公司名 + 被执行人/失信/仲裁 | 被执行人记录、失信名单、劳动仲裁次数和金额 |
| 经营异常 | 公司名 + 行政处罚/经营异常 | 处罚原因、处罚金额、是否在异常名录 |
| 行业口碑 | 公司名 + 招聘/评价/新闻 | 员工评价（看准/脉脉）、行业动态、负面新闻 |

### 怎么判断

**低风险**：
- 成立3年以上，注册资本与业务规模匹配
- 无被执行人记录，无失信记录
- 无行政处罚或经营异常
- 行业口碑正面或中性

**中风险**（需述标时重点验证）：
- 成立时间较短（<3年）
- 有小额被执行记录（已结清）
- 有1-2次行政处罚（非严重）
- 口碑中有管理类投诉

**高风险**（需特别关注）：
- 被执行人金额大或未结清
- 在经营异常名录中
- 劳动仲裁频繁（说明劳务管理有问题）
- 有重大负面新闻或合规问题

## 阶段2：材料评估逻辑

### 从 PDF/PPT 中提取什么

**案例数据（2.1 金融电销案例）**：
- 提及的同类项目数量、类型、时间范围
- 转化率、接通率等核心数据
- 合作甲方名称（可与搜索交叉验证）
- 数据是否有具体数字，还是模糊表述

**团队配置（2.2 团队经验 + 3.2 资源保障）**：
- 项目经理姓名、从业年限、过往项目
- 核心成员数量、配置合理性
- 是否承诺专职团队、冗余率、培训周期

**方案架构（3.1 目标达成策略 + 3.3 风险应对）**：
- 分阶段目标是否清晰
- 策略是否有针对性（vs 通用模板）
- 风控措施是否具体（合规质检点、投诉预案、数据安全）

**管理能力（4.1 质量管理体系 + 4.2 合作配合度）**：
- 质检覆盖率、闭环整改流程
- 汇报机制（日/周/月）
- 专人对接承诺

### 交叉验证规则

| 材料声称 | 搜索验证 | 判断 |
|----------|---------|------|
| "近3年服务5家银行" | 搜索到具体项目或甲方背书 | 可信 |
| "近3年服务5家银行" | 搜索无相关记录 | 存疑，需述标追问 |
| "无违规记录" | 搜索无行政处罚 | 一致 |
| "无违规记录" | 搜索有行政处罚 | 矛盾，重大风险 |
| "团队10年经验" | 搜索公司成立才2年 | 存疑，可能是个人经验非公司经验 |

## 阶段3：述标问题生成逻辑

### 问题来源

**材料短板**：材料中数据模糊、缺失、或过于笼统的部分
- 例：只说"转化率高"没给数字 → 问具体数字和计算口径

**搜索矛盾**：搜索结果与述标声称不一致的部分
- 例：声称"无仲裁"但搜索有记录 → 问具体事件和处理结果

**可行性存疑**：方案中承诺的指标或资源需要验证
- 例：承诺"5天培训上岗" → 问培训内容和考核标准

### 问题分类

**必问**（直接影响评分）：
- 数据真实性验证
- 方案可行性验证
- 团队经验验证

**建议问**（辅助判断）：
- 优势项的深度了解
- 行业认知和方法论

**风险验证**（针对搜索发现）：
- 司法/经营异常的解释
- 负面信息的回应

## 阶段4：转文字评估逻辑

### 从转文字中看什么

**沟通逻辑（影响 4.2 合作配合度）**：
- 回答是否清晰、有条理
- 是否主动展开细节，还是挤牙膏
- 遇到不确定的问题，是否诚实说不知道

**应变能力（影响 3.1 目标达成策略）**：
- 追问时能否调整思路
- 是否能针对具体场景给方案
- 面对质疑时的反应（防御 vs 接纳）

**专业深度（影响 2.1 + 2.2）**：
- 对金融电销业务的理解程度
- 对合规要求的认知
- 对具体操作细节的熟悉度

### 交叉验证

| 转文字表现 | 材料/搜索 | 判断 |
|-----------|----------|------|
| 主动提及具体案例数字 | 材料中有对应数据 | 高度可信 |
| 主动提及具体案例数字 | 材料中无对应数据 | 可能是真实经验，也可能是编造 |
| 回避某类问题 | 搜索有相关风险 | 风险信号 |
| 承诺过度（什么都能做） | 材料中无对应能力支撑 | 不务实，扣分 |
