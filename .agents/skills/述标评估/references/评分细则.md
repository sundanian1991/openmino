## 详细评分标准

### 二、业务案例与经验（30分）

#### 2.1 金融电销案例（20分）

**优秀（18-20分）**：
- 近3年 ≥3个同类金融电销项目（银行/金融/保险）
- 有具体转化率数据，且 ≥行业均值120%
- 可通过搜索或公开信息交叉验证项目真实性
- 有甲方推荐或可联系验证

**良好（14-17分）**：
- 近3年 1-2个同类项目
- 转化率数据 ≥行业均值
- 数据可信但无法独立验证
- 案例与我司场景有一定匹配度

**一般（1-13分）**：
- 无金融电销案例，或案例类型不匹配
- 数据模糊（"转化率很高"无具体数字）
- 案例时间久远（>3年）
- 无法验证数据真实性

#### 2.2 团队经验（10分）

**优秀（9-10分）**：
- 项目经理 + 核心成员均有3年以上金融电销经验
- 有可验证的过往项目经历
- 述标时能展示专业深度（转文字验证）
- 团队稳定性好（搜索无频繁劳动仲裁）

**良好（6-8分）**：
- 仅项目经理符合经验要求
- 核心成员经验 1-3年
- 团队配置合理但非顶尖

**一般（1-5分）**：
- 团队经验与金融电销不匹配
- 项目经理经验不足3年
- 核心成员无相关经验

### 三、业务承接方案（40分）

#### 3.1 目标达成策略（15分）

**优秀（13-15分）**：
- 明确分阶段放款目标（月度/季度拆解）
- 配套数据筛选策略（目标客群画像、数据来源）
- 话术迭代机制（AB测试、转化漏斗优化）
- 动态调优方案（数据驱动的策略调整）
- 策略针对我司产品特性定制

**良好（10-12分）**：
- 目标合理但策略单薄
- 有基本方案但缺乏细节
- 方案有一定针对性但偏通用

**一般（1-9分）**：
- 目标模糊或不合理
- 缺乏达成路径
- 方案为通用模板，换个甲方名字也能用

#### 3.2 资源保障方案（15分）

**优秀（13-15分）**：
- 承诺 ≥15人专职团队
- 坐席冗余率 ≥10%
- 培训周期 ≤5天，有明确培训内容
- 人员到岗时间明确

**良好（10-12分）**：
- 团队规模达标（≥15人）但无冗余
- 培训周期 5-10天
- 资源投入合理但缺乏弹性

**一般（1-9分）**：
- 团队规模不足
- 无冗余方案
- 培训安排不清晰

#### 3.3 风险应对机制（10分）

**优秀（9-10分）**：
- 预设合规质检点（关键节点检查机制）
- 投诉应急预案（分级处理、响应时效）
- 数据安全方案（加密、权限、脱敏）
- 有历史风控经验可佐证

**良好（6-8分）**：
- 仅覆盖基础合规要求
- 有基本投诉处理流程
- 方案偏框架性

**一般（1-5分）**：
- 未提及风控措施
- 合规意识薄弱
- 无数据安全方案

### 四、管理能力（10分）

#### 4.1 质量管理体系（6分）

**优秀（5-6分）**：
- 质检覆盖率100%
- 闭环整改流程（发现问题→整改→复查→归档）
- 有自建质检标准或高于行业标准
- 质检数据可追溯

**良好（3-4分）**：
- 自建质检标准
- 有质检流程但闭环不完整
- 覆盖率 >80%

**一般（1-2分）**：
- 无体系化质检方案
- 仅依赖甲方质检
- 无整改机制

#### 4.2 合作配合度（4分）

**优秀（4分）**：
- 承诺日/周/月三级汇报机制
- 专人对接策略调整
- 沟通响应及时（转文字验证）
- 主动性和配合意愿强

**良好（2-3分）**：
- 常规沟通机制（周报/月报）
- 有对接人但非专人
- 配合意愿一般

**一般（0-1分）**：
- 未明确协同方式
- 沟通机制模糊
- 配合意愿弱

## 依据引用规范

每个评分必须附带依据，格式：

```
[来源类型] 具体内容

来源类型：
- 搜索：来自 AnySearch 的公开信息
- PPT：来自述标 PPT 的内容
- PDF：来自述标 PDF 的内容
- 转文字：来自述标转文字的内容
- 交叉：来自多方交叉验证的结论
```

示例：
```
18分 | [PPT] 近3年服务建设银行、工商银行、平安保险3家金融机构电销项目，
转化率分别为12%、15%、11%，均高于行业均值8%。
[搜索] 建设银行项目在2024年行业报道中被提及。
```
