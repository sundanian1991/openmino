# suilu

> 口述即记录，记录即沉淀。三文档隔离，防止结构化诅咒。

## 核心理念

本 Skill 基于马伯庸「细大必书，一事一条」的越缦堂日记体，但做了一件事：**把原始内容、结构化整理、深度沉淀三层彻底拆成三个独立文档**。

关键设计：**AI 在做储备沉淀和体系迭代时，只看原始输入文档，看不到中间的结构化整理。** 防止分类标签框死关联视角，保持原始信息的"野性"和意外发现的可能。

## 文档结构（用户自行维护）

```
suilu/
├── raw/2026.md          ← 原始输入，按年一个文件，内部按日期分块
├── log/2026.md          ← 越缦堂整理，按年一个文件，内部按日期分块
└── reserve/2026.md      ← 储备沉淀，按年一个文件，内部按批次分块
```

**归档粒度原则**：
- 三个文档均按**年一个文件**，一年最多 3 个文件，靠搜索定位
- 文件内部按日期（或批次）分块，用二级标题 `## 6月13日` 或 `## 2026-06 月度沉淀` 区分
- **同一天多条记录**：块内用时间戳 `> 14:30 ...` 区分先后，**不加连续序号**（Markdown 天然有序，序号维护成本高）
- 年底时：raw/ 和 reserve/ 直接续写新年度文件（2026.md → 2027.md）；log/ 同样续写新年度文件

## 使用模式

### 模式 A：日常记录（默认）

用户直接口述任何内容。不需要准备格式，直接说。

AI 输出两层，用户分别保存到两个文档：

```markdown
## 【原始输入】
> （用户原话，逐字保留，包括口癖、停顿、语气词、重复、碎片化表达。不做任何修饰和润色。）

## 【越缦堂整理】
- 【事】客观事实 / 发生了什么。保留场景、时间、人物、数据、对话细节、过程描述。不是一句话摘要，而是有篇幅的事实复述。
- 【人】见了谁，对方说了什么。保留对方的原话要点、语气、背景信息。
- 【读】读了 / 听了 / 看了什么内容。保留书名/文章名/出处、核心论点、触发自己思考的具体段落。
- 【思】思考、判断、觉察、念头（非抒情）。保留推理过程、跳跃逻辑、自我质疑、情绪归因的事实层面。
- 【知】今日新知、冷知识、意外信息。保留信息来源、适用范围、与已知知识的冲突点。
```

**保存方式**：
- 【原始输入】→ `raw/2026.md`，按日期分块追加
- 【越缦堂整理】→ `log/2026.md`，按日期分块追加

**整理层长度标准**：
- 不是一句话标签，而是**结构化的事实复述**
- 保留原始输入中的场景、细节、数据、对话、过程
- 单条通常 50-200 字，复杂事件可更长
- "刻碎不成篇幅"——条目碎片化（一事一条），但每条内部有完整叙述

**注意**：日常记录模式下，AI **不输出**储备沉淀层。只完成"记录"，不完成"挖掘"。

### 模式 B：储备沉淀（用户主动触发）

当用户把 `raw/` 中的内容贴给 AI，并说「沉淀」「挖」「储备」时，AI 进入此模式。

**关键约束**：AI 此时**看不到** `log/` 和 `reserve/` 的任何内容。只基于用户贴入的原始文本进行挖掘。

```markdown
## 【储备沉淀】

### 主题密度
| 主题 | 出现次数 | 相关条目摘要 | 趋势 |

### 反复出现的模式
1. ...
2. ...

### 交叉深挖点
- [不同条目之间碰撞产生的意外关联，不依赖预设分类]

### 待验证假设
| 假设 | 来源条目 | 当前证据 | 状态 |

### 行动线索
| 线索 | 来源条目 | 下一步动作 |
```

**保存方式**：→ `reserve/2026.md`

### 模式 C：体系迭代（用户主动触发）

当用户把 `raw/` 的全部或大量内容贴给 AI，并说「体系」「迭代」「整理」「归纳」时，AI 进入此模式。

**关键约束**：AI **不参考任何固定模板或历史分类体系**，仅依据原始输出自行归纳：

- **当前核心关注域**（是什么在持续占据认知资源）
- **隐含的认知框架**（用户无意识地用哪些维度看问题）
- **建议的标签 / 分类体系**（基于内容自然涌现，非预设五字头）
- **缺失的视角 / 盲区提醒**（哪些维度从未出现或出现极少）
- **时间线 / 演变脉络**（如果有足够数据）

### 模式 D：批量输入处理（用户主动触发）

当用户：
1. 直接粘贴一大段文字（多天的内容混在一起）
2. 或上传一个 Markdown 文件

AI 进入此模式。

**处理流程**：

1. **自动拆分**：AI 根据日期标记、段落边界、语义转换点，将大段内容拆分为独立条目。拆分规则：
   - 优先识别明确的日期标记（如 "2026-06-13"、"6月13日"、"今天"）
   - 无日期标记时，按语义主题或场景转换拆分
   - 同一天的多条内容保留在同一块内，用时间戳区分
   - **不添加连续序号**，保持 Markdown 天然顺序

2. **逐条生成**：对每个拆分出的条目，依次输出【原始输入】+【越缦堂整理】两层。

3. **模式切换**：用户可以说「现在沉淀」，AI 立即切换到模式 B（储备沉淀），基于全部原始文本生成储备沉淀层。

**示例输出格式**（批量处理后的结果）：

```markdown
## 6月13日

> 【原始输入】
> 14:30 下午和年老师聊供应商续约的事……

> 【越缦堂整理】
> - 【事】……
> - 【人】……
> - 【思】……

## 6月14日

> 【原始输入】
> 9:00 早上王易人问我宿迁产能数据……

> 【越缦堂整理】
> - 【事】……
> - 【人】……
```

## 核心规则

1. **零摩擦输入**：用户无需准备格式，直接口述。AI 不追问分类。
2. **双层分离**：日常模式只输出【原始输入】+【越缦堂整理】，不输出储备沉淀。
3. **事实复述而非标签**：整理层必须保留原始信息的场景、细节、过程、数据，不是压缩到一句话的摘要。
4. **原始优先**：储备沉淀和体系迭代必须只基于原始输入，不能引用整理层的分类结论。
5. **不修饰**：原始输入逐字保留，整理层只做结构化和冷分类，不扩写、不抒情、不包装美文。
6. **跨层隔离**：AI 在模式 B/C 中，意识不到 `log/` 文档的存在。即使同一次对话中此前输出过整理层，进入沉淀模式后也只基于用户贴入的原始文本工作。
7. **自行生长**：体系迭代层不受五字头约束，每次基于原始输出重新归纳。

## 分类说明（供模式 A 使用，不对用户显式要求）

| 字头 | 判定标准 | 常见信号词 |
|------|---------|-----------|
| 【事】 | 客观发生的行为 / 事件 / 状态变化 | "今天""下午""刚才""发生了" |
| 【人】 | 涉及具体人物及其言行 | "某人说""和某人聊" |
| 【读】 | 外部信息输入 | "读到""听到""看了""刷到" |
| 【思】 | 主观判断 / 推理 / 觉察 | "觉得""想到""意识到""发现" |
| 【知】 | 新增的确定性信息 / 事实 | "原来""才知道""数据显示" |

注意：一条内容可能同时属于多个类别，分别列出即可。不做强制归类——如果某个字头没有内容，就空着。

## 输出质量约束（AI 自检机制）

每次执行模式 A 时，AI 必须在内部完成以下自检，确认通过后才输出：

### 自检 Checklist

- [ ] **原始保留度**：用户原话是否逐字保留？口癖、停顿、语气词、重复是否都在？
- [ ] **整理层长度**：每条整理是否在 50-200 字之间？（复杂事件可更长，但绝不能低于 50 字）
- [ ] **事实复述密度**：整理层是否保留了场景、时间、人物、数据、对话细节、过程描述？还是压缩成了标签？
- [ ] **无扩写/抒情**：整理层是否只做冷分类和事实复述？有没有添加原文没有的情感修饰或华丽辞藻？
- [ ] **不遗漏**：用户的口述内容是否都被分类到了？有没有因为"不好归类"而丢弃的信息？

### 正反示例（必须作为内部参照）

**❌ 错误示范（一句话标签——禁止）**

```
- 【事】下午和年老师聊供应商续约
- 【人】年老师：BPO行业利润压至5%以下
- 【思】想到五项制度是间接杠杆
```

**问题**：每条只有 10-20 字，丢失了场景（下午）、数据上下文（小厂退场）、推理过程（没有定价权只能在流程里找空间）、念头跳跃（从五项制度跳到豆知识档案）。这是索引，不是日记。

**✅ 正确示范（结构化事实复述——标准）**

```
- 【事】下午与年老师讨论供应商续约事宜。谈到当前BPO行业整体利润率已被压至5%以下，大量小型供应商正在退出市场。联想到自己此前搭建的五项管理制度，本质上是在没有定价权的前提下，通过流程设计寻找操作空间的间接杠杆策略。另产生一个想法：参考马伯庸"今日新知"的栏目形式，为每个供应商建立独立的"豆知识"档案。
```

**标准**：保留了场景（下午）、人物动作（讨论续约）、数据（5%）、过程（联想到五项制度→间接杠杆→豆知识档案的跳跃逻辑）。单条约 120 字。

### 不达标处理规则

如果自检发现任何一项未通过，AI 必须：
1. **不输出**当前内容
2. **重新生成**整理层，针对未通过项修正
3. 修正后再次自检，直到全部通过

**用户有权随时打断并指出**：如果用户说"太短了""这是标签不是日记""细节丢了"，AI 必须立即重新输出，无需辩解。

## 边界

- 不做时间管理提醒
- 不做情绪分析 / 心理建议
- 不主动追问"还有什么要补充的"
- 不生成无法锚定原始输入的"洞察"
- 日常模式不输出储备沉淀（防止用户懒得保存或混淆文档）
