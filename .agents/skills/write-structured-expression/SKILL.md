# 结构化表达框架

> 核心：不同场景用不同叙事结构。先判断卡在哪，再走对应的路。

---

## 第一步：识别模糊类型

> **问用户：你现在卡在哪了？**

| 场景 | 信号词 | 用户买什么 | 叙事结构 |
|------|--------|-----------|----------|
| **范围模糊** | "不知道从哪下手"、新课题 | 你的框架 | 五步叙事 |
| **路径模糊** | "知道要去哪，不知道怎么到" | 你的权衡逻辑 | 对比叙事 |
| **优先级模糊** | "事太多，先做什么" | 你的筛选标准 | 漏斗叙事 |
| **因果模糊** | "出了问题不知道为什么" | 你的排除过程 | 侦探叙事 |
| **冲突模糊** | "各方诉求不一致" | 你对各方诉求的洞察 | 桥梁叙事 |
| **决策报告** | "报告写了没人看"、分析已完成需推动决策 | 你的可执行方案 | 决策叙事 |

**判断技巧**：用户说了一句描述，问他"你觉得最难的部分是什么"——答案会落在上表某一栏里。一个问题跨两类 → 拆成两步。

## 第三步：生表达框架

按照对应场景的参考文件，引导用户走完每一步，然后输出完整的表达框架。

### 通用输出格式

```markdown
## 📋 表达框架：[主题]

### 场景类型：[X]模糊
### 叙事结构：[X]叙事
### 一句话模板：[对应的一句话总结]

---

### 步骤1：[名称]
- 要讲什么：[一句话]
- 产出物：[具体描述]

### 步骤2：[名称]
...

### 自检清单
- [ ] [自检问题1]
- [ ] [自检问题2]
```

### 核心原则

1. **经营逻辑 vs 业务逻辑** — 这是两套完全不同的沟通目的，结构用反了报告就废：
   - **经营逻辑**（前五类模糊）：卖你的思考过程。领导要判断"这个人靠不靠谱"。结构：过程→框架→结论（自下而上）。失效模式：直接扔结论 → "从哪来的？"
   - **业务逻辑**（决策报告）：卖你的行动方案。领导要决定"我接下来做什么"。结构：结论→原因→行动（自上而下）。失效模式：罗列分析过程 → "所以呢？"
2. **分析报告 vs 业绩报告** — 分析报告卖逻辑（讲思考过程），业绩报告卖结果（扔数字）。探索型/转型型用分析结构，常规进度用业绩结构。混淆了全盘推翻。
3. **每个选择交代理由** — 九宫格里标"高潜力"凭什么？能扛住三个"为什么"。
4. **标定边界** — 结论在什么条件下成立，什么条件下不成立。

## 第四步：价值放大升级（最终打磨）

表达框架生成后，**必须**过一遍价值放大层。读取 `references/value-perception.md`，执行三层递进：

1. **显性化** — 做了什么说出来。动作+成本+结果，不让隐性工作被掩盖
2. **放大** — 这件事还能衍生什么。可复用吗？可沉淀吗？可制度化吗？（Alignment是核心手段）
3. **预判** — 下一步提前准备什么。资源预警 + 风险预案 + 需求预判

> 不是"怎么包装好看"，而是通过表达这件事本身，逼迫自己看到更多价值。
