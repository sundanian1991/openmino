# 场景2：路径模糊 — "知道要去哪，不知道怎么到"

**触发信号**：目标明确，但团队对"怎么到"意见不一致，或者自己也没想清楚。
**叙事结构**：对比叙事——先终点，再展开路径对比，最后推荐+理由
**一句话模板**：

> "目标是[终点画像]。有三条路——A[路径1]重但稳，B[路径2]快但浅，C[路径3]居中。我推荐[某条]，因为[权衡理由]。接受的风险是[什么]，应对方式是[什么]。"

**实战举例**：供应商制度体系化——A制度建设先行（稳但慢）、B系统工具先行（快但可能脱离实际）、C试点先行（快速验证再推广）。

---

## 步骤详解

### 步骤1：画终点

| 要素 | 说明 |
|------|------|
| **动作** | "到了"长什么样 |
| **产出** | 成功画像（3-5个具体特征） |
| **工具** | 目标画像法 |
| **自检** | 别人听了能想象出画面吗？ |

**引导话术**：
> "如果这件事做成了，你希望看到什么具体的画面？用3-5个特征描述一下。"

---

### 步骤2：列路径

| 要素 | 说明 |
|------|------|
| **动作** | 至少3种走法 |
| **产出** | 路径清单（A/B/C） |
| **工具** | Issue Tree（路径穷举） |
| **自检** | 有没有遗漏明显的路径？ |

**引导话术**：
> "从现状到终点，你能想到几种走法？至少列出3条。"

---

### 步骤3：评路径

| 要素 | 说明 |
|------|------|
| **动作** | 每条路的成本、风险、时间 |
| **产出** | 多维对比表 |
| **工具** | 多维评估矩阵（成本×风险×时间） |
| **自检** | 每条路的评估标准一致吗？ |

**引导话术**：
> "每条路分别看：投入多少？风险是什么？多久见效？用同样的标准评估。"

---

### 步骤4：推荐

| 要素 | 说明 |
|------|------|
| **动作** | 选哪条、为什么 |
| **产出** | 推荐路径 + trade-off说明 |
| **工具** | 权衡说明（为什么不是另外几条） |
| **自检** | 如果不选推荐的那条会怎样？ |

**引导话术**：
> "你推荐哪条路？为什么不是另外两条？你愿意为此放弃什么？"
