# 场景5：冲突模糊 — "各方诉求不一致，推不动"

**触发信号**：会议反复讨论无果、邮件来回扯皮、推进卡在某个点上。
**叙事结构**：桥梁叙事——承认各方合理性→暴露分歧根源→搭桥（共识+折中）→推行动
**一句话模板**：

> "A方要[X]，B方要[Y]。表面看是对立的，但根源是[分歧根源]。共识是[Z]。分歧点我设计了[方案]，trade-off是[什么]。"

**实战举例**：供应商考核标准争议——根源不是标准松紧，是数据口径不一致（人头 vs 有效工时）。先对齐口径，标准自然好谈。

---

## 步骤详解

### 步骤1：呈现立场

| 要素 | 说明 |
|------|------|
| **动作** | 各方到底要什么 |
| **产出** | 利益地图 |
| **工具** | 利益地图（谁要什么、为什么） |
| **自检** | 我有没有偏袒某一方？ |

**引导话术**：
> "各方的诉求分别是什么？他们各自为什么坚持？你客观描述一下。"

---

### 步骤2：找根源

| 要素 | 说明 |
|------|------|
| **动作** | 分歧到底在哪一层 |
| **产出** | 分歧归类（利益/原则/信息差） |
| **工具** | 三层归因（利益冲突 vs 原则分歧 vs 信息不对称） |
| **自检** | 是真的利益冲突还是信息不对称？ |

**引导话术**：
> "分歧的根源是什么？是真的利益对立，还是信息不对称？或者是原则问题？"

---

### 步骤3：找共识

| 要素 | 说明 |
|------|------|
| **动作** | 大家都同意的部分 |
| **产出** | 共识清单 |
| **工具** | 共识清单法 |
| **自检** | 共识够不够作为起点？ |

**引导话术**：
> "大家有没有都同意的部分？哪怕很小，先找共识。"

---

### 步骤4：设计方案

| 要素 | 说明 |
|------|------|
| **动作** | 在分歧点上怎么折中 |
| **产出** | 2-3个折中选项 |
| **工具** | trade-off分析（每个选项各方得失） |
| **自检** | 每个选项各方能接受吗？ |

**引导话术**：
> "在分歧点上，你能设计几个折中方案？每个方案的得失是什么？"

---

### 步骤5：推落地

| 要素 | 说明 |
|------|------|
| **动作** | 怎么执行 |
| **产出** | 执行方案 + 责任分摊 |
| **工具** | RACI矩阵（谁执行/谁负责/谁咨询/谁告知） |
| **自检** | 谁干什么、什么时候干完？ |

**引导话术**：
> "定了方案之后，谁干什么？什么时候完成？怎么确认？"
