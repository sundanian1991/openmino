# VISUAL STREAMS — 9 种视觉气质

> nian-design 的 9 种气质定义。**7 视觉型 + 2 结构型**，全部独立。
> 视觉型管"页面内怎么表达"，结构型管"整页怎么编排"。
> 一份报告可以同时是"斜切入场（视觉型）+ 黑条书签（结构型）"。

---

# 视觉型（7 种）

> 管"页面内的视觉表达方式"。决定 Hero 怎么切、节奏怎么走。

---

## 1 · Statement 宣言对齐

**频次**：25（最高）
**一句话**：粗体断言 + 单行大字 + 大量留白
**来源**：V4 hero variant + S04 Split Statement 演化

### 典型骨架

| 字段 | 值 |
|---|---|
| Hero | V4-Statement（左大断言 + 右侧数据/装饰） |
| Layout | S04 Split Statement（7fr 5fr） |
| 适用 | 品牌宣言 / 价值观 / 决策建议开头 |

### 克制规则

- 标题字号 ≥ 72px（强制视觉冲击）
- 一句话力量，不超 2 行
- 副文案 ≤ 60 字
- 不用插图（字就是图）

### 反模式

- 三行以上的"宣言"（变说教）
- 副文案超过 1 段
- 配合花哨插图

### 典型 showcase

- H062-Haglofs早期v2-statement
- H001-品牌展示-statement
- H052-数据分析-通用报告-statement

---

## 2 · Diagonal 斜切入场

**频次**：13
**一句话**：clip-path 切出深色斜块，135deg 微渐变
**来源**：V1 Diagonal hero + S01 Hero Cover 演化

### 典型骨架

| 字段 | 值 |
|---|---|
| Hero | V1-Diagonal（左 5fr + 右 7fr 切角） |
| Layout | S01 Hero Cover |
| 适用 | 品牌展示 / 工作汇报 / 高调开屏 |

### 克制规则

- 切角范围 **8-15%**（polygon 第二点 8%-15%）
- 深色块宽度 **45-55%**（不能超过一半也不能低于 1/3）
- 渐变用 **135deg 2 色相近**（#2D2A26 → #1A1816）
- 不用 3 色或彩虹渐变（变成 SaaS 落地页）

### 反模式

- 切角 > 20%（设计过度）
- 渐变色相跨度过大
- 在正文中也用斜切（只在 Hero 用）

### 典型 showcase

- R4-工作汇报-diagonal
- R3-品牌展示
- H029-品牌系统-视觉标识-diagonal
- H058-供应商管理-分量策略-diagonal

---

## 3 · Split 对分屏

**频次**：11
**一句话**：左右 50/50 或 60/40 平分，**无斜切**
**来源**：V2 Split hero + S09 双栏演化

### 典型骨架

| 字段 | 值 |
|---|---|
| Hero | V2-Split（1fr 1fr） |
| Layout | S09 对照双栏 |
| 适用 | 对比 / 新旧 / 方案 A vs B |

### 与 Diagonal 的关键区别

| 特征 | Split | Diagonal |
|---|---|---|
| 边缘 | 直边 | 斜切 |
| 视觉冲击力 | 中 | 强 |
| 适用 | 对比 / 引用 | 开场 / 焦点 |

**绝不能混用**：要么 Split 要么 Diagonal，没有"斜切分屏"。

### 克制规则

- 比例 1:1 / 1:2 / 2:1 三选一，**不要 1:1.3**（奇怪比例）
- 中间不画线（用空白分隔）
- 文字对齐跟随内容：左对比左对齐，左对比右对齐

### 反模式

- 比例 1:1.5 等奇怪值
- 中间加竖线
- 两侧字体大小一致（应该有主次）

### 典型 showcase

- R1-数据分析报告-split
- H006-工作汇报-split
- H011-数据分析-split
- H021-知识管理-split

---

## 4 · Numeral 数字开屏

**频次**：3
**一句话**：3-5 个大数字开屏，第一秒看数字
**来源**：V6 Numeral hero + S03 Hero Numeral 演化

### 典型骨架

| 字段 | 值 |
|---|---|
| Hero | V6-Numeral（3-5 个 KPI 横排） |
| Layout | S03 Hero Numeral |
| 适用 | 数据报告 / 看板 / 季度业绩 |

### 克制规则

- 数字 **3-5 个**（少于 3 显单薄，多于 5 显拥挤）
- 数字字号 ≥ 64px
- 单位字号 ≤ 数字字号 1/3
- 必须有 1 个数字用 --yellow 高亮（"看这里"）

### 反模式

- 6+ 数字（变数据墙）
- 数字字号 < 48px
- 全部数字同色（必须有 1 个高亮）

### 典型 showcase

- H004-供应商管理-numeral
- H020-操作手册-numeral
- 场景模板-numeral

---

## 5 · Entrance 入场沉浸

**频次**：3
**一句话**：整屏沉浸 + 极简元素，铺垫情绪
**来源**：与 Long-form Hero 同源但用途不同

### 典型骨架

| 字段 | 值 |
|---|---|
| Hero | 100vh 整屏 + 居中 1 行大字 |
| Layout | 整页 1 个 section 后续延展 |
| 适用 | 沉浸式品牌页 / 落地页 / 故事入口 |

### 与 Long-form Hero 的区别

| 特征 | Entrance | Long-form Hero |
|---|---|---|
| 后续内容 | 延展多 section | 立即进入长文阅读 |
| Hero 字号 | 96-120px | 48-72px |
| 阅读时长 | < 10 秒 | > 60 秒 |
| 节奏 | 慢、铺垫 | 快、直入 |

**两者都用整屏 Hero，但下游节奏不同。**

### 克制规则

- 100vh 整屏
- 元素 ≤ 3 个（标签 + 标题 + 1 个装饰）
- 数字编号 1 处（如 "CHAPTER 01"）
- 不用滚动条引导

### 反模式

- 100vh 内塞 ≥ 5 个元素
- 没有过渡直接跳到正文

### 典型 showcase

- H053-数据分析-深潜Part1-split
- 深度阅读-entrance

---

## 6 · Pulse 节奏脉动

**频次**：2
**一句话**：时间线 / 脉动 / 节奏可视化
**来源**：S08 时间线 + S10 流程步骤 + S20 闭环流程

### 典型骨架

| 字段 | 值 |
|---|---|
| Hero | Numeral 或 Split 简化 |
| Layout | S08 时间线 / S20 闭环流程 |
| 适用 | 项目里程碑 / 飞轮 / 周期循环 |

### 克制规则

- 时间节点 ≤ 7 个（多则分页）
- 节点间距均匀
- 当前节点用 --yellow 高亮
- 不用动画（用静态节奏感）

### 反模式

- 节点 > 7（变成"项目大事记"）
- 加滚动动画
- 节点之间画曲线（直线即可）

### 典型 showcase

- R1-数据叙事-pulse
- 品牌系统-数据分析中心-pulse

---

## 7 · Dashboard 看板

**频次**：2
**一句话**：数据密集、信息密度高，多指标多视图
**来源**：S14 格子卡 + S22 Bento

### 典型骨架

| 字段 | 值 |
|---|---|
| Hero | Numeral（缩到 60vh） |
| Layout | S14 格子卡 / S22 Bento |
| 适用 | 业务看板 / 监控大屏 / 实时数据 |

### 克制规则

- 至少 **3 种视觉形态**（规则 5 强约束）
- 信息块之间用 1px border 分隔（不用色块）
- **不用色编码身份**（单色系统，色只编码状态）
- 文字密度大 → 用 9-11px 标签字号

### 反模式

- 全用同一种图表（看腻）
- 用 6 色编码 6 个供应商（违反 nian 单色约束）
- 框线加色块（双重强调）

### 典型 showcase

- 仪表盘-dashboard
- 数据分析-FT粉红-dashboard

# 结构型（2 种）

> 管"整页怎么编排"。与视觉型正交，可叠加。

---

## S1 · 黑条书签

**频次**：H062 1 个，但范式独立
**一句话**：64px 固定深色长条 + 竖排章节链接
**来源**：H062-Haglofs早期v2-statement

### 适用场景

- 内容 ≥ 3 章节
- 需要持久导航
- 品牌希望"有书卷气"

### 克制规则

- 宽度 **48-80px**（不能更宽）
- 黑条里**只放**：品牌名（竖排）+ 3-5 个章节链接（竖排）+ TOP 回到顶部
- 整页全黑条（不只是某一屏）
- 移动端 < 1024px 必须隐藏
- 不用图标、不用 hover 弹出

### 反模式

- 黑条加图标
- 章节链接 > 7 个
- 在窄屏继续占位
- 用黑条做"短报告"（< 3 章节用就是装饰过度）

### 典型 showcase

- H062-Haglofs早期v2-statement

---

## S2 · 长文导航

**频次**：R5 / H010 等多个
**一句话**：sticky 侧栏 + 1:2 grid + TOC 编号
**来源**：R5-知识管理.html

### 适用场景

- 文档 ≥ 3 章节
- 文档字数 ≥ 1500 字
- 读者需要"我在哪"的导航

### 克制规则

- 侧栏 ≤ 主内容的 1/2（1:2 grid 起步）
- 侧栏**只放 3 类**：标题 / 元信息 / TOC
- TOC 项数 ≤ 8
- sticky top ≥ 60px（给顶部 nav 留位）
- 章节之间间距 ≥ 80px
- 移动端塌成单列，侧栏不再 sticky

### 反模式

- 短内容（< 1500 字）也用长文导航
- 侧栏塞图片 / 按钮 / 搜索
- TOC 项数 > 8

### 典型 showcase

- R5-知识管理
- H010-知识管理-statement
- H021-知识管理-split

# 9 种气质选择流程

## 何时用哪种

```
1. 看内容长度
   - < 1500 字 → 不用结构型
   - ≥ 1500 字 → 考虑 S2 长文导航
   - ≥ 3 章节 + 持久导航 → 考虑 S1 黑条书签

2. 看内容性质
   - 数据密集（≥ 3 指标）→ Dashboard
   - 单一震撼指标 → Numeral
   - 时间/阶段 → Pulse
   - 对比 → Split
   - 主张/价值观 → Statement
   - 品牌展示 → Diagonal / Entrance

3. 看气质叠加
   - Diagonal + S1 = 高调品牌 + 持久导航
   - Statement + S2 = 长文 + 主张
   - Numeral + Dashboard = 数字开屏 + 数据延展
```

## 与 nian 美学的对应

| 气质 | 用的字体 | 用的色 | 用的特征 |
|---|---|---|---|
| Statement | Playfair Display 96-120px | --darkgray 文本 | 留白 ≥ 30% |
| Diagonal | Playfair Display 64-84px | --darkgray 深色块 | clip-path 切角 |
| Split | Playfair 56px + Inter 16px | --bg + --darkgray | 50/50 或 60/40 |
| Numeral | JetBrains Mono 64-96px | --yellow 高亮 1 个 | 数字 + 单位 |
| Entrance | Playfair Display 96-120px | --darkgray | 100vh 整屏 |
| Pulse | JetBrains Mono 标签 + Inter 文字 | --olive 节点 | 时间线 |
| Dashboard | Inter + JetBrains | --rock / --olive | ≥ 3 形态 |
| 黑条书签 | JetBrains Mono 9-10px | --darkgray | 64px 竖排 |
| 长文导航 | Inter + JetBrains | --text-secondary TOC | sticky 侧栏 |
