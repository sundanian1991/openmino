# 用户上下文收集协议 (Collection Protocol)

> 用于在本地 Agent 中引导用户系统性梳理个人工作上下文。
> **目标**：用 30-45 分钟完成第一轮收集，产出一份结构化的用户画像文档。

---

## 一、前置说明（给用户看的）

### 为什么要做这个？

你和 AI 助手的每次对话，都等于在"重新介绍自己"。这个协议帮你一次性梳理清楚"我是谁、我在哪、我管什么、我怎么想"，以后 AI 就能：**少问废话、精准理解、不犯常识性错误**。

### 三个原则

1. **不追求完美**：第一轮先抓骨架（60分即可），细节可以后续补充
2. **不纠结实时数据**：只存"结构"，不存"状态"（比如存"我管供应商"，不存"目前27家"）
3. **诚实面对模糊**：如果你"好像知道但说不清楚"，标记为"待澄清"——这比瞎写一个答案更有价值

### 预计时间

- 第一轮（L0-L2 骨架）：30-45 分钟
- 第二轮（L3 偏好 + 补充细节）：15-20 分钟
- 文档扫描（如果本地有现成材料）：10-15 分钟

## 二、信息分层清单

### L0 基础画像（你是谁）

**必答：**
- [ ] 你的角色/职位名称是什么？（如：电销团队运营管理、供应商管理专员）
- [ ] 向谁汇报？（职级+部门，如：向销售运营部总监汇报）
- [ ] 你的核心职责用一句话怎么概括？
- [ ] 你在这个岗位多长时间了？

**选答：**
- [ ] 你的职业背景路径是什么？（什么专业毕业→哪些岗位→现在）
- [ ] 你的核心能力标签是什么？（别人评价你最擅长的3件事）
- [ ] 你的职业发展目标（短期1年/中期3年）是什么？

**头脑自检：**
> 如果你的同事要用一句话向别人介绍你，他会怎么说？这个描述和你自己的认知一致吗？

---

### L1 工作结构（你在什么组织中）

**必答：**
- [ ] 你所在的部门/团队全称是什么？在组织架构中处于什么位置？
- [ ] 你的直接上下游是谁？（谁给你输入、你给谁输出）
- [ ] 你的**权限边界**是什么？（你能决定什么、不能决定什么——这是最容易被 AI 误解的点）
- [ ] 你的**资源边界**是什么？（预算、人力、时间、系统权限）

**选答：**
- [ ] 部门的核心KPI/OKR是什么？
- [ ] 你的个人绩效是怎么被评估的？
- [ ] 部门内有哪些关键角色？（与你协作最密切的3-5人，他们的角色）
- [ ] 跨部门协作中，哪些部门和你交互最多？

**头脑自检：**
> 有没有一个决策你"想做但做不了"？这个约束条件背后的原因是什么？（缺权力、缺信息、缺资源、缺流程？）

**权限边界特别重要，请详细展开：**
```
你能做的：
- ...

你不能做但可以影响的：
- ...

你不能做且影响不了的：
- ...
```

---

### L2 业务知识（你处理什么）

**必答：**
- [ ] 你的日常工作分为哪几大类？（如：供应商管理、合同签署、业绩分析、团队协调）
- [ ] 每类工作的**标准流程**是什么？（不用写SOP，写"谁→做什么→输出什么→给谁"）
- [ ] 你工作中最常用的**业务术语**有哪些？（列10-15个，每个给一句话定义）
- [ ] 你工作中涉及的**关键指标**有哪些？（名称+计算公式/口径+目标值范围）

**选答：**
- [ ] 你所在行业的核心逻辑是什么？（如果向一个外行解释"这个行业怎么赚钱"，你怎么说？）
- [ ] 你工作中最常见的3类问题/痛点是什么？
- [ ] 你工作中"一犯错就后果严重"的环节有哪些？
- [ ] 有没有一些"行规"或"潜规则"？（不成文但重要的做事方式）

**头脑自检：**
> 你有没有一些术语，自己以为很清楚，但让你给一个新手解释时发现"好像也不是那么清楚"？把这些标记为"术语待澄清"。

**术语模板（每个术语请按此格式）：**
```
术语：XXX
一句话定义：...
使用场景：...
计算方式（如果是指标）：...
常见问题/误区：...
```

---

### L3 思维偏好（你怎么思考）

**必答：**
- [ ] 你做决策时最依赖什么？（数据/直觉/经验/他人意见/规则）
- [ ] 你最喜欢的思维方式/框架是什么？（如：咏春拳哲学、第一性原理、系统思维）
- [ ] 你偏好的信息呈现方式是什么？（表格/图表/文字/口头）
- [ ] 你在沟通中倾向于直接还是委婉？在写作中呢？

**选答：**
- [ ] 你的"认知舒适区"是什么类型的思考？（分析/创意/人际/执行）
- [ ] 你容易陷入的思维陷阱是什么？（过早下结论/追求完美/回避冲突/过度分析）
- [ ] 什么类型的任务会让你进入心流状态？
- [ ] 你最欣赏的决策者/思考者是谁？为什么？

**头脑自检：**
> 回忆最近一个重要的正确决策，你是怎么做出那个决策的？回忆一个错误的决策，当时缺了什么视角？

---

### L4 工具与环境（你用什么）

**必答：**
- [ ] 你日常工作中使用哪些系统/工具？（每个标注用途）
- [ ] 你最常用的信息来源是什么？（内部系统/邮件/IM/文档/外部网站）
- [ ] 你的文档习惯是什么？（用什么工具写、怎么命名、怎么分类）

**选答：**
- [ ] 有没有你特别依赖但不太稳定的工具/系统？
- [ ] 你的工作环境偏好是什么？（独处/协作、安静/有背景音、纸质/电子）

**⚠️ 注意：只记录"用什么系统"，不记录账号、密码、具体URL。**

## 三、本地文档速查映射

如果你本地有工作文档，不用全文阅读，按以下清单**快速扫描提取**：

| 文档类型 | 扫描重点 | 对应L层级 | 预计时间 |
|---------|---------|----------|---------|
| 岗位说明书/职责文档 | 职责范围、权限边界、汇报线 | L0-L1 | 3分钟 |
| 部门介绍/组织架构图 | 团队构成、上下游关系 | L1 | 3分钟 |
| 过往方案/报告 | 常用术语、指标口径、分析框架 | L2 | 10分钟 |
| 制度/SOP文档 | 标准流程、关键节点、checklist | L1-L2 | 5分钟 |
| 绩效评估/复盘文档 | 目标指标、评估维度 | L1-L2 | 3分钟 |
| 个人笔记/知识库 | 思维模型、个人方法论 | L3 | 5分钟 |
| 邮件模板/常用话术 | 沟通风格、表达习惯 | L3-L4 | 3分钟 |

**扫描方法**：
1. 打开文档，用搜索功能搜关键词（如"KPI""供应商""流程""指标"）
2. 只看标题、小标题、加粗部分、表格
3. 把发现的术语/流程/指标复制到对应层级的收集表中
4. **不要试图读完全文**——你不是在复习，是在"淘金"

## 四、自检清单（收集完成后对照）

完成以上清单后，用以下问题检查完整性：

**角色清晰度：**
- [ ] 看完这份画像，一个陌生人能大致理解"这个人是干什么的"吗？
- [ ] 我的权限边界写得够清楚吗？AI 会因此误判我能做什么吗？

**术语清晰度：**
- [ ] 每个术语的定义，一个外行能看懂吗？
- [ ] 有没有"我以为大家都知道但其实不是"的术语？

**冲突最小化：**
- [ ] 我有没有写了一些"现在是这样但可能很快会变"的信息？（如果有，标记为"动态"）
- [ ] 我有没有写了一些过于具体的数字/人名/日期？（如果有，删除或泛化）

**偏见暴露：**
- [ ] 我写的"我的职责"是我希望的职责，还是实际在做的职责？（两者不一致是常见陷阱）
- [ ] 我写的"我们的流程"是纸面上的流程，还是实际运行的流程？（后者更重要）

## 六、更新机制

### 何时更新

| 触发条件 | 操作 |
|---------|------|
| 岗位/职责/汇报线变更 | 立即更新 L0-L1 |
| 新增/变更业务术语 | 下次对话时更新 L2 |
| 发现知识库与当前情况冲突 | 以当前情况为准，标记知识库"待更新" |
| 每季度末 | 主动Review一次，清理过时信息 |
| 每次使用 thinking-lenses 后 | 检查是否有新增洞察需要沉淀到 L3 |

### 更新原则

1. **追加而非覆盖**：保留历史版本，新信息追加到变更日志
2. **标记不确定性**：不确定的信息标注"待核实"
3. **删除过时信息**：季度Review时，删除已失效的内容，不要只是标注"已过时"
4. **精简至上**：如果某个信息项连续3个月没被对话引用，考虑删除
