# 项目日志与方法回收

## 概述

使用这个 skill 防止长期策展项目丢失判断过程。它记录会影响后续内容、结构、设计和实现的关键决定。

## 记录什么

记录判断，而不是流水账。应包括：

- 本轮工作背景和测试的问题。
- 被比较的方案。
- 用户或团队反馈。
- 某个方向为什么失败、收窄或被保留。
- 已固定的结论及其适用范围。
- 未解决问题和下一步。
- 可迁移到下一个项目的方法经验。

## 日志结构

采用简洁结构：

- 日期和阶段。
- 工作背景。
- 测试方向。
- 发现的问题。
- 固定结论。
- 余留问题。
- 下一步工作。
- 可复用方法说明。

## 自检问题

每轮复盘时询问：

- 策展对象是否仍然清楚？
- 人物、物件、图像或界面效果是否开始替代真正主体？
- 每种页面类型是否仍有明确角色？
- 文本是否已经能落位到模块？
- 材料是在承担证据、定位、分析还是氛围？
- 视觉和技术选择是否服务阅读路径？
- 哪条规则应被带入下一个项目？

## 输出

输出日志条目、固定结论列表、短自检和可复用方法说明。常规文件修改细节不必记录，除非它影响后续判断。
