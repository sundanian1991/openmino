# 04-研究转策展文本

## 概述

这一阶段做的是翻译工作：把研究稿和原始材料，翻译成有页面功能的字段。不是缩短原文，是重新组织——决定每个字段放什么、承担什么功能。

---

## 输入

- 01-定题的策展地图
- 03-叙事规划的叙事方案
- 用户提供的原始材料

---

## 核心区分

| 类型 | 功能 | 特点 |
|------|------|------|
| **研究稿** | 证明、探索、保存复杂性 | 完整、严谨、允许不确定 |
| **策展文本** | 选择、组织、落位到页面 | 有明确功能、指向观众体验 |

不要只是缩短研究稿。要把它转译成有功能的字段。

---

## 字段模型

每个策展单元（主题/页面/卡片）可以包含以下字段：

| 字段 | 功能 | 必要性 |
|------|------|--------|
| **标题** | 一眼知道这一块讲什么 | 必须 |
| **定位句** | 一句话说明这个单元的价值 | 建议有 |
| **导言** | 开场，设定语境 | 建议有 |
| **入口层** | 观众最先遇到的具体内容（人物、数据、图像、动作） | 必须 |
| **主题层** | 这个单元真正要讲的问题或判断 | 必须 |
| **展开层** | 解释性正文 | 视需要 |
| **证据层** | 数据、图表、引用、来源 | 视需要 |
| **图像提示** | 配图说明（如果需要图） | 可选 |

不需要每个单元都填满所有字段。小型项目可能只有标题+入口层+主题层就够了。

---

## 工作步骤

### 第一步：逐主题转译

对策展地图中的每个主题/子主题，按 03-叙事规划确定的结构，写出策展文本。

**关键原则：**
- 先写标题和定位句。如果写不出来，说明这个主题还没想清楚，退回 01-定题。
- 入口层要具体。"Q1 运营数据概览"是标题，不是入口层。入口层应该是"27 家供应商，1 月到 3 月，首次外呼人力 5174 人"——具体的、可感知的。
- 主题层要有判断。不是"下面是 Q1 数据分析"，而是"Q1 的核心矛盾是规模扩张和效率下降并存"。
- 遇到材料不确定时，保留来源和不确定性，而不是写成无依据的确定叙述。
- 不写占位句（"之后补充资料""这里插入来源"）。已知来源直接写出，存疑处标注为编辑说明。
- 识别研究材料中的核心判断，先决定观众最先遇到什么，再展开。

### 第二步：检查完整性

转译完后，检查：

1. 每个主题都有标题和入口层？
2. 标题之间有没有重复或混淆？
3. 主题层的判断是否和策展地图一致？
4. 证据层是否有支撑？
5. 有没有"之后补充"的占位句？有的话要么补上，要么标注为存疑。

### 第三步：交互确认

把策展文本展示给用户，问：

1. 每个标题和定位句对不对？
2. 入口层是否吸引人？
3. 主题层的判断是否准确？
4. 有没有需要调整的地方？

确认后，进入 05-视觉语言。

---

## 输出

策展文本集，包含每个单元的字段：

```
单元 A1：标题 / 定位句 / 导言 / 入口层 / 主题层 / 展开层 / 证据层
单元 A2：标题 / 定位句 / 入口层 / 主题层
单元 B1：标题 / 定位句 / 导言 / 入口层 / 主题层 / 展开层 / 证据层
```

确认后进入 05-视觉语言。
