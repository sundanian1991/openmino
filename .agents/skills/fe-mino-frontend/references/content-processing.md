# 内容整理方法论

> **用途**：Step 0 内容整理的完整方法论。SKILL.md 只保留触发条件和核心规则，详细步骤在此文件。
>
> **触发**：每次收到材料时自动读取。

---

## 1. 读取材料

**先判断材料形态，再选读取策略**：

| 材料形态 | 读取策略 | 常见坑 |
|---------|---------|--------|
| **完整文章/报告** | 先读首尾段抓主旨，再扫中间找关键数据和结论 | 照搬原文结构，没有提炼 |
| **数据表格/KPI** | 先找最大/最小/异常值，再看列头关系（排名？对比？趋势？） | 平铺所有数据，没有聚焦 |
| **对话记录/聊天** | 提取事实+观点，忽略寒暄和重复 | 把对话原文搬上页面 |
| **清单/列表**（装备/步骤/要求） | 按逻辑分组，删重复项 | 逐条罗列，没有结构 |
| **URL/网页** | 先用 webReader 抓内容，再按文章策略处理 | 没抓到正文，用的是导航文字 |
| **口述/一句话** | 先追问细节（3 个问题以内），确认后才处理 | 脑补内容，偏离用户意图 |

**读取完成后，必须回答**：
1. 这份材料的**核心结论**是什么？（一句话）
2. 读者看完应该**记住什么**？（1-3 个点）
3. 材料里有什么**数据/证据**支撑这个结论？

答不出这三个问题 = 没读懂，不要往下走。

---

## 2. 拆解信息点

把内容拆成**原子信息点**。每个信息点 = 可以独立存在的一个事实/观点/数据/步骤。

**拆解规则**：
- 一个长句里包含多个信息 → 拆成多条
- "毅航产能 3000 万，排名第一，环比增长 15%" → 拆成 3 个信息点
- 修饰词和背景信息 → 保留但标记为 P2/P3

**给每个信息点打标签**：

| 标签 | 判断方法 | 设计表达 |
|------|---------|---------|
| **数据** | 包含具体数字、百分比、排名 | 数字放大、进度条、表格 |
| **洞察** | 包含结论性判断（"所以""说明""意味着"） | 标题、引用块、高亮 |
| **流程** | 有先后顺序（"先/后""步骤""阶段"） | 时间线、步骤条 |
| **清单** | 并列条目，无顺序要求 | 卡片网格、标签组 |
| **对比** | A vs B、前后变化 | 左右分栏、对比表格 |
| **背景** | 前提、上下文、情境 | 小字辅助说明 |

**标签判断示例**：

| 原文 | 拆解 | 标签 |
|------|------|------|
| "毅航 3 月产能 3000 万，排名第一" | 毅航产能 3000 万 / 排名第一 | 数据 / 数据 |
| "头部两家占比超 60%，存在集中风险" | 头部两家占比超 60% / 存在集中风险 | 数据 / 洞察 |
| "先培训，再上岗，最后考核" | 步骤1培训 / 步骤2上岗 / 步骤3考核 | 流程 × 3 |
| "需准备：泳镜、泳帽、泳衣、毛巾" | 泳镜 / 泳帽 / 泳衣 / 毛巾 | 清单 × 4 |
| "刘伟佳表面说管理问题，实际维护定价权" | 表面说管理问题 / 实际维护定价权 | 背景 / 洞察 |

---

## 3. 排优先级

不是所有信息同等重要。强制排 P0-P3：

**判断 P0（核心）的方法**：

| 信号 | 说明 |
|------|------|
| 材料的标题/第一句话 | 通常就是核心 |
| "所以""因此""结论是"后面的内容 | 结论性陈述 |
| 如果删掉这个信息，整个材料失去意义 | 不可删除 = 核心 |
| 数字中最大/最小/异常的值 | 数据型材料的核心焦点 |

**优先级定义**：

| 级别 | 含义 | 处理方式 | 每页配额 |
|------|------|---------|---------|
| **P0** | 这页想让读者记住的 | 用标题或最大视觉元素表达 | 最多 1 个 |
| **P1** | 支撑 P0 的关键证据 | 数据放大、图标、卡片 | 2-4 个 |
| **P2** | 辅助理解的背景信息 | 小字、次要位置 | 按需 |
| **P3** | 删掉不影响理解 | **删掉** | 0 |

**常见错误**：
- ❌ 把所有数据都标 P0 → 没有焦点，全是重点等于没有重点
- ❌ 保留所有原始文字 → 页面拥挤，信息密度失控
- ❌ 自己脑补了材料里没有的洞察 → 只整理，不创造

## 4. 分组与分页

信息点拆好后，先**分组**再**分页**。

**分组逻辑 — 找信息点之间的关系**：

| 关系 | 识别方法 | 分组方式 |
|------|---------|---------|
| **因果** | A 导致了 B | 放同一页，A 在上 B 在下 |
| **并列** | A 和 B 是同级的不同方面 | 放同一页，左右或网格排列 |
| **递进** | A 是 B 的前提，B 是 C 的前提 | 分成不同页，按顺序排列 |
| **总-分** | A 是结论，B/C/D 是支撑 | 一页：P0 做标题，P1 做内容 |
| **时间序列** | A 发生在 B 之前 | 按时间排页，或一页时间线 |

**分页规则**：

| 规则 | 做法 |
|------|------|
| **一页一主题** | 每页只承载 1 个 P0 + 2-4 个 P1 |
| **密度不超载** | 一页视觉块≤7 个 |
| **独立观点独立页** | 两个不相关的 P0 必须分到两页 |
| **≥4 页加封面** | 封面 = 标题 + 副标题 + 元信息 |
| **≥6 页加总结页** | 最后一页：核心结论 + 行动建议 |

**分页数量判断**：

| 信息量 | 页数 | 结构 |
|--------|------|------|
| 1-2 个 P0 | 1-2 页 | 无需封面 |
| 3-5 个 P0 | 3-6 页 | 封面 + 内容页 |
| 6+ 个 P0 | 7+ 页 | 封面 + 目录 + 内容页 + 总结页 |

---

## 5. 整理结果输出

完成 1-4 后，输出整理结果：

```
【内容整理结果】

核心结论：[一句话]

分页计划：
  Page 1（封面）：[标题]
  Page 2：P0=[核心观点] | P1=[支撑点1, 支撑点2] | 版式=[卡片/表格/对比/...]
  Page 3：P0=[核心观点] | P1=[支撑点1, 支撑点2] | 版式=[...]
  ...

删除的 P3 信息：[列出删掉的内容]
```

**检查清单**（不通过不进入 Step 1）：

- [ ] 能用一句话说清每页的核心观点吗？
- [ ] 删掉了所有不影响核心观点的信息吗？（P3 已清除）
- [ ] 每页信息点≤7 个吗？
- [ ] 页与页之间有逻辑关系吗？（因果/递进/并列）
- [ ] 分页数量合理吗？（不挤不空）
- [ ] 没有脑补材料里没有的内容吗？
