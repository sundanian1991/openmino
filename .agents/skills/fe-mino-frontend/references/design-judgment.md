## 一、内容元素识别（Layer 1）

### 判断方法

逐项检查内容，可多选：

| 元素 | 判断关键词 | 确认方法 |
|------|-----------|---------|
| **数据** | 数字、排名、百分比、同比、环比、达成率、KPI | 内容中是否包含可量化的指标 |
| **流程** | 先、后、步骤、阶段、流程、时间线、里程碑 | 内容是否有时间或逻辑顺序 |
| **清单** | 清单、列表、装备、要求、事项、分类 | 内容是否有并列条目需逐项查看 |
| **洞察** | 结论、分析、动机、建议、判断、策略、原因 | 内容是否有需要思考的观点 |

### 组合示例

| 实际内容 | 包含元素 | 主导元素 |
|---------|---------|---------|
| 供应商 KPI 排名表 | 数据 | 数据 |
| 一天训练计划 | 流程 | 流程 |
| 装备清单 + 注意事项 | 清单 + 洞察 | 清单 |
| 刘乾坤行为模式 + 动机分析 | 流程 + 洞察 | 洞察 |
| 赛马结果 + 切量方案 | 数据 + 流程 | 数据 |
| 安全禁止事项 | 清单 | 清单（紧迫情绪下） |

## 二、情绪气质判断（Layer 2）

### 判断依据

**通过内容语义和关键词判断**：

| 气质 | 触发关键词 | 典型场景 |
|------|-----------|---------|
| **紧迫** | 禁止、危险、风险、异常、警告、必须、立即 | 安全警告、风险预警、合规要求 |
| **中性** | 清单、步骤、流程、数据、说明、计划 | 装备清单、训练计划、操作指南 |
| **庆祝** | 成功、达成、里程碑、成果、进步、完成 | 成功标准、业绩成果、目标达成 |
| **沉思** | 洞察、为什么、本质、判断、结论、观点 | 战略判断、动机分析、一句话结论 |

### 情绪 × 视觉参数速查

| 气质 | 留白 | 字号 | 边框 | 陶土色用量 | 图形风格 |
|------|------|------|------|-----------|---------|
| **紧迫** | 10-20% | 正常或偏小 | 粗、实线 | 可多处（每项标记） | 锐利、实心 |
| **中性** | 30-40% | 标准 | 细线、1px | ≤2 处 | 简洁、空心 |
| **庆祝** | 50-60% | 偏大 | 无或极细 | ≤1 处 | 圆润、柔和 |
| **沉思** | 60%+ | 大 | 无 | ≤1 处 | 极简、单色 |

### 边界情况

| 情况 | 判断方法 | 举例 |
|------|---------|------|
| 同一页内有紧迫又有庆祝 | 以**主要情绪**为准 | 风险清单（紧迫为主，底部成果是次要） |
| 内容语气不明确 | 默认**中性** | 流程说明既不紧急也不庆祝 |
| 封面/目录页 | **中性**，偏向内容的整体气质 | 游泳训练 = 中性偏积极 |

## 三、阅读方式判断（Layer 3）

### 判断依据

**问自己：读者拿到这页，第一反应是什么？**

| 方式 | 读者行为 | 判断方法 |
|------|---------|---------|
| **扫视** | "我要找 XX 在哪" | 读者带着目的来，快速定位 |
| **阅读** | "让我看看这页说的什么" | 读者从头到尾线性理解 |
| **凝视** | "嗯，让我想想" | 读者停下来思考、消化 |

### 速查表

| 内容类型 | 通常是 | 但也可能是 |
|---------|--------|-----------|
| 排名表/数据表 | 扫视 | — |
| 装备清单 | 扫视 | — |
| 操作步骤 | 阅读 | — |
| 流程说明 | 阅读 | — |
| 核心结论 | 凝视 | — |
| 行为模式分析 | 阅读 | 凝视（如果结论很深） |
| 成功标准 | 凝视 | 阅读（如果需要逐项确认） |

## 四、密度与版式决策（Layer 4）

### 密度查表（情绪 × 阅读方式）

| | 扫视 | 阅读 | 凝视 |
|---|------|------|------|
| **紧迫** | 高密度（紧凑表格） | 中高密度（紧凑列表） | —（紧迫不配凝视） |
| **中性** | 中高密度（网格/表格） | 标准密度（正常列表） | 中低密度（留白 + 小焦点） |
| **庆祝** | —（庆祝不配扫视） | 中低密度（宽间距） | 低密度（大留白 + 可视化） |
| **沉思** | —（沉思不配扫视） | 低密度（宽留白） | 极低密度（全可视化或金句） |

### 版式查表（结构关系）

**先判断内容元素之间的结构关系**：

| 关系特征 | 版式 | 判断方法 |
|---------|------|---------|
| 行列结构（行=实体，列=属性） | **表格** | 有明确的"表头"维度 |
| 对立/对比（A vs B） | **左右对比** | 内容天然分两阵营 |
| 多个并列（独立但平等） | **卡片网格** | 3-6 个独立模块 |
| 线性展开（1→2→3→4） | **标准列表** | 单一主题逐步展开 |
| 时间顺序（阶段/里程碑） | **时间线** | 有明确的时间节点 |
| 层级关系（总→分→子） | **树形/缩进** | 有层级嵌套 |
| 因果关系（A 导致 B） | **流程图/因果图** | 有箭头/因果逻辑 |

### 组合示例

| 页面 | 情绪 | 阅读 | 密度 | 结构 | 版式 |
|------|------|------|------|------|------|
| 安全禁止事项 | 紧迫 | 扫视 | 高 | 对立（禁止 vs 必须） | 左右对比 |
| 装备清单 | 中性 | 扫视 | 中高 | 并列（必备 vs 建议） | 双列表格 |
| 训练计划 | 中性 | 阅读 | 标准 | 时间顺序 | 时间线 |
| 常见错误 | 紧迫 | 阅读 | 中高 | 对比（错误 vs 纠正） | 对比表格 |
| 成功标准 | 庆祝 | 凝视 | 低 | 并列 + 递进 | 卡片 + 可视化 |
| 核心洞察 | 沉思 | 凝视 | 极低 | 单一焦点 | 金句页 / 关系图 |

## 五、可视化叠加（Layer 5）

### 渐进级别详解

#### Level 1：微点缀（5-10%）

**在不改变文字布局的前提下，加入最轻量的视觉标记。**

| 手段 | CSS 实现 | 适用场景 |
|------|---------|---------|
| 色块标记关键数字 | `background: var(--accent); color: white; padding: 2px 6px;` | 表格中标记异常值 |
| 条件格式 | `border-left: 3px solid var(--accent);` | 列表中标记重点行 |
| 数字放大 | `font-size: 2em; font-weight: 700; color: var(--accent);` | 核心数据突出 |
| 小图章/徽章 | `background: var(--accent); border-radius: 50%; width: 24px;` | 状态标记（完成/未完成） |

**示例**：装备清单表格中，"必备"行左侧加陶土色竖线，"建议"行左侧加灰色竖线。

#### Level 2：轻混合（20-30%）

**在文字布局中加入小型可视化元素，帮助快速识别。**

| 手段 | 实现方式 | 适用场景 |
|------|---------|---------|
| 手绘 SVG 图标 | `stroke: var(--accent); stroke-width: 2; stroke-linecap: round;` | 装备类别标识 |
| 进度条 | `height: 4px; background: linear-gradient(...);` | 目标完成度 |
| inline 标签 | `background: var(--accent-light); border-radius: 2px; padding: 2px 8px;` | 分类标签 |
| 小型状态指示器 | 圆点 + 文字 | 在线/离线、正常/异常 |

**示例**：装备清单中，每项装备前加手绘 SVG 图标（泳镜=两个圆圈+连线）。

#### Level 3：中混合（40-50%）

**文字和可视化各占一半，互相补充。**

| 手段 | 实现方式 | 适用场景 |
|------|---------|---------|
| 时间线 | 左侧时间节点 + 右侧内容卡片 | 训练计划、项目进度 |
| 矩阵图 | 2x2 或 3x3 网格 + 象限标注 | 优先级矩阵、波士顿矩阵 |
| 卡片墙 | 等宽卡片网格 + 分类标题 | 装备清单、特性对比 |
| 进度/状态组 | 多个进度条 + 标签 | 多维度达标情况 |

**示例**：训练计划用纵向时间线，左侧时段标记（上午/下午/晚上），右侧内容卡片。

#### Level 4：重混合（60-70%）

**可视化为主，文字只做标注和补充。**

| 手段 | 实现方式 | 适用场景 |
|------|---------|---------|
| 流程图 | SVG 节点 + 连线 + 箭头 | 审批流程、决策流程 |
| 关系图 | 节点 + 连线 + 力导向布局 | 人物关系、组织结构 |
| 泳道图 | 水平泳道 + 流程节点 | 跨部门流程 |
| 架构图 | 层级方块 + 连接线 | 系统架构、能力模型 |

**示例**：刘乾坤行为模式分析用关系图，中心是"刘乾坤"，外圈是行为模式，连线标注触发条件。

#### Level 5：全可视化（80-90%）

**几乎全图，文字只保留标题和最少的标注。**

| 手段 | 实现方式 | 适用场景 |
|------|---------|---------|
| 技能树 | 中心节点 + 分支 + 叶节点 | 能力成长路径 |
| 里程碑图 | 路径 + 标记点 + 距离标注 | 项目进度、学习进度 |
| 信息图 | 多种图表组合 + 叙事线 | 综合报告、年度总结 |
| 概念图 | 中心概念 + 辐射关联 | 核心洞察、战略全景 |

**示例**：成功标准用技能树，中心是"学会游泳"，三个分支是三个里程碑，底部虚线延伸到后续计划。

### 可视化级别由主导元素决定

| 主导元素 | 推荐级别 | 理由 |
|---------|---------|------|
| 数据 | 微点缀 → 轻混合 | 数据本身是焦点，可视化只做增强 |
| 流程 | 中混合 → 重混合 | 流程天然适合可视化 |
| 清单 | 轻混合 → 中混合 | 清单需要快速识别，图标/卡片有帮助 |
| 洞察 | 重混合 → 全可视化 | 洞察需要视觉隐喻来帮助理解 |

### 情绪对可视化风格的影响

| 情绪 | 可视化风格 | 具体表现 |
|------|-----------|---------|
| **紧迫** | 锐利、高对比 | 直线、实心填充、粗边框、密集排列 |
| **中性** | 简洁、平衡 | 细线、空心、正常间距、对称布局 |
| **庆祝** | 柔和、圆润 | 圆角、渐变底色、大间距、曲线 |
| **沉思** | 极简、大量留白 | 细线、单色、中心对称、大留白 |

## 5.5 空白激活决策

> **核心问题**：版式选对了，但页面有大量"剩余空白"，看起来空洞、不专业。
>
> **核心论点**：空白不是"剩余"的，是"设计"的。需要用元素激活空白，让它变成"设计留白"而非"空洞空白"。

### 空白类型诊断

**问自己：这个空白是什么性质？**

| 空白类型 | 识别方法 | 问题 |
|---------|---------|------|
| **结构性空白** | 版式决定的位置（左右对比中间的空） | ✅ 不需要激活，保持呼吸感 |
| **内容稀疏空白** | 内容量不足以填满版式 | ❌ 需要激活或换版式 |
| **边缘空白** | 固定 padding 导致边缘过大 | ❌ 需要响应式 padding |
| **底部空白** | 内容结束后的剩余空间 | ❌ 需要填充元素 |

### 空白激活决策表

**根据空白类型选择激活元素**：

| 空白类型 | 激活元素 | 实现方式 | 示例 |
|---------|---------|---------|------|
| **内容稀疏** | 进度条 | 顶部 3px 陶土色条，跟随滚动 | 刘乾坤分析 |
| **内容稀疏** | 徽章系统 | high/medium/low 三色徽章 | 风险评估表 |
| **内容稀疏** | 装饰分割线 | 2px 陶土色线 | 总结页 |
| **边缘空白** | 响应式 padding | `clamp(1.5rem, 4vw, 3rem)` | 所有页面 |
| **底部空白** | 底部信息条 | 深色色块 + 关键信息 | 目录页 |
| **底部空白** | 导航提示 | "滚动继续" + 箭头 | 封面页 |
| **中间空白（对比版式）** | 判定区 | VS 徽章 + 箭头 + 结果标签 | 权责对比 |

### 激活元素详解

#### 1. 进度条（顶部装饰线）

**触发条件**：
- 页面数 ≥ 3 页
- 内容有线性阅读顺序
- 不适用于单页或非线性内容

**实现方式**：
```css
.progress {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  height: 3px;  /* 不是 2px，不是 4px */
  background: transparent;
  z-index: 100;
}
.progress-bar {
  height: 100%;
  background: var(--accent);
  width: 0%;
  transition: width 0.2s;
}
```

**为什么是 3px**：
- 2px 太细，滚动时看不清
- 4px 太粗，抢占视觉焦点
- 3px 刚好：存在感足够，不抢焦点

#### 2. 徽章系统

**触发条件**：
- 表格中有"等级/状态/风险"字段
- 需要快速识别优先级

**实现方式**：
```css
.badge-high {
  background: rgba(214, 51, 51, 0.1);
  color: #d63333;
}
.badge-medium {
  background: rgba(245, 158, 11, 0.1);
  color: #f59e0b;
}
.badge-low {
  background: rgba(37, 99, 235, 0.1);
  color: #2563eb;
}
```

**配色原则**：
- 不用陶土色（陶土色留给焦点）
- 用语义色：红=高风险，黄=中，蓝=低

#### 3. 装饰分割线

**触发条件**：
- 需要强调章节边界
- 标题下方需要视觉锚点

**实现方式**：
```css
.divider-line {
  width: 100%;
  height: 2px;  /* 不是 1px */
  background: var(--border);
}
.divider-line.accent {
  background: var(--accent);  /* 陶土色强调线 */
}
```

**为什么是 2px**：
- 1px 在高分辨率屏幕上看不清
- 2px 有存在感但不抢焦点

#### 4. 响应式 padding

**触发条件**：
- 所有页面（默认）

**实现方式**：
```css
--page-padding: clamp(1.5rem, 4vw, 3rem);  /* 24-48px */
```

**为什么不是固定 56px**：
- 固定值在小屏幕上太宽，内容被挤压
- 固定值在大屏幕上可能不够宽
- clamp 让空白自适应屏幕

#### 5. 底部信息条

**触发条件**：
- 目录页、概览页底部有大面积空白
- 有需要强调的总结信息

**实现方式**：
```css
.bottom-bar {
  background: var(--text-primary);  /* 深色色块 */
  color: white;
  padding: 1rem 1.5rem;
  margin-top: auto;  /* 推到底部 */
}
```

**效果**：
- 填充底部空白
- 深色色块比细线条更有"重量感"
- 内容更充实

### 空白激活检查清单

- [ ] 页面是否有进度条？（≥3 页时必须有）
- [ ] 表格是否有徽章系统？（有等级字段时必须有）
- [ ] 分割线是否够粗？（2px，不是 1px）
- [ ] padding 是否响应式？（clamp，不是固定值）
- [ ] 底部空白是否填充？（信息条或导航提示）
- [ ] 对比版式中间是否有内容？（判定区，不是空白）

## 5.6 可视化形式选择：底层原理驱动

> **定位**：Layer 5 的 5 个级别解决"用多少可视化"，本节解决"用什么形式的可视化"。
>
> **一个核心论点**：可视化形式由底层原理决定，不是经验选择。
>
> **结构**：
> 1. 底层原理（前注意特征编码 + 格式塔原理）
> 2. 预计算速查表（7 条映射 = 底层原理的推导结果，每条附证明）
> 3. 4 步推导流程（遇到速查表未覆盖的新场景时，用底层原理从零推导）

### 一、底层原理 A：前注意特征编码

> **核心论点**：人类视觉系统在意识参与之前（200-500ms），就能自动提取某些视觉特征。设计师的任务是选择正确的视觉变量来编码正确的信息。

#### 四大前注意特征通道

| 通道 | 可编码的视觉变量 | 适合表达的信息属性 | 克制专业主义的实现 |
|------|---------------|-----------------|----------------|
| **大小** | 高度、宽度、面积 | 重要性等级、数量级、优先级 | 字号层级（T1-T6）、卡片面积差异 |
| **位置** | 2D 坐标（点/线/盒） | 类别归属、时序、空间关系 | 网格定位、时间线位置、表格行列 |
| **色调** | 色相（单色/多色）、饱和度、明度 | 类别区分、异常标记、状态 | 陶土色=焦点，灰色层次=层级，黑白=基线 |
| **强度/形态** | 线条粗细、填充/空心、实/虚线 | 确定性、状态、边界 | 粗线=确定/强调，细线=辅助，虚线=关联/可选 |

#### 编码原则（3 条铁律）

**铁律 1：一个视觉变量只编码一个信息属性**

| ✅ 正确 | ❌ 错误 |
|--------|--------|
| 字号只编码层级（T1=页标题，T2=模块标题） | 字号既表示层级又表示紧急程度 |
| 陶土色只编码焦点（异常/风险/行动） | 陶土色同时用于标题强调 + 图标填充 + 背景底色 |
| 粗细只编码确定性（实线=确定，虚线=可选） | 边框粗细随机变化 |

**铁律 2：最稀缺的视觉变量留给最重要的信息属性**

克制专业主义中，陶土色是最稀缺资源 → 只用于最高优先级的信息（P0 异常/P1 排名/P2 行动建议）。

**铁律 3：同类信息用同一通道编码**

| 信息类型 | 编码通道 | 示例 |
|---------|---------|------|
| 所有层级关系 | 大小通道 | T1 > T2 > T3 > T4 |
| 所有焦点标记 | 色调通道 | 陶土色标记 |
| 所有关联关系 | 强度通道 | 虚线 = 关联，实线 = 确定 |

#### 编码决策流程

```
第一步：识别需要编码的信息属性
├── 这页有哪些信息需要视觉区分？
├── 例：层级（标题/正文/注释）
├── 例：焦点（哪个最重要）
├── 例：状态（正常/异常/建议）
└── 例：结构（并列/对比/层级）

第二步：为每个属性分配视觉通道
├── 最重要的属性 → 最稀缺的通道（色调 = 陶土色）
├── 第二重要的属性 → 次稀缺的通道（大小 = 字号层级）
├── 第三重要的属性 → 充足的通道（位置 = 布局定位）
└── 辅助属性 → 最充足的通道（强度 = 线条粗细/虚实）

第三步：检查通道冲突
├── 是否有两个属性用了同一个通道？
├── 如果有 → 改其中一个到其他通道
└── 例：层级和焦点都用字号 → 改焦点为色调（陶土色）
```

#### 编码示例

**示例 1：安全禁止事项**

| 信息属性 | 优先级 | 分配通道 | 具体实现 |
|---------|--------|---------|---------|
| 禁止 vs 必须 | P0 | 色调 | 禁止=陶土色标记，必须=深灰 |
| 条目层级 | P1 | 大小 | 标题 T2，正文 T4 |
| 安全等级 | P2 | 强度 | 绝对禁止=粗实线，一般注意=细实线，建议=虚线 |
| 位置关系 | P3 | 位置 | 左侧禁止，右侧必须 |

**示例 2：刘乾坤总结页**

| 信息属性 | 优先级 | 分配通道 | 具体实现 |
|---------|--------|---------|---------|
| 金句 vs 要点 | P0 | 大小 | 金句 1.5rem 加粗，要点 0.85rem 常规 |
| 金句是焦点 | P1 | 色调 | 金句用陶土色 |
| 4 要点并列 | P2 | 位置 | 2x2 网格等面积 |
| 要点内结构（标题 vs 内容） | P3 | 强度 | 标题 font-weight:600，内容 font-weight:400 |

### 二、预计算速查表（内在结构 → 形式映射 + 底层原理证明）

> **定位**：7 条常见映射，每条都能用底层原理（前注意特征 + 格式塔）推导出来。
> 不是经验速查，是底层原理的"预计算结果"。日常直接查表，需要时回溯证明理解"为什么"。

| 内在结构 | 映射形式 | 识别方法 | 前注意特征 | 格式塔原理 |
|---------|---------|---------|-----------|-----------|
| **行列** | 表格 | 有明确的"表头"维度 | 位置通道：行列坐标编码实体×属性交叉 | 围合律(表头分组) + 相似律(同列对齐) |
| **对立/对比** | 左右对比 | 内容天然分两阵营 | 位置通道：左 vs 右编码对立 | 相近律(两侧各自聚合，中间间距大=断裂) |
| **并列** | 卡片网格 | 3-6 个独立等权模块 | 位置(网格坐标) + 大小(等面积=等权) | 围合律(边框=一个单元) + 相似律(等宽=同类) |
| **线性** | 标准列表 | 单一主题逐步展开 | 位置(纵向)编码顺序 + 大小编码层级 | 连续律(纵向对齐) + 相似律(统一圆点) |
| **时间** | 时间线 | 有明确的时间节点 | 位置(纵向坐标)编码时序 | 连续律(共享竖线=连续) + 围合律(节点卡片) |
| **层级** | 树形/缩进 | 有嵌套关系 | 位置(缩进深度)编码层级 + 大小编码重要性 | 相似律(同级同缩进) + 围合律(子集含于父集) |
| **因果** | 流程图/因果图 | 有箭头/因果逻辑 | 位置(箭头方向)编码因果方向 | 连续律(箭头线=因果链) + 围合律(节点框) |

**用法**：当你能明确识别结构关系时，直接映射。需要理解"为什么是这个形式"时，回溯对应的底层原理列。

**速查表未覆盖的场景**（需要用 §四 的 4 步推导流程从零推导）：
- 等权并列但内容量差异大（如刘乾坤总结页 4 个要点长短不一）
- 多种结构混合（同一页有对比 + 流程）
- 需要选择"表格 vs 卡片"这种同构选择时

### 三、底层原理 B：格式塔原理

> **核心论点**：人类视觉系统自动将元素组织为有意义的整体。设计师利用这些组织规律，引导读者"自动看到"正确的结构和关系。

#### 四大核心格式塔原理

| 原理 | 含义 | 设计应用 | 克制专业主义实现 |
|------|------|---------|----------------|
| **围合律** | 被边界包围的元素被视为一组 | 卡片、色块区域、边框分组 | 1px 边框卡片 = 一个信息单元 |
| **连续律** | 沿直线或曲线排列的元素被视为整体 | 时间线、流程线、对齐线 | 左侧竖线时间线 = 阶段连续 |
| **相近律** | 距离更近的元素被视为一组 | 间距决定分组，大间距=分组边界 | 卡片间距 > 内部间距 = 卡片是一组 |
| **相似律** | 外观相似的元素被视为一类 | 颜色、字号、样式一致性 | 同字号同色 = 同层级，陶土色 = 同类焦点 |

#### 格式塔应用决策

**问自己：我需要读者看到什么样的分组/关系？**

| 需要表达的关系 | 使用的格式塔原理 | CSS 手段 |
|-------------|---------------|---------|
| 这些是一组 | 围合律 | `border: 1px solid #e5e5e5; padding: 1rem;` |
| 这些是同类的 | 相似律 | 统一的 `font-size`、`color`、`font-weight` |
| 这两组是分开的 | 相近律 | 组间距 `gap: 1.5rem` >> 组内间距 `gap: 0.5rem` |
| 这些是连续的流程 | 连续律 | 共享的左侧竖线 `border-left: 3px solid` |
| 这个是特殊的 | 围合律 + 色调差异 | 陶土色边框 `border: 2px solid var(--accent); background: var(--accent-light);` |

#### 格式塔冲突检查

**常见问题：格式塔原理互相打架**

| 冲突 | 表现 | 解决 |
|------|------|------|
| 相近律 vs 围合律 | 卡片间距太近，看不出分组 | 加大卡片间距，或增加卡片边框 |
| 相似律 vs 焦点需求 | 所有卡片同色，看不出哪个重要 | 用陶土色打破相似性，制造"异类" |
| 连续律 vs 并列关系 | 时间线样式用在并列内容上 | 并列用网格，不用竖线连接 |

**诊断方法**：眯起眼看页面，第一秒看到什么分组？这就是格式塔在起作用。如果看到的分组和内容的真实结构不一致，就说明格式塔用错了。

### 四、4 步推导流程（底层原理的展开形式）

> **定位**：速查表 §二 是"预计算结果"，这个流程是"推导过程"。
> - 日常用速查表直接映射（因为底层原理已经帮你算好了）
> - 遇到速查表未覆盖的场景，用这个流程从零推导
> - 也可以用来验证某个形式选择是否正确（回溯证明）

```
Step 1：识别编码对象
├── 这页有哪些信息属性需要视觉编码？
├── 按重要性排序（P0 > P1 > P2 > P3）
└── 输出：编码对象列表 + 优先级

Step 2：前注意特征编码
├── 为每个编码对象分配视觉通道
├── 检查：一个通道只编码一个属性？
├── 检查：最稀缺的通道（陶土色）给了最重要的属性？
└── 输出：视觉通道分配表

Step 3：格式塔组织布局
├── 读者应该看到什么样的分组？
├── 用围合律/相近律建立分组边界
├── 用连续律/相似律建立组内关系
├── 检查：眯眼测试 → 看到的分组和内容结构一致？
└── 输出：布局方案

Step 4：情绪调节风格参数
├── 根据 Layer 2 的情绪气质，调节：
│   ├── 线条粗细（紧迫=粗，沉思=细）
│   ├── 填充方式（紧迫=实心，沉思=空心）
│   ├── 间距大小（紧迫=紧凑，庆祝=宽松）
│   └── 陶土色用量（紧迫=可多处，沉思=≤1处）
└── 输出：最终视觉参数
```

#### 4 步流程验证示例

**验证：刘乾坤总结页（V3 成功版本）**

| Step | 决策 | 结果 |
|------|------|------|
| **S1 编码对象** | ① 金句（P0）② 4 要点的并列关系（P1）③ 要点内标题 vs 内容（P2） | 3 个编码对象 |
| **S2 前注意特征** | ① 金句→大小（1.5rem bold）+色调（陶土色）② 并列→位置（2x2 等面积网格）③ 标题 vs 内容→强度（600 vs 400 weight） | 无通道冲突 |
| **S3 格式塔** | 4 卡片用围合律（1px 边框）+相近律（组间 gap:2rem >> 组内紧密），金句与卡片用大间距（3.5rem）制造分组断裂 | 眯眼测试：看到 4 个方块 + 底部一行红字 ✓ |
| **S4 情绪调节** | 沉思气质 → 细线 1px、大留白 60%+、陶土色≤1 处（金句）、空心卡片 | 参数匹配 ✓ |

**反例：V2 关系图为什么失败？**

| Step | 问题 |
|------|------|
| **S1** | 编码对象是 4 个等权要点，没有"中心-外围"关系 |
| **S2** | 用了位置通道强制制造"中心节点=刘乾坤"的视觉权重，但 4 个要点实际等权 |
| **S3** | 格式塔连续律（连接线）暗示了因果关系，但 4 个要点是并列关系。原理和内容结构冲突 |
| **结论** | 形式和内容的结构关系不匹配 → V2 失败 |

## 六、焦点选择与表达（Layer 6）

### 6.1 焦点优先级

| 优先级 | 适用场景 | 示例 |
|--------|---------|------|
| **P0** 异常/风险 | 下滑、超标、警告、禁止 | 业绩下滑的供应商行 |
| **P1** 排名/等级 | 第一/最后、A/C 级 | KPI 排名第一 |
| **P2** 行动建议 | 底部卡片、话术、下一步 | 底部行动建议卡片 |

### 6.2 焦点表达形式选择

> **核心问题**：焦点确定后，用什么视觉形式表达？
>
> **核心论点**：表达形式由焦点类型决定，不是随机选择。

**焦点类型 → 表达形式映射**：

| 焦点类型 | 推荐形式 | 次选形式 | 不推荐 | 示例 |
|---------|---------|---------|--------|------|
| **异常值（表格内）** | 条件格式（左侧色条） | 徽章 | 背景色填充 | 风险供应商行 |
| **等级标记** | 徽章（高/中/低） | — | 背景色填充 | 风险评估表 |
| **排名突出** | 数字放大 + 陶土色 | 徽章 | 边框 | KPI 排名第一 |
| **核心结论** | 金句样式（大字号 + 陶土色） | 卡片内强调 | 徽章 | 总结页金句 |
| **行动建议** | 陶土色边框卡片 | 深色背景卡片 | 徽章 | 底部行动卡片 |
| **对比结果** | 判定区（结果标签） | 陶土色边框 | 徽章 | 权责对比 |

### 6.3 表达形式详解

#### 形式 1：条件格式（表格内）

**适用场景**：标记表格中的重点行

**实现方式**：
```css
tr.highlight td {
  border-left: 3px solid var(--accent);
  background: var(--accent-light);
}
```

**特点**：
- 不打断表格整体结构
- 左侧色条醒目但不抢眼
- 适用于多行标记

#### 形式 2：徽章（Badge）

**适用场景**：等级、状态、风险标记

**实现方式**：
```css
.badge {
  display: inline-block;
  padding: 0.2rem 0.5rem;
  font-size: 0.7rem;
  font-weight: 600;
  border-radius: 2px;  /* 锐利，不是圆润 */
}
```

**颜色规则**：
- 高风险：红 `#d63333`
- 中风险：黄 `#f59e0b`
- 低风险：蓝 `#2563eb`
- **不用陶土色**（陶土色留给 P0 焦点）

#### 形式 3：数字放大 + 陶土色

**适用场景**：单个核心数据突出

**实现方式**：
```css
.big-number {
  font-size: 2em;
  font-weight: 700;
  color: var(--accent);
}
```

**注意**：
- 只用于单个数字，不用于文字
- 放大比例 ≤ 2em，再大就失衡

#### 形式 4：金句样式

**适用场景**：总结页核心结论

**实现方式**：
```css
.key-insight {
  font-size: 1.5rem;
  font-weight: 700;
  color: var(--accent);
  line-height: 1.4;
}
```

**特点**：
- 字号比正文大 50%+
- 陶土色强调
- 通常单独一行，有呼吸感

#### 形式 5：陶土色边框卡片

**适用场景**：底部行动建议、核心判断

**实现方式**：
```css
.card.highlight {
  border: 1px solid var(--accent);
  background: var(--accent-light);
}
```

**特点**：
- 边框 1px（不是 2px，太粗抢焦点）
- 背景用 8% 透明度的陶土色（不是纯色）
- 适用于单个卡片

#### 形式 6：判定区

**适用场景**：左右对比版式的中间区域

**实现方式**：
```css
.compare-center {
  display: flex;
  flex-direction: column;
  justify-content: center;
  align-items: center;
  gap: 12px;
}
.vs-badge {
  width: 32px;
  height: 32px;
  border-radius: 50%;
  background: var(--text-primary);
  color: white;
}
.result-badge {
  background: var(--accent-light);
  border: 1px solid var(--accent);
  padding: 8px 12px;
}
```

**特点**：
- 填充左右对比中间的空白
- VS 徽章 + 箭头 + 结果标签
- 视觉上有"判定"感

### 6.4 焦点表达检查清单

- [ ] 焦点类型识别正确？（异常/等级/排名/结论/行动/对比）
- [ ] 表达形式匹配焦点类型？（查上表）
- [ ] 陶土色只用在正确位置？（P0/P1/P2）
- [ ] 徽章颜色是否语义正确？（红/黄/蓝，不是陶土色）
- [ ] 一页是否只有一处陶土色焦点？（紧迫情绪可放宽到 2-3 处）

### 6.5 使用规则

- 一页一处陶土色强调（紧迫情绪下可放宽到 2-3 处）
- 不用于装饰（分隔线、背景色块、图标填充）
- 不用于正文（只用于标题、数据、标签）
- 微点缀级别的陶土色标记不受"一页一处"限制（如表格内条件格式）

### 灰色层次使用

| 用途 | 色值 | 使用场景 |
|------|------|---------|
| 主标题 | #111111 | 页面标题、卡片标题 |
| 辅助文字 | #666666 | 正文、描述、次要信息 |
| 浅灰 | #999999 | 日期、页码、注释 |
| 边框/分割 | #E5E5E5 | 卡片边框、表格分割线 |
| 底色 | #FAFAFA | 底部卡片背景、表头背景 |

## 八、克制专业主义参数系统

> **核心论点**：方法论是骨架，参数系统是血肉。骨架相同但血肉不同，输出就不稳定。
>
> **三层参数模型**：
> - Layer 1：全局参数（所有风格共享）
> - Layer 2：风格参数（每种风格独有）
> - Layer 3：内容参数（由内容特征决定）

### 8.1 全局参数（所有风格共享）

#### 色彩系统

```css
:root {
  /* 文字层级 */
  --text-primary: #111111;
  --text-secondary: #666666;
  --text-tertiary: #999999;

  /* 背景 */
  --bg: #FFFFFF;
  --surface: #FAFAFA;

  /* 边框 */
  --border: #E5E5E5;

  /* 陶土色（焦点色） */
  --accent: #E2725B;
  --accent-light: rgba(226, 114, 91, 0.08);
}
```

#### 间距基准（4pt 系统）

```css
:root {
  --space-xs: 4px;   /* 元素内部最小间距 */
  --space-sm: 8px;   /* 相关元素间距 */
  --space-md: 12px;  /* 分组内部间距 */
  --space-lg: 16px;  /* 分组之间间距 */
  --space-xl: 24px;  /* 区块之间间距 */
  --space-2xl: 32px; /* 大区块之间间距 */
  --space-3xl: 48px; /* 页面边缘间距 */
}
```

### 8.2 克制专业主义风格参数

> **基准案例**：刘乾坤带节奏分析（年老师满意的版本）

#### 字体层级系统

| 层级 | 用途 | clamp 值 | 基准值 |
|------|------|----------|--------|
| **T1a 封面标题** | 封面页大标题 | `clamp(2.2rem, 5.5vw, 3.5rem)` | 35-40px |
| **T1b 页面标题** | 内页标题 | `clamp(1.8rem, 4.5vw, 2.8rem)` | 28-30px |
| **T2 模块标题** | 区块标题 | `clamp(1rem, 2vw, 1.3rem)` | 16-18px |
| **T3 卡片标题** | 卡片/表格标题 | `clamp(0.85rem, 1.3vw, 1rem)` | 14-15px |
| **T4 正文** | 正文内容 | `clamp(0.75rem, 1.2vw, 0.9rem)` | 12-13px |
| **T5 辅助** | 辅助说明 | `clamp(0.7rem, 1vw, 0.8rem)` | 11-12px |
| **T6 标签** | 页码/标签 | `clamp(0.6rem, 0.8vw, 0.7rem)` | 10px |

**关键修正点**：
- 封面标题：35-40px（不是 28px）
- 层级差：T1a → T1b 约 10px，T1b → T2 约 12px

#### 间距系统

| 参数 | clamp 值 | 基准值 | 说明 |
|------|----------|--------|------|
| **页面 padding** | `clamp(1.5rem, 4vw, 3rem)` | 24-48px | 上限 48px（不是 32px） |
| **卡片 padding** | `clamp(1rem, 2vw, 1.5rem)` | 16-24px | 响应式 |
| **卡片 gap** | `clamp(1rem, 2vw, 1.5rem)` | 16-24px | 响应式 |
| **表格表头 padding** | `0.5rem 0.75rem` | 8px 12px | 不是 12px |
| **表格单元格 padding** | `0.6rem 0.75rem` | 10px 12px | |

**关键修正点**：
- 页面 padding 上限：48px（不是 32px）
- 表头 padding：8px 12px（不是 12px）

#### 边框系统

| 参数 | 值 | 说明 |
|------|-----|------|
| **卡片边框** | 1px solid #E5E5E5 | 标准边框 |
| **表头分割线** | 2px solid #111111 | 粗黑线 |
| **分割线高度** | 2px | 不是 1px |
| **进度条高度** | 3px | 不是 2px 或 4px |
| **卡片圆角** | 2px | 锐利（不是 4px） |
| **强调边框** | 1px solid var(--accent) | 陶土色 |

**关键修正点**：
- 圆角：2px（不是 4px）— 更锐利
- 分割线：2px（不是 1px）— 更明显
- 进度条：3px — 刚好可见

#### 陶土色使用规则

| 场景 | 实现方式 | 限制 |
|------|---------|------|
| **进度条** | 顶部 3px，跟随滚动 | ≤ 5px 高 |
| **分割线强调** | `.divider-line.accent` 2px | 非封面页可用 |
| **关键词强调** | `color: var(--accent); font-weight: 600;` | 仅关键词 |
| **徽章** | **不用陶土色** | 用语义色（红/黄/蓝） |
| **焦点卡片** | `border: 1px solid var(--accent); background: var(--accent-light);` | 一页一处 |

### 8.3 内容参数（由内容特征决定）

#### 密度调整

| 情绪 × 阅读 | 密度 | 页面 padding | 卡片间距 |
|------------|------|-------------|---------|
| 紧迫 × 扫视 | 高密度 | 24px | 12px |
| 中性 × 阅读 | 标准密度 | 24-32px | 16px |
| 沉思 × 凝视 | 低密度 | 24-48px | 24px |

#### 陶土色用量

| 情绪 | 陶土色使用量 |
|------|------------|
| 紧迫 | 2-3 处（密集标记） |
| 中性 | 1 处（标准焦点） |
| 庆祝 | 1 处（大字号金句） |
| 沉思 | 1 处（极简焦点） |

### 8.4 执行检查清单

- [ ] 所有页面使用相同的 CSS 变量定义
- [ ] T1-T6 字号值在所有文件中一致
- [ ] 页面 padding 使用 clamp（不是固定值）

- [ ] 圆角使用 2px（不是 4px）

- [ ] 一页只有一处陶土色焦点？
