# 手绘 SVG 标准化工作流程

> 算法生成基础结构 + 人工微调关键细节
> 目标：结合算法效率和人工审美

---

## 核心理念

**算法 70% + 人工 30%**

- 算法负责：基础结构、坐标计算、色彩填充、线条分层
- 人工负责：控制点间距、小数点节奏、不对称微调、气质调整

---

## 工作流程

### Step 1：分析需求（人工）

**三个问题**：
1. 画什么？（杯子、虾、猫）
2. 复杂度？（简单 10-30 path / 中等 30-60 / 复杂 60-90）
3. 焦点在哪？（陶土色用在哪里）

**输出**：
- 物体类型（生物/器物/建筑/工具）
- 关键点列表（如杯身的四个角）
- 复杂度级别

---

### Step 2：算法生成基础结构（算法）

**使用 `organic-curve-generator.js`**：

```javascript
const { generateBezierPath, getStrokeWidths } = require('organic-curve-generator');

// 定义关键点
const keyPoints = [
  { x: 75, y: 65 },   // 左上
  { x: 75, y: 155 },  // 左下
  { x: 125, y: 155 }, // 右下
  { x: 125, y: 65 }   // 右上
];

// 生成曲线
const path = generateBezierPath(keyPoints, {
  complexity: 'medium',
  isLeft: true,
  isRight: true
});

// 获取线条粗细
const widths = getStrokeWidths('medium');
```

**输出**：
- 基础 path d 属性
- 线条粗细配置
- 色彩填充代码

---

### Step 3：人工微调关键细节（人工）

**微调清单**：

| 细节 | 算法输出 | 人工微调 | 示例 |
|------|----------|----------|------|
| **控制点间距** | 均匀分布 | 手动调整节奏 | 4.1→2.01→-6.1 改为 4.1→11.14→25.3 |
| **小数点** | 规则分布 | 随机化位置 | 4.1 改为 4.72 |
| **不对称** | 左右对称 | 添加 ±1-5px 偏差 | 左边 75 改为 73.2 |
| **节奏** | 单一趋势 | 波浪式节奏 | 递增→递减→递增 |

**微调方法**：
1. 找到控制点密集的地方（算法可能生成了太多小间距）
2. 合并或删除一些控制点
3. 调整关键控制点的间距，形成呼吸感
4. 手动修改小数点，避免规律性

---

### Step 4：验证与迭代（人工）

**验证清单**：
- [ ] 曲线有呼吸感吗？（紧-松-紧节奏）
- [ ] 小数点不规则吗？（没有 .00，1-2 位混合）
- [ ] 左右有微妙不对称吗？
- [ ] 色彩分配正确吗？（陶土色 ≤2 处）
- [ ] 线条分层清晰吗？（主/结/细）

**如果不通过**：
- 回到 Step 3，继续微调
- 或回到 Step 1，重新定义关键点

---

### Step 5：输出 SVG（算法）

**使用 SKILL.md 的输出格式**：

```xml
<svg xmlns="http://www.w3.org/2000/svg" width="250" height="250" viewBox="0 0 250 250">
  <!-- 填充层 -->
  <g id="fill-layer">
    <path d="..." fill="#FEFFFE"/>
    <path d="..." fill="#D6654B"/>
  </g>

  <!-- 描边层 -->
  <g id="stroke-layer" fill="none" stroke="#1A1612" stroke-linecap="round" stroke-linejoin="round">
    <path d="..." stroke-width="3.0"/>
    <path d="..." stroke-width="2.0"/>
    <path d="..." stroke-width="1.0"/>
  </g>
</svg>
```

## 示例：杯子生成

### Step 1：分析需求
- 物体：杯子（器物类）
- 复杂度：简单（15-20 path）
- 焦点：杯身高光（陶土色）

### Step 2：算法生成
```javascript
const keyPoints = [
  { x: 75, y: 65 }, { x: 75, y: 155 }, { x: 125, y: 155 }, { x: 125, y: 65 }
];
const path = generateBezierPath(keyPoints, { complexity: 'medium' });
```

**输出**：
```svg
m75 65c4.1 25.79 2.01 23.21 -6.1 41c16.2 0.41 10.6 -1 23.3 0.58c5.5 -18.3 -3.6 -24.7 -1.85 -47
```

### Step 3：人工微调
- 控制点间距：`4.1` 改为 `11.14`（增加呼吸感）
- 小数点：`75` 改为 `75.2`（避免整数）
- 不对称：右边 `125` 改为 `124.8`（微妙偏差）

**微调后**：
```xml
m75.2 65c11.14 25.79 8.25 23.21 -6.1 41c16.2 0.41 15.7 -1 23.3 0.58c5.5 -18.3 -3.6 -24.7 -1.85 -47
```

### Step 4：验证
- ✅ 曲线有呼吸感（11.14→8.25→15.7）
- ✅ 小数点不规则（75.2, 11.14, 15.7）
- ✅ 色彩分配正确（米白主色）
- ✅ 线条分层清晰（3px/2px/1px）

### Step 5：输出 SVG
（完整代码略）

## 关键洞察

### 算法优势
- 快速生成基础结构
- 自动处理色彩填充
- 自动分层线条粗细

### 算法局限
- 控制点间距可能太均匀
- 小数点分布可能太规律
- 左右可能完美对称

### 人工价值
- 注入"呼吸感"（波浪式节奏）
- 打破"规律性"（随机小数点）
- 制造"人性化"（不对称偏差）
