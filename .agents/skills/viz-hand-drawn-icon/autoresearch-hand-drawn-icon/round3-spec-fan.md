# Round 3 规格书 — 电扇（复杂器物）

## API 调用信息

- **目标物体**: 电扇（台式电风扇）
- **复杂度预期**: 60-80 path
- **测试重点**: 生物类模式（5组件）是否可迁移到器物类

---

## 设计哲学（给 Quiver 的指令）

### 视觉气质
**温暖、有机、功能性** — 不是冷冰冰的工业产品，是有温度的日常用品。手绘感让金属部件不那么僵硬。

### 视角决策
**略俯视（20-25°）** — 展示：
1. 扇叶的立体感（能看到前后的扇叶重叠）
2. 底座的稳定性（圆形或方形底座完整呈现）
3. 整体比例（扇罩、底座、支架的关系）

### 色彩哲学

**主色：米白 #FEFFFE** — 电扇主体（扇罩、扇叶、底座）
- 理由：无生命物体用米白作为整体色调
- 应用：扇罩、扇叶、底座、支架统一用米白

**点缀：陶土色 #D6654B** — 功能性部件（开关、按钮、商标）
- 理由：点缀色不超过 10%，用于吸引注意力
- 应用：调速开关、电源按钮、品牌标识

**阴影：深陶土 #B03A21** — 结构深度（扇罩网眼、底座阴影）
- 理由：制造立体感和层次
- 应用：扇罩网眼交叉点、底座底部阴影

**描边：墨黑 #1A1612** — 所有轮廓线

### 结构分解（5 组件模式迁移）

**Layer 1: 扇罩（保护网）**
- 外圈：圆形轮廓，带手绘不规则感
- 网眼：交叉线条，不必太规整（手绘感）
- 中轴：圆形，固定扇叶的中心

**Layer 2: 扇叶（叶片）**
- 3-4 片叶片：放射状排列
- 透视：略俯视，能看到前后叶片重叠
- 曲线：叶片边缘有弧度，不是直线

**Layer 3: 支架（连接杆）**
- 高度：连接扇罩和底座
- 形状：细长圆柱或扁杆
- 调节：可显示角度调节关节（可选）

**Layer 4: 底座（稳固平台）**
- 形状：圆形或椭圆形底座
- 稳定：底部加宽或加重感
- 功能：开关、按钮集中在底座上

**Layer 5: 装饰（功能标识）**
- 开关：调速旋钮或按钮
- 品牌：简单文字或图形
- 电源线：从底座延伸出的曲线

### 线条分层

- **主轮廓**: 4.0px（扇罩外圈、底座轮廓）
- **结构线**: 3.0px（扇罩网眼、支架、扇叶边缘）
- **细节线**: 1.7px（开关按钮、商标、电源线）

### 曲线节奏

- 扇罩外圈：70% 小偏差 ±5px，30% 大偏差 ±15px（手绘圆的不规则感）
- 扇叶：中等曲线，展现叶片的弧形轮廓
- 支架：相对直，但带轻微弯曲（非机械直线）
- 网眼：交叉线条，允许交叉点不完全规整

### 细节密度控制

**器物类特殊性**：
- 功能性细节优先（开关、扇叶结构）
- 装饰性细节最小化（品牌标识简洁）
- 网眼可以简化（不必画出所有交叉点）

**预期 Path 分配**：
- 主体结构（扇罩+扇叶+支架+底座）：约 40-50 path（70%）
- 功能细节（开关、按钮）：约 8-12 path（15%）
- 装饰纹理（网眼、品牌）：约 8-12 path（15%）

## 学习重点

### 核心问题
1. **5 组件模式是否适用？**
   - 生物：头/身/四肢/尾/装饰
   - 器物：扇罩/扇叶/支架/底座/功能
   - 验证：结构分解逻辑是否一致？

2. **可动部件如何表达？**
   - 扇叶的透视关系（前后重叠）
   - 调节角度的关节（如果有）
   - 开关按钮的位置和大小

3. **色彩分配差异**
   - 生物：陶土色贯穿（虾）或米白贯穿（猫）
   - 器物：米白贯穿 + 陶土点缀
   - 验证：点缀色 10% 原则是否有效？

4. **网眼线条的简化策略**
   - 完整网眼：每个交叉点都画（太复杂）
   - 示意网眼：只画部分交叉线（推荐）
   - 完全省略：用纹理代替（太简单）

## 预期产出

**成功标准**：
- ✅ 5 组件模式成功迁移到器物类
- ✅ 可动部件（扇叶）透视正确
- ✅ 米白主色 + 陶土点缀 ≤10%
- ✅ 功能性细节清晰（开关可见）
- ✅ 手绘感自然（无机械对称）

**失败信号**：
- ❌ 扇叶透视错误（看起来平的）
- ❌ 陶土色过多（>10%，喧宾夺主）
- ❌ 网眼过于规整（机械感）
- ❌ 整体过于复杂（>100 path）
