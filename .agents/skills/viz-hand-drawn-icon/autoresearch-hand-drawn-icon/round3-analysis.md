## 1. 失败原因分析

### 为什么会生成人脸？

**可能原因 1: Prompt 混淆**
- Prompt 中提到了 "fan"（风扇），但 "fan" 在英文中也有"粉丝、狂热者"的意思
- Quiver 可能误解为"狂热者的人脸"而非"电风扇"

**可能原因 2: 描述过于复杂**
- Prompt 包含大量技术细节（5 components、40-60 sub-components、line hierarchy）
- 这些细节可能分散了 Quiver 对核心物体的注意力

**可能原因 3: 缺少明确的视觉锚点**
- 没有提供参考图片或更具体的视觉描述
- "electric desk fan" 可能不够明确

## 3. 关键学习

### ✅ Prompt 设计必须避免歧义

**错误示例**:
```
A hand-drawn electric desk fan...
```
- "fan" 有多种含义
- "desk fan" 不够具体

**正确示例**（应改为）:
```
A hand-drawn electric FAN for cooling air.
It has rotating blades, a protective grille, a stand, and a base.
NOT a human face, NOT a sports fan.
```

### ✅ 视觉锚点的重要性

**R1 虾的成功**: "side view to show body curves"
**R2 猫的成功**: "raised right paw", "seated posture"
**R3 电扇失败**: 缺少强视觉特征描述

**应该添加**:
- "rotating blades"（旋转的扇叶）
- "circular protective grille"（圆形保护网）
- "stand connecting to base"（连接底座的支架）

### ✅ 否定约束的必要性

对于有歧义的词，必须明确"NOT XXX"：
```
NOT a human face
NOT a sports fan
NOT a hand-held fan
```

## 7. 核心洞察总结

### 设计决策规则的普适性

| 规则 | R1 虾 | R2 猫 | R3 电扇（失败） |
|------|------|------|----------------|
| **视角选择** | ✅ 有效 | ✅ 有效 | N/A |
| **结构分解** | ✅ 有效 | ✅ 有效 | ✅ 有效（但对象错） |
| **色彩哲学** | ✅ 有效 | ✅ 有效 | ✅ 有效 |
| **线条分层** | ⚠️ 偏细 | ⚠️ 偏细 | ❌ **严重偏细** |
| **Prompt 设计** | ✅ 清晰 | ✅ 清晰 | ❌ **有歧义** |

### 关键教训

**1. Prompt 无歧义是第一优先级**
- 使用具体、明确的名词
- 避免多义词（fan → cooling fan）
- 添加否定约束（NOT XXX）

**2. 视觉锚点必须明确**
- 描述核心视觉特征（rotating blades）
- 说明功能状态（spinning, stationary）
- 提供比例关系（grille larger than base）

**3. 线条粗细需要重新校准**
- Quiver 的默认线条比我们预期的细
- 可能需要在 instructions 中更明确地指定 stroke-width

## 8. 对技能的改进建议

### 新增 Prompt 设计原则

**歧义词处理规则**:
```
如果名词可能有多重含义：
1. 使用更具体的描述（cooling fan vs fan）
2. 添加功能说明（machine that blows air）
3. 明确否定约束（NOT a human face）
```

**视觉锚点规则**:
```
必须包含 2-3 个强视觉特征：
- 核心部件（rotating blades）
- 独特形状（circular grille）
- 功能状态（spinning motion）
```
