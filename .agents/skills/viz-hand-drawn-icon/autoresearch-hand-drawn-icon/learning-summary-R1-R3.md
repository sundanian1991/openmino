### 2. 结构分解（三层结构法）

**第一层：主体轮廓** — 一笔到底的超长 path

### 3. 色彩哲学（核心验证）

**整体色调原则** — 颜色是整体色调，不是分段涂色

| 物体 | 主色 | 占比 | 点缀色 | 占比 | 验证 |

**关键发现**: Quiver 完美执行"整体色调"原则

### 4. 线条分层（⚠️ 需要调整）

**我们预期的标准**:
- 主轮廓: 4.0-5.2px
- 结构线: 3.5-4.0px
- 细节线: 1.7-2.9px

**Quiver 实际输出**:
- 主轮廓: 1.08-3px（整体偏细）
- 结构线: 0.5-1.5px
- 细节线: 0.25-0.7px

**发现**: Quiver 的线条整体比我们预期的细约 60%

**建议**:
- 接受 Quiver 的线条粗细（自然、有机）
- 或在 instructions 中明确指定 stroke-width 数值

### 5. 曲线节奏

**Quiver 完美执行**:
- 70% 小偏差 ±5px
- 30% 大偏差 ±15-30px
- 无机械对称
- 坐标不规则（无 .00 结尾）

**验证状态**: ✅ R1、R2 都呈现真实手绘感

### 6. 5 组件模式（复杂场景）

**生物类 5 组件**: 头/身/四肢/尾/装饰
**验证状态**: ✅ R2（猫）完美验证

## 🚨 新发现的问题与应对

### 1. 词汇偏见（R3 失败原因）

**问题**: "fan" 有多重含义（风扇/粉丝/狂热者）
**表现**: 两次尝试都生成了人脸
**应对策略**:
```
避免多义词:
- fan → electric cooling appliance
- bat → flying mammal / sports equipment
- match → fire starter / competition
```

### 2. 否定约束的局限性

**假设**: "NOT a human face" 可以避免错误生成
**现实**: 对于强联想词，否定约束无效
**建议**: 不要依赖否定约束，从源头避免歧义

### 3. 线条粗细差异

**问题**: Quiver 默认线条比我们预期的细
**应对**:
- 方案 A: 接受 Quiver 的自然线条（推荐）
- 方案 B: 在 instructions 中明确指定 stroke-width
- 方案 C: 调整我们的标准，匹配 Quiver 输出

## 📐 提炼的设计决策规则（v5.0）

### 新增：词汇选择原则

```markdown
## 词汇选择原则（新增）

### 高风险多义词（必须避免）

| 词汇 | 多重含义 | 替代词 |
|------|----------|--------|
| fan | 风扇/粉丝/狂热者 | electric cooling appliance |
| bat | 蝙蝠/球棒 | flying mammal / sports equipment |
| match | 火柴/比赛/匹配 | fire starter / competition |
| bank | 银行/河岸 | financial institution / river bank |

### 验证方法

在选择测试物体前，自问：
1. 这个词有多个含义吗？
2. 是否可能被误解为其他物体？
3. 有无更明确的替代词？
```

### 新增：否定约束原则

```markdown
## 否定约束原则（新增）

### 适用场景

否定约束适用于：
- ❌ 弱联想词（如：tree — 不太可能误解为其他物体）
- ✅ 可以添加 "NOT a XXX" 增强明确性

否定约束不适用于：
- ✅ 强联想词（如：fan — 人脸联想权重太高）
- ❌ 不要依赖否定约束解决根本问题

### 核心原则

**从源头避免歧义 > 事后否定约束**
```

### 调整：线条粗细标准

```markdown
## 线条分层（调整）

### Quiver 自然标准（推荐）

Quiver API 的默认线条粗细：
- 主轮廓: 1.5-3px
- 结构线: 0.7-1.5px
- 细节线: 0.25-0.7px

**优势**: 自然、有机、符合手绘风格

### 自定义标准（可选）

如果需要更粗的线条，在 instructions 中明确：
```
stroke-width="4.0" for main outline
stroke-width="3.0" for structure lines
stroke-width="1.7" for detail lines
```
```
