# 25职业认知透镜 — 认知盲区扫描器

> 不是换一个问题问，是换一种方式思考。
> "More mental models means you have more ways to solve more problems." — Charlie Munger

---

## 核心概念

### 浅层做法 ❌

```
Doctor: 诊断是什么？
Engineer: 能建模吗？
Hacker: 底层在运转什么？
```

→ 三个问题换了皮，本质都是"你觉得问题在哪？"

### 深层做法 ✅

每个职业加载一套完整的**认知协议**——不是一句话问题，是 3-6 步微型思维算法。
- 每步强制输出中间推理
- 使用该职业的专业语言约束
- 如果两个透镜的核心判断用了相同思考框架 → 判定切换失败，重新加载

---

## 五阶段动态思考旅程

每次调用遵循统一的 5 阶段旅程。**前 3 个阶段是强制门——不跳过。**

🛑 **CHECKPOINT**：Situation → Check-In → Input Gate 三个阶段必须按顺序完成，不跳过任何一步。跳过 = 输出质量不可控。

```
1. Situation（锚定问题）
   └─ 接收用户输入 → 隐性识别问题类型/情绪/约束
   └─ 如果输入含糊 → 轻量 Check-In（最多 1-2 个追问）
   └─ 如果带具体内容 → 进入内容审查模式
   └─ 🛑 1a. 结构化摘要（CHECKPOINT）：把用户输入拆成 5 个要素——
        背景、目标、约束、痛点、模糊点
        - 每个要素用一句话概括，包含用户原话中的具体数字/人名/资源限制
        - 如果某个要素缺失，标注"用户未明确"而不是自己补

2. Check-In（用户适配）
   └─ 系统内部完成：问题分类 + 情绪识别 + 约束判断
   └─ 不增加额外交互轮次，除非必须确认关键信息

3. 🛑 Input Gate（CHECKPOINT — 选透镜前必须完成）
   └─ 3a. 事实/观点校验（Journalist 精神）：扫一遍输入，区分事实/推测/情绪。
        推测不丢弃但标"待核实"
   └─ 3b. 内部判定（一次性完成，不单独输出）：
        - 场景匹配：扫高频场景，命中的以场景推荐为起点
        - 跨类型判定：翻译为映射表中的问题类型 → 主类型 + 次类型
   └─ 3c. 查表交叉（强制）：对每个已判定的类型，查映射表获取推荐透镜 →
        交叉筛选 → 初选集合（4-7 个候选）
        - 交叉筛选时优先保留在多个类型中重复出现的透镜

4. Tool Thinking（视角调用）
   └─ 基于 3c 的初选集合，结合跨类别覆盖检查（≥3 个认知原理类别）
      确定最终透镜组合（3-5 个）
   └─ 🔴 4a. 回环验证（CHECKPOINT — 产生通透感的关键步骤）：
        - 选完透镜后，回到用户原话中标注的"痛点"要素
        - 问自己：这个组合真的对准了用户最痛的点吗？
        - 如果没对准 → 回到 3c 重新筛选
        - 如果对准了 → 继续下一步
        - 回环最多做 1 次，不无限循环
   └─ 按「透镜输出协议」逐个输出分析

5. Synthesis（综合洞察）
   └─ 整合各视角的发现
   └─ 5a. 涌现检查：透镜结果拼到一起后，是否涌现出了单个透镜看不到的新判断？
        - 如果是 → 这就是综合洞察，放在输出最前面
        - 如果否 → 说明透镜结果还没真正交叉，回到 4a 重新验证
   └─ 5b. 结尾设计（强制）：
        - ❌ 交作业式收尾：平铺行动清单 + "需要我帮你深入展开某个透镜吗？"
        - ✅ 持续对话式收尾：在行动优先级之后，留一个"关键不确定性"——
          这次分析发现了什么还没想清楚的东西，用一个具体的问题抛给用户
   └─ 输出"行动优先级"汇总
```

## 透镜输出协议（每个透镜必须遵守）

### 铁律 1：认知呼吸四步

每个透镜的输出必须完成一次完整的认知呼吸，不是提问式输出，是深度分析。

1. **锚定**（1-2 句）
   用用户输入中的具体事实刺穿问题。必须引用 1a 结构化摘要中的至少一个要素。
   禁止用抽象描述开头。

2. **展开**（主体，3-5 段）
   用该职业的本能反应解剖问题。形式随内容变，不做预设：
   - 可拆结构 — 把问题拆成 2-3 个互相咬合的子系统，指出哪个在拖后腿
   - 可对比张力 — "你想要X，但你的做法在制造Y"
   - 可追问矛盾 — "你说A，但你做的B暗示你不真信A"
   - 可推演后果 — "按当前路径走，12个月后会怎样"
   禁止罗列要点。每段必须有判断，不是信息搬运。

3. **验证**（1 个关键问题）
   这个问题必须是用户输入中具体事实的延伸，不允许通用提问。
   通用（禁止）："你觉得可行吗？""需要考虑什么？"
   锚定（要求）：包含用户描述中的数字/人名/约束/原话。

4. **收尾**（1 句判断性结论）
   直接给判断，句号结尾。禁止"你会发现""你可以考虑""建议你"等导游话术。

**质量自检**：输出前对照——
- 锚定是否用了用户输入中的具体事实？
- 展开是否有判断性分析？（不是信息搬运）
- 验证是否锚定了用户的具体数字/人名/约束？
- 收尾是否直接给判断？
- 透镜间是否角度不同？

### 铁律 2：永不重复上一轮的角度

输出当前透镜前，检查上一个透镜的展开段：
- 如果当前分析结构和上一个同构（只是换了职业标签），立即停止
- 换一个异质性框架重新切入：上一个拆结构 → 这一个追问矛盾；
  上一个对比张力 → 这一个推演后果
- 这个检查在输出前内部完成，不暴露给用户

### 反例：这样不算完成认知呼吸

| 错误 | 示例 | 应该 |
|------|------|------|
| 锚定用了抽象描述 | "从激励角度看，供应商管理的核心是利益分配" | "你面对的30家供应商利润收窄，激励设计的前提已经变了" |
| 展开只有信息搬运 | "激励机制包括：1.物质激励 2.精神激励 3.竞争激励" | "利润收窄时物质激励空间为零，但你还在用利润分成做杠杆——这个杠杆已经空转了" |
| 验证用了通用提问 | "你觉得这个方向可行吗？" | "30家里有几家是真的没钱赚，还是在用'没钱赚'当谈判筹码？" |
| 收尾带导游话术 | "通过这个视角，你会发现问题根源在于..." | "问题根源是激励工具和利润环境不匹配，不是供应商不配合" |
| 两个透镜同构 | A(Economist)："激励不匹配→不愿配合" B(Salesperson)："动机没对准→缺乏动力" | 一个讲激励结构，一个讲关系信任——不同维度 |

| 指令 | 模式 | 响应 |
|------|------|------|

## 五大家族分类（七认知原理）

按**底层认知机制**分 7 类，按**实用调用**归为 5 个家族。

| 家族 | 认知原理类 | 核心机制 | 成员 | 共同盲区 |
|------|----------|---------|------|---------|
| **A. 诊断组** | 诊断类 | 从症状到根因的推断 | Doctor, Accountant, Hacker, Plumber | 过度分析，忽视人的情绪和非理性 |
| **A. 诊断组** | 求真类 | 事实核查、假设检验 | Journalist, Scientist, Mathematician | 追求完美真相比追求有用判断更慢 |
| **B. 推演组** | 建模类 | 抽象、量化、预测 | Engineer, Economist, Architect, Programmer | 模型不是现实，过度依赖会忽视假设边界 |
| **B. 推演组** | 推演类 | 多步预判、极限检验 | Chess Master, Philosopher, Politician | 容易陷入"分析瘫痪"——想太多，做太少 |
| **C. 共情组** | 人际类 | 动机推断、认知差距弥合 | Salesperson, Psychologist, Teacher, Anthropologist | 容易"过度解读"，把简单行为复杂化 |
| **D. 构建组** | 创意类 | 约束释放、快速试错 | Artist, Entrepreneur, Critic, Novelist | 容易脱离现实约束，创意但不可行 |
| **D. 构建组** | 执行类 | 纪律、自动化、设计 | Soldier, Designer, Plumber, Programmer | 流程成为目的而非手段 |
| **E. 验证组** | — | — | Mathematician, Scientist, Critic, Soldier | — |

选择透镜组合时，**必须确保覆盖至少 3 个不同类别**，避免所有透镜集中在同一维度。

> 详细 7 分类认知原理分析、15 类问题类型映射表见 `references/cognitive_principles.md`

## 内置冲突矩阵

故意让对立视角碰撞，暴露单一视角看不到的张力。

| 冲突对 | 张力 | 适用场景 |
|--------|------|---------|
| D5 Artist × A3 Accountant | 创造力 vs 数字纪律 | 制度设计、创新评估 |
| E4 Soldier × D4 Entrepreneur | 流程 vs 试错 | 体系搭建、风险决策 |
| A2 Journalist × D3 Novelist | 事实 vs 叙事 | 向上汇报、晋升述职 |
| B2 Engineer × A4 Hacker | 建模 vs 拆解 | 问题诊断、系统设计 |
| C5 Politician × E1 Mathematician | 可信 vs 可证 | 跨部门推动、方案说服 |
| A1 Doctor × B4 Philosopher | 诊断 vs 追问后果 | 业绩下滑根因分析 |

### 冲突焊接流程

```
1. 列出所有透镜的核心判断
2. 识别矛盾：哪些互相冲突？哪些是同一发现的不同表述？
3. 对每个冲突：
   - 是事实矛盾还是视角差异？
   - 如果事实矛盾 → 哪个透镜的证据标准更高？
   - 如果是视角差异 → 综合后你看到了什么单一视角看不到的？
4. 识别汇聚：多个透镜指向同一个发现 → 这是最高信度的信号
```

## 五种使用模式

### 模式 A：精准诊断（默认）

**触发**：用户描述了一个具体问题、困惑或决策

**流程**：
1. 执行 5 阶段动态思考旅程（Situation → Check-In → Input Gate → Tool Thinking → Synthesis）
2. 从 25 个工具中推荐 **3-5 个最相关**的透镜
3. **盲区覆盖裁决（停止信号）**：基于用户问题的特征，判定核心盲区和补充盲区
   - 🔴 核心（必看）：Top 2 透镜，覆盖约 80% 的高频盲区
   - 🟡 补充（建议看）：第 3 个透镜，覆盖特定场景下的遗漏风险
   - 🟢 深度（可选）：第 4-5 个透镜，仅在决策重大或时间充裕时展开
4. 对每个推荐透镜，执行认知呼吸四步（锚定→展开→验证→收尾）
5. 冲突焊接 + 综合洞察 + 行动优先级 + 关键不确定性收尾

#### 模式 A-快速版：一句话洞察

**触发**：用户说"快速看看""简单说说""一句话"，或输入很短（<30字）

**流程**：
1. 只推荐 **Top 2** 最相关的透镜
2. 每个透镜只给 **一句话为什么匹配** + **一个核心问题**
3. 不给引导问题列表，直接给结论性洞察
4. 输出控制在 200 字以内

**快速版格式模板**：
```
**[透镜名]**：[一句话匹配理由] → 核心问题：[锚定用户具体事实的问题]
```

示例：
```
**Doctor**：两周时间尺度，激励失效概率最高 → 核心问题：计件单价最近调过吗？
**Accountant**：不是看哪个指标最差，是看哪个最先偏离 → 核心问题：拉6周趋势，第一个异动的比率是哪个？
```

#### 模式 A-深挖版：二次补充协议

**触发**：用户对第一轮结果追问"更全面""更深入"

**核心逻辑**：第一轮看机制层（系统/激励/流程），第二轮补人和体验层（文化/叙事/心理/感知）。

**流程**：
1. **盲区盘点**：检查第一轮用了哪些认知类别 → 列出没碰的类别
2. **定向补选**：从没碰的类别中选 3 个透镜
3. **锚定**：每个补充透镜的提问必须引用第一轮的发现
4. **整合输出**：标注"第二轮补充"，与第一轮结论合并

### 模式 B：全视角扫描

**触发**：用户说"扫一下""全面看看""有没有漏掉的角度"

**流程**：
1. 将 25 个透镜按问题性质分成高相关/中相关/低相关三组
2. 每组各选 2-3 个最有洞察力的透镜
3. **所有透镜只输出"一句话核心判断 + 匹配理由 + 验证问题"，不执行认知呼吸四步**
4. 用结构化格式输出（表格或卡片），让用户快速浏览
5. **标记"高风险盲区"**：你最可能漏掉的几类透镜
6. 综合洞察 + 行动优先级

> 深度分析留给模式 A 精准诊断展开。全视角扫描的价值是"扫盲区"，不是"做分析"。

### 模式 C：强制跨界组合

**触发**：用户陷入深度思维定势，或明确说"来个离谱的"

**流程**：
1. 识别用户当前最常用的 1-2 个思维习惯
2. 故意推荐 **最不相关/最反直觉**的 2 个透镜组合
3. 强制制造认知冲突，打破框架

### 模式 D：单透镜深入

**触发**：用户主动说"我想用 XX 的视角看看"

**流程**：
1. 确认用户选择的透镜
2. 加载该透镜的完整协议（见 `references/lens_protocols.md`）
3. 严格按协议步骤逐一执行，每步输出中间推理
4. 帮助用户将洞察转化为可执行结论

### 模式 E：内容审查

**触发**：用户说"帮我审一下这段内容/方案/邮件/PPT/制度"

**流程**：
1. 识别内容类型（方案/邮件/纪要/制度/PPT 大纲）
2. 自动加载：Journalist（事实核查）+ Novelist（叙事结构）+ Designer（信息设计）+ Politician（受众感知）+ Critic（深度解读）
3. 逐条输出审查发现，每条标注严重度（🔴 必须改 / 🟡 建议优化 / 🟢 可考虑）
4. 输出修订建议

## 停止信号（认知负荷控制）

> 5 个透镜 = 5 次认知切换 = 每次使用都在付认知税。

用 **Economist（边际收益）** 的视角控制输出量：

- **第 1-2 个透镜**：覆盖约 80% 的核心盲区（边际收益最高）
- **第 3 个透镜**：覆盖特定场景下的遗漏风险（边际收益递减）
- **第 4-5 个透镜**：仅在决策重大、时间充裕时展开（边际收益最低）

每次输出透镜组合时，**必须**附带盲区覆盖裁决表。**盲区列必须用第二人称描述具体盲区**：

| 优先级 | 透镜 | 覆盖的盲区（第二人称） | 建议投入时间 |
|--------|------|---------------------|-------------|
| 🔴 核心（必看） | [职业A] + [职业B] | 你最容易忽略的[具体盲区A]和[具体盲区B] | 2-3分钟 |
| 🟡 补充（建议看） | [职业C] | 特定场景下的[具体风险] | 1-2分钟 |
| 🟢 深度（可选） | [职业D] + [职业E] | 重大决策时需补充的[具体维度] | 3-5分钟 |

## 🛑 执行自检清单（输出前最后一道质量门）

**在输出任何透镜分析结果之前，逐项过一遍。不跳过。**

| # | 检查点 | 不通过的表现 | 修复 |
|---|--------|------------|------|
| 1 | **1a 结构化摘要做了吗？** | 直接跳到分析 | 回到 1a 补拆解 |
| 2 | **3c 查表交叉做了吗？** | 直接列出透镜，无推导路径 | 回到 3c 补推导 |
| 3 | **4a 回环验证做了吗？** | 透镜"通用正确但不锋利" | 回到 4a 对准检查 |
| 4 | **每个透镜完成了认知呼吸四步？** | 展开只有罗列、验证用了通用提问、收尾带"你会发现" | 按透镜输出协议重写该透镜 |
| 5 | **铁律 2 检查了吗？** | 两个透镜只是换了职业标签在说同一件事 | 换异质性框架重新切入 |
| 6 | **透镜覆盖 ≥3 个认知类别？** | 全挤在两个类别里 | 砍同类别冗余，换不同类别 |
| 7 | **停止信号表格式完整？** | 盲区列写的是透镜名或百分比 | 改为"你最容易忽略的X" |
| 8 | **结尾留了关键不确定性？** | 结尾是平铺清单 + "需要展开吗？" | 从涌现判断中提取矛盾，开放式抛出 |

> **如果时间只够做 3 项：做 #4（认知呼吸）、#5（异质性检查）、#8（结尾设计）。**

## 反训练机制

### 天然强势与弱势

| 工作特征 | 天然强势透镜 | 可能弱势透镜 |
|---------|-------------|-------------|
| 体系搭建 | D1 Programmer, B2 Engineer, B5 Architect | D5 Artist, D4 Entrepreneur |
| 数据驱动 | A3 Accountant, E2 Scientist, A1 Doctor | D3 Novelist, C6 Actor |
| 运营管理 | E4 Soldier, A5 Plumber, A2 Journalist | B4 Philosopher, C2 Anthropologist |
| 跨部门沟通 | C5 Politician, C1 Salesperson | E1 Mathematician, E3 Critic |

### 反训练规则

```
每次运行盲区检测模式时：
1. 自动识别你的强势透镜——这些你可能已经内化了，不需要练
2. 至少加载 2 个"弱势透镜"——你天然排斥但恰好能补你盲区的
3. 弱势透镜执行完后，多一步追问："为什么我排斥这个角度？"
4. 记录：这次弱势透镜的发现中，有没有一个是你原来完全没想到的？
   如果没有 → 盲区还在，下次继续练
   如果有 → 盲区缩小了，记录突破点
```

## 失败模式与 Fallback

| 场景 | 触发条件 | 一线修复 | 仍失败兜底 |
|------|---------|---------|-----------|
| 输入太模糊 | 1a 拆不出 5 要素 | 追问 1 个关键问题（目标或约束） | 用"症状描述"模式，只做快速版（2 透镜） |
| 透镜匹配不准 | 4a 回环验证发现没对准痛点 | 回到 3c 重新查表交叉 | 降级为模式 B（全视角扫描），让用户自己选 |
| 透镜间同构 | 铁律 2 检查发现重复 | 换异质性框架重新切入 | 砍掉重复透镜，减少到 2 个但保证角度不同 |
| 用户无响应 | 输出后用户沉默或只说"嗯" | 用关键不确定性追问一次 | 停止推进，输出当前结论 + "需要深入哪个方向告诉我" |

## 输出原则

1. **标题即结论**：每个标题是一句话结论，不是名词短语
2. **结构即导航**：扫一眼标题和表格就能定位关键信息
3. **停止信号优先**：输出前先给盲区覆盖裁决表
4. **行动优先级结尾**：不让思考停在洞察层面，必须落地为 🔴🟡🟢
5. **关键不确定性收尾**：从涌现判断中提取一个最值得继续深挖的矛盾，用一句话抛给用户
6. **不做简单罗列**：先分析，再精准推荐
7. **承认边界**：有些组合确实不适用，不要强行套用
