## Demo Uniqueness Audit

- Previous-work audit: Prior supplier-management HTML used Nian-design process with glacier deep Hero + numeral-grid + step-sequence. Current work uses Fincher cinematic language instead.
- Recurring traits to avoid: Two-column hero with metrics on right, step-sequence with monotone borders, card-quad in olive background
- Shell-ban list: Standard Nian template sections, surface-raised section backgrounds, border-only card separation
- Primary composition family: Full-bleed stage with evidence-board density
- Why this family differs: Evidence-board framing (inspired by Fincher's "data as evidence" visual language) replaces Nian's clean template sections. Each section is a "case file" rather than a report section.

## Research Notes

### Film Palette (The Social Network → Nian Design)
- Primary: #2A4A5A (scene-glacier — maps to Fincher's deep cool tones)
- Secondary: #4A5D3A (brand-olive — muted, grounded complement)
- Accent: #FFD100 (accent-yellow — maps to Social Network's warm interior/amber lighting)
- Shadow: none (Nian constraint — depth via borders, not shadows)
- Text: #2C2C2C on light surface; rgba(255,255,255,0.85) on dark

### Director Signatures (Web Translations)
1. Precision framing → Every section has a deliberate asymmetry, no centered generic grids
2. Information density → Dense data presentation with clear hierarchy (Answer→Argument→Evidence)
3. Controlled coolness + selective warmth → Cool glacier base, warm yellow only for key data

### Film Translation Notes
- Framing: Asymmetric, edge-anchored, "evidence board" composition — data feels like case files
- Rhythm: Slow reveal → dense data → breath → dense data → breath → resolution
- Materiality: Matte surfaces, no gloss, no blur. Depth through borders and opacity, never shadow
- Restraint: One break element (large ghost number). Fincher doesn't over-decorate
