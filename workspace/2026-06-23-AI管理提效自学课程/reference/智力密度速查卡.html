三个低密度陷阱（见之即改）

  1. 全量历史塞 context：同样的信息反复读 → Context 层解决
  2. 套固定模板的思维链：空转不增加判断 → Prompt 层解决
  3. 失败的工具调用重试：同一错误反复烧 token → Harness 层解决
