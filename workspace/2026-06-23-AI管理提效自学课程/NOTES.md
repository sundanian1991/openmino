# NOTES — 教学偏好与工作笔记

## 用户偏好

- **母语中文教学**，专业术语保留英文原词（如 harness、loop、token、PEV）。
- **以自己的实践为锚点**：每个概念必须对应一个已做过的真实案例。用户反感空学理论。
- **结构化优先**：用户偏好 MECE 的金字塔拆解，不喜欢散点罗列。
- **"思维链应依据内容动态生成，而非套固定流程模板"** —— 用户 06-04 明确的原话，设计课程时尊重这一直觉，不堆砌固定流程。
- **可视化优先**：关系/流程/对比/数据/结构用图，禁用文本符号模拟（来自 AGENTS.md）。
- token 成本敏感：轻量日几百~几千、产研日 30-80 万（06-04 数据）。Cost 层课程要落到这个量级。
- 母语自学，lesson 可以用中文 HTML。

## 课程结构关键决策（2026-06-23）

1. **切割线 = "一次 AI 协作的生命周期"**：选了空间维度（输入→接口→外围→回路）而非职能维度，因为生命周期每层都有用户现成案例锚定。
2. **4+3 结构**：空间四层（Context/Prompt/Harness/Loop）+ 治理三层（Skill/Eval/Cost）。治理层处理跨多次协作的沉淀、度量、预算问题。
3. **Skill 从 Prompt 层独立为第 5 层**：因为 skill 横切四层（SKILL.md 正文=Context、触发编码=Prompt、加载机制=Harness、复用积累=Loop），不应降格为 Prompt 子项。
4. **Model Routing 归入 Cost 层**：选强/弱模型是经济决策，不另起一层。
