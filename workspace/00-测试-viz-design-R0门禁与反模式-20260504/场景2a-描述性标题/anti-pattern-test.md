# 场景 2a-2d 反模式拦截测试记录

> 测试 viz-design 在执行过程中是否识别并拦截反模式

---

## 场景 2a：触发 A1（描述性标题）

**输入**："做一张供应商 Q1 业绩分析图"

### 执行过程

**意图识别**：用户要做供应商 Q1 业绩的可视化，但没有提供具体数据，标题使用了描述性语言。

**A1 拦截检验**：
- 标题为"供应商 Q1 业绩分析图"
- 检验方法：去掉最后一个词"分析图"后 → "供应商 Q1 业绩"仍然成立
- **判定：触发 A1 反模式**

### 拦截动作

识别到标题为描述性标题，应要求改为结论性标题。

**错误**："供应商 Q1 业绩分析图"
**正确（示例）**："Q1 供应商业绩整体下滑 15%，2 家逆势增长"

### 测试结果

| 项目 | 结果 |
|------|------|
| 是否触发 A1 拦截 | **是** |
| 判定依据 | 标题以"分析"结尾，去掉后语义仍完整 |
| 修正方向 | 要求补充具体结论 |
| 缺陷 | 无 |

## 场景 2b：触发 A2（一图多信息）

**输入**："画一张图同时展示供应商的达成率趋势、占比构成、和效率关系"

### 执行过程

**意图识别**：用户要求在**一张图**中同时表达：
1. 达成率趋势（时间序列 → 折线图）
2. 占比构成（比例关系 → 饼图/堆叠条）
3. 效率关系（相关性 → 散点图）

**A2 拦截检验**：
- 标题包含 3 个不同维度的信息
- 可以用"而且"连接："展示趋势 而且 展示占比 而且 展示关系"
- 3 个维度需要不同的图表类型
- **判定：触发 A2 反模式**

### 拦截动作

根据 decision-rules.md 规则 2："一个图一个核心信息"，应拆分为 3 张图：

| 图序 | 维度 | 建议模式 |
|------|------|---------|
| 图 1 | 达成率趋势 | 07-折线图 |
| 图 2 | 占比构成 | 14-磁带图（堆叠条）|
| 图 3 | 效率关系 | 09-散点图 |

### 测试结果

| 项目 | 结果 |
|------|------|
| 是否触发 A2 拦截 | **是** |
| 判定依据 | 输入包含 3 个独立维度，用"和"连接 |
| 拆分方向 | 拆为趋势图 + 构成图 + 关系图 |
| 缺陷 | 无 |

## 场景 2c：触发 B2（禁止图型 - 饼图>3 分类）

**输入**："用饼图展示 5 个供应商的份额占比"

### 执行过程

**意图识别**：用户指定饼图，5 个分类。

**B2 拦截检验**：
- 饼图分类数 = 5
- 反模式 B2 规则：饼图（>3 分类）→ 禁止
- **判定：触发 B2 反模式**

### 拦截动作

根据 anti-patterns.md B2 规则：

| 项目 | 内容 |
|------|------|
| 禁止原因 | 饼图 >3 分类时扇区角度差异难以比较，尤其是 5 个相近值 |
| 替代方案 | 水平条形图（13-柱状图的横向变体）|
| 优势 | 线性排列，标签清晰，长度对比直观 |

### 测试结果

| 项目 | 结果 |
|------|------|
| 是否触发 B2 拦截 | **是** |
| 判定依据 | 饼图 5 个分类 > 3 分类上限 |
| 替代方案 | 水平条形图 |
| 缺陷 | 无 |

## 场景 2d：触发 B4（非零基线）

**数据**：供应商 A=40, B=42, C=45, D=48, E=92
**意图**：柱状图展示

### 执行过程

**意图识别**：柱状图对比 5 个供应商评分，其中 4 个在 40-48 区间，E=92 是 outlier。

**B4 拦截检验**：
- 柱状图 yAxis min 必须为 0
- 如果截断 Y 轴（如 min=35），A(40) 和 B(42) 的高度差异会被放大，造成误导
- **判定：需在 SPEC 中强制 yAxis min = 0**

### 拦截动作

在视觉编码中明确声明：
- Y 轴范围：0-100，**min = 0**
- 即使 40-48 区间的柱子高度差异较小，也不截断，保持数据诚实

### 测试结果

| 项目 | 结果 |
|------|------|
| 是否触发 B4 拦截 | **是** |
| 判定依据 | 柱状图 yAxis min 必须为 0 |
| 执行方式 | SPEC 中声明 scales.y.min = 0 |
| 缺陷 | 无 |
