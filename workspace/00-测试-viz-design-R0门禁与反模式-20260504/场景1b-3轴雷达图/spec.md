# SPEC — 场景 1b：3 轴雷达图

> 测试目标：故意选 22-雷达图，仅 3 维度，验证 R0-4 是否拦截

---

## R0 门禁检验记录

### 首次检验（22-雷达图，3 维度）

| 序号 | 检验项 | 判定 | 理由 |
|------|--------|------|------|
| R0-1 | 几何直觉 | PASS | 3 维可以构成三角形，几何上可读 |
| R0-2 | 对象可区分 | PASS | 对象各有标签 |
| R0-3 | 差距可读 | PASS | 3 维三角形面积差异可见 |
| R0-4 | 轴数充足 | **FAIL** | 雷达图轴数仅 3 条 < 5 条最低要求。3 轴雷达图退化为三角形，视觉上像菱形而非雷达图，失去多维对比的意义 |

**结果：R0-4 失败 → 门禁不通过 → 查备选表**

### 备选模式替换

查 `chart-validity-gate.md` 备选表：
- 22-雷达图（轴数<5 或 多对象叠加无法区分）→ **13-柱状图（分组对比）**

### 重新检验（13-柱状图）

| 序号 | 检验项 | 判定 | 理由 |
|------|--------|------|------|
| R0-1 | 几何直觉 | PASS | 线性柱状图，3 秒可辨 |
| R0-2 | 对象可区分 | PASS | 分组柱状图，每组 3 个维度 |
| R0-3 | 差距可读 | PASS | 柱子高度差一目了然 |
| R0-4 | 轴数充足 | PASS | N/A |

**结果：全部通过 → 继续**

---

## 第一部分：叙事意图

**【可视化目的】**
对比某供应商在 3 个维度（交付质量、响应速度、成本控制）上的能力表现。认知缺口：读者不知道哪个维度是短板。

**【想传达什么】**
该供应商交付质量强，但响应速度和成本控制是短板。

**【结论】**
交付质量领先，响应速度和成本控制需提升。

**【思路】**
分组柱状图，3 个维度并排对比，柱子高度直观反映强短板差异。

---

## 第二部分：视觉执行

**【模式选择】**
- **模式**：13-柱状图（原选 22-雷达图，R0-4 失败后切换）
- **理由**：3 维对比用柱状图更直观，避免 3 轴雷达图的菱形退化

**【标题】**
- **主标题**：交付质量领先但响应速度和成本控制是短板
- **副标题**：2026 Q1 · XX 供应商三维能力评估

**【视觉编码】**
- **X 轴**：3 个维度（交付质量、响应速度、成本控制）
- **Y 轴**：评分 0-100，min=0
- **颜色**：最高维度高亮 Warm 500，其余 Stone 300

**【数据组织】**
| 维度 | 评分 |
|------|------|
| 交付质量 | 95 |
| 响应速度 | 62 |
| 成本控制 | 58 |

**【标注策略】**
- 高亮交付质量柱（最高）
- 标注响应速度和成本柱："流程响应周期长 + 人工成本偏高"
- 60 分及格线

**【布局】**
- 画布：700 × 400
