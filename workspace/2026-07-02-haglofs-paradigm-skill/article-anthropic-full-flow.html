上周 Anthropic 发了一份报告，本来没太当回事——这家公司最近新闻太多了。一会儿搞实名认证，一会儿顶级模型给看不给用，还有跟美国政府的对线。现在一看到他们家名字都快免疫了。

但当看完这份报告的详细内容后，还是忍不住想分享给你。因为这不是那种「我们觉得未来会怎样」的趋势预测报告，而是实打实的数据报告。

一份前所未有的数据。他们分析了约 40 万次真实的 Claude Code 交互会话，来自约 23.5 万个用户，时间跨度从 2025 年 10 月到 2026 年 4 月。不是跑测试题，不是问卷调查，不是分析公开代码仓库——是真实的人机协作过程：人说了什么、AI 做了什么、最后成没成。

Claude Code 是过去一年全球知识工作者广泛使用的 AI 编程产品之一。甚至连它的竞争对手谷歌的工程师也在做类似的产品，还公开称赞 Claude Code 好用。注意，这已经不是「写代码的工具」，而是「完成复杂任务」的 AI 产品。你给一个指令，它会自己读文件、改代码、跑命令、检查结果，然后再回到你这里。

一条指令下去，它平均触发约 10 个动作，输出约 2400 字。这种数据在 AI 行业里之前几乎没有——只有像 Anthropic 这样既做模型、又能看到真实使用数据的公司，才拿得到。

Core Finding 01

人管判断，
AI 管执行

这份报告最有价值的发现，是第一次用真实使用数据证明了一件事。

Human Decision
~70%
规划：做什么 / 走哪条路线 / 怎样算完成。人决定方向和质量标准。
AI Decision
~80%
执行：改哪些文件 / 写什么代码 / 跑哪些命令。AI 决定实现路径。

70% 和 80%，不是猜测，不是推演，是 40 万次对话里真实存在的分工。人管 what，AI 管 how——这句话以前很多人在说，但从来没有人拿过数据证明。现在有了。

举个小例子：你说「帮我修复这个登录页面的 bug」，Claude Code 自己读了三个配置文件，改了两处代码，跑了一遍测试，测试没过，又回去改了一处，再跑，过了，最后把改动列出来给你看。你全程只说了一句话，它做了十几个动作。

而且七个月下来，用户行为在变：

修 bug
33% → 19% 不再只是修修补补
运行软件
14% → 21% 开始部署和跑流程
写作+分析
~10% → ~20% 非程序员开始进场

七个月下来，用户开始把更接近最终交付的任务交给它了。任务平均估算价值上升了 27%。AI 编程工具的进化方向，不是「更会补全代码」，而是用户开始让它干代码之外的活了。

Core Finding 02

在 AI 面前，
你的资历归零了

报告里最锋利的发现来了，连定义本身都很有意思。

Anthropic 给每个会话里的用户打了一个「专业度」评分，从新手到专家五个级别。但这个「专家」的定义跟我们以为的完全不一样——它不看职位头衔，不看从业年限，不看学历证书。它只看你在这一次具体任务里的具体表现，三个信号：

01
指令精确度
目标、边界、验收标准有没有说清
02
验证意识
知不知道该让 AI 验证什么
03
纠偏能力
AI 理解错了能不能发现并纠正

一个不会 Python 的会计，能清清楚楚地告诉 Claude Code 对账脚本必须执行哪些规则，能抓住月末结账时的边界错误——在这个任务上，她就是专家。反过来，一个资深软件工程师，如果对 Rust 不熟悉，那他在 Rust 相关任务上就是新手。不是资历浅，是在这个具体任务上指令质量就是新手水平。

"一大批只会管人的中高层领导，在 AI 面前，就要被归为「职场新手」了。做了 20 年产品经理，但如果只是跟 AI 说「你帮我做一下某个功能的产品需求文档」——在 AI 眼里就是一个产品经理新手，因为指令质量太低。"

报告做了控制变量：把工作类型、任务价值、时间、职业、模型家族全部拉平之后，专业度每升一级，Claude Code 的动作数增加 9%，输出增加 13%。趋势依然显著。好指令不是微操——好指令是高质量授权。

甩手掌柜型
帮我写个脚本
AI 就给你写个脚本。对不对账、规则对不对，它不管。结果不能用。
高质量授权型
帮我写个对账脚本，必须校验这三条规则，月末结账的边界情况要单独处理
AI 拿到就能自己跑出一条完整的路来。一次通过。

管理职业的验证成功率比软件工程师还高（37% vs 34%）。拆任务、说清目标、委派、检查结果——这些原本是管理动作，现在也成了操作 AI 的技术动作。以前能管好人，现在也能管好 AI。

Core Finding 05

翻车后，
才是真正分水岭

所谓「遇到麻烦」，就是报错、测试失败、反复重试、用户表达不满。在复杂任务里，AI 出错是常态。但在遇到麻烦之后，不同专业度的用户，结局完全不同。

4%
新手救回率
VS
15%
专家救回率

遇到麻烦后，19% 的新手直接放弃——判定为失败且没写任何代码。而其他专业度级别的用户，放弃率只有 5-7%。4% 是什么概念？100 次翻车里，新手只有 4 次能救回来。

新手放弃率高，不是因为 AI 帮不了他们——恰恰相反，AI 就在那里等着，只要你给对方向它就能跑。但新手给不出方向，于是得出「AI 不好用」的结论。而专家遇到同样的麻烦，会说「你这里理解错了，应该是这样的」，然后继续推进。

Ending

AI 面前资历归零。
但你的专业能力没有。

"生产软件，可能正在从一种程序员专属的职业能力，变成一种普通工作能力。"
—— Anthropic 报告原文

律师会写合同检查脚本，还是一名律师；会计指挥 Python 对账，还是一名会计。编程不再是程序员的专利，而是各行各业解决自己问题的工具。方向是对的——未来不是人人都成为程序员，而是各行业懂问题的人，可以直接生产脚本、自动化流程、内部工具和技术文档。
