## 一、工作流执行记录（Step 1-4，每步卡壳点）

### Step 1 · 选 Hero（首命中标原则压测点）

**决策**：选 **V5 Pulse Hero**（深底 charcoal + 脉动线 + 3 KPI 条）。
**理由**：按 rules-hero 决策树「首命中标原则」，第 1 分支「数据/指标/仪表板封面 → Pulse」即命中。内容主轴优先于调性感受——本页主轴是「用性能数据证明品牌实力」，数据先命中；「传承仪式感」是调性词，只做 tie-breaker。这正是技能 v1.2 想验证的冲突裁决点：**「数据 + 仪式感」→ 走 Pulse（数据是主轴）**，规则裁决清晰可用。✓

**3 KPI 选择**：28000mm HT 防水 / 520h+ 实测 / 70% 回收材质——直接落入 Pulse 母版的 3 格 KPI 占位符，母版够用。

**卡壳点 H-S1（轻微）**：Pulse 母版的 KPI `<small>` 单位标签默认样式未定义下限，我自行补了 `font-size:1.4rem;color:var(--text-inverse-3)`。母版可补一个 `.kpi__num small` 默认态以防施工者遗漏。

### Step 2 · 取品牌 DNA / 页面类型裁决（⚠️ 核心盲区暴露）

**这是本次压测暴露的最大结构性缺口。**

paradigm-boundary Step 3 的 B 表给出**四类纯净页面预设**：品牌叙事页 / 数据仪表盘页 / 设计系统文档页 / 知识模板。**没有「品牌 + 数据混合」预设档**。Arc'teryx 品牌数据简报本质上就是混合页，规则无法直接套用任何一组联动选择：

| 维度 | 品牌叙事组建议 | 数据组建议 | 混合页的真实需求 | 规则状态 |
|------|--------------|-----------|----------------|---------|
| padding | 96-120px | 80px | 品牌段要大呼吸、数据段要高密度 | **未定义** |
| 分隔 | 背景色切换 | 发丝线 gap:1px | section 间背景色 + 组件内发丝线 | **W5 禁混用，但混合页必须混** |
| 宽度 | 单一 1120px | 三档 1120/960/720 | 品牌段单一、数据段需中/宽档 | **未定义** |
| 动画 | 无或 fade-in | pulse + hover | Hero pulse（数据）+ 品牌段无 hover | 部分可推 |
| 网格 | 大间距留白 | 发丝线 | 品牌段大间距、数据段发丝线 | **W9 也禁混用** |
| .cmp | 可省 | 强烈推荐 | 数据段有、品牌段省 | **B11 未定义混合处理** |

**我的裁决（无规则支撑，凭施工经验）**：
1. **页面级主分隔**统一用「背景色切换」（品牌叙事语言，贯穿全页，保持 section 间语言统一）；
2. **数据组件内部**（Data Table 行、Ring 卡间隙）局部用发丝线（数据语言，限定在组件边界内）；
3. **section padding 统一 96px**（`--s-4xl`），取两组的折中——既不低于品牌页底线，又比 120px 紧凑以容纳数据段；Statement Quote 段单独拔到 120px 做引言大呼吸。
4. **.cmp 自标注**：仅数据段（④）加，品牌段省。

这是一个「分层分隔」妥协方案：section 级不混用（全背景色切换），组件级可借用另一语言。**但规则里完全没写这种分层策略**，属于纯经验裁决——这是 v1.3 必须补的盲区（见缺陷 BLANK-1）。

### Step 3 · 叙事施工（衔接是主战场）

**8 段深浅节奏**（深段孤立 ✓，A10/N4 通过）：

```
① Pulse Hero(深) → ② 概念Prose(浅) → ③ Product Card(cream) → ④ Data+Ring(浅)
→ ⑤ Statement Quote(cream) → ⑥ Heritage Timeline(深) → ⑦ Principle(浅) → ⑧ Footer(深 black)
深段：①⑥⑧ 全部被浅/cream 隔开，无连续深色 ✓
```

**关键衔接：数据段(④) → 品牌段(⑥Timeline) 的过渡**
我用了 **⑤ Statement Quote（cream 浅底引言）做过渡桥**——④ 数据段结束后，不直接跳深色传承段，而是插一句浅底引言做「情绪缓冲」，让读者从「读数据」模式软切到「读故事」模式。

**卡壳点 N-S1（缺口）**：Statement Quote 的规则用途是「中段引言呼吸」，**没有明确说它能充当「数据→叙事过渡桥」**。这是我临时借用的，规则没背书。技能缺少一个明确的「数据段→品牌段过渡」衔接指引（见缺陷 GAP-1）。

**卡壳点 N-S2（深段配额冲突）**：混合页里，唯一的深段配额该给 Manifesto 还是 Heritage Timeline？
- Manifesto（价值观升华）需要深底 charcoal
- Heritage Timeline（传承时间线）在深底视觉冲击更强
- 但 A10 规定深段孤立，一页能放的深段有限（Hero 已占 1 个，Footer 占 1 个）
- 我的处理：放弃 Manifesto（把价值观用浅底 Principle Cards 承载），把深段配额给 Timeline
- **规则盲区**：没有「混合页深段配额优先级」指引（见缺陷 BLANK-2）

**padding 切换判断（B4 量化规则压测）**：B4 说「section 数 ≤7 且元素 ≤5 → 120px；section 数 8+ 或元素 6+ → 96px」。本页 8 段，按规则应 96px。但「元素数」在混合页里怎么算？Product Card 4 卡算 1 个 section 还是 4 个元素？Data Table 4 行 + 2 Ring 算几个？**规则没定义混合组件的元素计数法**（见缺陷 BLANK-3）。我统一取 96px 规避歧义。

## 二、数据组件与品牌组件共存的衔接方案

这是本次压测的核心交付。我的处理分三层：

### 层 1：section 级——背景色切换统一（不混用分隔语言）
全页 8 段的 section 间分隔**全部用背景色切换**（深→浅→cream→浅→cream→深→浅→深），这是品牌叙事语言，保持页面级统一。**不**在 section 间用发丝线分隔——避免 W5「一页混用两种 section 分隔策略」违规。

### 层 2：组件级——数据组件内部允许发丝线
Data Table 的行间 `border-top:1px solid var(--border-rest)`、Ring 卡之间的 grid gap——这些是**组件内部**的数据语言，不算 section 级分隔。这样既满足数据段的高密度发丝线需求，又不破坏 section 级的统一性。

**这是「分层分隔」策略**：section 边界用背景色，组件内部用发丝线。规则没写这种分层，但这是混合页唯一可行的解法。

### 层 3：过渡桥——用品牌组件做数据→叙事的软切
④ 数据段（Data Table + Ring）结束后，用 ⑤ Statement Quote（浅底引言）做过渡。引言内容是「我们不为了轻而轻……每一处取舍都必须能在风暴里自证」——既呼应数据（可量化），又开启叙事（价值观），是天然的语义桥梁。

**衔接点的 padding 处理**：④ 数据段 96px（紧）、⑤ 引言段 120px（松）。这个「紧→松」的 padding 切换本身就是过渡信号——读者在数据段感受密度，到引言段突然获得呼吸空间，心智模式完成切换。

**结论**：现有组件能完成衔接，但**缺少明确的过渡组件/指引**。Statement Quote 是被「借用」的，不是被「设计」来做过渡的。
