# 叙事节奏规则卡

> 从 6 个 Haglöfs 范式样本中提取的叙事节奏与 Section 组织规则。
> 样本：H042(Volvo 传承) · H065(案例研究) · H066(传承展馆) · R3-数据分析中心 · R3-品牌声明 · R3-品牌展示
>
> 来源背景（阶段1已验证）：section 数稳定 7-8 个；间距梯队 2/4/8/16/24/32/48/64/96px；Hero 均 100vh。

---

## 1. Section 组织模式（典型顺序模板）

### 1.1 品牌叙事模板（H042 / H065 / H066 / R3-声明 / R3-展示）

6 个品牌类样本的 Section 顺序归纳出一条通用弧线：

```
① Hero（100vh，品牌声明）
  → ② 概念建立（核心张力 / 核心脉络 / 引言金句）
  → ③ 资产展示 ×2-4（色彩 / 字体 / 产品线 / 材料 / 标识…）
  → ④ 时间维度（传承时间线 / 里程碑，可选）
  → ⑤ 价值观升华（原则 / 可持续 / CTA 金句）
  → Footer（深色收尾）
```

各样本的实际 Section 序列对照：

| 序位 | H042-Volvo | H065-案例研究 | H066-展馆 | R3-品牌声明 | R3-品牌展示 |
|------|-----------|-------------|----------|-----------|-----------|
| ① Hero | Reveal Hero L112 | Statement Hero L584 | Gallery Entrance L286 | 经典居中 Hero L312 | 色卡 Hero L650 |
| ② 概念 | Grille 产品线 L129 | Quote 金句 L630 | Heritage Frieze L303 | Tension 张力 L351 | Tension 组件 L677 |
| ③ 资产 | Safety 安全 L159 | Tension L641 | Artifact 展品 L347 | Statement 金句 L383 | List 列表 L722 |
| ④ 资产 | Interior 材料 L193 | Identity 视觉 L687 | Spotlight 聚光 L377 | Color 色彩 L393 | Prose 排版 L760 |
| ⑤ 资产 | Performance 数据 L221 | Color 色彩 L725 | Era 时代 L396 | Typography L446 | Components L798 |
| ⑥ 时间 | Heritage 时间线 L251 | Stats 数据 L749 | Collection 藏品 L441 | — | — |
| ⑦ 价值观 | Sustainability L281 | Timeline L771 | Trail 路线 L482 | — | — |
| ⑧ 收束 | Footer L306 | Principles→CTA→Footer L801/836/845 | Footer L541 | Footer L471 | Footer L881 |

**可提取的结构规则：**

- **R1-1 概念建立必在第二个位置**：6 个样本中 5 个（H065/H066/R3-声明/R3-展示 对应概念 section，H042 用产品线做概念锚点）的第二段都是"概念建立"——核心张力 / 金句 / 时间线概览 / 组件概览。**Hero 之后不允许直接跳到细节展示。**
- **R1-2 价值观/CTA 收束段是品牌页的标配**：H042 用 Sustainability，H065 用 Principles+CTA，R3-声明用 Statement 金句。品牌页必须有"升华"段，不能以资产展示直接收尾。
- **R1-3 时间维度段是可选层**：H042(Heritage Track)、H065(Timeline)、H066(Frieze+Era) 三者有；R3-声明/展示 没有——因为它们是设计系统展示而非品牌历史叙事。**传承类品牌页必须有时间线段；系统展示类不需要。**

### 1.2 数据叙事模板（R3-数据分析中心）

```
① Pulse Hero（100vh，深色 + 底部嵌 3 KPI 数据条）
  → ② 方向指标（Compass 趋势：方向箭头 + 柱状微图）
  → ③ 趋势曲线（Elevation 剖面：3 条 SVG 路径）
  → ④ 目标对比（Seam 基线：进度条 + 目标标记线）
  → ⑤ 数据分层（Layer Stack：5 层 + hover 反转）
  → ⑥ 指标卡（Swatch Cards：4 卡 + 点阵进度）
  → ⑦ 频度分布（Grid Matrix：12 格密集网格）
  → ⑧ 发现与行动（Callout 关键发现 + Action 行动列表）
  → Progress 进度条 + Footer
```

**数据叙事的弧线本质**：从"方向"（应该往哪走）→ "趋势"（历史在变好）→ "目标差距"（还差多少）→ "结构"（数据由什么构成）→ "分布"（细节频度）→ "洞察行动"（所以呢）。这是一条**从宏观到微观再到决策**的漏斗。

## 2. 信息密度梯度（量化）

### 2.1 量化方法

「屏元素数」= 一个 section 内用户可识别的信息单元数量（卡片 / 数字 / 段落 / 图表 / 数据格）。密度等级：极疏(1-2) / 疏(3-4) / 中(5-8) / 密(9-15) / 极密(16+)。

### 2.2 品牌叙事密度曲线

```
H042-Volvo:     疏(4) → 中(4) → 中(4) → 中(3) → 中(4) → 中(4) → 中(3) → 疏(2)
R3-品牌声明:     疏(2) → 中(4) → 极疏(1) → 中(5) → 疏(3) → 疏(2)
R3-品牌展示:     中(4) → 中(4) → 中(3) → 中(5) → 中(8) → 疏(2)
H065-案例研究:   中(6) → 极疏(1) → 中(4) → 中(4) → 密(10) → 中(4) → 中(4) → 中(5) → 疏(2) → 疏(1)
H066-展馆:       疏(4) → 密(7) → 中(3) → 极疏(1) → 中(3) → 中(4) → 密(7) → 疏(2)
```

**品牌页密度规律：**

- **R2-1 Hero 恒疏**：所有品牌页 Hero 元素数 2-6 个，标题 + 描述 + 1-2 装饰元素。**不允许 Hero 内堆数据网格。** 唯一例外是 H065 的 hero-meta（4 个数字），但那是叙事性元数据（1914/5/12/4）而非数据可视化。
- **R2-2 金句/引述段是"呼吸谷"**：H065 的 Quote（1 元素）、R3-声明的 Statement（1 元素）、H066 的 Spotlight（1 元素）——密度降到极疏（1 个核心元素）。品牌页每隔 2-3 个中密段必须插入一个极疏的"呼吸段"。
- **R2-3 品牌页峰值 ≤ 10 元素**：H065 色彩 section 10 个色块是品牌页最高密度，没有超过 10 的。品牌页信息载体是"概念卡"（3-5 个/段），不是"数据格"（12+ 格）。

### 2.3 数据叙事密度曲线

```
R3-数据分析中心: 密(4) → 密(6) → 密(6) → 密(8) → 密(5) → 密(4) → 极密(12) → 中(5) → 疏(2)
                  Hero    Compass  Elev    Seam    Layer   Swatch  Matrix  Insight  Footer
```

**数据页密度规律：**

- **R2-4 数据页 Hero 内嵌数据条**：R3 的 Pulse Hero 底部 `.hero__strip`（L318-334）直接放 3 个 KPI（94% / 82 / 10），不等用户滚动就给信息密度。品牌页 Hero 从不这样做——这是两类叙事的分水岭。
- **R2-5 数据页持续高压无呼吸谷**：8 个 content section 中 7 个是密/极密，仅在 Insight 段降至中等（callout + action）。**与品牌页的"呼吸式"（疏密交替）相反，数据页是"持续加压直到洞察释放"。**
- **R2-6 极密段（12 格矩阵）出现在第 ⑦ 位**：数据页的信息峰值不放在开头（Hero 虽密但有标题占位），而是放在倒数第二段——先用 6 段铺垫语境，最后用矩阵做全景概览，然后用 Insight 降密收束。

### 2.4 密度曲线对比（ASCII 波形）

```
品牌页:    ▁▂▂▁▂▁▂▁▂    (疏-中-疏-中交替，有呼吸谷)
           H C V C B C F

数据页:    ▃▃▃▃▃▃▅▂▁    (持续中高密 → 极密峰 → 降密收束)
           H C E S L S M I F
```

## 3. 留白节奏（量化规则 + 用什么变量）

### 3.1 两套间距体系并存

| 体系 | 样本 | 变量定义 | section padding |
|------|------|---------|----------------|
| **Token 化体系（规范）** | R3-数据中心 / H066-展馆 | `--s1:4px → --s24:96px`（12 级） | `var(--s20) var(--s8)` = 80px × 32px |
| **硬编码体系（遗留）** | H042 / R3-声明 / R3-展示 / H065 | 无空间 token | 120px（H042/R3-声明/R3-展示）/ 96px（H065） |

**R3-1 推荐统一为 Token 化体系**（R3-数据中心 / H066 的 `--s*` 梯队），原因：两个最新且密度最高的样本都用它，且支持三档容器宽度。

### 3.2 空间梯队（Token 化样本的完整定义）

R3-数据中心 L20 / H066 L15：

```css
--s1:4px;  --s2:8px;   --s3:12px;  --s4:16px;  --s5:20px;
--s6:24px; --s8:32px;  --s10:40px; --s12:48px; --s16:64px;
--s20:80px;--s24:96px;
```

### 3.3 量化留白规则

| 层级 | 间距值 | 用途 | 使用频率 | 出处 |
|------|--------|------|---------|------|
| **section 上下内边距** | 80px (`--s20`) | section 垂直呼吸 | 每 section | R3 L36 / H066 L34 |
| **section 左右内边距** | 32px (`--s8`) | section 水平安全边距 | 每 section | R3 L36 / H066 L34 |
| **section label → 内容** | 32px (`--s8`) | 标签到正文的间距 | 每 section | R3 L32 / H066 L27 |
| **section 标题 → 网格** | 32px (`--s8`) | 标题到主内容的间距 | 每 section | R3 `.sec--w` margin-top |
| **网格内卡片间距** | 16-24px (`--s4`-`--s6`) | 卡片 / 网格项之间 | 每网格 | R3 `.matrix__grid` gap:1px / `.swatch__grid` gap:`--s6` |
| **卡片内部 padding** | 24-48px (`--s6`-`--s12`) | 卡片内容到边框 | 每卡片 | R3 `.callout` `--s8` / H066 `.art` `--s12 --s8` |
| **段落间间距** | 12-16px (`--s3`-`--s4`) | 文字段落 / 列表项间 | 文本区 | R3 `.act-item` gap:`--s3` |
| **大数字到标签** | 4-8px (`--s1`-`--s2`) | KPI 数字与标签紧贴 | 数据卡 | R3 `.hm__num→.hm__lbl` `--s1` |

**R3-2 section 垂直间距 = 80px 是基准线**。两个 Token 化样本一致用 `--s20`=80px，不是 96px（阶段1 骨架测绘中 H061 用 96px，但那是另一个样本集；本批 6 个样本中 Token 化的 2 个都用 80px）。

**R3-3 硬编码样本用 120px**（H042 / R3-声明 / R3-展示），说明品牌展示类页面倾向比数据页更大的呼吸——品牌页 section padding 120px，数据页 80px。差异 ~33%。

**R3-4 网格间距采用"发丝线分隔"策略**：R3 数据页大量使用 `gap:1px` + `background:var(--bd)` 做网格分隔（`.matrix__grid` L237, `.elevation__charts` gap:`--s8`），而非用大间距留白。这是数据页压缩留白、提升密度的手法——用 1px 线替代 16px 间隙。

**R3-5 品牌页用大间距留白**：品牌页网格 gap 普遍用 40-80px（H042 `.material-row` gap:60px L52，`.sustain-grid` gap:60px L87），用空气感而非线条做分隔。

### 3.4 三档容器宽度（仅数据页）

```css
.sec--w { max-width:1120px }  /* 宽档：网格 / 矩阵 / 时间线 */
.sec--m { max-width:960px }   /* 中档：卡片 / 指标 */
.sec--n { max-width:720px }   /* 窄档：金句 / 长文 / 聚光灯 */
```

**R3-6 品牌页用单一宽度（无分档）**：H042 / R3-声明 / R3-展示 不设 max-width，靠 section padding 控制。品牌页不需要窄档因为内容以卡片网格为主。

## 4. 滚动叙事手法（每种：手法 → 样本出处 → 实现方式）

### 4.1 深浅背景色交替（制造滚动节奏的听觉）

- **样本**：R3-数据中心 / H042 / H065 / H066
- **实现**：section 背景色在深色（`--pk`/`--td`/`--bg-dark`）和浅色（`--bg`/`--surface`/`--bg-alt`）间交替
- **出处**：
  - R3-数据中心：Hero 深(L43 `.hero` bg:`--pk`) → Compass 浅(L84 `.compass` bg:`--sf`) → Elevation 深(L119 `.elevation` bg:`--pk`) → Seam 浅(L143) → Layer 浅(L181 `--sr`) → Swatch 浅(L208) → Matrix 深(L235 `--pk`) → Insight 浅(L254)。模式：**深-浅-深-浅-浅-浅-深-浅**
  - H042：Hero 深(L16 `--td`) → Grille 浅 → Safety 浅(alt) → Interior 浅 → Performance 深(glacier L63) → Heritage 浅 → Sustainability 深(L84 `--td`) → Footer 深
  - H065：Hero 浅 → Quote 浅 → Tension 深(L189 `.section-dark`) → Identity 浅 → Color 浅(alt) → Stats 深 → Timeline 浅 → Principles 浅(alt) → CTA 深 → Footer

**R4-1 深色 section 不连续出现超过 1 个**：所有样本的深色 section 都是孤立的，前后必有浅色 section 隔开。深色 section 的功能是"视觉重音"，连续深色会失去节奏。

### 4.2 渐显入场动画（Fade-up on load）

- **样本**：R3-品牌展示
- **实现**：`@keyframes fadeUp` (L587-590) 从 opacity:0 + translateY(30px) 到 opacity:1 + translateY(0)，配合 `.animate` 类和 `.delay-1/2/3` 阶梯延迟（0.1s/0.2s/0.3s）
- **出处**：R3-品牌展示 L587-599（CSS）/ L651/666/667/668/673（HTML 应用）
- **用途**：仅用于 Hero 区元素（tag / h1 / desc / year / dot-pattern）的入场，**不用于 section 滚动触发**（无 IntersectionObserver / scroll-trigger）

**R4-2 渐显仅限 Hero**：6 个样本中仅 R3-品牌展示 使用渐显，且仅作用于 Hero 内部元素。滚动途中的 section 不做渐显动画——靠背景色交替 + section padding 制造节奏，而非靠动画。

### 4.3 脉冲呼吸动画（Pulse）

- **样本**：R3-数据分析中心
- **实现**：`@keyframes pulse` (L55-56) opacity 0.3↔1，3s 周期，ease-in-out；应用于 `.hero__pulse`（L51-53），一条 absolute 定位的 1px 橄榄色横线在 Hero 顶部
- **出处**：R3-数据中心 L51-56
- **用途**：象征"活的数据"——仪表盘的心跳线。**仅用于数据页 Hero，品牌页不用。**

### 4.4 Sticky 定位（滚动时固定元素）

- **样本**：R3-品牌展示 / H065-案例研究
- **实现**：
  - R3-品牌展示：`.nav { position:sticky; top:0 }` (L48) 导航栏粘性 + `.type-anchor { position:sticky; top:80px }` (L251) 字体大字母滚动时固定
  - H065：`nav { position:fixed; top:0 }` (L36) 导航固定（fixed 而非 sticky）
- **用途**：导航始终可见 + 大字母在滚动字体列表时形成视差锚点
- **限制**：sticky 大字母仅用于"长列表浏览"场景（字体展示 / 产品列表），不适合卡片网格场景

### 4.5 横向滚动（Horizontal scroll track）

- **样本**：H066-传承展馆
- **实现**：`.frieze__track { display:flex; overflow-x:auto; scrollbar-width:none }` (L77-81)，7 个 `.frieze__item`（flex-shrink:0, width:200px）水平排列，隐藏滚动条
- **出处**：H066 L77-81 / L307-343
- **用途**：时间线横向浏览——打破纵向滚动的单一节奏，用横向维度承载"时间流动"语义

**R4-3 横向滚动仅用于时间线**：6 个样本中仅 H066 的 Heritage Frieze 使用横向滚动，且语义是"时间轴"。不要在普通卡片网格中使用横向滚动。

### 4.6 Hover 反转（密度感交互）

- **样本**：R3-数据分析中心 / H042
- **实现**：
  - R3 `.lyr:hover` (L188)：Layer Stack 的行从浅色背景 + 深色文字反转为深色背景 + 白色文字 + 橄榄色数字变橙色——全行视觉反转
  - H042 `.grille-item:hover` (L31)：背景色微变（hover bg）
  - H042 `.grille-item.active` (L35)：选中态整块深色反转
- **用途**：数据页用 hover 反转制造"可探索"密度感，不增加额外元素

### 4.7 进度指示器（滚动位置反馈）

- **样本**：R3-数据分析中心 / H066-传承展馆
- **实现**：
  - R3 `.ptrl` (L271-279)：7 个圆点 + 1 个当前态（椭圆），absolute 在 section 之间独立浮动（L653-662）
  - H066 `.trail__path` (L221-250)：7 个站点 + 连接线 + 已访问/当前/未访问三态，作为独立 section
- **用途**：告诉用户"你在叙事的哪个位置"——数据页用进度点（已完成/当前），展馆页用参观路线（空间隐喻）

### 4.8 巨字水印（装饰性深度层）

- **样本**：全部 6 个（标配手法）
- **实现**：absolute 定位的超大字号文字，极低透明度（0.02-0.15），pointer-events:none
- **出处**：
  - H042 `.hero-reveal__bars` (L17-18)：1px 横线条纹阵列
  - R3-数据中心 `.hero::before` (L46-50)：content:'ANALYTICS' clamp(8rem,20vw,22rem) opacity:0.02
  - H065 `.hero-year` (L171-181)："1914" 140px opacity:0.06
  - H066 `.entrance__seal` (L68-73)：圆形印章 EST 1914
  - R3-声明 `.hero-year` (L100-112)："1914" 180px opacity:0.12
  - R3-展示 `.hero-year` (L146-157)："2024" 180px opacity:0.08

**R4-4 Hero 必须有至少 1 个装饰性深度层**：6 个样本无一例外。裸标题（无水印 / 无图案 / 无装饰块）的 Hero 违反范式。

## 5. 品牌叙事 vs 数据叙事对比表

| 维度 | 品牌叙事（H042/H065/H066/R3-声明/R3-展示） | 数据叙事（R3-数据中心） | 规则编号 |
|------|------------------------------------------|----------------------|---------|
| **Section 数量** | 4-7 个 content section（弹性大） | 8 个 content section（固定） | R5-1 |
| **Hero 信息量** | 标题 + 描述 + 装饰（2-6 元素） | 标题 + 3 KPI 数据条（4 元素，但含数字） | R5-2 |
| **Hero 底部** | 装饰水印 / 角落 badge / dot 图案 | **3 列 KPI 数据仪表条**（`.hero__strip`） | R5-2 |
| **中段节奏** | 疏密交替，每 2-3 段插入极疏"呼吸谷" | 持续中高密度，无呼吸谷 | R5-3 |
| **金句/引述段** | 必有（独立 section，1 个核心元素） | 无（融入 Callout 内） | R5-4 |
| **时间线段** | 传承类必有（Frieze/Track/Timeline） | 不需要 | R5-5 |
| **色彩/字体段** | 常见（资产展示） | 不需要 | R5-5 |
| **数据可视化** | 少（仅简单数字/静态卡片） | 7 种：柱状微图 / SVG 曲线 / 进度条 / 目标线 / 分层条 / 点阵 / 矩阵 | R5-6 |
| **收尾方式** | CTA 按钮 / 价值观金句 / Footer 金句 | Callout 关键发现 + Action 行动列表 | R5-7 |
| **section 标签** | `.section-label`（通用标签） | `.cmp` 组件自标注（S2·*Compass Trend*—中文说明） | R5-8 |
| **section padding** | 120px（大呼吸） | 80px（高密度） | R5-9 |
| **网格间距** | 40-80px 大间距留白 | 1px 发丝线分隔 | R5-10 |
| **容器宽度** | 单一（靠 padding 控制） | 三档（1120/960/720px） | R5-11 |
| **背景色交替** | 深-浅交替，品牌资产段多为浅色 | 深-浅-深交替，数据密集段倾向深色 | R5-12 |
| **动画类型** | 渐显入场（仅 Hero） | 脉冲呼吸（Hero） + hover 反转（全程） | R5-13 |
| **进度指示** | 无（或展馆路线隐喻） | 进度圆点（已完成/当前） | R5-14 |
| **字体角色** | 衬线标题叙事 + 正文 + Mono 标签 | 同左，但 Mono 大量用于数字/数据 | R5-15 |
| **叙事弧线** | 声明 → 概念 → 证据 → 价值观（归纳式） | 方向 → 趋势 → 差距 → 结构 → 分布 → 决策（漏斗式） | R5-16 |

## 6. 禁忌

### 硬禁忌（违反则破坏范式 DNA）

- **❌ F1 不能连续 2 个深色背景 section**：所有样本的深色 section 前后必有浅色隔开。深色是"视觉重音"，连续重音 = 无节奏。（出处：6 个样本的背景色序列均无连续深色）
- **❌ F2 Hero 不能无装饰层**：裸标题（无水印 / 无图案 / 无几何块 / 无 hero-tag 线）的 Hero 违反全部 6 个样本的实践。Hero 必须有 ≥1 个装饰性深度层（R4-4）。
- **❌ F3 Hero 后不能直接跳到高密度数据展示**：品牌页 Hero 后第二段必须是"概念建立"（张力 / 金句 / 概述），不能直接放数据网格 / 表格 / 12 格矩阵。（R1-1）
- **❌ F4 不能连续 3 个高密度 section（品牌页）**：品牌页每 2-3 个中密段必须插入一个极疏"呼吸谷"（金句 / 引述 / 聚光灯）。（R2-2）数据页不受此限——数据页的叙事本质是持续加压。
- **❌ F5 Footer 不能用浅色背景**：6 个样本的 Footer 全部使用深色背景（`--pk` / `--td` / `--bg-dark` / `--text-display`）。浅色 Footer 打破了"深色收尾"的闭合感。（出处：H042 L94 / R3 L282 / H066 L253 / H065 L525 / R3-声明 L275 / R3-展示 L555）
- **❌ F6 section 数量不能超过 8 个 content section**：6 个样本的 content section 数：H042=7, H065=9(含 quote/CTA), H066=7, R3-数据中心=8, R3-声明=4, R3-展示=5。中位数为 7。H065 达到 9 是因为它含 quote 和 CTA 两个过渡段。**超过 8 个实质 content section 需拆页。**（与阶段1结论一致：7-8 个是叙事容量基准）

### 软禁忌（违反则降低品质，不致命）

- **⚠️ W1 品牌页不应在 Hero 内嵌 KPI 数据条**：Hero 内嵌数据仪表条（`.hero__strip`）是数据页 Pulse Hero 的专属手法。品牌页 Hero 放数据条会混淆叙事类型。（R5-2）
- **⚠️ W2 数据页不应有"纯金句"独立 section**：品牌页的 Statement/Quote/Spotlight（1 元素极疏段）是呼吸节奏的产物。数据页不需要——数据页的"呼吸"是 Insight 段的 callout，而非空旷的金句页。（R5-4）
- **⚠️ W3 不应用硬编码 px 替代空间 token（新页面）**：H042/R3-声明/R3-展示 用硬编码 120px 是遗留风格。新建页面应使用 `--s*` token 体系。（R3-1）
- **⚠️ W4 section padding 不应低于 80px**：80px（`--s20`）是数据页的压缩极限。品牌页用 120px。低于 80px 会导致 section 间无呼吸。（R3-2）
- **⚠️ W5 网格 gap 不应混用"大间距"和"发丝线"两种策略**：一个页面内要么全用大间距留白（品牌页），要么全用 1px 发丝线（数据页），不要在同一页面混用——会破坏间距语言的统一性。（R3-4 / R3-5）
- **⚠️ W6 横向滚动不应用于非时间线场景**：`.frieze__track` 横向滚动仅适用于时间线/时间轴语义。卡片网格横向滚动会破坏纵向阅读习惯。（R4-3）
- **⚠️ W7 渐显动画不应超出 Hero 区**：`.animate` + `delay-*` 仅限 Hero 入场。section 滚动途中的渐显需要 IntersectionObserver，但 6 个样本均未实现——靠背景色交替 + padding 制造节奏。（R4-2）
