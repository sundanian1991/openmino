The Method

为软件团队设计的纪律。

Linear 的产品哲学不是功能堆叠，而是四条互相咬合的工作纪律——它们决定了每一次按键如何被响应、每一次同步如何发生。

01

键盘优先

每个操作都有快捷键，手不离键盘。命令面板、⌘K、流式输入——工具应跟随手的速度，而非拖慢思考。

02

实时同步

变更在毫秒内推送到每个协作者。没有刷新、没有冲突提示，团队始终在同一张事实表上工作。

03

原生集成

GitHub、GitLab、Figma、Slack 双向同步。代码提交即关闭 issue，设计变更即更新任务，无中间人。

04

极速渲染

50ms 内完成界面渲染。性能不是优化项，是从第一行代码就锁定的工程约束——慢的工具会拖慢团队。

三个原语，一套节奏。

Linear 的产品不是功能列表，而是三个互相衔接的工作原语——周期驱动节奏，项目承载交付，路线图指向方向。

Design Tension

两条纪律之间的张力。

Linear 的克制不是减法，是对立的平衡——键盘的效率与界面的可读、自动化的速度与人的可控，在每一处交互里同时成立。

01 · Keyboard
键盘优先
Keyboard First

每个动作都有快捷键，命令面板 ⌘K 一键触达。工具跟随手的肌肉记忆，而非强迫眼睛去找按钮。

02 · Pointer
鼠标亦通
Pointer Friendly

所有键盘能做的事，鼠标也能做。效率面向专家，可读性面向新人——不牺牲任何一方。
