01 · Principles

设计原则

四条原则定义 Mino 的立场。它们不是限制创意，而是保护一致性——当两个方案都"可以"时，原则替你做决定。

01
Function First

功能优先

每个元素必须先回答"它解决什么问题"，再谈美感。装饰是功能的副产品，不是出发点。好看的界面来自好用的结构。

02
Restraint is Intent

克制表达

层次靠色调、边框与间距，不靠渐变与阴影。留白不是没想好放什么，是决定不放。填充是最容易的冲动，也是最廉价的。

03
System Coherence

系统一致

间距走梯队、色值走 token、字体走配方。一致性来自纪律而非灵感——同一个按钮在任何页面都该长一样。

04
Sustainable

可持续

为长寿命设计。克制的视觉不易过时，稳定的系统降低维护成本。今天少加一处装饰，明天少还一笔技术债。

04 · Spacing

间距梯队

11 级 8px 基准梯队。任何 padding / margin / gap 必须取自此梯队——禁用 5/7/10/12/13/20/40/80px 这类非梯队值。条形长度即真实间距比例。

05 · Components

组件目录

8 个核心组件，覆盖界面 90% 的常见模式。点击展开查看用途与禁忌。

触发主要操作的载体。哑光克制语境下，按钮靠边框与字重区分层级，不靠阴影。主按钮实色填充 forest，次按钮描边，文字按钮仅 hover 显底色。 交互核心
表单数据采集。默认 1px sand 边框，focus 态边框转 stone 并轻微提亮底色。错误态用 signal-red 边框 + 红色提示文字，禁用态置灰。 表单
信息聚合容器。white 底 + 1px sand 边框 + radius 4px。hover 时边框转 stone 或底色转 cream，禁止用阴影做悬浮感。 容器
状态或分类标记。JetBrains Mono 等宽 + 全大写 + 0.06em 字距，1px 描边 pill 形。状态语义染 forest / stone / signal-orange，禁用裸文字色。 标记
结构化数据陈列。表头 mono 大写，行 hover 高亮 cream，列数 ≤5。状态列用 Tag 组件承载，不可裸色文字。 数据
临时任务焦点。遮罩用 black 低透明度，弹体 white 底 + radius 8px（最大允许）。克制使用——打断流式操作是高成本交互。 覆盖层
操作反馈通知。charcoal 底 + offwhite 文字，顶部或底部居中浮现，自动消失。不强制打断，区别于 Modal。 反馈
页面或区域定位。顶部 sticky 或左侧 sidebar 两种范式，当前页本侧边栏即其实现。锚点走 smooth scroll，当前项高亮 moss。 结构
06 · Usage

使用规范

核心规则速查。违反任意一条视为出系统——一致性来自纪律。

规则 要求 数值约束 级别
底色 页面主底用 offwhite 温润米白，禁纯白 / 冷灰 #F5F3EF 红线
间距 所有间距走 11 级梯队 token，禁非梯队值 1/2/4/8/16/24/32/48/64/96/120 红线
色值 全 var(--color-*) 引用，无裸 hex 硬编码 15 色 + 别名 红线
forest 主强调仅小面积点缀，无满铺底色 ≤8 处 约束
Signal 信号色稀缺，blue 仅用于链接 ≤2 处 约束
字体 四字体角色不混，正文不用衬线 4 配方 红线
圆角 哑光克制，无大圆角 ≤8px 约束
层次 靠色调 + 边框 + 间距，禁渐变 / 模糊 / 装饰阴影 — 强约束
