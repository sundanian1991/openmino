在 AI 转型方案中的定位

L1

数据底座

jdme-local-tools（京ME 群聊采集 + 加密数据库直读）+ read-joymeeting（慧记会议全文 + 逐字稿）+ read-email（邮件完整访问）→ 覆盖方案中「群消息采集通道」「每日业绩自动化」「邮件报告生成」三个数据入口

L2

可信知识库

全量消息存档 + 完整会议转写 + 邮件正文归档 → 对应方案中五库知识库体系的数据填充

L3

Skills 能力层

personal-insight-summary → 日/周工作简报（/日简报 P1）+ jingME → 供应商承诺追踪 + 会议待办闭环（/整改跟踪 + /供应商沟通准备）→ 这是准确率从 21% 到 95%+ 的关键杠杆

L4

验证层

人工抽样（每周）+ 离线评估（每月）+ 衰变监控（每月）+ 规则校准（每季度）

jdme-local-tools · macOS 验证结果（2026-06-27）

  • chat.data 和 session.data 可读，密钥 md5("sundanian").hexdigest() 有效
  • 发现 247 个有名字的群聊（含供应商管理相关群：大额税票贷新供应商述标、非金融供应商寻源合规流程、JD-零犀代运营项目沟通群等）
  • summary 命令成功拉取最近 3 天消息流，messages 和 chat-history 均可正常输出 JSON Lines
  • sessions 命令列出完整会话索引（含 kind=512 群聊、kind=1024 商家群、kind=0 单聊）
