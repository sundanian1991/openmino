定位

本地 JD ME / 京 ME 聊天数据的只读本地检查工具。macOS 和 Windows 双平台支持。三个文件分工明确：入口（13 行）→ 核心（2659 行）→ 平台（140 行），OS 特定路径发现独立于核心数据逻辑。

核心能力（★★★★★ 数据深度）

数据库直读

SQLCipher 加密 chat.data / session.data 完整访问。密钥 = md5(ERP账号).hexdigest()，匹配 JD ME 自身 db-helper。先试 sqlcipher CLI，Windows 上无 CLI 时用 ctypes 调 DLL 回退。

日志恢复

从 sdk-event.log + renderer.log 恢复消息（容忍 JD ME 自身畸形 JSON 转义——正文里嵌未转义双引号）。Windows 上额外扫 renderer.log 快照。

实时监听

watch（日志 tail）+ chat-watch（数据库轮询）。支持关键词/@我/多 session 过滤。常驻进程，新消息即推。

LLM 专供

summary 命令专为 LLM 日报设计：按时间窗口拉消息，自动分类 private/named_group/at_me_other，默认排除告警/通知/robot 噪音群。输出 JSON Lines，可直接喂给 LLM。

13 个命令完整清单

通道 1：日志恢复（无需 sqlcipher）

命令功能数据源
discover打印发现的 JD ME 应用、数据、日志路径和账号。报告 app root、ME binary、support roots、Electron roots、log roots、SQLCipher 候选、数据账号、日志账号和 recommended_account文件系统扫描
messages从 sdk-event.log 恢复纯文本消息（容忍 JD ME 的畸形 JSON）。支持 --contact/--session/--keyword/--at-me/--limit/--rotations 过滤sdk-event.log + renderer.log
watch监控 renderer.log + sdk-event.log，解析消息推送、会话更新和 selected-session 详情到稳定 JSON。支持 --keyword/--at-me/--self-pin/--session/--timeout日志 tail

通道 2：数据库直读（需 sqlcipher CLI 或 DLL）

命令功能数据源
sessions列出 session_info（id、name、kind、type、option）。支持 --name-contains 和 --kind 过滤（0=p2p, 512=group）session.data（SQLCipher）
chat-history从 chat.data 读取解密消息。支持 --session/--sender/--since/--until/--msg-type 过滤。输出 JSON Lineschat.data（SQLCipher）
chat-search在 msg_json 和 search 列中做子串搜索。支持 --keyword（OR 匹配，可重复）。输出 JSON Lineschat.data（SQLCipher）
chat-watch轮询 chat.data，匹配关键词/@我/发送者/会话后实时输出。支持 --keyword/--at-me/--self-pin/--sender/--session/--interval/--timeoutchat.data 轮询
chat-latest显示最近 N 条跨会话解密消息。可选 --session 过滤chat.data（SQLCipher）
summary按时间窗口拉消息，自动分类 private/named_group/at_me_other，专为 LLM 设计。默认包含 Relay|Zero|基础服务，排除告警/通知/robot 噪音群chat.data（SQLCipher）

通道 3：混合诊断

命令功能数据源
history报告可读的导出 .txt 文件和 chat.data* 数据库候选。非 SQLite 可打开的标注 encrypted_or_unknown文件系统扫描
query在可读的导出文件和账户日志中搜索联系人 ERP/姓名，返回片段、直接会话 ID 和加密数据库边界导出文件 + 日志
preview诊断回退：解析会话 ID，先试运行/原生化 JD ME bridge，再回退到日志预览。sdk_context_unavailable 或 native bridge not found 视为桥接诊断JD ME SDK bridge + 日志

消息过滤能力

--contact

按联系人 ERP/pin/姓名匹配 sessionId, senderPin, receiverPin, content

--session

精确会话 ID 过滤（可重复）

--keyword

内容子串过滤（可重复，OR 匹配）

--at-me + --self-pin

@我 过滤

--limit

限制返回条数

--rotations

扫描轮转日志文件（sdk-event.1.log..N.log）

summary 命令：专为 LLM 日报设计

默认包含

Relay | Zero | 基础服务

默认排除

告警 | 预警 | 监控 | 超时 | 失败 | 事件 | 通知 | 审核中心 | robot | 爱回收 | 客服

输出字段

sessionId, sessionName, sessionKind, categories（private / named_group / at_me_other）, sender, timestamp, text, msgType, mid

安全边界（重要）

  • 不修改 JD ME 应用包/安装目录
  • 不注入 Electron
  • 不从运行进程内存提取密钥
  • 不写入 JD ME 聊天数据库
  • 不尝试凭证提取
  • 只读本地文件，用户已有权限访问
