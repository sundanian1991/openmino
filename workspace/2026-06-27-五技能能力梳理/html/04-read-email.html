定位

Mail 邮件统一读取、查询、梳理、总结、回复、发送、转发、文件夹管理与批量处理入口。优先用于绝大多数邮件相关场景。使用统一 SOAP 查询入口完成邮箱目录浏览、邮件列表查询、邮件详情读取、邮件总结展示、基于原邮件回复及常见高级筛选场景。

27 个命令完整清单

读取查询（6 个）

命令功能
folders浏览目录树，支持 --parent/--parent-id/--traversal/--limit
search在单个文件夹中抓取邮件列表并做客户端过滤（支持 --folder/--sender/--recipient/--subject/--body-keyword/--after/--before/--read/--unread/--category/--has-attachments/--importance/--max-pages）
detail按 ItemId 获取邮件详情（正文、收发件人、附件元数据）
daily-stats按天统计邮件（日期/星期/邮件总数/未读数/已读数），默认按 date_time_received 统计
pending-important未回复的重要邮件（importance=High 或分类含 important/重要/urgent/待办，且同一 ConversationId 下没有更晚的已发送邮件）
person-unhandled特定人员发来的未查看未处理邮件（发件人匹配，邮件当前未读，同一 ConversationId 下没有更晚的已发送邮件）

写入操作（4 个）

命令功能
reply基于一封已有邮件回复（支持 --reply-all/--include-original/--attachments），发送前必须二次确认
send新建一封全新邮件（支持 --to/--cc/--bcc/--subject/--body/--body-type/--importance/--attachments），发送前必须二次确认
forward转发一封已有邮件（支持 --to/--cc/--bcc/--body/--attachments），发送前必须二次确认
lookup-recipient通过 ERP / 姓名 / 模糊关键词查收件人（调 POST http://joymail-skill.jd.com/hr/user/queryByErp），发送前必须解析

批量编辑（9 个）

命令功能
batch-mark-read批量标记已读
batch-mark-unread批量标记未读
batch-add-category批量打标签
batch-move批量移动到指定文件夹
batch-delete批量删除
flag单封旗标（支持 --start-date/--due-date）
unflag单封取消旗标
batch-flag批量旗标
batch-unflag批量取消旗标

智能总结（2 个）

命令功能
summarize从 stdin 读取 search/pending-important/person-unhandled/detail 的 JSON 输出，生成 Markdown 总结（核心统计/总览表格/建议优先处理/简要观察）
search + summarize 管道search --folder inbox --unread --limit 20 | summarize，直接生成可读摘要而非原始 JSON

确认规则（所有修改操作必须二次确认）

适用范围：send / reply / forward / batch-mark-* / batch-add-category / batch-move / batch-delete

鉴权链路（自动化，最多重试 3 次）

1. SSO Token

优先 SSO_TOKEN，其次 ME_TOKEN/SO_TOKEN，再尝试从 ~/.joyclaw/openclaw.json 中提取

2. clawJwtLogin

POST http://joymail-skill.jd.com/sso/clawJwtLogin
Cookie: sso.jd.com=<sso_token>。取 data.token

3. mailpost

auth: <token> + Authorization: Bearer <token>
默认 endpoint：https://mail-skill.jd.com/mail/api/clawmail/mailpost

4. 401? 自动重试

重新调用 clawJwtLogin 刷新 token，默认最多重试 3 次
