## 三、问题归类与根因分析

### A. 渲染层问题（4/5 数据集受影响）

| 问题 | 影响数据集 | 根因 |
|------|-----------|------|
| **高亮编码丢失**：spec JSON 契约中 highlight 字段未被渲染器执行，全部回退默认色 #c26d3a | #81, #92 | viz-echarts 渲染器未正确解析 visualEncoding.highlight 字段 |
| **标注缺失**：spec JSON 中 annotations 数组未被渲染 | #03, #81, #92 | viz-echarts 未实现 annotations 渲染 |
| **参考线缺失**：spec JSON 中 referenceLines 未被渲染 | #03, #92 | viz-echarts 未实现 referenceLines 渲染 |
| **chartType 错误**：bubble_chart 被渲染为 bar，data 为 null | #98 | viz-echarts 对 bubble_chart 支持不完整或契约字段不匹配 |
| **多图缺失**：spec 要求 2 图，HTML 只有 1 图 | #98 | viz-echarts 不支持多图并列渲染 |
| **X轴标签异常**：#03 的日期数据被渲染为 1-28 数字索引 | #03 | 时序数据处理问题 |

### B. Phase 4 清单自身可检测到的问题

清单在检查项 7（视觉编码）和 9（高亮）时，有效识别了渲染层编码丢失。检查项 10（标注）有效识别了标注缺失。说明**清单的检测能力是有效的**。

### C. 上游规范问题

| 问题 | 说明 |
|------|------|
| spec JSON 契约与 viz-echarts 实际支持能力不匹配 | 契约定义了 annotations、referenceLines、多图表等字段，但渲染器未实现。Phase 3 产出了渲染器无法执行的规格。 |
| Phase 3 到 Phase 4 之间缺少"渲染能力校验"环节 | spec 写了但渲染器做不到，直到 Phase 4 才发现 |

## 四、Checklist 自身缺陷评估

### 检测到的缺陷

| # | 缺陷 | 严重程度 | 说明 |
|---|------|---------|------|
| 1 | **无渲染能力校验项** | 高 | 清单检查的是"渲染结果 vs spec 意图"的偏差，但没有一项检查"spec 中的契约字段是否被渲染器支持"。这导致 #81/#92 的高亮丢失、#03/#92 的标注/参考线缺失，虽然被标记为 FAIL，但根因不在 viz-design 而在 viz-echarts。清单无法区分"设计错误"还是"渲染错误"。 |
| 2 | **无多图校验项** | 高 | 清单假设单图场景，没有检查"spec 要求的图表数量是否全部渲染"。#98 要求 2 图只渲染 1 图，如果不是人工核对很容易漏掉。应新增"图表数量与 spec 一致"检查项。 |
| 3 | **Item 14 Source ID 检查模糊** | 中 | 标准是"每项决策有出处可追溯"，但所有 5 个数据集的 spec.md 都有 Source ID 清单表，所以全部 PASS。但实际上 source ID 应该能从 chart.html 追溯到具体决策，HTML 中不包含任何 source ID。检查项应该明确检查目标文件。 |
| 4 | **Item 18 视觉重量差在单色场景误判** | 中 | 当全部元素同色时（#81/#92），仅靠长度/面积差异判定视觉重量差。按严格标准"hero 面积 ≥ background 面积 3倍"，最长条/最短条长度比确实 ≥3倍，所以 PASS。但这与"高亮编码丢失"的 FAIL 矛盾 — 同一个颜色问题，在 Item 7 判 FAIL，在 Item 18 却可能 PASS。 |
| 5 | **缺少"数据渲染完整性"检查** | 中 | 清单没有专门检查"spec 中的数据是否全部渲染到 HTML 中"。#98 的 data:[null] 如果不是人工检查 ECharts option 很容易漏。应新增"数据完整性：spec 数据点数 = HTML 数据点数"。 |
| 6 | **PARTIAL 判定无标准** | 低 | 清单只有 PASS/FAIL 二值判定，但实际检查中出现了 PARTIAL（如 #03 的 Item 19）。应定义 PARTIAL 的判定标准和计分方式。 |
| 7 | **反模式自检与正式检查项重叠** | 低 | 反模式自检中的"标注只重复数字"与 Item 10 重复，"高亮过多"与 Item 9 重复。应整合或明确差异。 |

### 建议改进

1. **新增 Item 21：渲染能力匹配** — "spec 中所有契约字段（annotations/referenceLines/highlight/multi-chart）均被渲染器支持并正确渲染"
2. **新增 Item 22：图表数量校验** — "HTML 中实际渲染的图表数量 = spec 中定义的图表数量"
3. **明确 Item 14** — 改为"source ID 可从 HTML 追溯到 spec 中的 Source ID 清单"
4. **修复 Item 18** — 加入颜色维度：颜色区分应贡献视觉重量，同色时仅靠尺寸不够
