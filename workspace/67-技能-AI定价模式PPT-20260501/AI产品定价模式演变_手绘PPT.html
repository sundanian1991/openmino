演变方向：API 次数 → Prompt → Token/Credit → 统一 Credits。越来越透明、越来越可预测。

现在最让用户头疼的是什么？计量单位。

五种计量方式同时存在：API 请求次数、Prompt 次数、Token 计量、Credit 计量、时间窗口配额。

直接比数字毫无意义。阿里百炼给 18000 次 API 请求，MiniMax 给 40 个 Prompt——看起来百炼多 15 倍，但 1 个 Prompt 等于 15 到 20 次模型调用，等于 1200 到 1600 次 API 请求。

90% 的开发者会踩坑。这不是设计缺陷，这是信息壁垒——模糊计量有利于厂商。
