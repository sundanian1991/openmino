# 里程碑审计清单

> 每个里程碑完成后，用此清单验证是否可以进入下一个。

---

## 审计检查

### 完成度验证
- [ ] 里程碑的所有任务都标记为完成
- [ ] 预期结果已实现（对照 GOALS.md 中的描述）
- [ ] 没有"差不多完成"的模糊状态——要么完成，要么没完成

### 证据验证
- [ ] 所有"继续前进前所需的证据"已收集
- [ ] 证据是可验证的（截图、日志、测试结果），不是主观判断
- [ ] 证据与预期结果匹配

### 代码质量
- [ ] 代码审查已完成
- [ ] 没有未解决的审查意见
- [ ] 测试通过（如有测试）
- [ ] 没有新增的 lint 警告或错误

### 路线图一致性
- [ ] 下一个里程碑的范围仍然正确
- [ ] 没有遗漏的依赖或前置条件
- [ ] 新发现是否改变了工作优先级？
- [ ] "完成"的定义是否仍然成立？

### 阻塞项
- [ ] 所有已知阻塞项已解决或有明确的解决计划
- [ ] 没有隐藏的阻塞（worker 可能知道但没报告）

---

## 审计输出

审计完成后，更新 GOALS.md：

1. 将完成的里程碑移至"已完成的里程碑"区域
2. 更新"活跃目标"为下一个里程碑
3. 记录任何新发现或变更到"变更记录"
4. 如果发现遗漏，添加新的里程碑或任务

---

## 常见问题

**问题**：里程碑"基本完成"但有些小问题
**处理**：明确是修复后继续，还是记录为已知问题并继续。不要让"基本完成"阻塞整个项目。

**问题**：新发现改变了下一个里程碑的范围
**处理**：更新 GOALS.md，记录变更原因，调整预期。

**问题**：worker 报告了与里程碑无关的发现
**处理**：记录到"变更日志"，评估是否需要新的里程碑。
