> 活跃里程碑：M1 — 周四演示版

## 项目定义

把已有的人机协同方法论（信息密度过高的 HTML 指南），转化成目标用户能**开箱即用、逐步上手**的工作包，并通过周四的周进展演示验证价值。

**一句话**：别人装上能用、看了知道怎么做、用完觉得有价值。

### 核心约束

| 约束 | 值 |
|------|-----|
| 目标用户工具 | 内部 agent 产品 "zero"（类 Codex） |
| 硬节点 | 7月17日（周四）周进展演示 |
| 投入预算 | 碎片时间为主，每天约 1-2 小时（假设，待确认） |
| 内容范围 | 三大核心模块（见下），非全套 8 section |
| 产出形态 | 可安装技能 + 模板 + 流程引导 |

### 三大核心模块

1. **3A 分诊法**（你的工作该怎么用 AI / 协同的本质）
2. **5 问提取判断样本**（判断档案方法论）
3. **AI 长周期项目协作方法论**（三大锚点 + 七步工作流 + 模型分工）

### 决策记录

- **D1 (7/13)**：工作包本身就是最好的演示材料，不单独做 PPT。演示 = 现场跑工作包。
- **D2 (7/13)**：聚焦三个高价值模块，砍掉自测定位、演进阶段等辅助内容（可保留为附录）。
- **D3 (7/13)**：MVP 目标是"一个人装上、跟着引导走、30 分钟内出第一个成果"。
- **D4 (7/13)**：工作包形态 = 项目模板（一键初始化）。含 AGENTS.md + GOALS.md 模板 + 三个技能 + 快速上手引导。用户安装后即有完整环境。
