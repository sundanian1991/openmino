# C 盘清理脚本 使用说明

> `Clean-CDrive.ps1` — 三阶段执行 + 性能诊断，只删可重建缓存

---

## 能解决什么 / 不能解决什么

| 问题 | 能不能解决 |
|---|---|
| C 盘空间不够（临时文件、更新缓存、浏览器缓存等占用） | 能 |
| 系统卡（启动项过多、磁盘老、进程吃内存） | **只诊断不自动改**，给出报告和方向 |
| 休眠文件、Windows.old、系统还原点占空间 | 不碰（不可逆），报告里列大小，需手动处理 |
| WinSxS 组件库膨胀 | 不碰，给命令让你自己判断后再执行 |

---

## 运行前准备

1. **以管理员身份运行 PowerShell**
   - 开始菜单搜 "PowerShell" → 右键 → 以管理员身份运行
   - 脚本开头有管理员权限硬门，非管理员直接退出

2. **临时放行脚本执行**（只对当前窗口有效，不改全局策略）
   ```powershell
   Set-ExecutionPolicy -Scope Process Bypass
   ```

3. **把脚本放到 Windows 机器上**任意目录，比如 `C:\Scripts\Clean-CDrive.ps1`

---

## 两种运行模式

```powershell
# 模式一：只扫描评估，不出删除确认（推荐第一次先跑这个）
.\Clean-CDrive.ps1 -ScanOnly

# 模式二：完整三阶段（扫描 → 确认 → 删除）
.\Clean-CDrive.ps1

# 可选：指定报告输出目录（默认桌面）
.\Clean-CDrive.ps1 -ReportPath D:\Reports
```

## 三个阶段看什么

### 阶段 0：环境自检
自检管理员权限、系统版本，创建报告目录（桌面 `C盘清理报告_时间戳`）。

### 阶段 1：扫描评估（只读，不改）
- **可清理项**：逐项列出名称、大小、备注。末尾给"总可释放空间"
- **参考项**：Windows.old、hiberfil.sys、pagefile.sys 的大小（不删，让你知道大头在哪）
- **各用户 Downloads**：大小（不删）
- **性能诊断报告**（单独文件）：
  - 磁盘：C 盘容量、SSD/HDD、健康度、剩余百分比（<15% 标红）
  - 开机时长：最后开机时间、已运行天数
  - 启动项：名称、命令、位置、用户（共 N 项）
  - 自启服务：Automatic 类型数量、运行中数量
  - 进程 Top 10：按内存排
  - C 盘大文件夹 Top 10：顶层目录大小

这一阶段**不删任何东西**，只读。`-ScanOnly` 模式跑完就停在这里。

### 阶段 2：预览确认
列出"将删除的项 + 释放空间"，给三个选项：
- `Y` 全部执行
- `S` 选择编号（输入如 `1,3,5`）
- `N` 取消

提示：浏览器缓存项需先关闭浏览器，否则部分文件被占用会跳过。

### 阶段 3：执行删除
逐项删，每项容错（被占用跳过不中断）。每项给状态：
- `完成`：释放 ≥ 90% 预计值
- `部分`：释放一部分（有文件被占用）
- `跳过`：没删动
- `无内容`：本来就空

末尾对比清理前后可用空间，算出实际释放量。

## "卡"怎么排查

性能诊断报告里按这个顺序看：

1. **磁盘是不是 HDD**
   - 物理磁盘 MediaType 列。HDD 老盘本身就慢，清理空间帮助有限，考虑换 SSD
   - HealthStatus 不是 Healthy 要警惕硬件问题

2. **可用空间是不是 < 15%**
   - 空间太满磁盘 I/O 雪崩式变慢，先清空间
   - 清完跑一次脚本看是否回到 25% 以上

3. **开机多久没重启**
   - 已运行几十天的话，重启一次能解决很多卡顿

4. **启动项数量**
   - 启动项 20+ 算偏多。任务管理器 → 启动 选项卡禁用不需要的
   - 脚本不自动改启动项（不可逆，需你自己判断）

5. **进程 Top 10**
   - 看有没有内存占用异常的程序（比如某软件吃 2GB+）
   - 针对性关掉或卸载，比无脑清空间有用

6. **自启服务**
   - Automatic 服务过多拖慢开机。`services.msc` 里把不需要的手动启动
   - 不确定的服务别动，乱停服务会出问题

> 清理空间和解决卡顿是两件事。如果清完空间还是卡，重点看上面 1/3/4/5 项。

## 安全边界（脚本绝不碰这些）

| 不删的项 | 为什么不删 | 想删怎么办 |
|---|---|---|
| 休眠文件 `hiberfil.sys` | 关休眠才释放，不可逆 | 管理员运行 `powercfg /h off` |
| `Windows.old` | 删了不能回退旧系统 | 磁盘清理工具 `cleanmgr` → 清理系统文件 → 以前的 Windows 安装 |
| 系统还原点 | 删了没退路 | `SystemPropertiesProtection` → 清理 |
| 下载文件夹 / 桌面 | 用户数据 | 自己手动整理 |
| WinSxS / DISM ResetBase | ResetBase 后不能卸载更新 | 先 `Dism /Online /Cleanup-Image /AnalyzeComponentStore` 看体积，再决定是否 `Dism /Online /Cleanup-Image /StartComponentCleanup` |
| 页面文件 `pagefile.sys` | 系统管理 | 不建议动 |

## 常见问题

**Q: 脚本卡在"扫描 C 盘大文件夹"很久？**
A: 正常，C:\Windows 等大目录扫描需要 1-2 分钟。等就行。

**Q: 浏览器缓存显示"部分"或"跳过"？**
A: 浏览器开着，文件被占用。关掉浏览器重新跑那几项。

**Q: 实际释放比预计少很多？**
A: 正常。被占用的文件跳过了，且部分缓存正在被系统使用。看执行报告里每项状态。

**Q: 清完空间没怎么变？**
A: 两种可能：① C 盘大头是 Windows.old / WinSxS / 下载文件夹，这些脚本没动（看参考项）；② 清理后有程序立刻写入了新文件。看参考项决定要不要手动清那些大头。

**Q: 想再清深一点？**
A: 跑 `cleanmgr`（系统自带磁盘清理）勾选更多项，或用 `Dism /Online /Cleanup-Image /StartComponentCleanup` 清 WinSxS。这些偏不可逆，自己判断。

**Q: 能不能定时自动跑？**
A: 能，但自动跑删除有风险（万一清理了正在用的缓存）。建议只把 `-ScanOnly` 模式加到任务计划，定期看报告，手动执行删除。

## 执行策略提示

脚本未签名。Windows 默认执行策略可能阻止运行。两种处理：

- **临时**（推荐，只对当前窗口）：`Set-ExecutionPolicy -Scope Process Bypass`
- **当前用户**（永久，需确认安全）：`Set-ExecutionPolicy -Scope CurrentUser RemoteSigned`

跑完想恢复：`Set-ExecutionPolicy -Scope Process Undefined`（窗口关掉自动恢复）。
