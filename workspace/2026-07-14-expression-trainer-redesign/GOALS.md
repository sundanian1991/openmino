## 产品定义

### 一句话
离线实时中文语音 → 表达训练 + 多类型文档产出，一个本地桌面工作站。

### 核心架构
```
引擎层（共享底层）
  ├── ASR 引擎（Sherpa-ONNX 离线实时识别）
  ├── 词库分析引擎（情感/填充/犹豫/笼统检测）
  └── AI 后端（云端 API / 本地 Ollama，可切换）
       ↓
模式层（各自独立）
  ├── 训练模式：实时反馈 + 表达分析报告
  ├── 文档·正式报告：口述要点 → 结构化正式文档
  ├── 文档·会议纪要：讨论 → 纪要 + 待办 + 决策
  ├── 文档·邮件消息：口述大意 → 正式邮件/消息草稿
  └── 文档·笔记周报：碎片想法 → 整理笔记/周报
```

### 用户画像
自用工具——用户是产品经理/管理者，日常需要：
1. 提升自己的口头表达精准度
2. 快速把口述内容变成正式文档（报告、邮件、纪要、周报）

### 差异化
- 离线隐私（非云端 ASR）→ 可用于敏感场景
- 词库知识库 → 不只听写，理解表达质量
- 训练 + 文档融合 → 文档模式本身也是表达训练（改了哪里、为什么改）

### 视觉协调性硬约束（用户提醒 · M2/M3 实现时遵守）
1. **元素比例协调**：图标按钮（如设置⚙）不能比周围文字小太多；统计数字不能比同区文字大太多。同区域内所有元素的视觉权重要协调。
2. **间距节奏统一**：录制按钮与底部统计条的距离，要与统计条和其他元素的距离保持一致的节奏感。
3. **字号梯度收敛**：字幕、统计标签、面板标题、底部条的字号差异不能过大。用 Haglöfs Type Scale 的相邻级过渡，不跳级。
4. **内容边界对齐**：面板内边距、卡片内边距、栏目宽度要形成统一的安全区。
