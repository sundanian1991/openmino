# 工作流追踪与分析 — AI 增强方案

> 用 AI 真正理解工作内容，而不是统计词频

---

## 一、为什么需要 AI

### 1.1 词频统计的局限

| 方式 | 能做什么 | 做不到 |
|------|----------|--------|
| **词频统计** | "供应商"出现50次 | 不知道在处理供应商的什么问题 |
| **规则引擎** | 识别"飞书"=沟通 | 不知道沟通的目的和结果 |
| **时间统计** | 在Chrome停留2小时 | 不知道是在学习还是在摸鱼 |

### 1.2 AI 能做什么

| 能力 | 示例 |
|------|------|
| **内容理解** | 这张截图是在分析供应商KPI数据 |
| **任务识别** | 这段时间在撰写AI提效方案 |
| **逻辑推理** | 从搜索记录推断：用户在研究BPO联盟机制 |
| **洞察生成** | 下午2-3点频繁被IM打断，建议设置专注时段 |

## 二、AI 介入架构

### 2.1 三层处理模式

```
第一层：规则引擎（快速、便宜）
  └→ 时间戳统计、应用识别、基础分类

第二层：本地小模型（可选）
  └→ 文本聚类、简单语义理解

第三层：云端大模型（深度、准确）
  └→ 工作内容理解、任务识别、洞察生成
```

### 2.2 数据流向

```
截图序列 → 规则引擎 → 预处理数据
                ↓
            智能采样（选哪些用AI分析）
                ↓
            AI批量分析（聚合上下文）
                ↓
            深度理解报告
```

## 三、AI 应用场景

### 3.1 工作内容理解（核心）

**问题**：用户今天做了什么？

**不用 AI**：
- 词频：供应商(50)、KPI(30)、数据(20)
- 无法知道：在做什么？

**用 AI**：
- 输入：一组截图 + OCR 文本
- 提示词："分析这20分钟的工作内容"
- 输出："用户在分析供应商Q1的KPI数据，对比毅航和毛毛虫的表现"

**实现方式**：
```python
# 将30分钟内的截图聚合
screenshots_30min = group_by_time(screenshots, window=30)

# 批量发给 AI
prompt = f"""
分析以下工作片段：
时间：{time_range}
截图内容：
{extract_text_and_context(screenshots_30min)}

请回答：
1. 用户在做什么任务？
2. 任务完成度如何？
3. 工作类型是什么（深度/沟通/会议）？
"""

result = call_ai_api(prompt)
```

### 3.2 工作流重建

**问题**：一天的工作逻辑是什么？

**不用 AI**：
- 简单分段：9-11点在某应用
- 无法知道：这2小时在做什么

**用 AI**：
- 输入：全天工作块序列
- 提示词："理解这一天的工作流"
- 输出：
```
上午9-11点：供应商KPI分析（深度工作）
11-12点：与刘伟佳沟通定价策略（谈判）
14-16点：撰写AI提效方案（文档编写）
16-18点：准备老板汇报材料（整合）
```

---

### 3.3 智能洞察生成

**问题**：怎么改进工作？

**不用 AI**：
- 规则：碎片化>0.5 → "减少切换"
- 太泛化，没针对性

**用 AI**：
- 输入：全天数据 + 工作模式
- 提示词："分析工作模式，给出3条具体建议"
- 输出：
```
1. 下午2-3点被打断12次，建议设置14:00-15:00为专注时段
2. 供应商沟通占60%，但深度分析不足，建议提高深度工作比例
3. 周五下午效率下降明显，建议将重要会议安排在周四
```

---

### 3.4 趋势理解

**问题**：这周比上周有变化吗？

**不用 AI**：
- 简单对比：效率72→78分
- 无法解释：为什么提升？

**用 AI**：
- 输入：两周数据对比
- 提示词："分析工作模式的变化"
- 输出：
```
本周相比上周：
- 深度工作增加2小时，因为减少了临时会议
- 供应商相关活动增加15%，进入Q1总结期
- 效率提升6分，主要因为下午时段安排更合理
```

## 四、成本优化策略

### 4.1 智能采样

**问题**：每张截图都问 AI，成本太高

**方案**：只问 AI 有价值的问题

| 采样策略 | 触发条件 | AI 调用 |
|----------|----------|----------|
| **密集采样** | 深度工作块 | 每10分钟一次 |
| **稀疏采样** | 碎片化时段 | 每30分钟一次 |
| **跳过采样** | 明确的低效时段 | 不调用 |

### 4.2 批量聚合

**单张分析** vs **批量分析**：

```python
# 方式1：每张截图单独问 AI（贵）
for screenshot in screenshots:
    result = ai.analyze(screenshot)
    # 成本：100张 × $0.001 = $0.1

# 方式2：批量聚合后问 AI（便宜）
group = group_screenshots(screenshots, window="30min")
result = ai.analyze_batch(group)
# 成本：4组 × $0.01 = $0.04
```

### 4.3 分层决策

```
规则引擎 → 需要深度理解？
    ↓ 是
智能采样 → 这组值得问 AI 吗？
    ↓ 是
AI 分析 → 生成洞察
```

## 六、实现方案

### 6.1 技术架构

```python
class AIEnhancedAnalyzer:
    def __init__(self):
        self.rule_engine = RuleEngine()  # 规则引擎
        self.ai_client = AIClient()      # AI 客户端

    def analyze_day(self, screenshots):
        # 1. 规则引擎预处理
        basic_data = self.rule_engine.process(screenshots)

        # 2. 智能采样
        work_blocks = self.identify_work_blocks(basic_data)
        important_blocks = self.sample_by_importance(work_blocks)

        # 3. AI 批量分析
        ai_results = []
        for block in important_blocks:
            result = self.ai_client.analyze_block(block)
            ai_results.append(result)

        # 4. 整合报告
        report = self.generate_report(basic_data, ai_results)
        return report
```

### 6.2 AI 调用示例

```python

要求：
- 建议要具体可执行
- 要基于数据，不要泛泛而谈
- 要有优先级
"""
    return call_claude_api(prompt)
```
