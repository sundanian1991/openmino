## 二、哲学层迁移

### 2.1 Boil the Lake → 完整性优先

**原义**：AI让"完整实现"的边际成本趋近于零，所以每次都做完整版本

**迁移**：供应商管理的"完整性"是什么？

| 原场景 | 迁移场景 | 完整性定义 |
|--------|----------|------------|
| 代码实现 | 供应商准入 | 不只看资质，要看：产能、意愿、团队、背调、试单 |
| 测试覆盖 | 业绩评估 | 不只看KPI，要看：趋势、原因、对比、预测、建议 |
| 错误处理 | 风险管理 | 不只看表面问题，要看：根因、影响、概率、预案 |

**你的工作应用**：
- 供应商准入：用"准入诊断管线"完整走完，不跳步
- 业绩评估：用"业绩诊断管线"挖到底，不满足于"还行"
- 风险管理：用"风险信号框架"系统扫描，不遗漏

### 2.2 Search Before Building → 先了解再决策

**原义**：造东西前先搜索已有方案，三层知识框架

**迁移**：供应商管理的"搜索"是什么？

| 原场景 | 迁移场景 | 搜索内容 |
|--------|----------|----------|
| 搜索已有代码/库 | 供应商选择 | 行业口碑、历史合作、竞品对比、内部推荐 |
| 搜索最佳实践 | 制度设计 | 行业标准、内部历史、其他团队经验 |
| 第一性原理 | 问题诊断 | 为什么选这家？为什么出问题？本质是什么？ |

**三层知识框架迁移**：

| 层级 | 原定义 | 供应商管理迁移 |
|------|--------|----------------|
| Layer 1 | 已验证模式 | 行业成熟做法（资质审核、合同模板、KPI体系） |
| Layer 2 | 新流行趋势 | 新的供应商类型、新的管理模式、数字化工具 |
| Layer 3 | 第一性原理 | 这家供应商为什么存在？核心竞争力是什么？ |

## 三、方法层迁移

### 3.1 管线式设计迁移

**原模式**：Phase 1 → Phase 2 → Phase 3 → Phase 4 → Phase 5 → Phase 6

**迁移到供应商管理**：

#### 供应商准入诊断管线

```
Phase 1: 背景采集
  - 基本信息、资质、团队、产能、过往合作
  - 输出：供应商画像初稿

Phase 2: 核心验证（6个问题）
  - Q1 需求匹配度：为什么需要这家供应商？
  - Q2 替代方案：没有这家会怎样？
  - Q3 具体场景：哪家业务线最需要？谁会急？
  - Q4 最小合作：最小化试单是什么？
  - Q5 过往验证：有没有真实合作案例？
  - Q6 长期适配：3年后还需要吗？

Phase 3: 前提挑战
  - 这是必须的吗？能内部消化吗？能合并到其他供应商吗？

Phase 4: 方案生成
  - 方案A：全面合作（高风险高收益）
  - 方案B：试点合作（低风险验证）
  - 方案C：暂缓准入（观察等待）

Phase 5: 输出文档
  - 准入评估报告

Phase 6: 信号综合
  - 推荐等级：强推/推荐/观望/不推
```

#### 供应商业绩诊断管线

```
Phase 1: 数据采集
  - KPI数据、趋势、异常点、对比数据

Phase 2: 核心诊断（6个问题）
  - Q1 业绩现状：核心指标是什么状态？
  - Q2 趋势原因：为什么会这样？根本原因是什么？
  - Q3 具体影响：哪条业务线受影响最大？谁在投诉？
  - Q4 改进可能：能改进吗？需要什么支持？
  - Q5 观察验证：现场看过吗？聊过一线吗？
  - Q6 未来预测：3个月后会怎样？

Phase 3: 前提挑战
  - 这是偶发还是趋势？是供应商问题还是我们问题？

Phase 4: 方案生成
  - 方案A：支持改进（给资源、给时间）
  - 方案B：观察警告（设红线、限期改进）
  - 方案C：降级/清退（启动替代）

Phase 5: 输出文档
  - 业绩诊断报告 + 行动计划

Phase 6: 信号综合
  - 风险等级：高/中/低
  - 行动优先级：立即/本周/本月
```

### 3.2 逼迫性问题迁移

**核心原则**：一次只问一个，推到答案具体、有证据、让人不舒服

#### 供应商准入的6个逼迫性问题

| # | 问题 | 目的 | 好答案特征 | 红旗信号 |
|---|------|------|------------|----------|
| Q1 | **为什么必须是这家？** | 验证需求真实性 | 具体业务线、具体需求、没有替代方案 | "多家比比"、"先接触看看" |
| Q2 | **没有这家会怎样？** | 验证紧迫性 | 业务会卡住、会损失、会延迟 | "有也行没有也行" |
| Q3 | **谁最需要？说具体名字** | 验证需求来源 | 业务线负责人姓名、痛点描述 | "业务线都说需要" |
| Q4 | **最小试单是什么？** | 验证可操作性 | 具体业务、具体量、具体时间 | "先签合同再说" |
| Q5 | **有成功案例吗？** | 验证能力 | 具体客户、具体项目、可核实 | "他们说做过" |
| Q6 | **3年后还需要吗？** | 验证长期价值 | 行业趋势、战略匹配、成长空间 | "先解决眼前问题" |

#### 供应商业绩诊断的6个逼迫性问题

| # | 问题 | 目的 | 好答案特征 | 红旗信号 |
|---|------|------|------------|----------|
| Q1 | **核心指标到底怎样？** | 验证现状 | 具体数字、趋势、对比 | "还行"、"差不多" |
| Q2 | **根本原因是什么？** | 验证理解深度 | 可验证的原因、数据支撑 | "可能是..."、"他们说..." |
| Q3 | **谁受影响最大？** | 验证影响范围 | 具体业务线、具体人、具体损失 | "影响不大" |
| Q4 | **能改进吗？要什么？** | 验证改进可能 | 具体方案、资源需求、时间线 | "需要我们一起努力" |
| Q5 | **去现场看过吗？** | 验证真实性 | 具体观察、一线反馈 | "问了他们负责人" |
| Q6 | **3个月后会怎样？** | 验证趋势预判 | 具体预测、依据、风险点 | "应该会好" |

### 3.3 信号采集框架迁移

**原模式**：采集创始人信号 → 计数 → 决定收尾力度

**迁移**：采集供应商信号 → 计数 → 决定管理力度

#### 供应商准入信号清单

| 信号类别 | 正向信号（+1） | 负向信号（-1） |
|----------|----------------|----------------|
| **需求真实性** | 业务线主动提需求、有具体使用场景 | 需求模糊、无人认领 |
| **能力验证** | 有可核实的成功案例、团队背景清晰 | 只有PPT、案例无法核实 |
| **意愿程度** | 主动配合背调、愿意试单、价格合理 | 推诿、要求先签合同 |
| **风险可控** | 财务健康、团队稳定、合规完善 | 财务问题、人员流失、合规瑕疵 |
| **长期价值** | 战略匹配、成长空间、差异化优势 | 同质化、可替代性强 |

**评级规则**：
- ≥4：强推，快速推进
- 2-3：推荐，正常流程
- 0-1：观望，需要更多信息
- <0：不推，终止准入

#### 供应商风险预警信号清单

| 信号类别 | 高风险信号 | 中风险信号 |
|----------|------------|------------|
| **业绩信号** | 连续2个月不达标、核心指标下滑>20% | 单月不达标、指标波动大 |
| **团队信号** | 核心人员离职、团队缩减>20% | 人员流动增加、招聘冻结 |
| **财务信号** | 欠薪、供应商投诉、诉讼 | 付款周期拉长、成本削减 |
| **合作信号** | 响应慢、推诿增加、承诺不兑现 | 沟通变少、会议取消 |
| **外部信号** | 行业负面、监管处罚、竞品挖人 | 行业下滑、政策变化 |

## 四、认知模式迁移

### 4.1 18条CEO认知模式 → 供应商管理认知模式

**决策类**

| 原 | 迁移 | 应用场景 |
|----|------|----------|
| 分类本能：可逆×量级 | 供应商决策分类：可逆×影响 | 新增供应商（可逆）vs 清退供应商（不可逆） |
| 速度校准：70%信息够 | 供应商决策速度 | 准入慢（影响大）、日常快（影响小） |
| 逆向反射：怎么失败 | 供应商风险逆向 | 如果这家供应商出问题，我们会怎样？ |
| 聚焦即减法：不做什么 | 供应商管理减法 | 哪些供应商可以合并？哪些可以砍掉？ |

**战略类**

| 原 | 迁移 | 应用场景 |
|----|------|----------|
| 偏执扫描：转折点 | 供应商生态扫描 | 行业变化、供应商动态、竞品动态 |
| 时间深度：5-10年 | 供应商长期规划 | 供应商梯队建设、备选培养 |
| 战时意识：诊断阶段 | 供应商状态诊断 | 哪些在战时（紧急）、哪些在和平（稳定） |
| 意志力即战略 | 供应商关系推进 | 关键供应商的深度合作推进 |

**管理类**

| 原 | 迁移 | 应用场景 |
|----|------|----------|
| 人才优先序列 | 供应商优先序列 | 核心供应商 > 战略供应商 > 普通供应商 |
| 杠杆执念 | 供应商杠杆 | 哪家供应商投入产出比最高？ |
| 代理怀疑论 | KPI怀疑论 | KPI是否还反映真实价值？ |

## 五、你的个人工作应用

### 5.1 日常工作中如何使用

**准入评估时**：
1. 打开"准入诊断管线"，按Phase走
2. 每个Phase必须输出具体内容
3. 用"6个逼迫性问题"验证每个关键假设
4. 采集信号，计算评级

**业绩诊断时**：
1. 打开"业绩诊断管线"，按Phase走
2. 用"6个诊断问题"挖到底
3. 采集风险信号，判断等级
4. 输出诊断报告+行动计划

**沟通准备时**：
1. 先跑一遍"业绩诊断管线"的Phase 1-2
2. 带着具体问题和数据去沟通
3. 沟通中验证假设、采集更多信号
4. 沟通后更新诊断

### 5.2 我（Mino）如何支持你

**日常对话中**：
- 你提到某个供应商 → 我自动触发"业绩诊断管线"
- 你提到要新增供应商 → 我自动触发"准入诊断管线"
- 你提到风险 → 我自动采集风险信号

**定期分析时**：
- 每月供应商业绩数据 → 我自动跑诊断，输出报告
- 每季度供应商盘点 → 我自动采集信号，输出评级

**决策支持时**：
- 你需要做供应商决策 → 我提供决策框架和信号分析
- 你需要准备沟通 → 我生成沟通要点和问题清单
