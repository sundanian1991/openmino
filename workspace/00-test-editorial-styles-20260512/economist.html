供应商分级盘点

以九宫格为工具，从经验判断走向数据驱动的供应商分级体系

对应HRBP原版：业务侧人才盘点（九宫格） · 月度评估 + 季度盘点 · 优先级 P0

对30余家供应商进行综合评估——当月贡献、环比趋势与供管自评三维叠加——输出分级清单与三张清单（保留、培养、优化），为资源分配、培养优化与淘汰清退提供决策依据。这是一套从"凭感觉分级"到"用数据说话"的系统性转变。

现状：为什么需要分级

目前供应商管理存在以下结构性问题：

  • 覆盖30+供应商，横跨金条、企金、信用卡、财富四条产线
  • 头部6家（毅航、毛毛虫、伽玛、赛维斯、岐力、翰锐）贡献绝大部分产能
  • 缺乏系统性的分级评估机制，分级多凭经验和直觉
  • 没有正式的"三张清单"（保留/培养/优化）
  • 赛马机制有相对排名，但只看业绩一个维度
  • 合规有巡检整改记录，但无量化的合规分数
  • 交付稳定性（人员流动、产能波动、SLA）无结构化数据
"分级多凭经验和直觉"——当30家供应商的命运由感觉决定，资源错配就是必然结果。

评估体系设计

总体架构：两套体系

体系用途频率
月度评估供应商分级 + 三张清单每月
季度盘点战略维度校准 + 格局审视每季度

月度评估：得分计算

最终得分 = [赛马基础分（扣分后 × 稳定性系数）] + 供管自评加分（0~4分）

第一步：赛马基础分

直接采用现有赛马成绩（满分100）。

第二步：合规扣分

事件扣分说明
黄牌整改-10分/次巡检发现问题并下发整改函
投诉/一般违规-5分/次质检通报、客户投诉
红线违规当月直接降级安全、数据泄露等严重事件

第三步：交付稳定性系数

月底扫三张检查线，ABC三档：

档位系数触发条件（满足任一即降级）
稳定 A 1.0 人员无大幅变动、产能无异常波动、SLA全达成
波动 B 0.9 任一触发：人员流失>10%、产能波动>20%、SLA一次未达成
失控 C 0.75 任一触发：核心人员离职、产能断崖、SLA两次以上未达成、突发掉链子

第四步：供管自评加分（0~4分）

供管主观评价，四项各评高中低，高=1分，中低=0分：

维度高=1分的触发条件
战略协作主动提出建设性建议/参与规划讨论、与业务方向一致
响应质量24h内响应、反馈有实质内容（非沟通频率）
高效执行临时任务按时按质完成，无返工
共同成长产线会议积极贡献、分享经验、对整体产线有额外贡献

供应商赛马合规扣分稳定性自评最终得分

九宫格映射（月度）

两轴定义

轴含义切分方式
X轴：当月贡献 最终得分（赛马×稳定性+自评） 按当月30+家供应商得分的高中低三分位切分
Y轴：环比趋势 本月最终得分 vs 上月最终得分的变化率 上升≥5% / -5%~+5%持平 / 下降≥5%
注：第一个月只有贡献无趋势，全部标记为"基线"，从第二个月开始有趋势。

九宫格定义

趋势下降 ↓趋势持平 →趋势上升 ↑
高贡献预警（稳住别掉）明星（稳产主力）加速器（重点投）
中贡献关注（为什么掉）中坚（基本盘）成长（重点培养）
低贡献淘汰（持续恶化）观察（给期限）潜力（给机会）

季度盘点（校准层）

月度评估捕捉变动，但有些维度月度看不出来。每季度追加校准：

季度校准维度

维度说明评估方式
战略意义 是否在关键产线/关键项目有不可替代性 管理层评估
长期潜力 规模扩张意愿、技术升级能力、团队稳定性 访谈+观察
风险敞口 过度依赖单一大客户、财务状况、舆情 公开信息+调研

季度校准的作用

月度表现好 + 季度校准好 → 纳入保留清单，深度绑定

月度表现好 + 季度校准差 → 标记"高贡献但脆弱"，准备替代方案

月度表现差 + 季度校准好 → 标记"低贡献但值得救"，给资源倾斜

月度表现差 + 季度校准差 → 纳入优化清单，启动清退

输出：三张清单

保留清单（头部）

  • 来源：明星 + 加速器 + 中坚（月度高贡献或中贡献且趋势稳定）+ 季度校准良好
  • 策略：资源倾斜、优先分配、深度绑定
  • 预计：6-10 家

培养清单（腰部）

  • 来源：成长 + 潜力 + 关注 + 观察（月度中/低贡献但有上升趋势）+ 季度校准有潜力
  • 策略：针对性辅导、明确提升方向、月度跟踪进度
  • 预计：12-15 家

优化清单（尾部）

  • 来源：淘汰 + 预警（趋势下降的高贡献）+ 季度校准无潜力
  • 策略：限期整改、减少份额、准备替代
  • 预计：5-8 家

实施路径

第一步：基线建立（第1个月）

  • 收集当月赛马成绩、合规事件、稳定性评估、供管自评
  • 计算30+家供应商当月得分
  • 按三分位切分高中低，绘制首次九宫格（趋势标记"基线"）

第二步：月度滚动（第2个月起）

  • 每月重复第一步
  • 计算环比趋势，更新九宫格位置
  • 输出三张清单（保留/培养/优化）

第三步：季度校准（每季度末）

  • 追加战略意义、长期潜力、风险敞口评估
  • 修正月度评估结果
  • 向易人汇报分级结果，确认分类

里程碑

时间里程碑完成标志
第1周（5月）基线数据收集完成30+供应商得分计算完毕
第2周（5月）首次九宫格绘制完成每家供应商有明确位置（基线）
第2周（5月）三张清单输出保留/培养/优化清单确认
第3周（5月）汇报完成易人确认分级结果
6月起月度评估滚动每月末自动更新九宫格
6月底首次趋势可见5月→6月环比趋势可用
Q2末（6月）首次季度校准战略/潜力/风险校准完成

量化指标

指标定义目标
评估覆盖率 完成评估的供应商数/总供应商数 100%
头部产能占比 头部供应商产能/总产能 建立基线（当前未知）
腰部转化率 季度内从腰部进入头部的供应商数 Q2末≥1家
尾部淘汰率 季度内从优化清单清退的供应商数 根据业务实际
稳定性预警率 稳定性评为B或C的供应商占比 监控用，无目标
自评区分度 供管自评得分分布（不应集中在3-4分） 至少20%供应商得2分以下

风险与应对

风险应对
交付稳定性无历史数据，首次评估可能不准 第1个月标注"待校准"，从第2个月开始有趋势后可修正
合规扣分与赛马成绩中的合规指标可能重复 确认赛马是否已包含合规，如已包含则合规扣分项减半或只计红线事件
供管自评变成"人情分"，全部高分无区分度 每项有明确触发条件，月底过检查表；连续两月无中低分的维度需复核
分级变成"贴标签"，腰部供应商失去积极性 强调分级是动态的，月度调整，每次公布"进步榜"
精力集中在分级本身，后续动作跟不上 分级只是起点，重点是三张清单对应的后续动作
外部储备供应商未纳入体系 储备供应商独立评估，对标存量层级后决定是否引入
"分级只是起点，重点是三张清单对应的后续动作。"
