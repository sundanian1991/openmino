# Obsidian + Claude 深度阅读工作流完整指南

> 将扫描版PDF转化为中英双语深度阅读笔记，全程可视化，无需代码。

---

## 一、方案总览

```
PDF扫描版 → MinerU OCR → Markdown → Obsidian → Claude处理 → 双语笔记
   (1)           (2)            (3)         (4)         (5)        (6)
```

### 核心优势
- ✅ **本地运行**：所有数据都在本地，不上传云端
- ✅ **可视化操作**：在Obsidian界面完成，无需终端
- ✅ **结构保留**：表格、公式、图片都能保留
- ✅ **双语对照**：原版+中文翻译+总结归纳
- ✅ **随时交互**：可以在任何位置提问、添加笔记

---

## 二、环境准备

### 2.1 安装Obsidian

1. 下载：https://obsidian.md/
2. 创建一个vault（文件夹），命名为"深度阅读"或其他

### 2.2 安装Claudian插件（核心）

这是连接Obsidian和Claude的桥梁。Claudian是专门为Obsidian设计的插件。

**步骤**：

**1. 下载插件文件**
- 访问：https://github.com/zhumengkai/Claudian/releases
- 下载以下三个文件：
  - main.js
  - manifest.json
  - styles.css

**2. 创建插件文件夹**
- 在电脑上创建一个名为 `claudian` 的文件夹
- 将上述三个文件放入该文件夹中

**3. 放入Obsidian插件目录**
- 打开Obsidian
- 进入 设置 → 第三方插件
- 点击"打开插件文件夹"按钮
- 在plugins文件夹中，放入刚才创建的claudian文件夹

**4. 启用插件**
- 回到Obsidian设置
- 如果看到安全模式提示，先关闭安全模式
- 在第三方插件列表中找到Claudian
- 打开开关启用

**5. 配置Claude Code（可选）**
如果要与Claude Code集成，编辑配置文件：
- Mac/Linux: `~/.claude/settings.json`
- Windows: `%USERPROFILE%\.claude\settings.json`

### 2.3 安装MinerU（OCR工具）

```bash
# 安装
pip install mineru

# 验证
pdf2md --help
```

## 三、工作流执行

### 第1步：PDF转Markdown

```bash
# 基础用法
pdf2md your-book.pdf -o book.md

# 高级用法（保留图片）
pdf2md your-book.pdf -o book.md --images-dir ./images
```

输出示例：
```markdown
# 第一章 导论

这是一段文本...

表格：

| 列1 | 列2 |
|-----|-----|
| A   | B   |
```

---

### 第2步：导入Obsidian并按章节分割

**操作方法**：

1. **创建章节文件**
   - 在Obsidian中新建文件夹，如`书名/`
   - 创建主文件`书名.md`，粘贴完整Markdown内容

2. **手动分割章节**
   - 读Markdown，找到章节标题（通常是`# 第一章`这种）
   - 从每个章节开始，剪切到新文件
   - 命名规范：`01-第一章.md`, `02-第二章.md` ...

3. **章节文件结构**
```markdown
---
title: 第一章 导论
---

# 第一章 导论

章节内容...
```

---

### 第3步：连接Claude

**在Obsidian中启动Claudian**：

1. 确保Claudian插件已启用
2. 在Obsidian中打开任意笔记
3. 使用Claudian的命令或快捷键启动Claude
4. Claude会直接在Obsidian内读取和处理你的笔记

**Claudian使用方式**：
- 可以直接让Claude读取当前文件
- 可以让Claude搜索整个vault
- 可以让Claude创建或修改笔记
- 所有操作都在Obsidian界面内完成

---

### 第4步：单章节处理（标准化流程）

这是最关键的一步，建立标准化Prompt。

**Prompt模板**：

```
请帮我处理这个章节：

1. **分段理解**：把章节分成几个主题段落
2. **每段做三件事**：
   - [总结] 用中文概括核心观点
   - [翻译] 把原文翻译成中文（保留英文原文）
   - [归纳] 提炼关键概念、方法、结论

3. **输出格式**：
```
## [段落标题]

### 原文
[英文原文]

### 中文翻译
[翻译内容]

### 总结归纳
- 核心观点：[一句话概括]
- 关键概念：[列出3-5个]
- 实践启示：[如何应用到实践中]
```

4. **要求**：
   - 翻译准确、自然
   - 总结简洁、有价值
   - 在原文件上直接修改，保留原文

```

**示例输出**：

```markdown
# 第一章 导论

## 1.1 什么是深度学习

### 原文
Deep learning is a subset of machine learning that uses neural networks with multiple layers to progressively extract higher-level features from raw input.

### 中文翻译
深度学习是机器学习的一个子集，它使用具有多层结构的神经网络，从原始输入中逐步提取更高级别的特征。

### 总结归纳
- **核心观点**：深度学习通过多层神经网络逐层提取特征，实现从低级到高级的信息抽象。
- **关键概念**：
  - 神经网络：模拟人脑神经元连接的计算模型
  - 多层结构：每层负责提取不同抽象级别的特征
  - 特征提取：从原始数据中识别有意义模式的过程
- **实践启示**：在设计深度学习模型时，需要考虑每一层的作用，确保特征能够有效传递和增强。
```

---

### 第5步：审查与迭代

**审查要点**：
- 翻译是否准确、自然？
- 总结是否有价值？
- 有没有遗漏关键信息？
- 归纳是否清晰？

**迭代方法**：
- 在Obsidian中直接修改
- 针对问题部分，重新让Claude处理：
  > "这段的总结太简略了，请补充更多细节"
  > "这个翻译不太准确，请重新翻译"

---

### 第6步：批量处理所有章节

**基于单章节的经验，让Claude批量处理**：

```
请按照第一章的处理方式，处理所有章节：

1. 保持相同的输出格式
2. 每个章节生成单独的.md文件
3. 文件命名：01-第一章.md, 02-第二章.md ...
4. 处理完成后告诉我完成情况
```

**关键**：Claude会学习第一章的模式，应用到所有章节。

## 四、创建Skill（标准化流程）

把这套流程转化为可复用的经验，生成一个skill。

### 4.1 Skill内容框架

```markdown
# 深度阅读工作流 - Obsidian+Claude

## 触发场景
- 收到一本扫描版PDF书籍
- 需要进行深度阅读和笔记
- 希望有中英双语对照

## 工作流步骤

1. **PDF转Markdown**
   - 使用MinerU：`pdf2md input.pdf -o output.md`
   - 或使用PaddleOCR：`paddleocr --det_model_dir ...`

2. **导入Obsidian**
   - 创建vault文件夹
   - 导入Markdown文件

3. **章节分割**
   - 识别章节标题（`# 第X章`）
   - 手动或自动分割成多个文件

4. **单章节处理**
   - 使用标准化Prompt模板
   - 原文 + 翻译 + 总结归纳

5. **批量处理**
   - 基于单章节模式，处理所有章节

6. **审查迭代**
   - 在Obsidian中直接修改
   - 针对问题重新处理

## Prompt模板

[这里放完整的Prompt]

## 注意事项

- OCR工具选择：MinerU适合大多数情况，PaddleOCR适合需要GPU加速的场景
- 章节识别：注意多级标题的处理
- 翻译质量：先处理一章节，建立标准，再批量处理
```

### 4.2 创建Skill

在Obsidian中：
1. 新建文件`.claude/skills/深度阅读工作流.md`
2. 粘贴上述内容
3. 保存

下次使用时：
> "/skill 深度阅读工作流"

## 五、常见问题

### Q1: Claudian插件连接不上？
**A**:
- 确保Claudian插件已启用
- 检查插件文件夹是否正确放置
- 确认三个文件（main.js, manifest.json, styles.css）都已下载

### Q2: OCR效果不好怎么办？
**A**:
- 尝试不同工具：MinerU → PaddleOCR → Marker
- 调整参数：`--dpi 300` 提高分辨率
- 对于扫描质量差的，可能需要手动处理

### Q3: 章节分割太麻烦？
**A**: 可以写个Python脚本自动分割，或者让Claude帮你在Obsidian内分割。

### Q4: 翻译质量不好？
**A**:
- 提供更明确的Prompt："用专业术语，保持原文风格"
- 分段处理，避免一次性处理太多内容
- 迭代优化，建立你的翻译标准

## 六、进阶技巧

### 技巧1：自动生成目录

让Claude基于章节文件生成目录：

```
请基于所有章节文件，生成一个目录（TOC）：
```
# 目录

- [第一章 导论](./01-第一章.md)
- [第二章 基础概念](./02-第二章.md)
...
```

### 技巧2：添加笔记和提问

在阅读过程中，随时在Obsidian中添加：

```markdown
### 我的笔记
这个观点很有启发，它让我想到...

### 待解决问题
1. 这里的XX概念需要进一步理解
2. 能否找到实际案例？
```

然后让Claude解答这些问题。

### 技巧3：跨章节关联

让Claude分析章节之间的关系：

```
请分析第一章和第三章之间的关联，它们如何互为补充？
```

## 八、工具对比总结

| 工具 | 优势 | 劣势 | 推荐场景 |
|------|------|------|---------|
| **MinerU** | 简单、直接PDF→MD | 图像可能不完美 | 首选 |

| **Marker** | 结构保真度高 | License限制 | 复杂文档 |
