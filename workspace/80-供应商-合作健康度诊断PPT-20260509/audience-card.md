# Audience Card — 卞海军（军哥）

## 基本信息

- **职位**：数据科技业务部负责人（二级部门）
- **与汇报人关系**：年老师的隔级上级（年老师 → 王易人 → 卞海军）
- **汇报频率**：关键事项直接汇报，日常通过王易人间接

## 管理风格：穿透式高压

| 维度 | 特征 |
|------|------|
| **追问方式** | 连环追问根因，不接受表面解释 |
| **数据要求** | 数据前置，没有数据就没有发言权 |
| **时间要求** | 明确时间节点，"最大接受4-6周"，超过换人 |
| **情绪特征** | 问题严重时激动，会定性追责 |
| **行动导向** | 要具体动作和方案，不要计划和想法 |

## 信息接收偏好

| 要素 | 要求 |
|------|------|
| **结构** | 结论先行 → 数据支撑 → 根因分析 → 行动方案 → 时间线 |
| **密度** | 高信息密度，不说废话，不铺垫 |
| **形式** | 结构化表达，能一眼看清全局 |
| **态度** | 有立场有判断，不说"看情况" |

## 审批关注点

- 这件事的影响面多大？
- 数据在哪？能不能量化？
- 谁负责？什么时候搞定？
- 有什么风险？最坏情况是什么？
- 为什么是这个方案而不是别的？

## 对"供应商管理BP"的期望

- BP 项目要有可量化成果
- 不要空对空的方法论，要落地的动作
- 季度内能看到效果
- 能解决他关心的实际问题（供应商业绩、稳定性、合规）

## 沟通雷区

- 只讲问题不讲方案 → 被质疑"你要我做什么？"
- 模糊时间线 → "什么时候能搞定？"
- 没有数据支撑 → "依据是什么？"
- 问题累积到爆发才上报 → "为什么不早说？"
