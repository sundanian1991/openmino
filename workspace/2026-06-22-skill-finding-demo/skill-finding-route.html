Step 01 · 英文关键词
把需求变成英文关键词
词越具体,结果越真实

Skill 生态现在主要还是英文内容。直接搜中文,结果通常会少很多。

比如想找"设计前端页面"的 Skill,不要只搜"前端设计",要把需求拆成词:frontend、web design、ui design、react。词越具体,搜出来的东西越接近真实需求。

Key Insight
词越具体,结果越接近真实需求
Example · 拆词对照
"前端设计" → 4 个英文词
一个模糊中文词,拆成 4 个具体英文词,每个都能独立搜出不同维度的 Skill。
frontend · web design · ui design · react
Step 02 · skills.sh 初筛
看 skills.sh 网站
人气和方向,不是最终结论

Skills.sh 是 Vercel 推出的开放 AI Agent 技能目录与排行榜,开发者可通过一条命令将封装为 Markdown 的 Skill 安装到 Claude Code、Cursor 等主流 AI 编程助手中。

里面有排行榜、安装量、Trending 和来源信息,适合先做判断:某个方向有没有现成 Skill?哪些 Skill 更多人用?来自哪个仓库?

Key Insight
这里看的是"人气"和"方向",不是最终结论。安装量高,只能说明被更多人试过,不代表没风险,也不代表一定适合你的场景。
Site · 地址
skills.sh
Vercel 推出的开放 AI Agent 技能目录与排行榜。看排行榜、安装量、Trending 和来源信息。
https://www.skills.sh/
Step 03 · 补充搜索
三个站,各有侧重
01
Force 01 · 安全质量
agentskill.sh
看安全和质量信号。会给 Skill 做安全评分和质量评分,也有安全面板。分数有参考价值,特别是几个同类 Skill 放在一起比较时——但评分只能当参考。
https://agentskill.sh/
02
Force 02 · 大范围搜索
SkillsMP
更像全网 Skill 搜索引擎,可按关键词、职业、创作者、GitHub 来源去找。优势是覆盖广,代价是结果更杂。
https://skillsmp.com/
03
Force 03 · 补充浏览
LobeHub Skills
页面比较好逛,分类、趋势、最近更新这些信息也比较清楚。但也是多来源聚合,出现在 LobeHub 上不等于被背书,同样要看指向哪个 GitHub 仓库。
https://lobehub.com/skills

Step 05 · 校验 GitHub
安装前,看三类信息
Argument 01 · 看说明
SKILL.md + description
重点看 SKILL.md 和 description。SKILL.md 会告诉你这个 Skill 解决什么问题、什么时候会被触发、是否依赖外部工具、有没有脚本。description 决定 Agent 什么时候会调用这个 Skill。
01
Argument 02 · 看脚本
scripts/ 风险信号
如果有的话,重点看 scripts/(非必项)。说明型 Skill 风险相对低。但如果看到要跑 bash/shell/python/node 脚本,要做浏览器自动化、文件访问,或需要 API key、token 凭证,就需要注意——可能执行代码、访问文件、控制浏览器。
02
Argument 03 · 看维护
README / 更新 / 反馈
README 看作者有没有认真说明使用方式;最近更新时间 看是否还在维护;stars/forks/issues 看有没有真实用户反馈。脚本读不懂的,就先别装到重要项目里。
