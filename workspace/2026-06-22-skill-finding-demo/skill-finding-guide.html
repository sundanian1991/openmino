01破题 · 普通用户的困境

对于不是精通 Agent 上下文领域的普通用户来说，在开始想用 Skill 的时候，一般都会遇到一个场景：在自己的场景下有没有一个现成的 Skill 可以解决当前的问题？不知道该去哪找。

网上搜，可能会出来很多结果，更迷茫了——哪个 Skill 更好用？哪个更受大家认可？来源靠不靠谱？有没有安全风险？找到以后能不能直接装？

03关键词 · 把需求变成英文

Skill 生态现在主要还是英文内容。直接搜中文，结果通常会少很多。

比如你想找"设计前端页面"的 Skill，不要只搜"前端设计"，要把需求拆成词：frontend、web design、ui design、react，词越具体，搜出来的东西越接近真实需求。

04初筛 · Skills.sh 看人气

Skills.sh 是 Vercel 推出的开放 AI Agent 技能目录与排行榜，开发者可通过一条命令将封装为 Markdown 的 Skill 安装到 Claude Code、Cursor 等主流 AI 编程助手中。

里面有排行榜、安装量、Trending 和来源信息，适合先做判断：某个方向有没有现成 Skill？哪些 Skill 更多人用？来自哪个仓库？

"安装量高，只能说明 Skill 被更多人试过，不代表没有风险，也不代表一定适合你的场景。"

Trust · 来源判断
按信任层级排序
01 官方组织 OpenAI / Anthropic / Vercel / GitHub 官方仓库，可信度最高。
02 大厂账号 知名项目或大厂官方账号维护的 Skill。
03 社区精选 官方组织维护的社区精选库，如 awesome-copilot。
04 高 Star 仓库 高安装量、高 star 的社区仓库，有真实用户反馈。
Verify · 安装前校验
看三类信息，再决定装不装
Checklist
01看说明 02看脚本 03看维护

01看说明

重点看 SKILL.md 和 description。SKILL.md 会告诉你这个 Skill 解决什么问题、什么时候会被触发、是否依赖外部工具、有没有脚本。

description 也很关键，决定 Agent 什么时候会调用这个 Skill。

02看脚本

重点看 scripts/，是非必选项。如果一个 Skill 只是说明型，风险相对低——像一份给 Agent 看的工作说明。

但如果看到这些内容就需要注意：bash、shell、python、node 脚本，浏览器自动化、文件访问，或者需要 API key、token 这类凭证。这些不代表一定有问题，只是说明可能会执行代码、访问文件、控制浏览器。

03看维护

包括 README、最近更新时间、stars / forks / issues。README 能看出作者有没有认真说明使用方式；最近更新时间能看出是否还在维护；stars、forks、issues 里能看到有没有真实用户反馈。

"脚本读不懂的，就先别装到重要项目里。"
Deduction · 推演
假如没有这些平台会怎样
Thought Experiment
设想一下：如果 skills.sh、agentskill.sh 这些平台都不存在。你只能在 GitHub 上盲搜，看到一个 Skill 不知道安全不安全，不知道有没有人用过，不知道质量如何。这些平台的价值，是把"选"的门槛从"靠运气"降到"靠信息"。但最终的判断，还是要靠你自己的眼睛——看来源、看 SKILL.md、看脚本、看维护。
Closing · 收束
当 AI 把"做"的门槛降到零，"选"的智慧就成了分水岭。精挑细选一件趁手工具的耐心，或许就是 AI 时代留给我们人类最不可替代的竞争力。
