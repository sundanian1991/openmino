§ 01

把需求变成英文关键词

Skill 生态主要还是英文内容，直接搜中文，结果通常会少很多。

比如你想找"设计前端页面"的 Skill，不要只搜"前端设计"，要把需求拆成具体的词。词越具体，搜出来的东西越接近真实需求。

Keyword A
frontend
前端方向
Keyword B
web design
网页设计
Keyword C
ui design
界面设计
Keyword D
react
技术栈
§ 02

在 skills.sh 做第一轮初筛

看的是"人气"和"方向"，不是最终结论。

skills.sh 是 Vercel 推出的开放 AI Agent 技能目录与排行榜。开发者可通过一条命令，将封装为 Markdown 的 Skill 安装到 Claude Code、Cursor 等主流 AI 编程助手中。里面有排行榜、安装量、Trending 和来源信息，适合先做判断：某个方向有没有现成 Skill？哪些 Skill 更多人用？来自哪个仓库？

定位：skills.sh 适合做第一轮初筛。
安装量高，只能说明 Skill 被更多人试过，不代表没有风险，也不代表一定适合你的场景。初筛看人气，终审看来源。
§ 03

补充搜索与候选

skills.sh 不一定能一次找到合适的。这三个站，每个侧重点不同。

agentskill.sh

安全优先
https://agentskill.sh/

会给 Skill 做安全评分和质量评分，也有安全面板。分数有参考价值，特别是几个同类 Skill 放在一起比较时。

优势：看安全和质量信号

SkillsMP

大范围
https://skillsmp.com/

更像全网 Skill 搜索引擎，可以按关键词、职业、创作者、GitHub 来源去找。

优势：覆盖广；代价：结果更杂

LobeHub Skills

适合浏览
https://lobehub.com/skills

页面比较好逛，分类、趋势、最近更新这些信息也比较清楚。但也是多来源聚合。

优势：分类清晰，适合补充浏览
组合策略：skills.sh 找到候选 → 拿名字去其他站搜来源/评分；skills.sh 没找到 → 拿原关键词去其他站继续搜。一个 Skill 出现在 LobeHub，不等于被官方背书——还是要看指向哪个 GitHub 仓库。
§ 04

判断来源

一个 Skill 靠不靠谱，看背后的来源仓库。同样搜索结果，openai/skills 和陌生个人仓库，可信度不一样。

信任阶梯 · 从高到低

§ 05

安装前，校验 GitHub 源文件

找到合适的 Skill，先别急着装。点进 GitHub 仓库，看这三类信息。

01
Look · README

看说明

重点看 SKILL.md 和 description。SKILL.md 告诉你这个 Skill 解决什么问题、什么时候被触发、是否依赖外部工具。description 决定 Agent 什么时候调用它。

说明型 Skill风险相对低——它像一份给 Agent 看的工作说明。
02
Look · Scripts

看脚本

如果有 scripts/ 目录，重点看会执行什么。说明型无脚本 → 风险低。

注意这些信号：跑 bash / shell / python / node、浏览器自动化、文件访问、需要 API key 或 token。
03
Look · Activity

看维护情况

包括 README 完整度、最近更新时间、stars / forks / issues。README 看作者态度；更新时间看是否还在维护；stars/forks/issues 看有没有真实用户反馈。

铁律：脚本读不懂，就先别装到重要项目里。
选
Final Thought

当 AI 把"做"的门槛降到零，
"选"的智慧就成了分水岭。

安装量、安全评分、star 数，这些数字能帮我们快速筛选。但最终是否使用，还是要靠我们自己的判断力。在人人都可以发布工具的时代，辨别什么是真正靠谱的东西，是越来越珍贵的能力——精挑细选一件趁手工具的耐心，或许就是 AI 时代留给我们最不可替代的竞争力。
