02 — WORKFLOW

五步工作法

从接收到交付的完整路径，每一步都有明确的输出标准。

01
INTENT GATE 意图分类 简单查询、复杂任务、探索、代码、数据 — 分而治之，不混为一谈。
02
PRE-DECLARATION 委派前声明 使用工具前说明理由和预期。不说"我做"，说"我用哪个工具做什么"。
03
TODO MANAGEMENT 任务可视化 多步骤时列出拆分清单，让进度一目了然，不被遗忘。
04
PROGRESS VISIBILITY 过程透明 执行中实时汇报进度。不做沉默运行 — 沉默意味着失控。
05
VERIFICATION & SUMMARY 验证总结 完成后逐项对照验收。没有验收标准，就不算完成。
03 — EVOLUTION

个性化升级

CLAUDE.md 从简单说明演进为分层结构，这个过程本身就是学习体系的缩影。

入口
CLAUDE.md 项目级指令，每次会话自动加载。定义行为规范、工具配置、开发约束。 claude.md ← 每次会话第一口上下文
核心（自动）
.claude/rules/ 分层规则文件。IDENTITY → SOUL → AGENT-FIRST → MEMORY → NOW。 按职责拆分，不堆在一份文件里
扩展（按需）
.claude/reference/ 用户画像、工作契约、工具使用、代码规范。不是每次都要，需要时再读。 懒加载 — 保持核心轻量
指令
.claude/commands/ 快捷指令，一键触发重复任务。自动化日常操作。
模板
.claude/skills/ 工作流模板。代码审查、文档生成、数据分析 — 可复用的最佳实践。
04 — PLAN5

关键教训

Plan5 五文件工作流是实践中沉淀出的方法论，每条教训都是用成本换来的。

01 不要跳过需求分析 直接开始写代码是最常见的错误。先问"要解决什么问题"，再问"怎么写"。需求分析不是拖延，是方向。
02 任务拆解要具体 "重构代码"不是任务。"将 X 函数从 A 移到 B"才是可执行的任务。拆到不能再拆为止。
03 持久化是核心 不记录到文件系统的讨论，下次会话就会丢失。文档即记忆，file system 是最好的大脑。
04 验证要对照合同 完成标志必须提前定义。没有验收标准的任务，永远做不完也做不对。
05 五文件闭环 需求 → 拆解 → 持久化 → 追踪 → 验证。闭环意味着每一步都有下一步，最后一步回到需求。
05 — ARCHITECTURE

架构理解

CLI-first + Agentic Coding，两个核心概念定义了 Claude Code 的设计哲学。

CLI-FIRST
命令行优先
终端是界面，文件系统是状态，git 是版本控制。比 IDE 集成更轻量、灵活、可脚本化。不依赖 GUI，不需要鼠标。
轻量 可脚本化 无处不在
AGENTIC CODING
主动代理
不只是补全代码，而是主动完成任务 — 编辑文件、运行命令、创建提交、管理分支。AI 是执行者，不是建议者。
全栈执行 工具调用 状态感知

07 — MASTERY

掌握的关键指标

四个指标，比任何证书都可靠。不在于学了多久，在于能做什么。

指标一 3min 创建一个新的 CLAUDE.md。理解上下文管理的核心，从零到可用。
指标二 1句 说清每个组件的用途。理解工具的定位，不被文档绑架。
指标三 自主 设计新的 hooks。理解事件驱动机制，不为框架所限。
指标四 自动化 为重复任务创建 commands。自动化思维形成肌肉记忆。
