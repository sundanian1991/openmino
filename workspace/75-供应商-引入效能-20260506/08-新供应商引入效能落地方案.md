# 项目8：新供应商引入效能 — 落地执行方案

> 作者：Mino | 日期：2026-05-06 | 状态：待执行 | 优先级：P8

## 一、项目定位

### 1.1 为什么要做

当前供应商引入流程从初筛到准入平均耗时 30-45 天，其中存在大量非增值等待时间。30+ 供应商中，近三年新引入转正的仅 8 家，通过率不足 40%。引入效能低下直接导致：核心业务线供应商池深度不足（金条人工仅 6 家核心供应商扛 3 亿+产能），AI 产线供应商池更薄（集中度上限 70% / 3 家），一旦头部供应商出现合规或交付异常，无即刻替补。

### 1.2 解决什么问题

1. **引入周期过长**：从接触到入库 ≥30 天，关键审批节点（C3 ≤3天、C2 ≤5天）常被拖延
2. **筛选标准模糊**：初筛阶段缺乏量化评分卡，依赖主观判断，导致"看着不错、进来不行"
3. **测试期管理粗放**：核心业务测试 3-6 个月（10-100 人）、创新业务 ≤12 周（5-10 人），但过程无里程碑监控
4. **准入决策低效**：评分 ≥85 入库、<85 需特批（C3+C2），但评分表字段与业务线实际诉求不完全匹配
5. **引入与运营脱节**：新供应商入库后 90 天内异常率 35%，远高于成熟供应商 12%

## 二、现状分析

### 2.1 当前状态

引入流程分为五阶段：需求提出 → 市场初筛 → 资质审核 → 测试验证 → 准入审批。对应制度文件：
- `docs/供应商/管理制度/供应商引入管理规范.md`（31 页，24984 字）
- `docs/供应商/管理规范/供应商引入SOP-流程与模板汇编.docx`
- `docs/工具箱/管理场景梳理/E1-新供应商初筛.md`（决策卡片）
- `docs/工具箱/管理场景梳理/E2-准入评审决策.md`（决策卡片）

准入硬性门槛：
- 注册资本 ≥1000 万
- 持有呼叫中心经营许可证
- 资产负债率 ≤60%
- 在岗人员 ≥50 人

审批权限：C3（≤3 天）→ C2（≤5 天），逐级流转。

### 2.2 已知痛点

| 痛点 | 影响 | 根因 |
|------|------|------|
| 初筛靠经验，无量化评分 | 引入质量不稳定，进来再筛成本高 | 缺少标准化评分卡 |
| 资质审核与实地看场分离 | 看场安排在资质审核后 1-2 周，拉长周期 | 流程串行而非并行 |
| 测试期无里程碑检查 | 3-6 个月测试期中间无 checkpoint，到期才发现问题 | 过程管理缺位 |
| 评分表字段与业务线诉求不匹配 | AI 产线和人工产线用同一套评分表 | 未分产线定制 |
| 特批流程使用率高 | <85 分仍需入库的占比 ~30%，特批频繁 | 评分标准与市场实际脱节 |
| 新供应商入驻后 90 天无人盯 | 异常率高，运营团队接手晚 | 引入与运营交接不清晰 |

## 三、实施框架

### 3.1 核心方法论

**"三步前置 + 两步并行 + 一步闭环"**

```
三步前置：需求前置（明确要什么）→ 标准前置（准入门槛量化）→ 风险前置（合规尽调提前）
两步并行：资质审核 ∥ 实地看场（同周完成）→ 测试执行 ∥ 过程评估（里程碑同步）
一步闭环：入库 → 90天护航 → 正式交接运营
```

### 3.2 评估维度与权重

引入评分表（满分 100 分，≥85 入库）：

| 维度 | 权重 | 子项 | 评分标准 |
|------|------|------|----------|
| **资质实力** | 25% | 注册资本（5 分）| ≥5000 万=5，≥2000 万=4，≥1000 万=3 |
| | | 人员规模（5 分）| ≥200 人=5，≥100 人=4，≥50 人=3 |
| | | 场地规模（5 分）| 金条人工：二职场 ≥40 工位=4，≥60=5；三职场 60+ 且 top50%=5 |
| | | 资质证照（5 分）| 呼叫中心许可证+ISO27001=5，仅许可证=3 |
| | | 财务状况（5 分）| 资产负债率 ≤40%=5，≤50%=4，≤60%=3 |
| **业务能力** | 30% | 同业经验（10 分）| 有金融电销经验=8，头部机构经验=10 |
| | | 技术能力（10 分）| AI+人工混合运营能力=10，纯人工=5 |
| | | 管理团队（5 分）| 项目经理 PMP/6σ=5，无认证=3 |
| | | 质量体系（5 分）| 有 QA 团队+质检流程=5，仅事后质检=2 |
| **合作意愿** | 20% | 投入承诺（5 分）| 承诺专职团队+专属职场=5 |
| | | 价格竞争力（5 分）| 单价低于市场均价 10%+=5，持平=3 |
| | | 响应速度（5 分）| 24h 响应机制=5，48h=3 |
| | | 战略契合（5 分）| 认同我方管理理念=5，仅商业合作=3 |
| **合规记录** | 15% | 历史合规（10 分）| 无处罚记录=10，轻微=7，有红线=0（直接淘汰） |
| | | 数据安全（5 分）| 通过等保三级=5，等保二级=3 |
| **创新能力** | 10% | AI 应用（5 分）| 有自研 AI 工具=5，使用第三方=3 |
| | | 优化提案（5 分）| 能主动提出流程优化建议=5 |

### 3.3 分级/分类标准

按业务线差异化引入标准：

| 业务线 | 测试期 | 测试规模 | 准入分数 | 特殊要求 |
|--------|--------|----------|----------|----------|
| 金条-人工 | 3-6 个月 | 10-100 人 | ≥85 | 二职场 ≥40 工位，n-1 原则 |
| 金条-AI | ≤12 周 | 5-30 人 | ≥80 | 需有 AI 运营经验，集中度 ≤3家/70% |
| 企金 | 3-6 个月 | 10-50 人 | ≥85 | 企金行业经验优先 |
| 信用卡 | 3-6 个月 | 10-50 人 | ≥85 | 信用卡催收经验 |
| 财富 | 3-6 个月 | 10-50 人 | ≥80 | 理财资质优先 |
| 创新业务 | ≤12 周 | 5-10 人 | ≥75 | 快速验证，灵活准入 |

### 3.4 决策矩阵

| 评分区间 | 决策 | 审批流 | 说明 |
|----------|------|--------|------|
| ≥90 | 直接入库 | C3 审批 | 优质供应商，快速通道 |
| 85-89 | 正常入库 | C3 审批 | 达标，标准流程 |
| 75-84 | 有条件入库 | C3 + C2 特批 | 需在合同中约定改进项和时限 |
| 60-74 | 进入储备库 | C3 备案 | 暂不入库，持续跟踪，6 个月后复评 |
| <60 | 淘汰 | 无需审批 | 记录淘汰原因，12 个月内不再接触 |

红线一票否决（任一项触发即终止流程）：
- 历史数据泄露事件
- 资质证书造假
- 数据造假记录
- 录音篡改/篡改监控数据

**动作 3：审批 SLA 规则确认**（年老师与 C3/C2 确认）
- 明确 C3 审批 ≤3 天、C2 ≤5 天的具体起止时间定义

- 目标：每个业务线至少 2 家储备候选
- 渠道：行业推荐、招标网、同业交流
- 产出：储备供应商候选名单（Excel）

### 4.2 第 3-6 周：执行阶段

**动作 5：试点供应商引入**（选择 1-2 家正在接触中的供应商）
- 使用新版评分表和流程执行完整引入
- 资质审核与实地看场并行安排（同周完成）
- 记录每个节点的实际耗时，与旧流程对比

**动作 6：测试期里程碑设置**
- 核心业务（3-6 个月测试）：设置 3 个 checkpoint
  - 第 4 周：人员到岗率 ≥80%，基础培训完成率 100%
  - 第 8 周：转化效率达到基准线 70%，合规检查 0 红线
  - 第 12 周：全面评估，决定是否继续/提前转正/终止
- 创新业务（≤12 周）：设置 2 个 checkpoint
  - 第 4 周：MVP 验证通过
  - 第 8 周：核心指标达标

**动作 7：90 天护航机制建立**
- 新供应商入库后 90 天内：
  - 周度：专项监控报表（出勤、业绩、合规三项）
  - 月度：专项评估（独立于月度综合评估）
  - 第 30 天：首次复盘会（年老师 + 供应商负责人）
  - 第 60 天：中期复盘
  - 第 90 天：转正评估 → 移交日常运营

### 4.3 第 7-8 周：验收阶段

**动作 8：试点复盘**
- 对比新旧流程指标（周期、通过率、90 天异常率）
- 产出：试点复盘报告

**动作 9：制度固化**
- 将验证有效的流程更新至 `docs/供应商/管理制度/供应商引入管理规范.md`
- 新增模板正式编号入册

**动作 10：培训宣贯**
- 对内：服务组全员培训新流程
- 对外：向核心供应商发送新版管理要求（SM-TPL-012 确认书）

| 历史处罚记录 | 供应商声明+自查 | 信用中国/监管公开信息 |

### 5.2 评分/评估表（详细字段）

**引入评分表**（共 5 维度 16 子项）：

评分公式：`总分 = Σ(子项得分 × 权重系数)`

每子项评分说明（以"资质实力-注册资本"为例）：
- 5 分：注册资本 ≥5000 万，且实缴 ≥80%
- 4 分：注册资本 ≥2000 万，且实缴 ≥60%
- 3 分：注册资本 ≥1000 万（最低门槛），实缴 ≥50%
- 2 分：注册资本 ≥1000 万，实缴 <50%
- 1 分：注册资本 <1000 万（触发硬性淘汰）

评分人：初筛评分（服务组专员）→ 复核评分（年老师）→ 终评（C3）

### 5.3 输出模板（报告结构）

**新供应商引入评估报告**结构：

```
一、供应商基本信息（1页）
   - 公司概况、股权结构、核心管理团队
二、资质审查结论（1页）
   - 硬性门槛逐项验证结果（通过/不通过）
   - 资质证书有效性验证
三、评分详情（2页）
   - 五维度得分雷达图
   - 各子项评分明细
   - 与同业供应商对比
四、实地看场记录（2页）
   - 场地照片、工位统计、设施评价
   - 一线员工访谈摘要
五、测试期计划（1页）
   - 测试业务线、规模、周期
   - 里程碑节点和验收标准
六、风险与关注项（0.5页）
   - 已知风险点
   - 建议的合同中特别约定
七、审批意见（0.5页）
   - C3 意见 + 日期
   - C2 意见（如需特批）+ 日期
```

### 5.4 沟通话术（对内/对外）

**对内 — 向 C3/C2 发起审批**：

> "XX 公司已完成引入评估全流程，综合评分 XX 分（≥85 达标 / <85 申请特批）。该公司在 [核心优势] 方面表现突出，主要风险在 [风险点]，已通过 [措施] 缓释。附件为完整评估报告，请审批。C3 审批时限 ≤3 天。"

**对内 — 向服务组同步新供应商信息**：

> "XX 公司将于 [日期] 正式进入测试期，业务线 [XX]，测试规模 [XX 人]。请各位在测试期间重点关注：1）[关注点1]；2）[关注点2]。周报中新增该供应商专项数据。"

**对外 — 向供应商发送准入通知**：

> "恭喜贵公司通过我方供应商准入评审。请于收到本通知后 5 个工作日内签署《供应商管理办法确认书》（SM-TPL-012）及《SLA 确认书》（SM-TPL-013），并反馈扫描件。测试期将于 [日期] 启动，具体安排见附件《测试期计划书》。"

**对外 — 向供应商发送不通过通知**：

> "感谢贵公司参与我方供应商准入评审。经综合评估，贵公司在 [具体维度] 方面暂未达到我方要求。我们已将贵公司纳入储备库，将于 6 个月后邀请复评。如有异议，可在 10 个工作日内提交补充材料申请复议。"

## 七、风险与预案

### 7.1 预期风险清单

| 风险 | 概率 | 影响 | 预警信号 |
|------|------|------|----------|
| 审批超时 | 高 | 中 | C3 超过 2 天未处理 |
| 供应商提供虚假材料 | 中 | 高 | 资质验证不一致 |
| 测试期指标不达标 | 中 | 高 | 第 4 周 checkpoint 未通过 |
| 市场优质供应商不足 | 中 | 高 | 3 个月无新候选 |
| 新供应商 90 天内出合规问题 | 低 | 极高 | 首次质检即发现红线 |

### 7.2 触发信号

| 信号 | 级别 | 响应时限 |
|------|------|----------|
| 审批超时 48h | 黄色 | 自动提醒 + 升级 |
| 资质验证不一致 | 橙色 | 24h 内启动复核 |
| 测试期第 4 周未通过 | 橙色 | 48h 内制定改进计划 |
| 发现虚假信息 | 红色 | 立即终止流程 |
| 测试期红线问题 | 红色 | 立即终止 + 列入黑名单 |

### 7.3 应对措施

**审批超时**：
1. 自动提醒（第 2 天）→ 升级提醒（第 3 天）→ 直批通道（第 4 天）
2. 月度统计超时率，纳入审批人绩效考核

**虚假材料**：
1. 立即终止引入流程
2. 列入供应商黑名单，12 个月内不再接触
3. 如已签署合同，启动违约条款

**测试期不达标**：
1. 第 4 周未通过 → 给予 2 周改进期
2. 第 6 周复评仍未通过 → 终止测试
3. 记录终止原因，作为储备库复评参考
