# 策展大纲 · handoff-1

> Step 0a 产出。按 curatorial-topic-framing + curatorial-narrative-modules 流程。
> 素材：42章经播客《少有的深度参与过字节、美团组织建设的人》——曲凯 × 魏小康对话，原文 18000 字，微信删减版约 9000 字。

---

## 对象界定（topic-framing 产出）

### 策展对象
一场关于"组织建设与招聘"的**深度播客访谈的中文删减稿**。注意——策展对象不是"魏小康的组织建设方法论体系"，是**这篇经过 42章经编辑删减的访谈文本本身**：它把一场 18000 字的播客压成 9000 字，保留了曲凯的提问框架 + 魏小康的回答 + 少量编辑过渡。

### 真实主体（关键）
表面在讲"魏小康谈组织建设"，真实主体是：**一份"创业公司该怎么把组织搭起来"的操作地图——核心不是文化/绩效/培训这些 HR 大词，是"把 80% 的时间花在招聘的前两步（想需求 + 找供给）"这个反直觉的方法论**。整篇所有的"文化/绩效/培训/职级"内容都在被"砍掉"——魏小康反复在说这些不重要、不要花时间。真正的重心只有一个：**找供给是组织建设里最重要的事，没有之一**。

### 进入尺度
**方法尺度 + 反直觉尺度**——通过"该砍什么 vs 该重压什么"的对比进入。不是按"组织建设的几个模块"铺陈（那是教科书），是按"创业者最容易搞错的轻重缓急"进入：你以为面试重要，其实需求+供给重要；你以为职级体系重要，其实没必要搞；你以为培训能培养人，其实人是打仗打出来的。

### 目标受众
正在创业或在大厂带团队的决策者——尤其那些"正在为招人/管人发愁"的人。他们读完应该带走：**一个明确的优先级重排**（招聘前两步 > 一切），而不是"魏小康说了哪些观点"。

### 材料边界
- **可靠**：魏小康的任职履历（2017-2020 字节、2020-2026 美团）、721 比例、Google/Amazon/字节/美团的组织对比、"字节给 1.5x 干 3x 活"的策略、"99% 蓝领被骗过"的估计——这些是访谈的事实层
- **存疑**：
  - "985 前 1% 就够做几十亿美元公司"——魏小康的经验估计，不是统计结论，需标注为"个人判断"
  - "创业时最少三分之一看走眼"——同样是经验估计
  - "Google 几千人的招聘团队"——常识性数字但未给精确来源
  - "蓝领求职者 99% 都被骗过"——极强的断言，无数据来源，必须标注存疑
- **缺失**：原文是删减版（9000/18000 字），完整播客未提供；魏小康的 AI 招聘创业公司名称、产品细节未提及
- **三层声音**（这篇访谈特有的 fidelity 风险）：
  - **魏小康的原话观点**（访谈主角，方法论持有者）
  - **曲凯的提问框架**（曲凯不是中性采访者，他的提问在引导方向——比如"是不是该招一批人淘汰""去了智谱 MiniMax 走掉的人挺多"）
  - **42章经的编辑裁剪**（删减掉一半内容，编辑判断哪些留哪些砍）
- **排除**：魏小康个人履历细节（具体在字节哪个部门、美团哪个业务线）——不是这篇访谈的重心

### 排除方向（★设计判断的关键输入）
- **排除教科书式组织管理框架**（选/用/育/励/汰 平铺五块）——魏小康自己都在砍这些，长文不能反而把它们当结构
- **排除 HR 大词堆砌**（文化/绩效/职级/培训 各开一段）——这些是魏小康明确说"不重要/不要搞"的，长文要 preserve 这个"砍"的态度
- **排除纯语录集**——访谈有明确的因果论证（为什么招聘前两步最重要），不能压成金句墙
- **排除人物专访体例**（"魏小康是谁→他说了什么→总结"）——履历只是信用背书
- **必须保留三层声音的视觉区分**——这是访谈体（不是文章体）的核心 fidelity 问题：魏小康的方法论断言 vs 曲凯的引导性提问 vs 编辑删减的过渡，必须视觉可分
- **必须保留"反直觉"的论证张力**——每个"不要做 X"的建议，都对应一个"创业者以为该做 X"的常识假设，这个张力不能抹平

## 叙事结构（narrative-modules 产出）

### 内容形式
**CT-1 长文 / 深度访谈解读**。三声部访谈体（魏小康方法论 + 曲凯提问 + 编辑裁剪）。和 socher 那篇的"双声部"（作者归纳 + 引用）不同——这篇访谈里曲凯不是中性组织者，他的提问有引导性（"是不是该淘汰""00后能不能脱颖而出"），所以是**三声部**：主角的方法论 + 提问者的框架引导 + 编辑的删节痕迹。

### 叙事弧线
```
引子（魏小康是谁 + 这场对话的核心增量：组织=把人选对）
  → 重新定义"创始人特质"（4点个人层 + 1个组织层——组织才是决定性的）
  → 砍掉 HR 大词（文化/绩效/职级/培训逐个砍——每个"不要做"都是反直觉）
  → 重压招聘前两步（想需求 + 找供给 = 80%时间——这是全文重心）
  → 谈判谈预期（钱不是核心，三五年想要什么才是）
  → 海外标杆对比（Google/字节 vs Amazon/美团——两套逻辑三维坐标）
  → AI 时代的反主流判断（OPC 不靠谱 + 00后打不过90后 + AI coding 是硬条件）
  → 收束（魏小康自己的创业——从方法论到产品）
```

**弧线特点**：**沙漏型**——上宽（创始人特质→砍 HR 大词，是"减法铺垫"），中窄（招聘前两步是唯一重心，全文收束到这一个点），下宽（海外对比→AI时代→创业，是"这个方法论在不同尺度的应用"）。**不是平铺五模块**（选育用励汰），是**所有内容都收束到"找供给是唯一重要的事"这个针尖**。

### 页面类型表（映射展览模型 → 长文章节）

| 策展页面类型 | 在这篇长文里的映射 | 角色 |
|-------------|------------------|------|
| 入口页 | Article Header | 标题主张"组织=把人选对" + 三声部 meta |
| 核心专题页（铺垫层）| 章节 01-02 | 创始人特质 + 砍 HR 大词（为重心蓄力）|
| **物件页（核心展品）** | **章节 03-04** | **招聘前两步 + 谈判预期——全文重心，视觉最强** |
| 跨单元分析页 | 章节 05 | 海外标杆对比（把方法论放到全球坐标系）|
| 当代延伸页 | 章节 06 | AI 时代的变与不变 |
| 关于页 | 章节 07 + Article Footer | 魏小康的创业 + 受访者信息 |

**核心展品是章节 03-04**——这是整篇访谈的"针尖"，其他章节都是它的铺垫或延伸。视觉上这两章必须有更强的重量感（不能和铺垫章节同等对待）。

### 交互逻辑
线性阅读 + Reading Progress。章节 03（招聘前两步）用 Callout 强调"80% 时间"这个核心发现。章节 05（海外对比）用 Data Table 做"两套逻辑"的并列。无跳转、无展开。

## 一句话压缩（ljg 式检验）

> 魏小康借一场深度访谈，把"组织建设"从 HR 大词（文化/绩效/职级/培训）砍回到一件事——把 80% 的时间花在招聘的前两步（想清楚需求 + 找到供给）；人不是培养出来的，是打仗打出来筛出来的；AI 时代加了"会 AI coding 是硬条件"这把新尺子，但"每个方向都要一个好手"的判断没变。

**压得成，不糊弄。** 关键词："砍回到""前两步""打仗打出来""AI coding 是硬条件但判断没变"——每个词都排除了错误的内容形式（"砍"排除教科书式平铺，"前两步"排除五模块均衡，"打仗"排除培训体系叙事，"判断没变"排除"AI 颠覆一切"的恐慌叙事）。

## 自检（策展阶段常见偏差扫描）

策展 SKILL.md 列了 5 个常见偏差，逐一对照：

| 偏差 | 这次有没有踩 |
|------|-------------|
| 人物成为全部故事，背后结构消失 | ⚠️ 风险——魏小康履历（字节+美团双背景）很有故事性，容易做成"大佬访谈录"。已通过"真实主体=招聘前两步方法论"对冲 |
| 材料变数据库，缺策展判断 | ⚠️ 风险——访谈覆盖面广（创始人/文化/绩效/招聘/海外/AI），容易做成"魏小康观点汇编"。已通过"沙漏型弧线收束到找供给"对冲 |
| 历史图像/视觉复原变成项目成立理由 | ✅ 不适用 |
| 主题过宽，每页都要重释背景 | ✅ 没踩——各章节独立，不需互相解释 |
| 对象未稳定就先决定交互/视觉 | ✅ 没踩——视觉决策（章节03-04加重、Callout、Data Table）留给施工图 |
