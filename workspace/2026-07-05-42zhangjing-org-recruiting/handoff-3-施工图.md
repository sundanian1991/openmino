## 内容形式（来自策展）
CT-1 长文 / 三声部访谈解读（魏小康方法论 + 曲凯引导提问 + 编辑删减痕迹）。沙漏型弧线：铺垫（砍 HR 大词）→ 针尖（招聘前两步 = 80% 时间）→ 延伸（海外对比 + AI 时代 + 创业）。
→ 视觉形式：**CT-1 长文组件链**，但需为"三声部"做视觉区分，且针尖章节（03-04）视觉权重大于铺垫章节。

## 施工硬约束（来自审查 🔴/🟡，必须 preserve）

1. **三层声音视觉区分**（Journalist 🔴）：
   - 魏小康的方法论断言 → 主文 prose，strong 强调
   - 曲凯的引导性提问 → dialogue 块明确标"曲凯"，视觉降权（次要色 + 小字）
   - 编辑删减过渡 → meta/eyebrow 处理，不放正文
2. **未核实数字 inline 标记**（Journalist 🔴）：用 `<sup class="cite-est">※</sup>` 标注所有经验估计数字（985 1% / 三分之一 / 99%蓝领 / 5-10倍等），文末统一脚注"※ 魏小康个人经验估计，无公开数据来源"
3. **"99% 蓝领被骗过"特别处理**（Journalist 🔴 最高优先级）：这个数字即使加了 ※ 标记也够强，要在正文里紧跟一句"这是一个极强的判断，无公开数据支持"——用编辑性括注（【】或脚注形式），让读者明确意识到这是断言不是事实
4. **章节 03 针尖视觉加重**（Novelist 🔴）：招聘前两步这章用 Callout 强化"80% 时间花在前两步"这个核心发现，且这章的 chapter-header 可以略重于铺垫章节
5. **曲凯的提问保留并角色显式化**（Novelist 🔴）：曲凯不是中性话筒，他的提问（淘汰/00后/智谱MiniMax）在引导方向——dialogue 块要保留，且通过视觉降权（次要色）暗示"这是提问者的常识假设，不是事实陈述"
6. **"打仗筛人 vs 没资格淘汰"张力保留**（Critic 🟡）：章节 03 用魏小康原话"组织是服务业务需要的，不是服务淘汰需要的"做解药，不抹平矛盾

## 施工弹性区（来自审查，可权衡）

## 进入 8 问

### Q1 内容本质上在做什么？
**传递一个"优先级重排"的方法论**——把"组织建设"从 HR 五模块（选育用励汰）砍回到一件事（招聘前两步 = 80% 时间）。一句话本质：用"砍刀 + 重压"的姿态，把创业者的注意力从"该搞文化/绩效/职级"重定向到"该花时间想需求 + 找供给"。
**→ CT-1 长文。Article Header 开篇，非 Hero 母版**（长文阅读优先，不占满屏 100vh——这是 CT-1 规则，socher 那篇的实践）。

### Q2 primary（唯一锚点）？
**"把 80% 的时间花在招聘的前两步——想清楚需求 + 找到供给。"** 这是整篇访谈所有内容收束的针尖。其他章节（创始人特质/砍HR大词/海外对比/AI时代）都是这个 primary 的铺垫或延伸。
- Article Header 标题主张必须指向这个 primary（不是"魏小康谈组织"，是"组织砍回到找供给"）
- 眯眼测试：第一眼看到的是"砍/重压/前两步"这个动作，不是"魏小康的方法论体系"

### Q3 情绪温度？
**精确/权威 + 反直觉冲击**。不是温润叙述（socher 那篇的工具箱温度），是**砍刀手的权威感**——魏小康在连续砍（文化/绩效/职级/培训），读者要感到"这个人在重排我的优先级"。不是恐慌（不是"你的组织完蛋了"），是**被校正**（"你以为重要的事其实不重要"）。
→ 配色承诺：**中克制**，主底 offwhite + forest-soft 浅底绿点缀。深段 ≤1 处（footer 收尾）。章节 03 针尖可以用 cream 底色微染强化（不进深段）。

### Q4 信息层次？
- **primary**：招聘前两步 = 80% 时间（章节 03，全文重心）
- **secondary**：
  - 砍掉的 HR 大词（章节 02：文化/绩效/职级/培训逐个砍）
  - 谈判谈预期（章节 04：钱不是核心，三五年想要什么才是）
  - 海外两套逻辑（章节 05：Google/字节 vs Amazon/美团）
- **tertiary**：
  - 创始人四特质（章节 01：铺垫，非重心）
  - AI 时代判断（章节 06：延伸）
  - 魏小康创业（章节 07：收束）

**关键**：章节 01（创始人特质）是 tertiary——访谈开篇聊创始人特质，但这不是访谈的重心，重心是后面的组织/招聘。不能让章节 01 抢了章节 03 的视觉权重。

### Q5 叙事弧线？
**沙漏型**：
```
引子（魏小康是谁 + 核心增量：组织=把人选对）
  ↓ 蓄力
章节 01 创始人四特质（个人层铺垫）
章节 02 砍 HR 大词（连续砍：文化/绩效/职级/培训）
  ↓ 收束到针尖
章节 03 招聘前两步 = 80% 时间 ★全文重心★
章节 04 谈判谈预期（针尖的延伸——找到人后怎么谈）
  ↓ 延伸到不同尺度
章节 05 海外标杆对比（全球坐标系）
章节 06 AI 时代的变与不变
章节 07 魏小康创业（从方法论到产品）
```
**不是平铺五模块**，是**所有内容收束到"找供给是唯一重要的事"这个针尖**。Novelist 已警告：不能所有章节一个模子，针尖要重。

### Q6 克制度？
**中克制**。
- 120px section padding（CT-1 长文标准）
- 留白为主，正文收窄到 720px（阅读宽度）
- 章节间用 chapter-header rule（forest 2px 短线）分隔，不用粗分隔条
- **针尖章节 03 用 Callout 强化"80% 时间"**（cream 底 + forest 左边框，但不进深段）
- **章节 05 用 Data Table 做两套逻辑对比**（结构化，但不滥用数据组件）
- 深段：仅 footer 收尾（black 底），全文 1 处深段

### Q7 视觉气质？
**建筑感为主（结构、砍刀、权威）+ 测绘感点缀（章节编号、数据精度）**。
- 不是手写感（这不是温润叙述，是方法论校正）
- 不是有机感（不是品牌叙事）
- → chapter-header 编号 + forest rule + mono 标签（测绘基准线）做测绘感
- → prose 收窄 720px + Drop Cap 开篇 + dialogue 三层声音区分（建筑感的结构清晰）
- 装饰层选择：**前缀线 + forest rule**（结构型，不花哨）

### Q8 换品牌成立吗？
**否**。核心是"招聘前两步 = 80% 时间"这个反直觉方法论 + 三层声音的访谈体 fidelity 处理。换成别的话题（比如不是组织建设是产品方法论），这个"砍+重压"的沙漏结构不成立——它的形态服从"把宽主题砍回到一件事"这个特定内容。

### 反同质化检查（craft-checklist C4）

近 3 次 haglofs 长文 Hero/Header 选型：
- socher 那篇：Article Header + 6 个并列方法块（工具箱）
- anthropic 那篇：Article Header + 归纳式弧线（数据→反转→收束）
- 这篇：**Article Header + 沙漏型弧线（铺垫→针尖→延伸）+ 三层声音 dialogue 区分**

**判定**：弧线形态不同（沙漏 vs 工具箱 vs 归纳），且三层声音处理是这篇独有的 fidelity 创新。✅ 不触发强制换型。
