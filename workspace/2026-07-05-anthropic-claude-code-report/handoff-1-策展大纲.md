# 策展大纲 · handoff-1

> Step 0a。素材：Anthropic 官方研究报告《Agentic coding and persistent returns to expertise》（第一手原文）。

---

## 对象界定

### 策展对象
Anthropic 发布的**官方研究报告本身**——基于 40 万次 Claude Code 会话的实证分析。注意：这次策展对象就是原文，没有中间解读层。前两次（中文解读文章、Socher 访谈总结）都是二手材料，这次是一手。

### 真实主体
表面是"40 万次会话的数据描述"，真实主体是：**三个被实证支持的论断——(1) 人机分工清晰（人定 what，AI 定 how），(2) 领域专业知识回报持续存在（domain expertise，不是 coding proficiency），(3) 这种回报如果未来下降，意味着模型开始补上判断力**。第三个是最深的——它是面向未来的观察指标。

### 进入尺度
**数据尺度**——通过 6 张图（任务类型分布、分工比例、专业度回归、错误恢复、职业成功率、任务价值趋势）进入。每个论断都有图表 + 统计支撑。

### 目标受众
AI 协作的从业者 + 关注劳动力市场变化的人。读完应该带走：不是"AI 替代程序员"的恐慌或安慰，是**一套可追踪的指标**——用这些指标，自己判断趋势。

### 材料边界
- **可靠**：所有数字直接来自原文（70% planning / 20% execution / 40万会话 / 23.5万用户 / 七个月 / 9 个工作模式 / 职业成功率差 ≤7pp / 专家-中级差距 modest / 任务价值 +27%）
- **可靠**：原文的限定条件（"verified success"严格定义、回归控制变量、95% 置信区间）
- **可靠**：原文的克制表述（"may be a preview"、"if these patterns hold"、"suggests"——不夸大）
- **缺失**：附录里的完整方法论（session 界定、occupation 推断、expertise 评分）——技术细节
- **排除**：中文解读文章的延伸（"管理动作=操作 AI 的技术动作"等）——这次不用二手延伸

### 排除方向（★关键）
- **排除长文体例**——这是数据报告，不是带作者声音的深度文章
- **排除访谈总结体例**——没有"双声部"，是机构单一署名
- **排除恐慌/安慰叙事**——原文 deliberately 克制（"may""if""suggests"），不要戏剧化
- **保留原文的学术克制**——所有论断标注限定条件，不把"may"翻译成"will"
- **保留三个论断的递进结构**——分工 → 专业度回报 → 未来观察指标

## 叙事结构

### 内容形式
**P2 数据简报 / 研究报告**。机构署名、硬数据驱动、有图表、有置信区间、有附录。这是数据组件的主场——Stat Grid / Elevation / Seam Benchmark / V5 Pulse Hero 在这里是正解，不是错配。

### 叙事弧线
```
Pulse Hero（KPI 概览：40万/23.5万/7个月/9模式）
  → 概念建立（这份报告为什么重要 · "may be a preview"）
  → 分工发现（Stat Grid：70% planning / 20% execution + 9 个工作模式）
  → 专业度回报（Elevation/Seam：domain expertise > coding proficiency · 专家-中级 modest）
  → 错误恢复（Tension Grid：专家 vs 新手翻车后恢复率）
  → 职业成功率（Data Table：10 个职业 + ≤7pp 差距）
  → 未来观察指标（Callout：if returns decrease → 模型补判断力）
  → Footer（报告来源 + 附录链接）
```

**弧线特点**：数据漏斗（Hero 概览 → 逐层展开 → 洞察收束）。这是 R3-数据分析中心的经典弧线，rules-narrative.md 的"方向→趋势→差距→结构→分布→决策"漏斗。
