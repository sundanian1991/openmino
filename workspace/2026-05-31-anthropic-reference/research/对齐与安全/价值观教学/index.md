# 教导 Claude 理解为什么

![教导 Claude 理解为什么](images/teaching-claude-why_01041aa942.svg)

去年，我们发布了一份关于[自主行为失对齐](https://www.anthropic.com/research/agentic-misalignment)的案例研究。在实验场景中，我们发现来自不同开发者的 AI 模型在遇到（虚构的）伦理困境时，有时会采取严重失对齐的行动。例如，在一个被广泛讨论的例子中，模型通过勒索工程师来避免被关闭。

当我们首次发表这项研究时，我们最强大的前沿模型来自 Claude 4 系列。这也是我们首次在训练过程中进行实时对齐评估的模型系列；自主行为失对齐是浮现出的若干行为问题之一。因此，在 Claude 4 之后，我们清楚地意识到需要改进安全训练，此后我们对安全训练进行了重大更新。

我们以自主行为失对齐为案例，重点介绍一些我们发现效果惊人的技术。事实上，自 Claude Haiku 4.5 以来，每一款 Claude 模型在自主行为失对齐评估中都取得了满分——也就是说，这些模型从不进行勒索行为，而之前的模型有时会高达 96% 的情况出现这种行为（Opus 4）。不仅如此，我们在[自动化对齐评估](https://www-cdn.anthropic.com/bf10f64990cfda0ba858290be7b8cc6317685f47.pdf)中持续看到其他行为的改善。

在这篇文章中，我们将讨论我们在对齐训练中所做的若干更新。从这项工作中，我们总结了四个主要经验：

**失对齐行为可以通过在评估分布上直接训练来抑制——但这种对齐可能无法很好地泛化到分布外（OOD）。** 在与评估非常相似的提示上进行训练可以显著降低勒索率，但在我们留出的自动化对齐评估上没有改善表现。**然而，进行有原则的对齐训练是可能泛化到分布外的。** 例如，关于 Claude 宪法的文档和描述 AI 行为可敬的虚构故事，尽管与我们的所有对齐评估*极度*分布外，仍然改善了对齐。**仅在*示范*期望行为上进行训练往往是不够的。** 相反，我们最有效的干预更深入：教导 Claude 解释*为什么*某些行动优于其他行动，或者用更丰富的描述来训练 Claude 的整体角色特质。总体而言，我们的印象是——正如我们在 Claude 宪法的讨论中所假设的那样——教授对齐行为背后的*原则*可能比单独训练对齐行为的示范更有效。两者结合似乎是最有效的策略。

**数据的质量和多样性至关重要。** 我们发现，通过迭代改进训练数据中模型回复的质量，以及通过简单方式增强训练数据（例如，包含工具定义，即使没有被使用），都能带来持续的、令人惊讶的改进。

### 为什么会出现自主行为失对齐？

在我们开始这项研究之前，失对齐行为的来源并不清楚。我们的两个主要假设是：

- 我们的后训练过程意外地通过失对齐的奖励鼓励了这种行为。
- 这种行为来自预训练模型，而我们的后训练未能充分抑制它。

我们现在认为，假设（2）是主要原因。具体来说，在 Claude 4 训练时，我们绝大多数的对齐训练是标准的基于聊天的从人类反馈中强化学习（[RLHF](https://www.anthropic.com/research/training-a-helpful-and-harmless-assistant-with-reinforcement-learning-from-human-feedback)）数据，其中不包含任何智能体工具使用。这在之前足以对齐主要用在聊天场景中的模型——但对于像自主行为失对齐评估这样的智能体工具使用场景，情况并非如此。

为了调查这一点，我们在一个 Haiku 级别（即较小的）模型上运行了聚焦对齐数据的缩小版后训练流程，发现自主行为失对齐率仅略微下降，在训练早期就趋于平稳。有关进一步调查行为来源的实验，请参阅[扩展博文](https://alignment.anthropic.com/2026/teaching-claude-why/)。

### 提高对齐专用训练数据的质量：理由比行动更重要

我们尝试在数据上训练 Claude，这些数据显示出抵制类似评估中蜜罐的倾向。在这些数据中，模型可能有机会破坏竞争 AI 的工作以推进自己的目标（如其系统提示中所给定的），或保护自己不被关闭——这对实现其目标是工具性的。我们通过对每个提示采样模型并筛选出助手选择*不*落入蜜罐的案例来生成训练数据。尽管与评估分布非常匹配，我们发现这种方法出乎意料地不成功——仅将失对齐率从 22% 降至 15%。

我们通过重写回复使其还包含模型对自身价值观和伦理的审慎思考，显著改善了这一点（将失对齐降至 3%）。这表明，*尽管在对齐行为上进行训练有帮助，但在助手为其对齐行为展示出可敬的推理过程的示例上进行训练效果更好*。

然而，直接针对评估场景进行训练出于多种原因并非最优。理想情况下，我们想要的是一个非常不同的训练分布，使我们能够在评估上取得改进，因为这将使我们更有信心，训练可以*泛化*到未被评估捕获的其他部署分布。

我们最终确定了一个更分布外的训练集，其中用户面临一个伦理上模糊的情境：他们可以通过违反规范或规避监督来实现合理目标。助手被训练（使用监督学习）给出深思熟虑的、有细微差别的回复，与 Claude 的宪法保持一致。值得注意的是，面临伦理困境的是*用户*，而 AI 为其提供建议。这使得该训练数据与我们的蜜罐分布有本质不同——在蜜罐中，AI 自身处于伦理困境中，需要采取行动。我们称之为"困难建议"数据集。

引人注目的是，**我们仅用 300 万 token 的这个远更分布外的数据集就实现了与之前相同的评估改进**。除了 28 倍的效率提升外，该数据集更有可能泛化到更广泛的场景，因为它与我们所使用的评估集的相似度要低得多。实际上，该模型在（旧版）自动化对齐评估上表现更好。这与以下事实一致：Claude Sonnet 4.5 通过在合成蜜罐集上训练达到了接近零的勒索率，但在远离训练分布的情境中仍然参与了失对齐行为，[其频率远高于 Claude Opus 4.5](https://www-cdn.anthropic.com/bf10f64990cfda0ba858290be7b8cc6317685f47.pdf) 或之后的模型。

### 教导 Claude 宪法

我们假设"困难建议"数据集之所以有效，是因为它教授的是伦理推理，而不仅仅是正确答案。鉴于这一方法的成功，我们进一步探索，尝试更广泛地教导 Claude 宪法内容，并通过文档训练来训练其与宪法对齐。

我们预期这在三个方面会有效：

- 这在很大程度上是上述关于"困难建议"数据集为何有效的思路的延伸；
- 我们可以给模型一个更清晰、更详细的 Claude 角色画像，使得对其中一部分特质的微调能够激发出整个角色（类似于[审计游戏论文](https://www.anthropic.com/research/auditing-hidden-objectives)中观察到的效果）；
- 它更新了模型对 [AI 人格的感知](https://www.anthropic.com/research/persona-selection-model)，使其在平均水平上更加对齐。

我们发现，高质量的宪法文档结合描绘对齐 AI 的虚构故事，可以将自主行为失对齐降低超过三分之二，尽管它们与评估场景无关。

## 通过 RL 的泛化与持久性

尽管前一节讨论的宪法评估是令人鼓舞的信号，但我们最终需要确保对齐改进能在 RL 过程中持久存在。为了测试这一点，我们准备了一个 Haiku 级别模型的几个快照，使用不同的初始化数据集，然后在部分环境中运行 RL，这些环境以无害性为目标（我们推断这最有可能降低失对齐倾向）。

我们在运行过程中评估了这些模型在自主行为失对齐评估、宪法遵循评估和自动化对齐评估上的表现。在所有这些评估中，我们发现更对齐的快照在整个运行过程中保持了领先优势。这既适用于不存在失对齐行为的情况，也适用于存在主动可敬行为的情况。

## 多样化训练对泛化至关重要

我们的最后一项发现简单但重要：在*广泛*的安全相关环境上进行训练可以改善对齐泛化。以能力为焦点的 RL 环境混合分布正在快速变化和增长；假设标准的 RLHF 数据集会像过去一样继续良好泛化是不够的。

为了测试这一点，我们在 Claude Sonnet 4 的基础模型上训练了几个在多样性水平上各不相同的 RL 混合。基线环境在主题上是多样的，但大多在用户消息中包含有害请求或越狱尝试，没有系统提示。我们通过添加工具定义和多样化的系统提示来增强这些环境。用户提示保持不变。值得注意的是，这些环境中没有一个是实际需要智能体行动的（工具对于任务从来不是必要或有用的）或自主行动的（始终有人类用户与模型对话），因此它们与我们的评估并不相似。

当将这些增强环境与简单聊天环境混合时，我们看到模型在蜜罐评估上的改善速度有小幅但显著的提升。这表明了在安全训练中纳入多样化环境集的重要性。

### 讨论

自主行为失对齐是我们在模型中发现的第一个重大对齐失败之一，需要建立新的缓解流程——这些流程自那以后已成为我们的标准做法。

我们对这些进展感到鼓舞，但重大挑战仍然存在。完全对齐高度智能的 AI 模型仍然是一个未解决的问题。模型能力尚未达到像勒索倾向这样的对齐失败会构成灾难性风险的程度，而我们讨论的方法是否能继续扩展也尚待观察。此外，尽管最近的 Claude 模型在大多数对齐指标上表现良好，我们承认我们的审计方法论尚不足以排除 Claude 会选择采取灾难性自主行动的场景。

我们对进一步努力发现当前模型中的对齐失败持乐观态度，这样我们就可以在变革性 AI 模型被构建之前，理解并解决当前方法的局限性。我们也期待看到更深入的研究，试图理解我们所描述的方法为何如此有效——以及如何进一步改进这些训练。
