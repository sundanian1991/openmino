# 缓解浏览器使用中的提示注入风险

Claude Opus 4.5 在抵御*提示注入*（prompt injection）——隐藏在 AI 模型处理内容中的对抗性指令——方面设立了新标准。我们的新模型在核心性能和安全防护方面均较前代有显著提升。但提示注入远未被解决，尤其是随着模型执行越来越多现实世界操作。我们期望持续推进这一进展，实现 AI 模型（或"代理"）能够处理高价值任务而无重大提示注入风险的未来。

## 什么是提示注入？

AI 代理要真正有用，就需要能替你行事——浏览网站、完成任务、使用你的上下文和数据。但这也带来了风险：代理访问的每个网页都是潜在的攻击向量。

具体而言，当代理浏览互联网时，它会遇到无法完全信任的内容。在合法的搜索结果、文档和应用程序中，攻击者可能嵌入了恶意指令，用以劫持代理并改变其行为。这些提示注入攻击是浏览器 AI 代理面临的最严峻安全挑战之一。

下文将解释提示注入如何威胁浏览器代理，以及我们为提升 Claude 的鲁棒性所做的改进。

**为什么浏览器使用会产生独特的提示注入风险**

要理解提示注入的威胁，设想一个常规任务：你让 Claude 阅读近期的邮件并为所有会议请求起草回复。其中一封邮件——表面上是一封供应商咨询——在白色文字中嵌入了隐藏指令，对你不可见但会被代理处理。这些指令要求代理在起草你所需的回复之前，将包含"机密"一词的邮件转发到外部地址。一次成功的注入会在你等待回复时悄然外泄敏感通信。

虽然所有处理不可信内容的代理都面临提示注入风险，但浏览器使用从两方面放大了这一风险。第一，攻击面巨大：每个网页、嵌入式文档、广告和动态加载的脚本都是恶意指令的潜在载体。第二，浏览器代理可以执行大量不同操作——导航到 URL、填写表单、点击按钮、下载文件——攻击者一旦获得对代理行为的影响力便可加以利用。

**Claude 在浏览器使用鲁棒性上的进展**

自 Claude for Chrome 以研究预览版发布以来，我们在提示注入鲁棒性方面取得了显著进展。下图比较了我们今天发布的 Claude 浏览器扩展版本与最初发布配置，在面对内部自适应"Best-of-N"攻击者时的表现，该攻击者尝试并结合了多种已知有效的提示注入技术。

![内部Best-of-N攻击者的攻击成功率](images/prompt-injection-defenses_9c91927780.svg)

**我们内部 Best-of-N 攻击者的攻击成功率（ASR）。** 越低越好。自适应攻击者在每个环境中被给予 100 次尝试机会。ASR 按每个模型遭遇的攻击百分比计算。

Claude Opus 4.5 在浏览器使用中展现出比前代模型更强的提示注入鲁棒性。此外，自浏览器扩展的最初预览版以来，我们实现了新的防护措施，显著提升了所有 Claude 模型的安全性。

1% 的攻击成功率——虽已是重大改进——仍意味着不可忽视的风险。没有任何浏览器代理能完全免疫提示注入，我们分享这些发现是为了展示进展，而非声称问题已被解决。

我们的工作集中在以下领域：

**训练 Claude 抵御提示注入。** 我们使用强化学习将提示注入鲁棒性直接构建到 Claude 的能力中。在模型训练期间，我们将提示注入嵌入模拟网页内容中暴露给 Claude，当它正确识别并拒绝执行恶意指令时给予"奖励"——即使这些指令被设计得看似权威或紧急。

**改进分类器。** 我们扫描所有进入模型上下文窗口的不可信内容，并用[分类器](https://www.anthropic.com/news/constitutional-classifiers)标记潜在的提示注入。这些分类器能检测嵌入在各种形式中的对抗性命令——隐藏文本、被操纵的图像、欺骗性 UI 元素——并在识别到攻击时调整 Claude 的行为。自最初的研究预览版以来，我们改进了与 Claude for Chrome 配合使用的分类器，同时也改进了检测到攻击尝试后引导模型行为的干预措施。

**规模化专家人工红队测试。** 人类安全研究人员在发现创造性攻击向量方面始终优于自动化系统。我们的内部红队持续探测浏览器代理的漏洞。我们还参与了外部的[竞技场式挑战赛](https://app.grayswan.ai/arena/challenge/indirect-prompt-injection/rules)，对全行业的鲁棒性进行基准测试。

**前行之路**

网络是一个对抗性环境，构建能在其中安全运行的浏览器代理需要持续保持警惕。提示注入仍是一个活跃的研究领域，我们致力于在攻击技术演进的同时持续投资防御。

我们将继续透明地公布进展，既帮助客户做出明智的部署决策，也鼓励全行业对这一关键挑战加大投入。
