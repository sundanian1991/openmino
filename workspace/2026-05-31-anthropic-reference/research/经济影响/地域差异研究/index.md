# Anthropic 经济指数：不均衡的地域与企业 AI 采用

![Anthropic Economic Index hero](images/anthropic-economic-index-september-2025-report_94e5df8dd0.svg)

## 引言

AI 与以往技术的不同之处在于其前所未有的普及速度。仅在美国，40%的员工报告在工作中使用 AI，较两年前 2023 年的 20%翻了一番。1 这一快速普及反映了该技术在广泛应用领域中的实用性、在现有数字基础设施上的可部署性以及其易用性——只需打字或语音，无需专门培训。前沿 AI 的快速进步很可能在每个维度上进一步强化这种快速普及。

历史上，新技术需要数十年才能实现广泛普及。电力在城市电气化后花了 30 多年才到达农村家庭。第一台面向大众市场的个人电脑在 1981 年到达早期采用者，但直到 20 年后才进入大多数美国家庭。即使普及快速的互联网，也花了大约五年时间才达到 AI 仅用两年就实现的普及率。2

为什么会这样？简而言之，新技术——哪怕是变革性的——需要时间才能在经济中扩散，消费者采用需要时间才能变得地域上不那么集中，企业需要时间重塑业务运营以最好地释放新的技术能力。企业采用，首先是在狭窄的任务集上，然后在更通用的应用上，是重大技术传播并产生变革性经济效应的重要途径。3

**换句话说，早期技术采用的标志是集中——既集中在少数地理区域，也集中在企业的少数任务上。** 正如我们在本报告中记录的，AI 采用似乎正在遵循 21 世纪的类似模式，尽管时间线更短、强度更大，超过了 20 世纪技术的扩散速度。

为研究这种 AI 早期采用模式，我们在两个重要维度上扩展了 [Anthropic 经济指数](https://www.anthropic.com/economic-index)的分析范围：引入了对 Claude.ai 对话的地理分析，以及首次对企业和开发者 API 使用情况的考察。我们展示了 Claude 使用随时间如何演变，不同地区的采用模式如何不同，以及——首次——企业如何部署前沿 AI 来解决商业问题。

### Claude.ai 使用模式随时间的变化

在本报告第一章中，我们识别了 Claude.ai 在过去八个月中的显著变化，这些变化伴随着底层模型能力的提升、新产品的发布以及 Claude 消费者群体的扩大。

**我们发现：**

**教育和科学任务使用份额在上升：** 虽然 Claude 用于编程的份额仍以 36%占据主导地位，但教育任务从 9.3%飙升至 12.4%，科学任务从 6.3%增至 7.2%。**用户正在赋予 Claude 更多自主权：** "指令式"对话——用户将完整任务委托给 Claude——从 27%跃升至 39%。我们看到编程中程序创建增加（+4.5pp），调试减少（-2.9pp），表明用户可能能够在单次交互中完成更多目标。

### AI 采用的地理分布

我们首次发布了 Claude.ai 在 150 多个国家和美国所有州的使用数据。为研究扩散模式，我们引入了 Anthropic AI 使用指数（AUI），用于衡量一个经济体中 Claude.ai 使用相对其劳动年龄人口是过度代表还是不足代表。

**我们发现：**

**AUI 与各国收入强相关：** 与以往技术一样，我们看到 AI 使用在地理上集中。新加坡和加拿大是按人均计算使用率最高的国家之一，分别达到基于其人口预期的 4.6 倍和 2.9 倍。相比之下，新兴经济体，包括印度尼西亚（0.36 倍）、印度（0.27 倍）和尼日利亚（0.2 倍），使用 Claude 较少。**在美国，地方经济因素塑造使用模式：** 华盛顿特区人均使用领先（3.82 倍人口份额），但犹他州紧随其后（3.78 倍）。我们看到地区使用模式反映了地方经济的独特特征：例如，加州 IT 使用较高，佛罗里达金融服务使用较高，DC 文档编辑和职业协助使用较高。**领先国家使用更多元化：** 低采用国家倾向于更多编程使用，而高采用地区在教育、科学和商业方面展现多样化的应用。例如，编程任务在印度占所有使用的一半以上，而全球约三分之一。**高采用国家显示更少自动化、更多增强型使用：** 在控制各国任务组合后，低 AUI 国家更倾向于委派完整任务（自动化），而高采用地区更倾向于学习和人机迭代（增强）。

AI 早期采用的不均衡地理分布引发了关于经济趋同的重要问题。19 世纪末和 20 世纪初的变革性技术——广泛电气化、内燃机、室内管道——不仅开启了现代经济增长时代，还伴随着全球生活水平的巨大分化。4

如果生产力提升对高采用经济体更大，那么当前的使用模式表明 AI 的收益可能集中在已经富裕的地区——可能加剧全球经济不平等，逆转近几十年来观察到的增长趋同。5

### 企业系统性部署 AI

在最后一章中，我们首次提供了关于大量第一方（1P）API 流量的洞察，揭示了企业和开发者正在使用 Claude 完成的任务。重要的是，API 用户通过编程方式访问 Claude，而非通过网页用户界面（如 Claude.ai 的情况）。这展示了早期采用的企业如何部署前沿 AI 能力。

**我们发现：**

**1P API 使用虽与 Claude.ai 使用相似，但在专业化方面存在差异：** 1P API 使用和 Claude.ai 使用都高度集中于编程任务。然而，1P API 使用在编程和办公/管理任务上更高，而 Claude.ai 使用在教育和写作任务上更高。**1P API 使用以自动化为主导：** 77%的商业使用涉及自动化使用模式，而 Claude.ai 用户约为 50%。这反映了 API 使用的程序化特性。**能力似乎比成本在塑造商业部署中更重要：** API 数据中使用最多的任务往往成本高于较少使用的任务。总体而言，我们发现了弱价格敏感性的证据。模型能力和可行自动化给定任务的经济价值似乎在塑造商业使用模式中发挥更大作用。**上下文约束复杂使用：** 我们的分析表明，为模型策划正确的上下文对于在复杂领域中高影响力部署 AI 至关重要。这意味着对一些企业来说，昂贵的数据现代化和提取上下文信息的组织投资可能成为 AI 采用的瓶颈。

### 开源数据以促进独立研究

与此前报告一样，我们已将底层数据开源以支持对 AI 经济效应的独立研究。这一综合数据集包括 Claude.ai 和 1P API 流量的任务级使用模式（映射到 O*NET 分类体系及自底向上类别）、按任务划分的协作模式分类，以及我们方法的详细文档。目前，地理使用模式仅适用于 Claude.ai 流量。

我们希望这些数据能帮助其他人研究的关键问题包括：

- AI 使用与采用对工人和企业的本地劳动力市场影响是什么？
- 什么决定了各国和美国各州内部 AI 采用的差异？如何确保 AI 的收益不只流向已经富裕的经济体？
- 每任务成本在塑造企业部署模式中扮演什么角色（如果有的话）？
- 为什么企业能够自动化某些任务而不能自动化其他任务？这对哪些类型的工人将面临更好或更差的就业前景有何影响？

## 第一章：Claude.ai 使用随时间的变化

### 概述

理解 AI 采用如何随时间演变有助于预测其经济影响——从生产力提升到劳动力结构变化。借助从 2024 年 12 月和 2025 年 1 月（我们首份报告，"V1"）到 2025 年 2 月和 3 月（"V2"），再到 2025 年 8 月最新数据（"V3"）的数据，我们能够追踪过去八个月中 AI 使用的变化，在此过程中能力和产品功能有所提升、新类型的用户开始采用该技术、使用方式变得更加成熟。我们认为以下呈现的证据表明，新产品功能催生了新的工作形式，而非仅是加速了现有任务的采用。

### Claude.ai 经济任务使用如何变化

#### 教育和科学任务的相对重要性持续上升

虽然计算机和数学任务仍以 36%的份额主导总体使用，但我们在知识密集型领域看到了持续增长。教育教学和图书馆任务从 V1 的 9%上升到 V3 的 12%。生命、物理和社会科学任务从 6%增至 7%。与此同时，商业和金融运营任务的相对份额从 6%降至 3%，管理类从 5%降至 3%。

这一分化表明，AI 使用可能在知识综合和解释相关的任务中扩散得特别快，相比之下传统商业运营任务增长较慢——这可能是因为这些任务从 Claude 的推理能力中获益更多。

> **图 1.1：Claude.ai 使用随时间的变化。** 每个面板显示抽样 Claude.ai 对话中与各 SOC 主要组别任务相关的份额。我们看到科学和教育任务的使用显著增加。SOC 主要组别按我们首份报告中的使用量排序。

#### 新能力正在塑造使用模式

在更细粒度的层面上，我们记录了任务构成的变化，这些变化似乎与 V2 和 V3 之间发布的功能相关联。例如，搜索电子来源和数据库的任务大幅增长（0.03% → 0.49%），很可能反映了我们 3 月发布的[网页搜索功能](https://www.anthropic.com/news/web-search)。此外，我们还看到基于互联网的研究任务增加（0.003% → 0.27%），这与我们 4 月发布的[研究模式](https://www.anthropic.com/news/research)相吻合。1

我们还看到其他类型的变化。与开发教学材料相关的任务增加了 1.3 个百分点，从 0.2%增长到 1.5%——超过 6 倍的增长，可能反映了[教育者采用 Claude 的增长](https://www.anthropic.com/news/anthropic-education-report-how-educators-use-claude)。2 创建多媒体文档上升了 0.4 个百分点，从 0.16%增长近三倍到 0.55%，可能受到我们 [Artifacts 功能](https://www.anthropic.com/news/build-artifacts)持续使用的推动，该功能可在 Claude.ai 内构建传统和 AI 驱动的应用。

有趣的是，创建新代码的任务份额增长了一倍以上，增加了 4.5 个百分点（从 4.1%到 8.6%），而调试和纠错任务下降了 2.8 个百分点（从 16.1%到 13.3%）——净 7.4 个百分点的向创建而非修复代码的转变。这可能表明模型变得越来越可靠，用户花更少时间修复问题，更多时间在单次交互中创造。3

#### 指令式自动化正在加速

与以往报告一样，我们不仅追踪人们用 Claude 做什么，还追踪他们如何在 Claude.ai 上与 Claude 协作或将任务委托给 Claude。

在高层次上，我们区分了使用 Claude 的自动化（automation）和增强（augmentation）模式：

- **自动化**包含以任务完成为重点的交互模式：
  - **指令式**：用户给 Claude 一个任务，Claude 在最少来回交互中完成
  - **反馈循环**：用户自动化任务并根据需要向 Claude 提供反馈

- **增强**关注协作式交互模式：
  - **学习**：用户向 Claude 询问各种主题的信息或解释
  - **任务迭代**：用户与 Claude 协作迭代任务
  - **验证**：用户向 Claude 寻求对其工作的反馈

Claude.ai 对话样本中指令式对话的份额从 2024 年底 V1 的 27%跃升至 V3 的 39%。这一增长主要以任务迭代和学习交互的减少为代价，意味着展示自动化使用模式的对话份额净增显著——在短短八个月内的显著增长。这是首份自动化使用超过增强使用的报告。

> **图 1.2：Anthropic 经济指数各报告中的协作模式频率。** 左面板计算展示自动化或增强使用形式的对话份额。右面板按协作模式细分。随时间推移，Claude 倾向于以更自动化的方式被使用，主要由指令式使用的增加驱动。

一种解释是这是模型能力提升的结果。随着模型在预测用户需求和首次尝试产生高质量输出方面改进，用户可能需要更少的后续修正。指令式使用的跃升也可能表明用户对将完整任务委托给 AI 的信心增强，这是一种"做中学"的形式。4

指令式使用增长是归因于模型能力改进还是"做中学"，可能意味着截然不同的劳动力市场影响。如果更先进的模型仅是扩展了可自动化任务的集合，那么执行这类任务的工人面临被取代的风险就增加了。然而，如果指令式使用的上升反映的是"做中学"，那么最能适应新的 AI 驱动工作流的工人可能会看到更大的需求和更高的工资。换句话说，AI 可能使一些工人比其他工人获益更多：它可能为那些最能适应技术变革的人带来更高的工资，哪怕那些适应能力较低的人面临工作动荡。5 这将是未来研究的重要领域。

### 展望

V3 数据揭示了 AI 能力和采用正在持续进步。基于知识的任务，包括教育和科学应用，继续保持快速增长，新产品功能似乎正在促成不同类型的工作，而不仅是加速现有任务。

最引人注目的是，数据指向了向 AI 系统任务委托的增加——也许是由于用户对技术的信任和底层模型能力的改善的某种组合。这也可能是由于底层用户群体的变化。本报告的下一章首次按地域分解使用情况，使我们能够更清晰地解开时间变化与地理变化。我们将在未来的报告中继续密切追踪这些趋势。

1. "搜索电子来源，如数据库或存储库，或手工来源获取信息"从 0.03%增加到 0.49%。"进行基于互联网和图书馆的研究"从 0.003%增加到 0.27%。
2. 统计指标由任务中包含字符串"开发教学材料"的任务计算得出。
3. 任务汇总自频率变化幅度大于或等于 0.2 个百分点的任务集。编程创建任务包括："编写新程序或修改现有程序"（1.5% → 4.9%）、"设计、构建或维护网站"（1.2% → 2.0%）、"编写、分析、审查和重写程序"（1.2% → 0.5%）、"开发新软件应用"（0.06% → 0.6%）、"开发事务性 Web 应用"（0.1% → 0.3%）以及"开发特定应用软件"（0.05% → 0.3%）。调试/纠错任务包括："修改现有软件以纠正错误"（两个变体：2.5% → 3.8%和 4.8% → 2.7%）、"通过做出适当更改来纠正错误"（3.0% → 2.1%）、"执行初始调试程序"（2.0% → 0.9%）、"诊断、排除和解决硬件/软件问题"（1.6% → 2.5%）、"审查和分析计算机输出来定位代码问题"（1.3% → 0.9%）以及"确定网页或服务器问题的来源"（0.9% → 0.4%）。
4. 我们注意到 V3 使用 Claude Sonnet 4 进行分类，而 V2 使用 Sonnet 3.7，这使直接比较变得复杂。为解决此问题，我们使用 Sonnet 3.7 重新运行了 V3 数据，仍然发现指令式交互显著增长（虽然自动化水平较低，为 45%对比 Sonnet 4 的 49%）。我们还验证了这一趋势不是由任务组合变化驱动的——指令式交互的转变在各职业类别中广泛出现，表明它反映了人们与 Claude 交互方式的真实变化，而非构成效应。
5. Nelson 和 Phelps, 1966 *Investment in Humans, Technological Diffusion, and Economic Growth* 是关于教育在帮助工人适应变化方面价值的经典参考文献。另见 Goldin 和 Katz, 2008 *The Race between Education and Technology*。我们感谢 Anton Korinek 提出的观察，即 AI 本身可能加速 AI 的传播和在经济上的影响，到它能够扮演熟练工人过去在摸索如何在新环境中有效运用新技术时所扮演的角色的程度。


## 第二章：Claude 在美国和全球的使用

### 概述

AI 首先在哪里被采用——以及它如何被使用——将塑造全球的经济结果。通过分析 Claude 在 150 多个国家和美国所有州的使用模式，我们发现了三个关键动态：早期采用者在哪里、他们用 AI 做什么、以及随着采用成熟使用如何演变。这些地理模式为 AI 的经济扩散提供了现实证据，有助于追踪不同地区在 AI 采用上是趋同还是分化，并揭示了地方经济特征如何形塑技术部署。

我们的数据依赖于对 100 万条 Claude.ai 对话的[隐私保护](https://www.anthropic.com/research/clio)分析1，证实了一些我们的预期，同时也挑战了其他预期。美国以 21.6%的主导总使用量位居首位，鉴于其规模和较高收入，这并不令人意外。但即使在调整劳动年龄人口规模后，高收入国家也倾向于有更高的使用量。例如，新加坡的使用率是其劳动年龄人口所提示的 4.5 倍，而全球大片地区显示极少的使用量。有趣的是，在美国内部，DC 和犹他州在人均使用上超过了加州。2

我们还观察到随着人均采用深化，AI 用例发生变化。人均 AI 采用较低的国家主要集中于编程任务——在印度超过一半的使用，而全球约为三分之一。随着采用成熟，使用变得更加多元，教育、科学和商业运营的占比上升。

更引人注目的是：成熟市场倾向于更协作地使用 AI，而新兴市场更可能将完整任务委托给它——可能反映了处于不同结构转型阶段的经济体部署 AI 的方式差异。我们的数据提供了一个观察这些地理模式的窗口，今后将使我们能够追踪这些采用差距是缩小、扩大还是在结构上发生变化。

### Claude 在全球的扩散

**Claude 总使用量在美国最高**

Claude 采用总体上高度地理集中。在全球总使用量方面，美国占比最高（21.6%），其次高使用量的国家份额显著较低（印度 7.2%，巴西 3.7%，见图 2.1）。然而，这种集中受到每个国家人口规模的影响3——大国可能纯粹因为人口规模而拥有更大的使用份额。

> **图 2.1：全球 Claude.ai 使用份额领先的国家。** 数据包含 Claude.ai Free 和 Pro 对话。

**Claude 人均使用集中在技术先进国家**

为考虑人口规模差异，我们分析了经劳动年龄人口调整后的使用量，引入了一项新指标 Anthropic AI 使用指数（AUI）：对于每个地理区域，我们计算其 Claude 使用份额与劳动年龄人口（15-64 岁）份额的比值。

> Anthropic AI 使用指数 = Claude 使用份额 / 劳动年龄人口份额

该指数揭示了各国使用 Claude 是否多于或少于基于其劳动年龄人口预期的水平。AUI > 1 的地区在调整人口后使用高于预期，AUI < 1 的地区使用低于预期。

**聚焦领先和新兴国家的人均使用**

这种在人口规模有限的发达经济体中的集中反映了它们作为技术先驱的既定模式。例如，以色列和新加坡都在[全球创新指数](https://www.wipo.int/web-publications/global-innovation-index-2024/en/gii-2024-at-a-glance.html)中排名靠前——该指数衡量全球各经济体的创新程度——表明对信息技术的总体投资为经济体快速采用前沿 AI 奠定了良好基础。总体而言，这些经济体能够利用其受过良好教育的劳动力、健全的数字基础设施和创新友好政策为 AI 创造有利条件。

值得注意的是主要发达经济体在 Claude 使用中的位置。美国（3.62）在人均采用方面位列领先国家，加拿大（2.91）和英国（2.67）相对于其人口有较高但更为温和的采用率。其他主要经济体显示较低的采用率，包括法国（1.94）、日本（1.86）和德国（1.84）。

Claude 使用的差异可能反映了多种因素的共同作用，其中一些与收入相关：

- **数字基础设施：** 高使用国家通常拥有访问 AI 助手所需的健全互联网连接和云计算能力。
- **经济结构：** 如本报告及此前报告所述，Claude 能力很适合各种典型的脑力劳动者任务。先进经济体倾向于在此类岗位中拥有更大的劳动力份额，相比之下，较低收入经济体在制造业方面的就业份额更大。
- **监管环境：** 各国政府在积极鼓励各行业使用 AI 方面以及在多大程度上监管该技术方面存在差异。
- **认知和可及性：** 与硅谷和 AI 研究社区联系更紧密的国家可能对 Claude 有更高的认知度和访问能力。
- **信任和舒适度：** 公众对 AI 的信任舆论在各国间[差异显著](https://hai.stanford.edu/ai-index/2025-ai-index-report/public-opinion)。

### 各国的任务使用模式

我们观察到不同国家使用 Claude 的方式存在显著差异。与以往报告一样，我们使用两种不同的方法来分析这些趋势。首先，我们根据 O*NET（美国职业分类体系，将具体任务映射到职业和职业组）将对话分类为任务。

其次，我们使用 Claude 构建 Claude.ai 用户请求的自底向上分类法，这为不适合现有分类法的使用模式提供了洞察。例如，请求聚类"帮助撰写和改进求职申请的求职信"（最低层级）汇入更高层级的聚类"帮助求职申请、简历和职业文件"（中间层级），再汇入聚类"帮助求职申请、简历和职业发展"（最高层级）。这两种互补方法使我们既能报告与标准劳动统计一致的结果，又能灵活捕捉标准分类法遗漏的任务。

**人机协作中的地理模式**

此前各节考察了人们用 Claude 做什么任务，而同样揭示性的一种模式是在他们如何与它交互中浮现。这里，我们使用第一章中定义的相同的增强和自动化协作模式。

各国有不同的任务组合，意味着它们专注于不同的经济任务，这可能部分解释了自动化模式的差异。在本节中，我们研究即使在控制任务组合差异的情况下，低人均采用经济体和高人均采用经济体之间自动化使用是否存在系统性差异。9

我们发现，即使在控制一个国家的任务组合的情况下，来自不同国家的用户对自主委托与协作交互表现出显著不同的偏好。随着人均 Claude 使用的增加，各国从自动化导向转向增强导向的使用。这有些违反直觉，因为我们控制了不同国家之间更多样化的任务构成。我们推测文化和经济因素可能影响自动化份额，或者每个国家的早期采用者倾向于以更自动化的方式使用 AI——但这方面需要更多研究。

### 结论

我们对跨地理区域 Claude 使用模式的分析揭示了几个关键洞见。其中最引人注目的是 Claude 使用的地理集中。美国整体和加州在 Claude 总使用量方面的领先，以及 Claude 使用与人均收入的强相关性，表明与过去技术相似的模式，其中初始地理集中和专业化使用是关键特征。将当前模式与以往技术的扩散模式进行类比，可能有助于我们更好地理解 AI 的扩散和影响。

令人惊讶的是，地理不仅塑造了 AI 工具被用于什么，还塑造了它们如何被使用。人均使用相对较低的经济体的用户相对更倾向于将任务委托给 Claude（自动化），而人均使用较高的经济体用户则在一定程度上更倾向于偏好更协作或基于学习的与 Claude 的交互（增强），即使在控制任务组合的情况下也是如此。与任务使用中的地方专业化类似，AI 协作模式的地方专业化表明 AI 的影响在不同地区可能非常不同。

AI 采用的地理模式——在哪里使用、用于什么任务、如何交互——表明，为实现 AI 造福全球人们的潜力，政策制定者需要关注 AI 使用和采用的本地集中，并应对加深数字鸿沟的风险。

7. 在进一步调查犹他州的活动时，我们发现其使用量的一个显著部分似乎可能与协同滥用有关。这也反映在远超平均水平的"指令式"自动化得分上。然而，我们进行了稳健性检查并相信此活动不驱动结果。
8. 请求被过滤为在全球层面至少占 1%且在当地层面至少占 1%。
9. 为在考虑任务构成差异的同时隔离自动化偏好与 Claude 使用之间的关系，我们执行以下操作：首先，通过加权平均计算每个国家的预期自动化百分比。对于每个 O*NET 任务（例如，编程、写作或分析），我们将该任务在该国使用中的份额乘以该任务类型的全球自动化率（该任务通过指令式/反馈循环模式由 Claude 完成的全球百分比）。将这些加总得到给定国家在给定其特定任务组合下的自动化百分比预期值。然后，我们将实际自动化%和 AUI 分别对该预期自动化%进行回归。这些回归的残差代表每个变量中无法由任务构成解释的变异。通过考察这些残差之间的关系（称为偏回归分析），我们可以确定 AI 使用高于其任务组合所预测水平的国家是否也倾向于具有高于预测水平的自动化。


## 第三章：API 企业部署 Claude

### 概述

前沿 AI 能力是否使我们更具生产力、重塑劳动力市场并加速增长，将取决于企业选择何时以及如何部署 AI。即使企业认识到 AI 的潜力，有利可图地采用它可能需要昂贵的生产流程重组、培训新工人以及其他促进有效部署的沉没成本投资。1

为了理解 AI 的商业采用模式，我们转向一个新的数据来源：Anthropic 的第一方（1P）API 客户——再次依赖[隐私保护方法](https://www.anthropic.com/research/clio)。2 我们的 API 允许客户将 Claude 直接集成到他们自己的产品和应用中，并按使用的 token 收费，而非固定订阅费。这代表了与 [Claude.ai](http://claude.ai) 根本不同的产品体验，后者的使用我们已在前两章中重点分析。

制度惯性，加上采用的固定成本，表明 AI 企业使用的早期案例很可能集中在部署容易、能力稳健且采用带来的经济效益高的专业任务上。

事实上，我们在本章呈现的数据中看到了符合这一逻辑的证据。我们的分析揭示了几个模式：

- **企业使用 Claude 的方式与个人用户相似但更专业：** 企业将使用集中在 AI 部署特别适合程序化访问的任务上，如编程或行政任务。与 Claude.ai 用户相比，企业较少使用 Claude 进行教育或创意任务，总体上以更自动化的方式使用。
- **API 客户倾向于更高成本的任务：** 尽管任务在成本上差异巨大，最昂贵的任务往往有更高的使用量，表明模型能力、部署便利性和自动化的经济价值远比交互本身的成本在决定采用模式上更重要。
- **获得适当的上下文信息是复杂部署的必要条件：** 我们发现了 AI 对企业有用性的一项重要潜在瓶颈的证据。使用 Claude 进行复杂任务的 API 客户倾向于向 Claude 提供冗长的输入。这可能代表了对一些重要任务的更广泛企业部署的障碍，这些任务依赖于尚未集中化或数字化的分散上下文。纠正这一瓶颈可能需要企业重组其组织、投资新的数据基础设施并集中信息以实现有效的模型部署。

### 背景设定：公开数据中的 AI 采用模式

在深入分析 API 数据之前，值得建立一下商业 AI 采用的更广泛图景。根据人口普查局的商业趋势与展望调查，美国企业中的 AI 采用率在过去两年中增长了一倍以上，从 2023 年秋季的 3.7%上升到 2025 年 8 月初的 9.7%（图 3.1）。3 尽管增长迅速，美国绝大多数企业仍报告未在生产流程中使用 AI。

但这些总体数字掩盖了各行业之间的巨大差异。例如，在 2025 年 8 月初，信息行业每四家企业中就有一家报告使用 AI，这大约是住宿和餐饮服务行业的十倍。4

公开数据所呈现的图景是清晰的：企业使用 AI 正在快速增长，但我们仍处于 AI 采用的早期阶段。使用在经济中分布不均，最能快速采用并从这项技术中获益的行业正在这样做。

如下文所见，我们的 1P API 数据得出了一个互补的结论：Claude 的早期企业使用同样在经济中分布不均，主要部署在信息行业职业的典型任务上。

> **图 3.1：美国企业的 AI 采用率，商业趋势与展望调查（人口普查局）。** 注意：AI 采用率计算为对"在过去的两个星期里，该企业是否在生产商品或服务时使用了人工智能（AI）？（AI 示例：机器学习、自然语言处理、虚拟代理、语音识别等）"回答"是"的企业份额。

### Anthropic API 客户的专业化使用

为分析 API 流量，我们应用了与前几章相同的隐私保护分类方法——将匿名化的 API 记录按 O*NET 任务分类并归入自底向上分类法。浮现的模式显示企业使用集中在特别适合自动化的专业任务上。

总体而言，软件开发主导全局。在前 15 个使用聚类中——代表约一半的 API 流量——大多数与编程和开发任务相关。调试 Web 应用和解决技术问题各占使用量的约 6%，而构建专业商业软件是另一个重要部分。值得注意的是，约 5%的 API 流量专门集中在开发和评估 AI 系统本身上（图 3.2）。

但并非所有 API 使用都是用于编程。API 客户也部署 Claude 来创建营销材料（4.7%）和处理商业与招聘数据（1.9%）。这两个类别揭示了 AI 不仅被部署用于商品和服务的直接生产，还用于人才招聘和对外传播。

> **图 3.2：抽样 1P API 记录中 Claude 使用的自底向上分类法。** 我们使用隐私保护方法将 1P API 记录分类为反映底层使用的自底向上分类法。此图报告了此分类法最宽泛层级的主要用例。

O*NET 分类使这些模式更加清晰。略少于一半的所有 API 流量映射到计算机和数学任务——比 Claude.ai 使用高出 8 个百分点以上。办公和行政管理任务排在第二位，约占记录的 10%，反映了它们对自动化的适应性。

另一方面，几个在 Claude.ai 上突出的互动密集型任务在 API 使用中的份额要小得多：教育和图书馆任务从 12.3%降至 3.6%，艺术和娱乐从 8.2%降至 5.2%。

然而，在许多情况下，职业类别在 Claude.ai 和 API 数据之间相当接近，表明底层模型能力而非具体产品形态在许多情况下驱动采用。

> **图 3.3：按总体使用划分的主要职业类别：Claude.ai vs 1P API。** 确定任务的使用份额后，我们计算了分配给 O*NET 分类法中顶级职业的 Claude.ai 和 1P API 客户流量份额。例如，此图显示我们样本中 44%的 API 流量匹配到计算机和数学职业的典型任务。

### 职业细分 vs. 任务专业化

尽管服务不同的用户群体、具有不同的界面，API 和 Claude.ai 使用在任务分布上遵循非常相似的幂律分布。在 Claude.ai 对话中，底部的 80%任务类别仅占总使用的 12.7%；对 API 客户而言，这一数据更为集中，为 10.5%（图 3.4）。这些极端的集中度（基尼系数5 分别为 0.84 和 0.86）揭示了 AI 与任务匹配的巨大差异——匹配最好的任务比匹配差的任务多出数个数量级的使用量。

不同平台之间的相似性尤为引人注目，因为它们的用户群体和用例不同。两者汇聚到可比的集中度水平，表明 AI 能力和相关经济任务之间存在一个共同的匹配过程。

像代码生成这样的任务主导使用，因为它们达到了一个"甜点"——模型能力突出、部署障碍极小、员工可以快速采用新技术。很少使用的任务的长尾分布可能反映了几个因素。6 例如，某些任务根本就是较少见的——调试软件远比谈判马戏团合同频繁得多。极端的集中度也暗示了 O 形环（O-Ring）力量的潜在作用：如果一个任务需要 Claude 无法处理的推理水平、企业无法访问的内部数据或不存在的监管审批，任何单一障碍都可能阻止采用。7

> **图 3.4：使用在少量任务中的集中度可视化：Claude.ai vs 1P API。** 左侧面板计算 Claude.ai 和 1P API 样本中 O*NET 任务的洛伦兹曲线。曲线上的高亮点指示底部 80%任务占总使用的比例。右侧面板针对在样本中占总使用至少 0.1%的任务绘制任务排名与任务使用份额的关系。齐普夫定律（最佳拟合线系数等于 -1）在各种经济环境中规律性地出现。

### API 记录中的自动化 vs. 增强

API 和 Claude.ai 使用之间最清晰的区别在于人类和 AI 如何分工。当企业将 Claude 嵌入到他们的应用中时，他们大多是委托单个任务，而非与模型进行迭代式协作。

在我们的数据中，77%的 API 记录显示自动化模式（尤其是完整任务委托），而增强（例如，协作完善和学习）仅占 12%。基于 Claude.ai 对话样本，自动化和增强之间的分割几乎是对等的。跨经济任务来看，通过 API 的 Claude 自动化程度更加鲜明：97%的任务在 API 使用中显示出以自动化为主的模式，而在 Claude.ai 上仅为 47%（图 3.6）。

这在直觉上是合理的。程序化的 API 访问自然适合自动化：企业提供上下文，Claude 执行任务，输出直接流向最终用户或下游系统。

这一模式回响了有经济意义的重大技术如何变得具有变革性：嵌入系统中，让工人无需专业技能即可获取生产力提升。虽然增强和自动化方法都能提升人类能力，但系统级自动化可能带来整个经济中更大的生产力提升，同时也带来劳动力市场更显著的变化：完全自动化一些任务，改变哪些任务对各种工作重要，甚至产生全新的工作形式。

> **图 3.5：O*NET 任务中的自动化 vs 增强协作模式：Claude.ai vs 1P API。** 此图报告了每个 O*NET 任务中展示自动化或增强使用模式的 Claude.ai 对话和 1P API 记录的份额。自动化和增强模式在第一章中定义。当出于隐私保护原因我们未观察到特定协作模式的使用份额时，我们在该图中给该类别赋予 0%的值。自动化主导定义为任务具有更大的自动化使用观察份额。增强主导同理。

### Claude 做得越多，就需要知道越多

为什么 API 客户在某些任务上使用 Claude 比在另一些任务上更多？除了基本的模型能力外，一个潜在的重要解释是，对某些任务而言，为成功部署提供 Claude 所需的信息更容易。

例如，如果目标是让 Claude 重构复杂软件开发项目中的一个模块，Claude 可能需要阅读——或至少探索——整个代码库以理解需要做哪些更改以及在哪里更改。对于具有集中代码仓库的软件开发，获取这些信息在原则上是简单的。

对于其他任务，适当的上下文可能无法随时获得，或者可能难以访问。例如，让 Claude 为关键客户制定销售策略可能需要 Claude 不仅能够访问客户关系管理系统中的信息，还需要访问位于客户经理、市场营销人员和外部联系人头脑中的隐性知识。在其他条件相同的情况下，缺乏此类上下文信息将使 Claude 能力下降。

我们通过考察各任务之间平均 API 输入长度（即提供给 Claude 的上下文）和 Claude 平均输出长度（即模型响应生成的内容）之间的关系来探讨这一问题。8

对于样本中的每个 O*NET 任务，我们计算相关 API 记录的平均输入和输出长度。然后我们将这些值除以样本中所有任务的平均长度。这为每个任务生成了一个输入 token 指数和一个输出 token 指数。例如，指数值 1.5 意味着与该任务相关的 API 记录比跨任务的平均值长 50%。

Claude 的 API 输出长度在不同任务之间存在相当大的差异。例如，输出长度第 90 百分位的任务比第 10 百分位的任务长 4 倍以上。表 3.1 提供了示例 O*NET 任务，以及 Claude Sonnet 4 对分布中该部分任务组的总结。9 图 3.7 显示输出长度在职业类别之间也存在系统性差异。

> **表 3.1：具有较短和较长输出长度的示例 O*NET 任务及 Claude 的总结。** 对于每个匹配到 1P API 流量的 O*NET 任务，我们计算了一个输出 token 指数：将该任务相关记录的平均输出长度除以样本中所有任务的（不加权）平均值。Claude 被提示识别输出 token 指数分布中第 10、50 和 90 百分位的任务，指导最少："列应为'示例任务'、'指数值'、'总结'，你提供总结"。Claude 将输出长度与任务复杂性相关联。

> **图 3.6：主要职业类别中跨 O*NET 任务的平均输出 token 指数。** 对于每个匹配到 1P API 流量的 O*NET 任务，我们计算输出 token 指数：将该任务相关记录的平均输出长度除以样本中所有任务的（不加权）平均值。然后我们对给定 O*NET 分类法顶级职业类别中的任务取平均值，针对顶级使用职业组。'所有其他'将剩余职业组合并为一个类别。

从 Claude 对任务的评估中脱颖而出的是，较长输出的任务往往代表越来越复杂的用途。当然，输出长度并不涵盖任务复杂性的所有维度，但它似乎是一个合理的、易于衡量的代理变量。

由于 API 客户在输入 token 和输出 token 上都是按边际定价的，他们在使用 Claude 时有动力优化模型提示以最小化输入和输出 token。反过来，Claude 产生的输入长度和输出之间的任何系统性关系部分反映了在部署 Claude 进行复杂任务时的底层上下文约束。换种方式说，API 客户有动力仅向 Claude 提供刚好足够的上下文来达成他们的目标，而不会更多。因此我们了解到不同输出长度任务的上下文需求。

跨任务来看，我们看到 API 客户向 Claude 提供多少上下文与 Claude 实际产生多少之间存在非常稳定的关系。跨经济任务，输入长度每增加 1%，输出长度增加小于比例关系的 0.38%（图 3.7）。这个 0.38 的弹性表明在将这些有经济价值的任务的更长上下文输入转化为更长输出方面存在强烈的边际收益递减。10

> **图 3.7：O*NET 任务中输出 token 指数和输入 token 指数的散点图。** 对于每个匹配到 1P API 流量的 O*NET 任务，我们计算输出 token 指数：将该任务相关记录的平均输出长度除以样本中所有任务的（不加权）平均值。输入 token 指数以类似方式构建。0.38 的弹性意味着输入 token 指数每增加 1%，输出 token 指数增加 0.38%。

结论是，为复杂任务部署 AI 可能更多受到信息获取的约束，而非底层模型能力。不能有效收集和组织上下文数据的公司可能在复杂 AI 部署中遭遇瓶颈，为更广泛的企业采用创造潜在瓶颈——特别是对于隐性、分散知识对业务运营至关重要的职业和行业。

### 每任务成本与跨任务替代模式

API 客户按 token 付费，为使用 Claude 部署不同任务创造了成本差异。更复杂的任务由于输入和输出 token 数量更高，往往成本更高。这种差异帮助我们探讨成本是否是决定企业选择用 Claude 自动化哪些任务的主要因素。

数据表明它不是，至少在相对意义上如此。11 例如，计算机和数学职业的典型任务成本比销售相关任务高出 50%以上，但仍在使用中占主导地位。

总体而言，我们发现成本与使用之间存在正相关：成本更高的任务往往有更高的使用率（图 3.9）。12

成本与使用之间的正相关表明，成本在塑造企业 AI 部署模式中扮演无关紧要的角色。相反，企业可能优先在模型能力强大且 Claude 驱动的自动化产生足够经济价值（超过 API 成本）的领域使用。

> **图 3.8：各职业类别的每任务 API 成本与使用份额。** 对于每个匹配到 1P API 流量的 O*NET 任务，我们计算 API 成本指数：将该任务相关记录的平均 API 成本除以样本中所有任务的（不加权）平均值。此图绘制了给定职业类别中任务的 API 成本指数平均值与使用份额的关系。估计弹性为 3 意味着任务的平均成本每增加 1%，在我们的样本中的流行度增加 3%。

虽然这一正相关在总体上成立，我们接下来问：在其他方面相似但成本更高的任务中，对 Claude 能力的需求是否更低？作为一项重要的前提提示，这应被视为初步探索——我们发现了以下情况。

在控制任务特征后，我们发现每 1%的成本增加与 API 记录样本中使用频率下降 0.29%相关（图 3.10）。13 虽然与标准经济理论一致，即更高价格导致更低需求，但使用对成本下降的隐含增加是有限的。根据此估计，某个特定任务成本降低 10%只会使使用量增加约 3%。

除特定任务使用 Claude 的成本之外，其他因素对使用模式似乎更重要。

> **图 3.9：控制任务特征后每任务 API 成本与使用份额的散点图。** 对于每个匹配到 1P API 流量的 O*NET 任务，我们计算 API 成本指数：将该任务相关记录的平均 API 成本除以样本中所有任务的（不加权）平均值。然后我们将样本限制为同时出现在 1P API 和 Claude.ai 样本中的任务。此偏散点图控制了以下任务级特征：职业类别的固定效应、来自 Claude.ai 的协作模式份额，以及 Claude.ai 样本中给定协作模式是否因隐私保护原因被截断的指标。估计弹性为 -0.29 意味着，在控制任务特征后，给定任务的 API 成本指数每增加 1%，在我们的样本中的流行度下降 0.29%。

### 结论

我们的 API 数据捕捉了企业 AI 采用在初期的图景：高度集中、以自动化为主、且令人惊讶地对价格不敏感（至少在我们的 API 客户使用 Claude 的任务中是如此）。

77%的自动化率表明企业使用 Claude 来委托任务，而非作为协作工具。这种系统性部署很可能是 AI 在经济中带来更广泛生产力提升的重要渠道。鉴于商业部署中清晰的自动化模式，这也可能在劳动力市场带来颠覆，潜在地取代那些角色最可能面临自动化的工人。

但对劳动力市场的影响并非完全明确。如上所述，复杂任务需要不成比例更多的上下文。这些信息可能分散在组织各处。在这种条件下，拥有关于业务运营隐性知识的工人可能作为复杂 AI 驱动的自动化的互补品而获益。14 理解 AI 采用在劳动力市场中不均衡的影响是未来研究的重要领域。

希望有效采用 AI 的企业可能需要重塑他们组织和维护前沿系统所依赖的信息的方式。今天狭窄的、以自动化为主的采用是会演变为更广泛的部署，将可能决定 AI 未来的经济影响。

## 总结语

这第三版 Anthropic 经济指数报告捕捉了 AI 采用在关键时刻的图景。Claude 和其他前沿 AI 系统的现有能力已经准备好转变经济活动，鉴于该技术的广泛适用性。快速发展的 AI 能力只会强化"巨大变革即将到来"这一结论。然而，AI 早期采用却惊人地不均衡。使用目前集中在少量任务中，具有很强的与收入高度相关的地理差异——特别是在各国之间。这种集中反映了 AI 能力、部署便利性和经济价值相一致的领域：编程和数据分析有高使用量，而需要分散上下文或复杂监管导航的任务则较为落后。

AI 的早期商业采用与消费者使用既有相似之处（编程是两者最常见的用途），也有几个重要方面的不同。特别是，通过 API 程序化访问 Claude，企业倾向于以更高的自动化程度使用 Claude。这种系统性企业部署反映了 AI 将如何重塑经济活动：提高总体生产力，但对那些现有职责已被自动化取代的工人产生不确定的影响。

这些模式带来了分化的风险。如果 AI 的生产力提升集中在已经繁荣的地区和适合自动化的部门，现有不平等可能扩大而非缩小。如果 AI 自动化提高了拥有隐性组织知识的工人的生产力——正如我们的部分证据所示——那么更有经验的工人可能看到需求上升和更高工资，哪怕入门级工人面临更差的劳动力市场前景。1

基于我们之前的发布，这版指数报告标志着范围和透明度的显著扩展。我们现在将综合的 API 使用数据与我们现有的 Claude.ai 消费者数据（现在包括州和国家层面的地理细分）一起开源，都与详细的任务级分类交叉呈现。

通过公开这些数据，我们希望使其他人能够研究我们未曾考虑过的问题，检验关于 AI 经济影响的假设，并基于经验证据制定政策应对方案。

**最终，变革性 AI 的经济效应将由技术能力和社会的政策选择共同塑造，程度相当。**

历史表明，技术采用的模式并非固定不变：它们随着技术成熟、互补创新的出现以及社会对其部署做出的审慎选择而变化。我们今天观察到的高度集中使用的模式可能仍会演变向更广泛的分布——一种能捕捉更多 AI 提升生产力潜力、加速落后部门的创新，并促成新形式经济价值创造的分布。我们仍处于这场 AI 驱动的经济转型的早期阶段。政策制定者、商业领袖和公众现在采取的行动将塑造未来岁月。随着 AI 能力不断进步，我们将继续追踪这些模式，并为我们这个时代最重要的经济转型提供经验性依据。

Six Facts about the Recent Employment Effects of Artificial Intelligence* 记录了自 2022 年末以来高 AI 暴露度的入门级工人就业前景相对较差的明确证据。暂且不谈因果问题，直接的解释是这是由于 AI 替代了原本由早期职业工人完成的工作。另一种解释由 Gans 2025, *If AI and workers were strong complements, what would we see?* 提出：经验丰富的工人就业增长相对更快反映了 AI 使此类工人更具生产力从而需求高涨。AI 是对工作形成互补还是替代，可能是我们希望数据能帮助回答的最重要的问题。
