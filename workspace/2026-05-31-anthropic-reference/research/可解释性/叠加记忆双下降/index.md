在[近期一篇论文](https://transformer-circuits.pub/2022/toy_model/index.html)中，我们发现简单神经网络在玩具任务上训练时，经常出现一种称为**叠加**的现象——即它们表示的特征数量超过了神经元的数量。彼时我们的研究仅限于无限数据、欠拟合的范畴。但有理由相信，如果我们想在机制可解释性上取得成功，理解过拟合可能很重要，而叠加可能是这个问题的核心。

为什么机制可解释性需要关心过拟合？尽管过拟合是机器学习中的核心问题，但我们对深度学习模型过拟合或记忆样本时到底发生了什么，几乎没有机制层面的理解。此外，先前的工作暗示，过拟合与学习可解释特征之间可能存在重要关联。

那么，理解过拟合很重要，但它为什么与叠加相关？考虑一个逐字记忆文本的语言模型——它是如何做到的？一个朴素的想法是，它可能使用神经元创建一个查找表，将序列映射到任意延续。对于它想要记忆的每一段 token 序列，它可以专门分配一个神经元来检测该序列，然后在神经元激活时执行任意行为。这种方法的问题是效率极低——但它似乎是叠加的完美候选，因为每种情况都是互斥的，不会相互干扰。

在这篇文章中，我们对在有限数据集上训练相同的玩具模型进行了非常初步的研究。尽管极其简单，这个玩具模型却成为过拟合研究的出人意料地丰富的案例。具体来说，我们发现了以下几点：

- 过拟合对应于在叠加中存储**数据点**，而非特征。
- 根据数据集大小，我们的模型分为两种不同的机制：过拟合机制（以叠加方式存储数据点）和泛化机制（以叠加方式存储特征）。
- 当模型在这两种机制之间转换时，我们观察到了**双下降**现象。

### 实验设置

我们假说真实的神经网络在一个稀疏、高维的"特征"空间中执行操作，但这些特征因被存储在叠加中而难以直接观察。基于此，我们试图使用合成的输入向量 x 来模拟这个特征空间，x 是稀疏、高维且非负的（类似于我们[之前的论文](https://transformer-circuits.pub/2022/toy_model/index.html#demonstrating-setup-x)）。具体而言，x ∈ ℝⁿ 是一个 n=10,000 维的向量。我们令每个特征 x_i = 0 的概率为 S=0.999，否则在 [0, 1] 上均匀分布。然而与之前的工作不同，我们随后对 x 进行缩放，使得 ||x||² =1，这将使训练样本的记忆更容易实现。我们还考虑了有限大小 T 的训练集，而我们之前的工作只考虑 T=∞。我们用 X ∈ ℝⁿˣᵀ 表示训练数据矩阵，其中每一列 X_i 是一个训练数据点。

我们采用之前论文中的["ReLU Output"玩具模型](https://transformer-circuits.pub/2022/toy_model/index.html#demonstrating-setup-model)，定义为：

**ReLU输出模型**

```
h = Wx
x' = ReLU(Wᵀh + b) = ReLU(WᵀWx + b)
```

其中 W 采用 Xavier 初始化。模型通过最小化均方重构误差进行训练：

```
L = (1/T) ∑_x ∑_i I_i (x_i - x'_i)²
```

在本文中，我们限制为均匀重要性 I_i = 1, ∀i。

我们使用 50,000 次全批量更新（而非小批量），采用 AdamW 优化器。学习率调度包含 2,500 步线性预热至 1e-3，然后余弦衰减至零。训练更新次数、全批量优化的使用以及退火学习率对我们的结果都很重要。我们专注于极低维的隐藏空间，这似乎使优化问题更加困难。此外，极其稀疏的特征使得梯度噪声大得多。因此，使用全批量进行大量步骤的优化对产生这些结果至关重要。在一些实验中，我们会进一步训练多个使用不同随机种子进行参数初始化的模型，并选择训练损失最低的模型以避免局部最小值。但我们定性发现，当 m>3 时这些敏感性大大降低。

除非另有说明，我们使用的权重衰减为 1e-2。与先前工作一致，我们发现双下降在权重衰减较低时最强，反之亦然。

### "数据点特征"与"泛化特征"

在之前论文描述的"正常叠加"中，我们发现模型嵌入的特征数量超过其维度，通常将它们映射到正多面体上。例如，如果模型有一个二维隐藏空间，稀疏特征将被组织成一个五边形：

![正常叠加：特征组织成五边形](data:image/PNG;base64,...[内嵌图片 - 见原始页面](https://transformer-circuits.pub/2023/toy-double-descent/index.html#datapoints-vs-features)

然而，当我们训练模型在有限数据集上重构时，行为变得非常不同。我们不再看到特征形成正多面体，而是看到一些混乱无结构的情况——除非我们观察隐藏向量 h_i = Wx_i，即训练数据点在隐藏空间中的投影。令人惊讶的是，这些投影呈现出非常干净的结构。

具体来说，对于小的训练集，隐藏向量形成正 T 边形（称为"T-gons"），即训练数据点在隐藏空间中被排列成正多边形。模型不是在叠加中存储"泛化特征"，而是在叠加中存储单个数据点。

当我们增加训练样本数量时，行为发生了质变。在某个临界数据集大小以上，模型不再将数据点存储在叠加中，而是切换到我们熟悉的泛化特征模式。在过渡区域，模型表现混乱，特征维度和数据点维度都很低。

### 叠加双下降

模型在两种不同机制中表现迥异、中间出现奇怪行为的现象，令人联想起**双下降**，尤其是"数据双下降"。数据双下降的一个惊人现象是：测试损失在变好之前先变差——这违背了"更多数据总是减少过拟合"的朴素直觉！

对于给定的训练集大小 T，模型的解取决于随机选择的训练集，其中一些训练集有利于记忆（例如正交的训练样本），而另一些有利于泛化。为了确保我们的结果不是偶然，我们为每个数据集大小训练了 4 个使用不同数据集随机种子的模型。然后我们绘制平均测试损失，揭示了在"叠加中的数据点"机制和"叠加中的泛化特征"机制之间过渡处的明显凸起。

值得注意的是，我们是在没有标签噪声的情况下观察到双下降的。也就是说，输入和目标完全相同。这里的"噪声"来自下投影中发生的损失压缩。即使在线性投影的稀疏极限下，将 10,000 个特征编码到 2 个神经元中也是不可能的。因此重构必然不完美，产生不可避免的重构误差，进而导致双下降。

### m 对双下降的影响

此时，人们自然会问：双下降是否仅仅是 m=2 个隐藏维度的产物，或者是优化困难造成的。在本节中，我们确认情况并非如此，同时探索了双下降文献中的一个主题——将其理解为一个多维现象，而非一维现象，即模型大小、数据集大小和训练之间的多维交互。

我们将双下降可视化为一个二维函数，同时变化训练样本数 T 和隐藏维度数 m。所有其他超参数同上。我们为每个 (T, m) 配置训练四个模型，取平均损失。我们经验发现，当 m>3 时优化结果显著更加一致。

结果清晰地显示了"双下降"发生的区域——即更大的模型或更多的数据反而损害性能的区域。与先前关于双下降的工作一致，这些结果对权重衰减和训练轮数敏感。在附录中，我们展示了对 m=4 的模型：
- 训练更多轮数时，测试损失凸起增大
- 将权重衰减从 1e-2 增加到 1.0 则完全消除了凸起

### 最大数据维度

Adam Jermyn 进一步探索了模型对训练数据点赋予的维度如何随数据点重复次数变化。他发现在记忆区域，重复的数据点获得显著更高的维度——不仅与随机数据点相比，而且与"类似"的数据点相比。这意味着模型学习的是具体的"单数据点特征"，而不是对相似数据点有意义的特征。

这些结果表明，模型可以同时使用"数据点特征"和"泛化特征"——例如在泛化特征之上的"记忆挂件"。这也暗示，在真实模型中，区分这两种情况需要解决叠加问题。

### 讨论

我们在玩具模型中展示，记忆可以理解为模型在叠加中学习"单数据点特征"。当从这种表示数据点的策略过渡到表示特征的策略时，这些模型表现出双下降。

这项工作的几个启示：

首先，我们有一个过拟合的例子——在一个并非专门为过拟合/记忆调参的问题中——从朴素的角度看，它看起来极其混乱和复杂，但结果却非常简单和干净。虽然是玩具问题，但这非常有前景。

其次，我们观察到的东西恰好是"朴素假说 + 叠加"。这在回顾时非常合理：记忆是叠加的理想使用场景。从定义上讲，单个数据点特征是你能拥有的最稀疏的特征。

第三，Adam Jermyn 对重复数据的扩展表明，"单数据点特征"和"泛化特征"可以共存。双下降的漂亮相变只是锦上添花。重要的是拥有这两种表示数据点和表示特征的机制。

这项工作还可能对**机制异常检测**产生影响。也许最清晰的例子是 Adam Jermyn 的重复数据扩展。这里，模型同时具有"泛化机制"和"很少激活的硬编码特殊情况"。如果不理解叠加结构，区分它们将非常困难。

如果叠加是普遍存在的，这些结果表明机制异常检测需要解决叠加问题。我目前的猜测（尽管非常不确定）是，叠加是机制可解释性中最困难的问题。这意味着异常检测可能并非比整个机制可解释性显著更容易的问题。

### 权重衰减和训练轮数对双下降的影响（附录）

这里我们使用 m=4 的模型，因为它们比 m=2 的模型更不容易出现优化失败，使我们对结果更有信心。保持其他超参数与正文相同，我们发现减小权重衰减会增大测试损失凸起，而将其增加到 1.0 则完全消除了凸起。这与先前的双下降工作一致。

我们还变化了训练轮数（仍使用前 5% 训练的线性预热，随后余弦衰减学习率）。同样，我们发现结果与先前的双下降工作一致：更多轮数导致更大的测试损失凸起。
