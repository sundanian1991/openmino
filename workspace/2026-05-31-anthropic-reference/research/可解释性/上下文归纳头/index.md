## 摘要

随着 Transformer 生成模型的持续扩展和在现实世界中的日益广泛应用，解决其相关的安全问题变得越来越重要。** mechanistic interpretability（机制可解释性）**——试图逆向工程模型执行的详细计算——为解决这些安全问题提供了一条可能的途径。如果我们能理解导致 Transformer 模型产生其输出的内部结构，那么我们可能能够更系统地应对当前的安全问题，并预见未来更强大模型中的安全问题。

在过去，机制可解释性主要聚焦于 CNN 视觉模型，但最近，我们在[先前的工作](https://transformer-circuits.pub/2021/framework/index.html)中展示了 Transformer 语言模型机制可解释性的一些非常初步的进展。具体来说，我们在先前的工作中发展了一个分解 Transformer 操作的数学框架，使我们能够理解小型（1层和2层仅注意力）模型，并对其功能给出近乎完整的描述。也许最有趣的发现是**归纳头（induction head）**，这是一个电路，其功能是在序列中回溯寻找当前 token（记为 `A`）的先前实例，找到上次紧随其后的 token（记为 `B`），然后预测相同的补全将再次出现（例如形成序列 `[A][B] … [A] → [B]`）。换句话说，归纳头通过复制和补全先前出现过的序列来"完成模式"。

在本文中，我们迈出了构建间接证据的第一步，提出了一个引人入胜的假设：**归纳头可能构成了大型 Transformer 模型中绝大多数上下文学习（in-context learning）的实际机制**。具体来说，论文的核心论点是存在与 2 层归纳头具有相同或相似机制的电路，它们执行"模糊"或"最近邻"版本的模式补全，完成 `[A*][B*] … [A] → [B]`，其中 `A* ≈ A` 且 `B* ≈ B` 在某个空间中相近；此外，这些电路实现了大型模型中的大部分上下文学习。

我们获取这一证据的主要方式是通过发现和研究一个**相变（phase change）**现象，它在各种规模的 Transformer 模型训练早期（只要它们具有多于一层）都会出现，并且在训练损失曲线上表现为一个凸起。在此相变期间，大部分的上下文学习能力（通过序列早期和晚期 token 的损失差异来衡量）被获取，同时模型内部形成了能够实现相当抽象和模糊模式补全的归纳头。

具体来说，本文提出了六条互补的证据线索，论证归纳头可能是任意规模 Transformer 模型中通用上下文学习的机制来源：

1. **论据1（宏观共现）**：Transformer 语言模型在训练早期经历"相变"，在此期间归纳头形成，同时上下文学习急剧改善。
2. **论据2（宏观共扰动）**：当我们改变 Transformer 架构以改变归纳头能否形成（以及何时形成）时，上下文学习的急剧改善以精确匹配的方式随之移动。
3. **论据3（直接消融）**：当我们在测试时直接"敲除"小型模型中的归纳头时，上下文学习的量大幅减少。
4. **论据4（归纳头通用性的具体示例）**：尽管我们对归纳头的定义在复制字面序列方面非常狭窄，但我们经验性地观察到这些相同的头也似乎实现了更复杂的上下文学习类型，包括高度抽象的行为。
5. **论据5（归纳头通用性的机制合理性）**：对于小型模型，我们可以从机制上解释归纳头如何工作，并能展示它们对上下文学习的贡献。此外，实际的运作机制暗示了它可以被自然地重新利用以执行更通用的上下文学习。
6. **论据6（从小模型到大模型的外推）**：在前5个论据中，归纳头解释上下文学习的案例对于小型模型比对大型模型更强。然而，与归纳头和上下文学习相关的许多行为和数据在小型模型到大型模型之间是平滑连续的，这表明最简单的解释是机制相同。

## 关键概念

### 上下文学习

在现代语言模型中，上下文中较后位置的 token 比上下文中较早位置的 token 更容易预测。随着上下文变长，损失下降。从某种意义上说，这正是序列模型被设计来做的（使用序列中较早的元素来预测较晚的元素），但随着从较早 token 预测较晚 token 的能力变得更好，它可以越来越多地以有趣的方式被使用（例如指定任务、给出指令或要求模型匹配模式），这表明它可以被有益地视为一种独立的现象。当以这种方式思考时，它通常被称为**上下文学习（in-context learning）**，有时也被称为**元学习（metalearning）**，尽管这个术语隐含着更强的声明，即模型正在学习*如何*执行新能力（而不是"定位"一种能力，即学习它*应该*做什么）。

涌现的上下文学习在 GPT-2 中被注意到，并在 GPT-3 中获得了显著关注。仅仅通过调整"提示（prompt）"，Transformer 就可以被适配来执行许多有用的任务而无需重新训练，如翻译、问答、算术和许多其他任务。

文献中至少存在两种重要的、不同的概念化和测量上下文学习的方式。第一种概念化——以 Brown 等人为代表——关注特定任务的**少样本学习（few-shot learning）**。第二种概念化——以 Kaplan 等人为代表——关注观察不同**token 索引**处的损失，以衡量模型在接收更多上下文时预测能力的提升。

在本工作中，我们采用 Kaplan 等人的概念：**在增加的 token 索引处损失下降**。我们这样做是因为它比"少样本学习"是更通用的现象框架。贯穿本工作，我们计算一个简单的上下文学习启发式度量：

- **上下文学习得分**：上下文中第500个 token 的损失减去第50个 token 的平均损失，在数据集示例上取平均。

最后值得注意的是，上下文学习对安全可能具有特殊的相关性。上下文学习使得在长上下文之后预测模型的行为变得更加困难。从更长远来看， mesa-optimization 或 inner-alignment 等概念假定有意义的学习或优化可能在测试时发生（不改变权重）。上下文学习将是这种隐藏优化的一个显而易见的未来机制。

### 归纳头

在我们[先前的论文](https://transformer-circuits.pub/2021/framework/index.html)中，我们在两层仅注意力模型中发现了一种特殊的注意力头——我们将其命名为**归纳头（induction head）**。归纳头由一个电路实现，该电路由不同层中的一对注意力头组成，它们协同工作以复制或补全模式。第一个注意力头将信息从前一个 token 复制到每个 token 中。这使得第二个注意力头能够基于之前发生的事情来关注 token，而不是基于它们自己的内容。具体来说，第二个头（我们称之为"归纳头"）在序列中搜索当前 token `A` 先前出现的位置，并关注下一个 token（记为 `B`），复制它并使模型更有可能输出 `B` 作为下一个 token。也就是说，两个头共同作用使序列 `…[A][B]…[A]` 更有可能被补全为 `[B]`。

归纳头通过类比归纳推理（inductive reasoning）来命名。在归纳推理中，我们可能推断如果在上下文中 `A` 后面较早跟随的是 `B`，那么在同一上下文中 `A` 更有可能再次后面跟随 `B`。归纳头将这种推断具体化。

注意，归纳头实现的是一个简单算法，而**不是**记忆一个固定的 n-gram 统计表。规则 `[A][B] … [A] → [B]` 无论 `A` 和 `B` 是什么都适用。归纳头似乎在很大程度上解耦了 `A` 和 `B`。这意味着归纳头在某种意义上可以**在分布外工作**，只要上下文早期的局部统计数据能代表后期的统计数据。

在本文中，我们的目标是提供证据证明一个更广泛的主张：归纳头在**通用**上下文学习中扮演主要角色（不仅仅是字面的 `[A][B]...[A]→[B]` 复制），适用于**大型**模型而不仅仅是小型2层仅注意力模型。

**正式定义**：我们将归纳头定义为在重复随机 token 序列上展示以下两个特性的头：

- **前缀匹配（Prefix matching）**：该头关注之前的、被当前和/或最近的 token 所跟随的 token。也就是说，它关注归纳推理所建议的下一个 token。
- **复制（Copying）**：该头的输出增加了对应于被关注 token 的 logit。

作为推论，归纳头将倾向于擅长完整地重复序列。例如，给定"The cat sat on the mat. The cat …"，归纳头将促进补全"sat on the mat"。

### 逐 Token 损失分析

为了更好地理解模型在训练过程中如何演变，我们分析所谓的"逐 token 损失向量"。其核心思想追溯到 Erhan 等人使用的方法，更一般地追溯到数学中的"函数空间"思想。

我们从一组模型开始。接下来，我们收集每个模型对一组一致的10,000个随机 token（每个取自不同的示例序列）分配的对数似然。我们将这些对数似然组合成一个"逐 token 损失向量"，并应用主成分分析（PCA）。

通过将这种方法应用于多个模型在训练过程中的快照，我们可以可视化和比较不同模型的训练轨迹如何在其输出方面演变。

## 归纳头是多数上下文学习机制的六大论据

在我们逐一展开论据之前，需要界定证据在哪些方面更具决定性、哪些方面较不具有决定性。对于小型仅注意力模型，我们有强有力的证据表明注意力头是多数上下文学习的机制。相反，对于所有模型，我们可以有力地论证归纳头在上下文学习中发挥**某种作用**。然而，模型越大，就越难确定归纳头构成了上下文学习的实际**多数**来源。对于带 MLP 的大型模型，我们主要依赖相关性证据，这可能存在混杂因素。

### 论据1：宏观共现

Transformer 语言模型在训练期间经历"相变"，在此期间归纳头形成，同时上下文学习急剧改善。

我们的第一个观察是，如果我们在训练过程中测量 Transformer 模型的上下文学习，它会在训练早期的一个狭窄窗口（大约25亿到50亿 token）内突然发展，然后在剩余的训练中保持恒定。在此窗口之前，上下文学习不到0.15 nats，之后大约0.4 nats，这一数量在剩余的训练中保持恒定，并且在许多不同的模型规模上也保持恒定（除了一层模型，其中从未形成太多上下文学习）。

事实证明，上下文学习的突然改善并不是在这个窗口中唯一改变的东西。如果我们遍历模型的注意力头并对它们是否为归纳头进行评分，我们发现归纳头在上下文学习发展的完全相同的窗口期间突然形成。例外是一层模型，归纳头从不形成——正如一层模型的上下文学习从未实质性发展一样。

这已经强烈暗示归纳头和上下文学习之间存在某种联系，但不仅如此，这个窗口似乎是训练过程中的一个关键点：无论发生什么，在训练曲线上都可见为一个凸起。这实际上是训练中损失不是凸的（单调递减斜率）的唯一位置。

以下所有现象在同一个突然窗口期间共同出现：
1. 上下文学习能力急剧改善（通过上下文学习得分衡量）
2. 归纳头形成
3. 损失经历一个小"凸起"
4. 模型的轨迹突然改变（在逐 token 损失空间中，通过 PCA 可视化）

总体而言，这些结果表明在训练早期的大约25亿到50亿 token 窗口期间发生了一些重要的转变。我们称这种转变为"相变"。

**评估证据**：尽管有上述所有共现证据，事实仍然是我们尚未证明归纳头是上下文学习的主要机制。我们仅仅展示了归纳头与上下文学习同时形成，并且此后上下文学习不再改善。

---

### 论据2：宏观共扰动

论据1的不足之处在于我们只是观察归纳头和上下文学习共同变化；像任何观察性研究一样，它不如我们主动改变一件事并测量另一件事发生了什么更有说服力。在本节中，我们做了一个更"干预性"的实验，我们以某种方式改变模型架构，使归纳头更容易形成，并观察对上下文学习的影响。

我们从观察到相变和相应的上下文学习改善仅出现在具有多于一个层的 Transformer 中开始。这正是如果归纳头是多数上下文学习的机制时我们所预测的：归纳头需要注意力头的组合，这仅在具有两个或更多层时才有可能。

我们定义了一种"smeared key"架构，具有一个非常简单的修改，应使任意深度的 Transformer 轻松表达归纳头。在我们的修改模型中，对于每个头 `h`，我们引入一个可训练的实参数 `α^h`，我们使用 `σ(α^h) ∈ [0, 1]` 在当前 token 和前一个 token 的 key 之间进行插值：`k^h_j = σ(α^h) k^h_j + (1-σ(α^h)) k^h_{j-1}`。归纳头是上下文学习主要机制的假设预测，在这种变化下，一层模型将发生相变，更多层模型可能更早发生。

结果与预测一致：当使用 smeared-key 架构时，上下文学习确实为一层模型形成（之前不会），并且为两层和更大的模型更早形成。

---

### 论据3：直接消融

对于它们所涵盖的情况，消融是我们最有力的证据。基本论据是敲除归纳头会减少我们在模型中观察到的上下文学习量。

事实上，在小型仅注意力模型中，**几乎所有的**上下文学习似乎都来自这些归纳头！这从相变开始就成立，并一直保持到训练结束。事实上，消融大多数其他注意力头似乎会**增加**上下文学习。

不幸的是，我们没有对我们全规模模型进行消融。对于我们有消融的模型，这一证据对于归纳头增加上下文学习（至少按照我们选择评估的方式）似乎是决定性的。

---

### 论据4：归纳头通用性的具体示例

一个完全不同的角度是找到归纳头实现看似困难的上下文学习行为的例子；这将使归纳头能够解释多数上下文学习变得可信。请注意，我们通过经验性复制任意 token 序列的注意力模式来**定义**归纳头。我们的目标是找到满足这个定义但**也**执行更有趣和更复杂行为的头。

在本论据中，我们展示了来自更大 Transformer（我们的40层、130亿参数模型）的归纳头的一些轶事性例子，这些头展示了如下行为——即字面复制、翻译和一种特定类型的抽象模式匹配。

#### 行为1：字面序列复制

我们从字面复制重复文本的最简单情况开始。

#### 行为2：翻译

众所周知，语言模型可以在语言之间翻译。有趣的是，我们遇到了许多可以进行翻译的归纳头的例子。在这里，我们探索了在我们的40层模型第7层中发现的一个头，展示了英语、法语和德语之间的翻译。

#### 行为3：模式匹配

在最后一个例子中，我们展示了一个进行更复杂模式匹配的注意力头。每个行遵循四个模板之一，后跟一个标签指示它从哪个模板中抽取。这个头关注回到正确类别的先前实例，超过一半的时间是正确的。

#### 关于同时是归纳头的更抽象头

我们再次强调，上述描述的注意力头同时实现了我们描述的抽象行为，并且这些**完全相同的**注意力头也满足归纳头的形式定义。这种比较不是隐喻或模糊定义：归纳头通过它们复制字面序列的能力来**定义**，**结果也**有时匹配更抽象的模式。

### 论据5：机制合理性

对于小型模型，我们可以从机制上解释归纳头如何工作，并能展示它们对上下文学习的贡献。此外，实际的运作机制暗示了它可以被自然地重新利用以执行更通用的上下文学习。

**复制**是由 OV（"输出-值"）电路完成的。**前缀匹配**是在 QK（"查询-键"）电路中使用 K-composition（以及在较小程度上 Q-composition）实现的。

在我们之前论文中研究的小模型中，所有归纳头都具有推动其注意力模式所描述的 QK 项和一个执行复制的正特征值电路。

**关于更复杂的归纳头**：我们假设它们在两个方面不同：（1）使用更复杂的 QK 项，而不仅仅匹配前一个 token；以及（2）匹配和复制更抽象和复杂的语言特征，而不是精确的 token。

---

### 论据6：从小模型到大模型的连续性

这个论据实际上是来自上述所有论据的扩展推论。论据1-5为小型 Transformer（尤其是小型仅注意力模型）中归纳头负责多数上下文学习提供了相当强有力的证据，而对于大型 Transformer，证据没有那么强。从多大程度上可以合理地从小模型中推断出同样的事情正在更大模型中发生？

模型分析表中的测量在小注意力模型、带 MLP 的小模型和全规模模型之间看起来完全类似。只要多于一层，所有模型都经历相变。所有模型都具有相同的上下文学习急剧增加，转变前后的粗略量相同。所有模型在 PCA 空间中追踪相似的路径。所有模型都形成归纳头。

我们最令人信服的替代可能性是其他组合机制也可能在相变期间形成。如果所有"组合头"在相变期间同时形成，那么在超过某个规模时，非归纳头的组合头可能共同解释了比归纳头更多的相变和上下文学习改善。

## 模型分析表

上述论据基于对34个仅解码器 Transformer 语言模型的分析，包括四组不同的模型系列：

- **小型仅注意力模型**：没有 MLP 的一系列模型（从1层到6层）
- **带 MLP 的小型模型**：同时具有注意力和 MLP 层的一系列模型（从1层到6层）
- **全规模模型**：一系列连续更大的带 MLP 模型（从4层、13M参数到40层、13B参数）
- **Smeared key 模型**：一个定向的架构实验，旨在允许任意深度的 Transformer 表达归纳头

训练数据集包括经过滤的 common crawl 数据和互联网书籍，以及包括大约10% python 代码在内的几个较小的分布。

## 未解释的奇特现象

### 看似恒定的上下文学习得分

本文中一个更奇怪的观察是，**上下文学习得分**在相变之后对所有模型几乎相同。似乎模型是小型的两层模型还是相当大的130亿参数模型无关紧要，模型是刚经历相变还是训练了更长时间也无关紧要。唯一重要的似乎是模型是否经历了相变。

大型模型仍然在所有索引处比小模型更好地预测 token，并且它们在较晚的 token 处预测得最好。正在发生的是，大型模型在上下文的非常早期就获得了对小型模型的所有优势。事实上，大部分差异在前十个 token 中形成。

### 相变对损失导数的影响

另一个我们发现非常引人注目的观察是，如果查看不同规模模型的损失曲线导数，它们的顺序似乎在相变时发生切换。在小模型比大模型学习得更快之前，在相变之后则相反。

## 讨论

### 安全影响

我们研究的最终动机是逆向工程神经网络可能帮助我们对它们的安全性感到自信的理论。相变行为使得研究人员和社会更难为未来的问题做准备。上下文学习强调了模型行为可以在推理过程中"改变"，而无需进一步训练。

### 连接学习动力学、缩放定律和机制可解释性

上下文学习相变可能是一个有用的"罗塞塔石碑"，连接机制可解释性、学习动力学和神经网络的统计物理类经验属性（如缩放定律或相变）。

## 相关工作

### 上下文学习

涌现的上下文学习在 GPT-3 中被令人信服地展示。一些发现与归纳头假设一致，或支持我们的方法论。然而，也有实验似乎与归纳头假设存在张力。

### 缩放定律

缩放定律与机制可解释性之间的关系可以被视为类似于热力学与单个粒子物理学之间的关系。归纳头相变是我们意识到的第一次连接。它们给了我们一个在宏观属性（损失）层面的现象，可以在电路和机制可解释性层面来解释。

### 相变与不连续模型行为

我们观察到的归纳头的不连续相变行为最类似于 Power 等人的"grokking"。我们认为我们对这一文献的主要贡献是将我们观察到的变化与归纳头的形成和涉及电路的参数层面理解联系起来。

### 学习动力学

如果神经网络确实可以在电路的层面被机械地理解，似乎几乎**必须有**某种方式来以电路变化的动力学理解学习过程。

### 普遍性

在可解释性和电路的背景下，"普遍性"是指多个模型发展出相同的特征和电路。我们对 OV 和 QK 电路的分析将这种普遍性声明扩展到电路层面。归纳头模式作为普遍的注意力模式出现。

## 附录

### 归纳头形成的位置

在本节中我们考察：归纳头在模型中位于何处？它们通常是在第二层形成，还是最后一层，还是其他地方？

- **小型仅注意力模型**：归纳头在最后一层形成。
- **带 MLP 的小型模型**：到5层和6层规模时，归纳头更多在倒数第二层形成。
- **全规模模型**：在我们全规模扫描的模型中，归纳头形成得更早。事实上，在24L和40L模型中，大多数归纳头在模型深度的一半之前形成。

### 分数分布

在我们的小型模型中分数分布比全规模模型更尖锐。在这两种情况下，具有正前缀匹配分数的头更可能具有正复制分数，而这种相关性对于最高的前缀匹配分数甚至更强。

### 验证头激活评估器

我们的复制评估器与每个头的 OV 矩阵相关性良好。我们的前缀匹配评估器与前一个 token QK 电路项的 QK 特征值迹相关性良好。

### 数据收集

我们测量模型的几个整体行为属性和每个组成注意力头的属性：

1. 未修改模型的逐 token 损失
2. 消融单注意力头的逐 token 损失
3. 使用头激活评估器的经验头测量
4. 前一个 token QK 电路项的 QK 特征值迹

### 消融方法

- **全消融**：运行模型，将被消融注意力头的结果向量替换为零向量。
- **模式保留消融**：先运行模型一次保存所有注意力模式。第二次运行时将被消融头的结果向量替换为零向量，但强制所有注意力模式使用第一次运行记录的版本。

### 头激活评估器

我们构建了三个注意力头的启发式测量：

1. **复制**：该头对残差流的直接影响是否增加与被关注 token 相同的 token 的 logits？
2. **前缀匹配**：在重复随机 token 序列上，该头是否关注与当前 token 匹配的较早 token 之后的 token？
3. **先前 token 注意力**：该头是否关注紧接当前 token 之前的 token？

### QK 特征值迹

我们度量与专门对关注前一个 token 的头进行"纯 K-composition"的 QK 电路项中的 `W` 矩阵的特征值整体大小和正性。`W` 的正特征值表示"相同匹配"。

### 分析

我们从逐 token 损失数据中展示了以下"视图"：损失曲线、2D 上下文学习图、2D 损失导数图、上下文学习得分和 PCA。

对于消融数据，我们展示了"前后"向量的归因和上下文学习得分的归因。
