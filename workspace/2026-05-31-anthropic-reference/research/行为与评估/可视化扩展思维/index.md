# Claude 的扩展思维

> 来源：Anthropic Research，2025-02-24

有些事我们几乎瞬间就能反应过来："今天是星期几？"另一些则需要更多脑力，比如解一道字谜或调试一段复杂的代码。我们能够根据手头任务选择投入或多或少的认知努力。

现在，Claude 也具备了同样的灵活性。通过全新的 [Claude 3.7 Sonnet](https://www.anthropic.com/news/claude-3-7-sonnet)，用户可以开启或关闭"扩展思维模式"，让模型对更棘手的问题进行更深入的思考。开发者甚至可以设置"思考预算"，精确控制 Claude 在某个问题上花费的时间。

扩展思维模式并不是切换到另一个采用不同策略的模型，而是让同一个模型给自己更多时间、投入更多努力来得出答案。

Claude 全新的扩展思维能力带来了令人印象深刻的智能提升，但也为那些关注 AI 模型如何工作、如何评估以及如何提升其安全性的人提出了许多重要问题。在这篇文章中，我们分享一些我们获得的洞见。

## 可见的思考过程

除了让 Claude 拥有更长时间思考、从而解答更难问题的能力之外，我们还决定以原始形式展示其思考过程。这带来了几个好处：

- **信任。** 能够观察 Claude 的思考方式，使得理解和检查其答案变得更容易——也可能帮助用户获得更好的输出。
- **对齐。** 在我们此前的一些[对齐科学研究](https://www.anthropic.com/research/alignment-faking)中，我们曾利用模型内心所想与外在表达之间的矛盾，来识别它何时可能涉入欺骗等令人担忧的行为。
- **趣味。** 观察 Claude 思考常常令人着迷。我们一些有数学和物理背景的研究者注意到，Claude 的思考过程与他们自己推理难题的方式惊人地相似：探索多个不同角度和推理分支，反复进行双重和三重检查。

但可见的思考过程也有几个缺点。首先，用户可能会注意到，展示出来的思考比 Claude 的默认输出更抽离、更缺乏个人色彩。这是因为我们没有对模型的思考过程执行标准的[角色](https://www.anthropic.com/research/claude-character)训练。我们希望给 Claude 最大自由度，让它能进行任何必要的思考以得出答案——而正如人类的思考一样，Claude 有时会在思考过程中产生一些不正确、误导性或半生不熟的想法。许多用户会觉得这很有用；其他人则可能对此（以及思考过程中较少角色特质的内容）感到困扰。

另一个问题是被称作"忠实性"的问题——我们并不确定思考过程中显示的内容是否真正代表了模型内部的运行情况（例如，英文单词，如思考过程中所展示的，可能根本无法描述为什么模型表现出某种特定行为）。忠实性问题——以及如何确保忠实性——是我们活跃的研究领域之一。到目前为止，我们的结果表明，模型往往基于其思考过程中*未*明确讨论的因素做出决策。这意味着我们不能依赖监控当前模型的思考来对其安全性做出有力论断。

第三，它带来了若干安全和安保隐患。恶意行为者可能利用可见的思考过程来构建更好的越狱策略。更推测性地看，如果模型在训练中了解到其内部思考将被公开展示，它们可能被激励以不同的、更难预测的方式进行思考——或刻意隐藏某些想法。

这些后一种担忧对未来更强大的 Claude 版本尤为尖锐——如果这些版本存在对齐偏差，将构成更大的风险。对于未来的发布，我们将权衡揭示思考过程的利弊。在此期间，Claude 3.7 Sonnet 中的可见思考过程应被视为研究预览。

## Claude 思考能力的新测试

### Claude 作为智能体

Claude 3.7 Sonnet 受益于我们可称之为"行动扩展"的能力——一种增强的能力，使其能够迭代地调用函数、响应环境变化，并持续执行直到完成一项开放任务。此类任务的一个例子是使用计算机：Claude 可以发出虚拟鼠标点击和键盘按键，替用户解决任务。与其前身相比，Claude 3.7 Sonnet 能够为计算机使用任务分配更多回合——以及更多时间和计算资源——其结果通常也更好。

我们可以从 Claude 3.7 Sonnet 在 [OSWorld](https://os-world.github.io/) 上的改进看到这一点。OSWorld 是一个衡量多模态 AI 智能体能力的评估基准。Claude 3.7 Sonnet 起步时略好，但随着模型持续与虚拟计算机交互，其性能差距会随时间扩大。

### Claude 玩宝可梦

Claude 的扩展思维和智能体训练共同帮助它在 OSWorld 等许多标准评估中表现更好。但它们也在一些或许更出人意料的、其他类型的任务上带来了巨大提升。

玩宝可梦——具体而言，Game Boy 经典游戏*宝可梦 红*——正是这样一个任务。我们为 Claude 装备了基础记忆、屏幕像素输入以及按下按钮和导航屏幕的函数调用能力，使其能够超越通常的上下文限制持续玩宝可梦，通过数万次交互维持游戏进程。

在下图中，我们绘制了 Claude 3.7 Sonnet 与之前没有扩展思维选项的 Claude Sonnet 版本的宝可梦进度对比。可以看到，之前的版本在游戏很早期就卡住了，Claude 3.0 Sonnet 甚至未能离开故事开始地真新镇的房子。

但 Claude 3.7 Sonnet 改进后的智能体能力帮助它走得更远，成功击败了三位宝可梦道馆馆主（游戏的 Boss）并赢得了徽章。Claude 3.7 Sonnet 在尝试多种策略和质疑先前假设方面非常高效，这使其能够在推进过程中不断提升自身能力。

### 串行与并行测试时计算扩展

当 Claude 3.7 Sonnet 使用其扩展思维能力时，可以描述为受益于"串行测试时计算"。也就是说，它在产生最终输出之前使用多个顺序推理步骤，在此过程中投入更多计算资源。总体上，这以可预测的方式提升其性能：例如，其在数学题上的准确率随着允许采样的"思考 token"数量的增加而对数级提升。

我们的研究者还在尝试使用*并行*测试时计算来提升模型性能。他们通过采样多个独立的思考过程，并在事先不知道正确答案的情况下选出最佳答案来实现这一点。一种方法是[多数](https://arxiv.org/abs/2206.14858)投票或共识投票：选出最常出现、被视为"最佳"的答案。另一种方法是使用另一个语言模型（如第二个 Claude 副本）来检查其工作，或使用一个学习到的评分函数来选择它认为最佳的答案。类似这样的策略已被多个其他 AI 模型的评估结果所报告。

我们在 [GPQA 评估](https://arxiv.org/abs/2311.12022)上使用并行测试时计算扩展取得了显著改进。GPQA 是一组常用的生物、化学和物理难题。使用相当于 256 个独立样本的计算量、一个学习到的评分模型以及最大 64k token 的思考预算，Claude 3.7 Sonnet 在 GPQA 上达到了 84.8% 的分数（其中物理子项分数为 96.5%），并且在超越多数投票极限后仍能继续从扩展中受益。我们在下方报告了评分模型方法和多数投票方法的结果。

## Claude 3.7 Sonnet 的安全机制

### AI 安全等级

Anthropic 的[负责任扩展政策](https://www.anthropic.com/news/announcing-our-updated-responsible-scaling-policy)要求我们在未实施适当安全和安保措施的情况下，不得训练或部署模型。我们的前沿红队和对齐压力测试团队对 Claude 3.7 Sonnet 进行了广泛测试，以确定它是否需要与之前模型相同级别的部署和安保防护措施——即 AI 安全等级（ASL）2 标准——还是需要更强的措施。

我们对 Claude 3.7 Sonnet 的全面评估确认，当前的 ASL-2 安全标准仍然适用。与此同时，该模型在所有领域都展现出了更高的成熟度和增强的能力。在考察与化学、生物、放射性和核（CBRN）武器制造相关任务的对照研究中，我们观察到模型辅助参与者相比无辅助参与者表现出一定的性能"提升"。也就是说，参与者能够比仅使用网上可获取的信息走得更远。然而，所有执行这些任务的尝试都包含了严重失败，完全阻碍了成功。

对该模型的专家红队测试产生了褒贬不一的反馈。虽然一些专家注意到模型在 CBRN 过程的某些领域知识有所改进，但他们也发现严重失败的频率过高，无法成功完成端到端任务。我们正通过加速开发和部署针对性分类器及监控系统，积极增强 ASL-2 措施。

此外，未来模型的能力可能需要我们迈向下一阶段：ASL-3 防护措施。我们最近在[宪法分类器](https://www.anthropic.com/research/constitutional-classifiers)方面防止越狱的工作，以及其他努力，为我们近期实施 ASL-3 标准的要求奠定了良好基础。

### 可见思考过程

即使在 ASL-2 级别，Claude 3.7 Sonnet 的可见扩展思维功能也是全新的，因此需要新的、适当的防护措施。在极少数情况下，Claude 的思考过程可能包含具有潜在危害的内容（主题包括儿童安全、网络攻击和危险武器）。在此类情况下，我们将对思考过程进行加密：这不会阻止 Claude 在思考过程中包含这些内容（这些内容对于最终产生完全良性的回复可能仍然重要），但思考过程的相关部分将对用户不可见。相反，用户将看到消息"此回复的其余思考过程不可用"。我们的目标是让这种加密极少发生，且仅在潜在危害较高的情形下触发。

### 计算机使用

最后，我们增强了 Claude 计算机使用能力的安全措施（上文已讨论：允许 Claude 查看用户计算机屏幕并代表用户执行操作）。我们在防御"提示注入"攻击方面取得了实质性进展，这种攻击是指恶意第三方将秘密信息隐藏在 Claude 使用计算机时可能看到的地方，可能诱使其执行用户非预期的操作。通过新的抗提示注入训练、包含忽略此类攻击指令的新系统提示，以及在模型遇到潜在提示注入时触发的分类器，我们现在能够 88% 地阻止此类攻击，而无缓解措施时为 74%。

以上仅是我们对 Claude 3.7 Sonnet 广泛安全工作的一部分简要总结。更多信息、分析结果以及多个防护措施运行示例，请参阅我们完整的[系统卡](http://anthropic.com/claude-3-7-sonnet-system-card)。
