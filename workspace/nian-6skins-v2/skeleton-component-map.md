## 建议优先级

### P0 — 直接替换（24 个骨架的 inline style → 组件 class）

所有骨架的 inline style 都应迁移到目录组件的 class，这是主要工作量。

### P1 — 建议补充引用的组件

| 骨架 | 建议增加 | 理由 |
|------|----------|------|
| S01 | C73 对角切割 Hero | 比 C75 更有视觉冲击力，适合封面 |
| S02 | C10 暗底强调块 | 当前 KPI 卡片缺少暗底组件的标准样式 |
| S09 | C40 幽灵水印 | 暗底收束页缺少背景装饰层 |
| S20 | C44 优劣对比 | Before/After 可用绿色/红色标记增强对比 |
| S25 | 新增空状态组件 | 目录中无此组件，需补充 |

### P2 — 目录缺口

| 缺口 | 说明 |
|------|------|
| 空状态组件 | S25 需要，目录中无。建议新增 C94 空状态页（居中图标+引导文案+CTA） |
| 阅读专用组件（C84-C93）| 全部未被骨架引用。这些组件适用于长文阅读场景，当前 27 骨架以展示/汇报为主，暂不需要 |
| 深色分屏变体 | C74 默认左浅右深，但 S03/S23/S27 需要左深右浅或双暗色变体，需确认 C74 是否支持 |
