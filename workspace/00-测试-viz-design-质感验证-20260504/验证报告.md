## 发现的问题与建议

1. **高亮比例边界**：5 个数据点 × 1 个高亮 = 20%，超过 10% 绝对比例。建议：规则改为"≤10% 或最少 1 个关键点"，取较大者。
2. **雷达图 JSON x/y 映射**：雷达图的 mapping.x = null，mapping.y = null 是合理的（极坐标无笛卡尔轴），但应在 JSON Schema 中明确允许 null。
3. **风险指数计算**：图 4 的风险指数公式 (100-FCI)×0.5 + 流失率×5 中，流失率权重过高（5× vs 0.5×），导致流失率主导了结果。建议增加权重合理性说明或在标注中拆分展示。
