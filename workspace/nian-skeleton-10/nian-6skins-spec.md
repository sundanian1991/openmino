# nian · 6 套皮肤 × 21 个骨架

> 每个场景一层皮肤，覆盖感知/决策/计划/执行/沉淀/对外全部场景。
> 骨架是结构，皮肤是气质。同一骨架换皮肤 = 同一结构不同气质。

---

## 6 套皮肤

### Skin 1 · 感知层（Perception）

**气质**：冷峻、精确、数据驱动
**适合**：行业研究报告、竞品分析、数据看板、全景地图

| 维度 | 设定 |
|------|------|
| 标题字体 | Inter 200 |
| 正文字体 | Inter 400 |
| 数据字体 | JetBrains Mono 700 |
| 主色 | `#2B5C8A`（蓝灰） |
| 辅色 | `#4A90D9` / `#7AB3E8` / `#A8D1F5`（同色阶） |
| 信号色 | `#C92A2A`（红·预警）/ `#2D6A4F`（绿·正向） |
| 背景 | `#F8F9FA`（冷灰白） |
| 卡片 | 直角、细边框、无圆角 |
| 装饰 | CSS 网格纹理（48px 间距） |
| 网格 | 12 列，gap 12px |

## 使用方式

1. **选场景** → 确定用哪套皮肤
2. **选骨架** → 从该皮肤的可用骨架中选
3. **定密度** → 低/中/高/全（插槽系统）
4. **填内容** → 替换 `[必填]` 占位符
