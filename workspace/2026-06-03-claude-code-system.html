WORKFLOW PRINCIPLE

透明，不是黑盒

透明工作流的核心价值：让 AI 的思考过程可见，不再是黑盒操作。五步法建立信任，每一步都经得起追问。

02 WORKFLOW

五步工作法

透明工作流（TRANSPARENT-WORKFLOW）定义了从接收到交付的完整路径，每一步都有明确的输出标准。

01 Step 1 Intent Gate 意图分类。简单查询、复杂任务、探索、代码、数据，分而治之。
02 Step 2 Pre-Declaration 委派前声明。使用工具前说明理由和预期，消除黑盒。
03 Step 3 Todo Management 任务可视化。多步骤时列出拆分清单，让进度一目了然。
04 Step 4 Progress Visibility 过程透明。执行中实时汇报进度，不做沉默运行。
05 Step 5 Verification & Summary 验证总结。完成后逐项对照验收，确保交付质量。

PLAN5 关键教训
01 不要跳过需求分析
直接开始写代码是最常见的错误。先问"要解决什么问题"，再问"怎么写"。
02 任务拆解要具体
"重构代码"不是任务。"将 X 函数从 A 文件移到 B 文件"才是可执行的任务。
03 持久化是核心
不记录到文件系统的讨论，下次会话就会丢失。文档即记忆。
04 验证要对照合同
完成标志必须提前定义。没有验收标准的任务，永远做不完。
05 五文件工作流
需求分析 → 任务拆解 → 文件持久化 → 执行追踪 → 验证总结。闭环。
04 ARCHITECTURE

架构理解

Claude Code 采用 CLI-first 路线，比 IDE 集成工具更轻量、灵活、可脚本化。AI 不只是补全代码，而是主动完成任务。

CLI-FIRST 命令行优先
比 IDE 集成更轻量、灵活、可脚本化。终端是界面，文件系统是状态，git 是版本控制。三层为基，不依赖 GUI。
AGENTIC 主动代理
不只是补全代码，而是主动完成任务 — 编辑文件、运行命令、创建提交。Agent 即执行者。
