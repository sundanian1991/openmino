# Visual Storyboard — 供应商年度评审

> 2026-05-02 — Phase 2 视觉叙事设计

---

## 站点级视觉语法

- **整体叙事弧线**：02-对比式（全景 → 差距 → 分层 → 建议）
- **视觉一致性规则**：
  - 主色调：Warm（Warm 500 高亮，Stone 300 灰化背景数据）
  - 辅助色：Coral 400（标注风险/不达标项）
  - 字体家族：系统无衬线（-apple-system, sans-serif）
  - 圆角/间距：圆角 4px，间距 16px 基准
- **什么必须统一**：供应商颜色跨 3 张图保持一致（毅航 Warm500，其余 Stone300 递减）
- **什么必须变化**：每张图的视觉编码方式不同（雷达/子弹/金字塔）
- **什么可以复用**：供应商名称、基准线数值

## 单图场景设计

### Scene 1: 雷达图 — 六家全貌

- **场景论文**：用雷达图一张图放完 6 家 4 维数据，强弱自现
- **在叙事中的角色**：开场定调 + 全景展示
- **叙事节拍**：
  - 第一眼看到什么 → 毅航的雷达面积最大、覆盖最全（颜色最亮）
  - 第二眼看到什么 → 尾部 3 家的面积明显缩小，形成视觉落差
  - 最终理解什么 → 6 家差距不是"差一点"，是"不在一个量级"
- **签名视觉元素**：灰色虚线基准圆（四项基准值连线），作为雷达图的"及格线"
- **克制声明**：只高亮毅航（最佳），不逐一标注每家数值；图例放右上，不占主视觉区

### Scene 2: 子弹图 — 差距量化

- **场景论文**：把"差多少"变成直观的距离感
- **在叙事中的角色**：证据支撑 — 量化雷达图中的模糊感知
- **叙事节拍**：
  - 第一眼看到什么 → 哪些条超过了基准线（绿色/暖色），哪些没到（灰色/Coral）
  - 第二眼看到什么 → 尾部 3 家几乎全部未达标，且差距幅度大
  - 最终理解什么 → 不是"个别指标差"，是"系统性落后"
- **签名视觉元素**：基准线垂直标记 + Coral 色标注"缺口"段落
- **克制声明**：4 项指标各一张子弹图，但合并为同一行/列显示；不画数据标签，只标关键缺口

### Scene 3: 金字塔 — 分层决策

- **场景论文**：用三层金字塔给出管理结论
- **在叙事中的角色**：总结收束 + 行动建议
- **叙事节拍**：
  - 第一眼看到什么 → 三层大小不同（塔底大、塔尖小）
  - 第二眼看到什么 → 每层标注了供应商名单和管理策略
  - 最终理解什么 → 不同层级需要不同管理方式，不能一刀切
- **签名视觉元素**：塔层颜色由下至上递增（深→浅），对应"关注度递增"
- **克制声明**：不加装饰性图标，不画背景纹理；纯文字标注策略

## 阅读路径设计

| 顺序 | 图名 | 功能 | 为什么在这个位置 |
|------|------|------|-----------------|
| 1 | 雷达图 | 全景开场 | 先让读者建立"全局观"，知道有哪几家、大概什么水平 |
| 2 | 子弹图 | 证据支撑 | 在全景感知差距后，用精确数据量化差距幅度 |
| 3 | 金字塔 | 结论收束 | 有了全景和证据，给出"所以应该怎么做"的分层建议 |

## 版面规划

- **排列方式**：垂直堆叠（适合滚动阅读 / PPT 页）
- **尺寸分配**：雷达图 800×550，子弹图组 800×400，金字塔 600×500
- **间距规范**：图间间距 32px，四周留白 24px
