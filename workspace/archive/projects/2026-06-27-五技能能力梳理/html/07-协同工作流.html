四层架构——在 AI 转型方案中的定位

L1

数据底座

jdme-local-tools（京ME 群聊采集 + 加密数据库直读）+ read-joymeeting（慧记会议全文 + 逐字稿）+ read-email（邮件完整访问）。覆盖方案中「群消息采集通道」「每日业绩自动化」「邮件报告生成」三个数据入口。

L2

可信知识库

全量消息存档 + 完整会议转写 + 邮件正文归档。对应方案中五库知识库体系（SOP 库 / 供应商档案库 / 经验案例库 / 规则库 / 商务沟通库）的数据填充。

L3

Skills 能力层

personal-insight-summary → 日/周工作简报（/日简报 P1）。jingME → 供应商承诺追踪 + 会议待办闭环（/整改跟踪 + /供应商沟通准备 P1）。这是准确率从 21% 到 95%+ 的关键杠杆（Anthropic 经验第 2 条：Skill 是最大杠杆）。

L4

验证层

人工抽样（每周）+ 离线评估（每月）+ 衰变监控（每月）+ 规则校准（每季度）。对应 Anthropic 经验第 3 条：系统会衰变，维护是严肃工程任务。

四个典型工作场景

场景 1：早上 9:00 日简报

personal-insight-summary 自动触发（默认配置：昨天 0:00-24:00）。并行调用 jdme-local-tools（summary --since yesterday）、read-joymeeting（--date yesterday）、read-email（search --after yesterday）。输出 6 板块日报：项目参与、优先处理、重点关注、风险提醒、已定决策、待定问题。

场景 2：供应商沟通前准备

jingME 触发：查某供应商群最近对话 + 相关会议记录。jdme-local-tools：chat-history --session <供应商群ID> --limit 200。read-joymeeting：--date last7days + 按群名过滤相关会议。输出：谁承诺了什么、截止何时、当前状态、建议提醒文案。

场景 3：周报生成

personal-insight-summary 周报模式（扩展方向）。聚合 5 个工作日的数据源。输出：本周关键事项、跨天主题归纳、持续跟进事项。

场景 4：供应商巡检准备

jdme-local-tools：chat-search --auto --keyword 巡检 --limit 50。read-joymeeting：--date lastmonth + 搜索历史巡检相关会议。jingME：提取巡检中发现的问题和整改承诺。

三层数据通道（技术视角）

日志恢复层

messages / watch：最近几天消息，无需 sqlcipher。从 sdk-event.log + renderer.log 恢复。

数据库直读层

chat-* / sessions / summary：完整历史记录，需 sqlcipher CLI。密钥 md5(pin).hexdigest()。

实时监听层

chat-watch：新消息推送，常驻进程。chat-watch --auto --keyword 跟进 --timeout 60。

智能加工层

personal-insight-summary + jingME。并行调度 3 个数据源 → 交叉分析 → 6 板块输出 + 跟进清单。
