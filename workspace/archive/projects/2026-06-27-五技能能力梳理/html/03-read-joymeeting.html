定位

从京东 JoyMinutes 平台提取用户参与的会议记录。采用 headless 浏览器 + 原生 CDP 协议的方式，一个脚本完成全部操作：启动浏览器 → 继承登录态 → 调用 API → 返回数据 → 清理浏览器。无需 Playwright MCP 或任何额外依赖。

核心能力（★★★★★ 数据深度）

完整会议记录 + 逐字稿转写文本
突破"只能看摘要"限制，拿到完整会议全文

Headless Chrome 自动继承 ERP SSO 登录态
无需手动登录，原生 CDP 协议直接调用 JoyMinutes API

日期参数解析规则

用户说法--date 参数说明
"今天" / 未指定today（默认）仅今天
"昨天"yesterday仅昨天
"本周"thisweek 或 week本周一到今天
"上周"lastweek上周一到上周日
"近 N 天" / "最近 N 天"lastNdays（如 last7days）从 N 天前到今天
"YYYY-MM-DD"--date YYYY-MM-DD指定单日
"X 月 X 日到 X 月 X 日"--start YYYY-MM-DD --end YYYY-MM-DD指定区间

核心命令

node fetch-meetings.js                                     # 今天的会议
node fetch-meetings.js --date yesterday                     # 昨天的会议
node fetch-meetings.js --date thisweek                      # 本周的会议
node fetch-meetings.js --date lastweek                      # 上周的会议
node fetch-meetings.js --date 2026-06-08                   # 指定单日
node fetch-meetings.js --date last7days                     # 近 7 天
node fetch-meetings.js --start 2026-06-03 --end 2026-06-09  # 日期区间
node fetch-meetings.js --date thisweek --detail             # 本周会议 + 逐字稿
node fetch-meetings.js --id 21ac80c26e507f76 --detail      # 按会议 ID 查询

返回字段

字段来源说明
titleminutes.detail会议标题
startTimeminutes.detail / list会议开始时间（毫秒时间戳）
durationlist会议录制时长（毫秒）
creatorlist.userInfo创建者姓名
membersminutes.detail参会人列表（姓名+ERP 账号）
speakersminutes.speakers.timelines发言人列表（姓名、发言占比、发言时长）
transcriptminutes.meetingrecord.query文字记录（需 --detail；数组格式，每项含 speaker/text/beginMs/endMs；为 null 表示无转写）
attachmentsminutes.detail会议附件（关联的 JoySpace 文档等）

transcript 字段解析

if (m.transcript && Array.isArray(m.transcript) && m.transcript.length > 0) {
  const fullText = m.transcript.map((item) => `[${item.speaker}]: ${item.text}`).join('\n')
  // 或按发言人分组统计
  const bySpeaker = {}
  m.transcript.forEach((item) => {
    if (!bySpeaker[item.speaker]) bySpeaker[item.speaker] = []
    bySpeaker[item.speaker].push(item.text)
  })
}

浏览器优先级

macOS

检测活跃窗口、运行进程、浏览器路径。优先级：当前活跃窗口的浏览器 → Chrome → Tabbit → Edge。

Windows

检测运行进程、浏览器路径。优先级：Chrome → Tabbit → Edge。

常见问题处理

  • SSO_REDIRECT：自动尝试当前活跃浏览器 → fallback 到其他浏览器 → 自动打开登录页
  • 会议数据为空但 totalInList > 0：日期过滤没有匹配到会议，检查日期或扩大范围
  • CDP not ready：端口被占用，脚本自动在 9333-9350 范围内查找可用端口

总结输出规范

总览（区间、会议总数、主要主题）+ 按日分组 + 综合要点（跨天主题归纳、持续跟进事项）
