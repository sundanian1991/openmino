# Reviewer Card — 卞海军（军哥）

## 角色：双重身份

军哥既是**听众**（听取方案汇报），也是**审批者**（决定是否推进、给资源）。

## 审批决策框架

他会用三个问题过滤你的方案：

### 1. 值不值得做？（价值判断）
- 这个问题有多大？（影响面）
- 不做会怎样？（不作为的风险）
- 做了能得到什么？（可量化收益）

### 2. 你想清楚了吗？（逻辑判断）
- 方案逻辑自洽吗？
- 有没有明显的漏洞？
- 数据能不能支撑你的判断？

### 3. 能落地吗？（执行判断）
- 谁来做？
- 多久做完？
- 中间卡住了怎么办？

## 对"合作健康度诊断"的具体关注

基于军哥的管理风格，他会重点关注：

| 关注点 | 他的潜台词 |
|--------|-----------|
| **访谈 5-8 家供应商** | "为什么是5-8家？不是全量？抽样依据？" |
| **Top 3 问题** | "只解决 3 个够吗？其他的怎么办？" |
| **90 天改善计划** | "90天太长了，能不能更快？" |
| **供应商满意度基线** | "现在的满意度是多少？你怎么知道？" |
| **争议处理缩短50%** | "50%的目标怎么来的？有依据吗？" |
| **季度追踪机制** | "谁负责追踪？什么时候第一次看数据？" |

## 指导类型预判

军哥给"指导"通常有两种：

| 类型 | 特征 | 应对 |
|------|------|------|
| **方向性指导** | "这个思路对/不对，你应该关注XX" | 记下来，调整方案方向 |
| **执行性指导** | "你去找XX，先搞定YY，ZZ之前给我看" | 当场确认时间节点，会后立即执行 |

## 汇报节奏建议

10分钟汇报，建议分配：

| 阶段 | 时长 | 内容 |
|------|------|------|
| **开场定调** | 1min | 为什么要做 + 核心结论 |
| **现状与问题** | 3min | 数据 + 痛点（用具体案例） |
| **方案与路径** | 3min | 怎么做 + 时间线 + 责任人 |
| **预期成果** | 2min | 量化目标 + 风险预判 |
| **请示** | 1min | 需要军哥支持什么 |

## 关键原则

- **结论先行**：第一页就要说清楚"我发现什么问题、打算怎么解决"
- **数据说话**：每个判断都有数据或案例支撑
- **主动预判质疑**：把他可能问的问题提前回答了
- **要资源要明确**：需要他做什么，一句话说清楚
