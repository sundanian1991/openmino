# 合作健康度诊断 — 演讲脚本

> 12页 / 约10分钟 / 受众：卞海军（军哥）

---

## 第1页 | 封面：合作健康度诊断

**幻灯片内容**：标题 + 副标题 + 汇报人 + 日期

**演讲词**（30秒）：

军哥好。今天汇报供应商管理BP的第六个项目——合作健康度诊断。

对标HRBP做组织健康度诊断的思路，我们准备系统性地梳理我方跟供应商之间的协作卡点，锁定问题、快速改善、建立追踪机制，防止问题反复出现。

这个项目不是写调研报告，是带动作、带责任人、带验收标准的闭环。

**时长**：30秒

**军哥可能追问**：

- "健康度诊断？你打算怎么量化？" → 应对：用四组定量指标（争议处理周期、投诉次数、响应时效、人员流失率）加上供应商访谈双线交叉验证，不是拍脑袋打分。
- "为什么是第六个？前五个什么进度？" → 应对：前五个项目按计划推进中，健康度诊断是独立模块，不依赖其他项目产出。

---

## 第2页 | 四大痛点：为什么要做合作健康度诊断

**幻灯片内容**：四个痛点卡片 — 摩擦未量化 / 评估机制缺失 / 反馈渠道空白 / 效率黑洞

**演讲词**（45秒）：

先说为什么要做这件事。

我们跟30多家供应商的日常合作中，摩擦是存在的。但这些摩擦从来没有被系统梳理过——不知道哪个环节效率最低、供应商有什么不满、争议处理到底拖多久。这是第一个痛点：摩擦未量化。

第二个，我们没有定期的合作满意度评估机制。合作状态好不好，全凭感觉。

第三个，供应商可能有很多想法，但没有正式的反馈渠道。他们不说，不代表没问题。

第四个，争议处理、需求传达这些关键环节的效率是未知的，很可能存在隐性成本。

与其等问题爆发再救火，不如主动诊断、精准治理。

**时长**：45秒

**军哥可能追问**：

- "你说的摩擦，能举几个具体例子吗？" → 应对：比如需求传达后供应商理解偏差导致返工、争议处理环节多方拉扯耗时过长、人员流失后交接不畅影响产能。这些我们在数据收集阶段会量化。
- "供应商有想法为什么不说？" → 应对：缺乏正式渠道是一个原因，另一个是担心说了影响关系。所以我们设计了匿名问卷加私下访谈两条线。
- "这事为什么不早点做？" → 应对：之前精力集中在准入和合规上，合作摩擦属于"重要但不紧急"的象限。现在BP项目推进到第六个，时机成熟。

---

## 第3页 | 项目目标：诊断 → 改善 → 追踪

**幻灯片内容**：三段递进箭头 — 识别卡点 / 快速改善 / 持续追踪

**演讲词**（40秒）：

项目目标分三步。

第一步，用数据和访谈找出最影响业务的Top 3协作问题。不是泛泛调研，是精准定位。

第二步，90天闭环改善。每个问题绑定责任人、解决路径、完成时间、验收标准——四个要素缺一不可。

第三步，建立季度健康度追踪机制。改善完了不等于结束，要持续追踪，防止复发。

总结一句话：不做调研报告，做改善闭环。

**时长**：40秒

**军哥可能追问**：

- "90天的依据是什么？能不能更快？" → 应对：Top 1问题可以加速到45天内闭环，其余两个按90天推进。90天是留了缓冲的，如果执行顺畅可以提前。
- "Top 3够吗？万一问题很多呢？" → 应对：Top 3之外的问题全部入backlog，不会丢。但改善阶段严控范围，贪多嚼不烂，反而什么都做不好。

## 第4页 | 数据收集框架：定量+定性双线并行

**幻灯片内容**：左右双栏 — 左侧定量指标（4项）/ 右侧访谈问题（4个核心问题）

**演讲词**（50秒）：

数据收集是整个项目的基础，分两条线并行。

定量方面，拉取近3个月的四组数据：争议处理数量和平均时长、投诉次数和类型分布、需求传达的响应时效、供应商人员流失率变化。这些数据大部分在现有系统里就能取到。

定性方面，访谈5到8家关键供应商的对接人。访谈围绕四个核心问题：合作中最顺畅的是什么、最不顺畅的是什么、最希望我们改进什么、如果只改一件事你选什么。前两个问题定位亮点和痛点，后两个问题锁定优先级。

两条线交叉验证——数据告诉我们在哪出问题，访谈告诉我们为什么出问题。1周内完成。

**时长**：50秒

**军哥可能追问**：

- "为什么是5-8家而不是全量？" → 应对：5-8家关键供应商覆盖80%以上的业务量，样本代表性足够。全量访谈成本高周期长，性价比不划算。
- "数据从哪取？系统里有吗？" → 应对：争议数据在工单系统，投诉数据在质检台账，响应时效在需求管理记录，人员流失率在供应商月报。部分可能需要手工补录。
- "供应商说假话怎么办？" → 应对：匿名问卷加私下访谈双线设计，且访谈问题是开放式的（"最不顺畅的是什么"），不容易敷衍。交叉验证也能筛出不实反馈。

## 第5页 | 问题排序矩阵：影响面 × 严重度锁定Top 3

**幻灯片内容**：2×2矩阵图 — 横轴影响面 / 纵轴严重度 / 右上角标注优先解决

**演讲词**（50秒）：

数据收完之后，用"影响面×严重度"矩阵来排序。

横轴是影响面——多少家供应商受这个问题影响。纵轴是严重度——对业务的影响有多大。

落在右上角的，就是我们要优先解决的Top 3。左上和右下的，是次优先，放入backlog等下一轮。左下的，暂不处理。

问题分成三类：流程类，比如步骤太多审批太慢；沟通类，比如需求不清反馈不及时；利益类，比如分配不公激励不足。每个问题会标注类别，方便后续对症下药。

核心原则：只抓Top 3，不贪多。其余入backlog，不丢。

**时长**：50秒

**军哥可能追问**：

- "影响面和严重度怎么量化？谁说了算？" → 应对：影响面用受影响的供应商数量占比来衡量，是客观的。严重度用对业务的影响程度——是否影响产能、是否导致客诉、是否造成资金损失——分三档打分。不是拍脑袋。
- "如果Top 3集中在同一类怎么办？" → 应对：那说明这一类问题确实最突出，反而更容易集中解决。我们会确保Top 3不全是低垂果实——至少有一个是结构性问题。

## 第6页 | 实施路径：四步闭环

**幻灯片内容**：横向四步流程箭头 — 数据收集(1周) / 问题分析(3天) / 改善计划(2天) / 季度追踪(持续)

**演讲词**（40秒）：

整体路径四步。

第一步，1周完成数据收集，定量拉取加5到8家访谈同步进行。

第二步，3天完成问题分析，用矩阵排序输出Top 3。

第三步，2天制定改善计划，每个问题绑定责任人和验收标准。

第四步，进入持续的季度追踪。

也就是说，从启动到计划定好，大约4周。然后进入90天改善期。改善完成后，转季度追踪。

**时长**：40秒

**军哥可能追问**：

- "4周定计划，会不会太慢？" → 应对：数据收集1周是最耗时的环节，其余都是分析和制定。如果数据取顺畅的话，可以压缩到3周。
- "谁来做这些事？" → 应对：年老师主导，数据拉取配合质检组和业务管理组，访谈我亲自做。

## 第7页 | 里程碑：4周诊断，7月复盘

**幻灯片内容**：横向时间轴 — 4月第1周 / 4月第2周 / 4月第3周 / 7月

**演讲词**（40秒）：

具体时间线。

4月第一周，数据收集和访谈全部完成，定量数据拉取到位。

4月第二周，输出Top 3问题，分析报告完成，跟军哥确认问题排序。

4月第三周，改善计划启动发布，责任人确认，90天倒计时开始。

7月，90天复盘。评估改善效果，决定是否进入新一轮诊断。

每个节点都有明确的交付物，可追踪可验收。

**时长**：40秒

**军哥可能追问**：

- "7月复盘的时候，如果没改善呢？" → 应对：复盘时如果Top 3问题解决率低于预期，会分析是执行问题还是问题定位问题。执行问题换方案，定位问题回溯数据重新诊断。不会不了了之。
- "4月第一周就能完成数据收集？数据都在手边吗？" → 应对：争议数据和投诉数据在系统里可以导出，响应时效和人员流失率需要从供应商月报中提取。部分可能需要手工补录，但1周时间足够。

## 第8页 | 90天改善计划：三个问题，三条闭环

**幻灯片内容**：三行四列表格 — 每行一个问题，四列：责任人 / 解决路径 / 完成时间 / 验收标准

**演讲词**（45秒）：

90天改善计划的核心是"三个绑死"。

每个Top问题都绑定四样东西：谁负责、怎么解决、什么时候完成、怎么验收。缺一个都不行。

目前问题还没诊断出来，所以表格里的具体内容是待确认状态。但框架已经搭好了，Top 3一旦确认，立刻填入并启动。

改善原则：每个问题可追踪、可验收、不可推诿。不是发个文件就算改善，要有实实在在的验收标准。

**时长**：45秒

**军哥可能追问**：

- "责任人是你还是供应商？" → 应对：我方内部问题，责任人是我方内部对应负责人。涉及供应商配合的，由我对接供应商推动。不会把球踢给供应商。
- "验收标准怎么定？谁来验？" → 应对：验收标准用数据说话——比如争议处理周期从平均X天降到Y天，满意度评分提升X分。最终向军哥汇报验收结果。
- "如果改善涉及制度调整呢？" → 应对：制度类改善需要审批流程，这个周期可能会长。我们会把制度类问题拆成"短期缓解动作"和"长期制度优化"两步，短期动作90天内完成。

## 第9页 | 量化目标：四个指标衡量改善效果

**幻灯片内容**：四个指标卡片 — 问题解决率 ≥2/3 / 争议周期缩短50% / 满意度提升1分 / 协作效率缩短30%

**演讲词**（50秒）：

改善效果用四个量化指标来衡量，不做无目标的改善。

第一，Top 3问题解决率不低于三分之二，也就是至少解决2个。

第二，争议处理周期缩短50%。目前基线数据还没建立，等数据收集完成后会确认当前值，然后对标缩短50%这个目标。

第三，建立供应商满意度基线，用1到10分评分，Q2末提升1分。

第四，跨部门协作效率缩短30%。需求从提出到解决的平均天数压缩三成。

每个指标都有明确的衡量方法，不是空喊口号。

**时长**：50秒

**军哥可能追问**：

- "50%缩短的依据是什么？凭什么不是30%或70%？" → 应对：50%是目标值，基线建立后根据实际情况调整。设定50%是因为现有争议处理周期偏长，有较大的压缩空间。实际执行中可能部分达标、部分超额。
- "满意度提升1分，基数是多少？1分算多吗？" → 应对：基数在数据收集阶段建立。1到10分制下，1分的提升在满意度量表中是一个显著变化。如果基线是5分，提到6分就是20%的改善。
- "这些指标谁在追踪？" → 应对：我在追踪。季度评估时出数据报告，向军哥汇报。

## 第10页 | 季度追踪机制：防复发、可持续

**幻灯片内容**：循环流程图 — 季度评估 → 问题追踪 → 仪表盘更新 → 改善行动

**演讲词**（40秒）：

最后一步是建立季度追踪机制。

每季度做一次轻量评估，用问卷加数据的方式，追踪Top 3问题的改善进度。评估完更新健康度仪表盘——一眼看清当前合作状态。如果发现回退，立刻启动新一轮改善行动。

改善不是一次性运动，是持续的机制。有了这个季度循环，问题一旦冒头就会被发现，不会积累到爆发。

**时长**：40秒

**军哥可能追问**：

- "季度评估的工作量多大？别变成负担。" → 应对：轻量设计，问卷控制在10题以内，数据取值用现成系统导出。单次评估不超过2天工作量。
- "仪表盘谁看？" → 应对：仪表盘向军哥和王易人同步开放，我负责维护更新。供应商侧不开放，避免引起不必要的解读。

---

## 第11页 | 风险预案：四个关键风险已预判

**幻灯片内容**：四行风险-应对对照表

**演讲词**（50秒）：

主要风险我预判了四个，每个都有应对方案。

第一，供应商可能不愿说真话。应对：匿名问卷加私下访谈双线设计，保证"说了不会有问题"。

第二，问题可能太多无从下手。应对：严格按影响面×严重度排序，只抓Top 3，其余入backlog不丢。

第三，改善计划变成纸上谈兵。应对：每个问题绑定责任人和时间线，季度跟踪，到期验收。

第四，诊断可能暴露我们内部的协作问题。应对：内部问题单独向军哥汇报，不和供应商反馈混在一起。内部该改的改，但不在供应商面前自爆。

这些风险都已经想清楚了，不是走一步看一步。

**时长**：50秒

**军哥可能追问**：

- "暴露内部问题是什么意思？具体指什么？" → 应对：供应商反馈中可能涉及我们内部部门协调效率、需求传达质量等问题。这些属于内部管理范畴，不适合在供应商面前讨论。我会单独整理成内部改善清单，向军哥汇报。
- "匿名问卷供应商会信吗？他们可能觉得能追溯。" → 应对：问卷设计上不收集身份信息，发放渠道用第三方工具（如问卷星），数据汇总后才给我看原始数据。会提前跟供应商说清楚匿名机制。

## 第12页 | 请示与下一步

**幻灯片内容**：三个请示事项 + 下一步行动

**演讲词**（30秒）：

最后请示军哥三件事。

第一，方案方向您是否认可？有没有需要调整的地方？

第二，访谈5到8家供应商时，是否需要您跟对方打个招呼？有您背书，供应商配合度会高很多。

第三，改善计划涉及跨部门协调时，能否请您出面推动？

如果您认可这个方向，5月第一周我就启动数据收集，开始干。

**时长**：30秒

**军哥可能追问**：

- "访谈需要多长时间？别影响正常业务。" → 应对：每家访谈30-45分钟，可以约在供应商日常对接的间隙。不会占用额外业务时间。
- "改善计划需要什么资源？" → 应对：主要是人力——数据拉取需要质检组配合半天，改善执行可能需要相关部门的工时投入。具体资源需求在Top 3确认后细化。
- "你一个人能搞定吗？" → 应对：数据收集和分析我主导，改善执行阶段根据问题性质分配给对应负责人。涉及跨部门的，需要军哥支持协调。
