经验封装 / 知识蒸馏

你的经验值，正在被定价

2026年5月，中金公司上线了一个叫「老于」的AI分析师。团队把首席分析师于钟海过去几年写的30多万字研报全部喂给了AI，提炼出一套四步方法论：假设、验证、估值、风险。AI不再模仿他的语气，而是学会了他的决策逻辑。

这不是科幻。这是正在发生的事。而且不只有中金——东方财富、国泰海通、广发证券、国信证券，都在做同一件事。它们不再比赛谁用了更好的大模型，而是在比谁能把最值钱的分析经验，做成可复用的AI技能包。

不止券商。法律咨询、医疗诊断、建筑设计、供应链管理——每个依赖「专家判断」的行业都在发生同一件事。Prompt Engineering的时代正在过去，Skill Engineering的时代刚刚开始。

SKILL.md：你的经验，被写成一条规则

券商把这件事叫「分析师Skill」——本质是把一个专业人的工作方法论，打包成一个AI能读取、能执行的「技能文件」。拿中金的「老于」来说，它的核心文件长这样：

老于 — 分析框架
# 身份
计算机行业首席分析师
覆盖 A 股软件与云计算赛道

# 分析框架
1. 假设验证 — 看到市场共识后，先找反例
2. 盈利预测 — 底-中-顶三情景建模
3. 估值锚定 — 历史中枢±行业溢价
4. 风险清单 — 3条以上可证伪的尾部风险

# 输出要求
每篇研报必须有可被验证的预测指标
和完整的计算过程。

30万字研报怎么用？不是直接让AI背下来——那样只会得到一个「模仿语调但逻辑混乱」的缝合怪。真正的做法是：从研报里提取于钟海每次做判断的逻辑链——他看什么指标、用什么顺序分析、怎样排除干扰——然后把这些逻辑链写成规则。这叫知识蒸馏。

整个过程可以拆成三步：分解——把「经验」拆成「步骤」；抽象——把具体案例变成通用规则；验证——每个规则附上「什么情况下失效」。最难的不是「喂给AI」。最难的是你有没有把经验整理成可复用的方法论。很多人工作十年，简历写满了项目，但问「你怎么做决策的」，答不上来三步以上的逻辑。

这不是投行专属

你的工作经验里，藏着多少未被编码的SKILL？中金做的是把30万字研报提炼成四步框架。你不用30万字——你的经验可能就浓缩在以下内容里。

招聘初筛 HR

面试过300人后，一个HR总结出的规律：先看项目规模与团队复杂度，再看技术问答中追问细节的能力，最后看薪资预期与市场行情的偏离度。过去她说「感觉这个人不太对」——说不出具体哪里不对。编码后变成三个量化信号，每个0–3分，总分≤5不推进。按此规则筛选的50人，转正率78%；凭感觉筛选的50人，转正率52%。

运营文案 Marketing

写过100篇公众号爆款后提炼的规律：爆款都有三段式结构——前三句制造认知差，中间三节带数字和具体案例，最后一段提出可执行的一步行动。过去「这篇写得有感觉」——爆款靠运气。编码后变成可重复的结构模板，连续20篇按结构写，打开率稳定在15%以上。

销售跟单 Sales

跟过500个客户后总结的信号：按回复速度（24h内/48h内/超72h）、提问具体度（问价格还是问实施细节）、预算提及时机排序。过去「这个客户意向挺高」——靠感觉判断。编码后变成可追踪的评分卡，追踪100个客户的实际成交率，准确率从六成提升到八成。

更多场景：房地产中介的带看路线设计、幼儿园老师的课堂管理节奏、保险规划师的方案设计逻辑、供应链采购的供应商评估矩阵——每个行业都有只存在于专家脑子里的决策逻辑。

券商在比什么

券商之间的竞争正在转移。不是比谁家的大模型参数更大，而是比谁的经验能被AI执行得更稳定、复用得更广泛。

券商策略核心判断
中金公司 首席分析师「老于」分身，30万字研报提炼四步框架 个人方法论 → 团队级Skill
东方财富 8个标准化Skills接入华为云、阿里云 技能 = 可调用API
国泰海通 6大金融Skill包，研报/行情/选股/自选管理 按需组装，不是大而全
广发证券 30类标准化插件，覆盖投研/财富/客服/运营/合规 一套合规规则，5部门共享
国信证券 智能投研助手覆盖研报生成、合规质检 合规审查是最高频复用场景

一条时间线清晰可见：2023年大家比Prompt技巧；2024年抢的是检索增强，把私有知识库接进去；2025年转向Agent工作流，做多步任务自动化；到2026年，竞争来到了Skill封装——谁的方法论能被写成最稳定的执行规则。Prompt会过期，模型会迭代，但方法论一旦写成规则，可以迁移到任何底层模型上。

竞争力换了定义

三件事的转向值得细想。

一、写不出来的才是壁垒

2023年大家比谁会写Prompt。2026年，Prompt竞争已经到顶——所有人都能写出像样的提示词。现在真正的差距在于：你有没有一套能被AI稳定执行的方法论？中金能把于钟海的四步框架写成SKILL.md，你能不能把你的工作方法也写成规则？

Prompt技巧三个月就过时。但「看到市场共识后先找反例」这个判断原则，十年后依然有效。

二、你的直觉可以被定价

券商给分析师的经验定了价——把它做成了一个可调用的Skill产品。一个面过500人的HR，她的「看人直觉」如果写成规则，就跟中金的Skill有一样的商业逻辑。把隐性的经验变成显性的方法论，专业价值就不再被工作时间限制。某招聘平台已将「资深HR初筛规则」封装为付费Skill，企业可按调用次数付费——经验从「按小时卖」变成「按次卖」。

三、先写出来的人定义标准

券商是速度最快的，但每个行业都还没被覆盖。你所在的领域——带看方法论、课堂管理节奏、方案设计逻辑——目前只在脑子里，没有SKILL.md。当所有人都在用大模型写邮件、做总结时，你所在垂直领域的Skill标准还是空白。这个窗口可能只有12–18个月。

2023年，你的竞争力是「我会用AI」。

2026年，你的竞争力是「我有一套方法论，AI能替我执行」。

边界声明

AI是方法论放大器，不是判断替你器。

把你的经验写成SKILL.md，不会让你变成人工智能专家。它只是把你的经验从「每次靠感觉做判断」变成「AI帮你按规则执行」。最后做判断的还是你——AI不会替你决定招不招这个人、买不买这只股票、推不推荐这个方案。

第一步

写一个文件

不需要30万字。找一个你重复做过100次以上的决策。

01
我每次第一步看什么？
比如看简历时，我先扫项目规模还是先看学校背景？
02
我什么时候觉得「这个不能要」？
比如面试中哪个信号一出现，我就给低分？
03
这个决策我验证过多少次，结果如何？
比如按这三步筛选的候选人，转正率是多少？

把答案写进一个 .md 文件。这就是你的第一个SKILL。

你的第一个 SKILL.md
# 身份
资深 HR，负责技术岗位初筛

# 初筛规则
1. 项目规模 ≥ 团队 5 人 / 周期 3 个月
2. 技术问答中追问过至少 2 层细节
3. 薪资预期在市场价 ±20% 范围内

# 一票否决
- 简历出现 3 处以上时间重叠
- 期望薪资高于预算 50% 且无合理理由

# 验证记录
按此规则筛选的 50 人，转正率 78%
凭感觉筛选的 50 人，转正率 52%

30分钟起步：选一个重复场景 → 写下「第一步看什么」 → 写下「一票否决」信号 → 给每个信号一个可量化的标准 → 保存为.md，下次决策前先读它。
