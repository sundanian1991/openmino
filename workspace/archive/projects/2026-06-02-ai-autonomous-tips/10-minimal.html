12

让 AI 替你自主干活

阳志平

第一组 / Task Orchestration

任务编排

01 合并同类项

把相似任务打包，减少AI上下文切换。把同类请求放在一起处理，而不是频繁切换上下文。

Batch similar tasks / Reduce context switching
02 确定性分流

能用规则解决的不要用AI判断。if/else能搞定的走确定性路径，不确定的才交给AI。

Rule-based first / AI for uncertainty only
03 先确认世界没变过

每次开工前检查前置条件。文件还在不在？依赖变了没？上次做到哪了？

Pre-flight checks / Validate assumptions
-- 以上完成规划端，以下进入执行端 --
第二组 / Execution

实际开工

04 干了再说

不要让AI一直规划不动手。先跑起来再迭代，比完美计划更有用。

Ship fast / Iterate over plan
05 worktree 并行分身

用git worktree让AI同时处理多个分支。一个改前端，一个改后端，互不干扰。

Parallel branches / Isolated workspaces
06 补齐测试

每完成一步就写测试。不要堆积到最后，那会变成技术债雪崩。

Test incrementally / Avoid debt avalanche
-- 执行完成，进入验证阶段 --
第三组 / Self-Review

自检与评审

07 对抗性自检

让AI扮演反对者，找自己方案的漏洞。"如果你要攻击这个方案，你会从哪里下手？"

Adversarial role-play / Attack your own plan
08 换不同谱系模型评审

不同训练数据的模型看到不同盲区。Claude和GPT的视角差异就是你的安全网。

Cross-model review / Diverse training = diverse blind spots
09 分级修复

不是所有bug都要立刻修。P0阻塞流程的先修，P3样式问题的攒着。

Severity-based triage / P0 first, P3 batch
-- 验证完成，进入自治保障 --
第四组 / Autonomy

自动续航

10 看门狗

设置监控，AI偏离轨道时自动触发干预。超时、异常输出、循环行为都是触发信号。

Automated guardrails / Timeout / Loop detection
11 意外处理矩阵

预定义异常场景和对应策略。出了问题不用临时想，按表执行。

Predefined fallback / Exception lookup table
12 状态持久化

每个阶段保存快照。出问题时回滚到最近的安全点，而不是从头开始。

Checkpoint everything / Rollback to safe point
