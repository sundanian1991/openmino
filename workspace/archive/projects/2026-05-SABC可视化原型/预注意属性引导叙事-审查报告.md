# 预注意属性引导叙事 · 视觉审查报告

> 审查对象：SABC 标准化深度诊断框架 · 完整版（15 张图/表）
> 审查视角：预注意属性（Preattentive Attributes）如何引导叙事
> 审查日期：2026-05-17

---

## 整体判断

叙事骨架（"数据→发现→行动"）完整，15 张图覆盖了诊断的每个角度。但从"预注意属性引导叙事"的角度看，目前处于**叙事意图写在了文档里，但没完全写进图里**的阶段——多数图表告诉读者看什么的方式是靠文字标注，而不是靠视觉属性本身在 200ms 内抢到注意力。

最有参考价值的模板是**图5（入职趋势）**：它的预注意层级是清晰的——红色（现象）> 虚线（参考）> 灰色（背景）——读者不需要先读标题就知道该看什么。其余 14 张图应以此为质量标准重排各自的预注意属性优先级。

---

## 一、逐图审查

### 图1：职场 S/A/B/C 构成（堆叠柱状图）

**预注意属性编排**：色相编码 S/A/B/C，按 C 级率降序排列。

**问题**：C 级用了最浅的灰色（#B8C4D0），在视觉层级中是最退让、最不可见的颜色。但 C 级是这个图表想让读者第一时间注意的东西——它是"问题"。颜色在跟文字打架：浅灰色告诉眼睛"这不重要"。

**建议**：如果叙事意图是"先看问题职场"，C 级段落在视觉上应该具有最高的预注意突显度。要么：
- C 级用暖红色（#E8734A 或更重的红褐），S/A/B 后撤为不同饱和度的灰蓝
- 或者反向：C 级保留最饱和色，S 级用最浅色

**关键原则**：**预注意属性的突显优先序应当与叙事优先级一致**——想让人先注意到什么，什么就应该最"抢眼"。

---

### 图2：首贷×复贷对比卡

**预注意属性编排**：空间位置（并排）+ 颜色标注差异项。

**评估**：优秀。对比表是预注意友好的格式——读者可以用位置快速扫描各维度差异。颜色标注作为后加的 pop-out 信号，用在差异显著处，信号密度合理。

**小建议**：标注逻辑应统一为"什么值得看"而非"全部差异都标"。

---

### 图3：司龄×等级热力图

**预注意属性编排**：颜色饱和度梯度（#FFF0EB → #C75A35）+ 空间网格。

**问题**：饱和度梯度太窄。#FFF0EB 几乎是白色，#C75A35 是中等深度的橙褐。最密到最疏的视觉区分度有限——两个相邻格子的值接近时，饱和度差异可能肉眼不可分辨。

**建议**：使用更宽的颜色梯度（接近白色 → 深红褐），或者用分段色阶代替连续渐变，让聚集效应更明显。如果希望读者先看到"S级集中在哪儿"，S 级列应该是整张图里饱和度最高的一列。

---

### 图4：司龄段绩效走势（折线图）

**预注意属性编排**：双线+双轴+拐点标注。

**问题**：双轴折线图是预注意属性最脆弱的图表类型之一。读者需要先在两条线之间切换注意力，再在两个 Y 轴之间做参考，认知负荷很高。拐点标注是用文字补偿视觉编码的不足。

**建议**：拆成两张图——一张画 GMV 走势（柱+拐点标记），另一张画转化率走势。每张图只承载一条叙事线。如果必须合在一张图里，用**上下镜像布局**（两个不同方向的 Y 轴，两条线在不同空间区域）比双轴更易读。

---

### 图5：月度入职趋势（条形图+均值线）

**预注意属性编排**：红色高亮关键月份 vs 灰色背景月份 + 虚线参考线 + 背景色块"集中入职期"。

**评估**：全框架中预注意属性用得最好的一张图。原因：
- 红色 vs 灰色的色相对比，让"要看的"和"背景"在 200ms 内分离
- 均值线用虚线而非实线，视觉层级退后一级，不抢柱子的注意力
- 数据标签只出现在红色柱子上，降低认知负荷
- 背景色块形成"形状封闭"的预注意信号

**已实现的 HTML 原型验证了这些策略**。

---

### 图6：新人产出追踪（分组条形图）

**预注意属性编排**：暖橙 vs 灰色分组 + 并排位置。

**问题**：4 组指标 × 2 组人群 = 8 个柱子，密度太高。分组条形图的叙事前提是"对每个指标做一次比较"，但 4 个指标等权呈现时注意力被均匀稀释。

**建议**：如果核心叙事是"3月入职新人已贡献6,970万GMV（占总量21.8%）"，则将 GMV 占比突出为主视觉，其他 3 个指标放在辅助 mini 图中。

---

### 图7：SABC四档画像表

**预注意属性编排**：表格 + S/C 倍差列颜色编码（红/黄/绿阈值）。

**评估**：表格本身不是高预注意性的格式，但 S/C 倍差列的颜色编码有效。

**建议**：核心叙事是"6.7×"，应让这个数字在视觉上成为整张表的锚点——更大字号、更饱和的颜色，或者在表外独立放一个"6.7×"的视觉标签，让眼睛先撞上这个数字再回落表格看细节。

---

### 图8：转化断层对比（条形图+断崖标注）

**预注意属性编排**：色相编码 + 红色箭头标注"下降67%"。

**评估**：红色箭头是最高效的预注意元素——形状（箭头）+ 颜色（红色）+ 方向（向下）的三重叠加。好的设计。

**问题**：A→B 断崖是主要叙事，但 B→C 也标注了下降幅度——两个标注在预注意层级上竞争。

**建议**：只保留 A→B 的红色箭头标注，B→C 的下降幅度放在副标题或图注里，不在主视觉区域出现。**每张图只应有一个预注意锚点**。

---

### 图9：非参评群体构成（堆叠图+直方图）

**预注意属性编排**：红色高亮零通话群 + 绿色标记隐藏高潜者 + 两图并排。

**问题**：整个框架中预注意属性冲突最严重的设计。红色和绿色是等权重的 pop-out 信号，零通话（问题）和高潜（机会）在抢注意力。两图并排本身也有注意力竞争。

**建议**：
1. 确定主叙事。如果核心是"零通话47人是首要问题"，只红色标注零通话群，绿色标记转为文字标注
2. 两图并排要有一张明显更大（主次关系），或者用上下排布代替左右

---

### 图10：通时×GMV 关联（散点图+回归线）

**预注意属性编排**：首贷灰色、复贷暖橙色 + 两条回归线 + 相关系数标注。

**评估**：颜色分配正确——复贷贡献 90% GMV（暖色调应占视觉主导）。两条回归线的斜率差异是最自然的预注意对比信号。

**问题**：标注太多。相关系数 r=0.47、r=0.83、全量均值线、回归线……信息密度太高。

**建议**：预注意锚点只留一个——"复贷的回归线斜率远比首贷陡"。其他标注藏到 tooltip 里。均值线可以虚线淡化。

---

### 图11：司龄通时效率（条形图/阶梯图）

**预注意属性编排**：条形图位置 + 倍数列。

**评估**：本质上是表格的视觉化。"6.1x" 应该是图中唯一的预注意锚点。

**建议**：如果不做独立图，将效率数据直接标注在图4的拐点上（"MOB12+效率=54,117元/小时，是新人的6.1倍"）。如果做独立图，让 6.1x 作为唯一的视觉突出元素。

---

### 图12：评分维度贡献权重（条形图）

**预注意属性编排**：暖橙（正相关）vs 冷灰（负相关）。

**评估**：优秀的预注意编码——用色相直接编码符号（正/负），冷暖对比显而易见。负权重用冷灰编码，在视觉上显得"不对劲"。

**小建议**：在负权重的柱子上增加形状标记（如斜线填充或感叹号图标），因为灰色柱子虽然在色相上"冷"，但饱和度低，容易被忽略。

---

### 图13：ATT×转化率矩阵（散点图）

**预注意属性编排**：首贷暖橙/复贷灰色 + 四象限分割线 + 相关系数标注。

**评估**：四象限分割线是预注意友好的设计。但核心叙事是"没有相关性"——预注意属性很难编码"不存在的关系"。文字标注是目前唯一的办法。

**建议**：在散点图上方加注释框或信息摘要，用一句话说明 r≈0 的结论（"ATT不应作为管理重点"），而不是依赖读者自己从散点图中"看不出相关性"的推理。

---

### 图14：职场C级率排名（水平条形图）

**预注意属性编排**：红/黄/灰分段颜色 + 水平条形（利于阅读排名）。

**评估**：水平条形图是排名的标准选择。红/黄/灰按阈值分段是合理的颜色编码。

**问题**：如果 8 个职场中 3 个红、2 个黄、3 个灰——红色柱子的预注意突显压过所有灰色柱子，可能导致读者只看红色。

**建议**：如果叙事意图是"整体C级率偏高"，可以用统一深色调，只在长度上区分排名。然后用一条整体 C 级率参考线（红色虚线）作为预注意锚点，让读者先看整体水平，再看每个职场与整体的差距。

---

### 图15：GMV帕累托曲线

**预注意属性编排**：暖橙面积折线 + 标注节点 + 参考线。

**评估**：帕累托曲线本身预注意友好——面积填充（颜色区域）是强 pop-out 信号。标注节点引导了阅读顺序。

**问题**：50% 和 80% 两条参考线同时存在，预注意层级没有区分。

**建议**：去掉 50% 线（或淡化），突出 80% 线。三个标注节点中，"前30%=76.4%"最应醒目——因为它支撑标题结论。

## 二、跨图共性问题

### 问题1：颜色编码在不同图表中含义不一致

#E8734A（暖橙）在本框架中至少有 6 种不同的编码含义：

| 图表 | 暖橙编码什么 | |
|------|-------------|---|
| 图1 | S级（最好的、正面） | |
| 图5 | 集中入职（现象） | |
| 图8 | S级（最好的） | |
| 图10 | 复贷（贡献 90% GMV） | |
| 图12 | 正向贡献（正相关） | |
| 图13 | 首贷（注意：图10 和图13 暖橙含义矛盾） | |

**图10 和 图13 中暖橙的含义是矛盾的**——图10 用暖橙编码"复贷"（应预注意），图13 用暖橙编码"首贷"（不应预注意）。同一个色相在两张相邻的图里含义翻转，读者在多图扫描时会被色相含义的跳跃混淆。

**建议**：定义全局色相语义——暖橙 = "需要被注意的信号"，灰蓝 = "比较参考"；或固定暖橙为"正面/人多/贡献大"，冷色为"负面/少/贡献小"。在每张图的图例中明确定义。

### 问题2：预注意锚点过多

多数图表有 3-5 个标注/高亮/箭头，在竞争同一个 200ms 的注意力窗口。**每张图应只有一个主要预注意锚点**。

### 问题3：标题叙事很强，但视觉编码没有承载相同的故事

图表标题是叙事性的——"XX职场C级率最高"、"前30%坐席贡献76%GMV"。但标题说的事情在视觉上往往不是最先被注意到的元素。理想状态是：去掉标题，只看图的预注意属性排列，也能告诉读者"图想让我先看什么"。目前多数图做不到这一点。

### 问题4：图形类型选择的预注意效率

| 预注意效率高（推荐） | 预注意效率低（需补偿） |
|---|---|
| 条形图（长度） | 双轴折线图 |
| 热力图（颜色饱和度） | 表格 |
| 帕累托曲线（面积+位置） | 散点图（依赖密度） |

该框架中图4（双轴折线）和图13（散点图）用低预注意效率的格式承载核心发现。图4可以拆图，图13应搭配更强的文字引导补偿。

### P1（强烈建议）

3. **图9 双图冲突** — 红绿并置 + 两图并排，预注意信号在打架
4. **图3 色阶太窄** — 扩大饱和度范围或使用分段色阶

6. **图6 多指标分组条** — 浓缩为一个主视觉 + 三个 mini sparklines

### P2（改进型）

7. **图8 只留一个箭头** — A→B 断崖一个锚点足够
8. **图12 负权重加形状标记** — 灰色柱子需额外视觉权重
9. **图15 去掉一条参考线** — 突出 80% 线
