# 36 模板矩阵方案（v2 · 精确矩阵）

> 6 场景 × 3 深度 × 2 气质变体 = **恰好 36 个模板**

---

## 设计目标

1. **组件全覆盖**：26 个组件族每个至少出现在 3 个模板中
2. **气质均匀**：4 种气质各覆盖 9 个模板
3. **Section 模块化**：每个 section 独立可复用，模板之间可跨文件引用
4. **质感下限**：每个模板通过 5 条硬规则
5. **场景全覆盖**：6 个场景覆盖年老师 90%+ 工作场景

---

## 九、Section 模块化设计

### 核心理念

**模板 = Section 的有序集合。Section 是独立可复用的 HTML 片段。**

```
模板 T-DATA-L3-ST
  ├── hero-statement (S01)     ← 可被 T-DECISION-L3-ST 复用
  ├── data-metrics (S05)       ← 可被 T-REPORT-L3-ST 复用
  ├── data-ranking (S06)       ← 可被 T-DATA-L3-NM 复用
  ├── data-table (S07)         ← 可被 T-SOP-L3-SP 复用
  ├── content-statement (S15)  ← 可被 T-BRAND-L3-ST 复用
  └── footer (S21)             ← 所有模板共用
```

### Section 复用规则

1. 每个 section 模块有独立的 HTML + CSS 文件
2. section 之间通过 CSS 变量（palette, accent）传递主题
3. 模板文件内 section 按顺序排列
4. 施工时可跨模板引用 section：`<!-- ref: T-DATA-L3-ST#data-metrics -->`
5. 被引用 section 保持原样，只替换内容数据
