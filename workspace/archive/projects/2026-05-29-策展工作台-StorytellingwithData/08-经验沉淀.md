# 08-经验沉淀：Storytelling with Data 策展

> 2026-05-30

## 可复用的认知

### 1. 参照系选择 = 定题逻辑的镜像

选择和判断参照系时，用的方法论和定题完全相同：列出候选 → 确定评估维度 → 打分 → 推荐 + 降级方案。这不是两次不同的决策，是同一种决策逻辑在不同层面的应用。

### 2. 页面规划不是可选项

03 叙事规划之后、04 研究转策展文本之前，必须插入页面规划层。没有这层，文本写出来不知道放哪、怎么放。页面规划定义了"容器"，04 填充"内容"。

### 3. "策展级别" = 可交互原型，不是设计规范

设计规范文档是设计师的工作文件，不是策展产出。策展级别的产出应该是观众可以直接体验的、可交互的、有视觉完成度的东西。对 HTML 项目来说，就是一页完整的页面。

### 4. Nothing Design 的 5 层结构值得借鉴

Nothing Design 的稳定不是因为风格好看，是因为结构完整：tokens → components → rules → anti-patterns → workflow。5 层互相约束，缺一层就容易翻车。以后做设计系统可以直接照搬这个结构。

### 5. 编辑叙事式 + Nothing Design 的混合

编辑叙事式（NY Times Snow Fall）提供长页面的信息节奏和叙事逻辑，Nothing Design 提供组件级的视觉一致性。两者不冲突，可以叠加使用：叙事逻辑用编辑叙事式，视觉实现用 Nothing Design。

| 点 | 怎么做更好 |

| 页面规划的模块表更早和年老师对齐 | 03.5 阶段就对完模块组合 |
