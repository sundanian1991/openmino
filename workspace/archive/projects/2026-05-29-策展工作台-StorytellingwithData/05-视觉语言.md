# 05-视觉语言：Storytelling with Data

> 2026-05-29 | 参照系确认：B（编辑叙事式），C（教学卡片式）作为降级方案

---

## 一、参照系选择

### 候选

| 候选 | 代表 | 核心特征 |
|------|------|----------|
| A：当代数据编辑式 | FiveThirtyEight、The Economist | 干净、数据优先、层次分明、静态阅读 |
| B：编辑叙事式 | 纽约时报 Snow Fall | 长页面滚动叙事、视觉故事线、信息逐层展开 |
| C：教学卡片式 | Notion、在线课程 | 卡片布局、步骤引导、知识模块、可跳转 |
| D：编辑叙事 + 教学卡片混合 | B + C | 长页面做骨架，卡片做模块 |

### 评估

| 维度 | A 数据编辑 | B 编辑叙事 | C 教学卡片 | D 混合 |
|------|-----------|-----------|-----------|--------|
| **与主题同构性** | 中 | 高 | 低 | 高 |
| **观众体验** | 看 | 体验 | 学 | 体验+学 |
| **开发成本** | 低 | 高 | 中 | 高 |
| **与材料匹配** | 中 | 高 | 高 | 高 |
| **风险** | 太平淡 | 过度设计 | 太像课程 | 两套逻辑打架 |

### 推荐：B（编辑叙事式）

理由：
1. 同构性最高——滚动体验 = 策展主题（从混乱到清晰）
2. 与材料匹配——7 个单元的递进关系天然适合长页面叙事
3. 观众体验——沉浸式，信息逐步揭示，有节奏感
4. 风险可控——通过约束视觉复杂度来避免过度设计

降级方案：C（教学卡片式）。如果实现时 B 的成本过高，退到 C，骨架逻辑不变。

## 二、材料身份

| 类型 | 本项目中的实例 | 处理方式 |
|------|--------------|----------|
| **证据材料** | 书中案例（IT 工单、Pew Research、零售定价、意面图）、原始图表 | 主角。占据视觉焦点，有呼吸空间 |
| **分析材料** | 上下文五问、格式塔六原则、前注意属性、三幕结构 | 配角。结构化呈现（卡片、流程图），不抢证据注意力 |
| **定位材料** | 7 个单元的导航、章节标记、来源标注 | 退居幕后。存在但不显眼 |
| **氛围材料** | 牡蛎与珍珠比喻、小红帽故事、"Pie charts are evil" | 点缀。入口层冲击力和出口层收束感 |

**核心规则**：证据材料永远是主角。分析材料服务证据解读。氛围材料只在入口层和出口层出现。定位材料始终存在但不抢戏。

## 三、设计系统

### 3.1 Tokens

参照系为编辑叙事式，tokens 如下：

#### 字体栈

| 角色 | 字体 | 用途 |
|------|------|------|
| **Display** | 系统无衬线（-apple-system, "Helvetica Neue"） | 单元标题、入口层冲击语句 |
| **Body** | 系统无衬线 | 正文、展开层、证据层叙述 |
| **Data** | "SF Mono", "Menlo", monospace | 数据、标签、来源、脚注 |

编辑叙事式不追求特殊的字体个性，追求的是**层级清晰**。字体少，字号梯度明确。

#### 字号梯度

| Token | 大小 | 行高 | 用途 |
|-------|------|------|------|
| `--display` | 48px | 1.1 | 单元标题 |
| `--heading` | 24px | 1.2 | 模块标题（展开层、证据层内部） |
| `--body` | 18px | 1.7 | 正文（编辑叙事式的正文偏大，适合长页面阅读） |
| `--caption` | 14px | 1.5 | 图注、标签、来源 |
| `--label` | 12px | 1.3 | ALL CAPS 标签、脚注 |

#### 色彩系统

| Token | 值 | 用途 |
|-------|-----|------|
| `--bg` | `#FAFAFA` | 页面背景（暖白，不是纯白） |
| `--surface` | `#FFFFFF` | 卡片、面板背景 |
| `--text-primary` | `#1A1A1A` | 正文 |
| `--text-secondary` | `#666666` | 标签、图注、来源 |
| `--text-disabled` | `#999999` | 最弱信息 |
| `--accent` | `#2563EB` | 强调色（蓝，非红——避免情绪化，保持理性） |
| `--accent-subtle` | `rgba(37,99,235,0.08)` | 强调色底色 |
| `--border` | `#E5E5E5` | 分隔线、卡片边框 |
| `--success` | `#16A34A` | 正面数据 |
| `--warning` | `#D97706` | 注意数据 |

**色彩使用原则**：
- 强调色 ≤10% 视觉面积（呼应书中的前注意属性原则）
- 同一页面不超过 3 种有彩色
- 色盲友好——不依赖红绿区分

#### 间距比例（8px 基准）

| Token | 值 | 用途 |
|-------|-----|------|
| `--space-xs` | 4px | 紧凑间距（图标与标签） |
| `--space-sm` | 8px | 组件内部间距 |
| `--space-md` | 16px | 标准元素间距 |
| `--space-lg` | 24px | 模块间距 |
| `--space-xl` | 48px | 单元内大段间距 |
| `--space-2xl` | 96px | 单元间间距 |

### 3.2 Components

策展项目常用组件规范：

#### 对比面板

用于 A vs B 的直接对比（Pew Research 两版、饼图替代方案等）。

```
┌──────────────┬──────────────┐
│   版本 A      │   版本 B      │
│   [图片/图表]  │   [图片/图表]  │
│   说明文字     │   说明文字     │
└──────────────┴──────────────┘
```

- 两栏等宽，视觉权重相等
- 用 `--border` 分隔，不用阴影
- 说明文字用 `--caption`，居中或左对齐
- 如果有"推荐"标记，用 `--accent` 标签，不改变面板本身

#### 卡片组

用于多个平行条目（上下文五问等）。

```
┌─────────┐ ┌─────────┐ ┌─────────┐
│  标签     │ │  标签     │ │  标签     │
│  内容     │ │  内容     │ │  内容     │
└─────────┘ └─────────┘ └─────────┘
```

- 卡片背景 `--surface`，边框 `1px solid --border`，圆角 8px
- 标签用 `--label` 样式（ALL CAPS，`--text-secondary`）
- 内容用 `--body`
- 卡片间距 `--space-md`
- 不超过 5 张一行，超过则换行

#### 流程图

用于步骤、过程（六步去杂、三幕结构等）。

```
步骤 1 ──→ 步骤 2 ──→ 步骤 3
  │          │          │
  说明       说明       说明
```

- 垂直排列（编辑叙事式适合纵向阅读）
- 步骤编号用 `--accent`，步骤标题用 `--heading`
- 说明文字用 `--body`
- 连接线用 `--border`，1px 虚线

#### 图文混排

用于案例 + 文字解读（"数 3"实验、零售定价拆解等）。

```
[图片/图表]
---
文字解读
```

- 图片占满内容宽度
- 解读文字在图片下方，用 `--body`
- 图注用 `--caption`，`--text-secondary`

#### 引文

用于书中原文引用（牡蛎与珍珠、Robert McKee 等）。

```
│ "Exploratory analysis is opening 100 oysters..."
│ — Cole Nussbaumer Knaflic
```

- 左边框 `3px solid --accent`
- 引文文字用 `--body`，斜体
- 来源用 `--caption`，`--text-secondary`

#### 表格

用于结构化对比（替代方案等）。

- 表头用 `--label` 样式，底部边框 `--border`
- 单元格用 `--body`
- 数字右对齐，文本左对齐
- 不用斑马条纹，用行间距分隔

### 3.3 Rules

#### 三层视觉层级

每个屏幕（每个滚动视口）只有三层：

| 层 | 内容 | 视觉处理 |
|----|------|----------|
| **Primary** | 当前单元的标题或核心入口元素 | `--display`，`--text-primary`，大量留白 |
| **Secondary** | 正在阅读的模块内容 | `--body`，`--text-primary`，紧凑间距 |
| **Tertiary** | 来源、标签、导航 | `--caption`/`--label`，`--text-secondary`，边缘位置 |

**测试**：眯起眼睛看屏幕，能否一眼分出三层？如果两层在竞争，其中一层需要缩小、变淡或移开。

#### 信息密度

编辑叙事式的信息密度是**中等偏松**：
- 每个滚动视口只传递一个核心信息
- 模块之间用 `--space-xl`（48px）分隔
- 单元之间用 `--space-2xl`（96px）分隔
- 关键信息周围有大量留白（"呼吸空间"）

#### 色彩使用

- `--accent` 只用于：强调色标签、流程图编号、引文边框、交互元素
- 不用于：背景、大面积填充、装饰
- 数据状态色（success/warning）只用于数据值本身，不用于标签或背景

#### 编辑叙事式的节奏

滚动过程中，信息密度有节奏变化：

```
入口层：密集（标题+定位句+导言+入口元素，一屏可见）
  ↓
展开层：中等（分析、框架、理论）
  ↓
证据层：密集（案例、数据、图表）
  ↓
收束层：稀疏（一句话收束，大量留白）
  ↓
过渡：空白（单元之间的呼吸空间）
```

这个节奏本身就是"从混乱到清晰"的视觉体现——每个单元内部有起伏，单元之间有呼吸。

### 3.4 Anti-patterns

| 禁止 | 理由 |
|------|------|
| 7 个单元用同一模板 | 流水线，不是策展。每个单元的模块组合应由迷你定题决定 |
| 阴影、渐变、3D 效果 | 书中明确禁止，参照系也不需要 |
| 装饰性边框和分隔线 | 用间距分隔，不用线条。线条只在数据表格中使用 |
| 深色背景 | 编辑叙事式用暖白背景，深色留给特定场景（如书中案例的演示） |
| 每屏超过 3 种有彩色 | 色彩是事件，不是默认 |
| 正文小于 16px | 编辑叙事式需要舒适的长页面阅读体验 |
| 入口层没有具体元素 | "Q1 数据概览"不是入口层，"27 家供应商，5174 人"才是 |

### 3.5 Workflow

```
1. 选择参照系 → 候选→分析→选择（已在本项目完成）
2. 映射 tokens → 从参照系推导字体、色彩、间距（本节已完成）
3. 确定组件 → 从页面规划的组件清单映射到组件规范（本节已完成）
4. 确定页面类型 → 入口页 / 单元页（03.5-页面规划）
5. 规划模块 → 每个页面的模块组合（03.5-页面规划）
6. 原型实现 → 用 tokens + components + 规则实现（06-原型实现）
7. 验证 → 检查三层视觉层级、信息密度、色彩使用
```

## 四、与可视化技能的协作

本项目使用 nothing-design 或 web-design-engineer 实现时：

1. **tokens 优先用本项目的**：参照系是编辑叙事式，不是 Nothing 的工业风，所以字体、色彩、间距用本节定义的 tokens
2. **组件结构可以参考 Nothing 的**：对比面板、卡片、表格等组件的结构逻辑（三层层级、间距即意义）可以借鉴
3. **Nothing 的 anti-patterns 大部分适用**：无阴影、无渐变、无装饰、无斑马条纹——这些和编辑叙事式一致
4. **不适用的部分**：Nothing 的暗色模式、Doto 字体、点阵装饰——这些是 Nothing 的美学，不是编辑叙事式的
