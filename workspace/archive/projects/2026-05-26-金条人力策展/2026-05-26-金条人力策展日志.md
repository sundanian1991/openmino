# 项目日志 — 金条人力供给数字策展

## 日期：2026-05-26
## 阶段：完整工作流验证（阶段 1-6）

---

### 工作背景

将一份供应商人力数据分析报告（17个月、11个职场）转化为数字展览。使用 research-digital-curation-skills 全部 7 个 skill 完整走一遍工作流。

### 测试方向

用策展方法论处理一份纯数据报告，看工作流是否适配非历史/文化类主题。

### 发现

1. **策展对象界定（topic-framing）适配良好**。从"模糊数据"中提炼出"结构性分化"作为真正主体，比单纯做数据看板更有叙事价值。
2. **研究转策展文本（to-curatorial-copy）需要重新理解字段模型**。"观众进入层"在这个场景下是数字锚点（5,174 / 746），而不是人物或物件。字段模型需要灵活解读。
3. **叙事结构（narrative-modules）的页面类型表很实用**。纵向滚动叙事 + 章节分隔符的方案，比之前的 KPI 面板更适合"展览"定位。
4. **视觉语言（visual-language）的参照系选择很关键**。"分析型仪表册"而非"艺术品"的定位，决定了最终输出的气质。
5. **原型实现（prototype-standardization）与之前 nothing-design skill 的设计系统有冲突**。策展工作流强调"视觉服从内容"，nothing-design 强调"信息密度的机械美学"。最终选择了策展工作流的指导——视觉服务阅读路径，用了更轻的排版和更多留白。
6. **方法回收（log-method-recovery）是本轮最有价值的环节**。没有这个步骤，上面的发现就会丢失。

### 固定结论

| 结论 | 适用范围 |
|------|---------|
| 策展工作流可用于纯数据主题，不限于历史/文化 | 数据分析类展览 |
| 进入尺度为"数字"时，Hero 区域用 Doto 超大数字建立锚点 | 数字主导型展览 |
| 纵向滚动叙事 > 多页面切换，适合决策类报告 | 单主题、线性叙事 |
| 职场排名用进度条 + 趋势标签，比纯表格信息密度更高 | 横向比较场景 |
| "分析型仪表册"参照系适合面向管理层的展览 | B2B 决策支持 |

### 余留问题

1. 这套 skill 没有覆盖数据可视化图表的技术选型（Canvas vs SVG vs Chart.js）。本次用纯 Canvas 手绘，因为与 nothing-design 设计系统一致，但通用场景需要补充图表组件指引。
2. 页面间的跨单元分析（比如"伽玛的月度波动"从排名页跳转到趋势图的对应区间）未实现，需要交互层面的补充。
3. 移动端适配做了基础处理，但没有做真正的移动端阅读测试。

### 可复用方法

**"数据策展三步法"**（适用于纯数据主题的数字展览）：
1. 从数据中提炼"结构性判断"作为策展主体（不是数据本身，是数据揭示的分化/趋势/异常）
2. 按判断的层级组织页面（总览 → 维度拆解 → 个体画像 → 归因）
3. 视觉参照系选"分析型仪表册"而非"艺术展览"，信息密度优先
