# KaaS 智能知识库中台 — 操作手册

## 一、系统简介

KaaS（智能知识库中台）是京东科技面向事业群级团队的知识底座。数据源为 Git 仓库，按 `domains/<一级域>/<主题>/{docs,wiki,artifacts}/` 层级组织，支持 PRD 文档、技术 wiki、代码产物等多类型内容。  
Web 入口：`http://ai-kaas.pre-apps.jd.com/project-management`

## 二、Web 界面操作指南

### 2.1 登录与概览
打开入口地址后，系统标题为「智能知识库中台」。侧边栏导航包含以下核心模块：

- **项目管理** — 一级域下项目维护
- **知识录入与自动归类** — 文档批量录入、目录批量入库、目录批量接入指引
- **知识加工流水线** — 加工流水线、三层消化架构
- **知识库与召回策略** — 召回策略配置、召回调试、实时测试召回效果
- **外部集成与管理** — 接入指南、外部集成、管理

### 2.2 新增知识库项目（项目管理）

路径：侧边栏「项目管理」 → 选择或创建一级域

1. 在「一级域下项目维护」页面，维护 `domains/<l1>` 层级的一级域名称。
2. 在对应一级域下新增「主题」（topic）。主题对应具体的业务线或产品模块，如 `voiceprint`、`data-science`。
3. 每个主题目录下自动包含三个标准子目录：
   - `docs/` — PRD、技术文档、需求说明等
   - `wiki/` — 角色 wiki、设计规范、技术方案
   - `artifacts/` — 代码产物、测试用例、配置模板

### 2.3 知识录入与自动归类

路径：侧边栏「知识录入与自动归类」

**单文档批量录入**
- 支持格式：Markdown、Word、PDF、纯文本
- 操作：选择目标主题 → 上传文件 → 系统自动解析并归入对应主题目录

**目录批量入库**
- 适用场景：已有完整目录结构的知识资产整体迁移
- 操作：选择「目录批量接入指引」 → 按模板整理目录结构 → 一键导入

> 注：目录结构需遵循 `domains/<l1>/<topic>/{docs,wiki,artifacts}/` 标准，否则召回时路径映射可能出错。

### 2.4 知识加工流水线

路径：侧边栏「知识加工流水线」

KaaS 采用「三层消化架构」对原始文档进行自动加工：

1.  **第一层（解析层）**：文件类型识别、格式转换、文本提取
2.  **第二层（结构化层）**：标题层级解析、语义分段、关键词提取
3.  **第三层（索引层）**：向量化存储、倒排索引、关联图谱构建

文档录入后，系统自动触发流水线加工，完成三层消化后方可被检索召回。

### 2.5 召回策略配置

路径：侧边栏「知识库与召回策略」 → 「召回策略配置」

**召回模式选择**
| 模式 | 适用场景 |
|------|----------|
| 片段检索（Snippet） | 快速定位相关段落，适合日常问答 |
| 全文检索（Full-text） | 获取完整文档内容，适合深度阅读 |
| 主题打包（Pack） | 一次性拉取模板 + wiki + PRD，适合写作前置准备 |

**召回调试**
- 点击「召回调试」/「实时测试召回效果」
- 输入测试 Query（如"声纹识别一期 PRD Java"）
- 查看返回的片段、相关路径、命中主题

### 2.6 外部集成

路径：侧边栏「外部集成与管理」 → 「接入指南」

KaaS 提供两类调用方式：
1. **API 接口**：RESTful 接口（详见具体项目的 API 文档）
2. **MCP Server**：在 Claude Code / 智能体环境中直接调用 `kb_search`、`kb_pack` 等工具

## 三、MCP 工具调用（智能体/开发层）

当在 AI 工具（如 Claude Code）中调用 KaaS 时，使用以下 6 个 MCP 工具：

| 工具 | 功能 | 典型用法 |
|------|------|----------|
| `kb_describe` | 查看系统能力、数据源状态 | 首次接入时了解 KaaS 全貌 |
| `kb_resolve` | 路由决策 + 列出某主题下所有文件清单 | 快速定位主题目录结构 |
| `kb_search` | 关键词检索，返回片段 + 引用路径 | 日常知识查找 |
| `kb_pack` | 主题 × 角色 × 意图 → 打包模板 + wiki + PRD | **写作前首选** |
| `kb_get_artifact` | 单文件读取（可分段） | 查看具体文档内容 |
| `kb_get_artifacts` | 批量读取多个文件 | 补齐 pack 后的剩余文件 |

### 3.1 按任务类型调用的强制工作流

**场景 A：写 PRD / 写 wiki / 二期需求**
1. `kb_pack(topic="<主题slug>", role="pm", intent="write_prd", query="<需求名>")`
2. 如 siblings 里有遗漏，再调用 `kb_get_artifacts(paths=[...])`
3. 禁止仅依据 `kb_search` 片段就动笔

**场景 B：改前端 Vue/React / 改 Java 服务端 / 写测试用例**
1. `kb_search(query=..., role="fe"/"be"/"qa")`
2. `kb_pack(topic=..., role="fe"/"be"/"qa")`
3. 仍有缺口 → `kb_get_artifacts(paths=[...])`

**场景 C：仅探索 / 调试路由**
1. `kb_describe` → `kb_resolve(query)` → 看 topics 文件清单
2. `kb_search` 看片段

### 3.2 数据组织规范

```
domains/
  <一级域>/
    <主题>/
      docs/        ← PRD、需求文档、接口文档
      wiki/        ← 角色 wiki、设计规范、技术方案
      artifacts/   ← 代码产物、测试用例、配置模板
```

## 四、常见问题

**Q1：为什么我上传了文档但检索不到？**  
文档需经过「知识加工流水线」的三层消化才会被索引，上传后请等待加工完成（通常数分钟内）。

**Q2：新增项目时如何避免命名冲突？**  
在「项目创建」前，先用 `kb_resolve` 查询目标 `domains/<l1>` 下已有主题列表。

**Q3：pack 和 search 的区别是什么？**  
- `search` 返回片段（适合快速查答案）
- `pack` 返回模板 + wiki + PRD 全文（适合写作前建立完整上下文）

**Q4：外部系统如何集成？**  
参考「外部集成 → 接入指南」，获取 RESTful API 地址和鉴权方式（通常需 SSO Token）。
