01 · Executive Summary

执行摘要

供应商管理长期处于事件驱动状态 — 出问题才处理，缺体系，判断凭经验。Q2 的核心目标是将供应商管理从被动响应转为主动管理，通过十大落地项目覆盖管理全生命周期，每项都有明确目标、实施路径和量化指标。

核心 Takeaways

  • 四大转变：被动响应 → 主动管理；经验判断 → 数据驱动；单点动作 → 体系化闭环；执行者 → BP 角色。
  • 三阶段推进：4月打底（分级盘点 + 健康度诊断），5月体系建设（SOP + 考核 + 保留 + 备份），6月深化闭环（赋能 + 文化 + 复盘）。
  • 量化目标：供应商分级覆盖率 100%，核心供应商保留率 ≥ 95%，新供应商 90 天留存率 ≥ 85%，协作卡点 Top3 全部有行动项。

3. 需要哪些数据、权限和协作支持？

02 · Background

背景与角色定位

供应商管理 BP 是连接业务与供应商的桥梁，偏 BP 角色 — 通过影响他人达成结果，而非直接考核。

我的定位

负责 30+ 供应商的全量管理（合规、准入、清退、分工）。头部 6 家 — 毅航、毛毛虫、伽玛、赛维斯、岐力、翰锐 — 占金条产线 3 亿+ 职场。部门归属数据科技业务部二级部门，服务组编制，上级王易人。

现状：事件驱动

出问题才处理，缺体系，判断凭经验。没有数据仪表盘，没有预警机制，没有标准化的供应商评估框架。

核心差异：HRBP 管内部员工，有制度约束力。我们管外部供应商，约束力来自合同和利益分配。不能直接考核，只能通过数据、分级和利益绑定来影响。

Q2 四个转变

  • 被动响应 → 主动管理
  • 经验判断 → 数据驱动
  • 单点动作 → 体系化闭环
  • 执行者 → BP 角色

04 · Timeline

三阶段推进路径

4-6月只做一件事：把供应商管理从被动响应变成主动管理。三阶段推进，每阶段有明确交付物。

阶段一：打底（4月）

目标：看清现状，识别卡点。

  • 供应商分级盘点：30+ 供应商三维评估，输出九宫格分级清单
  • 合作健康度诊断：识别 Top3 卡点，启动 90 天改善计划

阶段二：体系建设（5月）

目标：建立标准化 SOP 和评估体系。

  • 新供应商冷启动 SOP（30/60/90天路径）
  • 考核体系对齐业务目标（分层分级，每人3-5个指标）
  • 核心供应商保留与激励专项（一供应商一策略）
  • 供应商备份与产能冗余方案
  • 供应商关系精细化 + 风险前置机制

阶段三：深化闭环（6月）

目标：深化赋能，完成 Q2 复盘。

  • 供应商主管赋能（目标拆解、绩效面谈、辅导反馈）
  • 新供应商引入效能提升（缩短周期、提高匹配质量）
  • 供应商文化对齐（行为准则、标杆案例、季度复盘）
  • Q2 整体复盘与下半年规划
05 · Key Projects

重点项目详解

项目 1：供应商分级盘点（九宫格）

30+ 供应商，一张图看清全局。

方法：三维评估 — 业绩 40% + 合规 30% + 潜力 30%。按高/中/低业绩与高/中/低潜力交叉形成九宫格，分为明星（重点投入）、中坚（重点培养）、尾部（限期优化）三档。

交付物：三张清单 — 保留清单（头部 6-10 家，资源倾斜、深度绑定）、培养清单（腰部 12-15 家，针对性辅导、季度跟踪）、优化清单（尾部 5-8 家，限期整改、准备替代）。

时间线：第 1 周数据收集 → 第 2 周九宫格落位 → 第 3 周汇报确认。3 周完成。

分级是内部管理工具，不对外公布排名。

项目 2：合作健康度诊断

定量 + 定性，找准 Top3 卡点，90 天闭环。

定量维度：交付及时率、质量投诉次数、需求响应时长、人员流失率。

定性维度：内部 stakeholder 访谈、协作流程走查、沟通记录分析。

预期卡点：需求响应慢（设立 SLA，48h 内响应，纳入考核）；人员流失频繁（关键岗位备案制 + 替补人才池）；结算流程不透明（月度对账模板 + 异常 48h 闭环）。

交付物：每供应商健康度报告 + Top3 卡点 + 90 天改善计划，责任到人。

项目 3：核心供应商保留

一供应商一策略，不通用模板。

健康档案：头部 6 家建档跟踪，月度更新 + 季度复盘。跟踪业绩趋势（月度 GMV）、人员稳定性（核心岗位）、合规记录 + 合作满意度。

三级预警：

  • 绿色：正常，周报跟踪
  • 黄色：单一指标下滑，72h 跟进
  • 红色：多项异常，24h 启动保留方案

激励包：资源倾斜（优先分配高价值业务）、发展机会（纳入创新项目试点）、合作深化（长期框架协议）、认可机制（季度优秀供应商表彰）。

项目 4：新供应商冷启动

30/60/90 天带教路径，否决项触发即清退。

阶段核心目标关键动作否决项
30天 · 熟悉期了解业务 + 熟悉流程 指定导师、发放 SOP、2 次旁听、合规培训 合规不合格
60天 · 融入期独立执行 + 首次交付 独立承接小项目、首次校准会、收集反馈 交付质量不达标
90天 · 转正期全面评估，决定是否保留 第二次校准会、转正评估、纳入分级 评分低于阈值

四个维度的转变

  • 管理方式：从"出问题才处理"到"提前预判"
  • 决策方式：从"凭感觉"到"有数据"
  • 角色定位：从"执行者"到"BP"
  • 供应商关系：从"甲乙方对立"到"利益绑定"

需要确认的 4 件事

  1. 权限边界：分析类直做，制度类先跟易人确认后再推
  2. 数据源对接：业绩数据从哪个系统拉？谁协助打通？
  3. 考核体系对接：现有标准可复用还是重新设计？
  4. 汇报节奏：月度简报 + 季度深度复盘，是否认可？

资源需求

  • 精力：Q2 预计 50% 精力，4 月打底阶段需 70%
  • 协作：跨部门 — 数据团队对接 + 财务协助
  • 配合：供应商 — 提前打招呼，确保对接人参与
  • 工具：BI 工具 — 如无可先用 Excel 过渡
Call to Action

打底（4月）：分级盘点 + 健康度 → 看清现状。
体系（5-6月）：SOP + 数据 + 预警 → 有法可依。
产出：分级图、诊断报告、SOP 集、仪表盘、复盘报告。
