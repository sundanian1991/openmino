# 项目1：供应商分级盘点（九宫格） — 落地执行方案

> 作者：Mino | 日期：2026-05-06 | 状态：待执行 | 优先级：P0

---

## 一、项目定位

### 1.1 为什么要做

当前32家供应商、70个站点分散在12条业务线上，缺乏统一的分级视图。现有月度评估（A/B/C/D/E）仅反映单站点单月表现，无法回答以下战略问题：
- 哪些供应商是核心依赖，哪些是边缘替补？
- 哪些供应商在增长，哪些在衰退？
- 如果某家供应商突然清退，业务连续性风险有多大？
- 分量调整（赛马机制）该向谁倾斜、从谁手里收？

月度评估是"体温计"（当下健康度），九宫格是"体检报告"（全局结构+趋势）。两者缺一不可。

### 1.2 解决什么问题

| 问题 | 解决方式 |
|------|---------|
| **看不到全局结构** | 九宫格以"业绩水平 × 增长趋势"为两轴，32家供应商落入9个格子，一目了然 |
| **分量调整凭感觉** | 明星象（右上）自动获得扩项优先，衰退象（左下）自动进入缩减名单 |
| **缺乏供应商差异化管控** | 不同象限匹配不同的管理力度（高频巡检 vs 月度巡检 vs 季度巡检） |
| **续约决策无抓手** | 续约前直接读取九宫格位置，明星/金牛直接续约，瘦狗/衰退启动汰换评估 |

### 1.3 与其他项目的关系

| 项目 | 关系 |
|------|------|
| 项目2：合作健康度诊断 | 九宫格是结果，健康度诊断是原因。九宫格发现"这家在衰退"，健康度诊断回答"为什么衰退" |
| 项目3：新供应商冷启动 | 冷启动供应商在九宫格中固定落入"新进入"区域，满6个月后纳入正式盘点 |
| 月度评估（现有制度） | 九宫格的Y轴（业绩水平）直接引用月度评估的季度平均分 |

### 1.4 成功标准（量化指标）

| 指标 | 目标值 | 验证方式 |
|------|--------|---------|
| 盘点覆盖率 | 100%（全部32家供应商） | 供应商名单与盘点报告一一对应 |
| 数据采集完整率 | ≥95%（每家供应商≥5项指标有数据） | 数据完整性检查表 |
| 九宫格分类争议率 | ≤10%（争议家数/总家数） | 争议记录清单 |
| 盘点结果应用率 | 100%（盘点后30天内产出分量调整建议、续约建议、巡检计划调整） | 应用追踪表 |
| 首次执行完成时限 | ≤4周 | 项目启动到交付报告 |

|------|---------|------|

| **战略供应商识别不清** | 没有"不可替代性"评估维度 | 清退决策只看绩效不看替代成本 |

## 三、实施框架

### 3.1 核心方法论

九宫格采用 **"业绩水平 × 增长趋势"双轴矩阵**，将供应商划分为9个象限：

```
              增长趋势（环比季度）
              ↑
     ┌─────────┼─────────┐
     │  问题   │  明星   │  ← 高增长
     │ (低业绩  │ (高业绩  │
     │  高增长) │  高增长) │
高   ├─────────┼─────────┤
业   │  衰退   │  金牛   │  ← 零增长
绩   │ (高业绩  │ (高业绩  │
     │  零增长) │  零增长) │
低   ├─────────┼─────────┤
     │  边缘   │  观察   │  ← 负增长
     │ (低业绩  │ (中业绩  │
     │  负增长) │  负增长) │
     └─────────┼─────────┘
        低      中      高
              业绩水平
```

**注意**：上述为逻辑示意。实际九宫格3x3布局如下：

| | 增长趋势：负增长 | 增长趋势：零增长(±5%) | 增长趋势：正增长 |
|---|---|---|---|
| **业绩：高（前30%）** | 衰退（左上） | 金牛（中上） | 明星（右上） |
| **业绩：中（30%-70%）** | 观察（左中） | 稳健（正中） | 潜力（右中） |
| **业绩：低（后30%）** | 边缘（左下） | 停滞（中下） | 问题（右下） |

### 3.2 评估维度与权重

#### X轴：业绩水平（60%权重）

以**季度综合得分**为基准，取最近一个季度（或最近3个月）的月度评估平均分：

| 分级 | 标准 | 说明 |
|------|------|------|
| 高业绩 | 季度平均分 ≥85分 或 月度排名在前30% | 对应月度评级A或B+ |
| 中业绩 | 70 ≤ 季度平均分 <85 或 月度排名在30%-70% | 对应月度评级B或C |
| 低业绩 | 季度平均分 <70 或 月度排名在后30% | 对应月度评级C-/D/E |

**数据来源**：从 `SM-TPL-008_月度评估评分表.xlsx` 提取近3个月总分，计算季度平均。

#### Y轴：增长趋势（40%权重）

以**近两个季度的业绩变化率**为衡量：

| 分级 | 标准 | 说明 |
|------|------|------|
| 正增长 | 环比增长 ≥+5% | GMV、产能、评分任一提升≥5%即判定 |
| 零增长 | -5% ≤ 环比 ≤ +5% | 基本持平 |
| 负增长 | 环比下降 ≤-5% | 任一核心指标下降≥5%即判定 |

**增长率计算**：
> 增长率 = (本季度平均分 - 上季度平均分) / 上季度平均分 × 100%

**多指标交叉验证**：
- 主指标：月度评估季度平均分变化
- 辅指标1：GMV季度环比（从 `SM-TPL-002` 提取）
- 辅指标2：出勤人数季度环比（从 `SM-TPL-001` 提取）
- 判定逻辑：主指标+任一辅指标同方向变化，方可确认趋势

### 3.3 分级标准

| 象限 | 名称 | 特征 | 管理动作 |
|------|------|------|---------|
| **明星** | 高业绩+正增长 | 持续优秀且在变好 | 扩项优先、年度优秀评选候选、减少巡检频次（月度→季度） |
| **金牛** | 高业绩+零增长 | 稳定优秀但无提升 | 维持当前分量、关注是否进入平台期 |
| **衰退** | 高业绩+负增长 | 曾经优秀但正在下滑 | [P0]启动根因分析、约谈负责人、限制扩项 |
| **潜力** | 中业绩+正增长 | 表现中等但在变好 | 适当扩项、加大赋能支持 |
| **稳健** | 中业绩+零增长 | 中规中矩 | 维持现状、月度巡检 |
| **观察** | 中业绩+负增长 | 正在从中等滑向尾部 | 约谈、要求提交改进计划 |
| **问题** | 低业绩+正增长 | 基础差但正在改善 | 给予改进窗口、关注改善持续性 |
| **停滞** | 低业绩+零增长 | 长期尾部且无变化 | [P1]启动汰换评估、缩减分量 |
| **边缘** | 低业绩+负增长 | 持续恶化 | [P0]启动清退评估 |

### 3.4 决策矩阵

| 九宫格位置 | 分量调整 | 续约建议 | 巡检频次 | 审批级别 |
|-----------|---------|---------|---------|---------|
| 明星 | +10%~+20% | 优先续约 | 季度巡检 | C3 |
| 金牛 | ±0% | 续约 | 季度巡检 | C3 |
| 衰退 | -5%~-10% | 有条件续约 | 月度巡检+专项评估 | C2 |
| 潜力 | +5%~+10% | 续约观察 | 月度巡检 | C3 |
| 稳健 | ±0% | 续约 | 月度巡检 | C3 |
| 观察 | -5% | 暂缓续约 | 双周巡检 | C3 |
| 问题 | ±0%（观察改善） | 有条件续约 | 月度巡检 | C3 |
| 停滞 | -10%~-20% | 不续约 | 双周巡检 | C2 |
| 边缘 | 启动清退 | 不续约 | 每周巡检 | C2 |

### 4.2 第2周：九宫格绘制与分类

#### 动作1：坐标定位

对每家供应商执行以下判定：

```
IF 季度平均分 ≥85 OR 排名百分位 ≤30%:
    X轴 = 高业绩
ELSE IF 季度平均分 ≥70 OR 排名百分位 ≤70%:
    X轴 = 中业绩
ELSE:
    X轴 = 低业绩

IF 增长率 ≥ +5%:
    Y轴 = 正增长
ELSE IF 增长率 ≥ -5%:
    Y轴 = 零增长
ELSE:
    Y轴 = 负增长

九宫格位置 = (X轴, Y轴) 的组合
```

#### 动作2：绘制九宫格

输出格式：3x3表格，每格列出落入该格的供应商名称、季度得分、增长率。

**示例输出**：

| | 负增长 | 零增长 | 正增长 |
|---|---|---|---|
| **高业绩** | 衰退：XX(得分87, -8%) | 金牛：XX(得分91, +2%) | 明星：XX(得分88, +12%) |
| **中业绩** | 观察：XX(得分76, -7%) | 稳健：XX(得分74, +1%) | 潜力：XX(得分72, +9%) |
| **低业绩** | 边缘：XX(得分58, -15%) | 停滞：XX(得分62, 0%) | 问题：XX(得分55, +6%) |

### 4.3 第3周：分析报告撰写

#### 动作1：撰写《供应商分级盘点报告》

报告结构：

> **第一部分：总览**
> - 九宫格全景图（3x3表格+供应商分布）

>
> **第二部分：各象限详解**
> - 每个象限的供应商列表、关键指标、趋势解读
> - 重点关注：衰退象限的根因预判、边缘象限的清退时间表
>
> **第三部分：分量调整建议**
> - 建议扩项的供应商（明星+潜力）、建议缩减的供应商（停滞+边缘）

> **第五部分：风险提示**
> - 集中度风险（某象限过于集中）
> - 替代风险（某业务线只有一家供应商）

**自动判定公式**：
- X轴判定：`=IF(D2>=85,"高业绩",IF(D2>=70,"中业绩","低业绩"))`
- Y轴判定：`=IF(G2>=0.05,"正增长",IF(G2>=-0.05,"零增长","负增长"))`

## 七、风险与预案

### 7.1 预期风险清单

| 风险 | 概率 | 影响 | 触发信号 | 应对措施 |
|------|------|------|---------|---------|
| 历史数据缺失 | 中 | 中 | 某供应商<3个月评估数据 | 用现有月份推算，标注不确定性 |
| 供应商抗议分类结果 | 高 | 低 | 分类结果公布后48小时内收到申诉 | 开放3个工作日申诉窗口，按标准流程处理 |
| 分量调整引发供应商消极 | 中 | 高 | 缩减分量的供应商出勤突然下降 | 约谈+预警，必要时启动替代供应商 |
| 九宫格被当作"永久标签" | 低 | 中 | 连续两期同一供应商位置无变化但实际已改善 | 强调九宫格是动态快照，月度更新 |
| 过度集中某象限 | 中 | 中 | 某象限供应商数>总40% | 调整判定阈值或引入第三维度 |

### 7.2 触发信号

| 信号 | 含义 | 动作 |
|------|------|------|
| 连续2期衰退 | 可能进入清退通道 | 启动健康度诊断（项目2） |
| 某业务线只剩1家供应商在明星/金牛 | 集中度过高 | 启动备用供应商培育 |
| 季度盘点结果与月度排名差异>2个象限 | 数据质量问题 | 立即复核数据源 |

### 7.3 应对措施

**数据争议处理流程**：

```
供应商提出异议
  → 供管岗在1个工作日内回复是否受理
  → 受理：2个工作日内完成复核
  → 不受理：书面说明理由
  → 复核结果为最终结果
```

### 9.3 长期化建议（是否可Loop化）

**强烈建议Loop化**。九宫格盘点不是"一次性项目"，而应是**季度例行机制**。

**Loop化方案**：

| 要素 | 设计 |
|------|------|
| 频率 | 每季度执行一次（3/6/9/12月第1周） |
| 触发方式 | Cron定时提醒 |
| 自动化程度 | 数据采集→计算→分类自动完成，分析报告需人工撰写 |
| 存档路径 | `docs/供应商/运营数据/九宫格盘点/2026Q2/` |
| 与月度评估的关系 | 月度评估是九宫格的"数据燃料"，九宫格是月度评估的"战略视图" |

**首次执行后动作**：
1. 将九宫格模板存入 `docs/供应商/管理规范/`
2. 创建Cron任务：每季度第1周提醒执行
3. 在 `memory/thinking/buffer.md` 中记录首次盘点结果摘要
