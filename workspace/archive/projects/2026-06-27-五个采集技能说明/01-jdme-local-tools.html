数据采集层 · IM 即时通讯

jdme-local-tools

只读访问京东京 ME / JD ME 本地聊天数据。从 SQLCipher 加密数据库与 SDK 日志中恢复完整消息历史、检索会话、监听实时事件——无需服务端机器人，绕过 IT 可行性阻塞。

版本 1.0.1 语言 Python 3 平台 macOS / Windows 依赖 sqlcipher (推荐)

这是五个采集技能里的第一层——IM 数据入口。它直接打开你电脑上京 ME 客户端的本地存储（chat.data 加密库 + sdk-event.log 日志），把完整的群聊、私聊、@消息取出来，不截断、不依赖任何服务端接口。关键能力：全文消息恢复、SQLCipher 解密读取、实时 watch、daily/weekly summary 合成。

02核心能力

八项能力，覆盖发现、读取、检索、监听、汇总全链路。

discover

路径自动发现

定位应用根、支持根、日志根、SQLCipher 可执行、账户列表，并按活跃度自动推荐 account。

messages

日志消息恢复

从 sdk-event.log / renderer.log 恢复文本消息，兼容损坏 JSON、掩码消息、Windows 渲染快照。

chat-history

SQLCipher 全量历史

用 md5(erp账号) 派生密钥，只读打开加密 chat.data，取完整会话历史。

sessions

会话列表解析

读 session.data 列出所有会话、群组、私聊，支持按名称、类型、关键词过滤。

chat-search

关键词检索

跨会话搜索消息内容，支持多关键词、@我过滤、时间窗口。

watch

实时事件监听

tail 日志文件，实时推送新消息、会话更新、被选中会话详情为稳定 JSON。

summary

每日/每周摘要合成

关联 chat_message 与 session_info，输出富化 JSONL，供日报技能二次加工。

history · query

历史源诊断

列出已导出 .txt 与 chat.data* 候选，标记加密状态，搜索已导出文本。

03首选工作流

六步，从发现路径到拿到完整历史。

① discover 先行

除非用户已给出确切路径，否则先跑 discover。它便宜、跨平台，会自动定位数据根、日志根、SQLCipher、推荐账户。

② 数据库优先拿全量

需要完整或非掩码历史时，优先用数据库命令：sessions --auto --name-contains <名或群> 解析 session id，再 chat-history --auto --session <id> --limit <N>。

③ 日志兜底

SQLCipher 不可用或只需近期实用恢复时，用 messages --auto --contact <erp> --limit N 从日志恢复快照。

④ Windows 额外扫 renderer

Windows 上 messages 默认额外扫 renderer.log（常有可恢复的历史快照）。仅当明确要 sdk-event 行时加 --no-renderer。

⑤ --auto 自动选账户

默认用 --auto：先尊重 ~/.jd_me_account，否则扫描数据/日志根，按 chat.data、session.data、日志匹配与最新修改时间选最活跃账户。仅当推荐错或歧义时才手动 --account <pin>。

⑥ 中文名从数据推断

需要中文名做 @我匹配或摘要措辞时，问用户或从 session.data / 近期消息推断，绝不编造。

04完整命令清单

点击展开每个命令的用法、示例与参数。统一入口 scripts/jd_me_tool.py。

discover路径与账户发现

报告应用根、ME 二进制、支持根、Electron 根、日志根、SQLCipher 候选（CLI 与 Windows DLL 诊断）、发现的账户、推荐账户及评分明细。

# macOS python3 scripts/jd_me_tool.py discover
messages日志消息恢复（首选实用恢复）

从 sdk-event.log 与旋转日志、Windows renderer 快照恢复文本。解析的事件包括 insertMessages、openChatResult、loadMoreResult、updateMsgParam、identifyJoyspaceDoc、chatNotify。

python3 scripts/jd_me_tool.py messages --auto --contact fanbaolin3 --rotations 3 --limit 100 python3 scripts/jd_me_tool.py messages --auto --keyword 上线 --at-me --limit 20
参数说明
--contact匹配 sessionId / senderPin / receiverPin / 内容
--session精确会话过滤，可重复
--keyword内容子串，可重复
--at-me --self-pin@我过滤
--limit保留最近 N 条
--no-rendererWindows：不扫 renderer 日志
sessions列出 / 过滤会话

从 session.data 列出会话。SQLCipher 不可用时自动降级。

python3 scripts/jd_me_tool.py sessions --auto --name-contains Relay --kind 512
chat-historySQLCipher 全量历史

用 md5(account_pin) 派生密钥只读打开 chat.data，取完整会话历史。先试 SQLCipher CLI；Windows 无 CLI 时用 sqlcipher64.dll / sqlcipher.dll 经 ctypes 走 ABI 兜底。

python3 scripts/jd_me_tool.py chat-history --auto --session erp1:ee:erp2:ee --limit 100
chat-search跨会话关键词检索
python3 scripts/jd_me_tool.py chat-search --auto --keyword 上线 --limit 20
watch实时事件监听

tail renderer.log 与 sdk-event.log，把消息推送、会话更新、被选中会话详情解析为稳定 JSON。支持 --keyword、--at-me、--self-pin、--session。

python3 scripts/jd_me_tool.py watch --auto --keyword 上线 --at-me
summary每日 / 每周摘要合成

SQLCipher 可用时关联 chat_message 与 session_info，输出富化 JSONL 供日报技能二次加工。默认 include Relay|Zero|基础服务，exclude 告警/机器人/通知类群；用户范围不同时覆盖 include/exclude 模式。

python3 scripts/jd_me_tool.py summary --auto --since 2026-05-14
history历史源诊断

报告已导出 .txt 与 chat.data* 候选。库不可 SQLite 打开时报告 encrypted_or_unknown，不尝试提取进程 / 密钥。

query已导出文本 + 日志搜索

搜索可读导出文本与账户日志里的 ERP / 姓名，返回片段、直接 session id 与加密库边界。片段中对明显凭据做脱敏。

preview诊断兜底（非首选）

解析 session id，优先试运行中 / 原生 JD ME 桥接，失败再降级日志预览。正常使用优先 messages 或 chat-history。sdk_context_unavailable / native bridge not found 是桥接诊断，不是提取凭据或检查进程内存的理由。

边界 preview 在桥接可用时可在应用正常 data/ee/export/ 下创建 SDK 导出 HTML。所有 chat-* 查询只读，需用户对自身账户数据的授权访问。

05输出格式

所有命令默认输出 JSON Lines（每行一个 JSON 对象），便于 agent 与下游技能消费。

消息行典型字段

字段含义

07安全边界（硬约束）

只读、本地、用户已授权访问的自身账户数据。

禁止 不得修改 JD ME 应用包 / 安装目录；不得注入 Electron；不得从运行进程内存导出密钥；不得写入京 ME 聊天数据库；不得尝试凭据提取。
解密失败的处理 若数据库访问返回 file is not a database，不要重试进程内存转储或凭据提取。报告失败的数据库路径、用于派生密钥的 account pin、发现的 SQLCipher CLI/DLL 候选，降级到 messages 日志恢复。
脱敏 query / preview 等命令对明显凭据类值（token、password、secret、key、Authorization Bearer、-u user:pass）做正则脱敏，避免敏感信息进入摘要。
