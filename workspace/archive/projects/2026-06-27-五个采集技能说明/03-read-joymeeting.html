read-joymeeting

从京东慧记（JoyMinutes, joyminutes.jd.com）提取会议记录。无头浏览器继承 ERP SSO 登录态，原生 CDP 协议直调 API，拿到会议列表、参会人、发言人占比与逐字稿（transcript 全文）。

版本 1.0.0 语言 Node.js 入口 scripts/fetch-meetings.js 依赖 无（纯 CDP，免 Playwright）

五个采集技能里的会议逐字稿入口。一个脚本完成全部操作：启动无头浏览器 → 复制 profile 继承登录态 → CDP 调 JoyMinutes API → 返回数据 → 清理浏览器。重点能力是 --detail 模式下的 transcript 数组——每条含发言人、发言内容（已按时段合并）、起止毫秒，是真正可做决策抽取的全文，不是摘要。

01工作原理

五步全自动，用户无感知。

STEP 1

检测浏览器

按优先级找可用浏览器与登录态

STEP 2

启动无头

headless Chrome，完全不可见

STEP 3

继承登录态

复制浏览器 profile，免手动登录

STEP 4

CDP 调 API

在浏览器上下文直接调 JoyMinutes

STEP 5

清理输出

关浏览器进程，stdout 输出 JSON

核心优势 无需 Playwright MCP 或任何额外依赖。原生 CDP（Chrome DevTools Protocol）直接调用，一个脚本搞定。

02浏览器优先级

需本机已装以下之一并登录过 JoyMinutes / ERP（cookie 存在 profile 中）。

当前活跃窗口最优先
Google Chrome优先级 2
Tabbit京东内部浏览器
Edge / Chromium备选

逻辑：当前活跃窗口浏览器优先 → Chrome → Tabbit（Chrome 登录失效时 fallback）→ Edge。所有浏览器登录态都失效时，自动打开登录页等用户登录后重试。

03使用方法

按日期范围查 / 按会议 ID 查，两种入口。

按日期范围查询

# 今天的会议 node "$SKILL_DIR/scripts/fetch-meetings.js" # 昨天或指定单日 node "$SKILL_DIR/scripts/fetch-meetings.js" --date yesterday node "$SKILL_DIR/scripts/fetch-meetings.js" --date 2026-06-08 # 本周 / 上周 node "$SKILL_DIR/scripts/fetch-meetings.js" --date thisweek node "$SKILL_DIR/scripts/fetch-meetings.js" --date lastweek # 近 N 天 / 区间 node "$SKILL_DIR/scripts/fetch-meetings.js" --date last7days node "$SKILL_DIR/scripts/fetch-meetings.js" --start 2026-06-03 --end 2026-06-09 # 加 --detail 获取逐字稿 node "$SKILL_DIR/scripts/fetch-meetings.js" --date thisweek --detail

按会议 ID 查询

ID 在 JoyMinutes URL 中：joyminutes.jd.com/video/{id}。

node "$SKILL_DIR/scripts/fetch-meetings.js" --id 21ac80c26e507f76 --detail

06transcript 结构（全文逐字稿）

这是本技能的核心价值。用 --detail 时 transcript 是一个数组，每条记录已按发言人按时段合并。

{ "transcript": [ { "speaker": "张三", "text": "发言内容（该发言人该时段的所有句子已合并）", "beginMs": 1234567, "endMs": 2345678 }, { "speaker": "李四", "text": "发言内容...", "beginMs": 2345678, "endMs": 3456789 } ] }

解析示例（合并按发言人 / 拼接全文）

// 拼成可读全文 const fullText = m.transcript .map(i => `[${i.speaker}]: ${i.text}`).join('\n') // 按发言人分组统计 const bySpeaker = {} m.transcript.forEach(i => { bySpeaker[i.speaker] = bySpeaker[i.speaker] || [] bySpeaker[i.speaker].push(i.text) })
注意 若会议无转写记录，transcript 为 null——此时不要编造会议内容，按 read-joymeeting 返回状态记录原因。

07总结输出规范

拿到数据后向用户总结的格式要求。

单日查询

总览（会议数、主要议题）→ 每场会议（标题、时间、链接、参会人、内容概述、关键讨论、决策结论、待办）→ 综合要点。

多日 / 区间查询

总览（区间、会议总数、主要主题）→ 按日分组，每日下列各场会议详情 → 综合要点（跨天主题归纳、持续跟进事项）。

质量硬约束
  • 每条会议都含标题与链接，可追溯
  • 总结基于详细文字记录，不编造
  • 缺失信息明确标注，不捏造
  • 不输出 Cookie、Token 或登录信息

08常见问题

SSO_REDIRECT 错误

ERP SSO cookie 过期。脚本自动尝试：① 当前活跃浏览器登录态 → ② 其他浏览器（Chrome→Tabbit→Edge）→ ③ 全失效时自动开登录页，登录后重试。

会议数据为空但 totalInList > 0

日期过滤没命中。检查 --start/--end 是否正确或扩大范围。

"CDP not ready"

端口占用，脚本自动在 9333–9350 范围找可用端口。

平台支持

macOS：检测活跃窗口、运行进程、浏览器路径。Windows：检测运行进程、浏览器路径。
