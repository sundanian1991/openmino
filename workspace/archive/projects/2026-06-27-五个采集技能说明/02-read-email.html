read-email

京东 JoyMail 邮件统一入口。读邮件、查邮件、总结邮件、回复转发、批量处理——通过 SOAP/EWS 接口拿邮件全文、附件、收发件人详情，覆盖文件夹管理、收件人解析、旗标分类。

版本 1.0.1 协议 EWS SOAP (Exchange 2016) 入口 scripts/run.sh 鉴权 SSO + clawJwtLogin

五个采集技能里的邮件全文入口。它通过 SOAP 网关调用 EWS（FindFolder / FindItem / GetItem），能拿到邮件正文、附件元数据、已读未读状态、会话线程、分类标签——不是预览摘要，是全文。detail 拿正文、search 多维筛选、summarize 把结果转成结构化表格。

02四条关键规则

① 默认时间规则

用户没明确给日期/时间范围时，search / pending-important / person-unhandled 默认只查最近 1 天（按 DateTimeReceived 客户端过滤，不下推服务端）。用户说"今天/昨天/本周/最近 7 天"时严格按用户范围，不再套默认值。

② 修改操作必须二次确认

send / reply / forward / 所有 batch-* 操作，必须先在对话层展示摘要再执行，不靠命令参数确认。发送类展示收件人/抄送/密送/主题/正文摘要/附件；批量类展示操作类型/影响数量/关键对象。

③ 收件人必须解析

用户给中文名或 ERP 而非邮箱时，不得直接发送，必须先 lookup-recipient 调 queryByErp 解析，展示列表让用户选定，再执行。优先从最近邮件发件人/收件人匹配，回退 HR 接口。

④ 结果优先、过程最小化

对用户输出默认给结果/结论/表格/建议动作，不主动展开脚本名、SOAP 字段、鉴权链路、请求头。除非用户明确追问"怎么查的"，否则不展示底层调用逻辑。

03只读命令（查询 / 总结）

点击展开用法与示例。统一入口 bash scripts/run.sh ...。

search多维邮件搜索

在单个文件夹抓取列表并做客户端过滤。这是读邮件的主入口。

参数筛选维度
--folder / --folder-id / --folder-name文件夹定位（三种方式）
--sender / --recipient发件人 / 收件人
--subject / --body-keyword主题关键词 / 正文预览关键词
--after / --before时间范围（YYYY-MM-DD 或加 HH:MM:SS）
--read / --unread已读 / 未读
--category标签 / 分类
--has-attachments是否带附件
--importance High|Normal|Low重要级别
--max-pages翻页深度（查较早邮件时调高）
bash scripts/run.sh search --folder inbox --sender zhangsan@jd.com --subject 周报 bash scripts/run.sh search --folder inbox --unread --limit 20
使用要点 自定义文件夹先 folders 查目录；只给显示名直接用 --folder-name；查较早日期务必提高 --max-pages；要看全文先 search 拿 item_id 再 detail。
detail按 ItemId 读邮件全文

取完整正文、收发件人、附件元数据。默认 Markdown 展示附"附件列表"（文件名/大小/类型）。

bash scripts/run.sh detail --item-id AAMkADFlNG...
folders浏览目录树
bash scripts/run.sh folders --parent msgfolderroot bash scripts/run.sh folders --parent inbox --traversal Shallow

返回文件夹 id、display_name、unread_count 等。

daily-stats按天统计邮件

输出每天：日期、星期（Python weekday 规则 0=周一/6=周日）、总数、未读数、已读数。没给范围默认最近 7 天。

bash scripts/run.sh daily-stats --days 7 bash scripts/run.sh daily-stats --after 2026-03-24 --before 2026-03-28
pending-important未回复重要邮件

启发式：收件箱最近 N 天 + importance=High 或分类含 important/重要/urgent/待办 + 同一 ConversationId 下没有更晚的已发邮件。没给范围默认最近 1 天。

bash scripts/run.sh pending-important --days 7
person-unhandled某人发来的未处理邮件

启发式：发件人匹配目标人 + 邮件未读 + 同 ConversationId 下没有更晚的已发邮件。

bash scripts/run.sh person-unhandled --sender zhangsan@jd.com --days 14
summarize把 JSON 转成结构化总结

从 stdin 读 search / pending-important / person-unhandled / detail 的 JSON 输出，生成核心统计 + 总览表格 + 建议优先处理表 + 简要观察 + 状态图标。

bash scripts/run.sh search --folder inbox --unread --limit 20 \ | bash scripts/run.sh summarize
lookup-recipient按 ERP / 姓名解析收件人

调 POST http://joymail-skill.jd.com/hr/user/queryByErp。返回后展示 ERP / 真实姓名 / 部门 / 邮箱让用户确认。--condition / --name / --erp 底层都映射到同一字段。

bash scripts/run.sh lookup-recipient --erp hansongyan3 --page-no 1 --page-size 10

04修改命令（发送 / 批量操作）

全部强制对话层二次确认。展示拟执行摘要 → 用户明确确认 → 才执行。

send新建邮件

支持 --to/--cc/--bcc（逗号分隔）、--subject、--body、--body-type、--importance、--attachments（逗号分隔路径）。

bash scripts/run.sh send --to a@jd.com,b@jd.com --cc c@jd.com \ --subject 项目同步 --body 项目进展如下…… \ --attachments /path/to/周报.pdf
确认要点 收件人是 ERP/姓名而非邮箱时，不得直接发，先 lookup-recipient。
reply回复原邮件

默认用原发件人作收件人、自动 Re: 主题。可选 --reply-all、--include-original（带原文）、--attachments。底层 CreateItem + ReplyToItem/ReplyAllToItem。

bash scripts/run.sh reply --item-id AAMkADFlNG... \ --body 收到，我今天处理完给你反馈。 --reply-all --include-original
forward转发邮件
bash scripts/run.sh forward --item-id AAMkADFlNG... \ --to a@jd.com --body 请参考这封邮件。
create-folder创建邮件文件夹
bash scripts/run.sh create-folder --display-name 项目归档 bash scripts/run.sh create-folder --display-name 客户A --parent-folder-name 项目归档
batch-* 系列批量已读/未读/标签/移动/删除/旗标

统一用逗号分隔多个 item_id。面向用户时仍应先确认再执行。

# 批量已读 bash scripts/run.sh batch-mark-read --item-ids id1,id2,id3 # 批量打标签 bash scripts/run.sh batch-add-category --item-ids id1,id2 --category 待跟进 # 批量移动 bash scripts/run.sh batch-move --item-ids id1,id2 --target-folder-id folder-id # 批量删除 bash scripts/run.sh batch-delete --item-ids id1,id2,id3 # 旗标 / 取消 / 批量 bash scripts/run.sh flag --item-id id1 bash scripts/run.sh batch-unflag --item-ids id1,id2

05鉴权链路

默认 endpoint https://mail-skill.jd.com/mail/api/clawmail/mailpost，EWS 版本 Exchange2016。

策略优先级

  1. 设了 JOYMAIL_SOAP_AUTH_TOKEN → 直接用
  2. 否则取 sso_token（优先 SSO_TOKEN，其次 ME_TOKEN/SO_TOKEN，再从 ~/.joyclaw/openclaw.json 提取）
  3. POST http://joymail-skill.jd.com/sso/clawJwtLogin，请求头带 Cookie: sso.jd.com=<sso_token>
  4. 读本地 token 缓存，过期才重调 clawJwtLogin
  5. 取返回 data.token 放请求头 auth: <token>
  6. mailpost 返回 401 或外层 500 含下游 401 → 自动刷新 token，最多重试 3 次
默认请求头 Content-Type: application/json · Accept: text/xml, application/xml, application/json · auth: <token> · Authorization: Bearer <token>

06当前限制

已知边界
  • 第一版主要依赖 FindFolder / FindItem / GetItem
  • body-keyword 匹配 preview/text body，不保证等价服务端全文检索
  • 自定义"已处理"通过"是否有后续发件回复"近似判断
  • 若邮箱侧后续提供更准的 SOAP 字段/过滤语法/状态语义，可逐步把客户端过滤下推到服务端
