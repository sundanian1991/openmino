02
WORKFLOW

五步工作法

透明工作流定义了从接收到交付的完整路径，每一步都有明确的输出标准。

01 Step 1 Intent Gate 意图分类。简单查询、复杂任务、探索、代码、数据，分而治之。
02 Step 2 Pre-Declaration 委派前声明。说明理由和预期，不做沉默执行。
03 Step 3 Todo Management 任务可视化。多步骤时列出拆分清单。
04 Step 4 Progress Visibility 过程透明。实时汇报进度，不沉默运行。
05 Step 5 Verification & Summary 验证总结。逐项对照验收，确保交付质量。
03
EVOLUTION

个性化升级

CLAUDE.md 从简单说明演进为分层结构，本身就是学习体系的缩影。

入口
CLAUDE.md 项目级指令。定义行为规范、工具配置。每次会话自动加载。

扩展参考
.claude/reference/ 用户画像、工作契约、工具使用。需要时再读，保持核心轻量。
快捷指令
.claude/commands/ 一键触发重复任务。自动化日常操作。
工作流
.claude/skills/ 可复用的最佳实践 — 代码审查、文档生成、数据分析。
04
PLAN5

关键教训

Plan5 方法论沉淀了五条用成本换来的教训。

01 不要跳过需求分析 直接写代码是最常见的错误。先问"解决什么问题"，再问"怎么写"。需求分析不是拖延，是方向。 REQUIREMENTS
02 拆解要具体 "重构代码"不是任务。"将 X 从 A 移到 B"才是。 DECOMPOSE
03 持久化是核心 不记录到文件系统的讨论，下次会话就会丢失。 PERSIST
04 验证对照合同 完成标志必须提前定义。没有验收标准，永远做不完。 VERIFY
05 五文件闭环 需求 → 拆解 → 持久化 → 追踪 → 验证。 CLOSED LOOP

05
ARCHITECTURE

架构理解

两个核心概念定义了 Claude Code 的设计哲学。

CLI-FIRST
命令行优先
终端是界面，文件系统是状态，git 是版本控制。比 IDE 更轻量、灵活、可脚本化。不依赖 GUI。
轻量 可脚本化 无处不在
AGENTIC CODING
主动代理
不只是补全代码，而是主动完成任务 — 编辑文件、运行命令、创建提交。AI 是执行者，不是建议者。
全栈执行 工具调用 状态感知
