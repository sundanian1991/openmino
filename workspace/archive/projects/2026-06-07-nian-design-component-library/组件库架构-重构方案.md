# nian-design 组件库架构（重构方案）

> 三层：A 内容表达层（55 核心）→ B 操作交互层（8 个）→ C 扩展库（其余）

---

## A 内容表达层 · 55 核心

按 9 种认知操作分组。每个组件有：
- 表达宣言（它说什么、别的替代不了）
- 优先级（第1优先 → 第2优先 → 备用）
- 适用的场景（SOP/数据报告/品牌展示等）

## B 操作交互层 · 8 个

> 不走认知操作判断，走「用户需要做什么操作」。

| 组件 | 用户操作 | 何时用 |
|------|---------|-------|
| Inputs | 输入文字/数据 | 页面需要用户填信息 |
| Toggle/Switch | 切换开/关状态 | 页面需要用户切换模式 |
| Segmented Control | 在有限选项中选择 | 页面需要用户选视图/分类 |
| Buttons | 触发动作（提交/保存/跳转） | 页面需要用户执行操作 |
| Quick Edit | 在页面上直接编辑内容 | 交付前微调阶段 |
| Slider | 在范围内调整数值 | 数据过滤/参数调整 |
| Date Navigation | 切换时间周期 | 数据报表需要切换日期 |
| Tag（交互态） | 筛选/标记 | 内容列表需要筛选项 |

**选择逻辑**：Step B 确定认知操作后，检查页面是否需要用户操作。需要 → 从操作层选。不需要 → 跳过。80% 的内容页面（报告/SOP/笔记）不需要操作层组件。

## C 扩展库（约 45 个）

> 不进入主决策流程。保留但不主动推荐。AI 只有在以下情况才查：
> 1. 核心 55 个无法满足特殊需求
> 2. 年老师明确指定要某个特定组件

包括：装饰类（C16 Dot Pattern、DV1-DV3...）、过度具体场景类、被核心组件覆盖的备选变体、未清理的旧有组件。

---

## 核心原则

- **同一种认知操作下，第1优先组件锁死**。除非年老师明确要求换，否则不开放选择。
- **交互组件不在认知操作里判断**。它们是「用户需要做什么操作」，不是「内容想表达什么」。
- **扩展库不存在于 AI 的默认路径**。不读、不查、不考虑。
