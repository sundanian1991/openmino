# 内容意图 → 组件映射表（草稿）

> 第一层：内容意图（12种）
> 第二层：组件（每个意图下 1-3 个，锁死，不开放多族选择）

---

### 1. 排名 / 排序
**意图**：展示谁前谁后、差距多少、TOP-N

| 内容结构 | 组件 | 为什么是它 |
|---------|------|-----------|
| 有序号 + 名称 + 数值 | **rank-bar** | 条状长度直观展示差距，序号强调排名 |
| 例：「1.毅航 92 2.毛毛虫 87 3.伽玛 85」 | | |

### 2. 比例 / 构成
**意图**：展示各部分占总量的多少、份额分布

| 内容结构 | 组件 | 为什么是它 |
|---------|------|-----------|
| 名称 + 百分比/数值 | **seg-bar** | 分段条，每段代表一个子项的占比 |
| 例：「金条42% 企金28% 信用卡18%」 | | |
| 多层级百分比嵌套 | stacked-bar（备用） | 多层堆叠，复杂构成时使用 |

### 3. 对比 / 选择
**意图**：左 vs 右、方案选哪个、前后对比

| 内容结构 | 组件 | 为什么是它 |
|---------|------|-----------|
| 左右两栏 + 对应维度 | **comparison** | 天然双栏，同一维度左右对照 |
| 例：「方案A 功能85 成本70 vs 方案B 功能75 成本88」 | | |
| 四个象限、两组张力 | **tension-grid** | 2×2 网格，适合品牌核心张力展示 |

### 4. 趋势 / 时间序列
**意图**：随时间变化、上升下降、增长率

| 内容结构 | 组件 | 为什么是它 |
|---------|------|-----------|
| 时间 + 数值序列 | **sparkline / line-chart** | 折线天然表达趋势 |
| 例：「1月85 2月88 3月92」 | | |

### 5. 流程 / 步骤
**意图**：先做A、再做B、后做C、步骤序列

| 内容结构 | 组件 | 为什么是它 |
|---------|------|-----------|
| 步骤编号 + 阶段名 + 说明 | **flow-pipeline** | 横向箭头连接，线性流向 |
| 例：「Step1准入 → Step2分工 → Step3运营」 | | |
| 时间轴、大事记 | **timeline-track** | 纵向节点 + 连接线 |

### 6. 并列展示 / 卡片
**意图**：多个等重概念并列展示、能力项、原则

| 内容结构 | 组件 | 为什么是它 |
|---------|------|-----------|
| 3-4 个并列标题 + 描述 | **card-triad / card-quad** | 等大等高的卡片阵列 |
| 例：「原则一：克制 / 原则二：锐度 / 原则三：诚实」 | | |
| 不同大小、不同权重的卡片 | **bento-grid** | 不规则网格，有主次关系 |

### 7. 关键指标 / KPI
**意图**：展示几个关键数字、业绩达成、核心指标

| 内容结构 | 组件 | 为什么是它 |
|---------|------|-----------|
| 多个（名称 + 数值）并列 | **numeral-grid** | 大数字 + 小标签，一目了然 |
| 例：「供应商35家 环比+8.3% 合规率98.3%」 | | |
| 单个指标 + 进度 | **metric-card** | 带进度条的单指标卡 |

### 8. 明细 / 参数
**意图**：查看详细数据、多维度参数、精确值

| 内容结构 | 组件 | 为什么是它 |
|---------|------|-----------|
| 多行多列表格 | **data-table** | 行列式，精确查找 |
| 标签 + 值对 | **spec-table** | 左标签右值，适合规格参数 |

### 9. 引用 / 提示
**意图**：引用一句话、提示注意、方法论说明

| 内容结构 | 组件 | 为什么是它 |
|---------|------|-----------|
| 一句话 + 出处 | **callout** | 左边框，安静地提示 |
| 长引用 + 作者 | **pull-quote** | 大字号引文，占据视觉焦点 |

### 10. 品牌宣言 / Hero
**意图**：开门见山、品牌声明、封面

| 内容结构 | 组件 | 为什么是它 |
|---------|------|-----------|
| 标题 + 副标题 + 数据锚点 | **V1-V6（6种切割）** | 按决策路径选，不在意图层选 |

### 11. 检查 / 清单
**意图**：勾选、确认、逐项核对

| 内容结构 | 组件 | 为什么是它 |
|---------|------|-----------|
| 多行（状态 + 事项） | **checklist** | 带状态标记的逐行列表 |

### 12. 评分 / 评价
**意图**：打分、评级、多维度评估

| 内容结构 | 组件 | 为什么是它 |
|---------|------|-----------|
| 维度 + 分值 + 对比 | **comparison（评分变体）** | comparison 的评分模式 |
| 多维度雷达 | **radar-chart** | 雷达图，综合性评价 |

---

**核心原则：同一种内容结构不会有超过 2 个可选组件。**
如果有 → 说明这两个组件中有一个表达领地重叠，需要合并或删除。
