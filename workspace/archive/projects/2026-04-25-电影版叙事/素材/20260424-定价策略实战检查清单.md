# 定价策略实战检查清单

> 2026-04-24 | 从 AI 定价模式演变中提炼 | 可复用于任何订阅/资源型产品定价

---

## 新产品定价决策清单

- [ ] **定义计量单位**：用什么计费？（Token/请求/次数/credit/时间）
  - 模糊计量（Prompt、请求次数）→ 早期获客友好
  - 透明计量（Token、时间）→ 长期信任友好
  - 选错计量单位 = 选错定价权归属

- [ ] **选择获客策略**：补贴跑量 or 差异化高价？
  - 补贴 → 快但不可持续，必须有收紧计划
  - 差异化 → 慢但健康，需要真正的能力壁垒

- [ ] **设计分层结构**：几档？价差多少？
  - AI 行业经验：3-4 档，入门档 ~$20-50，Pro 档 3-5 倍
  - 档位太多用户选择困难，太少覆盖不了不同支付意愿

- [ ] **确定透明度**：额度描述要多透明？
  - "每 5 小时 N 次" → 用户能预估，但不精确
  - "N 万 Token" → 精确但用户难预估自己消耗
  - "倍数描述"（5x、20x）→ 最模糊，利润空间最大

- [ ] **规划演进路径**：补贴 → 收紧 → 分层
  - 补贴期多久？收紧节奏是什么？分层依据是什么？
  - 不要在补贴期就设计好终态定价，会过度保守

---

## 商业模式转型检查点

- [ ] **用户成本可预测性**：用户能不能算清楚自己要花多少？
  - 算不清楚 → 用户焦虑 → 流失
  - Cursor 事件：突然改变计费 → 退款风波

- [ ] **价格弹性测试**：收紧定价后用户留存如何？
  - 没有弹性测试就直接分层 = 拍脑袋定价

- [ ] **生态粘性**：从单一场景到多场景的升级是否自然？
  - MiniMax：编程 → 全模态 → 生态锁定
  - 不自然的捆绑 = 用户反感

- [ ] **竞争护城河**：定价结构是否容易被复制？
  - 计量单位可以抄，但生态（模型 + 工具 + 社区）难抄

---

## 反模式（别踩的坑）

- **突然切换计费模式**：Cursor 2025-07 事件 → 必须渐进迁移，提前通知
- **计量单位混乱**：90% 用户踩坑 → 信息不对称短期有利，长期伤害信任
- **饥饿营销常态化**：每天 10:30 抢购 → 短期有效，长期伤品牌
- **标价与实际脱节**：订阅费是地板不是天花板 → 实际 2-5 倍 → 用户被骗感
- **国内海外价差过大**：同产品两套价格 → 用户发现后信任崩塌

---

## 决策口诀

> **计量即权力，补贴即投资，收紧即筛选，分层即利润。**
>
> 入口获客，生态锁客。先跑量再算账，别在第一天就想好终价。
