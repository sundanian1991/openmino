终幕 · 决策训练
如果你是 CEO
你会怎么做？
2025 年底。你是 MiniMax 的 CEO。OpenClaw 突然爆火，GitHub 355K Stars。你的 Token 消耗量 6 个月内增长了 6 倍，成本线开始撑不住。Coding Plan 的"不限量"承诺面临挑战。
你会选哪条路？
CHOICE A
维持不限量，硬扛成本
相信规模效应会稀释成本，继续用不限量吸引用户。短期亏损换长期壁垒。
实际结果： Anthropic 选了这条路，2026 年 4 月从标准订阅移除 OpenClaw 访问——成本撑不住了。不限量在 Agent 时代是不可持续的承诺。
CHOICE B
收紧套餐，限时限量
从"不限量"切换到时间窗口配额，比如每 5 小时限额 + 每周上限。控制成本但不改变订阅模式。
实际结果： 智谱 GLM 和 Claude Code 选了这条路——5 小时窗口 + 每月上限。有效控制了成本，但用户开始算账，比价心理被激活。
CHOICE C
转 Token 精细计量
直接把 Coding Plan 升级为 Token Plan。按实际 Token 消耗计费，仅统计未命中缓存的 Token。
实际结果： MiniMax 选了这条路。2026 年 3 月正式更名 Token Plan，同时打包全模态能力——从"编程工具"升级为"AI 多媒体工作室"。这是主动转型，掌握了节奏。
谁先转谁主动，谁后转谁被动。
Cursor 2025-06 的教训：突然改变计费 = 退款风波。
