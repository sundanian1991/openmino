HALLMARK
不做好看的。
做想清楚的。

每一个 AI 都能调出好看的颜色。Hallmark 不关心这个。它关心的是——这个页面的结构是不是一个人想清楚了才动手做的，还是模型把默认值堆上去了。

01 — 问题

AI 生成的页面有一种共同的气味。

居中大标题。渐变按钮。三栏等宽卡片。玻璃拟态。来回跳的悬停动画。
你不说，它就往那里跑。因为训练数据里最多的就是这些。

问题不是丑。问题是千篇一律。两个完全不同brief的页面，长同一个骨架——只是换了个颜色。

02 — 拒绝清单

Hallmark 拒绝接受AI 的默认值。

60+ 条检测门禁，每一条都是一个 AI 味的视觉指纹。以下是几条最致命的：

THE PURPLE-GRADIENT HERO
紫蓝渐变背景 + 白色居中文字。这是被识别度最高的 AI 美学。没有之一。
INTER-EVERYWHERE
Inter 做标题也做正文，没有配对字体。单字体页面就是模板页面。
3-COLUMN FEATURE GRID
三等宽栏，图标在上，标题在中，正文在下。每个 LLM 都会输出这个。
THE GRADIENT HEADLINE
标题用 background-clip: text 填渐变色。比什么都快地暴露"AI 生成"。
CARD-IN-CARD
卡片里面套卡片，嵌套的容器没有语义理由。选一层，删另一层。
DEFAULT-ATTRACTOR SAMENESS
同一项目连续两个页面用同一个骨架。换颜色不等于换设计。
03 — 六条纪律

不是建议。
是纪律。

01
诚实文案
没数据就不编数据。"+47% 转化率"和"50,000+ 团队在用"一旦是编造的，就是 slop。用真数字，或留占位符。
02
锁定 Token
所有颜色、字体必须走 CSS 变量。渲染中途随手写一个 #ff6b35 是被禁止的。值不存在就先建 token 再引用。
03
不画假 Chrome
不造假的浏览器栏、手机壳、IDE 窗口。你的环境已经有真实的 chrome。用真截图，或让内容自己说话。
04
移动端必须过关
320 / 375 / 414 / 768 四个宽度全部验证过。没有水平滚动，没有两行可点击文字，图片网格用 minmax。
05
发前自评
哲学、层次、执行、具体性、克制、多样性——六轴打分 1–5。低于 3 就重做。分数印在文件顶部。
06
结构多样性
同一项目连续两次输出不能共享同一个宏结构。不是颜色变体，是骨架变体。两个 brief 应该看起来像两个网站。
04 — 四种风格谱系

Genre 不是滤镜。
是声音。

每种 Genre 有自己的字体纪律、配色边界、排版节奏、禁止清单。选了 Genre，就选了一整套审美契约。

Editorial
报纸 / 杂志感。衬线体展示，不对称排版，发丝线分隔。品牌故事、杂志、个人站点。
Modern-minimal
Stripe / Linear 学派。Geist 无衬线，纯白纸，药丸按钮，零色度灰。SaaS、API、开发者工具。
Atmospheric
环境氛围，沉浸式，大图驱动。旅游、生活方式、高端品牌。页面先说"看"，再说"读"。
Playful
色彩丰富，圆角，动效活泼。消费者 App、教育、玩具。但 Playful ≠ 随意——它有自己的克制。

05 — 怎么用

四个动词。
零配置。

1
/hallmark
默认动词。从 brief 开始，完整设计并输出 HTML。Genre、Theme、Macrostructure 自动选择。
/hallmark 做一个供应商管理系统的仪表盘
2
hallmark audit
只打分不动手。读取目标文件，返回六轴评分 + 问题清单。不编辑任何东西。
hallmark audit dashboard.html
3
hallmark redesign
保留内容和意图，重新设计视觉层。新的骨架、新的节奏、新的组件声音。
hallmark redesign landing-page.html
4
hallmark study
从你喜欢的设计中提取 DNA——宏观结构、字体搭配、配色锚点——生成诊断报告。
hallmark study screenshot.png
