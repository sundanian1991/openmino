## 二、阶段详细操作指引

### 2.1 ① 需求调研（培训前 3-4 周）

**核心目标**：精准识别「哪些痛点值得花一期培训讲」。不是「收集问题」，是「找到该讲什么」。

**操作步骤**：
1. **开放议题池**：列出可分享主题（数据策略/话术产能/企微运营/人员激励/招聘/薪酬/培训/管理），由供应商自主选择擅长的议题
2. **发放事前预习框架（供应商版）**：4 模块引导问题——数据策略与名单管理 / 话术结构与产能突破 / 企微客户运营 / 人员激励与团队管理——每个模块设 3 个引导问题，让供应商带着问题来听
3. **收集供应商「最想了解的问题」**：每供应商至少提 3 个具体问题（不是抽象需求），这些问题就是培训当天的「导航仪」
4. **问题分类确认优先级**：高频深痛主题 → 对应元主题 → 候选讲师方向。运营官拍板最终主题

**产出物**：培训主题确认单（含主题 / 讲师方向 / 时间 / 地点 / 覆盖人数）。

### 2.2 ② 讲师锁定（培训前 2-3 周）

**核心目标**：锁定主讲方，拿到讲师素材。不做内容预判——讲师没给的不补，不编造数据。

**操作步骤**：
1. **联系目标供应商**：确认讲师意愿和时间。原则：分享方毫无保留分享实战经验，内容全部贴近一线实操
2. **拿到讲师素材**：口述转写 / 微信草稿 / 现有 PPT / 笔记 / 录音——任何形式的素材
3. **初次梳理**：结构化整理素材，产出内容结构初稿（不是最终版，是梳理方向）
4. **确认课程框架**：和讲师对接确认课程脉络，提前打磨内容

**产出物**：讲师素材清单（含素材类型 / 核心模块 / 缺失内容 / 需要补充的细节 / 需要准备的案例）。

### 2.3 ③ 备课梳理（培训前 1-2 周）

**核心目标**：把讲师零散的口述/草稿/材料，结构化为完整的培训大纲。解决备课最耗时的「多轮打磨」——AI 完成初版结构化，运营官只需微调。

**操作步骤**：
1. **内容结构化**：按「5页模板」梳理——①为什么做（背景/痛点）②怎么做（方法论/框架）③做什么（具体动作/工具）④做多久（时间/节奏）⑤做完后（检查/追踪/迭代）
2. **提炼金句和记忆点**：每个模块至少提炼 1 条可传播的金句，用于宣传和记忆
3. **内审**：运营官确认内容方向——不做人选判断，不编造数据，不替代人际沟通
4. **预演**：讲师自行预演，确认时间节奏（60 分钟含讲解+讨论）

**产出物**：PPT 终稿（含讲师确认）；课程大纲确认单。

### 2.4 ④ 会前材料（培训前 1 周）

**核心目标**：批量生成会前/会中/会后的全套标准化材料。直接调用知识库模板填充，输出 docx（打印）+ md（编辑）。不发送通知（运营官发），不做视觉设计（交可视化）。

**操作步骤**：
1. **生成事前预习框架（供应商版）**：4 模块引导问题，每个模块 3 个引导问题，每个问题配 3 个「先想想」的勾子——让供应商对照自己团队现状写下一个具体问题
2. **生成事中对照记录表**：8 个模块（数据策略×3 / 话术产能×2 / 企微运营×2 / 人员激励×1）的逐项对照——「主讲方要点→我团队现状→可直接抄的动作→我的疑问」四栏。现场每听完一个模块立刻填
3. **生成现场 Q&A 速记表**：编号→问题→主讲方回答要点→是否解决——有疑问当场记，会后答疑或私下交流
4. **发放所有材料**：培训前 3 天发送给所有参会供应商
5. **收集回执**：每供应商至少提交 3 个最想了解的问题（不是抽象需求）

**产出物**：事前预习框架（已填）；问题清单；事中对照记录表（空白）；现场 Q&A 速记表（空白）。

### 2.5 ⑤ 培训执行（培训当日）

**核心目标**：现场全流程管理——从开场到收尾，每个模块的节奏、内容、互动、记录都有标准流程。

**操作步骤**：
1. **开场欢迎（10分钟）**：主办方代表欢迎供应商到场，说明本次培训背景和选址原因，明确会议目标
2. **主持人介绍（5分钟）**：说明本次培训的筹备过程、分享方、内容模块，提醒供应商使用事中对照记录表
3. **模块分享（每模块60分钟）**：主讲方分享→供应商现场对照记录→每模块结束留 5 分钟答疑
4. **现场 Q&A（30分钟）**：集中答疑——每个供应商至少提 3 个问题，现场解答
5. **收尾总结（15分钟）**：主办方总结，肯定分享方，布置后续任务（7天内提交行动回收表，30天+90天双节点跟踪）

**产出物**：事中对照记录表（现场填）；Q&A 速记；现场录音/转写。

### 2.6 ⑥ 事后回收（培训后 7 天）

**核心目标**：推动供应商从「学到」到「做到」——不是「作业」，是「落地契约」。

**操作步骤**：
1. **发放事后行动回收表**：5 部分——①回顾（从对照表选出 1-2 个可抄动作）②30 天计划（谁做/场景/方式/试多久/成功标准/最大阻碍）③落地所需支持（多选）④意外发现⑤反馈给主办方（内容覆盖度/下一场建议）
2. **收集回执**：培训后 7 天内收回
3. **统计落地计划**：汇总所有供应商的落地动作、阻碍、所需支持
4. **识别共性问题**：如果多个供应商都提了同一个阻碍（如名单筛选条件/CRM字段），作为联盟层面统一推动解决

**产出物**：事后行动回收表（已填）；落地计划清单；共性问题清单。

### 2.7 ⑦ 30 天回访

**核心目标**：强制节奏锚点——解决「没节奏被动带走」的问题。T+30 回访不是「来检查你」，是「来看看你还需要什么支持」。

**操作步骤**：
1. **回访时间**：培训后 30 天
2. **回访三问**：①你选的那个动作，落了吗？（落了的→效果如何？没落的→卡在哪？）②这 30 天你们团队有什么变化？（数据/行为/心态）③还需要联盟给你补什么支持？
3. **回访结果分类**：绿灯（落地了，效果好，继续推）→ 黄灯（部分落地，效果一般，需要补充支持）→ 红灯（完全没落地，卡在某个障碍，需要联盟层面干预）
4. **补充支持**：根据回访结果，针对性提供二次交流/工具支持/名单调整/激励配合

**产出物**：回访记录（含三问 + 结果分类 + 补充支持清单）。

### 2.8 ⑧ 90 天效果复盘

**核心目标**：评估培训效果，为下一期培训做准备。确保培训不白听。

**操作步骤**：
1. **数据对比**：对比培训前后各供应商核心指标（名产 / 加微率 / 授信成功率 / 提前还款率 / 人均业绩），识别培训是否带来实际效果
2. **收集反馈**：培训综合评价表（含内容覆盖度/讲师评分/下一场建议）
3. **讲师评价**：基于现场表现和内容质量，对讲师打分排名。表现优秀讲师可获得京东单独奖励
4. **下一期改进**：基于回收结果和讲师评价，确定下一期培训主题、讲师、改进方向

**产出物**：效果评估报告；讲师评分卡；下一期培训计划。

## 四、联盟培训标准与机制

### 4.1 分享频率

- **线上**：每月一次（60分钟讲解+讨论）
- **线下**：每三个月一次（全天，含培训+参观+聚餐）
- **覆盖**：京东合作供应商所有管理及独挡一面以上员工
- **议题**：开放由供应商自主选择擅长议题

### 4.2 内容标准

- 分享方毫无保留分享实战经验，内容全部贴近一线实操
- 提前大量时间对接梳理课程脉络
- 分享者自身也能梳理沉淀经验获得成长

### 4.3 讲师标准

- **S 级（联盟标杆讲师）**：内容完整度高 + 实战深度高 + 现场感染力强 + 金句产出多——建议每期培训做开场锚定
- **A 级（优秀）**：内容扎实，可继续邀请
- **B 级（良好）**：有提升空间，建议补充深度

### 4.4 效果验收

- 每个供应商回去后至少落地一个学到的点
- 双节点跟踪：30天中期检查 + 90天复盘会，确保培训不白听
- 最终实现京东/供应商/员工与客户三赢

### 4.5 联盟核心原则

**前期试点（金条业务已成功举办两期）→ 持续迭代（效果一期比一期好）→ 经验沉淀（培训前提前大量时间对接课程）→ 机制固化（每月线上+每季度线下+讲师库+评分卡+效果评估）**

核心价值：帮助供应商沉淀可复制的方法论，具备自主造血能力，避免随意腾挪成熟人接新项目，同时保障现有业务交付质量。分享者自身也能梳理沉淀经验获得成长，被行业同层级认可，锻炼管理队伍。最终输出行业认可的供应商能力标准。
