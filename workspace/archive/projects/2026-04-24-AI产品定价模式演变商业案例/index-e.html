序幕 — 一个数字的谎言
10,000,000,000
$15,000+

一个开发者 8 个月消耗的 Token 数，和它对应的 API 标价。
但他只付了 ¥49/月。

这不是滥用，是行业标准

¥49 订阅看起来便宜。但一个重度开发者实际消耗的算力，远超这个价格能覆盖的范围。Coding Agent 自动循环调用模型，每次提问触发 5-30 次 API 请求。8 个月累积下来，就是 100 亿。

更令人不安的是——其中 70% 是浪费。文件读取占 35-45%，代码生成只有 5-15%。你在为 AI 读文件付费，不是写代码。

  • 8 个月消耗100 亿 Token
  • API 标价$15,000+
  • 实际订阅费¥2,744 (¥49×56)
  • 节省比例93%
  • 浪费比例70%

这个数字告诉我们两件事：
第一，订阅制让 AI 变得可负担。
第二，旧的定价模式已经破产。

这一切是怎么发生的？

第一章 — 2024 Q4

定价机被打破了

一切从 DeepSeek 开始。它把百万 Token 的价格从几十元砸到了几毛钱。这不是定价调整——这是把整个行业的定价机砸碎了。

价格战的起点

2024 Q4，DeepSeek 带头降价。百万 Token 从几十元降到几毛钱——降了一个数量级。

这在 AI 行业是前所未有的。此前，所有厂商的定价逻辑是"模型很贵，按量付费"。DeepSeek 证明了：MoE 架构 + 国产芯片可以让成本暴降 73%。

一旦有人证明了价格可以这么低，所有人都被迫跟进。价格战启动了。

  • DeepSeek V4-Pro输出 24 元/百万
  • DeepSeek V4-Flash输出 2 元/百万
  • 对比 GPT-5输出 $15/百万
  • 成本降幅73%
芯片供给直接绑定定价天花板。
DeepSeek 官方说："受限于高端算力，
昇腾 950 批量上市后价格大幅下调。"
—— 华为昇腾 950PR 算力是 H20 的 2.87 倍，价格是 H200 的 1/3，2026 年 4 月量产

价格战打开了潘多拉魔盒。
便宜到让人用不完的算力——
接下来会发生什么？

第四章 — 上下文崩溃

70% 的 Token 是浪费

Agent 不是"用得多"，而是"浪费得多"。每次调用都重发完整上下文：系统提示、读过的文件、错误日志、对话历史。一个 typo fix 用了 23,000 tokens。一个 $0.50 的 fix，47 次迭代后变成 $30。

上下文膨胀——Agent 每一轮都在重复发送
第 1 轮
5,000 tokens
第 10 轮
15,000 tokens
第 30 轮
35,000 tokens
第 50 轮
上下文窗口满了
有效信息 重复上下文（浪费）
Token 构成——你在为什么付费？

DEV Community 的研究追踪了 42 个 Agent 运行记录：70% 的 Token 是浪费。文件读取占 35-45%，工具输出 15-25%，上下文重发 15-20%。代码生成——你真正想让 AI 做的事——只占 5-15%。

更糟糕的是，200K token 的对话成本是 20K 的 10 倍。Agentic session 比非 agentic 使用 贵 200 倍。

  • 文件读取35-45%
  • 工具输出15-25%
  • 上下文重发15-20%
  • 推理10-15%
  • 代码生成5-15%

Coding Plan 的"不限量"承诺
在 Agent 时代不可持续。

第五章 — 终态与启示

从 Coding Plan 到 Credit

定价模式的演变不是计划出来的，是被现实逼出来的。Coding Plan 是过渡产品，Token Plan 是下一步，Credit 可能是终态。

定价模式三步演进
Talking Plan
2025 Q4 — 2026 Q1

固定月费，不限用量。用户说"帮我写个功能"，AI 执行。像手机合约机——绑定特定场景（编程），深度优化。

智谱 GLM 两个月 ARR 破亿，证明了市场需求。但前提是用户手动调用，Agent 还没来。

类比：手机话费套餐——充 100 块用不完，因为你在 2G 网络下发短信。
Token Plan
2026-03 至今

MiniMax 将 "Coding Plan" 更名为 "Token Plan"——从编程工具升级为通用算力。打包图片、语音、视频。不限量消失，按 Token 计量。

阿里百炼跟进，企业版 ¥198/698/1398。Lite 档停售——补贴结束，分层开始。

类比：手机流量套餐——终于透明了，但你得算着用了。
Credit 时代
2026+ 趋势

Cursor 用 Credit 池，小米 MiMo 用 Credit，阿里百炼企业版也用 Credit。统一 Credits，不同模型不同倍率——抽象了一层，但可预测。

这是终态吗？可能。但 Agent 的浪费问题如果不解决，Credit 也不够。真正的终态是：上下文压缩 + 模型路由 + 精细计量。

类比：话费余额——充进去的钱，打电话和发短信扣的不一样多，但你知道还剩多少。

定义权
谁能定义计量单位，谁就掌握定价权。
可复用的商业洞察
洞察一：计量单位即定价权
谁定义计量单位，谁就掌握定价权。MiniMax 从 Prompt 转向 Token，Kimi 同理——都是在争夺用户信任。模糊计量有利于早期获客，透明计量有利于长期信任。
模式推出新产品时，先用模糊计量降低门槛，成熟后用透明计量建立信任。
洞察二：补贴 → 收紧 → 分层
所有互联网赛道都走这三步。2024 补贴跑量，2025 H2 收紧测试弹性，2026 精细分层。MiniMax 极速版 ¥98-899，Kimi Allegro ¥559——分层是终局。
模式进入新市场不要一开始就想好定价。先补贴跑量建立习惯，再收紧测试弹性，最后精细分层。
洞察三：Agent 时代的成本不可逆
70% Agent Token 是浪费。推理需求已达训练的 4-5 倍，IDC 预测 2028 年推理占比 73%。所有厂商最终都会转向精细计量——谁先转谁主动，谁后转谁被动。
模式当使用场景从"手动"变为"自动"时，定价必须从固定转向计量。不转就是 Cursor 退款风波的教训。
