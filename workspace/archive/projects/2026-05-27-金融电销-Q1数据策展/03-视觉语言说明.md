# 阶段3：视觉语言说明

> 2026-05-27 / curatorial-visual-language

---

## 设计参照系

**选定：当代数据编辑式**

介于学术期刊的数据呈现和商业报告的清晰传达之间。文字驱动叙事，数据图表作为证据材料，交互服务下钻和比较。

**最终实现：Nothing 风格**

遵循 nothing-design 技能的完整设计系统，以单色画布、结构即装饰、数据即美的哲学为核心。

## 材料身份规则

| 材料类型 | 定义 | 处理方式 |
|---------|------|---------|
| 证据材料 | 原始数据、指标、排名 | 原样呈现，Space Mono 等宽字体 |
| 定位材料 | 导航、标签、索引 | 低调，ALL CAPS，11px |
| 分析材料 | 图表、散点图、矩阵 | 视觉焦点，可交互 |
| 氛围材料 | 背景、边框、分隔 | 最小化，1px border |

## 字体系统

| 角色 | 字体 | 字号 | 字重 |
|------|------|------|------|
| Display | Space Mono | 48-72px | 700 |
| Heading | Space Grotesk | 24px | 600 |
| Body | Space Grotesk | 16px | 400 |
| Label | Space Mono | 11px | 400, ALL CAPS |
| Data | Space Mono | 12-48px | 700 |

## 色彩系统

**单色画布**：

| 角色 | 色值 | 用途 |
|------|------|------|
| 背景 | #F5F5F5 | 页面底色（暖白） |
| 表面 | #FFFFFF | 卡片、图表背景 |
| 边框 | #E0E0E0 | 分隔线、1px grid |
| 正文 | #000000 | 标题、关键数字 |
| 弱化 | #666666 | 正文、描述 |
| 元数据 | #999999 | 标签、来源、时间 |
| 强调 | #D71921 | 唯一红色，仅用于标杆/警示 |

**数据区分方式**：
- 不用颜色编码团队（没有团队色板）
- 用 **opacity**（0.5 dim）表示低于基准的数据
- 用 **形状**（圆/方块）区分好/差
- 用 **线型**（实线/虚线）区分表现

## 版式节奏

**1px border grid**：Nothing 标志性分割方式

- 模块之间：1px border 分隔
- 无阴影、无渐变
- 间距基于 8px 倍数
- 页面最大宽度 1120px

## 交互语法

| 交互行为 | 实现方式 | 视觉反馈 |
|---------|---------|---------|
| 页面切换 | 顶部固定导航 | 当前页高亮 |
| 数据聚焦 | Hover | 边框/文字变化 |
| 详情展开 | 点击 | 内容展开 |

## 排除方向

| 排除项 | 原因 |
|--------|------|
| 深色背景 | 数据策展需要高可读性 |
| 渐变色 | 装饰性大于功能性 |
| 阴影 | Nothing 设计哲学禁止 |
| 多色团队编码 | 破坏单色画布一致性 |
| emoji | 不专业 |

## 与 Nothing 技能的协作规则

**关键教训**（来自方法回收）：

1. **技能设计系统 > 策展视觉参数**：nothing-design 有自己的完整 tokens，视觉实现以技能系统为主
2. **策展视觉语言只提供结构指导**：页面角色、材料身份、交互逻辑照用；色彩、字体、间距交给技能自己的系统
3. **避免两套规则打架**：如果实现时需要在两套规则之间做选择，说明阶段3写得太细了
