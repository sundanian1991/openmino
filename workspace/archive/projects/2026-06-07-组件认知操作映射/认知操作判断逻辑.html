01扫SCAN · 概览
用户在做什么：不分析，只建立整体印象。看一眼就走。
扫的特征是不深入、不比较、只看数量级。数据必须是并列的、同质的、一眼能扫完的。
数据形态：并列指标
3-5 个数字并排，扫一眼就知道数量级。不是对比（没有 A vs B），不是排序（没有先后）。
组件为什么选它什么时候不选
stat-block P1大数字+小标签，扫的效率最高需要趋势 → metric-card
metric-card P1指标+迷你趋势线，比 stat-block 多一层变化方向不需要趋势 → stat-block
kpi-card P1带完成度进度，扫时同时知道做了多少不涉及完成度 → stat-block
hero-numeral P0Hero 内嵌数字，首屏就要扫不是 Hero 位置 → stat-block
数据形态：数字矩阵
6+ 个数字排成方阵，扫一眼知道分布。比并列指标信息量更大，但仍然是"扫"不是"看"。
组件为什么选它什么时候不选
numeral-grid P04列数字方阵，同质数据的空间排列数据异质 → bento-grid
dashboard-grid P1多指标仪表盘，扫的密度最高指标 ≤5 → stat-block
02比COMPARE · 对比
用户在做什么：A 和 B 放在一起，找差异、判断优劣。
比的特征是两个对象同时呈现，差异一目了然。数据必须是对称的、可对照的。
数据形态：左右对照
两个对象并排，视觉上直接比。不是排序（没有多个对象），不是度量（不是单个指标）。
组件为什么选它什么时候不选
comparison P1双栏非对称，比的意图最明确维度 >5 → data-table
tension-grid P02列对立网格，比的哲学性更强不是哲学对比 → comparison
light-dark-switch P1同一对象两面，before/after 型不是同一对象 → comparison
do-dont P1二元规则对比，结论最明确不是规则对比 → tension-grid
数据形态：表格对比
多维度逐项比，精确但需要阅读。左右对照是"扫着比"，表格是"读着比"。
组件为什么选它什么时候不选
spec-table P1技术参数逐项对比，精度最高不是技术参数 → comparison
data-table P1多维度交叉对比，信息密度最高维度 ≤3 → comparison
principle-card P1单卡内行对比（原vs新），粒度最细对象 >2 → data-table
03排RANK · 排序
用户在做什么：多个对象建立秩序——谁第一、谁最后、权重怎么分。
排的特征是有明确的序号或位置关系，高低/先后/权重一目了然。
数据形态：条形 + 序号
长度=大小，序号=位置，排的意图最直接。不是概览（有顺序），不是对比（有多个对象）。
组件为什么选它什么时候不选
rank-bar P1水平排名+进度+delta，排的意图最明确不涉及进度 → rank-panel
rank-panel P1带标题的排名容器，包装感更强需要精确数值 → ranking-table
ranking-table P1排名表格，精度最高需要视觉冲击 → rank-bar
数据形态：分段条
长度=权重，段=组成，排的是比例关系。不是排名（没有序号），是份额。
组件为什么选它什么时候不选
seg-bar P0签名工业分段条，排比例/权重不是比例关系 → rank-bar
weight-bar P1权重条，各项占比不是占比 → seg-bar
stacked-bar P1堆叠条，排的是组成结构不是组成 → weight-bar
progress-bar P1进度条，排的是完成度不是完成度 → seg-bar
04读FLOW · 流程
用户在做什么：理解顺序——先做什么、后做什么、经过几个阶段。
读的特征是有明确的时间或逻辑顺序，前后关系不可颠倒。
数据形态：节点 + 连线
节点=阶段，连线=顺序，流的意图最直接。不是排序（不是比大小），是比先后。
组件为什么选它什么时候不选
timeline-track P1链式节点，流的是时间上的先后不是时间线 → flow-pipeline
flow-pipeline P1横向流程线，done/active/pending 三态不涉及状态 → step-sequence
progress-trail P1进度路径，阶段+完成度不涉及完成度 → timeline-track
level-track P1等级轨道，等级晋升不是等级 → flow-pipeline
数据形态：编号步骤
编号=顺序，步骤=操作，流的是"怎么做"。不是"经过什么阶段"，是"执行什么动作"。
组件为什么选它什么时候不选
step-sequence P0编号步骤，流的是 SOP 怎么做不是操作流程 → flow-pipeline
05量MEASURE · 度量
用户在做什么：判断一个指标的好坏——高还是低、达标还是没达标。
量的特征是单个指标+判断标准，不是比两个对象，是判断一个对象。
数据形态：仪表
一个指针/弧形+一个阈值，量的意图最直接。不是概览（只有一个指标），不是对比（没有 A vs B）。
组件为什么选它什么时候不选
gauge-arc P1SVG 弧形仪表，视觉冲击最强不需要视觉冲击 → block-bar
gauge-segmented P1分段仪表，在哪个等级不需要等级 → gauge-arc
bullet-chart P1实际值 vs 目标 vs 范围带没有基准目标 → gauge-arc
数据形态：趋势线
一条线+时间轴，量的是"变化方向"。不是当前值好坏，是"在变好还是变坏"。
组件为什么选它什么时候不选
sparkline P1迷你趋势线，量变化趋势需要看当前值好坏 → gauge-arc
数据形态：评分
一段条+一个等级，量的是"在什么水平"。不是百分比，是等级划分。
组件为什么选它什么时候不选
block-bar P1段状评分，在什么水平需要精确百分比 → progress-row
progress-row P1进度条+数值，达成率不是达成率 → block-bar
06解READ · 解读
用户在做什么：理解一段文字/数据的含义——为什么这么说、背后的逻辑。
解的特征是有论证结构——观点+依据、结论+原因。
数据形态：边注 + 引用
边注=重点标注，引用=权威来源，解的意图最直接。不是度量，不是对比。
组件为什么选它什么时候不选
callout P1边注框，解的最常用形态需要引用原文 → pull-quote
pull-quote P0引用块，权威说了什么不是引用 → callout
blockquote P1内联引言，粒度最细需要独立展示 → callout
brand-statement P1全宽宣言卡，品牌立场不是品牌立场 → callout
数据形态：标注 + 清单
标注=逐段解读，清单=该做什么，解的是"怎么做"。
组件为什么选它什么时候不选
annotated-highlight P1pin+标签，图表/图片里的重点不是图表解读 → callout
action-list P1动作清单，该做什么不是行动建议 → callout
07信PROVE · 背书
用户在做什么：确认信息有依据——数据从哪来、谁说的、凭什么信。
信的特征是有来源标注——Logo、出处、代码，不是内容本身，是内容的"凭证"。
数据形态：来源 + Logo
Logo=谁在用，来源=数据出处，信的意图最直接。不是解读，是凭证展示。
组件为什么选它什么时候不选
logo-wall P0Logo 墙，一堆 logo 比文字更可信不是合作方背书 → source-row
source-row P1数据出处行，精确到来源+日期不是数据来源 → tag
tag P1标签，标注来源/分类不是来源标注 → callout
数据形态：数字即证据 / 代码即证据
数字本身就是证据不需要额外来源；代码本身就是证据技术场景的背书。
组件为什么选它什么时候不选
stat-block P1用数据证明不是用数据证明 → logo-wall
code-panel P1代码即证据，技术场景不是技术场景 → stat-block
08展DIG · 深入
用户在做什么：想看更多——点一下展开、切换视图、看详情。
展的特征是有交互——点击/悬停/切换，信息分层揭示。
数据形态：折叠 / 切换
点一下展开/收起，展的意图最直接。不是静态展示，不是概览。
组件为什么选它什么时候不选
accordion P1折叠面板，FAQ、多层详情需要并排切换 → tab-switcher
tab-switcher P1选项卡，同一空间多视图不需要多视图 → accordion
flip-card P1翻转卡片，正反两面不是正反 → accordion
hover-reveal P1悬停揭示，渐进式信息密度不是渐进 → detail-panel
detail-panel P1全屏详情面板不需要全屏 → accordion
数据形态：清单 / 跟踪
勾选=完成，清单=任务，展的是"进度"。不是展开详情，是跟踪状态。
组件为什么选它什么时候不选
interactive-checklist P1交互清单，任务跟踪不是任务 → accordion
segmented-control P1分段选择器，切换数据视图不是数据视图 → tab-switcher
09感FEEL · 氛围
用户在做什么：不判断，只感受。建立品牌印象、空间节奏、视觉冲击。
感的特征是不传递具体信息，传递情绪和气质。
数据形态：Hero（开场）
全屏、大字号、视觉冲击，感的意图最直接。不是内容区，是第一印象。
组件为什么选它什么时候不选
V1-V6 Hero P06种切割变体，首屏决定气质不是首屏 → 装饰元素
Marquee P0品牌宣言 Hero，品牌态度不是宣言 → V1-V6
Split Diptych P0产品+风景对屏，产品与自然不是产品 → V1-V6
Photographic P0全屏风光，自然气质不是自然场景 → Marquee
数据形态：装饰元素
幽灵文字、点阵、纹理，感的是"空间节奏"。不是 Hero，是氛围填充。
组件为什么选它什么时候不选
ghost-text P1幽灵水印，空间深度不是空间感 → dot-pattern
seal P1印章，仪式感不是仪式感 → ghost-text
dot-pattern P1点阵背景，科技感不是科技感 → texture
dot-matrix P1前景点阵字符，技术与手工不是技术感 → ghost-text
数据形态：品牌系统
色板、字体、标志，感的是"品牌 identity"。不是装饰，是品牌体系展示。
组件为什么选它什么时候不选
color-swatch P0单个色板，颜色本身要展示体系 → color-swatch-grid
typography-showcase P0字体展示，字体性格不是字体 → color-swatch
brand-signature P0品牌签名收尾，结束感不是收尾 → Hero
