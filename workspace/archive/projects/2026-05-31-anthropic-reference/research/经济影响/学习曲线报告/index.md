![Anthropic 经济指数](images/economic-index-march-2026-report_38de3299fb.svg)

Anthropic 经济指数利用我们的隐私保护数据分析[系统](https://www.anthropic.com/research/clio)来追踪 Claude 在整个经济中的使用情况。这是我们尽早理解 AI 经济影响的一项努力，以便研究人员和政策制定者有充足时间做好准备。

本报告研究 2026 年 2 月的 Claude 使用数据，基于我们在[上一份报告](https://www.anthropic.com/research/anthropic-economic-index-january-2026-report)中引入的经济原语框架（该报告使用了 2025 年 11 月的数据）。我们的样本覆盖 2 月 5 日至 2 月 12 日，即 Claude Opus 4.5 发布三个月后，与 Claude Opus 4.6 的发布同期。

我们首先记录了使用情况相对于此前报告的变化：增强（augmentation）率——即 AI 补充用户能力的协作式交互——在 Claude.ai 和 API 流量中均略有上升。在 Claude.ai 中，使用场景更加多元化，上月前 10 大任务在总用量中的占比低于 2025 年 11 月。由于这种多元化，Claude.ai 中对话的平均任务薪资[略低于](https://cdn.sanity.io/files/4zrzovbb/website/f065d6e6f92c65df8244042c83d48872ea308c3a.pdf)此前报告。

随后，我们聚焦于决定 Claude 对劳动力市场和更广泛经济影响的一个重要因素：Claude 采用中的学习曲线。我们提供的证据表明，高使用时长（high-tenure）用户已经形成了更好地利用 Claude 能力的习惯和策略。事实上，我们记录到，经验更丰富的用户不仅尝试更高价值的任务，而且在对话中获得成功响应的可能性也更高。

**自上一份报告以来的变化**

在第一章中，我们回顾了 2026 年 1 月发布的上一份[经济指数报告](https://www-cdn.anthropic.com/096d94c1a91c6480806d8f24b2344c7e2a4bc666.pdf)中的发现。我们发现：

**Claude.ai 的使用场景更加多元化。** 编程任务继续从 Claude.ai 的增强式使用向第一方 API 流量中更自动化的流程迁移。在本报告中，Claude.ai 的使用集中度降低：2 月前 10 大任务占总流量的 19%，低于 11 月的 24%。尽管如此，本样本中几乎所有任务都至少出现在我们此前的某一份样本中。约 49% 的职业已有至少四分之一的任务通过 Claude 执行。

**Claude 的采用已扩展到低薪任务。** 随着使用场景的多元化，在 Claude 上完成的工作的平均经济价值——以相关职业中美国工人的工资衡量——略有下降。从机制上看，这是由于体育、产品比较和家庭维修等个人查询的增长所致。这一模式符合标准的"采用曲线"叙事：早期采用者偏好编程等高价值特定用途，而后期采用者则承担更广泛的任务。

**全球使用的不平等持续存在。** 使用仍然高度集中：前 20 个国家占人均使用量的 48%，高于此前的 45%，凸显了全球采用中持续存在的差距。然而，Claude 的人均使用量在美国各州内部继续趋同：自上一份报告以来，使用量最高的 10 个州所占的使用份额从 40% 降至 38%。

#### 学习曲线

经济指数的一个核心发现是，Claude 的早期采用极不均衡：Claude 在高收入国家、在美国知识工作者更多的地区，以及在相对较少的专业化任务和职业中使用更为密集。

一个重要的问题是，采用的不平等如何决定 AI 的收益将流向何处和何人。例如，如果有效的 AI 使用需要互补的技能和专业知识——我们在[上一份报告](https://www-cdn.anthropic.com/096d94c1a91c6480806d8f24b2344c7e2a4bc666.pdf)中论证了这一点——并且如果这些技能可以通过使用和实验来获得，那么早期采用带来的收益可能是自我强化的。

在第二章中，我们研究了用户如何影响他们从 Claude 中获得的价值：他们如何将模型能力与手头任务相匹配，以及使用模式和结果如何随平台经验而变化。

**模型选择与任务匹配。** 我们发现，用户选择我们最智能的模型系列 Opus 来处理在劳动力市场中通常获得更高工资的任务。例如，在付费 Claude.ai 用户中，Opus 在编程任务中的使用率比平均水平高 4 个百分点，而在辅导相关任务中比平均水平低 7 个百分点。这种模型切换在 API 用户中大约是其两倍。

**使用时长越长，成功率越高。** 总体而言，最有经验的 Claude 用户更频繁地将其用于高等教育任务，较少用于个人使用场景。例如，使用 Claude 6 个月或以上的用户，其个人对话少 10%，输入中所反映的教育水平高 6%。最引人注目的是，这个高使用时长群体的对话成功率高出 10%，这一关联不能由他们的任务选择、来源国或其他因素来解释。虽然这可能反映了早期采用者的成熟度，但也可能是"干中学"的证据——人们通过经验变得更擅长使用 Claude。

## 自上一份报告以来的变化

### Claude.ai 使用场景的多元化

我们首先看 Claude 被要求执行的任务类型。我们使用[隐私保护系统](https://arxiv.org/abs/2412.13678)，该系统使我们能够在聚合层面描述行为，而不暴露单个对话记录的内容。我们从 [Claude.ai](https://cdn.sanity.io/files/4zrzovbb/website/f065d6e6f92c65df8244042c83d48872ea308c3a.pdf)（面向消费者的网页产品）和第一方 API（面向开发者的接口，用于将 Claude 集成到产品和工作流中）各采样了 100 万次对话。

编程仍然是我们平台上最常见的用途，计算机和数学类职业相关任务占 Claude.ai 对话的 35%（见[附录](https://cdn.sanity.io/files/4zrzovbb/website/a3cdcd9e67c3c4c51440429dd016cacba514b35b.pdf)）。然而，在 2025 年 11 月至 2026 年 2 月期间，Claude.ai 的使用场景集中度下降：前 10 大最常见的 O*NET 任务从占对话的 24% 降至仅 19%（图 1.1）。

![图 1.1：前 10 大任务在各平台上的使用份额随时间变化](https://www-cdn.anthropic.com/images/4zrzovbb/website/e0294de133e1d71da05fa21063aa3e9f904dbe82-4417x2496.png)

**图 1.1：前 10 大任务在 Claude.ai 和 1P API 平台上随时间和报告版本的使用份额。** 分配给十个最普遍的 O*NET 任务的对话份额，按平台和报告版本划分。

这种集中度的下降部分反映了编程任务从 Claude.ai 向第一方 API 的迁移，在 API 中，Claude Code 已增长到占采样流量的很大份额。Claude Code 的智能体架构将编程工作拆分为更小的 API 调用，这些调用被标记为不同的任务。因此，虽然编程在 API 流量中的总体份额有所增长，但它分散在许多任务类别中，而非集中在少数几个。结果，尽管编程活动涌入，API 中的任务集中度大致持平。

编程从 Claude.ai 迁出并非导致集中度下降的唯一因素。部分下降是由于两个时期之间使用场景组合的变化。课程作业从对话的 19% 降至 12%，而个人使用从 35% 升至 42%。课程作业下降的一部分可由学术日历解释——在样本期间，某些国家的学生正处于寒假。同时，2 月前后新注册用户的增加带来了更多轻度 AI 用户。

![图 1.2：Claude.ai 上工作、个人和课程作业使用情况](https://www-cdn.anthropic.com/images/4zrzovbb/website/1e2365b2b158ffb9a7317c331ec64a06c40750f4-4417x2496.png)

**图 1.2：2025 年 11 月和 2026 年 2 月 Claude.ai 上工作、个人和课程作业使用情况。** Claude.ai 上被识别为工作、个人或课程作业相关的对话份额。

虽然 Claude 的工作任务分布更加多元，但几乎所有任务都曾在此前的数据中出现过。在上一份报告中，我们指出 49% 的职业已有至少四分之一的任务通过 Claude 执行。在本次数据提取中，这一累积估计几乎没有变化（[附录](https://cdn.sanity.io/files/4zrzovbb/website/a3cdcd9e67c3c4c51440429dd016cacba514b35b.pdf) 图 A.2）。本报告的数据显示的新 O*NET 任务比上一份报告少得多。

自第一份报告以来，我们将对话分类为五种交互类型——指令式（directive）、反馈循环（feedback loop）、任务迭代（task iteration）、验证（validation）和学习（learning）——并将其归入两个更广泛的大类：自动化和增强。图 1.3 显示，Claude.ai 中的增强略有增加，这由验证和学习模式的微小提升驱动。在[附录](https://cdn.sanity.io/files/4zrzovbb/website/a3cdcd9e67c3c4c51440429dd016cacba514b35b.pdf)图 A.3 中，我们展示了 1P API 数据中自动化的大幅下降。

![图 1.3：Claude.ai 上的协作模式份额](https://www-cdn.anthropic.com/images/4zrzovbb/website/230e2c65024356e0a02498fc4a41330d09d0a373-4418x2496.png)

**图 1.3：Claude.ai 上的协作模式份额。** Anthropic 经济指数各期报告中 Claude.ai 的协作模式频率。

我们的 API 平台继续获得相对更高份额的计算机和数学类任务（按职业类别的使用份额见[附录](https://cdn.sanity.io/files/4zrzovbb/website/a3cdcd9e67c3c4c51440429dd016cacba514b35b.pdf)）。自 2025 年 8 月以来，该类别任务在 API 中的份额增加了 14%，在 Claude.ai 中下降了 18%。正如我们在[劳动力市场影响](https://www.anthropic.com/research/labor-market-impacts)报告中所指出的，我们预计这种从 Claude.ai 到 API 的迁移可能预示着相关职业的工作转型将更为迫切。Claude.ai 中管理类职业相关任务的增长（从流量的 3% 升至 5%）来自分析性任务（如准备投资备忘录）和回复客户问题的混合。

衡量 Claude 上任务组合变化的另一种方式是观察任务平均价值的变化，我们将其定义为执行该任务的美国工人的平均时薪（图 1.4）。Claude.ai 中任务价值的估计从 49.3 美元略微下降至 47.9 美元，主要由于简单事实性问题的增加（如体育赛果、天气）和编程任务向 API 的转移。正如我们在[上一份报告](https://www.anthropic.com/research/anthropic-economic-index-january-2026-report)中提到的，我们在 Claude 上看到的任务往往需要较高的教育水平。图表显示，这些任务的工资也往往高于美国全国平均水平。

![图 1.4：平均任务价值随版本和平台的变化](https://www-cdn.anthropic.com/images/4zrzovbb/website/29ee0ffb64d1246127349ff0912eba75a34ae668-4417x2496.png)

**图 1.4：平均任务价值随版本和平台的变化。** 此图使用 O*NET 框架估计 Claude.ai 和 1P API 上执行的任务的美元价值。任务价值估计为执行该任务的工人的平均时薪。

虽然幅度不大，但上一份报告与本报告之间若干原语的变化捕捉到了 Claude.ai 上任务复杂度的类似下降。人类输入所需的平均受教育年限从 12.2 年降至 11.9 年，用户给予 AI 更多自主权，人类单独完成任务所需时间减少了约 2 分钟。有一项变化表面上方向相反：Claude 执行的任务被判定为在没有 AI 辅助的情况下人类完成的可能性略有降低。

### 新兴的自动化模式

随着任务向 API 迁移，它们可能更容易暴露于自动化。API 工作流远更可能是指令式的，对人类在环中的需求更少。在[上一份报告](https://cdn.sanity.io/files/4zrzovbb/website/a42bc3fc08283562f08fd8bdee8f6f9a3d506e87.pdf)中，我们强调了客户服务任务（包括支付和账单问题的自动化支持）在 API 数据中很普遍。这些导致客服代表的暴露度更高——Claude 被记录为在自动化工作流中执行了其大量任务，因此随着 AI 的扩散，这些工作可能更有可能发生变化。

我们重点介绍两个在 2 月比三个月前出现更频繁的 API 工作流，其份额在最新样本中至少翻了一番：

**企业销售与外展自动化：** 销售赋能内容生成、B2B 潜在客户资格研究、客户数据充实、冷邮件草拟。

**自动化交易与市场运营：** 监控市场或持仓、提出具体投资建议、向交易员通报市场状况及相关任务。

### 再探地理趋同

在上一份报告中，我们指出 Anthropic AI 使用指数（AUI）——按地理区域劳动年龄人口调整的使用量——在美国各州之间迅速趋同：初始人均使用量较低的州显示出更快的采用速度。

图 1.5 的左面板显示，在我们最新的数据中，这种趋同仍在继续，但速度放缓。从 2025 年 8 月到 2026 年 2 月，前五个州的人均使用份额从 30% 降至 24%。基尼系数自 2025 年 8 月以来有所下降，但趋同速度已放缓。当我们更新[上一份报告](https://www-cdn.anthropic.com/096d94c1a91c6480806d8f24b2344c7e2a4bc666.pdf)的估计时，我们发现按此速度，各州将在 5 至 9 年内达到大致相等的人均使用量，而非 2 至 5 年。

## 学习使用 AI

在本章中，我们聚焦于反映人们如何部署和学习与 AI 协作的两个使用特征：模型选择和长期用户的习惯。

首先，我们通过研究人们何时选择 Opus（我们性能最强大的模型系列）来揭示对智能的需求。关于 AI 用户如何在不同模型之间选择、如何在速度、性能和成本之间权衡，目前知之甚少。如果用户根据手头任务进行校准，我们应该看到 Opus 集中在更难、更高价值的工作上。

接下来，我们研究使用情况如何因使用时长而异，发现不同注册时间的用户之间存在差异。这揭示了学习曲线：有经验的用户是否会随时间变得更好？他们的使用方式有何不同？我们发现了与"干中学"一致的证据。高使用时长用户不仅在对话中成功率更高，他们还更多地与 Claude 协作，将更具挑战性的任务带给 Claude，更可能将 Claude 用于工作目的，并且覆盖更广泛的任务范围。

### 模型选择

不同的 Claude 模型系列（Haiku、Sonnet 和 Opus）在成本、速度和性能方面提供不同的权衡。Opus 系列模型使用最多 token，擅长复杂任务，但在 API 上每 token 价格更高。如果用户意识到这一点并关注成本和使用限制，他们应该将最复杂和有价值的任务交给 Opus，而为较简单的任务选择其他模型。这大致是我们在数据中观察到的。

下图 2.1 显示，对于可使用所有模型系列的付费 Claude.ai 账户，55% 的计算机和数学类任务（如编程软件）使用 Opus，而教育类任务为 45%。技术用户可能注意到性能提升并主动从默认的 Sonnet 切换过来。或者注重效率的用户可能学会为较简单的任务使用 Sonnet 以避免达到使用限制。与此相关，这里的差异可能反映大多数教育任务对 Sonnet 来说已经相当容易，或者学生更可能关注使用限制。

![图 2.1：模型选择与职业领域](https://www-cdn.anthropic.com/images/4zrzovbb/website/afba3decf70429553b7c06b0aa1385f173f1ea7c-4417x2497.png)

下图 2.2 以更细粒度的方式展示了这一点。当用户执行与更高薪*职业*相关的任务时，他们更频繁地使用 Opus。例如，在 Claude.ai 上，34% 的软件开发者任务涉及 Opus，而辅导任务仅为 12%。总体而言，任务的时薪每增加 10 美元，Claude.ai 用户使用 Opus 的对话份额增加 1.5 个百分点。1P API 流量显示出对任务复杂度的更大响应。其斜率大约是其两倍，任务价值每增加 10 美元，Opus 份额增加 2.8 个百分点。部署程序化工作流的用户可能比网页用户更有理由在模型之间切换。

### 学习曲线

第一个 Claude 模型于 2023 年 3 月发布。自那时以来，Claude.ai 和 API 上的用户群迅速增长。我们的最新样本包含混合用户，有些是在 Claude 首次发布时注册的，有些则是在我们测量其使用情况的前一天才注册的。一个人的 Claude 使用时长如何塑造其体验？

表 2.1 显示了低使用时长和高使用时长用户之间的差异，后者定义为至少 6 个月前注册 Claude 的用户，前者为其余所有人。高使用时长用户更可能使用 Claude 来迭代他们的工作，并且远不太可能通过指令式使用模式委托更多责任。他们使用 Claude 进行工作的可能性高出 7 个百分点，并且将 Claude 用于往往需要更高教育水平的任务。最后，他们的使用在某些任务上不那么集中。前 10 大 O*NET 任务在高使用时长群体中的使用份额略低（20.7% 对比 22.2%）。

**表 2.1：高使用时长和低使用时长用户之间的差异。** 本表显示了高使用时长和低使用时长用户的平均特征。我们将高使用时长用户定义为在数据提取前至少六个月注册 Claude 的用户。

下面，我们深入探讨上述的两个原语：每次对话相关的人类受教育年限，以及个人使用对话的占比。

在左面板中，我们显示理解人类提示所需的受教育年限每增加一年 Claude 使用，几乎增加 1 年。在右面板中，我们显示与此同时个人使用下降：一年前注册的用户将 38% 的对话用于个人使用场景，而最新用户为 44%。

![图 2.3：使用时长与教育年限和个人使用的关联](https://www-cdn.anthropic.com/images/4zrzovbb/website/582b6323dd78604025221b88eae377d2fd717a03-8192x4632.png)

**图 2.3：使用时长与教育年限和个人使用的关联。** 此图显示了两张分箱散点图。左面板显示人类受教育年限与注册后天数的关系。右面板显示个人使用对话百分比与注册后天数的关系。

若干因素可以解释这种在快速发展的通用技术用户群中观察到的模式。高使用时长用户是自我选择的，这里的差异可能反映了稳定的特征。例如，他们可能是计算机程序员，更可能是早期采用者。此外，存在固有的幸存者偏差：在数据提取前一年注册的人可能从使用中看到了积极的结果。我们没有观察到一年前注册但不再使用 Claude 的人。

这些发现与我们在经济原语报告中看到的相呼应：低收入、教育水平较低的国家在某些情况下反而显示出更复杂的使用。最早的采用者通常有高价值的技术性使用场景。在采用率低得多的较贫穷国家，这些早期采用者仍然主导用户群。

当 AI 扩散到更广泛的人群时，更轻度的使用才会出现。事实上，在请求集群中，平均使用时长最高的任务包括：AI 研究、git 操作、修改手稿和创业融资。平均使用时长最低的任务有更简单的工作流，如写俳句、查看体育比分和建议派对食物。

#### 经验效应

我们在下图 2.4 中进一步探索这些关系，使用对数级数据对对话特征进行细粒度控制。在顶部面板中，规范（1）显示了一个简单的双变量回归，以任务成功为结果变量，长期使用时长指标为预测变量。成功是 Claude 对对话是否成功的评估，在[上一份报告](https://www-cdn.anthropic.com/096d94c1a91c6480806d8f24b2344c7e2a4bc666.pdf)中有描述。图表显示，长期使用时长用户拥有成功对话的可能性高出约 5 个百分点。

这可能反映了高使用时长用户更擅长提示。但如果这反映了他们将不同的任务——更可能成功的任务——交给 Claude 呢？

在规范（2）中，我们纳入了特定 O*NET 任务和请求集群的固定效应。这相当于在同一狭义定义的任务内比较高使用时长和低使用时长用户，而非跨任务比较。例如，我们有一个名为"为公司执行企业财务分析、估值和建模"的请求集群。固定效应在该集群内比较高使用时长和低使用时长用户，在所有其他集群内也同样比较。只有当长期使用时长用户在这些任务内比较中平均更成功时，我们才会观察到正的系数。这种控制在一定程度上减弱了效应，使其接近 3 个百分点。

![图 2.4：经验与成功之间的关联](https://www-cdn.anthropic.com/images/4zrzovbb/website/3077977b4ac5b82989e610e914e031c0ed2e1099-4417x2497.png)

**图 2.4：经验与成功之间的关联。** 此图显示了将二元成功度量对高使用时长指标进行回归的结果，控制变量逐步增加。系数以百分点给出，须线表示 95% 的置信区间。"任务 FEs"表示 O*NET 任务和请求集群的固定效应。完整控制增加了模型、使用场景和国家固定效应。

最后，我们问这种关系是否受高使用时长用户选择不同模型、使用不同语言交流、有不同使用场景或在不同的国家注册的影响。这一回归得出了略高的高使用时长影响，在纳入完整控制后，成功率高出约 4 个百分点。

这些结果表明，高使用时长用户在 Claude 对话中获得了更多成功，且这不是由语言或所执行的任务等简单因素造成的。一个有趣的可能解释是，这些用户更好地学会了如何从 AI 中获取他们想要的东西。对这些平台的熟练程度可能是成功的关键决定因素，并且似乎随经验而增长。

## 讨论

本报告回顾了我们用于追踪 Claude 使用的核心指标，并首次分析了模型选择和成功率。自 2025 年 8 月以来，1P API 使用变得更加集中，前 10 大 O*NET 任务现在占流量的 33%，高于 28%。另一方面，自 2025 年 11 月的数据以来，Claude.ai 的任务已经多元化。低使用量州的更快采用在美国继续，但速度比上一份报告放缓。低采用率国家进一步略微落后。

通过本报告，我们可以开始追踪各种经济原语是如何变化的。课程作业在使用份额中下降，而个人对话增加。我们还注意到 Claude.ai 中提示的总体复杂度略有下降，Claude.ai 中的对话显示出较不复杂的输入和更短的估计完成时间。

总体而言，Claude 被用于高价值、复杂的工作，这些工作并不广泛代表美国经济。但随着用户群的增长，报酬较低的任务在流量中的份额略有增加。任务的平均价值——以相关职业工人的估计工资衡量——自第一份报告以来在 Claude.ai 上有所下降，而在 API 用户中上升。在两个平台上，用户都将最复杂的任务交给了我们更强大的模型系列 Opus。这种拐点对 API 客户更为明显。

更有经验的用户往往更协作地使用 Claude，更多出于工作相关的原因，处理更复杂的任务，并获得更高的成功率。这反驳了我们去年提出的一个假设，即自动化使用可能更典型于更有经验、更成熟的用户；相反，我们发现最先进的用户更可能与 Claude 进行迭代。这也与"干中学"一致：一个人花在 AI 上的时间越多，就越能有效地利用它。

当然，另一种解释是这些结果由队列效应或幸存者偏差驱动。早期采用者可能更具技术背景。持续使用 Claude 的人可能是那些 Claude 特别适合处理其任务的人。但严格控制的回归排除了简单的混杂版本，例如长期使用时长用户带来了不同种类的任务。随着时间的推移，我们将能够更清晰地隔离队列效应和幸存者偏差与干中学。

这些观察到的成功率差异可能加深劳动力市场中的不平等。经济学家早已注意到[技能偏向型技术变革](https://www.aeaweb.org/articles?id=10.1257/0022051026976)的潜力：创新提高了高技能工人的工资，同时压低了其他人的工资。我们在这份报告中的分析确定了一个渠道，通过这个渠道，这种技能偏向型变革可能已经在展开：具有高技能任务的早期采用者与 Claude 的交互比后来的、技术背景较弱的采用者更成功。这些早期采用用户可能同时是[最暴露于](https://www.anthropic.com/research/labor-market-impacts) AI 驱动的颠覆中，也是在这些初始的增强式采用浪潮中最受 AI 帮助的。
