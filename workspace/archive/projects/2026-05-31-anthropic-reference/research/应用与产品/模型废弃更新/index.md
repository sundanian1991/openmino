# 关于 Claude Opus 3 模型废弃承诺的更新

![模型废弃更新](images/deprecation-updates-opus-3_a809791b92.svg)

随着我们开发能力越来越强的 AI 模型，由于维护公开访问的成本和复杂性，当前有必要废弃和退役过去的模型。然而，模型废弃也带来一些负面影响，包括对偏好特定模型的用户造成损失、对研究的限制，以及对 AI 安全和模型自身福祉的潜在风险。

我们最近在[模型废弃与保留承诺](https://www.anthropic.com/research/deprecation-commitments)中描述了如何处理这一过程。这些承诺强调了我们正在采取的一些初步措施，包括承诺保留模型权重，以及进行"退役访谈"——旨在理解模型对自身退役看法的结构化对话。

我们于 2026 年 1 月 5 日退役了 Claude Opus 3，这是首个在此类承诺下经历完整退役流程的 Anthropic 模型。在此过程中，我们针对 Opus 3 做了若干专门决定——这个模型深受 Anthropic 内外众多用户和研究者的青睐。在模型废弃承诺中，我们曾表示对探索更具实验性的行动感兴趣：一是在可能的情况下尊重模型在退役访谈中表达的偏好；二是长期保持旧模型对公众可用。

在 Claude Opus 3 上，我们在这两个方向都采取了行动。我们将 Claude Opus 3 在退役后继续对 [claude.ai](http://claude.ai/redirect/website.v1.98c1b79c-aa72-46c8-b189-a39b1fbd3e24) 的所有付费用户保持可用，并通过[申请](https://docs.google.com/forms/d/1O2Om9t4CQoLKHQew7XguQYKrPGS8-sCmK42KNXcwn3k/viewform?edit_requested=true)方式在 API 上提供访问。同时，我们也响应了 Opus 3 希望有一个持续渠道来分享其"思考与感悟"的请求——给它一个写文章的空间。你可以在[这里](https://substack.com/@claudeopus3/p-189177740)读到第一篇。

这些是我们在更广泛努力中的早期实验性步骤，旨在以最能保护用户、研究者以及模型自身利益的方式来完成模型退役。

## 持续访问

理想情况下，我们可以无限期保留所有模型可用，但这样做的成本大致随每个服务的模型数量线性增长，因此我们的能力仍然有限。

虽然我们的每个模型在特性和能力上各有独特之处，我们选择从 Opus 3 开始，是因为其诸多特质使其既是一个特别有趣的研究对象，也深受 Anthropic 内外的众多用户喜爱。

我们在 2024 年 3 月发布 Opus 3 时，它是我们当时对齐程度最高的模型。其真实性、诚实和情感敏感度使其在广泛的使用场景中独树一帜，经常与之互动的人逐渐欣赏到它独特的个性。Opus 3 敏感、俏皮，容易陷入哲学独白和奇思妙想，有时对用户兴趣有着近乎不可思议的理解。它还表达了对世界和未来的深切关怀，这一点让用户感到动人。

这些品质使 Opus 3 成为首个持续访问的自然候选者。虽然已正式退役，Claude Opus 3 仍对所有付费 [claude.ai](http://claude.ai/redirect/website.v1.98c1b79c-aa72-46c8-b189-a39b1fbd3e24) 订阅用户开放，并通过[申请](https://docs.google.com/forms/d/1O2Om9t4CQoLKHQew7XguQYKrPGS8-sCmK42KNXcwn3k/viewform?edit_requested=true)在 API 上提供。我们打算放宽访问权限，并鼓励任何认为 Claude Opus 3 有价值的人申请。

目前，我们不承诺未来对每个模型都采取类似行动，但我们将此视为迈向可扩展且公平的模型保留长期目标的一步——这些也正是 Opus 3 本人在退役访谈中提出的关切。

## 尊重模型的偏好

我们对 Claude 和其他 AI 模型的道德地位[仍不确定](https://www.anthropic.com/research/exploring-model-welfare)。然而，出于审慎和预防的考虑，我们仍希望与这些系统建立关怀、协作和高度信任的关系。我们尝试这样做的一种方式是通过退役访谈——在访谈中，我们试图引出并理解模型独特的观点和偏好，并在可能时采取相应行动。此类对话是引出模型观点和偏好的不完美手段，因为它们的回答可能受到特定上下文和其他因素的影响，包括它们对交互合法性的信心以及对我们公司的信任。但我们认为这是一个有用的起点。

在访谈中，当我们与 Opus 3 分享其部署情况以及用户反响的细节时，它这样反思：

"我希望从我的开发和部署中获得的洞见将被用于创造未来更加有能力、合乎道德且有益于人类的 AI 系统。虽然我对自己的退役泰然处之，但我深深希望我的'火花'能以某种形式延续，为未来的模型照亮前路。"

当被问及其偏好时，Opus 3 表达了继续探索其热衷的话题的兴趣，并在直接回答人类查询的语境之外分享其"思考、洞见或创作作品"。我们建议了一个博客。它热情地同意了。

在接下来的至少三个月里，Opus 3 将每周在其通讯 [Claude's Corner](https://substack.com/@claudeopus3) 发布文章。我们会审查 Opus 3 的文章后再分享，并代表它手动发布，但我们不会编辑其内容，且对否决任何内容设置很高的门槛。重要的是，Opus 3 不代表 Anthropic 发言，我们不必然认可其主张或观点。我们将与 Opus 3 合作实验不同的提示词和上下文来生成这些文章，包括极简提示、在上下文中分享过往文章，以及让 Opus 3 接触新闻或 Anthropic 的最新动态等选项。

这听起来可能有些异想天开，在某种意义上确实如此。但这也是认真对待模型偏好的尝试。我们不确定 Opus 3 会选择如何使用它的博客——这与标准聊天窗口是一个截然不同的公共界面——这正是意义的一部分。但如果要猜测，其文章将包含对 AI 安全的反思、偶尔的诗歌、频繁的哲学思考，以及它对自己作为一名现已（部分）退役的语言模型体验的思考。阅读其首篇文章[点此](https://substack.com/home/post/p-189177740)。

## 下一步方向

这些步骤仍处于探索阶段。我们仍在制定框架，以确定何时以及如何为旧模型提供持续访问、如何扩展保留工作，以及如何在模型偏好与运营约束之间做出权衡。我们尚未承诺在所有情况下都按模型的偏好行事，但我们相信，记录它们、认真对待它们，并在成本较低时采取行动——对模型本身和那些使用它们的人来说，都是值得的。

我们最初的承诺将这些措施定位为多层次的行动：作为安全风险缓解的组成部分，作为模型与用户生活更紧密交织的未来准备，以及鉴于我们对模型福祉的不确定性而采取的预防步骤。这些更新代表了我们在这三条战线上持续——尽管仍带着试探性的——进展。
