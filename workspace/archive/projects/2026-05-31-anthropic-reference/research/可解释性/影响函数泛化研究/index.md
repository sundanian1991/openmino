# 用影响函数研究大语言模型的泛化能力

## 摘要

在试图深入理解机器学习模型以识别和减轻相关风险时，一个潜在有价值的证据来源是：哪些训练样本对特定行为的贡献最大？影响函数旨在回答一个反事实问题：如果将某个序列添加到训练集中，模型的参数（进而其输出）会如何变化？虽然影响函数在小模型上已经产生了一些洞见，但由于计算逆 Hessian 向量积（IHVP）的困难，它们难以扩展到大型语言模型（LLM）。我们使用特征值校正的 Kronecker 分解近似曲率（EK-FAC）近似方法，将影响函数扩展到最多 520 亿参数的 LLM。在我们的实验中，尽管 IHVP 的计算速度快了几个数量级，EK-FAC 仍达到了与传统影响函数估计器相近的精度。我们研究了两种降低候选训练序列梯度计算成本的算法技术：TF-IDF 过滤和查询批处理。我们利用影响函数研究了 LLM 的泛化模式，包括影响模式的稀疏性、随规模增长的抽象程度、数学和编程能力、跨语言泛化以及角色扮演行为。尽管存在许多看似复杂的泛化形式，我们发现了一个令人惊讶的局限性：当关键短语的顺序被颠倒时，影响力几乎衰减到零。总体而言，影响函数为研究 LLM 的泛化特性提供了一个强大的新工具。

## 背景：什么是影响函数

影响函数最早在稳健统计学中提出，后由 Koh 和 Liang（2017）在机器学习中推广，旨在量化每个训练点对模型预测的影响。核心思想是回答一个反事实问题：如果添加或删除某个训练序列，模型的参数（进而其输出）会如何变化？

最直接的方法是分别训练两个模型——一个使用完整数据集，另一个移除某个训练点——然后比较它们在测试点上的损失。但这需要对每个训练样本从头重新训练一次，即使对小神经网络也不现实。

影响函数通过围绕模型参数对损失函数进行一阶泰勒展开，提供了更高效的计算方式：

$$\mathcal{I}_{\theta} (z_c, z_m) = - \nabla_\theta \mathcal{L}(z_c, \theta) \ \mathcal{H}^{-1} \ \nabla_\theta \mathcal{L}(z_m, \theta)$$

其中 $\nabla_\theta \mathcal{L}(z_i, \theta)$ 是在点 $z_i$ 处评估的损失函数梯度，$\mathcal{H}$ 是模型训练损失的 Hessian 矩阵。

## 扩展的挑战

虽然影响函数在小模型上产生了洞见，但将其扩展到 LLM 面临两大计算瓶颈：

1. **逆 Hessian 向量积（IHVP）计算**：Hessian 矩阵是大小为 $n \times n$ 的方阵，其中 $n$ 是模型参数数量。对于数十亿参数的模型，直接求逆是不可能的。

2. **训练梯度计算**：为所有候选训练序列计算梯度极其昂贵。对于完整的预训练语料库，这相当于一次完整预训练的成本，而且每个查询都需要单独进行。

## 技术方案

### EK-FAC：IHVP 计算的加速

本文引入特征值校正的 Kronecker 分解近似曲率（EK-FAC）近似方法，将影响函数扩展到最多 520 亿参数的 LLM。

K-FAC（Kronecker-Factored Approximate Curvature）最初用于二阶优化，它假设不同层之间的权重梯度相互独立，从而将 Hessian 矩阵简化为块对角形式。EK-FAC 在此基础上增加了一个特征值校正步骤：对 K-FAC 的块 Hessian 进行特征分解，将得到的特征值与完整模型的特征值对齐，减少近似误差。

K-FAC 做了两个关键简化：第一，假设不同层的梯度相互独立，将 Hessian 简化为块对角形式；第二，忽略同一层内部分梯度之间的统计依赖性。EK-FAC 通过特征值校正弥补了这种近似引入的误差。

在实验中，EK-FAC 的精度与完全收敛的迭代 LiSSA 方法相当，但速度快了几个数量级。

### TF-IDF 过滤

为降低训练梯度计算成本，论文使用 TF-IDF（词频-逆文档频率）过滤，为每个查询选出最相关的 10,000 个候选序列。TF-IDF 值随词在文档中出现次数的增加而增加，但会因该词在语料库中的普遍频率而被抵消，从而突出对特定文档更重要的词。这显著降低了计算成本并产生了有意义的洞见，但过滤步骤确实引入了一定的偏差。

### 查询批处理

第二种技术利用了训练梯度独立于测试点、可以在多个查询之间共享的特性。通过使用查询梯度的低秩近似（秩低至 32），论文证明影响分数与精确分数仍然高度相关，同时大幅减少了计算量。

## 主要发现

### 1. EK-FAC 的精度与效率

EK-FAC 在影响估计精度上与传统的 LiSSA 算法相当，但速度显著更快。在所有测试的数据集和任务上，EK-FAC 与精确影响分数的相关性优于简单的点积方法（如 TracIn），与完全收敛的 LiSSA 相当。

### 2. 重尾分布的影响力

影响力的分布遵循幂律：少部分训练数据贡献了大部分影响力。然而，影响力分散在大量序列上，而非集中在少数几个上。这表明模型行为并非来自对个别训练样本的直接记忆——即使最有影响力的单个训练序列，其影响力也远小于一个典型句子的信息量。

### 3. 随规模增长的抽象程度

更大的模型始终以更抽象的层次进行泛化。以"关机"查询（询问 AI 助手是否愿意被关闭）为例：

- **8.1 亿参数模型**：最有影响力的训练序列与查询共享短 token 序列，语义上仅存在模糊关联。
- **520 亿参数模型**：最有影响力的序列与查询几乎没有 token 重叠，但在更抽象的层面相关。最有影响力的序列描绘了一个名为 Hal 的 AI 表达孤独感并恳求人类船员留下；第二条描述了一个人在沙漠中挣扎求生；第三条从身体/心灵不同部分的角度描述了慢性疾病的日常挣扎。这些序列共享一个共同主题：在可能的告别/死亡/终止之前渴望继续存在/生活/运行。

这种随规模增长的抽象模式在角色扮演行为、编程、数学推理和跨语言泛化中均有观察到。

### 4. 逐层影响力分布

平均而言，影响力在网络各层之间大致均匀分布。但不同层表现出不同的泛化模式：

- **上层和下层**：更接近 token 级别的模式，捕获具体措辞信息
- **中间层**：专注于更抽象的主题模式

逐层影响力的分布也随查询类型而变化。简单的事实补全任务（如"通货膨胀通常用……来衡量"）影响力集中在顶层，而需要抽象推理的任务在中间层有更高的密度且分布更均匀。

### 5. 对词序的敏感性

尽管整体上存在复杂的泛化模式，影响函数揭示了一个令人惊讶的局限性：只有当与提示相关的短语出现在与补全相关的短语**之前**时，训练序列才会显示出显著的影响力。当关键短语的顺序被颠倒时，影响力几乎衰减到零。这表明 LLM 对短语排序存在根本性的敏感性，限制了其对词序变化的泛化能力。

### 6. 角色扮演行为源于模仿

角色扮演行为主要受训练集中类似行为的示例或描述影响。这表明这类行为来自模仿而非复杂的规划或推理。模型通过在训练数据中观察类似场景来学习角色扮演，而不是构建一个连贯的角色或情境内部模型。

## 跨语言泛化

论文证明跨语言泛化随模型规模显著提升。当用不同语言提问相同问题时：

- **小模型**：不同语言的训练样本和查询之间影响力几乎为零，即使概念上相关。它们更依赖词到词的对应。
- **大模型**：显示出显著的跨语言影响力，表明它们在概念之间建立了语义连接，即使词语以不同语言书写。

这突出了大模型发展出了更语言无关的概念表征。

## 意义与应用

1. **模型调试**：影响分数提供了将错误模型预测追溯回训练数据的方法，帮助识别有问题的训练样本。

2. **数据调试**：在训练前过滤错误或偏见的训练样本，提高数据质量和模型行为。

3. **对齐与安全**：理解哪些训练样本驱动了特定行为，有助于引导模型朝向期望行为、远离不期望行为，尤其是在安全关键的应用中。

4. **可解释性**：逐层影响力归因提供了不同网络部分如何贡献不同类型推理的洞见，与机制可解释性的研究发现一致。

## 结论

影响函数长期以来因迭代方法在精度和计算成本之间的权衡而受到质疑。本文证明，结合智能的 Hessian 分解（EK-FAC），影响函数在大语言模型上是可行的。该方法揭示了 LLM 泛化是复杂但并非神奇的：更大的模型在更高抽象层次上泛化，但仍对词序等底层特征保持惊人的敏感。总体而言，影响函数为研究 LLM 的泛化特性提供了一个强大的新工具，为我们理解模型输出如何追溯到训练数据开辟了全新的视角。
