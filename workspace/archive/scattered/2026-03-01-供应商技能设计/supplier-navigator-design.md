# 供应商管理技能架构设计

> 一张供应商管理的完整地图，覆盖全生命周期，提供精准指引

---

## 一、技能定位

**名称**：supplier-navigator（供应商导航）

**核心价值**：
- 像地图一样，为供应商管理提供精准指引
- 覆盖供应商管理全生命周期
- 根据上下文信号自动判断场景，提供针对性支持

**触发机制**：
- 提到任何供应商相关话题
- 提供供应商数据时
- 需要做供应商相关决策时
- 定期分析时（月度、季度）

---

## 二、上下文信号识别

### 2.1 触发信号

| 信号类型 | 具体信号 | 触发场景 |
|----------|----------|----------|
| **名词触发** | 供应商名称、BPO、外包团队 | 提到具体供应商 |
| **动作触发** | 准入、评估、沟通、清退、谈判 | 提到管理动作 |
| **数据触发** | KPI、业绩、排名、趋势 | 提供数据时 |
| **问题触发** | 风险、异常、投诉、下滑 | 提到问题时 |
| **时间触发** | 月初、季度末、年底、合同到期 | 定期任务 |

### 2.2 状态判断信号

| 信号组合 | 当前状态 | 推荐动作 |
|----------|----------|----------|
| 新供应商 + 无数据 | 准入评估阶段 | 启动准入诊断管线 |
| 有供应商 + 有数据 + 正常 | 日常管理阶段 | 定期业绩诊断 |
| 有供应商 + 数据异常 | 问题诊断阶段 | 启动业绩诊断管线 |
| 有供应商 + 风险信号 | 风险预警阶段 | 启动风险信号采集 |
| 合同到期 + 需续签 | 合同决策阶段 | 启动续签评估 |
| 多次异常 + 无改善 | 清退决策阶段 | 启动清退评估 |

### 2.3 供应商分类信号

| 信号组合 | 供应商类型 | 管理重点 |
|----------|------------|----------|
| 核心业务 + 高依赖 | 核心供应商 | 深度合作、风险预防 |
| 重要业务 + 有替代 | 战略供应商 | 价值创造、关系维护 |
| 一般业务 + 标准化 | 普通供应商 | 效率优先、成本控制 |
| 问题多 + 改善难 | 问题供应商 | 限期改进或清退 |

### 3.1 准入期

**场景**：评估新供应商是否准入

**触发信号**：
- 提到"新供应商"、"准入"、"引入"
- 提到具体供应商名称但系统中无记录
- 业务线提出新增需求

**核心管线**：准入诊断管线

```
Phase 1: 背景采集
  - 基本信息、资质、团队、产能
  - 输出：供应商画像初稿

Phase 2: 核心验证（6个问题）
  Q1: 为什么必须是这家？
  Q2: 没有这家会怎样？
  Q3: 谁最需要？说具体名字
  Q4: 最小试单是什么？
  Q5: 有成功案例吗？
  Q6: 3年后还需要吗？

Phase 3: 前提挑战
  - 这是必须的吗？
  - 能内部消化吗？
  - 能合并到其他供应商吗？

Phase 4: 方案生成
  方案A: 全面合作（高风险高收益）
  方案B: 试点合作（低风险验证）
  方案C: 暂缓准入（观察等待）

Phase 5: 输出文档
  - 准入评估报告

Phase 6: 信号综合
  - 推荐等级：强推/推荐/观望/不推
```

**输出物**：
- 准入评估报告
- 推荐等级
- 关键风险点
- 后续行动建议

---

### 3.2 成长期

**场景**：新供应商刚开始合作，需要关注磨合

**触发信号**：
- 新准入后3个月内
- 合作初期出现磨合问题
- 试单阶段

**核心管线**：磨合诊断管线

```
Phase 1: 数据采集
  - 试单数据、问题记录、反馈

Phase 2: 磨合评估
  Q1: 试单结果如何？
  Q2: 主要问题是什么？
  Q3: 对方响应如何？
  Q4: 是否需要调整合作方式？
  Q5: 是否继续合作？

Phase 3: 方案生成
  方案A: 扩大合作
  方案B: 维持现状观察
  方案C: 缩小范围
  方案D: 终止合作

Phase 4: 输出
  - 磨合评估报告
  - 合作调整建议
```

---

### 3.3 稳定期

**场景**：正常合作中的供应商日常管理

**触发信号**：
- 提到供应商名称 + 正常状态
- 月度/季度业绩数据
- 定期沟通

**核心管线**：业绩诊断管线

```
Phase 1: 数据采集
  - KPI数据、趋势、对比

Phase 2: 核心诊断（6个问题）
  Q1: 核心指标到底怎样？
  Q2: 根本原因是什么？
  Q3: 谁受影响最大？
  Q4: 能改进吗？要什么？
  Q5: 去现场看过吗？
  Q6: 3个月后会怎样？

Phase 3: 前提挑战
  - 这是偶发还是趋势？
  - 是供应商问题还是我们问题？

Phase 4: 方案生成
  方案A: 支持改进
  方案B: 观察警告
  方案C: 降级处理

Phase 5: 输出文档
  - 业绩诊断报告
  - 行动计划

Phase 6: 信号综合
  - 风险等级：高/中/低
  - 行动优先级：立即/本周/本月
```

**风险预警机制**：

| 风险信号 | 风险等级 | 推荐动作 |
|----------|----------|----------|
| 连续2月不达标 | 高 | 立即诊断+沟通 |
| 核心人员离职 | 高 | 现场了解+评估 |
| 单月不达标 | 中 | 关注趋势 |
| 沟通响应变慢 | 中 | 了解原因 |
| 小问题增多 | 低 | 记录观察 |

---

### 3.4 退出期

**场景**：评估是否需要清退供应商

**触发信号**：
- 多次异常无改善
- 提到"清退"、"替换"
- 风险等级持续高
- 合同到期不续签

**核心管线**：清退评估管线

```
Phase 1: 历史回顾
  - 合作历史、问题记录、改善尝试

Phase 2: 清退评估（6个问题）
  Q1: 核心问题是什么？尝试过什么？
  Q2: 还有改善可能吗？
  Q3: 清退的替代方案是什么？
  Q4: 清退的影响是什么？
  Q5: 过渡期需要多久？
  Q6: 清退后有什么风险？

Phase 3: 方案生成
  方案A: 立即清退
  方案B: 限期改进后清退
  方案C: 暂缓清退（无替代）

Phase 4: 输出文档
  - 清退评估报告
  - 过渡计划
  - 风险预案
```

## 四、日常应用场景

### 4.1 场景一：提到某个供应商

**信号**：你提到"岐力"、"阅文"等供应商名称

**我的响应**：
1. 读取该供应商的档案
2. 判断当前阶段（准入/成长/稳定/退出）
3. 判断当前状态（正常/问题/风险）
4. 提供针对性支持

**示例对话**：
```
你：岐力最近怎么样？
我：【状态判断】岐力 - 稳定期 - 正常
   【数据概览】上月KPI：92分（↑3），排名：第5
   【最近动态】无异常信号
   【建议】本周可做常规沟通，重点了解下季度产能规划
```

### 4.2 场景二：提供业绩数据

**信号**：你发送供应商KPI表格或排名

**我的响应**：
1. 解读数据，识别异常
2. 对比历史趋势
3. 识别需要关注的供应商
4. 输出分析报告和行动建议

**示例对话**：
```
你：[发送业绩表格]
我：【数据分析】
   - 岐力：92分（↑3），连续3月上升，表现稳定
   - 翰锐：78分（↓12），首次跌破80，需关注
   - 阅文：新准入，试单阶段

   【重点关注】翰锐 - 本月下滑最大
   【建议】本周沟通，了解原因
   【问题清单】
   Q1: 下滑原因是什么？
   Q2: 是偶发还是趋势？
   Q3: 需要什么支持？
```

### 4.3 场景三：需要做决策

**信号**：你问"这家供应商怎么样"、"要不要续签"等

**我的响应**：
1. 采集相关信号
2. 调用对应管线
3. 输出评估和建议

**示例对话**：
```
你：阅文这家供应商怎么样？
我：【准入诊断管线启动】

   Phase 1 背景采集：
   - 基本信息：浙江阅文，车抵贷新供应商
   - 状态：合同未完成

   Phase 2 核心验证：
   Q1: 为什么必须是这家？
       → 车抵贷新业务，需要有经验的供应商
   Q2: 没有这家会怎样？
       → 业务无法启动
   Q3: 谁最需要？
       → 信用卡业务线，具体负责人待确认

   【信号采集】
   +1 需求明确（新业务需要）
   +1 有具体业务线
   -1 合同未完成
   评级：观望（需完成合同后推进）

   【下一步】
   1. 确认业务线具体负责人
   2. 完成合同签订
   3. 设计最小试单方案
```

### 4.4 场景四：准备沟通

**信号**：你说"要和岐力沟通"、"准备周会"等

**我的响应**：
1. 读取供应商状态
2. 生成沟通要点
3. 准备问题清单

**示例对话**：
```
你：要和岐力沟通，帮我准备一下
我：【沟通准备】岐力 - 稳定期

   【最近状态】
   - KPI：92分（↑3），排名第5
   - 趋势：连续3月上升
   - 合作：稳定，无异常

   【沟通要点】
   1. 肯定近期表现（具体数据支撑）
   2. 了解下季度产能规划
   3. 询问是否需要支持
   4. 探讨合作深化可能

   【问题清单】
   Q1: 下季度人力规划如何？
   Q2: 有什么需要我们支持的？
   Q3: 对现有合作模式有什么建议？

   【预警问题】
   - 最近团队稳定性如何？
   - 有什么行业动态？
```

## 五、输出物定义

### 5.1 状态报告

**触发**：提到供应商名称

**格式**：
```
【供应商状态】{名称}
阶段：{准入/成长/稳定/退出}
状态：{正常/关注/风险}
数据：{最新KPI、趋势、排名}
动态：{最近事件、变化}
建议：{下一步行动}
```

### 5.2 诊断报告

**触发**：业绩诊断管线、准入诊断管线

**格式**：
```
【{诊断类型}报告】{名称}

一、背景
   {基本情况和问题陈述}

二、核心发现
   Q1: {问题和答案}
   Q2: {问题和答案}
   ...

三、信号分析
   正向信号：{列表}
   负向信号：{列表}
   评级：{等级}

四、方案对比
   方案A：{描述}
   方案B：{描述}
   推荐：{选择}

五、行动计划
   1. {行动项}
   2. {行动项}
```

### 5.3 沟通准备

**触发**：准备沟通

**格式**：
```
【沟通准备】{供应商名称}

状态概览：{阶段+状态}
数据摘要：{关键指标}

沟通要点：
1. {要点}
2. {要点}

问题清单：
Q1: {问题}
Q2: {问题}

预警问题：
- {问题}
```
