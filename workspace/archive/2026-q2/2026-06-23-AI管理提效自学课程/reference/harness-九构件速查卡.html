核心心智

核心命题 Model + Harness = Agent。 模型提议，harness 执行、校验、修复。
失败归因：90% 是 harness 没接住，不是模型笨。

PEV Loop（动态骨架）

阶段谁做关键约束
Propose 提议模型—
Execute 执行harness修复在前，执行在后
Verify 校验harness必须独立于提议的模型

三个设计原则（来自你的 Repair Layer）

  1. 宽容而非严格：schema 校验前修复，不当裁判判错
  2. 有 Schema 更精准：传 schema 时只修复已知类型
  3. 执行顺序有讲究：先还原结构，再补形状

常见反模式（见之即改）

  • 模型自己校验自己 → V 阶段失去独立性
  • 失败后无脑重试 → 低密度多 token 陷阱
  • schema 太严 → 可恢复噪声被拒绝而非修复
