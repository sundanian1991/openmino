# Plan

## Brief

- **目标读者**：对 AI 有初步使用经验、但尚未把 token 消耗当作核心能力的脑力工作者；包含产品经理、创业者、研究员、工程师、内容创作者。
- **目标语言**：中文（跟随源语言）
- **文章类型**：`longform`（完整长文 / 归档 / 深度阅读）
- **信息保留比例**：~100%（保留原文完整论证链、案例、计算、消费观、四条件与结尾宣言；只删重复过场句和微信底部 UI 噪音）
- **必须保留的信息**：
  - 标题与核心命题：「Token消耗」的本质是调用智能的能力
  - 项羽 vs 韩信的 ROI / 调用量对比
  - 2 亿 token ≈ 93 元的三段 source 计算（个人月包、Peter Steinberger API 账单、X 用户自测）
  - 月薪 12000 元以上脑力工作者、10% 可支配收入投入智能的消费观
  - 四个不变的第一性原理条件
  - 前额叶减负、谷氨酸减少的生理机制论点
  - 结尾关于「超体」与 10 万倍调兵能力差距的宣言
- **可删减的信息**：
  - 重复出现的「有人会说……」反问句式可适度合并
  - 文中关于迪迦奥特曼的玩笑可保留但不必重复
  - 作者简介中过长的头衔列表可精简为关键身份
- **语气**：克制而有锋芒的出版叙事；不煽情，但保留作者原作的断言感
- **主要观点**：
  1. Token 消耗量 = 调用智能的「用兵能力」，是未来个体竞争力的核心指标。
  2. 智能已极其便宜且持续降价；限制因素不是成本，而是个人调用能力。
  3. 先练到日耗 2 亿 token，再谈投入产出比。
- **阅读目标**：读完理解为什么「日耗 2 亿 token 是底线」，并愿意把 token 消耗作为刻意练习指标。
- **版式宽度**：`regular`
- **TOC**：开
- **配图策略**：`none`（不使用外部图片；所有视觉增强通过 Raw / SVG / 自定义排版实现，保持离线可打开）
- **封面**：开。构图想法：tufte 主题 + 模板 C「上下分屏」或模板 E「极简框」；主视觉为细线网格 + 抽象「token 流 / 兵符」SVG，标题与副题压图，保持低饱和与克制线条。

## Theme

- **选定主题**：`press`
- **理由**：用户二次确认切换。文章是长文宣言，核心论证依赖「成本计算」「ROI 对比」「四条件」等证据链；`press` 与 tufte 同宗共享 Data-Ink 纪律，但更像「被认真设计过的书」：衬线字体、氧化血红主色、更大字号对比，让长文在克制中更有阅读节奏，解决「太素」问题。
- **与源材料的冲突**：无。
- **当前信息密度下的表现建议**：
  - 信息保留从 `100% longform` 精简到 `~70% longform`：删除旁枝案例与重复论证，保留核心命题、计算链条与结尾宣言。
  - 正文仍是主体；Raw 作为「点亮关键概念」的内插图，数量克制、线条化、低装饰。
  - 封面与关键 section 加入少量 AI 生成的 editorial 暖调图片，避免纯 SVG 的单调，同时保持离线可打开的 single-file HTML。
