01 · Problem

AI 生成界面的一致性问题

当你让 Claude 写一个设置页面，每次生成的按钮样式、间距、圆角都不同。Aham UI 的解法是：把设计决策写成 AI 能解析的结构化数据。

没有 Aham UI
每次让 AI 生成界面 → 每次风格不同 → 手动调一致性 → 时间浪费
有 Aham UI
AI 读 AGENTS.md + tokens.json → 按规范生成 → 风格一致 → 零调优
扩展能力
不只 Web UI → PPT / Word / 数据报告都能套用同一套设计语言
02 · Architecture

七层规范体系

从设计原则到具体组件，每一层都有对应的文件。AI 按需读取，不全量加载。

L1
设计原则
克制、功能性、一致性。参考 Apple HIG 但用自定义值。冷色调、单 accent 色、无装饰。
DESIGN.md §1
L2
设计基础
三层 Token 体系：Primitive（原始色值）→ Semantic（语义映射，如 color-danger）→ Component（组件级，如 btn-primary-bg）。
tokens.json
L3
组件库
~65 个组件，8 大类：Navigation / Input / Data Display / Feedback / Layout / Pattern / Media / System。每个组件有 HTML+CSS+JS 实现。
aham-ui.css + aham-ui.js
L4
组合规则
组件怎么拼成页面。间距规则、网格系统、响应式断点。不告诉你"用什么组件"，告诉你"组件之间怎么配合"。
DESIGN.md §4
L5
页面模式
常见页面结构的参考实现：Dashboard / Settings / List Detail / Form / Landing。不是模板，是组合范式。
examples/
L6
媒体扩展
把同一套设计语言延伸到 Office 文档（PPT/Word）。颜色、字体、间距规则不变，只是输出载体变了。
aham-ui-office.md
L7
AI 入口
告诉 AI"你怎么用这套系统"。文件加载顺序、Token 引用方式、组件选择逻辑。这是 AI 的使用手册。
AGENTS.md

03 · Tokens

三层 Token 体系

从原始色值到组件级变量，三层映射让设计决策可追溯、可替换。

// tokens.json — 三层结构 { "primitive": { "blue-500": "#336EE8", // 原始色值 "gray-900": "#1A1A1A", "radius-md": "8px" }, "semantic": { "color-accent": "{primitive.blue-500}", // 语义映射 "color-text": "{primitive.gray-900}", "radius-base": "{primitive.radius-md}" }, "component": { "btn-primary-bg": "{semantic.color-accent}", // 组件级 "btn-primary-radius": "{semantic.radius-base}" } }

05 · Design Language

冷色调 · 克制 · 金属感

参考 Apple HIG 的方法论，但用自定义值。单 accent 色 #336EE8，其余全部灰阶。无渐变、无阴影、无装饰动画。

维度规则说明
配色
单 accent + 灰阶#336EE8 唯一强调色，其余全部灰度层级
圆角
4px / 8px / 12px按钮 999px pill，卡片 8px，输入框 4px
阴影
无 box-shadow层级通过边框和背景色阶实现
字体
系统字体栈-apple-system, SF Pro, Segoe UI，零加载
动效
150ms ease-out仅 hover/focus 状态变化，无过渡动画
暗色
完整暗色模式不是浅色取反，是独立设计的暗色层级
无障碍
WCAG AA所有文本对比度 ≥ 4.5:1
RTL
支持阿拉伯语/希伯来语从右到左布局

06 · Comparison

Aham UI vs nian-design

08 · Usage

AI 怎么用这套系统

1
AI 读 AGENTS.md
了解文件加载顺序、Token 引用方式、组件选择逻辑。这是 AI 的"使用手册"。
2
AI 读 tokens.json
加载设计基础：颜色、间距、圆角、字体。生成界面时引用 Token 名，不硬编码值。
3
AI 读 DESIGN.md 对应层
按需读取组件规范、组合规则、页面模式。不全量加载，按场景查表。
4
AI 生成 HTML/CSS
按规范生成代码。按钮用 btn-primary，间距用 Token，圆角用 Token。风格自动一致。
与 nian-design 的协作可能
Aham UI 的 Token 系统（JSON 三层）比 nian-design 的 CSS :root 更适合机器解析。如果把 Aham UI 的 Token 层接入 nian-design 的施工流水线，可以实现：AI 读 Aham Token → 选 nian 变体 → 注入 Aham 组件 → 校验输出。两套系统互补而非竞争。
