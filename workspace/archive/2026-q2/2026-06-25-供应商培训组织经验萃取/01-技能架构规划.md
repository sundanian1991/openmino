# 供应商培训组织 · 技能架构规划

> **定位**：把培训组织经验从"知识库（素材）"升级为"技能（引擎）"的完整架构。
> **关系**：知识库是静态素材 → 技能是动态引擎 → 物料/方案是可用产出。
> **状态**：架构规划完成，待逐个打磨技能边界后进入开发。
> **日期**：2026-06-25

---

## 一、核心架构：1 编排器 + 8 环节技能

不做"全流程一个技能"（太重跑不动），而是**1个编排器调度多个环节技能**，跟 `nian-orchestrator` 一个思路。

```
┌─────────────────────────────────────────────────────────────┐
│                     编排器（Orchestrator）                     │
│   识别当前在培训的哪一步 → 调度对应技能 → 串起数据流             │
└─────────────────────────────────────────────────────────────┘
       │
       ├── ⓪ 需求调研    → 供应商真实问题（分类清单）
       ├── ① 物料生成    → 会前/会中/会后全套材料
       ├── ② 备课辅导    → ②a 亮点挖掘 + ②b 内容梳理
       ├── ③ 现场引导    → 追问清单 + 突发应对 cheat sheet
       ├── ④ 总结转化    → 结构化纪要/要点卡/案例库/宣传素材/评价素材
       ├── ⑤ 宣传生成    → 圈内/圈外多种内容
       ├── ⑥ 讲师评价    → 打分/排名/荣誉建议
       └── ⑦ 效果追踪    → T+30 回访 + 复盘 + 回到⓪
```

---

## 二、数据流（技能间的串联）

技能不是孤立的，数据必须流转。完整链路：

```
⓪需求调研
   │ 输出：分类问题清单（按主题/痛点/频次）
   ↓
①物料生成 ← 培训基本信息
   │ 输出：会前材料包（含问题清单作为主讲备课输入）
   ↓
②a亮点挖掘 ← ⓪的问题清单（亮点要对应痛点）
   │ 输出：锁定主讲供应商 + 亮点定位
   ↓
②b内容梳理 ← ⓪的问题清单 + 供应商口述/草稿
   │ 输出：完整培训大纲（交可视化技能出PPT/议程图）
   ↓
③现场引导 ← ②b大纲 + 议程
   │ 输出：追问清单 + 时间预警 + 突发应对卡片
   ↓
[现场执行] → 现场记录（文本/录音转写）
   ↓
④总结转化 ← 现场记录
   │ 输出：结构化纪要 + 要点卡 + 案例库条目 + 宣传素材 + 讲师评价素材
   ↓
⑤宣传生成 ← ④的宣传素材
   │ 输出：4种圈内内容（群内/朋友圈/内部群/1v1）+ [可选]外部内容
   ↓
⑥讲师评价 ← ④的评价素材 + 听众打分 + 运营官观察
   │ 输出：讲师单场打分 + 累计排名 + 荣誉建议
   ↓
⑦效果追踪 ← T+30回访记录
   │ 输出：回访结构化记录 + 极简复盘表
   ↓
（回到⓪）→ 遗留问题进入下一期需求调研
```

**编排器职责**：管理这个数据流，确保每个技能拿到它需要的上游输入。

## 三、9个技能详细定义

### ⓪ 需求调研（research-collector）

| 维度 | 定义 |
|------|------|
| **做什么** | 每期培训前，主动1v1调研供应商的真实痛点，分类整理 |
| **输入** | 1v1调研记录（微信/语音/笔记）+ 往期遗留问题 |
| **输出** | 分类问题清单（按主题/频次）+ 候选讲师建议（哪家在这个痛点上做得好）|
| **触发时机** | 培训前3-4周 |
| **与知识库的关系** | 输出喂给②a②b作为备课方向；问题清单进入①的会前材料 |
| **核心价值** | 让备课从"拍脑袋定主题"变成"从痛点反推" |

**技能边界**：
- ✅ 做：收集、分类、归并相似问题、反推候选讲师
- ❌ 不做：不代替运营官做1v1沟通（人际沟通靠人）；不决定最终主题（运营官拍板）

---

### ① 物料生成（training-material-generator）

| 维度 | 定义 |
|------|------|
| **做什么** | 根据培训基本信息，生成会前/会中/会后的全套标准化材料 |
| **输入** | 培训基本信息（主题/时间/地点/主讲/议程/参会者）+ ⓪问题清单 |
| **输出** | 会前材料包（邀请函/会前须知/行程确认/课前预习表/议程图）+ 会中材料（事中对照记录表/签到表/追问清单）+ 会后材料（评价问卷/回访话术）|
| **触发时机** | 确定培训基本信息后 |
| **与知识库的关系** | 直接调用知识库的所有模板（培训通知模板/参会者信息表/行程设备清单等）|
| **核心价值** | 一次性生成全套材料，告别"散落难找、每次重做" |

**技能边界**：
- ✅ 做：基于模板+输入信息，填充生成完整文档
- ❌ 不做：不做视觉设计（议程图/数据看板交给可视化技能）；不发送通知（运营官手动发）

**输出格式**：docx（打印用）+ md（编辑用）+ 可交可视化技能的HTML骨架

---

### ② 备课辅导（分两个子技能）

#### ②a 亮点挖掘（insight-miner）

| 维度 | 定义 |
|------|------|
| **做什么** | 基于供应商数据和⓪的问题清单，识别"哪家在哪个痛点上做得好" |
| **输入** | ⓪分类问题清单 + 供应商业绩数据/日常观察笔记 |
| **输出** | 候选讲师 + 亮点定位（"毅航在复贷留存上S率50%，对应'人员留存'痛点"）|
| **触发时机** | ⓪完成后 |
| **核心价值** | 解决"全靠人找"的发现端，让讲师发现有数据依据 |

**技能边界**：
- ✅ 做：从数据/观察中识别异常亮点，匹配痛点
- ❌ 不做：不评价讲师能不能讲（转化靠②b和人际）；不接触供应商

#### ②b 内容梳理（content-organizer）

| 维度 | 定义 |
|------|------|
| **做什么** | 帮讲师把零散的口述/草稿/材料，梳理成完整的培训大纲 |
| **输入** | ⓪问题清单 + 讲师提供的素材（口述录音转写/微信草稿/现有PPT/笔记）|
| **输出** | 完整培训大纲（按5页模板结构：背景→做法→效果→坑→总结），交可视化技能出PPT |
| **触发时机** | 锁定讲师后 |
| **核心价值** | 解决备课最耗时的"多轮打磨"——技能完成初版结构化，运营官只需微调 |

**技能边界**：
- ✅ 做：结构化、补充逻辑、提炼要点、对照问题清单确保覆盖
- ❌ 不做：不编造数据（讲师没给的数字不补）；不替代多轮打磨的人际沟通

**关键约束**：输入的素材形态多样（录音转写/微信/草稿），技能要能处理多种格式。

---

### ③ 现场引导（facilitation-copilot）

| 维度 | 定义 |
|------|------|
| **做什么** | 培训前生成现场使用的"弹药库"——追问清单+时间预警+突发应对 |
| **输入** | ②b大纲 + 议程 + 参会者名单（层级） |
| **输出** | 追问清单（按模块+按层级分层）+ 时间预警线（80%/100%/120%）+ 突发应对cheat sheet（冷场/超时/主讲出状况）|
| **触发时机** | 培训前1-2天 |
| **形态** | 打印版cheat sheet（A4）或手机查看版 |
| **核心价值** | 把"现场靠随机应变有遗憾"变成"提前备好弹药库" |

**技能边界**：
- ✅ 做：基于大纲预生成追问和预案
- ❌ 不做：不实时陪会（不是AI开会）；不做现场记录（运营官手记，事后交④处理）

**追问分层逻辑**（核心能力）：对老板问战略、对主管问执行、对一线问方法——同模块不同层级不同追问粒度。

---

### ④ 总结转化（summary-hub）★ 中枢技能

| 维度 | 定义 |
|------|------|
| **做什么** | 一次培训的内容进来，派生5种产物（一鱼5吃）|
| **输入** | 现场记录（录音转文本/手记/聊天记录）+ ②b大纲 + 参会者反馈 |
| **输出** | ①结构化纪要 ②给学员要点卡 ③案例库条目 ④宣传素材 ⑤讲师评价素材 |
| **触发时机** | 培训后1-3天 |
| **核心价值** | 一次输入，5种产物，每种面向不同受众不同口径 |

**5种产物详述**：

| 产物 | 受众 | 口径 | 去向 |
|------|------|------|------|
| 结构化纪要 | 内容库 | 完整记录 | 案例库归档 |
| 给学员要点卡 | 参会者 | 可抄动作（1页） | 群内分发 |
| 案例库条目 | 新供应商 | 可学习的方法论 | 案例库 |
| 宣传素材 | ⑤宣传生成 | 故事化/数据化 | 喂给⑤ |
| 讲师评价素材 | ⑥讲师评价 | 讲授表现观察 | 喂给⑥ |

**技能边界**：
- ✅ 做：从原始记录提取结构、转化口径、生成多产物
- ❌ 不做：不替听众打分（打分是听众填问卷）；不做视觉版式（要点卡等交可视化）

**关键能力**：处理多种输入格式（录音转写/腾讯会议纪要/飞书妙记/手记），统一结构化。

### ⑤ 宣传生成（promotion-writer）

| 维度 | 定义 |
|------|------|
| **做什么** | 基于④的宣传素材，生成多种渠道的宣传内容 |
| **输入** | ④的宣传素材 + 培训基本信息 |
| **输出** | 群内预告/朋友圈/内部群推文/1v1私信（4种圈内）+ [可选]脱敏方法论文章（外部）|
| **触发时机** | 培训后3天内 |
| **与知识库的关系** | 调用[对外宣传模板与案例](../04-持续工具层/对外宣传模板与案例.md)的模板 |
| **核心价值** | 一次培训自动产出全套宣传，告别"每次从头写" |

**技能边界**：
- ✅ 做：基于模板和素材生成文案
- ❌ 不做：不决策渠道策略（圈内vs圈外是运营官决策，见宣传战略张力）；不发布（运营官手动发）

---

### ⑥ 讲师评价（instructor-evaluator）

| 维度 | 定义 |
|------|------|
| **做什么** | 收集听众打分+运营官观察，生成讲师单场评价+累计排名+荣誉建议 |
| **输入** | 听众问卷打分（4维度5分制）+ ④的评价素材 + 运营官现场观察 |
| **输出** | 讲师单场打分卡 + 累计排名表 + 荣誉触发建议（如"曹总累计2次，触发'联盟讲师'称号"）|
| **触发时机** | 培训后7天内（问卷回收后）|
| **与知识库的关系** | 输出更新讲师档案 + 荣誉体系（见[讲师发展机制](../01-筹备期/讲师发展机制.md)第五章）|
| **核心价值** | 让讲师评价和荣誉授予从"拍脑袋"变成"有数据依据" |

**4维度评价**（已确认）：
- 讲授能力（表达清楚/节奏控制）
- 案例质量（数据真实/可复制）
- 互动引导（引发讨论/回应提问）
- 专业深度（对主题的理解）

**荣誉触发逻辑**（技能自动判断）：
- 单场总分≥4.5 → "本场金牌讲师"
- 累计≥2次 → "联盟讲师"称号
- 累计≥3次含线下 → 金牌讲师候选
- 年度总分前3 → "京东供应商联盟年度金牌讲师"候选

**技能边界**：
- ✅ 做：算分、排名、触发荣誉规则、维护讲师档案
- ❌ 不做：不颁发荣誉（年度大会由人颁发）；不评价供应商轨道（那是单独算）

**技能边界**：
- ✅ 做：结构化回访记录、触发复盘提醒、归集遗留问题
- ❌ 不做：不替运营官做1v1回访（人际沟通靠人）；不替运营官做复盘判断

## 四、技能优先级（基于"先做出来、做减法"）

| 优先级 | 技能 | 为什么先做 |

| 🔴 P0 | ④ 总结转化 | 一次输入5种产物，效率提升最明显，是中枢 |
| 🔴 P0 | ① 物料生成 | 标准化程度最高，最容易技能化，模板已齐 |

## 五、与现有技能生态的衔接

这些新技能不是孤立的，跟已有技能生态有衔接点：

| 新技能 | 衔接的现有技能 | 衔接点 |
|--------|-------------|--------|
| ①物料生成 | docx / pptx / xlsx 技能 | 输出docx打印件、交ppt技能出议程图 |
| ②b内容梳理 | docx / word-formatter | 大纲输出docx |
| ③现场引导 | （无直接衔接） | 独立 |
| ④总结转化 | docx / word-formatter | 纪要/要点卡输出docx |
| ④→可视化 | nian-design / ppt:frontend-slides / viz-* | 要点卡/议程图/数据看板交可视化技能 |

**关键衔接**：④总结转化是枢纽，它的产物分流到⑤（宣传素材）、⑥（评价素材）、可视化技能（要点卡/议程图视觉化）。这是数据流的十字路口。

## 六、技能边界（已打磨确认）

### 贯穿原则：运营官是人决策中枢，技能只执行不越权

所有"带判断的决策"（选谁讲、讲什么主题、渠道策略、荣誉颁发）都是运营官拍板。技能的角色是**给建议/给素材/执行梳理**，不替代人的判断。

### 6个边界决策

| # | 边界问题 | 决策 | 说明 |
|---|---------|------|------|
| 1 | ②a vs ②b 输入边界 | **运营官拍板，②b只执行** | ②a给候选建议→运营官定最终讲师→②b拿到确认后只管梳理，不质疑人选 |
| 2 | ④ 输入格式 | **专门转写工具纯文本** | 只支持飞书妙记/讯飞等转写工具输出，不支持会议软件转写/手记/聊天记录。④核心能力=无结构文本→结构化提取 |
| 3 | ④→可视化交接物 | **文本+版式骨架** | ④输出含结构化标注（如"卡片组-3并列""数据高亮区"），可视化技能按骨架填充但保留视觉自由度 |
| 4 | ⑥ 打分来源 | **线上表单导出数据** | 听众用腾讯问卷/金数据/飞书表单填，运营官导出xlsx/csv丢给⑥，技能自动算分排名 |
| 5 | 编排器触发方式 | **半自动提醒** | 培训日期定后生成倒计时行动表，按节点提醒+确认执行。解决"没节奏被动带走"痛点 |
| 6 | 技能间状态共享 | **共享培训目录** | 每期培训建一个目录，所有技能读写，跟现有workspace习惯一致，天然形成归档 |
