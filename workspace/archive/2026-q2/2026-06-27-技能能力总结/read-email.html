Skill · v1.0.1 · JoyMail 统一入口

read-email

邮件全文读取、查询、梳理、总结、回复、发送、转发、文件夹管理——一个 SOAP 入口覆盖所有邮件场景。

核心突破

突破"只能读标题"限制——通过 EWS SOAP 协议获取邮件全文（正文 + 附件元数据 + 收发件人），支持多维度筛选和批量操作。

关键规则

默认时间规则

所有查询操作，用户未明确给出日期范围时，默认只查最近 1 天（按 DateTimeReceived）。 folders、detail 不受此限制。

二次确认规则

所有修改邮箱状态的操作（send / reply / forward / batch-*）必须对话交互层二次确认。 先展示摘要（收件人/主题/正文/附件），用户明确确认后才执行。

收件人解析规则

用户给中文名或 ERP 发邮件时，必须先调用 lookup-recipient 解析邮箱。 不要猜邮箱、不要跳过确认。

输出风格规则

结果优先、过程最小化。直接给邮件结果/结论/摘要/表格，不主动展开内部实现细节。 "查到 3 封未读，其中 1 封高优先级"，而不是"我先调用了 search 再调用了 summarize"。

search 筛选能力

维度参数说明
文件夹--folder / --folder-id / --folder-name指定搜索范围
发件人--sender邮箱地址匹配
收件人--recipient收件人匹配
主题--subject主题关键词子串匹配
正文--body-keyword预览/正文关键词（客户端过滤）
时间--after / --beforeYYYY-MM-DD 或 YYYY-MM-DD HH:MM:SS
已读/未读--read / --unread读取状态过滤
标签--category分类标签过滤
附件--has-attachments仅含附件的邮件
重要级别--importanceHigh / Normal / Low
翻页--max-pages控制扫描深度

常用示例

# 查看邮箱目录
bash scripts/run.sh folders --parent msgfolderroot

# 查询收件箱未读邮件
bash scripts/run.sh search --folder inbox --unread --limit 20

# 按发件人+主题查询
bash scripts/run.sh search --folder inbox --sender "zhangsan@jd.com" --subject "周报"

# 获取邮件全文
bash scripts/run.sh detail --item-id "AAMkADFlNG..."

# 最近7天未回复的重要邮件
bash scripts/run.sh pending-important --days 7

# 搜索→总结 管道
bash scripts/run.sh search --folder inbox --unread --limit 20 | bash scripts/run.sh summarize

# 回复邮件（带附件）
bash scripts/run.sh reply \
  --item-id "AAMkADFlNG..." \
  --body "收到，我今天处理完给你反馈。" \
  --attachments "/path/to/会议纪要.docx,/path/to/排期.xlsx"

# 新建邮件（带抄送）
bash scripts/run.sh send \
  --to "a@jd.com,b@jd.com" \
  --cc "c@jd.com" \
  --subject "项目同步" \
  --body "大家好，项目进展如下……"

# 按天统计
bash scripts/run.sh daily-stats --days 7

# 批量标记已读
bash scripts/run.sh batch-mark-read --item-ids "id1,id2,id3"

技术架构

组件说明
协议EWS SOAP（Exchange Web Services）
版本Exchange2016
入口scripts/run.sh
鉴权SSO Token → JWT Login → auth header
Token 缓存本地缓存 + 过期自动刷新 + 最多3次重试
默认端点https://mail-skill.jd.com/mail/api/clawmail/mailpost
