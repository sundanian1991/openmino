WORKFLOW · 全景

五个阶段，一条路径

01
定题与对象界定
TOPIC FRAMING
把模糊兴趣转化为清楚的策展对象。回答：展览真正研究什么、观众从哪里进入、哪些材料在范围内。
OUTPUT · 定题说明
02
叙事结构与页面模块
NARRATIVE MODULES
从策展文本进入页面结构。决定需要哪些页面、每个单元承载什么、观众如何移动。
OUTPUT · 页面类型表 + 模块 schema
03
策展视觉语言
VISUAL LANGUAGE
定义展览应如何被观看和操作。参照系、材料身份、字体层级、色彩系统、交互语法。
OUTPUT · 视觉语言说明
04
原型实现与标准化
PROTOTYPE
把确认的结构和视觉转化为可测试的原型。先测标准单元，再扩展到同类页面。
OUTPUT · 可运行代码 + 验证结果
贯穿

项目日志与方法回收 — 每轮迭代记录判断、固定结论、余留问题和可复用方法。不是可选项，是基础设施。

PRINCIPLES · 核心原则

四条规则

01
不跳阶段
每个阶段要求前置条件就绪才进入。定题没稳就做页面结构，后面一定返工。
02
可逆
任何阶段发现缺口都可以回退。阶段 4 暴露结构问题？回到阶段 2 补，不是用 CSS 掩盖。
03
可出入任意点
已有策展文本？从阶段 2 开始。已有视觉规范？从阶段 3 开始。不必从头走。
04
调度器不做实事
工作流只做诊断、引导、状态传递。实际工作按子技能的方法执行，不越俎代庖。
REFERENCE · 阶段对照

每个阶段回答什么

阶段 核心问题 前置条件 主要产出 常见偏差
01 定题 展览真正研究什么？ 无，可从零开始 定题说明 主题过宽 / 人物替代结构
02 叙事 需要哪些页面？怎么连？ 定题说明已确认 页面类型表 + 模块 schema 策展对象不清晰就做结构
03 视觉 展览应如何被观看？ 定题 + 页面结构就绪 视觉语言说明 一张图承担整页成立感
04 原型 怎么实现和验证？ 结构 + 视觉 + 一个成熟单元 代码 + 组件文档 + 验证 用 CSS 掩盖结构缺口
贯穿 日志 我们学到了什么？ 任何时候 判断记录 + 可复用方法 被当作可选项跳过

ENTRY POINT · 从哪进

五个问题，一个入口

01
策展对象明确了没有？
"我想做个关于 XX 的展览" → 已明确。"我有一堆材料" → 未明确，从阶段 1 开始。
02
有策展文本或大纲吗？
已有成文文本 → 可能有阶段 1 输出。只有零散材料 → 需要阶段 1。
03
页面结构定了没有？
知道有哪些页面、每个放什么 → 跳阶段 3。还没有 → 阶段 2。
04
视觉风格定了没有？
有设计规范、参考风格 → 跳阶段 4。还没有 → 阶段 3。
快速判断

只有模糊兴趣 → 阶段 1。有定题 → 阶段 2。有结构 → 阶段 3。有视觉 → 阶段 4。完成一轮要复盘 → 日志。

SUMMARY · 总结

先做出来，再做好。

01

不要在定题阶段纠结视觉风格——后面会回来

02

日志不是可选项——每个阶段结束记录一次判断

03

发现缺口就回退，不要用代码掩盖结构问题
