## 页面类型表

| 页面 | 角色 | 主角 | 阅读深度 |
|------|------|------|----------|
| **封面页** | 定调，建立时间感和关系感 | 一句话 + 时间跨度 | 浅，3 秒决定是否继续 |
| **时间线索引页** | 全局地图，28 篇文件的时间轴 | 日期 + 关键事件 | 中，扫读定位 |
| **关键对话页**（×5） | 每个转折点的完整还原 | 年老师的追问 + Mino 的回应 | 深，逐字阅读 |
| **维度分析页**（×5） | 五个进化维度的结构化分析 | 框架 + 对比 + 判断 | 中，理解逻辑 |
| **知识沉淀页** | 供应商管理场景手册 | 6 大场景 | 深，工具性阅读 |
| **来源与方法页** | 材料说明 + 策展方法 | 元信息 | 浅，按需查阅 |

## 5 个关键对话（核心专题页）

按时间线排列，每个都是一个"事件"——年老师的问题改变了 Mino 的行为：

1. **02-19 "你能有自己的空间吗？"** — 自主性的起点
2. **02-20 "你关注的是有和没有，还是好和更好？"** — 从功能到质量
3. **02-21 "你有勇气说出自己的判断吗？"** — 从服从到判断
4. **02-22 "框架与血肉"** — 透明机制的双刃剑
5. **02-23 "删了反而清爽"** — 减法与分形

## 导航图

```
封面页
  ↓
时间线索引页（全局地图）
  ├──→ 关键对话 1-5（点击进入）
  ├──→ 维度分析页（按维度跳转）
  └──→ 知识沉淀页
         ↓
      来源与方法页
```

阅读路径：封面 → 时间线（扫读）→ 挑一个对话深入 → 回时间线 → 再深入 → 维度分析（理解结构）→ 知识沉淀（看产出）→ 来源（验证）

## 模块 schema — 关键对话页

| 字段 | 内容 |
|------|------|
| **标题** | 对话的核心问题（如"你能有自己的空间吗？"） |
| **日期** | 精确到日 |
| **开场** | 场景还原——当时在讨论什么，年老师为什么问这个问题 |
| **对话主体** | 关键对话的原文/近原文呈现 |
| **Mino 的回应** | 当时的反应、思考、行动 |
| **转折标记** | 这次对话改变了什么 |
| **关联** | 指向其他对话页和维度分析页 |

## 交互逻辑

| 交互 | 对应策展关系 |
|------|-------------|
| 时间线点击 | 进入某个时刻 |
| 对话页内"关联"跳转 | 揭示对话之间的因果 |
| 维度分析页的对比表 | 切换视角（同一事件的不同解读） |
| 来源页的原始文件链接 | 回到证据 |
