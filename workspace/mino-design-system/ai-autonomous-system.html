Phase 01 — 准备
先核对现在的真相
开工前先确认当前的真实状态——最新的进度、版本、数据。别拿上一次缓存的旧状态当真相。基于过期状态干活，轻则白忙，重则覆盖掉别人在途的工作。
Phase 02 — 开工
干了再说，边干边对齐
别在计划上无限纠缠。先把最小可用的版本跑起来，让产出本身说话。需求往往是「干」出来的，靠「想」想不全。对齐是让意图和不断生长的产出互相补全。

Phase 01 · 准备阶段 — 先核对真相
01
Boot Principle
先核对现在的真相
AI 手里的「事实」往往是上一次对话开始时的快照。可这期间，别人可能已经改过了，后台可能已经自动合并过了。
指令：开工前先确认一下当前的真实状态
（最新的进度、版本、数据），别拿上一次
缓存的旧状态当真相。
02
State Persistence
状态持久化
AI 没有跨对话的长期记忆，上下文一重置就「失忆」。解法是把记忆外置成一个文件——这是整个系统的唯一真相源。
指令：请维护一个状态文件，把「已完成 /
进行中（当前在哪一步）/ 卡住 / 待开始」
实时写进去，每进入一个新阶段就更新一次。
03
Dogfooding
先喂自己
系统设计的第一原则：先在自己身上跑通。自己都不用的东西，别指望别人会用。
理念：让 AI 自主系统先服务自己的工作流，
验证后再推广到更复杂的场景。
Phase 02 · 开工阶段 — 边干边对齐
04
Iterate
干了再说，边干边对齐
传统工程教育告诉你先写详尽的计划再动手。但在 AI 自主系统里，计划永远不是唯一的真相，实际产出才是。
指令：别在计划上无限纠缠。先把最小
可用的版本跑起来，让产出本身说话。
05
Parallel
worktree 并行分身
把大活拆成几块，每个分身负责一块，利用 git worktree 创建隔离的工作副本，同时干、互不干扰，干完再合到一起。
指令：注意给每个分身都利用 git worktree
创建一个隔离的工作副本。
06
Reroute
失败绕行
为失败预设绕行路径，别让局部的卡顿拖垮整体。这个原则在设计 AI 自主系统的时候几乎处处可见。
原则：当一条路走不通时，系统应该
自动切换到备选路径，而不是停下来。

Checkpoint
阶段校准
过程中每隔一段，就拿最初的意图回来对一次，看跑偏没有。所谓对齐，是让意图和产出互相补全。

意图回来对一次，看跑偏没有。

Key Insights
核心原则
AI 自主干完一整夜，你最怕回来面对一个黑箱。所以这套流程要求它最后产出一份汇报：哪些完成了、哪些卡住了、做过哪些关键判断。让你一眼就能接管，而不是从头重新理解一遍。
隐藏陷阱
AI 干活的一个隐藏陷阱是：它手里的「事实」，往往是上一次对话开始时的快照。可这期间，别人可能已经改过了，后台可能已经自动合并过了。基于过期状态干活，轻则白忙，重则覆盖掉别人在途的工作。
