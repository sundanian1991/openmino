# Notion AI 产品页 蒸馏报告

> 来源: https://www.notion.com/product/ai
> 日期: 2026-05-10

---

## 这个对象是什么

Notion AI 的产品营销 Landing Page。定位是"你的 24/7 AI 团队"——不是 AI 功能集合，而是一支虚拟团队。页面从 Hero 到功能拆解到定价到信任背书再到 FAQ，完整 SaaS 产品页结构。

---

## 最值得拿的是什么

**"AI 团队"的叙事框架**。不是卖功能，而是卖"团队"——把你的 AI 拟人化成一支随时待命的团队。这个叙事转变是整个页面最有价值的东西。

> **不是"我们有 AI 功能"，而是"你有了一支 AI 团队"。**

---

## 分层拆解

### 结构层

```
Hero 区（全屏）
  主标题: "Meet your 24/7 AI team"
  副标题: 一句话价值主张
  双 CTA: "Try for free" + "Request a demo"
      ↓
社会证明栏: OpenAI/Figma/Ramp/Nvidia Logo 横排
      ↓
功能导航锚点: Agents / Enterprise Search / AI Meeting Notes / Admin Controls
      ↓
功能模块 1: Custom Agents（GIF 动图 + 客户引言 + Logo）
      ↓
功能模块 2: Notion Agent（GIF + 文案）
      ↓
功能模块 3: Enterprise Search（GIF + 文案）
      ↓
功能模块 4: AI Meeting Notes（GIF + 文案）
      ↓
管理功能网格（4 宫格）: Analytics / Permissions / Model Agnostic / ?
      ↓
信任背书区: 安全认证 5 项（No training / SOC2 / GDPR / Encryption / HIPAA）
      ↓
用例展示（4-up GIF + 6 个用例卡片）
      ↓
FAQ 手风琴（4 个问题，展开式）
      ↓
底部 CTA 收尾
```

### 机制层

| 机制 | 说明 |
|------|------|
| **GIF 演示** | 每个功能模块都用 16:9 GIF 动图展示实际使用效果，不是静态截图 |
| **客户引言+Logo** | 每个功能模块配一个真实客户的引用和公司 Logo |
| **锚点导航** | 顶部 4 个功能标签可点击跳转 |
| **定价透明** | Free vs Business 对比，Custom Agents 用 credits 计量 |
| **FAQ 手风琴** | 可展开/折叠，降低页面长度 |
| **社会证明前置** | Hero 下方紧跟 Logo 墙，建立信任 |

### 视觉层

| 元素 | 处理 |
|------|------|
| 背景 | 大面积浅色底 + 局部深色区块（信任区） |
| 动图 | 统一 16:9 圆角矩形 GIF，展示真实产品界面 |
| 字体 | 无衬线，标题粗体，正文常规 |
| 色彩 | 黑白灰为主，CTA 按钮用品牌色 |
| 间距 | 模块间距大，呼吸感强 |
| 图标 | 用产品截图/GIF 代替图标，不画抽象图标 |

### 节奏层

- **Hero 极简短**——一行标题 + 一行描述 + 两个按钮
- **功能模块均匀分布**——每个模块结构一致（GIF + 标题 + 描述 + 链接）
- **信任区突然变密**——从功能展示切换到密集的安全认证信息，节奏变化
- **用例区再次变轻**——回到 GIF + 短句，降低认知负担
- **FAQ 收尾**——手风琴折叠，页面变短

## 文案拆解

| 位置 | 文案 | 手法 |
|------|------|------|
| Hero 标题 | "Meet your 24/7 AI team" | 拟人化，"team"不是"tool" |
| Hero 副标题 | "Infinite minds, built for teamwork" | 诗意+实用 |
| Custom Agents | "Set them up once, and the entire team benefits" | 强调一次投入、全员受益 |
| Agent 能力 | "Anything you can do in Notion, your Notion Agent can do for you" | 能力对等，不是辅助 |
| 会议笔记 | "perfect meeting memory" | 一个词概括价值 |

**核心文案策略**：全程用"team/teammate/memory"等人类协作词汇，不用"automation/algorithm/model"等技术词。

## 四象限判断

### 可直接拿

| 条目 | 原因 |
|------|------|
| "AI 团队"叙事框架 | 通用的 SaaS 产品叙事方法 |
| GIF 演示代替截图 | 动态展示 > 静态截图 |
| 客户引言+Logo 配对 | 每个功能配一个真实客户背书 |
| 社会证明前置 | Hero 下方紧跟 Logo 墙 |
| 定价透明（credits 计量） | 用量可视化 |
| FAQ 手风琴 | 可折叠降低页面长度 |
| 锚点导航 | 顶部功能标签跳转 |

### 不可直接抄

| 条目 | 原因 |
|------|------|
| Notion 产品设计图/GIF | 版权保护 |
| 客户引言原文 | 属于 Notion 的品牌资产 |
| 具体的定价数字 | Notion 的商业数据 |
| 安全认证列表 | Notion 的合规成就 |

### 可适配后拿

| 条目 | 适配思路 |
|------|------|
| "AI 团队"叙事 | 用于我们的供应商管理——"每家供应商都是你的外延团队" |
| GIF 演示模式 | 用 GIF 展示供应商管理系统的实际操作流程 |
| 客户引言模式 | 每块业务配一个供应商/业务方的正面反馈 |
| 功能均匀模块 | 每个供应商 = 一个模块（数据 + 引言 + 状态） |
| 信任背书区 | 安全/合规/质量认证区——我们的供应商合规清单 |
| 用例展示 | 供应商管理典型场景：月度评估/清退流程/新供应商入驻 |
| 定价透明 | 供应商费用/折抵/成本的透明展示 |

### 证据不足

| 条目 | 缺什么 |
|------|------|
| 移动端适配 | 需要手机端截图 |
| 交互细节（锚点滚动、手风琴展开动画） | 静态抓取看不到 |
| 深色/浅色模式切换 | 不确定是否有 |

  - 权限控制

  - 数据安全

  - 月度评估自动化
