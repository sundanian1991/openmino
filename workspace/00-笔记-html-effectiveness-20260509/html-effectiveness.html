核心总结
12 / 13
01

Markdown 擅长线性阅读，但空间关系（diff、调用图、设计变体对比）天然需要二维展示。HTML 把信息从"读完"变成"看见"

02

每个示例都是自包含的一个 .html 文件，不需要服务器，不需要构建步骤，浏览器直接打开，Agent 生成后立刻可用

03

用完即弃的编辑器思路：为特定任务做一个临时 UI，但始终用导出按钮把操作结果变回文本，人始终在循环里

04

HTML 就是 HTML 应该交付的格式，设计系统渲染成色板，组件铺成联系单，产物直接喂回下一个 prompt，循环收紧
