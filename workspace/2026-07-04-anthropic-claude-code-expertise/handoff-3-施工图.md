## 内容形式（来自策展）
CT-1 长文 / 深度解读文章。
→ 视觉形式倾向：**CT-1 长文组件链**（Article Header / Drop Cap / Prose / Pull Quote / Reading Progress / Article Footer）。不锁定，Q1 后正式定。

## 施工硬约束（来自审查 🔴，必须 preserve）
1. **数据克制**：70%/80% 是中文解读的具体化，不是 Anthropic 原文精确数字。长文呈现时必须标注解读属性，或用定性表述。
2. **管理者成功率的"打折"**：管理者 > 工程师的延伸必须保留原文"即便打个折"——否则变煽动。
3. **作者声音连续性**：章节之间用作者过渡句桥接（"好，接下来的问题就是""你注意到没？"），不用学术编号章节标题。
4. **翻车机制的完整解释**：4%/15% 段必须解释"为什么专家能救回来"——不能只列对比数字。
5. **专家从名词变动词**：在"重新定义专家"段明确点出这个认知增量。
6. **开放钩子保留**：收束段加一句"如果这个差距消失了，是好事还是坏事？"（原文的 returns to expertise 下降观察）。

## 施工弹性区（来自审查 🟡，可权衡）
1. 是否用"会计"做"重新定义专家"段的钩子（Novelist 建议，可选）。
2. Pull Quote vs Statement Quote（审查未涉及，由 Q7 视觉气质决定）。

## 组件禁用清单（来自策展排除方向）
- ❌ V5 Pulse Hero（KPI 数据条是数据报告专属）
- ❌ Stat Grid（开屏数字冲击——长文不用）
- ❌ Tension Grid（二元对立——人/AI 是分工不是对立）
- ❌ Seam Benchmark（目标达成——专家/新手不是目标关系）
- ❌ Chapter Header 大编号（学术体例——长文用叙事钩子）
- ❌ Report Cover / TOC / Data Citation（报告体例——长文不用）

## 一句话本质（ljg 式压缩）
作者借 40 万次真实数据，重新定义"专家"——从资历职位（名词），变成具体任务里的指令质量（动词）；人管判断 AI 管执行，资历归零但专业能力换了接口。

## 进入 8 问

### Q1 内容本质上在做什么？
**讲一个故事 + 证明一个论点的混合**——但叙事载体是"故事"（作者解读视角），不是"报告"（中性陈列）。一句话本质已浮现：重新定义专家。**→ Hero 主选：不走 V5 Pulse，走 Article Header（CT-1 长文标配，不占满屏，标题后直接进正文）。**

### Q2 primary（唯一锚点）？
"专家从名词变动词"——这个认知增量是这篇文章比同类解读多出来的东西（Critic 验证）。标题"专家资历归零"是它的钩子表达。

### Q3 情绪温度？
温暖/共鸣（作者对话感）+ 精确（数据可信）。不是冷的仪表盘，是有体温的解读。→ 配色承诺：中克制，主底 offwhite + forest-soft 浅底绿点缀。1 深段（footer）。

### Q4 信息层次？
- primary：专家从名词变动词（handoff-3 第 5 条硬约束）
- secondary：人管判断AI管执行 / 翻车分水岭 / 管理者反转
- tertiary：趋势数据、职业排名、检查清单

### Q5 叙事弧线？
策展大纲已定：引子→论点→重新定义→5倍→反转→转折→收束。**含两个反转**（管理者>工程师、翻车分水岭）+ 一个开放收束（returns to expertise 钩子）。

### Q6 克制度？
中克制。长文默认。120px section padding（P6 长文预设），留白为主。

### Q7 视觉气质？
**手写感**（衬线温度 + 作者叙述 + 温润留白），测绘感只在数据 inline 呈现时点缀。→ Drop Cap 首字装饰 + Pull Quote 杂志金句 + Reading Progress。

### Q8 换品牌成立吗？
否。核心是"专家从名词变动词"这个独有论证 + 作者解读视角。换主题结构完全不适用。

## 施工图（hero/组件/section 序列）

### Hero：Article Header（非 V5 Pulse）
- 浅底 offwhite，不占满屏
- tag + 标题（"专家资历归零"钩子）+ 副标题 + meta（40万/23.5万/7个月）+ 收口线

### 组件链（CT-1 长文五件套）
1. Article Header
2. Drop Cap（引子首段，forest-soft 装饰绿）
3. Prose（720px 收窄正文，保留作者过渡句）
4. Pull Quote ×1（"专家一条指令顶新手五倍"段落浮动）
5. Reading Progress（顶部 2px 进度条，forest 填充）
6. Article Footer（作者位 + 相关阅读 + 标签）

### 数据 inline 呈现（不上独立组件段）
- 70%/80% 分工 → 用 data-pair 浅底并列块，标注"中文解读具体化"
- 5 倍差距 → inline 在 Prose 段落内，用 strong 强调
- 4%/15% 翻车 → data-pair，配机制解释（不只是数字）
- 管理者排名 → inline 文字 + 强调"打折"
- 收束 → 三件事清单 inline 在 Prose 流中 + 开放钩子余韵段

### 深段配额
仅 Footer 1 段深色（black）。全文浅底 offwhite 阅读流。

### 章节标题
叙事钩子式（"在 AI 面前，资历归零了"），带小序号 inline（01-06 作为 eyebrow，不作为 Chapter Header 大编号）。章节间用作者过渡句桥接。
