## 二、汇总表设计

### 汇总表1：供应商人力汇总（周度）

**数据来源**：所有供应商的 `人力明细表.xlsx` 最新一行

| 字段 | 来源 | 计算方式 | 用途 |
|------|------|---------|------|
| 供应商名称 | 文件夹名 | — | 标识 |
| 周期 | 人力明细表 | 周期起始~周期结束 | 时间维度 |
| 总人数 | 人力明细表 | 直接取值 | 规模监控 |
| 熟手人数 | 人力明细表 | 直接取值 | 产能计算分母 |
| 新人人数 | 人力明细表 | 直接取值 | 新人占比=新人/总数 |
| 新人占比 | 计算 | 新人÷总数 | >30%触发人力结构修正 |
| 本周新入职 | 人力明细表 | 直接取值 | 扩编跟踪 |
| 本周离职 | 人力明细表 | 直接取值 | 流失率计算 |
| 日均有效出勤率 | 人力明细表 | 直接取值 | 主管胜任评估(指标2) |
| 月度流失率 | 计算 | 本月离职总数÷月初总人数 | 主管胜任评估(指标1) |

**获取方式**：

```python
# 伪代码：从所有供应商人力明细表汇总
import glob
import openpyxl

suppliers = glob.glob("workspace/供应商空间模板/*/01-人力管理/人力明细表.xlsx")
summary = []
for f in suppliers:
    supplier_name = f.split("/")[-3].replace("-工作空间", "")
    wb = openpyxl.load_workbook(f)
    ws = wb.active
    # 取最后一行有效数据
    last_row = ws.max_row
    while last_row > 3 and not ws.cell(row=last_row, column=1).value:
        last_row -= 1
    if last_row <= 3:
        continue
    row = {
        "supplier": supplier_name,
        "total": ws.cell(row=last_row, column=5).value,
        "skilled": ws.cell(row=last_row, column=6).value,
        "new": ws.cell(row=last_row, column=7).value,
        "hired": ws.cell(row=last_row, column=8).value,
        "resigned": ws.cell(row=last_row, column=9).value,
        "attendance_rate": ws.cell(row=last_row, column=10).value,
    }
    row["new_ratio"] = row["new"] / row["total"] if row["total"] else 0
    summary.append(row)
```

### 汇总表5：整改任务台账（持续）

**数据来源**：所有供应商的 `整改计划表.xlsx`

| 字段 | 来源 | 用途 |
|------|------|------|
| 供应商名称 | 整改计划表 | 标识 |
| 约谈日期 | 整改计划表 | 触发时间 |
| 触发原因 | 整改计划表 | 问题类型 |
| 整改项总数 | 整改项编号数 | 承诺工作量 |
| 整改闭环率 | 整改进度表计算 | 已完成÷总项数 |
| 逾期项数 | 整改进度表计算 | 完成时间<今天且未完成的项数 |
| 整改执行力评分 | 闭环率×权重 | 主管胜任评估(指标5)、排名配合度 |

**整改闭环率计算**：

```
整改闭环率 = 按期完成的整改项数 ÷ 总整改项数

- ≥70% = 合格
- 50-70% = 预警
- <50% = 危险
```

## 四、数据质量控制

### 4.1 交叉验证规则

| 规则 | 验证方式 | 不通过的后果 |
|------|---------|------------|
| 总人数 = 熟手 + 新人 | 人力明细表内部校验 | 数据退回重填 |
| 本周离职数 = 本周离职登记表条数 | 人力表 vs 离职表 | 要求补齐离职登记 |
| 整改项编号连续性 | 整改计划表中编号是否连续 | 要求补充缺失项 |
| 整改进度跟踪对应整改计划 | 整改项编号是否存在于计划表 | 无效进度不计入执行力 |

### 4.2 迟报记录

| 表格 | 截止时间 | 迟报扣分 | 记录位置 |
|------|---------|---------|---------|
| 人力明细表 | 周一 12:00 | 每次扣1分 | 配合度评分 |
| 合规自查表 | 每月5日 | 每次扣2分 | 合规维度 |
| 整改计划表 | 约谈后48h | 每次扣5分 | 整改执行力 |
| 整改进度跟踪 | 周五 18:00 | 每次扣1分 | 整改执行力 |

## 五、自动化建议（第二期）

第一期手动汇总：打开每个供应商的 Excel，复制数据到汇总表。

第二期建议自动化：

1. **Python 脚本自动汇总** — 遍历所有供应商文件夹，读取最新数据行，写入汇总表
2. **异常自动报警** — 当人力汇总表中某供应商出勤率<85%或流失率>8%，自动标红
3. **整改闭环自动计算** — 读取整改计划+整改进度两张表，自动算闭环率
4. **一键导出排名** — 综合所有汇总表数据，自动计算加权排名
