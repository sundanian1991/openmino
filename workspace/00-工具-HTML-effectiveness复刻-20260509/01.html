一、项目定位

1.1 为什么要做

当前32家供应商、70个站点分散在12条业务线上，缺乏统一的分级视图。现有月度评估（A/B/C/D/E）仅反映单站点单月表现，无法回答以下战略问题：

  • 哪些供应商是核心依赖，哪些是边缘替补？
  • 哪些供应商在增长，哪些在衰退？
  • 如果某家供应商突然清退，业务连续性风险有多大？
  • 分量调整（赛马机制）该向谁倾斜、从谁手里收？
核心定位
月度评估是"体温计"（当下健康度），九宫格是"体检报告"（全局结构+趋势）。两者缺一不可。

1.2 解决什么问题

问题解决方式
看不到全局结构九宫格以"业绩水平 × 增长趋势"为两轴，32家供应商落入9个格子，一目了然
分量调整凭感觉明星象（右上）自动获得扩项优先，衰退象（左下）自动进入缩减名单
缺乏供应商差异化管控不同象限匹配不同的管理力度（高频巡检 vs 月度巡检 vs 季度巡检）
续约决策无抓手续约前直接读取九宫格位置，明星/金牛直接续约，瘦狗/衰退启动汰换评估

1.3 与其他项目的关系

项目关系
项目2：合作健康度诊断九宫格是结果，健康度诊断是原因。九宫格发现"这家在衰退"，健康度诊断回答"为什么衰退"
项目3：新供应商冷启动冷启动供应商在九宫格中固定落入"新进入"区域，满6个月后纳入正式盘点
月度评估（现有制度）九宫格的Y轴（业绩水平）直接引用月度评估的季度平均分

1.4 成功标准（量化指标）

指标目标值验证方式

3.2 评估维度与权重

X轴：业绩水平（60%权重）季度综合得分

以季度综合得分为基准，取最近一个季度（或最近3个月）的月度评估平均分：

分级标准说明
高业绩季度平均分 ≥85分 或 月度排名在前30%对应月度评级A或B+
中业绩70 ≤ 季度平均分 <85 或 月度排名在30%-70%对应月度评级B或C
低业绩季度平均分 <70 或 月度排名在后30%对应月度评级C-/D/E
数据来源
从 SM-TPL-008_月度评估评分表.xlsx 提取近3个月总分，计算季度平均。
Y轴：增长趋势（40%权重）环比变化率

以近两个季度的业绩变化率为衡量：

分级标准说明
正增长环比增长 ≥+5%GMV、产能、评分任一提升≥5%即判定
零增长-5% ≤ 环比 ≤ +5%基本持平
负增长环比下降 ≤-5%任一核心指标下降≥5%即判定
增长率计算公式
增长率 = (本季度平均分 - 上季度平均分) / 上季度平均分 × 100%

多指标交叉验证：

  • 主指标：月度评估季度平均分变化
  • 辅指标1：GMV季度环比（从 SM-TPL-002 提取）
  • 辅指标2：出勤人数季度环比（从 SM-TPL-001 提取）
  • 判定逻辑：主指标+任一辅指标同方向变化，方可确认趋势

第 2 周

九宫格绘制与分类

动作1：坐标定位判定逻辑

对每家供应商执行以下判定：

IF 季度平均分 ≥85 OR 排名百分位 ≤30%: → X轴 = 高业绩
ELSE IF 季度平均分 ≥70 OR 排名百分位 ≤70%: → X轴 = 中业绩
ELSE: → X轴 = 低业绩

IF 增长率 ≥ +5%: → Y轴 = 正增长
ELSE IF 增长率 ≥ -5%: → Y轴 = 零增长
ELSE: → Y轴 = 负增长

九宫格位置 = (X轴, Y轴) 的组合
