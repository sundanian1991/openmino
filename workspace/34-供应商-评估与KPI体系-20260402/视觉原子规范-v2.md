# 视觉原子规范 V2

> 极简、语义优先、克制
> 核心原则：颜色是信息，不是装饰

---

## 一、色彩系统（三色域）

### 1.1 中性色（Neutral）—— 背景与层级

仅保留5个层级，从白到黑：

| 名称 | 色值 | 用途 |
|------|------|------|
| **N-0** | `#FFFFFF` | 纯白背景、高亮卡片 |
| **N-1** | `#F5F5F5` | 主背景、容器底色 |
| **N-2** | `#E0E0E0` | 边框、分割线、次要背景 |
| **N-3** | `#999999` | 辅助文字、禁用状态 |
| **N-4** | `#333333` | 正文、标题 |

**使用原则**：
- N-0 和 N-1 交替使用，创造层级感
- 边框统一用 N-2
- 文字用 N-3（辅助）或 N-4（正文）

---

### 1.2 语义色（Semantic）—— 状态判断

仅4种状态，对应明确逻辑：

| 名称 | 色值 | 逻辑含义 | 使用场景 |
|------|------|----------|----------|
| **S-Good** | `#34A853` | 正常/通过/正向 | 达标、完成、健康 |
| **S-Warn** | `#FBBC04` | 警告/临界/待关注 | 接近阈值、需留意 |
| **S-Bad** | `#EA4335` | 危险/异常/失败 | 超出阈值、需行动 |
| **S-Neutral** | `#5F6368` | 中性/默认/进行中 | 无特定状态、过程 |

**透明度变体**（仅用于背景填充）：

| 名称 | 色值 | 用途 |
|------|------|------|
| S-Good-bg | `rgba(52, 168, 83, 0.08)` | 正常状态卡片底色 |
| S-Warn-bg | `rgba(251, 188, 4, 0.08)` | 警告状态卡片底色 |
| S-Bad-bg | `rgba(234, 67, 53, 0.08)` | 危险状态卡片底色 |

---

### 1.3 功能色（Functional）—— 强调与交互

仅2种，突出重点：

| 名称 | 色值 | 来源 | 用途 |
|------|------|------|------|
| **F-Accent** | `#C96442` | Anthropic 品牌橘 | 重点强调、当前选中、关键数据 |
| **F-Accent-light** | `rgba(201, 100, 66, 0.1)` | F-Accent 10% | 浅强调背景、选中态底色 |

**什么时候用 F-Accent**：
- 当前步骤（流程图中）
- 需要用户立即关注的数据
- 主按钮、主链接
- 当前选中的供应商

**什么时候不用**：
- 状态判断（用语义色）
- 普通背景和边框（用中性色）
- 辅助信息（用 N-3）

## 二、组件库

### 2.1 容器（Containers）

统一圆角：6px

#### C-1 基础卡片

```
┌─────────────────────┐
│                     │  ← N-0 背景
│      内容区域        │  ← 1px N-2 边框
│                     │  ← padding: 20px
└─────────────────────┘
```

| 属性 | 值 |
|------|-----|
| Background | N-0 或 N-1 |
| Border | 1px solid N-2 |
| Radius | 6px |
| Padding | 16px-24px |

#### C-2 状态卡片

左侧 4px 色条，表示状态：

```
┌─┬───────────────────┐
│█│                   │  ← 左侧色条: S-Good/S-Warn/S-Bad
│█│    内容区域        │  ← 背景: S-X-bg (8%透明度)
│█│                   │
└─┴───────────────────┘
```

**色条高度 = 卡片高度**，色条在卡片内部左侧，不是外边框。

#### C-3 数据卡片（Data Card）

小尺寸关键指标展示：

```
┌─────────────────┐
│ 标签            │  ← N-3, 12px
│                 │
│ 128             │  ← N-4, 28px, bold
│ ↑ 12%          │  ← S-Good/S-Bad, 11px
└─────────────────┘
```

**尺寸**：固定宽度 120px-160px，高度自适应内容。

---

### 2.2 节点（Nodes）

#### N-1 过程节点

```
┌──────────────┐
│   过程名称    │  ← N-0 背景, N-4 文字
└──────────────┘  ← 1px N-2 边框
```

**当前状态**：边框变为 F-Accent，背景变为 F-Accent-light。

#### N-2 决策节点

```
      /\
     /  \         ← 菱形
    / 判 \        ← 背景 N-0
   / 断  \        ← 边框 N-2
  /________\
```

**当前决策**：填充 F-Accent-light，边框 F-Accent。

---

### 2.3 线条（Lines）

统一 1.5px 实线：

| 类型 | 颜色 | 用途 |
|------|------|------|
| 主线 | N-2 | 正常流程连接 |
| 当前线 | F-Accent | 当前步骤的连接 |
| 弱化线 | N-3 (dashed) | 历史、已归档、可选路径 |

箭头末端统一 8px 三角箭头。

---

### 2.4 文字（Typography）

仅3个层级：

| 层级 | 字号 | 字重 | 颜色 | 用途 |
|------|------|------|------|------|
| **标题** | 16-18px | 600 | N-4 | 模块标题、卡片标题 |
| **正文** | 13-14px | 400 | N-4 | 主要内容 |
| **辅助** | 11-12px | 400 | N-3 | 标签、说明、数据来源 |

**数字数据**：16-24px，粗体，N-4 或语义色。

---

### 2.5 图标（Icons）

无 emoji，用简单几何符号：

| 符号 | 含义 | 颜色 |
|------|------|------|
| ● | 正常/活跃 | S-Good |
| ◐ | 警告/处理中 | S-Warn |
| ○ | 未开始 | N-2 |
| → | 正向流程 | N-3 |
| ↑ ↓ | 趋势 | S-Good/S-Bad |

## 三、布局原则

### 3.1 容器与内容的位置关系

**原则：容器必须包裹完整内容，不能让内容"穿出去"**

#### 正确做法

```
┌─────────────────────┐
│ ● 供应商名称         │  ← 图标+文字都在容器内
│                     │
│ 数据: 128           │
└─────────────────────┘
```

#### 错误做法（禁止）

```
      ● 供应商名称     ← 图标在容器外，文字被容器截断
┌───────┬─────────────┐
│       │ 数据: 128   │
└───────┴─────────────┘
```

**检查方法**：
- 每个内容元素（图标、文字、数字）必须完全在容器边界内
- 图标和文字之间留 6px 间距
- 容器 padding 至少 16px，确保不贴边

### 3.2 自适应规则

| 场景 | 规则 |
|------|------|
| 内容超长 | 换行，不截断；容器高度自适应 |
| 内容超短 | 保持最小高度，不塌陷 |
| 多列布局 | 列宽自适应，最小 120px |
| 移动端 | 单列堆叠，保持 padding |

## 四、使用模式

### 4.1 四象限矩阵

```
         高Y轴
            │
   [左上]   │   [右上]
   潜力型   │   标杆型
   (N-1底)  │   (S-Good-bg)
            │
 低X───────┼─────────高X
            │
   [左下]   │   [右下]
   问题型   │   波动型
   (S-Bad-bg)│  (S-Warn-bg)
            │
         低Y轴
```

每个象限底色用语义色 8% 透明度填充。

### 4.2 流程图

```
[开始] → [步骤1] → [步骤2] → [结束]
         ↑当前步骤
```

当前步骤：边框 F-Accent，背景 F-Accent-light。
其他步骤：边框 N-2，背景 N-0。

### 4.3 数据对比

```
         当前值          对比值
         128             96
         ↑ 33%           ↓ 12%
         S-Good          S-Bad
```

重点数据用 F-Accent 强调。

## 六、禁止事项

| 禁止 | 原因 | 正确做法 |
|------|------|----------|
| 莫兰迪色系 | 色彩过多，信息层级混乱 | 仅用中性色+语义色+功能色 |
| 超过3种主色 | 视觉焦点分散 | 中性打底，语义表状态，功能表重点 |
| 阴影效果 | 装饰性，非功能性 | 用边框和背景色区分层级 |
| 文字截断/穿出 | 信息丢失 | 换行或自适应高度 |
| 圆角超过8px | 过于柔和，失去稳重 | 统一6px |
| 渐变装饰 | 除非是信息衰减语义 | 纯色填充 |
