# 供应商站点调研框架

> 目标：验场 + 业务诊断 + 竞品了解 → 产出汇报文档，推动调整

---

## 调研前准备

### 带的材料

- [ ] 该供应商近6个月业绩数据（人均产能、结算、达标率）
- [ ] 同业务线其他供应商同期数据（做对比基准）
- [ ] 企业主贷大额税票贷产品要素表（利率/期限/准入条件/审批时效）
- [ ] 竞品产品要素表（如果有的话）
- [ ] 该供应商合同/结算标准

### 要见的人

| 对象 | 目的 | 时长建议 |
|------|------|----------|
| 供应商老板/总经理 | 了解经营困难、合作意愿、减量诉求的真实原因 | 30-40min |
| 现场负责人/主管 | 了解日常运营、团队状态、执行层面问题 | 30min |
| 一线坐席 2-3人 | 了解实际打电话遇到的问题、产品卖点/难点 | 各15min |
| 培训/质检岗 | 了解培训周期、合规情况、常见问题 | 15min |

## 二、业务诊断（核心）

> 这是重点。目标：搞清楚"为什么人均收入不理想"，找到不是靠减量能解决的真问题。

### 2.1 数据摸底

**先看数据，再听说法。用数据引导对话。**

| 问题 | 要拿到的数据 |
|------|-------------|
| 产能 | 人均日拨打量、有效通话量、接通率、转化率 |
| 结算 | 人均月结算、近6个月趋势、与达标线差距 |
| 人员 | 在岗人数、月度流失率、新人出单周期 |
| 名单 | 名单来源、质量、复打率、有效率 |

**关键对比**：这家的数据 vs 最大那家（人均结算10000）的差距在哪一步？是拨打量不够？接通率低？还是转化率差？

### 2.2 问题诊断6问

> 对着数据问，不笼统问

| 问题 | 目的 | 追问方向 |
|------|------|----------|
| 人均结算从多少降到多少？ | 量化问题 | 什么时候开始降的？降了多少？ |
| 降在哪个环节？ | 定位根因 | 是拨打量↓？接通率↓？还是转化率↓？ |
| 是所有人都降了，还是部分人？ | 区分共性vs个性 | 新人还是老人？哪个组？ |
| 名单有没有变化？ | 排除外因 | 名单量、名单质量、名单来源 |
| 产品有没有变化？ | 排除外因 | 利率、准入条件、审批时效 |
| 你们自己觉得问题在哪？ | 听供应商归因 | 对比数据，看归因是否准确 |

### 2.3 供应商减量诉求评估

> 年老师的核心判断点：要不要同意减量？

**要搞清楚的真实问题**：

| 问题 | 为什么重要 |
|------|-----------|
| "人多亏得多"具体亏多少？ | 量化亏损，判断是真亏还是预期管理 |
| 亏的是固定成本还是变动成本？ | 固定成本（房租/管理）减人也解决不了 |
| 减到多少人能盈亏平衡？ | 判断合理区间 |
| 减下来的人去哪？ | 是转项目还是裁员？ |
| 减量后业绩会不会更差？ | 人少了产能可能更低（士气/管理成本摊薄） |

**年老师的顾虑验证**：

| 顾虑 | 需要验证的 |
|------|-----------|
| "短期波动不应调整" | 近4个月是趋势性下滑还是波动？看6个月以上数据 |
| "开内部转项目口子" | 他们有没有提转项目？转去哪？ |
| "后续结算波动出大问题" | 如果同意减量，什么条件下可以加回来？ |

### 2.4 一线坐席访谈

> 坐席说的和管理层说的经常不一样。要单独聊。

| 问题 | 目的 |
|------|------|
| 每天打多少电话？ | 验证管理层给的数据 |
| 客户最常见的拒绝理由是什么？ | 产品问题还是话术问题 |
| 你觉得这个产品好卖吗？为什么？ | 一线视角的产品竞争力 |
| 培训教的东西能用上吗？ | 培训有效性 |
| 有没有遇到过合规问题？怎么处理的？ | 合规风险 |
| 你在这里多久了？之前做什么？ | 人员稳定性、经验背景 |
| 收入满意吗？和之前比怎么样？ | 真实收入感受 |
| 知道公司在做其他项目吗？想转吗？ | 内部转项目意愿 |

## 调研输出物

回去后产出一份文档，结构：

1. **调研概况**：时间、地点、人员、目的
2. **站点验收结论**：合格/有条件合格/不合格
3. **业务诊断**：数据+问题+根因
4. **竞品对比**：产品差异+运营差异+可借鉴点
5. **减量诉求评估**：数据支撑+判断+建议
6. **行动建议**：具体可执行的改进措施
7. **附录**：原始数据、访谈记录

给老板汇报用的版本压缩到：结论→问题→建议→需要支持，4页以内。
