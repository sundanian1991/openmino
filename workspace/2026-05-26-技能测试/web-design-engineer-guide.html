01

这是什么

一个注入到 AI 决策过程中的设计品位系统。不是设计工具，是让 AI 从"能用"进阶到"惊艳"的结构化提示词。灵感来自 Claude Design 系统提示词，可移植到任何项目。

适用

  • 网页和落地页
  • 交互式原型（带设备框架）
  • HTML 幻灯片 1920x1080
  • 数据可视化仪表盘
  • CSS/JS 动效演示
  • UI Mockup / 设计系统探索

不适用

  • 后端 API
  • CLI 工具
  • 数据处理脚本
  • 纯逻辑开发
  • 性能优化
  • 无视觉需求的终端任务

六步工作流

核心逻辑：先确认事实，再理解需求，建立设计系统后才写代码。每一步都有检查点。

0
验证事实
提到具体产品/品牌/SDK → 先搜索确认，不凭训练数据猜测。
搜索优先于猜测

理解需求
信息充足就开干，不足才提问。模糊需求 → 切换 Design Direction Advisor。
2
收集设计上下文
截图/Figma/代码库 > 现有页面 > 行业实践 > 配方库。Code >> Screenshots。
有上下文才能设计
3
声明设计系统
写代码前，用 Markdown 声明配色、字体、间距、动效。用户确认后才开始。
STOP — 等用户确认
4
展示 v0 草稿
占位符 + 布局 + token。让用户纠偏方向，而不是等一个"完美 v1"。
STOP — 等用户纠偏
5
完整构建
v0 通过后，写完整组件、状态、动效。关键决策点再次暂停确认。
6
验证 + 评分
逐项过交付清单。可选五维评分：哲学对齐 / 视觉层次 / 工艺品质 / 功能性 / 原创性。

04

核心原则

四条不可协商的设计铁律。不是审美偏好，是保护品牌识别度的工程约束。

避免
紫粉蓝激进渐变、Inter 字体做标题、emoji 当图标、CSS 剪影代替产品图、编造数据、#0D1117 暗底 + 赛博霓虹、左侧彩色边框卡片
替代方案
oklch 感知均匀色彩从品牌色派生。占位符方块 + 标签代替假图标。真实图片或 AI 生成图。占位符标注"需要真实数据"。25 套真实品牌配方代替默认审美。
占位符哲学
缺图标 → 方块 + [icon]。缺头像 → 首字母圆圈。缺图片 → 比例占位卡。缺 logo → 停下来问用户。占位符说"这里需要真实素材"，假货说"我偷懒了"。
目标是惊艳
大胆字号对比（h1 与正文 4-6x）。纹理叠加、mix-blend-mode、backdrop-filter。不写注释——代码自己说话。不总结——前面没说清楚是没写好。

触发方式

三种入口，覆盖从明确需求到完全模糊的全场景。

自动激活

请求涉及网页、原型、幻灯片、可视化、动画、Mockup 时自动加载。无需手动启用。

"帮我做一个落地页"

指定风格

说锚点名 → 自动加载对应 recipe，注入具体设计 token。25 套配方，7 个学派。

"Linear 风" / "Aesop 感觉" / "MUJI 安静感"

模糊需求

不知道要什么风格 → 切换 Design Direction Advisor，给出 3 个差异化方向供选择。

"做个好看的页面" / "不知道要什么风格"
07

三条技术硬规则

React + Babel 内联 JSX 场景下不可违反的约束。

01
不用 styles 变量名

多个组件文件的 styles 会互相覆盖。必须命名空间隔离：terminalStyles、headerStyles，或直接内联 style={{...}}。

02
Babel 块不共享作用域

每个 <script type="text/babel"> 独立编译。跨文件组件用 Object.assign(window, {...}) 挂到全局。

03
不用 scrollIntoView

iframe 预览中会干扰外层滚动。用 element.scrollTop 或 window.scrollTo 替代。
