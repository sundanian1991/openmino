## 二、各维度详细结果

### 2.1 Phase 4 验收清单（新增测试）

**测试方法**：取 5 个已有数据集的 chart.html，按 checklist.md 20 项（扩展为 27 项）逐项验收。

| 数据集 | 通过率 | 关键问题 |
|--------|--------|---------|
| #29 传染病 | 27/27 (100%) | 唯一无缺陷 |
| #03 汇率 | 26/27 (95%) | 标注/参考线未渲染 |
| #81 钢铁 | 18/27 (65%) | 多色高亮编码全部丢失 |
| #92 降水量 | 16/27 (60%) | 灰化编码丢失 + 标注/参考线缺失 |
| #98 HDI | 11/27 (40%) | bubble_chart 渲染为 bar + 缺少第 1 张图 |

**根因**：viz-echarts 渲染层未正确消费 JSON 契约中的 highlight/annotations/referenceLines/多图表/bubble_chart 等字段。

**checklist 模板缺陷（7 项）**：
1. 缺少渲染能力校验（渲染技能是否消费了 JSON 声明）
2. 缺少多图完整性检查（是否所有 chart 都渲染了）
3. 缺少多图风格一致性检查
4. Source ID 检查模糊（"有出处"不够具体）
5. 缺少高亮/灰化实际渲染的验证
6. 缺少标注是否遮挡数据的检查
7. 最终判断未区分"设计层问题"和"渲染层问题"

### 2.2 入口模式 A（逐步引导）

- 列出了 4 个实质不同的分析角度（排名/画像/风险/关系）
- 未直接出图
- 符合 SKILL.md 中模式 A 规范

### 2.3 入口模式 C（自定义/指定）

核心功能通过，5 个偏差：

**场景 C1（气泡图指定）**：
- 高亮比例 20% > 10% 规则（5 个数据点下最小高亮单位为 1 个 = 20%，规则边界问题）
- v2 JSON 中 scales color type 错误：连续变量用了 `"type": "manual"` 而非合适的 scale type

**场景 C2（柱状图指定）**：
- `params.color: null` 应 omit 而非设为 null
- x 轴 scale type 用了 `"linear"` 但实际是分类轴
- 轻量模式正确合并了 Phase 1+2，spec.md 包含 v2 JSON

### 2.4 多图全局风格锁定（Phase 0）

## 三、发现的规则/规范缺口

### 3.1 已修复（本轮优化已处理）

| 缺口 | 影响 | 修复 |
