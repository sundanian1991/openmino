## 核心循环

```
Explore → Validate → Build → Iterate
  探索       验证      构建     迭代
   │                         │
   └─────── 用真实反馈驱动 ───┘
```

## 使用方式

1. 读 `00-方法论/开发流程框架.md` 了解完整流程
2. 从 `01-探索` 开始，写一页纸问题陈述
3. 做 3 个原型，验证核心假设
4. 决定继续/调整/放弃
5. 如果继续，进入 `03-构建`，按迭代开发
6. 持续 `04-迭代`，用真实反馈驱动

## 关键区别

| 瀑布式（旧版） | 迭代式（AI 时代版） |
|----------------|---------------------|
| Phase 0 → 1 → 2 → 3 → 4 | Explore → Validate → Build → Iterate |
| 先写完整 PRD | 先写一页纸问题陈述 |
| 先画完整架构图 | 先搭能跑的骨架 |
| 先定完整 UI 规范 | 先做能用的页面 |
| 追求完美第一版 | 快速出半成品，迭代优化 |
