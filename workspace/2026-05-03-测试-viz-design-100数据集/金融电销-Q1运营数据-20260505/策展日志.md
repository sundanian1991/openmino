# 策展日志 · 金融电销Q1运营数据

> 2026-05-26 | 阶段：完整流程（1→6）

## 工作背景

年老师要给管理层做Q1运营复盘汇报，需要将18行结构化表格数据转化为可视化报告。原始数据包含12个指标、6个团队、3个月。

## 测试方向

**方向A**：直接使用数据.md中的5条既有结论做可视化
**方向B**：独立分析数据，重新提取洞察，再进入策展流程

**选择B**。既有结论是事实性判断（如"日均拨打量与转化率负相关"），但缺乏因果链和行动建议。管理层需要的是"为什么"和"怎么办"，不只是"是什么"。

## 发现的问题

1. **F组数据冻结**：1月/3月成交金额、接通量、意向客户数几乎相同。既有可能是数据录入问题，也可能是团队真实停滞。策展时标注但不做因果推断。
2. **"量大≠质高"是核心洞察**，但既有结论中只作为第5条出现，权重不够。
3. **因果链缺失**：既有结论是并列的5条，没有呈现指标之间的传导关系。

## 固定结论

| 编号 | 结论 | 适用范围 |
|------|------|---------|
| C1 | 三层分化：B/D第一、A/E第二、C/F第三 | Q1整体 |
| C2 | 接通率是最强前置指标，每提升1pp，转化率约提升0.3pp | 本数据集内 |
| C3 | 量大≠质高：C组日均拨打最高但转化率垫底 | 反例证明 |
| C4 | 效能瓶颈在"质"不在"量" | 行动方向 |
| C5 | B组优势：名单质量好+话术能力强，可复制 | 经验提炼 |
| C6 | E组2月逆势增长，值得复盘 | 异常值 |
| C7 | F组数据冻结，需核实 | 存疑项 |

## 余留问题

- F组数据真实性需业务侧确认
- 缺少个体坐席层面数据，无法验证"话术能力"的因果假设
- 无行业基准，无法判断绝对水平

## 可复用方法

1. **三层叙事结构**：定义边界 → 拆解逻辑 → 交叉分析 → 聚焦重点 → 结论建议。适用于所有数据汇报类策展。
2. **因果链可视化**：过程指标 → 中间指标 → 结果指标的传导路径图，比并列条目更有效。
3. **异常值单独成章**：不混入主体分析，降低干扰。
4. **逆向验证**：让子代理独立分析，再与既有结论对照，发现遗漏和权重偏差。
