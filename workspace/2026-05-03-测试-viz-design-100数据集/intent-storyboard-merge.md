# 中间文件精简方案

## 当前问题

SKILL.md 规定 4 个中间文件：intent.md + storyboard.md + spec.md + checklist.md。

但 **intent.md 和 storyboard.md 重叠严重**。对比：

| 维度 | intent.md | storyboard.md |
|------|----------|--------------|
| 一句话灵魂 | TOP-T Takeaway | 一句话灵魂 |
| 阅读路径 | Guide（引导路径） | 阅读路径 |
| 克制声明 | 不需要 | 克制声明 |
| 为什么选这个图 | 模式匹配 + 备选 | 无 |
| 场景论文 | 无 | 场景论文 |

单图场景下，填完 intent.md，storyboard 的内容 80% 已经有了。为了"走流程"多写一个文件，产出大量重复内容。

## 精简方案

**合并为 2 个文件：**

| 文件 | 内容 | 作用 |
|------|------|------|
| **intent.md**（合并） | 意图决策 + 叙事设计 | 为什么做、选什么图、怎么讲、克制什么 |
| **spec.md** | 规格参数 | 数据、色值、标注坐标、布局 |
| checklist.md | 验收清单 | 出图后逐项检查 |

**storyboard.md 的内容合并到 intent.md 里**，作为"叙事设计"章节。这样 intent.md 变成：

```
1. 意图决策（为什么做、选什么模式）
2. 叙事设计（一句话灵魂、阅读路径、克制声明）
3. 版式选择
4. 配色/字体
```

不再单独写 storyboard.md。

## 三循环工作流不变

中间文件数量变了，但循环不变：
- 循环1：写 intent + spec，出初稿
- 循环2：做减法，更新 spec
- 循环3：checklist 验收
