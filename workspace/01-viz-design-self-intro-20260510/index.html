viz-design v1.0

可视化设计器

不是画图工具，是可视化架构师

把抽象概念、非结构化数据、文字描述，变成完整的可视化设计方案——选什么图、为什么选、怎么突出关键信息、讲什么故事，全部设计好，交给下游渲染技能画出来。

一句话理解

viz-design 做什么？不做什么？

你告诉我想表达什么，我负责决定用什么图表、怎么组织数据、突出什么、隐藏什么、按什么顺序讲故事，然后委托专业的渲染技能把图画出来。我不自己写画图代码。

核心能力

六大能力模块，覆盖从"不知道怎么画"到"出图验收"的全流程

意图匹配

9509 条 DNA 案例库，按"你想表达什么"匹配最佳图表类型和完整设计策略，不是从零想。

视觉叙事

设计阅读路径、叙事弧线、视觉节拍——先讲什么后讲什么，让图自己"说话"。

全局风格锁定

多图场景先统一风格学派、色系、字体，避免各自选色导致的不协调。

版式策略

40 种版式模板 + 解读策略（KPI 卡片 / 洞察卡片 / 纯展示），按受众和场景自动匹配。

SPEC 编译

产出机器可读的渲染约束：标注清单、颜色映射、布局约束、视觉权重，下游技能直接消费。

质量验收

出图后按 checklist 逐项验收，4 项以上不通过自动回退，不交付残次品。

完整工作流

从"我想要一张图"到"图已验收交付"，共 6 个阶段

R0

必要性门禁 — 真的需要画图吗？

信息能一句话说清 → 不做图。精确数值/清单 → 用表格。只有涉及分布、趋势、关联、复杂比较时才继续。

门禁
P0

全局风格锁定（多图场景）

锁定风格学派（麦肯锡 / FT / Bloomberg 等）、全局色系（1 主色 + 1 强调色）、字体配对。后续每张图从全局继承，不各自选色。

表现层
P1

DNA 意图匹配 — 选什么图？

读 9509 条 DNA 索引，按表达意图匹配最佳图表类型。命中案例 → 继承全策略（高亮 / 标注 / 构图 / 阅读路径 / 色板）。未命中 → 依次查概念模式库 / 咨询架构库 / 自主推理。同时决定版式和解读策略。

内容层
P2

视觉叙事设计 — 怎么讲故事？

设计叙事弧线、一句话灵魂、签名视觉元素、克制声明。决定阅读路径：眼睛先看什么、再看什么、最后落在哪。

内容层
P3

SPEC 编译 — 渲染约束

将设计决策编译为渲染技能可消费的规格：数据组织、视觉编码、标注策略、布局参数，以及机器可读的渲染约束代码块。然后委托下游渲染技能执行。

表现层
嗅探

渲染结果嗅探 — 出图后快速扫描

在正式验收前先扫一遍常见问题：内容截断、代码泄露、中文乱码、元素重叠。发现则自动处理或告知，无则进入验收。

质量保障
P4

验收 — 逐项检查

出图后按 checklist 逐项验收。4 项以上不通过 → 回退 Phase 1 重新设计。同一修复策略连续失败 2 次 → 升级为架构级变更（换图表类型 / 拆分多图 / 重写叙事）。

门禁
轻量模式（快速通道）：单图明确需求（如"画个柱状图对比 5 个供应商业绩"）→ 跳过 Phase 0/2，不写中间文件，直接写 spec.md 委托渲染。迭代时只改表现层（风格/配色/版式），内容层（数据/结论/叙事）全程复用。

三种入口模式

A

逐步引导

"先看看数据能讲什么"
B

"你帮我决定" / "最佳方案"
C

自定义 / 快速路径

"帮我画个柱状图对比 X"

技能边界

明确 viz-design 负责什么、不负责什么，避免期望错位

viz-design 负责 viz-design 不负责
✓ 概念 → 模式匹配（选什么图表类型） ✗ 实际画图（不写 ECharts / SVG 代码）
✓ SPEC 编写（标题 / 标注 / 布局 / 叙事） ✗ 配色设计（各渲染技能自带配色体系）
✓ 视觉叙事策略（先讲什么后讲什么） ✗ 数据清洗（交给上游数据技能）
✓ 版式选择 + 解读策略决策 ✗ 多图 HTML 组装（委派给容器技能）
✓ 出图后质量验收 ✗ 猜下游渲染技能的输入格式

适用场景 vs 不适用场景

不是所有信息都适合可视化——viz-design 会在 R0 门禁直接劝退不需要的请求

适合使用

  • 有分布、趋势、关联关系需要呈现
  • 多层级数据需要一张图讲清全貌
  • 抽象概念需要视觉化（机制 / 架构 / 流程）
  • 管理层汇报需要结论性图表
  • 多图需要统一风格叙事
  • 有对比 / 排名 / 差距需要突出
  • 复杂关系需要降维表达

不适合使用

  • 信息一句话说得清 → 直接文字
  • 精确数值 / 系数 / 清单 → 用表格
  • 只有单一数据点，无比较维度
  • 需要实时数据流 / 动态刷新
  • 需要用户交互编辑数据 → 用 BI 工具
  • 需要打印高精度的照片级图片
  • 纯装饰性图片，不传达数据关系

核心设计原则

所有设计决策都遵循的六条铁律

标题写结论

不是"供应商业绩对比"，而是"B 组领先 F 组 62%"——标题本身就是判断。

颜色是信号

颜色编码重要性，不是装饰。高亮不超过 10%，色系不超过 2 种。

一图一洞察

每张图只传达一个核心判断。多余的拆到下一张图，不挤在一起。

视觉层级

最重要的元素 = 视觉上最突出。禁止多个元素同等突出互相抢夺注意力。

删除非数据墨水

删除所有不影响表意的元素，最大化数据墨水比例。网格线要淡，边框要去。

色盲友好

禁用红绿对比，使用色盲安全色板。颜色不是唯一的信息编码手段。

快速上手

告诉 viz-design 你想要什么，剩下的交给它

有数据

直接告诉我数据和想表达什么：

"对比 6 家供应商的月度成交额，看看谁在增长谁在停滞"

只有概念

描述你想表达的抽象关系：

"画一个机制图，说明供应商管理中的三个环节是如何联动的"

多图需求

给出主题，viz-design 先锁定全局风格再逐张设计：

"做一份供应商季度分析报告，需要 4-5 张图讲清全貌"

只有概念

没有数据，只有文字描述的关系或机制：

"用概念图说明供应商准入、考核、清退三个环节是怎么联动的"

快速出图

直接指定图表类型，跳过设计阶段：

"帮我画个柱状图，对比 A/B/C 三组转化率：4.2%, 3.8%, 2.1%"
