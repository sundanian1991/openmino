可视化设计器

不是画图工具，是可视化架构师
你告诉我想表达什么，我负责决定用什么图表、怎么组织数据、突出什么、隐藏什么、按什么顺序讲故事，然后委托专业的渲染技能把图画出来。我不自己写画图代码。

完整工作流

从"我想要一张图"到"图已验收交付"，7 个阶段。简单需求可走轻量通道。

R0 必要性门禁 门禁

先问是否需要图。不是所有信息都适合可视化。

  • 信息能一句话说清 → 不做图，文字表达
  • 精确数值/系数/清单 → 用表格，不走 Phase 0
  • 分布/趋势/关联/复杂比较？→ 继续进入后续阶段
P0 全局风格锁定 表现层

仅多图场景执行。单图跳过此步。

  • 风格学派：hash(主题名) → 匹配 McKinsey / FT / Bloomberg / Excalidraw 等
  • 全局色系：1 套主色 + 1 套强调色
  • 全局字体：锁定 1 对字体配对
  • 多图衔接：编号连续、视觉引导、配色统一

产出：global-style.md

P1 DNA 意图匹配 内容层

核心决策：选什么图？

  • 读 DNA 索引（15 类意图，每类 3-6 种推荐图表）
  • 命中案例 → 继承全策略：chart_type / highlight / annotation / composition / reading_path / color_palette
  • 未命中 → fallback 链：concept-pattern-map → consulting-archetypes → 自主推理（须写依据）
  • 同时决定版式（40 种模板）和解读策略

命中 DNA 施工图模板时 → 直接套用，只替换数据

产出：intent.md

P2 视觉叙事设计 内容层

怎么讲故事？

  • 叙事弧线：选择 12 种叙事弧线之一（问题解决型 / 对比型 / 趋势型等）
  • 一句话灵魂 + 签名视觉元素 + 克制声明
  • 阅读路径：眼睛先看什么 → 再看什么 → 最后落在哪

产出：storyboard.md

用户确认点（非快速路径时执行）：向年老师确认模式选择 + 推荐理由，提供 2-3 个备选。确认后才进入下一步。
P3 SPEC 编译 表现层

将设计编译为渲染技能可消费的规格。

  • 数据组织：字段清单 / 排序规则 / 聚合规则
  • 视觉编码：X/Y 轴编码什么 / 颜色编码什么 / 大小编码什么
  • 标注策略：高亮 ≤10% / 标注写原因+幅度
  • 渲染约束代码块：标注清单 / 视觉权重 / 颜色映射 / 布局约束

产出：spec.md → 委托下游渲染技能执行

嗅探 渲染结果扫描 质量保障

委派后、验收前的快速扫描。

快速检查常见问题：内容截断 / 代码泄露 / 中文乱码 / 元素重叠。发现问题自动处理或告知年老师，无问题则进入验收。

P4 验收 门禁

逐项检查，不通过 = 打回重做。

  • 出图前自检：读 anti-patterns.md
  • 按 checklist 逐项验收，标注通过/不通过
  • 4 项以上不通过 → 回退 Phase 1 重新设计
  • 同一修复策略连续失败 2 次 → 升级为架构级变更（换图表类型 / 拆分多图 / 重写叙事）

产出：checklist.md

轻量模式（快速通道）：单图明确需求（如"画个柱状图对比 5 个供应商业绩"）→ 跳过 P0/P2，不写中间文件，口头确认后直接写 spec.md 委托渲染。迭代时只做表现层（风格/配色/版式），内容层（数据/结论/叙事）全程复用。

三种入口模式

根据你说的不同类型，自动选择最合适的执行方式

列出 2-5 个分析角度，逐一确认后再选图表。适合数据丰富、不确定表达方向的场景。

"先看看数据能讲什么"

AI 自主决策最佳方案，直接出图。适合信任 AI 判断、追求效率的场景。

"你帮我决定" / "最佳方案"

按指定图表或角度直接执行。单图明确需求跳过中间文件，直接写 SPEC 委托渲染。

"帮我画个柱状图对比 X" / "做个漏斗图看转化瓶颈"

技能边界

明确负责和不负责的，避免期望错位

viz-design 负责

  • ✓ 概念 → 模式匹配（选图表类型）
  • ✓ SPEC 编写（标题/标注/布局/叙事）
  • ✓ 视觉叙事策略（先讲什么后讲什么）
  • ✓ 版式选择 + 解读策略决策
  • ✓ 出图后质量验收

viz-design 不负责

  • ✗ 实际画图（不写 ECharts/SVG 代码）
  • ✗ 配色设计（渲染技能自带配色）
  • ✗ 数据清洗（交给数据技能）
  • ✗ 多图 HTML 组装（委派容器技能）
  • ✗ 猜下游渲染技能的输入格式

适用 vs 不适用场景

不是所有信息都适合可视化——R0 门禁会直接劝退不需要的请求

适合使用

  • 有分布、趋势、关联关系需要呈现
  • 多层级数据需要一张图讲清全貌
  • 抽象概念需要视觉化（机制/架构/流程）
  • 管理层汇报需要结论性图表
  • 多图需要统一风格叙事
  • 有对比/排名/差距需要突出
  • 复杂关系需要降维表达

不适合使用

  • 信息一句话说得清 → 直接文字
  • 精确数值/系数/清单 → 用表格
  • 只有单一数据点，无比较维度
  • 需要实时数据流/动态刷新
  • 需要用户交互编辑数据 → 用 BI 工具
  • 纯装饰性图片，不传达数据关系

核心设计原则

所有设计决策都遵循的六条铁律

标题写结论

不是"供应商业绩对比"，而是"B 组领先 F 组 62%"。

颜色是信号

颜色编码重要性，不是装饰。高亮 ≤10%，色系 ≤2 种。

一图一洞察

每张图只传达一个核心判断。多余的拆到下一张图。

视觉层级

最重要的元素 = 视觉上最突出。禁止多个元素同抢注意力。

删除非数据墨水

删除所有不影响表意的元素。网格线要淡，边框要去。

色盲友好

禁用红绿对比，使用色盲安全色板。颜色不是唯一的编码手段。

快速上手

告诉 viz-design 你想要什么，剩下的交给它

有数据 + 有明确想表达

直接告诉我数据和想表达什么：

"对比 6 家供应商的月度成交额，看看谁在增长谁在停滞"
只有概念 / 没有数据

描述你想表达的抽象关系：

"画一个机制图，说明供应商管理中的三个环节是如何联动的"
多图需求

给出主题，先锁定全局风格再逐张设计：

"做一份供应商季度分析报告，需要 4-5 张图讲清全貌"
快速出图

直接指定图表类型，跳过设计阶段：

"帮我画个柱状图，对比 A/B/C 三组转化率：4.2%, 3.8%, 2.1%"

常见问题

viz-design 能直接画图吗？
不能。它产出四个 md 文档（intent / storyboard / spec / checklist），然后委派 viz-echarts 或 viz-svg-flow 执行渲染。它是脑，渲染技能是手。
工作流一定要走完所有 Phase 吗？
不一定。单图明确需求走轻量模式：跳过 P0/P2，口头确认直接写 spec.md。只有多图复杂场景才走完整流程。
改个配色要重跑全部流程吗？
不需要。资产分层机制：内容层（数据/结论/叙事）不重写，全程复用；表现层（风格/配色/版式）每次迭代可调。迭代时只改表现层。
DNA 库里找不到匹配怎么办？
三级 fallback：先查概念模式库（纯概念场景）→ 再查咨询架构库（逻辑关系场景）→ 最后自主推理，但必须在 intent.md 中写明推理依据。
