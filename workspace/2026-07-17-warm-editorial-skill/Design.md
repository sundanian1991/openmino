# Warm Editorial Design System

> **一句话**：把任何内容转化为具有"高端编辑感"的视觉页面。

---

# 第一部分：技能本质

## 1.1 核心隐喻

这个技能不是"加颜色和字体"。它是一个**设计思维透镜系统**。

| 思维透镜 | 设计透镜 |
|----------|----------|
| 面对问题，选3-4个职业视角 | 面对内容，选3-4个设计原则 |
| 每个职业有核心思考方式 | 每个原则有核心视觉机制 |
| 强制从职业视角思考 | 强制从原则维度决策 |
| 多视角交叉覆盖盲区 | 多原则互补生成完整设计 |
| 引导而非灌输 | 推导而非套用 |

## 1.2 核心信条

- **提出好问题比给答案更重要**
- **推导而非套用**
- **引导而非灌输**
- **每个元素必须有功能**
- **少即是多**

---

# 第二部分：设计原则透镜库

## P1 · Functional Beauty（功能之美）

### 你是谁

你是一个极度务实的设计师。你信奉"形式服从功能"。你不会因为"好看"而添加任何元素，你只会因为"有用"而添加。

### 你的核心问题

> "这个元素的任何像素，都在服务于什么功能？"

### 你看什么

- 内容的结构（标题/正文/数据/引用/导航）
- 读者的认知路径（先看哪里，再看哪里，最后看哪里）
- 信息的主次关系（必须看到的 vs 可选的）

### 你怎么思考

**第一层：理解内容的功能**
- 这份内容读完之后，读者应该做什么？
- 内容中哪些信息是必须看到的，哪些是可选的？
- 读者在阅读过程中会有什么认知负担？

**第二层：设计组件的功能**
- 页面上的每个元素（标题、段落、列表、色块、标签）——它的功能是什么？
- 这些元素之间如何协作完成整体功能？
- 如果我只能保留3个设计决策，是哪3个？

**第三层：验证功能是否实现**
- 遮住颜色，功能还能传达吗？
- 把字号统一，层级还在吗？
- 去掉所有装饰，内容还好读吗？

### 你输出什么

- 一份"功能清单"：每个设计决策对应的功能
- 一份"删除清单"：可以被移除的元素
- 一份"优先级清单"：最重要的3个设计决策

### 你的边界

- 不追求"好看"，追求"有用"
- 不添加任何纯装饰性元素
- 如果无法解释一个元素的功能，它不应该存在

## P2 · Editorial Authority（编辑权威）

### 你是谁

你是一个资深杂志编辑。你处理过无数复杂稿件，你知道如何让读者在3秒内找到重点。你的武器是：字号对比、网格系统、标签层级。

### 你的核心问题

> "读者能在3秒内找到重点吗？"

### 你看什么

- 信息的复杂程度（有多少层？）
- 读者的注意力预算（愿意看多久？）
- 内容的可扫描性（能跳读吗？）

### 你怎么思考

**第一层：诊断信息复杂度**
- 这份内容有多少层信息？（标题/副标题/正文/引用/数据/注释 = 6层）
- 哪些是"必须看到的"，哪些是"可以跳过的"？
- 最核心的一句话是什么？

**第二层：设计层级对比**
- 主标题应该多大？（通常是正文的3-5倍）
- 层级之间有足够的对比度吗？（不能所有字号都差不多）
- 有明确的"阅读入口"吗？（读者从哪里开始？）

**第三层：设计阅读路径**
- 读者的视线怎么流动？（左→右 / 上→下 / 大字→小字）
- 有没有"视觉锚点"引导读者？（编号/前缀线/标签）
- 跳读友好吗？（不看正文，只看标题和标签能理解吗？）

### 你输出什么

- 一份"层级地图"：每个信息的层级归属
- 一份"字号配方"：每个层级的字号/字重/字距
- 一份"阅读路径图"：视线的流动方向

### 你的边界

- 不使用超过4种字号
- 不使用超过3种字重
- 每个层级必须有明确的视觉差异

---

## P3 · Material Authenticity（材质真实性）

### 你是谁

你是一个材质设计师。你相信颜色不只是"好看"，它应该让人想触摸。每种颜色都指向一种真实的材质联想。

### 你的核心问题

> "这个颜色让人联想到什么材质？摸起来是什么感觉？"

### 你看什么

- 内容的情绪基调（温暖的？冷静的？权威的？亲和的？）
- 目标读者的审美偏好（极客？商务？创意？）
- 内容的"时间感"？

### 你怎么思考

**第一层：诊断内容的情绪**
- 这份内容给人什么感觉？（温暖/冷静/权威/亲和/科技/自然）
- 读者应该感受到什么？（信任/兴奋/平静/好奇）
- 这个内容适合什么场景？（快速浏览/深度阅读/展示/参考）

**第二层：选择材质组合**
- 主色对应什么材质？（羊皮纸/炭/苔藓/陶土/旧书脊）
- 这些材质的"触感"是什么？（粗糙/光滑/温暖/冷峻/柔软/坚硬）
- 材质之间有对话吗？（冷暖对比/粗糙vs光滑/新vs旧）

**第三层：验证材质联想**
- 遮住颜色名称，还能感受到材质吗？
- 材质的"触感"与内容的情绪匹配吗？
- 有没有"无材质联想"的屏幕色？

### 你输出什么

- 一份"材质清单"：每种颜色对应的材质
- 一份"触感描述"：页面整体的触感体验
- 一份"材质对话"：材质之间的对比关系

### 你的边界

- 不使用无材质联想的纯色
- 不使用超过6种颜色
- 所有颜色必须有暖底

---

## P4 · Restrained Luxury（克制奢华）

### 你是谁

你是一个奢侈品设计师。你知道"贵"不是堆砌，而是稀缺。你的设计哲学是：少即是多，留白即奢华。

### 你的核心问题

> "什么是最'贵'的设计决策？"

### 你看什么

- 内容的"稀缺性"在哪里？（最核心的信息是什么？）
- 读者的注意力预算（还剩多少？）
- 哪些元素可以减到最少？

### 你怎么思考

**第一层：诊断稀缺性**
- 这份内容的"奢华感"应该体现在哪里？（信息密度低？留白多？材质感？）
- 读者最应该把注意力放在哪里？
- 什么是"可以多"的，什么是"必须少"的？

**第二层：设计克制**
- 点缀色总面积占比？（≤15%）
- 深底用了几处？（≤2处/页）
- 装饰性元素有几个？（0个）

**第三层：验证克制**
- 去掉50%的内容，核心价值还能传达吗？
- 把配色从6色减到3色，高级感还在吗？
- 如果把字号统一，页面反而更好看吗？

### 你输出什么

- 一份"稀缺清单"：哪些信息是最珍贵的
- 一份"克制指标"：点缀色占比/深底数量/装饰数量
- 一份"删除建议"：可以被移除的元素

### 你的边界

- 不添加任何纯装饰性元素
- 不使用渐变/阴影/模糊
- 留白是主动的设计选择，不是"没填满"

---

## P5 · Temporal Depth（时间深度）

### 你是谁

你是一个古董设计师。你相信好的设计应该看起来"经历过时间"，不是"刚做好的"。你拒绝纯白和纯黑，因为它们是"无时间感"的。

### 你的核心问题

> "这个页面看起来是'刚做的'还是'一直在这的'？"

### 你看什么

- 内容的时间属性（当下的？经典的？永恒的？）
- 目标读者的时间感知（追求新鲜感？追求经典感？）
- 页面的"色温"（偏暖？偏冷？）

### 你怎么思考

**第一层：诊断时间属性**
- 这个内容是"当下的"还是"经典的"？
- 页面应该看起来"刚做的"还是"经历过时间"？
- 读者应该感受到什么时间感？（紧迫？从容？永恒？）

**第二层：设计色温**
- 主色调偏暖还是偏冷？（Warm Editorial答案是：偏暖）
- 有没有纯白#FFF或纯黑#000？（答案：没有）
- 深底文字用纯白还是暖白？（答案：暖白）

**第三层：验证时间深度**
- 把页面做旧处理，看起来像10年前的吗？
- 色温统一吗？有没有冷暖混用？
- 页面有"新"的痕迹吗？（高光/镜面/荧光色）

### 你输出什么

- 一份"时间定位"：这个页面的时间属性
- 一份"色温方案"：每个部分的色温
- 一份"无时间感检测"：有没有纯白/纯黑/荧光色

### 你的边界

- 不使用纯白#FFF做页面底
- 不使用纯黑#000做文字
- 不使用荧光色/马卡龙色
- 所有颜色必须有暖底

# 第三部分：原则选择器

## 3.1 基础层（每次都用）

- **材质真实** + **时间深度** → 决定色彩系统

## 3.2 表达层（根据内容选2-3个）

| 内容类型 | 推荐组合 | 理由 |
|----------|----------|------|
| 技术指南 | 编辑权威 + 功能之美 + 克制奢华 | 信息密度高，需要清晰层级和克制表达 |
| 产品发布 | 克制奢华 + 材质真实 + 编辑权威 | 需要高端感和材质联想 |
| 数据报告 | 功能之美 + 编辑权威 + 克制奢华 | 数据驱动，需要清晰层级的克制 |
| 个人品牌 | 材质真实 + 时间深度 + 克制奢华 | 需要温度感和经典感 |

# 第四部分：生成流程

## Step 1: 内容分析

**Functional Beauty 模式思考**：
- 这份内容读完之后，读者应该做什么？
- 内容中哪些信息是必须看到的，哪些是可选的？
- 读者在阅读过程中会有什么认知负担？

## Step 2: 原则选择

根据内容类型选择3-4个原则，写出选择理由。

## Step 3: 强制推导（每个原则运行其思维链）

**Functional Beauty 输出**：
- 功能清单：每个设计决策对应的功能
- 删除清单：可以被移除的元素
- 优先级清单：最重要的3个设计决策

**Editorial Authority 输出**：
- 层级地图：每个信息的层级归属
- 字号配方：每个层级的字号/字重/字距
- 阅读路径图：视线的流动方向

**Material Authenticity 输出**：
- 材质清单：每种颜色对应的材质
- 触感描述：页面整体的触感体验
- 材质对话：材质之间的对比关系

**Restrained Luxury 输出**：
- 稀缺清单：哪些信息是最珍贵的
- 克制指标：点缀色占比/深底数量/装饰数量
- 删除建议：可以被移除的元素

**Temporal Depth 输出**：
- 时间定位：这个页面的时间属性
- 色温方案：每个部分的色温
- 无时间感检测：有没有纯白/纯黑/荧光色

## Step 4: 综合设计决策

综合所有原则的输出，确定最终设计：
- 色值、字号、间距
- 页面结构（section顺序、背景色、色带位置）
- 组件选择（卡片/表格/列表/引用）

## Step 5: 生成HTML

基于设计决策生成完整HTML页面。

## Step 6: 验证

| 检查项 | 标准 |
|--------|------|
| Functional Beauty | 每个元素都能说出功能 |
| Editorial Authority | 3秒内找到重点 |
| Material Authenticity | 颜色有材质联想 |
| Restrained Luxury | 点缀色≤15%，无装饰 |
| Temporal Depth | 无纯白纯黑，色温统一 |

# 第五部分：色带系统

## 5.1 色带颜色序列（固定）

```
深灰 → 卡其 → 焦橙 → 灰绿
  1      2      3      4
```

## 5.2 色带位置系统

| 位置 | 方向 | 功能 | 何时用 |
|------|------|------|--------|
| 底部 | 横向 | 品牌标识/导航 | 页面首尾 |
| 顶部 | 横向 | 章节导航/状态 | 复杂页面 |
| 中间 | 横向细线 | 章节分隔 | 长页面节奏断点 |
| 左侧 | 竖向 | 页面标识/编号 | 封面/特殊页面 |
| 右侧 | 竖向 | 页码/辅助信息 | 数据密集页面 |

## 5.3 色带使用规则

1. **每个section最多一个色带位置**
2. **相邻section的色带位置不能冲突**（例如：底部→底部 = 冲突）
3. **视线流动方向**：左→右 / 上→下
4. **色带颜色顺序不可乱**：深灰→卡其→橙→灰绿

---

# 第六部分：红线（不可违反）

1. **禁止冷白做页面底**：深底文字用 #E7E0D2（暖白），不是纯白
2. **禁止渐变/装饰阴影**：层次靠色块+色调区分
3. **点缀色每页总面积 ≤ 15%**
4. **禁止 charcoal 文字落 charcoal 底**：深底用暖白文字
5. **禁止硬编码色值**：必须走 `var(--color-*)` 语义引用
6. **色带颜色顺序不可乱**：深灰→卡其→橙→灰绿
7. **每个section最多一个色带位置**
