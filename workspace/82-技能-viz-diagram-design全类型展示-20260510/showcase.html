viz-diagram-design · Type Catalog

13 种图类型与子形态

每个类型 = 自包含 HTML + inline SVG · 设计系统：语义色 · 4px 网格 · 复杂度预算（max 9 节点 / 12 箭头 / 2 强调元素）

Quadrant · 象限

type-quadrant

两轴优先级映射，适用于 30+ 项分类、决策矩阵、影响/投入评估

2×2 Classic 默认

四象限基础布局。右上 focal（accent 色），其余 muted。项以圆点/数字标注，不写大段文字。

例：影响 × 投入矩阵

3×3 Nine-box 九宫格

三行三列均匀网格，右上明星唯一 accent。轴端标签标注高中低。适合供应商分级盘点。

例：业绩 × 潜力 九宫格

Bubble Distribution 气泡

圆/椭圆散点分布，大小映射第三维度（如金额/人数）。不强制象限线，视觉密度传递信息。

例：供应商规模 × 业绩散点

Venn · 交集

type-venn

集合重叠关系，适用于共享能力、交叉领域、交集定位

2-Circle 默认

两个圆相交。交集区域标注共享属性。最简洁的集合表达。

例：技术 × 业务交叉能力

3-Circle 经典

三圆交叠，中心三重交集区域为 focal。标注各交集区域的共享属性。

例：产品 × 技术 × 商业交集

4-Set 进阶

四集合使用椭圆而非正圆，通过角度旋转实现交叠。中心为 focal 点。复杂度较高，慎用。

例：四维能力模型交集

Pyramid · 金字塔

type-pyramid

层级递进/过滤结构，适用于能力成熟度、信息过滤、漏斗转化

Pyramid 默认

标准三角分层。顶部最窄=focal=强调层。每层水平切分，宽度递减。

例：需求层次金字塔

Funnel 漏斗

漏斗形状，顶部宽底部窄。每层标注转化率/流失率。强调过滤和递减。

例：招聘漏斗、转化漏斗

Bar Pyramid 条形

用水平条代替三角切分，宽度递减。更易标注文字，适合层数较少的场景。

例：优先级分层

Tree · 树

type-tree

层级展开/归属关系，适用于组织架构、分类体系、分解结构

Top-down 默认

根节点在顶部，向下分叉。经典树形，适合宽度适中的层级。

例：决策树分解

Left-right 横向

根在左侧，向右展开。适合宽屏展示，节点文字从左到右阅读自然。

例：WBS 工作分解

Org Chart 组织

标准组织结构图形式。上级→下级→再下级。实线=直接汇报，虚线=虚线汇报。

例：部门组织架构

Radial Tree 辐射

根在中心，向外辐射分叉。适合对称性强的层级，视觉冲击力强。

例：生态系统辐射图

Timeline · 时间线

type-timeline

时间序列事件排列，适用于项目里程碑、产品路线图、历史回顾

Horizontal 默认

水平主线，事件节点左右交替标注。适合事件较少的短周期时间线。

例：Q2 项目里程碑

Vertical 纵向

垂直主线，事件节点左右交替。适合事件较多的长周期时间线。

例：公司发展历史

Gantt-style 甘特

时间条+行分组。每行一条时间带，条宽=持续时间。适合多线并行的项目计划。

例：多项目并行排期

State · 状态/循环

type-state

状态转换或闭环流程，适用于生命周期、飞轮效应、状态机

Linear Chain 默认

线性链式排列，状态依次推进。箭头连接相邻节点。

例：入职→培训→上岗→考核

Hub & Spoke 中心辐射

中心状态向外辐射连接多个外围状态。中心=focal。

例：供应商→多维度评估

Swimlane 泳道

多行并行状态链，每行一个角色/阶段。适合多角色协作的闭环流程。

例：供应商准入多角色流程

Flowchart · 流程图

type-flowchart

决策与流程混合，适用于审批流、异常处理、条件分支

Linear Flow 默认

从左到右或从上到下单向流动。矩形=步骤，菱形=决策。focal 在起始或关键决策点。

例：订单处理流程

Decision Tree 决策树

菱形决策节点带"是/否"分支。决策=珊瑚色，路径=muted。适合审批流。

例：供应商准入审批

Swimlane Flowchart 泳道

按角色分泳道的流程图。每行一个角色，流程跨泳道时用箭头连接。

例：跨部门协作流程

Sequence · 时序交互

type-sequence

参与者之间的调用时序，适用于 API 交互、系统间消息流

Simple Call-Response 默认

两个参与者，一问一答。请求=实线箭头，响应=虚线箭头。

例：HTTP 请求→响应

Nested Calls 嵌套调用

A→B→C→B→A 的嵌套调用。展示中间层转发逻辑。

例：API Gateway→Service→DB

Async Sequence 异步

包含异步事件/消息队列的时序。异步箭头带方框标记，回调=虚线。

例：事件驱动架构消息流

Swimlane · 泳道

type-swimlane

多角色并行任务分配，适用于跨部门流程、责任矩阵、RACI

Horizontal 默认

水平泳道，每行一个角色。从左到右时间推进，跨泳道箭头标注交接点。

例：跨部门协作流程

Vertical 纵向

垂直泳道，每列一个角色。从上到下时间推进。适合列数较少的场景。

例：供应商管理三角色分工

Kanban 看板

看板列分组（待办/进行中/完成）。卡片=任务项，泳道变体。

例：供应商任务看板

Layer Stack · 层叠

type-layers

抽象层或技术栈分层，适用于 OSI 模型、CSS 级联、内存层次

Horizontal Bands 默认

水平带堆叠，每层一行。左标注索引标签，中为层名，右为备注。4-6 层。

例：技术栈分层

Nested Layers 嵌套

同心圆角矩形嵌套。外层=低抽象，内层=高抽象。类似 nested 类型的逻辑。

例：内存层次 L1→L2→L3

Tech Stack (with icons) 带图标

每层带小图标（数据库、服务器、浏览器）。可选层间箭头表示数据流向。

例：全栈技术架构

Nested · 嵌套包含

type-nested

范围嵌套与层级包含，适用于作用域边界、信任区域、影响范围

Concentric Rectangles 默认

3-5 层同心圆角矩形，嵌套内缩。外环细，内环粗。最内层 focal = accent。

例：CLAUDE.md 级联作用域

Concentric Circles 同心圆

同心圆嵌套。适合"影响半径"或"爆破范围"类示意，圆形暗示径向扩散。

例：供应商影响半径

File Tree Nested 文件树

每层包含折叠角文件图标，标签为文件/文件夹名。适合目录结构展示。

例：.claude/ 目录嵌套

ER · 数据模型

type-er

实体关系与数据模型，适用于数据库 Schema、API 资源关系、领域模型

Entity Boxes 默认

两节实体框：header=类型标签+名，body=字段列表。PK 前缀 #，FK 前缀 →。

例：User → Org 关系

Crow's Foot 乌鸦脚

用视觉乌鸦脚标记替代文字基数。单线=one，三叉=many，圆+叉=zero or many。

例：Order 1:N Item

Minimal ER 极简

只有实体名，无字段列表。关系线+基数标注。适合 >8 个实体时的概览。

例：RBAC 模型概览

Architecture · 架构

type-architecture

系统架构与集成地图，适用于基础设施拓扑、数据流、集成关系

Tiered Architecture 默认

按层分组（frontend → backend → data）或信任边界。主流向左→右或上→下。

例：三层应用架构

Hub-and-Spoke 中心辐射

中心平台/枢纽节点，外围系统辐射。Hub=focal=accent。外围=muted 箭头。

例：供应商集成拓扑

Data Flow Architecture 数据流

箭头是主角，带数据类型标签（JSON, SQL, events）。组件框最小化（仅名称）。

例：数据管道流向
