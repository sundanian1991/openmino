## 第一章 总则

### 1.1 目的

规范京东2B电销BPO供应商的日常管理动作，明确各站点"谁做什么、什么时候做、怎么做"，确保业务连续、交付可控、信息通畅。

### 1.2 适用对象

- 所有承接京东2B电销业务的BPO供应商站点
- 站点负责人、对接人、一线坐席
- 供应商管理岗

### 1.3 基本原则

- **标准化**：所有站点执行同一套标准，不因人而异
- **及时性**：信息报送有明确时间要求，超时即为违规
- **闭环性**：每一项管理动作都有发起、执行、反馈、归档四个环节
- **可追溯**：所有重要事项留存书面记录，事后可查

### 1.4 角色定义

| 角色 | 所属方 | 定义 | 职责 |
|------|--------|------|------|
| 站点负责人 | 供应商 | 供应商派驻各职场的最高管理者 | 统筹本站点全部业务，对供应商负责 |
| 站点对接人 | 供应商 | 各站点指定的唯一日常对口人 | 信息报送、沟通协调、异常报备，同时对接京东业务对接人和供应商管理岗 |
| 业务对接人（业管） | 京东 | 京东侧负责业务交付管理的岗位 | 产能管理、业务目标、日常交付监督 |
| 供应商管理岗 | 京东 | 京东侧负责供应商管理的岗位 | 制度制定、评估考核、合规监督 |

## 第二章 信息报送规范

### 2.1 报送总则

- 所有信息通过指定渠道报送，不得使用个人微信或口头传达代替书面记录
- 报送格式按本手册附录模板执行，不得自行变更格式
- 站点对接人为报送责任人，对报送数据的及时性和准确性负责

### 2.2 日常报送

#### 2.2.1 每日出勤报送

| 项目 | 要求 |
|------|------|
| 报送内容 | 当日实际出勤人数、请假人数、异常缺勤情况 |
| 报送时间 | 每日 **10:00 前** |
| 报送人 | 站点对接人 |
| 报送方式 | 群内指定文档/共享表格"到岗情况"页 |
| 格式要求 | 按附件模板《每日出勤报送表》填报 |

**填报要求：**
- 须区分正常出勤、病假、事假、旷工四类
- 异常缺勤（旷工、未提前请假）须在备注栏说明原因
- 数据须与考勤系统一致，不得瞒报虚报

#### 2.2.2 每日排班报送

| 项目 | 要求 |
|------|------|
| 报送内容 | 次日各班次人员安排 |
| 报送时间 | 每日 **下班前** |
| 报送人 | 站点对接人 |
| 报送方式 | 共享表格"排班表"页 |

#### 2.2.3 每日业绩复盘

| 项目 | 要求 |
|------|------|
| 报送内容 | 当日核心业务数据（按产品线分列） |
| 报送时间 | 每日 **下线前** |
| 报送人 | 站点对接人 |
| 报送方式 | 群内发送 |

**复盘内容须包含：**
1. 当日产能达成情况（实际值 vs 目标值）
2. 关键指标波动说明（与前一日对比）
3. 次日重点安排

#### 2.2.4 群昵称规范

| 项目 | 要求 |
|------|------|
| 格式 | `职场-职务-姓名` |
| 示例 | `金条-站点负责人-张三` |
| 时限 | **首次要求后24小时内**完成修改 |
| 未完成处理 | 清除出群，由站点负责人重新申请入群 |

### 2.3 周期报送

#### 2.3.1 周报

| 项目 | 要求 |
|------|------|
| 报送内容 | 本周出勤汇总、产能完成情况、异常事项、下周计划 |
| 报送时间 | 每周 **周五下班前** |
| 报送人 | 站点对接人 |
| 报送方式 | 邮件或共享文档 |
| 格式要求 | 按附件模板《周报模板》填报 |

**周报内容结构：**
1. 本周出勤概况（日均出勤率、请假率、异常缺勤情况）
2. 产能完成情况（日均产能、达成率、与目标差距）
3. 质量情况（如已有质检数据，简要说明）
4. 异常事项清单（已处理/待处理）
5. 下周重点事项及排班预告

#### 2.3.2 月度总结

| 项目 | 要求 |
|------|------|
| 报送内容 | 月度业务数据、人员变动、合规自查、下月计划 |
| 报送时间 | 次月 **3日前** |
| 报送人 | 站点负责人 |
| 报送方式 | 邮件 |
| 格式要求 | 按附件模板《月度总结模板》填报 |

**月度总结内容结构：**
1. 月度核心数据汇总（出勤率、产能达标率、数据准确率）
2. 人员变动汇总（入职、离职、调岗）
3. 培训完成情况
4. 合规自查结果
5. 异常事项回顾与改进
6. 下月重点事项

#### 2.3.3 人员花名册

| 项目 | 要求 |
|------|------|
| 报送内容 | 在岗人员完整信息（姓名、工号、入职日期、岗位、资质状态） |
| 报送方式 | 共享表格，**实时更新** |
| 更新要求 | 人员变动后 **3个工作日内** 更新 |
| 报送人 | 站点对接人 |

**花名册必填字段：**
- 姓名、身份证号（仅供应商管理岗可查阅，站点对接人通过密封信封提交）
- 入职日期、所属站点、岗位
- 资质证书编号及有效期
- 培训记录（入职培训、合规培训、业务培训）
- 当前状态（在岗/试用/离职）

> **身份证号保密实现方式**：站点对接人在花名册表格的"身份证号"列填写时，将号码密封信封提交，或通过加密邮件直接发送供应商管理岗。共享表格中该列权限设为"仅供应商管理岗可编辑"，站点对接人仅可见"已填写/未填写"状态。

## 第三章 人员管理

### 3.1 人员准入

#### 3.1.1 上岗前要求

| 事项 | 要求 | 完成时限 |
|------|------|----------|
| 入职培训 | 完成公司统一入职培训课程 | 上岗前 |
| 合规培训 | 完成信息安全、话术规范等合规培训 | 上岗前 |
| 岗位考核 | 通过岗位技能考核（笔试+模拟通话） | 上岗前 |
| 持证上岗 | 取得岗位所需资质证书 | 上岗前 |
| 花名册入档 | 完成花名册信息登记 | 上岗后3日内 |

#### 3.1.2 试用期管理

- 新入职人员设有 **30天试用期**
- 试用期内：质检抽检比例提高至常规标准的 **2倍**
- 试用期不达标：站点须提交改进计划或调整安排
- 试用期满：站点出具考核意见，供应商管理备案

### 3.2 在岗管理

#### 3.2.1 定期复训

| 培训类型 | 频率 | 说明 |
|----------|------|------|
| 话术更新培训 | 按需（话术变更时） | 话术变更后X个工作日内完成全员培训 |
| 合规复训 | 每季度1次 | 信息安全、录音合规等 |
| 业务技能复训 | 每月1次 | 由业务方或站点自行组织 |

#### 3.2.2 岗位权限管理

- 坐席权限由站点负责人统一管理
- 人员离职或调岗后 **当日** 回收系统权限
- 权限变更须同步报备供应商管理岗

#### 3.2.3 终端设备管理

- 所有坐席终端须安装公司指定的安全软件
- 禁止使用个人设备接入业务系统
- 终端设备定期安全检查（每季度1次）

### 3.3 人员变动

#### 3.3.1 变动类型与报备要求

| 变动类型 | 报备时限 | 报备内容 | 报备方式 |
|----------|----------|----------|----------|
| 新入职 | 上岗前3个工作日 | 人员信息、培训完成情况、替代安排 | 书面 |
| 主动离职 | 提前3个工作日 | 离职原因、工作交接方案、到岗替代人员 | 书面 |
| 被动辞退 | 即时 | 辞退原因、交接方案、合规审查结果 | 书面+电话 |
| 岗位调换 | 变动前3个工作日 | 调换原因、新岗位培训安排 | 书面 |
| 大面积离职（≥3人/月） | 第一人离职时 | 情况说明、影响评估、补员计划 | 书面+电话 |

#### 3.3.2 离职交接清单

| 序号 | 交接事项 | 责任人 |
|------|----------|--------|
| 1 | 客户信息/工单交接 | 离职人员 → 接手人 |
| 2 | 系统权限回收 | 站点负责人 |
| 3 | 终端设备归还 | 离职人员 |
| 4 | 花名册状态更新 | 站点对接人 |
| 5 | 三方确认（离职人员、站点、供应商管理） | 站点负责人 |

### 3.4 异常人员管理

以下情形触发异常人员预警：

| 情形 | 处理方式 |
|------|----------|
| 连续2个月质检评分低于站点平均分20% | 站点出具改进计划，供应商管理备案 |
| 单月投诉≥3次 | 停岗检查，提交情况说明 |
| 资质证书过期未续 | 立即停岗，补齐资质后方可复岗 |
| 严重合规违规（信息安全红线） | 立即停岗，启动调查 |

## 第四章 异常处理

### 4.1 异常分级

| 级别 | 定义 | 典型场景 |
|------|------|----------|
| **一般异常** | 对业务影响有限，可自行处理 | 个别人迟到、系统偶发卡顿 |
| **重要异常** | 对业务有明显影响，需升级处理 | 大面积请假、系统故障、批量客户投诉 |
| **紧急异常** | 对业务或安全造成严重威胁，需立即处理 | 信息安全事件、安全事故、群体性事件 |

### 4.2 响应要求

| 级别 | 响应时间 | 报备方式 | 升级路径 |
|------|----------|----------|----------|
| 一般异常 | 当日处理 | 群内报备 | 站点自行处理 |
| 重要异常 | **2小时内** | 书面+电话 | 升级至供应商管理岗 |
| 紧急异常 | **立即** | 电话先行，书面补报 | 升级至供应商管理岗+业务对接人 |

### 4.3 异常处理流程

```
发现异常
  │
  ├─ 一般异常 → 站点自行处理 → 群内报备 → 记录台账
  │
  ├─ 重要异常 → 立即报备（2h内）→ 供应商管理介入 → 制定处理方案 → 执行 → 闭环记录
  │
  └─ 紧急异常 → 立即电话报备 → 启动应急响应 → 供应商管理+业务方联合处理 → 事后复盘
```

### 4.4 异常记录要求

每条异常须在台账中记录以下信息：

- 异常发生时间
- 异常类别与级别
- 影响范围（涉及人员/业务量）
- 处理措施
- 处理结果
- 根因分析与改进措施
- 闭环时间

## 第五章 沟通与会议机制

### 5.1 日常沟通

| 场景 | 渠道 | 响应要求 |
|------|------|----------|
| 日常信息 | 群消息 | **30分钟内回复** |
| 重要/紧急事项 | 电话 + 群内补发 | **立即电话**，挂断后群内补发书面说明 |
| 文件/资料 | 邮件或共享文档 | 按要求格式提交 |

### 5.2 会议机制

#### 5.2.1 周度业务复盘会

| 项目 | 要求 |
|------|------|
| 参与方 | 业务对接人、供应商管理岗、各站点对接人 |
| 频率 | 每周1次 |
| 议程 | 各站点周报同步、关键异常讨论、下周重点 |
| 会议纪要 | 由供应商管理岗记录并群内发送 |

#### 5.2.2 月度评估沟通会

| 项目 | 要求 |
|------|------|
| 参与方 | 供应商管理岗、各站点负责人 |
| 频率 | 每月1次（次月第1周） |
| 议程 | 月度评估结果公布、排名公示、一对一沟通 |
| 输出 | 月度评估报告、排名公告 |

#### 5.2.3 专项沟通会

| 触发条件 | 说明 |
|----------|------|
| 月度评估C级及以下 | 约谈站点负责人，讨论改进方案 |
| 触发预警 | 要求提交整改计划 |
| 重大业务调整 | 由业务方或供应商管理发起 |

## 第六章 台账与档案管理

### 6.1 台账体系

| 台账类别 | 内容 | 责任人 | 更新频率 |
|----------|------|--------|----------|
| 人员台账 | 花名册、培训记录、考核成绩 | 站点对接人 | 实时更新 |
| 业务台账 | 每日出勤、排班、业绩数据 | 站点对接人 | 每日 |
| 合规台账 | 质检结果、投诉记录、整改记录 | 供应商管理岗 | 按月汇总 |
| 沟通台账 | 会议纪要、书面往来、异常处理记录 | 供应商管理岗 | 事件触发 |

### 6.2 档案保存要求

| 档案类型 | 保存期限 | 存储方式 |
|----------|----------|----------|
| 人员档案 | 供应商合作期间+1年 | 共享文档 |
| 业务数据 | 12个月滚动 | 共享表格 |
| 合规记录 | 供应商合作期间+2年 | 共享文档 |
| 会议纪要 | 12个月滚动 | 群文件/共享文档 |
| 异常处理记录 | 供应商合作期间+1年 | 共享文档 |

### 6.3 数据保密

- 所有台账数据仅限供应商管理岗、站点负责人、站点对接人查阅
- 涉及客户信息的台账不得通过即时通讯工具传输
- 人员花名册中身份证号等敏感信息仅供应商管理岗可查阅

**权限保障细则**：

| 场景 | 实现方式 |
|------|----------|
| 共享表格中身份证号列 | 权限设为"仅供应商管理岗可编辑"，站点对接人可见"已填写/未填写"标记 |
| 站点对接人提交身份证号 | 通过密封信封或加密邮件直接提交供应商管理岗 |
| 花名册查阅权限 | 仅供应商管理岗可查阅完整信息，站点负责人和站点对接人仅可查阅脱敏版本（姓名+岗位+状态） |

**违规处理**：如发生信息泄露且查明责任方，追究相应责任。
