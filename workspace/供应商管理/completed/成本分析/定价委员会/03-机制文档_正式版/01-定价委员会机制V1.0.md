# 供应商定价调整委员会机制

**版本号**：V1.0
**生效日期**：2026-03-01
**解释权**：定价委员会秘书处
**起草人**：供应商管理岗

---

## 第一章 总则

### 1.1 目的与目标

为规范供应商定价调整流程，建立有约束的决策机制，特成立供应商定价调整委员会（以下简称"委员会"）。

**核心目标**：
- 从"事后救火"转向"事前防火"
- 保障供应商/坐席合理收益
- 维护电销生态健康稳定

### 1.2 适用范围

本机制适用于所有供应商定价调整场景，包括：
- 单价下调（降本）
- 单价上调（激励）
- 结算规则变更（考核门槛、奖惩系数等）

### 1.3 基本原则（红线）

| 原则 | 标准 | 说明 |
|------|------|------|
| **单次降幅上限** | ≤30% | 超过需分 2 次执行 |
| **生效延迟** | 次月 1 日生效 | 当月审批，避免突发冲击 |
| **收入底线** | 按业务线分别制定 | 需与行业拉齐 + 供应商调研 |
| **调整周期** | 3个月调整一次 | 避免频繁调整 、特殊调整单独申请 |
| **双向公平** | 涨价/降价统一审批 | 涨价需配套利好，降价需严格审批 |

## 第二章 组织架构

### 2.1 委员会构成

| 角色 | 人员 | 主要职责 |
|------|------|----------|
| 秘书长 | 供应商管理岗（1 人） | 组织会议、收集材料、追踪落地、归档记录 |
| 委员 | 业务督导代表（1 人） | 坐席层面信息（收入增减对业务的影响，管理坐席的影响） |
|      | 量化代表（1 人） | 数据测算、ROI 分析 |
|      | 服务组负责人 C3（1 人） | 业务运营、职场管理 |
|      | 策略组负责人 C3（1 人） | 定价策略、规则设计 |
| 终审 | 数据科技业务部负责人 C2 | 每项调整的最终审批 |

### 2.2 职责分工

| 角色 | 职责 |
|------|------|
| 供应商管理岗 | 组织会议、收集材料、初审提案、追踪落地、归档记录；收集供应商反馈 |
| 业务督导 | 代表坐席声音，提供坐席层面信息（收入增减对业务的影响，管理坐席的影响） |
| 量化代表 | 提供数据测算、ROI 分析、影响评估 |
| 服务组 C3 | 评估业务运营影响，协调职场安排 |
| 策略组 C3 | 提供定价策略建议，设计调整方案 |
| 终审（C2） | 每项调整的最终审批，掌握最终决策权 |

### 2.3 议事规则

**会议频率**：
- 常规评审：每月 1 次（第二周的周三）
- 临时会议：遇紧急调整，秘书长召集

**表决规则**：
- 常规调整（降幅≤15%）：5 人委员会，3/5 通过
- 涨价调整：5 人委员会，3/5 通过（与常规调整一致）
- 重大调整（降幅>15%）：5 人委员会，4/5 通过后，报 C2 终审

**回避原则**：
- 委员与调整涉及供应商有关联的，应主动回避

## 第三章 定价调整流程

### 3.1 流程概览

```
阶段 1：提案（T-5 天）→ 阶段 2：初审+评审（T-2 天）→ 阶段 3：终审（T-1 天）→ 阶段 4：调整后宣贯（生效后）→ 阶段 5：生效（次月 1 日）→ 阶段 6：复盘（+30 天）
```

### 3.2 阶段 1：提案（T-5 天）

**提案人**：策略组或量化代表

**必需材料**：《定价调整提案》（包含调整原因、数据支撑、影响测算、配套措施汇总）

**提案材料结构**：
1. **调整概述**：调整类型、幅度、时间、影响范围
2. **调整原因**：市场变化、成本变动、业务需求等
3. **数据支撑**：量化分析、对比数据、趋势分析
4. **影响测算**：对供应商、坐席、业务的影响预测
5. **配套措施**：如有，说明利好绑定、过渡方案等

### 3.3 阶段 2：初审+评审（T-2 天）

**审查人**：秘书长（初审）+ 5 人委员会全体（评审）

**审查要点**：材料完整性、降幅合规性、收入底线、配套措施、供应商反馈

**协同审查**：业务督导评估承受力、量化复核测算逻辑

**表决规则**：
- 常规调整（降幅≤15%）：5 人委员会，3/5 通过
- 涨价调整：5 人委员会，3/5 通过（与常规调整一致）
- 重大调整（增幅>15%）：5 人委员会，4/5 通过后，报 C2 终审

### 3.4 阶段 3：终审（T-1 天）

**审批人**：数据科技业务部负责人 C2

**审批范围**：所有定价调整提案（无论大小）

**审批重点**：战略合理性、整体影响、风险控制

### 3.5 阶段 4：调整后宣贯（生效后）

**宣贯对象**：供应商管理层

**内容重点**：
- 供应商对其员工的同步情况
- 员工层面的反馈和意见
- 是否存在较大反弹或负面情绪
- 供应商的应对措施

**目的**：了解执行效果，为下次调整提供参考

### 3.6 阶段 5：生效（次月 1 日）

**关键动作**：系统配置、通知结算、首周监控

### 3.7 阶段 6：复盘（生效后 30 天）

**输出**：《定价调整复盘报告》（包含执行效果、反馈汇总、改进建议）

## 第四章 核心规则详解

### 4.1 单次降幅≤30%

超过 30% 需分 2次执行。

### 4.2 价格调整频次

**调整频次**：3个月调整一次

**申请时间**：每次调整前1个月提交

### 4.3 次月生效原则

当月 15 日前提交→当月 17 日前评审→当月 18 日前终审→次月 1 日生效

### 4.4 收入底线保障

需调研后制定各业务线标准，测算时确保预计月收入≥底线。

### 4.5 价格调整申请材料结构

**必需要素**：
1. **基本信息**：调整类型、幅度、时间、影响范围
2. **调整原因**：市场、成本、业务等具体因素
3. **数据支撑**：量化分析、历史对比、趋势预测
4. **影响测算**：供应商、坐席、业务影响预测
5. **配套措施**：利好绑定、过渡方案、风险控制

### 4.6 定期评估日

**固定评估日**：每月第二个周三

**目的**：
- 集中处理待审提案
- 统一沟通反馈
- 提高决策效率

### 4.7 涨价与降价流程一致

- 涨价调整：委员会 3/5 通过
- 降价调整：按幅度决定（常规≤15%为3/5，重大>15%为4/5+C2终审）

### 4.8 供应商反馈申请通道

**设立目的**：为供应商提供正式渠道，对即将实施的定价调整提出异议或建议

**申请时机**：定价调整方案确定后、正式生效前

**申请方式**：
- 电子邮箱：pricing-committee@JD.com

**申请材料要求**：
1. **供应商基本信息**：公司名称、联系人、联系方式
2. **异议说明**：针对哪项定价调整、具体异议点
3. **影响论证**：详细说明调整对业务、员工、运营的具体影响
4. **数据佐证**：财务数据、运营数据、市场分析等客观证据
5. **替代建议**：如有可能，提供更合理的调整方案建议
6. **紧急程度**：□ 紧急（1天内处理）□ 重要（3天内处理）□ 一般（5天内处理）

**处理流程**：
1. **接收登记**（T+0）：秘书长接收并登记申请
2. **初步审查**（T+1）：材料完整性、合规性检查
3. **评估分析**（T+2）：量化代表、业务督导分析影响
4. **委员会讨论**（T+3）：如情况复杂，召开紧急会议
5. **决策反馈**（T+4）：告知供应商处理结果

**处理结果类型**：
- **采纳建议**：调整定价方案
- **部分采纳**：采纳合理建议，适当调整
- **维持原案**：说明理由并提供补偿措施
- **延期执行**：给予缓冲期或分期执行

## 第五章 风险防控

### 5.1 早期预警信号

| 信号 | 预警线 | 响应 |
|------|-------|------|
| 投诉率上升 | >20% 环比 | 暂停，重新评估 |
| 供应商利润 | <5% | 暂停，重新测算 |
| 坐席收入<底线 | >10% 占比 | 暂停，调整方案 |
| 离职率 | >行业平均 2 倍 | 暂停，调研原因 |

### 5.2 应急预案

触发条件：群体性事件、投诉率>50% 上升、离职率>20%

响应：24 小时内紧急会议→决策暂停/调整/继续
