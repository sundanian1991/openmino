# 供应商日常配合管理机制（讨论稿）

> **文档性质**：团队讨论稿
> **创建时间**：2026-03-21
> **适用范围**：所有BPO供应商

---

## 一、背景与目的

### 1.1 现状问题

当前供应商日常管理中存在以下痛点：

| 问题类型 | 具体表现 | 影响 |
|----------|----------|------|
| **响应不及时** | 数据回复延迟；周末无人值守 | 影响决策时效 |
| **配合不到位** | 出行安排不到位；人员未按时到岗 | 影响业务连续性 |
| **问题处理频繁** | 各种小问题都需要即时处理 | 管理精力分散 |

### 1.2 解决思路

**从"即时处理"转向"制度管理"**：

| 传统方式 | 新方式 |
|----------|--------|
| 发现问题 → @负责人 → 即时处理 → 重复发生 | 发现问题 → 记录在案 → 月度评估 → 影响排名 → 自动改进 |

### 1.3 核心目标

1. **建立长效机制**：用制度约束行为，不依赖人工盯防
2. **释放管理精力**：不即时处理每个小问题，聚焦月度结果
3. **赛马驱动改进**：排名公开，驱动供应商主动改进
4. **与赛马机制挂钩**：配合度得分纳入月度综合评估

## 二、适用范围

### 2.1 适用对象

- 所有BPO供应商（金条、企金、信用卡、财富）
- 所有日常配合场景（数据汇报、人员管理、应急响应等）

### 2.2 不适用场景

- **紧急重大事件**：仍需即时上报和处理
- **首次发生问题**：以沟通提醒为主，首次不扣分
- **不可抗力**：系统故障、自然灾害等

## 三、供应商日常配合要求

### 3.1 数据汇报配合

| 要求项 | 具体标准 | 责任方 |
|--------|----------|--------|
| **汇报时间** | 严格按照约定时间（工作日9:00、周末10:00等） | 供应商 |
| **数据质量** | 准确、完整、格式符合要求 | 供应商 |
| **周末值守** | 周末有人查看群消息，超时未响应需说明原因 | 供应商 |
| **异常上报** | 发现数据异常及时主动上报，不隐瞒 | 供应商 |

### 3.2 人员管理配合

| 要求项 | 具体标准 | 责任方 |
|--------|----------|--------|
| **出勤管理** | 按约定人数到岗，不得擅自减少 | 供应商 |
| **人员变更** | 人员变动提前报备，新人到位前做好交接 | 供应商 |
| **培训赋能** | 新人入职前完成培训，确保合格上岗 | 供应商 |
| **离职管理** | 人员离职提前报备，做好数据交接 | 供应商 |

### 3.3 业务执行配合

| 要求项 | 具体标准 | 责任方 |
|--------|----------|--------|
| **SOP执行** | 严格按照作业指导书执行 | 供应商 |
| **质量标准** | 达到约定的质量指标（转化率、准确率等） | 供应商 |
| **风险上报** | 发现业务风险及时主动上报 | 供应商 |
| **应急响应** | 突发情况下配合应急调度 | 供应商 |

### 3.4 沟通协同配合

| 要求项 | 具体标准 | 责任方 |
|--------|----------|--------|
| **群消息响应** | 工作时间1小时内响应，非工作时间4小时内响应 | 供应商 |
| **问题反馈** | 对管理方提出的问题及时响应和整改 | 供应商 |
| **信息同步** | 重要事项及时同步，不隐瞒 | 供应商 |
| **会议配合** | 按时参会，提前准备材料 | 供应商 |

## 四、问题发现与报告流程

### 4.1 问题发现渠道

| 渠道 | 发现人 | 示例 |
|------|--------|------|
| **日常监控** | 业管、质检、供管 | 发现数据未按时回复 |
| **群消息观察** | 任何人 | 看到供应商长时间未读群消息 |
| **数据核查** | 数据分析 | 发现数据异常 |
| **现场检查** | 现场管理人员 | 发现人员不在岗 |

### 4.2 问题报告流程

```
发现问题 → 记录在共享文档 → 报告供管
          ↓
供管确认 → 是否属实？是否首次？是否可抗力？
          ↓
处理决定 → 记录在案（扣分/提醒/免责）
          ↓
月度汇总 → 统计当月所有记录 → 计算得分
```

### 4.3 问题记录要素

每次问题记录需包含：

| 要素 | 说明 | 示例 |
|------|------|------|
| **供应商** | 哪个供应商 | 博岳 |
| **问题类型** | 数据延迟/人员缺失/质量不达标等 | 数据回复延迟 |
| **发生时间** | 什么时间发生的 | 2026-03-21 10:45 |
| **具体描述** | 问题的详细情况 | 约定10:00前回复，实际10:45回复 |
| **影响程度** | 轻微/一般/严重 | 一般（延迟45分钟） |
| **是否首次** | 是否首次发生该类问题 | 否（本月第2次） |
| **处理方式** | 扣分/提醒/免责 | 扣1分 |

## 五、月度评估与扣分机制

### 5.1 评分框架

| 项目 | 设置 |
|------|------|
| **基础分** | 10分 |
| **评分周期** | 按月累计，每月1日清零重新计算 |
| **影响范围** | 月度排名、分量调整、业绩评定 |
| **排名公布** | 每月5日前公布上月得分和排名 |

### 5.2 扣分标准（待讨论）

**数据汇报类**：

| 违规行为 | 扣分标准 | 说明 |
|----------|----------|------|
| 数据延迟 ≤30分钟 | -0.5分 | 以约定时间为准 |
| 数据延迟 30-60分钟 | -1分 | |
| 数据延迟 >60分钟 | -2分 | |
| 数据质量错误（非首次） | -0.5~1分 | 按影响程度 |
| 周末无人值守（非首次） | -1分 | 超时未响应且无说明 |

**人员管理类**：

| 违规行为 | 扣分标准 | 说明 |
|----------|----------|------|
| 人员未按时到岗 | -0.5~1分 | 按人数和影响 |
| 人员变动未报备 | -0.5分 | 非首次 |
| 新人未培训上岗 | -1分 | |

**业务执行类**：

| 违规行为 | 扣分标准 | 说明 |
|----------|----------|------|
| 未按SOP执行（非首次） | -0.5~1分 | 按影响程度 |
| 质量指标不达标 | 已有质量考核体系，本机制不重复扣分 | |

**沟通协同类**：

| 违规行为 | 扣分标准 | 说明 |
|----------|----------|------|
| 群消息超时未响应（非首次） | -0.5分 | 工作时间1小时、非工作时间4小时 |
| 问题整改不及时 | -0.5~1分 | 按延迟程度 |

### 5.3 免责条款

以下情况不扣分：

| 情形 | 说明 |
|------|------|
| **首次发生** | 同类问题首次发生，以提醒为主 |
| **主动上报** | 供应商主动发现问题并及时上报 |
| **不可抗力** | 系统故障、自然灾害等 |
| **已提前报备** | 提前报备并获得批准的调整 |

### 5.4 得分应用

月度得分应用于：

| 应用场景 | 说明 |
|----------|------|
| **月度排名** | 纳入供应商月度综合排名 |
| **分量调整** | 影响下月任务分量分配 |
| **年度评估** | 作为年度综合评估参考 |
| **赛马机制** | 排名公开，驱动供应商主动改进 |

## 六、实施保障

### 6.1 工具支持

- **问题记录表**：建立共享文档/表格，所有人可填写
- **自动统计**：每月自动汇总各供应商得分
- **排名公示**：每月在供应商群公示排名

### 6.2 沟通机制

- **启动会**：与供应商召开启动会，讲解规则
- **月度反馈**：每月向供应商反馈得分情况和改进建议
- **季度回顾**：每季度回顾机制运行情况，优化规则

### 6.3 责任分工

| 角色 | 职责 |
|------|------|
| **发现问题的人** | 及时记录问题，报告给供管 |
| **供管** | 确认问题、判定处理方式、月度汇总评估 |
| **供应商** | 查看问题记录、改进内部管理、提升配合度 |
