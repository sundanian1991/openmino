## 一、项目背景与目标

### 1.1 核心问题

当前供应商定价调整存在以下问题：

- **定价调整随意** — 策略/量化单方面决策，缺乏供应商声音
- **降幅无约束** — 单次调整超过 50%，超出心理承受
- **事后救火** — 供应商管理岗只有安抚，没有决策权
- **无复盘机制** — 调整完不追踪，下次还犯同样错误

### 1.2 建设目标

从"**事后救火**"转向"**事前防火**"，建立：

- 有约束的决策机制（单次≤30%、次月生效）
- 有代表的委员会（供应商、业务、策略、量化、督导）
- 有复盘的闭环（每次调整后追踪效果）

## 二、机制设计概览

### 2.1 组织架构

**5人委员会 + C2终审**


| 角色     | 人员            | 主要职责                |
| ------ | ------------- | ------------------- |
| 秘书长    | 供应商管理岗        | 组织会议、收集材料、追踪落地、归档记录 |
| 委员     | 业务督导代表        | 坐席层面信息（收入增减对业务的影响）  |
| &nbsp; | 量化代表          | 数据测算、ROI 分析         |
| &nbsp; | 服务组负责人 C3     | 业务运营、职场管理           |
| &nbsp; | 策略组负责人 C3     | 定价策略、规则设计           |
| 终审     | 数据科技业务部负责人 C2 | 每项调整的最终审批           |


### 2.2 核心流程

```
阶段 1：提案（T-5 天）→ 阶段 2：初审+评审（T-2 天）→ 阶段 3：终审（T-1 天）→ 阶段 4：调整后宣贯（生效后）→ 阶段 5：生效（次月 1 日）→ 阶段 6：复盘（+30 天）
```

### 2.3 决策权限


| 调整类型             | 审批人         | 通过条件            |
| ---------------- | ----------- | --------------- |
| **常规调整**（降幅≤15%） | 委员会 5 人     | 3/5 通过          |
| **涨价调整**         | 委员会 5 人     | 3/5 通过（与常规调整一致） |
| **重大调整**（降幅>15%） | 委员会 + C2 终审 | 4/5 通过 + C2 终审  |
| **每项调整**         | C2          | 每项调整都需要C2终审     |

## 三、机制亮点

### 3.1 三条红线


| 红线       | 标准      | 例外处理         |
| -------- | ------- | ------------ |
| **单次降幅** | ≤30%    | 超过需分 2 次执行   |
| **生效时间** | 次月 1 日  | 当月审批，避免突发冲击  |
| **调整频次** | 3个月调整一次 | 每次调整前1个月提交申请 |


### 3.2 风险防控机制

#### 早期预警信号


| 信号      | 预警线       | 响应      |
| ------- | --------- | ------- |
| 投诉率上升   | >20% 环比   | 暂停，重新评估 |
| 供应商利润   | <5%       | 暂停，重新测算 |
| 坐席收入<底线 | >10% 占比   | 暂停，调整方案 |
| 离职率     | >行业平均 2 倍 | 暂停，调研原因 |


#### 应急预案

触发条件：群体性事件、投诉率>50% 上升、离职率>20%
响应：24 小时内紧急会议→决策暂停/调整/继续

### 3.3 供应商反馈通道

设立正式渠道，让供应商对即将实施的定价调整提出异议或建议：

- **申请时机**：定价调整方案确定后、正式生效前
- **处理流程**：接收登记→初步审查→评估分析→委员会讨论→决策反馈
- **处理结果**：采纳建议、部分采纳、维持原案、延期执行

## 四、申请与复盘体系

### 4.1 定价调整申请材料结构

**必填要素**：

1. **基本信息**：调整类型、幅度、时间、影响范围
2. **调整原因**：市场、成本、业务等具体因素
3. **数据支撑**：量化分析、历史对比、趋势预测
4. **影响测算**：供应商、坐席、业务影响预测
5. **配套措施**：利好绑定、过渡方案、风险控制

### 4.2 定价调整复盘报告

每次调整生效30天后，出《定价调整复盘报告》，包含：

- 执行效果分析
- 供应商/坐席反馈汇总
- 改进建议

## 八、Q&A

### Q1: "这个机制是否会削弱我的决策权？"

**A**: 您仍掌握最终决策权，机制的作用是提供更科学的前置评估。每项调整都需要您终审，但您将获得更充分的信息支撑。

### Q2: "如何确保机制不会影响业务灵活性？"

**A**:

- 保留了您的紧急决策权，可绕过常规流程
- 设置了"临时会议"机制处理紧急需求
- 定价调整频次为3个月一次，给予足够灵活性

### Q3: "如何衡量机制效果？"

**A**:

- 量化指标：投诉率、离职率、供应商利润、业务产能变化
- 时间节点：每次调整后30天进行复盘
- 闭环反馈：复盘结果纳入下次调整参考
