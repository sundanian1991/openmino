# 定价委员会机制建设思路汇报

**汇报对象**：数据科技业务部负责人 C2
**汇报人**：供应商管理岗
**时间**：2026-02-27

---

## 一、为什么建这个机制

**核心问题**：
1. **定价调整随意** — 策略/量化单方面决策，无供应商声音
2. **降幅无约束** — 单次调整超过 50%，超出心理承受
3. **事后救火** — 供应商管理岗只有安抚，没有决策权
4. **无复盘机制** — 调整完不追踪，下次还犯同样错误

### 建设目标

从"**事后救火**"转向"**事前防火**"，建立：
- 有约束的决策机制（单次≤30%、次月生效）
- 有代表的委员会（供应商、业务、策略、量化、督导）
- 有复盘的闭环（每次调整后追踪效果）

---

## 二、机制核心设计

### 1. 组织架构（5 人委员会）

| 角色 | 人员 | 主要职责 |
|------|------|----------|
| 秘书长 | 供应商管理岗（您） | 组织会议、收集材料、追踪落地、归档记录 |
| 委员 | 业务督导代表 1 人 | 坐席层面信息（收入增减对业务的影响，管理坐席的影响） |
|      | 量化代表 1 人 | 数据测算、ROI 分析 |
|      | 服务组负责人 C3 | 业务运营、职场管理 |
|      | 策略组负责人 C3 | 定价策略、规则设计 |
| 终审 | 数据科技业务部负责人 C2 | 每项调整的最终审批 |

### 2. 决策权限

| 调整类型 | 审批人 | 通过条件 |
|---------|--------|---------|
| **常规调整**（降幅≤15%） | 委员会 5 人 | 3/5 通过 |
| **涨价调整** | 委员会 5 人 | 3/5 通过（与常规调整一致） |
| **重大调整**（降幅>15%） | 委员会 + C2 终审 | 4/5 通过 + C2 终审 |
| **每项调整** | C2 | 每项调整都需要C2终审 |

### 3. 三条红线

| 红线 | 标准 | 例外处理 |
|------|------|---------|
| **单次降幅** | ≤30% | 超过需分 2 次执行 |
| **生效时间** | 次月 1 日 | 当月审批，避免突发冲击 |
| **调整频次** | 3个月调整一次 | 每次调整前1个月提交申请 |

### 4. 流程概览

```
提案（T-5 天）→ 初审+评审（T-2 天）→ 终审（T-1 天）→ 生效（次月 1 日）→ 调整后宣贯（生效后）→ 复盘（+30 天）
```

**关键节点**：
- **提案**：策略/量化提交《定价调整提案》（含影响测算、配套措施、供应商反馈）
- **初审+评审**：秘书长形式审查 + 5人会议讨论，投票表决（2-3天完成）
- **终审**：C2对每项调整进行最终审批
- **生效**：次月 1 日执行，首周监控异常
- **调整后宣贯**：了解供应商对其员工的同步情况，员工层面反馈
- **复盘**：30 天后出《复盘报告》，纳入下次参考

### 5. 定期评估日

**固定评估日**：每月第二个周三

**目的**：集中处理待审提案，提高决策效率

### 6. 价格调整申请材料结构

**必填要素**：
1. **基本信息**：调整类型、幅度、时间、影响范围
2. **调整原因**：市场、成本、业务等具体因素
3. **数据支撑**：量化分析、历史对比、趋势预测
4. **影响测算**：供应商、坐席、业务影响预测
5. **配套措施**：利好绑定、过渡方案、风险控制

### 7. 供应商反馈申请通道

**设立目的**：为供应商提供正式渠道，对即将实施的定价调整提出异议或建议

**申请时机**：定价调整方案确定后、正式生效前

**申请材料要求**：
1. **供应商基本信息**：公司名称、联系人、联系方式
2. **异议说明**：针对哪项定价调整、具体异议点
3. **影响论证**：详细说明调整对业务、员工、运营的具体影响
4. **数据佐证**：财务数据、运营数据、市场分析等客观证据
5. **替代建议**：如有可能，提供更合理的调整方案建议
6. **紧急程度**：□ 紧急（1天内处理）□ 重要（3天内处理）□ 一般（5天内处理）

**处理流程**：接收登记→初步审查→评估分析→委员会讨论→决策反馈

## 三、需要 C2 支持的事

### 1. 确认 C2 作为委员会终审

- **职责**：每项定价调整的最终审批
- **参与方式**：所有调整都需C2审批，确保决策权集中
- **好处**：掌握最终决策权，确保调整战略符合业务目标

### 2. 确认收入底线调研计划

- **目标**：每条业务线制定收入底线（如：A 业务≥4000 元，B 业务≥5000 元）
- **方法**：
  - 与行业拉齐（同业供应商成本结构调研）
  - 与供应商充分调研（座谈会、问卷）
- **时间**：预计 2-3 周完成
