Chapter 01

全景概览

SenseNova Skills 是商汤科技推出的开源 AI Agent 技能集合，基于 SenseNova Platform API 构建。它不是一个单体应用，而是 24 个独立工具 的有机组合 — 每个工具专注解决一个特定领域的复杂任务。

一个 API Key，五大能力域，24 个专业工具。从"给我一张图"到"帮我做一份行业研究报告"，覆盖内容生产的完整链路。

Chapter 02

架构与工作原理

统一 API 入口

所有技能共享同一个 SenseNova Platform API。Base URL 为 https://token.sensenova.cn/v1，只需一个 API Key（SN_API_KEY）即可调用全部能力。

输入 自然语言 / 文件 / URL → SenseNova API → 输出 图片 / PPTX / 分析报告 / HTML

环境变量优先级

API Key 的解析遵循明确的优先级链，越靠前优先级越高：

优先级 环境变量 适用场景
1 CLI --api-key 参数 命令行临时覆盖
2 SN_*_API_KEY（技能专用） 某个技能单独指定 Key
3 SN_CHAT_API_KEY 对话类技能共用 Key
4 SN_API_KEY 全局默认 Key（推荐）
推荐配置：绝大多数场景只需设置 SN_API_KEY，所有技能会自动 fallback 到这个全局 Key。只有需要隔离计费或使用不同模型时，才需要设置技能专用变量。

Chapter 03

图像生成能力

五个技能组成从底层能力到上层应用的完整图像生产链。底层负责与 API 交互，上层面向具体业务场景。

sn-image-base
→
文生图 / 图像识别 / 文本优化
LAYER 0

基础设施层

sn-image-doctor 环境诊断工具。检查 Python 依赖、环境变量、API 连通性。安装后第一步建议先跑这个，确认环境就绪。
sn-image-base 核心工具层。封装了三个原子能力：文生图（自然语言描述 → 图片）、图像识别（图片 → 文字描述）、文本优化（提示词润色）。其他图像技能都依赖它。
LAYER 1

应用层

sn-infographic 信息图生成器。内置 87 种布局 和 66 种风格，支持 VLM 审稿。输入文字内容，自动选择布局和风格，输出专业信息图。
sn-image-imitate 风格模仿。给一张参考图 + 提示词，生成保持原图风格的新图片。适合品牌视觉延续、系列配图等场景。
sn-image-resume 简历图片生成。针对简历场景优化的人像生成能力。
Chapter 04

PPT 演示文稿

五个技能覆盖从输入解析到最终交付的完整 PPT 制作流程。双轨道设计满足不同质量要求。

内容输入
→
sn-ppt-entry
统一入口
→
Creative / Standard
→
PPTX / PNG
ENTRY

入口与诊断

sn-ppt-entry PPT 统一入口。解析用户输入的内容类型和需求，自动分发到 Creative 或 Standard 模式。用户只需调用这一个技能，不需要手动选择模式。
sn-ppt-doctor PPT 环境诊断。检查 SN_API_KEY、Node.js、sn-image-base 依赖。制作 PPT 前建议先运行。
MODE A

创意模式 — 快速视觉冲击

sn-ppt-creative 每页生成一张 16:9 PNG。速度快，视觉表现力强，适合对排版精度要求不高、追求快速出图的场景（如社交媒体传播图、创意提案）。
MODE B

标准模式 — 专业交付级

sn-ppt-standard 逐页生成 HTML → VLM 视觉审稿 → 修正 → 导出 PPTX。流程更严谨，输出可编辑的 .pptx 文件。适合正式汇报、客户提案等需要后续微调的场景。
Creative
快速出图，PNG 输出
Standard +
专业级，PPTX 可编辑
VLM Review
视觉质量自动审查
Chapter 05

数据分析引擎

三个技能构成轻量级数据分析工作站。从 Excel 多 Sheet 处理到万行级大文件，覆盖日常数据工作的核心痛点。

CORE

核心技能

sn-da-excel-workflow Excel 全流程处理。支持多 Sheet 读取、数据清洗（去重/空值/异常值/格式统一）、统计分析（分组/趋势/KPI/透视表）、可视化（柱状图/折线图/环形图），最终导出为新的 Excel 文件。内置 40+ 子能力模块，按需组合。
sn-da-image-caption 图像理解。专门处理表格截图的 OCR 提取和图表解读。当你拿到的数据不是 Excel 而是 PDF 截图时，这个技能可以把图片中的表格转成结构化数据。
sn-da-large-file-analysis 大文件处理。专门应对 10,000 行以上的数据集。使用流式读取和 Parquet 中间格式，避免内存溢出。常规 Excel 处理不了的，交给它。
适用场景：日常数据报表制作、多源数据合并清洗、截图表格数字化、季度/年度数据分析、异常数据排查。
Chapter 06

深度研究编排

七个技能组成五阶段研究流水线。从"帮我调研 XX 行业"到输出完整的 HTML 报告，全程自动化编排。

计划
→
维度研究
→
综合判断
→
报告撰写
→
格式排版
ORCHESTRATOR

编排器

sn-deep-research 深度研究统一入口。接收研究课题，自动编排后续四个阶段的执行。用户只需与这一个技能交互，不需要手动串联。
STAGE 1-3

研究过程

sn-research-planning 研究计划生成。分析课题，拆解研究维度，制定搜索策略和信息源优先级。
sn-dimension-research 按维度搜集证据。每个维度独立执行搜索 → 提取 → 归纳，输出结构化的维度发现。
sn-research-synthesis 综合判断合成。跨维度交叉验证，识别矛盾点和共识区，形成研究结论。
STAGE 4-5

报告输出

sn-research-report 最终报告撰写。基于综合结论，生成结构化的 Markdown 研究报告，包含摘要、正文、数据支撑和结论建议。
sn-report-format-discovery 报告格式发现。根据报告类型（行业分析/竞品研究/趋势预测等）自动匹配最佳排版模板。
sn-md-to-html-report Markdown 转 HTML。将 Markdown 报告转换为可直接浏览器阅读的离线 HTML 文件，支持目录导航和响应式布局。
Chapter 07

学术与社交搜索

四个垂直搜索引擎，每个专注一个信息源类型。不是通用搜索的包装，而是针对每个平台特性做了深度优化。

学术搜索 sn-search-academic

覆盖 ArXiv、Semantic Scholar、PubMed、Wikipedia。支持按引用量排序、时间范围筛选、摘要提取。科研工作者的第一站。

开发者搜索 sn-search-code

覆盖 GitHub、Stack Overflow、Hacker News、HuggingFace。精准定位代码仓库、技术问答、模型权重和行业讨论。

中文社交 sn-search-social-cn

覆盖 B站、知乎、抖音。专攻中文互联网的内容生态，适合市场调研、用户洞察、热点追踪。

英文社交 sn-search-social-en

覆盖 Reddit、Twitter、YouTube。国际舆论场的实时脉动，产品出海和竞品分析的关键信息源。

Chapter 08

使用方式

前置准备

使用前需要完成两项配置：

步骤 操作 说明
1 设置 API Key 将 SN_API_KEY 写入环境变量或 Claude Code 的 .claude/settings.local.json
2 运行诊断 调用 sn-image-doctor 或 sn-ppt-doctor 确认环境就绪

日常使用

技能安装后会被 Claude Code 自动识别。直接用自然语言描述需求即可：

示例对话：
"帮我用 sn-infographic 生成一张关于 AI 行业趋势的信息图"
"用 sn-deep-research 帮我调研一下 2026 年大模型定价策略"
"用 sn-ppt-entry 把这份报告做成 PPT，标准模式"
"用 sn-search-academic 搜索关于多模态大模型的最新论文"

技能更新

sn-update 技能更新工具。一键检查并拉取上游最新版本的 SenseNova Skills。保持工具链与商汤官方同步。

安装完成。环境就绪。直接开始使用。
