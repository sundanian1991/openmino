# Design Intent — 2026年技术路线图

> 测试场景 5：多图 + 5+ 张图

---

## 基本信息

- **任务**：2026 年技术路线图可视化
- **输入类型**：文字描述 + 概念
- **数据状态**：混合（时间节点数据 + 概念性路线图）
- **受众**：技术团队 + 管理层
- **使用场景**：汇报 + 教育

## 意图识别

### 你要表达什么？

| 维度 | 判断 | 理由 |
|------|------|------|
| 核心关系 | 趋势 + 层级 + 时间序列（路线图本质是时间线上的技术演进） | 技术路线图需要展示时间、技术分层、演进方向 |
| 叙事目的 | 展示技术演进路径和关键里程碑 | |
| 认知缺口 | 读者现在不知道技术发展路径和优先级，看完应知道什么时间做什么、为什么 | |

### 模式匹配

> 多图场景（5+ 张图），每张图独立意图匹配：
> 1. 技术全景总览 → 18-元素周期表（分类+属性二维网格）
> 2. 时间线演进 → C04-时间线+里程碑
> 3. 技术成熟度 → 22-雷达图（多维能力评估）
> 4. 技术依赖关系 → 20-桑基图（流入→流出→转化/依赖）
> 5. 投资优先级 → C03-2x2 矩阵（两维度四象限）

- **主选模式**：见上方多图拆分表
- **备选模式**：各图有备选
- **为什么没选备选**：意图匹配度

## 自动选择记录

- **输入**：2026年技术路线图
- **Hash**：H = hash("2026年技术路线图")（UTF-8 字节累加和 mod 1000000）
- **L1 内容锁死**：
  - 模式：每张图独立意图匹配（不走 hash），见多图拆分表
  - 弧线：03 演进式（历史/趋势 = 技术路线图天然路径，意图匹配，**不走 hash**）
  - 构图：07 水平时间轴（5+ 图 + 时间序列 = 水平时间轴，**不走 hash**）
    - 或构图 10 仪表盘（4+ 图 = 仪表盘）。此处选择 07 因为技术路线图有明确的时间递进关系
- **L2 场景约束**：
  - 场景：高管汇报（技术路线图用于汇报）→ Tier 1 = [01 麦肯锡, 02 贝恩, 03 BCG]，Tier 2 = [04 Tufte, 05 FT]
  - 风格学派：H % 5 = 余数 → 从候选池选择
    - **注：多图场景若存在 global-style.md，直接从全局读取，跳过 hash**
  - 开场骨架：时间故事 → Tier 1+2 = [03 全景时间线, 06 演变轨迹]
    - 若 global-style 未锁定开场：(H/10) % 2 = 余数 → 选择（回退 hash）
    - 若 global-style 已锁定开场：从全局读取，跳过 hash
- **L3 风格自由**：
  - **多图场景：全部从 global-style 继承，不 hash**
  - 配色：从 global-style 读取（不 hash）
  - 字体：从 global-style 读取（不 hash）
  - 节拍：走弧线自带序列（不 hash，03 演进式 = 起点→变化→拐点→终态）
