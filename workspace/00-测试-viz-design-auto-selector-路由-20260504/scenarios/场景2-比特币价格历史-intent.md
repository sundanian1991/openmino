# Design Intent — 比特币价格历史

> 测试场景 2：单图 + 时间序列

---

## 基本信息

- **任务**：比特币价格历史可视化
- **输入类型**：数据
- **数据状态**：有精确数据
- **受众**：通用
- **使用场景**：分析

## 意图识别

### 你要表达什么？

| 维度 | 判断 | 理由 |
|------|------|------|
| 核心关系 | 趋势（时间序列） | 价格随时间的变化 |
| 叙事目的 | 展示比特币价格的历史走势和关键拐点 | |
| 认知缺口 | 读者现在不知道价格变化趋势，看完应了解整体走势和重要时间节点 | |

### 模式匹配

- **主选模式**：07-折线图（趋势变化）
- **为什么选这个**：时间序列 + 单一指标（价格）→ 折线图是最直观的趋势表达
- **备选模式**：08-面积图
- **为什么没选备选**：面积图强调累积量，比特币价格不需要表达"累积"概念

### 多图拆分（不适用）

> 单图场景，不拆分。

## 自动选择记录

- **输入**：比特币价格历史
- **Hash**：H = hash("比特币价格历史")（UTF-8 字节累加和 mod 1000000）
- **L1 内容锁死**：
  - 模式：07-折线图（意图匹配：趋势变化 = 时间序列数据 → 折线图，**不走 hash**）
  - 弧线：无（单图不走弧线，**不走 hash**）
  - 构图：01 单中心聚焦（1 张图 = 单中心，**不走 hash**）
- **L2 场景约束**：
  - 场景：数据新闻（通用数据分析场景）→ Tier 1 = [05 FT, 06 Economist, 07 NYT]，Tier 2 = [10 OWID, 17 Cairo]
  - 风格学派：H % 5 = 余数 → 从 5 个候选中选择（hash 选择）
  - 开场骨架：时间故事 → Tier 1+2 = [03 全景时间线, 06 演变轨迹]
    - (H/10) % 2 = 余数 → 选择（hash 选择）
- **L3 风格自由**：
  - 配色：从已选风格学派的色系方向 → (H/100) % 色系数量 → 色系选择 → (H/1000) % 25 → color-themes #N（hash 选择）
  - 字体：从已选风格学派 → (H/10000) % 12 → typography #N（hash 选择）
  - 节拍：H % 20 → 选择 2-3 个节拍组合（hash 选择）。预期节拍：C-05 趋势拐点 + C-12 运动方向
