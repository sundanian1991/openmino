# Task Specification
教"会用 AI 但没做过产品的职场人"用人机共驾三层协同做出完整产品。
产品 = 可复制学会的教学包（看/点/试/抄四能力），不是"一个网页"。
三层协同：执行层 vibe / 编排层 orchestrate-projects / 接力层 task-handoff。

# Key Decisions（按时间）
1. **三层协同定位**（非只教 vibe 流水线）——灵感来自 Gabriel Chua《Collaborating with Codex》
2. **项目自指**：用三层协同开发出来的全套产物本身是教学样本
3. **走法 B**：先 orchestrate-projects 编排，再在编排框架下走 vibe（三层从第一天就在场）
4. **MVP 形态**：站会大盘单页 + 走读 + 接力实战；最终形态后定
5. **dashboard 共驾是硬约束**
6. **站会大盘视觉**：Nothing Light + B+C 融合（主卡浮网格点+数字矩阵hero+briefing+三段式聚合+卡外附录）
7. **M2 裁剪**：只写主页，taxonomy 只用 popover+inline-expand，a11y/动效/四态延后
8. **技术栈 = 纯静态教学包**（最大决策）：双击即开零门槛；三层分离（JSON+HTML+JS）；放弃 Stitch/CloudBase/shadcn
9. **页面生成机制**：数据+模板+渲染三层分离（AI 只改 JSON）
10. **M7 提前**：在 M6 前先演示 task-handoff，让用户早体验接力
