一、项目定位

1.1 为什么要做

30+ 供应商、3000+ 人团队、4 条业务线，当前供应商关系管理主要依赖"出问题后处理"的被动模式。头部 6 家供应商（毅航、毛毛虫、伽玛、赛维斯、岐力、翰锐）扛起金条 3 亿+产能，但关系维护深度不均——部分供应商只在月度评估时才有系统互动，中间数周处于"盲飞"状态。

风险事件（合规问题、业绩下滑、人员流失、合同纠纷）往往在发生后 1-2 周 才被发现，错过了最佳干预窗口。精细化关系管理和风险前置的核心目标是：把被动响应变成主动感知，把事后补救变成事前预防。

1.2 解决什么问题

#问题具体表现
1关系维护深度不够仅靠月度评估和异常事件驱动互动，缺乏日常温度管理
2风险发现滞后合规问题、业绩异常、人员波动平均发现延迟 7-14 天
3合同/证照到期无预警已有 2 起供应商证照过期未及时续期的先例
4供应商情绪/合作意愿无量化供应商满意度仅凭主观感受，无数据支撑
5分级管理未落地ABC 分级和 PIP 流程已有框架，但未形成日常执行节奏

三、实施框架

3.1 核心方法论

"三层心跳 + 风险日历 + 闭环管理"

三层心跳核心机制
  • 日心跳 → 自动数据采集（出勤、业绩、异常信号）
  • 周心跳 → 人工扫描（周报分析 + 情绪判断 + 合同到期检查）
  • 月心跳 → 深度评估（月度综合评估 + 关系健康度评分 + 分级调整）
风险日历提醒规则
  • 合同到期 → 提前 60 天提醒 + 30 天预警 + 7 天紧急
  • 证照到期 → 提前 90 天提醒 + 30 天预警
  • 合规检查 → 月度固定日期（每月 10 号）
  • 出差巡检 → 季度固定计划（Q1:3月 Q2:6月 Q3:9月 Q4:12月）

3.4 预警分级与响应机制

级别触发条件响应时限响应动作升级路径
黄色 单项指标偏离基准 10-20%（如业绩下滑 15%、出勤率降至 85%） 3 天内 发送预警通知 + 要求说明 3 天未响应 → 升级橙色
橙色 单项指标偏离 >20% 或多项同时偏离 / 黄色未响应 24 小时内 电话沟通 + 现场核查 + 整改计划 24h 未响应 → 升级红色
红色 触发红线（数据泄露、资质造假、录音篡改）/ 橙色未响应 立即 暂停业务 + 启动调查 + 法务介入 直接启动清退评估

  • C 级：约谈会议（参考 A2-约谈准备决策卡片），明确改进方向
  • D 级：立即启动 PIP 流程
第 5-8 周

常态化运行

动作8：心跳机制常态化
  • 日心跳：自动化运行，异常自动标记
  • 周心跳：每周五下午执行
  • 月心跳：每月评估后 3 个工作日内完成健康度更新
动作9：风险日历自动化
  • 设置提醒规则，到期自动推送
  • 与 SM-TPL 报送系统对接（如有条件）
动作10：分级动态调整
  • 每月根据健康度评分重新分级
  • 连续 2 月升级 → 上调一级
  • 连续 2 月降级 → 下调一级并启动对应措施
动作11：出差巡检计划执行

7.3 应对措施

预警阈值调整季度复审
  • 每季度回顾一次预警准确率
  • 准确率 <50% → 提高阈值（减少误报）
  • 准确率 >80% 但漏报率高 → 降低阈值（增加敏感度）
分级争议处理申诉流程
  1. 供应商可在收到分级结果后 5 个工作日内提出申诉
  2. 服务组在 3 个工作日内复核并回复
  3. 如确有误判，立即调整并通知供应商
PIP 执行保障质量保障
  • PIP 目标必须 SMART（具体、可衡量、可实现、相关、有时限）
  • 每周检查 PIP 进度
  • 改进期中提供必要支持（培训、资源）
  • 期满严格评估，不达标即启动清退
