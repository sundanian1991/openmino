# Workspace 重构方案 — Lifecycle + 工作流状态

**创建时间**：2026-03-16
**核心逻辑**：按文件生命周期组织 + 工作流状态管理

---

## 设计原则

### 1. 三区域架构

| 区域 | 定位 | 内容类型 | 留存周期 |
|------|------|---------|---------|
| **Workspace** | 入口区 | 临时文件、草稿、当前工作 | 周度清理 |
| **Docs** | 归档区 | 正式文档、已完成项目 | 长期保留 |
| **Memory** | 核心区 | 洞察、偏好、决策 | 永久保留 |

### 2. 工作流状态

每个主题下分三个状态：

| 状态 | 图标 | 说明 | 示例 |
|------|------|------|------|
| **进行中** | 🔄 | 当前活跃工作 | 正在写的评估报告 |
| **待处理** | ⏳ | 排队等待处理 | 待回复的邮件草稿 |
| **已完成** | ✅ | 完成暂存 | 完成的项目（待周回顾） |

### 3. 知识库独立

**知识库**不属于工作流，是稳定的参考文档，放在主题目录下独立管理。

## 流转规则

### 日常使用流程

```
新工作 → ⏳待处理 → 🔄进行中 → ✅已完成
                              ↓
                           周回顾判断
                              ↓
                    ┌─────────┼─────────┐
                    ↓         ↓         ↓
                docs/     memory/    删除
               （归档）    （记录）
```

### 具体操作

| 场景 | 操作 |
|------|------|
| **新任务** | 在对应主题下创建文件，放入 `⏳待处理/` |
| **开始处理** | 移动到 `🔄进行中/` |
| **完成工作** | 移动到 `✅已完成/` |
| **周回顾** | 检查 `✅已完成/`，判断去向 |
| **有价值文档** | 移至 `docs/项目归档/` |
| **有洞察内容** | 提炼到 `memory/active/my-thoughts/` |
| **无价值** | 删除 |
