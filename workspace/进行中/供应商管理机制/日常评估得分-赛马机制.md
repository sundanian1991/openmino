# 日常评估得分 — 赛马机制

> **核心思路**：用制度管，不是用人盯
> **创建时间**：2026-03-21
> **状态**：方案初稿

---

## 一、背景

### 问题描述

1. **周末被打扰**：年老师双休日不常上班，但刘乾坤会在周末不断@处理各种供应商问题
2. **责任边界不清**：供应商日常出勤管理，到底是谁制定制度并监管？是否所有事都要年老师亲自完成？
3. **小问题不断上报**：晚10分钟、晚半小时、出行安排不到位——各种日常小问题都要即时处理

### 年老师的困惑

> "无论是工商这边晚了10分钟、晚了半小时，还是出行安排不到位，都属于同样的情形。如果所有这些问题都必须上报并处理，那么可以对整体行为进行综合评分。"

### 解决思路

**不即时处理每个小问题**，而是建立一套**日常评估得分制度**：
- 出问题记录为扣分项
- 扣分影响整体业绩、排名、分量
- 让制度自动管，而不是人工盯

---

## 二、方案设计

### 核心原则

| 原则 | 说明 |
|------|------|
| **用制度管，不用人盯** | 不实时盯群，定期看数据 |
| **记录-扣分-影响结果** | 不即时处理，但结果上体现 |
| **赛马逻辑** | 排名公开，让供应商自己卷 |
| **归口供应商管理** | 制度由年老师制定，不归刘乾坤 |

### 评分框架

#### 1. 分数设置

| 项目 | 设置 |
|------|------|
| **总分** | 10 分 |
| **扣分项** | 每次出现问题扣 0.5-2 分 |
| **评分周期** | 按月累计 |
| **影响范围** | 整体业绩、排名、分量 |

#### 2. 扣分项类型（初版）

| 类型 | 扣分标准 | 说明 |
|------|----------|------|
| **数据回复延迟** | 晚 ≤30分钟：-0.5 分<br>晚 30-60分钟：-1 分<br>晚 >60分钟：-2 分 | 以约定时间为准 |
| **周末无人值守** | -1 分/次 | 周末消息未读或超时未回 |
| **数据质量问题** | -0.5-1 分/次 | 错误、遗漏、格式不对 |
| **出行安排不到位** | -0.5-1 分/次 | 人员未按时到岗 |
| **其他响应问题** | -0.5-1 分/次 | 按严重程度判定 |

#### 3. 运作流程

```
供应商发生问题 → 年老师记录在案（不即时处理）
      ↓
月度汇总统计 → 计算当月得分
      ↓
得分纳入整体评估 → 影响排名、分量、业绩
      ↓
供应商看到结果 → 自己想办法改进
```

## 三、与现有体系的关系

### 现有体系

| 体系 | 用途 | 阶段 |
|------|------|------|
| **引入评分** | 供应商引入决策 | 引入前 |
| **日常评估得分** | 持续运营管理 | **运营中** ← 新增 |
| **月度/季度评估** | 综合业绩评定 | 定期 |

### 日常评估得分的定位

```
引入评分 → 是否准入
  ↓
日常评估得分 → 每月运营表现 ← 新增
  ↓
月度/季度评估 → 综合业绩、排名
  ↓
年度评估 → 分量调整
```

## 四、实施要点

### 对年老师

- **周末不秒回**：让供应商习惯"年老师周末有自己节奏"
- **记录而非即时处理**：看到问题记下来，月底统一算账
- **用结果说话**：不盯过程，看最终得分

### 对供应商

- **告知规则**：提前公布扣分项和标准
- **月度公布**：每月公布得分和排名
- **自己想办法**：不教他们怎么做，只告诉结果

### 对刘乾坤

- **制度明确**：告诉他"我已经建立日常评估制度，不需要每次@我"
- **归口清晰**：供应商管理归年老师，不是周末随叫随到
