## 8 件逐个超越策略

### 1. 联盟分享会毅航合肥（叙事+gauge）

**Nothing 核心**：hero 半圆 gauge + 圆形打破 + 红色 accent + sticky nav + 分享会叙事
**nian 超越**：
- gauge 换成 nian olive 描边 + ghost 大字"分"做打破
- 红色 accent 换成 yellow（功能性识别）+ orange（视觉焦点）
- 分享会叙事用三层金字塔：Answer（一句话总结）/ Argument（3 个亮点）/ Evidence（数据来源）
- 衬线 Playfair 做 hero 标题，分享会"人"的温度回来

### 2. 设计系统自我介绍（元设计宣言）

**Nothing 核心**：dot grid + Doto 120px Hero + OLED 黑底 + 元设计
**nian 超越**：
- 黑底改 nian darkgray #2C2C2C（不是纯黑，保留温度）
- Doto 120px 换 Playfair italic 120px（衬线宣言更有"刻"感）
- dot grid 换 SVG noise 布纹（有机质感替代机械点阵）
- ghost 用 Playfair italic "NIAN" opacity 0.05

### 3. 关联矩阵（6×4 数据密度）

**Nothing 核心**：6 供应商×4 产线 + 3 级灰阶 + 红角标
**nian 超越**：
- 3 级灰阶换成 nian 4 级：olive（主力）+ darkgray（协同）+ surface-raised（试水）+ bg（未覆盖）
- 红角标换 yellow 圆点（合规标杆）+ orange 圆点（预警）
- 三层金字塔 hero 替代 64px Doto 数字
- **已在《6供应商分级评估.html》实现**

### 4. 区位矩阵（8 城市热力）

**Nothing 核心**：8 城市×4 指标 + bar fill 嵌入 + 热力灰阶
**nian 超越**：
- bar fill 用 nian olive→earth→glacier 渐变（替代单黑）
- best/worst 标记用 olive/orange（替代绿/红）
- hero 用三层金字塔：Answer（最佳职场）/ Argument（4 指标对比）/ Evidence（数据来源）
- ghost 用 Playfair italic "8" 做打破
- **将在《8职场区位矩阵.html》实现**

### 5. 分面条形（6 维度对比）

**Nothing 核心**：6 维度×3 供应商 + 替代雷达 + gap callout
**nian 超越**：
- bar fill 用 nian 4 级：olive（rank1）+ darkgray（rank2）+ earth（rank3）+ orange（预警）
- gap callout 用 orange 左边框（替代红）+ ghost 大字"差"
- 三层金字塔 hero：Answer（最大差异维度）/ Argument（3 家对比）/ Evidence（评分依据）
- 衬线维度名 + Mono 数据值

### 6. 暗色模式（OLED 黑）

**Nothing 核心**：OLED 黑 + gauge + segmented + LED 数码管
**nian 超越**：
- OLED 黑换 nian darkgray #2C2C2C + glacier #2A4A5A 冷调叠加（不是纯黑，有海拔感）
- LED 数码管换 Playfair italic 数字（衬线温度）
- gauge 用 olive 描边 + ghost 大字"夜"做打破
- segmented 用 4 级：olive/darkgray/earth/orange
- **将在《暗色模式-nian版.html》实现**

### 7. 趋势线（Sparkline）

**Nothing 核心**：SVG polyline 160×32 + sparkline 表格 + sparkline 卡片
**nian 超越**：
- sparkline 用 olive（上升）+ orange（下降）+ darkgray（平稳）替代绿/红/黑
- 末端圆点用 yellow（创新高）+ orange（新低）
- 三层金字塔 hero：Answer（趋势结论）/ Argument（12 月走势）/ Evidence（数据周期）
- 表格用衬线供应商名 + Mono 数据值
- **将在《趋势线-nian版.html》实现**

### 8. 非对称构图（设计教学）

**Nothing 核心**：对称 vs 不对称对比 + 96px hero number + 设计教学
**nian 超越**：
- 96px Doto 换 Playfair italic 96px（衬线教学感）
- 对称/不对称对照保留，但用 nian 7 色重新着色
- ghost 大字"构"做打破
- 三层金字塔解释"为什么不对称"：Answer（结论）/ Argument（原理）/ Evidence（案例）

## 衔接切口建议（4 条）

### 切口 1：nothing ↔ nian 气质切换
nian-decision-card 增 `visualStream: Industrial` 时回退 nothing 色板（黑白灰+单红），避免气质打架。

### 切口 2：决策卡→施工契约
9 气质强制对应 Hero 类型，禁止跨气质混搭：
- Statement → V4-Statement（Playfair 断言）
- Diagonal → V1-Diagonal（斜切入场）
- Editorial → V3-Editorial（衬线长文）
- ...

### 切口 3：showcase 对标库
nian showcase 旁挂 `references/showcase-archive/compare/` 放 nothing 29 分对标件，做参照系。每个 nian 新产出必须能回答"它比 nothing 对标件强在哪"。

### 切口 4：组件缺口补齐
nian 当前 32 组件无以下高级形态，需补 6 族：
- **关联矩阵**（已在《6供应商分级评估》验证）
- **区位矩阵**（热力+bar fill 嵌入）
- **分面条形**（多维对比）
- **Sparkline 表格**（趋势+数据行）
- **对称/非对称对照**（设计教学）
- **Gauge 仪表盘**（半圆+刻度）
