# CLAUDE.md — Workspace 工作区

> **所有工作文件的唯一入口**

---

## 核心定位

Workspace 是**入口区**，存放：
- 临时文件和草稿
- 当前正在处理的工作
- 待处理的任务
- 完成待归档的内容

**留存周期**：周度清理（每周回顾后流转到 docs/ 或 memory/）

---

## 目录结构

```
workspace/
├── inbox/              # 新文件入口（必须清零）
│   └── {刚下载/刚生成/未分类的文件}
│
├── drafts/             # 草稿（编辑中，尚未定稿）
│   └── {草稿文件}
│
├── active/             # 进行中项目（必须带日期）
│   ├── {主题}-{YYYY-MM-DD}/
│   │   ├── {工作文件}.md
│   │   ├── CLAUDE.md   # 项目说明（可选）
│   │   └── ...
│   └── ...
│
├── output/             # 已交付产出物
│   ├── {主题}-{YYYY-MM-DD}/
│   │   ├── {主题}-v1.html
│   │   ├── {主题}-v2.html
│   │   └── ...
│   └── {主题}-v{数字}.html  # 独立产出
│
├── archive/            # 已归档（按周期）
│   ├── 2026-W{周数}/
│   └── 2026-MM/        # 月度归档
│
└── docs/               # workspace 专用文档
    └── {模板、SOP、检查清单}
```

> **通用模板/SOP 在 `docs/workspace-knowledge/`，不在此处**

---

## 生命周期

```
新文件 → inbox/         （自动识别放入）
  ↓ 判断用途
  草稿 → drafts/         （编辑中）
  项目 → active/{主题}-{日期}/（有明确项目归属）
  ↓ 产出交付
  产出 → output/{主题}-{日期}/  （HTML/PPTX/PDF 等终版）
  ↓ 定期清理
  旧版本 → keep 最近的 2-3 个版本，其余归档
  过期项 → archive/{周期}/
```

**每周清理一次**：inbox/ 必须清到 0，active/ 评估状态变更

## 文件命名规范

| 场景 | 格式 | 示例 |
|------|------|------|
| 主题文件夹 | `{YYYY-MM-DD}-{主题}/` | `2026-04-05-成本调研/` |
| 有版本号 | `{YYYYMMDD}-{主题}-v{数字}.{ext}` | `20260405-供应商通报-v2.html` |
| 带日期 | `{YYYYMMDD}-{主题}.{ext}` | `20260402-Q2通报.html` |
| 项目工作文件 | `{内容}.md` （在 active/{项目名}/ 下） | `调研报告.md` |
| 归档 | `{原名}` （在 archive/{周期}/ 下） | `2026-Q1供应商评估.md` |

**禁止命名**：
- `完整版`、`最终版`、`最终` — 用 v{数字} 代替
- `校准过的精确`、`分析性精确` — 用有意义的版本标识
- `_` 作为主题分隔符 — 统一用 `-`（下划线只用于英文/拼音连接，如 `2026-W12`）
- 空格 — 统一用 `-` 连接

**版本号规则**：
- v1 是首个交付版本
- 每次修改提升版本号（不需要每次交付都升）
- 不需要 `v0`、`v0.1`

## 新增文件规则

1. **所有新增文件先进入 workspace/**
2. **先归类到 inbox/**，明确用途后再移动到目标目录
3. **如果文件没有前置约定**（不属于已有项目/主题），放 inbox/ 并在文件名中用 `-` 描述内容
4. **产出物统一放 output/**，不直接挂 workspace/ 根目录

## AI 访问指南

当 AI 需要在 workspace/ 创建文件时：

### 第一步：判断文件类型

| 文件类型 | 存放位置 | 命名规则 |
|---------|---------|---------|
| 草稿/编辑中 | drafts/ | `{主题}-{简述}.md` |
| 项目工作文件 | active/{主题}-{YYYY-MM-DD}/ | `{内容}.md` |
| 产出物（HTML/PPTX/PDF） | output/{主题}-{YYYY-MM-DD}/ | `{主题}-v{数字}.{ext}` |
| 临时文件 | inbox/ | `{描述}.{ext}` |

### 第二步：检查目录是否存在

```bash
# 检查目录
ls -la workspace/active/
ls -la workspace/output/

# 创建目录（如需要）
mkdir -p workspace/active/{主题}-{YYYY-MM-DD}
mkdir -p workspace/output/{主题}-{YYYY-MM-DD}
```

### 第三步：创建文件

- 确认命名符合规范（无禁止词汇、用 `-` 连接）
- 确认版本号从 v1 开始（如有）
- 确认文件放在正确位置

## 与其他目录的区别

| 目录 | 内容 | 典型文件 | 留存周期 |
|------|------|---------|---------|
| **workspace/** | 工作文件与产出 | 草稿、项目文件、HTML/PPTX 产出 | 周度清理 |
| **docs/** | 长期知识资产 | FAQ、SOP、方法论、制度 | 长期保留 |
| **memory/** | 记忆与索引 | 回忆、人物画像、项目索引 | 永久保留 |
| **plans/** | 计划与思考过程 | brainstorm、research、review | 项目结束后归档 |
| **projects/** | 长期项目追踪 | 职业辅导、供应商管理体系 | 长期保留 |
| **reference/** | 参考代码与资料 | Claude Code SDK 源码 | 长期保留 |
