# workspace/ — 工作区

> 入库，git 跟踪。按大类前缀组织，同类主题聚合。

---

## 核心原则

**一个文件夹 = 一个主题**

- 同类内容放一起
- 不同内容必须分开
- 主题清晰，一看就知道是什么

---

## 命名规则

**格式**：`NN-大类前缀-主题名称-YYYYMMDD`

- `NN` = 两位数字序号（00-99），按项目启动时间倒序编号，最新的编号最小
- 新建文件夹时，取当前最大序号 +1
- 日期格式：`YYYYMMDD`（年月日紧凑格式，不含分隔符）

**大类前缀**：

| 前缀 | 含义 | 示例 |
|------|------|------|
| **供应商** | 供应商管理相关工作 | 01-供应商-巡检整改函-20260422 |
| **成长** | 个人成长、能力提升 | 10-成长-Plan模板可视化-20260419 |
| **技能** | 具体技能学习与实践 | 06-技能-全量可视化分析-20260421 |
| **生活** | 个人生活相关 | 30-生活-游泳学习-20260404 |
| **记录** | 学习记录、历史提炼 | 29-记录-700会话历史提炼-20260405 |

**命名要求**：
- 主题要具体，禁止模糊命名
- 日期用项目启动日期或最早文件日期
- 数字序号保证按时间倒序排列，最新的编号最小

---

## 新增文件夹流程

**第一步：判断大类**
- 跟供应商管理相关 → `供应商-`
- 个人成长、职场能力 → `成长-`
- 具体技能学习 → `技能-`
- 个人生活 → `生活-`
- 历史记录提炼 → `记录-`

**第二步：检查是否已有同类主题**
- 有 → 合并进去，作为子文件夹或追加文件
- 没有 → 新建文件夹

**第三步：编号**
- 查看当前最大序号：`ls -1 | grep -oE '^[0-9]+' | sort -n | tail -1`
- 新文件夹序号 = 当前最大 +1

**第四步：命名**
- 格式：`NN-大类前缀-主题名称-YYYYMMDD`
- 主题名称不超过 10 个字

**第五步：组织内容**
- 简单项目（≤5个文件）：直接放文件
- 复杂项目（>5个文件）：用子文件夹组织

---

## 防呆检查（新建前必做）

新建文件夹前，必须执行：

1. `ls -1 | grep "主题关键词"` — 确认无同名/近似文件夹
2. 序号不冲突 — 取最大序号 +1
3. 大类前缀从 5 个中选，不自创新前缀
4. 日期用 `YYYYMMDD` 格式，不带分隔符

---

## 分类原则

### 什么时候合并

同类主题 → 合并到一个文件夹

示例：
- 已有 `33-供应商-站点开发-20260403`
- 新增站点看板 → 放进去作为子文件夹

### 什么时候新建

不同主题 → 新建文件夹

示例：
- `供应商-成本调研` 和 `供应商-评估与KPI体系` 是不同的事
- 即使都是供应商相关，也要分开

---

## 操作清单

新增文件夹前，检查：

1. [ ] 大类前缀选对了吗？（只从 5 个中选）
2. [ ] 是否已有同类主题？→ 合并
3. [ ] 命名清晰吗？→ 看一眼就知道是什么
4. [ ] 日期正确吗？→ 用项目启动日期，YYYYMMDD 格式
5. [ ] 序号正确吗？→ 取当前最大序号 +1
