# Workspace — 工作区

> 所有工作文件的唯一归属目录。

---

## 目录结构

```
workspace/
├── inbox/      — 新文件入口（刚下载/刚生成/未分类的文件）
├── drafts/     — 草稿（正在编辑，尚未定稿）
├── active/     — 进行中项目
│   ├── {项目名}/     # 项目工作文件
│   └── ...
├── output/     — 已交付产出物
│   ├── {主题}/       # 按主题分子目录
│   │   ├── {主题}-v1.html
│   │   └── {主题}-v2.html
│   └── {主题}.ext   # 无子目录的独立产出
├── archive/    — 已归档（按日期/周/月命名）
│   └── 2026-W12/
│   └── 2026-W13/
└── _templates/ — 模板、SOP、参考规范
```

---

## 生命周期

```
新文件 → inbox/         （自动识别放入）
  ↓ 判断用途
  草稿 → drafts/         （编辑中）
  项目 → active/{项目名}/ （有明确项目归属）
  ↓ 产出交付
  产出 → output/{主题}/  （HTML/PPTX/PDF 等终版）
  ↓ 定期清理
  旧版本 → keep 最近的 2-3 个版本，其余归档
  过期项 → archive/{周期}/
```

**每周清理一次**：inbox/ 必须清到 0，active/ 评估状态变更。

## 文件命名

| 场景 | 格式 | 示例 |
|------|------|------|
| 有版本号 | `{主题}-v{数字}.{ext}` | `供应商排名通报-v2.html` |
| 带日期 | `{主题}-{YYYYMMDD}.{ext}` | `Q2通报-20260402.html` |
| 项目工作文件 | `{内容}.md` （在 active/{项目名}/ 下） | `调研报告.md` |
| 归档 | `{原名}` （在 archive/{周期}/ 下） | `2026-Q1供应商评估.md` |

**禁止命名**：
- `完整版`、`最终版`、`最终` — 用 v{数字} 代替
- `校准过的精确`、`分析性精确` — 用有意义的版本标识
- `_` 作为主题分隔符 — 统一用 `-`（下划线只用于英文/拼音连接，如 `2026-W12`）
- 空格 — 统一用 `-` 连接

**版本号规则**：
- v1 是首个交付版本
- 每次修改提升版本号（不需要每次交付都升）
- 不需要 `v0`、`v0.1`

---

## 新增文件规则

1. **所有新增文件先进入 workspace/**
2. **先归类到 inbox/**，明确用途后再移动到目标目录
3. **如果文件没有前置约定**（不属于已有项目/主题），放 inbox/ 并在文件名中用 `-` 描述内容
4. **产出物统一放 output/**，不直接挂 workspace/ 根目录

---

## 与其他目录的区别

| 目录 | 内容 | 典型文件 |
|------|------|---------|
| **workspace/** | 工作文件与产出 | 草稿、项目文件、HTML/PPTX 产出 |
| **docs/** | 长期知识资产 | FAQ、SOP、方法论、制度 |
| **memory/** | 记忆与索引 | 回忆、人物画像、项目索引 |
| **plans/** | 计划与思考过程 | brainstorm、research、review |
| **projects/** | 长期项目追踪 | 职业辅导、供应商管理体系 |
| **reference/** | 参考代码与资料 | Claude Code SDK 源码 |
