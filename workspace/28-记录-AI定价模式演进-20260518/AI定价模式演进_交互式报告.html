AI 编程工具定价范式转移

从 Coding Plan 到 Token Plan — 一场成本透明化的行业运动

2026-05-18  |  AI 商业/定价策略  |  横纵分析法深度研究

从 2025 年中到 2026 年中，AI 编程工具行业经历了一场几乎同步的定价范式转移：从按固定功能/请求数计费的「Coding Plan」转向按 Token 消耗量计费的「Token Plan」。这场转变的本质不是涨价，而是成本结构的透明化——厂商不再用 VC 的钱为重度用户兜底。

$128B ▲ CAGR 27% 2026 年市场规模
70% 同时使用 2-4 个 开发者多工具率
70% Agent 会话浪费 Token 非代码生成占比

一、纵向分析：从诞生到当下

VC 补贴期：不限使用的幻觉（2022 末 ~ 2024 中）

一切要从一个现在看来近乎天真的假设说起：AI 编程工具的成本可以被扁平月费覆盖。2023 年初，GitHub Copilot 以 $10/月的价格横空出世，不限请求次数。背后是 Microsoft 的 Azure 基建兜底和 OpenAI 的早期补贴。同一时期，Cursor 定价 $20/月，Codeium（后来的 Windsurf）干脆全功能免费。

这个阶段的定价逻辑很简单——先证明 PMF，再谈赚钱。VC 出钱补贴 token 成本，厂商用不限使用的承诺快速吸用户。

但这里有一个致命的结构性问题。一个简单补全和一个多文件重构，对平台来说成本差 10 到 100 倍，但用户付一样的钱。这意味着两个截然不同的经济画像被塞进了同一个 $20 的盒子里：安静的用户一天零星补全，实际成本可能不到 $0.10/天；重度用户用 Agent 做小时级编辑，实际成本可能是 $10-50/天。

裂缝出现：请求数上限（2024 末 ~ 2025 初）

Agent 模式的出现是第一个裂缝。传统的代码补全是单步交互，token 消耗在几百到几千之间。Agent 模式完全不同：一个多步自主编码会话中，AI 要先读取上下文文件、搜索代码库、理解依赖关系、执行多轮编辑、验证结果——这个过程中 70% 的 token 不是用来生成代码的，而是用来维持 Agent 循环的。上下文随对话轮次线性增长：第 1 轮 5K token，到第 50 轮可能膨胀到 200K token。

2025-01 Cursor 将 500 次 Premium Requests 上限写入定价条款——隐性限制显性化
2025-04-21 Windsurf（Codeium）推出 Credit 计量模型，迈出扁平订阅转向用量计费第一步
2025-06 GitHub Copilot 引入 Premium Request 上限：Pro 300 次/月，Enterprise 1000 次/月，超出 $0.04/次
2025-06-16 ★ Cursor 切换到 Token 计费——行业第一枪。Pro 从"500 次 Fast Requests"改为"$20 API 信用额度"，推出 Ultra $200/月
2025-07-04 Cursor CEO Michael Truell 发布道歉博客，承认"沟通不清"，为意外收费提供全额退款
2025-08~09 Cursor Teams 切换到 API 计费；Auto 模式计费调整
2026-04-02 OpenAI Codex 从 per-message 切换到 Token-based 定价
2026-04 ★ GitHub 宣布 Copilot 全面转向 GitHub AI Credits（6/1 生效）；Anthropic 悄悄上调费用估算（$6→$13/天）

引爆点：Cursor 的第一枪

Cursor 是整个行业第一个从固定请求数切换到 Token 计费的主流 AI 编程 IDE。而且这一枪打得不够准，打在了自己脚上。

问题是，绝大多数用户不知道自己一天会消耗多少 API 信用。一个开发者报告单周超支 $350。有用户发现用 Claude 模型时，几次 Prompt 就把 $20 额度烧光了。Reddit 和 X 上爆发了大规模抱怨，TechCrunch 等主流媒体跟进报道。

危机数据：一个开发者 8 个月消耗 100 亿 token，API 价格折合 $15,000+，Max 计划只需 $800，实际节省 93%。

新模型在长 horizon 任务中每个请求消耗更多 token。最难的请求比简单请求贵一个数量级。基于 API 的定价是反映这一点的最佳方式。

—— Cursor 官方公告

但真正的原因是，Cursor 当时的 ARR 已经超过 $5 亿（后来在 2025 年中突破 $5B，大约每两个月翻倍），VC 补贴再也兜不住了。Simon Willison 对此的判断是："VC 补贴 token 的时代可能正在结束。"

成本透明化的过程本身就是一个信任风险。用户不怕贵，怕的是不知道自己会花多少钱。

切换的根因

表面上看，这场转变是厂商受不了 VC 补贴、要转嫁成本。但根因有三层：

第一层：Agent 工作负载彻底改变了成本结构。模型成本本身在下降——Claude Opus 4.0/4.1 定价 $15/$75 per MTok，4.6 降至 $5/$25（降了 67%）——但总 token 消耗的增长完全抵消了降价。

第二层：厂商无法继续为"同价不同量"买单。当安静用户和重度用户的成本差异从 10 倍拉到 100 倍时，任何扁平定价都会崩溃。

第三层：整个行业的盈利时间表在推后。OpenAI 2025 年收入约 $4B，成本约 $9B，预计 2029 年才能盈利。Anthropic 2025 年收入约 $1B，成本约 $5.6B，预计 2027 年盈利。

三、横向分析：竞争图谱

市场全景

85% 的开发者已经在使用 AI 编程工具，73% 日常使用。但最关键的一个数据是：70% 的工程师同时使用 2-4 个工具，15% 使用 5 个以上。"一个工具通吃"的假设已经死了。用户不是选一个忠诚度最高的工具，而是在构建自己的 AI 开发工具栈。

四大玩家：各活各的

Cursor

AI-first IDE 领导者。ARR 突破 $5B，100 万+付费用户，社区最活跃。但定价危机后品牌信任度受损。

Pro $20/月 · Ultra $200/月

Claude Code

Agent 能力最强，增长最猛。首次尝试准确率 ~80%，91% CSAT，54 NPS。工作场景使用率 12 个月翻了 6 倍。

Pro $20/月 · Max 20x $200/月

GitHub Copilot

企业分发王者，470 万付费用户，Fortune 100 中 90% 在用。但满意度仅 9%，垫底。

Individual $10/月 · 6/1 转向用量计费

Windsurf

性价比守门员，独立开发者首选。但被 Cognition 收购后反复折腾，用户信任受损，大量流失到 Cursor。

Pro $20/月 · Quota 配额制

用户迁移趋势：三大阵营

用户类型 首选工具 原因
初创公司 (<50 人) Claude Code (75%) Agent 多步任务处理能力最强
中型企业 (500-5K) Cursor (~50%) IDE 体验最好，Composer 强大
大型企业 (10K+) GitHub Copilot (56%) 微软分发、采购便利、合规

主流的使用模式不是选一个，而是堆叠：Cursor 用于日常编辑（主 IDE），Claude Code 用于复杂多步任务（重构、跨文件调试），GitHub Copilot 用于行内补全（响应速度仍然最快）。

四、横纵交汇洞察

四条路径，四种定价哲学

Cursor
被动防御型
"我们发现成本兜不住了，所以改"——被迫转型，信任代价最大
Claude Code
原生透明型
"从第一天就按真实成本结构定价"——没有心理落差，最干净
Copilot
圈地收缩型
"先圈用户，再慢慢收紧"——短期有效，长期伤害品牌
Devin
心理缓冲型
"用抽象单位让用户感觉不像在烧 token"——ACU 把"消耗"变成"工作时间"

Claude Code 的原生透明型最干净，因为它没有"从不限到限量"的心理落差。Cursor 的被动防御型代价最大，因为它是被迫的，不是主动的。Copilot 的圈地收缩型短期有效但长期伤害品牌——用户会觉得自己被"骗进来再宰"。Devin 的心理缓冲型最聪明，ACU 把"消耗"重新定义为"工作时间"，完全改变了用户的感知框架。

定价的本质不是定数字，是定框架。

未来推演：三个剧本

最可能的剧本：多元定价并存，工具栈化成为常态

Credits、Quota、ACU、按任务付费会共存。70% 的开发者使用 2-4 个工具的趋势会加强，厂商会从"争夺单一忠诚度"转向"争夺工具栈中的份额"。

核心逻辑：定价模式的多样性反映了用户工作负载的多样性。

最危险的剧本：成本转嫁引发大规模用户反弹

当用户月费从 $20 变成 $200+ 时，大量中小开发者会被挤出市场，削弱整个生态的开发者基数。

触发条件：模型成本停止下降 + Agent 用量持续暴增 + 经济环境收紧

最乐观的剧本：按结果付费成为主流

从"按投入（token）计费"到"按产出（完成任务数）计费"，成本和收益完全对齐。需要 Agent 首次尝试准确率突破 95%+。

触发条件：首次尝试准确率 95%+ + 任务完成定义足够清晰

更大的图景

这场定价转变背后是一个更大的问题：当 AI 的边际成本不是零的时候，什么样的商业模式是可持续的？

云计算从包月按需付费的演进花了 15 年才成熟。手机套餐从无限流量到分级套餐经历了无数用户抗议。每一次从"固定"到"按量"的转变都是痛苦的，因为用户天然偏好确定性，而按量计费天然带来不确定性。

AI 编程工具的这次转型，本质上是在回答一个所有 AI 应用最终都要面对的问题：当使用量的方差从 10 倍拉到 100 倍时，谁来承担这个方差？

目前的回答是：用户承担，但厂商用订阅 tiers 来封顶。这个答案不完美，但在模型成本继续下降之前，它是最现实的。

五、可复用的 6 条商业经验

经验 1：成本透明化不是涨价，沟通方式决定信任存亡

Cursor 的案例证明，用户最大的敌人不是价格本身，而是不可预测性。存活下来的厂商都做到了两件事：提供封顶选项（如 Claude Code 的 Max 计划），以及用"批量折扣"的叙事替代"限制消费"的叙事。

经验 2：原生定价 > 转型定价

Claude Code 从零开始就是 Token 计费，用户没有心理落差。Cursor 从不限使用转型过来，用户有被"骗进来再宰"的感觉。教训：第一天就用可持续的定价模型，不要用 VC 补贴制造虚假的用户预期。

经验 3：心理框架比数字更重要

Devin 的 ACU（15 分钟工作时间）是一个绝妙的定价创新。同样的成本，用"token 消耗"的框架用户觉得在烧钱，用"工作时间"的框架用户觉得在雇佣一个员工。定价的本质不是定数字，是定框架。

经验 4：从投入计费到产出计费是必然方向

Token Plan 是过渡态。用户最终不想为 AI 的"思考过程"付费，只想为"结果"付费。谁能先解决可靠性问题（首次尝试准确率 95%+），谁就能率先完成这个定价跃迁。

经验 5：70% 的用户用 2-4 个工具——不要试图独占

厂商之间的竞争不是争夺忠诚度，而是争夺工具栈中的份额。$20/月的标准线之所以成为行业共识，就是因为它是用户愿意为"一个工具"支付的舒适价格带。

经验 6：先发者承担教育成本，后发者收割成熟用户

Cursor 承担了定价教育的成本（包括那 $350 的退款），后来者直接享受已经接受 Token 计费的用户。先动者吃亏，但也在定义规则。后动者安全，但失去了定价锚点的定义权。
