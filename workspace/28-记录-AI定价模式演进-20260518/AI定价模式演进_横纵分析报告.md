# AI 编程工具定价范式转移：从 Coding Plan 到 Token Plan

> 研究时间：2026-05-18 | 所属领域：AI 商业/定价策略 | 研究对象：AI 编程/Agent 工具定价模式演进

---

## 一、一句话定义

从 2025 年中到 2026 年中，AI 编程工具行业经历了一场几乎同步的定价范式转移：从按固定功能/请求数计费的「Coding Plan」（用户支付固定月费，不限或按次数使用），转向按 Token 消耗量计费的「Token Plan」（用户的月费与底层模型 API 的实际调用量挂钩）。这场转变的本质不是涨价，而是**成本结构的透明化**——厂商不再用 VC 的钱为重度用户兜底。

---

## 二、纵向分析：从诞生到当下

### VC 补贴期：不限使用的幻觉（2022 末 ~ 2024 中）

一切要从一个现在看来近乎天真的假设说起：AI 编程工具的成本可以被扁平月费覆盖。

2023 年初，GitHub Copilot 以 $10/月的价格横空出世，不限请求次数。背后是 Microsoft 的 Azure 基建兜底和 OpenAI 的早期补贴。同一时期，Cursor 定价 $20/月，500 次 Fast Premium Requests，用完转 Slow（不限次数但降速），Codeium（后来的 Windsurf）干脆全功能免费。

这个阶段的定价逻辑很简单——**先证明 PMF，再谈赚钱**。VC 出钱补贴 token 成本，厂商用不限使用的承诺快速吸用户。

但这里有一个致命的结构性问题。一个简单补全和一个多文件重构，对平台来说成本差 10 到 100 倍，但用户付一样的钱。这意味着两个截然不同的经济画像被塞进了同一个 $20 的盒子里：

- 安静的用户一天零星补全，实际成本可能不到 $0.10/天
- 重度用户用 Agent 做小时级编辑，实际成本可能是 $10-50/天

这种差异在早期用户量不大、Agent 模式还没普及的时候可以靠 VC 的钱盖住。但当 Agent 模式成为默认工作方式时，这个盒子就炸了。

### 裂缝出现：请求数上限（2024 末 ~ 2025 初）

Agent 模式的出现是第一个裂缝。

传统的代码补全是一个单步交互：用户写一行，AI 补下一行，token 消耗在几百到几千之间。Agent 模式完全不同：一个多步自主编码会话中，AI 要先读取上下文文件、搜索代码库、理解依赖关系、执行多轮编辑、验证结果——这个过程中 70% 的 token 不是用来生成代码的，而是用来维持 Agent 循环的。上下文随对话轮次线性增长：第 1 轮 5K token，到第 50 轮可能膨胀到 200K token。

厂商开始设上限了。

2025 年 1 月，Cursor 在定价条款中明确写入了 500 次 Premium Requests 上限。这个上限之前一直存在，但是隐性的——写进条款意味着他们已经开始认真监控用量，准备收紧。

2025 年 4 月 21 日，Windsurf（Codeium）推出基于 Credit 的计量模型，迈出了从扁平订阅转向用量计费的第一步。

2025 年 6 月，GitHub Copilot 引入了 Premium Request 上限：Pro 用户 300 次/月，Enterprise 1000 次/月，超出按 $0.04/次计费。这是 Copilot 第一次对"不限使用"动刀。

### 引爆点：Cursor 的第一枪（2025 年 6 月）

Cursor 是整个行业第一个从固定请求数切换到 Token 计费的主流 AI 编程 IDE。而且这一枪打得不够准，打在了自己脚上。

2025 年 6 月 16 日，Cursor 发布定价公告，Pro 计划从"500 次 Fast Requests"改为"$20 的 API 信用额度"。同时推出了 Ultra 计划（$200/月，20 倍用量），给重度用户留了出口。

问题是，绝大多数用户不知道自己一天会消耗多少 API 信用。

一个开发者报告单周超支 $350。有用户发现用 Claude 模型时，几次 Prompt 就把 $20 额度烧光了。Reddit 和 X 上爆发了大规模抱怨，TechCrunch 等主流媒体跟进报道。

两周后，Cursor CEO Michael Truell 发了道歉博客，承认"沟通不清"，并为 6 月 16 日到 7 月 4 日之间的意外收费提供全额退款。

Cursor 官方的切换理由是：

> 新模型在长 horizon 任务中每个请求消耗更多 token。最难的请求比简单请求贵一个数量级。基于 API 的定价是反映这一点的最佳方式。

但真正的原因是，Cursor 当时的 ARR 已经超过 $5 亿（后来在 2025 年中突破 $5B，大约每两个月翻倍），VC 补贴再也兜不住了。Simon Willison 对此的判断是："VC 补贴 token 的时代可能正在结束，尤其是像 Cursor 这样已经过了 PMF 验证阶段的产品。"

这次定价危机给行业留下了一个血淋淋的教训：**成本透明化的过程本身就是一个信任风险**。用户不怕贵，怕的是不知道自己会花多少钱。

### 多米诺效应：全行业跟随（2025 下半年 ~ 2026）

Cursor 引爆后，全行业用了不到一年完成了几乎同步的转向。

**Anthropic Claude Code**从发布第一天就是纯 Token 计费（API 模式），用订阅层做封顶。Pro $20/月约 45 条消息/5 小时窗口，Max 5x $100/月，Max 20x $200/月。2025 年底官方称平均 $6/开发者/天，90% 用户低于 $12/天。但 2026 年 4 月，Anthropic 悄悄将估算上调至 $13/天，90% 用户低于 $30/天——被 Ed Zitron 在 X 上曝光，引发透明度争议。一个开发者 8 个月消耗 100 亿 token，API 价格折合 $15,000+，但 Max 计划只需 $800，实际节省了 93%。

**GitHub Copilot**在 2026 年 4 月宣布，6 月 1 日生效，全面切换到 GitHub AI Credits。所有计划包含与订阅价格等值的 Credits，超出可按 API 价格购买。CPO Mario Rodriguez 的解释最直白：

> Copilot 已经不是去年的产品了……Agentic 使用正在成为默认。今天，一个快速聊天问题和一个多小时的自主编码会话对用户来说花一样的钱。GitHub 已经吸收了大部分不断增长的推理成本，但当前的 Premium Request 模型不再可持续。

**Windsurf**是这个过程里最折腾的。2025 年 4 月第一次改为 Credit 计量，2025 年 12 月被 Cognition 收购（据报道 $3B），2026 年 3 月第二次大改从 Credit 换成 Quota 配额，Pro 从 $15 涨到 $20。Reddit 上用户大规模抗议，有人花了 $2500 后成本暴涨，大量用户流失到 Cursor。

**OpenAI Codex**在 2026 年 4 月 2 日从 per-message 定价切换到 Token-based 定价，使用 Credits 作为结算单位，消耗率改为输入 token × 费率 + 缓存 token × 缓存费率 + 输出 token × 费率。

**Devin（Cognition）**走了另一条路。它用 ACU（Agent Compute Units）作为计费单位，1 ACU 约等于 15 分钟活跃工作。$20/月入口价，$2.25/ACU。本质仍是用量计费，但用 ACU 这个抽象单位替代了裸露的 token 数字，降低了用户的心理阻力。

### 切换的根因：不只是钱的问题

表面上看，这场转变是厂商受不了 VC 补贴、要转嫁成本。但根因有三层：

**第一层是 Agent 工作负载彻底改变了成本结构。** Agent 模式把单次交互的 token 消耗从几百拉到几十万。模型成本本身在下降——Claude Opus 4.0/4.1 定价 $15/$75 per MTok，4.6 降至 $5/$25（降了 67%）——但总 token 消耗的增长完全抵消了降价。

**第二层是厂商无法继续为"同价不同量"买单。** Greyhound Research 分析师说得很清楚：一个安静的用户一天零星补全几乎不花钱，一个重度用户在前沿模型上做小时级编辑成本可能高出一个数量级。当这种差异从 10 倍拉到 100 倍时，任何扁平定价都会崩溃。

**第三层是整个行业的盈利时间表在推后。** OpenAI 2025 年收入约 $4B，成本约 $9B，预计 2029 年才能盈利。Anthropic 2025 年收入约 $1B，成本约 $5.6B，预计 2027 年盈利。在这种情况下，继续用低于成本的价格卖 token 是不可持续的。

### 定价模式演进的四个阶段

回看整个历程，AI 编程工具的定价经历了四个阶段：

| 阶段 | 时间 | 特征 | 代表 |
|------|------|------|------|
| **VC 补贴期** | 2022 末 ~ 2024 中 | 扁平订阅或免费，不限使用 | 早期 Copilot、Cursor、Codeium |
| **请求数上限** | 2024 末 ~ 2025 初 | 固定月费 + 次数上限，超出付费 | Cursor 500 次、Copilot 300 次 |
| **Token 计费** | 2025 中 ~ 2026 | 订阅绑定 API 信用额度，按量计费 | Cursor Pro API Credits、Claude Code |
| **多元定价** | 2026 ~ 未来 | Credits/Quota/ACU/按任务付费并存 | Devin ACU、Cosine Genie 按任务 |

每个阶段都比上一个更贴近真实成本，但也给用户带来了更大的不确定感。

## 三、横向分析：竞争图谱

### 市场全景

AI 编程助手市场 2026 年达到 128 亿美元，预计 2032 年 301 亿美元，年复合增长率 27%。85% 的开发者已经在使用 AI 编程工具，73% 日常使用。但最关键的一个数据是：70% 的工程师同时使用 2-4 个工具，15% 使用 5 个以上。

"一个工具通吃"的假设已经死了。用户不是选一个忠诚度最高的工具，而是在构建自己的 AI 开发工具栈。

### 四大玩家：各活各的

**Cursor — AI-first IDE 领导者**

Cursor 在收入端领先，ARR 突破 $5B，100 万+ 付费用户，拥有最活跃的社区。Tab 补全速度比 Copilot 快约 30%，多文件工作生产力提升 1.42 倍。

但定价危机后 Cursor 的品牌信任度受损。积分制下重度用户的单周超额 $350 是整个行业的标志性事件。尽管 CEO 道歉并退款，这个伤疤还在社区记忆里。

当前定价：Pro $20/月（API Credits）、Ultra $200/月（20 倍用量）、Business $40/人/月。$20 和 $200 两个价位后来成了整个行业的标准线。

**Claude Code — Agent 能力最强，增长最猛**

Claude Code 是这场定价转变中最大的受益者。它从第一天就是 Token 计费，所以没有经历 Cursor 那种"从不限到限量"的用户阵痛。

数据最能说明问题：Agent 首次尝试准确率约 80%（对比 Cursor 65%、Windsurf 55%、Copilot 40%），91% CSAT，54 NPS。工作场景使用率 12 个月从 3% 飙到 18%，翻了 6 倍。JetBrains 2026 年 4 月满意度调查中，Claude Code 以 46% 的"最受喜爱"率排名第一，Cursor 19%，Copilot 仅 9%。

定价方面，与 Claude 消费者订阅绑定：Pro $20/月（5x 基础用量，Sonnet 模型）、Max 5x $100/月（含 Opus 模型）、Max 20x $200/月。一个开发者 8 个月消耗 100 亿 token，API 价格约 $15,000，Max 计划只需 $800，省了 93%。

这个"省了 93%"的叙事是 Anthropic 最聪明的定价传播策略。它没有说"我们便宜"，而是说"如果你用量大，反而更划算"——把 Token 计费从"限制消费"变成了"批量折扣"。

**GitHub Copilot — 企业分发王者，但满意度垫底**

Copilot 有 470 万付费用户，75% 的 YoY 增长，Fortune 100 中 90% 在用。$10/月的价格是主流工具里最低的，免费层也慷慨（2000 次补全 + 50 次聊天/月）。

但满意度只有 9%，垫底。2026 年 4 月 20 日起暂停 Pro/Pro+/学生新注册并收紧周用量限制，2026 年 6 月 1 日全面转向用量计费。IDC 预测 Global 1000 企业会低估 AI 基建成本 30%，这意味着 Copilot 的用量计费转型对企业客户来说会是一个更大的财务冲击。

**Windsurf — 性价比守门员，但被收购后反复折腾**

Windsurf 是独立开发者的首选，$15/月最低入场价（后来涨到 $20），免费层给 25 积分/月。Cascade agent 的整体项目理解力不错。

但被 Cognition 收购后的反复折腾伤害了用户信任。2025 年 4 月 Credit 计量，2026 年 3 月换成 Quota 配额，Pro 从 $15 涨到 $20。Reddit 热帖抗议后，大量用户流失到 Cursor。

### 用户迁移趋势：三大阵营

| 用户类型 | 首选工具 | 原因 |
|---------|---------|------|
| 初创公司 (<50 人) | Claude Code (75% 采用率) | Agent 多步任务处理能力最强 |
| 中型企业 (500-5K) | Cursor (~50%) | IDE 体验最好，Composer 强大 |
| 大型企业 (10K+) | GitHub Copilot (56%) | 微软分发、采购便利、合规 |

主流的使用模式不是选一个，而是堆叠：Cursor 用于日常编辑（主 IDE），Claude Code 用于复杂多步任务（重构、跨文件调试），GitHub Copilot 用于行内补全（响应速度仍然最快）。

### 新的定价模式正在孕育

Token Plan 不是终局。新的定价创新已经在酝酿：

- **BYOK（Bring Your Own Key）**：Cline/Aider/Continue.dev 开源免费 + API 直连，轻度用户月付 $10-20，重度用户可能超 $200。适合重视数据控制的用户。
- **ACU 抽象单位**：Devin 用 Agent Compute Units 计费，1 ACU 约 15 分钟活跃工作。本质是 Token 计费，但包装成了"工作时间"而不是"消耗量"。
- **按任务/结果付费**：Cosine（Genie）提出固定月费按完成的任务数计费，不按 token 计费。这是从"按投入计费"到"按产出计费"的方向性转变。
- **配额刷新制**：Windsurf 2026 年 3 月的 Quota 模式，每日/每周配额自动刷新，降低了用户的用量焦虑。

### 用户视角的真实声音

定价转变中，用户的痛点不是价格本身，而是**不可预测性**。

Cursor 用户说："我完全不知道这个月会花多少钱，每次用都有心理负担。"

Windsurf 用户在 Reddit 上写道："我花了 $2500 买积分，结果发现新配额系统下这些根本不够用。"

Claude Code 用户则说："至少我知道 $200 一个月封顶，超过了我可以升级计划。"

这个差异很关键。Claude Code 的订阅 tiers 虽然也是 Token 计费，但用户知道每层的价格和用量上限，可预测性远高于 Cursor 的纯 API Credit 模式。

## 四、横纵交汇洞察

### 历史如何塑造了当下的竞争位置

Cursor 作为第一个切换到 Token 计费的产品，付出了最大的信任代价，但也获得了最大的先发优势。它的 $20/$200 两个价位成了行业标准，这个定价锚点是后来者都绕不开的参考系。但代价是，Cursor 的品牌信任度在社区里始终带着一道伤疤——那次定价危机没有被忘记，只是被覆盖了。

Claude Code 的幸运在于它没有历史包袱。它从第一天就是 Token 计费，用户进来的时候心理预期已经是"按量计费"。这就像第一家做按需流媒体收费的公司和后来者的区别——先来者承担了教育用户和引发反弹的双重成本，后来者直接享受已经接受规则的用户。

GitHub Copilot 的企业优势不是产品优势，是分发优势。470 万付费用户里有大量不是因为产品最好而选择的，而是因为"微软买了它"。当用量计费转型到来时，这些企业客户会面临最大的财务冲击（IDC 预测低估 30%），但因为采购惯性，他们可能也不会立刻迁移。

### 竞品的纵向路径有何不同

四条路径代表了四种定价哲学：

- **Cursor**："我们发现成本兜不住了，所以改"——被动防御型
- **Claude Code**："从第一天就按真实成本结构定价"——原生透明型
- **Copilot**："我们先圈用户，再慢慢收紧"——圈地收缩型
- **Devin**："我们用一个抽象单位让用户感觉不像在烧 token"——心理缓冲型

这四条路径里，Claude Code 的原生透明型最干净，因为它没有"从不限到限量"的心理落差。Cursor 的被动防御型代价最大，因为它是被迫的，不是主动的。Copilot 的圈地收缩型短期有效但长期伤害品牌——用户会觉得自己被"骗进来再宰"。Devin 的心理缓冲型最聪明，ACU 把"消耗"重新定义为"工作时间"，完全改变了用户的感知框架。

### 优势与劣势的历史根源

Claude Code 今天的最大优势是用户信任（91% CSAT），这源于它从一开始就诚实定价。但它的劣势是重度用户的配额消耗极快（单任务 5 万-20 万 token），这也是 Token 计费的天然矛盾——你越用效果越好，但成本越高。

Cursor 今天的最大优势是 IDE 体验和社区规模，源于它作为独立 IDE 产品的完整性和先发优势。但它的劣势是信任伤疤和不可预测的计费模式，源于那次定价危机的后遗症。

Copilot 今天的最大优势是企业分发和价格最低，源于 Microsoft 的渠道。但劣势是满意度垫底（9%），源于它长期的"先给甜头再收紧"策略。

### 未来推演：三个剧本

**最可能的剧本（概率最高）：多元定价并存，工具栈化成为常态。**

Token Plan 会持续，但不会是唯一模式。Credits、Quota、ACU、按任务付费会共存，因为不同类型的用户需要不同的确定性水平。70% 的开发者使用 2-4 个工具的趋势会加强，厂商会从"争夺单一忠诚度"转向"争夺工具栈中的份额"。企业会正式采用"AI 开发工具栈"采购模式，而不是单一供应商选择。

这个剧本的核心逻辑是：**定价模式的多样性反映了用户工作负载的多样性**。没有一种计费方式能同时满足轻度补全用户和重度 Agent 用户的需求。

**最危险的剧本：成本转嫁引发大规模用户反弹，行业增长放缓。**

如果模型成本不能继续下降，而用量继续暴增，厂商可能被迫进一步转嫁成本给用户。当用户的月费从 $20 变成 $200+ 时（这在纯 API 直连场景下已经发生），大量中小开发者会被挤出市场。这会直接削弱整个生态的开发者基数，反过来影响模型的训练数据质量和社区生态。

触发条件：模型成本停止下降 + Agent 用量持续暴增 + 经济环境收紧企业 IT 预算。

**最乐观的剧本：按结果付费成为主流，定价与价值完全对齐。**

Cosine Genie 的"按任务付费"方向如果被验证，会彻底改变这个行业的定价逻辑。从"按投入（token）计费"到"按产出（完成任务数）计费"，意味着用户不再为 AI 的"思考过程"买单，只为"结果"买单。这对用户是最理想的，因为成本和收益完全对齐。

但这需要 AI 的可靠性和首次尝试准确率大幅提升——目前的 80%（Claude Code）还不够，需要到 95%+ 才能让用户放心地"按结果付费"。触发条件：Agent 首次尝试准确率突破 95% + 任务完成的定义足够清晰。

### 更大的图景

这场定价转变让我想到一个更大的问题：**当 AI 的边际成本不是零的时候，什么样的商业模式是可持续的？**

云计算从包月按需付费的演进花了 15 年才成熟。手机套餐从无限流量到分级套餐经历了无数用户抗议。每一次从"固定"到"按量"的转变都是痛苦的，因为用户天然偏好确定性，而按量计费天然带来不确定性。

AI 编程工具的这次转型，本质上是在回答一个所有 AI 应用最终都要面对的问题：当使用量的方差从 10 倍拉到 100 倍时，谁来承担这个方差？厂商？用户？还是通过某种新的定价机制来稀释？

目前的回答是：用户承担，但厂商用订阅 tiers 来封顶。这个答案不完美，但在模型成本继续下降之前，它是最现实的。

## 五、可复用的商业经验

### 经验 1：成本透明化不是涨价，但沟通方式决定信任存亡

Cursor 的案例证明，从"不限"到"限量"的过程中，用户最大的敌人不是价格本身，而是不可预测性。那些在定价变革中存活下来的厂商，都做到了两件事：一是提供封顶选项（如 Claude Code 的 Max 计划），二是用"批量折扣"的叙事替代"限制消费"的叙事。

### 经验 2：原生定价 > 转型定价

Claude Code 从零开始就是 Token 计费，用户没有心理落差。Cursor 从不限使用转型过来，用户有被"骗进来再宰"的感觉。这个教训对任何新产品都适用：第一天就用可持续的定价模型，不要用 VC 补贴制造虚假的用户预期。

### 经验 3：心理框架比数字更重要

Devin 的 ACU（15 分钟工作时间）是一个绝妙的定价创新。同样的成本，用"token 消耗"的框架用户觉得在烧钱，用"工作时间"的框架用户觉得在雇佣一个员工。**定价的本质不是定数字，是定框架。**

### 经验 4：从投入计费到产出计费是必然方向

Token Plan 是过渡态。用户最终不想为 AI 的"思考过程"付费，只想为"结果"付费。Cosine Genie 的按任务付费和 Devin 的按 ACU 计费都在朝这个方向走。谁能先解决"可靠性"问题（首次尝试准确率 95%+），谁就能率先完成这个定价跃迁。

### 经验 5：70% 的用户用 2-4 个工具——不要试图独占

厂商之间的竞争不是争夺忠诚度，而是争夺工具栈中的份额。这意味着定价策略要考虑用户的"多工具预算"，而不是假设用户只买你一个。$20/月的标准线之所以成为行业共识，就是因为它是用户愿意为"一个工具"支付的舒适价格带。

### 经验 6：先发者承担教育成本，后发者收割成熟用户

Cursor 承担了定价教育的成本（包括那 $350 的退款），后来者直接享受已经接受 Token 计费的用户。这是所有行业定价转型的共同规律：先动者吃亏，但也在定义规则。后动者安全，但失去了定价锚点的定义权。
