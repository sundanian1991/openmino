## 一、原始困惑

年老师的核心焦虑：

> 我有比较深的焦虑，担心自己跟不上时代的发展。自己的效率不够高。还是用 1V1 的模式在和 agent 交流，审核 AI 的结果。在使用 agent 上没有放大自己的效率。受限于自己的注意力。在模式上没有放大自己的兴趣，遇到问题还是对话，而不是开发类 10X 的提高。ai 似乎可以实现一切，在会用的人手里。我也学了一些，但是后续没有持之以恒。用了一段时间的方式就放弃了。在 agent 的调用和使用上不成体系。类似 hook、skill、agent 组合使用下的深度和广度，我认为自己没有彻底改造完成。

年老师的自我评估：60分水平。认为从60到80的跃迁不是"多用几次"能解决的，而是要工程化、优化工作环节。

## 二、项目背景

**项目名称**：任务三 — 供应商管理数字化
**目标**：构建完整的供应商管理数字人体系，向职场提供可咨询、可记录、可提醒的供应商管理 Agent
**时间线**：6月W2-W4（4周），分3个阶段

### 三大交付目标

1. **供应商档案数字化**：通过 Agent 实现供应商档案入库，搭建完整的 5 库知识库体系
2. **标准化管理规则执行**：5维考核评分 / 分量建议 / 异常告警 / 合规处理，自动推送管理举措建议
3. **全生命周期管理**：引入评估 → 日常监控 → 清退复盘，关键节点自动推送与闭环跟踪

### 4周计划

| 周次 | 交付物 | 不可完成（需后续） |
|------|--------|-------------------|
| **W2（6/9-6/13）** | 12家金条供应商档案入库 + 信息收集模板落地 + 标签体系定义 + 老板画像开始收集 | 全量32家完成 |
| **W3（6/16-6/20）** | 5维考核模型定稿 + 评分细则写清楚 + 分量规则框架 + 红黄牌机制文档 | 考核自动化运行 |
| **W4（6/23-6/27）** | /档案查询 原型可跑通 + /日简报 原型可跑通 + 5个知识库框架搭建完毕 | 全量Skill上线、全生命周期闭环 |

**7月（Phase 2）**：接入规则引擎，考核+分量+诊断自动化
**8月（Phase 3）**：引入/清退流程上线，数字人全模式运行

## 四、核心诊断

### 瓶颈不在框架，在数据

**框架已完成的部分：**
- ✅ 27个Skill设计（10个P1）
- ✅ 5个知识库结构规划
- ✅ 5维考核模型定义（30/25/15/15/15）
- ✅ 供应商信息收集模板
- ✅ 供应商自助填报系统
- ✅ 方案汇报材料

**数据缺失的部分：**
- ❌ 35家供应商的业绩数据 — 从未录入
- ❌ 35家供应商的合同信息 — 从未录入
- ❌ 35家供应商的配合数据 — 从未录入
- ❌ 33家供应商的老板画像 — 只有2家有深度记录
- ❌ 17家画像的标签全部未勾选
- ❌ 联系人信息几乎全空

**结论**：即使把Skill全部写完、知识库搭好，Agent去查某家供应商的业绩趋势——库是空的。

### 年老师的"60分"问题

实际上年老师的问题不是"不会用agent"，而是：
1. **有基础设施但没有跑通闭环** — skill装了、模板做了，但没有形成日常使用的工作流
2. **有框架但数据没灌入** — 方案写了585行，但供应商数据还在脑子里
3. **有想法但没有持续执行** — 学了一些方法，用了一段时间就放弃

## 八、待讨论的问题

以下是需要其他 AI 给建议的方向：

### 1. 执行策略
- 42家供应商的数据量不小，批量导入 getnote 知识库的最优方式是什么？
- 12家金条供应商优先录入，应该按什么顺序？（建议：先6家重点→再6家非重点）
- 老板画像只有2家有深度数据，其余33家怎么高效收集？

### 2. agent 使用模式
- 年老师目前是1V1对话模式，如何过渡到"开发类10X"模式？
- hook + skill + agent 组合使用的最佳实践是什么？
- 如何避免"学了一些但没持之以恒"的问题？

### 3. 项目管理
- 4周计划的拆解粒度是否合适？是否需要按天拆？
- 哪些环节适合用 cron 定时任务自动执行？
- 进度追踪应该用 task 系统还是其他方式？

### 4. 数据工程
- 从 Excel 到 getnote 知识库的数据流转最佳路径？
- 42家×67字段的数据质量如何保证？
- 业绩数据需要从数仓取，如何与知识库打通？

### 5. Skill 开发
- /档案查询 和 /日简报 的原型应该做到什么程度算"可跑通"？
- Skill 开发是否应该先做最小可用版本再迭代？
- 如何验证 Skill 的输出质量？
