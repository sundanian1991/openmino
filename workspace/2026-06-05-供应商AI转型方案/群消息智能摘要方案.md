```
京ME群（35+个）
    │
    ▼ [joyclaw jmechat 插件]
search_group + message_sumary

    ▼ [joyclaw Agent / Cron]

    ▼ [AI 聚合分析]

    ▼ [输出]
1. 京ME群内推送（数字人消息）
2. getnote 知识库归档
3. 年老师 Claude Code 可查
```

## 四、Phase 1 详细步骤

### D1：数据采集验证

在 joyclaw 里执行：

```
1. search_group("岐力") → 获取群ID
2. message_sumary(群ID, 最近2天) → 获取消息摘要
3. 同理获取翰锐、伽玛的群消息
4. 把三份原始结果复制给我
```

我需要确认：
- `message_sumary` 返回的粒度（逐条？按小时？按天？）
- 摘要中是否包含问题/情绪/关键人等有效信号
- 数据格式（JSON/文本？字段有哪些？）

### D2：聚合分析验证

我拿到样本后：
1. 设计问题识别规则（关键词 + 语义判断）
2. 设计分类体系（按业务线/问题类型/紧急程度）
3. 设计跨群关联逻辑（同一问题出现在多个群 → 合并标注）
4. 输出一份样本简报，你确认信息是否有价值

### D3：自动化配置

在 joyclaw 配置 Cron 任务：

```yaml
时间: 每天 08:30
步骤:
  1. 遍历 3 个目标群，调用 message_sumary
  2. 将摘要交给 Agent 做聚合分析
  3. 输出到指定位置（getnote / 本地文件）
  4. 可选：推送到京ME群
```
