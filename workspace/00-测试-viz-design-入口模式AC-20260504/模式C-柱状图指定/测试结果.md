### Phase 3：spec.md

```markdown
# Compiled SPEC — 供应商 Q1-Q2 业绩对比

## Page: Q2 业绩整体回暖

### 叙事意图

**【可视化目的】**
- 认知缺口：哪些供应商 Q2 增长了，哪些下滑了？

**【想传达什么】**

- **高亮点**（≤10%）：赛维斯（增幅最大）
- **标注内容**："增幅 22%，最大增长"
- **基准线/参考线**：无

## 测试评估

### 1. 是否尊重了用户的图表指定
**通过**。年老师指定"柱状图"，执行使用了分组柱状图（模式 13），没有替换。

### 2. 简单数据是否走了轻量模式
**通过**。Phase 1+2 合并为口头确认，没有写 intent.md 和 storyboard.md。数据简单（5 行 × 3 列），符合轻量模式条件。

### 3. spec.md 是否仍然包含渲染契约 JSON
**通过**。spec.md 末尾附带了完整的 v2 格式渲染契约 JSON。

### 4. 模式 C 是否没有过度分析
**通过**。没有提出额外的分析角度，没有建议其他图表，直接执行用户的指定。

### 5. 数据格式合理性

**发现问题**：
1. **⚠️ `params.color: null`**：v2 JSON Schema 要求 params 中的 color 是标量，null 虽然是标量，但更好的做法是 omit 这个字段或设为具体色值。这取决于渲染管线的容错能力。

2. **scales 中 x 轴 type 问题**：`"aesthetic": "x"` 的 type 设为 `"linear"` 但 x 轴是分类变量（供应商名），应该使用 `"type": "ordinal"` 或 `"type": "category"`。当前 scales 类型定义中没有 ordinal 选项，这是一个 schema 空白。

3. **数据组织方式**：将宽表（Q1/Q2 两列）转换为长表（季度列）是正确的做法，因为 ECharts 分组柱状图需要这种格式。
