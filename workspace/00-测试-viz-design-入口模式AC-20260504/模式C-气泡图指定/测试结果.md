### Phase 3：spec.md

```markdown
# Compiled SPEC — 供应商规模与效率气泡图

## Page: 规模不决定效率

- **场景论文**：从 storyboard 继承（轻量模式跳过）
- **签名视觉元素**：Custom
- **为什么不能简化为默认模板**：需要传达"规模与效率非线性关系"的洞察

### 叙事意图

**【可视化目的】**
- 认知缺口：团队规模越大是否人均产出越高？

**【想传达什么】**
- 核心信息：毛毛虫人均产出最高但规模只有毅航的 62.5%

**【结论】**
- 读者应得出的判断：中等规模供应商效率最优，规模不决定效率

**【思路】**
- 视觉叙事路径：第一眼看到气泡位置分布 → 第二眼注意到毛毛虫在右上象限 → 理解规模与效率的非线性关系

### 视觉编码

- **X 轴编码**：团队规模（150-800人）
- **Y 轴编码**：人均产出（25-52万）
- **颜色编码**：客户满意度（Warm 色阶，62→92）
- **大小编码**：利润率（5%→22%，气泡面积）

### 数据组织

- **字段清单**：供应商、团队规模、人均产出(万)、客户满意度、利润率
- **排序规则**：无（散点图不需要排序）
- **聚合规则**：无

### 标注策略

- **高亮点**（≤10%）：毛毛虫（效率最高，右上象限）
- **标注内容**："人均产出最高，规模非最大"
- **基准线/参考线**：均值线（规模均值400，人均产出均值38）

### 布局

- **画布**：800 × 550
- **配色**：restrained-warm

## 测试评估

### 1. 是否尊重了用户的图表指定
**通过**。年老师指定"气泡图"，执行使用了气泡图（模式 05），没有替换为其他图表。

### 2. 简单数据是否走了轻量模式
**通过**。Phase 1+2 合并为口头确认，没有写 intent.md 和 storyboard.md，直接进入 Phase 3。

### 3. spec.md 是否仍然包含渲染契约 JSON
**通过**。spec.md 末尾附带了完整的 v2 格式渲染契约 JSON。

### 4. 模式 C 是否没有过度分析
**通过**。没有展开多维度分析建议，没有列出替代方案，直接执行用户的指定。

### 发现的问题

1. **⚠️ 高亮比例超标**：5 家供应商中高亮了 1 家（毛毛虫），占比 20%，超过 SKILL.md 规定的 maxHighlightRatio ≤ 0.1（10%）。**修正**：对于 5 个数据点，10% 意味着最多 0.5 个，实际上至少需要高亮 1 个才能有效。这是规则在小样本场景下的边界问题，应允许 1 个高亮（最小单位）。

2. **JSON scales 中 color values 问题**：scales 中 `"aesthetic": "color"` 配了 5 个离散色值，但 mapping 中 color 映射到"客户满意度"是连续变量。应该使用 `"type": "linear"` 色阶而非 `"type": "manual"` 离散色值。
