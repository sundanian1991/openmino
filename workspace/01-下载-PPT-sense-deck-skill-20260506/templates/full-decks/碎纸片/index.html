AI GAME PIPELINE

GPT Image 2 + Codex

把视觉资产工坊和编码 agent 连成一条生产线，从概念图推进到浏览器里可玩的网页游戏。

开场强调这不是只生成几张图，而是把创意、资产、代码、验证连成一条生产线。

WORKFLOW

流程不是魔法，是四个可控工位

每一站都有输入、产物和验收标准，AI 负责提速，人负责定方向。

01定义游戏体验

玩家目标、核心循环、胜负条件、平台约束。

02生成视觉资产

用 GPT Image 2 形成角色、场景、UI 和状态变体。

03Codex 搭建迭代

生成项目骨架，实现规则，接入素材，运行调试。

04浏览器验证交付

检查交互、移动端、性能、截图和源码包。

把流程拆成工位，降低听众对 AI 工作流不可控的担忧。

ROLE SPLIT

GPT Image 2 负责视觉可能性，Codex 负责工程闭环

GPT Image 2

把风格变成资产

  • 角色、场景、道具、UI 风格探索
  • 透明背景素材和状态变体
  • 快速统一画风、光源、色板和氛围
Codex

把资产变成玩法

  • 搭建 HTML/CSS/JS 或框架项目
  • 实现规则、碰撞、状态机、音效触发
  • 运行测试、修 bug、优化移动端
避免把模型能力混在一起讲：一个偏视觉生成，一个偏代码执行和工程反馈。

ASSET BOARD

视觉资产先做成一张资产板，再进入代码仓库

让每个素材都能被命名、裁切、复用和替换，Codex 才能稳定接入。

主角精灵idle / run / hit
敌人与障碍variant / warning
背景层parallax / depth
按钮与 HUDscore / lives / timer
胜负反馈win / lose / combo
图标与粒子collect / burst
建议把资产命名、尺寸、透明背景、状态枚举在进入 Codex 前整理清楚。

CODEX LOOP

Codex 的价值在于边写边跑，而不是只吐代码

读取需求
生成骨架
接入资产
实现玩法
运行检查
反馈迭代
const game = createGame({
  assets: manifest,
  loop: "jump-dodge-score",
  target: ["desktop", "mobile"]
});

await codex.run("npm test");
await codex.inspect("http://localhost:3000");
重点讲 Codex 能在本地工作区里改文件、起服务、看错误、再修复。

ARCHITECTURE

一个轻量网页版游戏可以拆成五层

01输入层

键盘、触摸、手柄、暂停。

02游戏状态

分数、生命、关卡、计时器。

03规则系统

移动、碰撞、生成、胜负判断。

04渲染层

Canvas、DOM、Pixi 或 Phaser。

05资产与音效

精灵图、背景、UI、反馈音。

技术页只讲边界，不陷入某个框架细节。Canvas、DOM、Pixi、Phaser 都能套这个结构。

QUALITY GATES

交付前至少过三道质量门

01

玩法闭环

30 秒内能理解并完成一局，失败后愿意再来一次。

02

视觉一致

尺寸、描边、光源、色板、UI 密度统一。

03

浏览器稳定

桌面/移动端无重叠、无空白、无控制台错误。

强调网页版游戏最容易翻车的是视觉拼贴不统一和移动端布局。

HANDOFF

最终交付的是一套可继续迭代的游戏包

可运行源码

能本地启动，也能部署到静态站点。

资产目录

命名规范、尺寸约定、来源说明。

玩法说明

目标、操作、计分、胜负条件。

测试清单

浏览器、移动端、键盘、触摸、性能。

迭代 backlog

新关卡、新机制、留存点和商业化实验。
