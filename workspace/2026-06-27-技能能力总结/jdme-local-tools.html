概述

核心突破：突破"只能读摘要"限制，通过 SQLCipher 解密直接访问京ME本地数据库，获取完整原始聊天消息——包括消息体 JSON、会话结构、时间戳等全部字段。

本技能提供对京ME（JD ME）本地数据的只读检查能力。通过统一入口脚本 jd_me_tool.py 调用共享核心逻辑，平台相关路径发现封装在独立模块中。支持路径发现、日志文本恢复、SQLCipher 加密数据库读取、以及实时消息事件监听。

🔍 路径发现

自动发现京ME应用根目录、支持根目录、Electron根目录、日志根目录、SQLCipher可执行文件位置、数据账户和推荐账户。

🔐 数据库解密

通过 SQLCipher CLI 或 Windows DLL fallback（ctypes）读取加密的 chat.data / session.data。密钥为 md5(account_pin)。

📋 日志恢复

从 sdk-event.log 和 Windows renderer.log 中恢复消息快照，作为数据库不可用时的后备方案。

👁️ 实时监听

通过 watch 命令 tail 日志文件，将消息推送、会话更新解析为稳定 JSON 输出。

推荐工作流

1. discover → 2. sessions → 3. chat-history
  1. 先运行 discover——快速、跨平台，除非用户已提供精确路径。
  2. 需要完整历史时优先用数据库命令：sessions --auto --name-contains <群名> 解析会话ID，然后 chat-history --auto --session <id> --limit <N>。
  3. SQLCipher 不可用或只需近期记录时，用 messages --auto --contact <erp> --limit <N> 从日志恢复。
  4. Windows 上 messages 默认同时扫描 renderer.log 和 sdk-event.log。
  5. 默认使用 --auto：先检查 ~/.jd_me_account，再扫描数据/日志根目录，选最活跃的本地账户。

python3 scripts/jd_me_tool.py messages --auto --session erp1:ee:erp2:ee --limit 100

聊天记录读取

messages —— 日志恢复（推荐首选）

从以下来源恢复消息文本：

  • sdk-event.log 及其轮转文件 sdk-event.N.log
  • Windows 渲染快照：renderer.log、renderer.N.log、renderer.old.log
  • 事件类型包括：insertMessages、openChatResult、loadMoreResult、updateMsgParam、identifyJoyspaceDoc、chatNotify

过滤参数

参数说明
--contact <erp-or-name>匹配 sessionId / senderPin / receiverPin / 内容
--session <session-id>精确会话过滤，可重复
--keyword <text>内容子串匹配，可重复
--at-me@我的消息过滤（需配合 --self-pin）
--limit <N>保留最近 N 条
--no-rendererWindows：不扫描 renderer 日志

chat-history —— 数据库完整历史

通过 SQLCipher 解密读取完整消息。输出 JSON Lines，每行包含：sessionId、timestamp、sender、msgType、原始 msgJson、以及解析后的 text。

⚠ 数据库解密失败处理

若返回 file is not a database，不要重试进程内存转储或凭据提取。报告失败数据库路径、密钥派生账户、发现的 SQLCipher CLI/DLL 候选，回退到 messages 日志恢复。

其他命令详解

watch —— 实时监听

tail renderer.log 和 sdk-event.log，将消息推送、会话更新、选中会话详情解析为稳定 JSON。支持 --keyword、--at-me、--self-pin、--session。

summary —— 日报/周报摘要

SQLCipher 可用时 join chat_message + session_info，输出增强 JSON Lines 用于日报/周报合成。默认包含 Relay|Zero|基础服务，排除告警/机器人/通知类群。

query —— 文本搜索

搜索可读导出文本文件和账户日志，返回片段、直接会话ID、加密数据库边界。自动脱敏凭据类值。

preview —— 诊断回退

先尝试运行中/原生京ME桥接，失败后回退到日志预览。将 sdk_context_unavailable 或 native bridge not found 视为桥接诊断，非凭据提取理由。

架构说明

文件职责
scripts/jd_me_tool.py稳定命令入口，SKILL.md 示例统一使用此路径
scripts/jd_me_tool_core.py共享数据访问、解析、CLI 行为（平台无关）

所有命令输出 JSON Lines（除非另有说明）。

安全边界

本工具不得修改京ME应用包/安装目录、注入 Electron、从运行进程内存转储密钥、写入京ME聊天数据库、或尝试凭据提取。它是操作在用户已有文件上的本地只读助手。

preview 在桥接可用时可能在应用 data/ee/export/ 下创建 SDK 导出 HTML。所有 chat-* 数据库查询均为只读，需要用户对其账户数据的授权访问。
