## MB 模块化网格系统

**核心理念：** 不只是列对齐，还有行对齐。页面被分成"模块场"，内容占据整个模块或模块组合。

**参数：**
```css
--cols: 12;        /* 列数 */
--rows: 8;         /* 行数（默认） */
--bl: 8px;         /* 基线单位 */
--lh: 24px;        /* 行高 = 3 × 基线 */
--gutter: 24px;    /* 列间距 */
--margin: 48px;    /* 页边距 */
--maxw: 1200px;    /* 最大宽度 */
--module-h: 96px;  /* 模块高度 */
```

**每个章节的网格配置：**

| 章节 | 网格配置 | 说明 |
|------|----------|------|
| Hero | 4×2 | 标题 + 核心指标 |
| 第一章 | 4×3 + 6×4 | KPI（4列3行）+ 图表（6列4行） |
| 第二章 | 6×4 | 图表 + 文字混排 |
| 第三章 | 8×6 | 复杂表格 + 图表 |
| 第四章 | 6×4 | 图表 + 文字混排 |
| 第五章 | 4×3 + 6×4 | 异常卡片（4列3行）+ 图表（6列4行） |
| 第六章 | 4×3 + 8×6 | 行动卡片（4列3行）+ 表格（8列6行） |

## 模块化布局示例

**4×3 模块化网格（KPI 仪表盘）：**
```
┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐
│ A1  │ │ B1  │ │ C1  │ │ D1  │
└─────┘ └─────┘ └─────┘ └─────┘
┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐
│ A2  │ │ B2  │ │ C2  │ │ D2  │
└─────┘ └─────┘ └─────┘ └─────┘
┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐
│ A3  │ │ B3  │ │ C3  │ │ D3  │
└─────┘ └─────┘ └─────┘ └─────┘
```

**6×4 模块化网格（图表 + 文字）：**
```
┌─────────────────────────────────────┐
│              标题（跨6列）            │
└─────────────────────────────────────┘
┌─────────────────────────────────────┐
│              图表（跨6列，跨3行）     │
│                                     │
│                                     │
└─────────────────────────────────────┘
┌─────────────────────────────────────┐
│              文字说明（跨6列）        │
└─────────────────────────────────────┘
```

## 对比查看

**v1（12 列 + 基线）：** 只有列对齐，没有行对齐
**v2（模块化网格）：** 行和列都对齐，内容占据整个模块

建议并排打开 `sabc-mb-grid.html`（v1）和 `sabc-mb-grid-v2.html`（v2），按 `G` 键显示网格，观察差异。
