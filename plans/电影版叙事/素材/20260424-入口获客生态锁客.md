# 入口获客，生态锁客

用高频刚需场景获客，用多场景生态提高 ARPU 和粘性。MiniMax 从 Coding Plan 改名 Token Plan 就是这个逻辑。

## Context

MiniMax 在 2026-03 将 "Coding Plan" 正式更名为 "Token Plan"，因为它的套餐不只卖编程能力，而是打包了图片、语音、音乐、视频等全模态 AI 能力。¥29 买到的不只是代码补全，是"AI 多媒体工作室"。

这个转变的商业逻辑：
- **编程**是高频刚需入口 → 获客
- **全模态**是低频高价值场景 → 提高 ARPU
- **一个套餐打包** → 生态锁定，用户不会为每个场景单独付费

## Implication

产品定价的升级路径：
1. 找到一个高频刚需场景作为入口（编程、聊天、搜索）
2. 用低价/免费策略快速获客
3. 逐步叠加更多场景到同一个套餐/平台
4. 从"卖工具"升级为"卖生态"
5. 用户一旦在你的生态里用了多个场景，切换成本急剧上升

注意：场景捆绑要自然，强行捆绑会引起反感。MiniMax 的全模态打包之所以成立，是因为同一个 Token 可以用于所有模态。
