Part 03

为什么比不了

五种计量体系并存是最大的信息壁垒。直接比数字毫无意义，90% 的开发者在这一步踩坑。你以为阿里百炼比 MiniMax 多 15 倍，其实计量单位完全不同。

#1

API 请求次数

阿里百炼 · 火山方舟 · 腾讯云 · 百度千帆

1 次用户提问 = 5-30 次 API 调用。数字看起来很大，实际消耗不透明。有利于厂商的模糊计量。

透明度：低
#2

Prompt 次数

智谱 GLM · MiniMax（早期）

1 Prompt ≈ 15-20 次模型调用 ≈ 1200-1600 API 请求。比 API 次数直观，但仍然是打包后的数字。

透明度：中
#3

Token 计量

Kimi（2026.1 切换后）· MiniMax Token Plan

直接按输入输出 Token 计费，仅统计未命中缓存的 Token。最透明，但用户难以预估自己的消耗量。

透明度：高
#4

Credit 计量

Cursor · 小米 MiMo · 阿里百炼 Token Plan

统一 Credit 池，不同模型不同倍率。抽象了一层但可预测——你知道选什么模型贵，但不知道具体贵多少。

透明度：中
#5

时间窗口配额

智谱 GLM（5h 窗口）· Claude Code（5h+7d）

每 5 小时限额 + 每月上限。限制节奏而非总量——不是你不能多用，是不能用太快。

透明度：中
关键发现：同样 ¥40-50/月，阿里百炼给 18000 次 API 请求，MiniMax 给 40 prompts/5h（约 1200 API 请求/5h）。看起来百炼多 15 倍，但实际消耗完全不同。Agent 场景下一个开发者 8 个月消耗 100 亿 Token（API 价 $15,000+），Max plan 只花 $800（省 93%）。选对模型是成本优化的最大杠杆。
Part 04

本质是什么

Coding Plan 到 Token Plan 的转变，不是换个名字。是三层逻辑同时驱动的商业进化。

LAYER 01 — 成本驱动

算力成本决定了定价下限

算力成本→ Token 成本→ 套餐定价

国产芯片放量（华为 950 年内 75 万片）、MoE 架构效率革命（1.6T 参数只激活 49B）、推理专用芯片竞争——算力成本持续下降，价格战才有底气。没有成本优势的低价是烧钱，有成本优势的低价是竞争力。

LAYER 02 — 竞争驱动

七家在一个池子抢用户，不降价就被淘汰

价格战→ 补贴圈用户→ 收紧利润

开发者选择多（7+ 平台），切换成本低（换 API Key 就行），买方议价力极强。但价格战不是终态——所有互联网赛道都走"烧钱→收紧→分层"三步，只是时间问题。收紧的节奏比补贴的力度更关键。

LAYER 03 — 价值驱动

从"卖 Token"到"卖生态"的升级

卖 Token→ 卖工作流→ 卖生态

MiniMax 把 Coding Plan 改名 Token Plan，因为它打包了图片、语音、音乐、视频。¥29 买到的不只是代码补全，是"AI 多媒体工作室"。用高频刚需场景（编程）获客，用低频高价值场景（视频/音乐）提高 ARPU。用户一旦在你的生态里用了多个场景，切换成本急剧上升。

类比：电信套餐 20 年演进史

电信阶段AI 对应核心逻辑
2009 话费套餐2024 API 按量供给过剩，用户感知不到成本
2013 流量套餐2025 Coding Plan用量激增，包月降低感知
2017 "不限量"达量降速2026 分层套餐精细化分层，按速度定价
2020 按 GB 精打细算2026+ Token 精细计量用户算账，厂商利润回归
Part 05

所有赛道都走这三步

AI 定价只是最新案例。电信套餐、网约车、外卖——全走这个节奏。

01

补贴期

2024 – 2025

DeepSeek 带头降价，百万 Token 从几十元降到几毛钱。首月 7.9 元入局。目的是抢用户规模，建立使用习惯。不要在第一天就想好终态定价，会过度保守。

02

收紧期

2025 H2

首购优惠取消、低档套餐停售、限时抢购取代敞开供应。补贴撑不住了，开始测试价格弹性。收紧的节奏比补贴的力度更关键——太快用户跑，太慢利润撑不住。

03

分层期

2026 – 至今

MiniMax 推极速版 ¥98-899，Kimi 上线 ¥559 Allegro。精细分层，匹配不同支付意愿，利润最大化。入口获客，生态锁客。先跑量再算账。

计量即权力，补贴即投资，收紧即筛选，分层即利润。
CASE STUDY

OpenClaw：一个框架如何改变行业

OpenClaw 是一个开源 AI Agent 框架，2026 年 2 月爆火，迅速积累 355,000 GitHub stars。它做的事情很简单：让 Agent 自动循环调用模型。但就是这个简单行为，直接把 Token 消耗量推高了 6-10 倍。以前写代码是手动调 API，现在 Agent 自动跑——一个任务就是几十次模型调用。

这不是用户用得更多了，而是使用方式彻底变了。70% 的 coding agent token 是浪费（DEV Community 研究），但用户愿意为这种浪费付费，因为套餐给了"成本安全感"。于是厂商的套餐收入涨了，但算力成本涨得更快。

Part 06

芯片供给即定价天花板

谁控制算力供给，谁就控制定价下限。这个被低估的变量，决定了 Token 价格的下行空间。

英伟达在中国的退缩 ↓

  • 市占率：66%(2024) → 54%(2025)
  • 2025-04 全面禁止所有 AI GPU 对华出口
  • 后续松绑 H20/H200，但严苛采购限制，订单落地严重滞后

华为昇腾的崛起 ↑

  • 2025 年交付 81.2 万片，市占率 ~20%
  • 950PR：算力是 H20 的 2.87 倍，定价仅 1/3
  • 2026 目标：出货 75-100 万片，营收超 800 亿
  • 路线图：950PR → 950DT → 960 → 970
含义：算力供给不再被英伟达卡脖子 → 国内厂商成本结构和海外产生分化 → 定价策略可以更激进。DeepSeek V4 官方明确说"下半年昇腾 950 批量上市后价格大幅下调"——芯片供给直接绑定定价天花板。观察一个 AI 产品的定价走势，不要只看市场竞争，要看它的芯片供应链。算力供给是定价的硬约束，市场竞争只是软约束。
Part 07

七条可复用洞察

这个案例最大的价值不是知道发生了什么，而是提炼出可以复用于任何产品定价的模式。

INSIGHT 01

计量单位即定价权

谁定义计量单位，谁就掌握定价权和用户感知。MiniMax 从 Prompt 切换为 Token、Kimi 同样切换——这个看似技术性的变化，本质是定价权的争夺。模糊计量（API 请求次数）利于早期获客，透明计量（Token）利于长期信任。五种计量体系并存是最大的信息壁垒，90% 用户踩坑。趋势是从 API 次数 → Prompt → Token/Credit → 统一 Credits，方向是越来越透明。

可复用模式：新产品定价时，计量单位选择决定了用户感知和利润空间。选错计量单位 = 选错定价权归属。
INSIGHT 02

补贴-收紧-分层的三段式节奏

所有互联网赛道的定价都走这三步：烧钱抢量 → 测试弹性 → 精细分层。AI 定价只是最新案例。电信套餐（话费→流量→不限量→按 GB）、网约车（补贴→涨价→动态定价）、外卖（满减→会员→精准券）——完全一致的节奏。进入新市场时不要一开始就想好定价，先补贴跑量，再收紧测试弹性，最后精细分层。

可复用模式：收紧的节奏比补贴的力度更关键。太快用户跑，太慢利润撑不住。
INSIGHT 03

入口获客，生态锁客

编程是高频刚需入口 → 获客。全模态是低频高价值场景 → 提高 ARPU。一个套餐打包 → 生态锁定，用户不会为每个场景单独付费。¥29 买到的不只是代码补全，是"AI 多媒体工作室"。用户一旦在你的生态里用了多个场景，切换成本急剧上升。场景捆绑要自然，强行捆绑会引起反感。

可复用模式：用高频刚需场景获客，用低频高价值场景提高 ARPU。从"卖工具"升级为"卖生态"。
INSIGHT 04

开源模型的定价权悖论

DeepSeek V3.2 在 OpenRouter 月调用 5.35 万亿 token，全球第三。但 DeepSeek 没有 Coding Plan/Token Plan，只卖 API。开源模型的免费部署能力持续压制定价上限——当用户可以免费部署同级别模型时，厂商定价权只能来自"服务便利性"而非"模型独占性"。定价权应该建立在"服务生态"（多模态打包、Agent 编排、行业解决方案）上，而非"模型能力"上。

可复用模式：如果你的定价权依赖于"模型独占"，开源模型会持续侵蚀你的护城河。
INSIGHT 05

订阅费是地板不是天花板

一个开发者 8 个月 Claude Code 消耗 100 亿 Token，API 价 $15,000+，Max plan 只花 $800。但反过来看：Cursor 2025-06 转 credit-based 后，重度用户一周超支 $350，引发退款风波。Token 超额、premium 模型加价、agent 循环消耗 → 实际账单是标价的 2-5 倍。订阅费只是入场券，真正的成本在使用中暴露。

可复用模式：定价模式切换必须渐进，提前通知。突然改变计费 = 用户反弹。
INSIGHT 06

国内外定价逻辑的根本差异

国内：七家在一个池子里打价格战 → "你买到的便宜，是因为有人在替你卷"。海外：面对 Claude/GPT/Gemini 三巨头 → 需要证明"我不比 Claude 差"，比谁更强。国内比便宜，海外比能力。小米 MiMo 国内定价 $1/$3 极低价，但海外需要证明能力不输 Claude。同一产品两套价格体系的合理性建立在竞争环境差异上，但价差过大时用户发现后信任崩塌。

可复用模式：定价策略必须匹配竞争环境。不同市场的定价不能只看成本，要看对手。
INSIGHT 07

跨越鸿沟：从实验到生产的信任挑战

Gartner：90% 企业推进 AI 实验，仅 41% 成功落地生产环境。预计 2027 年 40% Agentic AI 项目因无法证明 ROI 被终止。企业在实验阶段愿意尝试低价 Coding Plan → 进入生产环境后需要 SLA、稳定性、合规性 → 这是 Token Plan 企业版溢价的基础。跨越鸿沟的关键不是模型能力，而是"可信赖度"。定价权最终来自信任，不是来自低价。

可复用模式：为跨越鸿沟提前准备——SLA、合规、稳定性这些"可信赖度基础设施"才是进入主流市场的门票。
Part 08

未来预判

不是猜测，是基于当前驱动力和已公开路线图的合理推断。

Token 成本将持续下降

驱动力：国产芯片放量（华为 950 年内 75 万片）→ 算力成本下降；DeepSeek V4 开源 MoE 架构效率革命（1.6T 参数只激活 49B）→ 模型效率提升；推理专用芯片竞争（英伟达 Rubin CPX / 谷歌 TPU v8 / 华为 950PR）→ 推理成本加速下降
→ Token Plan 价格战继续，但利润空间来自"服务溢价"而非"算力差价"

英伟达市占率持续下降（66% → 54% → 可能低于 50%）；华为昇腾 + 寒武纪 + 阿里平头哥形成国产算力三角；软件生态从 CUDA 迁移到 CANN，迁移效率已超 90%；DeepSeek V4 同时适配华为 + 寒武纪，Day 0 支持
→ 算力供给不再被卡脖子 → 国内厂商成本结构和海外产生分化 → 定价策略可以更激进

开源模型重塑定价权

→ 开源模型的免费部署能力持续压制定价上限 → 定价权来自"服务便利性"而非"模型独占性"

Agent 时代的成本指数级增长不可逆

OpenClaw 证明：一个 Agent 任务 = 几十次模型调用；推理需求 2026 年已达训练的 4-5 倍且在加速；IDC 预测 2028 年推理工作负载占比 73%；70% coding agent token 是浪费
→ 所有厂商最终转向 Token 精细计量。谁先转谁主动，谁后转谁被动（Cursor 2025-06 的教训）
