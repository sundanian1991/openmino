Token 正在走一模一样的路。

Section 03

七个可复用的商业模式

从这个案例中能提炼什么跨场景可用的定价模式。

01
计量单位即定价权
谁定义计量单位，谁就掌握定价权。
五种计量体系并存是最大的信息壁垒。模糊计量利于早期获客，透明计量利于长期信任。
02
补贴-收紧-分层的三段式
烧钱抢量 → 测试弹性 → 精细分层。所有互联网赛道都走这三步。
收紧的节奏比补贴的力度更关键——太快用户跑，太慢利润撑不住。
03
入口获客，生态锁客
用高频刚需场景获客，用多场景生态提高 ARPU。
MiniMax 从 Coding Plan 改名 Token Plan：编程是入口，全模态是生态。¥29 买到的是"AI 多媒体工作室"。
04
国内外定价逻辑的根本差异
国内比谁更便宜，海外比谁更强。
7 家在同一池子打价格战。"你买到的便宜，是因为有人在替你卷。"
05
Agent 时代的成本放大器
70% coding agent token 是浪费。订阅费是地板不是天花板。
8 个月 Claude Code 消耗 100 亿 Token，API 价 $15,000+，Max plan 只花 $800（省 93%）。
06
OpenClaw：Token 消耗的催化剂
一个 Agent 任务 = 几十次模型调用。
MiniMax M2 用量增长 6 倍，Coding Plan 消耗增长 10 倍+。阿里云算力告急，Anthropic 移除 OpenClaw——成本撑不住了。
07
芯片供给即定价天花板
观察一个 AI 产品的定价走势，不要只看市场竞争，要看它的芯片供应链。
英伟达中国市占率 66% → 54%。华为昇腾 950PR 算力 2.87x H20，价格仅 1/3。DeepSeek V4 官方直接说"预计下半年昇腾 950 批量上市后价格大幅下调"——算力供给是定价的硬约束，市场竞争只是软约束。
Section 04

实战清单

从洞察到行动。定价决策清单 + 反模式。

  • 定义计量单位：选错 = 选错定价权归属。模糊→早期获客，透明→长期信任
  • 选择获客策略：补贴快但不可持续，差异化慢但健康
  • 设计分层结构：3-4 档，入门 ~$20-50，Pro 档 3-5 倍
  • 确定透明度："每 5h N 次" vs "N 万 Token" vs "5x/20x" 倍数
  • 规划演进路径：补贴→收紧→分层，别在第一天就想好终态
突然切换计费 — Cursor 2025-07 退款风波。必须渐进迁移。
计量单位混乱 — 90% 用户踩坑。信息不对称短期有利，长期伤信任。
饥饿营销常态化 — 每天 10:30 抢购。短期有效，长期伤品牌。
标价与实际脱节 — 订阅费是地板不是天花板，实际 2-5 倍。
国内海外价差过大 — 同产品两套价格，用户发现后信任崩塌。

"计量即权力，补贴即投资，收紧即筛选，分层即利润。
入口获客，生态锁客。先跑量再算账，别在第一天就想好终价。"

Section 05

四个趋势

价格战会继续，但利润空间来自"服务溢价"而非"算力差价"。

  1. Token 成本持续下降
    国产芯片放量（华为 950 年内 75 万片）+ DeepSeek V4 MoE 效率革命（1.6T 参数只激活 49B）+ 推理专用芯片竞争加速。
  2. 国产芯片替代改变供给格局
    英伟达市占率 66% → 54% → 可能低于 50%。华为昇腾 + 寒武纪 + 阿里平头哥形成国产算力三角。DeepSeek V4 同时适配华为+寒武纪，Day 0 支持。
  3. 开源模型重塑定价权
    DeepSeek 模式：开源 + 极致低价 + 不卖订阅。V3.2 月调用 5.35 万亿 token，全球第三。厂商定价权来自"服务便利性"而非"模型独占性"。
  4. Agent 时代成本指数级增长不可逆
    推理需求 2026 年已达训练的 4-5 倍。Coding Plan "不限量"模式不可持续。谁先转 Token 精细计量谁主动，谁后转谁被动。

Coding Plan → Token Plan 的转变，本质上是从"卖工具使用权"到"卖算力资源包"的商业模式升级。

Coding Plan ≈ 手机合约机（绑定特定工具）
Token Plan ≈ 手机流量包（通用算力，不限场景）
