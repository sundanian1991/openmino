## 图 1：定价演进时间线

**模式**：C04 时间线+里程碑
**渲染目标**：viz-echarts / viz-svg-flow

### 第一部分：叙事意图

**【可视化目的】**
读者以为"定价变化就是一堆数字调整"。这张图要让他们看到——这是有节奏的行业演进，精确重演了电信行业 20 年的三步曲。认知缺口：把"乱"重新定义为"有规律的演进"。

**【想传达什么】**
AI 定价不是乱打价格战，是"补贴→收紧→分层"的三段式节奏，每个阶段都有标志性事件。

**【结论】**
从 2024 到 2026，AI 定价用两年走完了电信行业二十年的路——Coding Plan 只是过渡，Token Plan 才是终态。

**【思路】**
水平时间轴，从左到右。用颜色区分三个阶段（蓝色=补贴、黄色=收紧、绿色=分层）。第一眼看到密集的事件分布（2025-2026），第二眼看到颜色变化（阶段切换），最终理解"这不是乱战，是节奏"。右侧加电信类比标注"2009→2013→2017→2020"作为对照。

## 图 2：套餐对比表（同样 ¥40-50，你能买到什么？）

**模式**：C05 对比表+结论
**渲染目标**：viz-echarts / viz-svg-flow

### 第一部分：叙事意图

**【可视化目的】**
读者看到"¥39、¥49"以为各平台差不多。这张图要让他们看到——同样价格，实际可用量差了 3 倍以上。认知缺口：把"价格对比"重新定义为"计量混乱下的信息不对称"。

**【想传达什么】**
90% 的开发者踩坑，不是因为他们不会算，是计量单位根本不统一。

**【结论】**
同样 ¥40-50/月，不同计量方式下实际可用量差异巨大——智谱 80 次提问 vs 腾讯云 70 轮 vs Cursor 225 次请求。

**【思路】**
横向对比表，每行一个平台。关键动作：把所有平台换算成"等效提问次数"，让读者一眼看出差距。最大差距的用信号色标注，最小差距的正常显示。

### 第二部分：视觉执行

**【模式选择】**
- **模式**：C05 对比表+结论
- **理由**：多选项多维度评估，需要横向对比 + 结论列

**【标题】**
- **主标题**：同样 ¥40-50/月，实际可用量最大差 3 倍——计量混乱让 90% 开发者踩坑
- **副标题**：2026-04 · 入门档套餐等效对比

**【视觉编码】**
- **行**：5 个平台（智谱 GLM、MiniMax、阿里百炼、腾讯云、Cursor）
- **列**：月费 | 计量单位 | 额度 | 等效提问/天 | 结论（高亮列）
- **颜色**：等效提问列用热力条——最高 Teal（绿色），最低 Coral（红色），中间渐变
- **排序**：按等效提问从多到少（让读者先看到"最划算"）

**【数据组织】**

| 平台 | 月费 | 计量 | 额度 | 等效提问/天 | 结论 |

## 图 3：定价模式演进的三层逻辑（瀑布分析）

**模式**：C02 瀑布分析
**渲染目标**：viz-echarts

### 第一部分：叙事意图

**【可视化目的】**
读者以为定价变化是"厂商想多赚钱"。这张图要让他们看到——定价是被三层力量推着走的：成本→竞争→价值。认知缺口：把"厂商贪心"重新定义为"多重力量驱动的必然"。

**【想传达什么】**
定价不是厂商一拍脑袋定的，是成本驱动→竞争驱动→价值驱动的三层演进。

**【结论】**
从 API 按量到 Token Plan，成本下降 90%（正向），但 Agent 消耗增长 10 倍（负向），最终迫使厂商从"模糊打包"转向"精细计量"。

**【思路】**
瀑布图：左柱=2024 API 定价（高），中间多个正负浮动柱（成本下降、竞争降价、Agent 消耗、收紧利润），右柱=2026 Token Plan（低但利润空间更大）。第一眼看到左右柱高度差，第二眼看到中间的最大驱动因素。

### 第二部分：视觉执行

**【模式选择】**
- **模式**：C02 瀑布分析
- **理由**：需要展示从 A（API 定价）到 B（Token Plan 定价）的驱动因素分解

**【标题】**
- **主标题**：Token 成本下降 90%，但 Agent 消耗增长 10 倍——定价从模糊打包到精细计量的必然
- **副标题**：2024 → 2026 · 定价驱动因素拆解

**【视觉编码】**
- **Y 轴**：单位 Token 成本（相对值）
- **柱**：正向驱动（成本下降，向下浮动，Teal 色）vs 负向驱动（消耗增长，向上浮动，Coral 色）
- **累计线**：连接各柱顶部
- **排序**：按贡献绝对值从大到小

**【数据组织】**

| 驱动因素 | 方向 | 幅度（相对值） | 说明 |
|---------|------|--------------|------|
| 起始：2024 API 百万 Token 定价 | — | 100 | 基准 |
| 国产芯片替代 | ↓ -25 | 华为昇腾 + 寒武纪压低算力成本 |

| 价格战补贴 | ↓ -30 | DeepSeek 带头，7 家抢用户 |

| 分层溢价 | ↑ +10 | 极速版、企业版溢价 |
| 终点：2026 Token Plan | — | 90 | 单价低但结构更优 |

**【标注策略】**
- **高亮**：最大的正向驱动（Agent 消耗增长 ↑+40）和最大的负向驱动（价格战补贴 ↓-30）
- **标注**：Agent 柱旁标"OpenClaw 催化"，价格战柱旁标"DeepSeek 带头"

- **图例**：不需要，颜色自解释（Teal=降本，Coral=增耗）
- **脚注**：Source: 公开数据估算，相对值非绝对金额

## 图 5：企业 AI 落地漏斗

**模式**：C06 漏斗+瓶颈标注
**渲染目标**：viz-echarts

### 第一部分：叙事意图

**【可视化目的】**
读者以为"AI 定价问题就是贵"。这张图要让他们看到——真正的问题不是价格，是 90% 的企业根本走不到生产环境。认知缺口：把"定价贵"重新定义为"落地难"。

**【想传达什么】**
Gartner 说 90% 企业推进 AI 实验，仅 41% 成功落地。跨越鸿沟的关键不是模型能力，是可信赖度。

**【结论】**
企业在实验阶段愿意尝试低价 Coding Plan，但进入生产环境后需要 SLA、稳定性、合规性——这才是 Token Plan 企业版溢价的真正基础。

**【思路】**
标准漏斗：实验意向（90%）→ 实际实验（60%）→ 试点成功（45%）→ 生产落地（41%）→ 持续付费（?%）。瓶颈在"实验→试点"这一步，标注"能力-准备度鸿沟"。
