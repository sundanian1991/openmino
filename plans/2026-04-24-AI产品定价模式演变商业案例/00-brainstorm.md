### 芯片供给线（算力成本的底层变量）

**英伟达在中国的退缩：**
- 2024 年市占率 66% → 2025 年 54% → 2025 年底跌破 60% 关口
- 2023 年美国禁止顶级 AI 芯片出口 → 2025-04 全面禁止所有 AI GPU 对华出口
- 后续松绑 H20/H200，但严苛采购限制，订单落地严重滞后

**华为昇腾的崛起：**
- 2025 年累计交付 81.2 万片，市占率 ~20%，国产第一
- 950PR：FP4 算力 1.56 PFLOPS（H20 的 2.87 倍），定价 ~7 万元（H200 的 1/3），4 月量产
- 950DT：2026Q4 推出，面向训练+推理 Decode，144GB HBM
- 三年路线图：950PR(26Q1) → 950DT(26Q4) → 960(27Q4) → 970(28Q4)
- 2026 年目标：AI 芯片市占率 50%，出货 75-100 万片，营收超 800 亿

**国产芯片整体格局：**
- 2025 年国产自给率 41%（英伟达 55%，AMD 4%）
- 伯恩斯坦预测：2027 年国产化率从 2023 年 17% → 55%
- 阿里平头哥 25.6 万片（第二）、百度昆仑芯/寒武纪各 11.6 万片
- 关键突破：华为自研 HBM（HiBL/HiZQ），绕开 HBM 供应链限制

**算力成本下降对定价的影响：**
- 国产芯片性价比提升 → 算力成本下降 → Token 成本下降空间打开
- DeepSeek V4 已适配华为昇腾 + 寒武纪，明确说"受限于高端算力，预计下半年昇腾950批量上市后价格大幅下调"
- 推理需求爆发（2026 年推理需求 = 训练的 4-5 倍）→ 推理芯片竞争加剧 → 成本进一步下降

### 跨越鸿沟：AI 从实验到生产的信任挑战

**数据：**
- Gartner：90% 企业推进 AI 实验，仅 41% 成功落地生产环境
- 预计 2027 年 40% Agentic AI 项目因无法证明 ROI 被终止
- 斯坦福 2026 AI 指数报告："能力-准备度鸿沟"，AI 事故年增 55%，仅 31% 美国人信任政府 AI 监管

**行业判断：**
- 真格基金戴雨森：Agent 产品仍处 early market，被创新者和早期采用者接受，跨越鸿沟需要模型能力和产品形态继续进步，"依然需要几年的时间"
- 港股 AI 上市潮（Q1 IPO 募资 79 天破 1000 亿港元），但资本市场的热情与实际落地之间存在落差

**对定价的含义：**
- 企业在实验阶段愿意尝试低价/免费的 Coding Plan → 但进入生产环境后需要 SLA、稳定性、合规性 → 这是 Token Plan 企业版溢价的基础
- 跨越鸿沟的关键不是模型能力，而是"可信赖度" → 定价策略需要从"低价获客"转向"价值证明"
- 智谱和 MiniMax 的招股书都在强调"企业客户"和"商业化落地" → 说明资本市场也在等待跨越鸿沟的证据

## 未来预判

### 趋势 1: Token 成本将持续下降

驱动力：
- 国产芯片放量（华为 950 年内 75 万片）→ 算力成本下降
- DeepSeek V4 开源 → MoE 架构效率革命（1.6T 参数只激活 49B）
- 推理专用芯片竞争（英伟达 Rubin CPX / 谷歌 TPU v8 / 华为 950PR）→ 推理成本加速下降

结果：Token Plan 的价格战会继续，但利润空间来自"服务溢价"而非"算力差价"。

### 趋势 2: 国产芯片替代加速改变供给格局

- 英伟达市占率持续下降（66% → 54% → 2026 年可能低于 50%）
- 华为昇腾 + 寒武纪 + 阿里平头哥形成国产算力三角
- 软件生态从 CUDA 迁移到 CANN，迁移效率已超 90%
- DeepSeek V4 同时适配华为 + 寒武纪，Day 0 支持

含义：算力供给不再被英伟达卡脖子 → 国内 AI 厂商的成本结构会和海外产生分化 → 定价策略可以更激进。

### 趋势 3: 开源模型重塑定价权

DeepSeek 模式：开源 + 极致低价 + 不卖订阅
- V3.2 在 OpenRouter 月调用 5.35 万亿 token，全球第三
- 开源模型占所有开源 Token 的一半以上
- 但 DeepSeek 没有 Coding Plan/Token Plan，只卖 API

冲击：开源模型的免费部署能力，会持续压制定价上限。厂商的定价权来自"服务便利性"而非"模型独占性"。

### 趋势 4: Agent 时代的成本指数级增长不可逆

- OpenClaw 爆火证明：一个 Agent 任务 = 几十次模型调用
- 推理需求 2026 年已达训练的 4-5 倍，且在加速
- IDC 预测 2028 年推理工作负载占比 73%
- Coding Plan "不限量"模式在 Agent 时代不可持续

结果：所有厂商最终都会转向 Token 精细计量。谁先转谁主动，谁后转谁被动（Cursor 2025-06 的教训）。

### 参与者分类

**模型厂商（专卖店路线）**：智谱 GLM、Kimi（月之暗面）、MiniMax、小米 MiMo —— 只提供自家模型，深度优化

**云平台（超市路线）**：阿里云百炼、火山方舟（字节）、腾讯云 —— 聚合多家模型，一个 Key 自由切换

### 五种计量体系并存

| 计量方式 | 采用者 | 换算关系 | 透明度 |
|----------|--------|----------|--------|
| API 请求次数 | 阿里百炼、火山方舟、腾讯云、百度千帆 | 1 次用户提问 = 5-30 次 API 请求 | 低（数字大但实际消耗不透明） |
| Prompt 次数 | 智谱 GLM、MiniMax（早期） | 1 Prompt ≈ 15-20 次模型调用 = 1200-1600 API 请求 | 中（比 API 次数直观） |
| Token 计量 | Kimi（2026.1.28 切换后）、MiniMax Token Plan | 直接按输入输出 Token 计费，仅统计未命中缓存的 Token | 高（最透明，但用户难预估） |
| Credit 计量 | Cursor、小米 MiMo、阿里百炼 Token Plan | 统一 Credit 池，不同模型不同倍率（MiMo: Omni 1x, Pro 256k 2x, Pro 1M 4x） | 中（抽象了一层但可预测） |
| 时间窗口配额 | 智谱 GLM（5h 窗口）、Claude Code（5h+7d 双层） | 每 5 小时限额 + 每周/每月上限 | 中（限制节奏而非总量） |

**计量单位演变趋势**：API 次数 → Prompt 次数 → Token/Credit → 统一 Credits。方向是越来越透明、越来越可预测。

## 关键发现

### 1. 手机流量套餐的精确重演

一位日耗 10 亿 Token 的工程师的观察：

> - 2009：充 100 块话费，根本用不完
> - 2013：9.9 块买几 G，2G-3G 网络
> - 2017：各种"不限量"，但 4G 达量降速
> - 2020：真正无限流量消失，按 GB 精打细算
>
> Token 正在走一模一样的路。

### 2. 定价不透明是最大信息壁垒

各平台计量单位完全不同，直接比数字毫无意义。90% 开发者容易踩坑。

### 3. 国内 vs 海外定价逻辑差异

- **国内**：七家在一个池子里抢用户，价格战打得凶 → 便宜
- **海外**：面对 Claude/GPT/Gemini，不需要打价格战，需要证明"我不比 Claude 差"

### 4. Agent 时代的成本放大器

- 70% coding agent token 是浪费（DEV Community 研究）
- 一个开发者 8 个月 Claude Code 消耗 100 亿 Token，API 价 $15,000+，Max plan 只花 $800
- Cursor 2025-06 转 credit-based，重度用户一周超支 $350，引发退款风波

### 5. 订阅费是地板不是天花板

Token 超额、premium 模型加价、agent 循环消耗 → 实际账单是标价的 2-5 倍

### 6. OpenClaw 爆火 = Token 消耗的催化剂

- OpenClaw GitHub 355K stars，开源 AI Agent 框架
- 直接催热 Coding Plan 需求 —— 以前写代码手动调 API，现在 Agent 自动循环调用
- MiniMax M2 token 用量：2 月 vs 12 月增长 6 倍，Coding Plan 消耗增长 10 倍+
- 阿里云算力告急，限量供应（Lite 档 4-13 停售的直接原因）
- Anthropic 2026-04 从标准订阅移除 OpenClaw 访问 —— 成本撑不住了

**含义**：OpenClaw 这类 Agent 框架是 Token 消耗的放大器。一个 Agent 任务 = 几十次模型调用。Coding Plan 的"不限量"承诺在 Agent 时代不可持续，必须转向 Token 精细计量。

## Themes（反复出现的主题）

### Theme 1: 成本可控性是用户核心痛点

用户从"按 Token 付费的不确定性恐惧"转向"固定月费的安全感"，再到"精细分层的按需选择"。

### Theme 2: 计量单位是定价权的争夺

谁能定义计量单位（Token/Prompt/请求/credit），谁就掌握定价权。MiniMax 从 Prompt 转 Token、Kimi 从 Prompt 转 Token，都是在争夺这个定义权。

### Theme 3: 补贴 → 收紧 → 分层的经典三段式

和互联网所有赛道一样：烧钱圈用户 → 市场份额稳定 → 精细分层收割。Token 价格战只是新一轮。

### Theme 4: 编程只是入口，生态才是目标

MiniMax 把 Coding Plan 改名 Token Plan，因为它打包了图片、语音、音乐、视频。从"编程工具"升级为"AI 工作方式"。

### Theme 5: Agent 框架是定价模式的颠覆者

OpenClaw 爆火直接加速了 Coding Plan → Token Plan 的转变。Agent 自动循环调用让"不限量订阅"不可持续，倒逼厂商转向精细计量。Anthropic 移除 OpenClaw 访问是标志性事件。

### Theme 6: 新入局者用低价破局，但终局必然是收紧

小米 MiMo V2-Pro 定价 $1/$3（Claude 的 1/5），但官方同时说"低价是陷阱"。这和 DeepSeek 当年带头降价后各家跟进再收紧的节奏完全一致。

## 关键判断

基于事实，核心问题是：

> **Coding Plan → Token Plan 的转变，本质上是从"卖工具使用权"到"卖算力资源包"的商业模式升级。**

类比：
- Coding Plan ≈ 手机合约机（绑定特定工具/场景）
- Token Plan ≈ 手机流量包（通用算力资源，不限场景）

MiniMax 改名 Token Plan 的逻辑：不只卖编程能力，卖的是全模态 AI 能力的"算力通票"。
