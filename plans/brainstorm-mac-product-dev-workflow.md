# Brainstorm: Mac 桌面产品开发全流程方法论

> 日期：2026-06-29 | 状态：已完成（两轮合并）

---

## What I Heard

**核心问题：** 有具体产品（Insight Asset OS）在做，但缺少可复用的产品开发全流程方法论——从产品策划到架构到 UI 到交付，而不是每次靠对话即兴堆功能。

**已有资产：**
- Tauri 版本（更成熟，7 页面 + 14 API + SQLite + LLM + OCR + 语义搜索）
- Electron 版本（Phase 1 骨架规划，5 步执行计划，packages/ 下无源文件）
- 产品设计（06-25/06-28 日志，"采集 → 整理 → 输出"三步闭环，Insight Kernel）
- 四阶段方法论框架（workspace/2026-06-29-Mac产品开发流程/）——已写但未在产品上验证
- Phase1 实施计划（workspace/2026-06-29-Phase1-实施计划/）——Electron 技术方案
- 手头 50+ 技能（brainstorming、Plan、frontend-design、impeccable 等）——单次调用模式，缺串联

## 核心发现

**不是缺少开发流程，是三样东西没配对：**

1. **四阶段方法论没在 Insight Asset OS 上实际跑过**——理论未被验证
2. **技能调用时机不清晰**——头脑风暴在哪个阶段用？Plan 在哪个阶段用？UI 设计何时介入？没有对照表
3. **两个技术路线（Tauri 已有代码 vs Electron 新计划）没有做取舍和记录**

## Decisions to Make

- Tauri vs Electron → **先都放一放**，等方法论定下来再选
- 通用方法论 vs 具体产品 → **两者都要**，先抽象流程再用产品落地
- 已删资源 → **AI 技能/bundle**，需要搜 GitHub 找现成的

## Constraints

- Mac 平台，个人项目，无团队
- 经验偏轻办公（HTML/可视化），桌面端开发是新领域
- 之前删除的 AI 技能/bundle 需要找回或重建

## Themes

1. **流程断裂 vs 工具堆积** —— 有零散技能但缺连接层（"产品全链路 × 技能调用时机"对照表）
2. **产品已有，方法论缺失** —— 边做边想，缺标准开发顺序
3. **两个版本悬而未决** —— Tauri 更完整，Electron 有计划，需决策
4. **理论和实践的断裂** —— 有方法论（workspace），有代码（insight-asset-os），有实施计划（Phase1），但三者之间没有流转关系

## Tensions

- 通用方法论 vs 具体产品（两者兼顾，方法论先行）
- 深度 vs 速度（完整流程每个阶段都需要时间）

## Gaps

- 不知道之前删除的 AI 技能具体是什么
- 不确定 UI/UX 能力边界——Mac 桌面端 UI 设计有哪些可复用的范式
- 没有正式 PRD
- 缺少一张"产品全链路 × 技能调用时机"对照表

## 承重问题（待确认）

- **技术路线**：坚持 Tauri 已有代码，还是切到 Electron？还是先不管，跑流程优先？
- **核心目标**：做出一个完整产品？沉淀可复用的流程方法论？还是双目标以产品为锚点？
- **切入方式**：从现有代码出发验证方法论？还是先画流程地图再动手？

## 方向

创建工作区目录 + 完整的流程框架文档，然后开始 Phase 0 产品定义。
独立的目录结构，把完整的开发生命周期拆成可执行的阶段，每个阶段有明确的输入/输出/检查点。

核心三件事：
1. 画一张"产品全链路 × 技能调用时机"对照表
2. 以 Insight Asset OS 为锚点，从头走一遍 Explore → Validate → Build → Iterate 四阶段
3. 走完后提炼成可复用的开发流程模板
