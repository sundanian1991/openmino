# AI长周期任务协作方法论 — 教学材料重构计划

**Type:** Operations
**Status:** Draft
**Created:** 2026-07-21

---

## Recommendation

**问题诊断**：呈现方式不对，不是内容问题。两个版本都犯了同一个错——试图在一个页面里塞下所有信息，结果用户不知道从哪开始看、看完能做什么。

**核心改进**：从"信息展示"转向"任务驱动"。用户打开页面的第一眼应该知道：这东西能帮我解决什么问题 → 我现在该做什么 → 做完能得到什么。

---

## Current State

**已尝试的两个版本：**

| 版本 | 结构 | 用户反馈 |
|------|------|---------|
| 旧版（7章理论框架） | 概述→实验→Agent Loop→5习惯→对比→可迁移框架→工具箱→范式升级 | "不是很好理解。不好落地。" |
| 新版（5个操作+代码块） | 建GOALS.md→读文件→更新文件→拆里程碑→三句话汇报 | "不够清晰。" |

**失败原因分析：**

1. **信息密度过高** — 两篇文章的内容太多，塞在一个页面里让人窒息
2. **视觉层次不清晰** — 没有明确的"先看什么、再看什么"
3. **缺少"啊哈时刻"** — 用户打开页面，不知道这东西能帮他解决什么具体问题
4. **交互为炫技而非理解** — 展开/收起、导航栏等交互没有服务于"让人更容易理解"

---

## Proposed Approach

### 方案：任务卡片式布局

**核心理念**：不是"教你一套理论"，而是"帮你解决一个具体问题"。

**页面结构**：

```
┌─────────────────────────────────────────────────┐
│  Hero: 一句话说清楚这东西能帮你做什么            │
├─────────────────────────────────────────────────┤
│  问题选择器: 你遇到了哪个问题？                  │
│  [AI跑偏了] [不知道从哪开始] [做完不知道质量]    │
├─────────────────────────────────────────────────┤
│  解决方案卡片:                                   │
│  ┌─────────────────────────────────────────────┐│
│  │ 问题: AI跑着跑着就偏了                       ││
│  │ 原因: 目标只存在对话里，context会清空         ││
│  │ 解法: 用GOALS.md文件锚定目标                 ││
│  │ 操作: [3步，每步可复制]                       ││
│  │ 验证: 怎么知道做对了                         ││
│  └─────────────────────────────────────────────┘│
├─────────────────────────────────────────────────┤
│  工具箱: 可直接复制的Prompt和模板               │
├─────────────────────────────────────────────────┤
│  底部: 来源链接                                 │
└─────────────────────────────────────────────────┘
```

### 交互设计

1. **问题选择器** — 用户点击自己遇到的问题，页面滚动到对应的解决方案
2. **解决方案卡片** — 每个问题一张卡片，包含：问题描述→原因分析→解决步骤→验证方法
3. **一键复制** — 每个操作步骤都有可复制的prompt
4. **进度保存** — 用户完成的步骤可以打勾，下次打开还能看到

### 视觉设计

- **极简** — 去掉所有不必要的装饰，只保留内容
- **大字** — 标题28px+，正文16px+，确保可读性
- **留白** — 每个区块之间有足够的呼吸空间
- **颜色** — 只用三种：蓝色（主色）、绿色（已完成）、红色（问题）
