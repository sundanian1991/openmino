# 03.5-页面规划：Storytelling with Data

> 2026-05-29 | 03-叙事规划之后，04-研究转策展文本之前

---

## 页面类型

| 页面类型 | 角色 | 本项目实例 |
|----------|------|-----------|
| **入口页** | 建立基调，一句话告诉观众这是什么 | "从混乱到清晰——数据可视化的知行鸿沟" |
| **单元页** | 承载一个完整单元的内容，是策展的主体 | 7 个单元，每单元一个页面 |
| **索引页** | 全局导航，展示 7 个单元的关系和进度 | 可选，编辑叙事式可以不单独做，用顶部导航替代 |

本项目不需要：跨单元分析页（单元之间已通过递进关系串联）、来源页（来源嵌入各单元证据层）、关于页（入口页已包含定位信息）。

---

## 单元页模块 Schema

每个单元页包含以下模块，按阅读顺序排列：

```
┌─────────────────────────────────┐
│  单元标题                         │  ← 大号，冲击力
│  定位句                           │  ← 中号，一句话说明价值
├─────────────────────────────────┤
│  导言                            │  ← 开场，设定语境
├─────────────────────────────────┤
│  入口层                           │  ← 观众最先遇到的具体内容
│  （案例 / 实验 / 对比 / 命题）      │     每个单元不同，由迷你定题决定
├─────────────────────────────────┤
│  主题层                           │  ← 核心判断或问题
├─────────────────────────────────┤
│  展开层                           │  ← 解释性正文、框架、理论
├─────────────────────────────────┤
│  证据层                           │  ← 数据、图表、引用、案例细节
├─────────────────────────────────┤
│  收束 / 过渡                      │  ← 一句话收束 + 引向下一单元
├─────────────────────────────────┤
│  来源脚注                         │  ← 数据来源、引用出处
└─────────────────────────────────┘
```

### 各单元模块明细

| 单元 | 入口层模块 | 展开层模块 | 证据层模块 |
|------|-----------|-----------|-----------|
| 1 上下文 | 对比面板（错误 vs 正确汇报） | 卡片组（上下文五问） | 引文（牡蛎与珍珠原文） |
| 2 图表 | 文字叙述（一句话反叛） | 图表（视觉通道示意图） + 对比面板（4 种替代方案） | 图文混排（Fox News + 零售商案例） |
| 3 去杂 | 文字叙述（抛出问题） | 图表（格式塔原理） + 流程图（六步去杂） | 对比面板（每步前后） |
| 4 注意力 | 图文混排（"数 3"实验） | 图表（前注意属性列表） | 对比面板（彩虹色 vs 策略色） |
| 5 设计 | 对比面板（Pew Research 两版） | 文字叙述（追问） + 图表（视觉层次） | 对比面板（行动标题 vs 描述标题） |
| 6 故事 | 图文混排（红帽姑娘实验） | 流程图（三幕结构 + Bing-Bang-Bongo） | 图文混排（零售定价 9 页拆解） |
| 7 案例 | 文字叙述（命题） | 对比面板（意面图三种解法） | 图文混排（深色背景 + 饼图替代） |

## 组件清单

本项目需要的组件类型：

| 组件 | 用途 | 出现位置 |
|------|------|----------|
| **对比面板** | 并列对比两种方案/状态/版本 | 单元 1/2/3/4/5/6/7 |
| **卡片组** | 多个平行条目（如五问） | 单元 1 |
| **流程图** | 步骤、过程、因果关系 | 单元 3/6 |
| **图文混排** | 案例 + 文字解读 | 单元 4/6/7 |
| **引文** | 书中原文引用 | 单元 1/5 |
| **表格** | 结构化对比（如替代方案） | 单元 2 |
| **文字叙述** | 解释、分析、讲故事 | 全部单元 |

## 交互逻辑

| 交互 | 对应的策展关系 | 实现方式 |
|------|--------------|----------|
| **滚动** | 逐层揭示（入口→展开→深入→出口） | 长页面自然滚动，信息逐步展开 |
| **并列对比** | A vs B 的直接比较 | 对比面板，左右或上下并列 |
| **步骤展开** | 流程的先后顺序 | 流程图，垂直排列 |
| **悬停揭示** | 来源、细节的按需展示 | 来源脚注默认折叠，悬停展开 |

本项目不需要：标签页切换（单元之间用滚动串联，不用切换）、地图交互（无地理数据）、筛选器（内容固定）、时间线交互（时间线是内容，不是交互）。

## 导航设计

| 导航元素 | 位置 | 行为 |
|----------|------|------|
| **顶部导航** | 固定在页面顶部 | 显示 7 个单元标题，点击跳转到对应单元 |
| **进度指示** | 顶部导航内 | 显示当前所在单元 |
| **单元间过渡** | 每个单元底部 | 收束句 + "下一个：XXX" 的文字引导 |
| **返回顶部** | 页面右下角 | 固定按钮，点击回到入口页 |

编辑叙事式的导航原则：**滚动为主，跳转为辅**。观众的主要行为是向下滚动，逐步发现内容。顶部导航是备用通道，让观众可以跳转到感兴趣的单元。
