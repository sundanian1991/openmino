## 8大风险维度（最新定义）

### 已确认定义

| # | 风险维度 | 定义/阈值 | 判断标准 |
|---|----------|-----------|----------|
| 1 | 微小站点 | headcount < 20 | 职场人力 < 20人 |
| 2 | 超大站点 | headcount > 100 | 职场人力 > 100人 |
| 3 | 单产品供应 | 某产品线供应商数 < 3 | 该产品线的供应商数量 < 3家 |
| 4 | 单产品线供应商 | 供应商只服务1条产品线 | 供应商的products.length === 1 |
| 5 | 供应链单薄 | 同产品线供应商数 < 3 | 与#3相同，可能需要合并 |
| 6 | 最大省份集中度 | 某省人力占总人力比例 | 最大省占比 > 40%中风险，>50%高风险 |
| 7 | **地理集中度** | **同城市站点数 > 2** | ⚠️ 从区县改为城市级别 |
| 8 | **项目单一风险** | **供应商只做1条产品线** | ⚠️ 与#4相同，需要合并或区分 |

### 待澄清

- **#4和#8的关系**：都是"单产品线供应商"，是否需要合并？
- **#3和#5的关系**：都是"某产品线供应商数<3"，是否需要合并？
- 是否需要新增其他风险维度？

## 数据结构说明

### RAW数据格式（108条）

```javascript
{
  id: 1,
  site: "颍州站点",
  company: "衍生",
  short: "衍生",      // 供应商简称
  province: "安徽",
  city: "阜阳",
  district: "颍州区",
  region: "华东",
  tier: "三线",
  product: "企业主贷",
  bizLine: "企金",
  mode: "人工",
  m1: 12,  // 1月人力
  m2: 8,   // 2月人力
  m3: 13   // 3月人力
}
```

### 聚合引擎（所有版本共享）

1. **mergeWorkplaces(data)**: 按short+site合并职场
2. **mergeSuppliers(data)**: 按short合并供应商
3. **mergeProducts(data)**: 按product合并产品线

### 筛选器逻辑

- 月份：currentMonth = 1/2/3
- 省份：filterProvince
- 供应商：filterSupplier
- 产品线：filterProduct

## 主题/张力/缺口

### 主题

1. **风险维度的准确性**：8大风险中有重复（#3/#5、#4/#8），需要去重或重新定义
2. **图表组合的差异化**：四个版本不能只是换颜色，必须有实质性内容差异
3. **实现复杂度**：旭日图、帕累托图、风险评级表需要ECharts高级用法

### 张力

1. **完整 vs 精简**：V6-RP要完整，V6-Swiss要精简，如何平衡？
2. **可视化 vs 表格**：V6-Data全表格，V6-Consult多图表，各有优劣
3. **自动化 vs 人工**：洞察面板是自动计算还是人工编辑？

### 缺口

1. **旭日图数据结构**：需要准备层级数据（区域>省份>供应商>人力）
2. **帕累托图逻辑**：需要确定按什么维度计算累计占比
3. **风险评级算法**：V6-Data的供应商风险评级表如何计算综合等级？
