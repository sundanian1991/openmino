## 看板布局设计

### 整体结构

```
┌─────────────────────────────────────────────────────┐
│  顶部标题栏                                          │
├─────────────────────────────────────────────────────┤
│  KPI 卡片行（5个）                                   │
├─────────────────────────────────────────────────────┤
│  筛选器栏（月份切换 + 供应商/产品线/省份筛选）          │
├──────────────────────┬──────────────────────────────┤
│  人力总览趋势（折线）  │  产品线分布（环形图）          │
│                      │                              │
├──────────────────────┼──────────────────────────────┤
│  供应商 TOP10（柱状） │  省份分布（柱状/地图）         │
│                      │                              │
├──────────────────────┴──────────────────────────────┤
│  站点明细表（可排序/可搜索/可导出）                     │
├─────────────────────────────────────────────────────┤
│  月度环比变化（增长率柱状图，正绿负红）                 │
└─────────────────────────────────────────────────────┘
```

### 模块详细规格

#### 模块 1：KPI 卡片（P0）
| 指标 | 计算逻辑 | 格式 |
|------|---------|------|
| 总人力 | 当月所有站点人力之和 | 大数字 |
| 总站点数 | 去重站点数 | 大数字 |
| 总供应商数 | 去重供应商数 | 大数字 |
| 产品线数 | 去重产品线数 | 大数字 |
| 月度变化 | 当月总人力 vs 上月 | 百分比 + 箭头 |

**交互**：点击月份切换时，KPI 实时更新

#### 模块 2：人力总览趋势（P0）
- **图表**：面积折线图（Chart.js）
- **X轴**：月份（1月-4月）
- **Y轴**：人力总数
- **叠加**：人工 vs 人机 分层
- **交互**：hover 显示具体数值

#### 模块 3：供应商 TOP10 排名（P0）
- **图表**：水平柱状图
- **数据**：按供应商汇总当月总人力，取前10
- **颜色**：按产品线分段着色
- **交互**：hover 显示详细数据

#### 模块 4：产品线人力分布（P0）
- **图表**：环形图（Doughnut）
- **数据**：按产品线汇总当月人力
- **标签**：产品线名称 + 人力数 + 百分比
- **交互**：hover 高亮

#### 模块 5：省份分布（P0）
- **图表**：水平柱状图（按省份汇总）
- **数据**：按省份汇总当月人力
- **颜色**：梯度色（人力越多越深）
- **交互**：hover 显示具体值

#### 模块 6：筛选器栏（P1）
- **月份选择器**：1月/2月/3月/4月（默认最新有数据的月份）
- **供应商筛选**：下拉多选
- **产品线筛选**：下拉多选
- **省份筛选**：下拉多选
- **重置按钮**

#### 模块 7：站点明细表（P1）
- **列**：序号、供应商、站点、产品线、模式、省份、城市、1月人力、2月人力、3月人力、4月人力、环比变化
- **功能**：排序、搜索、滚动
- **样式**：斑马纹、环比变化正绿负红

#### 模块 8：月度环比变化（P1）
- **图表**：垂直柱状图
- **数据**：每个站点当月 vs 上月变化率
- **颜色**：正增长绿色，负增长红色
- **交互**：hover 显示具体变化值

## 数据处理计划

### 原始数据 → BI 数据

```
1. 解析 Markdown 表格 → JSON 数组
2. 字段清洗：
   - 供应商名称统一（去除多余空格）
   - 省份名称标准化（"陕西省" → "陕西"）
   - 空值处理（无数据的月份填 null）
3. 派生字段计算：
   - 月度环比 = (当月 - 上月) / 上月
   - 供应商汇总 = 同供应商所有站点之和
   - 产品线汇总 = 同产品线所有站点之和
   - 省份汇总 = 同省份所有站点之和
4. 导出为 JSON，嵌入 HTML
```

## 技术实现计划

### 前端技术栈
| 组件 | 技术 | 理由 |
|------|------|------|
| 图表库 | Chart.js 4.x | 轻量、交互好、文档全 |
| 样式 | CSS Grid + Flexbox | 响应式布局 |
| 字体 | 系统字体栈 | 无依赖、加载快 |
| 主题 | CSS Variables | 方便切换深浅色 |
| 数据 | 内嵌 JSON | 离线可用、无后端 |

### 文件结构
```
单文件 HTML：
  <head> — CSS Variables + 样式
  <body>
    ├── <header> — 标题 + 月份切换
    ├── <section.kpi-cards> — 5 个 KPI 卡片
    ├── <section.charts-row-1> — 趋势 + 环形图
    ├── <section.charts-row-2> — 供应商排名 + 省份分布
    ├── <section.detail-table> — 站点明细表
    └── <section.monthly-change> — 月度环比
  <script> — Chart.js CDN + 数据 + 逻辑
```

## 设计风格规格

遵循 mino-frontend 克制专业主义：

| 属性 | 值 |
|------|---|
| 主色 | #1a1a2e（深色）或 #f8f9fa（浅色） |
| 强调色 | #e94560（珊瑚红） |
| 辅助色 | #0f3460（深蓝）、#16213e（墨蓝） |
| 圆角 | 2px |
| 分割线 | 1px solid rgba(0,0,0,0.08) |
| 字体 | -apple-system, BlinkMacSystemFont, "SF Pro", sans-serif |
| KPI 数字 | 36px, font-weight: 700 |
| 卡片间距 | 16px |
| 内边距 | 20px |
| 最大宽度 | 1400px 居中 |
