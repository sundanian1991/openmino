# 供应商站点看板 v5 — 完整需求文档

> 日期：2026-04-03
> 定位：管理决策工具 + 汇报展示（兼用）
> 产出物：`workspace/output/供应商站点看板-v5.html`
> 核心变更：分析维度从「业务线」改为「项目」

---

## 一、项目背景

### 1.1 问题

v1/v2 是「数据展示」思维——把 108 条记录堆上去，有图表有筛选，但年老师（管理 30+ 供应商、3000+ 人团队）日常决策时用不上。核心问题：

- 108 条记录是「站点×项目」维度，不是「物理职场」维度。一个物理职场可能有多条记录
- 没有风险信号——哪些站点人太少扛不住、哪些站点太大依赖过重、哪些供应商集中度太高
- 没有交叉分析——每个职场做了哪些项目、这些项目又分散在哪些地方
- 地理分析缺失——供应商之间是否有物理重叠、同一区域供应商密度如何

### 1.2 目标

**一句话**：年老师打开看板，5 分钟内知道「今天该关注什么、该找谁聊、有什么风险」。

| 维度 | v1/v2 | v5 |
|------|-------|----|
| 站点定义 | 原始 108 条 | 合并物理职场 67 个 |
| 分析维度 | 业务线（6 条） | **项目（15 个）** |
| 风险视角 | 无 | 8 个风险维度 |
| 核心洞察 | 图表堆砌 | 自动生成关键洞察 |
| 交叉分析 | 单向热力图 | 职场×项目 + 项目×职场 双向矩阵 |
| 数据驱动 | 展示数据 | 管理决策 |

### 2.2 数据清洗规则

| 问题 | 清洗规则 |
|------|---------|
| 城市名格式 | 已标准化（都带"市"），无需调整 |
| 省份格式 | "河北省" 保持完整（不缩写） |
| 人力空值 | 未填报月份标记为 `null`，图表中不参与计算、不显示 |
| Q1 平均人力 | 无缺失，直接使用 |

### 2.3 物理职场合并

**合并键**：`供应商简称 + 职场名称`（如"毅航 × 埇桥站点"）

**合并逻辑**：
```
合并后职场 = 唯一(供应商简称 + 职场)
合并人力（某月）= 该职场下所有记录该月人力之和
合并Q1平均 = 该职场下所有记录的 Q1 平均人力之和
项目列表 = 该职场下覆盖的所有项目（去重）
模式 = 人工（有任何一条是人工）/ 纯人机（全部是人机）
省份/城市/地址 = 取第一条（同一物理职场地址相同）
```

**合并后数量**：67 个物理职场（从 108 条合并）。

### 2.4 新增计算字段

| 字段 | 计算逻辑 | 用途 |
|------|---------|------|
| 合并人力（当月） | 同一职场所有记录当月人力之和 | 所有分析的基石 |
| 职场规模等级 | 合并人力 <20 微小 / 20-99 正常 / 100-199 大型 / ≥200 超大 | 风险分类 |
| 项目数量 | 职场覆盖的项目数 | 灵活性评估 |
| 月度环比 | 当月 / 上月 - 1 | 趋势判断 |
| 连续下降 | 连续 N 个月环比 < 0 | 流失预警 |
| 供应商集中度 | 供应商在某项目人力 / 该项目总人力 | 集中度风险 |

│  • 例："毅航埇桥站点人力已达 461 人，为最大单点依赖"     │

**KPI 卡片规格**：
- 4 列等宽网格
- 数值大字（28px 加粗）、标签小字（13px 灰色）
- 数值右侧显示环比：↑绿色 / ↓红色 / →灰色
- 环比计算：当月 vs 上月

**关键洞察生成规则**：

| # | 洞察类型 | 触发条件 | 文案模板 |
|---|---------|---------|---------|
| 1 | 总人力趋势 | 月度环比 ≠ 0 | "{月}总人力较{上月}{增/降}{X}%，主要由{项目/供应商}贡献" |
| 2 | 单点依赖 | 某职场合并人力 > 总人力 15% | "{供应商}{站点}人力已达{N}人，占总人力{X}%，为最大单点依赖" |
| 3 | 供应链单薄 | 某项目供应商数 < 2 | "{项目}仅靠{N}家供应商支撑，供应链单薄" |
| 4 | 微小站点警告 | 微小站点数 > 5 | "当前{N}个站点人力不足20人，运营成本效率偏低" |
| 5 | 新开/关闭 | 某职场某月从 0→有值 或 有值→0 | "{供应商}{站点}{月}新增/关闭，人力{N}" |

### 5.2 联动规则

- 清除筛选 = 重置所有筛选器

## 七、交互规范

### 7.1 Tab 切换

- 点击 Tab 标签切换
- 切换动画：fadeIn（200ms）
- Tab 标签固定在内容区顶部
- 当前 Tab 高亮（底部蓝色边线 + 文字变色）

### 7.2 图表交互

- ECharts 图表统一配置：
  - `tooltip.trigger: 'item'`
  - `legend.show: true`（位置根据图表调整）
  - 统一使用 PALETTE 配色

### 7.3 表格交互

- 热力图表格：hover 行高亮（`--card-hover` 背景色）
- 数据明细表：点击表头排序（升序/降序切换）
- 风险列表：hover 显示 tooltip 详情
