## 二、核心问题重述

**年老师管理 30+ 供应商、3000+ 人团队，打开看板后 5 分钟内要做什么？**

### 2.1 管理场景假设

| 场景 | 行为 | 看板需要提供什么 |
|------|------|-----------------|
| **日常监控** | 打开看板，扫一眼核心指标和风险信号 | KPI 卡片 + 风险概览 + 关键洞察 |
| **问题诊断** | 发现某供应商人力下降，想知道原因 | 供应商维度下钻 + 趋势对比 + 职场明细 |
| **决策支持** | 要给某条业务线新增供应商，想知道现状 | 产品线分析 + 供应链健康度 + 地域分布 |
| **汇报准备** | 要给领导汇报，需要展示材料 | 各 Tab 的图表 + 自动生成的洞察 |
| **数据查询** | 要查某个站点的详细人力数据 | 数据明细 Tab + 筛选 + 导出 |

### 2.2 设计张力

| 张力点 | 选项 A | 选项 B | 权衡 |
|--------|--------|--------|------|
| **定位** | 管理决策工具 | 汇报展示材料 | 两者兼用，但以决策为主 |
| **数据粒度** | 物理职场合并 | 原始站点记录 | 默认合并，可切换明细 |
| **风险展示** | 独立预警 Tab | 分散在各 Tab | 两者兼有：预警中心 + 标签高亮 |
| **洞察深度** | 自动生成 | 需要人工解读 | 自动生成 + 人工下钻 |
| **筛选维度** | 全局联动 | 各 Tab 独立 | 全局联动 |

## 四、需要澄清的问题

### 问题 1：看板定位

v3 需求说"管理决策工具 + 汇报展示（兼用）"，但这两个场景的设计重点不同：
- **决策工具**：强调风险信号、异常检测、下钻能力
- **汇报材料**：强调美观、完整、自解释

**年老师**：这个看板主要用来做什么？日常自己看，还是主要给领导汇报？

### 问题 2：预警中心是否需要？

人力看板案例有独立的"预警中心"Tab，包含：
- 综合风险评分（仪表盘）
- 风险类别分布（饼图）
- 高风险预警详情列表
- 快速操作按钮（导出报告、发送通知）

v3 需求没有这个 Tab。

**年老师**：是否需要新增预警中心 Tab？

### 问题 3：地理分析深度

人力看板案例有：
- 中国地图展示省份分布
- 热力地图展示站点密度

v3 需求没有地图展示。

**年老师**：是否需要地图可视化？如果需要，展示什么维度？

### 问题 4：桑基图是否需要？

人力看板案例有"供应商→产品线"桑基图，直观展示人力流向。

v3 需求没有桑基图，但 Tab 2 有"职场×业务线矩阵"（热力图表格）。

**年老师**：桑基图和热力图矩阵，哪个更有价值？还是两个都要？

### 问题 5：职场×月份热力图是否需要？

人力看板案例有"TOP 20 站点 × 月份人力热力图"，可以识别：
- 季节性模式（哪些月份人力高/低）
- 增长/下降趋势
- 异常波动

v3 需求没有这个图表。

**年老师**：是否需要新增这个图表？放在哪个 Tab？

## 五、初步建议

基于分析，我建议 v3 需求做以下调整：

### 5.1 结构调整

| 调整项 | 变更 |
|--------|------|
| Tab 1 名称 | "全局大盘" → "全局概览" |
| Tab 2 名称 | "职场分析" → "职场视角" |
| Tab 3 名称 | "供应商分析" → "供应商视角" |
| Tab 4 名称 | "产品线分析" → "产品线视角" |
| Tab 5 名称 | "数据明细" → "预警中心 + 数据明细"（合并） |

### 5.2 功能新增

| 新增项 | 位置 | 理由 |
|--------|------|------|
| 预警中心模块 | Tab 5 上半区 | 独立展示风险信号 |
| 风险仪表盘 | Tab 5 顶部 | 综合风险评分 |
| 供应商→产品线桑基图 | Tab 3 下半区 | 直观展示人力流向 |
| 职场×月份热力图 | Tab 2 下半区 | 识别季节性模式 |
| 中国地图 | Tab 2 右侧 | 地理分布可视化 |

### 5.3 风险体系调整

保留 v3 的 8 大风险维度，但调整展示方式：
- Tab 1：风险概览卡片（4 个核心风险指标）
- Tab 5：预警中心（完整的风险列表 + 仪表盘）
