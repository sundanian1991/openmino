## Q2 四件事

### 一、供应商绩效评价体系（P0 — 底层依赖）

**本质**：给每家供应商打一个"综合分"，用分数决定资源投向。

#### 5 个关键问题

| 问题 | 选项建议 |
|------|----------|
| **权重怎么定？** | 业绩完成（40%）> 规模（15%）= 稳定性（15%）> 响应速度（10%）= 配合度（10%）> 资源投入（5%）= 行为指标（5%）。业绩是硬指标，占大头；规模和稳定性是底盘；其余是软性加分 |
| **数据从哪来？** | 系统数据（业绩、通话量、异议处理率）+ 巡检记录（合规、职场状态）+ 日常反馈（业管、质检）+ 供应商自评。系统数据客观，日常反馈主观，两者结合 |
| **谁来评？** | 定框架 + 业管提供日常数据 + 质检提供合规数据。季度评一次，月度跟踪 |
| **多久评一次？** | 月度看趋势、季度定级、年度总调。季度评级是关键动作 |
| **评分标准怎么量化？** | 每个维度 1-5 分，配具体描述。比如"响应速度"：1 分 = 48h 无响应；3 分 = 24h 内响应；5 分 = 2h 内响应并给出方案 |

#### 评级结果怎么用

| 评级 | 资源策略 | 管理策略 |
|------|----------|----------|
| **S 级**（3家） | 优先分配业务量、优先续约、开放新产线试点 | 高层定期沟通，战略合作 |
| **A 级**（5家） | 正常业务量，有增量机会优先 | 月度复盘，重点培养 |
| **B 级**（~20家） | 基础业务量，不额外投入 | 季度检查，达标的可升A |
| **C 级** | 逐步缩减业务量，不再新增 | 列入观察，连续两季度C则启动清退评估 |

#### 与已有方案的关系

4月13日已做 `供应商ABC分层分级方案`，采用 60:30:10 打分模型：
- 业务督导 60%（产能达成率、人均产出、交付时效率、任务完成率、业务增长率）
- 供应商管理 30%（响应速度、配合度、团队稳定性、整改执行力、管理规范性）
- 质检 10%（质检合格率、客户投诉率、红线事件）

**需要升级的方向**：
- 从 ABC 三级扩展到 SABC 四级（增加 S 级战略合作层）
- 权重调整：从 60:30:10 调整为 7维度新权重
- 增加规模、资源投入力度等新维度
- 评级结果与资源分配策略挂钩

### 二、供应商储备库建设（P3 — 依赖评价体系）

**本质**：建一个"备胎池"，缺供应商时能快速补位。

#### 三步拆解

**Step 1：信息收集（5月）**

| 渠道 | 具体动作 |
|------|----------|
| 行业展会/论坛 | 收集参展商名单，重点关注有金融BPO经验的 |
| 同行推荐 | 通过联盟成员互相推荐 |
| BD主动开拓 | 列出目标区域，定向摸排 |
| 供应商自荐 | 通过现有供应商网络扩散需求 |
| BPO联盟 | 联盟成员推荐优质供应商 |

**Step 2：标准筛选（5-6月）**

用评价体系的 7 个维度做初筛，设最低门槛：
- 规模：≥30 人
- 存续：≥2 年
- 经验：有金融/电销相关案例
- 合规：无重大违规记录

预计：行业 1000 家 → 符合标准 ~100 家 → 进入储备库 30-50 家

**Step 3：储备库管理（持续）**

| 状态 | 含义 | 动作 |
|------|------|------|
| **待评估** | 刚进入储备库 | 电话访谈 + 资质审核 |
| **已评估** | 信息完整 | 每季度更新一次信息 |
| **已引入** | 正式合作 | 转入日常评价体系 |

### 三、BPO联盟持续推进（P2 — 相对独立）

**本质**：建立联盟运作机制，体现管理价值。不是自己做联盟，是建机制让联盟自己转。

#### Q2 关键动作

| 动作 | 说明 |
|------|------|
| **毅航牵头** | Q1 未参与，Q2 转为牵头角色。明确牵头的具体内容 |
| **汇讯牵头** | 同上，需要明确汇讯的分工 |
| **效果跟踪** | 量化联盟产出：解决了什么问题？达成了什么共识？ |
| **期望管理** | 定期收集联盟成员期望，避免"开会热闹、会后没下文" |

#### 需要产出的不是"联盟成果"，而是"联盟运作机制"

| 产出物 | 内容 |
|--------|------|
| **议题库** | 联盟要解决哪些问题？（3-5个核心议题） |
| **分工表** | 每个议题谁牵头、谁配合、交付什么 |
| **节奏表** | 多久开一次会、会前准备什么、会后跟进什么 |
| **成果标准** | 每个议题的交付物是什么 |

#### 建议的价值主张

| 价值主张 | 具体产出 |
|----------|----------|
| **资源共享** | 联盟成员之间的人力互补机制 |
| **标准共建** | 联合制定行业标准，形成行业话语权 |
| **联合采购** | 联盟成员联合采购培训资源、系统工具 |
| **信息互通** | 共享行业趋势、政策变化、竞品动态 |

建议从"资源共享"切入——最实际、最容易见效。

### 四、制度细化到场景（P1 — 依赖分层结果）

**本质**：把去年写的"大框架"变成"用的时候能直接查到"的操作手册。

#### 典型场景举例

| 场景 | 需要细化的内容 |
|------|----------------|
| **供应商人力不足** | 触发条件 → 响应流程 → 补人时效 → 违约处理 |
| **业绩连续下滑** | 预警线 → 沟通机制 → 改善计划 → 退出机制 |
| **职场巡检发现问题** | 问题分级 → 整改时限 → 复检机制 → 扣分规则 |
| **新供应商准入** | 申请流程 → 评估标准 → 试用期规则 → 正式合作条件 |
| **供应商主动退出** | 通知期 → 业务交接 → 人员安置 → 保证金处理 |

#### 核心：建立触发-响应闭环

| 环节 | 内容 | 角色 |
|------|------|------|
| **触发** | 什么数据到什么值触发预警？ | 定义阈值 |
| **响应** | 触发后谁做什么？什么时限？ | 定义流程 |
| **跟踪** | 执行情况怎么记录？ | 定义工具 |
| **复盘** | 闭环后怎么评估效果？ | 定义标准 |

#### 管理动作日历

| 周期 | 管理动作 | 责任人 | 产出物 | 联动对象 |
|------|----------|--------|--------|----------|
| **日** | 业绩数据看板更新 | 年老师 | 数据报表 | 业管 |
| **周** | 异常供应商沟通记录 | 年老师 | 沟通纪要 | 业管 |
| **月** | 供应商月度评分 | 年老师+业管 | 评分表 | 业管、质检 |
| **季度** | 供应商分级调整 | 年老师 | 分级报告 | 领导 |
| **季度** | 联盟进展汇报 | 年老师 | 联盟简报 | 领导 |

## 四件事的逻辑关系

```
评价体系（P0 — 基础）
    ├── 现有 34 家 → 评级 → 资源分配策略
    └── 储备库建设（P3）→ 用同一套标准筛选新供应商

制度细化（P1 — 保障）
    └── 把评级结果落到管理制度里（C 级怎么处理、S 级怎么激励）

BPO 联盟（P2 — 生态）
    └── 联盟成员的推荐也纳入储备库来源
```

**一句话**：建一个供应商管理的"操作系统"——分层评价体系 + 场景化管理制度 + 联盟运作机制。

## 导师视角补充

### 核心盲点

1. **缺少供应商视角**：分层管理不能只是单方面管控，需要透明化——把评分标准公开给供应商，让它们知道自己的得分和改进方向
2. **联盟缺核心价值主张**：机制只是手段，需要给联盟一个硬价值（资源共享最实际）
3. **制度细化不是写文档**：核心是触发-响应闭环，让管理动作不依赖记忆和判断
4. **联动业管不是共享看板**：核心是让业管觉得这套体系对他们也有用

### 认知模式检验

| 认知模式 | 检验问题 | 是否回答 |
|----------|----------|----------|
| 逆向反射 | 什么会导致分层管理失败？ | 需补充：供应商不认可、数据不准确、执行不到位 |
| 杠杆执念 | 三件事中哪件事投入产出比最高？ | 分层管理（有数据基础、见效快） |
| 聚焦即减法 | 有没有可以不做的事？ | 联盟可以先不"做大"，先"做实" |
| KPI怀疑论 | 分层标准会不会被操纵？ | 需交叉验证（数据+巡检+反馈） |

### 汇报故事线

> "我搭建了供应商管理操作系统 1.0，实现了从人盯人到体系管人的转变。分层评价覆盖X家供应商，Y个场景建立了触发-响应闭环，联盟Z个议题启动推进。"

## 第三轮讨论：坐席 SABC → 供应商分层

### 坐席 SABC 评级逻辑（已有数据）

- 首贷和复贷分池独立评级
- 参评条件：活跃度（7天内有外呼）+ 出勤量（≥22天）
- 分级：S=Top20%+底线，A/B/C按排名分布
- 首贷维度：ATT + 转化率 + 拨打量
- 复贷维度：ATT + GMV + 件均 + 拨打量

### 关键发现

| 供应商 | 首贷C占比 | 复贷C占比 | 信号 |
|--------|-----------|-----------|------|
| 伽玛 | 36.4% | 18.9% | 首贷管理有问题，复贷还行 |
| 赛维斯 | 34.1% | 24.1% | 两条线都偏高 |
| 翰锐曲靖 | 40.0% | 24.0% | 首贷问题严重 |
| 毅航合肥 | 20.6% | 3.4% | 体量大但占比正常 |

### 坐席 C 级 → 供应商管理动作

| C 级坐席占比 | 动作 | 责任人 |
|--------------|------|--------|
| ≤20%（正常分布） | 供应商自己管，你只看结果 | 供应商 |
| >30%（异常） | 要求供应商提交改善计划 | 你跟踪 |
| >40%（严重） | 约谈供应商负责人，限期整改 | 你介入 |

### 供应商分层逻辑（坐席 SABC → 供应商 SABC）

| 维度 | 权重 | 数据来源 |
|------|------|----------|
| 坐席 S+A 占比 | 40% | 坐席 SABC 评级表 |
| 坐席 C 占比 | 30% | 坐席 SABC 评级表 |
| 参评率 | 15% | 坐席 SABC 评级表 |
| 流失率 | 15% | 司龄统计表 |
| 响应速度/配合度 | 额外加减分 | 供应商管理评分 |

**好处**：坐席数据客观、系统可查，不依赖主观判断，供应商能看到自己的坐席分布。

## 开放性问题

1. S/A/B/C 四级 vs ABC 三级？四级更精细但管理成本更高
2. 权重调整：60:30:10（旧）vs 坐席分布推导（新）？坐席数据更客观
3. 联盟的核心价值主张选哪个？资源共享 / 标准共建 / 联合采购 / 信息互通
4. 分层结果要不要公开给供应商？透明化是双刃剑
5. 首贷和复贷是否需要分开评级供应商？（伽玛首贷C=36%但复贷C=19%，差异大）

## 关键约束

- Q2 内需要出可量化成果
- 涉及外部协调的事不可控，需要降低预期
- 已有资产可复用：4月13日方案、BPO联盟培训总结
- 业管联动需要对方配合，先从共享数据看板切入
