# 洞察1: 「数据叙事」写作框架

**类型**: playbook

## 核心洞察

电销复盘文档的核心写作方法是「数据叙事」—— 用数字讲故事，而非用文字描述。每个关键结论都有至少3个维度的数据支撑。

---

## 框架结构

### 1. 数据密度公式

> **每个核心结论 = 1个主数据 + 1个对比数据 + 1个占比数据**

**示例**（原文）:
> "金条首贷用户数40万，目标达成率111.8%，同比+22%，大盘占比16.1%（25年17.1%）"

拆解:
- 主数据: 用户数40万
- 对比数据: 目标达成率111.8%、同比+22%
- 占比数据: 大盘占比16.1%（对比去年17.1%）

### 2. 对比维度清单

| 对比类型 | 用途 | 示例 |
|---------|------|------|
| **目标达成** | 展示完成度 | 目标达成率111.8% |
| **同比** | 展示增长趋势 | 同比+22% |
| **环比** | 展示短期变化 | 环比+55% |
| **大盘占比** | 展示相对地位 | 大盘占比16.1% |
| **历史对比** | 展示长期趋势 | （25年17.1%）↓ |
| **VS对照组** | 展示实验效果 | 实验组VS对照组提升80% |

### 3. 数据呈现层次

```
第一层: 绝对值（用户数40万）
    ↓
第二层: 达成度（目标达成率111.8%）
    ↓
第三层: 变化趋势（同比+22%，大盘占比16.1%→17.1%）
```

## 应用指南

### 改写示例: 供应商月度通报

**原文（假设）**:
> 毅航本月业绩表现良好，完成既定目标，团队整体稳定。

**改写后**:
> 毅航本月首贷名产1850元，目标达成率118%，同比+15%，占大盘产能23.5%（上月21.3%）↑；团队稳定在280人，新人达标率85%，较上月提升5个百分点。

### 关键检查清单

- [ ] 每个核心结论有3个维度的数据
- [ ] 至少包含1个对比数据（同比/环比/VS）
- [ ] 占比数据标注变化趋势（↑↓）
- [ ] 数据单位统一、精确

## 适用边界

| 适用场景 | 不适用场景 |
|---------|-----------|
| 业绩汇报 | 战略方向讨论 |
| 数据复盘 | 创新性探索 |
| 进展同步 | 定性分析 |
