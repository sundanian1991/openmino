**创建时间**：2026-04-04
**目标**：在每周20次配额限制下，生成东京猫/李诞虾级别的SVG

## 核心约束

**API限制**：
- 每周20次请求（硬约束）
- 每次1 credit（包括n>1的情况）
- Rate limit: 20 requests/60 seconds

**质量目标**：
- 参考标准：东京猫（精细线条~50个path元素）、李诞虾（双层技法+陶土色点缀）
- 必须承载具体内容（真实人物、部门、问题），不是抽象模板

## API核心能力分析

### 已知能力（从文档学习）

**模型**：`arrow-preview` — Quiver旗舰SVG生成模型

**输入**：
- `prompt`（必需） — 主要文本描述
- `instructions`（可选） — 风格/格式指导
- `references`（可选） — 参考图数组（最多4张）
- `model` — "arrow-preview"
- `n` — 生成数量（1-16）

**采样参数**：
- `temperature` (0-2，默认1) — 越低越确定
- `top_p` (0-1，默认1) — nucleus sampling
- `presence_penalty` (-2到2，默认0) — 惩罚重复

**输出**：
- SVG代码（直接可用）
- 支持streaming（Server-Sent Events）

## 高质量Prompt的构成要素

### 从东京猫/李诞虾反推

**东京猫特征**：
- ~50个path元素（高精细度）
- `stroke-linecap="round"` `stroke-linejoin="round"`（手绘感）
- 双层：填充层+线条层
- 色彩：#BA3420（陶土）、#0B0800（墨黑）、#F7F3ED（米白）

**李诞虾特征**：
- 双层技法更明显（fill layer + ink line layer）
- 纸张纹理（feTurbulence filter）
- 渐变色彩深度（不同透明度）

### Prompt结构假设

```
prompt = [主体描述] + [风格描述] + [构图说明] + [细节要求]

示例：
"Japanese organizational chart showing power structure,
策略组 on left with pricing authority, 服务组 on right carrying risk,
供应商 in middle receiving conflicting signals.
Hand-drawn ink style with vermillion accents,
双层技法: fill layer for color weight + ink layer for fine outlines.
Use #BA3420 for accents, #0B0800 for lines, #F7F3ED for background.
约50个path elements, stroke-linecap round."
```

### instructions的作用

**推测**：`instructions`是对风格的技术指导，与`prompt`的内容描述分离

**假设结构**：
```
instructions = "色彩规范 + 线条技法 + 构图原则 + 输出要求"
```

## 一次性生成 vs 批量生成

### 场景分析

**一次性生成（n=1）**：
- 优点：精细控制、可迭代、节省配额
- 缺点：需要多次尝试才能得到好结果
- 适用：复杂场景、需要精细调整

**批量生成（n>1）**：
- 优点：一次得到多个变体、可选择最好的
- 缺点：消耗配额快（n个=1 credit，但只能选1个）
- 适用：简单场景、探索性尝试

### 配额策略

**每周20次 = 约3次/天**

**建议分配**：
- 周一：3次探索性测试（n=3，快速看方向）
- 周二-周四：每天2次精细生成（n=1，迭代优化）
- 周五：3次最终版本（n=1-2，确保质量）
- 预留：4次应急/重试

**关键原则**：
- 每次调用前必须充分准备（prompt + references）
- 失败立即分析原因，不要盲目重试
- 记录每次参数和结果，形成知识库

## 权责不对等研究内容的具体方案

### 需要可视化的核心场景

1. **结构性权责分离**
   - 策略组（刘伟佳）vs 服务组（王易人 → 年老师）
   - 定价权 vs 风险承担
   - 供应商夹层

2. **供应商夹层困境**
   - 矛盾信号：质量↑ + 价格↓
   - 后果：流失/不投入/质量下降

3. **刘伟佳责任边界管理**
   - 分析外化 + 模糊化处理
   - 风险转移给服务组

4. **REACT回应框架**
   - 5步回应流程
   - 具体对话示例

5. **联盟可行性评估**
   - 三种联盟类型（正式/议题/信息同步）
   - 可行性/风险/建议

### 每个场景的生成策略

**场景1-3**：复杂场景，用n=1精细生成
**场景4**：流程图，可能用n=2得到变体
**场景5**：对比图，用n=1确保一致性

### 统一风格要求

**色彩系统**：
- 陶土色：#BA3420（重点）、#E2725B（辅助）
- 米白：#F7F3ED（背景）、#F5F1EE（深色）
- 墨黑：#0B0800（线条）、#111111（文字）
- 灰度：#D4CFC7、#A8A098

**线条技法**：
- stroke-linecap: round
- stroke-linejoin: round
- 双层：fill layer + ink layer
- 主线：1.5-2px，细节：0.8-1.2px

**构图原则**：
- 信息密度高但不拥挤
- 左右对称或中心辐射
- 用色彩引导视线
- 标注清晰（中文支持？）
