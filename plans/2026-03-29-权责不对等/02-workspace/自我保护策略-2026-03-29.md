# 自我保护策略：在权责不对等下不背锅

> **基于**: 边界识别清单 + 权责不对等研究（2026-03-29）
> **目的**: 在结构性权责不对等的环境下，保护自己不承担不应承担的责任

---

## 核心原则

**不揽责，留证据，明边界。**

权责不对等的核心风险：决策权在策略组，但出问题时服务组容易被指责"管理不到位"。

---

## 风险场景识别

| 场景 | 风险 | 谁的实际责任 | 你容易被指责为 |
|------|------|-------------|---------------|
| 供应商流失 | 策略组压价导致 | 定价决策 | 没做好关系维护 |
| 产能下降 | 供应商不愿投入 | 激励机制 | 没推动能力提升 |
| 质量问题 | 培训资源不足 | 资源分配 | 没做好质检培训 |
| 费效比差 | 结算价格高 | 定价策略 | 没管理好供应商 |

**共同特征**: 策略组的决策（定价、名单、数据分配）导致的问题，容易被归因为服务组的"管理问题"。

---

## 保护策略三层框架

### 第一层：预防（在问题发生前）

#### 1.1 明确责任边界

**做法**: 在周报、沟通中明确区分"我能做的"和"需要策略组配合的"

**表达模板**:
```
✅ 好的表达：
"本月供应商流失X家，主要原因是结算价格调整后供应商ROI不匹配。
我这边已做的工作：逐家沟通、提供优化建议、协商补充激励。
需要策略组配合：重新评估X家供应商的结算价格。"

❌ 避免的表达：
"供应商流失严重，我正在努力沟通争取挽留。"
```

**关键**: 让阅读者看到"你在尽力" + "问题根源在策略组决策"

---

#### 1.2 建立记录习惯

**需要记录的内容**:
- 策略组的关键决策（定价、名单调整、资源分配）
- 你的建议和策略组的反馈
- 供应商的反馈（尤其是对定价、政策的意见）
- 你采取的行动和结果

**记录位置**:
- 周报中"问题与建议"部分
- 供应商沟通邮件（CC相关方）
- 关键会议的纪要或总结

**记录模板**:
```markdown
## 本月供应商流失分析

流失原因：
1. 毅航：结算价格从X降至Y，ROI不匹配（策略组03-15决策）
   - 我的行动：沟通2次，提供优化方案
   - 供应商反馈：价格无法接受
   - 策略组反馈：（记录原话或无反馈）

2. 汇讯：...
```

---

#### 1.3 向供应商"说明情况"

**目的**: 让供应商知道"某些事不是我决定的"，降低他们对你个人的不满

**做法**:
- 策略组决策后，及时向供应商说明
- 用客观语气："这是公司的统一决策"
- 不抱怨策略组，但也不把责任揽到自己身上

**表达模板**:
```
"关于结算价格调整，这是策略组基于费效比目标做出的决策。
我理解这会影响大家的ROI，已经把大家的反馈整理后提交给策略组。
后续如有新的政策调整，我会第一时间同步给大家。"
```

**关键**: 既不当策略组的"传声筒"（避免被解读为"你同意这个决定"），也不当"抱怨者"（避免被解读为"你对抗组织"）。

---

### 第二层：应对（在问题发生时）

#### 2.1 被追问时的回应框架

**场景**: 上级或同事问"为什么供应商流失/产能下降？"

**回应框架（REACT）**:

| 步骤 | 内容 | 目的 |
|------|------|------|
| **R**espond | 承认问题存在 | 不回避，建立对话基础 |
| **E**xplain | 说明根本原因（策略组决策） | 归因正确 |
| **A**ction | 你已采取的行动 | 展示尽力 |
| **C**onstraint | 当前约束（无法单边解决） | 说明边界 |
| **T**arget | 建议的下一步 | 展示思考 |

**示例**:
```
Q: 为什么毅航这月流失严重？

A: 确实，毅航本月流失X人，是一个需要关注的问题。（R）
   根本原因是3月结算价格调整后，毅航的ROI从1.2降到0.8，已经低于他们可接受水平。（E）
   我这边已采取的行动：沟通3次、提供优化方案、协商阶段性激励，但对方反馈需要价格回调才愿意继续。（A）
   当前约束：定价权在策略组，我无法单边调整价格。（C）
   建议：请策略组重新评估毅航的定价，或允许我在增量部分给予弹性空间。（T）
```

---

#### 2.2 被指责时的防御

**场景**: 有人暗示或明说"你管理不到位"

**防御原则**:
1. **不反击**: 不说"这是策略组的问题，你去找他们"
2. **摆事实**: 用数据说明根本原因
3. **明边界**: 说明你能控制的范围

**表达模板**:
```
"您提到的管理问题我理解。让我补充一些信息：

从数据看，供应商流失的主要驱动因素是结算价格（相关性0.8），
而非管理动作（相关性0.2）。

我能控制的是：沟通频次、培训资源、问题响应速度。
这些指标本月都有改善（具体数据...）。

但定价策略、名单分配不在我的控制范围内。
如果这些不调整，管理动作的效果会受限。

您觉得还有什么我可以在可控范围内改进的？"
```

**关键**: 承认你能改进的部分，但不承担不属于你的责任。

---

#### 2.3 周报中的责任边界管理

**原则**: 让读者看到"问题根源在策略组"，但不用指责语气

**写法对比**:

| ❌ 避免 | ✅ 推荐 |
|--------|--------|
| "策略组定价太高，导致供应商流失" | "结算价格调整后，供应商ROI不匹配，流失增加" |
| "刘伟佳不配合" | "需要策略组在XX方面给予配合" |
| "这问题我解决不了" | "在当前框架下，建议..." |

**示例**:
```markdown
## 本月挑战

1. 供应商流失加剧（-X人）
   - **驱动因素**: 结算价格调整后，头部供应商ROI下降
   - **我的应对**: 逐家沟通，提供优化方案，协商补充激励
   - **当前约束**: 定价调整需要策略组审批
   - **建议**: 请策略组重新评估X家供应商的定价模型

2. 产能提升乏力
   - **驱动因素**: 供应商培训投入不足（ROI低导致不愿投入）
   - **我的应对**: 组织X场培训，提供数据使用指导
   - **当前约束**: 培训资源分配权在策略组
   - **建议**: ...
```

### 第三层：记录（在事后保护自己）

#### 3.1 建立"决策追溯"文档

**目的**: 当未来出现问题时，能证明"当时我就指出了风险"

**内容**:
| 日期 | 策略组决策 | 你的建议 | 结果 |
|------|-----------|---------|------|
| 03-15 | 毅航结算价降至Y | 建议：保持X价格，先观察 | 毅航04月流失 |
| 03-20 | ... | ... | ... |

**存储位置**: 个人工作区、周报、或专门文档

---

#### 3.2 重要邮件CC策略

**原则**: 关键沟通要留痕，但不要过度CC（会被解读为"不信任"）

**需要CC的场景**:
- 策略组做出重大决策后的沟通
- 供应商的强烈反馈
- 你提出的重要建议

**CC对象**: 军哥（必要时）、王易人（如果是共同议题）

**邮件模板**:
```
主题：关于XX供应商结算价格调整的反馈

收件人：策略组相关人
CC：军哥（如需要）、王易人（如涉及）

佳姐，

关于3月15日XX供应商结算价格调整，供应商反馈如下：
[具体反馈，最好是原文]

我已与供应商沟通X次，提出[方案]，但对方反馈[...]。

建议：[你的具体建议]

年老师
```

## 红线（不要踩）

| 红线行为 | 原因 |
|---------|------|
| 公开说"这是策略组的问题" | 会被解读为"推卸责任、对抗组织" |
| 在周报中指责刘伟佳 | 职场大忌，会树敌 |
| 对供应商抱怨策略组 | 可能被传回，加剧矛盾 |
| 过度CC（每封邮件都CC军哥） | 会被认为"不成熟、不信任组织" |
| 消极应对（"这问题我解决不了"） | 会被认为"不作为" |

---

## 实战检查清单

### 周报写完时检查

- [ ] 问题原因是否明确指向策略组决策（不用指责语气）？
- [ ] 我的行动是否清晰列出？
- [ ] 是否说明了"我能控制的"和"需要配合的"？
- [ ] 建议是否具体（不是空泛的"建议策略组配合"）？

### 被追问时检查

- [ ] 我是否先承认问题存在（不回避）？
- [ ] 我是否用数据说明了根本原因？
- [ ] 我是否列出了已采取的行动？
- [ ] 我是否明确了边界（什么是我不能控制的）？
- [ ] 我是否提出了建设性的下一步？

### 重要沟通后检查

- [ ] 是否有邮件/文档记录？
- [ ] 是否在需要时CC了相关方？
- [ ] 是否在个人工作区留存了备份？

---

## 心态建设

**重要认知**:
1. 这不是"对抗策略组"，是"明确责任边界"
2. 你不是在推卸责任，是在让责任回归正确位置
3. 刘伟佳的行为模式（责任边界管理）你也可以用
4. 保护自己≠不作为，你仍然要在可控范围内做到极致
