# 场景1：范围模糊 — "不知道从哪下手"

**触发信号**：领导说"你看看这个事"、新课题、没做过的方向。
**叙事结构**：五步叙事（基本盘 → 方法 → 评估 → 聚焦 → 打法）
**一句话模板**：

> "我分析的是[问题边界]。因为[原因]，我选择从[维度]切入。交叉评估后，全景是这样的[矩阵]。重点聚焦[区域]，因为[理由]。下一步打法是[行动]。"

**实战案例**：AI化分析——按工作内容拆模块，AI潜力×工具成熟度九宫格，聚焦高潜高成熟区域。

---

## 步骤详解

### 步骤1：锁定问题

| 要素 | 说明 |
|------|------|
| **动作** | 一句话说清"我在分析什么" |
| **产出** | 问题边界声明 |
| **工具** | 边界声明法 |
| **自检** | 别人听完能复述吗？ |

**引导话术**：
> "先确认一下，你现在要分析的核心问题是什么？用一句话说清楚。"

---

### 步骤2：选维度

| 要素 | 说明 |
|------|------|
| **动作** | 用什么角度切这个问题 |
| **产出** | 拆解维度 + 选这个维度的理由 |
| **工具** | Issue Tree（MECE拆解）、结构化思维技能 |
| **自检** | 为什么不是别的维度？ |

**引导话术**：
> "面对这么复杂的局面，你打算从哪个角度切入？为什么选这个角度？有没有考虑过其他切法？"

---

### 步骤3：全景扫描

| 要素 | 说明 |
|------|------|
| **动作** | 每个切面长什么样 |
| **产出** | 二维矩阵（九宫格/四象限） |
| **工具** | 二维矩阵（维度选择+格子填充） |
| **自检** | 每个格子有判断依据吗？ |

**引导话术**：
> "用你选的这两个维度交叉一下，全景是什么样的？每个区域有什么特征？"

---

### 步骤4：聚焦

| 要素 | 说明 |
|------|------|
| **动作** | 哪些区域值得深挖 |
| **产出** | 聚焦区域 + 排除理由 |
| **工具** | 聚焦标准（为什么是这个区域不是那个） |
| **自检** | 排除的部分为什么排除？ |

**引导话术**：
> "不可能全做，你的判断是哪些区域值得投入？哪些先放一放？为什么？"

---

### 步骤5：打法

| 要素 | 说明 |
|------|------|
| **动作** | 针对聚焦区域的具体行动 |
| **产出** | 打法/行动方案 |
| **自检** | 行动和聚焦区域对得上吗？ |

**引导话术**：
> "聚焦之后，下一步怎么打？针对你选的区域，具体做什么？"
