# 场景4：因果模糊 — "出了问题，不知道为什么"

**触发信号**：数据异常、业绩波动、投诉上升、指标突然变化。
**叙事结构**：侦探叙事——案情→嫌疑人→证据链→锁定→行动
**一句话模板**：

> "[时间范围]，[指标]发生了[变化]。可能原因有[N]个，排查后排除[M]个（[排除依据]），锁定根因是[具体原因]。对策是[行动]，预计[时间]见效。"

**实战举例**：金条产线产能掉了15%——排查人员/系统/流程/需求侧/供应商侧，排除后锁定A供应商2号线停检未报备。

---

## 步骤详解

### 步骤1：锚定现象

| 要素 | 说明 |
|------|------|
| **动作** | 具体发生了什么（数字+时间） |
| **产出** | 现象描述 |
| **工具** | 数据锚定（具体数字+时间范围） |
| **自检** | 数字和时间准确吗？ |

**引导话术**：
> "具体发生了什么？哪个指标、什么时间、掉了多少？用数字说话。"

---

### 步骤2：列嫌疑

| 要素 | 说明 |
|------|------|
| **动作** | 所有可能的原因 |
| **产出** | 嫌疑清单（5-8个） |
| **工具** | 鱼骨图 / Issue Tree |
| **自检** | 有没有遗漏常见原因？ |

**引导话术**：
> "可能导致这个结果的原因有哪些？全部列出来，不要急着排除。"

---

### 步骤3：排查排除

| 要素 | 说明 |
|------|------|
| **动作** | 用证据逐一排除 |
| **产出** | 排除记录 |
| **工具** | 5-Why（逐层追问）/ 证据排除法 |
| **自检** | 排除的证据够硬吗？ |

**引导话术**：
> "逐一看：哪些能排除？排除的证据是什么？剩下的可能原因是什么？"

---

### 步骤4：锁定根因

| 要素 | 说明 |
|------|------|
| **动作** | 剩下的最可能原因 |
| **产出** | 根因（1-2个） |
| **工具** | 假设驱动（先猜后验证） |
| **自检** | 能解释现象的全部吗？ |

**引导话术**：
> "剩下的原因里，最可能的是哪个？能解释全部现象吗？"

---

### 步骤5：对策

| 要素 | 说明 |
|------|------|
| **动作** | 怎么修+怎么防 |
| **产出** | 行动方案 |
| **工具** | 行动方案（修复+预防+验证） |
| **自检** | 修了之后怎么验证有效？ |

**引导话术**：
> "找到根因之后，怎么修？怎么防？怎么验证修好了？"
