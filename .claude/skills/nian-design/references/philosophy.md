# Nian Design Philosophy

> 完整设计哲学、7 色体系、字体规则、间距系统、反模式、内容类型。
> 首次使用或需要刷新设计直觉时读取。日常使用只需知道 SKILL.md 中的 5 行摘要。

---

## 1. DESIGN PHILOSOPHY — 年 相信什么

**Landscape and signal.**
页面是一片风景，用户的目光应该被引导到信号上——不多，不杂，不吵。

**结构即是装饰。**
暴露网格、层级、数据本身。不靠阴影、渐变、毛玻璃制造深度——深度来自字号对比、间距、底色交替。

**7 色 Brand DNA 是身份。**
darkgray / olive / earth + yellow / orange + glacier / rock —— 不是随便选的配色，是身份的延伸。颜色传达情绪，情绪服务于内容。

**三层级，不多不少。**
每个 section 恰好三层：结论(Answer) → 支撑(Argument) → 证据(Evidence)。两层太平，四层太乱。

**8:1 是底线。**
Hero 字号 ÷ 正文 ≥ 8。达不到的不算 Hero。

**一个打破。**
每页恰好一处不守规则的地方——一个 120px 圆形、一段旋转虚线、一个出血图片。只有一处，才叫设计。多了叫失控。

## 2. CRAFT RULES — 怎么搭

### 2.1 三 层 金 字 塔

每个 section 包含且仅包含三个层次：

| 层级 | 字体 | 字号 | 角色 |
|------|------|:----:|------|
| **Answer** | Playfair Display | 96-120px | 唯一结论 |
| **Argument** | Inter | 12-14px | 支撑上下文 |
| **Evidence** | JetBrains Mono | 10-12px | 元数据、来源、推到边缘 |

眯眼测试：闭上一只眼，能否一眼看出最重要的东西？如果两个元素在竞争，其中一个需要缩小、褪色或移动。

### 2.2 7 色 Brand DNA

自然低饱和主色传递户外属性，高饱和亮色强化功能性识别。7 色体系，比例固定。

**使用比例：Primary 80% + Accent 10% + Scene 10%**

#### Primary — 自然色系（80%）

| Token | 色值 | 占比 | 用途 |
|-------|:----:|:----:|------|
| `--darkgray` | `#2C2C2C` | 45% | 品牌主视觉底色、正文、标识 |
| `--olive` | `#4A5D3A` | 20% | 产品主色、户外属性传递 |
| `--earth` | `#8B7355` | 15% | 产品主色、自然质感 |

#### Accent — 功能性警示色（10%）

| Token | 色值 | 占比 | 用途 |
|-------|:----:|:----:|------|
| `--yellow` | `#FFD100` | 5% | 功能性识别、安全警示、CTA |
| `--orange` | `#E55B2B` | 5% | 功能性识别、视觉焦点、高亮 |

#### Scene — 场景化色彩（10%）

| Token | 色值 | 占比 | 用途 |
|-------|:----:|:----:|------|
| `--glacier` | `#2A4A5A` | 5% | 冬季/高海拔场景、冷调氛围 |
| `--rock` | `#808080` | 5% | 攀岩/山地场景、中性过渡 |

#### Surface & Text

精确色值见 `references/tokens.md` §2 COLOR SYSTEM。简要说明：

| Token | 角色 |
|-------|------|
| `--bg` / `--surface` / `--surface-raised` / `--border` | 表面层级（从底到高） |
| `--text-display` | Hero 数字、品牌宣言，每屏唯一 |
| `--text-primary` | 正文、主要内容 |
| `--text-secondary` | 标签、说明、元信息 |
| `--text-disabled` | 禁用态、时间戳、提示 |

**颜色纪律**：
- Accent 不做装饰——Accent 只用于打断和聚焦
- 灰度先于颜色——4 级灰度建立层级，颜色只标记需注意的信息
- 从不使用 opacity/梯度来区分数据——用层级、间距、底色
- 圆角 ≤ 8px（pill 按钮 999px 除外）
- 无阴影、无渐变、无毛玻璃
- 颜色用在值上，不在标签行或背景上着色
- **80/10/10 原则**：Primary 80% + Accent 10% + Scene 10%，不越界

### 2.3 字 体 四 工

| 角色 | 字体 | 用途 | 场景 |
|------|------|------|------|
| Display | Playfair Display | 标题、Hero、宣言 | 主 Display，戏剧性时刻 |
| Display Light | Georgia | 小字号标题、副标题、quote | 轻量场景，系统字体零加载 |
| 装饰 | Doto | 大数字、Hero 时刻 | dot-matrix、装饰数字 |
| Body | Inter | 正文、标签 | 默认正文字体 |
| Data | JetBrains Mono | 数字、代码、来源 | 数据、标签、元信息 |

**字体选择规则**：
- Hero 标题（≥48px）→ Playfair Display
- 小标题（< 48px）→ Georgia（系统字体，x-height 更高，更清晰）
- 装饰数字 → Doto（点阵效果）
- 正文 → Inter
- 数据/标签 → JetBrains Mono

**纪律**：每页最多 2 个字重、3 个字號。想加新字号？先问能不能用间距解决。

### 2.4 间 距 即 含 义

| 间距 | 含义 |
|------|------|
| 4-8px | 这两个是一起的（图标+标签、数字+单位） |
| 16-24px | 同组不同项（列表项、表单字段） |
| 32-48px | 新组开始（section 分隔） |
| 64-96px | 新上下文（Hero → 内容、大块分隔） |

**如果需要分割线，间距可能错了。** 分割线是用间距解决不了的替代方案。只在数据密集列表中使用。

### 2.5 容 器 策 略（从轻到重）

1. 间距本身（就近分组）
2. 一条分割线
3. 边框 outline
4. 表面卡片（底色变化）

**越重要的元素越不要框住它**——让它浮在背景上。

### 2.6 Hero 4 维 联 锁

Hero = 底色 + 切割 + 布局 + 信息层。6 种核心切割：

| 编号 | 名称 | CSS 关键代码 |
|:----:|------|-------------|
| V1 | Diagonal 对角线 | `clip-path: polygon(25% 0, 100% 0, 100% 100%, 0% 100%)` |
| V2 | Split 中心劈开 | `grid-template-columns: 1fr 1fr` |
| V3 | Horizontal 上下劈开 | `grid-template-rows: 1fr 1fr` |
| V4 | Quadrant 四分之一 | grid + 右上深色方块 |
| V5 | Bevel 左斜切 | `clip-path: polygon(40% 0, 100% 0, 100% 100%, 0% 100%)` |
| V6 | Watermark 水印 | 居中 + 超大透明 ghost 字衬底 |

Hero CSS 代码从 `references/showcase/H/H061-组件展示-Hero品质标杆-hero.html` 取。

**联锁规则**：
- 改 V1 → 布局左对齐（对角切割右侧重）
- 改 V6 → 布局居中（水印对称）
- 底色深 → 信息层建议陈述型（深色底+密集数据难读）
- 底色图案底 → 切割建议 V6（再加切割过度了）

### 2.7 打 破 规 则

每页正好 **1 处** 打破：
- 一个 120px 圆形元素
- 一段旋转虚线圆环
- 一个出血图片
- 一个超大装饰数字
- **一个 ghost 水印**（优先选择，最有效的空间感来源）

打破不超过 1 处。0 处 = 平庸。≥2 处 = 失控。

**Ghost 水印规范**（详见 `references/tokens.md` §10）：
- 字号：`clamp(80px, 15vw, 280px)`
- opacity：0.03-0.06（比 Haglöfs 更克制）
- 字体：Playfair Display（温暖）或 Doto（点阵）
- 位置：绝对定位，右下角或中心
- 内容：年份、品牌首字母、单个词

## 3. ANTI-PATTERNS — 红线

### 视觉
- 渐变、阴影、模糊、毛玻璃
- 骨架屏、toast 弹窗、表情
- 深色/浅色切换
- 视差、滚动劫持、弹跳缓动
- 斑马纹、填充图标、emoji 做 UI
- Accent 做装饰
- 圆角 > 8px（pill 除外）
- 颜色直接代替 opacity/梯度

### 文案
- em-dash `—` 禁止出现，统一用短横 `-`
- 禁止套话：赋能、抓手、闭环、对齐、深耕、升华、落地、引爆
- 禁止假精确数字（92%、4.1×、48k）——除非有真实数据源
- 禁止虚构客户/供应商/人名
- 禁止"值得注意的是""总的来说""综上"等 AI 套话
- 全页统一称呼口径

### 布局
- 禁止装饰性横条（Hero 底部 BRAND.MOTION.SPATIAL 类）
- 禁止城市/时间/天气信息条
- 禁止滚轮提示（Scroll ↓）
- 禁止装饰性章节编号（001·能力、01/4）
- 禁止图片上叠标签（说明放图片下方）
- 禁止隔 section 换底色——底色服务于叙事节奏，不是自动模式

### 数据
- 长列表 >5 项禁止逐行加分割线——换卡片/分栏/标签页/折叠面板
- 禁止进度条+背景轨道当对比视觉——用数字+小图标
- 禁止纯数字罗列——每组数据必须带标注/单位/时间边界

## 4. 内容类型与密度规范

> 不只是品牌展示。根据内容类型调整页数、密度、可视化程度。

### 4.1 规则 vs 模板

**规则（必须遵守）**：
- 字号对比 ≥ 8:1
- 7 色 Brand DNA，80/10/10 比例
- 每页有核心信息，有呼吸节奏
- 用 token，不用阴影渐变
- 标签统一 JetBrains Mono + uppercase

**模板（灵活选择）**：
- 首页风格（双区切割/单区居中/数据冲击/洞察前置）
- 颜色主色（olive/earth/glacier/rock，根据内容调性选）
- 结构章数（根据内容特点定，不是固定 7 章）
- 图表类型（折线/条形/散点/热力图，根据数据定）
- 密度高低（低/中/高，根据内容定）

### 4.2 首页设计规则

首页根据**内容类型**选择，不是固定模板：

| 内容类型 | 首页风格 | 视觉锚点 | 示例 |
|----------|----------|----------|------|
| **数据报告** | 数据冲击型 | 大数字 + 趋势箭头 | 28 家供应商 · 96.4% 合规率 |
| **分析决策** | 洞察前置型 | 核心结论 + 关键发现 | "结构性分化正在加剧" |
| **品牌展示** | 品牌宣言型 | 双区切割 + ghost 水印 | "用温暖的手绘语言" |
| **知识沉淀** | 目录导航型 | 章节列表 + 核心问题 | "8 个维度，1 个结论" |

**首页必须有**：
- 一个视觉锚点（大数字/核心结论/品牌宣言/章节地图）
- 字号对比 ≥ 8:1
- 明确的"这是什么"（标签/副标题）

### 4.3 颜色选择规则

颜色根据**内容调性**选择主色：

| 内容调性 | 主色 | 说明 | 适用场景 |
|----------|------|------|----------|
| **权威/分析** | `--olive` | 数据驱动、专业感 | 业绩报告、供应商诊断 |
| **温暖/人文** | `--earth` | 自然质感、亲和力 | 品牌故事、用户研究 |
| **冷静/精确** | `--glacier` | 技术感、可信度 | 技术文档、系统设计 |
| **警示/异常** | `--orange` | 强调问题、行动导向 | 风险预警、异常报告 |
| **中性/客观** | `--rock` | 不带情绪、纯数据 | 数据看板、客观分析 |

**颜色使用规则**：
- 主色用于：Hero 切割区、强调元素、数据高亮
- 辅色用于：次要信息、状态指示
- 中性色用于：背景、文字、边框
- 每页最多 1 个主色 + 1 个辅色

### 4.4 四种内容类型

| 类型 | 页数 | 信息密度 | 可视化程度 | 字号策略 | 留白比例 |
|------|:----:|:--------:|:----------:|----------|:--------:|
| **品牌展示型** | 5-10 | 低（1-2 信息/页） | 高（纯视觉） | 大字号（96-120px Hero） | >60% |
| **数据报告型** | 10-20 | 高（5-10 信息/页） | 中（图表为主） | 中字号（48-64px Hero） | 40-50% |
| **分析决策型** | 8-15 | 中（3-5 信息/页） | 中（文字+图表） | 中字号（48-64px Hero） | 45-55% |
| **知识沉淀型** | 15-30 | 中（3-5 信息/页） | 低（文字为主） | 中字号（48-64px Hero） | 40-50% |

### 4.5 可视化程度选择

| 数据特征 | 可视化方式 | 组件选择 |
|----------|-----------|----------|
| 单一指标 | 大数字 + 趋势箭头 | S05 指标卡 |
| 对比数据 | 条形图 / 双栏对照 | S06 排名条形 / S09 对照 |
| 时间序列 | 时间线 / 折线图 | S08 时间线 / SVG-Charts |
| 多维数据 | 表格 / 热力图 | S12 数据表格 / SVG-Charts |
| 流程步骤 | 步骤卡 / 流程图 | S10 流程步骤 / S16 闭环 |
| 文字叙事 | 图文混排 / 金句 | S11 图文混排 / S15 金句 |

### 4.6 页数控制

**页数公式**：
```
页数 = 核心信息数 ÷ 每页信息密度 + 2（Hero + 收束）
```

**示例**：
- 5 个核心指标 → 5 ÷ 2 + 2 = 5 页（品牌展示型）
- 10 个供应商数据 → 10 ÷ 3 + 2 = 6 页（数据报告型）
- 8 个分析维度 → 8 ÷ 2 + 2 = 6 页（分析决策型）
- 20 个知识点 → 20 ÷ 3 + 2 = 9 页（知识沉淀型）

**页数上限**：
- 品牌展示型：≤ 10 页（多了注意力分散）
- 数据报告型：≤ 20 页（多了信息过载）
- 分析决策型：≤ 15 页（多了决策疲劳）
- 知识沉淀型：≤ 30 页（多了阅读负担）

### 4.5 密度与质感的平衡

**高密度不等于低质感**：
- 用 `--surface` 做数据 section 背景（纯白，数据更清晰）
- 用 1px border 建立层级（不用 box-shadow）
- 用 JetBrains Mono 做数据标签（统一感）
- 用 ghost 水印做空间感（即使高密度也需要）

**低密度不等于高质感**：
- 留白过多会显得空洞
- 装饰元素过多会显得杂乱
- 字号过大会显得粗暴

**平衡原则**：
- 每页至少 1 个视觉锚点（大数字、图表、ghost）
- 每 3-4 页插一个呼吸 section（金句、空白页）
- 收尾 section 必须深色（仪式感）
