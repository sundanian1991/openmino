# 视觉形式框架

> 按视觉形式组织，不是按功能。每个形式有一个核心形态（A）和若干变体（B/C...）。

---

## 色彩体系（统一）

所有形式共用同一套色彩变量：

```
场景色：olive / earth / glacier（三选一）
信号色：org（仅功能警示）
灰度：display(100%) / primary(90%) / secondary(60%) / disabled(40%)
```

## 选型规则

```
① 定核心形式：
  要震撼开场？      → Hero 切割族（H-A 到 H-F）
  要展示数据？      → 数据图表族（HM/BR/LN/PI/SC/RD/GG/MT）
  要精确对比？      → 表格族（TB-A/B/C）
  要信息容器？      → 卡片族（CD-A/B/C/D/E）
  要引导结构？      → 导航族（NV-A/B/C/D）
  要氛围装饰？      → 装饰族（DC-A/C）

② 定具体变体：在族内选 A 优先，A 不合适再选 B/C

③ 定场景色：olive（增长） / earth（工艺） / glacier（技术）

④ 检查节奏：相邻 section 不得使用同族形态
```
