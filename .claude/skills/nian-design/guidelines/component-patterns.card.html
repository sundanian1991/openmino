Component Patterns — 组件组合模式
数据总览 需要一眼扫完关键指标时
Section Header → Stat Grid (4-6 列) → Split Card (可选)
Stat Grid 用 gap:1px 分隔，每格放一个数字+标签+描述。如果指标间有对比关系，下面加 Split Card 做双栏对比。
/* Stat Grid */\n.sg { display: grid; gap: 1px; background: var(--border); }\n.sg4 { grid-template-columns: repeat(4, 1fr); }\n.sg-card { background: var(--bg); padding: 40px 32px; }
原则/要点列表 有 3-5 个并列的要点或原则时
Section Header → Key Points (垂直列表)
Key Points 用 gap:1px 分隔，每项用编号+标题+描述。编号用 glacier 色，标题加粗。
/* Key Points */\n.kp { display: flex; flex-direction: column; gap: 1px; background: var(--border); }\n.kp-item { background: var(--bg); padding: 32px; display: grid; grid-template-columns: 48px 1fr; gap: 24px; }
激励/属性矩阵 5 个并列的属性或激励类型时
Section Header → Incentive Strip (5 列)
水平排列 5 个卡片，每卡放图标+标题+描述。用 gap:1px 分隔。
/* Incentive Strip */\n.inc { display: grid; grid-template-columns: repeat(5, 1fr); gap: 1px; background: var(--border); }\n.inc-card { background: var(--bg); padding: 32px 24px; text-align: center; }
成长阶梯/层级 有明确的层级递进关系时
Section Header → Ladder (4 列)
水平排列 4 个阶梯，每级放级别标签+名称+条件。最后一级可以用深色背景强调。
/* Ladder */\n.ladder { display: grid; grid-template-columns: repeat(4, 1fr); gap: 1px; background: var(--border); }\n.ladder-step { background: var(--bg); padding: 32px 24px; }
内容占比/比例 展示几个类别的占比关系时
Section Header → Layers (水平条)
水平排列，宽度按比例分配（flex:6/2/1.5/0.5）。每格放百分比+名称+描述。主色用 olive，其他用 tint 背景。
/* Layers */\n.layers { display: flex; gap: 1px; background: var(--border); height: 160px; }\n.layer--1 { flex: 6; background: var(--olive); }
年度/季度节奏 展示时间阶段或项目进度时
Section Header → Timeline (垂直列表)
垂直排列，每行放季度标签+名称+描述+目标。用 gap:1px 分隔。当前阶段用 olive 背景高亮。
/* Timeline */\n.tl { display: flex; flex-direction: column; gap: 1px; background: var(--border); }\n.tl-row { display: flex; align-items: stretch; }\n.tl-q { width: 80px; font-family: var(--fm); }
金句转场 数据密集 section 之间需要呼吸点时
Glacier/Earth Tint → Quote Hero (居中)
居中排列，大字号（36px/300）+ 留白。强调词用 olive 色。底部加小标签。背景用色彩 tint。
/* Quote Hero */\n.qh { text-align: center; padding: 80px 64px; }\n.qh-text { font-family: var(--fd); font-size: 36px; font-weight: 300; }
组合规则：每份报告 = 1 个 Hero(dark) + N 个内容 section(tint) + 1 个金句转场(可选) + 1 个收尾(dark)。每个 section 内部用上述模式之一，不混用。内容密集的 section 之间插金句转场做呼吸点。
