INSIGHTS

六大核心商业洞察

从定价演变中提取的可复用模式。

01计量单位即定价权
谁定义计量单位，谁就掌握定价权。MiniMax 和 Kimi 从 Prompt 转 Token，都在争夺这个定义权。云厂商用 API 请求次数——数字看起来大但消耗不透明，信息差有利于厂商。推出新产品时，计量单位的选择决定了用户感知和利润空间。
02补贴→收紧→分层：三段式节奏
先补贴跑量（百万Token几毛钱）→再收紧测试弹性（限购、停售低档）→最后精细分层（极速版 98-899 元）。不要一开始就想好定价。先跑量，再测弹性，最后分层。
03从入口到生态的升级路径
Coding 是获客入口，Token 是生态载体。MiniMax 的 Token Plan 不只卖编程，打包了图片、语音、音乐、视频。29元买到的不只是代码补全，是全模态 AI 能力。用高频刚需（编程）获客，用低频高价值（视频/音乐）提高 ARPU。
04国内外定价逻辑的根本差异
国内 7 家在同一个池子打价格战，比谁更便宜。海外面对 Claude/GPT/Gemini 三巨头，比谁更强。国内定价权来自成本控制，海外来自能力溢价。
05Agent 时代成本结构剧变
70% 的 coding agent token 是浪费。一个开发者 8 个月消耗 100 亿 Token（API 价 $15,000+），Max plan 只花 $800（省 93%）。订阅费是地板不是天花板——实际账单是标价的 2-5 倍。
06开源模型重塑定价权
DeepSeek 模式——开源+极致低价+不卖订阅。V3.2 在 OpenRouter 月调用 5.35 万亿 token，全球第三。开源模型的免费部署能力持续压制定价上限。厂商定价权来自服务便利性而非模型独占性。
