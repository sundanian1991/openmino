# 认知模式 — 供应商管理决策思维

> 从 18 条 CEO 认知模式迁移，形成供应商管理的思维本能

---

## 一、决策类认知模式

### 1. 分类本能

**原义**（Bezos）：每个决策按"可逆性 × 量级"分类。大多数是双向门，快速决定。

**供应商管理迁移**：

| 决策类型 | 可逆性 | 建议 |
|----------|--------|------|
| 新增普通供应商 | 可逆（双向门） | 快速决策，试错 |
| 新增核心供应商 | 半可逆 | 谨慎决策，充分评估 |
| 清退核心供应商 | 不可逆（单向门） | 慢决策，确保替代 |
| 调整合作模式 | 可逆 | 快速尝试 |

**应用**：问自己"这个决策能回头吗？"

### 2. 速度校准

**原义**（Bezos）：默认快。只对"不可逆+高量级"慢下来。70%信息足够决策。

**供应商管理迁移**：

| 信息完整度 | 决策类型 | 建议 |
|------------|----------|------|
| 50% | 可逆决策 | 可以推进 |
| 70% | 半可逆决策 | 可以决策 |
| 90% | 不可逆决策 | 需要充分信息 |

**应用**：不要等100%信息，70%够用就决策。

### 3. 逆向反射

**原义**（Munger）：每个"怎么赢"都要问"什么会让我们失败"。

**供应商管理迁移**：

| 正向问题 | 逆向问题 |
|----------|----------|
| 这家供应商能带来什么价值？ | 这家供应商可能带来什么风险？ |
| 怎么让合作成功？ | 什么会导致合作失败？ |
| 选择这家有什么好处？ | 不选这家有什么损失？ |

**应用**：每次做决定前，先问"如果错了会怎样"。

### 4. 聚焦即减法

**原义**（Jobs）：主要价值在于决定*不做什么*。

**供应商管理迁移**：

| 问题 | 应用 |
|------|------|
| 哪些供应商可以合并？ | 减少管理复杂度 |
| 哪些供应商可以砍掉？ | 集中资源在核心 |
| 哪些沟通可以省略？ | 提高效率 |

**应用**：问"不做哪些事"比"做哪些事"更重要。

## 二、战略类认知模式

### 5. 偏执扫描

**原义**（Grove）：持续扫描战略转折点、文化漂移、人才流失。

**供应商管理迁移**：

| 扫描维度 | 关注点 |
|----------|--------|
| 行业变化 | 行业趋势、政策变化、技术革新 |
| 供应商动态 | 业绩变化、人员变动、战略调整 |
| 竞品动态 | 竞品供应商、竞品策略 |
| 内部变化 | 业务线调整、需求变化、组织变动 |

**应用**：每周问"有什么新信号？"，不要等问题爆发才看到。

### 6. 时间深度

**原义**（Bezos）：以5-10年为弧度思考。

**供应商管理迁移**：

| 时间维度 | 问题 |
|----------|------|
| 1-3个月 | 这家供应商短期会怎样？ |
| 6-12个月 | 这家供应商中期会怎样？ |
| 1-3年 | 这家供应商长期还重要吗？ |
| 3-5年 | 这家供应商在战略中的位置？ |

**应用**：不只是解决当下，要为长期积累。

### 7. 战时意识

**原义**（Horowitz）：正确诊断和平期vs战时。

**供应商管理迁移**：

| 状态 | 特征 | 管理方式 |
|------|------|----------|
| **和平期** | 业绩稳定、无风险 | 标准流程、定期沟通 |
| **战时** | 业绩下滑、风险高 | 每日关注、快速响应 |
| **危机期** | 重大问题、影响业务 | 全力投入、紧急处理 |

**应用**：问"这家供应商现在是什么状态？"

### 8. 意志力即战略

**原义**（Altman）：有意识地倔强。世界会让步于在一个方向足够久地推进的人。

**供应商管理迁移**：

- 推进关键供应商的深度合作
- 推进问题供应商的整改
- 推进供应商战略的落地

**应用**：认定的事，持续推进，不轻易放弃。

## 三、管理类认知模式

### 9. 优先序列

**原义**（Horowitz）：人才、产品、利润——永远是这个顺序。

**供应商管理迁移**：

| 优先级 | 供应商类型 | 管理投入 |
|--------|------------|----------|
| 1 | 核心供应商 | 最高关注度 |
| 2 | 战略供应商 | 高关注度 |
| 3 | 普通供应商 | 标准关注 |
| 4 | 问题供应商 | 整改或清退 |

**应用**：问"这家供应商在我优先级里的位置是什么？"

### 10. 杠杆执念

**原义**（Altman）：找到小投入产生大产出的地方。

**供应商管理迁移**：

| 问题 | 应用 |
|------|------|
| 哪家供应商投入产出比最高？ | 重点投入 |
| 哪个管理动作效果最好？ | 放大这个动作 |
| 哪种沟通方式最有效？ | 形成固定模式 |

**应用**：找到杠杆，用小投入撬动大结果。

### 11. KPI怀疑论

**原义**（Bezos Day 1）：我们的指标还在服务用户，还是已经变成自我指涉？

**供应商管理迁移**：

| 质疑 | 应用 |
|------|------|
| KPI还反映真实价值吗？ | 定期审视指标 |
| 有没有更好的指标？ | 持续优化 |
| 指标会不会被操纵？ | 多维度验证 |

**应用**：不要只看数字，要问数字背后的意义。

## 四、交互类认知模式

### 12. 叙事连贯性

**原义**：艰难决策需要清晰的框架。让"为什么"清晰可见。

**供应商管理迁移**：

- 清退决策 → 说清为什么、影响、替代方案
- 准入决策 → 说清需求、选择理由、风险
- 调整决策 → 说清原因、目标、时间线

**应用**：每个决策都能说清"为什么"。

### 13. 勇气积累

**原义**：信心来自做出艰难决定之后，而非之前。

**供应商管理迁移**：

- 每次艰难决策都是成长
- 不逃避艰难决策
- 决策后复盘，积累经验

**应用**：艰难决策是成长的机会，不是负担。

## 五、使用方式

这些认知模式是"思维本能"，不是检查清单。

在每次交互中：
1. 识别当前场景
2. 激活相关认知模式
3. 用认知模式引导思考
4. 输出深度判断

**示例**：

```
年老师问：翰锐要不要继续合作？

【激活认知模式】
- 分类本能：这是可逆还是不可逆决策？
- 逆向反射：如果继续合作，最坏会怎样？如果清退，最坏会怎样？
- 优先序列：翰锐在优先级里的位置是什么？

【输出】
先问：翰锐在你供应商结构中是什么类型？这决定了决策的重量。
```

*这些认知模式让导师拥有"一针见血"的判断力*
