# Presentation Planning Guide

> 框架级规范。版式选择、内容密度和结构规划。

## Standard Slide Structures

### 标准汇报（10–12 页）

```
Slide 1:  封面 (cover)
Slide 2:  目录 (toc)
Slide 3:  执行摘要 / 核心论点 (executive_summary 或 table_insight)
Slides 4-7: 支撑论证（每页一个论点，变换版式）
Slides 8-10: 案例/证据 (case_study 或 side_by_side)
Slide 11: 综合/路线图 (timeline 或 process_chevron)
Slide 12: 关键结论 + 结束 (key_takeaway 或 closing)
```

### 简短汇报（6–8 页）

```
Slide 1:  封面
Slide 2:  执行摘要 (executive_summary)
Slides 3-5: 核心内容（交替用 big_number, table_insight, side_by_side）
Slide 6:  综合/时间轴
Slide 7:  关键结论
Slide 8:  结束页
```

### 关键规则

- **最少 8 页**（实质性主题）
- **一次性生成全部**：不允许生成后截断
- **目录必须列出所有章节**

## Layout Selection Rules

### 开篇优先级（Slides 2–5 用高影响力版式）

1. **`table_insight`** — 结构化论点+右侧灰色面板洞见（首选）
2. **`big_number`** — 单个有影响力的数字
3. **`key_takeaway`** — 左侧详情+右侧摘要

### 数据必须用图表

当内容含有日期/时段 + 数值/百分比时，**必须**用图表版式（#37–#39, #48–#56, #64, #70），不得用文字表格代替。

### 图文版式要求

8页以上 PPT，至少1张使用图文版式（#40–#47, #68）。

### 版式多样性要求

**相邻幻灯片不得使用同一版式**。

### `two_column_text` 限制

全局不超过1张。是最低视觉吸引力的版式。

## Content Density Requirements

每张内容页（非封面/结束）必须：

1. **至少3个视觉块**（标题栏 + 内容区 + 底部摘要）
2. **内容区使用率 ≥ 50%**（1.4"–7.05" 之间的可用空间）
3. **Action Title 必须是完整句子**（表达洞见，不是标签）
   - ✅ `"连接组约束的AI模型将自由参数减少90%，实现单细胞精度预测"`
   - ❌ `"连接组约束的AI模型"`
4. **具体数字**：用户提供了数字时必须突出展示
5. **每页有 source 出处**
