# Workspace Tidy — 三区域文件生命周期管理

> **触发词**：整理文件、清理根目录、归档已完成工作、周回顾清理、文件混乱
>
> **扫描范围**：docs/、memory/、workspace/ 三个文件夹
>
> **核心原则**：先分类整理，再归档移动，最后删除冗余

---

## 三区域架构理解

项目采用三区域架构，每个区域有明确职责：

| 区域 | 职责 | 内容类型 | 留存周期 | 命名规则 |
|------|------|----------|---------|---------|
| **workspace/** | 入口区 | 草稿、进行中、产出物 | 周度清理 | `{主题}-{YYYY-MM-DD}/` |
| **docs/** | 归档区 | 正式文档、知识资产 | 长期保留 | `{业务域}/{子类}/` |
| **memory/** | 核心区 | 洞察、决策、人物画像 | 永久保留 | `{类型}/{实体}-{日期}/` |

## Phase 1: 扫描诊断

### 1.1 扫描三个区域

```bash
# workspace/ 扫描
find workspace/ -type f \( -name "*.md" -o -name "*.html" -o -name "*.pptx" -o -name "*.xlsx" \) -not -path "*/node_modules/*"

# docs/ 扫描
find docs/ -type f -name "*.md"

# memory/ 扫描
find memory/ -type f -name "*.md" -not -path "*/archive/*"
```

### 1.2 问题诊断

对每个发现的文件，判断：
- **位置是否正确** — 是否在合适的区域？
- **命名是否规范** — 是否遵循命名规则？
- **内容是否过时** — 是否有保留价值？
- **是否重复** — 是否有多个版本？

## Phase 2: 分类整理

### 2.1 Workspace/ 整理

**目标结构**：
```
workspace/
├── inbox/              # 新文件入口（必须清零）
├── drafts/             # 草稿（编辑中）
├── active/             # 进行中项目
│   └── {主题}-{YYYY-MM-DD}/
├── output/             # 已交付产出物
│   ├── {主题}-{YYYY-MM-DD}/
│   └── {主题}-v{数字}.html
├── archive/            # 已归档（按周期）
│   └── 2026-W{周数}/
└── docs/               # workspace 专用文档
```

**整理规则**：

| 当前位置 | 内容类型 | 目标位置 | 操作 |
|---------|---------|---------|------|
| `inbox/*` | 所有内容 | 判断用途后移动 | → drafts/ 或 active/ 或 output/ |
| `草稿/*` | 草稿文件 | → drafts/ | 移动 |
| `进行中/*` | 进行中项目 | → active/{主题}-{日期}/ | 重命名+移动 |
| `产出物/*` | 已产出 | → output/{主题}-{日期}/ | 移动 |
| `已归档/*` | 已归档 | → archive/ 或 docs/ | 判断价值 |
| `待处理/*` | 待处理 | → active/ 或 drafts/ | 判断状态 |

**统一命名**：
- 主题文件夹：`{YYYY-MM-DD}-{主题}/`
- 产出文件：`{YYYYMMDD}-{主题}-v{数字}.{ext}` 或 `{YYYYMMDD}-{主题}.{ext}`
- 禁止：`完整版`、`最终版`、`_` 分隔符（统一用 `-`）

### 2.2 Docs/ 整理

**目标结构**：
```
docs/
├── 供应商/
│   ├── 管理制度/       # 办法、规范、模板
│   ├── 历史项目/       # 已完成项目
│   ├── 供应商库/       # 档案、盘点
│   └── 运营数据/       # 成果、现场管理
├── 个人成长/
│   ├── 晋升述职/
│   ├── 职业发展/
│   └── 工作风格/
├── 学习资料/
│   ├── AI-Agent工作流解析/
│   ├── Claude Code/
│   └── AI协作/
├── 知识/              # 通用知识资产
│   ├── 供应商管理知识/
│   ├── 需求文档结构规范/
│   └── 层级设计方法论/
└── workspace-knowledge/  # workspace 专用知识
```

**整理规则**：

| 问题类型 | 处理方式 |
|---------|---------|
| 散落的临时文件 | 移到 workspace/ 或删除 |
| 重复内容 | 保留最新/最完整版 |
| 命名不规范 | 重命名为 `{业务域}-{内容}.{ext}` |
| 过时内容 | 移到 archive/ 或删除 |

### 2.3 Memory/ 整理

**目标结构**：
```
memory/
├── MEMORY.md           # 记忆索引
├── insights.md         # 洞察记录
├── projects/           # 项目记忆
│   └── {项目名}/
├── topics/             # 主题记忆
│   └── {主题}.md
├── daily/              # 日志（按月归档）
│   └── YYYY-MM/
│       └── YYYY-MM-DD.md
└── archive/            # 历史归档（不动）
```

**整理规则**：

| 问题类型 | 处理方式 |
|---------|---------|
| 散落在根目录的 md | 移到 topics/ 或 insights.md |
| 混乱的 daily/ | 按 YYYY-MM/YYYY-MM-DD 重组 |
| 过时记忆 | 移到 archive/ |

## Phase 4: 执行移动

### 4.1 执行前确认

**确认清单**：
- [ ] 年老师确认移动计划
- [ ] 备份重要文件（可选）
- [ ] 检查目标路径不存在

### 4.2 执行顺序

1. **先处理删除** — 清理备份、临时、调试文件
2. **再处理移动** — 按区域顺序（workspace → docs → memory）
3. **最后验证** — 检查移动结果

### 4.3 执行命令示例

```bash
# 删除冗余文件
trash "workspace/output/*-backup.html"
trash "workspace/output/*-debug*.html"
trash "inbox/~$*.xlsx"

# 移动文件
mv "workspace/进行中/成本调研-2026-03-24" "workspace/active/成本调研-2026-03-24"
mv "workspace/产出物/供应商通报-2026-04-01" "workspace/output/供应商通报-2026-04-01"

# 创建归档目录
mkdir -p "workspace/archive/2026-W14"
mv "workspace/已归档/2026-W13" "workspace/archive/2026-W13"

# 清理 inbox
# （确认后清空）
```

## 规范文档

每个区域需要 CLAUDE.md 说明规范：

### Workspace/CLAUDE.md
- 文件夹命名规则：`{主题}-{YYYY-MM-DD}/`
- 文件命名规则：`{主题}-v{数字}.{ext}`
- 生命周期规则：inbox → drafts → active → output → archive

### Docs/CLAUDE.md
- 一级目录：供应商/、个人成长/、学习资料/、知识/
- 文件命名：`{业务域}-{内容}.{ext}`
- 归档规则：历史项目按主题归档

### Memory/CLAUDE.md
- 目录结构：MEMORY.md、insights.md、projects/、topics/、daily/
- 日志规则：按 YYYY-MM/YYYY-MM-DD 归档
- 记忆提取：有价值内容提炼到 topics/

## 智能价值判断

### 价值判断流程

```
读取文件内容
    │
    ├─→ 包含"总结"/"报告"/"规范"/"机制"
    │   └─→ 文档价值高 → docs/归档
    │
    ├─→ 包含"洞察"/"发现"/"经验"/"教训"
    │   └─→ 洞察价值高 → memory/提炼
    │
    ├─→ 包含"草稿"/"临时"/"test"/"TODO"
    │   └─→ 价值低 → 删除或 drafts/
    │
    └─→ 内容空洞/重复/过时
        └─→ 无价值 → 删除
```

### 价值评分矩阵

| 内容特征 | 文档价值 | 洞察价值 | 建议操作 |
|----------|---------|---------|---------|
| 正式汇报/机制文档 | ⭐⭐⭐ | ⭐ | docs/归档 |
| 述职/复盘/总结 | ⭐⭐ | ⭐⭐⭐ | 提炼洞察 + 归档 |
| 规范/模板 | ⭐⭐⭐ | ⭐ | docs/知识/ |
| 草稿/临时文件 | ⭐ | ⭐ | 删除 |
| 测试/备份文件 | ⭐ | - | 删除 |

## 注意事项

1. **谨慎删除** — 删除前必须预览确认，优先用 `trash` 而非 `rm`
2. **保持结构** — 不破坏已有合理结构
3. **内容优先** — 必须读取内容判断价值，不能只看文件名
4. **周回顾节奏** — 建议每周执行一次，保持 workspace/ 清洁
5. **规范优先** — 新增文件必须遵循命名规范
