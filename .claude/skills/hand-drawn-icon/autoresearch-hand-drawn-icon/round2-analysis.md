## 3. 组件模式提炼

### ✅ 5 大组件模式验证成功

```
东京猫 = 头部(7) + 身体(8) + 四肢(10) + 尾巴(1) + 装饰(26)
           └─ 核心特征     └─ 姿态     └─ 动态   └─ 陪衬
```

**核心结构**（27 path）:
- 头部: 耳朵×2 + 眼睛×2 + 鼻子 + 嘴巴 + 脸型
- 身体: 躯干 + 花纹 + 坐姿底座
- 四肢: 抬起的右爪 + 坐着的左爪 + 隐形后腿
- 尾巴: 单条弯曲 path

**装饰层**（26 path）:
- 铃铛: 主体 + 挂绳 + 细节
- 项圈: 带子 + 扣环
- 硬币: 在爪中的暗示
- 胡须: 9 条细线

**学习**:
- 核心结构只占 50%，装饰占 50%
- 装饰是"可扩展层"，核心是"不可移除层"
- 设计时应先确保核心结构完整，再逐步添加装饰

## 4. 繁而不乱的平衡

### ✅ Quiver 的平衡策略

**1. 色彩分层**（避免视觉混乱）
- 米白 (#FEFFDF): 主体，占 70%
- 米灰 (#D7C9A8): 阴影/过渡，占 20%
- 朱红 (#B12D16): 焦点（项圈、铃铛），占 5%
- 墨黑 (#0A0500): 描边，占 5%

**2. 线条分层**（建立视觉层次）
- 主轮廓: 0.75px（耳朵、身体轮廓）
- 结构线: 0.5px（内部细节）
- 细节线: 0.25px（胡须、纹理）

**3. 空间分布**（避免拥挤）
- 头部集中: 上半 40% 区域
- 身体居中: 中间 40% 区域
- 四肢展开: 四角支撑
- 装饰点缀: 不干扰主体

**关键洞察**: 繁而不乱的 3 个原则
1. **色彩分层** — 主色占 70%，点缀色 ≤ 10%
2. **线条分层** — 粗中细比例约 3:2:1
3. **空间分区** — 头/身/四肢/装饰各占区域，不重叠

## 6. 对技能的改进建议

### 新增设计决策

**细节密度规则**（何时该停）:
```
简单物体（10-30 path）: 细节占比 ≤ 20%
中等物体（30-60 path）: 细节占比 30-40%
复杂物体（60-90 path）: 细节占比 40-50%

超出范围 = 细节过多，需要删减
```

**装饰层原则**:
```
装饰是"可扩展层"，不是"必须层"
先确保核心结构完整（50% path），再逐步添加装饰（50% path）
如果核心结构不完整，装饰再多也无用
```

**色彩分层验证**（补充到色彩系统）:
```
主色占 70%: 整体色调（米白/陶土色）
点缀色 ≤ 10%: 焦点（朱红/深陶土）
描边占 20%: 墨黑统一
```

### 向下兼容策略

**如果 API 学习了复杂场景（虾、猫），能否简化到简单场景？**

**答案**: 可以。设计决策规则是"从复杂到简单"可降维的：

- 简单: 3 组件（主体 + 结构 + 装饰）+ 最少细节

**学习策略**: 先掌握复杂场景（80-90 path），再向下兼容到简单场景（10-30 path）。
