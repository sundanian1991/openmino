# 思维透镜输出格式与排版规范

> 配套 `thinking-lenses` Skill 的视觉与格式标准文档。
> 所有文本输出和可视化产出遵循此规范。

---

## 一、设计哲学

### 核心原则

- **标题即结论**：每个标题都是一句话结论，不是名词短语。用户扫一眼标题就知道这一段说什么。
- **高信息密度**：目标密度 7-8/10。拒绝 cathrynlavery 风格的大量留白（4/10）。
- **无装饰化**：无阴影、无渐变、无圆角泛滥。扁平化区块，顶部细线分隔。
- **结构即导航**：标题层级、表格、分割线本身就是导航，不需要额外的目录或按钮。

### 配色体系（贝恩灰橙 + 暖白）

| Token | 色值 | 用途 |
|-------|------|------|
| 主色 Primary | `#CC6B1E` | 高亮、强调、行动按钮、激活状态 |
| 暖白背景 Warm White | `#FAF8F5` | 主背景色 |
| 中性灰 Neutral Gray | `#B8B8B8` | 次要文字、分隔线、未激活状态 |
| 深灰 Dark Gray | `#333333` | 主文字 |
| 浅灰背景 Light BG | `#F0EEEB` | 卡片背景、区块背景 |
| 强调背景 Accent BG | `#FFF8F0` | 高亮区块背景 |

### 字体规范

- **主字体栈**：`system-ui, -apple-system, "Segoe UI", Roboto, "Noto Sans SC", sans-serif`
- **英文标题**：使用无衬线，字重 600-700
- **中文正文**：字重 400，行高 1.6-1.8
- **代码/引用**：`"SF Mono", "Fira Code", monospace`

## 二、Markdown 文本输出规范

### 标题层级

```markdown
# 一级 —— 仅用于整个文档标题，对话中极少使用

## 二级 —— 模块结论。如"推荐透镜：5个"、"全角度扫描完成"
### 三级 —— 子模块。如"透镜① Economist"、"场景1：会议纪要"
#### 四级 —— 极少使用。仅在需要细分时使用
```

### 强调规则

| 格式 | 用途 | 示例 |
|------|------|------|
| `**加粗**` | 关键结论、标签、行动项、职业名称 | **为什么匹配**、**Economist** |
| `*斜体*` | 引用原文、外来术语、需要强调的从句 | *More mental models...* |
| `` `代码` `` | 快捷指令、文件名、数据值、记录模板 | `方案扫一遍` |
| `> 引用块` | 原文引用、核心论点、金句 | > "More mental models..." |

### 表格规范

表格是信息密度最高的格式，优先使用：

```markdown
| 列A | 列B | 列C |
|-----|-----|-----|
| 内容 | 内容 | 内容 |
```

- 表头使用加粗
- 对齐：文本左对齐，数字右对齐
- 分组标记：用 emoji 或颜色标记分组（🔴🟡🟢）
- 避免超过 6 列，超过则拆分或使用卡片

### 列表规范

- **有序列表**：用于步骤、流程、有先后顺序的内容
- **无序列表**：用于并列选项、特征列举
- **任务列表**：用于行动项 `- [ ] 行动`
- 列表层级不超过 3 层

### 段落与信息密度控制（排版铁律）

**核心原则：每段只讲一件事。**

- 长段落（超过3句）必须检查：里面是否有并列信息？有就拆
- 并列的2-4项 → 用列表（无序或有序）
- 并列的5项以上或有对比维度 → 用表格
- 有时间先后的步骤 → 用有序列表
- 一个判断 + 多个论据 → 判断加粗在前，论据用列表跟在后面

**拆段规则**：

| 原文特征 | 处理方式 | 示例 |
|---------|---------|------|
| 一段里有3个以上并列点 | 拆成列表 | "条件1...条件2...条件3..." → 1. 2. 3. |
| 一段里有"如果...则..."的多个分支 | 拆成表格 | 边界条件 → 条件/后果 两列表格 |
| 一段里有对比关系 | 拆成表格 | "A是X，B是Y" → A/B 两行表格 |
| 一段里有步骤流程 | 拆成有序列表 | "先做X，再做Y，最后Z" → 1. 2. 3. |
| 一段里有判断+多个论据 | 判断加粗，论据列表 | **核心判断** → - 论据1 - 论据2 |

**反例**：

```
❌ 系统在什么范围内有效？边界条件是：(1) 用户问题有明确的结构可以被分析；
(2) AI对该领域有足够的训练数据；(3) 用户有耐心走完流程。出了第一个边界
（纯情绪宣泄），系统退化为心理咨询；出了第二个边界（高度专业化领域），
系统的推理可能不准确；出了第三个边界（用户赶时间），系统被迫压缩步骤。

✅ 系统在什么范围内有效？

| 边界条件 | 超出后会怎样 |
|---------|-------------|
| 用户问题有明确结构可以被分析 | 纯情绪宣泄 → 系统退化为心理咨询 |
| AI对该领域有足够训练数据 | 高度专业化领域 → 推理可能不准确 |
| 用户有耐心走完流程 | 用户赶时间 → 被迫压缩步骤，退化为快速版 |
```

**信息密度目标**：7-8/10。不是字多就密，是每句话都有信息量。宁可多用表格和列表让结构清晰，也不要堆长段落让读者自己找重点。

### 分隔线使用

```markdown
---
```

- 用于模块之间的硬分隔
- 对话末尾的使用记录前
- 不滥用，同一页面不超过 3 条

## 三、可视化输出规范（HTML/信息图）

### 当用户请求可视化时

用户说"做成图""卡片""信息图"时，使用以下规范生成 HTML 或 SVG：

### 卡片组件

```
┌─────────────────────────────────────┐
│ #01  Artist  艺术家                    │  ← 顶部细线 #CC6B1E (2px)
│                                     │
│ What if Creativity Were             │  ← 核心问题，加粗
│ the Priority?                       │
│                                     │
│ 最佳场景：创意瓶颈、方案同质化          │  ← 次要信息，灰色
│                                     │
│ [用此视角分析]                        │  ← 按钮，暖橙背景
└─────────────────────────────────────┘
```

### 页面布局

- **最大宽度**：1200px（桌面），100%（移动端）
- **内边距**：24px（桌面），16px（移动端）
- **卡片间距**：16px
- **网格**：CSS Grid，响应式列数（桌面3列，平板2列，手机1列）

### 按钮样式

- **主按钮**：背景 `#CC6B1E`，文字白色，无圆角（或 2px），无阴影
- **次按钮**：背景 `#F0EEEB`，文字 `#333333`，无圆角
- **悬停状态**：主按钮变暗至 `#B85F1A`

### 箭头与连线

- 使用 SVG 绘制
- 箭头颜色：`#CC6B1E` 或 `#B8B8B8`
- 线条粗细：2px
- 虚线用于"弱关联"或"可选路径"

## 四、模式专用输出模板

### 精准诊断（模式A）

```
## 诊断结论：你当前最大的盲区是 [一句话结论]

### 透镜① [职业] — [核心问题]
**为什么匹配**：一句话解释关联
**你要回答的问题**：
- ...
- ...
**如果从这个视角看，你会发现**：...

### 透镜② [职业] — [核心问题]
...

---
## 行动优先级

🔴 立即做：...
🟡 本周做：...
🟢 持续观察：...

---
**下一步**：选一个透镜深入，或尝试强制跨界组合？
```

### 全视角扫描（模式B）

```
## 全视角扫描完成：发现 [N] 个高风险盲区

| 分组 | 透镜 | 核心提问 | 为什么可能是你的盲区 |
|------|------|---------|---------------------|
| 🔴 高相关 | Economist | ... | ... |
| 🟡 中相关 | Architect | ... | ... |
| 🟢 低相关（高风险盲区） | Artist | ... | 你从未从创意视角看过 |

---
## 行动优先级
🔴 ... | 🟡 ... | 🟢 ...
```

### 内容审查（模式E）

```
## 审查结论：[N] 处需要关注

### 🔴 [职业] [维度] — [N]处
- [位置] "..." → 具体问题
- ...

### 🟡 [职业] [维度] — [N]处
- ...

### 🟢 [职业] [维度] — [N]处
- ...

---
## 修订建议
[直接输出修改版或逐条建议]

---
## 行动优先级
🔴 必须改：... | 🟡 建议优化：... | 🟢 可考虑：...
```

### 认知偏差提示格式

```
你的判断是 [X]。如果用一个 [职业] 的视角，会看到一个有趣的点：
[职业] 会 [做/问] ...

这不是说你的判断错了，而是说这个视角可能会补充一个你还没考虑到的维度。
```

### 行动优先级格式规范

**每次输出必须有"行动优先级"**，不让思考停在洞察层面：

| 优先级 | 标记 | 含义 | 使用场景 |
|--------|------|------|---------|
| 立即 | 🔴 | 如果不做，问题会恶化或机会会流失 | 关键事实错误、核心风险未处理 |
| 本周 | 🟡 | 重要但不紧急，需要计划时间执行 | 优化建议、补充信息、调整策略 |
| 观察 | 🟢 | 值得注意，但目前无需行动 | 趋势信号、备选方案、长期积累 |

**规则**：
- 🔴 不超过 3 项
- 🟡 不超过 5 项
- 🟢 不超过 5 项
- 如果某优先级没有内容，不显示该分类

## 五、停止信号输出模板（认知负荷控制）

### 盲区覆盖裁决表（模式A）

每次精准诊断输出前，必须先给裁决表，让用户根据时间选择深度：

```markdown
### 📊 盲区覆盖裁决（停止信号）

| 优先级 | 透镜 | 覆盖的盲区 | 建议投入时间 |
|--------|------|-----------|-------------|
| 🔴 核心（必看） | [职业A] + [职业B] | 你最容易忽略的[X]和[Y] | 2-3分钟 |
| 🟡 补充（建议看） | [职业C] | 特定场景下的[Z]风险 | 1-2分钟 |
| 🟢 深度（可选） | [职业D] + [职业E] | 重大决策或时间充裕时展开 | 3-5分钟 |

> **时间不够？只看 🔴 核心盲区，覆盖约80%的关键遗漏。**
```

### 扫描收敛裁决表（模式B）

```markdown
### 📊 扫描收敛裁决（停止信号）

25 个透镜已全部扫描，但**不需要全部深入**。按投入产出比排序：

| 优先级 | 透镜 | 核心提问 | 为什么是你的盲区 | 投入建议 |
|--------|------|---------|----------------|---------|
| 🔴 核心（必看） | [职业A] | ... | ... | 3分钟 |
| 🔴 核心（必看） | [职业B] | ... | ... | 3分钟 |
| 🟡 补充（建议看） | [职业C] | ... | ... | 2分钟 |
| 🟢 深度（可选） | [职业D/E] | ... | ... | 有时间再展开 |

> **时间有限？只看 🔴 核心 2 个，覆盖 80% 的关键盲区。**
```

### 审查优先级裁决表（模式E）

```markdown
### 📊 审查优先级裁决（停止信号）

| 优先级 | 审查维度 | 核心检查点 | 建议投入时间 |
|--------|---------|-----------|-------------|
| 🔴 核心（必看） | Journalist 事实核查 | 数字/日期/人名/情绪化断言 | 1分钟 |
| 🔴 核心（必看） | Politician 受众感知 | 利益相关者的负面解读风险 | 1分钟 |
| 🟡 补充（建议看） | Novelist 叙事结构 | 开头是否回答"为什么重要" | 1分钟 |
| 🟢 深度（可选） | Designer 信息设计 + Critic 深度解读 | 格式优化+遗漏角度 | 2-3分钟 |

> **时间紧迫？只过 🔴 核心两项：事实有没有错，有没有人会被得罪。**
```

## 六、对话流中的格式切换

根据用户输入长度和复杂度，自动选择输出格式：

| 用户输入 | 推荐格式 | 原因 |
|---------|---------|------|
| 简短问题（<50字） | 紧凑列表 | 快速回应 |
| 具体问题 + 上下文 | 推荐透镜模板 | 结构化分析 |
| "帮我全面看看" | 全角度扫描表格 | 信息密度最高 |
| 贴了大段文档 | 核查点列表 + 表格 | 逐条对照 |
| "做成图/卡片" | HTML/SVG 可视化 | 视觉化呈现 |

---

## 七、禁止事项

- ❌ 不要使用衬线字体作为标题
- ❌ 不要使用深色/暗色主题
- ❌ 不要使用圆角超过 4px 的卡片
- ❌ 不要使用渐变背景
- ❌ 不要使用阴影效果
- ❌ 不要在标题中只放名词（如"分析结果"，应为"分析结果：5个盲区待补"）
- ❌ 不要一次性输出超过 3 个无关模块
- ❌ 不要把用户的情绪反应当成事实接受
