# 认知原理与问题分类映射

> 25透镜的深化层——从"职业视角"到"认知原理"，从"随机切换"到"策略匹配"。
> 配套 `thinking-lenses` Skill，按需加载。

---

## 第一部分：问题分类 → 最优透镜组合映射表（路径B）

### 使用说明

不是所有问题都需要25个视角。先判断问题类型，再调用对应的最小有效透镜组合（2-5个）。

如果问题**跨类型**（如"供应商不配合"既涉及人际又涉及系统），先选主类型，再补充次类型的1-2个透镜。

---

### 映射表

| 问题类型 | 核心盲区 | 推荐透镜组合 | 组合逻辑 |
|---------|---------|------------|---------|
| **人际冲突/沟通障碍** | 自我中心、知识诅咒、误解动机 | Psychologist + Teacher + Anthropologist + Salesperson | 先检验理解偏差→消除知识诅咒→沉浸对方文化→洞察真实动机 |
| **系统故障/根因不明** | 只看症状、忽视结构、不敢拆解 | Doctor + Engineer + Hacker + Plumber | 诊断根因→建模量化→看底层机制→动手拆解验证 |
| **战略决策/重大选择** | 过早收敛、忽视二阶效应、情绪绑架 | Economist + Philosopher + Chess Master + Politician | 算激励反应→推极限后果→预判多步→管感知预期 |
| **创意瓶颈/同质化** | 思维定势、路径依赖、缺乏差异化 | Artist + Entrepreneur + Critic + 强制跨界组合 | 解放约束→快速试错→二次创作→反直觉碰撞 |
| **执行偏差/流程失控** | 纪律松弛、标准模糊、关键遗漏 | Soldier + Programmer + Designer + Accountant | checklist保护→模式自动化→信息自解释→数据监控 |
| **方案/文档审查** | 叙事混乱、事实偏差、受众错位 | Journalist + Novelist + Designer + Politician + Critic | 事实核查→叙事结构→信息设计→感知管理→深度解读 |
| **业绩诊断/数据异常** | 归因错误、口径混乱、忽视比率 | Doctor + Accountant + Scientist + Engineer | 根因诊断→比率分析→假设验证→建模量化 |
| **团队激励/士气低落** | 激励错位、理解偏差、文化隔阂 | Economist + Psychologist + Teacher + Anthropologist | 对齐激励→检验理解→消除知识诅咒→沉浸文化 |
| **谈判/说服** | 误判筹码、忽视对方底线、未预判反制 | Salesperson + Chess Master + Politician | 洞察动机→预判多步→管理感知 |
| **制度/规则设计** | 理想化、忽视人性、激励错位 | Soldier + Programmer + Designer + Economist | 纪律保护→自动化→自解释→激励对齐 |
| **个人状态/决策畏缩** | 情绪干扰、恐惧未知、缺乏routine | Actor + Soldier + Plumber | 情绪代入→纪律routine→拆解恐惧 |
| **内容创作/表达优化** | 信息密度低、逻辑不清、缺乏说服力 | Novelist + Designer + Critic | 叙事结构→信息设计→深度解读 |
| **竞争策略/市场分析** | 忽视对手、只看自己、静态视角 | Chess Master + Anthropologist + Hacker | 预判对手→沉浸市场文化→看底层机制 |
| **资源有限/方向不明** | 完美主义、不敢试错、信息过载 | Entrepreneur + Artist + Journalist | 快速验证→解放约束→事实核查 |
| **跨部门协作/文化冲突** | 本位主义、信息不对称、目标错位 | Anthropologist + Teacher + Politician + Economist | 沉浸文化→消除知识诅咒→管理感知→对齐激励 |

---

### 组合逻辑速查

每种组合遵循"**诊断→建模→人际→推演**"的互补结构：

- **诊断类**（Doctor/Accountant/Engineer）：找到问题在哪
- **建模类**（Engineer/Architect/Programmer）：把问题结构化
- **人际类**（Salesperson/Psychologist/Teacher）：理解人的层面
- **推演类**（Chess Master/Philosopher/Politician）：看未来会怎样

一个有效组合通常包含**至少两类**透镜，避免同类型重复。

## 第二部分：职业 → 认知原理映射表（路径A）

### 核心发现

"职业"只是认知原理的**载体**，不是本质。真正起作用的是底层的**认知机制**。

---

### 映射表

| 透镜 | 职业标签 | 认知原理 | 原理一句话 |
|------|---------|---------|-----------|

| Economist | 经济学家 | **激励相容** | 任何涉及人的系统中，行为由激励驱动，改变激励就改变行为 |

| Entrepreneur | 企业家 | **快速验证** | 在不确定性中，速度和迭代比完美计划更有价值 |

| Salesperson | 销售 | **动机推断** | 通过语言和行为的不一致，推断对方未说出的真实需求 |
| Soldier | 军人 | **流程纪律** | 关键流程的严格执行比个人判断更可靠 |
| Chess Master | 象棋大师 | **多步推演** | 当前决策的价值取决于它引发的连锁反应 |
| Designer | 设计师 | **信息设计** | 形式本身就携带信息，好的设计不需要额外解释 |

| Anthropologist | 人类学家 | **文化沉浸** | 要理解一个群体，必须先成为其中一员 |
| Psychologist | 心理学家 | **认知偏差识别** | 人类的判断系统性偏离理性，识别偏差才能纠正 |

## 第三部分：认知原理分类体系

25个透镜按底层认知原理，可归为7大类：

### 1. 诊断类（找问题在哪）
**核心机制**：从症状到根因的推断

| 透镜 | 原理 | 独特贡献 |
|------|------|---------|
| Doctor | 根因分析 | 通过症状排除法找到最可能的病因 |
| Accountant | 比率诊断 | 通过数字间的关系发现隐藏模式 |
| Engineer | 系统建模 | 把模糊问题转化为可量化的变量关系 |
| Hacker | 系统分层 | 透过表象看底层机制的漏洞 |

**共同盲区**：容易过度分析，忽视人的情绪和非理性因素。

---

### 2. 建模类（把问题结构化）
**核心机制**：抽象、量化、预测

| 透镜 | 原理 | 独特贡献 |
|------|------|---------|
| Engineer | 系统建模 | 变量识别与关系构建 |
| Architect | 愿景构建 | 未来状态的完整 envision |
| Programmer | 模式自动化 | 重复流程的抽象与标准化 |
| Mathematician | 严谨性标准 | 结论的可证伪性检验 |

**共同盲区**：模型不是现实，过度依赖模型会忽视模型的假设边界。

---

### 3. 人际类（理解人）
**核心机制**：动机推断、认知差距弥合、文化理解

| 透镜 | 原理 | 独特贡献 |
|------|------|---------|
| Salesperson | 动机推断 | 通过行为反推未说出的需求 |
| Psychologist | 认知偏差识别 | 识别人类判断的系统性错误 |
| Teacher | 知识诅咒弥合 | 意识到"我知道"≠"你能懂" |
| Anthropologist | 文化沉浸 | 通过成为一员来理解群体逻辑 |

**共同盲区**：容易陷入"过度解读"，把简单行为复杂化。

---

### 4. 推演类（看未来）
**核心机制**：多步预判、极限检验、感知管理

| 透镜 | 原理 | 独特贡献 |
|------|------|---------|
| Chess Master | 多步推演 | 决策的连锁反应预判 |
| Philosopher | 极限推演 | 想法在极端条件下的崩溃点 |
| Politician | 感知管理 | 事实如何被不同群体解读 |

**共同盲区**：容易陷入"分析瘫痪"——想太多，做太少。

---

### 5. 创意类（破定势）
**核心机制**：约束释放、快速试错、意义重构

| 透镜 | 原理 | 独特贡献 |
|------|------|---------|
| Artist | 约束释放 | 暂时放下所有限制，探索可能性空间 |
| Entrepreneur | 快速验证 | 用速度和迭代替代完美计划 |
| Critic | 深度解读 | 在已有基础上构建新的意义层 |
| Novelist | 叙事构建 | 用故事赋予混沌以结构和意义 |

**共同盲区**：容易脱离现实约束，产生"创意但不可行"的方案。

---

### 6. 执行类（保落地）
**核心机制**：纪律、自动化、设计

| 透镜 | 原理 | 独特贡献 |
|------|------|---------|
| Soldier | 流程纪律 | checklist 和 routine 防止人为失误 |
| Programmer | 模式自动化 | 把复杂判断转化为可重复的规则 |
| Designer | 信息设计 | 形式本身引导正确行为 |
| Plumber | 动手拆解 | 通过实操获得真实理解 |

**共同盲区**：容易僵化——流程成为目的，而不是手段。

---

### 7. 求真类（去噪音）
**核心机制**：事实核查、假设检验、信源评估

| 透镜 | 原理 | 独特贡献 |
|------|------|---------|
| Journalist | 事实核查 | 独立来源验证，剥离情绪 |
| Scientist | 假设验证 | 可证伪的实验设计 |
| Mathematician | 严谨性标准 | 结论的前提条件和边界 |

**共同盲区**：追求完美的真相比追求有用的判断更慢，在紧急场景中可能过度延迟决策。

## 第四部分：与咏春拳哲学的对应关系

### 组合策略：咏春负责深度，透镜负责广度

| 咏春支柱 | 核心含义 | 对应的认知原理类别 | 具体透镜 |
|---------|---------|------------------|---------|
| **结构 > 力量** | 角度和框架优于蛮力 | **建模类** | Engineer、Architect、Programmer |
| **守中用中** | 控制核心通道，找到最优破局点 | **诊断类** | Doctor、Accountant、Hacker |
| **同时攻防** | 化解阻力同时直达目的 | **推演类** | Chess Master、Philosopher、Politician |
| **小念头** | 回到最原初的基本结构 | **求真类** | Journalist、Scientist、Mathematician |

### 实战组合公式

**先用透镜扫描（广度）→ 再用咏春破局（深度）**：

1. **问题出现** → 用**问题分类映射表**（路径B）选出2-5个最相关透镜
2. **透镜扫描** → 快速暴露盲区，收集多维度信息
3. **找到核心** → 用咏春"守中用中"定位最关键的1-2个破局点
4. **一击制胜** → 用咏春"结构>力量"设计最小调整撬动最大改变

**示例**：供应商不配合（跨类型问题）
- **广度扫描**：Economist（激励错位？）+ Salesperson（真实动机？）+ Hacker（组织层级卡在哪？）
- **找到核心**：发现是"结算周期"（Economist 发现的激励问题）
- **深度破局**：咏春"守中用中"→ 结算周期是核心通道，调整它同时解决激励和现金流
- **同时攻防**：调整结算周期的同时，建立新的评估机制（防止供应商得寸进尺）

## 第五部分：使用建议

### 对于新手（<20次使用）

- **直接套用映射表**：遇到问题→查表→调用推荐组合
- **不需要理解原理**：先用起来，在实战中感受每个透镜的"味道"

### 对于进阶（20-50次使用）

- **追问原理**：每次使用后，问自己"这个透镜的本质机制是什么？"
- **开始组合创新**：根据问题特殊性，微调映射表中的推荐组合

### 对于高手（50次+使用）

- **创造新透镜**：基于自己的职业经验，提炼第26、27个透镜
- **反向使用**：先用咏春找到破局点，再反推"哪个透镜本该帮我更早发现这个点"
