# 技能打磨

> 不是自动优化，是用户主导的迭代改进

---

## 核心原则

**用户确认是最终标准，不是分数。**

AI 可以提建议、跑测试，但只有用户说"更好"，改动才成立。

---

## 核心流程

```
1. 定义目标
   - 这个技能要解决什么问题？
   - "更好"意味着什么？

2. 准备测试用例
   - 触发测试：哪些输入应该触发？哪些不应该？
   - 输出测试：典型任务的预期输出是什么？

3. 运行基线测试
   - 用当前版本跑测试用例
   - 记录结果，作为对比基准

4. 提出改动
   - 单一改动原则：每次只改一个点
   - 说明假设：为什么这样改会更好？

5. 验证效果
   - 跑测试用例
   - 让用户确认：输出是否更好？

6. 用户决定
   - 用户说"更好" → 保留
   - 用户说"没区别/变差" → 撤回
   - 用户说"继续打磨" → 回到步骤 4

7. 用户决定停止
   - 用户说"够了"或"不用继续"
```

---

## 两类指标

### 形式指标（AI 辅助判断）

快速检查，辅助决策，**不是最终标准**。

示例：
- 触发描述是否清晰？
- 核心流程是否完整？
- 有没有明显冗余？

### 效果指标（用户最终判断）

核心标准，决定保留或撤回。

示例：
- 触发准确率是否提高？
- 输出质量是否更好？
- 使用体验是否改善？

**原则**：形式指标辅助，效果指标决定。

## 测试用例设计

### 触发测试

```yaml
场景: [技能名称] 触发测试

应该触发:
  - 输入: "帮我准备和军哥的汇报"
    预期: 触发（汇报场景）
  - 输入: "这个怎么说比较好"
    预期: 触发（润色意图）

不应触发:
  - 输入: "分析这个供应商的数据"
    预期: 不触发（供应商分析，不是表达润色）
  - 输入: "写个 Python 脚本"
    预期: 不触发（编程任务）
```

### 输出测试

```yaml
场景: [技能名称] 输出测试

测试 1:
  输入: "博岳这周不行了，达标率掉了很多"
  预期输出: 包含具体数据、原因分析、下一步行动
  评估: 用户打分 1-5

测试 2:
  输入: "我要和供应商谈价格，怎么说"
  预期输出: 专业、有数据支撑的谈判要点
  评估: 用户打分 1-5
```

### changelog.md 格式

```markdown
## 改动 #1

- **改动内容**: 压缩触发描述从 9 行到 3 行
- **假设**: 更精简的描述能提高触发准确率
- **测试结果**:
  - 触发测试: 5/5 通过
  - 输出测试: 用户打分 4/5（之前 3/5）
- **用户确认**: 更好
- **状态**: 保留

## 改动 #2

- **改动内容**: 添加智能推断规则
- **假设**: 根据上下文推断场景，减少追问
- **测试结果**:
  - 触发测试: 5/5 通过
  - 输出测试: 用户打分 3/5（没变化）
- **用户确认**: 没区别
- **状态**: 撤回
```

## 用户参与节点

| 节点 | 用户做什么 |
|------|------------|
| 定义目标 | 确认"更好"的标准 |
| 准备测试用例 | 确认测试用例是否覆盖关键场景 |
| 验证效果 | 对输出打分，说"更好/没区别/变差" |
| 决定保留/撤回 | 最终决定 |
| 决定停止 | 说"够了"或"继续打磨" |

---

## 常见问题

### Q: 要迭代多少轮？

没有固定轮数。用户说"够了"就停。

### Q: 测试用例要多少个？

最少：
- 触发测试：3-5 个（覆盖主要场景）
- 输出测试：2-3 个（典型任务）

### Q: 改动多少算"单一改动"？

原则：能说清"我改了什么"。

示例：
- "压缩触发描述"是一个改动
- "压缩触发描述 + 添加模板"是两个改动，应该分开

---

## 与原 prompt-optimizer 的区别

| 维度 | 原版 | 新版 |
|------|------|------|
| 标准 | AI 自己打分 | 用户确认 |
| 指标 | 只有形式指标 | 形式指标 + 效果指标 |
| 测试 | 没有强制要求 | 必须有测试用例 |
| 停止条件 | 连续 3 次达标 | 用户说停 |

## 使用示例

```
年老师：帮我打磨 work-expression 技能

AI：
好的，先定义目标。

【当前问题】
触发描述较长，可能影响触发准确率

【目标】
提高触发准确率，让该触发的触发，不该触发的不触发

【测试用例】
触发测试：
1. "帮我准备汇报" → 应触发
2. "这个怎么说" → 应触发
3. "分析供应商数据" → 不应触发
4. "写个脚本" → 不应触发

输出测试：
1. 输入"博岳这周不行了" → 预期：有数据、有原因、有下一步

确认这个目标和测试用例吗？
```
