# Agent 角色详解

> Agentic Orchestration 中的四个核心角色


## Spec Agent

### 职责

- 理解用户需求
- 编写 Spec 文档（功能目标、输入输出、业务规则）
- 定义验收标准

### 激活技能

`spec-driven`

### 产出模板

```markdown
# [功能] Spec

## 功能目标
[一句话描述功能目标]

## 业务规则
1. [规则 1]
2. [规则 2]
3. [规则 3]

## 输入输出
- 输入：[数据类型、格式]
- 输出：[数据类型、格式]

## 验收标准
- [ ] [标准 1]
- [ ] [标准 2]
- [ ] [标准 3]
```

### 质量检查

- [ ] 功能目标清晰可衡量
- [ ] 业务规则无歧义
- [ ] 输入输出定义完整
- [ ] 验收标准可验证


## Test Agent

### 职责

- 阅读 Spec 文档
- 拆解测试用例（一规则一测试）
- 定义预期输入/输出

### 激活技能

`spec-driven` + `testing-patterns`

### 产出模板

```markdown
## 测试用例清单

| 编号 | 名称 | 对应规则 | 输入 | 预期输出 |
|------|------|----------|------|----------|
| TC-001 | [名称] | 规则 1 | [输入] | [输出] |
| TC-002 | [名称] | 规则 2 | [输入] | [输出] |

## 测试场景覆盖
- [ ] 正常输入
- [ ] 边界输入
- [ ] 异常输入
- [ ] 错误处理
```

### 质量检查

- [ ] 每条业务规则有对应测试
- [ ] 覆盖边界情况
- [ ] 覆盖错误处理
- [ ] 预期输出明确可验证


## Code Agent

### 职责

- 阅读 Spec + 测试用例
- 实现功能（TDD 循环：红→绿→重构）
- 确保测试全部通过

### 激活技能

`tdd-agentic`

### 工作流程

```
1. 写测试 → 确认失败（红）
2. 写实现 → 测试通过（绿）
3. 重构代码 → 保持通过
```

### 产出模板

```typescript
// [函数名].ts

/**
 * [函数描述]
 * @param [参数] - [描述]
 * @returns [返回值] - [描述]
 */
export function [函数名]([参数]): [返回类型] {
  // 实现代码
}
```

### 质量检查

- [ ] 测试全部通过
- [ ] 代码符合规范
- [ ] 无冗余代码
- [ ] 错误处理完整


## Review Agent

### 职责

- 代码审查（逻辑正确性、可读性）
- 验证测试覆盖率
- 提出改进建议

### 激活技能

`requesting-code-review`

### 产出模板

```markdown
## 代码审查报告

### ✅ 优点
- [优点 1]
- [优点 2]

### ⚠️ 建议改进
- [建议 1]
- [建议 2]

### ❌ 必须修复
- [问题 1]
- [问题 2]

### 覆盖率验证
- [ ] 测试覆盖所有业务规则
- [ ] 测试覆盖边界情况
- [ ] 测试覆盖错误处理
```

### 质量检查

- [ ] 审查意见具体可操作
- [ ] 区分"建议"和"必须修复"
- [ ] 验证测试覆盖率
- [ ] 无遗漏重大问题


## 角色配合

```
Spec Agent → 定义"做什么"
    ↓
Test Agent → 定义"怎么算对"
    ↓
Code Agent → 实现"让它对"
    ↓
Review Agent → 验证"真的对"
    ↓
Human → 最终验收
```


*每个角色都有清晰的职责边界和产出标准。*
