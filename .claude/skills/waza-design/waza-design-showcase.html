waza:design v3.18

Design with
a point of view.

不是组件库，不是设计系统——是一套设计决策流程。确保每个界面都有态度，而不是 AI 默认生成的样子。

01

五问门控 — 写代码之前

五个问题全部回答完毕才能动手。不是建议，是硬性门槛。

Q1
谁在什么场景下使用？
分析师看板 ≠ 落地页 ≠ 引导流程
不同场景决定完全不同的设计策略。分析师看板要数据密度、扫描效率；落地页要情感共鸣、行动驱动；引导流程要认知负荷管理、进度暗示。

App Shell 特殊处理：如果答案是"侧栏+主工作区"（Slack/Linear/Notion 类），走 App Shell 模式：关闭装饰性背景、只用背景色阶梯建层次、所有交互元素加 active:scale-95。
Q2
美学方向是什么？
必须精确命名——"干净现代"不是方向
精确命名：密集编辑风、原始终端感、纸上墨迹、粗野主义网格、温暖模拟感。

如果用户说"像 Linear"：不接受"像 Linear"作为方向。从 Linear 提取 3 个具体属性：
① 按钮圆角哲学（小圆角，非 pill）
② 表面深度处理（微妙阴影阶梯，非边框）
③ 强调色家族（紫蓝色系，低饱和）

品牌快捷方式：Linear/Stripe/Claude/Vercel/Apple 等 66+ 品牌有预制 DESIGN.md，可问用户是否拉取。
Q3
留在记忆里的一样东西是什么？
一个字体、一个色彩系统、一个意外的动效、一个不对称布局
这个问题强制做减法。不是"这个页面有字体+颜色+动效+布局"，而是只有一个需要让人记住。其他的都是配角。

如果选了"一个大胆的 Display 字体"，色彩和布局就应该克制。如果选了"一个意外的动效"，排版就该安静。
Q4
硬约束是什么？
框架、包大小、对比度最低值、键盘可访问性
技术约束决定美学自由度。如果必须用 Tailwind，就不能同时用 CSS Modules。如果 bundle 不能超过 50KB，就不能引入大型动画库。

必须在第一个组件前确定 CSS 策略：Tailwind only / CSS Modules only / CSS-in-JS only。三选一，不混用。
Q5
标志性微交互是什么？
按下缩放、交错入场、上下文图标动画——选一个，精确实现
微交互是设计的签名。不能"都要"——选一个做到极致。

按下缩放：按钮 active 时 scale(0.96)，CSS transition 实现，可中断
交错入场：列表项依次出现，~100ms 间隔，opacity + translateY + blur
图标动画：切换时 scale(0.25)→1 + opacity 0→1，120ms 交叉淡入
02

从方向到交付

7 步流程。先问清楚、再定义方向、再动手。截图迭代走另一条路。

S0
五问门控 — 回答五个方向问题
▸
所有五个问题必须在写代码前回答完毕。详见「五问门控」。
S1
读基础文件 — 加载 brand-dna.md + tokens.md
▸
提取实际的色彩值、字体栈、间距刻度。不从记忆推断，从文件读取。
S2
竞品参照 — 列出 2-3 个同类产品
▸
每个产品写一句话说明它如何解决同一问题。Linear 靠微妙阴影建层次，Notion 靠留白组织信息，Stripe 靠数据密度传递专业感。
S2.5
视觉论文 — 三行代码前必须输出
▸
① 视觉论文：情绪、材质、能量——一句话（如"温暖粗野主义编辑风，高对比墨字配粗纸纹理"）
② 内容计划：Hero → 支撑 → 细节 → CTA（工具型跳过 Hero）
③ 交互论文：2-3 个具体动效想法
S3
按需组装 — 从规则库选取组件，不套模板
▸
生产级 UI 扩展为 9 段式 DESIGN.md（主题、色板、字体、组件、布局、深度、Do/Don't、响应式、Prompt 指南）。单组件用三行论文即可。
S4
截图迭代 — 发图说哪里不好，基于此改
▸
收截图 → 一句话诊断 → 等确认 → 查竞品解法 → 找代码 → 最小修复 → 浏览器验证。每次只改一个组件、一个问题。
S5
AI Slop 测试 + 交付 — 陌生人会觉得是 AI 做的吗？
▸
首屏扫一眼。如果答案是"是"，修复。全宽 + 375px 双验证。交付后停止。
03

截图迭代模式

发一张截图 + 一句抱怨（"这里很丑"、"这个不对"），走这条专用路径。跳过五问门控。

01
读截图 — 一句话诊断问题
▸
具体到视觉层面：是间距问题？对比度不足？对齐偏移？字体不搭？配色冲突？不能说"不好看"，要说"标题和正文的间距不够，导致层次感缺失"。
02
等确认 — 用户同意诊断再动手
▸
不假设。诊断可能是错的。等用户说"对，就是这个问题"再改。
03
查竞品 — 知名 UX 问题先调研
▸
如果诊断是已知 UX 问题（分屏同步、无限滚动、虚拟列表、粘性头部），先花一轮调研 2-3 个同类产品怎么做的。纯视觉问题（颜色、间距、文案）可跳过。
04
找代码 — grep 组件名，读实际文件
▸
不靠记忆猜文件位置。grep 组件名或 class，读实际源码。
05
最小修复 — 一个组件、一个问题
▸
如果修复需要改 3 个以上组件，或暴露的是方向问题而非具体 bug → 暂停，重新走五问门控。
06
浏览器验证 — 用户确认修复
▸
不在脑子里判断"看起来对"。代码在脑子里正确 ≠ 浏览器里正确。让用户在浏览器里验证。

重设计优先级

改造现有 UI 时，按这个顺序来——最大化视觉提升，最小化影响范围。

字体替换
色彩清理
悬停/激活态
布局间距
替换通用组件
加载/空/错状态
排版精修
04

绝对禁止清单

这些 CSS 模式和设计决策被硬性禁止。违反任何一条，设计即不合格。

紫蓝渐变 / cyan-on-dark
AI 生成界面的标志性配色。传达不了任何品牌信息。
从品牌关键词通过 OKLCH 规则推导色板
backdrop-filter: blur 作为默认卡片
低端设备性能差；过度使用；实线边框会打碎层次幻觉。
用背景色阶梯 + box-shadow 建立表面层次
transition: all
强制浏览器每帧重算布局，导致卡顿。
列出具体属性；高度展开用 grid-template-rows: 0fr→1fr
弹跳/弹性缓动 (bounce, elastic)
真实物体是平滑减速的，不是弹跳的。
cubic-bezier(0.16, 1, 0.3, 1) — 指数级 ease-out
Modal 作为溢出 UI 的逃避方案
打断流程，破坏浏览器后退导航。
内联展开、详情面板、或独立路由。Modal 只在需要焦点锁定时用
视差滚动 / scroll-jacking
剥夺用户控制权，违反滚动预期。
不使用。用户滚动就是滚动。
background-clip: text 渐变文字
装饰性大于可读性；打印和高对比模式下不可读；AI 设计标志。
用品牌纯色、色调中性色、或字重变化
圆角 > 12px（卡片最大 8px，按钮 4px 或 0）
大圆角是模板思维的信号。
在方向锁定时定义命名圆角刻度（3-4 级）
Inter / DM Sans / Playfair 等作为 Display 字体
LLM 训练数据中过度代表。用它们 = "没有做决定"。
字体选择流程：三个品牌词 → 列三个直觉选择 → 全部拒绝 → 从命名铸字厂选
AI 陈词滥调 (Elevate, Seamless, Unleash, Delve...)
传达不了产品价值。
说具体的价值，不说空洞的形容词
05

动效系统

有机的，不是机械的。只动画 transform 和 opacity。点击下方方块体验。

scale0.96
lift-4px
focusring
fillfade
类型时长缓动用途
微交互150msease-out按钮状态、悬停、聚焦
标准过渡250msease-out面板展开、标签切换
强调入场400msease-out页面元素入场、模态框
交错入场~100ms 间隔ease-out列表项、卡片依次出现
退场150msease-in固定 translateY(-12px)

入场序列

opacity 0→1 + translateY(12px)→0 + blur(4px)→0
同时进行，400ms ease-out。只动画 transform 和 opacity——合成器友好。
图标切换: scale(0.25)→1 + opacity 0→1
120ms 交叉淡入，两个图标都在 DOM 中（一个 absolute），CSS 切换。
高度展开: grid-template-rows: 0fr→1fr
不动画 height。CSS grid 行过渡是唯一正确方式。

技术约束

will-change 只用于 transform/opacity/filter
仅在首帧卡顿时添加，不预判性地给每个元素加。
尊重 prefers-reduced-motion
用户设置了减少动效 → 禁用或缩减。
按钮 active:scale(0.96) 用 CSS transition
可中断。JavaScript 动画库的弹簧效果不可中断。
06

内容真实性

看起来真实但不是真实的占位内容，用户一读就破功。

✕
数据样本
不用假名、假公司、假数据
不用 John Doe、Jane Smith → 用文化多样且具体的名字（Priya Mehta, Lars Eriksson, Nia Okafor）
不用 Acme Corp → 用有领域具体性的名字（Meridian Logistics, Hokkaido Ceramics）
不用 Lorem Ipsum → 写符合阅读水平的真实短文案
不用整数 → 99.94%、47.2%、$99.00 而非 100%、50%、$100
多个头像不共享同一张图
✕
UI 文案
句首大写、不感叹、不被动、不用 AI 词
所有标题用 Sentence case（Title Case 是最常见的 AI 痕迹）
成功状态去掉感叹号（"Saved" 不是 "Saved!"）
错误信息不用 "Oops!" 开头（居高临下）
错误信息不用被动语态 → "We couldn't load your data. Try refreshing."
禁用 AI 营销词：Elevate, Seamless, Unleash, Delve, Tapestry, Game-changer, Next-Gen
!
占位优于模仿
没有素材时用占位符，不用低质量 SVG 画图
灰色矩形 = hero 图片占位
字母组合 = 缺失 logo 的替代
虚线边框 = 未设计的组件

绝不用内联 SVG 画插图。SVG 只做图标和几何形状。摄影、插画、产品截图用占位符，请用户补素材。

08

DESIGN.md 骨架

生产级 / 多页 UI 必须在写第一个组件前输出这 9 段式设计文档。单组件用三行论文即可。

视觉主题与氛围 — 情绪、密度、设计哲学，2-3 句
色彩方案与角色 — 语义名称 + 色值 + 功能角色
字体规则 — 字体族、尺寸刻度、字重、行高、字间距；超过 4 级用表格
组件样式 — 按钮（所有状态）、卡片（如果用）、输入框、导航；每个写 default/hover/active/disabled
布局原则 — 间距刻度、栅格列数、留白哲学
深度与层级 — 阴影系统或背景色阶梯系统；描述每个层级
Do's and Don'ts — 5-10 条针对此项目的护栏，不是通用规则
响应式行为 — 断点、导航折叠方式、触控目标最小值
Agent Prompt 指南 — 色彩快速参考（名称:值对）+ 3-5 个可直接粘贴的组件 Prompt
09

AI Slop 测试

陌生人扫一眼首屏——会觉得"这是 AI 做的"吗？如果答案是"是"，修复排版、色彩或布局直到答案翻转。

✓
首屏有 reflex 字体？（Inter、DM Sans、Space Grotesk 等黑名单字体作为标题）
✓
有紫蓝渐变或 cyan-on-dark？
✓
居中大标题 + 两个 CTA 并排？
✓
三个完全相同的卡片（圆角、阴影、内边距一致）？
✓
通用顶栏导航（logo 左、链接中、操作右）？
✓
居中图标 + 标题 + 段落的三段式重复？
✓
Section 在白色和 #f9f9f9 之间交替？
✓
四列等宽页脚？

常见陷阱

如果换了完全不同的内容，布局不需要改就能用——你做的是模板，不是设计。重做。
