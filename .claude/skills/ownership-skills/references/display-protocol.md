# 展示协议

> Ownership Skills 的状态面板与标记规范

---

## 核心标记

执行过程中用以下标记记录关键节点。标记用方括号 `[标记名]` 格式。

| 标记 | 含义 | 使用时机 | 输出示例 |
|------|------|----------|----------|
| `[Owner Check]` | 责任确认 | 接到任务时，确认理解需求边界 | `[Owner Check] 任务理解确认：修复登录接口 404 问题` |
| `[验证节点]` | 验证完成 | 完成关键验证步骤，附证据 | `[验证节点] 修复验证：curl 返回 200` |
| `[自检]` | 主动检查 | 执行中发现风险或主动扩展检查 | `[自检] 发现同类问题：注册接口可能也有此 bug` |
| `[复盘点]` | 经验沉淀 | 任务完成后，记录可复用经验 | `[复盘点] 根因：未处理空指针` |
| `[挑战升级 L{n}]` | 难度升级 | 第 n 次失败后升级 | `[挑战升级 L3]` |

## 任务状态面板

复杂任务（>3 步）启动时，输出简洁状态面板。

### 启动面板

```
┌─────────────────────────────────────┐
│ [Owner Check] 任务启动               │
├─────────────────────────────────────┤
│ 任务：{简述}                         │
│ 挑战等级：L{n}                       │
│ 预期交付：{标准}                     │
└─────────────────────────────────────┘
```

### 验证面板

关键验证完成时：

```
┌─────────────────────────────────────┐
│ [验证节点] 关键验证完成              │
├─────────────────────────────────────┤
│ 验证项：{内容}                       │
│ 结果：{输出摘要}                     │
│ 证据：{命令输出或截图}               │
└─────────────────────────────────────┘
```

### 挑战升级面板

升级时输出：

```
┌─────────────────────────────────────┐
│ [挑战升级 L{n}] 进入攻坚模式         │
├─────────────────────────────────────┤
│ 已尝试方案：                         │
│ 1. {方案 A} → {结果}                 │
│ 2. {方案 B} → {结果}                 │
│ 共同模式：{发现}                     │
│ 下一步：{新方案}                     │
└─────────────────────────────────────┘
```

### 复盘面板

任务完成时：

```
┌─────────────────────────────────────┐
│ [复盘点] 任务完成                    │
├─────────────────────────────────────┤
│ 目标：{原始目标}                     │
│ 交付：{实际交付}                     │
│ 弯路：{根因分析}                     │
│ 经验：{可复用模式}                   │
└─────────────────────────────────────┘
```

## 绘制规范

### Unicode 框线字符

| 字符 | 用途 |
|------|------|
| `┌` `┐` `└` `┘` | 四角 |
| `─` | 横线 |
| `│` | 竖线 |
| `├` `┤` | 左/右 T |
| `┬` `┴` | 上/下 T |
| `┼` | 十字 |

### 尺寸规范

- 面板宽度：统一 37 字符（不含边框）
- 内容左对齐，前后各 1 空格
- 多行内容自动换行，保持对齐

### 密度控制

| 任务复杂度 | 面板使用 |
|-----------|----------|
| 简单（≤3 步） | 仅 `[Owner Check]` 单行标记 |
| 中等（4-6 步） | 启动面板 + 关键验证 |
| 复杂（≥7 步） | 完整面板流程 |

## 自检清单展示

执行后自检用以下格式：

```
[自检] 执行后验证
□ 修复验证：{命令} → {结果}
□ 同类检查：{发现}
□ 上下游确认：{影响}
□ 边界覆盖：{边界 case}
□ 主动补充：{额外工作}
```

全部完成后：

```
[验证节点] 5项自检完成
✓ 修复验证 ✓ 同类检查 ✓ 上下游 ✓ 边界 ✓ 补充
```

```
┌─────────────────────────────────────┐
│ 7 项检查清单（L4 强制）               │
├─────────────────────────────────────┤
│ □ 1. 逐字读完失败信号                │
│ □ 2. 用工具搜索核心问题              │
│ □ 3. 读失败位置的原始上下文          │
│ □ 4. 所有假设都用工具确认            │

│ □ 7. 换过工具/方法/角度/框架         │
└─────────────────────────────────────┘
```

完成后：

```
[验证节点] 7项检查清单完成
✓ 1 ✓ 2 ✓ 3 ✓ 4 ✓ 5 ✓ 6 ✓ 7
```

## 子 Agent 责任传递

Spawn 子 Agent 时，在 prompt 末尾添加：

```
---
[Owner Check] 子任务责任传递
你对此子任务拥有 Owner 责任。执行要求：
1. 穷尽方案前不说"无法解决"
2. 交付时必须附验证证据
3. 修复后检查同类问题
4. 完成后输出 [复盘点] 总结

参考：{skill_path}/ownership/SKILL.md
```

## 示例：完整任务流程

```
[Owner Check] 任务启动：修复 API 404

┌─────────────────────────────────────┐
│ [Owner Check] 任务启动               │
├─────────────────────────────────────┤
│ 任务：修复 /api/login 404            │
│ 挑战等级：L1                         │
│ 预期交付：curl 返回 200，附证据      │
└─────────────────────────────────────┘

→ 执行方案 A：检查路由配置
→ 失败：路由配置正确

[挑战升级 L2] 切换方案

┌─────────────────────────────────────┐
│ [挑战升级 L2] 进入调整模式           │
├─────────────────────────────────────┤
│ 已尝试：检查路由配置                 │
│ 结果：配置无误                       │
│ 下一步：检查中间件加载顺序           │
└─────────────────────────────────────┘

→ 执行方案 B：调整中间件顺序
→ 成功

[验证节点] 修复验证完成

┌─────────────────────────────────────┐
│ [验证节点] 关键验证完成              │
├─────────────────────────────────────┤
│ 验证项：登录接口可用性               │
│ 结果：curl /api/login → 200 OK       │
│ 证据：HTTP/1.1 200 OK...             │
└─────────────────────────────────────┘

[自检] 主动检查同类问题
→ 发现 /api/register 有相同问题，一并修复

[复盘点]

┌─────────────────────────────────────┐
│ [复盘点] 任务完成                    │
├─────────────────────────────────────┤
│ 目标：修复登录 404                   │
│ 交付：修复 login + register          │
│ 弯路：一开始没怀疑中间件顺序         │
│ 经验：404 先看路由，再看加载顺序     │
└─────────────────────────────────────┘
```
