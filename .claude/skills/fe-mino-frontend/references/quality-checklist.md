# 质量检查清单

> **用途**：高质量输出时的自检参考。mino-frontend 默认执行基础检查（视口适配/无阴影/无渐变），完整自检按需读取。
>
> **触发场景**：用户说"高质量"、"严格检查"、或使用克制专业主义等严肃风格时。
>
> **检查顺序**：设计 Litmus → AI Slop → 11 分制 → 反模式

---

## 零、设计 Litmus（高层检验，最先执行）

> 来源：OpenAI Codex Litmus Checks。快速判断设计是否到位。

逐项检查（任一不通过需返工）：

| # | 检验 | 通过标准 |
|---|------|---------|
| 1 | **首屏品牌** | 品牌/产品名在第一个视口中 unmistakable |
| 2 | **视觉锚点** | 页面有一个强视觉锚点（图像/大标题/数据矩阵） |
| 3 | **标题可懂** | 只扫标题就能理解页面全部内容 |
| 4 | **每节一职责** | 每个 section 只做一件事：explain / prove / deepen / convert |
| 5 | **卡片必要** | 每个卡片都有存在的理由（是交互对象？删除不影响信息？） |
| 6 | **动效有用** | 动效改善层级或氛围，不是纯装饰 |
| 7 | **去阴影仍高级** | 删掉所有装饰性阴影后，设计仍然感觉 premium |

## 一、AI Slop 自检（反 AI 味检测）

> **来源**：impeccable 技能的 AI Slop Test

**核心问题**：给别人看，会不会一眼认出"这是 AI 生成的"？

逐项检查（任一命中即需返工）：

| # | AI 痕迹 | 检测方法 | 正确做法 |
|---|---------|---------|---------|
| 1 | **渐变泛滥** | ≥2 处渐变 | 纯色 + 透明度变化，渐变仅特殊强调 |
| 2 | **圆角统一** | 所有元素同一圆角 | 按风格速查表取值 |
| 3 | **居中偏执** | 文字/卡片/数据全部居中 | 标题/正文左对齐，仅金句/封面居中 |
| 4 | **卡片堆砌** | 每页等大卡片网格 | 不对称布局、差异化信息层级 |
| 5 | **Hero 指标模板** | 大数字+小标签+渐变装饰 | 进度条/徽章/排名变化 |
| 6 | **间距均匀** | 所有区域相同 padding/margin | 4pt 基准间距制造节奏 |
| 7 | **阴影堆砌** | 多层 box-shadow | 边框/背景色差/留白区分层级 |
| 8 | **纯黑纯白** | 用了 `#000000` 或 `#FFFFFF` | `#111111` 和 `#FAFAFA` |
| 9 | **装饰性图标** | 每个标题上方都有大圆角图标 | 图标只在有语义时使用 |
| 10 | **Motion 过度** | 多元素同时弹跳/旋转/渐变 | 最多 1 个主入场动画 |

## 二、质量自检清单（11 分制）

### 第一层：内容编辑（3 分）

- [ ] **核心观点**：能用一句话说清楚这页的核心观点吗？
- [ ] **减法原则**：删掉了所有不影响核心观点的文字吗？
- [ ] **信息密度**：视觉块数量≤7 个吗？

### 第二层：设计判断（3 分）

- [ ] **密度选择**：简约/标准/丰富 的选择有明确理由吗？
- [ ] **版式匹配**：为什么用这个版式？内容特征匹配吗？
- [ ] **焦点唯一**：陶土色只用在一处吗？

### 第三层：视觉执行（5 分）

- [ ] **顶部装饰线**：2.4px 陶土色线存在吗？（克制专业主义风格）
- [ ] **页码徽章**：非封面页有右下角圆形页码吗？
- [ ] **无阴影无渐变**：检查所有元素
- [ ] **文字对比度**：所有文字能清晰阅读吗？
- [ ] **视口适配**：100vh, overflow: hidden 吗？

### 评分标准

| 层级 | 总分 | 通过线 |
|------|------|--------|
| 内容编辑 | 3 | ≥2 |
| 设计判断 | 3 | ≥2 |
| 视觉执行 | 5 | ≥4 |

**总计 ≥8 分可交付，<8 分需要返工**

## 三、反模式清单

> **来源**：ui-design-brain 的核心反模式 + 8px 网格最佳实践

| # | 反模式 | 正确做法 |
|---|--------|---------|
| 1 | 标题跳级（h1 直接到 h3） | 严格 h1→h2→h3 层级 |
| 2 | 间距不一致 | 同类元素用同一间距变量 |
| 3 | 触摸目标过小（<44px） | 移动端最小 44px |
| 4 | 卡片嵌套卡片 | 扁平化，一个层级一个容器 |
| 5 | 所有按钮都是主按钮 | 一个区域一个 primary |
| 6 | placeholder 当 label | label 永远可见 |
| 7 | 空状态只说"无数据" | 空状态 = 插图 + 标题 + 引导 |
| 8 | spinner 用太多 | 骨架屏 > spinner |

### 8px 网格基线

mino-frontend 已有 4pt 基准间距系统（4, 8, 12, 16, 24, 32, 48），与 8px 网格兼容。
- 所有间距值必须是 4 的倍数
- 紧密关联元素用 8px
- 分组内元素用 12-16px
- 层级分隔用 24px+

## 四、设计失败模式（Design Rejects）

> **来源**：OpenAI Codex "Reject These Failures"。设计层面的典型失败，与技术反模式互补。

| # | 失败模式 | 诊断方法 | 正确做法 |
|---|---------|---------|---------|
| 1 | **SaaS 卡片网格首屏** | 首屏是等大卡片网格，没有主视觉 | 一个强视觉锚点 + 层级差异 |
| 2 | **美图弱品牌** | 图片很强但品牌名看不清 | 品牌名必须是页面最响的文字 |
| 3 | **强标题无行动** | 标题很震撼但不知道要做什么 | 每个 hero 必须有明确 CTA |
| 4 | **忙图后文字** | 图片很忙（嵌入文字/标识）导致 UI 文字不可读 | 选稳定色调区域的图 |
| 5 | **重复 mood** | 多个 section 表达同一个情绪 | 每个 section 一个独特职责 |
| 6 | **无叙事 carousel** | 轮播图没有叙事推进，只是填充 | 用独立 section 或时间线 |
| 7 | **卡片堆叠** | App UI 用卡片堆叠代替布局 | 用表格/列表/纯布局 |

## 四、基础必查项（每次都执行）

不读这个文件也要检查的：

- [ ] 视口适配：100vh, overflow: hidden
- [ ] 无阴影
- [ ] 无渐变（品牌模式纸张纹理除外）
- [ ] 无 emoji 图标
- [ ] 字号使用 clamp()

## 五、可用性检查（Usability Test 三维度）

> 来源：UX Designer Skill Lab — Usability Test 方法论。适用于 Dashboard、App UI、Landing Page。

生成完成后，用以下三维度快速自检：

### 1. 有效性（Effectiveness）

| 检查项 | 通过标准 |
|--------|---------|
| **目标达成** | 用户能否完成核心任务？（找到信息/执行操作/理解内容） |
| **导航清晰** | 用户能否知道自己在哪、能去哪？ |
| **错误预防** | 是否有防错机制？（确认对话框、撤销操作、输入验证） |

### 2. 效率（Efficiency）

| 检查项 | 通过标准 |
|--------|---------|
| **路径最短** | 完成任务需要的最少步骤是多少？能否更少？ |
| **信息获取** | 关键信息是否一眼可见？不需要滚动或点击？ |
| **操作成本** | 高频操作是否触手可及？（< 2 次点击/交互） |

### 3. 满意度（Satisfaction）

| 检查项 | 通过标准 |
|--------|---------|
| **视觉愉悦** | 设计是否让人感觉专业、可信、舒适？ |
| **响应反馈** | 操作后是否有即时反馈？（hover 状态、加载指示、成功提示） |
| **情绪匹配** | 视觉风格是否匹配内容情绪？（严肃/庆祝/紧迫） |

**触发条件**：
- 类型为 Dashboard / App UI / Landing Page
- 用户说"检查可用性"、"用户体验检查"
