# Presentation Design

Design principles and best practices for creating effective presentation slides.

## Core Principles

### 1. One Idea Per Slide
- Each slide should communicate one key message
- If you have multiple points, split them into multiple slides
- The audience should grasp the main point in 3 seconds

### 2. Visual Hierarchy
```
Size + Color + Position = Importance
```
- Title: Largest, most prominent
- Key points: Medium size, accent color
- Supporting details: Smaller, neutral color

### 3. The 10-20-30 Rule (Guy Kawasaki)
- **10 slides** maximum
- **20 minutes** presentation length
- **30 point font** minimum

## Slide Structure

### Title Slide
- Presentation title (clear, specific)
- Your name and role
- Date or event
- Optional: subtitle or tagline

### Content Slides
```
┌─────────────────────────────┐
│         HEADLINE             │  ← Clear, action-oriented title
├─────────────────────────────┤
│                              │
│  • Key point 1               │  ← 3-5 bullet points max
│  • Key point 2               │  ← Use parallel structure
│  • Key point 3               │
│                              │
│  [Supporting visual/quote]   │  ← Reinforce message visually
└─────────────────────────────┘
```

### Transition Slides
- Use between major sections
- Simple format: "Part 2: Analysis"
- Helps audience reorient

## Typography

### Font Guidelines
- Sans-serif for slides (more readable)
- Consistent font family throughout
- Size hierarchy:
  - Title: 40-60pt
  - Headings: 32-40pt
  - Body: 24-28pt
  - Minimum readable: 18pt

### Font Pairings
| Category | Fonts |
|----------|-------|
| Safe | Arial, Calibri, Helvetica |
| Modern | Gotham, Proxima Nova, Open Sans |
| Professional | Garamond, Georgia (serif accents) |

## Color

### Color Guidelines
- **60-30-10 rule**:
  - 60% neutral (white/light gray)
  - 30% primary color
  - 10% accent color
- Limit to 2-3 colors total
- Ensure high contrast for text

### Accessible Combinations
- Dark text on light background
- Light text on dark background
- Avoid: red/green, blue/yellow together (colorblindness)

## Visual Design

### Images
- Use high-quality images only
- One image per slide maximum
- Images should support, not decorate
- Crop to focus on relevant portion

### Charts and Graphs
- Simplify: remove gridlines, axes labels
- Use direct labels (no legend)
- One data story per chart
- Annotate key insights

### Diagrams
- Flow from left to right
- 3-7 elements maximum
- Consistent shapes and colors
- Clear visual hierarchy

## Common Mistakes to Avoid

| Mistake | Solution |
|---------|----------|
| Too much text | Use bullets, cut words |
| Reading slides | Use notes, speak to audience |
| Low contrast | Test on projector, use dark text on light |
| Fancy animations | Keep simple or avoid |
| Clip art | Use professional photos or icons |

## Presentation Flow

### Opening
1. Hook: surprising fact, question, story
2. Preview: what you'll cover
3. Agenda: slide with 3-5 main points

### Body
- One idea per slide
- Transitions between sections
- Stories and examples
- Data and evidence

### Closing
1. Summary: 3 key takeaways
2. Call to action: what to do next
3. Q&A: invite questions
4. Contact: email/website

## Practice Tips

1. **Rehearse out loud** - timing and flow
2. **Get feedback** - test with 1-2 people
3. **Equipment check** - test on actual projector
4. **Backup plan** - PDF version, offline notes
5. **Arrive early** - set up without stress
