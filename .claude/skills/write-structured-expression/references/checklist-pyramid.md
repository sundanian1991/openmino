# Checklist九问与金字塔原理

> 动笔前的快速自检工具。独立于主流程，用户自行使用。

---

## 三场景九问Checklist

**使用时机**：动笔/开口前使用，不是写完后的收尾检查。

| 场景 | 核心问题 | 具体标准 | 原理 |
|------|----------|----------|------|
| **写作** | Q1: 开头给结论了吗？ | 删掉第一段，读者仍能直接抓到重点 | 结论先行 |
| | Q2: 分论点同一范畴吗？ | 避免业务/HR/技术等不同领域混为一谈 | 归类分组 |
| | Q3: 论据之间有顺序吗？ | 时间/结构/程度/因果，四选一 | 逻辑递进 |
| **汇报** | Q4: 30秒能抓到重点吗？ | 第一张slide必须传达中心结论 | TOP-DOWN |
| | Q5: 用了SCQA开场吗？ | 情境→冲突→疑问→回答 | 故事化表达 |
| | Q6: 每页能归入一个论点吗？ | 游离页要么合并，要么删除 | 纵向支撑 |
| **思考** | Q7: 所有要点能归纳成一句吗？ | 不能则说明思考不清晰 | 中心思想提炼 |
| | Q8: 分类是MECE的吗？ | 无重叠、无漏项 | MECE |
| | Q9: 用了演绎还是归纳？ | 默认归纳，除非必要才用演绎 | 推理类型 |

## 金字塔四法则

### 1. 结论先行

先抛中心思想，再展开论据。违反直觉但高效。

- ✅ "申请8万元预算挽留核心客户"
- ❌ 铺垫背景三分钟后才说要什么

### 2. 以上统下

```
中心思想
↓   ↓   ↓
论点A 论点B 论点C
↓↓    ↓↓    ↓↓
论据a1a2 论据b1b2 论据c1c2
```

自检：每个分论点是否回答上层问题？中心思想能否从分论点归纳得出？

### 3. 归类分组

- ❌ "明年三大战略：进军东南亚、调整组织架构、总部搬到浦东"（业务/管理/行政混为一谈）
- ✅ "明年三大业务战略：东南亚市场拓展、产品线下沉至SMB、与Top3渠道商合作"（同属业务范畴）

### 4. 逻辑递进

| 顺序 | 用法 |
|------|------|
| 时间 | 按流程步骤 |
| 结构 | 按空间/部门划分 |
| 程度 | 按重要性排序 |
| 因果 | 问题→原因→解决方案 |

## SCQA框架

| 要素 | 说明 | 示例 |
|------|------|------|
| S 情境 | 读者已知的事实 | "Q3营收已达成103%" |
| C 冲突 | 打破现状的变化 | "Q4核心客户提出退场" |
| Q 疑问 | 自然引出的问题 | "那我们该怎么办？" |
| A 回答 | 用中心思想回应 | "申请8万元预算挽留客户" |

## MECE原则

- ❌ 用户分"新用户/老用户/活跃用户"（活跃与前两类重叠）
- ✅ 用户分"注册≤30天/注册>30天"（边界清晰无重叠）

---

## 两种构建方法

| 方法 | 步骤 | 适用场景 |
|------|------|----------|
| 自上而下 | 中心思想→设想读者疑问→用论据回答 | 问题清晰、结论明确（日常汇报） |
| 自下而上 | 列出所有要点→按属性归类→提炼中心思想 | 信息杂乱、结论未定（行业研究） |

---

## 三大常见误区

1. **"三点式"陷阱**：为凑三点强行分组，忽视逻辑关联性
2. **有结论无论据**：只说"需要增加预算"，不说明"为什么/怎么用"
3. **分类不MECE**：如"质量差、包装丑、价格高、用户不喜欢"（最后一项与前三项重叠）

## 补充规则

- **7±2法则**：大脑短期记忆一次处理5-9个信息块，分论点控制在3-5个
- **30秒原则**：高管平均30秒决定是否继续关注，首屏必须直击核心
