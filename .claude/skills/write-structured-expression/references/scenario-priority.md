# 场景3：优先级模糊 — "事太多，先做什么"

**触发信号**：待办清单很长、资源有限、各方都在抢优先级。
**叙事结构**：漏斗叙事——先展示全量，再说筛选标准，最后落到优先列表
**一句话模板**：

> "总共[数量]个事项。按[维度1]和[维度2]评估后，第一批必做[数量]个——[具体清单]。暂缓的[数量]个计划在[时间点]重新评估。"

**实战举例**：30家供应商分级管理——按业绩体量×风险等级筛出头部6家第一批，按季度逐步覆盖。

---

## 步骤详解

### 步骤1：穷举

| 要素 | 说明 |
|------|------|
| **动作** | 所有可能要做的事 |
| **产出** | 完整清单 |
| **工具** | Issue Tree（MECE穷举） |
| **自检** | 有没有遗漏？ |

**引导话术**：
> "把所有可能要做的事列出来，不管重不重要，先全倒出来。"

---

### 步骤2：定标准

| 要素 | 说明 |
|------|------|
| **动作** | 按什么排优先级 |
| **产出** | 评估维度（通常2-3个） |
| **工具** | 二维矩阵（影响力×紧迫性×可行性） |
| **自检** | 为什么是这几个维度？ |

**引导话术**：
> "你打算按什么标准来排？为什么选这几个维度？"

---

### 步骤3：打分排序

| 要素 | 说明 |
|------|------|
| **动作** | 每个事项按标准评估 |
| **产出** | 排序结果 |
| **工具** | 评分表（量化打分或高/中/低标记） |
| **自检** | 分数能区分开吗？还是都差不多？ |

**引导话术**：
> "每个事项按你的标准打分，排出来看，能拉开差距吗？"

---

### 步骤4：分层

| 要素 | 说明 |
|------|------|
| **动作** | 必做/可做/暂缓 |
| **产出** | 三层清单 |
| **工具** | 三层分类法 |
| **自检** | 暂缓的事项什么时候拿回来？ |

**引导话术**：
> "按结果分三层：必须做、可以做、暂缓。暂缓的打算什么时候重新看？"

---

### 步骤5：排节奏

| 要素 | 说明 |
|------|------|
| **动作** | 按时间排执行节奏 |
| **产出** | 里程碑时间线 |
| **工具** | 甘特图/时间线 |
| **自检** | 第一批做完，第二批的信息从哪来？ |

**引导话术**：
> "第一批做什么、什么时候做？做完第一批后，第二批怎么调？"
