# 图标使用规范

> mino-frontend 图标系统 v5.0 Phase 2

---

## 核心原则

1. **无图标优先** — 能用文字/数字表达的，不用图标
2. **克制使用** — 图标是配角，不是装饰
3. **统一风格** — 手绘风格优先，简洁几何次之
4. **自动获取** — 优先调用 better-icons，降级时手绘

---

## 图标使用决策

```
内容分析 → 是否需要图标？
    ↓ 是
图标类型判断
    ├─ 抽象概念（增长/下降/风险）→ 简单几何图标
    ├─ 具体物体（奖杯/旗帜/钱）→ better-icons 搜索
    └─ 装饰性图标 → 不使用
```

### 不需要图标的情况

- 纯数据表格（数字本身就是信息）
- 文字已能清晰表达（"TOP1"不需要皇冠图标）
- 为装饰而装饰（"让页面更丰富"不是理由）

### 需要图标的情况

- 快速识别（⚠️ 警示、✓ 完成）
- 无文字空间（小标签、徽章）
- 视觉隐喻（🎯 目标、📈 增长）

## better-icons 整合

### 调用方式

```bash
# 在 mino-frontend 生成流程中
better-icons search <关键词>
```

### 关键词选择

| 场景 | 关键词示例 |
|------|-----------|
| 排名 | "trophy", "medal", "crown", "star" |
| 增长 | "trending-up", "arrow-up", "chart-line" |
| 风险 | "alert", "warning", "shield-alert" |
| 目标 | "target", "bullseye", "flag" |
| 金钱 | "money", "coin", "banknote" |
| 团队 | "users", "people", "group" |

### 图标处理

获取 SVG 后：
1. **颜色统一** — 改为当前情绪强调色或 `#666`
2. **尺寸统一** — 16px（小）、24px（中）、32px（大）
3. **嵌入方式** — inline SVG，不引用外部文件

---

## 手绘风格图标

当 better-icons 不可用时，使用手绘风格：

```css
.hand-drawn-icon {
  fill: none;
  stroke: currentColor;
  stroke-width: 2;
  stroke-linecap: round;
  stroke-linejoin: round;
}
```

### 常用手绘图标

| 图标 | SVG路径 |
|------|---------|
| 向上箭头 | `<path d="M12 20V4M5 11l7-7 7 7"/>` |
| 向下箭头 | `<path d="M12 4v16m7-7l-7 7-7-7"/>` |
| 警告 | `<path d="M10.29 3.86L1.82 18a2 2 0 001.71 3h16.94a2 2 0 001.71-3L13.71 3.86a2 2 0 00-3.42 0z"/><line x1="12" y1="9" x2="12" y2="13"/><line x1="12" y1="17" x2="12.01" y2="17"/>` |
| 勾选 | `<path d="M20 6L9 17l-5-5"/>` |

## 降级策略

| 情况 | 处理 |
|------|------|
| better-icons 不可用 | 使用手绘简化图标 |
| 搜索无结果 | 用文字替代 |
| 图标风格不符 | 调整颜色后使用或弃用 |
