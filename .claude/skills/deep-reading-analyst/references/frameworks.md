# 思维框架库

> **何时读取**：需要应用特定思维框架进行分析时


## 快速分析（15 分钟）

### SCQA 结构思考

**用途**：梳理文章逻辑结构

| 要素 | 问题 | 示例 |
|------|------|------|
| **S**ituation | 背景是什么？ | "电销行业竞争激烈" |
| **C**omplication | 遇到什么问题？ | "供应商配合度下降" |
| **Q**uestion | 核心问题是什么？ | "如何提升供应商配合度？" |
| **A**nswer | 解决方案是什么？ | "引入赛马机制" |


### 5W2H 完整性检查

**用途**：检查信息是否完整

| 维度 | 问题 | 检查项 |
|------|------|--------|
| **What** | 是什么？ | 核心观点/方案 |
| **Why** | 为什么？ | 理由/依据 |
| **Who** | 谁？ | 相关方/责任人 |
| **When** | 何时？ | 时间/时机 |
| **Where** | 何地？ | 场景/地点 |
| **How** | 怎么做？ | 方法/步骤 |
| **How much** | 多少？ | 成本/收益/数据 |


## 标准分析（30 分钟）

### 批判性思维

**用途**：评估论证质量

**检查清单**：
1. 核心论点是什么？
2. 论据是否充分？
3. 推理过程是否有漏洞？
4. 有没有替代解释？
5. 作者立场是否客观？

**常见逻辑谬误**：
- 因果倒置
- 以偏概全
- 虚假两难
- 诉诸权威
- 人身攻击


### 逆向思考

**用途**：识别风险和失败模式

**核心问题**：
1. 如何确保这个方案失败？
2. 哪些假设如果错了会致命？
3. 最坏情况是什么？
4. 怎么预防？

**输出**：
- 失败模式清单
- 风险缓解措施
- Pre-mortem 分析


## 深度分析（60 分钟）

### 思维模型库

**用途**：多学科视角分析

| 学科 | 核心模型 | 应用场景 |
|------|---------|---------|
| **经济学** | 机会成本、边际效应、激励相容 | 资源分配、定价策略 |
| **心理学** | 认知偏误、损失厌恶、社会证明 | 用户行为、决策设计 |
| **生物学** | 自然选择、生态位、协同进化 | 竞争策略、生态系统 |
| **物理学** | 临界点、杠杆效应、第一性原理 | 系统变革、根因分析 |
| **系统论** | 反馈回路、涌现、瓶颈 | 组织设计、流程优化 |


### 第一性原理

**用途**：剥离假设，回归本质

**步骤**：
1. 识别核心假设
2. 质疑每个假设
3. 回归不可再分的基本真理
4. 从基本原理重新构建

**示例**：
- 假设："供应商必须赚钱才能配合"
- 质疑："一定吗？有没有例外？"
- 原理："供应商追求长期利益最大化"
- 重构："设计长期激励机制而非短期让利"


### 系统思考

**用途**：理解复杂关系和动态

**核心概念**：
- **增强回路**：A→B→更多 A（复利效应）
- **调节回路**：A→B→减少 A（自我修正）
- **延迟**：行动和结果之间的时间差
- **杠杆点**：小改变带来大影响的位置

**绘制系统图**：
```
[供应商配合度] ←增强— [赛马排名]
       ↓
   [分量调整]
       ↓
  [供应商收入] —增强→ [配合意愿]
```


### 六顶思考帽

**用途**：结构化多角度思考

| 帽子 | 视角 | 问题 |
|------|------|------|
| 🤍 **白帽** | 事实 | 已知什么？需要知道什么？ |
| ❤️ **红帽** | 情感 | 直觉感受是什么？ |
| 🖤 **黑帽** | 谨慎 | 风险是什么？为什么可能失败？ |
| 💛 **黄帽** | 乐观 | 好处是什么？为什么能成功？ |
| 💚 **绿帽** | 创造 | 有什么新想法？替代方案？ |
| 💙 **蓝帽** | 控制 | 下一步？需要什么流程？ |


## 研究分析（120 分钟+）

### 跨源对比

**用途**：多源验证和综合

**步骤**：
1. 搜索 2-3 个相关来源
2. 提取各来源 SCQA
3. 对比核心论点
4. 识别共识和分歧
5. 综合形成完整视角

| 证据强度 | ... | ... | ... | - |
