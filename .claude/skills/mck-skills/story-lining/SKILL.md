# Story-Lining Skill

This skill provides workflows to construct compelling, audience-tailored narrative structures using the SCQR framework. It prevents the mistake of presenting information chronologically (like a cook) and instead forces a results-oriented structure (like a waiter).

## The Core Model: SCQR (空雨湿伞)
*   **S (Situation / 空)**: The context or background.
*   **C (Complication / 雨)**: The change, conflict, or challenge.
*   **Q (Question / 湿)**: The problem caused by the complication.
*   **R (Resolution / 伞)**: The solution or core takeaway.

## Audience Mapping & Strategies
Select the structure based on the audience. *Read the linked reference files ONLY if you need concrete examples of how to execute a specific structure.*

1. **Passerby (路人 / Short Videos)** -> **C(Q)SR**
   *Strategy*: Hook immediately with Conflict. No prior knowledge assumed.
   *Reference*: [references/passerby_cases.md](references/passerby_cases.md)
2. **Audience (观众 / General Presentations)** -> **SC(Q)R**
   *Strategy*: Classic buildup ("Hero's Journey"). Context -> Conflict -> Resolve.
   *Reference*: [references/audience_cases.md](references/audience_cases.md)
3. **Big Boss (大老板 / Execs / Clients)** -> **RSC(Q)**
   *Strategy*: BLUF (Bottom Line Up Front). Start with the Conclusion. Details later.
   *Reference*: [references/boss_cases.md](references/boss_cases.md)
4. **Direct Manager (直属领导 / Team)** -> **R**
   *Strategy*: Skip buildup entirely. Deliver Resolution immediately.
   *Reference*: [references/manager_cases.md](references/manager_cases.md)

## Workflow
1. **Gather Inputs**: Ask the user for: (1) Audience, (2) Core Message/Resolution, (3) Context (Situation/Complication). Do not proceed without these.
2. **Select Structure**: Map the audience to the correct SCQR variation above.
3. **Generate**: Output the structured story-line, clearly labeling the S, C, Q, and R sections.
