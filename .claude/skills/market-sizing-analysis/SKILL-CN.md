# 市场规模分析

计算 TAM（总可触达市场）、SAM（可服务可触达市场）和 SOM（可服务可获得市场）的完整方法论框架。

## 概述

市场规模分析是创业战略、融资和商业规划的基础。通过三种互补方法计算市场机会：自上而下（行业报告）、自下而上（客户分层计算）和价值理论（支付意愿）。

## 核心概念

### 三层市场框架

**TAM（Total Addressable Market，总可触达市场）**

- 假设 100% 市场份额时的总收入机会
- 定义潜在客户的总范围
- 用于长期愿景和市场验证
- 示例：全球所有邮件营销软件收入

**SAM（Serviceable Available Market，可服务可触达市场）**

- TAM 中当前产品/服务可以覆盖的部分
- 考虑地理、细分市场或能力限制
- 代表现实的可触达机会
- 示例：北美电商行业的 AI 邮件营销

**SOM（Serviceable Obtainable Market，可服务可获得市场）**

- 3-5 年内可实现的市场份额
- 考虑竞争、资源和市场动态
- 用于财务预测和融资
- 示例：基于竞争格局，SAM 的 2-5%

### 三种方法的适用场景

**自上而下分析**

- 适用于已有成熟市场研究的场景
- 最适合定义清晰的成熟市场
- 验证市场存在性和增长趋势
- 从行业报告出发逐步缩小范围

**自下而上分析**

- 适用于针对特定客户群体的场景
- 最适合新兴或小众市场
- 对投资人最具说服力
- 从客户数据和定价出发构建

**价值理论**

- 适用于开创新市场品类的场景
- 最适合颠覆式创新
- 基于价值创造进行估算
- 计算客户对问题解决方案的支付意愿

## 三方法框架

### 方法一：自上而下分析

从总市场规模出发，逐步缩小到可触达的细分市场。

**流程：**

1. 从研究报告中确定总市场品类规模
2. 应用地理过滤器（目标区域）
3. 应用细分过滤器（目标行业/客户）
4. 计算竞争定位调整

**公式：**

```
TAM = 总市场品类规模
SAM = TAM × 地理占比 × 细分市场占比
SOM = SAM × 现实获取率 (2-5%)
```

**适用场景：** 有成熟研究的市场（如 SaaS、金融科技、电商）

**优势：** 快速、使用可信数据、验证市场存在性

**局限：** 可能高估新品类规模、颗粒度不够

### 方法二：自下而上分析

从客户分层计算构建市场规模。

**流程：**

1. 定义目标客户细分
2. 估算每个细分的潜在客户数量
3. 确定每客户平均收入
4. 计算现实渗透率

**公式：**

```
TAM = Σ (细分规模 × 每客户年收入)
SAM = TAM × (可服务细分 / 总细分)
SOM = SAM × 现实渗透率 (第3-5年)
```

**适用场景：** B2B、小众市场、特定客户群体

**优势：** 对投资人最具说服力、颗粒度高、可防御

**局限：** 需要详细的客户调研、耗时

### 方法三：价值理论

基于创造的价值和支付意愿进行计算。

**流程：**

1. 识别解决的问题
2. 量化问题的当前成本（时间、金钱、低效率）
3. 计算解决方案的价值（节省、收益、效率）
4. 估算支付意愿（通常为价值的 10-30%）
5. 乘以可触达客户基数

**公式：**

```
每客户价值 = 问题成本 × 解决方案覆盖比例
每客户价格 = 价值 × 支付意愿比例 (10-30%)
TAM = 总潜在客户数 × 每客户价格
SAM = TAM × 满足购买条件的比例
SOM = SAM × 现实采纳率
```

**适用场景：** 新品类、颠覆式创新、现有市场不明确

**优势：** 展示价值创造、适用于新市场

**局限：** 需要假设、较难验证

## 分步流程

### 第一步：定义市场

明确指定要衡量的市场。

**需要回答的问题：**

- 解决什么问题？
- 目标客户是谁？
- 产品/服务品类是什么？
- 地理范围是什么？
- 时间跨度是什么？

**示例：**

- 问题：电商公司在邮件营销自动化方面遇到困难
- 客户：年收入超过 100 万美元的电商公司
- 品类：AI 驱动的邮件营销软件
- 地理：初期北美，后续全球扩展
- 时间：3-5 年机会窗口

### 第二步：收集数据来源

确定可信的计算数据来源。

**自上而下数据源：**

- 行业研究报告（Gartner、Forrester、IDC）
- 政府统计数据（人口普查、劳动统计、行业协会）
- 上市公司财报和业绩
- 市场研究机构（Statista、CB Insights、PitchBook）

**自下而上数据源：**

- 客户访谈和调查
- 销售数据和 CRM 记录
- 行业数据库（LinkedIn、ZoomInfo、Crunchbase）
- 竞争情报
- 学术研究

**价值理论数据源：**

- 客户问题量化
- 时间/成本研究
- ROI 案例研究
- 定价调研和支付意愿调查

### 第三步：计算 TAM

应用选定方法确定总市场。

**自上而下：**

1. 从研究报告中查找总品类规模
2. 记录数据来源和年份
3. 必要时应用增长率
4. 用多个来源验证

**自下而上：**

1. 统计总潜在客户数
2. 计算每客户平均年收入
3. 相乘得到 TAM
4. 按细分拆解

**价值理论：**

1. 量化总可触达客户基数
2. 计算每客户价值
3. 基于价值估算定价
4. 相乘得到 TAM

### 第四步：计算 SAM

将 TAM 缩小到可服务可触达市场。

**应用过滤器：**

- 地理限制（可服务的区域）
- 产品限制（当前具备的功能）
- 客户要求（规模、行业、使用场景）
- 分发渠道可达性
- 监管或合规限制

**公式：**

```
SAM = TAM × (满足所有过滤条件的比例)
```

**示例：**

- TAM：100 亿美元全球邮件营销
- 地理过滤：40%（北美）
- 产品过滤：30%（电商聚焦）
- 功能过滤：60%（需要 AI 能力）
- SAM = $10B × 0.40 × 0.30 × 0.60 = $720M

### 第五步：计算 SOM

确定可实现的市场份额。

**考虑因素：**

- 竞争对手当前市场份额
- 新进入者的典型市场份额（2-5%）
- 可用资源（资金、团队、时间）
- 市场推广效果
- 竞争优势
- 实现时间（通常 3-5 年）

**保守估算：**

```
SOM (第3年) = SAM × 2%
SOM (第5年) = SAM × 5%
```

**示例：**

- SAM：$720M
- 第3年 SOM：$720M × 2% = $14.4M
- 第5年 SOM：$720M × 5% = $36M

### 第六步：验证和三角交叉

使用多种方法交叉验证。

**验证技术：**

1. 比较自上而下和自下而上的结果（应在 30% 以内）
2. 与该领域上市公司收入对比
3. 验证客户数量假设
4. 常识检验定价假设
5. 与行业专家讨论
6. 与类似市场品类对比

**危险信号：**

- TAM 太小（VC 支持的创业公司 < $1B）
- TAM 太大（缺乏数据支撑）
- SOM 过于激进（新进入者 5 年内 > 10%）
- 方法论之间不一致（差异 > 50%）

## 行业特定考量

### SaaS 市场

**关键指标：**

- 目标细分中的潜在企业数量
- 平均合同价值（ACV）
- 典型市场渗透率
- 扩展收入潜力

**TAM 计算：**

```
TAM = 目标企业总数 × 平均 ACV × (1 + 扩展率)
```

### 平台/市场型

**关键指标：**

- 品类交易总额（GMV）
- 佣金率（你获取的 GMV 百分比）
- 总交易量或用户数

**TAM 计算：**

```
TAM = 品类总 GMV × 预期佣金率
```

### 消费者市场

**关键指标：**

- 总可触达用户/家庭数
- 每用户平均收入（ARPU）
- 使用频率

**TAM 计算：**

```
TAM = 总用户数 × ARPU × 年购买频次
```

### B2B 服务

**关键指标：**

- 按规模/行业分的目标企业数
- 平均项目价值或年费
- 典型购买频率

**TAM 计算：**

```
TAM = 目标企业总数 × 平均交易规模 × 年交易次数
```

## 展示市场规模

### 面向投资人

**结构：**

1. 市场定义和问题范围
2. TAM/SAM/SOM 及方法论
3. 数据来源和假设
4. 增长预测和驱动因素
5. 竞争格局背景

**要点：**

- 以自下而上计算为主导（最具说服力）
- 展示与自上而下的交叉验证
- 解释保守假设
- 关联到收入预测
- 突出市场增长率

### 面向战略

**结构：**

1. 可触达客户细分
2. 按机会规模排序
3. 按细分制定进入策略
4. 预期渗透时间线
5. 资源需求

**要点：**

- 聚焦 SAM 和 SOM
- 展示细分层级详情
- 关联到市场推广计划
- 识别扩展机会
- 讨论竞争定位

## 常见错误

**错误一：混淆 TAM 和 SAM**

- 不要把整个市场都当作可触达的
- 应用现实的产品/地理限制
- 诚实面对可服务市场

**错误二：SOM 过于激进**

- 新进入者很少在 5 年内获得 > 5% 份额
- 考虑竞争和资源
- 展示现实的增长时间线

**错误三：只用自上而下**

- 投资人更认可自下而上的验证
- 单独自上而下缺乏说服力
- 必须用多种方法交叉验证

**错误四：选择性使用数据**

- 使用一致且最新的数据来源
- 不要不当混合方法论
- 清晰记录所有假设

**错误五：忽视市场动态**

- 考虑市场增长/衰退
- 考虑竞争强度
- 考虑转换成本和壁垒

## 快速入门

执行市场规模分析：

1. **定义市场** — 问题、客户、品类、地理范围
2. **选择方法** — 自下而上（首选）或自上而下 + 交叉验证
3. **收集数据** — 行业报告、客户数据、竞争情报
4. **计算 TAM** — 应用方法论公式
5. **缩小到 SAM** — 应用产品、地理、细分过滤器
6. **估算 SOM** — 2-5% 现实获取率
7. **验证** — 用其他方法交叉验证
8. **记录** — 展示方法论、来源、假设
9. **展示** — 根据受众调整结构（投资人、战略、运营）
