# 年老师实物场景图

## 核心定位

把文章观点、用户处境、个人经历、项目过程和成长路径，转译成一张吉卜力风格的"自然场景小现场"或"自然探索线"：

```text
年老师 + 自然场景 + 观察动作 + 短中文标签 + 留白叙事
```

这不是 PPT 信息图，也不是商业插画。2.0 的目标是让读者先看到一个自然、安静、有思考感的场景，再在 1 秒内意识到"这说的就是我"。

本 Skill 有两种输出规格：

- **标准模式**：16:9 正文配图，一个自然场景小现场，使用自然户外场景背景。
- **彩蛋模式**：年老师长卷故事图，一条超横版自然探索线，用于个人经历、项目复盘、成长路径。它以 `assets/examples/07-long-scroll-story-master.png` 为核心母版，使用自然柔和背景。

## Before Starting

按任务需要读取，不要一次全部塞进上下文：

- `references/style-dna.md`：2.0 视觉 DNA、比例、留白、颜色、自然场景规则。
- `references/nian-teacher-ip.md`：年老师 IP 在 2.0 里的形体弹性、动作和禁忌。
- `references/prompt-template.md`：单张生图提示词模板。
- `assets/examples/`：高质量模板母版和出图质量标尺。

开始前确认：

- 用户是否明确要标准 16:9 正文图，还是彩蛋长卷故事图。
- 用户是否提供了必须保留的人名、品牌名、项目名或经历节点。
- 如果没有明确要求生成图片，只先输出 shot list 或配图方案。

## Core Flow

### 1. 消化内容

先提炼当前内容里的"处境"，不要急着找场景：

- 这张图要让谁产生共鸣？
- 他们正在被什么东西拉扯、思考、观察、记录？
- 抽象概念能变成什么自然场景动作？
- 哪 2-4 个短词能一秒戳中痛点？
- 哪个自然场景天然承载这个处境？

优先选择高共鸣节点：独处思考、自然观察、安静记录、探索发现、内心坚定。

如果用户要求彩蛋模式，先把内容拆成 5-8 个连续节点：

- 左侧起点：身份、起点、最初处境。
- 中间节点：关键经历、项目转折、成长节点。
- 右侧终点：现在在做什么、核心关注、结论或下一阶段。
- 每个节点都要有一个自然场景、年老师动作和 1-3 行短注释。

完成标记：已经明确读者处境、核心冲突、自然场景、年老师动作和短中文标签；彩蛋模式已经得到 5-8 个可视化节点。

### 1B. 事实锚定

涉及个人经历、品牌名、公司名、项目名、时间跨度、成绩数字时，只能使用用户输入、用户提供素材或用户明确确认的事实。

- 用户没有提供的事实，不要从母版继承，也不要补成看起来更完整的履历。
- 无法确认的内容，用概括性标签替代，例如"内容平台""项目节点""用户反馈""产品实验"。
- 如果用户要求保留但信息不完整，在 shot list 或交付说明里标注"待确认"，不要把它画成确定事实。

完成标记：所有具体事实都能回到用户输入或素材；不能回溯的事实已删除、抽象化或标注待确认。

### 2. 先出 shot list

如果用户说"分析 / 思考 / 怎么配图"，先给 shot list。每张写清：

- 图的主题
- 读者共鸣点
- 自然场景动作
- 自然主场景
- 年老师在做什么
- 中文短标签
- 为什么这张值得画

标准模式默认 4-8 张。彩蛋模式默认先出 1 张长卷效果图。用户明确要"看效果 / 输出 / 生成"时，直接生图，不要停在方案。

完成标记：每个候选画面都有主题、共鸣点、自然场景、年老师动作、短标签和成图理由。

### 3. 单张生成

每张图单独生成，不要把多张拼成一张。

默认用 16:9 横版。提示词必须包含：

- 吉卜力风格手绘插画美学。
- 自然户外场景背景（森林/草地/溪流/木屋/山径）。
- 场景色调统一和谐。
- 年老师参与核心观察动作。
- 2-4 个短中文手写标签。
- 自然色彩点缀（绿/棕/蓝/黄）。
- 中等场景覆盖面积，但视觉重量轻。
- 禁止截图、UI、Logo、PPT、商业插画。

完成标记：每张标准图都有独立提示词，且包含比例、背景、自然场景、年老师动作、标签、点缀和负面约束。

### 3B. 彩蛋长卷生成

当用户说"彩蛋模式 / 长卷故事图 / 超横版 / 个人经历 / 项目复盘 / 成长路径"时，使用彩蛋模式。

彩蛋模式的核心是：

- 超横版长卷，比例接近 `2.6:1` 到 `3:1`。
- 自然柔和背景，接近白色但保留自然暖调。
- 一条细黑色手绘弯曲路线贯穿全图。
- 5-8 个自然场景节点，每个节点有年老师参与动作。
- 节点沿路线形成自然、不规律的上下起伏。
- 左侧是起点身份/开场，右侧是现在关注/结论。

完成标记：长卷提示词包含超横版比例、自然柔和背景、曲线路径、5-8 个自然场景节点、年老师逐段参与、左起右收和事实锚定后的节点文案。

### 4. QA 和迭代

生成后重点看：

- 候选图是否 3 秒读懂；如果需要解释主题才懂，就是失败。
- 年老师是否承担核心观察动作。
- 场景色调是否统一和谐。
- 中文短标签是否准确、少、能读。
- 画面第一眼是否像吉卜力风格手绘插画。

第一张生成图永远只是候选图，不是交付图。必须先查看候选图。只要出现以下问题，就不交付，先重写提示词或重生成：

- 元素清单化：把主题名词都堆进去，缺少一个清楚的场景焦点。
- 读不懂：不看说明时，场景焦点不清楚。
- 年老师没有承担核心观察动作。
- 画面不像吉卜力风格手绘插画。
- 场景色调不和谐。

不合格就重生成或局部编辑，不要把低质量图展示为最终交付。

完成标记：已通过质量检查；未过时，交付状态标为 DONE_WITH_CONCERNS 并指出下一轮迭代方向。

### 5. 保存交付

如果用户在 workspace 内工作，把最终图复制到：

```text
assets/<topic-slug>-nian-teacher-scenes/
```

彩蛋长卷模式可以保存到：

```text
assets/<topic-slug>-nian-teacher-long-scroll/
```

按顺序命名：

```text
01-topic-name.png
02-topic-name.png
```

保留原始生成文件，不覆盖已有资产，除非用户明确要求替换。

完成标记：图片已展示或保存到目标目录，命名有序，未覆盖旧资产。

## 输出口径

生成前：短而准，给 shot list 或生成计划。

生成后：展示图片、说明用途、给保存路径，并标出哪些最稳、哪些可选或需要再收一轮。

不要长篇解释理论。让图先说话。

## Completion States

| 状态 | 含义 |
| --- | --- |
| DONE | 已完成 shot list 或图片生成，并按 QA 检查通过 |
| DONE_WITH_CONCERNS | 已交付，但存在文字稳定性、场景真实感等待优化项 |
| BLOCKED | 缺少主题/正文/节点，或用户要求的事实、人物、品牌信息不可确认 |

## Good vs Bad

**好的：**

> 一个自然场景 + 一个年老师动作 + 2-4 个短标签。读者先看到自然小现场，再读懂处境。

**差的：**

> 把观点做成 PPT 流程图，堆很多箭头、模块框、大标题和解释文字，年老师只是站在旁边。

**差在哪：**前者靠自然场景和观察动作讲故事；后者靠信息堆叠解释概念。
