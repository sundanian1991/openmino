> 基于 Anthropic 品牌的手绘 SVG 风格生成器
> 风格来源：李诞虾（脱胎于 Anthemic 色彩系统）

## 风格 DNA（已固化）

### 色彩系统（四色）

```css
/* 从 Anthemic 品牌衍生 */
--terra-cotta: #D6654B;    /* 陶土色 — 主体色调，从 #d97757 调暖 */
--ink-black: #1A1612;      /* 墨黑 — 描边，从 #141413 调深 */
--rice-white: #FEFFFE;     /* 米白 — 背景/高光，纸张感 */
--deep-terra: #B03A21;     /* 深陶土 — 阴影/深度 */
```

**色彩使用核心哲学**：

**原则 1：颜色是整体色调，不是分段涂色**
- 主体（如虾）用一种主色（陶土色）贯穿全身
- 不是"头一个色，身体一个色，尾巴一个色"
- 颜色一致性 = 生物真实感

**原则 2：米白是高光，不是主体填充**
- 米白用于高光、反光、空白背景
- 不是"主体米白 + 局部陶土"
- 而是"主体陶土 + 局部米白高光"

**原则 3：深陶土是阴影，不是强调**
- 深陶土用于阴影、深度、层次
- 不是"标记重点"的亮色
- 而是制造立体感的暗部

**原则 4：墨黑是唯一描边色**
- 所有轮廓用墨黑
- 粗细分层，但颜色统一

### 颜色配置表（快速参考）

| 颜色 | 色值 | 使用场景 | 占比 | 示例 |
|------|------|----------|------|------|
| **陶土色** | `#D6654B` | 生物主体、焦点强调、有生命感的物体 | 5-20% | 虾身、猫耳朵、眼睛 |
| **米白** | `#FEFFFE` | 无生命主体、背景、高光、器物填充 | 60-80% | 杯子、碗、盘子主体 |
| **深陶土** | `#B03A21` | 阴影、深度、重量感、底部支撑 | 3-10% | 杯底阴影、虾节暗部 |
| **墨黑** | `#1A1612` | 所有描边、轮廓、结构线 | 10-15% | 所有 stroke 属性 |

**快速决策流程**：
1. 这个物体是活的吗？→ 是 = 陶土色主体，否 = 米白主体
2. 需要强调哪个部分？→ 那部分用陶土色点缀（≤2处）
3. 底部需要重量感吗？→ 添加深陶土阴影
4. 所有线条用墨黑描边

### 线条系统（三分层）

```css
/* 从李诞虾提取的粗细分布 */
主轮廓: 1.5 - 3.0px;      /* Quiver 自然标准（推荐） */
结构线: 0.7 - 1.5px;      /* Quiver 自然标准（推荐） */
细节线: 0.25 - 0.7px;     /* Quiver 自然标准（推荐） */

/* 备选：更粗的线条（需在 prompt 中明确指定） */
主轮廓: 4.0 - 5.2px;      /* 手工绘制标准 */
结构线: 3.5 - 4.0px;
细节线: 1.7 - 2.9px;
```

**线条使用规则**：
- stroke-linecap="round"（强制，所有描边）
- stroke-linejoin="round"（强制，所有转折）
- 大物体用粗线，小细节用细线
- **Quiver API 默认使用较细线条** — 自然、有机效果

### 曲线 DNA（手绘不规则性）

**从李诞虾提取的节奏模式**：

```xml
<!-- 真实手绘感示例（李诞虾第11行） -->
<path d="m18.72 41.09c11.14 7.83 28.25 9.87 43.46 9.83 18.02-0.04 35.81-2.15 45.58-3.64..." />
```

**节奏特征**：
- 长曲线跨度：20-50px（如 `35.81-2.15`）
- 偏差分布：70% 在 ±5px 内，30% 突然偏离 ±15-30px
- 不对称：左右对应部分曲线有差异
- 避免：完美对称、均匀抖动、数学化坐标（如 100.00）

**反模式检测**：
- ❌ 机械对称：`m85 45h80v120h-80z`
- ❌ 规律抖动：`m85.27 45.27c12.37-1.83 25.18-0.91 37.62-0.87`（偏差都在 ±10px）
- ❌ 缺少 round：`stroke-linecap="square"`
- ✅ 真实手绘：`m18.72 41.09c11.14 7.83 28.25 9.87...`（节奏有变化）

## 生成流程

### Step 1: 阅读参考案例（理解 DNA）

**必读** `assets/` 目录：
- `lidan-shrimp.svg` — 风格源头，理解曲线节奏和色彩分布
- `svg-1-cat.svg` — 高复杂度案例
- `svg-4-cat.svg` — 简化案例
- `svg-5-dog.svg` — 中等复杂度

**观察方法**：
1. 数一数主轮廓用了几条 path（李诞虾 39 条 path）
2. 看陶土色用在哪里（李诞虾：虾身 6 处，眼睛 1 处）
3. 看线条粗细分布（5.2px → 4.6px → 1.7px）

### Step 2: 分析主题

**三个问题**：
1. **主体结构**：有哪些主要部分？（例：虾 = 头 + 身体 + 触须 + 腿）
2. **复杂度**：简单（10-20 path）、中等（20-40）、复杂（40+）
3. **焦点**：陶土色用在哪？（≤ 2 处）

### Step 3: 构建填充层

```xml
<g id="fill-layer">
  <!-- 主体填充（米白） -->
  <path d="..." fill="#FEFFFE"/>

  <!-- 陶土色焦点（1-2 处） -->
  <path d="..." fill="#D6654B"/>
</g>
```

**原则**：
- 主体用米白
- 陶土色 ≤ 2 处
- 能用一个 path 就不用两个

### Step 4: 构建描边层

```xml
<g id="stroke-layer" fill="none" stroke="#1A1612" stroke-linecap="round" stroke-linejoin="round">
  <!-- 主轮廓（4-5px） -->
  <path d="..." stroke-width="4.5"/>

  <!-- 结构线（3.5-4px） -->
  <path d="..." stroke-width="3.8"/>

  <!-- 细节线（1.7-2.9px） -->
  <path d="..." stroke-width="2.0"/>
</g>
```

**关键**：每条 path 的贝塞尔曲线都要有李诞虾的节奏

## 贝塞尔曲线：如何复制李诞虾的手绘感

### 核心原则（不是技巧）

**❌ 错误思维**：如何让曲线看起来"随机"？
**✅ 正确思维**：如何复制李诞虾的节奏？

### 李诞虾节奏的三条规则

**规则 1：长曲线要有呼吸**
- 控制点间距：20-50px（不均匀）
- 节奏：紧-松-紧（如 `35.81-2.15` → `45.58-3.64`）
- 示例：`c11.14 7.83 28.25 9.87 43.46 9.83`（递增）

**规则 2：70/30 偏差分布**
- 70% 的偏差在 ±5px 内（手稳定时）
- 30% 的偏差在 ±15-30px（手抖时）
- 示例：`18.02-0.04`（小偏差）→ `35.81-2.15`（中等）→ `45.58-3.64`（大）

**规则 3：避免完美对称**
- 左右对应部分曲线要有差异
- 坐标不用 .00（用 `18.72` 不用 `18.00`）

### 对照表

| 类型 | 特征 | 示例 |
|------|------|------|
| **✅ 李诞虾风格** | 节奏变化，70/30 分布 | `m18.72 41.09c11.14 7.83 28.25 9.87 43.46 9.83` |
| **❌ 机械对称** | 直线，90° 转折 | `m85 45h80v120h-80z` |
| **❌ 规律抖动** | 偏差都在 ±10px | `m85.27 45.27c12.37-1.83 25.18-0.91...` |
| **❌ 数学化** | .00 结尾 | `m100.00 50.00c20.00-5.00...` |

## 设计决策规则（新增）

### 视角选择

| 物体类别 | 推荐视角 | 理由 | 示例 |
|---------|---------|------|------|
| **生物类** | 侧视图 | 展示身体曲线、动态感、比例关系 | 虾、猫、狗 |
| **器物类** | 略俯视（15-30°） | 展示开口形状 + 体积感 + 使用场景 | 杯子、碗、盘子 |
| **建筑类** | 正视图 | 展示结构、层次、立面关系 | 房子、树、桌子 |
| **工具类** | 使用角度优先 | 展示其功能状态 | 剪刀（张开）、锤子（平放） |

**视角统一原则**：
- 同一图标内不要混用多个视角
- 如果不确定，用物体最"日常"的状态（杯子 = 放在桌上的略俯视）

### 结构分解顺序

**三层分解法**：

**第一层：主体轮廓（一笔到底）**
- 用一条长 path 勾勒外形
- 覆盖整个物体的边界
- 这条 path 决定"这是什么"

**第二层：内部结构（从大到小）**
- 先画大的内部划分（如杯子的把手、虾的头身分界）
- 再画小的细节（如虾的触须、杯子的纹理）
- 结构线是"功能性"的，不是装饰

**第三层：装饰纹理（点缀性质）**
- 最后才加的纯装饰线条
- 不影响物体识别
- 用来增加"手绘感"

**错误示例**：
- ❌ 先画细节纹理，再拼主体 → 散乱
- ❌ 所有部分用相同的粗细 → 没层次

### 色彩分配决策

**问题：这个物体用什么主色？**

**判断规则**：

| 物体 | 主色选择 | 理由 |
|------|---------|------|
| 活的/有生命的 | 陶土色 | 有温度、有生命感 |
| 无生命的/功能性的 | 米白 + 陶土点缀 | 干净、简洁 |
| 重/稳定的 | 深陶土为主 | 有重量感、稳重 |

**高光/阴影位置规则**：

**高光 = 视觉重心附近 + 光源方向**
- 通常在物体的"凸起"部分
- 杯子 = 杯身左侧（假设光从左来）
- 虾 = 虾背中央（最突出的部分）

**阴影 = 结构凹陷处 + 底部支撑**
- 杯子 = 杯底与把手连接处
- 虾 = 虾节之间、虾身与桌面的接触面
- 通用 = 物体最底部（支撑点）

**高光/阴影形式**：
- 高光 = 短曲线（2-5px 宽），米白色
- 阴影 = 填充块，深陶土色
- 不是描边，是填充

### 纹理线决策

**问题：要不要加纹理线？加在哪里？**

**判断规则**：

| 物体类型 | 纹理线 | 位置 | 示例 |
|---------|-------|------|------|
| 生物类 | 天然结构线 | 身体节、毛发纹理 | 虾节、猫毛 |
| 器物类 | 功能性结构 | 边缘、连接处、制造痕迹 | 杯沿、把手连接 |
| 抽象类 | 装饰性曲线 | 不要太规律，点缀即可 | 星星、圆点 |

**纹理线密度**：
- 简单物体（10-30 path）：≤20% 细节占比
- 中等物体（30-60 path）：30-40% 细节占比
- 复杂物体（60-90 path）：40-50% 细节占比

**避免**：
- 纹理线太密 → 像网格纸，不自然
- 纹理线太规律 → 像机械加工
- 纹理线不对称 → 手绘感变"手抖感"

### 词汇选择原则（新增）

**⚠️ 避免多义词**

| 多义词 | 多重含义 | 替代词 |
|--------|----------|--------|
| fan | 风扇/粉丝/狂热者 | electric cooling appliance |
| bat | 蝙蝠/球棒 | flying mammal / sports equipment |
| match | 火柴/比赛/匹配 | fire starter / competition |
| bank | 银行/河岸 | financial institution / river bank |

**验证方法**：
在选择物体前，自问：
1. 这个词有多个含义吗？
2. 是否可能被误解为其他物体？
3. 有无更明确的替代词？

**教训**：R3 电扇测试失败两次 — "fan" 被理解为人脸，即使加了 "NOT a human face" 约束也无效。

### 否定约束原则（新增）

**适用场景**：
- ✅ 弱联想词（如：tree）可以添加 "NOT a XXX"
- ❌ 强联想词（如：fan）否定约束无效

**核心原则**：从源头避免歧义 > 事后否定约束

## 中间执行层：算法 + 人工（v6.0 新增）

> 核心理念：算法 70% + 人工 30%
> 算法生成基础结构，人工微调关键细节

---

### 为什么要中间执行层？

**问题**：设计哲学理解到位，但执行时质量不稳定

| 层次 | 内容 | 状态 |
|------|------|------|
| **设计哲学**（SKILL.md） | 什么是好的、什么是坏的 | ✅ 已固化 |
| **中间执行层**（本文） | 如何把"好的"变成"可执行的" | ✅ 新增 |
| **底层算法**（organic-curve-generator.js） | 坐标计算、路径生成 | ✅ 已实现 |

**关键洞察**：控笔节奏这种"肌肉记忆"很难纯算法复制，需要算法+人工协同。

---

### 标准化工作流程

#### Step 1：分析需求（人工）

**三个问题**：
1. 画什么？（杯子、虾、猫）
2. 复杂度？（简单 10-30 path / 中等 30-60 / 复杂 60-90）
3. 焦点在哪？（陶土色用在哪里）

**输出**：物体类型、关键点列表、复杂度级别

---

#### Step 2：算法生成基础结构（算法）

**使用 `organic-curve-generator.js`**：

```javascript
const { generateBezierPath, getStrokeWidths } = require('./organic-curve-generator');

// 定义关键点
const keyPoints = [
  { x: 75, y: 65 },   // 左上
  { x: 75, y: 155 },  // 左下
  { x: 125, y: 155 }, // 右下
  { x: 125, y: 65 }   // 右上
];

// 生成曲线
const path = generateBezierPath(keyPoints, {
  complexity: 'medium',
  isLeft: true,
  isRight: true
});

// 获取线条粗细
const widths = getStrokeWidths('medium');
```

**输出**：基础 path d 属性、线条粗细配置、色彩填充代码

---

#### Step 3：人工微调关键细节（人工）

**微调清单**：

| 细节 | 算法输出 | 人工微调 | 示例 |
|------|----------|----------|------|
| **控制点间距** | 均匀分布 | 手动调整节奏 | 4.1→2.01→-6.1 改为 4.1→11.14→25.3 |
| **小数点** | 规则分布 | 随机化位置 | 4.1 改为 4.72 |
| **不对称** | 左右对称 | 添加 ±1-5px 偏差 | 左边 75 改为 73.2 |
| **节奏** | 单一趋势 | 波浪式节奏 | 递增→递减→递增 |

**微调方法**：
1. 找到控制点密集的地方（算法可能生成了太多小间距）
2. 合并或删除一些控制点
3. 调整关键控制点的间距，形成呼吸感
4. 手动修改小数点，避免规律性

---

#### Step 4：验证与迭代（人工）

**验证清单**：
- [ ] 曲线有呼吸感吗？（紧-松-紧节奏）
- [ ] 小数点不规则吗？（没有 .00，1-2 位混合）
- [ ] 左右有微妙不对称吗？
- [ ] 色彩分配正确吗？（陶土色 ≤2 处）
- [ ] 线条分层清晰吗？（主/结/细）

---

#### Step 5：输出 SVG（算法）

使用 SKILL.md 的输出格式（见"输出格式"章节）

## API Prompt 模板（完整示例：杯子）

> 用这个模板驱动 Quiver API 生成，确保输出符合年老师风格

```
Create a hand-drawn style SVG illustration of a ceramic coffee cup/mug.

**Subject Details:**
- Object: Simple ceramic coffee cup with handle
- View: Slight overhead angle (20-30°) to show the opening and volume
- Style: Organic, hand-drawn, warm aesthetic
- Complexity: Medium (30-50 paths)

**Color Palette (STRICT - use only these 4 colors):**
- Primary fill: #FEFFFE (rice white) - for the cup body
- Accent: #D6654B (terracotta) - for handle only (1 accent location)
- Shadow/depth: #B03A21 (deep terracotta) - for bottom contact shadow
- Stroke: #1A1612 (ink black) - for ALL outlines

**Design Philosophy:**
- "Organic Warmth" movement - imperfect, human-made feeling
- Lines should feel naturally hand-drawn, not geometrically perfect
- Curve rhythm: breathe between tight and loose control points
- Avoid: perfect symmetry, mechanical regularity, mathematical precision

**Line Specifications:**
- Main outline: 1.5-3.0px (natural/organic weight)
- Structure lines: 0.7-1.5px (handle connection, rim details)
- Detail lines: 0.25-0.7px (subtle texture)
- ALL strokes MUST have: stroke-linecap="round" stroke-linejoin="round"

**Composition:**
- Canvas: 250x250px, viewBox="0 0 250 250"
- Cup centered, occupying 60-70% of frame
- Handle on right side, slightly curved
- Show subtle shadow beneath cup for grounding

**Curve Characteristics (CRITICAL):**
- Control point spacing: 20-50px (irregular, not uniform)
- Deviation pattern: 70% within ±5px, 30% sudden ±15-30px
- Avoid: .00 coordinates, perfect mirror symmetry, evenly spaced points
- Target: Natural hand-drawn rhythm like "c11.14 7.83 28.25 9.87 43.46 9.83"

**Layer Structure:**
1. Fill layer: White cup body + terracotta handle + deep terracotta shadow
2. Stroke layer: Black outlines with varying weights (main/structure/detail)

**Negative Constraints:**
- NO gradient fills - use solid colors only
- NO perfect circles or ellipses - hand-drawn curves only
- NO decorative patterns - keep it simple and functional
- NO multiple accent colors - terracotta on handle ONLY

Output clean SVG code with proper indentation, no CSS classes, inline styles only.
```
