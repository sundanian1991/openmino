**结果**: **仍然生成人脸**
**原因**: Quiver 模型对 "fan" 有强烈的联想偏见

## 1. 根本原因分析

### Quiver 模型的联想偏见

**发现**: Quiver 的模型在训练数据中，"fan" 这个词可能与"人脸"、"粉丝"等概念强关联，即使加了明确约束也无法覆盖。

**证据**:
- 两次尝试都生成了人脸
- 第二次的 Prompt 已经非常明确（"electric COOLING FAN", "NOT a human face"），仍然失败
- 说明这不是 Prompt 设计问题，而是**模型偏见**

### 否定约束的局限性

**假设**: "NOT a human face" 约束应该能避免生成人脸
**现实**: 约束无效
**结论**: Quiver 的否定约束机制可能不够强大，或者"fan"的联想权重太高

## 3. 关键学习

### ✅ 多义词必须避免

**错误示例**:
```
fan → 人脸/粉丝/风扇
```

**正确做法**:
1. 完全避免多义词
2. 使用同义词（"electric cooling appliance"）
3. 或选择无歧义的测试对象

### ✅ 否定约束不是万能的

**假设**: "NOT XXX" 可以避免错误生成
**现实**: 对于强联想词，否定约束无效
**建议**: 不要依赖否定约束，从源头避免歧义

### ⚠️ Quiver 模型有局限性

**发现**: Quiver 的模型在某些词汇上有强烈的联想偏见
**建议**:
- 测试前先验证词汇是否有多重含义
- 优先选择独特、明确的物体
- 避免常见多义词

## 7. 核心洞察总结

### Quiver API 的局限性

| 局限性 | 表现 | 应对策略 |
|--------|------|----------|
| **词汇偏见** | "fan" → 人脸 | 避免多义词 |
| **否定约束弱** | "NOT XXX" 无效 | 从源头避免歧义 |
| **线条偏细** | 1.2px vs 4.0px | 调整期望或明确指定 |
| **联想权重高** | 强联想覆盖约束 | 选择无偏见词汇 |

### Prompt 设计原则（更新）

**1. 词汇选择原则**:
```
优先级:
1. 独特、无歧义的词汇（shrimp, lucky cat）
2. 描述性词汇（"machine with rotating blades"）
3. 避免多义词（fan, bat, match）
```

**2. 否定约束原则**:
```
否定约束不是万能的:
- 对于强联想词，否定约束可能无效
- 不要依赖否定约束解决根本问题
- 从源头避免歧义更可靠
```

**3. 视觉锚点原则**:
```
视觉锚点必须配合无歧义词汇:
- 仅有视觉锚点不够（R3 第二次证明了这点）
- 词汇本身必须清晰
```
