# 供应商绩效报告 — 一条龙工作流

> 上游：CRM 底表 Excel → 下游：完整 HTML 报告

---

## 定位

**不画单张图、不做零散分析、不生成中间产物**。唯一产出是一份完整的自包含 HTML 报告。

工作流：数据校验 → 多维分析 → 洞察提取 → 叙事编排 → 视觉渲染 → 报告组装。内部串联：
```
supplier:performance-report（本技能：工作流编排）
  ├→ viz-data-storytelling（叙事编排：数据→叙事大纲）
  │    └→ viz-design（视觉 SPEC + JSON）
  │         └→ viz-echarts-visualization（HTML 图表渲染）
  └→ 报告组装（HTML 母版 + 图表合并 + 结论写入）
```

---

## 六步工作流

```
CRM 底表（Excel）
  ↓
Phase 1：数据加载与校验
  ↓
Phase 2：多维度分析
  ↓
Phase 3：洞察提取
  ↓
Phase 4：叙事编排（委派 viz-data-storytelling）
  ↓
Phase 5：视觉渲染（委派 viz-design → 渲染技能）
  ↓
Phase 6：报告组装（HTML 母版 + 图表 + 结论 → 最终报告）
```

## Phase 1：数据加载与校验

用 Python（openpyxl）读取 Excel，执行以下校验：

| 校验项 | 检查内容 | 异常处理 |
|--------|---------|---------|
| 字段完整性 | 必需字段是否存在（坐席名称/等级/综合得分/转化率/T0GMV/ATT/通时/司龄/参评判定/最近外呼日期） | 缺失字段列出来，继续分析可用字段 |
| 坐席类型分池 | 区分首贷/复贷坐席 | 分池独立分析，必要时出双版本 |
| 参评判定重算 | 用规则：最近外呼≤7天 + 外呼天数≥22天 → 参评 | 校验是否与底表一致 |
| 数据范围确认 | 截止日期、数据天数 | 在报告头部标注 |
| 多职场检测 | 职场名称列是否有多个值 | 自动识别单职场/多职场模式 |

**输出**：清洗后的结构化数据内存对象 + 校验日志，传递给 Phase 2。

---

## Phase 2：多维度分析

### 2.1 分析维度矩阵

| 看板 | 分析维度 | 图表类型 | 必出/条件 |
|------|---------|---------|----------|
| **摘要** | 核心结论 + 关键数字（总人数/参评率/S占比/C占比/预警率/参评T0GMV占比） | 卡片 | 必出 |
| **团队结构** | 人力结构矩阵（司龄×参评×等级三维交叉） | 热力图矩阵 | 必出 |
| | 在职/预警司龄分布 | 堆叠条形图 | 必出 |
| | Top5 vs Bottom5 画像对比 | 分组条形图 | 必出 |
| **效率诊断** | 司龄段四维效率（ATT/GMV/T0转化/通时） | 热力图矩阵 | 必出 |
| | 等级四维对比（S/A/B/C） | 热力图矩阵 | 必出 |
| | 帕累托曲线（前20%贡献占比） | 折线+柱组合 | 必出 |
| **产出分析** | 各等级GMV贡献拆解 | 100%堆叠条形 | 必出 |
| | GMV驱动因子拆解（转化率/件均/接通量贡献） | 100%堆叠条形 | 必出 |
| | 转化漏斗（接通→有效→首贷）分等级 | 漏斗图 | 必出 |
| **风险** | C级四维短板（C级 vs S级基准） | 雷达图 | 必出 |
| | 预警流失扫描 | 矩阵图 | 必出 |
| **留存** | 各月留存率趋势（如有历史数据） | 折线图 | 条件 |

### 2.2 关键指标计算口径

| 指标 | 公式 | 备注 |
|------|------|------|
| 参评率 | 参评人数 ÷ 总人数 | |
| S占比 | S级人数 ÷ 参评总人数（非全职场） | 按规则，等级在参评池中排名 |
| 预警率 | 预警人数 ÷ 有效在岗人数 | 预警:最近外呼≥3天 |
| 流失率 | 流失人数 ÷ 有效在岗人数 | 流失:最近外呼>7天 |
| 参评T0GMV占比 | 参评坐席T0GMV总和 ÷ 全员T0GMV总和 | 衡量有效产能占比 |
| 转化漏斗 | 接通→有效→首贷 | 各等级分别计算转化率 |
| 帕累托贡献 | 按GMV排序计算前20%/50%的GMV占比 | |
| GMV驱动因子拆解 | 因子贡献 = (因子均值/基准均值 - 1) × 因子权重 | 确认高绩效由哪个因子驱动 |

**输出**：结构化分析结果 JSON，传递给 Phase 3 和 Phase 5。

## Phase 3：洞察提取

用 LLM 分析 Phase 2 的分析结果，提取关键洞察：

### 3.1 自动扫描项目

| 扫描维度 | 识别信号 |
|---------|---------|
| **等级异常** | S级占比异常高/低、非参评率突增、C级占比过大（>15%）|
| **效率瓶颈** | 某司龄段四维热力图全红、某等级在特定维度断崖式下降 |
| **帕累托偏离** | 前20%产出占比偏离80%过多（<60%或>90%都需要关注）|
| **GMV驱动异常** | 驱动因子在不同职场间方向不一致、某因子贡献为负 |
| **预警信号** | 预警/流失率超过阈值（预警>10%、流失>5%）|
| **转化断裂** | 某等级在漏斗某个环节转换率骤降（如有效→首贷转换率<0.3%）|
| **结构失衡** | 新老比倒挂（MOB1占比>50%）、某司龄段参评率极低 |

### 3.2 洞察输出格式

每条洞察包含：
- **发现**：具体数据事实
- **方向**：意味着什么（正面/负面/中性）
- **置信度**：高/中/低
- **建议行动**：如果置信度高且有明确方向，给出行动建议

### 3.3 摘要卡片生成

从所有洞察中提取 3-5 条最重要结论，写入摘要卡片。
同时提取 4-6 个关键数字（总人数/参评率/S占比/C占比/预警率/参评T0GMV占比）。

**输出**：洞察列表 + 摘要卡片内容，传递给 Phase 4 和 Phase 6。

---

## Phase 4：叙事编排

委派 **viz-data-storytelling** 技能，输入：
- Phase 2 的分析结果 JSON
- Phase 3 的洞察列表
- 数据口径说明

viz-data-storytelling 产出 Markdown 叙事大纲，包含：
- Big Idea（洞察+推理+行动建议）
- 数据推理
- 图表清单（每张图的叙事功能、类型、视觉策略、标题）
- 三幕叙事结构
- 反对证据与局限性

**输出**：叙事大纲 Markdown，传递给 Phase 5。

---

## Phase 5：视觉渲染

委派 **viz-design → viz-echarts-visualization / viz-svg-flow-diagram**，输入叙事大纲和 Phase 2 的分析数据，输出每张图的 HTML chart fragment。

### 5.1 图表渲染任务分配

| 图表类型 | 渲染技能 |
|---------|---------|
| ECharts 交互式图表（热力图/条形/折线/雷达/漏斗/散点） | viz-echarts-visualization |
| 流程图/概念图 | viz-svg-flow-diagram |
| 摘要卡片、文字标注 | 本技能直做（HTML + CSS） |

**输出**：每张图的 HTML fragment（独立的 chart div + 初始化 script）。

## Phase 6：报告组装

### 6.1 HTML 母版

使用 NYT 数据新闻风格：

```
颜色系统：
  灰阶：50:#F7F7F7 100:#D9D9D9 200:#969696 300:#525252 400:#1A1A1A
  强调：暖橙 #F4A261
  正向：teal #2A9D8F
  负向：红 #E63946
  底：纯白 #FFFFFF
  文：深灰 #1A1A1A

字体：系统无衬线（-apple-system, Helvetica Neue, Arial）
布局：960-1200px 居中，卡片分隔，高信息密度
```

### 6.2 报告结构

```
报告标题 + 数据时间戳 + 数据来源标注
  │
  ├── 摘要卡片（3-5条核心结论 + 6个关键数字）
  │
  ├── 一、团队结构
  │   ├── 1.1 人力结构矩阵（司龄×参评×等级热力图）
  │   ├── 1.2 司龄分布（在职/预警堆叠条形图）
  │   └── 1.3 Top5 vs Bottom5 画像对比
  │
  ├── 二、效率诊断
  │   ├── 2.1 司龄段四维效率热力图
  │   ├── 2.2 等级四维对比热力图
  │   └── 2.3 帕累托曲线
  │
  ├── 三、产出分析
  │   ├── 3.1 各等级GMV贡献拆解
  │   ├── 3.2 GMV驱动因子拆解
  │   └── 3.3 转化漏斗（分等级）
  │
  ├── 四、风险与留存
  │   ├── 4.1 C级四维短板雷达
  │   ├── 4.2 预警流失扫描
  │   └── 4.3 [如有历史数据] 各月留存率趋势
  │
  ├── 五、多职场对比（仅多职场模式）
  │   ├── 5.1 各职场等级构成对比
  │   ├── 5.2 各职场核心KPI横向排名
  │   └── 5.3 各职场预警率对比
  │
  └── 附录：数据口径与局限性
```

### 6.3 说明区域规范

每张图下方必须附带 **说明文字区域**，包含：
- **图基**：数据范围（"数据来源：伽玛职场CRM底表，截至2026-05-15"）
- **标注**：异常点、口径变化、需要注意的事项（如"注：转化率分母为用户数月均已接通用户数，非名单下发数"）

**格式**：图表下方两个段落，左对齐，使用灰阶 200 色。第一个段落是数据来源脚注，第二个段落（如有）是注意事项标注。字号对比正文小一级。

### 6.4 交付物

- 文件命名：`report-{职场名}-{YYYYMMDD}.html`
- 自包含：全部 CSS 内联、ECharts 通过 CDN 加载（不影响离线查看核心内容）
- 直接双击可打开
