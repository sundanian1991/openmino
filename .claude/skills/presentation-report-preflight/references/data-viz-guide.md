## 一、图表选型矩阵

| 想表达什么 | 推荐图表 | 替代选项 | 不要用 |
| --- | --- | --- | --- |
| **趋势变化**（时间序列）| 折线图 | 面积图、阶梯图 | 柱状图（除非对比基数）|
| **类别对比** | 横向柱状图 | 竖向柱状图、气泡图 | 饼图（除非 ≤5 块）|
| **部分与整体** | 环形图 / 100% 堆叠柱 | 饼图（≤5 块）、树状图 | 3D 饼图 |
| **目标 vs 实际** | 子弹图（Bullet Chart）| 双柱对比图、对比 KPI 卡 | 单纯数字（没有对比）|
| **进度/完成率** | 进度条、仪表盘环 | 完成的格数 | 3D 进度条 |
| **漏斗/转化** | 漏斗图 | 阶段柱状对比 | 折线图 |
| **地理分布** | 地图（分级统计图）| 区域热力图 | 散点图（无空间含义）|
| **两两关系** | 散点图 + 趋势线 | 气泡图（多一维度）| 柱状图 |
| **数据构成** | 树状图（Treemap）| 堆叠柱、饼图 | 多层饼图 |
| **相关性矩阵** | 热力图 | 散点图矩阵 | 柱状图 |
| **大数字 / KPI** | KPI 数字卡 + sparkline | 单数字 + 同比环比 | 复杂图表 |
| **计划 vs 实际** | 双柱对比图 + 差异线 | 子弹图 | 折线图 |
| **里程碑 / 时间线** | 时间线（横向）| 甘特图 | 表格 |

## 二、图表标题的黄金法则

### 2.1 标题 = 结论，不是主题

| ❌ 错误（描述性） | ✅ 正确（结论性） |
| --- | --- |
| "Q3 营收图表" | "Q3 营收同比增长 35%，超额完成目标 8 个百分点" |
| "用户留存率" | "3 个月内用户留存率从 42% 提升至 58%" |
| "市场分析" | "目标市场 300 亿美元，我们能拿下其中 8%" |
| "团队规模" | "团队规模 12 人扩至 24 人，工程师占比 70%" |

**心法**：标题就是 PPT 上唯一一句话能传达的信息——观众没时间读完整张图。

### 2.2 标题模板

- **变化类**：[指标] 从 X 提升/降低到 Y，幅度 Z%
- **对比类**：[指标 A] 比 [指标 B] 高/低 [数量]
- **占比类**：[部分] 占整体的 [百分比]，是最大的细分
- **预测类**：预计 [指标] 在 [时间] 达到 [数量]
- **归因类**：[因素] 是导致 [结果] 的主要原因，占 [百分比]

## 三、数据呈现三原则

### 3.1 每个图表只表达一个观点

- 一个图表只回答一个问题
- 不要在一张图上同时展示"趋势 + 占比 + 对比"
- 如果有多个观点，拆成多张图

### 3.2 删除所有装饰

- 3D 效果 → 删除（除了 3D 地图）
- 阴影 / 渐变 → 删除（除非有数据含义）
- 网格线 → 弱化或删除
- 边框 → 删除
- 配色 → 只保留 1-3 个有意义的主色

### 3.3 对比讲故事

- 没有对比的数字没有意义
- 对比维度：去年 / 目标 / 行业平均 / 竞品 / 自己过去
- 一个数 + 一个对比 = 一个 insight

## 四、配色规范

### 4.1 主色 + 强调色 + 灰阶

```
推荐：3-5 色
- 主色 1 个：品牌色 / 主题色
- 强调色 1-2 个：用于关键数据、对比
- 灰阶 2-3 个：用于次要信息、背景、轴
```

### 4.2 场景配色建议

| 场景 | 推荐色调 | 原因 |
| --- | --- | --- |
| 战略/咨询 | 深蓝 + 灰 + 强调橙/红 | 稳重、可信、专业 |
| 销售/营销 | 品牌色 + 白 + 暖色 | 友好、活跃、行动导向 |
| 路演/融资 | 黑 + 白 + 1 个亮色 | 高级、聚焦、有冲击力 |
| 技术分享 | 深灰 + 蓝 + 绿 | IDE 风格、清晰 |
| 内部汇报 | 商务蓝 + 浅灰 + 绿/红 | 正式、可读、对比清晰 |
| 客户案例 | 客户品牌色 + 中性色 | 友好、贴近客户 |

### 4.3 红绿使用的坑

- 国际惯例：红 = 不好 / 减少，绿 = 好 / 增加
- 中国股市惯例：红 = 涨 / 好，绿 = 跌 / 不好
- **对内汇报**：用红绿没问题（团队熟悉中国习惯）
- **对外/国际/混用场合**：用蓝/橙或加图标，避免色盲 + 习惯冲突

## 五、中文化场景的特殊考量

### 5.1 字号

- 数字字号应该比周围文字大 1.5-2x
- 中文字符本身视觉权重高，数字更要突出
- 同一张图表：主数字字号 48-60px，辅助文字 16-20px

### 5.2 数字格式

- 千分位：1,000,000 而非 1000000
- 大数缩写：1.2M 而非 1200000（除非正式财报）
- 货币：¥1,000 或 1,000 元（统一一种）
- 百分比：35% 而非 35 %

### 5.3 单位前置

- ✅ "¥1.2M"，"30%"，"500 人"
- ❌ "1.2M 元"，"35 个百分比"，"500 人次"
- 单位放前面视觉更聚焦

### 5.4 时间格式

- ✅ "2026 Q1"、"2026年3月"、"过去 6 个月"
- ❌ "3/15/2026"、"上周"、"最近"
- 用具体时间段，不用相对时间

### 5.5 数据来源标注

- 放在图表正下方，不用脚注
- 格式："数据来源：XX 公司 / XX 报告 / 截止 YYYY-MM"
- 来源要可追溯（增加可信度）
- 每条证据在 `evidence_plan` 里带 `source_status`（`verified | user_reported | assumed | to_verify`）：用户自报的数字标 `user_reported`，未核实的标 `to_verify`，推算的标 `assumed`——不要把它们呈现得像 `verified`。

## 六、HTML/CSS 数据可视化建议

### 6.1 用 SVG 而非 Canvas

- SVG 易编辑（前端友好）
- 矢量、可缩放
- CSS 可控制颜色和动画
- D3.js / ECharts 都可以输出 SVG

### 6.2 推荐的可视化库

| 库 | 适用 | 体积 |
| --- | --- | --- |
| **ECharts** | 通用图表，国民级 | 较大（~900KB）|
| **Chart.js** | 简单图表 | 中（~70KB）|
| **D3.js** | 高度定制 | 大（~250KB）|
| **纯 SVG + CSS** | KPI 卡、进度条、对比 | 0 |
| **Recharts** | React 生态 | 中 |
| **VChart** | 字节出品的 ECharts 继承者 | 中 |

**建议**：单文件 HTML 项目用纯 SVG + CSS 做简单图表；复杂图表用 ECharts（CDN）。

### 6.3 纯 CSS 数据可视化技巧

- **KPI 数字**：CSS counter + @keyframes 计数动画
- **进度条**：CSS animation width
- **对比柱**：CSS Grid + flex + 渐变
- **时间线**：CSS flex + :before/:after
- **环形进度**：SVG stroke-dasharray + stroke-dashoffset 动画

## 七、常见数据可视化错误

| 错误 | 正确做法 |
| --- | --- |
| Y 轴不从 0 开始的折线图 | 折线图必须从 0 开始，否则会夸张趋势 |
| 用饼图展示 8 个以上分类 | 改用横向柱状图 |
| 双 Y 轴迷惑观众 | 用两张图分别展示 |
| 3D 饼图 / 柱状图 | 全部用 2D |
| 颜色仅作装饰 | 颜色必须有含义（分类、对比）|
| 字号不一致（数据字号比轴标签小）| 数据字号 ≥ 轴标签 |
| 一张图塞 10 个数据系列 | 拆成多张图，每张 ≤5 个系列 |
| 用面积图但层叠混乱 | 面积图只用于 2-3 个系列 |
| 数字没单位 | 每个数字必须有单位 |

## 八、汇报场景的"数字故事"

不同场景下，数字呈现侧重不同：

| 场景 | 数字侧重 |
| --- | --- |
| 战略汇报 | 市场份额、行业排名、3 年 CAGR |
| 销售提案 | ROI、节省成本、提升幅度、案例数字 |
| 路演 | 用户增长、营收、毛利率、留存 |
| 年度汇报 | KPI 完成率、同比、亮点项目核心数字 |
| 晋升答辩 | 业务破局数字、影响力范围、团队效率提升 |
| 复盘 | 目标 vs 实际、问题量化、改进后的对比 |
| 技术分享 | Benchmark、性能提升、复杂度下降 |

## 十、可视化自检清单

生成图表后检查：

- [ ] 标题是结论而非主题？
- [ ] 图表只表达一个观点？
- [ ] Y 轴从 0 开始（折线/柱状）？
- [ ] 颜色有含义而非装饰？
- [ ] 数据来源已标注？
- [ ] 每条证据标了 `source_status`（自报/待核/推算不冒充已验证）？
- [ ] 单位已标（%、¥、人）？
- [ ] 字号层次正确（数据 ≥ 轴标签 > 注释）？
- [ ] 删除所有 3D / 阴影 / 渐变装饰？
- [ ] 标签可读（不重叠、不被截断）？
- [ ] 关键数据有对比基准？
