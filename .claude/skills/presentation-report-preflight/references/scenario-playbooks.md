## 一、销售提案 / 客户方案（Pitch）

**目标**：让客户认可价值、签单。

**推荐骨架**：Problem → Solution → Benefit

**必杀 3 招**：
1. **首页用客户的痛点说话**——不要说"我们能做什么"，说"你是否遇到 XX 问题"
2. **每页标题必须是客户视角**——"节省 40% 成本"而非"我们的新功能"
3. **必须有竞品对比表**——不贬低对手，只讲差异化（数据 + 案例）

**推荐结构**：
```
1. 客户痛点（共情开场）
2. 现状分析（数据 + 行业背景）
3. 我们的方案（核心 3 个优势，每个 1 页）
4. 客户案例（PAST 叙述法，2-3 个真实案例）
5. 竞品对比（差异化矩阵）
6. 实施路径（时间线 + 里程碑）
7. ROI 计算（量化收益）
8. 下一步（demo / 试点 / 合同）
```

**视觉建议**：客户品牌色为主色调；商务简约；数据可视化为主，少装饰。

**禁忌**：技术细节堆叠、自吹自擂、PPT 超过 20 页。

## 二、融资路演（Investor Pitch）

**目标**：5-10 分钟让投资人建立兴趣，愿意约下次。

**推荐骨架**：Pitch Deck 10 页 或 AI 路演 6 件事。

**必杀 3 招**：
1. **前 60 秒必须答 3 件事**——什么问题、你的解法、为什么是机会
2. **市场大小要有梯子**——TAM / SAM / SOM 三层数据
3. **Traction 要具体**——不说"用户增长很快"，说"6 个月从 0 到 10K DAU，付费率 8%"

**推荐结构**：
```
1. Hook（1 句话讲清项目）
2. Problem（真实场景，不要泛泛而谈）
3. Solution（你的核心创新点）
4. Product Demo（截图 / 视频）
5. Market（TAM/SAM/SOM + 切入路径）
6. Business Model（怎么赚钱）
7. Traction（已验证的数据）
8. Competition（格局图 + 你的差异化）
9. Team（互补性 > 学历堆叠）
10. Ask（融资金额 + 用途分配）
```

**视觉建议**：现代简约、icon 化、节奏快（每页 30 秒讲完）。

**禁忌**：技术细节、团队学历堆砌、空泛愿景。

## 三、产品发布（Product Launch）

**目标**：让用户/媒体记住亮点、形成心智。

**推荐骨架**：Hero's Journey + Sparkline（Nancy Duarte）

**必杀 3 招**：
1. **开场讲"旧世界"的痛**——让观众感到共鸣
2. **核心亮点只讲 3 个**——多一个观众记不住
3. **必须有"明星时刻"**（star moment）——一个能上热搜的视觉/数据点

**推荐结构**：
```
1. Hook（数据/故事/视频）
2. 现状痛点（3 个场景）
3. 我们的愿景（future world）
4. 产品发布（hero 登场）
5. 核心功能 1（每功能一页，配演示）
6. 核心功能 2
7. 核心功能 3
8. 体验流程（用户视角 demo）
9. 价格 / 上市时间 / 获取方式
10. 愿景收尾（一年后世界会怎样）
```

**视觉建议**：高对比、大字号、强动画、节奏感强。

**禁忌**：功能列表、参数对比（除非是 to B）。

## 四、年度/季度工作汇报

**目标**：让老板看到你的价值，争取升职加薪。

**推荐骨架**：BARO + 年终述职 6 段式

**必杀 3 招**：
1. **数字必须对比**——不说"做了 XX"，说"让 YY 提升了 ZZ%"
2. **必须有 1-2 个亮点项目**（深讲，不要每个项目浅尝辄止）
3. **结尾直接要资源**——升职 / 加薪 / 招人 / 预算，勇敢说

**推荐结构**：
```
1. 自我介绍 + 汇报范围
2. 核心数据看板（关键 3-5 个指标）
3. 亮点项目 1（深讲：背景→行动→结果）
4. 亮点项目 2（深讲）
5. 其他工作（简述）
6. 不足与反思（诚实、有方案）
7. 学习与成长
8. 下季度/年度计划（目标+路径）
9. 需要的支持（资源/人员/预算）
10. 总结与感谢
```

**视觉建议**：商务正式、数据可视化为主、统一配色。

**禁忌**：流水账、只讲辛苦、没数字、没要资源。

## 五、晋升答辩（Promotion Defense）

**目标**：让评委认定你已具备下一级别的能力。

**推荐骨架**：字节 SOP / 麦肯锡 STAR

**必杀 3 招**：
1. **业务方法论要抽象**——评委不一定懂你的业务，要讲"我做的这套方法可以推广到 XX"
2. **业务难度讲清楚**——自然做好的不算成绩，破局的才算
3. **必须有金句**——至少 1-2 句业务理解金句，重复 2-3 次让评委记住

**推荐结构**：
```
1. 自我介绍（1 min：姓名、岗位、汇报范围、带多少人）
2. 业务理解（5 min：行业认知、关键挑战、机会判断）
3. 重点项目 1（5 min：业务背景→拆解逻辑→关键动作→难题和解法→结果）
4. 重点项目 2（5 min：同上）
5. 业务方法论（3 min：抽象出来可复制的思考框架）
6. 团队管理（2 min：人才识别、梯队建设）
7. 未来计划（3 min：从 1 到 100 的路径）
8. 总结收尾（1 min：1-2 句金句）
9. Q&A
```

**视觉建议**：极简专业、突出"判断力"、每页不超过 5 个 bullet。

**禁忌**：把晋升答辩讲成业务成果汇报、流水账、内容太散没抽象。

## 六、项目复盘

**目标**：把经验转化为组织能力、避免重复犯错。

**推荐骨架**：复盘四步法（联想/华为）

**必杀 3 招**：
1. **必须有数据对比**——目标 vs 实际，亮点 vs 不足
2. **原因分析要深挖**——不要停在表面，多问 3 个"为什么"
3. **必须有可复制的规律** + **下次行动**

**推荐结构**：
```
1. 项目背景与目标
2. 实际结果（Highlights + Lowlights）
3. 原因分析（成功的关键因素 + 失败的根本原因）
4. 经验总结（可复制的方法论）
5. 下次行动计划（具体负责人 + 时间）
```

**视觉建议**：文档感、对比图、时间线、表格为主。

**禁忌**：甩锅、只讲成功、原因分析太浅。

## 七、技术分享

**目标**：让同行学到东西，建立专业影响力。

**推荐骨架**：Why → What → How → Demo → Limitation

**必杀 3 招**：
1. **必须有 takeaway**——观众离开后能记住的 1-3 句话
2. **代码 / 架构图必须可读**——字号、对比度、配色都不能含糊
3. **限制与未来要诚实**——同行最讨厌"我做了一个完美产品"

**推荐结构**：
```
1. Why（为什么要做这件事 / 痛点）
2. What（我们的方案是什么 / 核心概念）
3. How（怎么实现的 / 架构 / 关键算法）
4. Demo（实际演示）
5. 性能数据（benchmark）
6. Limitations（目前的限制、坑、教训）
7. Future Work（下一步）
8. Q&A
```

**视觉建议**：类 IDE / Dark Mode、代码块高亮、架构图清晰、技术细节密度高。

**禁忌**：炫技、没有 takeaway、避谈限制。

## 八、培训授课

**目标**：让学员掌握可应用的技能。

**推荐骨架**：What → Why → How → 练习 → 检验

**必杀 3 招**：
1. **每 15-20 分钟必须互动**——问答、练习、小组讨论
2. **概念必须配例子**——抽象概念不落地就是没意义
3. **每个章节末有 Takeaway**——可贴出来当复习卡

**推荐结构**：
```
1. 课程介绍 + 学习目标
2. 概念 1（理论 + 例子）
3. 概念 2（理论 + 例子）
4. 概念 3（理论 + 例子）
5. 综合案例（动手练）
6. 工具 / 模板分享
7. 课后练习
8. Q&A + 答疑
```

**视觉建议**：视觉化要点、案例丰富、节奏慢、留白多。

**禁忌**：超过 30 分钟不互动、纯理论、跳过练习。

## 九、咨询交付（Consulting Deliverable）

**目标**：让客户做出决策。

**推荐骨架**：SCQA + 金字塔原理

**必杀 3 招**：
1. **结论先行**——执行层没时间听推导
2. **MECE 强约束**——分类要不重不漏
3. **风险与替代方案**——决策者最想知道"如果错了怎么办"

**推荐结构**：
```
1. 执行摘要（1 页核心结论）
2. 背景与挑战（SCQA）
3. 诊断分析（数据 + 框架）
4. 解决方案（金字塔结构）
5. 实施路径（时间线 + 资源）
6. 风险与缓解
7. 预期收益（量化）
8. 建议下一步
9. 附录（详细数据）
```

**视觉建议**：严谨、数据驱动、决策导向、留白少。

**禁忌**：堆数据、没结论、没替代方案。

## 十、战略汇报

**目标**：让管理层对齐方向、达成共识。

**推荐骨架**：SCQA + 三年规划 + 资源/风险

**必杀 3 招**：
1. **外部环境变化要讲清楚**——为什么"现在"必须改变
2. **路径必须可执行**——分阶段、有里程碑、有负责人
3. **必须配资源需求**——告诉老板要花多少钱/多少人

**推荐结构**：
```
1. 现状回顾（外部环境 + 内部能力）
2. 战略选择（2-3 个候选方向 + 对比）
3. 推荐方向（1 个 + 理由）
4. 三年目标（关键指标）
5. 第一年路径（季度里程碑）
6. 资源需求（人/钱/时间）
7. 风险与应对
8. 决策点（请管理层决定什么）
```

**视觉建议**：大气稳重、信息密度高、图表丰富。

**禁忌**：乐观到不真实、没风险、没资源评估。

## 十一、内部周会 / 站会

**目标**：让团队同步进度、识别阻塞。

**推荐骨架**：状态 → 阻塞 → 需要支持

**必杀 3 招**：
1. **每人不超过 5 分钟**
2. **每页只有 3 个 bullet**——已完成 / 进行中 / 阻塞
3. **明确需要的支持**——不是抱怨，是 ask

**推荐结构**（单页 5 分钟版）：
```
1. 标题（团队 + 周次 + 主持人）
2. 上周完成（3 条）
3. 本周进行中（3 条）
4. 阻塞项（最多 2 条）+ 需要谁支持
5. 跨团队协作（信息同步）
```

**视觉建议**：极简、共享屏幕友好。

**禁忌**：念 PPT、超过 30 分钟、没 ask。

## 十二、客户案例（Case Study）

**目标**：让潜在客户产生代入感、形成信任。

**推荐骨架**：客户背景 → 挑战 → 方案 → 数据结果 → 经验

**必杀 3 招**：
1. **必须用 PAST 叙述**——真实人物、真实场景、真实对话
2. **数字要可对比**——"之前 X，做了之后 Y，提升 Z%"
3. **客户原话是金句**——直接引用客户评价

**推荐结构**：
```
1. Hook（一句客户原话 / 一个震撼数据）
2. 客户背景（行业、规模、当时状态）
3. 挑战（具体痛点 + 量化影响）
4. 我们的方案（做了什么、为什么）
5. 实施过程（关键里程碑 + 时间线）
6. 结果（数据对比 + 客户反馈）
7. 可复制的经验
```

**视觉建议**：故事化、人物真实、对比震撼、配图丰富。

**禁忌**：只讲自己产品不讲客户变化、虚构细节、没有数据。
