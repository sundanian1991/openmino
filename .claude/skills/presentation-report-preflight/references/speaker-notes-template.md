## 一、演讲者备注的 3 个核心要素

每页 speaker notes 应包含：

1. **演讲要点**（3-5 句话）
   - 这一页要讲什么
   - 关键数据/案例是什么
   - 怎么讲（语气、节奏）

2. **可能的 Q&A**（2-3 个听众可能问的问题 + 建议回答方向）
   - 提前准备，避免被问倒

3. **关键数据/出处**（如果这页有数字）
   - 原始数据来源
   - 应对数据被追问

## 二、按页面类型的备注模板

### 2.1 封面页

```markdown
## 演讲者备注

【开场问候】
"大家好，我是 [姓名]，[职位]，[部门]。今天想花 [N] 分钟跟大家分享 [主题]。"

【钩子】
- 抛出核心问题或震撼数据
- 例："过去 6 个月，我们的留存率下降了 15%。今天想跟大家聊聊发生了什么、为什么、以及我们打算怎么办。"

【互动】
- 如果是大型场合，简短问"在座有谁遇到过类似情况吗？"
- 不要拖超过 30 秒

【时长】
- 30 秒-1 分钟
```

### 2.2 目录 / Agenda 页

```markdown
## 演讲者备注

【过渡语】
"今天我会分 [N] 个部分来讲：[目录]。重点会放在第二、第三部分，欢迎大家随时提问。"

【节奏控制】
- 30 秒过完
- 重点部分强调（如"重点会放在 X"）
- 告诉听众可以随时提问（建立安全感）

【常见 Q&A】
- Q: "PPT 结束后可以共享吗？"
- A: "会后我发到 [群/邮件]。"
```

### 2.3 数据展示页

```markdown
## 演讲者备注

【核心数据】
- 这一页展示的是 [指标名]：[核心数字]
- 关键对比：[对比基准 + 差异]

【讲解节奏】
- 先讲结论："我们 Q3 营收是 12.4M，同比增长 35%。"
- 再讲对比："目标 11M，超额 1.4M。"
- 必要时讲原因："主要来自 XX 业务的爆发。"

【可能的 Q&A】
- Q: "这个数字怎么算的？"
  A: "[具体口径 + 数据来源]"
- Q: "为什么同比涨这么多？"
  A: "[主要原因 1/2/3]"

【数据出处】
- 数据来源：[XX 系统 / XX 报告]
- 统计口径：[定义]
- 截止时间：[YYYY-MM-DD]

【时长】
- 1-2 分钟
```

### 2.4 对比分析页

```markdown
## 演讲者备注

【对比维度】
- 左：[对象 A 的数据] / 右：[对象 B 的数据]
- 关键差异：[差异点 + 量化]

【讲解技巧】
- 不要平均讲，重点讲差异
- "我们比对手强的地方在于 X，比对手弱的是 Y，我们打算怎么补 Y。"

【可能的 Q&A】
- Q: "为什么不和 [第三方] 比？"
  A: "[解释选择对比对象的逻辑]"

【时长】
- 1-2 分钟
```

### 2.5 方案介绍页

```markdown
## 演讲者备注

【方案核心】
- 这一页讲的是我们的 [方案名] 解决 [痛点]
- 核心创新点：[1-2 个差异化点]

【讲解技巧】
- 先讲痛点共鸣："你是否遇到过 XX 问题？"
- 再讲方案："我们的做法是 YY"
- 最后讲好处："这样会让客户/我们 ZZ"

【可能的 Q&A】
- Q: "为什么不做 [替代方案]？"
  A: "[解释选择路径的理由]"
- Q: "实施成本是多少？"
  A: "[给具体数字或范围]"

【时长】
- 2-3 分钟（核心方案可适当延长）
```

### 2.6 案例页

```markdown
## 演讲者备注

【案例故事化】
- 客户：[客户名 / 脱敏描述]
- 背景：[行业、规模、当时状态]
- 挑战：[具体痛点 + 量化影响]
- 我们的方案：[做了什么、为什么]
- 结果：[数据 + 客户反馈]

【讲解技巧】
- 用 PAST 叙述法（地点-动作-对话-想法）
- 引用客户原话："客户当时说：'XXX'"
- 让观众产生代入感

【时长】
- 2-3 分钟（一个案例）

【延伸】
- 如果有时间，可以多讲一个对照案例
```

### 2.7 行动计划页

```markdown
## 演讲者备注

【行动要点】
- 短期（1-3 个月）：[行动 1/2/3]
- 中期（3-12 个月）：[行动 1/2/3]
- 长期（1-3 年）：[愿景]

【需要支持】
- 人：[需要多少人 / 什么角色]
- 钱：[需要多少预算]
- 资源：[其他支持]

【可能的 Q&A】
- Q: "如果资源不到位能完成吗？"
  A: "[优先级排序 + 最小可行方案]"

【时长】
- 1-2 分钟
```

### 2.8 收尾 / CTA 页

```markdown
## 演讲者备注

【金句收尾】
- 用 1 句可被引用的话总结全文
- 例："数据告诉我们故事，而故事改变世界。"

【明确号召】
- 告诉听众下一步做什么：
  - "接下来 1 周，请每个团队 leader 提交一份本季度的 [XX] 计划"
  - "下周一会有一场 demo，欢迎来参加"
  - "PPT 会发到群里，有任何问题随时找我"

【Q&A 引导】
- "以上就是今天的分享。大家有什么问题或建议？"

【时长】
- 1-2 分钟

【结束时】
- 不要说"今天就到这里"
- 要说"期待和大家一起 XX"
```

## 三、演讲总时长控制

### 3.1 每页时间预估

| 页面类型 | 建议时长 |
| --- | --- |
| 封面 | 30 秒 - 1 分钟 |
| 目录 | 30 秒 |
| 过渡页 | 15 秒 |
| 数据展示 | 1-2 分钟 |
| 方案介绍 | 2-3 分钟 |
| 案例 | 2-3 分钟 |
| 对比分析 | 1-2 分钟 |
| 行动计划 | 1-2 分钟 |
| 收尾 | 1-2 分钟 |
| Q&A | 总时长的 20-30% |

### 3.2 总时长建议

| 场景 | 总时长 | 页数建议 |
| --- | --- | --- |
| 内部周会 | 5-15 分钟 | 3-5 页 |
| 产品发布 | 30-60 分钟 | 20-35 页 |
| 季度汇报 | 15-30 分钟 | 10-20 页 |
| 销售提案 | 30-60 分钟 | 15-25 页 |
| 融资路演 | 5-10 分钟（核心）| 10-12 页 |
| 晋升答辩 | 30-60 分钟 | 15-25 页 |
| 技术分享 | 30-90 分钟 | 25-40 页 |
| 战略汇报 | 60-90 分钟 | 25-40 页 |

## 四、演讲前的彩排自检

演讲前一天：

- [ ] 通读备注 2-3 遍
- [ ] 对着镜子或录音讲一遍
- [ ] 检查每页时长是否超标
- [ ] 准备 5 个可能被问到的问题的回答
- [ ] 检查 PPT 文件能在投影仪/对方电脑上正常打开
- [ ] 准备备份方案（PDF 版 / 微信发链接）

演讲当天：

- [ ] 提前 15 分钟到场测试设备
- [ ] 准备一杯水（保护嗓子）
- [ ] 手机静音
- [ ] 第一页熟悉、不背稿
- [ ] 保持眼神交流，不一直看屏幕

## 五、Q&A 处理技巧

### 5.1 接收问题

- "好问题。"（给自己 5 秒思考时间）
- "你问的其实是两个问题，我先回答第一个……"
- "让我确认一下你的问题……"（复述确认）

### 5.2 不知道答案怎么办

- "这个问题我需要回去确认，会后给你回复。"
- "这个数据我没有，但我可以分享相关的是 XX。"
- "我不确定，但我可以猜一个方向……"（如果对方只是讨论）

### 5.3 挑战性问题

- "我理解你的担忧。"
- "你提到的这个风险我们也考虑过，目前的应对是……"
- "这是个很好的问题，我现在的回答是 X，但你的视角让我重新思考……"

## 六、附录：演讲风格参考

### 6.1 对话感 vs 权威感

| 对话感（推荐） | 权威感（避免）|
| --- | --- |
| "我们一起看看……" | "我告诉你……" |
| "我发现一个有趣的事……" | "重点是……" |
| "你可能会想……" | "你们应该……" |
| "我的看法是……" | "结论是……" |

### 6.2 节奏与停顿

- 关键句前停 0.5 秒
- 关键句后停 1 秒（让信息沉淀）
- 翻页前后停 1-2 秒（给观众阅读时间）
- 不用语气词填满停顿（"嗯"、"呃"）

### 6.3 眼神交流

- 重点段落看 2-3 个具体观众
- 不要盯着屏幕或 PPT
- 不要扫视全场（会让人紧张）
- 眼神在不同区域切换（前台左、中、右各选一个人）
