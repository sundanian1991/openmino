## 一、汇报结构（4 项）

### 1.1 首因效应：开头是否抓住观众？

- [ ] 第 1 页不是简单的标题页，而是有钩子（故事/数据/视频/悬念）
- [ ] 前 30 秒内让观众知道"为什么要继续听"
- [ ] 自我介绍简洁（姓名、岗位、汇报范围，30 秒内完成）

**自检问题**：如果我是一个完全不了解这个主题的人，前 30 秒会让我留下吗？

### 1.2 中段：是否有清晰的叙事骨架？

- [ ] 整体有一个可识别的叙事框架（如 BARO / SCQA / 复盘四步法 / PSB）
- [ ] 每页只讲一个核心观点
- [ ] 页与页之间有过渡（不是突然跳转）
- [ ] 整体节奏有起伏（不是一直平铺）

**自检问题**：如果把每页标题连起来读一遍，能讲出一个完整故事吗？

### 1.3 结尾：是否留下清晰的行动或记忆点？

- [ ] 最后一页有金句或行动号召
- [ ] 不是简单的"谢谢聆听"
- [ ] 告诉听众下一步做什么

**自检问题**：观众听完 1 小时后，最可能记住的是哪句话？

### 1.4 整体时长

- [ ] 总时长在合理范围（按场景）
- [ ] 每页时间分配合理（核心页 2-3 分钟，过渡页 15-30 秒）
- [ ] 预留 Q&A 时间（总时长的 20-30%）

## 二、内容质量（4 项）

### 2.1 标题

- [ ] 每页标题是结论而非"主题"
- [ ] 标题清晰可独立读懂（不依赖上下文）
- [ ] 标题字号一致、风格统一

**自检问题**：把每页标题连起来读，是不是一份连贯的"摘要"？

### 2.2 数据与图表

- [ ] 每个数字都有单位（%、¥、人）
- [ ] 每个图表都有标题（结论性，不是描述性）
- [ ] 数据来源已标注
- [ ] 关键数据有对比基准（去年 / 目标 / 行业 / 竞品）
- [ ] 没有孤立数字（每个数都"对比"出来才有意义）

**自检问题**：把数字列出来，能不能讲出"所以呢"？

### 2.3 文字

- [ ] 没有大段文字堆砌
- [ ] bullet 数量符合 skill 规范（4-6 条 / 页）
- [ ] 关键词加粗或变色突出
- [ ] 没有错别字、错译

**自检问题**：每页文字扫一眼 5 秒内能 get 到要点吗？

### 2.4 故事化

- [ ] 有 1-2 个具体案例或故事（不是抽象描述）
- [ ] 案例有真实人物 / 真实场景 / 真实数据
- [ ] 有金句 / 可被引用的句子

**自检问题**：观众听完能复述出至少一个故事吗？

## 三、场景适配（4 项）

### 3.1 视觉风格是否符合场景？

- [ ] 战略/咨询 → 稳重严谨
- [ ] 销售/营销 → 客户友好
- [ ] 路演 → 现代简洁
- [ ] 内部汇报 → 商务正式
- [ ] 技术分享 → IDE 风格可读
- [ ] 培训授课 → 视觉化要点

**自检问题**：把 PPT 拿给目标听众看，他们会觉得"这就是我们要的样子"吗？

### 3.2 页面数量

- [ ] 在合理范围（5-25 页为主，按场景调整）
- [ ] 没有"凑数"页（每页都有不可替代的作用）
- [ ] 没有"装饰"页（纯视觉但不承载内容）

**自检问题**：删除任意 1 页，整体逻辑会断裂吗？

### 3.3 演讲者备注

- [ ] 每页都有演讲者备注
- [ ] 备注包含：要点 / 节奏 / Q&A / 数据来源
- [ ] 备注长度合理（每页 100-300 字）

### 3.4 视觉一致性

- [ ] 全篇配色统一（不超过 3-5 个主色）
- [ ] 全篇字体统一（标题/正文/注释分别一致）
- [ ] 全篇排版节奏统一（对齐方式、行间距）
- [ ] 没有突然出现的"突兀页"

---

## 四、可演讲性（4 项）

### 4.1 物理可读性

- [ ] 字号足够大（正文 ≥ 24px，标题 ≥ 36px）
- [ ] 对比度足够（深色背景用浅色字，反之亦然）
- [ ] 远距离可读（5 米外能看清关键信息）
- [ ] 图表可读（标签不被截断）

### 4.2 演讲节奏

- [ ] 有停顿 / 互动设计（不是一直念）
- [ ] 关键页有"明星时刻"（一张图、一句话、一个数据）
- [ ] 演讲者备注里有时长标注
- [ ] 彩排过至少 1 遍

### 4.3 Q&A 准备

- [ ] 准备了 5 个可能被问到的问题的回答
- [ ] 数据口径已统一（不会自相矛盾）
- [ ] 知道哪些问题超出范围（提前准备"我会后会确认"的回答）

### 4.4 备份与分发

- [ ] PPT 文件能在不同设备打开（HTML 单文件最稳）
- [ ] 准备 PDF 备份
- [ ] 知道会后怎么共享（群、邮件、链接）

## 五、自检流程

### Phase 1：交付前 1 天

- [ ] 通读整份 PPT 的标题链，是否讲得通？
- [ ] 抽查 5 页，看标题是否都是结论
- [ ] 检查所有数据是否有对比
- [ ] 演讲者备注是否完整

### Phase 2：交付当天

- [ ] 在不同设备打开测试（手机、平板、电脑）
- [ ] 检查排版没有错乱
- [ ] 备份 PDF / HTML 文件

### Phase 3：演讲前 30 分钟

- [ ] 测试投影 / 共享屏幕
- [ ] 通读关键页的演讲者备注
- [ ] 准备 5 个 Q&A 的回答
- [ ] 喝口水，深呼吸

## 六、12 项终极自检（用户提问式）

> **这是本 skill 自检的单一事实源（SSOT）。** SKILL.md 的 Phase 8 与 content-spec-guide 的自检都引用这里，不另起一套。改自检口径只改这一处。

用这 12 个问题**扫描**全篇，但**不要逐项打绿勾交差**——一份全 ✅ 的自检视为无效自检，对用户没有价值，还会制造虚假安全感。

12 个扫描项：

1. **如果观众只能记住一句话，会是哪句？**
2. **如果只能保留 5 页，是哪 5 页？**
3. **每页标题都是结论吗？**
4. **每个数字都有对比基准吗？**
5. **整体叙事骨架是什么？（能 1 句话说清）**
6. **结尾有明确的下一步吗？**
7. **场景适配对吗？（路演 vs 内部汇报不能一个风格）**
8. **演讲者备注完整吗？**
9. **物理可读性 OK 吗？（远距离、字号、对比度）**
10. **准备了 5 个 Q&A 的回答吗？**
11. **首页有钩子吗？**
12. **彩排过吗？**

`presentation-report-preflight` 额外扫描 5 项（preflight 专属）：

1. **下游是否能仅凭 `content_spec` 做出每页，而不需要重新发明文案、图表或素材？**
2. **每条证据是否带 `source_status`？`assumed/to_verify` 是否已落到脚注、且没被呈现得像已验证？**
3. **每页是否标了 `spec_density`？核心页 full、常规页 compact、>20 页混合？**
4. **`adapter_resolution` 是否如实记录 requested/resolved/status？未找到的下游有没有降级为 `universal_only` 而不是猜私有契约？**
5. **契约是否通过 `handoff-contract.schema.json`（必填字段齐、枚举值合法）？`references_consulted` 是否如实列出？**

**输出要求——指认最弱环，而非打勾**：

- 扫完 12 项后，只产出**最可能让这场汇报翻车的 1–2 个薄弱点**：具体到哪一页、哪个数字、哪个断言，并说明它为什么会在现场出问题。
- 每个薄弱点配一条**具体修复动作**（参见第七节修复建议）。
- 通过的项简列即可，不必逐条复述。
- 如果你诚实扫描后真的找不出任何弱点，那说明扫描不够狠——回去找，每份 brief 都有最薄弱的一环。

## 七、自检失败的常见原因 + 修复建议

| 失败 | 原因 | 修复 |
| --- | --- | --- |
| 标题链讲不通 | 没有叙事骨架 | 选一个（见 narrative-frameworks.md）|
| 数字没对比 | 没用对比基准 | 每个数加一个对比维度 |
| 视觉风格不匹配 | 没按场景选 preset | 见 scenario-playbooks.md |
| 演讲者备注空着 | 没意识到价值 | 用 speaker-notes-template.md 自动生成 |
| 结尾没号召 | 没考虑行动路径 | 加 1 张"下一步"页 |
| Q&A 准备不足 | 没预测问题 | 列 5 个"如果被问 X"的问题 |
| content_spec 不可执行 | 只写了视觉方向，没写上屏文案/图表/素材 | 为每页补 `primary_message`、`on_slide_copy`、`visual_blueprint`、`data_refs`、`asset_refs` |
| 数字真假不分 | 用户自报/推算的数字被呈现得像已验证 | 给每条证据补 `source_status`；`assumed/to_verify` 落脚注并进最弱环 |
| 下游适配落空 | 凭记忆假设下游 skill 存在 / 猜私有字段 | 走 `downstream-adapter-protocol.md` 运行时解析，未命中则降级 `universal_only` |

## 八、下游 Skill 集成建议

下游 PPT 生成 skill 在交付前可调用本清单：

1. **自动检测**（skill 可做）：
   - 标题中是否包含数字（粗略判断是不是结论）
   - 页面数量是否在合理范围
   - 是否有"谢谢"页面（提醒用户改为 CTA）
   - 配色 / 字体一致性

2. **引导用户自检**（skill 引导）：
   - 输出 12 项终极自检问题
   - 让用户逐项勾选
   - 未勾选项提供修复建议

3. **生成 checklist HTML**（skill 输出）：
   - 在导出 HTML 的隐藏页里附带 checklist
   - 演讲者可以打印或查阅
