# Work 技能 — 供应商管理日常思考助手

## 核心价值

年老师是供应商管理岗，管 32+ 供应商、3200+ 人、4 大产线。每天遇到各种场景：排名异常、同事要数据、供应商来电、需要约谈、新项目对接...

**这个技能的作用**：你描述一个工作场景 → 我自动匹配对应的思考检查清单（Plan 模板）→ 帮你系统化梳理"第一步想什么、第二步想什么、关键变量有哪些"。

**工作链路**：
```
年老师：/work 毅航业绩下滑了怎么办
   ↓
Work 技能：识别场景 → 匹配 A1 模板 → 输出思考步骤
   ↓
年老师：按步骤分析，提供数据
   ↓
Work 技能：给出结构化判断 + 决策建议
   ↓
年老师：确认后调用 official-doc 生成公文
```

---

## 调用方式

### 方式1：直接指定模板编号（最准）
```
/work A1          # 业绩异常判断
/work D3          # 供应商来电要量
/work B1          # 新供应商准入
```
**22个可用编号**：见下文"模板索引"表格。

### 方式2：场景描述自动匹配（最方便）
```
/work 毅航排名后30%了          → 自动匹配 A1
/work 同事问我要排名数据       → 自动匹配 D1
/work 新供应商引入             → 自动匹配 B1
/work 供应商来电要量           → 自动匹配 D3
/work 月度复盘                 → 自动匹配 B4
```

### 方式3：对话中直接唤起
```
年老师：@Mino 用 work 分析一下毅航业绩
Mino：识别为业绩异常场景 → 加载 A1 模板 → 输出分析框架
```

## 匹配逻辑

### 关键词映射表（技能内建）

```
业绩异常判断 (A1)：
  OR: 业绩下滑、排名下降、后30%、预警、断崖、峰值比、头部差距
  场景: 看到排名异常需要判断根因

约谈准备 (A2)：
  OR: 约谈、整改、连续预警、A级骤降、沟通
  场景: 触发约谈条件需要准备

分量调整决策 (A3)：
  OR: 分量、增减量、资源分配、调整份额
  场景: 月度分量评估或临时增量申请

转正延期决策 (A4)：
  OR: 转正、延期、试运行、6个月、里程碑
  场景: 新供应商试运行期满评判去留

合规问题定级 (A5)：
  OR: 合规、客诉、违规、红线、A类、信息安全
  场景: 收到客诉或发现违规需定级处理

清退评估 (A6)：
  OR: 清退、终止、连续3个月、后30%、诚信红线
  场景: 供应商达到清退条件需评估

新供应商准入 (B1)：
  OR: 新供应商、寻源、引入、准入、新项目
  场景: 有寻源需求，需执行8阶段SOP

新项目对接 (B2)：
  OR: 新项目、需求对接、同事提需求、业务需求
  场景: 同事提出新项目需求需评估承接

出差巡检准备 (B3)：
  OR: 巡检、现场、FCI、检查、出差
  场景: 要去现场巡检前系统化准备

月度复盘 (B4)：
  OR: 月度复盘、排名出炉、每月初、月度总结
  场景: 每月初需系统复盘上月排名

向上汇报准备 (B5)：
  OR: 汇报、给军哥、给易人哥、周会、月会
  场景: 向上级汇报前组织材料

降价谈判准备 (B6)：
  OR: 降价、谈判、价格、续签、成本优化
  场景: 与供应商谈判价格需准备

问题拆解 (C1)：
  OR: 想不清楚、理不清、复杂问题、MECE
  场景: 遇到模糊问题需要结构化拆解

邮件起草 (C2)：
  OR: 写邮件、起草、通知、发函
  场景: 想清楚后需要写邮件/公文

数据整理分析 (C3)：
  OR: 数据分析、看数据、整理、洞察
  场景: 拿到一堆数据需要看懂并提取洞察

制度流程梳理 (C4)：
  OR: 制度、流程、梳理、更新、SOP
  场景: 需要更新制度或设计流程

被同事@数据进度 (D1)：
  OR: 同事@、要数据、要进度、刘伟佳、刘乾坤、郭鑫
  场景: 内部群@要数据或进度，需快速回复

被领导点名 (D2)：
  OR: 领导问、军哥问、易人哥点名、点名
  场景: 军哥或易人哥直接询问需快速组织口径

供应商来电来消息 (D3)：
  OR: 供应商来电、要量、要支持、投诉、反映问题
  场景: 供应商主动联系需现场判断回应

供应商群异常信号 (D4)：
  OR: 群异常、24h未回复、态度变化、推诿
  场景: 供应商群观察到异常需快速介入

新供应商初筛 (E1)：
  OR: 新供应商初筛、推荐、评估、寻源发现
  场景: 有人推荐或自己寻源到新供应商需快速评估

准入评审决策 (E2)：
  OR: 试运行到期、转正评估、6个月评审
  场景: 新供应商第6个月末决定转正/延期/清退
```

### 匹配策略
1. **精确匹配**：场景描述含明确关键词（"新供应商"、"巡检"、"汇报"）→ 直接命中
2. **多候选**：同时命中多个模板 → 列出选项让年老师选择
3. **无匹配**：未识别出场景 → 建议使用 C1-问题拆解 或手动指定模板编号

## 使用示例

### 示例1：自动匹配
```
年老师：/work 毅航这个月排名后30%了，业绩下滑
Mino：识别到"排名"、"后30%"、"业绩下滑" → 匹配 A1-业绩异常判断
输出：
【模板】A1-业绩异常判断
【用途】看到排名下降时系统化判断异常原因
【触发信号】月度排名后30% / 周度断崖>20% / 自身峰值比<65%
请提供数据：
  1. 近6个月排名趋势
  2. 本月自身峰值比、头部差距
  3. 新人占比、出勤率
  4. 主管胜任5项指标（如有）
我将按A1模板5步法分析...
```

### 示例2：指定编号
```
年老师：/work B1 新供应商引入
Mino：加载 B1-新供应商准入 模板
输出：
【模板】B1-新供应商准入
【用途】新供应商寻源时执行8阶段SOP流程（T+15）
请确认需求：
  1. 项目背景：什么项目/多少人/何时上线/预计业绩？
  2. 供应商候选：已有推荐/需主动寻源？
  3. 资源可用：现有供应商是否有空余名额？
我将按8阶段推进...
```

### 示例3：快速响应（D类）
```

【模板】D3-供应商来电来消息（快速响应）
30秒判断：
  Step1：识别诉求类型 → 要量 / 要支持 / 反映问题 / 投诉
  Step2：查关键数据 → 当前排名 / 5维打分 / 资源匹配度
  Step3：判断真问题vs撒娇 → [判断树]
回应策略：
  - 要量：看5维打分，达标可增量，不达标先整改
  - 要支持：区分甲方责任 vs 供应商能力
话术模板已就绪，请提供供应商名称和具体诉求...
```

## 输出格式

Work 技能输出统一为以下结构：

```
【模板】[模板编号]-[模板名]
【用途】[一句话说明解决什么问题]
【触发信号】[什么情况下用这个模板]

【思考步骤】
1. 第一步：想什么 → 检查什么数据 → 判断什么
2. 第二步：...
3. ...

【关键变量】
- 变量1：[说明]
- 变量2：[说明]

【常见误区】
- ❌ [典型错误1]
- ❌ [典型错误2]

【决策输出】
[结论格式 + 示例]

【下一步衔接】
- 想清楚后 → 调用 official-doc [模板名] 生成公文
- 或 → [其他后续动作]
```

**D类模板额外输出**：
```
【30秒快速判断】
- 这是谁？要什么？紧急吗？
- 我现在能回答吗？需要什么信息？
- 回复口径（一句话模板）

【5分钟深入版】
- 如果有时间深入了解的思考框架
```
