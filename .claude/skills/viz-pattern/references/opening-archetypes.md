# 开场图骨架

> 15 种"第一张图"的骨架类型，定义画布比例、区域划分、叙事作用和禁忌。对标 fe-cinematic 的 hero-archetypes.md。

---

## 01 | 数据冲击

| 字段 | 值 |
|------|------|
| 名称 | 数据冲击（Data Shock） |
| 适用场景 | 有一个让人震惊的数字要第一时间传达 |
| 骨架结构 | 画布 16:9 · 标题区 15% 顶部 · 超大数字区 60% 居中（48-72px 粗体） · 注释区 15% 底部（解释上下文） · 脚注 10% |
| 叙事作用 | 第一眼建立"这件事很重要"的认知，跳过细节直接冲击 |
| 代表案例 | NYT 疫情死亡数字大字报 · FT"实际薪资缩水 10%" · 麦肯锡"行业利润下降 40%" |
| 禁忌 | 数字本身不震撼时不要用 · 不要放多个大数字（>3 个会打架） |

---

## 02 | 指标阵列

| 字段 | 值 |
|------|------|
| 名称 | 指标阵列（Metric Array） |
| 适用场景 | 需要同时展示 4-8 个关键指标的全景 |
| 骨架结构 | 画布 16:9 · 标题区 10% · 指标卡片区 75%（2×4 或 4×2 网格） · 脚注 15% · 每张卡片：数字 32px + 标签 11px + 趋势 ±X% |
| 叙事作用 | 第一眼建立"全局概览"认知，让读者自行扫描感兴趣的部分 |
| 代表案例 | Stripe Dashboard KPI 行 · Bloomberg 终端顶部指标 · 供应商季度评审总览 |
| 禁忌 | 指标 >8 个时密度过高 · 指标之间无关联时不要硬凑 |

---

## 03 | 全景时间线

| 字段 | 值 |
|------|------|
| 名称 | 全景时间线（Panoramic Timeline） |
| 适用场景 | 叙事的核心是时间维度，需要展示完整历史 |
| 骨架结构 | 画布 21:9 宽幅 · 标题区 10% · 时间轴区 75%（中央水平线贯穿） · 脚注 15% · 节点直径 12px，间距 ≥ 画布宽 8% |
| 叙事作用 | 第一眼建立"这是一个有时间跨度的故事"认知 |
| 代表案例 | FT"全球利率四十年" · 公司发展历程图 · 产品迭代时间线 |
| 禁忌 | 时间跨度 <3 个数据点不要用 · 不要用竖直时间线做开场（水平更有冲击力） |

---

## 04 | 对比开场

| 字段 | 值 |
|------|------|
| 名称 | 对比开场（Contrast Opening） |
| 适用场景 | 叙事核心是两个对象的巨大差异或反转 |
| 骨架结构 | 画布 16:9 · 标题区 10% · 左图区 40% · 分隔线 2% · 右图区 40% · 脚注 8% · 左标题 + 右标题在各自区域顶部 |
| 叙事作用 | 第一眼建立"两者差异很大"或"前后变化巨大"的认知 |
| 代表案例 | 麦肯锡"引入新供应商前后 SLA 对比" · BCG"行业集中度 Before/After" |
| 禁忌 | 两个对象无显著差异时不要用 · 不要做 >2 方的对比开场（太复杂） |

---

## 05 | 漏斗开场

| 字段 | 值 |
|------|------|
| 名称 | 漏斗开场（Funnel Opening） |
| 适用场景 | 叙事核心是衰减/转化/筛选过程 |
| 骨架结构 | 画布 16:9 · 标题区 10% · 漏斗图区 70% 居中（左宽右窄或上宽下窄） · 关键瓶颈标注在右侧 · 脚注 10% · 每层标注百分比 |
| 叙事作用 | 第一眼建立"大量进入、少量留下"的认知 |
| 代表案例 | 麦肯锡"客户转化漏斗：100 万访客 → 2% 购买" · 招聘漏斗 |
| 禁忌 | 不是递减关系不要用 · 漏斗层级 >6 层时过于密集 |

---

## 06 | 地图全景

| 字段 | 值 |
|------|------|
| 名称 | 地图全景（Map Panorama） |
| 适用场景 | 叙事核心是地理分布或区域差异 |
| 骨架结构 | 画布 16:9 · 标题区 10% · 地图区 70% 左侧 · 图例+排行 20% 右侧 · 脚注 10% · 地图色阶 ≤5 级 |
| 叙事作用 | 第一眼建立"空间分布不均"的认知 |
| 代表案例 | Our World in Data 全球贫困地图 · FT 选举地图 · 供应商全国站点分布 |
| 禁忌 | 没有地理维度时不要用 · 地图数据点 <10 个时太空 |

---

## 07 | 瀑布桥接

| 字段 | 值 |
|------|------|
| 名称 | 瀑布桥接（Waterfall Bridge） |
| 适用场景 | 叙事核心是从 A 值到 B 值的驱动因素分解 |
| 骨架结构 | 画布 16:9 · 标题区 10%（标题即结论） · 瀑布图区 75% · 脚注 15% · 起点柱子加粗 · 终点柱子加粗 · 增量柱子正/负双色 |
| 叙事作用 | 第一眼建立"变化有明确原因"的认知 |
| 代表案例 | 贝恩"EBITDA 桥：+15M 来自 volume，-8M 来自 price" · 利润桥分析 |
| 禁忌 | 不是数值变化分解不要用 · 驱动因素 >8 个时过于密集 |

---

## 08 | 雷达画像

| 字段 | 值 |
|------|------|
| 名称 | 雷达画像（Radar Profile） |
| 适用场景 | 叙事核心是多维能力/特征的综合评估 |
| 骨架结构 | 画布 4:3 或 1:1 · 标题区 10% · 雷达图区 65% 居中 · 关键发现区 15% 底部 · 脚注 10% · 基准线虚线灰色 |
| 叙事作用 | 第一眼建立"多维能力全貌"认知 |
| 代表案例 | 供应商健康度评估 · 个人能力画像 · 产品特性对比 |
| 禁忌 | 维度 <4 或 >8 时效果差 · 不适合纯数值对比（用柱状更好） |

---

## 09 | 散点洞察

| 字段 | 值 |
|------|------|
| 名称 | 散点洞察（Scatter Insight） |
| 适用场景 | 叙事核心是两个变量的关系或异常值识别 |
| 骨架结构 | 画布 16:9 · 标题区 10% · 散点图区 75% · 异常值标注在右侧 · 脚注 15% · 回归线/目标线叠加（2px 虚线） |
| 叙事作用 | 第一眼建立"有相关性"或"有异常值"的认知 |
| 代表案例 | Gapminder 收入-寿命散点 · "人效 vs 成本"供应商评估 · 异常检测 |
| 禁忌 | 数据点 <15 个时过于稀疏 · 无明显相关性时不要强加回归线 |

---

## 10 | 叙事引用

| 字段 | 值 |
|------|------|
| 名称 | 叙事引用（Narrative Quote） |
| 适用场景 | 需要用人的声音/故事来开场，数据后续支撑 |
| 骨架结构 | 画布 16:9 · 标题区 8% · 引用区 40%（左侧引用框，左边框 3px 品牌色，背景 #F8F9FA，padding 24px） · 数据区 42%（右侧支撑数据） · 脚注 10% |
| 叙事作用 | 第一眼建立"这是一个有人的故事"认知，建立情感连接 |
| 代表案例 | NYT 人物数据故事 · 客户满意度报告开场引用 · Alberto Cairo 案例 |
| 禁忌 | 纯数据无故事性时不要用 · 引用文字 >50 字时过于冗长 |

## 11 | 热力矩阵开场（新增）

| 字段 | 值 |
|------|------|
| 名称 | 热力矩阵开场（Heat Matrix Opening） |
| 适用场景 | 叙事核心是多维度交叉关系（如供应商×指标健康度） |
| 骨架结构 | 画布 16:9 或 1:1 · 标题区 10% · 热力矩阵区 70% · 图例+发现 15% 右侧 · 脚注 5% · 单元格 60×60px · 热力色阶 5 级 |
| 叙事作用 | 第一眼建立"多维度交叉关系全貌"认知，快速识别热点/冷点 |
| 关键设计 | 行标签左对齐 · 列标签顶对齐 · 单元格内数值 12px · 热点用饱和色（Red 600）· 冷点用浅色（Stone 200） |
| 代表案例 | 供应商×KPI 健康度矩阵 · 产品×渠道销量矩阵 · 地区×品类渗透率 |
| 禁忌 | 维度 <3×3 时信息稀疏 · 单元格 >10×10 时过于密集 |

## 12 | 桑基流入开场（新增）

| 字段 | 值 |
|------|------|
| 名称 | 桑基流入开场（Sankey Flow Opening） |
| 适用场景 | 叙事核心是资源/资金/客户的来源流向分析 |
| 骨架结构 | 画布 16:9 宽幅 · 标题区 10% · 桑基图区 75% · 关键节点标注 10% · 脚注 5% · 节点宽 20px · 流向线宽按值比例 · 层级 ≤5 |
| 叙事作用 | 第一眼建立"资源从哪里来、到哪里去"的认知 |
| 关键设计 | 源节点左侧对齐 · 目标节点右侧对齐 · 每层用不同色系 · 最大流量线宽 ≤30px · 添加百分比标注 |
| 代表案例 | 客户来源渠道分解 · 资金流向分析 · 供应商→产线人力分配 |
| 禁忌 | 层级 >5 时过于复杂 · 节点 >15 个时线条交叉过多 |

## 13 | 进度里程碑开场（新增）

| 字段 | 值 |
|------|------|
| 名称 | 进度里程碑开场（Milestone Progress Opening） |
| 适用场景 | 叙事核心是项目/季度关键节点的进度状态 |
| 骨架结构 | 画布 21:9 宽幅或 16:9 · 标题区 10% · 时间轴区 70% · 里程碑详情 15% · 脚注 5% · 里程碑直径 24px · 完成=实线+填充 · 未完成=虚线+空心 |
| 叙事作用 | 第一眼建立"当前在哪、还剩多少"的认知 |
| 关键设计 | 当前里程碑用品牌色高亮 · 已完成用灰色 #999 · 待完成用浅色 #CCC · 里程碑间距 ≥80px · 添加完成度百分比 |
| 代表案例 | 项目实施进度 · 季度 OKR 完成度 · 供应商准入流程 |
| 禁忌 | 里程碑 <3 或 >8 个时效果差 · 没有明确时间节点不要用 |

## 14 | 气泡定位开场（新增）

| 字段 | 值 |
|------|------|
| 名称 | 气泡定位开场（Bubble Positioning Opening） |
| 适用场景 | 叙事核心是三维数据的市场定位（X 轴、Y 轴、气泡大小） |
| 骨架结构 | 画布 16:9 · 标题区 10% · 散点图区 75% · 象限说明 10% · 脚注 5% · X/Y 轴定位 · 气泡直径 20-80px（第三维） |
| 叙事作用 | 第一眼建立"竞争格局"或"市场定位"认知 |
| 关键设计 | 象限用背景色块区分（透明度 10%）· 气泡面积 ∝ 数值 · 关键气泡加边框高亮 · 添加平均线（虚线） |
| 代表案例 | "人效 vs 成本"供应商定位 · "增长率 vs 份额"产品矩阵 · 客户价值分层 |
| 禁忌 | 气泡 <10 或 >30 个时效果差 · 第三维数据不明显时退化为散点图 |

---

## 15 | 仪表盘/速度计开场（新增）

| 字段 | 值 |
|------|------|
| 名称 | 仪表盘/速度计开场（Gauge Dashboard Opening） |
| 适用场景 | 叙事核心是单指标 vs 目标达成度 |
| 骨架结构 | 画布 4:3 或 1:1 · 标题区 15% · 仪表盘区 65% 居中 · 目标值+说明 15% · 脚注 5% · 表盘直径 280-320px · 指针长 100-120px |
| 叙事作用 | 第一眼建立"当前进度 vs 目标"的直观认知 |
| 关键设计 | 表盘角度 240°（从 7 点到 5 点）· 分段着色：红色 <60% · 黄色 60-80% · 绿色 >80% · 目标标记线 3px · 当前值 48px 居中 |
| 代表案例 | 年度业绩完成度 · 预算执行率 · KPI 达成率 · 项目进度百分比 |
| 禁忌 | 不是百分比/比率不要用 · 多个仪表盘时不要 >4 个（考虑用指标阵列） |
