# Anti-Patterns · 反模式清单

> 可插拔模块。不同 AI 渲染技能的共性错误，出图前自检拦截。
> 目标：让不同 AI 产出一致，不是各做各的。

---

## 一、叙事意图层

### A1：标题写描述，不是结论

| 错误 | 正确 |
|------|------|
| "供应商健康度分析" | "毅航健康度全面领先，3 家低于基准线" |
| "Q1 业绩趋势" | "Q4 环比+37%，创 18 个月新高" |
| "各渠道转化对比" | "付费渠道转化率 3× 于自然流量" |

**拦截**：如果标题去掉最后一个词（"分析""趋势""对比"）后还能成立，就是描述性标题。

### A2：一个图塞多个信息

一张图表达一个洞察。同时问"趋势+占比+关系"→ 拆成多张图。

**拦截**：能写出"而且"连接的标题 → 拆图。

### A3：高亮过多

所有柱子半彩色、所有折线都用主色 → 没有高亮。

**拦截**：数高亮元素。≤ 总数的 10%。10 个柱子最多 1 个高亮。

## 二、视觉编码层

### B1：颜色没有语义理由

| 错误 | 正确 |
|------|------|
| 每个柱子不同颜色（好看） | 全部 Stone 300，只高亮异常值 |
| 彩虹色折线 | 全部灰，一条 Warm 500 |

**拦截**：问"这个颜色编码了什么信息？"回答不了 → 去掉。

### B2：用了禁止的图型

| 禁止 | 替代 |
|------|------|
| 饼图（>3 分类） | 水平条形图 |
| 双 Y 轴 | 上下分图共享 X 轴 |
| 3D 图表 | 2D 扁平 |
| 雷达图 >8 维度 | 平行坐标或小多图 |
| 意面图（>5 条线） | 小多图或单线高亮 |

### B3：渐变/阴影/发光

这些效果在流式渲染中会闪烁，且违反设计体系。

**拦截**：SVG 中无 `filter`、`gradient`、`drop-shadow`。ECharts 中 `animation: false`。

### B4：数据不从零基线开始

柱状图 Y 轴截断 → 放大差异，误导读者。

**拦截**：柱状图 yAxis min 必须为 0。

## 三、标注层

### C1：标注重复数字

| 错误 | 正确 |
|------|------|
| "9月：37%" | "新方案上线 → 效率↑37%" |
| "最高点" | "旺季+促销叠加" |
| "最低" | "供应链中断两周" |

**拦截**：标注里如果只有数字没有原因 → 补"为什么"。

### C2：标注位置遮挡数据

标注放在数据元素正上方，遮挡柱顶或数据点。

**拦截**：标注与数据元素间距 ≥12px，不重叠。

### C3：标注过多

每张图超过 3 个标注 → 噪音。

**拦截**：标注数 ≤ 3。超过 → 只保留解释高亮点的那个。

---

## 四、布局层

### D1：标题到内容间距不足

标题紧贴图表 → 呼吸感差。

**拦截**：标题到内容 ≥12px。

### D2：文字溢出容器

单行文字超出卡片宽度 → 被裁剪。

**拦截**：中文单行不超过 20 字（14px 字号），超出换行。

### D3：元素不对称导致视觉偏移

右边标注多 → 整张图重心偏右。

**拦截**：检查视觉重心。右侧标注面积 ≈ 左侧留白。

### D4：缺少脚注/来源

图没有数据来源和时间范围 → 不可信。

**拦截**：底部必须有 Stone 400 小字脚注。

---

## 五、数据层

### E1：数据未灰化先高亮

先画彩色数据再"灰化其余" → 容易残留不该有的颜色。

**拦截**：严格按六层绘制法——第 4 层全灰，第 5 层局部着色。

### E2：分类按字母排序，不按意图

数据排序影响阅读路径。字母排序无叙事。

**拦截**：按最优→最差、按时间、按叙事意图排序。

### E3：缺失数据用零填充

缺失值画成零 → 误导。

**拦截**：缺失值用 "?" 或灰色占位，不用 0。

## 六、AI 特有错误（不同模型各有）

### F1：过度装饰型

常见于"想做得好看"的 AI。表现：
- 装饰性图标泛滥
- 卡片渐变背景
- 数据点加阴影
- 标题加 emoji

**修正**：回到 Warm 50 + 白卡 + 单色线条的骨架。

### F2：信息填充型

常见于"想提供更多信息"的 AI。表现：
- 标题后面加一大段解释文字
- 图表下方追加分析段落
- 每个数据点都加标注

**修正**：图表本身说话。文字只放标题、标注、脚注三处。

### F3：保守型

常见于"怕出错"的 AI。表现：
- 全部灰色，不敢高亮
- 标题模糊（"数据概览"）
- 不加任何标注

**修正**：SPEC 的叙事意图已经决定了高亮什么。按 SPEC 执行。

### F4：自由发挥型

常见于"有创意"的 AI。表现：
- 自创图型（不在 72 种模式内）
- 自配色（不用 Warm/Teal/Coral/Stone/Sky）
- 自定字号（不用 10/11/12/14/15/18/26/42 级）

**修正**：所有决策从 SPEC + 13-VISUALIZATION.md + 本文件的参考表中查表，不自创。
