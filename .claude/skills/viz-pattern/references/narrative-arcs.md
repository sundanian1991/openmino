# 多图表叙事弧线

> 8 种多图表叙事结构，定义每张图的角色和节奏

---

## 弧线列表

### 01. 调查式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 抛出异常（一个反直觉的数据点或异常信号） → 图2: 追踪线索（拆解异常的维度或时间线） → 图3: 揭示真相（锁定根因或关键变量） → 结论: 行动建议 |
| 节奏 | 图1: 高冲击 → 图2: 深入 → 图3: 高密度 → 结论: 呼吸 |
| 适用场景 | 数据异常排查、问题根因分析、审计报告、安全事故复盘 |
| 代表案例 | "某供应商流失率突然飙升 → 按产线拆解发现仅金条线异常 → 定位到某班组排班变更导致离职潮 → 建议恢复原排班并增设过渡期" |
| 弧线长度 | 3-5 张图 |

调查式弧线的核心设计原则是"好奇心驱动"。第一张图必须制造认知冲突——数据与预期不符，迫使观众想追问"为什么"。第二张图承担拆解任务，将大问题切成可操作的维度，每一层都离真相更近。第三张图是"破案时刻"，所有线索汇拢到一个清晰的结论。这种弧线适合汇报场景中需要"说服上级关注某个问题"的时刻，关键在于第一张图的冲击力是否足够让听众放下手机。

---

### 02. 对比式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 建立基准（现状或对照组的全貌） → 图2: 呈现差异（实验组/对比组的差异点） → 图3: 量化影响（差异带来的实际影响） → 结论: 选择建议 |
| 节奏 | 图1: 呼吸 → 图2: 高冲击 → 图3: 高密度 → 结论: 深入 |
| 适用场景 | A/B测试汇报、方案选型、竞品分析、前后效果对比、供应商对标 |
| 代表案例 | "6家头部供应商当前人效基准 → 引入新培训体系后 vs 未引入的差异 → 差异折算为月度产能增量 → 建议全面推广" |
| 弧线长度 | 2-4 张图 |

对比式弧线的关键在于"公平感"。第一张图必须让双方站在同一起跑线——同样的指标、同样的时间段、同样的口径。如果基准不公平，后续所有对比都站不住。第二张图是最有戏剧性的一张，差异的呈现方式决定了说服力：绝对值差异适合业务决策，百分比差异适合战略判断。第三张图将差异翻译为"跟你有什么关系"——把抽象的差距变成具体的成本、收益或风险。对比式弧线不适合"微妙差异"的场景，差异不明显时用对比式反而暴露弱点。

### 03. 演进式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 起点（初始状态或历史基线） → 图2: 轨迹（变化趋势与关键转折点） → 图3: 终点或预测（当前状态或未来走势） → 结论: 阶段判断 |
| 节奏 | 图1: 呼吸 → 图2: 深入 → 图3: 高冲击 → 结论: 呼吸 |
| 适用场景 | 季度/年度复盘、成长曲线、市场趋势分析、项目里程碑回顾 |
| 代表案例 | "供应商准入数量12个月基线 → 每月净增减趋势及政策影响标注 → 预测Q3将达到产能瓶颈 → 建议提前启动新供应商储备" |
| 弧线长度 | 3-6 张图 |

演进式弧线的灵魂是"转折点"。一条平滑上升的曲线没有故事性，真正有力的是标注出"政策变化""人员调整""外部冲击"等关键节点。第二张图上每一个标注都是一次叙事机会，让观众理解"不是数据在变，是事件在驱动变化"。演进式弧线要警惕"后视偏差"——用历史数据讲故事很容易把随机波动解读为因果律。标注转折点时必须诚实区分"已确认的因果关系"和"时间巧合"。这种弧线在复盘会上效果最好，因为观众有亲历感，每个转折点都能引发讨论。

### 04. 诊断式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 症状呈现（表面问题的全景） → 图2: 逐层拆解（从表象到结构的原因分析） → 图3: 处方（针对根因的解决方案及预期效果） → 结论: 优先级排序 |
| 节奏 | 图1: 高冲击 → 图2: 高密度 → 图3: 呼吸 → 结论: 深入 |
| 适用场景 | 绩效诊断、流程瓶颈分析、供应商问题排查、质量事故分析 |
| 代表案例 | "3月供应商质检通过率骤降至82% → 按环节拆解发现培训环节缺失+排班过密双重因素 → 针对性方案及模拟效果 → 优先解决排班问题（影响占68%）" |
| 弧线长度 | 3-5 张图 |

诊断式弧线与调查式的区别在于"已知问题 vs 未知问题"。调查式是从异常出发找原因，诊断式是从已知症状出发做结构化分析。第一张图的任务是让所有人都对"问题有多严重"达成共识——用绝对值、同比、排名等让症状不可忽视。第二张图是技术含量最高的一张，需要建立清晰的归因框架（人/机/料/法/环，或按流程环节），让观众能跟着逻辑走而不是被数据淹没。第三张图的核心是"可操作性"，方案必须具体到"谁在什么时间做什么"，否则诊断就停留在分析层面。

---

### 05. 提案式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 机会识别（未被满足的需求或市场空白） → 图2: 方案展示（提案的核心逻辑和关键设计） → 图3: 收益预估（量化投入产出比） → 结论: 决策请求 |
| 节奏 | 图1: 深入 → 图2: 高密度 → 图3: 高冲击 → 结论: 呼吸 |
| 适用场景 | 新项目立项、预算申请、战略提案、供应商引入建议 |
| 代表案例 | "金条产线Q4产能缺口达15% → 引入新供应商A的三阶段方案 → 预估ROI及风险对冲 → 申请启动准入流程" |
| 弧线长度 | 3-4 张图 |

提案式弧线的核心矛盾是"你的热情 vs 决策者的风险厌恶"。第一张图必须证明"不做这件事的代价"，而不是"做这件事的好处"——缺口和痛点比机会更有说服力。第二张图需要平衡"足够详细以建立信心"和"不过于复杂以免引发更多质疑"。第三张图是关键的"数值锚定"时刻，用一个清晰的数字（ROI、回本周期、成本节约）让决策变成算术题。提案式弧线最忌讳的是第三张图用模糊的"预计将大幅提升"收尾——没有具体数字的提案不是提案，是愿望。

### 06. 漏斗式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 全景入口（大范围、高视角的全量数据） → 图2: 层层筛选（按条件逐级过滤） → 图3: 精准聚焦（最终目标群体或关键发现） → 结论: 聚焦行动 |
| 节奏 | 图1: 呼吸 → 图2: 高密度 → 图3: 高冲击 → 结论: 深入 |
| 适用场景 | 用户分群、供应商分层、市场细分、资源分配、风险排查 |
| 代表案例 | "30+供应商全量画像 → 按产能/合规/人效三维筛选 → 锁定5家高风险供应商 → 逐家制定处置方案" |
| 弧线长度 | 3-5 张图 |

漏斗式弧线的视觉特性很强——每一张图的范围都比上一张小，数据密度却越来越高。这种"广→窄"的结构天然适合资源有限、需要聚焦的场景。设计时要注意每一层筛选的"理由"必须自洽：如果第一层按产能筛、第二层按合规筛、第三层按人效筛，三者的逻辑关系需要交代清楚（是并列维度还是递进维度）。漏斗式弧线的常见陷阱是"筛到最后只剩1-2个目标"——太少的结论让人觉得筛选条件是倒推的。保留3-5个目标最合适，既体现聚焦又保留选择空间。

### 07. 金字塔式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 核心结论（先亮观点，一句话结论的可视化） → 图2: 支撑论据（3-5个关键证据的并列展示） → 图3: 底层数据（原始数据或方法论） → 结论: 重申立场 |
| 节奏 | 图1: 高冲击 → 图2: 高密度 → 图3: 深入 → 结论: 呼吸 |
| 适用场景 | 高管汇报、电梯演讲配套材料、战略决策会议、董事会报告 |
| 代表案例 | "结论：应优先投资供应商培训体系 → 三大论据：人效提升27%、流失率下降15%、ROI达3.2倍 → 原始数据附录及计算方法 → 确认投资决策" |
| 弧线长度 | 3-4 张图 |

金字塔式弧线与所有其他弧线的区别是"先说结论"。这种结构来自麦肯锡的SCQA框架，核心理念是决策者的时间有限，必须在前30秒就让他知道"你到底要说什么"。第一张图是最难设计的一张——要把复杂分析浓缩为一个有力的视觉锚点（一个大数字、一个明确的趋势箭头、一个红绿灯式的判断）。第二张图是"可信度建设"，每个论据必须独立成立且相互支撑。第三张图是"备查层"，大部分人不会看，但看了就必须能还原你的分析过程。金字塔式弧线不适合学术讨论（学术需要先展示推理再得出结论），但非常适合商业决策场景。

### 08. 螺旋式

| 字段 | 值 |
|------|------|
| 叙事结构 | 图1: 初始认知（第一轮观察） → 图2: 深入后反转（新的视角推翻或修正初始认知） → 图3: 升维整合（将矛盾统一到更高框架） → 结论: 认知升级 |
| 节奏 | 图1: 呼吸 → 图2: 高冲击 → 图3: 深入 → 结论: 高密度 |
| 适用场景 | 战略复盘、认知升级型汇报、研究总结、复杂问题的多轮分析、年度反思 |
| 代表案例 | "初步观察：供应商数量与产能正相关 → 深入后发现：超过20家后管理成本吞噬规模效应 → 升维框架：最优供应商数量存在甜蜜点（12-18家） → 重新定义供应商策略" |
| 弧线长度 | 3-5 张图 |

螺旋式弧线是8种中最有"智力含量"的结构，也是风险最高的一种。它的力量来自"推翻自己"——第二张图否定第一张图的结论，制造认知张力，然后用第三张图在更高维度统一矛盾。这种弧线不适合时间紧张的汇报（观众需要时间消化反转），但非常适合深度讨论、战略研讨或内部分享。设计螺旋式弧线的核心技巧是"反转的诚实性"：第二张图的反转必须来自新的数据或新视角，而不是为了制造戏剧性而故意设一个弱的第一轮结论。如果第一轮结论本身就站不住，反转就失去了力量。
