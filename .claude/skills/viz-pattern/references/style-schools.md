## 学派详情

### 01. 麦肯锡极简

| 字段 | 值 |
|------|------|
| 创始人/代表 | McKinsey & Company 咨询团队 |
| 核心特征 | 1. 标题即结论 — 每张图表的标题是一句完整判断句，不是"销售额趋势"而是"华东区 Q3 销售额同比增长 23%" 2. 去装饰最大化 — 无网格线、无图例（直接标注）、无背景色、无阴影 3. 前对比后 — 用灰色淡化历史数据，用蓝色突出当前/目标数据 4. 每张图只讲一个观点 — 多指标拆成多张图，不堆叠 |
| 主色 | #051C2C（深海军蓝） |
| 辅助色 | #6BACD0（浅蓝）、#A9A9A9（灰色，用于对比淡化） |
| 背景色 | #FFFFFF |
| 文字色 | #333333（正文）、#051C2C（标题） |
| 字体倾向 | Sans-serif 为主。标题 14-16px 加粗，正文 10-12px 常规，注释 8-9px 浅灰。避免 Serif，保持商务利落 |
| 构图偏好 | 左上标题 + 左侧 Y 轴 + 底部 X 轴 + 右上角直接标注数据值。留白充裕，图表占页面 60-70%。横向条形图优于纵向柱状图（标签可读性） |
| 适用场景 | 高管月报、战略委员会汇报、投资分析备忘录、CEO 一页纸 |
| 反面案例 | 社交媒体信息图（太素）、品牌展示（缺乏情感）、创意提案（太严肃） |

### 02. 贝恩洞察

| 字段 | 值 |
|------|------|
| 创始人/代表 | Bain & Company 咨询团队 |
| 核心特征 | 1. 渐进揭示 — 从大图到细节逐层展开，每层只加一个维度 2. 绿色暗号 — 主色绿色贯穿，配合深灰和白色 3. 金字塔叙事 — 结论在最顶端，支撑数据在下方逐层展开 4. 瀑布图专精 — 用于展示从基准到目标的增量分解（营收 = 基准 + 增量A + 增量B - 流失） |
| 主色 | #CC0000（贝恩红） |
| 辅助色 | #00875A（贝恩绿）、#333333（深灰） |
| 背景色 | #FFFFFF |
| 文字色 | #1A1A1A（正文）、#CC0000（强调） |
| 字体倾向 | Sans-serif。标题 16-18px 加粗，子标题 12-14px，数据标签 10px。瀑布图和瀑布分解是标志性图表类型 |
| 构图偏好 | 标题在左上，图表居中偏左，右侧留注释空间。瀑布图从左到右流动，正负值用绿红区分。金字塔图自上而下收窄 |
| 适用场景 | 竞争分析报告、变革管理叙事、绩效提升路线图、董事会简报 |
| 反面案例 | 实时监控（不适合静态叙事）、社交传播（配色太企业）、学术研究（结论先行≠假设检验） |

### 03. BCG 矩阵

| 字段 | 值 |
|------|------|
| 创始人/代表 | Bruce Henderson / Boston Consulting Group |
| 核心特征 | 1. 矩阵思维 — 万物皆可四象限，用两条轴切割出策略空间 2. 增长-份额经典 — 纵轴市场增长率、横轴相对市场份额，四格命名（明星/现金牛/问号/瘦狗） 3. 气泡大小 = 第三维度 — 通常代表收入规模或资源投入 4. 战略色块 — 每个象限用不同深浅表示战略优先级 |
| 主色 | #00A651（BCG 绿） |
| 辅助色 | #5B6770（深灰）、#8CC63F（亮绿） |
| 背景色 | #FFFFFF |
| 文字色 | #333333（正文）、#00A651（标题强调） |
| 字体倾向 | Sans-serif。象限标签 12-14px 加粗，气泡标签 9-10px，轴标题 11px。保持网格清晰可读 |
| 构图偏好 | 严格的四象限布局，十字分割线清晰。横轴从高到低（右到左），纵轴从低到高（下到上）。气泡不重叠，错开排列 |
| 适用场景 | 产品组合分析、投资优先级决策、业务单元战略定位、资源分配讨论 |
| 反面案例 | 连续数据分布（硬分四格会丢失信息）、时间序列（矩阵是静态快照）、公众传播（术语门槛高） |

### 04. Tufte 数据墨水

| 字段 | 值 |
|------|------|
| 创始人/代表 | Edward Tufte（《视觉化展示信息的定量原则》） |
| 核心特征 | 1. 数据墨水比最大化 — 墨水只花在数据上，每一滴非数据墨水都是浪费 2. 抹去非数据元素 — 无网格线、无图例框、无装饰边框、无背景渐变 3. 微观/宏观同时阅读 — 整体可见趋势，局部可见细节 4. sparkline — 内联微型折线图，嵌入正文段落中 5. 数据密度优先 — 一张图容纳尽可能多的数据点 |
| 主色 | #000000（纯黑，用于数据和文字） |
| 辅助色 | #666666（灰色，用于坐标轴和辅助标注） |
| 背景色 | #FFFFFF |
| 文字色 | #000000 |
| 字体倾向 | Serif 优先（如 Georgia、Garamond），尊重传统排版。正文 10-12px，注释 8-9px。sparkline 无需标签，靠上下文理解 |
| 构图偏好 | 极简。数据点直接标注值而非依赖 Y 轴。坐标轴极细（hairline，0.5px 以下）。用数据本身的形态说话，不靠框架 |
| 适用场景 | 学术论文图表、研究报告、高信息密度仪表盘、任何追求信息纯度的场景 |
| 反面案例 | 营销展示（太素无冲击力）、公众传播（门槛高）、需要情感共鸣的场景（极致理性） |

### 05. Financial Times

| 字段 | 值 |
|------|------|
| 创始人/代表 | Financial Times 视觉新闻团队（Alan Smith、John Burn-Murdoch 等） |
| 核心特征 | 1. 粉红底色 — 延续 FT 报纸传统，图表底色微带粉调 2. 多色系但有序 — 使用 6-8 色调色板，每色有明确对应关系 3. 注释即叙事 — 在图表关键转折点放置标注框，解释发生了什么 4. 时间线即故事 — X 轴时间跨度大时，用灰色区间标注关键事件 |
| 主色 | #333333（深灰/黑） |
| 辅助色 | #E6772E（橙）、#BA5A53（砖红）、#6AAFE6（蓝）、#A2AA44（橄榄绿） |
| 背景色 | #FFF1E5（FT 粉白） |
| 文字色 | #333333 |
| 字体倾向 | Sans-serif（MetricWeb FT 专用字体）。标题 14-16px 加粗，注释 9-10px 斜体，轴标签 9px。注释框是核心叙事工具 |
| 构图偏好 | 横向宽图（16:9 或更宽），左侧留 Y 轴空间，底部时间轴。注释框浮于图表之上，带细线指向对应数据点。小多图（small multiples）排列展示多国/多品类对比 |
| 适用场景 | 经济数据趋势分析、政策影响评估、行业周期跟踪、财经媒体图表 |
| 反面案例 | 品牌视觉（粉白底色抢品牌）、暗色环境（底色不适配）、纯内部汇报（风格偏媒体化） |

### 06. The Economist

### 08. Gapminder 气泡

| 构图偏好 | 图表上方保留"阅读指南"区域。误差带（confidence band）是标志性元素。不追求美观冲击，追求诚实可读 |

| 反面案例 | 营销传播（诚实标注削弱说服力）、品牌展示（风格偏学术）、追求视觉冲击的场景 |

| 文字色 | #202124（浅色模式）/ #E8EAED（深色模式） |
| 字体倾向 | Sans-serif（Roboto / Google Sans）。标题 16-18px 加粗，轴标签 11px，悬停提示 12px。数字用等宽字体（Roboto Mono）对齐 |
