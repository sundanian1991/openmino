## 模板选择指南

### 通用公文（templates/general/）

**决策请示（decision-request.md）**

**适用场景**：供应商选型、方案选择、资源申请等需要领导决策的事项

**核心结构**：
- 事项背景
- 备选方案（表格对比）
- 我方建议
- 需要决策
- 预期影响

---

**汇报（report.md）**

**适用场景**：周报、月报、项目总结、专项汇报

**核心结构**：
- 工作概况
- 核心成果
- 数据分析
- 问题与风险
- 下一步计划

---

**通知（notification.md）**

**适用场景**：会议通知、变更通知、紧急事项通报

**核心结构**：
- 通知事项

- 注意事项
- 联系方式

**适用场景**：跨部门协作、方案审阅、资源协调

**核心结构**：
- 协作事项

## 写作标准

### 核心原则

**段落逻辑优先，视觉装饰为零，适度留白舒适**

### 具体规则

1. **开头称呼**：称呼单行，空一行后接主体

2. **关键词标识**：每个段落开始用 `<p><strong>关键词：</strong>` 标识

3. **禁止装饰**：不使用分割线、边框等视觉装饰

4. **数据结构化**：时间、人员、数据等用列表/表格清晰呈现

5. **适度留白**：段落之间空一行，保持阅读节奏

6. **结尾行动**：明确的行动请求或请示

### 标准格式示例

```
<p>尊敬的XXX：</p>

<p>一句话概括邮件内容。</p>

<p><strong>事实部分：</strong></p>

<p>具体事实描述，用列表或表格呈现关键数据（时间、人员、数据）。</p>

<p><strong>分析判断：</strong></p>

<p>基于事实的分析，使用"综合评估认为"、"这说明"等专业表述。</p>

<p><strong>建议/结论：</strong></p>

<p>明确的建议或结论，附上理由和依据。</p>

<p><strong>风险提示/后续安排：</strong></p>

<p>必要的风险说明或后续行动步骤。</p>

<p>结尾总结句和行动请求（请审阅/请批示/如有问题请反馈）。</p>
```

## 常见问题

**Q: 公文和邮件有什么区别？**

A: 公文侧重内容结构和专业性，邮件侧重发送渠道。本技能专注公文内容起草，如需发送可用 email 技能。

**Q: 模板不够用怎么办？**

A: 可以基于现有模板修改，或告诉我需要什么场景，我会新增模板。

**Q: 如何让公文更专业？**

A: 遵循"段落逻辑优先"原则，用表格/列表呈现数据，用专业表述（"综合评估认为"、"这说明"等）。
