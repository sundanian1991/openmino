# 公文起草检查规则体系

> 所有模板必须遵守的检查规则，确保公文专业、规范、得体

---

## 一、语言风格检查（L1 - 基础层）

**检查方式**：关键词检测 + 语义分析

| 检查项 | 规则 | 禁用词/模式 | 正确表达 |
|--------|------|------------|---------|
| **口语化表达** | 禁止 | 这个、那个、咱们、还行、差不多、挺、挺好、挺不错的 | 本项目、该方案、双方、良好、符合预期 |
| **模糊词** | 禁止 | 看情况、也许、可能、大概、应该可以、估计 | 预计、预计约、待定、视情况而定 |
| **情绪化表达** | 禁止（建议部分除外） | 我觉得、我希望、我认为、我建议 | 建议、拟、综合评估认为 |
| **废话开场** | 禁止 | 好的、收到、没问题、明白 | 直接进入主题 |
| **感叹号** | 禁止（紧急通报除外） | 任何"!" | 使用句号或省略号 |
| **emoji** | 禁止 | 任何表情符号 | 纯文本 |
| **网络用语** | 禁止 |YYDS、绝绝子、躺平、内卷 | 正式表述 |
| **缩写不标准** | 禁止 | ASAP、BTW、FYI | 尽快、顺便、供参考 |
| **反问句** | 禁止 | 难道...吗？怎么会...呢？ | 陈述句 |
| **设问句** | 谨慎使用 | 为什么...呢？（除非有明确答案） | 直接给出结论 |

---

## 二、场景适当性检查（L2 - 核心层）

**检查方式**：场景识别 + 内容审核

### 2.1 向上审批场景

| 禁用内容 | 理由 | 替代方案 |
|---------|------|---------|
| 内部供应商负面标签：配合度低、反馈慢、态度反复、前科 | 政治敏感，会被解读为对内部工具支持不够 | 不说配合度问题，说"时间差异"、"排期对齐" |
| "技术对接排期"作为唯一理由 | 站不住脚，老板会问"为什么不让内部也对接？" | 必须有明确的排期冲突证据 |
| 只说优点不说风险 | 不客观，误导决策 | 优点/风险/成本都要呈现 |
| 情绪化抱怨 | 不专业 | 客观陈述事实 |
| 细节过多 | 领导不需要知道细节 | 提炼核心信息 |

### 2.2 对外沟通场景

| 禁用内容 | 理由 | 替代方案 |
|---------|------|---------|
| 公司内部数据、未公开信息 | 保密要求 | 使用已公开信息或脱敏数据 |
| 贬低竞争对手 | 商业风险 | 客观对比，突出自身优势 |
| 承诺无法兑现的内容 | 法律风险 | 留有余地，使用"预计"、"力争" |

### 2.3 决策请示场景

| 禁用内容 | 理由 | 替代方案 |
|---------|------|---------|
| 只给一个选项 | 领导没有选择空间 | 至少两个备选方案 |
| 理由与结论不匹配 | 逻辑不成立 | 理由要支撑结论 |
| 优缺点不平衡 | 不客观 | 每个方案都要有优缺点 |
| 没有预期影响 | 领导无法评估风险 | 必须说明优点/风险/成本 |

## 三、政治敏感性检查（L3 - 关键层）

**检查方式**：敏感词检测 + 上下文分析

| 检查项 | 规则 | 检测方式 | 处理建议 |
|--------|------|---------|---------|
| **内部供应商表述** | 不给内部供应商贴负面标签 | 检测"配合度"、"反馈慢"、"态度"、"前科"等词 + 内部供应商名称 | 改用中性表述或删除 |
| **内部工具支持** | 避免被解读为"对内部工具支持不够" | 检测选第三方不选内部的理由 | 理由要充分，或考虑赛马机制 |
| **内部部门** | 不点名批评内部部门 | 检测部门名称 + 负面描述 | 改用"相关部门"或删除负面描述 |
| **赛马机制触发** | 两家都没把握时优先赛马 | 检测"两家"、"无把握"、"无参考" + 选一家 | 建议改为赛马机制 |
| **内部供应商决策** | 遵循"三原则" | 检测决策请示 + 内部供应商 | 应用内部供应商决策三原则 |

### 内部供应商决策三原则

```
1. 不给内部供应商贴负面标签
2. 无充分把握时优先"赛马机制"
3. 决策理由要经得起"为什么不选内部供应商"的追问
```

### 供应商选择决策树

```
两家都有把握 → 选更合适的
一家有把握 → 选有把握的
两家都没把握 → 赛马机制，数据说话
```

## 四、结构完整性检查（L4 - 形式层）

**检查方式**：模板匹配 + 必填字段检查

### 4.1 通用结构要求

| 文档类型 | 必需部分 | 缺失提示 |
|---------|---------|---------|
| **决策请示** | 事项背景、备选方案、我方建议、需要决策、预期影响 | "缺少XX部分，请补充" |
| **汇报** | 工作概况、核心成果、数据分析、问题风险、下一步计划 | "缺少XX部分，请补充" |
| **通知** | 通知事项、时间地点、参与人员、注意事项 | "缺少XX部分，请补充" |
| **协作** | 协作事项、需求说明、时间要求、预期产出 | "缺少XX部分，请补充" |

### 4.2 决策请示特殊要求

| 检查项 | 规则 |
|--------|------|
| **备选方案数量** | 至少2个方案，单方案需说明"唯一可行" |
| **备选方案表格** | 必须包含：方案、优点、缺点、风险 |
| **我方建议** | 必须明确"建议采用XX" |
| **需要决策** | 必须明确"请确认是否XX" |
| **预期影响** | 必须包含：优点、风险、成本（至少2项） |

## 五、逻辑一致性检查（L5 - 内容层）

**检查方式**：逻辑分析 + 数据对比

| 检查项 | 规则 | 检测方式 |
|--------|------|---------|
| **理由与结论匹配** | 理由要支撑结论 | 理由列表与结论对照分析 |
| **优缺点平衡** | 不能只说优点 | 检测"缺点"、"风险"部分是否为空 |
| **数据一致性** | 同一数据多处引用需一致 | 提取所有数字，对比同一指标 |
| **时间逻辑** | 时间线不能矛盾 | 提取所有日期，排序检查 |
| **因果逻辑** | 不能因果倒置 | "因为A所以B"逻辑检查 |
| **量化表述** | 能量化尽量量化 | 检测"很多"、"很少"、"大部分" → 改为具体数字或比例 |

---

## 六、格式规范性检查（L6 - 呈现层）

**检查方式**：HTML结构检查 + 格式验证

| 检查项 | 规则 | 检测方式 |
|--------|------|---------|
| **表格格式** | 必须有表头、边框 | `<table>` + `<th>` + `border` |
| **段落结构** | 段落间空一行 | `<p>`标签之间无`<br>` |
| **关键词标识** | 关键段落用`<strong>`加粗 | 检测`<p><strong>XXX：</strong>` |
| **分割线** | 禁止使用`<hr>` | 检测`<hr>`标签 |
| **HTML标签闭合** | 所有标签正确闭合 | HTML解析器检查 |
| **列表格式** | 使用`<ul>`或`<ol>`，不用手动编号 | 检测`1.` `2.`手动编号 |

---

## 七、检查流程（执行顺序）

```
起草完成
    ↓
L1 语言风格检查（基础）
    ↓ 通过
L2 场景适当性检查（核心）
    ↓ 通过
L3 政治敏感性检查（关键）← 失败 → 重大问题，必须修改
    ↓ 通过
L4 结构完整性检查（形式）
    ↓ 通过
L5 逻辑一致性检查（内容）
    ↓ 通过
L6 格式规范性检查（呈现）
    ↓ 通过
✅ 检查通过，可以发送
```

## 九、检查规则使用方式

**模板文件末尾格式**：

```markdown
---

## 本模板检查规则

### L1 语言风格检查
- [ ] 无口语化表达（这个、那个、咱们、还行、挺好）
- [ ] 无模糊词（看情况、也许、可能、大概）
- [ ] 无情绪化表达（我觉得、我希望、我认为）
- [ ] 无废话开场（好的、收到、没问题）
- [ ] 无感叹号、emoji、网络用语

### L2 场景适当性检查
- [ ] 无内部供应商负面标签（配合度低、反馈慢、态度反复）
- [ ] 理由经得起追问（"为什么不选内部供应商"）
- [ ] 优缺点平衡，不只说优点
- [ ] 细节适度，领导不需要知道细节

### L3 政治敏感性检查
- [ ] 不给内部供应商贴负面标签
- [ ] 无充分把握时优先考虑赛马机制
- [ ] 决策理由要经得起追问
- [ ] 避免被解读为"对内部工具支持不够"

### L4 结构完整性检查
- [ ] 有事项背景
- [ ] 有备选方案表格（至少2个方案）
- [ ] 有我方建议
- [ ] 有需要决策
- [ ] 有预期影响（优点/风险/成本）

### L5 逻辑一致性检查
- [ ] 理由与结论匹配
- [ ] 数据一致性（同一数据多处一致）
- [ ] 时间逻辑无矛盾
- [ ] 能量化尽量量化

### L6 格式规范性检查
- [ ] 表格有表头、边框
- [ ] 段落间有空行
- [ ] 关键段落用<strong>加粗
- [ ] 无分割线、手动编号
```

## 十、常见问题速查

| 问题 | 检查层级 | 解决方案 |
|------|---------|---------|
| 被领导质疑"为什么不选内部供应商" | L3 | 应用内部供应商决策三原则 |
| 被领导说"太口语化" | L1 | 去除口语化表达，使用正式表述 |
| 被领导说"理由站不住脚" | L5 | 理由要支撑结论，经得起追问 |
| 被领导说"只说好的不说坏的" | L2/L5 | 优缺点平衡，风险必须呈现 |
| 被领导说"看不懂你想说什么" | L4 | 结构完整，逻辑清晰 |
| 被领导说"格式不专业" | L6 | 遵循格式规范，表格/段落/关键词 |
