# 公文起草技能

> 专业化公文起草模板库，持续打磨，越来越好用

---

## 技能定位

从 `email` 技能中提取的公文模板部分，独立成专门技能，便于持续打磨和扩展。

---

## 适用场景

**需要起草以下类型公文时**：
- 向上汇报/决策请示
- 供应商全生命周期沟通
- 内部协调/跨部门协作
- 合规管控/风险通报
- 商务谈判/合同讨论

## 使用方式

**触发方式**：
```
用户说："帮我起草一个XXX的公文/邮件/请示"
   ↓
根据类型选择对应模板
   ↓
读取模板，结合用户需求填充
   ↓
输出完整公文内容
```

---

## 核心原则

**邮件写作标准（专业简洁，空间舒适）**：

1. **段落逻辑优先**：每个段落开始用 `<p><strong>关键词：</strong>` 标识
2. **视觉装饰为零**：不使用分割线、边框等装饰
3. **数据结构化**：时间、人员、数据等用列表/表格清晰呈现
4. **适度留白**：段落之间空一行，保持阅读节奏

**标准格式**：
```
<p>尊敬的XXX：</p>

<p>一句话概括邮件内容。</p>

<p><strong>事实部分：</strong></p>

<p>具体事实描述，用列表或表格呈现关键数据。</p>

<p><strong>分析判断：</strong></p>

<p>基于事实的分析。</p>

<p><strong>建议/结论：</strong></p>

<p>明确的建议或结论，附上理由和依据。</p>

<p><strong>风险提示/后续安排：</strong></p>

<p>必要的风险说明或后续行动步骤。</p>

<p>结尾总结句和行动请求。</p>
```

## 与 email 技能的区别

| 维度 | email 技能 | official-docs 技能 |
|------|-----------|------------------|
| **定位** | 发送邮件（SMTP） | 起草公文内容 |
| **功能** | 模板 + 发送 | 专注模板打磨 |
| **用途** | 需要发送邮件时 | 需要起草内容时 |
| **依赖** | 需要 SMTP 配置 | 无依赖，即用即走 |
