# 深度阅读工作流 - Obsidian+Claude

> 将扫描版PDF转化为中英双语深度阅读笔记的可复用工作流

---

## 何时使用

当你需要：
- 深度阅读一本扫描版PDF书籍
- 建立中英双语对照笔记
- 在Obsidian中可视化地整理知识
- 随时与AI交互，补充笔记内容

---

## 核心价值

- ✅ **本地运行**：所有数据都在本地，不上传云端
- ✅ **可视化操作**：在Obsidian界面完成，无需终端
- ✅ **结构保留**：表格、公式、图片都能保留
- ✅ **双语对照**：原版+中文翻译+总结归纳
- ✅ **随时交互**：可以在任何位置提问、添加笔记

---

## 工作流步骤

### 第1步：PDF转Markdown

**推荐工具**：MinerU（简单、直接、效果好）

```bash
# 安装
pip install mineru

# 转换
pdf2md your-book.pdf -o book.md

# 保留图片
pdf2md your-book.pdf -o book.md --images-dir ./images
```

**备选工具**：PaddleOCR（多语言、GPU加速）

```bash
# 安装
pip install paddlepaddle paddleocr

# 转换
paddleocr --img_dir book.pdf --output_dir ./output
```

---

### 第2步：导入Obsidian

1. 创建vault文件夹（如`深度阅读/书名/`）
2. 导入Markdown文件
3. 在Obsidian中预览，确认内容完整

---

### 第3步：章节分割

**手动分割**（推荐）：
1. 读Markdown，找到章节标题（`# 第一章`、`## 1.1`等）
2. 从每个章节开始，剪切到新文件
3. 命名规范：`01-第一章.md`, `02-第二章.md` ...

**自动分割**（复杂文档）：
让Claude帮你分割：
```
请把这个Markdown文件按章节分割成多个文件：
1. 识别所有一级标题（# 开头的）
2. 每个章节生成单独的.md文件
3. 文件命名：序号+章节名
```

---

### 第4步：单章节处理（标准化Prompt）

这是最关键的一步，使用以下Prompt：

```
请帮我处理这个章节：

1. **分段理解**：把章节分成几个主题段落
2. **每段做三件事**：
   - [总结] 用中文概括核心观点
   - [翻译] 把原文翻译成中文（保留英文原文）
   - [归纳] 提炼关键概念、方法、结论

3. **输出格式**：
```
## [段落标题]

### 原文
[英文原文]

### 中文翻译
[翻译内容]

### 总结归纳
- 核心观点：[一句话概括]
- 关键概念：[列出3-5个]
- 实践启示：[如何应用到实践中]
```

4. **要求**：
   - 翻译准确、自然
   - 总结简洁、有价值
   - 在原文件上直接修改，保留原文
```

**示例输出**：

```markdown
# 第一章 导论

## 1.1 什么是深度学习

### 原文
Deep learning is a subset of machine learning that uses neural networks with multiple layers to progressively extract higher-level features from raw input.

### 中文翻译
深度学习是机器学习的一个子集，它使用多层神经网络，从原始输入中逐步提取更高级别的特征。

### 总结归纳
- **核心观点**：深度学习通过多层神经网络逐层提取特征，实现从低级到高级的信息抽象。
- **关键概念**：
  - 神经网络：模拟人脑神经元连接的计算模型
  - 多层结构：每层负责提取不同抽象级别的特征
  - 特征提取：从原始数据中识别有意义模式的过程
- **实践启示**：在设计深度学习模型时，需要考虑每一层的作用，确保特征能够有效传递和增强。
```

---

### 第5步：审查与迭代

**审查要点**：
- 翻译是否准确、自然？
- 总结是否有价值？
- 有没有遗漏关键信息？
- 归纳是否清晰？

**迭代方法**：
- 在Obsidian中直接修改不满意的部分
- 针对问题部分，重新让Claude处理：
  > "这段的总结太简略了，请补充更多细节"
  > "这个翻译不太准确，请重新翻译"

---

### 第6步：批量处理所有章节

**基于单章节的经验，批量处理**：

```
请按照第一章的处理方式，处理所有章节：

1. 保持相同的输出格式
2. 每个章节生成单独的.md文件
3. 文件命名：01-第一章.md, 02-第二章.md ...
4. 处理完成后告诉我完成情况
```

**关键**：Claude会学习第一章的模式，应用到所有章节。

---

### 第7步：生成目录

让Claude基于章节文件生成目录：

```
请基于所有章节文件，生成一个目录（TOC）：
```
# 目录

- [第一章 导论](./01-第一章.md)
- [第二章 基础概念](./02-第二章.md)
...
```
```

## Obsidian + Claudian 连接配置

### 安装Claudian插件

Claudian是专门为Obsidian设计的插件，提供直接在Obsidian内使用Claude的能力。

**步骤**：

**1. 下载插件文件**
- 访问：https://github.com/zhumengkai/Claudian/releases
- 下载以下三个文件：
  - main.js
  - manifest.json
  - styles.css

**2. 创建插件文件夹**
- 创建一个名为 `claudian` 的文件夹
- 将上述三个文件放入该文件夹中

**3. 放入Obsidian插件目录**
- 打开Obsidian
- 设置 → 第三方插件 → 打开插件文件夹
- 在plugins文件夹中，放入claudian文件夹

**4. 启用插件**
- 关闭安全模式
- 在第三方插件列表中找到Claudian
- 启用插件

**5. 使用Claudian**
- 在Obsidian中打开任意笔记
- 使用Claudian的命令启动Claude
- Claude会直接在Obsidian界面内操作

## 工具对比

| 工具 | 优势 | 劣势 | 推荐场景 |
|------|------|------|---------|
| **MinerU** | 简单、直接PDF→MD | 图像可能不完美 | 首选 |
| **PaddleOCR** | 多语言、GPU加速 | 需要配置 | 需要GPU时 |
| **Marker** | 结构保真度高 | License限制 | 复杂文档 |
| **Claudian** | 专门为Obsidian设计、直接在界面操作 | 需手动下载 | 必选 |

## 常见问题

**Q: Claudian插件连接不上？**
- 确保Claudian插件已启用
- 检查插件文件夹是否正确放置
- 确认三个文件（main.js, manifest.json, styles.css）都已下载
- 运行`claude mcp list`检查连接

**Q: OCR效果不好？**
- 尝试不同工具：MinerU → PaddleOCR → Marker
- 调整参数：`--dpi 300`提高分辨率
- 扫描质量差的需要手动处理

**Q: 翻译质量不好？**
- 提供更明确的Prompt："用专业术语，保持原文风格"
- 分段处理，避免一次性处理太多内容
- 迭代优化，建立翻译标准

在阅读过程中随时添加：
```markdown

这个观点很有启发，它让我想到...

### 待解决问题

```

### 技巧2：跨章节关联
```
请分析第一章和第三章之间的关联，它们如何互为补充？
```

### 技巧3：知识图谱
让Claude分析全书的关键概念和关系：
```
请分析这本书的核心概念体系，画出它们之间的关系图
```
