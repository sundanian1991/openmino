# 母版选择与拦截协议

本文件用于修复两个关键风险：

- 元素清单化：模型把主题词、物件和标签当成清单堆进画面。
- 母版复刻化：模型把高质量案例当成可描摹模板，直接复制手机、纸条、挡板、沙漏等空间拓扑。

正确做法是先选择一个案例作为**质量锚点**，再抽取它的抽象不变量，最后为当前内容写出必须变异的地方。母版不是可复制画面，也不是空间模板；它只提供质量标尺和动作类型。

## 母版使用总原则

每次使用母版必须同时满足：

- **保留不变量**：比例克制、留白、真实物件质感、小黑参与核心动作、标签少而准。
- **必须变异**：主物件、空间方向、动作关系、道具组合、标签位置、视角或叙事重心中，至少 3 项必须和母版不同。
- **先读懂再好看**：不看说明，读者 3 秒内也能说出“小黑正在被什么困住 / 挡住 / 拉扯 / 审查 / 筛掉”。

如果生成图第一眼像某张案例的换皮版，就判失败，即使它背景、标签和小黑都对。

## 标准模式母版索引

### 01 meeting-pull-in

文件：

```text
assets/examples/01-meeting-pull-in.png
```

适合：

- 会议、同步、对齐、临时拉人、下班前被叫回。

抽象骨架：

- 多个外部请求从一侧施加拉力。
- 小黑被拉向一个工作入口或任务入口。
- 留白很大，配件很少。

可复用关系：

- “任务/会议/消息”作为卡片来源。
- 小黑被拉进主物件或工作现场。
- 标签在来源附近，不要每个物件都贴满字。

失败信号：

- 直接复制“左卡片 + 中间小黑 + 右电脑”的拓扑。
- 电脑太大，变成商品图。
- 线太多，像蜘蛛网。
- 纸条太多，像信息流合集。

### 02 message-overload

文件：

```text
assets/examples/02-message-overload.png
```

适合：

- 消息过载、任务涌出、已读不等于完成、催促。

抽象骨架：

- 一个真实物件作为“输出口 / 涌出口 / 源头”。
- 小黑正在挡、推、接、压住或整理涌出的东西。
- 画面表达“东西出来太多，人挡不住”。

可复用关系：

- 主物件是“出口”，不是信息展示屏；出口不必是手机。
- 小黑必须挡、推、接或整理，不是站着看。
- 标签只点出 2-4 个真实话术。

失败信号：

- 直接复制“左侧手机 + 中间纸条 + 右侧小黑挡板 + 沙漏”的拓扑。
- 当前内容不是消息或手机经验，却仍然强行用手机。
- 出现聊天 UI 或截图。
- 纸条文字太多。
- 主物件过大，画面像手机广告。

### 03 production-alert

文件：

```text
assets/examples/03-production-alert.png
```

适合：

- 报警、返工、打结、问题回流、边界情况、下班失败被拽回。

抽象骨架：

- 一个真实长条物或路径形成卡点、结、回流或缠绕。
- 小黑的动作集中在卡点上。
- 画面表达“本来应该顺着走，但在这里卡住 / 回来了”。

可复用关系：

- 线缆就是冲突的物理路径。
- 小黑动作集中在“结”上。
- 标签贴近问题点，不要把全流程画出来。

失败信号：

- 直接复制“横穿线缆 + 中央结 + 右侧小黑”的拓扑。
- 线缆变成装饰曲线，和小黑没有力学关系。
- 标签散太远，看起来像海报注释。
- 配件过多，冲淡“打结”的核心现场。

### 04 code-review-rework

文件：

```text
assets/examples/04-code-review-rework.png
```

适合：

- 审查、校对、复核、返工、验收、谁来兜底。

抽象骨架：

- 一个真实检查工具把内容局部放大、压住或框住。
- 小黑在检查区域里执行修、查、改、盖、圈。
- 画面表达“表面完成了，但还要被复核 / 兜底”。

可复用关系：

- 放大镜是“被审查/要再看”的物理工具。
- 小黑必须在被检查或检查别人之间形成清楚关系。
- 标签要像审查意见，不像流程名称。

失败信号：

- 直接复制“中央纸堆 + 大放大镜 + 右侧红印章”的拓扑。
- 放大镜太大，主体变成单一物件特写。
- 小黑只是站在镜子里，没在做动作。
- 纸堆太满，像办公素材拼贴。

### 05 ai-automation-badge

文件：

```text
assets/examples/05-ai-automation-badge.png
```

适合：

- AI 先做、人来兜底、岗位重组、自动化牵引、身份被重新贴标签。

抽象骨架：

- 一个真实标签、身份牌或纸带正在重新定义某个东西。
- 小黑用身体阻止、拉回、撕开或按住这个标签变化。
- 画面表达“系统给了新名字，但人还在承担后果”。

可复用关系：

- 标签/纸带代表系统给人的新定义。
- 小黑和挂绳/标签发生拉扯。
- 主体横向铺开，配件极少。

失败信号：

- 直接复制“左小黑 + 中工牌 + 右修正带”的拓扑。
- 做成 AI 工具广告或机器人图。
- 写太多“自动化/岗位/重组”概念词。
- 小黑站在旁边，无法看出它在抵抗或兜底。

### 06 ai-resume-filter

文件：

```text
assets/examples/06-ai-resume-filter.png
```

适合：

- 筛选、过滤、关键词、用户分层、机会被筛掉。

抽象骨架：

- 一个真实过滤工具制造入口和出口。
- 小黑和被筛物发生接触：推入、接住、救回或卡住。
- 画面表达“还没真正被看见，就先被规则过滤”。

可复用关系：

- 过滤器是主物件，其他元素都服务“被筛”。
- 小黑动作要接触卡片或筛网。
- 颜色只做几个小节奏点。

失败信号：

- 直接复制“中央筛子 + 下方纸堆 + 左侧小黑递纸”的拓扑。
- 筛子之外又加多个主物件。
- 画成流程图或招聘系统 UI。
- 标签解释过度，失去情绪按钮感。

## 必填母版锁定字段

每张标准模式图在生成前必须写出：

```text
母版：
抽取的不变量：
当前内容的变异点：
画面 3 秒读懂句：
当前内容适配：
要避免的失败信号：
```

如果这四项说不清，不生成，先重写 shot list。

变异点至少写 3 个，例如：

- 换掉主物件类别。
- 改变左右/中心空间关系。
- 改变小黑动作，不再做同一种推/拉/挡。
- 删除母版里的标志性配件。
- 改变标签出现方式。
- 改变视角或物件姿态。

## 出图拦截规则

生成后的第一张图永远只是候选图，不是交付图。必须和所选母版做视觉比对：

- 主体比例是否接近母版，而不是商品特写或缩略摆件。
- 留白是否接近母版，而不是元素铺满。
- 真实物件是否像同一张摄影棚照片。
- 小黑是否承担母版级的物理动作。
- 标签是否像情绪按钮，而不是概念说明。
- 是否出现“主题元素清单化”：把电脑、手机、纸条、线、标题都堆上去。
- 是否出现“母版复刻化”：主体、方向、配件、小黑动作和标签位置都像旧案例换字。
- 是否 3 秒读懂：不看解释，读者是否能说出画面冲突。

只要出现“元素清单化”“母版复刻化”“不像母版正文配图”或“看不懂冲突”，就必须重写提示词并重生成，不得交付为最终图。
