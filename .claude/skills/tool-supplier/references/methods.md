# 方法层 — 供应商诊断管线

> 可执行的思考流程，每次都用，形成习惯

---

## 一、六步诊断管线

这是所有诊断的通用框架，适用于准入、业绩、风险等场景。

```
Phase 1: 背景采集
  ↓
Phase 2: 核心验证（6个逼迫性问题）
  ↓
Phase 3: 前提挑战
  ↓
Phase 4: 方案生成
  ↓
Phase 5: 输出文档
  ↓
Phase 6: 信号综合
```

---

## 二、供应商准入诊断管线

### Phase 1: 背景采集

采集内容：
- 基本信息：公司、规模、产线匹配
- 资质证书：合规性、有效期
- 团队情况：核心人员、稳定性
- 产能状况：人力、站点、设备
- 过往合作：历史项目、表现

输出：供应商画像初稿

### Phase 2: 核心验证

**6个逼迫性问题**（一次只问一个）：

| # | 问题 | 目的 | 好答案特征 | 红旗信号 |
|---|------|------|------------|----------|
| Q1 | 为什么必须是这家？ | 验证需求真实性 | 具体业务线、具体需求、没有替代 | "多家比比" |
| Q2 | 没有这家会怎样？ | 验证紧迫性 | 业务会卡住、损失、延迟 | "有也行没有也行" |
| Q3 | 谁最需要？说名字 | 验证需求来源 | 负责人姓名、痛点描述 | "业务线都说需要" |
| Q4 | 最小试单是什么？ | 验证可操作性 | 具体业务、量、时间 | "先签合同再说" |
| Q5 | 有成功案例吗？ | 验证能力 | 具体客户、项目、可核实 | "他们说做过" |
| Q6 | 3年后还需要吗？ | 验证长期价值 | 战略匹配、成长空间 | "先解决眼前" |

**智能路由**：
- 新业务/新产品 → Q1, Q2, Q3, Q4
- 替换现有供应商 → Q2, Q4, Q5
- 长期战略合作 → Q1, Q5, Q6

### Phase 3: 前提挑战

挑战核心假设：
- 这是必须的吗？→ 能内部消化吗？
- 能合并吗？→ 现有供应商能扩展吗？
- 真的没有替代吗？→ 行业还有谁？

### Phase 4: 方案生成

**至少生成2-3个方案**：

| 方案 | 描述 | 风险 | 收益 |
|------|------|------|------|
| A全面合作 | 直接签大合同 | 高 | 高 |
| B试点合作 | 先小范围试单 | 低 | 中 |
| C暂缓准入 | 等待更多信息 | 极低 | 极低 |

### Phase 5: 输出文档

准入评估报告包含：
- 基本情况
- 6个问题答案
- 信号分析
- 推荐等级
- 风险点
- 下一步行动

### Phase 6: 信号综合

**准入信号清单**：

| 信号类别 | 正向信号(+1) | 负向信号(-1) |
|----------|--------------|--------------|
| 需求真实性 | 业务线主动提、有场景 | 需求模糊、无人认领 |
| 能力验证 | 有案例、团队清晰 | 只有PPT、无法核实 |
| 意愿程度 | 主动配合、愿意试单 | 推诿、要求先签合同 |
| 风险可控 | 财务健康、团队稳定 | 有财务/人员问题 |
| 长期价值 | 战略匹配、差异化 | 同质化、可替代 |

**评级规则**：
- ≥4：强推
- 2-3：推荐
- 0-1：观望
- <0：不推

## 三、供应商业绩诊断管线

### Phase 1: 数据采集

采集内容：
- KPI数据：核心指标、趋势、对比
- 异常点：什么时间、什么问题
- 排名变化：与同行对比

### Phase 2: 核心诊断

**6个逼迫性问题**：

| # | 问题 | 目的 | 好答案特征 | 红旗信号 |
|---|------|------|------------|----------|
| Q1 | 核心指标到底怎样？ | 验证现状 | 具体数字、趋势、对比 | "还行"、"差不多" |
| Q2 | 根本原因是什么？ | 验证理解深度 | 可验证的原因、数据支撑 | "可能是..." |
| Q3 | 谁受影响最大？ | 验证影响范围 | 具体业务线、人、损失 | "影响不大" |
| Q4 | 能改进吗？要什么？ | 验证改进可能 | 具体方案、资源、时间 | "需要一起努力" |
| Q5 | 去现场看过吗？ | 验证真实性 | 具体观察、一线反馈 | "问了负责人" |
| Q6 | 3个月后会怎样？ | 验证趋势预判 | 具体预测、依据 | "应该会好" |

**智能路由**：
- 业绩下滑 → Q1, Q2, Q4, Q6
- 突发问题 → Q1, Q2, Q3, Q5
- 长期观察 → Q1, Q2, Q6

### Phase 3: 前提挑战

挑战核心假设：
- 这是偶发还是趋势？
- 是供应商问题还是我们问题？
- 是能力问题还是意愿问题？

### Phase 4: 方案生成

| 方案 | 描述 | 适用场景 |
|------|------|----------|
| A支持改进 | 给资源、给时间 | 有意愿、有能力 |
| B观察警告 | 设红线、限期改进 | 意愿不明、能力不足 |
| C降级清退 | 启动替代 | 无意愿、无能力 |

### Phase 5: 输出文档

业绩诊断报告包含：
- 数据概览
- 问题诊断
- 原因分析
- 改进建议
- 跟进计划

### Phase 6: 信号综合

**风险信号清单**：

| 信号类别 | 高风险信号 | 中风险信号 |
|----------|------------|------------|
| 业绩信号 | 连续2月不达标、下滑>20% | 单月不达标、波动大 |
| 团队信号 | 核心人员离职、缩减>20% | 人员流动增加 |
| 财务信号 | 欠薪、诉讼 | 付款周期拉长 |
| 合作信号 | 响应慢、推诿、不兑现 | 沟通变少 |
| 外部信号 | 行业负面、监管处罚 | 行业下滑 |

**风险等级**：
- 高风险信号≥2：立即行动
- 高风险1个或中风险≥2：本周关注
- 仅中风险1个：本月观察
- 无风险信号：正常跟进

## 四、清退决策管线

### Phase 1: 历史回顾

- 合作历史
- 问题记录
- 改善尝试
- 承诺与兑现

### Phase 2: 清退评估

**6个问题**：

| # | 问题 |
|---|------|
| Q1 | 核心问题是什么？尝试过什么？ |
| Q2 | 还有改善可能吗？ |
| Q3 | 替代方案是什么？ |
| Q4 | 清退的影响是什么？ |
| Q5 | 过渡期需要多久？ |
| Q6 | 清退后有什么风险？ |

### Phase 3: 方案生成

| 方案 | 描述 |
|------|------|
| A立即清退 | 已有替代，影响可控 |
| B限期清退 | 给最后机会，同时准备替代 |
| C暂缓清退 | 无替代，先维持 |

---

*这些管线在交互中隐式运行，不需要年老师意识到自己在用"方法"*
