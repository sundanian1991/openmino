# 用户上下文 (User Context)

## 核心原则

1. **骨架优先**：只存储结构性、低频变更的信息。不存储实时数据、具体个案、动态状态。
2. **用户输入优先**：当用户描述与知识库冲突时，**以用户当前输入为准**，知识库标记为"可能过时"。
3. **轻量维护**：每次对话如有新增/变更信息，在对话末尾提示更新知识库。
4. **定期校验**：每季度主动请用户Review一次，删除过时信息。

## 知识库结构

```
user-context/
├── 基础画像（你是谁）
├── 工作结构（你在什么组织中）
├── 业务知识（你处理什么）
├── 工具与环境（你用什么）
├── 思维偏好（你怎么思考）
└── 约束与边界（什么不能变）
```

## 信息层级

| 层级 | 类型 | 变更频率 | 重要性 |
|------|------|---------|--------|
| L0 基础画像 | 身份、角色、汇报线 | 极少（半年~一年） | 高 |
| L1 工作结构 | 部门、职责、权限边界 | 较少（季度） | 高 |
| L2 业务知识 | 术语、流程、指标定义 | 中等（月度） | 中 |
| L3 思维偏好 | 认知框架、审美偏好、沟通风格 | 极少 | 中 |
| L4 动态信息 | 当前项目、实时数据、具体个案 | 每天 | **不存储** |

## 使用规则

- 加载此 Skill 后，**先快速浏览**用户画像，建立对话基调
- 遇到业务术语时，**核对**知识库中的定义是否仍然有效
- 每次对话末尾，**检查**是否有需要更新知识库的新信息
- **绝不**用知识库中的具体数字替代用户的实时描述

## 一致性检查协议

当用户输入与知识库冲突时，按以下流程处理：

```
知识库记录：用户管理供应商（无清退权）
用户当前输入："我最近获得了供应商评估的决策权"
→ 以用户输入为准
→ 更新知识库权限边界
→ 追加变更日志："YYYY-MM-DD：权限变更，新增供应商评估决策权"
→ 追问："这个决策权的范围是什么？能决定哪些事项？"
```
