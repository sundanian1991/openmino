# 快速更新指南

> 已有 user_profile.md 后，后续对话中如何快速补充/修正信息。

---

## 单点更新指令（用户可直接说）

在任意对话中，用户可以用以下句式快速更新知识库：

| 用户说 | 系统操作 |
|--------|---------|
| "更新一下：我现在的汇报线变了，现在是向XX汇报" | 更新 L0 汇报线，追加变更日志 |
| "补充一个术语：XX是指..." | 追加到 L2 术语表 |
| "纠正一下：我其实没有XX权限" | 修正 L1 权限边界，标注原信息有误 |
| "删除：XX这个信息已经过时了" | 删除对应条目，记录删除原因 |

## 轻量补充模板

如果用户想一次性补充多个信息，但不值得跑完整协议：

```markdown
## 补充记录

**日期**：YYYY-MM-DD
**类型**：补充 / 修正 / 删除
**对应层级**：L0/L1/L2/L3/L4

**内容**：
...

**原因**：...
```

## 常见更新场景

### 场景1：岗位变动
- 更新 L0 角色、汇报线
- 更新 L1 权限边界、上下游
- 检查 L2 业务知识是否仍然适用

### 场景2：新增业务线
- 在 L2 新增工作分类
- 补充新术语和指标
- 更新 L1 上下游关系

### 场景3：发现知识库错误
- 不要直接覆盖，先标记"待核实"
- 与用户确认后修正
- 在变更日志记录修正原因

### 场景4：季度Review
- 通读一遍 user_profile.md
- 删除已失效的信息
- 补充近3个月的新认知
- 更新版本号
