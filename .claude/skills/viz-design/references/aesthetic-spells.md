# 12 AI 审美咒语 → viz-design 参数映射

> 12 条"AI 设计咒语"不是新能力，是**现有 viz-design 参数的轻量表达入口**。
> 用户说"新极简"时，系统知道对应的风格/配色/间距/标注规则。
> 也适用于 AI 自主决策时快速锁定"高级但不冰冷"的视觉风格。

---

## 咒语映射表

| # | 咒语 | viz-design 落点 | 具体参数 |
|---|------|----------------|---------|
| 01 | **新极简主义** neo-minimalism | 风格学派 → 01 麦肯锡极简 + 04 Tufte 数据墨水 | 去装饰、标题即结论、每张图一个观点、无网格线或极浅虚线、无图例（直接标注） |
| 02 | **亲近的高级感** approachable sophistication | 风格学派 → 贝恩/FT 变体 + 温暖色系 | 不是冷冰冰的纯理性，用暖灰/暖白底色（Stone 50 `#faf8f5`）、圆角 8-12px、不用纯黑（用 `#333333`） |
| 03 | **低饱和柔和色** muted and pastel | 配色 → color-themes 中低饱和度主题 | 禁用高饱和荧光色，灰粉/雾蓝/奶白/浅咖等安静色调，色阶 300 作为主数据色而非 500 |
| 04 | **单色系** monochromatic | 配色 → 单色 ramp + Stone 辅助 | 1 主色 ramp（5 阶）+ Stone 灰阶，不引入第二 ramp，语义色按需启用 |
| 05 | **充足留白** generous whitespace | 间距 → composition-templates 版式间距 | 四周 ≥40px（不是最小 20px），图表间间距 ≥24px，标题-内容 ≥16px |
| 06 | **杂志式版面** editorial layout | 版式 → #08 L 型 / #14 主次左右 | 标题/数据/解读的成熟排版关系，右侧解读区宽度 30-35%，字号层级清晰 |
| 07 | **卡片式设计** card based design | 组件 → KPI 卡片 / 解读卡片 | 卡片圆角 10px、卡片间距 12px、内边距 16px、背景 Stone 50 区分于主图 |
| 08 | **分层元素** layered elements | 视觉层级 → hero/medium/light 三级 | 前景（hero 高亮）/ 中景（参考线+上下文）/ 背景（灰色淡化），空间层次明确 |
| 09 | **视觉层级** visual hierarchy | 权重 → spec.md 视觉权重铁律 | 主标题 > 副标题 > 数据的清晰权重，hero 视觉面积 ≥ background × 3 |
| 10 | **干净字体** clean typography | 字体 → typography-moods 配对 | 字号/字重/行距/字距的舒适配比，标题 16/600、正文 11/400、标注 11/600、脚注 10/400 |
| 11 | **轻微阴影** subtle shadow | 卡片 → subtle shadow（非数据元素） | 卡片/悬浮元素的柔和阴影，避免厚重投影。仅卡片使用，数据元素不用阴影 |
| 12 | **精致细节** refined details | 全局 → 间距统一/圆角一致/线条粗细 | 边角/线条/间距/图标的考究处理，全局统一值：圆角 10px、线条 1px、标注偏移 ≥12px |

## 负面排除清单（反模式）

> 咒语系统的核心在于"减法"。以下元素除非用户明确要求，否则**默认不出现**。

| 排除项 | viz-design 对应规则 |
|--------|-------------------|
| 杂乱背景 | spec.md → 非数据墨水检验：无装饰性背景 |
| 高饱和颜色 | color-themes → 禁用荧光色，走 muted ramp |
| 廉价渐变 | 全局 → 不使用渐变，纯平色 |
| 过度装饰 | anti-patterns → 无渐变/阴影/emoji |
| 拥挤排版 | composition-templates → 四周 ≥40px，间距统一 |

## 与自动选择器的关系

> 咒语不是独立的选择维度，是**风格学派的选择入口**。

```
用户输入 "新极简 + 低饱和"
  ↓
映射为：风格学派 → 01 麦肯锡极简（Tier 1）
        配色 → 低饱和度 ramp（L3 hash 范围内限定）
        间距 → 充足留白变体（composition 间距 × 1.5）
  ↓
进入 L1/L2/L3 正常流程
```

当用户提到 12 咒语中的任意组合时：
1. 先查本映射表 → 得到 viz-design 参数约束
2. 在 L2 风格学派候选池中优先选择匹配的
3. L3 配色/字体在约束范围内 hash
