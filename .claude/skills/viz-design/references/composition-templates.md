# 构图模板

> 13 种单图构图模板，定义画布分区、坐标占比和间距规则。所有坐标基于 px，画布默认内边距 40px。

---

## 模板列表

### 01. 单图居中

| 字段 | 值 |
|------|------|
| 画布比例 | 800×500（16:10） |
| 标题区 | y: 0-70px，左对齐，padding-left: 48px，padding-top: 32px |
| 内容区 | y: 70-440px，x: 40-760px，单张图片/图表水平垂直居中，最大尺寸 680×340px |
| 标注区 | 无独立标注区，标注叠加在内容区上方（position: absolute，bottom: 20px） |
| 脚注区 | y: 440-500px，padding-left: 48px，字号 12px |
| 视觉重心 | 画面正中央偏上（y 约 40% 处），标题作为入口锚点 |
| 间距规则 | 标题-内容: 24px；内容-脚注: 16px；内容区内边距: 40px；元素间距: 0（单图无间距） |
| 适用模式 | 单张数据图表、产品主视觉、强调型信息图、KPI 大数展示 |

单一主体占满画面，最大优势是聚焦。适合需要一眼抓住核心信息的场景。图片或图表不宜填满整个内容区，保留 5-10% 呼吸空间避免压迫感。标注如需出现，用半透明底色叠加在内容区底部，不单独占空间。

---

### 02. 左右对比（50/50）

| 字段 | 值 |
|------|------|
| 画布比例 | 960×540（16:9） |
| 标题区 | y: 0-72px，居中，padding-top: 28px |
| 内容区 | y: 72-480px，x 分两栏：左栏 40-460px（宽 420px），右栏 500-920px（宽 420px） |
| 标注区 | 两栏各自内部底部，y: 420-480px，背景 rgba(0,0,0,0.05) |
| 脚注区 | y: 480-540px，居中 |
| 视觉重心 | 先看标题，再左右扫视，最终停在差异最大的对比点 |
| 间距规则 | 左右栏间距: 40px；标题-内容: 20px；栏内边距: 24px；标注高度: 60px |
| 适用模式 | A/B 对比、前后对比、竞品对比、优劣对比、同期环比 |

两栏严格等宽，制造对称感。中缝 40px 间隔既分隔又关联。如果两栏内容量不等，不要强行 50/50，改用 60/40 变体。对比的关键是"同维度对齐"——左栏标题和右栏标题在同一 y 坐标，数据标签对齐，便于视觉扫描。

### 03. 左右对比（60/40）

| 字段 | 值 |
|------|------|
| 画布比例 | 960×540（16:9） |
| 标题区 | y: 0-72px，居中 |
| 内容区 | y: 72-480px，左栏（主）x: 40-580px（宽 540px），右栏（辅）x: 620-920px（宽 300px） |
| 标注区 | 左栏底部 y: 400-480px；右栏整栏作为注释/标注区 |
| 脚注区 | y: 480-540px |
| 视觉重心 | 先落左栏（占 60% 面积），右栏作为补充说明 |
| 间距规则 | 栏间距: 40px；左栏内边距: 24px；右栏内边距: 16px |
| 适用模式 | 图表+说明、主图+数据摘要、流程+注释、大图+小表 |

左大右小，主次分明。左栏放需要仔细看的图表或流程，右栏放辅助性的数据卡片、图例说明或要点提炼。右栏字号可以比左栏小 2px，视觉层级自动降一级。

### 04. 上下分栏

| 字段 | 值 |
|------|------|
| 画布比例 | 800×600（4:3） |
| 标题区 | y: 0-72px，左对齐，padding-left: 48px |
| 内容区-上 | y: 72-340px，x: 40-760px（高 268px） |
| 分隔线 | y: 340-344px，宽 720px 居中，1px 实线，颜色 #E0E0E0 |
| 内容区-下 | y: 344-556px，x: 40-760px（高 212px） |
| 脚注区 | y: 556-600px，padding-left: 48px |
| 视觉重心 | 上半区偏重（占 56%），先看上再看下 |
| 间距规则 | 上下区间距: 24px（含分隔线）；内容区内边距: 20px；脚注区上间距: 12px |
| 适用模式 | 概览+详情、趋势+明细、标题+正文、图表+数据表 |

上重下轻的分配（约 56:44）让视觉自然从上往下流。如果下半区更重要（如结论区），可以反转比例为 44:56。分隔线用 1px 细线，不要用粗色块——这会打断阅读流。

### 05. 小多图矩阵（2×2）

| 字段 | 值 |
|------|------|
| 画布比例 | 960×720（4:3） |
| 标题区 | y: 0-72px，居中 |
| 内容区 | y: 72-660px，4 个格子：行间距 24px，列间距 24px。每格 x/y 起止：左上 (40,72)-(448,348)、右上 (512,72)-(920,348)、左下 (40,396)-(448,660)、右下 (512,396)-(920,660) |
| 标注区 | 每格内部底部 36px，半透明底色 |
| 脚注区 | y: 660-720px，居中 |
| 视觉重心 | 左上格优先（阅读起点），对角线扫视路径：左上→右上→左下→右下 |
| 间距规则 | 格间距: 24px；格内边距: 16px；标题-矩阵: 20px |
| 适用模式 | 四象限分析、多维对比、同类图表并排、季度/区域分组 |

2×2 矩阵天然制造象限感。每格保持独立标题（12-14px），与主标题形成层级。如果四格内容量差异大，不要用等分矩阵——改用"主次嵌套"。格子之间用背景色差异（深浅交替）加强分组感，不要用边框。

---

### 06. 小多图矩阵（3×3）

| 字段 | 值 |
|------|------|
| 画布比例 | 960×800（6:5） |
| 标题区 | y: 0-64px，居中 |
| 内容区 | y: 64-740px，9 个格子：行间距 16px，列间距 16px。每格尺寸约 280×200px，起始坐标：第1列 x:40，第2列 x:340，第3列 x:640；第1行 y:64，第2行 y:280，第3行 y:496 |
| 标注区 | 每格内部底部 28px |
| 脚注区 | y: 740-800px |
| 视觉重心 | 中心格（第2行第2列）或左上格，取决于内容逻辑 |
| 间距规则 | 格间距: 16px；格内边距: 12px；每格标题高度: 28px |
| 适用模式 | 九宫格分析、多维度矩阵、时间×维度交叉、雷达图阵列 |

3×3 格子多但单格空间有限（280×200px），每个格子必须高度精简：一个迷你图表+一个数据标签。省略网格线和次要标注，只留核心数据。如果某格需要更多空间，可以跨格合并（如中心格占 2×2）。

### 07. 主次嵌套

| 字段 | 值 |
|------|------|
| 画布比例 | 960×540（16:9） |
| 标题区 | y: 0-64px，左对齐，padding-left: 48px |
| 内容区-主 | y: 64-460px，x: 40-640px（宽 600px，高 396px），占画面约 70% |
| 内容区-次 | y: 64-280px，x: 680-920px（宽 240px，高 216px）；y: 300-460px，x: 680-920px（宽 240px，高 160px） |
| 标注区 | 主图区底部叠加，y: 420-460px |
| 脚注区 | y: 480-540px |
| 视觉重心 | 先看主图（左大区），再看右侧小图（补充信息） |
| 间距规则 | 主-次间距: 40px；次图之间间距: 20px；主图内边距: 24px |
| 适用模式 | 总览+局部放大、主图表+支撑数据、地图+统计卡片、核心指标+趋势图 |

左大右小的嵌套布局，主图占 70% 面积传达核心信息，右侧叠放 1-3 个小图补充维度。右侧小图高度不必等分——第一张可以比第二张大，制造节奏感。主图区和次图区用微妙背景色差区分（如主区白底、次区 #F8F8F8）。

---

### 08. L型（图表+说明）

| 字段 | 值 |
|------|------|
| 画布比例 | 960×540（16:9） |
| 标题区 | y: 0-64px，左对齐 |
| 内容区-图表 | y: 64-420px，x: 40-620px（宽 580px，高 356px） |
| 内容区-说明 | y: 64-420px，x: 660-920px（宽 260px），文字列表或要点 |
| 内容区-底部 | y: 420-500px，x: 40-920px（全宽注释栏或数据表） |
| 脚注区 | y: 500-540px |
| 视觉重心 | 图表区中心偏左，右侧说明作为引导视线 |
| 间距规则 | 图表-说明: 40px；底部栏-上方内容: 16px；说明区内条目间距: 12px |
| 适用模式 | 图表+解读、流程图+步骤说明、数据+洞察、分析+结论 |

L 型布局的核心是"图说分离"：左侧图表让读者自己看数据，右侧文字提炼要点。底部全宽条带可以放注释、数据来源或补充表格。右侧说明文字用列表式，每条不超过 2 行，配合小圆点或数字标记。

---

### 09. 卡片阵列

| 字段 | 值 |
|------|------|
| 画布比例 | 960×540（16:9） |
| 标题区 | y: 0-72px，居中 |
| 内容区 | y: 72-480px，卡片排列：横向 3-4 张，纵向 1-2 行。单卡尺寸 260×180px，间距 20px。第一行 y: 72-252，第二行 y: 272-452 |
| 标注区 | 每张卡片内部底栏，高 40px，含图标+数据 |
| 脚注区 | y: 480-540px |
| 视觉重心 | 第一张卡片（左侧起点），横向扫视 |
| 间距规则 | 卡片间距: 20px；卡片内边距: 16px；卡片圆角: 8px；阴影: 0 2px 8px rgba(0,0,0,0.08) |
| 适用模式 | KPI 仪表盘、指标总览、团队成员展示、产品功能展示、数据摘要 |

每张卡片是一个独立信息单元：顶部图标/数字，中部标题，底部趋势或状态。卡片之间等间距排列，制造秩序感。关键指标用大号数字（32-48px），辅助信息用小号（12-14px）。单卡不要塞超过 3 个信息层级。

---

### 10. 仪表盘式

| 字段 | 值 |
|------|------|
| 画布比例 | 1200×680（约 16:9） |
| 标题区 | y: 0-56px，左对齐，右侧放时间筛选器或刷新按钮 |
| 内容区-核心指标 | y: 56-200px，x: 40-1160px，横向 4 个指标卡片，每卡宽 270px，间距 20px |
| 内容区-主图表 | y: 216-500px，x: 40-720px（宽 680px） |
| 内容区-侧图表 | y: 216-500px，x: 760-1160px（宽 400px） |
| 内容区-底图表 | y: 516-640px，x: 40-1160px（全宽辅助图表） |
| 脚注区 | y: 640-680px |
| 视觉重心 | 核心指标栏 → 主图表区域，F 型扫视路径 |
| 间距规则 | 区块间距: 16px；卡片间距: 20px；图表内边距: 20px |
| 适用模式 | 运营看板、数据大屏、多维度监控、日报/周报仪表盘 |

仪表盘的信息密度最高，分区严格。顶部指标栏用数字+趋势箭头，一秒可读。主图表区放最重要的趋势或分布图，侧图表放辅助维度。底部全宽条带放次要但需要完整展示的数据。所有图表共享同一套色彩编码，避免视觉混乱。

### 11. 时间线横向

| 字段 | 值 |
|------|------|
| 画布比例 | 1200×480（5:2） |
| 标题区 | y: 0-64px，左对齐 |
| 内容区-时间轴 | y: 64-240px，x: 60-1140px。轴线 y: 160px，节点圆点 r: 8px，节点间距均分或按时间比例 |
| 内容区-说明 | y: 240-420px，x: 60-1140px。每个节点下方对应一段说明文字，宽 200px，对齐节点 x 坐标 |
| 标注区 | 节点上方 y: 100-140px，显示日期/阶段名 |
| 脚注区 | y: 420-480px |
| 视觉重心 | 轴线中点或最右节点（当前/最近状态），从左向右扫视 |
| 间距规则 | 节点间距: 最小 120px；上下区间距: 24px；标注-轴线: 20px；说明文字行高: 20px |
| 适用模式 | 项目进度、历史沿革、产品迭代、流程阶段、里程碑展示 |

横向时间线天然匹配从左到右的阅读习惯。节点数量控制在 4-7 个——太少显得空，太多挤不下说明文字。当前阶段用强调色（品牌主色），过去阶段用灰色，未来阶段用虚线。说明文字不超过 3 行，超出用省略号。

### 12. 金字塔分层

| 字段 | 值 |
|------|------|
| 画布比例 | 800×600（4:3） |
| 标题区 | y: 0-72px，居中 |
| 内容区 | y: 72-540px，居中三角形。第1层（顶）y: 72-180px，宽 240px，x: 280-520；第2层 y: 180-300px，宽 400px，x: 200-600；第3层 y: 300-420px，宽 560px，x: 120-680；第4层（底）y: 420-540px，宽 720px，x: 40-760 |
| 标注区 | 每层右侧 x: 760-800px 外，或每层内部居中文字 |
| 脚注区 | y: 540-600px |
| 视觉重心 | 顶部（最窄层），从上往下层级递增 |
| 间距规则 | 层间距: 0（紧贴堆叠）；层内边距: 16px；文字居中 |
| 适用模式 | 需求层次、组织架构、优先级分层、漏斗分析、能力模型 |

金字塔的层级从上到下递增宽度，视觉上自然形成"重要→次要"的优先级感知。层数固定 3-5 层，每层用递进色彩（从深到浅或从品牌色到灰色）。每层文字居中，字号统一（14-16px）。如果需要标注右侧数据，在金字塔右侧对齐每层 y 坐标放置。

### 13. 环形/辐射

| 字段 | 值 |
|------|------|
| 画布比例 | 800×800（1:1） |
| 标题区 | y: 0-64px，居中 |
| 内容区 | y: 64-740px，中心圆 r: 80px，圆心 (400, 400)。外圈节点 5-8 个，均匀分布在 r: 220px 圆上。每个节点卡片宽 140px，高 80px |
| 标注区 | 连线上标注关系/权重，字号 11px |
| 脚注区 | y: 740-800px |
| 视觉重心 | 中心圆（核心概念），辐射至外圈节点 |
| 间距规则 | 中心-节点连线宽: 1.5px；节点间最小角距: 30°；节点卡片内边距: 12px；圆心到画布中心对齐 |
| 适用模式 | 关系网络、能力雷达、影响因素、核心+分支、知识图谱概览 |

辐射布局的核心在中心——放核心概念或主指标。外圈节点围绕分布，与中心用连线关联。节点数量严格控制在 5-8 个，超过 8 个连线会交叉，视觉混乱。1:1 正方形画布保证圆形不被压缩。节点之间的连线如果需要标注权重，用线宽差异（粗=强关联）而非颜色，因为颜色已用于区分类别。

---

### 14. 瀑布流

| 字段 | 值 |
|------|------|
| 画布比例 | 800×900（竖版叙事） |
| 标题区 | y: 0-70px，居中，padding-top: 32px |
| 内容区 | y: 70-840px，逐段垂直排列，每段高度自适应（最小 120px，最大 300px），段间间距 40px |
| 标注区 | 每段右侧内嵌，x: 内容区右边界 -200px，宽度 180px |
| 脚注区 | y: 840-900px，居中 |
| 视觉重心 | 顶部标题→逐段下流，每段有独立小标题 |
| 间距规则 | 段间距: 40px；段内边距: 24px；段宽: 720px（居中）；段内图表最大宽: 480px |
| 适用模式 | 长叙事滚动、渐进式披露、多步骤流程、数据故事 |

竖版叙事布局，每段独立呈现一个数据洞察或分析步骤。段落间用有意留白（40px）制造视觉暂停，让读者"消化完一段再看下一段"。每段内部可以嵌入单图或小图，配右侧文字解读。总段落数控制在 3-6 段，超过 6 段应考虑拆分为多页。底部脚注放数据来源和备注。

---

### 15. 地图+嵌套

| 字段 | 值 |
|------|------|
| 画布比例 | 960×600（16:10） |
| 标题区 | y: 0-60px，左对齐，padding-left: 48px |
| 内容区 | y: 60-560px，左侧主地图 x: 40-660px（宽 620px），右侧 inset 面板 x: 680-920px（宽 240px） |
| 标注区 | 地图上叠加标注（气泡/图标），inset 面板内放图例+排名列表 |
| 脚注区 | y: 560-600px，居中 |
| 视觉重心 | 地图中心区域，右侧 inset 辅助 |
| 间距规则 | 地图-inset 间距: 20px；inset 内卡片间距: 12px；地图边距: 20px |
| 适用模式 | 地理分布+数据明细、区域对比+排名、热力图+统计数据 |

左侧占 70% 放主地图（ECharts geo 或 SVG 地图），右侧 30% 放 inset 面板。inset 内通常放：图例/色阶说明、Top5 排名列表、汇总统计数字。地图上的标注用气泡（面积∝数值）或图标，避免文字密集。色阶 ≤5 级。适合"看分布+看排名"的双需求场景。
