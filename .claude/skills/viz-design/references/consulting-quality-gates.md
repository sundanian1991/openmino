# 质量标准（12 问）

> SPEC 编写完成后，逐项检查。P0 不通过 → 打回重修。P1 不通过 → 建议优化但不阻塞。
> 不限于咨询场景，所有可视化均适用。

---

## P0 必须通过项（5 项）

### Q1：标题 = 结论？

**检查**：标题是完整判断句，不是描述性短语。

| 不通过 | 通过 |
|--------|------|
| "供应商健康度分析" | "毅航健康度全面领先，3 家低于基准线" |
| "各季度业绩趋势" | "Q4 业绩加速增长，环比+37%" |
| "团队能力分布" | "产品能力强、交付能力弱，缺口在中间层" |

**判定方法**：标题能否独立作为一条结论？遮住图只看标题，读者能得出明确判断吗？

---

### Q2：一图一洞察？

**检查**：一张图只有一个核心结论。

- 不能写两个结论用"并且"连接
- 不能试图在一张图里同时表达"趋势"和"占比"
- 如果输入包含多个洞察 → 拆成多图

**判定方法**：能否用一句话说出"这张图想说啥"？如果说出来需要"而且"，就拆图。

---

### Q3：高亮 ≤ 10%？

**检查**：被强调的元素（颜色、粗细、标注）不超过总元素的 10%。

- 10 个柱子 → 最多高亮 1 个
- 5 条线 → 最多高亮 1 条
- 20 个气泡 → 最多高亮 2 个
- 其余全部灰化

**判定方法**：数高亮元素数量 / 总元素数量 ≤ 0.1

---

### Q5：遮住标题只看图，能得出与标题一致的判断？

**检查**：图表本身自证结论，不依赖标题。

- 高亮位置是否对应结论中的关键对象？
- 颜色对比是否支撑结论的方向（正向/负向）？
- 数据排序是否让结论一目了然？

**判定方法**：假设只看图不看标题，你能得出什么判断？这个判断和标题一致吗？

---

### Q8：无违禁项？

**检查**：以下项目绝对禁止：

- 饼图超过 3 类（改用柱状图或堆叠条）
- 双 Y 轴（左右两个不同的纵轴）
- 3D 效果（3D 柱状图、3D 饼图）
- 渐变填充（linear-gradient）
- 阴影效果（box-shadow、drop-shadow）
- Emoji 图标

**判定方法**：逐项扫描，发现任一违禁项 → 不通过。

---

## P1 建议优化项（7 项）

### Q4：颜色 ≤ 2 ramp？

**检查**：不超过 2 条色带。

- 一条主色带（数据色）+ 一条信号色带（正/负）= 2 ramp
- 不能用 5 种完全不同的颜色

---

### Q6：数据来源可追溯？

**检查**：脚注包含：

- 数据来源（系统名/报告名）
- 样本量（N = XXX）
- 时间范围（YYYY-MM-DD 至 YYYY-MM-DD）

**格式**：`Source: [来源], N=[样本量], [时间范围]`

---

### Q7：标注解释 WHY 而非重复数字？

**检查**：每个标注都写"为什么"，不是"是多少"。

| 不通过 | 通过 |
|--------|------|
| "9 月：37%" | "9 月新方案上线→效率↑37%" |
| "拐点" | "引入自动化流程" |
| "最高" | "旺季+促销叠加" |

---

### Q9：数据 MECE？

**检查**：分类互斥且穷尽。

- 同一层级不能有重叠分类
- 总和 = 100% 或总量（或明确标注"不含 XX"）
- 如有遗漏，必须加"其他"类

---

### Q10：排序有意图？

**检查**：排序不是字母序或随意排列。

- 按最佳到最差（让读者一眼看排名）
- 按最相关到最不相关（让读者先看重点）
- 按时间顺序（仅时间序列）
- 按逻辑顺序（流程/层级）

---

### Q11：眯眼测试通过？

**检查**：眯眼看图（或缩小到 30%），只有以下元素清晰可见：

- 标题
- 高亮部分
- 趋势方向

其余细节模糊不影响理解 → 通过。
如果眯眼后看不出重点 → 高亮不够或灰化不够。

---

### Q12：有 So What？

**检查**：SPEC 包含"So What 洞察框"或结论摘要。

- 1-2 句话，不是数据摘要，是分析结论
- 例："方案确认环节的 54% 转化率低于均值 65%，说明问题在流程设计而非供应商能力。"
- 如果图表本身已有明确结论，可省略 So What 框

---

### Q13：渲染契约 JSON 是否完整可消费？

**检查**：SPEC 后附带渲染契约 JSON（viz-design-spec-v1），且字段完整可消费。

- `version`：`"viz-design-spec-v1"`
- 单图：`renderTarget` / `chartType` / `title` / `subtitle` / `canvas` 齐全
- 多图：`globalStyle`（colorRamps ≤2, palette, typography, spacing）+ `charts[]` 数组
- `data.series`：每个系列含 name / values / highlight
- `visualEncoding.highlight`：高亮系列名 + 色值完整
- `visualEncoding.grayscale` / `maxHighlightRatio`：已设置，≤ 0.1
- `annotations`：标注文字写原因+幅度，非纯数字

**判定方法**：将 JSON 直接交给渲染技能，能否不依赖文本 SPEC 独立完成渲染？如果不能 → 补全缺失字段。

---

## 检查执行流程

```
SPEC 编写完成
  ↓
按 Q1→Q2→Q3→Q4→Q5→Q6→Q7→Q8→Q9→Q10→Q11→Q12 逐项检查
  ↓
P0 不通过？→ 标注失败项，说明原因，打回 SPEC 重修
  ↓
P1 不通过？→ 标注建议优化项，记录到 SPEC 的"待优化"段
  ↓
全部通过 → 输出 SPEC 供年老师确认
```
