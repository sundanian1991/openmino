# 视觉节拍

> 25 种单图视觉节拍，定义阅读路径、对应技法、关键参数值。对标 fe-cinematic 的单图节拍系统。

---

## 一、数字与锚定

### 01 | 锚定数字

| 字段 | 值 |
|------|------|
| 名称 | 锚定数字（Anchor Number） |
| 阅读路径 | 第一眼：超大数字 → 第二眼：数字标签/单位 → 最终：上下文解释文字 |
| 对应技法 | 字号 48-72px · 粗体 · 品牌主色或数据驱动色 · 周围留白 ≥ 数字宽度的 1.5 倍 |
| 场景 | KPI 大字报、关键增长率、总数冲击 |
| 对应原则 | "最大的数据 = 最大的视觉权重" — Tufte |

### 02 | 基准线穿越

| 字段 | 值 |
|------|------|
| 名称 | 基准线穿越（Baseline Crossing） |
| 阅读路径 | 第一眼：基准线（0 线/目标线/去年同期线）→ 第二眼：哪些在上方/下方 → 最终：幅度大小 |
| 对应技法 | 基准线 1.5px 虚线 #999 或实线 #666 · 正负区域双色填充（透明度 15%） |
| 场景 | 业绩 vs 目标、增速 vs 0、实际 vs 预算 |
| 对应原则 | Reference lines reduce cognitive load — Tufte |

### 03 | 排序阶梯

| 字段 | 值 |
|------|------|
| 名称 | 排序阶梯（Sorted Ladder） |
| 阅读路径 | 第一眼：从左到右/从上到下的渐变色条 → 第二眼：最高/最低值 → 最终：排名和差距 |
| 对应技法 | 严格降序或升序 · 颜色渐变（最大=饱和色，最小=浅色）· 标签在左侧/上方 |
| 场景 | 供应商排名、地区业绩排序、品类贡献排行 |
| 对应原则 | "Sorting is the simplest and most powerful data transformation" — Cairo |

## 二、颜色与对比

### 04 | 色彩弹出

| 字段 | 值 |
|------|------|
| 名称 | 色彩弹出（Color Pop） |
| 阅读路径 | 第一眼：唯一的彩色元素 → 第二眼：彩色元素的标签 → 最终：与灰色背景元素的对比关系 |
| 对应技法 | 全局仅 1 个元素使用饱和色（HSL 色相 220° 蓝或 0° 红），其余灰度 30-50% · 强调色饱和度 ≥ 80% |
| 场景 | 突出某个供应商/地区/品类 |
| 关键约束 | 同图不可多个色彩弹出，否则无重点 |

### 05 | 对比揭示

| 字段 | 值 |
|------|------|
| 名称 | 对比揭示（Contrast Reveal） |
| 阅读路径 | 第一眼：整体相似性 → 第二眼：关键差异区域 → 最终：差异量化值 |
| 对应技法 | 双系列重叠图 · 差异区域用阴影或颜色填充（透明度 20-30%）· 差异值标注在区域中心，字号 ≥ 14px |
| 场景 | 今年 vs 去年、A vs B 供应商、实际 vs 预测 |

### 06 | 异常闪烁

| 字段 | 值 |
|------|------|
| 名称 | 异常闪烁（Anomaly Flag） |
| 阅读路径 | 第一眼：异常标记（红色/图标/边框）→ 第二眼：异常值具体数字 → 最终：偏离正常范围的程度 |
| 对应技法 | 正常范围用灰色阴影带 #F0F0F0 透明度 50% · 超出范围用红色 #E74C3C 边框/填充 · 添加 ↑↓ 箭头 · 偏差百分比直接标注 |
| 场景 | KPI 超标、合规异常、SLA 违约标记 |

## 三、标注与引导

### 07 | 标注箭头

| 字段 | 值 |
|------|------|
| 名称 | 标注箭头（Annotation Arrow） |
| 阅读路径 | 第一眼：箭头指向的数据点 → 第二眼：箭头尾部注释文字 → 最终：理解因果关系 |
| 对应技法 | 箭头线宽 1.5px · 箭头头部 8px · 注释文字 11px · 注释框半透明或无框 |
| 场景 | 标注政策变化点、异常值解释、拐点原因 |
| 关键约束 | 注释文字 ≤ 25 字 · 写原因+幅度，不写数字 |

### 08 | 趋势拐点

| 字段 | 值 |
|------|------|
| 名称 | 趋势拐点（Trend Inflection） |
| 阅读路径 | 第一眼：趋势线整体走向 → 第二眼：拐点标记 → 最终：拐点前后的斜率变化 |
| 对应技法 | 拐点处画圆点（直径 8px，白色描边 2px）· 拐点前后分段用不同颜色或线型 · 辅助阴影区域标记阶段 |
| 场景 | 增长率转折、政策干预前后、季节性拐点 |

### 09 | 叠加融合

| 字段 | 值 |
|------|------|
| 名称 | 叠加融合（Overlay Fusion） |
| 阅读路径 | 第一眼：底层图表 → 第二眼：叠加层（趋势线/目标线/预测带）→ 最终：两层之间的关系 |
| 对应技法 | 底层：柱状图/面积图（不透明度 100%）· 叠加层：线型（2px 实线或虚线）· 预测带：半透明区域（20%） |
| 场景 | 柱状图+目标线、散点+回归线、面积图+预测区间 |

## 四、空间与组织

### 10 | 密度变换

| 字段 | 值 |
|------|------|
| 名称 | 密度变换（Density Shift） |
| 阅读路径 | 第一眼：整体密度模式 → 第二眼：高密度聚集区 → 最终：低密度空白区（同样重要） |
| 对应技法 | 散点图/热力图 · hexbin 或 kernel density · 颜色梯度 5 级（浅色→深色） |
| 场景 | 客户分布、供应商能力雷达、地域覆盖度 |
| 对应原则 | "Empty space is data too" — Tufte |

### 11 | 隔离突出

| 字段 | 值 |
|------|------|
| 名称 | 隔离突出（Isolation） |
| 阅读路径 | 第一眼：被隔离的元素 → 第二眼：隔离元素的数据 → 最终：为什么它被隔离 |
| 对应技法 | 物理隔离（独立卡片/放大框/callout 框）· 隔离区背景色 #F8F9FA · 微妙阴影 |
| 场景 | Top 1 / Bottom 1 单独展示、异常值单独分析、冠军/倒数 |

### 12 | 分组归类

| 字段 | 值 |
|------|------|
| 名称 | 分组归类（Grouping） |
| 阅读路径 | 第一眼：组间间距/分隔线 → 第二眼：组内元素相似性 → 最终：分组逻辑标签 |
| 对应技法 | 组间距 ≥ 组内间距的 2 倍（Gestalt proximity）· 每组浅底色背景框 · 组标签在顶部 11px 灰色 |
| 场景 | 供应商按等级分组、指标按维度分组、时间按季度分组 |
| 对应原则 | Gestalt: Proximity + Similarity + Common Region |

### 13 | 近邻关联

| 字段 | 值 |
|------|------|
| 名称 | 近邻关联（Proximity Pairing） |
| 阅读路径 | 第一眼：靠近的两个元素被视为一组 → 第二眼：两者关系 → 最终：这种关系的含义 |
| 对应技法 | 两元素间距 < 元素宽度的 30% · 细线或箭头连接 · 共享同色系但明度区分 |
| 场景 | 原因→结果并置、投入→产出并排、问题→方案配对 |

## 五、流动与编码

### 14 | 面积编码

| 字段 | 值 |
|------|------|
| 名称 | 面积编码（Area Encoding） |
| 阅读路径 | 第一眼：最大的面积块 → 第二眼：面积大小排序 → 最终：占比关系 |
| 对应技法 | 面积 ∝ 数值（严格比例）· 矩形树图或气泡图 · 最大气泡直径 ≤ 画布宽 40% |
| 场景 | 市场份额、预算构成、资源分配 |
| 警告 | 人类对面积感知误差约 15% · 必须配合数字标签 |

### 15 | 流动引导

| 字段 | 值 |
|------|------|
| 名称 | 流动引导（Flow Guidance） |
| 阅读路径 | 第一眼：流动方向（箭头/连线）→ 第二眼：流量大小（线宽/颜色）→ 最终：源→目标路径 |
| 对应技法 | 桑基图/流线图 · 线宽 ∝ 流量 · 颜色区分流向 · 节点标签在节点中心或旁边 |
| 场景 | 资金流向、用户路径、供应商→客户映射 |

### 16 | 时序脉冲

| 字段 | 值 |
|------|------|
| 名称 | 时序脉冲（Temporal Pulse） |
| 阅读路径 | 第一眼：时间轴方向 → 第二眼：脉冲波峰/波谷 → 最终：周期性和异常点 |
| 对应技法 | 面积图或柱状图 · 时间轴等距刻度 · 季节性用背景色带标记 · 异常点红色标记+注释 |
| 场景 | 月度/季度业绩脉冲、呼叫量波动、季节性分析 |

## 六、探索与叙事

### 17 | 缩放聚焦

| 字段 | 值 |
|------|------|
| 名称 | 缩放聚焦（Zoom Spotlight） |
| 阅读路径 | 第一眼：全局缩略图 → 第二眼：高亮框选的局部 → 最终：放大后的细节 |
| 对应技法 | 左侧小图（30% 宽度）带矩形选择框 · 右侧大图（65% 宽度）为选区放大 · 选择框颜色与高亮区一致 |
| 场景 | 长趋势线中的关键区间、大范围数据的局部细节 |

### 18 | 维度旋转

| 字段 | 值 |
|------|------|
| 名称 | 维度旋转（Dimension Rotation） |
| 阅读路径 | 第一眼：当前维度全貌 → 第二眼：切换维度后的变化 → 最终：哪个维度最关键 |
| 对应技法 | 同一数据集的多视图切换（Tab 或翻页）· 每个视图使用相同坐标轴范围 |
| 场景 | 同一指标按地区→按产品→按时间的多维分析 |

### 19 | 层级折叠

| 字段 | 值 |
|------|------|
| 名称 | 层级折叠（Hierarchy Fold） |
| 阅读路径 | 第一眼：顶层汇总 → 第二眼：展开/折叠的交互暗示 → 最终：下钻后的细节 |
| 对应技法 | 树状缩进+展开图标（▶）· 每层缩进 24px · 层级线 1px #E0E0E0 · 展开状态用动画或分页 |
| 场景 | 组织架构+业绩、供应商→团队→个人的逐层分析 |

### 20 | 叙事锚点

| 字段 | 值 |
|------|------|
| 名称 | 叙事锚点（Narrative Anchor） |
| 阅读路径 | 第一眼：人物/引用/故事元素 → 第二眼：支撑数据 → 最终：数据赋予故事的意义 |
| 对应技法 | 引用框（左边框 3px 品牌色，背景 #F8F9FA，padding 16px）· 数据图表在右侧/底部 · 引用文字斜体 13px |
| 场景 | 客户故事+数据、供应商反馈+评分、管理者引言+业绩 |
| 对应原则 | "Data without context is noise; context without data is opinion" — Cairo |

## 七、阅读路径与注意力（新增）

### 21 | F 型阅读锚点

| 字段 | 值 |
|------|------|
| 名称 | F 型阅读锚点（F-Pattern Anchor） |
| 阅读路径 | 第一眼：左上角关键信息 → 第二眼：横向扫描 → 第三眼：下移后再横向扫描 |
| 对应技法 | 左上(x=15%,y=20%)放核心数字 · 第一水平线 y=20% 放标题+主指标 · 第二水平线 y=60% 放详细数据 |
| 场景 | 西方阅读习惯优化、信息密集型仪表板、执行摘要 |
| 来源验证 | Nielsen Norman Group 眼动研究 |

### 22 | Z 型视觉流

| 字段 | 值 |
|------|------|
| 名称 | Z 型视觉流（Z-Pattern Flow） |
| 阅读路径 | 第一眼：左上起点 → 第二眼：右上 → 第三眼：左下 → 第四眼：右下终点 |
| 对应技法 | 起点(x=10%,y=15%)放标题/大数字 · 右上(x=90%,y=15%)放趋势/对比 · 左下(x=10%,y=70%)放详情 · 右下(x=90%,y=85%)放结论 |
| 场景 | 带图片的叙事布局、执行摘要、多步骤流程 |
| 来源验证 | 眼动追踪研究，适用于视觉层次丰富的页面 |

## 八、标注策略（新增）

### 23 | 三色标注策略

| 字段 | 值 |
|------|------|
| 名称 | 三色标注策略（Three-Tier Annotation） |
| 阅读路径 | 描述性标注（解释数据）→ 强调性标注（突出洞察）→ 上下文标注（补充背景） |
| 对应技法 | **描述性**：灰色 #666，11px，无框 · **强调性**：品牌色，12px，左边框 2px · **上下文**：浅灰 #999，10px，括号内 |
| 场景 | 复杂图表的多层标注、需要区分信息优先级 |
| 来源验证 | Datawrapper 最佳实践 |

## 九、渐进揭示（新增）

### 24 | Sparkline 优先

| 字段 | 值 |
|------|------|
| 名称 | Sparkline 优先（Sparkline First） |
| 阅读路径 | 第一眼：简化微型趋势 → 第二眼：识别异常/模式 → 第三眼：展开详细视图 |
| 对应技法 | Sparkline 尺寸：宽 60-120px，高 20-40px · 线条 1.5px · 无轴无标签 · 悬停/点击展开全尺寸图表 |
| 场景 | 高密度仪表板、概览+详情模式、渐进信息披露 |
| 来源验证 | "Sparkline first, details on demand" — 渐进揭示原则 |

## 十、无障碍与合规（新增）

### 25 | 对比度合规

| 字段 | 值 |
|------|------|
| 名称 | 对比度合规（Contrast Compliance） |
| 阅读路径 | 确保所有读者能清晰辨识层次（不仅是视力正常者） |
| 对应技法 | 文字对比度 ≥ 4.5:1（AA 级）· 图形对比度 ≥ 3:1 · 大文字(18px+)对比度 ≥ 3:1 |
| 场景 | 对外发布、政府报告、需要无障碍合规的场景 |
| 来源验证 | WCAG 2.1 无障碍标准 |

---

### 26 | 视觉暂停

| 字段 | 值 |
|------|------|
| 名称 | 视觉暂停（Visual Pause） |
| 阅读路径 | 密集数据区 → 有意空白区 → 再进入密集区 |
| 对应技法 | 在高密度区之间插入 ≥80px 无数据区域 · 背景色与画布一致 · 可配分隔线（0.5px #e5e5e5） |
| 场景 | 多图同屏、长滚动叙事、信息密度过高的报表 |
| 对应原则 | "White space is not empty space — it is the pause between notes" — Tufte |

### 27 | 留白强调

| 字段 | 值 |
|------|------|
| 名称 | 留白强调（Negative Space Emphasis） |
| 阅读路径 | 负空间边界 → 被包围的关键元素 → 意义理解 |
| 对应技法 | 关键元素周围 padding ≥ 元素自身尺寸 50% · 去除所有辅助线 · 无标注干扰 |
| 场景 | 单一关键结论强调、数字冲击、极简风格图 |
| 对应原则 | "What you leave out defines what you leave in" — Cairo |

## 节拍组合禁忌

| 禁止组合 | 原因 |
|----------|------|
| C-04 色彩弹出 + C-04 色彩弹出（同图多个） | 多个"重点" = 无重点 |
| C-01 锚定数字 + C-01 锚定数字（>3 个同区） | 视觉打架，无主次 |
| C-06 异常闪烁 + C-04 色彩弹出 | 冲突：哪个是异常？哪个是重点？ |
| C-11 隔离 + C-17 缩放聚焦（同元素） | 隔离已足够，缩放冗余 |
| C-03 排序阶梯 + C-15 流动引导 | 排序暗示序列，流动也暗示序列，冲突 |
