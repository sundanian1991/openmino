# Anti-Patterns · 反模式清单

> 可插拔模块。不同 AI 渲染技能的共性错误，出图前自检拦截。
> 目标：让不同 AI 产出一致，不是各做各的。

---

## 一、叙事意图层

### A1：标题写描述，不是结论

| 错误 | 正确 |
|------|------|
| "供应商健康度分析" | "毅航健康度全面领先，3 家低于基准线" |
| "Q1 业绩趋势" | "Q4 环比+37%，创 18 个月新高" |
| "各渠道转化对比" | "付费渠道转化率 3× 于自然流量" |

**拦截**：如果标题去掉最后一个词（"分析""趋势""对比"）后还能成立，就是描述性标题。

### A2：一个图塞多个信息

一张图表达一个洞察。同时问"趋势+占比+关系"→ 拆成多张图。

**拦截**：能写出"而且"连接的标题 → 拆图。

### A3：高亮过多

所有柱子半彩色、所有折线都用主色 → 没有高亮。

**拦截**：数高亮元素。≤ 总数的 10%。10 个柱子最多 1 个高亮。

## 二、视觉编码层

### B1：颜色没有语义理由

| 错误 | 正确 |
|------|------|
| 每个柱子不同颜色（好看） | 全部 Stone 300，只高亮异常值 |
| 彩虹色折线 | 全部灰，一条 Warm 500 |

**拦截**：问"这个颜色编码了什么信息？"回答不了 → 去掉。

### B2：用了禁止的图型

| 禁止 | 替代 |
|------|------|
| 饼图（>3 分类） | 水平条形图 |
| 双 Y 轴 | 上下分图共享 X 轴 |
| 3D 图表 | 2D 扁平 |
| 雷达图 >5 维度或 ≥4 对象 | 水平条形图组或小多图矩阵 |
| 意面图（>5 条线） | 小多图或单线高亮 |

### B3：渐变/阴影/发光

这些效果在流式渲染中会闪烁，且违反设计体系。

**拦截**：SVG 中无 `filter`、`gradient`、`drop-shadow`。ECharts 中 `animation: false`。

### B4：数据不从零基线开始

柱状图 Y 轴截断 → 放大差异，误导读者。

**拦截**：柱状图 yAxis min 必须为 0。

## 三、标注层

### C1：标注重复数字

| 错误 | 正确 |
|------|------|
| "9月：37%" | "新方案上线 → 效率↑37%" |
| "最高点" | "旺季+促销叠加" |
| "最低" | "供应链中断两周" |

**拦截**：标注里如果只有数字没有原因 → 补"为什么"。

### C2：标注位置遮挡数据

标注放在数据元素正上方，遮挡柱顶或数据点。

**拦截**：标注与数据元素间距 ≥12px，不重叠。

### C3：标注过多

每张图超过 3 个标注 → 噪音。

**拦截**：标注数 ≤ 3。超过 → 只保留解释高亮点的那个。

---

## 四、布局层

### D1：标题到内容间距不足

标题紧贴图表 → 呼吸感差。

**拦截**：标题到内容 ≥12px。

### D2：文字溢出容器

单行文字超出卡片宽度 → 被裁剪。

**拦截**：中文单行不超过 20 字（14px 字号），超出换行。

### D3：元素不对称导致视觉偏移

右边标注多 → 整张图重心偏右。

**拦截**：检查视觉重心。右侧标注面积 ≈ 左侧留白。

### D4：缺少脚注/来源

图没有数据来源和时间范围 → 不可信。

**拦截**：底部必须有 Stone 400 小字脚注。

---

## 五、数据层

### E1：数据未灰化先高亮

先画彩色数据再"灰化其余" → 容易残留不该有的颜色。

**拦截**：严格按六层绘制法——第 4 层全灰，第 5 层局部着色。

### E2：分类按字母排序，不按意图

数据排序影响阅读路径。字母排序无叙事。

**拦截**：按最优→最差、按时间、按叙事意图排序。

### E3：缺失数据用零填充

缺失值画成零 → 误导。

**拦截**：缺失值用 "?" 或灰色占位，不用 0。

## 六、AI 特有错误（不同模型各有）

### F1：过度装饰型

常见于"想做得好看"的 AI。表现：
- 装饰性图标泛滥
- 卡片渐变背景
- 数据点加阴影
- 标题加 emoji

**修正**：回到 Warm 50 + 白卡 + 单色线条的骨架。

### F2：信息填充型

常见于"想提供更多信息"的 AI。表现：
- 标题后面加一大段解释文字
- 图表下方追加分析段落
- 每个数据点都加标注

**修正**：图表本身说话。文字只放标题、标注、脚注三处。

### F3：保守型

常见于"怕出错"的 AI。表现：
- 全部灰色，不敢高亮
- 标题模糊（"数据概览"）
- 不加任何标注

**修正**：SPEC 的叙事意图已经决定了高亮什么。按 SPEC 执行。

### F4：自由发挥型

常见于"有创意"的 AI。表现：
- 自创图型（不在 72 种模式内）
- 配色无章法（随意混搭、超过合理色数）
- 自定字号（不用规范级数）

**修正**：所有决策从 SPEC + 本文件的参考表 + 渲染技能自有规范中查表，不自创。

## 七、平庸反模式（图没有错，但不够商业级）

### G1：标题不够锋利

| 平庸 | 锋利 |
|------|------|
| "Q1 供应商综合评分"（14px） | "毅航健康度全面领先"（16-18px, 600） |
| "各部门业绩趋势" | "Q4 环比+37%，创 18 个月新高" |

**拦截**：标题字号 < 16px 或字重 < 600 → 不合格。标题不含判断结论 → 重写。

### G2：灰化不彻底

| 平庸 | 合格 |
|------|------|
| 用 `#b0b0b0` 或 Tailwind 默认灰 | 非高亮元素用 Stone 300（`#ada599`）或更浅 |
| 次要数据线 `#999` | 次要数据 Stone 500（`#857d74`），背景 Stone 300 |

**拦截**：眯眼看图，非高亮点应几乎不可见。灰化色值必须是 Stone 色阶中的值。

### G3：轴标签不真实

| 平庸 | 合格 |
|------|------|
| X 轴是 "1","2","3"... 编号 | X 轴是真实日期/名称（"2024-01","毅航"） |
| 数值标签只显示整数 | 精度与数据源一致（如汇率保留 4 位小数） |

**拦截**：轴标签必须是原始数据值，不是索引编号。

### G4：标注无洞察

| 平庸 | 合格 |
|------|------|
| "9月：37%"（只重复数字） | "旺季+促销叠加 → 效率↑37%"（原因+幅度） |
| "最高点" | "04/15 新系统上线"（事件锚定） |

**拦截**：标注里如果只有数字没有原因 → 补"为什么"。标注数 ≤3。

### G5：网格线非虚线

| 平庸 | 合格 |
|------|------|
| splitLine 实线（`type: 'solid'`） | splitLine 虚线（`type: 'dashed'`） |
| 网格线颜色过深（`#e0e0e0`） | 网格线 Stone 100（`#f2f0eb`），浅到刚好看得见 |

**拦截**：ECharts splitLine 必须设 `type: 'dashed'`，color 为 `#f2f0eb`。

### G6：拥挤排版

| 平庸 | 合格 |
|------|------|
| 标题紧贴图表（<12px） | 标题到内容 ≥16px，图表到脚注 ≥12px |
| 卡片挤满不留缝 | 卡片间距 ≥16px，内边距 ≥20px |
| 所有空间填满数据 | 至少 15% 画布面积为留白（generous whitespace） |

**拦截**：版式内容密度 > 85%（内容面积 / 画布总面积）→ 拥挤。至少保留 15% 留白。
**来源**：12 审美咒语 — generous whitespace / editorial layout

### G7：缺乏留白策略

| 平庸 | 合格 |
|------|------|
| 所有间距统一 12px | 按层级设间距：标题区 24px / 内容区 16px / 区块间 20px |
| 卡片背景铺满无边距 | 卡片内边距 ≥20px，四周呼吸空间 |
| 多图之间无分隔 | 多图间距 ≥24px，或用虚线/底色区分区块 |

**拦截**：检查间距是否分层（不是一刀切）。关键信息周围留白 ≥ 内容宽度的 1.2 倍。
**来源**：12 审美咒语 — generous whitespace / visual-beats Beat 26-27

### G8：粗糙细节

| 平庸 | 合格 |
|------|------|
| 圆角随手写 4px / 8px | 圆角统一级数：卡片 10px / 节点 12px / 柱子 6px |
| 线条粗细凭感觉 | 线条统一级数：边框 1px / 描边 1.5px / 强调线 2px |
| 标注随意摆放 | 标注与数据元素间距 ≥12px，不重叠，统一对齐 |

**拦截**：出图前扫描 roundness / linewidth / alignment 三个维度。同一类元素必须用相同值。
**来源**：12 审美咒语 — refined details

### G9：AI 审美负面排除（12 咒语铁律）

> 12 AI 审美咒语的核心在于"减法"。以下五项除非用户**明确要求**，否则默认不出现。

| 排除项 | 表现 | 修正 |
|--------|------|------|
| **杂乱背景** | 装饰性纹理、图片水印、复杂渐变背景 | 纯白/暖白/Stone 50 单色背景 |
| **高饱和颜色** | 荧光色、霓虹色、纯 `#FF0000` 类刺眼色 | 色阶 300 作为主数据色，走 muted ramp |
| **廉价渐变** | 卡片渐变背景、数据渐变填充、彩虹渐变 | 纯平色，不渐变 |
| **过度装饰** | 装饰性图标、emoji 标题、数据点阴影、花边框 | 回到骨架：白卡 + 单色线条 + 充足留白 |
| **拥挤排版** | 标题紧贴图表、卡片挤满不留缝、所有空间填满数据 | 四周 ≥40px，内容密度 ≤85%，15%+ 留白 |

**拦截**：出图后问自己："这张图像知识海报还是像儿童手抄报？" 前者合格，后者打回。
**来源**：12 审美咒语 — neo-minimalism / muted and pastel / generous whitespace / refined details
