# 构图模板（扩展版）

> 27 种新增构图模板（编号 14-40），与原有 13 种共同构成 40 种
> 定义画布区域划分、间距规范和适用场景

## 14 | 主次左右（70/30）

| 字段 | 值 |
|------|------|
| **名称** | 主次左右（70/30 Split） |
| **骨架结构** | `grid-template-columns: 7fr 3fr; grid-template-rows: 60px 1fr 44px` |
| **适用场景** | 左侧主图表需要较大空间，右侧放关联但次要的信息（图例、摘要、关键数字）。比 60/40 更强调主次差 |
| **间距规范** | 标题到内容 ≥20px，四周 ≥40px，栏间距 32px，图表内边距 ≥20px |
| **最大图表数** | 3（左 1 大图 + 右 2 小卡片） |
| **代表案例** | 年度销售趋势大图 + 右侧 3 家头部供应商 KPI 卡片 |
| **禁忌** | 右侧内容重要到需要等宽时，改用 02 左右对比（50/50） |

```
┌─────────────────────────────────────────────┐
│              标  题  区  (60px)              │
├──────────────────────────┬──────────────────┤
│                          │   辅助信息区      │
│                          │   (30%)          │
│     主图表区 (70%)        │                  │
│                          │  ┌────────────┐  │
│                          │  │  数据卡片1  │  │
│                          │  └────────────┘  │
│                          │  ┌────────────┐  │
│                          │  │  数据卡片2  │  │

├──────────────────────────┴──────────────────┤
│              脚 注 区  (44px)                │
└─────────────────────────────────────────────┘
```

## 15 | 极窄侧边栏（80/20）

|------|------|

| **适用场景** | 几乎全屏给主图表，右侧极窄条仅放筛选器、图例或状态指示。适合数据密集型大屏 |
| **间距规范** | 标题到内容 ≥16px，四周 ≥32px，栏间距 24px，侧边栏内边距 12px |

| **禁忌** | 侧栏内容超过 3 个信息单元时，改用 14（70/30） |

| 字段 | 值 |

| **适用场景** | 内容高度不一致的卡片或图表混合排列，制造有机节奏感。适合数据故事中的"发现集锦"页 |

| **禁忌** | 需要严格对齐比较的场景（如同期对比），用等分网格代替 |

## 17 | 三层叠加

| 字段 | 值 |
|------|------|
| **名称** | 三层叠加（Background-Data-Annotation Overlay） |
| **骨架结构** | 三层叠加：`position: absolute` 堆叠，z-index: 背景(1) → 数据(2) → 注释(3) |
| **适用场景** | 地图+数据点+标注、热力图+高亮区域+文字说明。需要在同一视觉区域传达多层信息 |
| **间距规范** | 四周 ≥40px，注释文字与对应数据点间距 ≥8px，注释背景透明度 0.9 |
| **最大图表数** | 1（但含 3 层信息） |
| **代表案例** | 全国供应商分布地图 + 各点位业绩气泡 + 重点区域文字标注 |
| **禁忌** | 三层内容都极其密集时，信息过载，改为拆分到多张图 |

```

## 22 | 水平步骤流

| **骨架结构** | `grid-template-columns: repeat(N, 1fr); grid-template-rows: 56px 1fr 40px`，N=步骤数(3-6)。箭头用 `::after` 伪元素或 SVG |
| **适用场景** | 线性流程、阶段演进、操作步骤。3-6 步最佳，超过 6 步改用垂直排列 |

| **禁忌** | 流程存在分支/并行时，线性步骤无法表达，改用流程图 |

## 25 | 镜像对称

| 字段 | 值 |
|------|------|
| **名称** | 镜像对称（Mirror Symmetry） |
| **骨架结构** | `grid-template-columns: 1fr 2px 1fr; grid-template-rows: 56px 1fr 40px`，中间 2px 分隔线为对称轴 |
| **适用场景** | 严格对称的对比场景：男/女、国内/海外、线上/线下。镜像感强化"两面一体"的概念 |
| **间距规范** | 标题跨两栏居中 ≥56px，对称轴 2px 实线或渐变色，栏内边距 24px，四周 ≥32px |
| **最大图表数** | 2（左右各 1 主图，严格对称） |
| **代表案例** | 供应商绩效：左侧上半年 vs 右侧下半年，同维度同尺度 |
| **禁忌** | 两侧数据量纲不同或尺度不同时，镜像会误导 |

```
┌──────────────────────────────────────────┐
│           标  题  区 (居中跨栏)            │
├──────────────────────┬───────────────────┤
