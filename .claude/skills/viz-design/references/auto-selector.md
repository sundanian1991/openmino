# 自动选择器

> viz-pattern 的选择机制。对标 fe-cinematic 的 Director Preference Pool + hash selection。
> **核心原则：不是所有维度都能 hash。内容锁死的维度走意图匹配，风格自由的维度走 hash。**

---

## 三层选择模型

```
输入（主题/数据/概念）
  ↓
Layer 1 内容锁死层 — 意图匹配，不走 hash
  模式、弧线、构图 → 数据/意图决定
  ↓
Layer 2 场景约束层 — Tier 过滤 + 有限 hash
  风格学派、开场骨架 → 场景限定候选池，hash 二选一
  ↓
Layer 3 风格自由层 — 全 hash
  配色、字体、视觉节拍 → 多个选项都正确，纯个性化
  ↓
输出完整组合
```

### 各维度归属

| 维度 | 层级 | 选择方式 | 原因 |
|------|------|---------|------|
| **模式** | L1 内容锁死 | 意图匹配 | 趋势数据不能用饼图，内容决定 |
| **叙事弧线** | L1 内容锁死 | 意图匹配 | 单图无弧线概念，多图由叙事意图决定 |
| **构图** | L1 内容锁死 | 意图匹配 | 1图=单中心，2图对比=左右，数量决定 |
| **风格学派** | L2 场景约束 | Tier过滤+hash | 高管汇报不能用Excalidraw，但Tier1内可hash |
| **开场骨架** | L2 场景约束 | Tier过滤+hash | 全景概览可在指标阵列/漏斗间选 |
| **配色** | L3 风格自由 | hash | 麦肯锡风格下仍有多种蓝色系变体 |
| **字体** | L3 风格自由 | hash | 选定风格后，同风格内有多种字体配对 |
| **视觉节拍** | L3 风格自由 | hash | 同一张图可以有不同阅读路径引导 |

---

## 核心原理

### Hash 铁律

- **确定性**：同一输入永远得到相同组合
- **差异化**：不同输入得到不同组合
- **不越权**：hash 只在 L2/L3 层使用，不碰 L1

### Hash 计算

简化版（AI 可直接心算）：
1. 取输入主题的 UTF-8 字节序列
2. 计算简单累加和
3. 对 1000000 取模

**不需要精确实现**——AI 只要保证同一输入每次得到"不同但稳定"的数字即可。

### 变体种子

同一输入生成不同变体：
```
hash = H + seed
seed = 0 默认，1/2/3 生成变体 A/B/C
```

## Layer 1：内容锁死层（意图匹配，不走 hash）

### 一、模式选择

走 SKILL.md 的意图识别 + 模式匹配决策树。不走 hash。

```
有精确数据？→ 维度数决定图表类型
抽象概念？→ 关系本质决定类比模式
逻辑关系？→ 分解/变化/对比/因果决定 C01-C16
```

**为什么不走 hash**：用饼图表达趋势是错的，不是个性化问题，是正确性问题。

### 二、叙事弧线选择

弧线只适用于**多图叙事**（≥2 张图）。单图不走弧线。

| 叙事意图 | 弧线 | 为什么 |
|---------|------|--------|
| 揭示问题 | 01 调查式 | 震撼→扫描→深挖→结论，问题诊断天然路径 |
| A vs B | 02 对比式 | A→B→对比→解读，对比内容天然结构 |
| 历史/趋势 | 03 演进式 | 起点→变化→拐点→终态，时间序列天然路径 |
| 根因分析 | 04 诊断式 | 信号→分解→定位→方案，因果分析天然结构 |
| 方案评估 | 05 决策式 | 选项→标准→评分→推荐，决策流程天然结构 |
| 综合汇报 | 06 全景式 | 总览→分解→重点→行动，汇报天然结构 |
| 推动行动 | 07 说服式 | 问题→证据→方案→号召，说服天然结构 |

**选择规则**：叙事意图由输入内容决定，不走 hash。如果输入内容不足以判断弧线，默认 06 全景式。

### 三、构图选择

| 图表数量 | 叙事关系 | 构图 |
|---------|---------|------|
| 1 图 | 任何 | 01 单中心聚焦 |
| 2 图 | 对比 | 02 左右对比 |
| 2 图 | 因果/总量+分解 | 03 上下分层 |
| 3-9 图 | 同类并列 | 04 小 multiples 网格 |
| 3-5 图 | 主次分明 | 05 非对称平衡 |
| 5+ 图 | 时间序列 | 07 水平时间轴 |
| 3-4 图 | 递进 | 09 递进阶梯 |
| 4+ 图 | 仪表盘 | 10 仪表盘 |

**选择规则**：图表数量 + 叙事关系决定构图，不走 hash。

## Layer 2：场景约束层（Tier 过滤 + 有限 hash）

### 四、风格学派选择

**步骤**：
1. 判断场景类型 → 得到 Tier 1（首选）和 Tier 2（可用）
2. 合并 Tier 1 + Tier 2 → 候选池（通常 3-6 个）
3. hash % pool_size → 选中索引
4. 如果 Tier 1 只有 1 个 → 直接选，不走 hash

| 场景类型 | Tier 1 | Tier 2 |
|---------|--------|--------|
| 高管汇报 | 01 麦肯锡, 02 贝恩, 03 BCG | 04 Tufte, 05 FT |
| 数据新闻 | 05 FT, 06 Economist, 07 NYT | 10 OWID, 17 Cairo |
| 学术报告 | 04 Tufte, 10 OWID, 15 Nightingale | 17 Cairo |
| 仪表板 | 08 Bloomberg, 12 IBM Carbon, 19 深色终端 | 13 Stripe, 11 Material |
| SaaS/产品 | 13 Stripe, 11 Material, 12 IBM Carbon | 04 Tufte |
| 金融分析 | 08 Bloomberg, 05 FT, 06 Economist | 01 麦肯锡, 02 贝恩 |
| 脑暴/草图 | 20 Excalidraw, 18 Pudding | 07 NYT, 14 Airbnb |
| 创意叙事 | 18 Pudding, 07 NYT, 16 Gapminder | 17 Cairo |
| 地理数据 | 14 Airbnb, 10 OWID | 07 NYT |
| 周期数据 | 15 Nightingale, 10 OWID | 05 FT, 17 Cairo |
| 供应商管理 | 01 麦肯锡, 02 贝恩, 03 BCG | 04 Tufte |

### 五、开场骨架选择

适用于**多图叙事的第一张图**或**单图的呈现方式**。

**步骤**：
1. 从叙事意图得到 Tier 1 + Tier 2 → 候选池
2. 候选池 = 1 → 直接选
3. 候选池 > 1 → hash % pool_size → 选中

| 叙事意图 | Tier 1 | Tier 2 |
|---------|--------|--------|
| 震惊/冲击 | 01 数据冲击 | 04 对比开场 |
| 全景概览 | 02 指标阵列 | 05 漏斗开场 |
| 时间故事 | 03 全景时间线 | 06 演变轨迹 |
| 对比差异 | 04 对比开场 | 01 数据冲击 |
| 问题诊断 | 05 漏斗开场 | 07 根因分解 |
| 趋势变化 | 06 演变轨迹 | 03 全景时间线 |
| 异常发现 | 08 异常标注 | 01 数据冲击 |
| 目标差距 | 09 目标进度 | 04 对比开场 |
| 构成分析 | 10 构成拆解 | 02 指标阵列 |

## Layer 3：风格自由层（全 hash）

### 六、配色选择

**步骤**：
1. 从已选的风格学派得到色系方向（首选/备选）
2. hash / 100 % 色系数量 → 色系选择
3. 在 color-themes.md 中，hash / 1000 % 25 → 具体配色编号

| 风格学派 | 色系 0 | 色系 1 |
|---------|--------|--------|
| 01 麦肯锡 | 深蓝+灰+蓝强调 | 深灰+单色 |
| 02 贝恩 | 海军蓝+绿正+红负 | 深蓝+灰 |
| 03 BCG | 深蓝+四色矩阵 | 深灰+高饱和 |
| 04 Tufte | 蓝灰+灰+红异常 | 黑白灰 |
| 05 FT | 鲑鱼粉底+深蓝 | 暖橙+深蓝 |
| 06 Economist | 红+白+黑 | 红+蓝+灰 |
| 07 NYT | 蓝+白+红强调 | 灰+蓝 |
| 08 Bloomberg | 近黑底+荧光绿红 | 深蓝+亮色 |
| 13 Stripe | 紫+绿+红+橙 | 深蓝+亮色 |
| 20 Excalidraw | 纸张色+柔和彩 | 白底+手绘线 |

### 七、字体选择

**步骤**：
1. 从风格学派得到字体方向
2. typography-moods.md 中，hash / 10000 % 12 → 具体字体配对

| 风格学派 | 字体配对 |
|---------|---------|
| 01 麦肯锡 | Inter Tight + Inter |
| 02 贝恩 | DM Serif + DM Sans |
| 03 BCG | Space Grotesk + Space Mono |
| 04 Tufte | Libre Baskerville + Source Serif |
| 05 FT | Newsreader + Newsreader |
| 06 Economist | Roboto Condensed + Roboto |
| 07 NYT | Manrope + Inter |
| 08 Bloomberg | Orbitron + Roboto Mono |
| 12 IBM Carbon | Inter Tight + Inter |
| 13 Stripe | Plus Jakarta Sans |
| 20 Excalidraw | Amatic SC + Lato |

### 八、视觉节拍选择

视觉节拍定义**读者看图的阅读路径**。同一张图可以有不同的引导顺序。

| 节拍编号 | 名称 | 适用场景 |
|---------|------|---------|
| C-01 | 锚定数字 | KPI 大字报、关键指标 |
| C-02 | 色彩弹出 | 突出单个元素 |
| C-03 | 标注箭头 | 引导注意力 |
| C-04 | 基准线穿越 | vs 目标/平均/同期 |
| C-05 | 趋势拐点 | 带注释的时间序列 |
| C-06 | 对比揭示 | A vs B 差异高亮 |
| C-07 | 密度变换 | 从概览到细节 |
| C-08 | 隔离突出 | 噪音中找信号 |
| C-09 | 分组归类 | 分类后的子组对比 |
| C-10 | 排序阶梯 | 排名感知 |
| C-11 | 大小暗示 | 面积/气泡的大小对比 |
| C-12 | 运动方向 | 从左到右/从大到小的阅读流 |
| C-13 | 空间远近 | 核心在中心，边缘在外围 |
| C-14 | 层次递进 | 从标题→图例→数据→标注 |
| C-15 | 留白呼吸 | 用空白区域制造节奏变化 |
| C-16 | 色彩渐变 | 从冷到暖/从灰到彩的情绪变化 |
| C-17 | 形状区分 | 圆/方/三角的语义差异 |
| C-18 | 线型区分 | 实线=实际，虚线=预测 |
| C-19 | 标签密度 | 密集标签 vs 精简标签的节奏 |
| C-20 | 视觉重量 | 用粗细/大小/饱和度控制注意力 |

**选择步骤**：
1. 单图 → 从 C-01 到 C-20 中 hash % 20 选择 2-3 个节拍组合
2. 多图 → 按叙事弧线自带的节拍序列，不额外 hash

## 完整选择流程

### 示例：单图场景

```
输入: "供应商健康度评估"
  ↓
hash("供应商健康度评估") = H
  ↓
L1 内容锁死:
  模式: 22-雷达图（意图匹配：多维能力画像）
  弧线: 无（单图不走弧线）
  构图: 01 单中心聚焦（1 张图）
  ↓
L2 场景约束:
  场景: 高管汇报/供应商管理 → Tier 1 = [01, 02, 03]
  H % 3 = 0 → 选 01 麦肯锡
  开场: 全景概览 → Tier 1+2 = [02, 05]
  H/10 % 2 = 0 → 选 02 指标阵列
  ↓
L3 风格自由:
  配色: 麦肯锡色系0 → hash → color-themes #3
  字体: hash → typography #1
  节拍: hash → C-02 色彩弹出 + C-04 基准线穿越
```

### 示例：多图场景

```
输入: "2025年度供应商评审报告"（6 张图）
  ↓
hash("年度供应商评审报告") = H
  ↓
L1 内容锁死:
  弧线: 06 全景式（综合汇报天然路径）
  构图: 10 仪表盘（4+ 图）
  ↓
L2 场景约束:
  场景: 高管汇报 → Tier 1 = [01, 02, 03]
  H % 3 = 1 → 选 02 贝恩
  开场: 全景概览 → [02, 05]
  H/10 % 2 = 1 → 选 05 漏斗开场
  ↓
L3 风格自由:
  配色: 贝恩色系0 → hash → color-themes #7
  字体: hash → typography #2
  节拍: 走弧线自带序列（不额外 hash）
```

## 选择记录格式

在 SPEC 头部追加：

```markdown
## 自动选择记录

- **输入**: {主题}
- **Hash**: {H}
- **L1 内容锁死**:
  - 模式: {模式编号}-{名称}（意图匹配）
  - 弧线: {弧线名称/无}（{理由}）
  - 构图: {构图编号}-{名称}（{图表数量}张图）
- **L2 场景约束**:
  - 风格: {学派编号}-{名称}（hash % {N} = {余数}）
  - 开场: {开场编号}-{名称}（hash % {N} = {余数}）
- **L3 风格自由**:
  - 配色: color-themes #{N}
  - 字体: typography #{N}
  - 节拍: {节拍编号组合}
```

## 何时跳过自动选择

| 情况 | 处理 |
|------|------|
| 用户明确指定 | 走用户指令，跳过该层 |
| 候选池 = 1 | 直接选，不 hash |
| 模式独占约束 | 如甘特图只兼容甘特构图，直接选 |
