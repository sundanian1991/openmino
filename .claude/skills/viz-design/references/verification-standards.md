# Verification Standards · 验收标准

> 可插拔模块。对标麦肯锡/贝恩级图表质量。
> 目标：出图后通读自审，不是跑 checklist。

---

## 一、出图前：SPEC 完整性检查

在委托渲染之前，确认 SPEC 的 12 项必填齐全：

### 第一部分：叙事意图（4 项）

| 序号 | 字段 | 通过标准 |
|------|------|---------|
| 1 | 可视化目的 | 写了"认知缺口"——读者看图前后的状态变化 |
| 2 | 想传达什么 | 只有一句话，不是多条 |
| 3 | 结论 | 是判断不是描述 |
| 4 | 思路 | 解释了"为什么选这个图型"，不是"选了什么" |

### 第二部分：视觉执行（8 项）

| 序号 | 字段 | 通过标准 |
|------|------|---------|
| 5 | 模式选择 | 有编号 + 名称 + 1 句话匹配理由 |
| 6 | 标题 | 主标题是结论，副标题含时间 + 对象 |
| 7 | 视觉编码 | X/Y/颜色/大小各编码什么维度 |
| 8 | 数据组织 | 字段清单 + 排序 + 聚合规则 |
| 9 | 标注策略 | 高亮点 + 标注内容（原因+幅度） |
| 10 | 图例与辅助 | 图例/脚注/特殊说明 |
| 11 | 布局 | 画布尺寸 + 标题位置 + 留白 |

**缺任何一项 → 补全后再委托。**

## 二、出图后：通读自审

> **不跑 checklist。** 通读一遍图，问自己三个问题：

### 第一问：第一眼落在该看的地方吗？

闭眼 2 秒，睁眼。第一眼应该落在 SPEC 指定的高亮点上。

- 如果第一眼被标题吸走 → 标题字号过大或颜色过深
- 如果第一眼落在脚注 → 视觉层级反了
- 如果第一眼分散在多个元素 → 高亮过多

### 第二问：不看标题能读懂结论吗？

遮住标题，只看图。应该能得出与标题一致的判断。

- 如果不能 → 图没有独立说话，依赖标题
- 如果能得出不同结论 → 视觉编码与叙事意图不一致

### 第三问：这张图能 standalone 存在吗？

拿给一个不了解背景的人看。10 秒内应该能建立完整认知：

- 时间范围？→ 副标题或脚注
- 数据对象？→ 副标题或轴标签
- 核心结论？→ 标题
- 数据可信？→ 脚注来源

## 三、六条硬约束

> 源自 13-VISUALIZATION.md，但非强制。各渲染技能可保留自有约束。

| # | 约束 | 验证方式 |
|---|------|---------|
| 1 | 一个图一个核心洞察 | 能写出"而且"连接的两个结论 → 失败 |
| 2 | 标题是结论 | 是描述（"分析""趋势""对比"）→ 失败 |
| 3 | 高亮 ≤10% | 数元素，高亮/总数 > 0.1 → 失败 |
| 4 | 颜色 ≤2 ramp | 用了 3+ 色系 → 失败 |
| 5 | 无禁止项 | 饼图/双Y轴/3D/渐变/阴影/emoji → 失败 |
| 6 | 有副标题或脚注 | 底部无来源说明 → 失败 |

## 四、麦肯锡/贝恩级质量标准

以上六条是底线。要达到咨询级质量，还需要：

### M1：标题能独立作为 slide headline

麦肯锡每张图的标题是一条完整的判断句，读者只看标题就能带走结论。

**检验**：把标题单独拎出来放进邮件，收件人不需要看图也能理解核心判断。

### M2：标注是洞察，不是数字

贝恩的图表上，标注解释"为什么"，不重复"是多少"。

**检验**：标注里的数字如果全部遮住，剩下的文字还能解释趋势。

### M3：数据源和样本量可追溯

脚注包含：数据来源 + 时间范围 + 样本量（如果适用）。

**检验**：脚注信息足以让别人复现这张图。

### M4：灰化彻底

非高亮元素必须彻底灰化。Stone 300 是上限，不是下限。

**检验**：眯眼看图，只有高亮点和标题清晰可见。

### M5：字号级差

标题/副标题/轴标签/标注/脚注之间的字号差必须 ≥ 3px，层级分明。

**检验**：标题 ≥ 16px，副标题 11-12px，轴标签 11px，标注 11px，脚注 10px。相邻层级差不小于 1px，跨层级（标题→脚注）差 ≥ 6px。

### M6：对比度达标

标题色与背景色的对比度 ≥ 4.5:1（WCAG AA 标准）。

**检验**：标题 Warm 800 `#6b3410` 在白底 `#ffffff` 上对比度 8.6:1，合格。轴标签 Stone 500 `#857d74` 在白底上 5.1:1，合格。

### M7：网格线虚线

所有 splitLine 必须为虚线（`type: 'dashed'`），颜色 Stone 100 `#f2f0eb`。实线网格视觉噪音过高。

### M8：数据墨水比

图表区面积中数据元素（柱/线/点）面积占比 ≥ 30%，其余为留白和参考线。Tufte 数据墨水比原则：去掉任何不携带信息的墨水。

**检验**：去掉网格线、去掉边框、去掉装饰图标后，信息是否还在？如果在 → 那些就是多余墨水。

## 五、不同渲染技能的专项验收

### viz-echarts 专项

| 检查项 | 标准 |
|--------|------|
| animation | `false` |
| 数据 | 内联，无 fetch |
| 柱状图零基线 | `yAxis.min = 0` |
| 直接标注 | 不依赖图例 |
| 字符 | `&` 已转义为 `&amp;` |

### viz-svg-flow 专项

| 检查项 | 标准 |
|--------|------|
| viewBox | 高度覆盖所有元素（maxY + 40） |
| 箭头 | `<polygon>` 画，不用 `<marker>` |
| 旋转 | 无 `<g transform="rotate">` |
| 换行 | 用 `<tspan dy>` 或独立 `<text>`，不用 `\n` |
| 特殊字符 | `&` `<` `>` 已转义 |

### viz-editorial 专项

| 检查项 | 标准 |
|--------|------|
| 字号层级 | 字号层级合理（主标题 > 副标题 > 正文 > 脚注） |
| 间距 | 标题到内容 ≥12px |
| 卡片圆角 | 10px |
| 配色 | 色系克制（≤2 ramp 或有语义理由） |

---

## 六、验收通过标志

同时满足以下条件 = 验收通过：

- SPEC 12 项完整
- 三条硬约束全部通过
- 通读三问没有异常
- 渲染技能专项检查通过
