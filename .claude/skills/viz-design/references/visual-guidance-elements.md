# Visual Guidance Elements · 视觉引导元素库

> 可插拔模块。渲染技能按需读取。
> 目标：标题、标注、箭头、框线、图标——每种元素有统一规格，不凭感觉选。

---

## 一、元素分类

视觉引导元素分五类，按 **叙事功能** 组织：

| 类别 | 作用 | 典型元素 |
|------|------|---------|
| **指向** | 引导眼睛到关键点 | 箭头、指示线、高亮框 |
| **解释** | 说明"为什么" | 标注文字、气泡说明 |
| **基准** | 提供参考系 | 基准线、参考线、阈值线 |
| **强调** | 视觉权重加码 | 加粗、放大、色块底 |
| **连接** | 表达因果/关联 | 虚线箭头、关系线 |

---

## 二、指向元素

### 箭头

| 类型 | 用途 | SVG 画法 |
|------|------|---------|
| 实线箭头 | 标注指向高亮点 | `<line stroke-width="1.5" stroke="Stone 500"/><polygon fill="Stone 500"/>` |
| 虚线箭头 | 因果关联（非直接指向） | `<line stroke-dasharray="4,3"/>` |
| 曲线箭头 | 流程方向 | `<path fill="none" stroke-width="1.5"/>` |

**箭头规格**：
- 线宽：1.5px
- 箭头头部：8×6px 三角形 `<polygon>`
- 颜色：Stone 500（默认），Coral 500（预警指向）
- 起点距元素边缘 16px，终点不遮挡元素

**ECharts 中用 `markLine` 实现**：
```js
markLine: {
  symbol: ['none', 'arrow'],
  lineStyle: { color: '#857d74', width: 1.5 },
  label: { show: true, position: 'middle', formatter: '{c}', fontSize: 10 }
}
```

### 指示线

从图表区域外的文字指向区域内元素：

```
标注文字(外部) ─── 指示线 ───▶ 目标元素
```

- 线型：1px Stone 300 虚线
- 终点小圆点：3px 直径，Stone 500

### 高亮框

在元素外围加框强调：

| 状态 | 边框 | 说明 |
|------|------|------|
| 选中/聚焦 | 2px Coral 500 实线 | 唯一高亮 |
| 次要参考 | 1px Stone 300 虚线 | 辅助 |

框到元素间距：4px。圆角与元素一致。

---

## 三、解释元素

### 标注文字

| 属性 | 值 |
|------|----|
| 字号 | 11px |
| 字重 | 600 |
| 颜色 | 依语义（Warm 700 / Coral 500 / Teal 500） |
| 位置 | 高亮点右侧或上方，间距 ≥12px |

**标注格式**：`原因 → 结果+幅度`

| 错误写法 | 正确写法 |
|---------|---------|
| "9月: 37%" | "新方案上线 → 效率↑37%" |
| "拐点" | "引入自动化流程" |
| "最高" | "旺季+促销叠加" |

### 气泡说明

在紧凑空间内，用小型卡片作为"气泡"：

- 圆角 8px
- 背景 `#faf9f7`（Warm 50）
- 描边 1px Stone 300
- 内边距 8px
- 最大宽度 160px

## 四、基准元素

### 基准线

| 类型 | 线型 | 颜色 | 用途 |
|------|------|------|------|
| 行业基准 | 2px 虚线 | Stone 400 `#ada599` | 行业平均/标准值 |
| 目标线 | 2px 虚线 | Teal 500 `#2e8b6e` | 目标/达标线 |
| 预警线 | 2px 虚线 | Coral 500 `#c25030` | 清退/预警阈值 |
| 分隔线 | 1px 实线 | Stone 300 `#d6d2c9` | 区域分割 |

**ECharts 中用 `markLine` 实现**。

### 参考区域（markArea）

用半透明色块标记有意义的区间：

```js
markArea: {
  silent: true,
  itemStyle: { color: 'rgba(194,80,48,0.08)' },  // 极淡 Coral
  data: [[{ xAxis: '2026-02' }, { xAxis: '2026-04' }]]
}
```

- 透明度 ≤ 0.1，不抢数据注意力
- 区域标签：9px Stone 400，右上角

## 五、强调元素

### 加粗

| 元素 | 默认 | 加粗后 |
|------|------|--------|
| 折线 | 1.5px | 3px |
| 描边 | 1px | 2px |
| 柱形 | 正常宽 | 宽 × 1.2 |

### 放大

| 元素 | 默认 | 放大后 |
|------|------|--------|
| 数据点 | 4px | 8px |
| 图标 | 16px | 24px |
| 数字 | 26px | 42px+ |

### 色块底

将目标元素放在带色底的卡片中：

| 用途 | 底色 | 描边 |
|------|------|------|
| 正面强调 | Teal 50 `#e6f5f0` | Teal 300 1px |
| 负面预警 | Coral 50 `#faeae5` | Coral 300 1px |
| 中性聚焦 | Warm 50 `#faf0e6` | Warm 300 1px |

---

## 六、连接元素

### 因果连线

表达"A 导致 B"：

- 线型：1.5px 虚线
- 颜色：Warm 500
- 两端箭头
- 线上标注原因（10px Warm 700）

### 关联线

表达"A 与 B 相关"：

- 线型：1px 点线
- 颜色：Stone 300
- 无箭头

---

## 七、图标（SVG 内联）

在图表中使用极简 SVG 图标辅助叙事：

| 图标 | SVG | 用途 |
|------|-----|------|
| 上升 | `<svg width="12" height="12"><polyline points="2,10 6,2 10,6" fill="none" stroke="#2e8b6e" stroke-width="1.5"/></svg>` | 增长信号 |
| 下降 | `<svg width="12" height="12"><polyline points="2,2 6,10 10,6" fill="none" stroke="#c25030" stroke-width="1.5"/></svg>` | 下降预警 |
| 警示 | `<svg width="12" height="12"><text x="6" y="10" text-anchor="middle" font-size="10" fill="#c25030">!</text></svg>` | 异常提醒 |
| 检查 | `<svg width="12" height="12"><polyline points="2,6 5,9 10,3" fill="none" stroke="#2e8b6e" stroke-width="1.5"/></svg>` | 通过/达标 |

图标统一 12×12px，放在标注文字前面。

## 八、元素组合规则

| 叙事意图 | 组合 |
|---------|------|
| "看这里，这是问题" | Coral 色块底 + 警示图标 + 实线箭头指向 |
| "看这里，这是成果" | Teal 色块底 + 检查图标 + 实线箭头指向 |
| "这个点和那个点有关" | Warm 500 虚线连接 + 线上标注 |
| "这个区域值得注意" | 半透明 markArea + 右上角小标签 |
| "这条线是标准" | Stone 400 虚线 + "基准"文字标注 |

---

## 九、元素优先级

| 优先级 | 元素 | 理由 |
|--------|------|------|
| 最高 | 标题 | 读者第一眼 |
| 高 | 高亮点 + 指向箭头 | 引导注意力 |
| 中 | 标注文字 + 基准线 | 提供解释和参考 |
| 低 | 脚注 + 图例 | 补充信息 |
