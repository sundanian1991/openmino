# 图表有效性门禁

> viz-design 的质量门禁。选图后、委托渲染前必须通过。
> 对应 auto-selector L1 之后、SPEC 编写之前的检查点。

---

## R0: 3 秒理解检验

选图后，遮住标题，只看图的几何形态和颜色分布，问自己：

1. **3 秒内能否识别"谁好谁差"？**
2. **能否看出"差在哪、差多少"？**
3. **是否需要读者先理解特殊几何（菱形/螺旋/极坐标）才能读数据？**

3 秒内无法得出核心判断 → **门禁失败** → 自动切换到备选模式。

---

## 门禁规则

| 序号 | 检验项 | 通过标准 |
|------|--------|---------|
| R0-1 | 几何直觉 | 不需要先理解特殊几何才能读数据 |
| R0-2 | 对象可区分 | 图中 ≥2 个对象时，每个都有可区分的视觉标识 |
| R0-3 | 差距可读 | 优劣差距在图上一眼可见，不需要看 tooltip 或数字 |
| R0-4 | 轴数充足 | 雷达图等维度图，轴数 ≥5（4 轴像菱形） |

---

## 失败 → 备选模式替换表

门禁失败时，从下表选备选模式。备选模式需重新过门禁。

| 原模式 | 触发失败条件 | 备选模式 | 备选优势 |
|--------|------------|---------|---------|
| 22-雷达图 | 轴数<5 或 多对象叠加无法区分 | 13-柱状图（分组对比） | 线性对比，3 秒可辨 |
| 22-雷达图 | 仅 1-2 个对象但需多维对比 | 26-并列比较（表格+色块） | 精确数值可见 |
| 05-气泡图 | 数量>10 时重叠严重 | 09-散点图（无大小编码） | 去重叠，保留两维 |
| 20-桑基图 | 分支>8 时线条交叉混乱 | 15-甘特图 + 19-弧线图组合 | 分解为两步 |
| 21-弦线图 | 节点>6 时线条密度过高 | 19-弧线图（仅关键连接） | 降复杂度 |
| 33-辐轮 | 分支>8 时扇区过窄 | 13-柱状图（径向→线性） | 空间充足 |
| 03-同心图 | 层级>3 时内圈文字放不下 | 27-抽象树 | 文字空间充足 |
| 04-圆形图 | 指标>6 时环形拥挤 | 26-并列比较 | 线性排列 |
| 23-极坐标网格 | 周期>12 时刻度密集 | 07-折线图 | 线性展开 |
| 24-螺旋图 | 迭代>5 时圈层重叠 | 07-折线图 + C04-里程碑 | 展开为线性 |

---

## 执行流程

```
L1 意图匹配 → 选出首选模式
  ↓
R0 门禁检验（4 项）
  ↓
全部通过 → 继续写 SPEC
  ↓
≥1 项失败 → 查备选表 → 选备选模式
  ↓
备选模式重新过 R0 门禁
  ↓
备选也失败 → 标记"需多图组合" → 走 decision-rules Rule 4
```

---

## 与 decision-rules 的关系

本文件是 decision-rules.md 的 **Rule 10**，是委托渲染前的最后一道防线。

decision-rules Rule 1-9 是选图时的判断规则，Rule 10 是选完之后的有效性验证。

**Rule 1-9 决定"选什么"，Rule 10 验证"选对了吗"。**
