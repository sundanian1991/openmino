# viz-design — 可视化设计器

> 核心定位：不是画图工具，是**可视化设计器**。
> 输入：抽象概念、非结构化数据、文字描述、复杂关系
> 输出：完整的可视化 SPEC（模式选择 + 标题 + 数据组织 + 标注策略 + 叙事结构）
> 执行：委托 viz-echarts / viz-chart / viz-svg-flow 等渲染技能画出来

---

## 一、核心原理

**图表思维是认知压缩。** 文字是线性的，图像是并行的。一张好图让大脑跳过解码阶段，直达模式识别。

**选图本身就是洞察。** 选漏斗说明你看到了衰减关系，选齿轮说明你看到了联动机制。图的选择先于图的绘制。

**9509 条 DNA = 9509 个"意图→图表→设计策略"的完整经验记录。** 不需要从零想，从 DNA 索引中匹配最佳实践。

### 视觉层级铁律

> **视觉突出程度必须与重要性严格匹配。** 最重要的元素 = 视觉上最突出。禁止多个元素同等突出相互抢夺注意力。

**提升关键元素可视性的手段**：
- 调整大小、类型、标签和高亮方式
- 使用参考线和分割区块组织信息
- 重要文本用更大字号，节制使用加粗/倾斜/下划线
- 为关键数据加标签、提升颜色强度
- 为重要事件添加注释或说明
- 对离群值进行标注，或将同一双 Y 轴的数据拆分为两张图

**建立层级时必须做出选择**：只保留有助于受众更清晰理解的内容，隐藏不利于结论的数据。常用参考信息：
- 统计量：平均数、拟合回归线、置信区间
- 目标值、事件时间点（发生前/发生后）
- 重要时间、重要事件、类别

**设计要点检查清单**：
1. **删除非数据墨水** — 删除所有不影响表意的元素，最大化数据墨水比例
2. **确保每个元素都有标签** — 文字足够大，受众能清晰理解
3. **建立视觉层级** — 最重要元素最突出、比较对象明确标示、离群值加注释

## 二、技能边界

| 本技能负责 | 不负责 |
|-----------|--------|
| 概念→模式匹配（选什么图） | 实际画图（viz-design 绝不自己写 ECharts/SVG 代码） |
| SPEC 编写（标题/标注/布局/叙事） | 配色设计（各渲染技能自带配色体系） |
| 视觉叙事策略（先讲什么后讲什么） | 数据清洗（交给数据技能） |

## 三、四阶段工作流

```
Phase 0  全局风格锁定 — 多图场景先锁 globalStyle（storyboard.md 中）
  ↓
Phase 1  DNA 意图匹配 — 读 dna-index-ai.md，按意图选模式，按 ID 查完整 DNA
  ↓ references/dna-index-ai.md（15 类意图索引，AI 一次读完）
  ↓ references/data/chart-dna-index.tsv（按 ID 查完整策略）
Phase 2  intent.md       ← 意图决策（为什么做、为什么选这个模式）
  ↓ 模板：references/templates/intent.md
Phase 3  storyboard.md    ← 视觉叙事设计（叙事弧线、阅读路径、克制声明）
  ↓ 模板：references/templates/storyboard.md
Phase 4  spec.md          ← 编译规格（渲染契约 + 机器可读 JSON）
  ↓ 模板：references/templates/spec.md
Phase 5  checklist.md     ← 验收清单（出图后逐项对照）
  ↓ 模板：references/templates/checklist.md
```

**渐进加载**：先读 `references/library-index.md` 决定下一步加载什么，不要一次性加载所有参考文件。

**资产分层**（内容层写一次，表现层可迭代）：

| 层级 | 文件/内容 | 迭代时 |
|------|----------|--------|
| **内容层** | 数据清单、分析结论、叙事逻辑 | 不重写，全程复用 |
| **表现层** | 风格、配色、版式、SPEC 参数 | 每次迭代可调 |

迭代时只重写表现层，不重跑内容层分析。

**DNA 优先**：Phase 1 读 `references/dna-index-ai.md`（15 类意图索引），按表达意图匹配最佳图表类型和设计策略。锁定后按案例 ID 去 TSV 查完整 DNA（highlight/annotation/composition/reading_path/色板），继承全策略。未命中时自主推理，不走独立决策树。

**速查优先**：DNA 匹配后，立即查 `references/dna-playbook.md`。如果该 DNA 已有施工图模板（完整 ECharts option + 精确色值 + 标注规则 + 避坑清单），**直接套用模板，只替换数据**。不重新发明轮子。这是保证输出稳定性的核心机制。

**多图场景先走 Phase 0**：锁定全局风格（色系/字体/风格学派），后续每张图从全局继承。

### 入口模式

| 年老师说的 | 模式 | 做法 |
|-----------|------|------|
| "先看看数据能讲什么" | A 逐步引导 | 列出所有分析角度（2-5 个）→ 确认 → 再选图表 |
| "你帮我决定"/"最佳方案" | B AI 推荐 | AI 自主决策最佳方案 → 直接出图 |
| 直接指定图表或角度 | C 自定义 | 按指定执行 |
| 单图需求且意图明确 | C（快速路径） | 直接执行 |
| 多图需求或复杂数据 | A（默认） | 先分析再逐个确认 |

### 工作流详解

```
收到输入 → 选择入口模式（A/B/C）
  ↓
Phase 0: 全局风格锁定（仅 ≥2 张图时执行）
  ├── 风格学派：hash(主题名) → 读 style-schools.md
  ├── 全局色系：1 套主色 + 1 套强调色 → 读 color-themes.md
  ├── 全局字体：锁定 1 对 → 读 typography-moods.md
  ├── 多图衔接协议（内容被拆分时自动触发）：
  │   ├── 编号连续：上篇①⑦，下篇⑧⑬，保持逻辑流完整
  │   ├── 视觉引导：底部标注"▼ 续下篇：XXX"，顶部标注"▲ 承接上篇：XXX"
  │   └── 配色统一：全局色系贯穿，不得各自选色
  └── 写入 global-style.md
  ↓
Phase 1: DNA 意图匹配（读索引 + 查 TSV）
  1. 读 dna-index-ai.md — 意图速查表匹配表达意图
     ├── "趋势变化" → 面积图/折线图/散点图
     ├── "分类对比" → 柱状图/棒棒糖图/蝴蝶图
     ├── "占比构成" → 柱状图/堆叠条/饼图
     ├── "驱动因素" → 瀑布图/桥接图
     └── 15 类意图，每类 3-6 种推荐图表
  2. 锁定图表类型 → 从索引中取最佳案例 ID
  3. 按案例 ID 查 chart-dna-index.tsv → 继承全策略：
     chart_type / highlight / annotation / composition / reading_path / color_palette
  4. DNA 未覆盖的抽象概念（层级/齿轮/冰山等）→ 自主推理，参考 pattern-index.md 中的 B/C/D 类模式
  5. 自动选择器 — 读 auto-selector.md
     多图：仅走 L1，L2/L3 从 global-style 继承
     单图：L1→L2→L3 全流程
  ↓
Phase 1.5: 版式 + 解读策略（新增 — 不写文件，口头决策）
  6. 版式选择 — 读 layout-selector.md（场景→版式映射表）
     ├── 只有核心图表？→ #01 单图居中
     ├── 需要辅助解读？→ #08 L 型 或 #14 主次左右 70/30
     ├── 多个关键指标？→ #28 顶部 KPI + 多图表
     ├── 多图对比？→ #02 左右对比 或 #26 三栏对比
     ├── 流程/阶段？→ #22 水平步骤流
     └── 更多见 layout-selector.md 完整映射
  7. 解读策略 — 决定解读区形式：
     ├── 有明确结论要传达？→ 解读卡片（带原因+影响的要点）
     ├── 有关键数字要突出？→ KPI 卡片（大数字+趋势箭头）
     ├── 有对比/排名？→ 高亮 + 脚注说明
     ├── 纯数据展示无需解读？→ 不加解读区
  ↓ 写入 intent.md（含版式编号 + 解读策略）
Phase 2: storyboard.md — 视觉叙事
  8. 读 narrative-arcs.md + visual-beats.md 定弧线和节拍
  9. 场景设计 — "一句话灵魂" + 签名视觉元素 + 克制声明
  10. 写入 storyboard.md
  ↓
Gate: 用户确认点（非快速路径时执行）
  向年老师确认模式选择 + 推荐理由（AskUserQuestion）
  格式：推荐 X 模式，因为 Y。提供 2-3 个备选。
  年老师确认后 → 进入下一步
  ↓
Gate: 图表有效性门禁
  读 chart-validity-gate.md → R0 检验
  失败 → 查备选模式表 → 切换 → 重新过门禁
  ↓
Phase 3: spec.md — 编译规格
  11. 读 templates/spec.md，按模板编写
  12. 写入 spec.md（数据、视觉编码、标注策略、布局、版式）
  13. 委托渲染 → 传递四个 md 文档路径给下游技能
  ↓
渲染结果嗅探（委派后、验收前）
  快速扫描出图常见问题：内容截断 / 代码泄露 / 中文乱码 / 元素重叠
  有则自动处理或告知年老师，无则进入验收
  ↓
Phase 4: checklist.md — 验收
  16. 出图前自检 — 读 anti-patterns.md
  17. 验收 — 读 verification-standards.md
  18. 写入 checklist.md，标注通过/不通过
  19. 4 项以上不通过 → 回退 Phase 1
      ├── 同一修复策略连续失败 2 次 → 升级为架构级变更
      │   ├── 换图表类型（DNA 索引中查备选模式）
      │   ├── 拆分多图（走 Phase 0 多图衔接协议）
      │   └── 改变叙事结构（重写 storyboard）
      └── 首次失败 → 调参数重试（标注/高亮/布局/字号）
```

### 轻量模式（单图快速场景）

简单单图需求（如"帮我画个柱状图对比 5 个供应商业绩"）：

- Phase 1+2 合并为口头确认（不写文件）
- Phase 3 必须写 spec.md
- Phase 4 验收后，**必须委托渲染技能执行，不得自己出图**

## 四、SPEC 编写规范

> SPEC 是**视觉叙事剧本**，不是渲染指令。详细模板见 `references/spec-template.md`。

### 叙事意图（必须先写）

```
【可视化目的】为什么要做这张图？认知缺口是什么？
【想传达什么】核心信息（只有一句话）
【结论】读者应该得出什么判断？
【思路】为什么选这个图？视觉叙事路径是什么？
```

### 视觉执行（渲染指令）

```
【模式选择】编号 + 名称 + 匹配理由（1 句话）
【标题】主标题：结论性（不是描述性）；副标题：时间范围 + 对象
【视觉编码】X/Y 轴编码什么 / 颜色编码什么 / 大小编码什么
【数据组织】字段清单 / 排序规则 / 聚合规则
【标注策略】高亮 ≤10% / 标注写原因+幅度（不写数字）
【布局】画布尺寸 / 配色参考 13-VISUALIZATION.md 或自选
```

### 委托渲染路由

**viz-design 不写渲染代码，不猜下游格式。** 产出四个 md 文档，下游技能自己读自己决定怎么做。

**委托方式**：传递四个文档路径 + 一句话委托指令给下游技能。

**路由优先级**：

| 场景 | 首选下游 | 理由 |
|------|---------|------|
| 管理层汇报/商业报告 | **viz-echarts** | HTML 输出、中文零问题、可交互、多图合页面 |
| 学术论文/印刷出版 | **nature-figure** | 印刷级 SVG/PDF（ggplot2），但 svglite 中文有 bug |
| 交互式图表（tooltip/缩放） | viz-echarts | HTML 交互 |
| 商业仪表盘 | viz-echarts | 多图联动 |
| 概念图（齿轮/金字塔/冰山/流程） | viz-svg-flow | 抽象类比 |
| 结构关系（组织/层级/网络） | viz-svg-flow | SVG 生成 |
| 杂志风信息图 | viz-editorial | 编辑排版 |

**DNA render_target 参考**：DNA 的 `render_target` 字段提供"上次用什么效果好"的经验，不是强制路由。

**AI 路由自由度**：AI 可根据实际场景自由选择下游技能（多图统一一个技能、嵌入报告走 viz-editorial 等）。

**委托指令格式**：

```
"读以下 viz-design 文档，按要求渲染图表：
- intent.md: [路径]
- storyboard.md: [路径]
- spec.md: [路径]
- checklist.md: [路径]
请遵循 checklist.md 验收标准。配色参考 13-VISUALIZATION.md 或自选。"
```

## 五、委派收口铁律

> **核心原则：viz-design 只做设计，不做渲染。产出四个 md 文档，下游技能自己读自己做。**

**铁律 1：viz-design 绝不自己写渲染代码。** 不写 ECharts option、SVG path、matplotlib 代码。

**铁律 2：viz-design 绝不猜下游输入格式。** 不写渲染契约 JSON、不指定 tool_name。下游技能自己决定怎么做。

**铁律 3：多图场景必须先锁 globalStyle。** 后续每张图从全局继承，不得各自选色。

**铁律 4：委托方式统一。** 传递四个 md 文档路径 + 一句话委托指令。不存在第二条路径。

**铁律 5：出图后必须按 checklist.md 验收。** 逐项检查，不通过 = 打回重做。

**反模式（禁止）**：
- 自己写 ECharts option / SVG path / matplotlib 代码
- 猜下游技能的 JSON 输入格式
- 多图各自选色，不锁全局
- 自己组装多图 HTML（应委派容器技能）
- 出图后不验收即交付

