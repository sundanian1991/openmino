# 范例：汴京十二时辰

## 项目概况

- **类型**：研究型数字策展（大型项目）
- **主题**：北宋汴京一天十二时辰的城市生活
- **规模**：大型——多时段、多场景、多层级
- **结构**：时间线为骨架，每个时辰独立成篇，整体串联

---

## 策展结构

│   └── 案例：文人雅集

## 值得参考的设计决策

### 1. 时间线作为叙事骨架

十二时辰天然有时间线，策展以时间为轴，每个时辰是一个独立的策展单元。观众可以按时间顺序浏览，也可以随机进入某个时辰。这是**线性推进 + 多入口**的组合。

### 2. 每个时辰的内部结构一致

每个时辰页面遵循相同的模块结构：
- 标题（时辰名 + 意象）
- 开场（感官细节：声音、光线、气味）
- 主题模块（这个时辰的核心活动）
- 展开（具体场景、人物、事件）
- 证据/来源（历史文献、图像）
- 导航（上一个时辰 / 下一个时辰）

一致性让观众学会一个模式后，可以在所有页面复用。

### 3. 视觉语言：当代学术编辑式

- **文字主导**：大量文字内容，图像作为证据而不是装饰
- **历史地图作为材料**：地图不是背景，是和文字同等地位的证据
- **交互服务阅读**：点击是为了看更多内容，不是为了炫技
- **排版节奏**：阅读列 + 侧栏材料 + 来源区域，有明确的信息层级

### 4. 材料身份清晰

- **证据材料**：历史文献、宋代画作、考古发现
- **定位材料**：时间线、地图、方位指示
- **分析材料**：社会结构图、经济关系图
- **氛围材料**：极少量，克制使用

没有让图像成为"背景氛围"。每张图都有明确的功能。

### 5. 色彩和排版

- 深色背景（学术感、历史感）
- 文字以白色/浅色为主
- 强调色克制使用，只在需要引导注意力时出现
- 整体气质：安静、严肃、有深度

---

## 可复用的方法

1. **先找天然结构**：汴京的时辰结构是材料自带的，不是策展人强加的。找材料里自然存在的组织方式。
2. **模块一致性降低认知成本**：观众学会一个页面的模式后，所有页面都能复用。
3. **材料身份分类**：区分证据、定位、分析、氛围四种材料，防止图像变成无功能的装饰。
4. **克制交互**：交互只在服务阅读时才出现，不为炫技。
5. **入口具体**：每个页面的入口不是抽象标题，是具体的感官细节或数据。

---

## 局限性

- 汴京十二时辰的材料本身有极强的叙事张力（时间线 + 空间 + 人物群像 + 历史纵深）。不是所有材料都有这种天然结构。
- 视觉语言（深色学术风）适合历史类策展，不一定适合数据驱动的商业分析。
- 这是一个范例，不是模板。不要照搬结构，要学习方法。
