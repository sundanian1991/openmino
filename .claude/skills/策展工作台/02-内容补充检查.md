# 02-内容补充检查

## 概述

定题完成后，进入这个阶段。它做的事情很简单：对着策展地图，逐个主题检查材料够不够。不够的指出来，问用户要不要补。

不是 AI 自己补，而是告诉用户"这里缺什么"，让用户决定补不补、怎么补。

---

## 输入

01-定题输出的策展地图，包含：
- 主题结构
- 每个主题已有的材料
- 标记为"需要补充"的缺口

---

## 工作步骤

### 第一步：逐主题检查

对策展地图中的每个主题和子主题，检查：

| 检查项 | 说明 |
|--------|------|
| **材料是否足够支撑这个主题** | 这个主题要讲清楚，现有材料够吗？ |
| **是否有关键信息缺失** | 比如：有数据但没有背景说明，有结论但没有数据支撑 |
| **材料质量是否达标** | 数据是否准确？来源是否可靠？时间是否过期？ |
| **是否有冗余材料** | 有些材料可能和主题无关，可以排除 |

### 第二步：输出缺口清单

把检查结果整理成清单：

```
主题 A1：效率分析
  ✅ 已有：Q1 各月外呼人力数据
  ❌ 缺失：各月首外呼人力（只有总量，没有拆分）
  ❌ 缺失：效率指标定义（什么叫"效率"？）

主题 A2：规模分析
  ✅ 已有：27家供应商名单和站点分布
  ⚠️ 存疑：站点数据是否包含已清退供应商

主题 B1：质量分析
  ❌ 缺失：质检数据完全没有
  ❌ 缺失：客户满意度数据
```

### 第三步：交互确认

把缺口清单展示给用户，问：

1. **"这些缺口需要补吗？"** — 有些缺口可能不重要，可以跳过
2. **"你手上有没有这些材料？"** — 如果有，用户可以提供
3. **"需不需要我帮你找？"** — 有些材料可能在其他地方可以找到
4. **"补完以后再继续，还是先用现有材料做？"** — 用户决定是等材料还是先开始

### 第四步：更新策展地图

根据用户的反馈，更新策展地图：
- 补上了材料的标记为"已有"
- 确认不补的标记为"排除"（注明原因）
- 更新后问用户："可以进入 03-叙事规划了吗？"

---

## 原则

### 不代替用户做判断

AI 只负责指出"这里缺什么"。补不补、怎么补、先补哪个，由用户决定。

### 有些缺口不致命

不是所有缺口都需要补。如果一个子主题的材料不够，但不影响整体策展，可以先跳过，后面再补。

### 材料可以后续补充

策展是一个迭代过程。现在缺的材料，可以在后续阶段补充。不需要所有材料齐备才能继续。

### 排除也是决策

用户说"这个不补了"或"这个不需要"，这也是一个重要的决策，要记录在策展地图里。后续不会有人奇怪"这里为什么没有"。

---

## 输出

更新后的策展地图，包含：
- 每个主题的材料状态（✅已有 / ❌缺失 / ⚠️存疑 / 🛑排除）
- 缺口处理结果（补了什么、跳过了什么、排除了什么）
- 确认进入 03-叙事规划
