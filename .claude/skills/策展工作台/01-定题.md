# 01-定题：策展地图

## 概述

定题是整个策展工作台的 Plan。它要做的是：读完所有材料，判断策展目的和规模，输出一张完整的策展地图——主题有哪些、它们之间的关系、每个主题需要什么材料、材料够不够。定题确认了，后面的环节才有方向。

定题不是"提炼一个角度"。定题是"画出这张地图"。

---

## 输入

用户提供的任何材料：
- 数据文件（Excel、CSV、JSON）
- 文档（Markdown、Word、PDF）
- 演示文稿（PPT）
- 网页链接
- 对话记录
- 图片、视频
- 或其他任何形式

**原则：材料是什么形式不重要，重要的是里面有什么内容。**

---

## 工作步骤

### 第一步：读材料，不预设

读完所有材料。不预设最终形态（网页？PPT？看板？）。记录：
- 材料总量（行数、章节数、信息密度）
- 内容类型（数据型、叙述型、混合型）
- 内部结构（有清晰分章？还是平铺？）
- 信息之间的关系（时间线？层级？对比？因果？）

**整理用户的兴趣、材料和既有想法，先不预设最终网页形式。**

### 第二步：判断策展目的

问用户（或根据材料推断）：这份策展是为了什么？

| 目的 | 说明 | 影响 |
|------|------|------|
| **汇报** | 给上级/管理层看，需要结论先行 | 结构紧凑，重点突出，减少探索性内容 |
| **学习** | 自己或团队深入理解材料 | 结构开放，可以展开，鼓励多角度切入 |
| **探索** | 材料还没消化，先看看有什么 | 宽进窄出，先铺开再收敛 |
| **展示** | 给外部观众看，需要叙事张力 | 观众视角优先，故事线清晰 |
| **分析** | 从数据中发现规律、异常、趋势 | 数据驱动，图表优先，结论导向 |

目的不是互斥的。可能同时是"汇报+分析"，或"学习+展示"。但需要知道主要目的是什么，它决定了策展的重心。

### 第三步：判断规模

根据材料的密度、深度和广度，判断策展的规模：

| 规模 | 材料特征 | 策展结构 |
|------|----------|----------|
| **小型** | 单一主题，信息量适中 | 一个主题贯穿始终，无需分层 |
| **中型** | 2-3 个可区分的主题，有一定深度 | 几个主题并行，每个可独立成篇 |
| **大型** | 多主题，有层级关系，材料丰富 | 金字塔结构：总主题 → 子主题 → 支撑单元 |
| **超大型** | 主题多且深，跨领域，材料量极大 | 多层金字塔 + 每个子策展可独立运作 |

规模不由材料行数单独决定。100 行但信息密度极高的数据可能是中型。10000 行但重复度高的数据可能只是小型。**判断标准是：材料里有多少个独立的"值得讲的事"。**

### 第四步：提取 2-4 个可能的策展对象

**不要只给一个角度。** 从材料中提取 2 到 4 个可能的策展对象——也就是说，这份材料可以从哪几个完全不同的方向去切入。

**关键区分：**

| 做法 | 结果 | 问题 |
|------|------|------|
| 提炼摘要 | "这份材料讲了数据可视化方法论" | 太泛，没有切入角度 |
| 只找一个角度 | "用六课结构讲这本书" | 收敛了，可能遗漏其他价值 |
| **提出多个候选** | "可以从教学法/方法论/案例拆解/实践应用四个方向切入" | 为比较和选择留出空间 |

**策展对象**是什么：你打算研究什么？不是"展示什么"，而是"通过展示去研究什么"。它可以是一种方法论、一个决策问题、一组对比关系、一段演变过程、一套工具体系。

**进入尺度**是什么：观众通过什么入口进入？是通过一个具体案例、一个核心问题、一组对比数据、还是一条时间线？

**每个候选对象都要评估：**

| 维度 | 问题 |
|------|------|
| **进入尺度** | 观众从哪里进入？ |
| **真实主体** | 进入尺度背后真正要展示的是什么？ |
| **证据条件** | 材料是否足以支撑这个角度？哪些可靠、哪些缺失、哪些存疑？ |
| **目标观众** | 谁会看？他们关心什么？ |
| **风险** | 这个角度可能滑向什么偏差？ |

### 第五步：比较并推荐一个主 framing

**对 2-4 个候选对象做比较，推荐一个，说明为什么它更稳。**

不是简单说"我觉得 A 最好"。要展示推理过程：

- A 的优势是什么？风险是什么？
- B 的优势是什么？风险是什么？
- 为什么最终推荐 A 而不是 B？

**写明阶段边界**：当前做到哪一步为止？下一步是什么？

### 第六步：构建策展地图

framing 确认后，展开为完整的策展地图：

```
策展主题 A（总主题/核心主题）
├── 子主题 A1
│   ├── 支撑材料：xxx
│   └── 支撑材料：xxx
├── 子主题 A2
│   ├── 支撑材料：xxx
│   └── 需要补充：缺 xxx
└── 子主题 A3
    └── 支撑材料：xxx

策展主题 B（平行主题）
├── 子主题 B1
└── 子主题 B2
```

**小型项目**可能只有一个主题，没有子层级。

**大型项目**可能有 3-5 个总主题，每个下面有 2-4 个子主题，每个子主题下面有支撑单元。

### 第七步：交互确认

定题不是 AI 单方面输出，需要和用户确认：

1. **展示候选对象和推荐**，问：这几个方向你觉得哪个最对？
2. **展示策展地图**，问：这个结构对不对？有没有遗漏？
3. **指出材料缺口**，问：这里缺 xxx，要不要补充？
4. **讨论规模**，问：你觉得这是一个小型还是中型的策展？
5. **确认方向**：可以进入 02-内容补充检查了吗？

## 迷你定题：每个子主题的独特角度

策展地图展开后，每个子主题需要做一次迷你定题。逻辑和整体定题一致——提出候选、评估、选择——只是颗粒度更轻。

### 为什么需要迷你定题

7 个子主题用同一个结构（概念→案例→总结），不是策展，是流水线。策展的价值在于**再发现**——同一份材料，从不同角度切进去，看到的东西不一样。每个子主题的材料不同，它的角度和结构应该自己长出来。

### 迷你定题的三步流程

对每个子主题，执行和整体定题一致的逻辑：

**第一步：提出 2-3 个候选切入角度**

不是找"最有趣的东西"，而是提出 2-3 个完全不同的进入方式。同一个子主题，可以从比喻进入、从错误进入、从对比进入、从实验进入、从命题进入。每个候选都值得被评估。

候选可以是：
- 一个具体的入口（比喻、案例、实验、对比）
- 一个判断或命题（"X 其实是 Y"）
- 一个反直觉的发现

**第二步：评估每个候选**

| 评估维度 | 问题 |
|----------|------|
| **进入尺度** | 观众从哪里进入？是通过一个案例、一个实验、一个对比、还是一句话？ |
| **真实主体** | 这个切入角度背后真正要揭示的是什么？ |
| **材料支撑** | 这个角度需要的材料，是否能从原始材料中提取？ |
| **是否偏离整体** | 这个角度会不会偏离整体策展的 framing？ |

不需要像整体定题那样评估五个维度。观众和风险在整体定题中已确定，子主题不再重复。

**第三步：比较，选一个**

对 2-3 个候选做比较，推荐一个，说明为什么它更稳。然后确定这个子主题的组织方式——不是套模板，而是由材料本身决定的逻辑。

### 迷你定题的产出格式

```
子主题名称

材料：这个子主题涉及哪些材料
候选切入角度：
  A：xxx（进入方式）→ 评估：xxx
  B：xxx（进入方式）→ 评估：xxx
  C：xxx（进入方式）→ 评估：xxx
推荐：选 A/B/C，理由
组织方式：用 xxx 逻辑展开（这个子主题独有的结构，不是模板）
```

### 和整体定题的对应关系

| 维度 | 整体定题 | 迷你定题 |
|------|----------|----------|
| 候选数量 | 2-4 个策展对象 | 2-3 个切入角度 |
| 评估维度 | 进入尺度/真实主体/证据条件/目标观众/风险 | 进入尺度/真实主体/材料支撑/是否偏离整体 |
| 比较 | 推荐一个主 framing | 推荐一个切入方式 |
| 产出 | 定题说明 + 策展地图 | 切入角度 + 组织方式 |

逻辑相同，颗粒度不同。

### 反模式（禁止）

| 反模式 | 说明 |
|--------|------|
| **统一模板** | 7 个子主题都用"概念→案例→总结"结构 = 流水线 |
| **照搬章节顺序** | 材料的原始组织方式 ≠ 策展的组织方式。策展要重新发现 |
| **只列内容不找角度** | "这个子主题包含 A、B、C"是目录，不是定题 |
| **跳过候选直接给答案** | 没有比较就没有选择的重量 |
| **跳过迷你定题直接进叙事规划** | 没有角度的叙事规划 = 没有灵魂的结构 |

### 什么时候做

- 整体策展地图确认后，立即对每个子主题做迷你定题
- 迷你定题确认后，才进入 03-叙事规划
- 迷你定题可以和用户交互确认，也可以先做完再一起看

### 三层定题

```
整体定题：读所有材料 → 判断规模 → 提出候选 → 推荐 framing → 展开策展地图
  │
  └─ 迷你定题：对每个子主题 → 提出 2-3 候选 → 评估 → 选一个 → 确定组织方式
       │
       └─ 页面级定题：对每个页面 → 一句话点题（可选）
```

整体定题决定"讲什么"。迷你定题决定"每个部分怎么讲"。页面级定题决定"这一页的核心是什么"。三层思维，同一套方法，不同颗粒度。

---

## 输出

两份产出物：

**产出物一：定题说明**（包含）

- **策展目的**：这份策展为了什么
- **候选对象**：2-4 个可能的策展方向，每个附带进入尺度、真实主体、证据条件、目标观众、风险
- **推荐主 framing**：选哪个、为什么、排除了什么
- **阶段边界**：当前做到哪一步为止

**产出物二：策展地图**（定题确认后展开）

- **规模判断**：小型/中型/大型/超大型
- **主题结构**：总主题、子主题、支撑单元的层级关系
- **材料清单**：每个主题已有哪些材料、缺什么
- **下一步**：进入 02-内容补充检查（如果缺材料），或进入 03-叙事规划（如果材料够了）

**产出物三：子主题迷你定题**（策展地图确认后逐个展开）

每个子主题包含：
- **材料**：涉及哪些材料
- **最有趣的东西**：从材料中发现的最有意思的点
- **切入角度**：从哪里进入
- **组织方式**：这个子主题独有的展开逻辑（不是模板）

## 常见偏差

需要警惕这些情况：

| 偏差 | 说明 |
|------|------|
| **人物/工具成为全部故事，背后的结构消失** | 比如把"这本书讲了什么"当策展对象，而不是"这本书解决了什么问题" |
| **材料变成数据库，缺少策展判断** | 罗列了所有章节内容，但没有策展角度，没有"为什么这样组织" |
| **视觉呈现替代概念基础** | 因为"图表好看"就选了某个角度，而不是因为"这个角度最能揭示材料本质" |
| **主题过宽，每一页都必须重新解释全部背景** | 说明策展对象还不够聚焦，需要收窄进入尺度 |
| **在策展对象稳定之前，先决定网页交互和视觉风格** | 先想清楚讲什么，再想做成什么形态 |
| **只提炼一个角度** | 材料可能有多个值得展开的维度，不要收敛太早——先提出 2-4 个候选，比较后再选 |
| **把目的当主题** | "汇报 Q1 数据"是目的，不是主题。主题是"Q1 数据里有什么值得讲的" |
| **结构先于材料** | 先看完材料再决定用什么结构，不要先选金字塔再塞内容 |
| **忽略材料缺口** | 定题时就要发现缺什么，不要等到叙事规划时才发现 |
| **把所有东西都做成大型** | 小型项目一个主题贯穿就够了，不要硬分层 |
