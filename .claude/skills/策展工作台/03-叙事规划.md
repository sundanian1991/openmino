# 03-叙事规划

## 概述

定题解决了"讲什么"。叙事规划解决"怎么讲"。

这一阶段不决定做网页还是做 PPT。它决定的是：每个主题用什么叙事结构、信息怎么组织、观众的阅读路径是什么。这些确定了之后，04-研究转策展文本负责把叙事方案落位到具体字段，05-视觉语言提供参考风格，06-原型实现负责实际产出。

---

## 输入

- 01-定题输出的策展地图
- 02-内容补充检查确认后的材料状态

---

## 工作步骤

### 第一步：为每个主题选择叙事结构

不同主题适合不同的叙事方式。不是所有主题都需要同样的结构。

| 叙事结构 | 适用场景 | 特点 |
|----------|----------|------|
| **线性推进** | 有时间线或因果链的材料 | 先后顺序，一步接一步 |
| **对比结构** | 有两组或多组可对比的数据/方案 | Before/After、A vs B、理想与现实 |
| **层级展开** | 有总分关系的材料 | 先给结论，再逐层展开 |
| **案例串联** | 有几个独立但相关的案例 | 每个案例独立成篇，之间有共性 |
| **问题驱动** | 材料围绕一个核心问题 | 先抛问题，再逐步拆解 |
| **数据叙事** | 数据密集型材料 | 图表为主，文字为辅，数据讲故事 |
| **探索路径** | 材料复杂，观众需要自己选择路径 | 多入口，非线性，观众自主探索 |

一个策展里可以有多种叙事结构。不同主题可以选不同的方式。关键是匹配材料本身的特点。

### 第二步：规划每个主题的信息层级

对每个主题，规划信息的展开方式：

```
入口层：观众最先看到什么？（标题、核心结论、引人注目的数据）
  → 展开层：接下来看到什么？（背景、分析、论证）
    → 深入层：想继续看什么？（细节、证据、数据表、原始材料）
      → 出口层：看完以后去哪？（下一个主题、总结、行动建议）
```

不需要每一层都很重。小型项目可能入口层+展开层就够了。大型项目需要四层都做扎实。

### 第三步：规划呈现方式

针对每个主题，决定用什么方式呈现：

| 呈现方式 | 适用场景 |
|----------|----------|
| **图表** | 数据型内容，需要可视化趋势、对比、分布 |
| **表格** | 需要精确数值，或多维度对比 |
| **文字叙述** | 需要解释、分析、讲故事 |
| **图文混排** | 数据+分析结合，图表配文字解读 |
| **时间线** | 有时间维度的事件或变化过程 |
| **流程图** | 有步骤、流程、因果关系 |
| **对比面板** | A vs B 的直接对比 |
| **卡片组** | 多个平行案例或条目 |

同一主题内可以混合使用。关键是每种呈现方式都有明确的理由——不是因为"好看"，而是因为"这样讲最清楚"。

### 第四步：交互确认

把叙事方案展示给用户：

1. **展示每个主题的叙事结构 + 呈现方式**，问：这样讲对不对？
2. **展示信息层级**，问：入口层放这个合适吗？
3. **展示主题之间的关系**，问：先讲 A 再讲 B，还是反过来？
4. **确认后**，进入 04-研究转策展文本

---

## 分形原则

叙事规划也可以在每个层级重复使用：

```
整体策展 → 叙事规划（主题之间的讲述顺序和关系）
  └─ 每个主题 → 迷你叙事规划（这个主题内部怎么展开）
       └─ 每个页面 → 迷你叙事规划（这个页面的信息层级）
```

---

## 输出

每个主题的叙事方案，包含：

- **叙事结构**：选了哪种方式，为什么
- **信息层级**：入口 → 展开 → 深入 → 出口
- **呈现方式**：每个层级用什么方式呈现
- **主题间关系**：先讲什么、后讲什么、怎么串联

确认后进入 04-研究转策展文本。
