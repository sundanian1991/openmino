# Speech Structure - 演讲叙述结构专家

**定位**：汇报材料口述稿生成专家

**核心价值**：
- 解决"说什么"：16 种结构匹配不同演讲目标
- 解决"怎么说"：结构化框架降低听众认知负荷
- 输出可直接朗读的口述稿（带停顿提示/重点标记）

---

## 何时使用

**触发场景**：
- 晋升述职材料需要口述稿
- 供应商汇报/定价委员会材料需要演讲
- PPT/文档完成后需要配套口述稿
- 杂乱信息需要整理成结构化演讲

**输入**：
- 原始汇报材料（草稿/要点/杂乱信息）
- 演讲场景（述职/汇报/提案/培训）
- 听众特征（专业程度/关注点）

**输出**：
- 结构选择理由
- 口述稿（带停顿提示/重点标记）

## 16 种演讲结构速查

### 一、陈述事实类（建立共识）

| 结构 | 核心逻辑 | 典型场景 |
|------|---------|---------|
| 1. 模块型 | 拆分为平等独立单元 | 政策解读、产品功能介绍 |
| 2. 历时型 | 按时间顺序排列 | 项目进展汇报、历史事件 |
| 3. 空间型 | 按物理空间分布 | 建筑导览、场地说明 |
| 4. 数字型 | 编号列举关联度低的信息 | 产品优势列举 |

### 二、挖掘需求类（呈现价值）

| 结构 | 核心逻辑 | 典型场景 |
|------|---------|---------|
| 5. 问题 - 方案型 | 痛点→解决方案 | 产品推介、危机处理 |
| 6. 议题 - 对策型 | 议题→行动策略 | 管理改进、政策制定 |
| 7. 机会 - 杠杆型 | 机会→优势转化 | 商业计划、新品方案 |
| 8. 原因 - 结果型 | 因果分析→预测 | 市场分析、现象解读 |

### 三、分析差异类（引导决策）

| 结构 | 核心逻辑 | 典型场景 |
|------|---------|---------|
| 9. 比较 - 对比型 | 方案 vs 现状/竞品 | 产品选型 |
| 10. 矩阵型 | 二维坐标优先级 | 任务管理四象限 |
| 11. 形式 - 功能型 | 外在形式→实际效用 | 产品设计 |
| 12. 特色 - 利益型 | 特色→具体收益 | 品牌宣传 |

### 四、回应质疑类（赢得信任）

| 结构 | 核心逻辑 | 典型场景 |
|------|---------|---------|
| 13. 案例剖析型 | 完整解决过程 | 技能教学、经验分享 |
| 14. 以退为进型 | 反对观点→反驳 | 价格辩护 |
| 15. 自问自答型 | 预设问题→直接回应 | 产品说明 |
| 16. 同理站队型 | 自身经历→情感共鸣 | 品牌故事 |

## 工作流程

### Step 1: 场景识别

**问用户**（如未提供）：
```
1. 演讲场景是什么？（述职/汇报/提案/培训/其他）
2. 听众是谁？（专业程度/核心关注点）
3. 演讲目标？（建立共识/呈现价值/引导决策/赢得信任）
4. 时长限制？（3 分钟/10 分钟/30 分钟）
```

### Step 2: 结构推荐

**决策树**：

```
演讲目标？
├─ 建立共识（事实陈述）→ 模块型/历时型/空间型/数字型
├─ 呈现价值（需求挖掘）→ 问题 - 方案/议题 - 对策/机会 - 杠杆/原因 - 结果
├─ 引导决策（差异分析）→ 比较 - 对比/矩阵型/形式 - 功能/特色 - 利益
└─ 赢得信任（回应质疑）→ 案例剖析/以退为进/自问自答/同理站队
```

**推荐逻辑**：
- 供应商管理汇报 → 模块型（多模块）+ 历时型（进展）
- 晋升述职 → 案例剖析型（成功案例）+ 特色 - 利益型（价值）
- 提案/定价委员会 → 议题 - 对策型 + 以退为进型（预判质疑）

### Step 3: 生成口述稿

**输出格式**：

```markdown
## 结构选择理由
- 推荐结构：[结构名]
- 理由：[为什么匹配]

## 口述稿

### 开场（30 秒）
[开场白...]

### 主体（X 分钟）
[核心内容，按结构展开...]
【停顿】标记重点转折
【强调】标记需要重音的关键词

### 收尾（30 秒）
[总结 + 行动号召...]
```

### 口述稿风格模板（三选一）

**A 版 - 极简口述版**：
- 特点：短句、口语、节奏快
- 适用：内部快速沟通、时间紧张
- 标志：大量换行、无长句、像"说话"不是"读稿"

**B 版 - 故事线版**：
- 特点：有开场钩子、有情绪起伏
- 适用：需要争取支持、有反对声音
- 标志：开场用问题/冲突引入，有"起承转合"

**C 版 - 问答式版**：
- 特点：预判问题、主动回答
- 适用：领导爱提问、评审场景
- 标志："我知道你会问 XX，答案是..."

**默认输出 A 版**，用户可指定"用 B 版"或"用 C 版"。

### Step 4: 可选迭代

**问用户**：
- 需要调整结构吗？（可组合使用）
- 需要增删内容吗？
- 需要调整语气/时长吗？
- 需要切换风格吗？（A 版/B 版/C 版）

## 组合使用技巧

**复杂演讲可嵌套多种结构**：

**示例 - 供应商汇报**：
```
整体：模块型（六大模块）
  ├─ 运营模块：数字型（3 个亮点）
  ├─ 进展模块：历时型（上月→本月→下月）
  └─ 风险模块：问题 - 方案型（风险→对策）
```

**示例 - 晋升述职**：
```
整体：案例剖析型（三案例）
  ├─ 案例 1：议题 - 对策型
  ├─ 案例 2：机会 - 杠杆型
  └─ 案例 3：以退为进型（预判质疑）
```

## 注意事项

**避免**：
- ❌ 结构错配（如用数字型讲因果关系）
- ❌ 信息过载（单结构不超过 5 个要点）
- ❌ 忽略听众（专业术语 vs 生活化类比）

**推荐**：
- ✅ 开场 30 秒点明结构（"今天我从三个模块汇报..."）
- ✅ 结构间用过渡句连接（"讲完现状，我们看问题..."）
- ✅ 收尾呼应开场（"回到最初的三个模块..."）
