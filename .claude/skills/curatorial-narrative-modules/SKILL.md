# 叙事结构与页面模块

## 概述

使用这个 skill 从策展文本进入页面结构。它决定项目需要哪些页面或内容单元，每个单元承载什么，观众如何移动，哪些交互真正对应策展关系。

## 页面类型先行

先判断页面类型，再讨论具体布局。常见类型包括：

- 入口页或封面页。
- 索引页或导览页。
- 核心专题页、案例页、物件页、地点页或时间单元页。
- 跨单元分析页。
- 来源页、档案页、术语页或研究页。
- 关于页、方法页或鸣谢页。

每种页面类型都需要说明页面角色、页面主角、材料身份、阅读深度和导航关系。

## 模块拆解

为每个页面或单元定义：

- 稳定标题和定位文本。
- 开场材料或经验层。
- 主题或问题模块。
- 主体解释文本模块。
- 媒体、地图、档案或分析图栏位。
- 证据和来源模块。
- 进入、返回、下一步和跨页跳转动作。

## 交互逻辑

内容层级清楚之后再选择交互方式。点击、滚动、标签页、地图、覆盖层、抽屉、筛选器、时间线和网络高亮，都应对应某种策展关系：进入、比较、切换、揭示证据、标记当前位置或进入下一层。

## 输出

输出页面类型表、模块 schema、导航图和实现说明。此阶段不以具体视觉风格为中心；视觉决策应服从页面角色和内容层级。
