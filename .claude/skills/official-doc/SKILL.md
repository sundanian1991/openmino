# 公文编写

## Overview

撰写专业商务公文，基于场景选择合适模板，确保内容规范、逻辑清晰、专业得体。

## When to Use

```
需要撰写公文？
    ├─ 向上汇报？ → 使用 formal-report 模板
    ├─ 发通知？ → 使用 notification 模板
    ├─ 跨部门协作？ → 使用 collaboration 模板
    ├─ 内部沟通？ → 使用 internal 模板
    └─ 供应商管理？ → 选择对应生命周期模板
```

**Use when:**
- 撰写正式公文（报告、通知、协作函）
- 供应商管理文档（准入、运营、评估、清退、结算）
- 内部协调文档（决策请示、会议邀请、进度同步）
- 合规风险文档（培训通知、案例通报、风险预警）

**Don't use for:** 个人闲聊文档、非正式沟通

## Remix 模式（快速变体生成）

> **核心理念**：基础模板 → 4-6个语气变体 → 快速适配不同场景

### 语气变体矩阵

| 变体 | 特点 | 适用场景 | 关键词示例 |
|------|------|---------|-----------|
| **正式** | 严谨、规范、有据可依 | 向上审批、决策请示、供应商正式通知 | "根据规定"、"综合评估"、"特此通知" |
| **柔和** | 委婉、协商、留有余地 | 跨部门协作、供应商协调、问题沟通 | "建议考虑"、"是否可以"、"麻烦确认" |
| **紧急** | 直接、明确、限时要求 | 故障通报、风险预警、时限任务 | "紧急"、"请于X时前"、"立即处理" |
| **协商** | 平等、双赢、共同目标 | 商务谈判、激励方案、合同讨论 | "共同探讨"、"双赢方案"、"建议调整为" |
| **告知** | 客观、信息传递、无需反馈 | 信息同步、结果公布、数据通报 | "特此告知"、"供参考"、"无需回复" |
| **质询** | 严肃、要求解释、追责 | 违约问询、异常情况、整改要求 | "请说明"、"请解释"、"将采取" |

### Remix 工作流

```
1. 确定场景（向上/向下/平级/供应商）
2. 选择基础模板
3. 生成 4-6 个语气变体
4. 根据具体情况选择最合适的变体
5. 二次微调（补充具体数据、风险说明）
```

### 示例：结算通知公文的6种变体

| 变体 | 开头示例 | 使用时机 |
|------|---------|---------|
| 正式 | "根据合同约定，现就X月结算事宜通知如下..." | 标准月度结算 |
| 柔和 | "关于X月结算，有个小问题需要和您确认一下..." | 结算金额有差异需要协商 |
| 紧急 | "【紧急】X月结算务必于今日17点前确认，否则将影响付款..." | 逾期未确认、影响付款时限 |
| 协商 | "关于X月结算金额，我们建议调整为...，您看是否可以？" | 结算争议、需要协商 |
| 告知 | "X月结算已完成，明细如下，供参考..." | 结算已完成、无需对方确认 |
| 质询 | "X月结算数据存在异常，请于X时前说明原因..." | 数据异常、需要解释 |

## 写作标准

### 公文写作规范（专业简洁，空间舒适）

**核心原则：段落逻辑优先，视觉装饰为零，适度留白舒适**

| 正确做法 | 错误做法 |
|----------|----------|
| 段落开始用**加粗关键词**区分 | 用分割线/边框等装饰 |
| 表格用于数据展示 | 用表格做视觉分隔 |
| 段落靠内在逻辑推进 | 依赖分段/标题堆叠 |
| 适度留白保持阅读节奏 | 段落过密或过疏 |

**标准格式示例：**

```markdown
尊敬的XXX：

一句话概括公文内容。

**事实部分：**

具体事实描述，用列表或表格呈现关键数据（时间、人员、数据）。

**分析判断：**

基于事实的分析，使用"综合评估认为"、"这说明"等专业表述。

**建议/结论：**

明确的建议或结论，附上理由和依据。

**风险提示/后续安排：**

必要的风险说明或后续行动步骤。

结尾总结句和行动请求（请审阅/请批示/如有问题请反馈）。
```

**6条具体规则**：

1. **开头称呼**：称呼单行，空一行后接主体
2. **关键词标识**：每个段落开始用`**关键词：**`标识
3. **禁止装饰**：不使用分割线、边框等视觉装饰
4. **数据结构化**：时间、人员、数据等用列表/表格清晰呈现
5. **留白适度**：段落之间空一行，保持阅读节奏
6. **结尾行动**：明确的行动请求或请示

### 标题规范

- 以"关于..."开头，明确主题
- 简洁具体，避免模糊
- 时效性内容需包含时间范围

### 内容结构

1. **开头**：称呼+目的
2. **事实部分**：列表/表格呈现（时间、人员、数据等）
3. **过渡句**：段落间用"基于上述..."、"综合评估..."等连接
4. **分析判断**："我们认为..."、"这说明..."等加入观点
5. **结尾**：明确请求（"请审阅"、"如有问题请反馈"）

### 格式技巧

- 结构化数据用列表/表格
- 密集数据用表格呈现
- 叙述部分用段落+过渡句
- **重点内容用加粗**
- 明确的行动请求（Call to Action）
- 段落间使用过渡句
- 包含分析判断，不只是事实陈述

### 语调要求

- 专业正式，不随意
- 清晰直接，不机械
- 逻辑严密，结构清晰

## Common Mistakes

| ❌ 错误 | ✅ 修正 |
|-----------|-------|
| 模糊标题如"汇报" | 具体："关于供应商Q1评估汇报" |
| 没有行动请求 | 添加："请于周五前反馈" |
| 全文加粗 | 仅加粗关键项 |
| 缺少数据支撑 | 补充具体数据、案例 |
| 观点模糊 | 给出明确判断和依据 |
