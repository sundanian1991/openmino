# 供应商管理周报模板
## 使用说明
**适用场景**：供应商商务管理周会汇报（1-2分钟）
**汇报对象**：上级领导
**汇报重点**：商务进展、卡点支持、下周计划

---
## 标准周报格式

XX总好：

本周供应商管理周报如下。

**一、本周重点进展：**

**1. [标题]**

● 具体进展：XXX（用一句话描述关键成果）

● 关键数据：XXX（如涉及X家供应商，完成XX项沟通）

● 当前状态：XXX（进行中/已完成/待跟进）

**2. [标题]**

● 具体进展：XXX

● 关键数据：XXX

● 当前状态：XXX

**二、需支持/卡点事项：**

**●1. [标题]**

卡点情况：XXX（简述问题本质）

需要的支持：XXX（如：决策、资源协调、跨部门协助）

建议方案：XXX（如适用）

**●2. [标题]**

卡点情况：XXX

需要的支持：XXX

**三、下周重点计划：**

1. XXX（具体任务 + 预期成果）

2. XXX

3. XXX

**四、简要风险提示：**

● XXX（如果有需要关注的风险，如无则写"无重大风险"）

以上，请审阅。

## 写法要点
### 一、本周重点进展（必填）
**适合描述的场景**：
- 商务谈判进展（价格、合同、激励方案）
- 供应商沟通（准入、清退、争议处理）
- 评估考核完成（季度排名、月度评估）
- 分量调整完成（增量、减量、调配）
- 重大问题解决（整改完成、故障恢复）
**写法示例**：
```
1. 中乾价格谈判
● 具体进展：已完成2轮谈判，供应商接受X元/通目标价
● 关键数据：涉及XX席/月，预计节省XX元/月
● 当前状态：待合同签署

2. Q1评估结果通知
● 具体进展：已完成X家供应商评估结果通知
● 关键数据：A级X家、B级X家、C级X家
● 当前状态：供应商确认中
```
### 二、需支持/卡点事项（按需填写）
**适合描述的场景**：
- 需要上级决策的事项（清退、重大处罚、合同变更）
- 需要跨部门协调的事项（法务、财务、合规）
- 供应商配合度低、进展缓慢
- 资源冲突、时间紧张
**写法示例**：
```
1. 博彦清退决策
● 卡点情况：该供应商连续2个月C级，但现有100席需承接
● 需要的支持：是否批准清退？备选方案为新动力（80席）
● 建议方案：X月X日前完成清退，X月X日前完成切换

2. 法务合同评审滞后
● 卡点情况：汇讯补充协议已提交X天，仍未收到评审意见
● 需要的支持：协调法务部门X月X日前完成评审
● 建议方案：XXX
```
### 三、下周重点计划（必填）
**写法示例**：
```
1. 完成锦瑞增量谈判（目标：增量30席，价格XX元/通）
2. 组织Q2评估准备会（参会：评估团队、核心供应商）
3. 推进创博清退流程（目标：X月X日前完成交接）
```
### 四、简要风险提示（按需填写）
**适合描述的场景**：
- 合规风险（数据安全、监管检查）
- 运营风险（系统故障、人员短缺）
- 商务风险（合同到期、争议升级）
**写法示例**：
```
● 汛讯3项投诉未解决，可能引发监管关注
● 新动力100席清退计划，业务承接压力较大
```

## 快速填写模板
**周报编号**：WR-YYYYMMDD-XXX
**本周核心数据快览**（可选，放在邮件主题后）：
- 沟通X家供应商：XXX
- 完成X项关键任务：XXX
- 解决X个卡点问题：XXX
**填写流程**：
1. 先写本周重点进展（最多3项，按优先级排序）
2. 再写需支持事项（有就写，没有就写"无"）
3. 最后写下周计划（3项核心任务）
4. 风险提示按需填写

## 注意事项
### 周报撰写原则
- **简洁直接**：每项3-5行，1-2分钟可读完
- **数据说话**：用具体数字和事实，不用模糊描述
- **突出卡点**：需支持事项要清晰具体，明确诉求
- **风险意识**：及时提示需要关注的风险
### 常见错误
- ❌ 写成流水账（做了很多琐事）
- ✅ 只写重点进展（核心成果）
- ❌ 没有数据支撑（"进展顺利"、"沟通良好"）
- ✅ 有具体数据（"完成X家沟通"、"达成XX协议"）
- ❌ 卡点描述模糊（"需要协调"）
- ✅ 卡点描述具体（"需要法务X日前评审"）
### 时间控制
- 1-2分钟读完 = 约200-300字
- 最多3项重点进展 + 2项需支持事项
