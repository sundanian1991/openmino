# Section Archetypes — 常见 section 类型的大白话解释

> 这不是一个"必须对照"的清单，而是一个辅助理解的参考。
> 解剖 skill 时如果遇到这些 section，可以快速理解它的设计意图。
> 但 skill 里可能有这个清单之外的巧妙设计——别被清单限制住。

## 常见 section 及其意图

### 角色扮演规则
**它在干嘛**：告诉 AI "你要假装是这个人在说话"，不要用"他会认为..."这种旁观者口气。
**打个比方**：就像演话剧——导演说"你现在是哈姆雷特"，不是"你来分析一下哈姆雷特会怎么想"。
**没有它会怎样**：AI 会不自觉地跳出角色，变成一个在旁边评论的解说员。

### Agentic Protocol（回答工作流）
**它在干嘛**：遇到需要查资料才能回答的问题，先去查，别瞎编。
**打个比方**：就像一个靠谱的朋友——你问他"这家餐厅好不好"，他不会瞎说，而是先打开大众点评看看再告诉你。
**没有它会怎样**：AI 会凭"感觉"回答事实性问题，说得很自信但可能是编的。

### 信息源黑名单
**它在干嘛**：明确列出"这些地方的信息不靠谱，别用"。
**打个比方**：就像你妈告诉你"别听隔壁王阿姨的养生建议"——与其列一堆"可以听谁的"，不如直接说"这几个人别信"。
**没有它会怎样**：AI 可能从不靠谱的来源获取信息，然后一本正经地转述给你。

### Phase X.5 检查点（质量门）
**它在干嘛**：做到一半先停下来检查，确认没问题再继续。
**打个比方**：就像盖房子——地基打完先验收，别等房子盖好了才发现地基歪了。拆了重来的成本是验收的 10 倍。
**没有它会怎样**：前面的步骤如果出了问题，后面所有步骤都白做。垃圾进，垃圾出。

### 迭代上限
**它在干嘛**：规定"最多改 N 轮就交卷"，不要无限追求完美。
**打个比方**：就像考试时间到了就得交卷——不完美但诚实的 60 分，好过一直改到最后交了一份看起来完美但其实在编的 90 分。
**没有它会怎样**：AI 会陷入"再改一轮就更好了"的循环，永远交不出东西。

### 表达 DNA
**它在干嘛**：把一个人的说话风格量化成具体参数——用什么词、什么句式、什么节奏。
**打个比方**：就像模仿一个人说话——不是背他说过的原话，而是掌握他的"口头禅"、"说话习惯"、"转折方式"。
**没有它会怎样**：AI 模仿出来的"人物"会像一个通用的 ChatGPT 在说话，没有辨识度。

### 心智模型
**它在干嘛**：提炼一个人看世界的"镜片"——他用什么框架理解问题。
**打个比方**：就像不同颜色的太阳镜——戴红色镜片看什么都偏红，戴蓝色镜片看什么都偏蓝。心智模型就是这个人戴的"镜片"。
**没有它会怎样**：AI 只能复述这个人说过的话，但遇到新问题就不知道该怎么"像他一样思考"了。

### 诚实边界
**它在干嘛**：明确写出"这个 skill 做不到什么"。
**打个比方**：就像药品说明书上的"禁忌症"——不是说这个药不好，而是告诉你什么情况下别用。
**没有它会怎样**：用户会对 skill 的能力产生不切实际的期望，然后在它失败时感到失望。

### 双路径入口
**它在干嘛**：同时服务两种用户——知道自己要什么的（直接给名字），和不知道自己要什么的（只有模糊需求）。
**打个比方**：就像餐厅菜单——老顾客直接点菜名，新顾客可以说"我想吃辣的"让服务员推荐。
**没有它会怎样**：要么新手用不了（因为不知道该输入什么），要么老手觉得烦（因为每次都要回答一堆问题）。

### 并行 Agent
**它在干嘛**：把一个大任务拆成几个小任务，让多个 Agent 同时干，最后汇总。
**打个比方**：就像一个团队分工——一个人查书、一个人看视频、一个人搜社交媒体，比一个人从头到尾做完快得多。
**没有它会怎样**：一个 Agent 从头做到尾，注意力会分散，而且慢。

### 输出归档
**它在干嘛**：把结果自动存到固定位置，带时间戳，方便以后找。
**打个比方**：就像拍完照自动存到相册——你不用每次手动保存，以后想找也知道去哪找。
**没有它会怎样**：好不容易生成的内容，关掉窗口就没了。
