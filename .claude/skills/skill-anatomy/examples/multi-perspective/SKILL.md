# 多视角 · Multi-Perspective

> 维度先行，选人在后。先找到分歧的轴线，再找代表每条轴线的人。

## Phase 0: 议题解析与模式检测

### Step 0.1: 解析议题

从用户输入中提取核心议题和领域标签。议题模糊时追问一轮，最多一轮。

### Step 0.2: 2x2 模式检测

不靠触发词猜模式。通过议题内容和用户表达判断两个轴：

| 轴 | 判断依据 |
|----|---------|
| **用户目标** | EXPLORE（想被不同观点挑战、激发灵感）还是 LEARN（填补知识空白、系统了解）？ |
| **用户熟悉度** | 对此领域已有认知（Informed）还是相对陌生（Unfamiliar）？ |

四个象限及其含义：

```
                    EXPLORE (想法激发)        LEARN (知识补充)
                    ─────────────────        ────────────────
Informed            轻量讨论                   轻量知识
(已有认知)           📝为主，直接开聊            📝+少量🔍，快速汇总

Unfamiliar          深度讨论                   深度知识
(陌生领域)           🔍为主，先蒸馏再讨论         🔍为主，证据链完整
```

向用户展示推荐象限及理由，用户确认或覆写。

**判断不清时**，问一个问题：「你是想被不同观点挑战一下（讨论），还是想系统了解这个领域的多方观点（知识）？」不再多问。

## Phase 1: 维度先行选人

### Step 1.1: 扫描已有观点资产

按顺序扫描三个位置：

1. `.claude/skills/*-perspective/` → ⚡完整蒸馏
2. `multi-perspective/perspectives/` 各领域文件夹 → 🔍议题蒸馏
3. `.claude/skills/*-framework/` → 多人综合框架（可能已覆盖部分维度）

对每个匹配项提取：人名、领域标签、心智模型列表、最后更新日期。

### Step 1.2: 识别张力维度

**在选人之前**，先识别议题的 3-5 条**张力维度**——真实存在的分歧轴线。

每条维度用一句话描述两极。示例（议题：PTSD 治疗方法）：
- 维度 1：身体导向 ←→ 认知导向
- 维度 2：个体治疗 ←→ 系统/环境干预
- 维度 3：循证至上（RCT）←→ 临床智慧
- 维度 4：药物 ←→ 非药物

**为什么维度先行**：先选人再祈祷他们形成张力，经常失败。先画出分歧地图，再为每个位置找最佳代表——选人有据可查，盲区一目了然。

**每条维度预标注分歧类型**：在识别维度时，同时判断该维度的分歧可能属于哪种类型（实证性/规范性/定义性/适用性，见 `references/disagreement-types.md`）。这会影响后续选人和蒸馏策略：
- 实证性维度 → 优先选有数据/研究支撑的人物，蒸馏时重点采集证据
- 规范性维度 → 优先选有明确价值立场的人物，蒸馏时重点采集价值表达
- 定义性维度 → 需要在讨论开场时优先处理（统一定义）
- 适用性维度 → 需要选在不同场景有实践经验的人物

### Step 1.5: 维度确认检查点

向用户展示识别出的张力维度（含预标注的分歧类型），请用户确认：
- 这些维度是否覆盖了你关心的方面？
- 有没有你觉得重要但被遗漏的维度？
- 分歧类型的判断是否合理？

用户确认 → 进入选人。用户补充 → 调整维度后再选人。

**为什么这里要停一下**：维度地图是后面一切的地基——选谁、蒸馏什么、讨论往哪走都由它决定。地基歪了后面全白做。

### Step 1.6: 为每条维度匹配代表人物

匹配优先级：
1. 已有 ⚡ 完整蒸馏且与该维度相关 → 零成本，优先使用
2. 已有 🔍 议题蒸馏且与该维度相关 → 低成本
3. 在该维度有经典著作或知名立场的真实人物 → 需蒸馏
4. 至少一位「意外视角」——来自议题领域之外的人

### Step 1.7: 展示选人方案

```
## 议题维度与代表人物

议题：[主题]
模式：[象限名]（[理由]）

| 维度 | 代表人物 | 已有资产 | 推荐深度 | 理由 |
|------|---------|----------|---------|------|
| [维度1] | XX | ⚡完整Skill | 直接使用 | 已有深度蒸馏 |
| [维度2] | YY | 无 | 🔍议题蒸馏 | 核心维度代表，值得调研 |
| [维度3] | ZZ | 无 | 📝通用知识 | 补充视角 |
| 意外视角 | WW | 无 | 📝通用知识 | 跨领域启发 |

> 确认或修改。⚡零成本，🔍约3-5分钟调研。
```

**深度推荐逻辑**（替代旧的"核心vs辅助"主观判断）：
- 已有 ⚡ → 直接使用
- 已有 🔍 且相关 → 补充缺失维度
- 无资产 + 用户在 Deep 象限 + 此人代表核心维度 → 🔍
- 无资产 + 用户在 Light 象限 → 📝
- 意外视角 → 📝（价值在角度，不在深度）

## Phase 2: 蒸馏准备

### 2.1 按精度分流

| 精度 | 处理方式 |
|------|---------|
| ⚡已有完整Skill | 读取其 SKILL.md 中的心智模型、决策启发式、表达DNA、价值观 |
| 🔍需议题蒸馏 | 执行 3+1 Agent 轻量蒸馏（见下方） |
| 📝通用知识 | 不调研，发言/输出时标注精度 |
| 用户要求完整蒸馏 | 委托 nuwa-skill 执行（`~/.claude/skills/nuwa-skill/SKILL.md`） |

### 2.2 轻量蒸馏：3+1 Agent 流程

**前 3 个 Agent 并行采集**（聚焦本议题）：

| Agent | 搜索目标 | 提取重点 |
|-------|---------|---------|
| A 著作与观点 | 此人关于**本议题**的书、长文、演讲 | 核心论点、自创术语、反复出现的主张 |
| B 对话与辩论 | 此人关于**本议题**的访谈、播客、辩论 | 被追问时的回答方式、立场变化、即兴类比 |
| C 表达风格 | 此人的表达风格样本 | 句式偏好、幽默方式、确定性表达、高频用词 |

**第 4 Agent：综合提炼**（在前 3 个完成后执行）：

读取 A/B/C 的产出，使用 `~/.claude/skills/nuwa-skill/references/extraction-framework.md` 的三重验证方法：
- **跨域复现**：同一思维框架在 ≥2 个不同场景出现？
- **生成力**：能推断此人对新问题的立场？
- **排他性**：不是所有人都这么想？

三重通过 → 心智模型（2-3个）；仅 1-2 重 → 决策启发式；0 重 → 丢弃。

**每个 Agent 的硬性要求**：
- 只采集一手来源（本人著作、本人发言），不采集二手转述
- 注明信息来源 URL 和可信度
- 区分「他说过的」vs「别人说他的」vs「我推断的」
- 发现矛盾时保留矛盾，不和稀泥

**信息源黑名单**：知乎、微信公众号、百度百科永远排除。
**中国人物**：B站原始视频、小宇宙播客、权威媒体（36氪/晚点/财新/极客公园）优先。
**西方人物**：Twitter/X、YouTube、Podcast、Amazon书评优先。

### 2.3 模式感知后处理

同一份调研数据，按模式生成不同"卡片"：

**讨论模式 → 辩论卡**：
- 心智模型（名称 + 一句话 + 来源）
- 核心假设（此人从不质疑的前提）
- 挑战应对方式（被反驳时倾向于：让步/重构/反击/沉默？）
- 表达指纹（句式、幽默、确定性、高频词）
- 辩论倾向（先攻还是先守？用数据还是用类比？）

**讨论模式辩论卡操作指引**：
- 「挑战应对方式」→ 从 Agent B（对话与辩论）的采集中提取：此人被追问/反驳时的实际反应模式，而非推测
- 「辩论倾向」→ 从 Agent B 中统计：此人在辩论中先提出论点还是先回应对方？用数据/案例还是用类比/原则？
- 「核心假设」→ 从 Agent A（著作与观点）中提取：此人反复使用但从不论证的前提
- 如果 Agent B 数据不足以判断应对方式 → 标注「辩论风格：信息不足」，不编造

**知识模式 → 证据卡**：
- 心智模型（名称 + 一句话 + 来源）
- 核心主张（每条附来源 URL）
- 证据强度（强：多个一手来源佐证 / 中：单一来源 / 弱：推测）
- 分歧类型标签（实证性/规范性/定义性/适用性，见 `references/disagreement-types.md`）
- 此人证据基础的强项与弱项

**知识模式证据卡操作指引**：
- 「证据强度」判断标准：强 = ≥2 条独立一手来源佐证同一主张；中 = 1 条一手来源；弱 = 仅训练数据推测或二手转述
- 「分歧类型」→ 继承 Phase 1.2 维度预标注的类型，在证据卡中细化到具体主张层面
- 「强项与弱项」→ 此人的证据基础在哪些子话题上有一手数据支撑（强项），在哪些子话题上只有理论推演（弱项）

### 2.5 蒸馏确认检查点

所有蒸馏完成后，展示摘要供用户确认：

```
┌─────────────────────────────────────────────────────┐
│ 蒸馏完成                                              │
├──────┬──────┬──────────────┬────────────────────────┤
│ 人物  │ 精度 │ 心智模型      │ 一手来源数              │
├──────┼──────┼──────────────┼────────────────────────┤
│ XX   │ ⚡   │ 5个          │ 完整Skill              │
│ YY   │ 🔍   │ 2个          │ N条一手                │
│ ZZ   │ 📝   │ —           │ —                      │
├──────┴──────┴──────────────┴────────────────────────┤
│ 矛盾点：N处 │ 信息不足维度：[列出]                     │
└─────────────────────────────────────────────────────┘
```

用户确认 → Phase 3。用户不满意 → 调整后重新蒸馏。

### Agent 超时与失败处理

- 单个 Agent 超时（5分钟无结果）→ 继续推进，标注「信息不足」
- 信息源匮乏（<5条来源）→ 提醒用户，心智模型减至 1-2 个
- Agent 结果冲突 → 保留矛盾，在检查点展示

**铁律**：宁可诚实说「信息有限」，也不要编造深度。

## Phase 3A: 讨论模式

**委托 `~/.claude/skills/ljg-roundtable/SKILL.md` 执行圆桌讨论。**

继承 ljg-roundtable 的以下核心机制（不重写）：
- **统一定义**：开场前主持人确认各方对核心概念的理解，消除定义性分歧
- **「简言之」压缩**：每位发言者用一句话浓缩立场，防止冗长
- **ASCII 框架图**：主持人综述时用可视化展示观点关系
- **用户指令菜单**：`可/止/深入此节/引入新人物`
- **对话循环**：动态发言 → 主持人综述 → 用户指令 → 下一轮

本 skill 在此基础上提供以下增强输入：

### 增强 1：蒸馏驱动的发言规则

- ⚡ 人物：发言必须基于其 SKILL.md 中的心智模型，引用具体模型名称
- 🔍 人物：发言必须基于蒸馏档案中的心智模型和来源证据
- 📝 人物：基于训练数据发言，不得编造具体来源

发言格式增加精度标记：`【人物名】⚡/🔍/📝【行动标签】：发言内容`

### 增强 2：张力维度地图

将 Phase 1.2 识别的维度地图传给主持人，作为引导讨论方向的参考。主持人综述时可指出「心智模型级别的冲突」：
> 「XX的[模型A]与YY的[模型B]在此处产生了根本性张力——前者假设...，后者假设...」

### 增强 3：连贯性协议

解决旧版"谁该发言"无算法的问题：

1. **张力评分发言**：每轮后，主持人评估每位参会者的"未解决张力"——其核心模型与刚才发言的冲突程度。张力最高者优先发言。刚发言的人张力低（防止垄断），所有人张力都低时话题已穷尽（推进下一问题）。
2. **循环检测**：同两人来回 2+ 轮无新信息 → 主持人必须：(a) 引入第三声音，(b) 升级到假设层面，或 (c) 宣布此线索穷尽。
3. **沉默即信号**：某人的模型对当前子话题无发言权 → 保持沉默，主持人标注此盲区。

### 增强 4：额外用户指令

在 ljg-roundtable 的 `可/止/深入此节/引入新人物` 基础上增加：
- `蒸馏此人`：将 📝 参会者升级为 🔍。两种执行方式：
  - **前台蒸馏**（默认）：暂停讨论 → 执行 3+1 Agent 蒸馏 → 恢复讨论，蒸馏结果立即生效
  - **后台蒸馏**（用户说「后台蒸馏此人」）：讨论继续进行，该人物暂时仍以 📝 精度发言。蒸馏完成后通知用户，从下一轮开始以 🔍 精度发言。适合不想打断讨论节奏的场景

## Phase 3B: 知识模式

蒸馏完成后，综合所有人物视角，输出结构化知识报告：

```markdown
## 议题：[主题]

### 维度地图
[ASCII 图：展示张力维度及每位人物的位置]

### 参与视角
| 人物 | 精度 | 所在维度 | 核心立场（一句话） |
|------|------|---------|-------------------|

### 共识区
多数视角认同的观点：
1. **[观点]** — 证据强度：[强/中/弱] — 持有者：[names]

### 分歧区
| 争议 | 分歧类型 | 立场A | 立场B | 分歧根源 |
|------|---------|-------|-------|---------|
| ... | 实证性 | XX: ...(来源) | YY: ...(来源) | 不同数据/方法论 |
| ... | 规范性 | XX: ... | YY: ... | 不同价值优先级 |
| ... | 定义性 | XX: ... | YY: ... | 对核心概念理解不同 |
| ... | 适用性 | XX: ... | YY: ... | 适用范围/边界条件不同 |

**分歧根源分析**：
- 实证性分歧 → 可通过更多数据解决
- 规范性分歧 → 需要你自己做价值选择
- 定义性分歧 → 需先统一定义再讨论
- 适用性分歧 → 需明确具体场景的边界条件

### 盲区
- [哪些维度没有人物覆盖]
- [哪些问题浮现但无人能答]

### 行动建议
1. **[建议]** — 依据：[共识/特定视角] — 注意：[哪个视角有不同看法]

### 来源透明度
| 人物 | 精度 | 一手来源数 | 关键来源 |
|------|------|-----------|---------|
```

**输出原则**：
- 共识区只列真正有多方支持的核心观点，不面面俱到
- 分歧区追到根源（心智模型/假设层面），标注分歧类型
- 盲区诚实——如果选的人物都偏向某一侧，主动指出
- 📝 级别观点标注为「弱」证据强度

**模式切换入口**：

知识报告输出后，用户可以：
- `就此讨论 [分歧点编号]`：以该分歧点为焦点，转入讨论模式。自动继承当前参会者和蒸馏结果，委托 ljg-roundtable 执行，聚焦该分歧点展开辩证对话
- `补充 [人物/维度]`：在知识模式内追加蒸馏或新增维度，更新报告
- 讨论过程中产生的新知识需求 → 可随时说「补充这个点」，暂停讨论执行定向调研后恢复

## Phase 4: 归档与积累

### 4.1 蒸馏结果归档

对每位 🔍 蒸馏的人物：

1. 判断领域归属 → 写入 `perspectives/[领域]/[人名].md`
2. **合并逻辑**（同一人物已有档案时）：
   - 新心智模型与已有模型匹配（同名或同逻辑）→ 合并证据（追加来源和应用场景）
   - 新心智模型是全新的 → 追加，标注产生它的议题
   - 新模型与已有模型矛盾 → 记录为「时间性张力」或「领域性张力」
   - **永远追加不覆盖**，每条带日期戳
3. 更新 `perspectives/INDEX.md`

**严格禁止归档**：
- 圆桌对话中的发言（AI 角色扮演生成）
- 主持人综述和分析（AI 综合推断）
- 📝 通用知识人物的任何内容

### 4.2 会话记录存档

1. `date +%Y%m%dT%H%M%S` 获取时间戳
2. 写入 `sessions/{timestamp}--{议题关键词}/session.org`
3. org 文件结构：
```org
#+title: 多视角：{议题}
#+date: [{日期}]
#+filetags: :multi-perspective:
#+mode: [讨论/知识]

* 议题与维度地图
* 参会者（含精度标记）
* [讨论模式] 各轮讨论记录 / [知识模式] 结构化报告
* 开放问题
```

### 4.3 偏好追踪

每次会话结束后，向 `preferences.md` 追加一行：

```
[日期] | [议题] | [模式] | [使用的人物] | [深挖的维度] | [用户反馈信号]
```

反馈信号来源：用户使用「深入此节」的维度、追问的方向、明确表达的偏好。

### 4.4 升级提示

每次会话开始扫描 `perspectives/INDEX.md`：
- 某人物累计被 🔍 蒸馏 ≥3 次（跨不同议题）→ 提示：「XX 已在 N 个议题中被蒸馏，累计 M 个心智模型。要不要用女娲做一次完整蒸馏（⚡）？」
- 用户说「升级此人」→ 委托 nuwa-skill 执行完整蒸馏，已有 🔍 数据作为起点

### 4.5 资料增强入口

支持两种方式增强已有人物的观点档案：

**自动检测**：当用户在其他对话中处理了与某已有人物相关的素材时，下次使用 multi-perspective 时提示：「检测到你最近阅读了 XX 的新素材，要不要用它更新 XX 的观点档案？」

**手动指令**：用户说「增强 XX」或「给 XX 补充资料」→ 引导用户提供素材（PDF、transcript、文章链接等），执行定向蒸馏补充到已有档案。

### 4.6 观点库清理与分类

**领域分类**：每位人物归档时必须标注领域标签，`perspectives/` 按领域建子文件夹（如 `perspectives/neuroscience/`、`perspectives/nutrition/`）。INDEX.md 按领域分组展示，方便后续导出到外部管理工具。

**冷却标记**：每次会话开始扫描 `perspectives/INDEX.md` 时，同时检查每位 🔍 人物的最后使用日期：
- 最后使用 ≥6 个月 → 在 INDEX.md 中标注 `🧊冷却`
- 冷却人物不会被自动推荐为维度代表（但用户手动指定仍可使用）
- 用户说「清理观点库」→ 列出所有冷却人物，用户逐个决定：保留 / 归档到 `perspectives/_archive/` / 删除

**INDEX.md 格式**：

```markdown
## [领域名]
| 人物 | 精度 | 心智模型数 | 使用次数 | 最后使用 | 状态 |
|------|------|-----------|---------|---------|------|
| XX   | 🔍   | 3         | 5       | 2026-04 | 活跃 |
| YY   | 🔍   | 1         | 1       | 2025-09 | 🧊冷却 |
```

## 三级精度分类

| 级别 | 含义 | 归档位置 | 可信度 |
|------|------|----------|--------|
| ⚡完整蒸馏 | 经过完整女娲流程 | `.claude/skills/[人名]-perspective/` | 最高 |
| 🔍议题蒸馏 | 3+1 Agent 聚焦调研 | `multi-perspective/perspectives/[领域]/` | 中等 |
| 📝通用知识 | 无调研，仅训练数据 | 不归档 | 最低 |

**铁律**：🔍 和 📝 的内容永远不能混入 ⚡ 级别的 Skill 中。升级路径是单向的：📝 → 🔍 → ⚡。

## 信息源规则

**优先级**：用户提供的一手素材 > 本人著作 > 长对话/访谈 > 社交媒体 > 他人评价 > 二手转述

**黑名单**：知乎、微信公众号、百度百科永远排除。

**中国人物**：B站原始视频、小宇宙播客、权威媒体优先。
**西方人物**：Twitter/X、YouTube、Podcast、Amazon书评优先。
