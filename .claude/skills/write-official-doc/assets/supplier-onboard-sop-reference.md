## 各阶段审计检查点

### 阶段 1：需求发起
- 《需求清单》是否存档
- 需求是否有数据支撑
- 是否有策略岗负责人签字/确认

### 阶段 2：需求拆解
- 《寻源需求表》是否存档
- 是否有现有供应商评估记录
- 外部寻源决策是否有依据

### 阶段 3：寻源触达
- 寻源邮件是否存档（简单版/详细版）
- 回复供应商名单是否记录
- 筛选标准是否客观、可追溯
- 验收标准：≥3 家有效回复，有明确意向

### 阶段 4：方案审批
- xbp 方案是否存档
- 审批流程是否完整
- 领导审批意见是否记录
- 补充材料是否完整

### 阶段 5：结果同步
- 《结果同步邮件》是否存档
- 发送名单是否完整（策略岗、业管、供应商）
- 供应商通知是否规范（入选/未入选分别发）

### 阶段 6：供应商准备
- CID 信息是否存档
- 站点报备是否有记录
- 合同模板是否确认并存档
- 资质文件是否完整

### 阶段 7：内部操作
- 京音系统是否有供应商档案
- 账号开通记录是否存档
- 权限配置是否符合规范
- 是否有账号测试记录

### 阶段 8：培训上线
- 培训记录是否存档
- 培训考核结果是否记录
- 上线外呼时间是否记录
- 首日质量报告是否存档

## 供应商评估维度（xbp方案用）

| 维度 | 评分(1-5) | 评估要点 |
|------|----------|---------|
| 资质合规 | X | 注册资金、成立时间、无不良记录 |
| 项目经验 | X | 电销/呼叫中心运营案例 |
| 招聘能力 | X | 可投入人力规模、招聘渠道 |
| 培训体系 | X | 培训流程、质检培训能力 |
| 质量管理 | X | 质检体系、合规管理 |
| 配合意愿 | X | 响应及时度、合作态度 |
| **综合评分** | **X/30** | |

## 集中度检查

引入新供应商时必须检查：
- 当前 TOP3 供应商占比
- 引入后 TOP3 供应商占比
- 是否符合集中度要求

---

## CID 信息字段

公司全称、简称、统一社会信用代码、注册地址、实际经营地址、法人代表、法人身份证号、联系人（姓名/职务/电话/邮箱）、公司官网、成立时间、注册资金、员工规模、主营业务、现有合作甲方。

## 站点报备字段

站点编号、名称、省份、城市、详细地址、职场面积、最大坐席数、当前使用坐席数、站点负责人、启用时间、职场类型（自建/租赁）、电力保障、网络运营商、备用网络。

## 使用指南

**写引入相关公文时**：
1. 引用本文件获取流程阶段、时效标准、审计检查点
2. 配合 `supplier-onboard.md`（引入上线邮件模板）和 `supplier-reserve.md`（寻源储备模板）使用
3. 涉及寻源邮件时参考阶段3的要求（≥3家有效回复）
4. 涉及审批方案时参考评估维度表和集中度检查
5. 涉及通知时区分入选/未入选两种模板
