# 分量调整计算公式

> **何时读取**：需要计算减量幅度、绩效排名、集中度时


## 核心公式

### 1. 减量幅度计算

```
减量幅度 = (减前人数 - 减后人数) / 减前人数 × 100%
```

**示例**：
- 减前：94 人
- 减后：62 人
- 减量幅度 = (94-62)/94 × 100% = 34%


### 2. 按绩效排名分配减量

**分层减量比例**：

| 层级 | 排名 | 减量比例 | 说明 |
|------|------|----------|------|
| 头部 | 前 30% | 10-20% | 断崖缓冲，保留优质资源 |
| 腰部 | 中间 40% | 30-40% | 适度减量 |
| 尾部 | 后 30% | 50-60% | 优先减量，淘汰低效 |

**计算公式**：
```
某供应商减量 = 该供应商排名 × 对应层级减量比例
```


### 3. 绩效排名计算

**综合绩效** = 定量指标（70%）+ 定性指标（30%）

**定量指标**（70%）：
- GMV 完成率（30%）
- 转化率（25%）
- 名单产能（15%）

**定性指标**（30%）：
- 质检得分（15%）
- 合规记录（10%）
- 配合度（5%）

```
综合得分 = Σ(各指标得分 × 权重)
排名 = 按综合得分降序排列
```


### 4. 集中度检查

**AI 场景**（按供应商数量动态）：

| 供应商数量 | 单一供应商上限 |
|------------|---------------|
| ≤3 家 | ≤70% |
| 4 家 | ≤60% |
| ≥5 家 | ≤50% |

**人工场景**（人数上限 + 占比上限，取孰低）：

| 业务线 | 人数上限 | 占比上限 |
|--------|---------|---------|
| 金条人工 | 360 人 | 30% |
| 金条人机 | 120 人 | 40% |
| 信用卡 | 220 人 | 40% |
| 财富 | 100 人 | 40% |

**检查公式**：
```
集中度 = 单一供应商人数 / 总人数 × 100%

要求：集中度 ≤ 上限
```


### 5. 保护机制检查

**新职场保护**：
- 测试期内职场（核心业务 3-6 个月）不参与减量
- 合作<3 个月的职场不参与减量
- 保护期结束后首个周期：减量幅度≤20%（断崖缓冲）

**检查公式**：
```
合作月数 < 3 → 不参与减量
3 ≤ 合作月数 < 6 → 减量≤20%
合作月数 ≥ 6 → 正常减量
```

## 快速计算工具

### Python 脚本

```python
# 减量计算
def calculate_reduction(current_headcount, reduction_rate):
    return int(current_headcount * reduction_rate)

# 集中度检查
def check_concentration(headcount, total, limit):
    ratio = headcount / total * 100
    return ratio <= limit, f"{ratio:.1f}%"
```
