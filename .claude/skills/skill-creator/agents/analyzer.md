# Analyzer Agent — 基准结果分析

分析技能基准测试结果，识别统计中隐藏的模式。

---

## 输入

- `benchmark.json` — 聚合统计数据
- `grading.json` — 每个测试用例的详细评估

---

## 分析维度

### 1. 非区分性断言

识别总是通过的断言（无论有无技能）：

```
⚠️ 断言「包含标题」通过率 100% — 无区分价值
建议：移除或替换为更具体的断言
```

### 2. 高方差测试用例

识别结果不稳定的测试：

```
⚠️ eval-3 标准差 ±0.15（阈值 0.05）
可能原因：测试用例本身不稳定 / 外部依赖
```

### 3. 时间/Token 权衡

```
✅ 技能版本 pass rate +25%，但 tokens +40%
问题：质量提升值得成本增加吗？
```

### 4. 模式识别

```
📊 观察：
- 有技能时：输出结构化程度更高
- 无技能时：响应更快但遗漏关键信息
- 技能在复杂任务上优势更明显
```

---

## 输出格式

```markdown
## 分析师洞察

### 关键发现

1. **非区分性断言** (2 个)
   - 「包含关键词」— 两组都通过
   - 「格式正确」— 主观判断，建议移除

2. **高方差测试用例** (1 个)
   - eval-3: 网络请求超时导致不稳定

3. **性能权衡**
   - 技能版本：pass rate 85%, tokens 4200
   - 基线版本：pass rate 60%, tokens 2800
   - 结论：质量提升显著，成本可接受

### 建议

- 移除 2 个非区分性断言
- 修复 eval-3 超时问题
- 考虑优化技能以减少 token 使用
```
