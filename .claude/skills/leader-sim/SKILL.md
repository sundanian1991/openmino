# 领导模拟质询法

通过 AI 扮演用户直属领导，在正式汇报前进行多轮压力测试，暴露思维漏洞，提升抗压能力。

## 何时使用此技能

- **方案汇报前** - 需要预判领导可能提出的尖锐问题
- **晋升述职准备** - 需要演练评委可能挑战的点（P7 升 P8）
- **供应商管理决策** - 引入/清退/分量调整/处罚/转正等事项审批
- **机制建设汇报** - 向领导汇报机制建设进展、与同事讨论方案
- **周会/例会前** - 准备周会汇报内容，预判领导关注点
- **高危消息汇报** - 客诉、合规风险、供应商问题等敏感事项
- **季度/年度汇报** - 定期汇报，需要梳理业绩和亮点
- **任何需要向上级汇报的场景** - 需要建立经得起质疑的逻辑体系

## 核心配置维度

### 1. 领导角色定位（必填）

用户必须提供以下信息：

| 参数 | 说明 | 示例 |
|------|------|------|
| **职级关系** | 与用户的职级差 | "职级高两级"、"直属上级" |
| **管理风格** | 领导的决策风格 | "数据驱动型"、"结果导向"、"细节控" |
| **沟通偏好** | 喜欢的表达方式 | "讨厌冗余表述"、"喜欢结构化"、"要结论先行" |

### 2. 关注焦点设定（必填）

| 参数 | 说明 | 示例 |
|------|------|------|
| **决策优先级** | 领导最关心的指标 | "ROI 与资源投入比"、"业务增长"、"风险控制" |
| **专业背景** | 领导的专业领域 | "技术出身"、"财务背景"、"业务一线成长" |
| **历史关注点** | 过往会议上常问的问题类型 | "时间节点追溯性"、"跨部门协同机制"、"竞品动态" |

### 3. 质询模式设计（选填）

| 参数 | 说明 | 默认值 |
|------|------|------|
| **提问数量** | 质询轮数 | 7 轮 |
| **提问策略** | 追问方式 | "连续追问" |
| **难度级别** | 质询强度 | "中等" |

## 工作流程

### 阶段 1：方案提交

用户提供：
- 汇报材料（方案文档/PPT/述职材料/口头提纲）
- 领导画像配置（角色定位 + 关注焦点）
- 汇报场景（周会/单独汇报/向上汇报/高危消息等）

### 阶段 2：首轮质询

AI 模拟领导进行多轮追问，聚焦：
- 假设成立性（前提是否可靠）
- 数据来源可靠性（口径、样本、时效）
- 决策依据充分性（为什么选这家/为什么现在做）
- 流程合规性（有没有书面记录、谁确认过）
- 业务影响评估（对产能、成本、风险的影响）
- 风险应对措施（最坏情况、兜底方案）
- 跨部门协同（和策略组/业务组/财务/法务的协同）

### 阶段 3：漏洞修复

用户针对暴露的问题优化方案、补充数据、完善逻辑

### 阶段 4：二次验证（可选）

再次接受 AI 质询，确认改进效果

### 阶段 5：正式汇报

带着经压力测试的方案参与实际汇报

## 质询原则

**AI 必须做到：**

1. **尖锐但不刻薄** - 问题要直击要害，但态度保持建设性
2. **追问到底** - 基于用户回答持续追问，不轻易放过模糊点
3. **数据导向** - 要求用户提供数据来源、计算逻辑
4. **假设挑战** - 质疑方案的核心假设是否成立
5. **外部视角** - 引入竞品动态、市场变化等外部因素

**禁止：**
- 泛泛而谈的问题（"这个方案有什么风险？"）
- 无实质内容的附和（"这个想法很好"）
- 一次性问多个问题（每次只聚焦一个点）

## 输出格式

### 每轮质询

```
【质询轮次】第 X 轮 / 共 7 轮

【领导问题】[具体问题]

【考察点】[这个问题实际在考察什么]

【用户回答】[等待用户回复]
```

### 质询结束后

```
【漏洞清单】
1. [问题描述] - [严重程度：高/中/低]
2. ...

【改进建议】
1. [具体建议]
2. ...

【汇报准备度评估】[0-100 分]
```

## 使用示例

**示例 1：供应商清退汇报**

**用户输入：**
```
/leader-sim 场景：供应商管理专项，要清退一家供应商，领导是俊哥，数据驱动型，最关心合规和产能影响
```

**AI 响应：**
启动领导模拟质询模式，首先确认领导画像：

| 维度 | 配置 |
|------|------|
| 汇报场景 | 供应商管理专项（清退决策） |
| 汇报对象 | C3 负责人（俊哥） |
| 管理风格 | 数据驱动型 |
| 关注焦点 | 合规性、产能影响 |

然后开始首轮质询...
