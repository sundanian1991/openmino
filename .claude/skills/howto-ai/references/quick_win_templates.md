# Quick Win 模板库

按职业角色组织的开箱即用 AI 方案模板。使用时根据用户的实际职业选择最接近的模板进行定制。

## 通用办公效率

### 模板：邮件/报告草拟
- **场景**：每天写大量格式相近的邮件、周报、汇报
- **工具**：ChatGPT / Claude / Kimi / 豆包
- **Prompt 模板**：
  ```
  你是我的[职位/角色]助手。请根据以下信息帮我写一份[邮件/周报/汇报]：

  1. 背景：[一句话背景]
  2. 关键数据/进展：[列出3-5点]
  3. 下一步计划：[列出2-3点]
  4. 需要支持/决策：[如有]
  5. 语气风格：[正式/半正式/轻松]
  6. 字数限制：[如 300 字以内]

  请用简洁清晰的段落结构，避免套话。
  ```

### 模板：会议纪要整理
- **场景**：每次会后手动整理纪要
- **工具**：飞书妙记 / 通义听悟 / Otter.ai
- **Prompt 模板**：
  ```
  请将以下会议录音转文字内容整理为结构化纪要：

  [粘贴转写文本]

  请按以下结构输出：
  - 会议主题：
  - 参会人员：
  - 核心结论（3条以内）：
  - 待办事项（负责人 + 截止日期格式）：
  - 未解决的问题：
  ```

## 人力资源 (HR)

### 模板：简历筛选
- **场景**：从大量简历中筛选特定条件的候选人
- **工具**：ChatGPT / Claude（上传文件功能）
- **Prompt 模板**：
  ```
  请帮我从以下简历中筛选符合条件的候选人。

  筛选条件：
  - [条件1，如：有海外工作/学习背景]
  - [条件2，如：3年以上XX行业经验]
  - [条件3，如：英语可作为工作语言]

  对每位符合条件的候选人，请输出：
  1. 姓名
  2. 匹配条件说明（逐条对应）
  3. 亮点（1-2句话）
  4. 风险点（如有）

  [粘贴简历内容或上传PDF]
  ```

### 模板：岗位描述 (JD) 撰写
- **场景**：每次招人重写 JD，耗时且风格不一致
- **工具**：ChatGPT / Claude
- **Prompt 模板**：
  ```
  请帮我写一份[岗位名称]的招聘 JD。

  基本信息：
  - 公司/团队背景：[1句话]
  - 岗位职责：[列出3-5条]
  - 硬性要求：[列出2-3条]
  - 加分项：[列出2-3条]
  - 薪资范围：[如有]
  - 我们想要的人格特质：[如：自驱、细节控]

  要求：
  - 语言风格：[轻松互联网风 / 专业正式 / 极客风]
  - 总字数不超过 500 字
  - 在开头用一句话说明"为什么你应该加入我们"
  ```

## 内容创作 / 新媒体

### 模板：公众号/小红书文章撰写
- **场景**：每周固定内容产出，灵感枯竭时启动困难
- **工具**：Claude（长文）/ 秘塔写作猫
- **Prompt 模板**：
  ```
  请帮我写一篇关于[主题]的[公众号/小红书]文章。

  要求：
  - 目标读者：[如"30岁左右在职女性，关注个人成长"]
  - 文章目的：[如"提供可操作的时间管理技巧"]
  - 字数：[如 1500-2000 字]
  - 风格：口语化、有观点、避免 AI 腔。用具体数据和场景代替抽象描述。
  - 结构建议：痛点引入 → 方法论（3-4 点，每点配案例）→ 行动清单

  请先给出 3 个写作角度供我选择，选定后展开全文。
  ```

### 模板：多平台内容改编
- **场景**：一篇文章要适配公众号、小红书、知乎等不同平台
- **工具**：ChatGPT / Claude
- **Prompt 模板**：
  ```
  请将以下文章内容改编为[目标平台，如：小红书]风格：

  [粘贴原文]

  平台风格要求：
  - 小红书：短段落、多 emoji、强互动感、话题标签
  - 知乎：深度分析、数据支撑、观点鲜明
  - 朋友圈：一句话金句 + 简短感悟，不超过 200 字
  ```

## 数据分析

### 模板：自然语言查询数据库
- **场景**：业务方频繁提数据需求，但每次都要写 SQL
- **工具**：ChatGPT Code Interpreter / Claude（上传 CSV）+ Text-to-SQL
- **Prompt 模板**：
  ```
  我有以下数据表结构：

  表名：[表名]
  字段：
  - [字段1]：[类型]，含义：[说明]
  - [字段2]：[类型]，含义：[说明]
  - ...

  请帮我写 SQL 查询：[自然语言描述需求]
  数据库类型：[MySQL / PostgreSQL / Hive]

  要求：考虑性能优化，避免全表扫描。
  ```

### 模板：数据报告自动化生成
- **场景**：每周/每月固定输出数据报告，格式重复
- **工具**：ChatGPT Code Interpreter / Claude + CSV 上传
- **Prompt 模板**：
  ```
  请分析我上传的数据，生成一份[周报/月报]的核心内容。

  重点关注：
  1. [指标1]的变化趋势及原因分析
  2. [指标2]的异常波动（如有）
  3. 与上月/上周的对比

  输出格式：
  - 一句话总结（50字以内）
  - 3 个关键发现（每个配数据说明）
  - 2-3 条行动建议
  ```

## 运营 / 客服

### 模板：智能客服知识库搭建
- **场景**：每天回复相同类型问题
- **工具**：Coze / Dify（无代码平台）
- **操作步骤**：
  1. 整理最常见的 20-30 个问题及标准回答
  2. 在 Coze 创建 Bot → 配置知识库（上传 Q&A 文档）
  3. 设置触发规则（关键词匹配 → 自动回复）
  4. 复杂问题转人工
- **预期效果**：减少 60-80% 重复咨询工作量

### 模板：差评/投诉回复
- **场景**：处理客户差评需要斟酌措辞
- **工具**：ChatGPT / Claude
- **Prompt 模板**：
  ```
  我们收到了以下客户投诉，请帮我起草回复：

  投诉内容：
  [粘贴投诉原文]

  背景：
  - 产品/服务：[说明]
  - 公司态度：[如"愿意赔偿但不承认错误" / "真诚道歉并改进"]

  请提供 2 个版本的回复：
  - 版本 A：安抚为主（适合情绪化的客户）
  - 版本 B：解决为主（适合理性的客户）

  要求：真诚、不推卸责任、提供具体解决方案。
  ```

## 项目管理

### 模板：需求文档 (PRD) 初稿生成
- **场景**：每次写 PRD 重复结构，格式不统一
- **工具**：Claude / Notion AI
- **Prompt 模板**：
  ```
  请帮我生成一份[功能/项目名称]的产品需求文档初稿。

  核心信息：
  - 要解决的问题：[1句话]
  - 目标用户：[描述]
  - 核心功能：[列出3-5点]
  - 技术约束：[如有]
  - 优先级：[P0/P1/P2]

  请按以下结构输出：
  1. 背景与目标
  2. 用户故事
  3. 功能详情（含交互描述）
  4. 非功能需求
  5. 验收标准
  ```

## 翻译与处理文本

### 模板：中英双语内容制作
- **场景**：需要产出中英双语版本的内容
- **工具**：DeepL + ChatGPT / Claude 润色
- **Prompt 模板**：
  ```
  请将以下中文翻译为英文，要求：
  1. 先直译，再润色为 native speaker 水准
  2. 保留原文的语气和风格
  3. 专业术语保持一致
  4. 标注任何需要文化适配的表述

  [粘贴中文原文]
  ```
