# 汇报类型详解（Report Types）

> 不同汇报场景，领导关注点不同，质询重点不同

---

## 1. 向上汇报

**场景**：向部门负责人/C2 汇报，决策导向

**典型主题**：
- 资源申请（HC、预算）
- 方案审批（供应商管理、系统升级）
- 战略对齐（年度目标、季度规划）

**领导关注点**：
1. ROI — 投入产出比是多少？多久收回成本？
2. 战略对齐 — 如何支撑部门年度目标？
3. 风险预案 — 最大风险是什么？如何应对？
4. 数据支撑 — 数据来源？样本量？统计周期？

**必备要素**：
- 结论先行（SCQA 开场）
- ROI 测算（投入、产出、回收周期）
- 风险矩阵（概率×影响）
- 对标分析（行业/竞品）

**常见挑战**：
- "这个假设的底层逻辑是什么？"
- "有没有更轻量的方案？"
- "如果 X 不成立，Plan B 是什么？"

**质询强度**：⭐⭐⭐⭐⭐（尖锐）

---

## 2. 周会/例会

**场景**：常规工作汇报，效率优先

**典型主题**：
- 进度同步（本周完成、下周计划）
- 数据同步（KPI 达成、异常说明）
- 问题协调（需要支持的事项）

**领导关注点**：
1. 进度 — 相比计划是提前还是滞后？
2. 异常 — 影响范围多大？是否可控？
3. 下一步 — 下周关键里程碑是什么？

**必备要素**：
- 进度条（计划 vs 实际）
- 异常说明（根因、影响、措施）
- 下一步（责任人、时间点）

**常见挑战**：
- "这个异常的根本原因找到了吗？"
- "滞后了怎么追回来？"
- "需要我协调什么？"

**质询强度**：⭐⭐（引导式）

---

## 3. 单独事项汇报

**场景**：专项审批，深度讨论

**典型主题**：
- 供应商清退
- 合规风险处理
- 跨部门协同方案

**领导关注点**：
1. 方案细节 — 具体步骤、时间节点
2. 替代方案 — 有没有其他选择？为什么选这个？
3. 成本 — 人力、时间、机会成本

**必备要素**：
- 背景说明（5W2H）
- 方案对比（A/B 方案优劣）
- 推荐方案及理由

**常见挑战**：
- "为什么选 A 不选 B？"
- "这个方案的边界条件是什么？"
- "最坏情况想过没有？"

**质询强度**：⭐⭐⭐⭐（挑战式）

---

## 4. 临时汇报

**场景**：突发事件，快速决策

**典型主题**：
- 业务异常（数据暴跌、系统故障）
- 客户投诉（重大客诉、舆情）
- 供应商问题（断供、涨价）

**领导关注点**：
1. 事件描述 — 什么时候发生？影响多少人？
2. 应急措施 — 现在做了什么处理？
3. 需要支持 — 需要我协调什么资源？

**必备要素**：
- 5W2H 描述（What/When/Where/Who/Why/How/How much）
- 应急措施（已执行、执行中、计划执行）
- 资源需求（具体、可执行）

**常见挑战**：
- "什么时候发现的？为什么现在才报？"
- "现在控制住了没有？"
- "还会不会扩大？"

**质询强度**：⭐⭐⭐⭐（压力式）

---

## 5. 高危消息汇报 ⚠️

**场景**：负面信息，风险传导

**典型主题**：
- 供应商清退（核心供应商）
- 合规风险（监管处罚、法律纠纷）
- 业务中断（系统宕机、数据丢失）
- 重大客诉（群体性投诉、媒体曝光）

**领导关注点**：
1. 根因分析 — 根本原因？偶发还是系统性？
2. 止损措施 — 现在控制了没有？还会不会扩大？
3. 问责处理 — 谁的责任？怎么处理？
4. 预防机制 — 怎么保证不再发生？

**必备要素**：
- 根因分析（5 Why）
- 止损措施（已控制、待控制）
- 问责建议（责任人、处理方式）
- 预防机制（流程、系统、培训）

**话术框架**：SCQA + 三明治
```
S（情境）：正常情况下...
C（冲突）：但今天发生了 X...
Q（问题）：核心问题是...
A（答案）：我们的处理方案是...

然后：
- 先说结论（坏消息 + 已控制）
- 再说根因和处理
- 最后说预防措施
```

**常见挑战**：
- "这么重要的事，为什么现在才知道？"
- "这个供应商当初是谁选的？"
- "怎么保证下次不发生？"

**质询强度**：⭐⭐⭐⭐⭐（压力式，追问到底）

---

## 6. 跨部门协同汇报

**场景**：多方配合，责任划分

**典型主题**：
- 系统对接（多部门参与）
- 流程优化（跨部门流程）
- 资源协调（共享资源分配）

**领导关注点**：
1. RACI — 谁负责、谁审批、谁咨询、谁知会
2. 协同机制 — 怎么对齐进度？冲突谁拍板？
3. 冲突处理 — 如果对方不配合怎么办？

**必备要素**：
- RACI 矩阵
- 协同机制（例会、同步频率）
- 冲突升级路径（谁最终拍板）

**常见挑战**：
- "如果对方不配合，Plan B 是什么？"
- "这个责任划清楚没有？"
- "时间节点对齐了吗？"

**质询强度**：⭐⭐⭐（挑战式）

---

## 7. PRD/方案评审

**场景**：专业评审，挑漏洞

**典型主题**：
- 产品需求文档
- 技术方案设计
- 运营活动方案

**领导关注点**：
1. 逻辑严密性 — 边界条件、异常处理
2. 用户体验 — 使用路径、异常情况
3. 技术可行性 — 实现难度、排期

**必备要素**：
- 流程图（正常流程 + 异常流程）
- 边界条件（最大值、最小值、极限情况）
- 异常处理（报错、回滚、降级）

**常见挑战**：
- "如果用户点了两次怎么办？"
- "这个接口的超时时间是多少？"
- "并发量大时会不会崩？"

**质询强度**：⭐⭐⭐⭐（挑战式）
