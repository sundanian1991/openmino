## 高危消息汇报框架

### 三明治结构

```
第一层（缓冲）：
"领导，有个重要情况跟您汇报一下..."

第二层（核心）：
"X 供应商因为 Y 原因，需要清退。
我们已经控制了 Z 影响，业务不会中断。"

第三层（方案）：
"具体处理方案是...
根因是...
问责建议...
预防措施..."
```

### 5W2H 描述

```
What（什么事）：核心供应商 A 要清退
When（什么时候）：今天发现，已沟通 3 轮
Where（哪儿出的问题）：合规审查发现 X 问题
Who（谁负责）：供应商负责人是 X，我方对接人是 Y
Why（为什么）：违规转包，触碰红线
How（怎么处理）：启动备选供应商 B，本周切换
How much（多大影响）：影响 5% 订单，可控
```

## 资源申请话术

### ROI 先行

```
领导，申请 2 个 HC，一年成本 60 万。
预期收益：
1. 效率提升 30%，节省人力成本 40 万/年
2. 业务增长 15%，新增收益 100 万/年
3. 风险降低，减少潜在损失 50 万/年
投入产出比 1:3.2，6 个月收回成本。
```

### 对比论证

```
方案 A（申请 2 个 HC）：
- 成本：60 万/年
- 收益：190 万/年
- 风险：可控

方案 B（不申请）：
- 成本：0
- 收益：维持现状
- 风险：业务瓶颈，Q3 目标可能不达标

建议：选 A，6 个月收回成本。
```

## 周会汇报结构

### 进度 + 异常 + 下一步

```
1. 本周完成：
   - 事项 A（100%）
   - 事项 B（80%，滞后 1 天）

2. 异常说明：
   - 事项 B 滞后原因：供应商交付延迟
   - 影响：整体进度延迟 1 天，可控
   - 措施：已催货，明天到货

3. 下周计划：
   - 周一：完成事项 B
   - 周三：启动事项 C
   - 周五：完成 D 评审
```

### 数据同步模板

```
KPI 达成：
- 目标：100 万
- 实际：95 万
- 达成率：95%
- 同比：+20%
- 环比：+5%

异常说明：
- 未达标原因：X 供应商掉量
- 影响：5 万缺口
- 补救：Y 供应商下周补量
```

## 临时汇报模板

### 5W2H 快报

```
领导，有个紧急情况：

What：系统故障，下单接口报错
When：14:30 发现，已持续 30 分钟
Where：下单服务
Who：技术 X 在排查
Why：初步判断是数据库锁表
How：已切换备用库，正在恢复
How much：影响约 100 单，预估损失 5 万

需要支持：
- 技术负责人到场
- 客服准备话术
```

## 跨部门协同话术

### RACI 确认

```
这个项目涉及 3 个部门：
- 产品部（R）：需求设计
- 技术部（R）：开发实现
- 运营部（C）：上线配合
我（A）：整体负责
您（I）：关键节点知会

协同机制：
- 每周一 10 点对齐会
- 关键节点您拍板
```

### 冲突升级路径

```
正常协同：
- 执行层对接（每天）
- 负责人对齐（每周）

冲突处理：
- 小冲突：负责人协调
- 大冲突：升级到您和 X 总拍板
```

## PRD 评审应答话术

### 边界条件

```
这个问题我们考虑了：
- 最大值：支持 10 万并发
- 最小值：0 数据时不报错
- 极限情况：降级方案已设计
```

### 异常处理

```
异常情况分 3 类：
1. 可恢复：自动重试 3 次
2. 不可恢复：报错 + 记录 + 告警
3. 用户误操作：引导 + 撤销
```

## 应答技巧

### 模糊表述转化

| 模糊表述 | 精确表述 |
|---------|---------|
| "可能需要 2 周" | "预计 2 周，最晚 3 周" |
| "应该没问题" | "已测试 3 轮，通过率 100%" |
| "大概 50 万" | "50 万±10%，基于 XX 数据" |
| "尽量完成" | "目标 100%，保底 80%" |

### 被问住时的回应

```
"这个问题我确认一下，下午 3 点前给您准确答复。"
（不要现场编，记下来会后确认）
```

### 引导式回应

```
"您提到的 X 风险我们考虑了，具体方案是..."
（把问题引导到自己准备好的答案）
```

## 高危消息话术禁忌

| ❌ 禁忌 | ✅ 建议 |
|--------|--------|
| "我不知道" | "我确认后给您准确答复" |
| "这不是我的责任" | "我的责任是 X，已处理 Y" |
| "应该不会再发生了" | "已增加 Z 机制，确保不发生" |
| "都是供应商的问题" | "供应商有责任，我们也有 X 疏漏" |
| "我已经很努力了" | "结果是 X，改进措施是 Y" |
