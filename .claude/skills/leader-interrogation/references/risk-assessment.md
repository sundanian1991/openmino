# 风险评估矩阵（Risk Assessment）

> 高危消息怎么定级？风险怎么传导？责任怎么划分？

---

## 高危消息分级标准（P0-P3）

| 等级 | 定义 | 汇报时限 | 汇报对象 | 示例 |
|------|------|---------|---------|------|
| **P0** | 业务中断/重大损失/法律责任 | 15 分钟内 | C2+ 部门负责人 | 核心系统宕机、监管处罚、群体性投诉 |
| **P1** | 业务影响>10%/损失>50 万 | 1 小时内 | 部门负责人 | 核心供应商清退、数据泄露、重大客诉 |
| **P2** | 业务影响 5-10%/损失 10-50 万 | 4 小时内 | 直属领导 | 供应商断供、系统故障、VIP 客诉 |
| **P3** | 业务影响<5%/损失<10 万 | 24 小时内 | 直属领导 | 一般投诉、小故障、流程异常 |

---

## 风险传导路径分析

### 纵向传导（向上）

```
P3 → 直属领导（内部消化）
P2 → 部门负责人（备案）
P1 → C2（决策）
P0 → CEO/董事会（危机）
```

### 横向传导（跨部门）

```
供应商问题 → 业务部门 → 客服 → 公关
系统故障 → 技术 → 业务 → 客服
合规风险 → 法务 → 业务 → 财务
```

### 外部传导

```
内部问题 → 供应商 → 客户 → 媒体 → 监管
```

## 问责边界划分

### 责任类型

| 类型 | 定义 | 示例 | 处理方式 |
|------|------|------|---------|
| **直接责任** | 直接导致问题发生 | 操作失误、违规操作 | 警告/记过/辞退 |
| **管理责任** | 管理不到位导致问题 | 流程缺失、培训不足 | 通报/降薪/降职 |
| **领导责任** | 领导决策失误 | 方向错误、资源不足 | 通报/引咎辞职 |
| **连带责任** | 相关部门未尽责 | 协同不到位、监督缺失 | 通报/扣绩效 |

### 问责比例参考

| 问题类型 | 直接责任 | 管理责任 | 领导责任 |
|---------|---------|---------|---------|
| 操作失误 | 70% | 20% | 10% |
| 流程缺失 | 30% | 40% | 30% |
| 决策失误 | 10% | 30% | 60% |
| 系统性风险 | 20% | 30% | 50% |

## 风险评估矩阵

| 概率\影响 | 轻微 (1) | 中等 (2) | 严重 (3) | 致命 (4) |
|----------|---------|---------|---------|---------|
| **高 (A)** | P2 | P1 | P0 | P0 |
| **中 (B)** | P3 | P2 | P1 | P0 |
| **低 (C)** | P3 | P3 | P2 | P1 |

**使用方法**：
1. 评估问题发生概率（高/中/低）
2. 评估影响程度（轻微/中等/严重/致命）
3. 查表定级（P0-P3）
4. 按级别启动相应汇报和应急预案

## 高危消息汇报清单

### 汇报前自检

- [ ] 问题定级准确吗？（P0-P3）
- [ ] 影响范围清楚吗？（数据支撑）
- [ ] 根因找到了吗？（5 Why 分析）
- [ ] 止损措施执行了吗？（已完成/进行中）
- [ ] 问责建议有吗？（责任人、处理方式）
- [ ] 预防措施想好了吗？（流程/系统/培训）

### 汇报材料准备

- [ ] 事件描述（5W2H）
- [ ] 影响分析（数据、范围）
- [ ] 根因分析（5 Why）
- [ ] 处理进展（时间线）
- [ ] 问责建议（责任人、方式）
- [ ] 预防措施（具体动作）

### 汇报后跟进

- [ ] 领导指示记录了吗？
- [ ] 待办事项分解了吗？
- [ ] 责任人明确了吗？
- [ ] 时间节点确定了吗？
- [ ] 复盘会议安排了吗？

## 常见高危场景及应对

### 场景 1：核心供应商清退

**风险定级**：P1

**汇报要点**：
1. 清退原因（违规转包/质量问题/合规风险）
2. 业务影响（订单占比、切换周期）
3. 备选方案（供应商 B 已就绪）
4. 切换计划（时间表、责任人）

**领导可能问**：
- "这个供应商当初是谁选的？"
- "为什么现在才发现问题？"
- "切换期间业务会不会中断？"

### 场景 2：系统故障导致业务中断

**风险定级**：P0-P1（视影响范围）

**汇报要点**：
1. 故障时间（发现时间、持续时长）
2. 影响范围（订单量、损失金额）
3. 处理进展（已恢复/待恢复）
4. 根因分析（技术原因、人为原因）

**领导可能问**：
- "什么时候发现的？为什么现在才报？"
- "现在控制住了没有？"
- "怎么保证不再发生？"

### 场景 3：重大客户投诉

**风险定级**：P1-P2（视客户等级）

**汇报要点**：
1. 客户等级（VIP/普通）
2. 投诉内容（产品质量/服务态度）
3. 处理进展（已沟通/待解决）
4. 升级风险（媒体/监管）

**领导可能问**：
- "客户诉求是什么？"
- "我们有没有过错？"
- "最坏情况想过没有？"

---

*风险评估不是推卸责任，是为了更有效地处理问题。*
