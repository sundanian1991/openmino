# 质询检查清单（Checklist）

> 质询过程中逐项检查，确保覆盖所有维度

---

## 质询前检查

### 配置确认
- [ ] 领导风格配置完整（职级/风格/焦点/强度）
- [ ] 方案材料已提交（文档/大纲/核心逻辑）
- [ ] 汇报场景明确（向上/跨部门/资源申请）
- [ ] 预期目标清晰（审批/资源/共识）

### 基准确认
- [ ] 复述用户方案核心逻辑（确认理解无误）
- [ ] 确认质询轮次（默认 7 轮）
- [ ] 确认质询强度（温和/中等/尖锐）

---

## 质询中检查

### 8 大维度覆盖

| 维度 | 检查项 | 已完成 |
|------|--------|--------|
| **假设成立性** | 方案底层假设是否验证 | [ ] |
| **数据来源** | 数据可靠性、样本量、统计周期 | [ ] |
| **竞品动态** | 竞品动作、市场趋势、对标分析 | [ ] |
| **ROI 质疑** | 投入产出比、衡量指标、回收周期 | [ ] |
| **资源投入** | 资源需求合理性、轻量化替代方案 | [ ] |
| **时间节点** | 时间选择理由、早晚影响分析 | [ ] |
| **协同机制** | 跨部门协同、RACI、Plan B | [ ] |
| **风险预案** | 最大风险识别、应对措施、触发条件 | [ ] |

### 追问质量检查
- [ ] 问题基于用户上一轮回答（不是预设问题）
- [ ] 暴露用户回答中的矛盾点
- [ ] 不允许模糊表述（"可能"、"应该"、"大概"）
- [ ] 问题难度递增（逻辑→数据→执行→风险）

---

## 漏洞报告检查

### 报告结构
- [ ] 逻辑漏洞（高危）— 影响方案成立性
- [ ] 数据缺失（中危）— 影响说服力
- [ ] 执行风险（中危）— 影响落地可行性
- [ ] 话术优化（低危）— 影响表达效果

### 建议质量
- [ ] 每条建议可执行（具体动作，不是抽象建议）
- [ ] 建议有优先级（高危优先修复）
- [ ] 建议有示例（给出参考模板/话术）

---

## 最终评估检查

### 评估维度
- [ ] 逻辑严密性（1-5 分 + 评语）
- [ ] 数据充分性（1-5 分 + 评语）
- [ ] 风险预案（1-5 分 + 评语）
- [ ] 表达清晰度（1-5 分 + 评语）
- [ ] 综合评分 + 是否建议进入正式汇报

### 离场建议
- [ ] 明确"可汇报"或"需修复"
- [ ] 列出必须修复的 Top 3 问题
- [ ] 给出汇报开场白建议（如何预判领导问题）

---

## 使用后复盘

### 用户反馈
- [ ] 质询强度是否合适
- [ ] 问题是否击中要害
- [ ] 报告是否有 actionable 建议
- [ ] 是否愿意推荐给同事

### 持续改进
- [ ] 记录本次暴露的高频漏洞类型
- [ ] 更新问题库（新增尖锐问题）
- [ ] 优化建议模板（更具体、更可执行）

---

* checklist 不是形式，是确保质量的底线。*
