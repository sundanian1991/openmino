## 覆盖场景（7 大汇报类型）

1. **向上汇报** — C2 汇报、部门负责人汇报、晋升述职
2. **周会/例会** — 常规工作汇报、数据同步、进度更新
3. **单独事项** — 资源申请、方案审批、决策请示
4. **临时汇报** — 突发事件、应急响应、业务异常
5. **高危消息** — 供应商清退、合规风险、业务中断
6. **跨部门协同** — 多方配合、资源协调、责任划分
7. **PRD/方案评审** — 产品需求、技术方案、运营方案

---

## 核心流程

1. 选择汇报场景 → 7 大类型
2. 配置领导风格 → 职级/风格/焦点/强度
3. 提交方案材料 → 文档/口述/大纲
4. 首轮质询 → 5-7 轮压力测试
5. 生成报告 → 漏洞报告 + 话术建议
6. 修复优化 → 根据建议改进
7. 二次验证 → 确认修复效果（可选）
8. 最终评估 → 汇报准备度评分
