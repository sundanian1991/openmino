# Workspace Tidy — 文件生命周期管理

> 基于 LIFECYCLE-WORKFLOW.md 的文件整理和归档技能
> **扫描范围**：整个项目目录（根目录 + workspace/ + mino/ + docs/）

---

## 核心架构理解

项目采用三区域架构：

| 区域 | 用途 | 路径示例 | 留存周期 |
|------|------|----------|---------|
| **根目录** | 全局配置、临时文件 | `/my-agent/*.md` | 立即整理 |
| **workspace/** | 入口区，业务任务 | `/workspace/[主题]/🔄 进行中/` | 周度清理 |
| **mino/** | 个人工作区、脚本 | `/mino/*.js, *.py` | 按需整理 |
| **docs/** | 归档区，正式文档 | `/docs/项目归档/` | 长期保留 |
| **memory/** | 核心区，洞察决策 | `/memory/` | 永久保留 |

### 工作流状态

每个主题下分四个状态：

| 状态 | 图标 | 说明 | 示例 |
|------|------|------|------|
| **进行中** | 🔄 | 当前活跃工作 | 正在写的评估报告 |
| **待处理** | ⏳ | 排队等待处理 | 待回复的邮件草稿 |
| **已完成** | ✅ | 完成暂存 | 完成的项目（待周回顾） |
| **知识库** | 📖 | 稳定参考，不随工作流移动 | 管理规范、模板 |

## 扫描范围

**全项目扫描**，不只 workspace/：

```bash
# 1. 根目录（检查是否有任务文件放错位置）
ls -la /Users/sundanian/Documents/projects/ai-agents/my-agent/*.md /Users/sundanian/Documents/projects/ai-agents/my-agent/*.json /Users/sundanian/Documents/projects/ai-agents/my-agent/*.xlsx 2>/dev/null

# 2. workspace/ 目录（业务任务）
find /Users/sundanian/Documents/projects/ai-agents/my-agent/workspace/ -type f \( -name "*.md" -o -name "*.json" -o -name "*.xlsx" \)

# 3. mino/ 目录（个人工作区）
find /Users/sundanian/Documents/projects/ai-agents/my-agent/mino/ -type f \( -name "*.md" -o -name "*.js" -o -name "*.py" -o -name "*.json" \)

# 4. docs/ 目录（归档）
find /Users/sundanian/Documents/projects/ai-agents/my-agent/docs/ -type f -name "*.md"
```

对每个文档：
1. 读取内容，理解用途
2. 判断文件状态（活跃/完成/过时/位置错误）
3. 评估价值（高/中/无）
4. **检查是否在正确位置**（根目录只保留全局配置）

### 阶段 2：分类决策

根据文件位置和内容进行分类：

**根目录文件处理**：
```
根目录/*.md, *.json, *.xlsx
    │
    ├─→ 是规划文件？（task_plan.md, findings.md, progress.md）
    │   └─→ 移到 workspace/[相关主题]/🔄 进行中/[任务名]/
    │
    ├─→ 是业务文件？（台账、报表）
    │   └─→ 移到 workspace/[相关主题]/🔄 进行中/[任务名]/
    │
    ├─→ 是测试/临时数据？
    │   └─→ 移到 mino/ 或删除
    │
    └─→ 是全局配置？
        └─→ 保留在根目录（CLAUDE.md, README.md 等）
```

**workspace/ 文件处理**：
```
文件分析
    │
    ├─→ 📖 知识库内容？
    │   └─→ 保持现状（知识库不随工作流移动）
    │
    ├─→ 🔄 进行中状态？
    │   └─→ 保持现状
    │
    ├─→ ⏳ 待处理状态？
    │   └─→ 判断是否需要启动
    │
    └─→ ✅ 已完成状态？
        └─→ 进入归档判断
            │
            ├─→ 有文档价值？
            │   └─→ docs/项目归档/[主题名称]/
            │
            ├─→ 有洞察价值？
            │   └─→ memory/insights.md
            │
            ├─→ 过时无价值？
            │   └─→ 删除
            │
            └─→ 需要继续？
                └─→ 移回 🔄 进行中
```

### 阶段 3：预览确认

生成整理计划，展示给用户确认：

```
## 文件整理计划

### 供应商管理/⏳ 待处理/人力规划
- 状态：待处理超过 2 周
- 建议：评估是否需要启动，或移回知识库
- 操作：[移至 📖 知识库/06-晋升述职/]

### 供应商管理/📖 知识库/01-基础信息/
- 状态：稳定知识
- 建议：保持现状

### docs/项目归档/定价委员会/
- 状态：已完成项目
- 建议：保持归档

---
总计：移动 3 个文件，删除 1 个文件，保持 8 个文件

确认执行？
```

### 阶段 4：批量执行

用户确认后，执行批量操作：

```bash
# 移动文件
mv "workspace/供应商管理/✅ 已完成/xxx" "docs/项目归档/供应商管理/"

# 提炼洞察（如果有）
# 追加到 memory/insights.md

# 删除无价值文件
trash "workspace/供应商管理/✅ 已完成/过时文件.md"
```

## 分类规则参考

### 判断"已完成"的标准

- ✅ 文档明确标注"已完成"、"已归档"
- ✅ README 中有完成时间、汇报记录
- ✅ 内容是历史材料，不再活跃修改

### 判断"文档价值"的标准

**归档到 docs/**：
- 正式汇报材料（C2 汇报、机制文档）
- 完整项目过程记录（五文件工作流）
- 有参考价值的模板和规范

**提炼到 memory/**：
- 职业洞察和经验总结
- 重要的决策记录
- 偏好和行为模式发现

**删除**：
- 临时草稿
- 重复文件
- 已过时的中间版本

## 智能价值判断

**核心原则**：必须读取文件内容才能判断价值，不能只看文件名。

### 价值判断流程

```
扫描文件
    │
    ├─→ 读取文件内容（前 100 行或全部）
    │
    ├─→ 分析内容类型
    │   ├─ 包含"总结"/"报告"/"规范"/"机制" → 文档价值高
    │   ├─ 包含"洞察"/"发现"/"经验"/"教训" → 洞察价值高
    │   ├─ 包含"草稿"/"临时"/"test"/"TODO" → 价值低
    │   └─ 内容空洞/重复/过时 → 无价值
    │
    └─→ 结合后续表现
        ├─ 文件修改时间 > 30 天 → 可能过时
        ├─ 文件大小 < 1KB 且 内容单薄 → 价值低
        └─ 文件被多次引用/修改 → 活跃度高
```

### 价值评分矩阵

| 内容特征 | 文档价值 | 洞察价值 | 建议操作 |
|----------|---------|---------|---------|
| 正式汇报/机制文档 | ⭐⭐⭐ | ⭐ | docs/归档 |
| 述职/复盘/总结 | ⭐⭐ | ⭐⭐⭐ | 提炼洞察 + 归档 |
| 规范/模板 | ⭐⭐⭐ | ⭐ | 📖 知识库 |
| 草稿/临时文件 | ⭐ | ⭐ | 删除 |
| 测试数据 | ⭐ | - | mino/ 或删除 |

### 判断示例

**高文档价值**：
```markdown
# Q1 供应商评估报告
## 评估结果
博岳：达成率 92%，排名上升 2 位...
```
→ 包含完整数据和分析 → 归档到 docs/

**高洞察价值**：
```markdown
# 晋升述职总结
## 关键洞察
1. 职业发展的关键是持续学习...
```
→ 包含经验总结 → 提炼到 memory/insights.md

**低价值**：
```markdown
# 临时草稿
TODO: 整理一下数据
```
→ 内容空洞 → 删除

## 定期提醒机制

### 自动检测触发条件

每次会话启动时，自动检测以下条件：

| 条件 | 阈值 | 提醒级别 |
|------|------|---------|
| workspace/ 文件数 | > 50 | ⚠️ 需要整理 |
| ✅ 已完成/ 文件数 | > 10 | ⚠️ 需要归档 |
| 距上次整理时间 | > 7 天 | 💡 建议周度清理 |
| 根目录临时文件 | > 5 | 🔴 立即处理 |

### 提醒输出格式

```markdown
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【Workspace 状态提醒】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📊 当前状态：
- workspace/ 文件数：52（⚠️ 超过 50）
- ✅ 已完成/ 待归档：12 个

- 执行 /workspace-tidy 进行整理
- 或说"整理一下"开始清理

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 技能联动规则

不同技能产出的文件有特定存放位置：

| 技能 | 产出类型 | 存放位置 | 注意事项 |
|------|---------|---------|---------|
| **person-observer** | 人物观察记录 | `memory/projects/关键人画像/` | 永久保留，不随工作流移动 |
| **official-doc** | 公文模板 | `.claude/skills/official-doc/assets/` | 技能文件，不归档 |
| **official-doc** | 公文产出 | `workspace/[相关主题]/` | 按正常工作流处理 |
| **evolution-protocol** | SESSIONS.md | 各 skill 目录下 | 超 30 条自动裂变 |
| **prompt-optimizer** | 优化配置 | `workspace/prompt-optimizer/` | 长期保留 |

### 联动冲突处理

当文件同时涉及多个技能时：

```
文件同时是：
├─ 人物观察记录 → 优先放到 memory/
├─ 公文产出 → 按工作流处理
└─ 进化记录 → 保留在原 skill 目录

优先级：memory/ > docs/ > workspace/
```

### 整理时不触碰的目录

以下目录由特定技能管理，整理时跳过：

```
.claude/skills/          ← 技能文件，由 skill-creator 管理
memory/projects/关键人画像/ ← 人物观察，由 person-observer 管理
```

## 注意事项

1. **知识库不动**: 📖 知识库/ 下的文件是稳定参考，不随工作流移动
2. **谨慎删除**: 删除前必须预览确认，优先使用 `trash` 而非 `rm`
3. **保持结构**: docs/ 下的归档应按主题分类
4. **周回顾节奏**: 建议每周执行一次，保持 workspace/ 清洁
5. **内容优先**: 必须读取文件内容才能判断价值，不能只看文件名
6. **技能联动**: 注意不同技能的文件存放规则，避免冲突
