# Workspace Tidy — 文件生命周期管理

> 基于 LIFECYCLE-WORKFLOW.md 的文件整理和归档技能

---

## 核心架构理解

项目采用三区域架构：

| 区域 | 用途 | 留存周期 |
|------|------|---------|
| **workspace/** | 入口区，临时工作 | 周度清理 |
| **docs/** | 归档区，正式文档 | 长期保留 |
| **memory/** | 核心区，洞察决策 | 永久保留 |

### 工作流状态

每个主题下分四个状态：

| 状态 | 图标 | 说明 | 示例 |
|------|------|------|------|
| **进行中** | 🔄 | 当前活跃工作 | 正在写的评估报告 |
| **待处理** | ⏳ | 排队等待处理 | 待回复的邮件草稿 |
| **已完成** | ✅ | 完成暂存 | 完成的项目（待周回顾） |
| **知识库** | 📖 | 稳定参考，不随工作流移动 | 管理规范、模板 |

---

## 工作流程

### 阶段 1：扫描分析

遍历 workspace/ 下所有文档，收集信息：

```bash
# 扫描所有 README/CLAUDE 文档
find workspace/ -name "README.md" -o -name "CLAUDE.md" -o -name "*.md"

# 扫描所有主题目录
ls -la workspace/*/
```

对每个文档：
1. 读取内容，理解用途
2. 判断文件状态（活跃/完成/过时）
3. 评估价值（高/中/无）

### 阶段 2：分类决策

根据文件内容和建议规则进行分类：

**决策树**：

```
文件分析
    │
    ├─→ 📖 知识库内容？
    │   └─→ 保持现状（知识库不随工作流移动）
    │
    ├─→ 🔄 进行中状态？
    │   └─→ 保持现状
    │
    ├─→ ⏳ 待处理状态？
    │   └─→ 判断是否需要启动
    │
    └─→ ✅ 已完成状态？
        └─→ 进入归档判断
            │
            ├─→ 有文档价值？
            │   └─→ docs/项目归档/[主题名称]/
            │
            ├─→ 有洞察价值？
            │   └─→ memory/insights.md
            │
            ├─→ 过时无价值？
            │   └─→ 删除
            │
            └─→ 需要继续？
                └─→ 移回 🔄 进行中
```

### 阶段 3：预览确认

生成整理计划，展示给用户确认：

```
## 文件整理计划

### 供应商管理/⏳ 待处理/人力规划
- 状态：待处理超过 2 周
- 建议：评估是否需要启动，或移回知识库
- 操作：[移至 📖 知识库/06-晋升述职/]

### 供应商管理/📖 知识库/01-基础信息/
- 状态：稳定知识
- 建议：保持现状

### docs/项目归档/定价委员会/
- 状态：已完成项目
- 建议：保持归档

---
总计：移动 3 个文件，删除 1 个文件，保持 8 个文件

确认执行？
```

### 阶段 4：批量执行

用户确认后，执行批量操作：

```bash
# 移动文件
mv "workspace/供应商管理/✅ 已完成/xxx" "docs/项目归档/供应商管理/"

# 提炼洞察（如果有）
# 追加到 memory/insights.md

# 删除无价值文件
trash "workspace/供应商管理/✅ 已完成/过时文件.md"
```

## 分类规则参考

### 判断"已完成"的标准

- ✅ 文档明确标注"已完成"、"已归档"
- ✅ README 中有完成时间、汇报记录
- ✅ 内容是历史材料，不再活跃修改

### 判断"文档价值"的标准

**归档到 docs/**：
- 正式汇报材料（C2 汇报、机制文档）
- 完整项目过程记录（五文件工作流）
- 有参考价值的模板和规范

**提炼到 memory/**：
- 职业洞察和经验总结
- 重要的决策记录
- 偏好和行为模式发现

**删除**：
- 临时草稿
- 重复文件
- 已过时的中间版本

## 命令行工具（可选）

如果需要批量操作，可以使用：

```bash
# 查看当前状态
ls -la workspace/*/

# 移动文件（使用 mv 或 trash）
trash "不需要的文件.md"

# 移动到归档
mv "源文件.md" "docs/目标位置/"
```

---

## 注意事项

1. **知识库不动**: 📖 知识库/ 下的文件是稳定参考，不随工作流移动
2. **谨慎删除**: 删除前必须预览确认，优先使用 `trash` 而非 `rm`
3. **保持结构**: docs/ 下的归档应按主题分类
4. **周回顾节奏**: 建议每周执行一次，保持 workspace/ 清洁
