# Icon Syntax — Four Visual Languages for Design Modes

> **核心命题**：图标不是装饰，是设计模式的视觉语法的延伸。每个模式有独特的美学定位，图标必须服从这个定位，而不是用一套通用方案敷衍所有模式。

---

## 品牌模式 — Warm Touch

**美学定位**：人文温度 × 手工感 × 极简克制

**图标哲学**：手绘线条是温暖的，不是机械的。每个图标都应该像是有人用心画上去的，有呼吸感。

**视觉规范**：
- **线条**：炭灰色 `#3D2C29`，2.5px 粗细，`stroke-linecap: round`
- **点缀**：陶土色 `#E2725B` 小面积填充，仅用于视觉焦点
- **形状**：有机曲线，微微波动（Q 贝塞尔），非完美几何
- **容器**：可选双层偏移边框（灰色 3px 偏移 + 黑色主框）

**示例概念**：
- 书籍：两个圆角矩形代表书页，连接点用陶土色圆点
- 人物：圆形头部 + 曲线身体，填充陶土色
- 合作：两只手的手绘线条，陶土色握手区域

**禁止**：
- ❌ 直角、锐利转角
- ❌ 完美直线
- ❌ 机械对称

---

## 数据模式 — Graphic Precision

**美学定位**：精确 × 克制 × 数字原生

**图标哲学**：图标是代码的延伸，应该像编程符号一样精确、无歧义。不追求装饰性，追求语义准确性。

**视觉规范**：
- **线条**：纯黑 `#111111`，1.5px 粗细，`stroke-linecap: square`
- **点缀**：陶土色 `#E2725B` 仅用于强调关键节点
- **形状**：完美几何（圆形、正方形、直线），网格对齐
- **容器**：1px 黑色边框方形容器，8px 圆角

**示例概念**：
- 串行：→→（两个箭头连续）
- 并行：→||→（两条并行线）
- 路由：⟲（分支符号）
- 循环：↻（循环箭头）

**禁止**：
- ❌ 曲线、有机形状
- ❌ 手绘感
- ❌ 装饰性元素

---

## 金融时报 — Typographic Authority

**美学定位**：权威 × 可信 × 出版物质感

**图标哲学**：图标应该像报纸信息图表的一部分，精确、克制、有历史感。衬线字体和几何形状结合。

**视觉规范**：
- **线条**：青绿色 `#0d7680`，2px 粗细
- **点缀**：深绿 `#2e7d32`（正向）、红 `#cc0000`（负向）
- **形状**：矩形、箭头、图表元素，有衬线字体的气质
- **容器**：无独立容器，图标与文字排版一体化

**示例概念**：
- 趋势上升：↑ 粗箭头，青绿色
- 趋势下降：↓ 粗箭头，红色
- 排名：123（衬线数字，青绿色）
- 对比：█ █（矩形条，高低对比）

**禁止**：
- ❌ 手绘线条
- ❌ 圆润形状
- ❌ 过度装饰

---

## 咨询模式 — Strategic Geometry

**美学定位**：结论性 × 权威感 × 战略思考

**图标哲学**：图标是战略框架的视觉化，应该像麦肯锡图表一样清晰、有力、服务于结论。抽象几何优于具象。

**视觉规范**：
- **线条**：深灰 `#333333`，2px 粗细
- **点缀**：低饱和红 `#B85450`，用于关键路径或问题点
- **形状**：箭头、三角形、方块，抽象概念具象化
- **容器**：深灰边框，4px 圆角，与卡片统一

**示例概念**：
- 增长：↗ 粗箭头向上
- 下降：↘ 粗箭头向下
- 问题：⚠ 三角形警告，红色填充
- 机会：◇ 菱形，空心
- 关联：— 直线连接
- 分支：┬ 树状分叉

**禁止**：
- ❌ 可爱化
- ❌ 过度细节
- ❌ 装饰性复杂度

---

## 通用原则（跨模式）

1. **尺寸一致性**：所有图标 48×48 viewBox，保持视觉重量平衡
2. **语义优先**：图标必须服务于内容理解，不能为了美观牺牲准确性
3. **克制使用**：不是每个卡片都需要图标，图标用在需要快速识别的地方
4. **工艺标准**：每个图标都应该是精心设计的，不是随手画的

## 技术实现模板

```css
/* 品牌模式 */
.hand-drawn {
  fill: none;
  stroke: #3D2C29;
  stroke-width: 2.5;
  stroke-linecap: round;
  stroke-linejoin: round;
}
.hand-drawn-accent {
  fill: #E2725B;
  stroke: none;
}

/* 数据模式 */
.icon-precision {
  fill: none;
  stroke: #111111;
  stroke-width: 1.5;
  stroke-linecap: square;
}
.icon-accent {
  fill: #E2725B;
  stroke: none;
}

/* 金融时报 */
.icon-ft {
  fill: none;
  stroke: #0d7680;
  stroke-width: 2;
}
.icon-positive { fill: #2e7d32; }
.icon-negative { fill: #cc0000; }

/* 咨询模式 */
.icon-strategy {
  fill: none;
  stroke: #333333;
  stroke-width: 2;
}
.icon-alert { fill: #B85450; }
```

## 结语

这四套图标风格不是互斥的，而是各自服务于设计模式的美学定位。

**品牌模式**的图标温暖、手工、有人情味。
**数据模式**的图标精确、克制、像代码符号。
**金融时报**的图标权威、可信、像报纸图表。
**咨询模式**的图标有力、清晰、像战略框架。

选择哪一套，不取决于"哪个更好看"，而取决于"哪个更符合当前设计模式的灵魂"。
