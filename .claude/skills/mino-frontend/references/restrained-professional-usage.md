# 克制专业主义 — 高效复用方案

> **目标**：让"克制专业主义"风格成为年老师的默认风格指纹，实现一键调用、稳定输出

---

## 🎯 核心目标

1. **一键调用** — 无需重复描述风格细节，一个命令即可生成
2. **稳定输出** — 每次输出品质一致，不依赖临场发挥
3. **优先采用** — 成为默认风格，其他风格向此对齐

**步骤**：

1. 在 `mino-frontend/SKILL.md` 中添加新风格条目
2. 在 `mino-frontend/references/styles.md` 中添加完整规范
3. 在 `mino-frontend/references/design-philosophy-12modes.md` 中添加为第 13 种模式

**添加内容**：

```markdown
## 13. 克制专业主义

### 运动名称
**Restrained Professionalism** — 克制专业主义

### 设计哲学宣言

**空间与形式**：极简到极致的黑白灰世界，每个像素都为数据服务。无阴影、无渐变、无装饰 — 只有纯粹的网格和边框。顶部 2.4px 陶土色装饰线作为唯一的"大声"元素，右下角圆形页码徽章作为导航锚点。信息密度可以高，但视觉噪音必须为零。

**色彩与情绪**：陶土色 (#E2725B) 是唯一的例外，唯一的温度，唯一的焦点。一个页面的陶土色使用面积控制在 10% 以内 — 顶部装饰线、页码徽章、关键数据强调。这是冷峻理性世界中的唯一温暖，是经过计算的视觉引导。

**工艺标准**：每个数字都对齐到像素，每个边框的粗细都经过计算。卡片没有阴影，依靠 1px 边框和充足内边距创造层次。表格 hover 效果只用 `#fafafa`，最微妙的反馈。

### 概念线索

| 内容类型 | 概念线索 | 视觉翻译 |
|---------|---------|---------|
| 问题诊断 | 冷静、分析 | 黑白灰基调、表格拆解 |
| 事件分析 | 冲突、张力 | 左右对比、强调色焦点 |
| 战略演示 | 权威、可信 | 编辑式排版、结论性标题 |
| 数据报告 | 精确、客观 | 网格对齐、单一强调 |

### 工艺检查清单

**视觉工艺**：
- [ ] 顶部 2.4px 陶土色装饰线
- [ ] 右下角圆形页码徽章（非封面）
- [ ] 无阴影、无渐变
- [ ] 表头 2px 黑色底边框
- [ ] 陶土色每页≤10% 面积

**色彩工艺**：
- [ ] 主背景：`#FFFFFF`
- [ ] 文字层级：`#111` / `#666` / `#999`
- [ ] 边框：`#E5E5E5`
- [ ] 强调：`#E2725B`（慎用）

**字体工艺**：
- [ ] 中文：Microsoft YaHei
- [ ] 英文/数字：Arial
- [ ] 字号：T1(40px) / T2(28px) / T3(14-16px) / T4(12-13px) / T5(10-11px)

**版式工艺**：
- [ ] 5 种核心版式（封面/标准/对比/表格/卡片）
- [ ] 每页≤7 个视觉块（Miller's Law）
- [ ] 视口适配：`100vh`, `overflow: hidden`
```

```
- "年老师风格"
```

## 🧠 判断层规范（核心）

> **质量差异的根源不是美学执行，而是设计判断。**
>
> 本章节定义"内容→设计"的转译规则，确保输出质量稳定。

### 一、内容编辑五问（必须先回答，再动手）

**在渲染任何内容之前，强制回答：**

| 问题 | 目的 | 高质量答案示例 |
|------|------|--------------|
| **1. 这页的核心观点是什么？** | 防止多焦点 | "刘乾坤的动机是表演型预警 + 转移视线" |
| **2. 支撑观点需要几个视觉块？** | 控制密度（≤7） | "5 个：表头 +4 行动机行" |
| **3. 哪个信息是焦点？** | 单一陶土色使用 | "可能性列的'极高'徽章" |
| **4. 哪个版式最合适？** | 内容→形式匹配 | "表格（有明确对比维度）" |
| **5. 哪些内容可以删？** | 减法原则 | "删掉重复的解释性文字" |

**❌ 错误流程**：原始内容 → 直接套用版式 → 输出
**✅ 正确流程**：原始内容 → 编辑重构 → 版式匹配 → 输出

### 二、密度决策树（简约/标准/丰富 如何选择）

```
第一步：判断内容类型
├── 数据报告（KPI、排名、趋势）
│   └── 默认【标准密度】→ 表格版式
│   └── 如果只展示 1-2 个核心 KPI → 【简约密度】→ KPI 卡片
│
├── 问题分析（诊断、拆解、归因）
│   └── 默认【标准密度】→ 左右对比或表格
│   └── 如果只有一个核心洞察 → 【简约密度】→ 单卡片 + 大留白
│
├── 战略演示（方案、建议、计划）
│   └── 默认【标准密度】→ 卡片网格
│   └── 如果有 4 个以上并列项 → 【丰富密度】→ 多卡片分页
│
└── 事件分析（背景、行为、动机）
    └── 默认【标准密度】→ 对比版式
    └── 如果有复杂多维度 → 【丰富密度】→ 多页拆解

第二步：检查信息量
├── 核心指标 ≤ 3 个 → 【简约】
│   └── 大留白、大字号（T1 40px+）
│   └── 每页 3-4 个视觉块
│
├── 核心指标 4-7 个 → 【标准】
│   └── 正常留白、正常字号（T2 28px）
│   └── 每页 5-7 个视觉块
│
└── 核心指标 > 7 个 → 【丰富】→ 但必须分页！
    └── 不要塞进一页！拆成 2 页【标准密度】
```

**密度选择示例**：

| 场景 | 原始内容 | 密度选择 | 理由 |
|------|---------|---------|------|
| 供应商月度 KPI | 10 家×5 项指标 | 【丰富】→ 分 2 页 | 50 个数据点，单页超载 |
| 事件背景对比 | 左边 4 点、右边 4 点 | 【标准】 | 8 个视觉块，刚好 |
| 核心结论 | 1 句话 | 【简约】 | 焦点唯一，大留白强化 |
| 风险预警清单 | 7 项风险 | 【标准】 | 7 是上限，刚好 |

### 三、焦点选择原则（陶土色用在哪里）

**陶土色 #E2725B 的优先级决策：**

```
第一优先级：异常值/风险信号
├── 业绩下滑的供应商
├── 超阈值的风险指标
└── 需要立即注意的预警

第二优先级：排名/等级
├── 第一名/最后一名
├── A 级/C 级（两极）
└── 完成/未完成（二元状态）

第三优先级：行动建议
├── 底部行动卡片
├── 话术建议
└── 下一步计划

禁止使用：
❌ 装饰性元素（分隔线、图标填充）
❌ 正文文字（只用于标题/数据）
❌ 多处同时使用（一页一处原则）
```

**焦点选择示例**：

| 场景 | 正确用法 ✅ | 错误用法 ❌ |
|------|-----------|-----------|
| 供应商排名表 | 只强调第一名（陶土色分数） | 每行都用陶土色背景 |
| 风险清单 | 只强调最高风险项 | 所有风险项同等强调 |
| 动机分析表 | 只强调"可能性"列 | 动机名称 + 逻辑 + 可能性全强调 |
| 对比页 | 只强调右侧（真实情况） | 左右都用陶土色 |

### 四、版式选择逻辑（为什么用这个版式）

**内容特征 → 版式匹配决策树：**

```
【表格版式】
触发条件：
├── 有明确的行×列结构
├── 需要对比多个维度
└── 数据项≥4 个，且有统一的表头

示例：
- 供应商排名（行=供应商，列=指标）
- 动机分析（行=动机，列=逻辑/可能性）
- 风险清单（行=风险，列=描述/等级）

---

【左右对比版式】
触发条件：
├── 有两个对立/对照的概念
├── 需要凸显差异/张力
└── 每侧内容≤4 个要点

示例：
- 刘乾坤的说法 vs 真实情况
- 问题表象 vs 根本原因
- 短期影响 vs 长期影响

---

【卡片网格版式】
触发条件：
├── 有多个并列的独立模块
├── 每个模块有标题 + 内容 + 行动
└── 模块数量=2/4/6（偶数网格）

示例：
- 影响评估（4 个相关方卡片）
- 应对策略（4 个场景话术）
- 风险预警（短期/中期/长期）

---

【标准内容版式】
触发条件：
├── 单一主题的内容展开
├── 列表项≤6 个
└── 不需要复杂对比

示例：
- 行为模式拆解（5 种模式列表）
- 核心原则（3-5 条原则）
- 总结回顾（要点列表）

---

【封面版式】
触发条件：
├── 演示文稿首页
└── 章节分隔页

必备元素：
- 大标题（T1, 40px）
- 副标题（T3, 16px）
- 装饰短线（陶土色）
- 日期/页脚
```

### 五、内容减法清单（什么该删）

**删除优先级（从最该删到可保留）：**

| 内容类型 | 删除建议 | 理由 |
|---------|---------|------|
| **重复解释** | 删除 | 同一个观点不说第二遍 |
| **过渡性文字** | 删除 | "综上所述"、"总的来说" |
| **空洞修饰词** | 删除 | "非常重要的"、"显著地" |
| **长段落** | 拆成列表 | 超过 2 行的文字转成 bullet points |
| **次要数据** | 删除或移入附录 | 与核心观点无直接关系的数字 |
| **多焦点强调** | 只保留一个 | 一页只能有一个陶土色焦点 |

**减法原则**：
> 如果删掉后不影响核心观点传递，就删掉。

### 场景 1：供应商月度 KPI 报告

**输入特征**：
- 10+ 家供应商
- 每家 5-8 项指标（完成值、目标值、环比、同比）
- 需要展示排名和趋势

**转译规则**：

| 决策维度 | 选择 |
|---------|------|
| **密度** | 【丰富】→ 分 2-3 页 |
| **版式** | 表格（主表）+ KPI 卡片（摘要页） |
| **焦点** | 第一名（陶土色分数）+ 最后一名（陶土色警告） |
| **编辑** | 摘要页只展示 Top3+  Bottom3 |

**输出结构**：

```
页 1（摘要页）：
├── T1 标题：供应商月度 KPI 总览
├── 4 个 KPI 卡片（整体完成率、环比、同比、达标家数）
└── 底部：红榜（Top3）/ 黑榜（Bottom3）

页 2（排名表上）：
├── T2 标题：头部供应商排名（1-5 名）
├── 表格：供应商 | 完成值 | 目标 | 完成率 | 环比
└── 焦点：第一名分数用陶土色

页 3（排名表下）：
├── T2 标题：中腰部供应商排名（6-10 名）
├── 表格：同上
└── 焦点：最后一名用陶土色警告标记
```

**正反示例**：

| ✅ 正确 | ❌ 错误 |
|--------|--------|
| 摘要页只展示 4 个核心 KPI | 摘要页展示 10 家×8 项=80 个数据 |
| 表格分 2 页，每页 5 行 | 10 家供应商塞进一页，文字小到看不清 |
| 只强调第一名和最后一名 | 每家供应商都用陶土色强调 |

### 场景 2：事件/问题分析

**输入特征**：
- 有对立的双方/两种说法
- 需要揭示真相/根本原因
- 有核心矛盾需要凸显

**转译规则**：

| 决策维度 | 选择 |
|---------|------|
| **密度** | 【标准】 |
| **版式** | 左右对比（核心矛盾用底部卡片） |
| **焦点** | 右侧（真相/根因）用陶土色标题 |
| **编辑** | 每侧≤4 个要点，超出则分组 |

**输出结构**：

```
页 1（对比页）：
├── T2 标题：XXX 事件 — 表象 vs 真相
├── 左侧卡片（浅灰背景）：表象/说法
│   ├── T3 标题：XXX 的说法
│   └── 4 个要点（灰色文字）
├── 右侧卡片（浅粉背景）：真相/事实
│   ├── T3 标题（陶土色）：真实情况
│   └── 4 个要点（正常文字）
└── 底部卡片（浅灰）：核心矛盾（1-2 句话）
```

**正反示例**：

| ✅ 正确 | ❌ 错误 |
|--------|--------|
| 左右各 4 个要点，对齐清晰 | 一侧 3 点、另一侧 8 点，视觉失衡 |
| 右侧标题用陶土色强调 | 左右标题都用陶土色 |
| 底部一句话总结核心矛盾 | 底部空着，或写满 3 行解释 |

### 场景 3：动机/归因分析

**输入特征**：
- 多个并列的动机/原因
- 每个有名称 + 逻辑描述 + 可能性等级
- 需要凸显优先级

**转译规则**：

| 决策维度 | 选择 |
|---------|------|
| **密度** | 【标准】 |
| **版式** | 表格（动机 | 逻辑 | 可能性） |
| **焦点** | "可能性"列（极高/高用陶土色徽章） |
| **编辑** | 按可能性排序，极高的放前面 |

**输出结构**：

```
页 1（分析表）：
├── T2 标题：XXX 的动机分析
├── 表头：动机 | 逻辑 | 可能性
├── 表格内容（按可能性降序）：
│   ├── 动机 1（极高）→ 红色徽章
│   ├── 动机 2（极高）→ 红色徽章
│   ├── 动机 3（高）→ 橙色徽章
│   └── 动机 4（中）→ 蓝色徽章
└── 底部卡片（浅粉）：核心判断（1 句话）
```

**正反示例**：

| ✅ 正确 | ❌ 错误 |
|--------|--------|
| 按可能性降序排列 | 按随机顺序排列 |
| 可能性列用彩色徽章 | 所有徽章都用同样颜色 |
| 底部一句话总结核心判断 | 底部空着或写满 3 行 |

### 场景 4：应对策略/行动计划

**输入特征**：
- 多个场景/情况
- 每个场景有对应的话术/行动
- 需要可执行性

**转译规则**：

| 决策维度 | 选择 |
|---------|------|
| **密度** | 【标准】 |
| **版式** | 卡片网格（2x2 或 2x3） |
| **焦点** | 最高优先级场景（陶土色边框/背景） |
| **编辑** | 每个场景≤3 个要点 +1 句话术 |

**输出结构**：

```
页 1（策略页）：
├── T2 标题：应对策略 — 4 个核心场景
├── 卡片网格（2x2）：
│   ├── 卡片 1：场景 A + 3 要点 + 话术
│   ├── 卡片 2：场景 B + 3 要点 + 话术
│   ├── 卡片 3：场景 C + 3 要点 + 话术
│   └── 卡片 4（浅粉背景）：场景 D（最高优）+3 要点 + 话术
└── 底部（可选）：核心原则（1 句话）
```

**正反示例**：

| ✅ 正确 | ❌ 错误 |
|--------|--------|
| 每个卡片 3 要点 +1 句话术 | 每个卡片 8 要点 +3 段解释 |
| 最高优卡片用浅粉背景 | 所有卡片同样背景，看不出优先级 |
| 4 个卡片 2x2 网格 | 4 个卡片竖着排成一列 |

### 场景 5：风险预警/问题清单

**输入特征**：
- 多个风险项/问题项
- 每项有描述 + 等级/时间范围
- 需要凸显紧急程度

**转译规则**：

|---------|------|

| **版式** | 表格 或 时间线卡片（按时间分短期/中期/长期） |
| **焦点** | 高风险项/短期项（陶土色标记） |

```
页 1（风险表）：
```

```
│   ├── 短期（1-2 周）×2-3 项，浅粉背景
│   ├── 中期（1-2 月）×2-3 项，浅灰背景

```

**正反示例**：

| 按等级/时间排序 | 随机顺序 |
| 短期风险用浅粉背景 | 所有时间段同样背景 |

## 📋 质量自检清单（输出前强制回答）

> **使用方法**：渲染完成前，必须回答以下 5 个问题。任何一个回答"否"，都需要重新编辑。

### 第一层：内容编辑检查

- [ ] **核心观点**：能用一句话说清楚这页的核心观点吗？
- [ ] **减法原则**：删掉了所有不影响核心观点的文字吗？
- [ ] **信息密度**：视觉块数量≤7 个吗？

### 第二层：设计判断检查

- [ ] **密度选择**：简约/标准/丰富 的选择有明确理由吗？
- [ ] **版式匹配**：为什么用这个版式（表格/对比/卡片）？内容特征匹配吗？
- [ ] **焦点唯一**：陶土色只用在一处吗？这处是最重要的信息吗？

### 第三层：视觉执行检查

- [ ] **顶部装饰线**：2.4px 陶土色线存在吗？
- [ ] **页码徽章**：非封面页有右下角圆形页码吗？
- [ ] **无阴影无渐变**：检查所有元素，有违规吗？
- [ ] **文字对比度**：所有文字能清晰阅读吗？
- [ ] **视口适配**：100vh, overflow: hidden 吗？

### 自检评分

| 层级 | 总分 | 得分 | 判定 |
|------|------|------|------|
| 内容编辑 | 3 分 | __ | ≥2 分通过 |
| 设计判断 | 3 分 | __ | ≥2 分通过 |
| 视觉执行 | 5 分 | __ | ≥4 分通过 |

**总计**：___ / 11 分 → **≥8 分可发布，<8 分需要返工**
