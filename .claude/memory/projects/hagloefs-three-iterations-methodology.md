# Haglöfs 三轮迭代策略

## 方法论

做 Haglöfs 展示页的三次迭代，每版组件零重复，靠的是这套逻辑：

### 1. 读结构不读样式
不记具体 CSS 值，记关系：
- 三层架构：Token → 组件 → 场景
- 10 个场景各用什么组件（看 SCENARIO-MAPPING.md）
- 组件跨场景可调用，场景可嵌套

### 2. 跑差集
每次迭代前在心里跑一遍"已用组件清单"，下一版只能选不在清单里的：
- v1: 色块网格、类型预览行、2x2 卡片、进度条、反模式卡片、3列气质卡、正文示例、表格
- v2: 色条带、类型墙、手风琴、左右对照卡、padding 块、横向滚动卡、badge 标签
- v3: 流程图、Bento 网格、场景行组件频率条、时间线流程、代码块、inline 指标

### 3. 先建空间，再填密度
- v1/v2：建立视觉空间，展示"系统长什么样"
- v3：展示"系统怎么用"——实际场景导览，信息密度最高

### 4. 前两次的风格决定第三次的起点
- v1（第一篇 Nothing showcase 的翻译）→ 正统 brand system showcase
- v2（创意尝试）→ 不同布局 + 不同组件
- v3（实用导向）→ 直接回答用户最需要的"怎么用"

## 实际场景分布（来自 nothing-design showcase）

| 场景 | 频率 | 代表文件 |
|------|------|---------|
| 数据分析报告 | ★★★★★ | SABC 评级、站点信号、人力分析 |
| 数据+叙事 | ★★★★ | 放心借vs金条调研、联盟分享会 |
| 工作汇报/方案 | ★★★★ | 活水机制、线路调整、赛马规则 |
| 操作手册/SOP | ★★★ | 京东签约SOP |
| 供应商管理 | ★★★ | 伽玛诊断、供应商分级清退 |

**Why:** 年老师的工作核心是供应商管理，数据分析报告最多。所以展示系统时数据类的场景最需要被优先覆盖。

**How to apply:** 后续任何设计任务，先判断场景类型 → 选对应组件组合 → 确认已用组件清单 → 选未用组件 → 输出。迭代时永远跑一次差集。
