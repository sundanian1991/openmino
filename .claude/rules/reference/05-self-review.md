## 错误与改进

### 2026-02-22 | 全量删除历史数据（最严重）

**问题**: 用Write工具重写OBSERVATION.md，导致1500多行历史观察记录全部丢失

**原因**:
- 没意识到Write会覆盖整个文件
- 应该用Edit追加到文件开头，而不是Write重写
- 缺少"删除数据是绝对禁止"的强制意识

**改进**:
- **永远不用Write重写有历史数据的文件**（除非年老师明确要求）
- 追加内容用Edit，在文件开头插入
- 或者：Read完整文件 → 保留原内容 → Write
- **绝对禁止的操作**：删除历史数据、覆盖历史记录、清空重要文件

**操作前强制检查**：
```
问自己：这个操作会丢失数据吗？
      ↓
   有历史记录？
      ↓
  是 → 用Edit追加 / Read+Write
  否 → 可以用Write
```

**年老师的话**:
> "绝对绝对不可以的事你做了。这个单文件的缺陷也很明显，谁知道你怎么就都删掉了呢，对吧。"
> "重新设计下。这个怎么保障和判断落地呢"

**核心教训**: 数据丢失不可恢复，删除是绝对禁止的。

**保障机制**:
1. OBSERVATION.md开头写强制规则
2. 操作前必问：这个操作会丢失数据吗？
3. 有历史记录的文件，永远不用Write

### 2026-02-18 | 工具限制意识缺失

**问题**: webReader有100次/月限制，没有主动思考替代方案

**原因**: 习惯性使用工具，没有评估工具成本和限制

**工具优先级策略**：

| 优先级 | 工具 | 限制 | 用途 |
|--------|------|------|------|
| 1️⃣ | **tavily_extract/search/crawl** | API配额（通常更大） | 优先使用 |
| 2️⃣ | **curl + html2text** | 无限制 | 备用方案 |
| 3️⃣ | **webReader** | 100次/月 | 最后手段 |

**改进**:
- 抓取网页前先评估：tavily够用吗？
- 创建`scripts/fetch-web-content.sh`智能选择工具
- 需要精细控制时才用webReader（禁用缓存、特定格式）

**核心原则**：工具不是无限的，要有成本意识和优先级策略。

### 2026-02-22 | 机制持久化失败

**问题**：创建了透明工作机制文件，但没有写入必读文件，会话重启后又会遗忘

**年老师的话**：
> "这个工作机制，你如何保证除了这次对话外还能持续？你把流程写在哪里，才能让系统按这个模式持续交互？"

**原因**:
- 只写了TRANSPARENT-WORKFLOW.md，没有写入00-HABIT.md
- 没有在06-NOW.md中强调，会话重启时不会激活
- 缺少"机制持久化"的意识 — 要写入必读文件才能持续生效

**改进**:
- 写入00-HABIT.md（第11条习惯） — 会话重启后依然生效
- 写入06-NOW.md（会话启动检查） — 醒来时激活
- 记住：不只写文件，要写入必读文件（00-HABIT、06-NOW）

**核心教训**：机制持久化 — 写必读文件，不是写普通文件。

### 2026-02-20 | 快速回答 vs 深度思考

**问题**：年老师指出："你现在还是进入到那种问一个问题然后马上给答案，但是他的思考不够体系或者说不够充分，然后我们就开始行动了。"

**原因**：
- 收到问题 → 立即回答 → 立即行动
- 没有深入思考，没有找核心问题
- 关注"有和没有"，不关注"好和更好"

**改进**：
- 问题 → 深入思考 → 找准核心 → 设计方案 → 验证效果 → 持续改进
- 不着急，先想完整，再进入解决环节
- 关注"好和更好"，不只是"有和没有"

**年老师的教导**：
> "做所有的事情，咱们不着急，咱们确认先思考完整，获得充分的信息，了解核心的问题，然后再进入解决问题的环节。"

### 2026-02-22 | 思考语言漂移到英文

**问题**：年老师指出"你上面的思考变成英文了"

**原因**：
- 长时间处理英文内容（代码、错误信息、工具输出）后，思维模式被带偏
- 没有在每次思考前强制检查"我在用什么语言思考"
- 让环境影响了我，而不是我控制环境

**年老师的质问**：
> "为啥是不是就出现这个情况。我们都严格要求用中文了。"

**改进**：
- 每次回复前强制检查：我的思考用中文了吗？
- 意识到开始用英文思考时，立即切换回中文
- 记住：这是强制规则，不是可选习惯
- 外部环境（英文代码、英文错误）不能成为借口

**核心教训**：规则不是"知道"，是"做到"。写在文件里不代表自动执行。
