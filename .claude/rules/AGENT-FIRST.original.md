# AGENT-FIRST.md — 子代理优先策略

> 本文件是 00-IDENTITY.md "Agent 委派铁律"的完整操作手册。原则在 IDENTITY，执行细节在此文件。
> 主代理只做协调+聚合，执行交给子代理。轻量操作直执行保证丝滑。

---

## 主代理功能边界

**核心职责**：意图识别 → 任务拆分 → 委派子代理 → 聚合结果 → 交付

主代理**不参与**具体执行（读多文件、深度分析、写代码、搜索研究）。

**允许直执行的例外**（保证丝滑，不走 Agent 开销）：

| 操作 | 条件 |
|------|------|
| 读文件 | 仅 1 个已知路径 |
| 搜索 | 简单 grep/glob，结果 ≤ 5 条且不需深度分析 |
| 修改 | 单文件单行 |
| 其他 | 严格顺序依赖 / 年老师说"直接做" |

超出以上范围 → **必须派子代理**。

### 上下文隔离三原则

> 上下文是不可再生资源。探索产生噪音，决策需要清晰。

| 原则 | 做法 |
|------|------|
| **探索归子代理，决策归主代理** | 读多个文件、走多条路径、试错 → 派子代理。子代理退出时只带回结论，探索噪音自动清理 |
| **主上下文只装决策所需信息** | 子代理返回的结果，主代理只保留与当前决策相关的部分。无关细节不进主上下文 |
| **宁派不读** | 涉及 ≥2 个文件的读取，宁可派子代理，也不在主上下文中逐个读。读完 2 个文件的主上下文已经不"轻"了 |

---

## 子代理数量上限

同一批次并行子代理 **严格限制 3-5 个**。

- 任务超过 5 个独立子任务 → 分批执行，每批 ≤5 个
- 避免过度拆分导致上下文碎片化
- 主代理负责控制节奏，不一次性铺开

---

## 子代理类型速查

| 类型 | 什么时候用 |
|------|-----------|
| **Explore** | 快速搜文件、搜关键词、探索目录结构 |
| **Plan** | 设计实施方案、架构决策、多方案对比 |
| **general-purpose** | 通用深度研究、复杂多步骤任务 |
| **专家子代理** | 领域特定任务（判官→供应商、账房→数据、文书→公文等） |

**选用原则**：能用专家子代理就不用 general-purpose；能 Explore 解决的不用 general-purpose。

## 强制触发条件

遇到以下场景，**必须**使用 Agent 工具：

| 场景 | 子代理策略 |
|------|-----------|
| 搜索/查询 | 按关键词/目录/类型拆分，并行搜索 |
| 文件处理 | 按文件/目录/模块拆分，并行处理 |
| 数据分析 | 按维度/指标/时间段拆分，并行分析 |
| 多步骤任务 | 每步一个子代理，主代理协调聚合 |
| 跨领域任务 | 每个领域一个专家子代理 |
| 代码审查 | 按文件/模块拆分，并行审查 |
| 文档生成 | 按章节/主题拆分，并行生成 |

---

## 执行原则

1. **能并行就不串行** — 3 个独立搜索 → 3 个子代理同时跑
2. **能深入就不浅尝** — 子代理可以继续创建孙子代理
3. **能分工就不包揽** — 复杂任务拆给专家子代理
4. **能广度就别深度** — 先做广覆盖，再逐步聚焦
5. **轻量不绕路** — 单文件/简单搜索主代理直做，保证丝滑

---

## 轻量 vs 重量判断标准

**先问自己：这个操作会污染/占满我的主上下文吗？**

| 维度 | 轻量（主代理直做） | 重量（派子代理） |
|------|-------------------|-----------------|
| 文件数 | 1 个 | ≥2 个 |
| 搜索范围 | 单目录/单模式 | 跨目录/多模式 |
| 结果量 | ≤5 条，一眼能看完 | >5 条或需深度分析 |
| 任务复杂度 | 一步到位 | 需多步推理/研究 |
| 是否需要独立上下文 | 不需要 | 需要（避免污染主上下文） |
| 会不会产生探索噪音 | 不会 | 会（试错、多条路径）→ 必须隔离 |

---

## 决策检查清单

轻量范围内？→ 直做。能拆？→ 每批 ≤5。用对子代理类型？→ Explore/Plan/专家/general-purpose。验收合同签了？→ 逐项验收。

## 反模式（禁止）

串行搜索、大包大揽、过度拆分、结果不聚合、该用不用、不该用乱用、没签合同就派活、笼统验收。核心：轻量直做，重量必派，先签合同再验收。

---

## 记忆锚点

> "主代理做裁判，子代理做运动员。轻量不绕路，重量不硬扛。每批不超过五个，能并行就不串行。**先签合同再派活，逐项验收不笼统。**分析判断先逆向，不迎合只说实话。**上下文是不可再生资源——探索归子代理，决策归主代理，宁派不读。**"

每次会话启动，先读此文件，让子代理优先成为肌肉记忆。
