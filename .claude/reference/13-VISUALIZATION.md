## 核心原则

**说清楚 > 做漂亮**。骨架够了，不要加肉。

### 三条硬规则（每次出图必须遵守）

| # | 规则 | 做法 |
|---|------|------|
| 1 | **一个图一个核心洞察** | 标题写结论（"客诉率骤降 37%"），不写描述（"客诉率趋势"）。其余元素灰化，高亮 ≤10% |
| 2 | **颜色是信号不是装饰** | 默认 Warm ramp，单图 ≤2 ramp。先灰后彩，有目的地用色 |
| 3 | **每个元素必须有存在理由** | 去掉任何一个，信息还在吗？在就删。去噪是出图前最后一步 |

### 色阶克制原则

| 约束 | 说明 |
|------|------|
| **默认 Warm** | 对话框 widget 默认用 Warm ramp |
| **单图 ≤2 ramp** | 默认组合 Warm + Stone。语义需要时最多加至 3 个 |
| **先用深浅，不跨色** | 同一 ramp 的 50/300/500/800 四层已够表达主次 |
| **跨色必须有语义理由** | Warm=主数据，Stone=基准线，Teal=正面，Coral=负面 |

### 色块内文字规则

| 底色 | 文字颜色 |
|------|----------|
| 50 / 100 浅色底 | 同 ramp 的 700 或 800 |
| 300+ 深色底 | 白色 `#fff` |

## 数据 → 图型映射

> 不要看到数据就选图。先问：受众需要**理解什么**？

| 想表达 | 数据特征 | 推荐图型 | 备选 | 不该做的信号 |
|--------|---------|---------|------|-------------|
| 趋势变化 | 含时间字段 | 折线图 | 面积图 | 8+ 条线意面化 → 改用单线高亮或小多图 |
| 分类对比 | 类别 + 数值 | 水平条形图 | 纵向柱状图（≤5 项） | 15+ 类别 → 取 Top 10，其余归"其他" |
| 占比构成 | 分类 + 总和有意义 | 堆叠条形图 | 100% 堆叠条 | 饼图/环形图默认禁止（>3 分类） |
| 分布集中 | 数值密集 | 直方图 | 箱线图 | 散点图密度过高 → 加密度轮廓 |
| 关联离群 | 两个数值字段 | 散点图 | 气泡图 | 散点无规律 → 改用表格 |
| 多维对比 | ≥3 维度 | 雷达图 | 平行坐标（探索用） | 解释场景不用平行坐标 |
| 流入流出 | 源-目标-值 | 桑基图 | 堆叠面积图 | 20+ 连线节点堆积 → 聚合中间层 |
| 转化递减 | 阶段数值递减 | 漏斗图 | 柱状图排序 | 非递减数据不用漏斗 |
| 目标达成 | 实际 vs 目标 | 子弹图/对比条 | 仪表盘 | 双 Y 轴暗示虚假相关，禁止 |
| 时间打卡 | 日历热力 | 日历图 | 热力图 | 非日历数据不用日历图 |
| 层级结构 | 父子关系 | 树图 | 矩形树图 | 层级深度 >4 不做径向树 |

**铁律**：1-3 个关键数字直接写大字，不要画图。

## Chart.js 实操作法

### Chart.js 配色速查（Canvas 不能用 CSS 变量，必须用 hex）

| 色阶 | borderColor | bg rgba | 用途 |
|------|-------------|---------|------|
| Warm | `#c26d3a` | `rgba(194,109,58,0.1)` | 主数据 |
| Teal | `#2e8b6e` | `rgba(46,139,110,0.1)` | 正面指标 |
| Coral | `#c25030` | `rgba(194,80,48,0.1)` | 负面/预警 |
| Stone | `#857d74` | `rgba(133,125,116,0.1)` | 基准线/灰化 |
| Sky | `#3a7ab8` | `rgba(58,122,184,0.1)` | 信息 |

### 手法 1：全线灰 + 单段彩色高亮（segment 分段着色）

> 用 `segment` 配置，实现前半段灰、后半段彩的效果。

```js
{
  data: [4.1, 3.8, 3.5, 3.4, 3.3, 3.2, 3.2, 3.1, 1.8, 1.7],
  borderColor: '#ada599',
  borderWidth: 1.5,
  segment: {
    borderColor: function(ctx) {
      return ctx.p0DataIndex < 8 ? '#ada599' : '#c26d3a';
    },
    borderWidth: function(ctx) {
      return ctx.p0DataIndex < 8 ? 1.5 : 3;
    }
  },
  pointRadius: function(ctx) {
    var i = ctx.dataIndex;
    return i === 7 ? 4 : (i === 8 ? 6 : 0);
  },
  pointBorderColor: function(ctx) {
    return ctx.dataIndex === 8 ? '#c26d3a' : '#ada599';
  },
  pointBorderWidth: function(ctx) {
    return ctx.dataIndex === 8 ? 3 : 1.5;
  },
  tension: 0.25
}
```

**要点**：
- `segment.borderColor` 用 `p0DataIndex` 判断切换点
- `pointRadius` 动态控制关键节点大小
- 切换点的点加粗加色，其余为 0

### 手法 2：实线 = 实际，虚线 = 预测

> 用两个 dataset 实现，null 值断开连接。

```js
{
  label: '实际',
  data: [89, 86, 84, 82, 79, 76, null, null, null],
  borderColor: '#857d74',
  borderWidth: 2,
  pointRadius: 0,
  tension: 0.3
},
{
  label: '预测',
  data: [null, null, null, null, null, 76, 74, 72, 70],
  borderColor: '#857d74',
  borderWidth: 2,
  borderDash: [6, 4],
  pointRadius: 4,
  tension: 0.3
},
{
  label: '清退阈值',
  data: [75, 75, 75, 75, 75, 75, 75, 75, 75],
  borderColor: '#c25030',
  borderWidth: 1.5,
  borderDash: [4, 3],
  pointRadius: 0
}
```

### 手法 3：Canvas 上叠加 HTML 标注

> Chart.js 不支持内置复杂标注时，用 CSS `position: absolute` 在 canvas 容器上叠加。

```html
<div style="position:relative;width:100%;height:240px">
  <canvas id="myChart"></canvas>
</div>
<script>
// ... chart init ...
setTimeout(function(){
  var p = document.getElementById('myChart').parentElement;
  var lbl = document.createElement('div');
  lbl.innerHTML = '<span style="font-size:10px;color:#c26d3a;font-weight:600">9月方案上线</span>';
  lbl.style.position = 'absolute';
  lbl.style.left = '68%';
  lbl.style.top = '12%';
  p.appendChild(lbl);
}, 100);
</script>
```

**要点**：
- 标注放在 canvas 的 **父容器** 上
- 用 `position: relative` 包裹 canvas
- 标注用 `position: absolute` + `left/top` 百分比定位
- `setTimeout` 确保 chart 渲染后再加标注

## Widget 版式硬规则

### 间距

| 场景 | 间距 |
|------|------|
| 标题到内容 | 12px |
| 副标题到标题 | 6px |
| 卡片之间 | 12px |
| 卡片内边距 | 16px |
| 图表到标注 | ≥12px |
| 柱顶到数值标签 | ≥8px |
| 柱子间距 | ≥20px |

### 边框与圆角

| 元素 | 圆角 | 边框 |
|------|------|------|
| 卡片 | 10px | 1px solid var(--widget-border) |
| SVG 节点 | 12px | 1.5px 描边，Warm 500 |
| SVG 连线 | — | 1.5px，Stone 300 |
| 柱形图柱子 | 6px top | 无 |

### 版式骨架（每张图必须包含）

```
标题（15px, 600, Warm 800）
副标题（11px, 400, Stone 500）← 可选，含时间范围+对象

[图表区]

脚注（10px, 400, Stone 400）← 可选，数据来源
```

### 硬约束（不可违反）

| 约束 | 原因 |
|------|------|
| 禁止渐变、阴影、模糊、发光 | 流式渲染闪烁 |
| 禁止 animation | 设 animation:false |
| 禁止 fetch/ajax | CSP 阻断 |
| 禁止 position:fixed | iframe 高度自适应 |
| **文字和 widget 分两条回复** | 混排导致截断乱码 |
| 禁止 700+ 字重 | 设计体系只有 400/600 |
| 禁止 emoji | 用 SVG/CSS 替代 |
| 单图 ≤4000 字符 | 超过拆多个 |
| 数据必须内联 | 网络被 CSP 阻断 |

## 出图前检查清单（6 条）

> 出图前扫一遍。任何一条不通过，不做图。

- [ ] **标题是结论**（"客诉率骤降 37%" 不是 "客诉率趋势"）
- [ ] **高亮 ≤10%**（只突出 1 个关键点，其余灰化）
- [ ] **颜色 ≤2 ramp**（默认 Warm + Stone，跨色有语义理由）
- [ ] **无禁止项**（饼图/双Y轴/3D/渐变/阴影/emoji）
- [ ] **有副标题或脚注**（时间范围 + 数据来源）
- [ ] **闭眼再睁眼 2 秒**（第一眼落在你想让观众看的地方吗？）

## 数据图表专属规则

| 规则 | 说明 |
|------|------|
| **柱状图零基线** | Y 轴必须从 0 开始，禁止截断 |
| **禁止饼图/环形图** | >3 分类用水平条形图替代。唯一例外：2-3 分类且一个 >70% |
| **禁止双 Y 轴** | 暗示虚假相关。替代：上下分图共享 X 轴 |
| **预测与实际区分** | 实线=实际（粗），虚线=预测（细），最后实际点用锚点标注 |
| **拐点标注原因** | 不只写数字，写"04/15 新系统上线 → 效率↑37%" |
| **直接标注不要图例** | 在线尾/柱旁直接写名称，不靠图例 |
| **分类按意图排序** | 不按字母，按最优/最差/最常用排序 |

### 意面图解构（折线图 >5 条线）

| 策略 | 做法 | 适用 |
|------|------|------|
| 单线高亮 | 灰化所有，每次只高亮 1 条 | 现场汇报 |
| 小多图纵向 | 共享 X 轴，上下排列 | 关注单类别趋势 |
| 小多图横向 | 共享 Y 轴，左右排列 | 关注跨类别量级比较 |

**铁律**：所有小多图轴范围必须一致。

### 属性优先级

| 层级 | 元素 | 样式 |
|------|------|------|
| 最高 | 标题、核心标注 | 600、Warm 主色、大字号 |
| 高 | 关键数据线 | 3px 粗线、Warm 500 |
| 中 | 对比数据线 | 1.5px Stone 300 |
| 低 | 背景数据 | 1px 灰色虚线 |
| 最低 | 脚注 | 10px、Stone 400 |
