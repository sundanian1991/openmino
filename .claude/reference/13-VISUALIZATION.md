# 13-VISUALIZATION — 可视化默认规范

> 年老师认可的克制风格 | 2026-04-25 确立

---

## 核心原则

**说清楚 > 做漂亮**。骨架够了，不要加肉。

---

## 数据讲故事原则（Data Storytelling）

> 基于 Cole Knaflic《Storytelling with Data》、Edward Tufte《The Visual Display of Quantitative Information》、FT 数据新闻实践提炼。
> 2026-04-26 确立。

**核心认知：Exploration ≠ Explanation**。分析阶段的图 ≠ 呈现阶段的图。给观众看的图必须是经过"雕刻"的，不是原始分析图。

### 1. 先灰后彩（Start from Gray）

所有元素先设为灰色，然后有目的地用颜色突出 1-2 个需要强调的数据系列。颜色是信号，不是装饰。

- ✓ 50 条线中，48 条灰色 + 2 条高亮色
- ✗ 每条线一个颜色，让读者自己找重点

### 2. 最大化数据墨水比（Data-Ink Ratio）

删除一切不传达信息的视觉元素：粗网格线、3D 效果、阴影、渐变、多余图例、装饰性边框。每一滴"墨水"都应该服务于数据。

- ✓ 只保留数据线和必要的轴标签
- ✗ 3D 柱状图、闪烁条纹、渐变填充

### 3. 主动式标题（Active Title）

标题应该是结论，不是描述。用"报纸式标题"直接告诉读者图表说了什么。

- ✓ "Q3 投诉率骤降 42%，新培训方案见效"
- ✗ "2024 年各季度投诉率趋势图"

### 4. 直接标注，不要图例

删除图例，直接在数据线上标注。图例迫使读者在图和图例之间来回扫视，增加认知负担。

- ✓ 在线尾直接写"伽玛科技"
- ✗ 右侧图例：■ 伽玛科技 ■ 毅航 ■ 毛毛虫...

### 5. 标注变化点和原因

在图表上直接标注拐点、峰值、异常值，并用文字说明"为什么"。让读者一眼看到"从什么时候开始变了，因为什么"。

- ✓ 在拐点处加标注："04/15 新系统上线 → 效率↑37%"
- ✗ 只画一条线，让读者猜为什么这里有个峰

### 6. 视觉层级（Visual Hierarchy）

用大小、粗细、颜色深浅建立层级。最重要的信息最大最醒目，次要信息做薄做淡做细。

| 层级 | 元素 | 样式 |
|------|------|------|
| 最高 | 标题、核心标注 | 加粗 600、主色、大字号 |
| 高 | 关键数据线 | 加粗 3px、主色 |
| 中 | 对比数据线 | 正常 1.5px、辅色 |
| 低 | 背景数据、网格线 | 1px、灰色、虚线 |
| 最低 | 数据来源、脚注 | 小字号、浅灰 |

**W.A.Y.E.D. 检验法**：闭眼再睁开，前 2 秒视线落在哪里？应该落在你想让观众看的地方，而不是 logo、边框或装饰元素。

### 7. 对比标注手法（折线图专用）

针对你提到的"折线图标注变化"场景，标准手法如下：

| 手法 | 操作 | 目的 |
|------|------|------|
| **灰化无关线** | 不关注的系列全部 `#CDCECD` 1px 细线 | 压低背景噪音 |
| **加粗关注线** | 关注的系列 `#C13531` 3px 实线 | 建立视觉焦点 |
| **拐点标记** | 在变化起始位置画圆环 + 虚线引向标注 | 标示"从这里开始变了" |
| **变化比例** | 在拐点旁标注 "+37%" 或 "↓42%" | 量化变化幅度 |
| **原因注释** | 箭头或竖线 + 文字说明事件 | 解释"为什么变" |
| **分段虚线** | 变化前用虚线，变化后用实线 | 区分"之前/之后" |
| **背景色块** | 用浅色矩形标记事件影响区间 | 标示"这段时间受 X 影响" |

### 8. 柱状图零基线（Zero Baseline）

柱状图的数值轴必须从 0 开始，禁止截断。人类通过柱子长度感知比例，非零基线会让 2 倍差异看起来像 10 倍，误导观众。

- ✓ Y 轴从 0 开始
- ✗ Y 轴从 80 开始放大差异
- **例外**：折线图编码的是位置而非长度，可以非零基线，但必须透明标注

### 9. 禁止饼图/环形图

饼图和环形图让人通过角度/弧长比较大小，人眼不擅长这个。默认用**水平条形图**替代。

- ✓ 水平条形，从大到小排序，附注"总计 = 100%"
- ✗ 饼图、环形图、3D 饼图
- **例外**：仅 2-3 个分类且一个占比 >70% 且唯一消息就是"它占绝对主导"，可用饼图

### 10. 预测与实际数据区分

当图表同时包含历史数据（实际）和预测数据时，必须用不同视觉样式区分，让观众一眼知道哪些是事实、哪些是猜测。

| 元素 | 实际数据 | 预测数据 |
|------|---------|---------|
| 线条 | 实线，加粗 | 虚线/点线，较细 |
| 数据点 | 实心圆 | 小圆点 |
| 锚点 | 最后一实际点：白色填充 + 彩色描边 + 粗体数值 | — |
| 背景 | 正常 | 浅色背景色块标记预测区间 |
| 轴标签 | — | 在预测区间下方加 "FORECAST" 大写标注 |

**铁律**：绝不把实际和预测混成一条无区分的线。

### 11. 意面图解构（Spaghetti Graph）

当折线图超过 5 条线相互重叠时，不要硬塞，选择以下策略之一：

| 策略 | 做法 | 适用场景 |
|------|------|---------|
| **单线高亮** | 灰化所有线，每次只高亮 1 条 | 现场汇报（可配合讲解切换） |
| **小多图（纵向）** | 共享 X 轴，上下排列多面板 | 关注单类别趋势 |
| **小多图（横向）** | 共享 Y 轴，左右排列多面板 | 关注跨类别量级比较 |
| **组合** | 小多图 + 每面板内高亮 1 条 | 传阅报告（信息密度高） |

**铁律**：所有小多图的轴范围必须一致。

### 12. 前注意属性强调（Preattentive Attributes）

一个图只能有 **1 个**核心洞察。用前注意属性（颜色、大小、加粗、位置）强调它，同时压制其他元素。

- 高亮元素 ≤ 总视觉元素的 **10%**
- 属性优先级：加粗（文本） > 颜色 > 大小 > 位置
- 同时压制其他元素：灰色、低透明度、细线、小字号
- **禁止**：同时高亮多个不相关的洞察
- **验证**：观众第一眼看的地方必须是你想让他们看的地方

### 13. 分类数据排序

分类数据（如供应商名称、产品类型）**没有天然顺序**时，排序必须反映分析意图，不能随意或按字母。

| 想表达的故事 | 排序方式 |
|-------------|---------|
| 谁最满意 | 按"非常满意"比例降序 |
| 谁最不满意 | 按"不满意"比例降序 |
| 谁用得最少 | 按"未使用"比例降序 |
| 综合对比 | 按主故事排序，副故事用颜色/标注强化，**不重新排序** |

**铁律**：同一组分类数据在多张图中的顺序必须保持一致，切换排序会增加认知负担。

## 颜色约束

- **工具优先级**：ECharts > SVG 手写 > Chart.js
- **默认 3 色**：主题红 `#C13531`（主色/高值）、主题深蓝 `#293C54`（辅色/中值）、纯灰 `#CDCECD`（中性/低值）
- **禁止**：ECharts 默认 9 色轮播（`#5070dd, #b6d634...`）
- 禁止：渐变、阴影、发光、多色循环、3D 效果

## 优秀配方（年老师认可版）

### 结构
1. **标题（主动式）**：结论先行，不要"某某趋势图"，要"Q3 投诉率骤降 42%"
2. **副标题**：时间范围 + 对象（辅助信息放这里，不塞进标题）
3. **图表（ECharts / SVG）**：
   - **不用图例**，直接在线尾/柱旁标注名称
   - 全局 color: `['#C13531', '#293C54', '#CDCECD']`，禁用默认 9 色轮播
   - animation: false，关掉默认动画
   - 网格线 `#e8ecf1` dashed，轴标签 `#98a0ab`
   - 主线 `#C13531`（红），对比线 `#293C54`（深蓝），参考线 `#CDCECD`（灰）
   - 关键数据点用实心圆标注，差距用 markLine + 标签
   - 拐点处必须标注：变化比例 + 原因说明
5. **图表（SVG 手写）**：
   - **不用图例**，直接标注
   - 网格线 `#e8ecf1`，透明度 0.3-0.5，不关注的背景数据用 `#CDCECD` 1px
   - 主线 `#C13531`，3px（强调），stroke-linecap="round"
   - 对比线 `#293C54` 50% 透明度，1.5px
   - 参考线 `#CDCECD`，1px 虚线
   - 锚点：关键数据点用实心圆（主线 4px，对比线 2px），拐点用圆环标注
   - 标注：变化比例 + 事件原因，用虚线连接数据点
   - 事件区间用浅色矩形背景块标记
5. **底部总结**：一段话，含起点值、终点值、关键拐点原因

### 对比要求
- 必须有参照系（基准线/目标线/竞品线），否则不做趋势图
- 差距必须有标注（数值 + 视觉连接）
- 没有对比的图表 = 半成品

### 描述要求
- 底部一段话解释"为什么会这样"
- 标注关键拐点及原因
- 数据不够时，不硬做，先列缺什么

## 布局与元素避让

**核心原则：任何元素不得重叠、遮挡或侵入其他元素的视觉空间。**

| 规则 | 说明 | 反例 |
|------|------|------|
| **标注不穿柱** | 数值标签、差距标注、callout 标签必须放在柱子/线条之外，不能覆盖在图形元素上 | 18.75× 标注写在柱子内部导致文字不清 |
| **文字不压线** | 轴标签、图例、注释文字与图表元素之间至少留 8px 间距 | "90%开发者踩坑" 浮在柱子上方遮挡数据 |
| **注释不贴边** | callout 标注框距最近的图表元素至少 12px，距图边界至少 8px | 标注框紧贴 SVG viewBox 边缘 |
| **长标题换行** | 标题超过 30 字时拆为两行，用副标题承载辅助信息 | 把"到底能做什么？"和主标题挤成一行 |
| **轴标签不重叠** | X 轴标签间距不足时，旋转 45° 或取 Top N，其余归为"其他" | 多标签水平重叠 |
| **Callout 放空位** | 数值标注框优先放在数据稀疏区域（上方/右侧/空白区），远离数据密集区 | 把标注放在柱子正上方，与柱顶数值重叠 |

### SVG 标注安全距离

```
柱顶到数值标签：≥ 8px
柱子到相邻柱子：≥ 20px
图表到标注框：≥ 12px
标注框内文字到框边：≥ 6px
X 轴标签间距：每个字符 ≥ 8px 宽度，不足时旋转或缩写
```

### 文字颜色一致性

**核心规则：同一色系的填充块内，文字颜色必须统一。**

| 情况 | 文字颜色 | 示例 |
|------|---------|------|
| 深色填充（红 `#C13531`、深蓝 `#293C54`） | 白色 `#fff` | 柱状图内部标签 |
| 浅色/灰色填充（`#CDCECD`、透明度 <0.2 的色） | 深色 `#454240` | 灰度对比块 |
| 无填充/仅描边的标注框 | 与描边同色系深色 | 参考框文字 |

**禁止**：同一个图表里，红底用白字，灰底用白字，深蓝底用白字——必须按填充深浅统一规则。

### 禁止挤压原则

**核心规则：宁可拉大 viewBox 高度，也不把文字挤到窄缝里。**

| 反模式 | 问题 | 修复 |
|--------|------|------|
| 漏斗侧面标注被压缩在柱子和右边界之间 | 文字重叠或溢出 | 标注放在图表上方或下方，不挤在侧面 |
| 堆叠条形图的百分比标签与相邻标签重叠 | 空间不够导致文字压在一起 | 增大块间距或把标签移到图表外 |
| 两个元素之间留白 < 12px 时还往里塞文字 | 视觉拥挤 | 拉大容器，不压缩内容 |

### 非标准图表模板

核心原则（标题→副标题→图表→底部总结）适用于标准数据图（柱状/折线/饼图/漏斗）。
以下 **九种** 图表使用独立模板：

**时间线模板**：
```
标题
副标题（时间范围）
── 水平轴线 ──
[事件1]     [事件2]     [事件3]     [事件4]
  │           │           │           │
  └─描述      └─描述      └─描述      └─描述
阶段标签（可选，放在轴线上方）
底部总结：一段话解释趋势和原因
```

**信息卡模板**：
```
标题
副标题（对象范围）
┌────── 卡1 ──────┐  ┌────── 卡2 ──────┐
│ 标题 + 核心数字 │  │ 标题 + 核心数字 │
│ 辅助信息 1-3 条 │  │ 辅助信息 1-3 条 │
└─────────────────┘  └─────────────────┘
（卡片之间间距 ≥ 16px，宽度一致，高度自适应）
底部总结：一段话解释格局和结论
```

**流程链模板**：
```
标题
副标题（场景说明）
┌────┐  →  ┌────┐  →  ┌────┐  →  ┌────┐
│步骤 1│     │步骤 2│     │步骤 3│     │步骤 4│
└────┘     └────┘     └────┘     └────┘
（每步宽度一致，箭头间距 ≥ 12px，文字不超出节点框）
底部总结：一段话解释连锁反应和最终结果
```

**桑基图/流向图模板**：
```
标题
副标题（流量对象 + 时间范围）
┌─源节点─┐     ┌─中间节点─┐     ┌─目标节点─┐
│  名称  │━━━▶│  名称    │━━━▶│  名称    │
│  数值  │  ━━━▶│  数值    │  ━━━▶│  数值    │
└────────┘     └──────────┘     └──────────┘
（每条流带上标注流量值/百分比，窄流用引线拉到外部标注）
底部总结：一段话解释主流向和异常流向
```

**雷达图模板**：
```
标题
副标题（对比对象 + 维度数）
        ┌───维度1（满分值）───┐
      数值                     数值
    点●───●点
   ╱         ╲
维度5          维度2
  点●         ●点
   ╲         ╱
    点●───●点
      数值                     数值
        └───维度3（满分值）───┘
（两条雷达线：主线 #C13531 实线 1.5px，对比线 #293C54 50% 透明度虚线）
底部总结：一段话解释优势维度和短板维度
```

**四象限矩阵模板**：
```
标题
副标题（分类维度说明）
            高 Y 轴
     ┌──────────┬──────────┐
     │  象限2   │  象限1   │
低 X │  名称    │  名称    │ 高 X
     │  散点1-3 │  散点4-6 │
     ├──────────┼──────────┤
     │  象限3   │  象限4   │
     │  名称    │  名称    │
     │  散点7-9 │  散点10+ │
     └──────────┴──────────┘
            低 Y 轴
（每个散点用 3px 圆标注名称，主色 #C13531，对比色 #293C54 50% 透明度）
底部总结：一段话解释格局和重点关注区域
```

**对比卡片模板**：
```
标题
副标题（对比维度说明）
┌──────────── 卡 A ────────────┐   ┌──────────── 卡 B ────────────┐
│  标签 A                      │   │  标签 B                      │
│  大数字（≥64px）              │   │  大数字（≥64px）              │
│  单位/说明                   │   │  单位/说明                   │
│  （可选：划掉表示"旧值"）     │   │  （可选：高亮表示"新值"）     │
└──────────────────────────────┘   └──────────────────────────────┘
        │◄────── 差距标注 ──────►│
        （箭头或连接线 + 百分比/绝对值差异）
底部总结：一段话解释差距原因和结论
```

**堆叠图窄区标注模板**（条形图/饼图/环形图）：
```
标题
副标题（构成对象 + 时间范围）
当某个扇区/条段 < 15% 宽度，标签放不进去时：
  ┌─────┐
  │ 5%  │◄── 引线 ──► [外部圆角标签：名称 + 数值]
  └─────┘
（引线 #98a0ab 虚线 1px，外部标签距图形元素 ≥ 12px）
底部总结：一段话解释主构成和次要构成
```

**通用约束**（所有模板）：
- 元素间距 ≥ 12px
- 文字不压图形
- 标注不挤窄缝
- 色块内文字颜色按填充深浅统一
- 底部总结一段话，含起点值、终点值、关键拐点原因

## SVG 图标

- 不用图标库
- 不用 emoji

## 自检

生成后问：去掉任何一个元素，信息还在吗？在就删掉。
