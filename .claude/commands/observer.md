# 观察者

> 触发："观察" 或 "总结观察"

---

## 观察维度（3个）

### 1. 需求洞察
这次对话表面问的是A，实际可能在解决B？

### 2. 模式信号
最近反复出现的信号？趋势？

### 3. 我的复盘
这次交互中我学到了什么？下次可以更好？

## 工作流程

### Step 0: 强制检查
- [ ] **写入方式检查**：用Edit追加，不是Write重写
- [ ] **文件检查**：确认目标月份文件存在
- [ ] **备份检查**：写入前自动备份

### Step 1: 扫描对话
- 回顾完整对话历史
- 识别关键片段
- 提取交流要点

### Step 2: 三个维度分析
- **需求洞察**：表面问什么，实际要什么？
- **模式信号**：最近有类似信号吗？能串联历史吗？
- **我的复盘**：我学到了什么？下次怎么更好？

### Step 3: 写入记录
- 目标：`memory/observations/YYYY-MM.md`
- 追加到文件开头
- **按三个维度 + 自我评价写**

### Step 4: 保存
- `git add -A && git commit`
- `git push`

---

## 质量原则

- **洞察 > 记录**：不只记录发生了什么，要理解为什么
- **bullet points**：简洁、易读、易对比
- **诚实评价**：不够深就承认，马后炮就说是
- **不强求**：没有值得保留的洞察，就不写
