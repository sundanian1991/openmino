# Workflow: 内容创作

> 半自动化全流程方案 - 创意生成→结构规划→内容创作→视觉呈现→发布优化

**核心理念**：好内容 = 有观点 + 有结构 + 有呈现

---

## 为什么需要内容创作工作流

**问题**：
- 不知道写什么
- 有想法但没结构
- 写了但没人看
- 视觉呈现不专业

**解决**：
- 系统化生成创意
- 结构化组织内容
- 优化传播效果
- 提升视觉质量

**价值**：
- 效率：从创意到成品不卡壳
- 质量：专业的内容+视觉
- 影响：更多人看到并认同

---

## 工作流程

```
创意生成 → 结构规划 → 内容创作 → 视觉呈现 → 发布优化
   ↓          ↓          ↓          ↓          ↓
brainstorm  writing-  content-  baoyu-    copy-
          plans      creation  cover/     editing
                    writing-   pptx
                    clearly
```

---

## 执行步骤

### 阶段1：创意生成（10分钟）
**调用**：`brainstorming`

**输出**：
- 核心观点：想表达什么
- 目标受众：写给谁看
- 传播价值：读者能得到什么
- 题目方向：可能的角度

**创意框架（RIDE）**：
- **R**aising（提出问题）：什么痛点？
- **I**nterest（激发兴趣）：为什么重要？
- **D**esire（创造欲望）：有什么好处？
- **E**ntry（行动引导）：接下来怎么做？

**交互**：
- 我问你几个问题
- 共同打磨核心观点
- 确定题目方向

---

### 阶段2：结构规划（10分钟）
**调用**：`writing-plans` + `brainstorming`

**输出**：
- 内容大纲：章节安排
- 信息层次：先讲什么、后讲什么
- 关键素材：数据、案例、引用

**内容结构模板**：

**文章类（SCQA）**：
```
1. 情境（Situation）：描述现状
2. 冲突（Complication）：提出问题
3. 问题（Question）：为什么是问题
4. 答案（Answer）：解决方案
```

**教程类（三步走）**：
```
1. 为什么学：价值/意义
2. 怎么做：步骤/方法
3. 实践：案例/练习
```

**观点类（PREP）**：
```
1. 观点（Point）：核心论点
2. 理由（Reason）：为什么
3. 例证（Example）：证据/案例
4. 观点（Point）：总结强调
```

**交互**：
- 我提出大纲方案
- 你确认或调整
- 确定最终结构

---

### 阶段3：内容创作（30-60分钟）
**调用**：`content-creation` + `writing-clearly` + `copywriting`

**输出**：
- 完整内容：可直接发布的版本
- 多版本：长版/短版/摘要版

**写作原则**：
- **简洁**：能一句话不说两句
- **具体**：用数据、案例说话
- **有力**：有观点，不模糊
- **可读**：段落短、标题清晰

**交互**：
- 我按大纲写作
- 你审阅，提修改意见
- 迭代优化

---

### 阶段4：视觉呈现（15-30分钟）
**调用**：`baoyu-cover-image` / `baoyu-infographic` / `pptx`

**输出**：
- 封面图：吸引眼球
- 信息图：可视化关键信息
- PPT：演示版本（如果需要）

**视觉类型选择**：
| 内容类型 | 推荐视觉 |
|---------|---------|
| 观点文章 | 封面图（文字+图形） |
| 数据报告 | 信息图（图表+说明） |
| 教程指南 | 步骤图（流程+说明） |
| 汇报演示 | PPT（完整演示） |

**交互**：
- 你选择视觉类型
- 我生成视觉方案
- 你确认或调整

---

### 阶段5：发布优化（10分钟）
**调用**：`copy-editing` + `seo-audit`

**输出**：
- 标题优化：更吸引点击
- 摘要优化：更易传播
- SEO优化：更易被发现
- 发布建议：时间、渠道

**标题优化技巧**：
- 数字：《7个XX方法》 > 《XX方法》
- 痛点：《为什么XX失败》 > 《XX的问题》
- 利益：《XX能帮你YY》 > 《介绍XX》
- 好奇：《XX的秘密》 > 《XX介绍》

**交互**：
- 我优化标题和摘要
- 你确认最终版本
- 给出发布建议

# 从救火到防火：我的供应商管理方法论

## 每天都在救火

早上9点，A供应商电话：物料延迟
10点，B供应商邮件：质量问题
下午2点，C供应商上门：价格谈不拢

这是我2025年的日常。管理35家供应商，每天就像打地鼠——一个问题刚解决，另一个又冒出来。

我就在想：为什么总是救不完的火？

## 问题的本质

后来我意识到：不是供应商不行，是供应商"太舒适"。

我选的3家长期合作，没有竞争压力，服务越来越差。
我催一下，动一下；不催，就不动。

**问题的本质**：供应商没有危机感，而我也在"舒适区"——熟悉的供应商，沟通成本低，但效果越来越差。

## 转折点：引入竞争

2025年底，我决定引入8家新供应商。

不是替换，是**增量竞争**：
- 同样的业务，给2-3家供应商做
- 谁做的好，后续业务倾斜
- 谁做的差，自然淘汰

3个月后，结果让我意外：

**75%的新供应商进入了季度前三。**

原因很简单：新供应商有危机感，做得更用心。而老供应商看到新对手来了，也开始紧张。

**竞争激活了质量。**

## 防火方法论：五大支柱

基于这次经验，我建立了供应商管理的"五大支柱"：

### 1. 标准体系
- 不只说"你做得不好"，要明确"什么是好"
- 建立《供应商管理手册》作为"宪法"
- 让评估有据可依

### 2. 指标体系
- 16项KPI：质量、成本、交付、服务...
- 数据说话，不靠感觉
- 每季度量化评估

### 3. 监控体系
- 事前监控：不是出问题再看，是提前预警
- 风险画像：识别隐性断裂风险
- 把问题消灭在萌芽

### 4. 汇报体系
- 周报：让问题可见
- 事件通报：让风险透明
- 价值备忘录：让贡献被看见

### 5. 选汰体系
- **核心**：通过竞争激活
- 新供应商进，老供应商有压力
- 后3名有淘汰风险，自然会改进

## 结果验证

引入选汰机制后：

| 指标 | 之前 | 之后 |
|------|------|------|
| 需要我催的供应商 | 80% | 20% |
| 主动改进的供应商 | 10% | 70% |
| 我的日均处理问题 | 10+ | 2-3 |

**最重要的是**：我从"救火队员"变成了"生态经营者"。

## 你可以怎么做

如果你也在管理供应商，可以试试：

**第1步：盘点现有供应商**
- 哪些在"舒适区"？
- 哪些需要竞争激活？

**第2步：引入1-2家新供应商**
- 不是替代，是增量
- 给新供应商机会，给老供应商压力

**第3步：建立季度评估**
- 明确规则：怎么评估？
- 明确后果：后3名会怎样？

**第4步：让规则说话**
- 数据说话，不靠关系
- 该淘汰就淘汰，不手软

## 核心心法

供应商管理不是"管供应商"，是"经营生态"。

让竞争激活质量，让数据驱动决策，让规则替代人情。

从救火到防火，你只需要：引入竞争。
