# Learn 指令

分析当前会话，提取值得保存为技能的可复用模式。

## 触发时机

在会话中任何时刻运行 `/learn`，当你解决了一个非平凡问题时。

## 提取什么

寻找以下模式：

1. **错误解决模式**
   - 发生了什么错误？
   - 根本原因是什么？
   - 什么修复了它？
   - 这对类似错误可复用吗？

2. **调试技巧**
   - 非显而易见的调试步骤
   - 有效的工具组合
   - 诊断模式

3. **变通方法**
   - 库的怪癖
   - API限制
   - 版本特定修复

4. **项目特定模式**
   - 发现的代码库约定
   - 做出的架构决策
   - 集成模式

## 输出格式

在 `~/.claude/skills/learned/[模式名].md` 创建技能文件：

```markdown
# [描述性模式名]

**提取时间：** [日期]
**上下文：** [适用场景简述]

## 问题
[这个模式解决什么 - 具体描述]

## 解决方案
[模式/技术/变通方法]

## 示例
[代码示例（如适用）]

## 使用时机
[触发条件 - 什么情况激活此技能]
```

## 流程

1. 审查会话，寻找可提取的模式
2. 识别最有价值/可复用的洞察
3. 起草技能文件
4. 询问用户确认后再保存
5. 保存到 `~/.claude/skills/learned/`

## 注意

- 不要提取琐碎修复（拼写错误、简单语法错误）
- 不要提取一次性问题（特定API故障等）
- 聚焦于能在未来会话中节省时间的模式
- 保持技能专注 - 一个模式一个技能
