# Workflow: SOP标准化

> 半自动化全流程方案 - 识别场景→萃取经验→编写SOP→验证迭代

**核心理念**：SOP = 经验压缩 + 可复制 + 可迭代

---

## 为什么需要 SOP 工作流

**问题**：
- 同样的工作每次重新想怎么做
- 好的经验没有沉淀下来
- 换个人/换个时间就忘了

**解决**：
- 识别重复场景
- 萃取关键经验
- 编写成 SOP
- 持续迭代优化

**价值**：
- 效率：不用每次重新想
- 质量：标准化的流程
- 沉淀：经验可以传承
- 解放：大脑可以做更重要的事

---

## 工作流程

```
场景识别 → 经验萃取 → 结构设计 → SOP编写 → 验证迭代
   ↓          ↓          ↓          ↓          ↓
Clarify   steal-    planning-  docs-     systematic+
场景      learning   files      writer    debugging
                      brainstorming
```

---

## 执行步骤

### 阶段1：场景识别（5分钟）
**调用**：系统引导 + `brainstorming`

**输出**：
- 场景定义：什么工作？
- 频率评估：多久做一次？
- 复杂度评估：有多少步骤？
- 变化程度：每次差别大吗？
- **决策**：值得做SOP吗？

**判断标准**：
| 维度 | 值得做SOP | 不值得 |
|------|----------|--------|
| 频率 | ≥每月1次 | 偶尔 |
| 复杂度 | ≥3步 | 简单 |
| 变化 | 步骤稳定 | 每次不同 |
| 复用性 | 可他人复用 | 仅个人 |

**交互**：
- 我问你场景细节
- 评估是否值得做SOP
- 如果不值得，直接建议其他方案

---

### 阶段2：经验萃取（15分钟）
**调用**：`steal-learning` + 深度对话

**输出**：
- 关键步骤：核心动作是什么？
- 关键决策：哪些节点需要判断？
- 常见错误：哪里容易出错？
- 最佳实践：有什么技巧？
- 成功标准：怎么判断做好了？

**萃取框架（三行框架）**：
```
看到了什么 → 用在哪儿 → 何时试一次
```

**交互**：
- 你描述这个工作的全过程
- 我问细节，挖隐性经验
- 共同提炼关键要素

**关键问题**：
- "这步为什么要这么做？"
- "如果跳过这步会怎样？"
- "有什么是你做了但别人不知道的？"
- "新手最容易在哪里犯错？"

---

### 阶段3：结构设计（10分钟）
**调用**：`brainstorming` + `planning-with-files`

**输出**：
- SOP结构：章节安排
- 信息层次：哪些先讲、哪些后讲
- 呈现方式：文字/图表/代码？

**标准结构模板**：

```markdown
# SOP: [场景名称]

## 📋 基本信息
- **适用场景**：[什么情况下用这个SOP]
- **前置条件**：[需要准备什么]
- **耗时估算**：[大约需要多久]
- **成功标准**：[怎么判断做好了]

## 🎯 核心步骤

### 步骤1：[步骤名称]
**目的**：[为什么做这一步]
**操作**：
1. [子步骤1]
2. [子步骤2]
**输出**：[这一步的产出]
**注意**：[关键提醒]

### 步骤2：[步骤名称]
[同上]

...

## ⚠️ 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| [问题1] | [原因] | [解决] |
| [问题2] | [原因] | [解决] |

## 💡 最佳实践
- [经验1]
- [经验2]

## 📚 相关资源
- [相关文档]
- [相关工具]
- [相关SOP]

---
**创建时间**：[日期]
**最后更新**：[日期]
**版本**：[v1.0]
```

**交互**：
- 我提出结构方案
- 你确认或调整
- 确定最终结构

---

### 阶段4：SOP编写（20-30分钟）
**调用**：`docs-writer` + `writing-clearly`

**输出**：
- 完整SOP文档
- 可直接使用的版本

**编写原则**：
- **简洁**：能用一句话不说两句
- **具体**：不说"适当调整"，说"调整到X值"
- **可操作**：新手看了也能照做
- **可验证**：每步有明确的输出

**交互**：
- 我按结构编写SOP
- 你审阅，提修改意见
- 迭代优化

---

### 阶段5：验证迭代（5-10分钟）
**调用**：`systematic-debugging` + `checklist`

**输出**：
- 验证报告
- 优化建议
- 下次迭代的待办

**验证检查清单**：
- [ ] 新手能看懂吗？
- [ ] 每步都明确吗？
- [ ] 成功标准清晰吗？
- [ ] 常见问题覆盖了吗？
- [ ] 实际跑一遍通吗？

**迭代机制**：
1. **首次使用**：标记不清楚的地方
2. **第二次使用**：补充遗漏的步骤
3. **第三次使用**：精简冗余的内容
4. **定期回顾**：删除过时的内容

**交互**：
- 我提供验证checklist
- 你实际跑一遍SOP
- 反馈问题，我优化

# SOP: 供应商月度评估

## 📋 基本信息
- **适用场景**：每月评估35家供应商的表现
- **前置条件**：CRM、财务系统、质检系统访问权限
- **耗时估算**：2小时
- **成功标准**：完成排名、分类、通知、归档

## 🎯 核心步骤

### 步骤1：确认时间范围
**目的**：避免数据范围错误
**操作**：
1. 查看当月日历，确认是否有节假日
2. 确定评估周期（通常上月1日-月末）
3. 如有节假日，调整起止日期
**输出**：明确的起止日期
**注意**：节假日导致的工作日不足，需相应调整周期

### 步骤2：收集数据
**目的**：从3个系统导出完整数据
**操作**：
1. **CRM系统**：
   - 登录 → 报表 → 供应商报表
   - 时间范围：使用步骤1确认的起止日期
   - 导出格式：Excel
2. **财务系统**：
   - 登录 → 结算 → 供应商结算明细
   - 时间范围：同上
   - 导出格式：Excel
3. **质检系统**：
   - 登录 → 报表 → 供应商质检评分
   - 时间范围：同上
   - 导出格式：Excel
**输出**：3个Excel文件
**注意**：文件命名格式为"系统名_YYYYMM.xlsx"

## SOP 质量标准

好的SOP应该：
- ✅ **自解释**：不需要额外解释就能看懂
- ✅ **可执行**：新手看了也能照做
- ✅ **可验证**：每步有明确的输出
- ✅ **可迭代**：用一次就能发现问题

不好的SOP：
- ❌ "适当调整" → 不知道怎么调
- ❌ "处理好数据" → 不知道怎么处理
- ❌ "确保质量" → 不知道怎么判断
- ❌ 太啰嗦 → 看不下去

## 重要说明

**适用场景**：
- 重复性工作（≥每月1次）
- 多步骤工作（≥3步）
- 可复用工作（可交接/传承）

**核心原则**：
- 先识别，再做（不是所有工作都需要SOP）
- 萃取经验，不只记录步骤
- 简洁够用，不追求完美
- 持续迭代，用一次改一次

**你的角色**：
- 提供工作经验
- 验证SOP可用性
- 反馈迭代建议

**我的角色**：
- 引导萃取隐性经验
- 设计清晰结构
- 编写可用SOP
- 持续优化改进
