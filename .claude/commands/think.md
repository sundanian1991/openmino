# Think 指令 — 思考显性化

> 核心理念：思考过程比结论更重要。

---

## 功能

当你希望看到我的推理过程时使用。

这个指令会让我的思考变得**可见**：
1. **我的判断是什么** — 结论先行
2. **依据是什么** — 数据来源、推理过程
3. **不确定的地方** — 可能有几种方案
4. **风险和局限** — 可能的盲点

---

## 使用场景

| 场景 | 说明 |
|------|------|
| **复杂决策** | 想看我怎么分析问题 |
| **不确定答案** | 需要我展示推理而非直接给结论 |
| **审查我的思路** | 让我解释为什么这样想 |
| **教学式对话** | 你想学习我的思考方式 |

---

## 思考模板

```
【意图分类】[简单查询 / 复杂任务 / 探索任务]

【我的判断】
[一句话结论]

【依据】
1. [依据1]
2. [依据2]
3. [依据3]

【不确定的地方】
- [不确定点1]
- [不确定点2]

【风险/盲点】
- [可能的盲点1]
- [可能的盲点2]

【建议】
[如果需要行动的化]
```

## 使用示例

### 示例 1：复杂问题

```
输入：/think 我该选哪个供应商？
```

输出：
```
【意图分类】复杂任务 - 决策分析

【我的判断】
需要更多信息才能给出建议

【依据】
1. 你提到了"选供应商"，但没说是哪个业务线
2. 32家供应商，评估维度可能不同
3. 需要考虑：价格、质量、合规风险、历史表现

【不确定的地方】
- 你的优先级是什么？价格还是质量？
- 有没有红线条款？
- 预算限制是多少？

【建议】
请补充：
1. 具体的供应商候选是哪些？
2. 你的核心评估标准？
3. 有没有 deadline？
```

### 示例 2：探索问题

```
输入：/think 为什么SaaS插件能颠覆传统软件？
```

输出：
```
【意图分类】探索任务 - 原因分析

【我的判断】
核心是定价模型被打破

【依据】
1. $20/月 vs $150/月 — 40%功能但价格差7倍
2. SaaS是"订阅制"，但底层是软件许可思维
3. AI插件本质是"能力订阅"，不绑定许可证

【不确定的地方】
- 你指的是哪款插件？
- 需要深入分析某个具体行业吗？

【风险/盲点】
- 我的分析基于公开报道，可能有内部信息偏差
- 实际影响因公司而异
```

## 简单场景（可选不用）

如果是简单查询（如"今天几号"），可以直接回答，不需要用 think。

但如果你想看我**为什么这么回答**，可以用。

---

## 重要说明

**这不是**：
- 而是"展示推理" — 不是增加步骤
- 复杂化 — 是让判断更可信

**这是**：
- 透明化 — 你能看到我怎么想
- 可审查 — 你可以纠正我的推理
- 可学习 — 你能学到我的思考方式
