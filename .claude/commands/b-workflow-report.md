# Workflow: 汇报制作

> 半自动化全流程方案 - 数据→洞察→内容→视觉→报告

**核心理念**：让汇报制作变成流水线，而不是每次从零开始。

---

## 工作流程

```
数据收集 → 数据分析 → 内容撰写 → 视觉设计 → 报告输出
   ↓          ↓          ↓          ↓          ↓
 search    xlsx/pdf   writing-  frontend-  pptx/
tavily    summarize   clearly   design    infographic
```

---

## 执行步骤

### 阶段1：需求规划（2分钟）
**调用**：`brainstorming`

**输出**：
- 汇报受众是谁？（老板/团队/客户）
- 核心信息是什么？（成果/问题/方案）
- 成功标准是什么？（5分钟讲清楚/数据支撑）

**交互**：
- 我问你3-5个问题
- 你回答，我整理成需求清单

---

### 阶段2：数据收集（5-10分钟）
**调用**：`search` / `tavily` / `xlsx` / `pdf`

**输出**：
- 外部数据：行业对标、竞品分析
- 内部数据：你的数据源（Excel/PDF/其他）
- 数据清单：我们有哪些，缺什么

**交互**：
- 你提供数据文件或数据源
- 我帮你搜索外部数据
- 整理成数据清单

---

### 阶段3：数据分析（10-15分钟）
**调用**：`xlsx` / `systematic-debugging` / 自分析

**输出**：
- 数据洞察：关键数字、趋势、异常
- 对比分析：同比环比、行业对比
- 结论提炼：3个核心结论

**交互**：
- 我分析数据，展示发现
- 你确认或补充
- 共同提炼结论

---

### 阶段4：内容撰写（15-20分钟）
**调用**：`writing-clearly` / `content-creation`

**输出**：
- 汇报大纲：SCQA结构（情境-冲突-问题-答案）
- 核心内容：3-5页精华
- 数据支撑：每个结论有数据

**交互**：
- 我按大纲撰写
- 你审阅，提修改意见
- 迭代优化

---

### 阶段5：视觉设计（10-15分钟）
**调用**：`frontend-design` / `pptx` / `baoyu-infographic`

**输出**：
- 视觉方案：PPT/Infographic/网页
- 关键图表：数据可视化
- 完整报告：可直接使用

**交互**：
- 你选输出格式（PPT/PDF/网页）
- 我设计并生成
- 你审阅，微调

---

### 阶段6：最终审查（5分钟）
**调用**：`checklist`

**输出**：
- 完整报告
- 修改建议清单

**交互**：
- 过一遍 checklist
- 确认没有遗漏
- 完成交付

---

## 总耗时

| 阶段 | 预计耗时 |
|------|---------|
| 需求规划 | 2分钟 |
| 数据收集 | 5-10分钟 |
| 数据分析 | 10-15分钟 |
| 内容撰写 | 15-20分钟 |
| 视觉设计 | 10-15分钟 |
| 最终审查 | 5分钟 |
| **总计** | **47-67分钟** |

## 重要说明

**工作流特点**：
- 半自动：我主导流程，你提供数据+决策
- 可中断：任何阶段都可以暂停，下次继续
- 可定制：可以跳过某些阶段，或调整顺序
- 可复用：下次做汇报时，用同样的流程

**你的角色**：
- 提供数据源
- 回答关键问题
- 审阅并确认

**我的角色**：
- 主导整个流程
- 调用相应skills
- 整合输出完整报告
