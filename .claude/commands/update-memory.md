# update-memory — 每周汇总

> 从insights.md提炼，判断是否更新长期记忆

---

## 工作流程

### Step 1: 读取来源

读取 `memory/insights.md` 中本周的内容

### Step 2: 判断是否更新 MEMORY.md

判断本周内容是否需要更新到 `memory/MEMORY.md`：

| 更新目标 | 判断标准 |
|----------|----------|
| 用户画像 | 偏好、习惯明显变化 |
| 当前待办 | 有新增或完成的待办 → 更新 `memory/context/todo.md` |
| 关键决策 | 有重要决策记录 |
| 项目索引 | 有新项目或项目状态变化 |
| **重要洞察** | 符合升级标准（见下方） |

### Step 2.5: 洞察升级判断

**升级标准**（满足任一即可）：

| 标准 | 说明 | 示例 |
|------|------|------|
| **重复出现 3 次+** | 跨会话多次出现，是稳定模式 | "结构化偏好"在多次对话中出现 |
| **有长期价值** | 不是一时一事，可复用的洞察 | "看人透过形看神" |
| **被年老师认可** | 年老师明确说"这个很重要" | "刘伟佳水平更高" |

**执行方式**：
1. 检查 insights.md 中的洞察
2. 判断是否符合升级标准
3. 符合的 → 迁移到 MEMORY.md 的"重要洞察"部分
4. 不符合的 → 保留在 insights.md，90天后归档

### Step 3: 清理过时记忆

从 MEMORY.md 中删除：
- 已完成的事项（同步到 `memory/context/todo.md` Done 区）
- 不再准确的决策
- 重复的内容

### Step 4: 提交

```bash
git add -A && git commit -m "chore: update-memory weekly"
git push
```

## 文件结构

```
memory/
├── MEMORY.md     # 详细记忆索引（启动加载）
│   ├── 用户画像
│   ├── 当前状态
│   ├── 关键决策
│   ├── 重要洞察 ← 新增：从 insights.md 升级而来
│   └── 项目索引
├── insights.md   # 洞察记录（短期记忆）
├── projects/     # 项目背景
└── archive/      # 历史归档
```

**洞察流转机制**：

```
insights.md（短期记忆）
    ↓ 升级条件满足（重复3次+ / 长期价值 / 年老师认可）
MEMORY.md → 重要洞察（长期记忆）
```
