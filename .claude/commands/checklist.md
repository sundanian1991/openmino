# Checklist 指令 — 上下文核实工具

> 核心理念：对话结束前，确认每件事都完成了。

---

## 功能

1. **上下文扫描** - 扫描对话历史，提取所有任务/问题
2. **分类整理** - 区分已完成、进行中、未开始
3. **状态核实** - 逐项确认是否真的完成了
4. **遗漏检查** - 提醒可能被忽略的事项

---

## 工作流程

```
输入 /checklist
    ↓
1. 扫描对话 - 提取所有任务/问题/承诺
    ↓
2. 分类整理 - 按状态和优先级分组
    ↓
3. 逐项核实 - 每项确认是否完成
    ↓
4. 识别遗漏 - 检查是否有未处理的事项
    ↓
5. 输出清单 - 供后续执行使用
```

---

## 输出格式

```markdown
# 对话事项核实清单

## 对话回顾
本次对话涉及以下话题：
1. [话题1]
2. [话题2]

## ✅ 已完成
- [x] [事项1] - [简述]
- [x] [事项2] - [简述]

## 🔄 进行中
- [ ] [事项3] - [当前状态]

## 📋 未开始
- [ ] [事项4] - [待处理]

## ⚠️ 可能遗漏
- [?] [可能被忽略的事项]

---

**下一步**：继续执行哪一项？或结束对话？
```

---

## 使用场景

| 场景 | 作用 |
|------|------|
| **对话结束前** | 确保没有遗漏事项 |
| **中断恢复** | 重新梳理之前进度 |
| **多话题对话** | 防止后面的挤掉前面的 |

---

## 与其他指令配合

| 指令 | 作用 |
|------|------|
| `/plan` | 复杂任务前先规划 |
| `/observer` | 观察对话过程，为checklist提供素材 |
| `/UPDATE_MEMORY` | 完成后固化经验 |

---

## 重要说明

**核心原则**：
- 不只是列清单，是核实状态
- 主动识别遗漏，不等用户提醒
- 多话题对话时特别有用（防止后面的挤掉前面的）
