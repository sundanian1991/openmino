# /plan5 - 五文件工作流

## 目的

**解决长任务执行中的三个问题**：
1. 需求记不住 → 写进 `prompt.md`
2. 计划会丢失 → 写进 `plans.md`
3. 进度无法追踪 → 写进 `documentation.md`

---

## 工作流

### 阶段 1：需求分析

**AI 行为**：
1. 理解用户需求
2. 澄清模糊点
3. 识别约束条件
4. 定义完成标准

**输出**：`docs/prompt.md`
```markdown
## 目标
## 排除项
## 交付物
## 完成标准
```

### 阶段 2：任务拆解

**AI 行为**：
1. 拆成小里程碑（每个 1-2 小时）
2. 定义验收命令
3. 识别依赖关系

**输出**：`docs/plans.md`
```markdown
### Phase 1
- [ ] 任务
验收命令：`npm run test`
状态：pending | in_progress | completed
```

### 阶段 3：执行追踪

**每次会话**：
1. 读 `plans.md` → 找下一个 pending 任务
2. 执行任务
3. 跑验收命令
4. 更新状态 → completed
5. 写 `documentation.md` → 记录进度

---

## 使用方式

```bash
# 新项目/新需求
/plan5 实现用户登录功能

# 继续执行
/plan5 继续
```

---

## 强制需求澄清机制（A+B+C）

**收到 /plan5 后，AI 必须按以下流程执行**：

| 步骤 | AI 动作 | 用户动作 |
|------|--------|----------|
| 1 | 初始化 docs/ + 五文件模板 | - |
| 2 | 编辑 `prompt.md` 草案 | - |
| 3 | 编辑 `plans.md` 草案 | - |
| 4 | **暂停，AskUserQuestion 确认** | 审阅/修改 |
| 5 | 用户说"开始执行" | 确认后执行 |

> **禁止**：AI 自己理解需求后直接执行

---

*收到 /plan5 后，按以下流程执行：*

1. **需求澄清** - 问清楚要做什么、不做什么
2. **创建 docs/** - 初始化五文件
3. **写 prompt.md** - 记录需求边界
4. **写 plans.md** - 拆解任务 + 验收
5. **等用户确认** - "开始执行"
6. **执行并追踪** - 做完一步更新一步
