# Workflow: 产品发布

> 半自动化全流程方案 - 研究→文案→设计→开发→发布

**核心理念**：让产品发布变成可复制的标准流程。

---

## 工作流程

```
市场研究 → 策略定位 → 文案创作 → 视觉设计 → 前端开发 → 上线发布
   ↓          ↓          ↓          ↓          ↓          ↓
 search/   brainstorming  copy-   web-design  senior-   github/
tavily    marketing-ideas writing  expert      frontend  testing
```

---

## 执行步骤

### 阶段1：市场研究（10分钟）
**调用**：`search` / `tavily` / `web-search`

**输出**：
- 竞品分析：谁在做，怎么做
- 市场机会：空白点在哪
- 用户痛点：待解决问题

**交互**：
- 我搜索市场信息
- 展示研究发现
- 你确认方向

---

### 阶段2：策略定位（10分钟）
**调用**：`brainstorming` / `marketing-ideas`

**输出**：
- 定位声明：一句话说清是什么
- 目标用户：谁会使用
- 核心卖点：3个独特价值

**交互**：
- 我提出定位方案
- 你反馈调整
- 共同确定

---

### 阶段3：文案创作（20分钟）
**调用**：`copywriting` / `content-creation` / `writing-clearly`

**输出**：
- 产品名：好记、有意义
- Slogan：一句话传播
- 产品描述：100字精华
- 详细文案：功能说明、使用场景

**交互**：
- 我产出文案初稿
- 你审阅修改
- 迭代优化

---

### 阶段4：视觉设计（20分钟）
**调用**：`web-design-expert` / `baoyu-cover-image` / `ux-designer`

**输出**：
- 品牌色：主色、辅色
- Logo/图标：产品标识
- 视觉风格：设计方向
- 界面草图：核心页面

**交互**：
- 我提出设计方向
- 你选择风格
- 确定视觉方案

---

### 阶段5：前端开发（30-60分钟）
**调用**：`senior-frontend` / `frontend-design` / `frontend-patterns`

**输出**：
- 技术选型：框架、工具
- 页面代码：可运行的代码
- 响应式：适配多端

**交互**：
- 我写代码
- 你测试反馈
- 修复问题

---

### 阶段6：测试上线（15分钟）
**调用**：`webapp-testing` / `github` / `testing-patterns`

**输出**：
- 测试报告：功能检查
- 代码提交：git记录
- 部署清单：上线步骤

**交互**：
- 我测试功能
- 你确认可用
- 部署上线

---

## 总耗时

| 阶段 | 预计耗时 |
|------|---------|
| 市场研究 | 10分钟 |
| 策略定位 | 10分钟 |

| 前端开发 | 30-60分钟 |

```
正在搜索竞品信息...
- 找到5个类似产品
- 分析其功能、定价、用户评价

研究结论：
✓ 竞品主要功能：KPI追踪、风险评估
✓ 空白点：没有选汰机制自动化
✓ 机会：你的"激活竞争"理念是差异化

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[调用 brainstorming]

基于市场研究，我建议定位：

产品名：VendorPulse（供应商脉搏）
定位：让供应商管理从静态变成动态
目标用户：中小企业的供应商管理负责人

核心卖点：
1. 自动选汰 - 不再手动评估
2. 竞争激活 - 用数据驱动竞争
3. 风险预警 - 提前发现问题

你觉得这个定位如何？

用户: 定位可以，名字要不要更接地气？

AI: 好建议。换个方向：

产品名：供应商管家 / 优选供应商
定位：让优质供应商自动浮现

你更喜欢哪个？

[继续讨论...]
```

## 重要说明

**适用场景**：
- 新产品发布
- 功能上线
- 营销页面制作
- 内部工具推广

**可以简化**：
- 如果已有产品，跳过阶段1-2
- 如果已有设计，跳过阶段4
- 如果不需要前端，跳过阶段5
