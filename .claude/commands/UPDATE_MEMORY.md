# UPDATE_MEMORY — 每周汇总

> 从observations/提炼精华到04-MEMORY

---

## 工作流程

### Step 1: 读取观察记录
读取 `memory/observations/` 当前月份.md

### Step 2: 找出精华
识别值得长期保留的内容：
- 反复出现的模式
- 解释了行为或决策的原因
- 改变了理解的洞察
- 可复用的经验

### Step 3: 提炼到04-MEMORY
更新 `.claude/rules/reference/04-MEMORY.md`：

**分层判断**：
- **L1 置顶**：每次会话必读，影响所有交互（核心协议、反复调用）
- **L2 高频**：30天内调用≥3次（工作方法、工具配置）
- **L3 时新**：最近7天（新学到的、刚发生的）
- **L4 知识**：体系化内容（方法论、框架、模板）

**操作**：
- 找到合适的层级和位置
- 精炼表达，去掉冗余
- 保持简洁

### Step 4: 删除过时
从 04-MEMORY.md 中删除：
- 超过30天未使用的内容
- 不再准确的内容
- 重复的内容

### Step 5: 提交
- `git add -A && git commit`
- `git push`

---

## 执行频率

- **默认**：每周执行一次
- **触发**：手动执行 `/UPDATE_MEMORY`
