## 概述

2026年4月初，年老师对mino-frontend技能进行了深度重构，解决了长期存在的"方法论有了但输出不稳定"问题。核心发现：方法论是骨架，参数系统是血肉——骨架相同但血肉不同，输出就不稳定。重构分两步走：第一步建立风格x情绪决策矩阵，将14种经验分类简化为5种基础风格x4种情绪变体的二维决策逻辑；第二步构建三层参数系统（全局参数+风格参数+内容参数），为每种设计风格定义了具体的字号、边框、间距、颜色值。这次重构将mino-frontend从"靠感觉调整"升级为"参数化控制"的设计系统。

## 一、问题诊断：为什么方法论够了但输出不稳定

### 1.1 用户走过的迭代历程

| 阶段 | 方法论 | 反馈 |
|------|--------|------|
| V1 | 跳过流程 | "形似神不似" |
| V2 | 三区模型（标题/内容/结论） | "空间上的呈现不是很有方法论的感觉" |
| V3-V4 | 均匀压缩间距 | "效果更差了" |
| V5 | 视觉层级法（层级数+权重） | "不够专业" |
| 最新 | 内容-位置-权重三步法 | "一直没有特别满意" |

### 1.2 核心张力

年老师自述的卡点揭示了一个深层问题：

> "方法论=骨架，视觉血肉=具体设计语言参数。骨架有了，血肉没有。"

具体缺失：
- 边框用多细？1px还是2px？
- 字体层级差多少？13px vs 11px够不够？
- 信息密度多高？松到什么程度叫"呼吸感"？
- "专业"到底是什么感觉？是像咨询报告还是像财务报表？

### 1.3 参数不一致的证据

对比不同输出的实际参数值：

| 参数 | 游泳 | 刘乾坤v3 | 权责不对等 | 问题 |
|------|------|----------|-----------|------|
| T1字号 | 2rem~2.5rem | 1.8rem~2.8rem | 1.5rem~2.25rem | 不一致 |
| T2字号 | 1.5rem~1.75rem | 1rem~1.3rem | 1.125rem~1.375rem | 不一致 |
| 页面padding | 1.5rem~3rem | 1.5rem~3rem | 2rem~4rem | 不一致 |
| 陶土色装饰线 | 2.4px | 2px（非陶土色） | 无 | 不一致 |

**诊断结论**：同样的方法论，参数值定义不一致，导致每次生成需要"调参数"。

## 二、第一步：风格x情绪决策矩阵

### 2.1 当前问题：14种风格是"经验分类"，不是"决策逻辑"

mino-frontend原有的14种风格：
> 克制专业主义、品牌模式、数据模式、金融时报、咨询模式、自信宣言、现代工坊、分类标签、柔和几何、趣味拼接、复古报刊、极简现代、纸墨文学、Anthropic品牌

问题：这是基于历史使用积累的分类，没有内在逻辑。新增风格没有依据，删除风格没有标准，选择风格没有方法。

### 2.2 新模型：二维决策

```
            情绪（内容气质）
         紧迫    中性    庆祝    沉思
       ┌───────┬───────┬───────┬───────┐
  数据 │ 数据模式 │ 数据模式 │ 金融时报 │ 数据模式 │
       │ (红)   │ (灰)   │ (青绿) │ (深灰) │
       ├───────┼───────┼───────┼───────┤
  品牌 │ 自信宣言 │ 品牌模式 │ 品牌模式 │ 纸墨文学 │
       │ (黑金) │ (陶土) │ (陶土) │ (墨色) │
       ├───────┼───────┼───────┼───────┤
  咨询 │ 咨询模式 │ 咨询模式 │ 咨询模式 │ 咨询模式 │
       │ (红)   │ (灰)   │ (蓝)   │ (灰)   │
       └───────┴───────┴───────┴───────┘

风格类型（设计系统）
```

关键洞察：
- **风格类型=设计系统**（数据/品牌/咨询=3种基础）
- **情绪=Header颜色变体**（红/灰/陶土/青绿/金/墨=6种颜色）
- **组合=3x4=12种**，比14种更简洁，且有逻辑

### 2.3 简化方案

将14种风格简化为5种基础风格x4种情绪变体：

| 基础风格 | 核心特征 | 情绪变体 |
|---------|---------|---------|
| **数据模式** | 表格、数字、黑白灰 | 紧迫/中性/庆祝/权威 |
| **品牌模式** | 陶土色、手绘图标、人文感 | 紧迫/中性/庆祝/权威 |
| **咨询模式** | So What结论、结构化 | 紧迫/中性/庆祝/权威 |
| **金融时报** | 青绿、KPI网格、信赖感 | 紧迫/中性/庆祝/权威 |
| **极简宣言** | 黑金/黑陶土、大字号、冲击力 | 紧迫/中性/庆祝/权威 |

删除7种重叠或过于细分的风格：现代工坊、分类标签、柔和几何、趣味拼接、复古报刊、纸墨文学、Anthropic品牌。标记为"扩展风格"，不删除但默认不推荐。

### 2.4 Header颜色动态选择

| 情绪标签 | Header颜色 | 使用场景 |
|---------|-----------|---------|
| 紧迫/警示 | #8B0000（深红） | 处罚、风险、异常 |
| 中性/专业 | #111（炭黑） | 数据通报、常规汇报 |
| 庆祝/激励 | #E2725B（陶土） | 业绩达成、激励海报 |
| 权威/正式 | #0d7680（青绿） | 金融数据、战略汇报 |
| 奢华/高端 | #D4AF37（金） | 颁奖、年度盛典 |

决策流程：内容→提取情绪关键词→匹配情绪标签→确定Header颜色→选择基础风格。

## 三、第二步：三层参数系统

### 3.1 参数系统结构

```
┌─────────────────────────────────────────────────────────────┐
│                    参数系统三层结构                          │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  【Layer 1：全局参数】                                       │
│  所有风格共享的基准参数                                       │
│  ├── 色彩系统（主色、辅助色、陶土色）                        │
│  ├── 间距基准（4pt系统）                                    │
│  └── 响应式断点                                              │
│                                                              │
│  【Layer 2：风格参数】                                       │
│  每种设计风格独有的参数                                       │
│  ├── 字体层级（T1-T6 px值）                                │
│  ├── 边框系统（粗细、圆角）                                  │
│  ├── 间距比例（页面padding、卡片gap）                       │
│  └── 陶土色使用规则                                          │
│                                                              │
│  【Layer 3：内容参数】                                       │
│  由内容特征决定的参数                                         │
│  ├── 信息密度（高/标准/低）→ 间距调整                        │
│  ├── 情绪气质（紧迫/中性/庆祝/沉思）→ 陶土色用量            │
│  └── 阅读方式（扫视/阅读/凝视）→ 字号层级差                  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

参数优先级：**内容参数 > 风格参数 > 全局参数**。

### 3.2 色彩系统（全局参数）

```css
:root {
  /* 文字层级 */
  --text-primary: #111111;
  --text-secondary: #666666;
  --text-tertiary: #999999;

  /* 背景 */
  --bg: #FFFFFF;
  --surface: #FAFAFA;

  /* 边框 */
  --border: #E5E5E5;

  /* 陶土色（焦点色） */
  --accent: #E2725B;
  --accent-light: rgba(226, 114, 91, 0.08);
}
```

### 3.3 字体层级系统（风格参数）

| 层级 | 用途 | 基准值 | clamp值 | 层级差 |
|------|------|--------|---------|--------|
| T1 | 页面标题 | 28px | clamp(1.75rem, 4vw, 2rem) | 基准 |
| T2 | 模块标题 | 18px | clamp(1.125rem, 2vw, 1.25rem) | -10px |
| T3 | 卡片标题 | 14px | clamp(0.875rem, 1.5vw, 1rem) | -4px |
| T4 | 正文 | 13px | clamp(0.8125rem, 1.3vw, 0.9rem) | -1px |
| T5 | 辅助文字 | 11px | clamp(0.6875rem, 1vw, 0.75rem) | -2px |
| T6 | 页码/标签 | 10px | clamp(0.625rem, 0.8vw, 0.7rem) | -1px |

层级差原则：T1→T2大跳跃（-10px，区分页面和模块），T2→T3中跳跃（-4px，区分模块和卡片），T3→T4小跳跃（-1px，卡片内层级）。

### 3.4 边框系统

| 元素 | 边框粗细 | 边框颜色 | 圆角 |
|------|---------|---------|------|
| 卡片 | 1px | #E5E5E5 | 4px |
| 表头分割线 | 2px | #111111 | 0 |
| 表格行分割线 | 1px | #E5E5E5 | 0 |
| 顶部装饰线 | 2.4px | #E2725B | 0 |
| 强调卡片 | 1px | #E2725B | 4px |

### 3.5 间距系统与密度映射

4pt基准系统：4px/8px/12px/16px/24px/32px/48px。

| 密度 | 页面padding | 卡片内padding | 卡片gap | 区块gap |
|------|------------|--------------|---------|---------|
| 高密度 | 24px | 12px | 12px | 16px |
| 标准密度 | 32px | 16px | 16px | 24px |
| 低密度 | 48px | 24px | 24px | 32px |

密度由内容特征决定：

| 情绪 x 阅读方式 | 密度选择 |
|----------------|---------|
| 紧迫 x 扫视 | 高密度 |
| 中性 x 阅读 | 标准密度 |
| 沉思 x 凝视 | 低密度 |

### 3.6 陶土色使用系统：一页一处规则

| 情绪 | 陶土色使用量 |
|------|------------|
| 紧迫 | 2-3处（高对比，密集标记） |
| 中性 | 1处（标准焦点） |
| 庆祝 | 1处（大字号金句） |
| 沉思 | 1处（极简，大面积留白+小焦点） |

使用场景和面积限制：

| 场景 | 使用方式 | 面积限制 |
|------|---------|---------|
| 顶部装饰线 | 2.4px实线 | ≤5px高 |
| 页码徽章 | 24px圆形填充 | ≤24px x 24px |
| 强调卡片边框 | 1px实线 | 卡片周长 |
| 强调卡片背景 | rgba(226,114,91,0.08) | 卡片面积 |
| 列表项圆点 | 4px圆点 | ≤4px x 4px |

## 四、参数协调规则

### 4.1 边框 x 间距协调

| 边框粗细 | 配套间距 | 视觉效果 |
|---------|---------|---------|
| 1px | 标准（16px） | 轻量、透气 |
| 2px | 加大（24px） | 强调、分隔 |
| 陶土色1px | 加大（24px） | 焦点、吸引 |

### 4.2 字号 x 间距协调

| 字号层级差 | 配套行高 | 配套段落间距 |
|-----------|---------|-------------|
| T1-T2（-10px） | 1.2 | 24px |
| T2-T3（-4px） | 1.3 | 16px |
| T3-T4（-1px） | 1.5 | 12px |

### 4.3 参数系统与六层决策的关系

```
【六层决策】                  【参数系统】

L1 内容元素识别    →    决定可视化级别（参数系统Layer 3）
L2 情绪气质        →    决定密度和陶土色用量（参数系统Layer 3）
L3 阅读方式        →    决定字号层级差（参数系统Layer 3）
L4 密度与版式      →    选择间距值（参数系统Layer 2+3）
L5 可视化叠加      →    选择边框和装饰（参数系统Layer 2）
L6 焦点选择        →    选择陶土色位置（参数系统Layer 3）

【执行流程】
六层决策 → 输出内容结构+选择参数值 → 生成代码
```

## 五、方法论价值：从"感觉驱动"到"参数驱动"

### 5.1 核心认知转变

- **方法论=骨架，参数=血肉**：骨架相同但血肉不同，输出就不稳定
- **参数系统不是限制创意，而是保证底线**：在参数基础上可以有创意发挥
- **参数一致性=品牌一致性**：同一风格的所有输出应该有相同的参数基准

### 5.2 参数系统的可迁移性

这套参数系统的设计方法可以迁移到任何需要"输出稳定性"的创意领域：

1. 定义全局基准（所有场景共享的常量）
2. 定义风格参数（每种风格的独特变量）
3. 定义内容参数（由内容特征决定的动态变量）
4. 建立协调规则（参数之间的依赖关系）
5. 建立执行检查清单（确保参数一致性）
