## 概述

本文提炼了从头脑风暴到知识沉淀的完整项目管理方法论。通过对多个实际项目（游泳学习计划、Quiver API 策略、跨技能设计原则、mino-frontend 重构、AI 定价研究）的文件分析，揭示了四阶段工作流的标准化结构和每个阶段的质量要求。

## 一、四阶段工作流架构

### 阶段1：Brainstorm（头脑风暴）

**目的**：明确问题、识别约束、浮出主题、建立初步方向。

**标准结构**：
1. What I Heard：问题描述、待决策事项、开放问题、约束条件、利益相关方
2. 自动搜索结果：搜索已有知识库中的相关内容
3. 主题识别：从初步信息中提炼核心主题
4. 假设列表：列出需要验证的关键假设

**关键产出**：清晰的问题定义和待验证假设清单。

**质量检查**：
- 问题是否一句话可描述？
- 约束条件是否全部列出？
- 是否列出了至少3个待验证假设？

### 阶段2：Research（研究计划）

**目的**：基于头脑风暴输出，制定结构化的研究计划。

**标准结构**：
1. 研究目标：一句话描述
2. 研究问题按优先级分级（P0/P1/P2）
3. 研究方法表：问题→方法→数据来源→时间框架
4. 预期输出：文件清单和内容概要
5. 质量检查点

**关键产出**：可执行的研究计划和预期交付物清单。

**质量检查**：
- P0 问题是否都能在计划中找到对应研究方法？
- 预期输出是否具体到文件路径？
- 是否有明确的质量检查点？

### 阶段3：Review（审查）

**目的**：在研究或执行后，进行系统性的质量检验。

**标准结构**：
1. 战略对齐审查：目标清晰度、假设可证伪性、范围比例性、资源意识、战略一致性
2. 数据准确性审查：数据来源、时效性、精确性、引用完整性
3. 执行确认：关键决策点和理由
4. P1/P2/P3 问题分级和修复建议

**审查分级标准**：
- P1：阻塞性问题，必须修复才能继续
- P2：重要问题，需修复但不阻塞
- P3：可优化项，建议修复

**关键产出**：审查报告，包含通过项、问题项和修复建议。

### 阶段4：Compound（知识沉淀）

**目的**：将项目中的洞察和模式提取为可复用的知识。

**标准格式**：
```
## 洞察 N：[标题]
**类型**：insight / pattern / framework
**核心洞察**：[一句话描述]
**问题**：[遇到什么问题]
**解决方案**：[怎么解决的]
**收益**：[带来了什么价值]
**置信度**：High / Medium / Low
**标签**：[#标签1] [#标签2]
```

## 二、各阶段的核心差异

### Brainstorm vs Research

| 维度 | Brainstorm | Research |
|------|-----------|----------|
| 思维模式 | 发散 | 收敛 |
| 输出形式 | 主题+假设 | 计划+方法 |
| 问题处理 | 列出所有问题 | 按优先级排序 |
| 关键动作 | 浮出可能性 | 制定执行路径 |

### Review 的双重角色

Review 可以发生在两个节点：
1. Research 后：检验研究计划的质量
2. 执行后：检验执行结果的质量

两种 Review 的侧重点不同：
- Research Review：关注目标清晰度、方法适当性
- Execution Review：关注数据准确性、结果有效性

## 三、实际案例分析

### 案例1：Quiver API 高质量 SVG 策略

**Brainstorm**：在每周20次配额限制下生成东京猫/李诞虾级别的SVG。核心约束是API次数有限，每次都必须成功。关键假设包括references接受格式、n>1时是重复还是变体等。

**Research**：深度分析参考SVG代码结构。发现东京猫有396个path元素（不是估计的50个），李诞虾采用严格的双层技法（填充层+描边层分离）。关键发现是线条粗细变化（1.6px主线、2.1px轮廓、0.7px细节）和手工贝塞尔曲线（每个d属性20-40个控制点）。

**关键决策**：一次性成功的策略——先用代码分析理解精细度标准，再构造精确的prompt。

### 案例2：跨技能设计原则体系

**Brainstorm**：建立跨技能的抽象设计原则体系，作为所有视觉产出物的审美元规范。核心约束是简洁优先（原则是"宪法"不是"地方法规"）、可执行（有可验证检查点）、跨技能通用。

**三大主题**：
1. 第一性原理：克制>装饰、锐度>模糊、诚实>套路
2. 信息设计铁律：一页一核心、空白是设计、强调色≤10%、无装饰性元素
3. 反AI味检查：无阴影、无圆角premium cards、无玻璃拟态

### 案例3：mino-frontend 风格情绪矩阵

**Brainstorm**：用"风格×情绪"二维决策矩阵替代14种经验分类。核心洞察是"14种风格是经验分类，不是决策逻辑"。

**新模型**：3种基础风格（数据/品牌/咨询）× 4种情绪（紧迫/中性/庆祝/沉思）= 12种组合，比14种更简洁且更有逻辑性。

**Review 结果**：目标清晰、假设可证伪、范围比例合理、资源意识到位。

### 案例4：AI 定价研究 Review

**P2 问题**：
1. 价格数据时效性——AI定价变化极快，需加注时间戳
2. "70% token 浪费"引用需给出具体链接
3. 电信类比的时间尺度差异需标注

**P3 问题**：
4. "国内便宜是因为有人在替你卷"是观点不是事实
5. Agent 成本数据的时间点需交叉验证

**结论**：0个P1，3个P2，2个P3。整体质量过关。

## 四、工作流的质量标准

### 每个阶段的出口标准

| 阶段 | 出口标准 |
|------|----------|
| Brainstorm | 问题定义+3个假设+约束清单 |
| Research | P0问题都有方法+预期输出文件清单 |
| Review | 0个P1+P2有修复计划 |
| Compound | 至少1条洞察+标签完整 |

### 反模式

- 跳过 Brainstorm 直接 Research → 方向可能错误
- 跳过 Review 直接执行 → 可能做无用功
- 不做 Compound → 经验无法复用
- Review 只有赞美没有批评 → 失去审查价值
