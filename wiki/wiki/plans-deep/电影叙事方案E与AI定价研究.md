## 概述

本文记录了 AI 定价商业案例的两个电影叙事方案——主方案（Fincher/Social Network 调查档案风格，26页五幕叙事）和方案E（Nolan/Tenet 倒叙杂志特稿风格，4页精简版）的完整设计决策过程。同时整合了 AI 定价研究的核心内容，展示如何将行业研究转化为具有电影感的叙事体验。

## 一、两个方案的对比分析

### 主方案：Fincher × Social Network

- **隐喻**：调查档案——翻阅一份关于 AI 定价变革的完整商业案例报告
- **结构**：26页，五幕叙事（幻觉→崩塌→调查→规律→未来）
- **色彩**：深色底（#0a0a0f）+ teal（#1a3a4a）+ amber（#d4a35f）
- **导演签名**：去饱和调色、精确构图、glitch/scan line
- **入场**：4种动画（fadeUp、slide-left、scale-up、clip-reveal）

### 方案E：Nolan × Tenet

- **隐喻**：一本从最后一页开始读的商业杂志
- **结构**：4页倒叙（2026.04终态 → 2026.02催化剂 → 2025-2026争夺战 → 2024 Q4起点）
- **色彩**：米白底（#f5f4f0）+ 铜金（#c9a84c）+ 深蓝（#1a2744）
- **导演签名**：时间倒流、因果倒置、模糊到清晰的揭示
- **导航**：顶部进度条从右到左填充（时间倒流）

### 为什么需要两个方案？

方案E 是在主方案多次修改不满意后产生的替代方案。核心差异在于：
- 主方案是"调查档案"（非对称、密集、粗粝、终端感）
- 方案E 是"杂志特稿"（对称克制、大量留白、优雅、印刷品质感）

两种方案代表了两种完全不同的叙事哲学——前者是"给你全部信息让你自己发现规律"，后者是"先给你结果再带你理解为什么"。

## 二、Uniqueness Audit 方法论

### Shell-ban List 的作用

每次项目启动前审计历史作品的重复模式，列入禁止清单。这是防止审美趋同的关键机制。

**方案E 的 Shell-ban List**：
- 禁止暗色背景（#0a0a0f/#12121a等深色系）
- 禁止警报红（#ff4433）作为主强调色
- 禁止 JetBrains Mono 字体
- 禁止 count-up 数字动画
- 禁止正序叙事
- 禁止卡片堆叠、slide滚动

**主方案的 Shell-ban List**：
- 禁止单页全屏滚动
- 禁止居中 hero + features grid
- 禁止通用 SaaS 卡片矩阵
- 禁止顶部导航

### 唯一性测试

"去掉颜色后应像'纪录片分镜'而非'产品官网'"——这是主方案的唯一性测试标准。方案E的测试标准是"应像高端商业杂志特稿而非SaaS落地页"。

## 三、幕色调递进

主方案五幕的温度弧线设计：
- 第一幕（幻觉）：偏暖，amber 占比略高，营造"舒适错觉"
- 第二幕（崩塌）：冷峻，teal 主导，amber 用于强调危机
- 第三幕（调查）：冷静，灰白主导，amber 仅用于关键数据
- 第四幕（规律）：冷中带暖，规律揭示时 amber 作为"洞察闪光"
- 第五幕（未来）：渐亮，底部出现微妙亮色，暗示终局

这种色彩递进不是装饰性的，而是叙事性的——观众在潜意识中感受到"温度变化"，从而更好地理解叙事弧线。

## 四、密度节奏与过渡页

### 密度曲线

封面（低）→ 个人体验（中）→ 时间线（高）→ 过渡（极低）→ 对比（高）→ 过渡（极低）→ 调查（高）→ 过渡（极低）→ 规律（中）→ 过渡（极低）→ 趋势（低）→ 收尾（极低）

### 过渡页的价值

五幕之间各有1页过渡页。这些过渡页的作用：
1. 给观众呼吸空间
2. 预告下一幕的核心问题
3. 维持叙事节奏感

没有过渡页的叙事是噪音——信息密度过高导致观众无法消化。

## 五、AI 定价研究核心内容

### 一句话结论

"AI 大模型定价，精确重演了手机流量套餐的 20 年历史。Coding Plan 是过渡产品，Token Plan 是终态。"

### 关键时间线

| 阶段 | 时间 | 关键事件 |
|------|------|----------|
| 价格战启动 | 2024 Q4-2025 Q2 | DeepSeek 带头降价，百万 Token 从几十元降到几毛 |
| Coding Plan 涌现 | 2025 Q4 | 智谱 GLM 率先推出固定月费 |
| 全面铺开 | 2026 Q1 | 阿里百炼7.9元首月入局，腾讯最后入局 |
| 收紧 | 2025 H2 | 限购、停售低档、首购优惠变限时抢购 |
| 命名转变 | 2026-03 | MiniMax 将 "Coding Plan" 更名为 "Token Plan" |
| 计量切换 | 2026-01 | Kimi 从 Prompt 计量切换为 Token 计量 |
| OpenClaw 爆火 | 2026-02 | GitHub 355K stars，Agent 消耗6-10倍增长 |
| 供给约束 | 2026-04 | 阿里百炼 Lite 下架，各家算力吃紧 |

### 核心洞察

1. **计量单位即定价权**：从 Prompt 到 Token 的转变是定价权的争夺
2. **三段式节奏**：补贴获客 → 收紧限制 → 分层定价——所有厂商都走这条路
3. **入口到生态**：低价 Coding Plan 是入口，真正的盈利在 Token 消耗分层
4. **Coding Plan 是过渡产品**：最终都会演化为 Token Plan 或 Credits 体系

### 两派参与者

| 类型 | 代表 | 策略 |
|------|------|------|
| 模型厂商（专卖店） | 智谱、Kimi、MiniMax | 只卖自家模型，深度优化 |
| 云平台（超市） | 阿里百炼、火山方舟、腾讯云 | 聚合多模型，一键切换 |
| 新入局者 | 小米 MiMo | 极低定价（$1/$3），但暗示后续收紧 |
