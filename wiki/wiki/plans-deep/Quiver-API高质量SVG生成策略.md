# Quiver API 高质量 SVG 生成策略

> Sources: mino, 2026-04-28
> Raw: ../../raw/plans/2026-04-04-Quiver-API高质量使用策略-00-brainstorm.md; ../../raw/plans/2026-04-04-Quiver-API高质量使用策略-01-research.md

## 概述

本文记录了在每周20次 Quiver API 配额限制下，如何一次性成功生成高质量手绘风格 SVG 的完整策略。核心挑战是配额有限（每周仅20次），每次调用都必须成功——不能靠试错来学习。

## 一、API 约束分析

### 硬约束

| 约束 | 值 | 影响 |
|------|------|------|
| 每周配额 | 20次 | 每次失败成本5%周预算 |
| Rate limit | 20 requests/60s | 不可批量爆发 |
| 每次消耗 | 1 credit（包括n>1） | 批量生成不额外消耗配额 |

### 质量目标

参考标准：东京猫（~396个 path 元素，精细线条）、李诞虾（双层技法+陶土色点缀）。必须承载具体内容（真实人物、部门、问题），不是抽象模板。

## 二、参考 SVG 代码结构分析

### 东京猫特征

- 总共 396 个 path 元素（不是最初估计的 50 个）
- 猫主体约 50-60 个 path，其余是页面装饰
- 核心技法：填充层 + 线条层分离

```xml
<path d="..." fill="#FEFFFA" stroke="#0B0800" stroke-width="1.5"/>
<path d="..." fill="#BA3420"/>  <!-- 纯填充 -->
<path d="..." stroke="#0B0800" stroke-width="1.5"/>  <!-- 纯描边 -->
```

### 色彩系统

| 颜色 | 色值 | 用途 |
|------|------|------|
| 暖白 | #FEFFFA | 背景、填充 |
| 米白 | #F7F3ED | 次要填充 |
| 陶土 | #BA3420 | 强调色 |
| 陶土浅 | #C4432A | 强调色浅色 |
| 墨黑 | #0B0800 | 线条 |

### 李诞虾双层技法

```xml
<svg viewBox="0 0 250 250">
  <!-- 填充层：纯色彩，无描边 -->
  <path d="..." fill="#BA3420"/>
  <!-- 描边层：纯线条，无填充 -->
  <g fill="none" stroke="#0B0800" stroke-linecap="round" stroke-linejoin="round">
    <path d="..." stroke-width="1.6"/>
    <path d="..." stroke-width="2.1"/>
    <path d="..." stroke-width="0.7"/>
  </g>
</svg>
```

### 关键发现

1. **严格的层分离**：先完整填充层，再完整描边层
2. **线条粗细变化**：1.6px 主线、2.1px 轮廓、0.7px 细节
3. **手工贝塞尔曲线**：每个 d 属性有 20-40 个控制点
4. **viewBox 精确定义**：封面 250×250，图标 72×72

## 三、高质量 Prompt 构造策略

### Prompt 构成要素

1. **主体描述**：精确描述对象的外形和姿态
2. **风格指令**：手绘风格、线条粗细、色彩系统
3. **技法要求**：层分离、贝塞尔曲线密度、viewBox 尺寸
4. **内容承载**：真实人物、具体场景、有意义的叙事

### 验证中的关键假设

| 假设 | 风险 | 验证方法 |
|------|------|----------|
| references 接受 URL 还是 base64 | 可能只接受一种 | 测试两种格式 |
| n>1 时是重复还是变体 | 可能浪费配额 | 生成后对比 |
| instructions 对中文 prompt 支持 | 可能英文更好 | 测试中英混合 |
| context_length 131072 限制 | 可能 token 限制 | 长 prompt 测试 |

## 四、一次性成功策略

### 策略1：先分析再生成

不直接调用 API，先分析参考 SVG 的代码结构（path 数量、色彩系统、线条参数），构造精确的技术规范。

### 策略2：分层构造

先构造填充层，验证通过后再加描边层。不试图一次性完成所有层次。

### 策略3：利用 n>1

一次调用生成多个变体（n=4 或更多），不额外消耗配额。从中选择最接近目标的一个。

### 策略4：references 引导

上传参考图（最多4张）引导 API 理解风格要求。参考图的质量直接决定输出质量。
