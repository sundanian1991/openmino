# 图表技能改造与分析先行工作流

> Sources: mino, 2026-04-25
> Raw: [plan](../../raw/plans/2026-04-25-图表技能改造-plan.md); [prompt](../../raw/plans/2026-04-25-图表技能改造-prompt.md); [result](../../raw/plans/2026-04-25-图表技能改造-result.md); [spec](../../raw/plans/2026-04-25-图表技能改造-spec.md)

## 概述

2026年4月25日，年老师完成了chart-visualization和echarts-visualization两大图表技能的系统性改造，核心变更是从"选图→出图"的简单模式升级为"分析→确认→出图→注解"的完整工作流。这次改造解决了图表生成领域最常见的一个问题：AI直接出图而不分析数据特征，导致图表类型选择不当、缺乏数据洞察、注解缺失。改造采用"不合并，各自增强"的策略——chart-visualization重写完整5步工作流（14种分析目标+主题色palette传递），echarts-visualization在前端加步骤0数据分析前摇（主题推荐度标记+配色体系加固），两者形成分析决策→技术实现的上下游工作流。

---

## 一、改造动机

### 1.1 改造前的问题

| 问题 | 表现 | 后果 |
|------|------|------|
| **无分析前摇** | AI直接推荐图表，不理解数据特征 | 图表类型选择不当 |
| **无确认环节** | 直接生成，无法调整方向 | 不满意需要重做 |
| **无注解** | 图表没有解读文字 | 读者看不懂图表含义 |
| **无主题色传递** | 每次图表颜色不同 | 整体视觉不一致 |
| **技能边界模糊** | chart和echarts功能重叠 | 不知道该用哪个 |

### 1.2 改造目标

| 目标 | 说明 |
|------|------|
| 分析先行 | 先理解数据，再选择图表 |
| 用户确认 | 生成前确认选择 |
| 自动注解 | 图表附带解读文字 |
| 主题统一 | 全链路传递配色方案 |
| 分工明确 | chart管分析决策，echarts管技术实现 |

## 二、chart-visualization重写方案

### 2.1 5步完整工作流

**步骤0：数据分析前摇**

```
输入：原始数据
输出：数据特征分析报告

分析内容：
1. 数据类型识别（时间序列/分类/关系/分布/组成）
2. 数据特征分析（极值、缺失、分布形态）
3. 分析目标确定（对比/趋势/占比/关联/异常）
```

**步骤1：图表推荐**

```
输入：数据特征分析报告
输出：2-3个图表方案及权衡分析

每个方案包含：
- 图表类型
- 推荐理由
- 适用/不适用场景
- 风险提示
```

**步骤2：确认**

```
输入：图表方案
输出：用户确认结果

确认内容：
- 图表类型选择
- 配色方案
- 标注方式
- 是否调整
```

**步骤3：出图**

```
输入：确认的图表方案
输出：图表文件

出图要求：
- 传递分析阶段的palette
- 应用主题色体系
- 生成高质量图表
```

**步骤4：注解**

```
输入：图表+数据特征分析报告
输出：图表注解

注解内容：
- 趋势说明（数据走向）
- 异常标注（离群值）
- 关键发现（洞察结论）
- 解读建议（读者指南）
```

### 2.2 14种分析目标库

| 分析目标 | 推荐图表 | 使用场景 | 避免场景 |
|---------|---------|---------|---------|
| 时间趋势 | 折线图/面积图 | 连续时间数据 | 离散分类数据 |
| 类别对比 | 柱状图/条形图 | 离散类别对比 | 趋势数据 |
| 占比分析 | 堆叠条形图 | 部分与整体关系 | 饼图（避免） |
| 关系分析 | 散点图/气泡图 | 两变量相关性 | 单变量数据 |
| 分布分析 | 直方图/箱线图 | 数据分布特征 | 汇总数据 |
| 层级关系 | 树图/旭日图 | 嵌套层级结构 | 扁平数据 |
| 流程分析 | 桑基图/漏斗图 | 流向和转化率 | 无流向数据 |
| 地理分析 | 热力地图 | 地理维度对比 | 非地理数据 |
| 矩阵分析 | 热力图 | 两维度交叉对比 | 单维度数据 |
| 异常检测 | 控制图 | 离群值识别 | 正常分布数据 |
| 排名分析 | 水平条形图 | TOP N排序 | 无序分类 |
| 目标达成 | 子弹图/仪表盘 | 实际vs目标对比 | 无目标值 |
| 多维对比 | 雷达图 | 多维度同时对比 | 单维度数据 |
| 组成分析 | 堆叠面积图 | 随时间变化的组成 | 静态组成 |

### 2.3 主题色Palette传递

```
分析阶段 → 确定palette（三色核心）
    ↓
推荐阶段 → 在方案中指定配色
    ↓
确认阶段 → 用户确认或调整配色
    ↓
出图阶段 → 应用确认的配色
    ↓
注解阶段 → 注解中引用配色含义
```

**三色核心**（13-VISUALIZATION.md标准）：
- 主题红 `#C13531`：高风险标记、核心高亮
- 深蓝 `#293C54`：主数据系列、对比元素
- 纯灰 `#CDCECD`：参考线、背景数据

## 三、echarts-visualization手术刀编辑

### 3.1 改动范围

保持原有的7步工作流不变，在前面新增步骤0：

**步骤0：数据分析前摇（新增）**

- 在生成ECharts图表前，先分析数据特征
- 根据分析结果推荐最适合的ECharts图表类型
- 标记每种图表类型的推荐度（推荐/可选/不推荐）
- 应用13-VISUALIZATION.md的三色配色体系

### 3.2 主题推荐度标记

| 图表类型 | 推荐场景 | 推荐度 |
|---------|---------|--------|
| 折线图 | 时间序列趋势 | 推荐 |
| 柱状图 | 类别对比 | 推荐 |
| 面积图 | 时间趋势+占比 | 推荐 |
| 堆叠条形图 | 多维占比 | 推荐 |
| 散点图 | 两变量关系 | 可选 |
| 桑基图 | 流向分析 | 可选 |
| 热力图 | 矩阵对比 | 可选 |
| 饼图 | 简单占比 | 不推荐（优先用堆叠条形图） |
| 3D图表 | 任何场景 | 不推荐（禁止3D效果） |

### 3.3 配色体系加固

**强制使用13-VISUALIZATION.md三色系统**：
- 所有ECharts图表必须使用三色核心配色
- 禁止使用ECharts默认9色轮播
- 禁止使用渐变、阴影、发光效果
- 高风险标记统一用主题红 `#C13531`

## 四、改造核心决策

### 4.1 合并 vs 各自增强

**决策**：不合并，各自增强。

| 考虑因素 | 分析 |
|---------|------|
| 定位差异 | chart侧重"选什么图"（分析决策），echarts侧重"怎么画好"（技术实现） |
| 用户场景 | chart用于数据探索阶段，echarts用于最终出图阶段 |
| 互补关系 | 两者形成工作流上下游：分析先行→ECharts实现 |
| 合并风险 | 合并后失去灵活性，用户可能只需要分析不需要出图 |

|--------|------|------|

## 五、方法论总结

### 5.1 分析先行的核心价值

"分析先行"的本质是：**图表是分析的结果，不是分析的替代品**。

这个原则可以迁移到任何数据可视化场景：
- 先理解"数据是什么"（数据类型、特征、分布）
- 再明确"要看什么"（分析目标、核心问题）
- 然后决定"用什么图"（图表类型、配色、标注）
- 最后才"画图"（技术实现）

跳过前两步直接画图，是AI生成图表最常见的错误。

### 5.2 技能分工的通用规律

1. **分析决策与技术实现应该分离**：分析决策需要理解业务，技术实现需要掌握工具。两者合并会失去各自的深度。
2. **上下游关系优于包含关系**：chart和echarts是上下游（分析→实现），不是包含关系（chart包含echarts）。
3. **模板化是知识沉淀的方式**：将成功经验沉淀为模板文件，方便后续复用。

### 5.3 与可视化技能矩阵的关系

改造后的chart-visualization和echarts-visualization在可视化技能矩阵中的定位更加清晰：

```
数据分析 → chart-visualization（分析决策：选什么图）
    ↓
echarts-visualization（技术实现：怎么画好）
    ↓
mino-frontend（页面整合：整体页面+组件）
    ↓
Quiver API / diagram-design（装饰元素：SVG插图/技术图表）
```

每个技能都有明确的定位和边界，形成完整的可视化工作流。
