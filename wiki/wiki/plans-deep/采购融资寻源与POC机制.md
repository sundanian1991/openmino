## Overview

2026年4月13日，年老师完成了采购融资（企业贷）项目的供应商寻源需求文档制定。这是供应商管理体系中"供应商引入"环节的核心工作，涵盖了CPS结算模式、POC测试机制、末尾淘汰机制三大核心制度。该项目从0寻源开始，计划5人团队于2026-04-17上线，通过3家供应商进场POC测试（2个月），执行末尾淘汰（5进3），建立了一套完整的"引入→测试→筛选→淘汰"供应商生命周期管理闭环。

---

## 一、项目背景与基本参数

### 1.1 项目信息

| 维度 | 内容 |
|------|------|
| 项目名称 | 采购融资（企业贷） |
| 上线时间 | 2026-04-17 |
| 团队规模 | 5人（初始寻源规模30-50人） |
| 业务产线 | 企金 |
| 寻源阶段 | 从0开始，无现有供应商 |

### 1.2 寻源特点

与已有供应商的项目不同，采购融资项目是全新业务线，需要：
- 从零筛选和引入供应商
- 建立新的结算规则
- 设计POC测试验证供应商能力
- 末尾淘汰确保最终合作质量

## 二、CPS结算模式

### 2.1 三档结算比例

| 档位 | 条件 | CPS比例 | 说明 |
|------|------|---------|------|
| 第一档 | 基础达标 | X% | 达到最低产能要求的结算比例 |
| 第二档 | 表现良好 | Y%（Y>X） | 超过基础目标的激励比例 |
| 第三档 | 优秀表现 | Z%（Z>Y） | 超出优秀目标的最高激励比例 |

### 2.2 排除规则

以下情况不计入CPS结算：
- 虚假进件（欺诈行为）
- 重复进件（同一客户多次进件）
- 违规操作（违反合规要求的进件）

### 2.3 结算周期

| 项目 | 规则 |
|------|------|
| 结算频率 | 月度结算 |
| 数据确认 | 次月5日前确认上月数据 |
| 结算时间 | 次月15日前完成付款 |
| 争议处理 | 数据确认期3个工作日内提出异议 |

## 三、POC测试机制

### 3.1 POC设计原则

| 原则 | 说明 |
|------|------|
| **公平性** | 所有供应商在相同条件下测试 |
| **可比性** | 统一的评估维度和标准 |
| **充分性** | 2个月周期，覆盖不同业务场景 |
| **透明性** | 测试规则提前公布，结果公开 |

### 3.2 POC测试流程

```
供应商筛选（3家） → 进场准备（1周） → POC执行（2个月） → 评估排名 → 淘汰决策
```

### 3.3 POC评估维度

| 维度 | 权重 | 评估指标 |
|------|------|---------|
| 产能 | 40% | 月均进件量、产能增长率 |
| 质量 | 30% | 进件通过率、退件率 |
| 合规 | 20% | 违规次数、质检合格率 |
| 配合度 | 10% | 问题响应速度、改善意愿 |

### 3.4 POC期间管理

| 管理动作 | 频率 | 内容 |
|---------|------|------|
| 周度复盘 | 每周 | 进度跟踪、问题识别 |
| 月度评估 | 每月 | 综合评分、排名 |
| 现场走访 | 至少1次 | 实地了解运营情况 |
| 数据共享 | 持续 | 向供应商反馈行业基准数据 |

## 四、末尾淘汰机制

### 4.1 淘汰规则

| 阶段 | 规则 | 时间 |
|------|------|------|
| POC期末 | 5家供应商中排名后30%的淘汰 | 第2个月末 |
| 正式运营期 | 季度≥2次综合赛马后30%→PIP→淘汰 | 每季度 |

### 4.2 5进3淘汰流程

```
POC结束 → 综合评分排名 → 前3家进入正式合作
                                  ↓
                          后2家进入PIP（1个月）
                                  ↓
                    PIP达标 → 保留（替补位置）
                    PIP不达标 → 淘汰
```

### 4.3 淘汰后的处理

| 事项 | 处理方式 |
|------|---------|
| 数据交接 | 被淘汰供应商的客户数据按约定移交 |
| 结算清算 | 按CPS比例结算至淘汰日 |
| 备选供应商 | 从POC候选名单中启动下一批测试 |

## 六、方法论总结

### 6.1 新业务线寻源的通用框架

1. **从0到1**：新业务线寻源的关键是"先试后选"，POC机制比直接签约更可靠
2. **赛马而非相马**：让供应商在实际业务中竞争，而非仅凭资质和报价筛选
3. **淘汰是常态**：末尾淘汰不是惩罚，而是确保合作质量的必要机制
4. **规则前置**：所有规则（结算、POC、淘汰）在进场前明确，避免后期争议

### 6.2 与供应商分层分级体系的关联

采购融资项目的POC+末尾淘汰机制与ABC分层体系是互补的：
- POC机制解决"怎么选"的问题（引入阶段）
- ABC分层解决"怎么管"的问题（运营阶段）
- 末尾淘汰是两者的连接点（从POC到正式分级的过渡）

### 6.3 可迁移的规律

- **新业务线的供应商管理比成熟业务线更重要**：因为没有历史数据做参考，筛选机制的设计质量直接影响后续运营
- **POC周期不宜过长**：2个月是合理窗口——足够看到能力，不会消耗太多资源
- **淘汰规则必须与结算规则配合**：如果结算比例不合理，淘汰再严格也无法吸引优质供应商
