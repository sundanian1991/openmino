# 供应商分层分级与ABC管理体系

> Sources: mino, 2026-04-13
> Raw:[brainstorm](../../raw/plans/2026-04-13-供应商分层分级体系-00-brainstorm.md); [方案要点](../../raw/plans/2026-04-13-供应商分层分级方案-方案要点.md); [result](../../raw/plans/2026-04-13-供应商分层分级方案-result.md)

## 概述

2026年4月13日，年老师推动建立了供应商ABC三级分层管理体系，替代原有的S/A/B/C/D五级评级。新体系核心变化：从"绝对分数切分"转向"相对排名切分"，引入60:30:10三维打分模型（业务督导60%定量+供应商管理30%定性+质检10%定量），并配套PIP（Performance Improvement Plan）末尾淘汰机制。这套体系解决了原有评级过细、管理动作不够聚焦的问题，实现了"分级简化、动作聚焦、数据驱动"的管理升级。

---

## 一、为什么需要重构评级体系

### 1.1 原有体系的四个痛点

| 痛点 | 具体表现 | 后果 |
|------|---------|------|
| **分级过细** | S/A/B/C/D五级，比例约14%/37%/29%/14%/6% | 管理动作分散，难以聚焦资源 |
| **淘汰规则单一** | 季度≥2次综合赛马后30%→PIP→淘汰 | 只看排名不看综合能力，误伤潜力供应商 |
| **评估维度片面** | 接通率/转化率/合格率/留存率/协作配合（100分） | 缺乏业务督导和质检的独立打分维度 |
| **赛马机制非标准化** | POC赛马有实践但缺标准化流程 | 每次规则不同，供应商无法预期 |

### 1.2 核心设计目标

新体系要解决的根本问题不是"怎么打分"，而是"怎么让管理动作聚焦"。五级评级中，S级和D级各有明确管理动作，但中间的A/B/C三级动作模糊，导致30%+29%=59%的供应商处于"被管理但不被重视"的状态。

## 二、ABC三级分层架构

### 2.1 分级逻辑选择：相对排名 vs 绝对分数

**方案A：绝对分数切分**
- A级：≥80分（优秀供应商）
- B级：60-79分（合格供应商，需关注）
- C级：<60分（不合格，进入PIP）

**方案B：相对排名切分（采用）**
- A级：排名前30%
- B级：排名30%-70%
- C级：排名后30%（进入PIP）

**选择方案B的理由**：
- 与现有赛马排名逻辑一致，降低切换成本
- 能自动适应供应商总数变化（新增或淘汰后自动重新切分）
- 避免"所有供应商都得高分"的通胀问题
- 保持竞争压力，后30%永远是C级

### 2.2 分级比例与管理动作映射

| 级别 | 比例 | 管理动作 | 资源倾斜 | 沟通频率 |
|------|------|---------|---------|---------|
| **A级** | 前30% | 战略伙伴，优先分配资源 | 高：优先项目、数据共享 | 月度深度沟通 |
| **B级** | 中间40% | 合格供应商，重点关注提升 | 中：按需支持 | 月度常规沟通 |
| **C级** | 后30% | PIP整改，限期改善 | 低：仅基础支持 | 周度跟进 |

### 2.3 与原五级的映射关系

| 原评级 | 新评级 | 变化 |
|--------|--------|------|
| S级（14%） | A级 | 保留精英地位 |
| A级（37%） | A级+B级（部分） | 拆分，头部留A，尾部降B |
| B级（29%） | B级+C级（部分） | 拆分，头部留B，尾部降C |
| C级（14%） | C级 | 保留，进入PIP |
| D级（6%） | C级 | 合并入C级，统一PIP处理 |

## 三、60:30:10三维打分模型

### 3.1 打分角色与权重分配

| 打分角色 | 权重 | 性质 | 核心关注 | 打分频率 |
|---------|------|------|---------|---------|
| **业务督导** | 60% | 定量 | 业务结果（产出、转化、目标达成） | 月度 |
| **供应商管理**（年老师） | 30% | 定性 | 管理能力、配合意愿、资源稳定性 | 月度 |
| **质检培训运营** | 10% | 定量 | 质检合格率、合规性 | 月度 |

### 3.2 业务督导维度（60%定量）

| 指标 | 权重 | 计算方式 | 目标值 |
|------|------|---------|--------|
| 产能达成率 | 25% | 实际产能/目标产能 | ≥100% |
| 转化率 | 15% | 有效转化/总接通 | 行业基准 |
| 接通率 | 10% | 接通量/分配量 | ≥85% |
| 目标达成率 | 10% | 完成指标/设定指标 | ≥100% |

### 3.3 供应商管理维度（30%定性）

| 指标 | 权重 | 评估标准 |
|------|------|---------|
| 管理能力 | 10% | 团队稳定性、培训计划、现场管理 |
| 配合意愿 | 10% | 对管理要求的响应速度、改进态度 |
| 资源稳定性 | 10% | 人员流失率、场地稳定性、资金投入 |

### 3.4 质检维度（10%定量）

| 指标 | 权重 | 计算方式 |
|------|------|---------|
| 质检合格率 | 5% | 合格录音/抽检总量 |
| 合规性 | 5% | 违规次数、投诉率 |

### 3.5 打分模板结构

```
供应商：XXX    月份：2026年X月

一、业务督导打分（60分）
  1. 产能达成率（15分）：___分
  2. 转化率（9分）：___分
  3. 接通率（6分）：___分
  4. 目标达成率（6分）：___分
  小计：___/60分

二、供应商管理打分（30分）
  1. 管理能力（10分）：___分
  2. 配合意愿（10分）：___分
  3. 资源稳定性（10分）：___分
  小计：___/30分

三、质检打分（10分）
  1. 质检合格率（5分）：___分
  2. 合规性（5分）：___分
  小计：___/10分

总分：___/100分
排名：___/___
级别：A/B/C
```

## 四、PIP（绩效改善计划）机制

### 4.1 PIP触发条件

| 条件 | 说明 | 自动触发 |
|------|------|---------|
| 季度综合排名后30% | 连续两个季度 | 是 |
| 单月评分<60分 | 绝对分数低于阈值 | 是 |
| 重大合规事件 | 质检严重不合格 | 是 |
| 连续两月人力下降>20% | 规模收缩预警 | 否（需人工判断） |

### 4.2 PIP流程

```
触发 → 通知（书面） → 制定改善计划（7天） → 执行改善（30天） → 评估结果
                                                                ↓
                                               改善成功 → 移出PIP，恢复正常管理
                                               改善失败 → 末尾淘汰
```

### 4.3 改善计划内容要求

| 要素 | 说明 | 示例 |
|------|------|------|
| 问题描述 | 具体存在的问题 | 3月产能达成率仅72% |
| 改善目标 | 量化的改善目标 | 4月产能达成率≥90% |
| 改善措施 | 具体行动方案 | 增加培训频次、优化排班 |
| 资源支持 | 年老师提供的帮助 | 数据分析支持、标杆学习 |
| 评估标准 | 如何判断改善成功 | 4月评分≥70分 |
| 时间节点 | 关键里程碑 | 第1周诊断、第2周执行、第4周评估 |

### 4.4 末尾淘汰规则

| 条件 | 后果 | 时间窗口 |
|------|------|---------|
| PIP后连续2月未达标 | 退出合作 | 第3个月启动 |
| PIP后1月未达标但改善趋势明显 | 延长PIP 1个月 | 最多延长1次 |
| 重大合规事件 | 立即退出 | 即时 |

### 4.5 与原淘汰规则的对比

| 维度 | 原规则 | 新规则 |
|------|--------|--------|
| 触发条件 | 季度≥2次赛马后30% | ABC排名后30%+PIP |
| 改善周期 | 无明确周期 | 30天标准周期 |
| 评估维度 | 仅看排名 | 综合评分（60:30:10） |
| 淘汰缓冲 | 直接淘汰 | PIP改善+延长机会 |
| 灵活性 | 刚性 | 可延长1次 |

## 六、实施节奏

### 6.1 第一阶段：准备期（2周）

- [ ] 确认打分维度和权重
- [ ] 制作打分模板（Excel）
- [ ] 培训业务督导和质检打分人员
- [ ] 通知供应商新体系上线

### 6.2 第二阶段：试运行（1个月）

- [ ] 用历史数据模拟打分，验证分级结果
- [ ] 对比新旧评级结果，检查合理性
- [ ] 根据试运行结果调整阈值和权重

### 6.3 第三阶段：正式上线

- [ ] 首次正式打分和排名
- [ ] 识别首批C级供应商，启动PIP
- [ ] 月度复盘，持续优化

### 6.4 关键风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 打分标准不统一 | 高 | 分级结果失真 | 打分人培训+交叉验证 |
| 供应商反弹 | 中 | 合作关系恶化 | 充分沟通、透明规则 |
| 王易人审批延迟 | 中 | 上线推迟 | 提前沟通、方案对齐 |
| PIP供应商大面积流失 | 低 | 供应能力不足 | 提前储备备选供应商 |

## 七、方法论总结

### 7.1 可迁移到供应商管理其他场景的规律

1. **分级数量与管理效率成反比**：分级越细，管理动作越分散。3级是最优平衡点——够区分、好聚焦。
2. **相对排名优于绝对分数**：绝对分数会导致通胀，相对排名保持竞争压力。
3. **淘汰机制必须有缓冲**：直接淘汰会吓跑所有供应商，PIP提供改善机会的同时保持压力。
4. **定性打分需要约束**：30%定性部分如果不定义清晰标准，会沦为"印象分"。必须有明确的评估维度。

### 7.2 与权责不对等体系的关联

ABC分层体系是年老师在可控范围内建立的管理工具。它不触及策略组的定价权（不可控），但强化了供应商管理的定性评估权（可控）。通过60:30:10模型，年老师的30%定性打分权有了制度化的表达方式——这不是与策略组争夺权力，而是在自己的职责范围内建立更科学的管理标准。
