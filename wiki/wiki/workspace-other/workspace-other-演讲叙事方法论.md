## 概述

AI 定价模式的 25 页演讲脚本不仅是一份行业分析材料，更是一份完整的"演讲叙事方法论"案例。25 分钟、25 页、6 个节奏段、3 个核心结论，每一步都有明确的设计意图。这份脚本展示了一个可复用的商业演讲叙事框架。

## 核心叙事结构：六段节奏

演讲按时间分为六个节奏段，每段有不同的信息密度和节奏控制：

| 时段 | 内容 | 节奏 | 目的 |
|------|------|------|------|
| 0-3 min | 封面 + 核心判断 + 时间线 | 快 | 建立全景感 |
| 3-8 min | 三股力量 + 格局 + 三段式 | 稳 | 铺背景知识 |
| 8-13 min | 计量混乱 + Agent 成本 + OpenClaw | 强 | 数据冲击 |
| 13-18 min | 三层逻辑 + 五个洞察 | 深 | 分析框架 |
| 18-22 min | 未来预判 + 芯片 + 开源 | 远 | 趋势判断 |
| 22-25 min | 决策清单 + 反模式 + 收尾 | 实 | 带走的东西 |

这是典型的"快-稳-强-深-远-实"六段节奏设计：
- **快**：开场不拖泥带水，3 分钟内让听众知道"这 25 分钟讲什么"
- **稳**：铺足背景，让所有人都能跟上
- **强**：用数据制造冲击感，唤醒注意力
- **深**：提供分析框架，让听众带走方法论
- **远**：看趋势，建立前瞻性
- **实**：回到实操，给出可执行的清单

## 开场策略：跨域类比

### 电信故事开场

演讲没有自我介绍，没有"大家好我是谁"，直接用一个类比开场：

> 2009 年，你充 100 块话费，根本用不完。2017 年，各种"不限量"套餐，但达量降速。2020 年，真正无限流量消失了，你开始按 GB 精打细算。这条路，AI 正在精确重演。

这个开场有三个设计精妙之处：

1. **共鸣感**：每个人都经历过电信套餐演变，不需要解释
2. **悬念感**："AI 正在精确重演"——怎么重演？听众想听下去
3. **时间压缩**：电信行业用 20 年走完的路，AI 行业可能 3 年走完

### 核心判断前置

第 2 页（开场后第 2 分钟）就给出核心判断：

> Coding Plan 是过渡产品，Token Plan 是终态。

这不是最后才揭示的结论，而是先说结论，后面 23 页全是证明。这种"先结论后论证"的结构比"先铺陈后总结"更有力——听众带着问题听下去。

## 分析框架的递进式设计

### 框架层级

演讲中使用了三个层级的分析框架：

**第一层：描述性框架**（建立共识）
- 三股力量：模型进步 / Agent 爆发 / 芯片供给
- 七家竞争格局：专卖店 vs 超市路线
- 三段式：补贴 → 收紧 → 分层

**第二层：分析性框架**（深度拆解）
- 定价三层逻辑：成本驱动 / 竞争驱动 / 价值驱动
- 波特五力分析

**第三层：洞察性框架**（可带走的方法论）
- 计量单位即定价权
- 从入口到生态的升级路径
- 国内 vs 海外两套定价逻辑

三层框架从"是什么"到"为什么"到"带走什么"，逐层深入。

## 数据冲击策略

### 数字选择的艺术

演讲中的每个数字都不是随意选择的，而是刻意制造"冲击感"：

| 数字 | 冲击点 |
|------|--------|
| 100 亿 Token / 8 个月 | Agent 的消耗量级 |
| 15000 美元 API 价值 vs 800 美元订阅费 | 订阅是地板不是天花板 |
| 355K stars（OpenClaw） | 生态规模 |
| 90% 开发者会踩坑 | 计量混乱的受害者规模 |
| 950PR 是 H20 的 2.87 倍，价格三分之一 | 芯片替代的经济性 |
| 66% → 54%（英伟达市占率） | 格局变化 |

### 对比制造冲击

"同样每月 40-50 块"的对比：
- 智谱：2400 个 Prompt
- MiniMax：每 5 小时 40 个 Prompt
- 阿里百炼：18000 次 API 请求
- 腾讯云：3500 万 Token，约 70 轮深度问答

同样的价格，完全不同的计量方式。这不是数字对比，是"不可比"的荒谬感展示。

## 首尾呼应策略

### 开场：电信类比

2009 话费 → 2017 不限量降速 → 2020 按 GB 计费

### 结尾：电信类比回归

2009 话费 → 2024 API 按量
2013 流量套餐 → 2025 Coding Plan
2017 达量降速 → 2026 分层套餐
2020 按 GB 计费 → 2026+ Token 精细计量

> 历史不会重复，但会押韵。电信行业用 20 年走完的路，AI 行业可能 3 年就走完。

首尾呼应让听众产生"故事闭环"的满足感。

## 结尾策略：问题而非总结

结尾不是"谢谢大家"，而是一个问题：

> 如果你现在要做一个 AI 产品的定价策略，你会选择先补贴跑量，还是从一开始就走差异化路线？

这个设计有三个目的：
1. 让听众带走思考（不是带走结论）
2. 为后续讨论留入口
3. 承认问题没有唯一正确答案

## 可复用模式总结

### 演讲叙事通用模板

| 阶段 | 页数 | 节奏 | 关键动作 |
|------|------|------|----------|
| 吸引 | 1-3 | 快 | 类比开场 + 核心判断 + 时间线 |
| 铺垫 | 3-8 | 稳 | 背景框架 + 竞争格局 |
| 冲击 | 8-13 | 强 | 具体数据 + 对比冲击 |
| 深挖 | 13-18 | 深 | 分析框架 + 核心洞察 |
| 远见 | 18-22 | 远 | 趋势预判 + 前沿信息 |
| 落地 | 22-25 | 实 | 决策清单 + 反模式 + 问题结尾 |

### 三个关键设计原则

1. **结论前置**：先说结论，后面全是证明。不要最后才揭晓答案。
2. **数据造冲击**：用具体数字和对比制造"不可比"的荒谬感。
3. **首尾呼应**：开场的类比要在结尾回归，形成闭环。

### 对年老师的应用

供应商季度汇报同样可以应用这个叙事结构：

- 开场：用类比引入（如"供应商管理就像足球队..."）
- 核心判断前置："本月最关键的发现是..."
- 数据冲击：用环比变化和横向对比制造冲击
- 分析框架：用三问原则拆解问题
- 结尾：不是"总结"，而是"下月我们需要重点关注什么？"
