## 概述

数据炼金师（Data Alchemist）是一个 AI 驱动的数据叙事工具，创建于 2026-04-26。用户上传真实数据（CSV/Excel），工具通过引导式问答明确受众和目标，AI 自动分析数据并应用 Storytelling with Data 的 30 个可视化技能，生成包含多图表、三幕叙事的完整 HTML 报告，可直接用于商业汇报。

## 产品定位与设计哲学

### 核心定位

**实用工具 + 游戏化外壳**。核心是出报告，外壳提供沉浸感。

- **用户画像**：商务分析师、运营经理、需要向管理层做数据汇报的人
- **核心价值**：降低"用数据讲故事"的门槛，让不懂可视化的人也能产出专业级报告
- **差异化**：不是 BI 工具（如 Tableau），不是图表库（如 ECharts），而是"数据 → 故事"的端到端转换器

### 游戏化外壳的设计意图

数据炼金师采用游戏化设计（炼金师角色、5 步进度条、通关粒子动画），目的是：

1. **降低焦虑感**：面对数据和 AI，用户容易紧张。游戏化让过程变得轻松
2. **建立期待**：进度条和动画让"等待"变成"期待"
3. **成就感**：评分系统（100 分制）和通关动画给用户即时正反馈

但核心原则明确：游戏化是外壳，不是核心。报告质量是唯一的验收标准。

## 技术架构深度解析

### 前端技术栈

| 层 | 技术 | 选择理由 |
|----|------|----------|
| 框架 | React 18 + TypeScript | 类型安全，组件复用 |
| 构建 | Vite | 开发体验好，打包速度快 |
| UI | shadcn/ui + TailwindCSS | 原子化 CSS，按需导入组件 |
| 图表 | ECharts for React | 图表类型丰富，社区活跃 |
| 动画 | Framer Motion | 声明式动画，layout 动画优秀 |
| 状态 | Zustand | 轻量，API 简洁，无 boilerplate |
| 路由 | React Router v6 | 标准选择 |

### 数据流架构

```
用户上传文件 → Papa Parse/SheetJS 解析 → DataStore 状态
                                      ↓
用户回答 AI 问题 → ChatStore 状态机推进 → Claude API 调用
                                      ↓
AI 返回 ChartSpec JSON → ChartFactory 翻译 → ECharts Option → 渲染
                                      ↓
NarrativeStore 构建三幕叙事 → 合并导出 → 独立 HTML 文件
```

### 分层 Prompt 策略

这是降低 AI 调用成本的核心设计：

| Layer | 内容 | Token 量 | 使用阶段 |
|-------|------|----------|----------|
| 1 | 角色定义 | ~500 | 全程 |
| 2 | 核心规则（禁止项+图表映射+配色+标题+高亮+排序） | ~2000 | 阶段 1-3 |
| 3 | 图表指南（功能→图表映射+标注策略+复杂模板） | ~3000 | 阶段 4+ |
| 4 | 叙事规则（Big Idea 公式+三幕模板+受众适配+数据伦理） | ~2000 | 阶段 4+ |
| 5 | 输出格式（JSON Schema + 图表规格结构） | ~500 | 阶段 4+ |

阶段 1-3（问答阶段）只注入 Layer 1-2（~2500 tokens）。阶段 4+（生成阶段）注入 Layer 1-5（~8000 tokens）。使用 prompt caching 降低重复调用成本。

### 图表工厂架构

AI 返回 JSON ChartSpec，图表工厂将其翻译为 ECharts Option。

ChartSpec 包含：
- 图表类型、标题、数据源
- 高亮规则（目标、颜色、其他元素颜色）
- 注释（类型、值、标签、颜色）
- 叙事文本
- 所属幕（1/2/3）

图表工厂强制执行的视觉规则：
- 条形图必须零基线
- 高亮不超过 10% 的元素
- 先灰后彩（高亮系列用主色，其余灰化）
- 直接标注，不使用图例
- 排序按故事焦点

### 对话状态机

```
IDLE → DATA_UPLOADED → Q1_AUDIENCE → Q2_GOAL → Q3_METRICS
     → CHART_RECOMMENDED → CHART_PREVIEW → NARRATIVE_COMPLETE → EXPORT_READY
```

每步交互的具体问题：
1. **DATA_UPLOADED**：AI 分析数据结构，反馈"我分析了 X 行 Y 列数据，包含[字段]。这份报告是给谁看的？"
2. **Q1_AUDIENCE**："决策者是谁？他们对你的数据持什么态度？（信任/中立/怀疑）"
3. **Q2_GOAL**："你希望受众看完报告后做什么决定？用一个强动词描述"
4. **Q3_METRICS**："哪些指标最关键？有基准线或目标值吗？"
5. **CHART_RECOMMENDED**：AI 返回完整图表推荐方案 + 三幕叙事

## 评分系统：五维度评估

5 维度 × 20 分 = 100 分：

| 维度 | 满分条件（20 分） | 低分条件（5 分） |
|------|-------------------|------------------|
| 受众定义 | 有具体人名/角色 | 模糊标签 |
| 行动目标 | 有强动词 | "分析一下" |
| 图表合理性 | 类型匹配功能目标 | 饼图/3D |
| 叙事完整度 | 三幕完整 | 缺某幕 |
| 数据伦理 | 有反对证据 | 没有 |

评分不仅是游戏化元素，更是对数据汇报质量的量化评估。

## 开发执行顺序

16 步执行计划，从项目搭建到端到端测试，每一步都有明确的交付物。关键里程碑：

- 第 3 步：数据上传 + 解析完成
- 第 8 步：图表工厂完成
- 第 9 步：首张图表渲染（重要里程碑）
- 第 12 步：报告导出完成
- 第 16 步：错误处理 + 端到端测试

## 对年老师的价值

数据炼金师的方法论可以直接应用于供应商月度汇报：

1. **引导式问答**：通过回答 3 个问题（受众、目标、关键指标）明确汇报方向
2. **图表选择**：根据数据类型自动推荐合适的图表类型
3. **叙事结构**：三幕叙事框架让汇报有起承转合
4. **质量评估**：五维度评分系统可以自我检视汇报质量
