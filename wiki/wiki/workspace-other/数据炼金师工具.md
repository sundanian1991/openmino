## 概述

数据炼金师（Data Alchemist）是一个 AI 驱动的数据叙事工具项目，创建于 2026-04-26。该项目是 workspace-other 目录下技术深度最高的项目之一，拥有完整的 SPEC 文档、技术栈选型、产品架构和功能规格。

## 产品定位

**核心定位**：实用工具 + 游戏化外壳。核心是出报告，外壳提供沉浸感。

**用户**：商务分析师、运营经理、需要向管理层做数据汇报的人。

**工作原理**：用户上传真实数据（CSV/Excel），工具通过引导式问答明确受众和目标，AI 自动分析数据并应用 Storytelling with Data 的 30 个可视化技能，生成包含多图表、三幕叙事的完整 HTML 报告，可直接用于商业汇报。

## 核心功能

### AI 对话引擎

5 步状态机：
1. **DATA_UPLOADED**：AI 分析数据结构，反馈"我分析了 X 行 Y 列数据"
2. **Q1_AUDIENCE**："决策者是谁？他们对你的数据持什么态度？"
3. **Q2_GOAL**："你希望受众看完报告后做什么决定？用一个强动词描述"
4. **Q3_METRICS**："哪些指标最关键？有基准线或目标值吗？"
5. **CHART_RECOMMENDED**：AI 返回完整图表推荐方案 + 三幕叙事

AI 调用策略采用分层 prompt 加载：阶段 1-3 注入 Layer 1-2（核心规则 ~2500 tokens），阶段 4+ 注入 Layer 1-5（完整 ~8000 tokens），使用 prompt caching 降低重复调用成本。

### 图表工厂

支持 10 种图表类型：水平条形图、垂直柱状图、折线图、面积图、堆叠条形图、发散条形图、散点图、热力图、坡度图。

视觉规则（从 SKILL 知识强制执行）：
- 条形图必须零基线
- 高亮 ≤10% 元素
- 先灰后彩（高亮系列用主色，其余灰化）
- 直接标注，不要图例
- 排序按故事焦点

### 三幕叙事

UI 结构分为三幕：
- **第一幕 — 设定**：场景描述 + 失衡点 + 2 张图表
- **第二幕 — 论证**：证据描述 + 不行动的后果 + 3 张图表
- **第三幕 — 行动**：行动号召 + 紧迫性 + 2 张图表 + 反对证据与局限性

### 评分系统

5 维度 × 20 分 = 100 分：
1. **受众定义**：有具体人名/角色（20）vs 模糊标签（5）
2. **行动目标**：有强动词（20）vs "分析一下"（5）
3. **图表合理性**：类型匹配功能目标（20）vs 饼图/3D（0）
4. **叙事完整度**：三幕完整（20）vs 缺某幕（5-10）
5. **数据伦理**：有反对证据（20）vs 没有（5）

### 游戏化外壳

- 5 步进度条（Framer Motion 过渡）
- "数据炼金师"角色人设
- 步骤切换动画
- 通关粒子动画
- 评分弹窗

## 项目意义

数据炼金师是"数据讲故事"方法论的技术化实现。它将 Storytelling with Data 的抽象原则转化为可执行的代码规则，将三幕叙事的文学结构转化为可填充的 UI 模板。这个项目展示了年老师对"让数据自己讲故事"这一理念的深度追求——不是简单的图表生成，而是从受众定义到行动号召的完整叙事闭环。
