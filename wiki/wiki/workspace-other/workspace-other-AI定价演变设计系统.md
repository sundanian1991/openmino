## 概述

AI 定价演变项目产出了一份 25 页演讲脚本和一份独立的设计规格文档。design-spec 记录了从 25 页 PPT 迭代过程中积累的视觉设计规范，涵盖了色彩系统、版式布局、字体层级、图标规范、开发流程等完整的设计体系。这份规格文档是从"踩过坑"中提炼出来的方法论，对后续同类 PPT 项目有直接复用价值。

## 色彩系统：暖米色主题

### 色彩角色定义

| 角色 | 色值 | 设计意图 |
|------|------|----------|
| 背景主色 | `#F5F0E8` | 暖米色，替代冷白，营造纸张质感 |
| 卡片背景 | `#EDE8DD` | 同色系深一层，形成柔和层次 |
| 文字主色 | `#2C2825` | 深暖灰，比纯黑柔和 |
| 文字次色 | `#6B6864` | 中暖灰，描述文字 |
| 文字弱化 | `#9A9590` | 浅暖灰，标签、页码 |
| 强调红 | `#C13531` | 关键数据、结论高亮 |
| 强调蓝 | `#293C54` | 对比数据、次要强调 |
| 分割线 | `rgba(44,40,37,0.12)` | 极淡，内部结构分隔 |

### 卡片颜色铁律

核心规则：卡片背景永远用 `#EDE8DD`，**禁止用 `#fff` 白色**。

原理：卡片背景色与页面背景色同色系不同深浅，形成柔和层次。白色卡片在暖米色底上会产生割裂感，破坏整体的纸张质感。

半透明强调卡片用 `rgba(色值, 0.08)` 或 `rgba(色值, 0.10)`，用于需要突出的重点区块。

### 禁止项

- 渐变（`linear-gradient`、`radial-gradient`）—— 封面光晕除外，需极淡
- `#fff` 白色卡片背景在暖色底页面上
- 彩色边框替代纯色背景（badge 用实心色块，不用透明+边框）

## 边框规则：极少主义

### 禁止使用

- 卡片外边框（`border: 1px solid ...`）
- IPO badge、价格框、市场条等区块的彩色边框
- 卡片内部的颜色分隔线用 border 实现

### 允许使用

- 表格/列表的内部结构分隔线（极淡 `rgba(154,149,144,0.25)` 以下）
- 价格对比表的行分隔（`border-bottom: 1px solid rgba(44,40,37,0.06)`）
- timeline 轴心线（`#CDCECD`）
- 封面 tag 的强调性边框（有意设计）

### Badge 设计规范

正确做法：实心色块，文字用反白。
```css
.badge.red { color: #F5F0E8; background: #C13531; }
```

错误做法：透明背景加彩色边框。
```css
.badge.wrong { color: #C13531; background: transparent; border: 1px solid #C13531; }
```

## 版式与布局

### scale-wrap 模式

所有页面统一使用 scale-wrap 模式实现 1920x1080 固定尺寸自适应缩放：

```html
<div class="scale-wrap">
  <!-- 页码/标签 -->
  <!-- 主内容 -->
  <script>fit()</script>
</div>
```

### 居中规则

- 所有内容**强制水平+垂直居中**
- scale-wrap 已有 `translate(-50%, -50%)` 确保居中
- 内容容器用 flex 布局：`display: flex; flex-direction: column; align-items: center; justify-content: center`
- **禁止**使用 `writing-mode: vertical-rl` 做侧边标注（容易 collapse 成单字）

### 间距系统

| 层级 | 值 | 场景 |
|------|----|------|
| 小间距 | 8-12px | 图标+文字、badge 之间 |
| 中间距 | 20-36px | 卡片之间、列表项之间 |
| 大间距 | 48-60px | 内容区块之间 |
| 超大间距 | 70-120px | 封面/结论页的大块留白 |

### 卡片网格布局

- 4 个卡片 → 2x2 网格，不用 4 列横排
- 3 个卡片 → 3 列横排（卡片宽度适中时）或 1 列垂直堆叠（内容多时）
- 2 个卡片 → 左右并排
- 对比型 → 左右两列 + 中间箭头/鸿沟

### 对称检查

五力图等放射布局：中心圆必须在卡片矩阵的几何中心。卡片数量不对称时，考虑用 grid 而非 flex。

## 字体层级

| 层级 | 字号 | 字重 | 字体 | 用途 |
|------|------|------|------|------|
| 超大标题 | 80-120px | 700 | Noto Serif SC | 结论页大字 |
| 页面标题 | 48-52px | 700-900 | Noto Serif SC | 每页标题 |
| 卡片标题 | 22-32px | 700-900 | Noto Serif SC | 卡片/区块标题 |
| 正文 | 16-22px | 400 | 默认 | 描述文字 |
| 强调正文 | 16-22px | 600-700 | 默认 | 需要加粗的描述 |
| 标签 | 12-16px | 600 | JetBrains Mono | 页码、badge、计量单位 |
| 数据大字 | 28-72px | 700 | JetBrains Mono | 关键数字 |

### 字号比例检查

- 页面标题与卡片标题的差距至少 16px
- 数据大字与描述文字的差距至少 12px
- 标签文字不得大于 16px，不得喧宾夺主

字体选择逻辑：Serif 用于标题营造权威感和学术感，Sans-serif/Mono 用于数据和标签营造精确感。

## 图标与 SVG 规范

### 线条画标准

- `fill="none"`，纯线条画
- `stroke-width="2.5"`，`stroke-linecap="round"`，`stroke-linejoin="round"`
- 颜色跟随主题色：红 `#C13531`、蓝 `#293C54`、灰 `#CDCECD`

### 尺寸规则

- 行内图标：36-48px
- 列表前小图标：6-12px

### 禁止

- 填充型图标
- emoji
- text-based logos
- 过于简单的圆/方框冒充图标

## 开发流程：从规格到交付

### 强制步骤

1. **先写设计规格**：确认色彩、版式、组件样式，再动手写 HTML
2. **做 3 页后合并预览**：不要 25 页全做完才合并，早期验证设计方向
3. **每页完成后自检**：
   - 卡片背景色是否为 `#EDE8DD`（不是 `#fff`）
   - 是否有多余的 `border`
   - 内容是否居中
   - 字号层级是否符合规范
   - SVG 图标是否统一风格
4. **最终全量合并后检查**：
   - `grep "background: #fff"` → 0 结果
   - `grep "border: 1px solid rgba"` → 0 结果
   - `grep "linear-gradient"` → 0 结果（除非封面有意）

### 迭代教训

这份规格文档标注"2026-04-26 初稿，源于 AI 定价演变 PPT 25 页迭代教训"，说明这些规则是从多次返工、不满意、反复修改中提炼出来的。核心教训：

- 先定规范再动手，避免全局返工
- 小步验证，不要一口气做完所有页面
- 用 grep 等工具做自动化检查，不要靠肉眼

## 对后续项目的指导意义

这套设计规格可以复用于所有"商业汇报型 PPT"项目：

- 供应商季度汇报
- 项目交接材料
- 管理层汇报
- 商业分析报告

核心价值：把"好看"变成可执行的规则，而不是模糊的审美判断。
