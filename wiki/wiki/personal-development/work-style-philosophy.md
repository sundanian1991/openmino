# 工作风格哲学

> Sources: 年老师, 2026-03-04
> Raw:[工作风格](../../raw/personal-growth/工作风格-04-工作风格.md); [工作风格-表达模板库](../../raw/personal-growth/工作风格-表达模板库.md)

## 概述

年老师的工作风格可以用三个关键词概括：结构化、数据驱动、生活化类比。核心信念是"逻辑清晰比花哨重要，接地气比高大上有效"。

## 表达框架工具箱

| 框架 | 适用场景 | 结构 |
|------|---------|------|
| **SCQA** | 问题汇报 | 情境 → 冲突 → 问题 → 答案 |
| **RIDE** | 风险沟通 | 风险 → 利益 → 差异 → 影响 |
| **STAR** | 案例讲述 | 情境 → 任务 → 行动 → 结果 |
| **GROW** | 辅导对话 | 目标 → 现状 → 方案 → 行动 |
| **PDCA** | 迭代改进 | 计划 → 执行 → 检查 → 行动 |
| **PREP** | 观点表达 | 观点 → 理由 → 例证 → 观点 |

## 沟通风格特征

**结构化表达**：不零散回答问题，先给框架再填充细节。

**用数据说话**：不用"大概"、"差不多"，用具体数字支撑判断。

**生活化类比**：
- 供应商管理 → "餐厅外包服务员"：你是餐厅老板，外包的服务员干得好不好，取决于你立的规矩和考核标准
- 选汰机制 → "给新人机会，但不养懒人"

**接地气、直白**：不绕弯，不说"看情况"，给明确判断。

## 定性+定量融合表达层级

| 层级 | 特征 | 一句话判断 |
|------|------|-----------|
| **L1** | 只有感受 | "我觉得不太好" → 能验证吗？不能 |
| **L2** | 有定性无定量 | "问题出在人员流失" → 有数据吗？没有 |
| **L3** | 有定量无定性 | "达标率降了15%" → 为什么？说不清楚 |
| **L4** | 定性+定量融合 | "达标率降15%，因人员流失，下周补充培训" → 有数据、有原因、有下一步 |

**目标始终在 L4 层级**。

## 典型场景表达模板

### 向上级汇报（军哥）

**业绩出问题**：
> "[供应商]这周/这月KPI掉了X个百分点，主要卡在[具体指标]。我查了一下，原因是[根因]。我的方案是[行动]，预计[时间]能追回来。"

**需要资源**：
> "现在[业务]量涨了X%，但供应商产能已经到Y%了，下个月会有Z%的缺口。建议[方案]，需要您[支持内容]。"

**清退决策**：
> "[供应商]连续[N]个月KPI都在[X]%以下，我们也给了整改机会，但效果不明显（具体数据）。我的判断是建议清退，转移大概需要[时间]。"

### 供应商沟通（对外）

**业绩预警**：
> "你们本月达成率X%，比目标低了Y个百分点。我看了下数据，差距主要在[时段/环节]。麻烦你们分析下原因，[时间]前给我一个整改计划。"

## 决策模式

**先想透再做稳**：面对复杂问题，先理清框架再动手，不在方向没确认时就投入执行。

**三问原则**：做任何决策前，问自己三个问题——这个问题该谁解决？我现在该解决什么？这个决定对供应商有什么影响？

**边界清晰**：明确什么是自己的责任，什么是别人的，不越界也不推卸。

## 问题解决模式

1. **定义问题**：不是"供应商不配合"，而是"我们没有建立让供应商愿意配合的机制"
2. **拆解维度**：从选用育留汰五个维度分别看
3. **数据验证**：用指标说话，不用感觉
4. **机制设计**：治本不治标，设计规则而非临时方案

## 设计审美偏好

| 维度 | 偏好 |
|------|------|
| 空间 | 充足留白（p-6/p-8），呼吸感 |
| 圆角 | 12-16px（rounded-xl），现代友好 |
| 主文本 | #0f172a |
| 次级色 | #64748b |
| 背景色 | #fafaf9 |
| 组件库 | Shadcn UI / Radix UI 风格 |
| 风格排序 | Clean > Minimalist > Modern |
| 过渡 | Hover 状态、transition-all duration-300 |

**色彩系统原则**：明亮配色 > 深色主题；正常比例 > 夸张设计；清晰层级 > 装饰元素。

## 工作理念

| 原则 | 做法 |
|------|------|
| 物尽其用 | 不是"要不要"，是"怎么用好"——删除是最后一步 |
| 机制简化优先于自动化 | 能手动就用手动，太复杂的机制反而增加负担 |
| 边界清晰 > 功能复杂 | 每个机制有明确职责，不重叠，不混乱 |
