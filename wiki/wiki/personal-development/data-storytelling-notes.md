# 数据讲故事笔记 — Storytelling with Data

> Sources: Cole Nussbaumer Knaflic, 2026-04-28
> Raw: [全书知识汇总](../../raw/personal-growth/Storytelling_with_Data_-_Cole_Nussbaumer_Knaflic-全书知识汇总-中文版.md)

## 概述

《Storytelling with Data》将数据沟通拆解为完整流程：理解受众 → 明确目标 → 选图表 → 去噪音 → 引导注意力 → 讲故事 → 迭代打磨。30 个具体 skill 可归纳为 6 个核心模块。

## 完整流程框架

```
理解受众 → 明确目标 → 选图表 → 去噪音 → 引导注意力 → 讲故事 → 迭代打磨
```

### 阶段一：建立上下文

1. **WHO**：谁会受你的数据影响
2. **WHAT**：受众需要知道什么、做什么
3. **HOW**：用什么指标支撑你的信息
4. **Big Idea**：一句话，包含洞察 + 建议
5. **验证**：所有设计选择是否服务于 Big Idea

### 阶段二：六个核心步骤

1. 理解上下文
2. 选合适的视觉形式
3. 消除噪音（Gestalt 原则）
4. 引导注意力（前注意属性）
5. 功能优先于形式
6. 用三幕结构讲故事

## 模块一：沟通策略

**定义受众**：找出真正的决策者。多类受众有分歧时，做分开的沟通，不要拼成一个谁都不满意的版本。

**行动导向目标**：用强动词（批准、调整、优先、启动），而非"了解、注意、关注"。不完美的建议也比被动的总结好。

**三幕叙事结构**：

| 幕 | 内容 | 关键动作 |
|----|------|----------|
| 铺垫 | 设定场景，指出失衡 | 回答"我为什么要关心" |
| 冲突 | 用证据展开问题 | 建立你的推荐案的论据 |
| 决议 | 明确的行动号召 | 具体到"做什么、谁做、何时" |

## 模块二：图表选择

**功能决定形式**：先定义受众该做什么/理解什么，再选图。

| 功能目标 | 图表形式 |
|----------|----------|
| 时间趋势 | 折线图 |
| 分类对比 | 条形图 |
| 相关性 | 散点图 |
| 一个关键数字 | 大字，不是图 |

**禁止清单**：

- 饼图/环形图：人脑不擅长比较角度和面积
- 双 Y 轴：暗示虚假关联
- 3D 视觉：扭曲感知

**饼图替代方案**：占比对比用水平条形图；部分与整体关系用 100% 堆叠条形图；关键数字用大字展示；比例变化用坡度图。

## 模块三：视觉清晰度

**最小化认知负荷**：每个视觉元素都要问："它的信息价值值得它的认知成本吗？"

**前注意属性高亮法则**：≤10% 的元素要突出，其余灰化。按效果排序：加粗 > 颜色 > 大小 > 位置。

**Gestalt 原则**：

| 原则 | 做法 |
|------|------|
| 接近性 | 相关项目靠近，不相关的拉开 |
| 相似性 | 同色/同形/同大表示关联 |
| 闭合 | 去掉外框，人脑自动感知完整 |
| 连续性 | 沿平滑路径对齐，去不必要轴线 |

**多线图解构三步法**：评估数据必要性 → 选择解构策略（逐个高亮/垂直小多图/水平小多图）→ 强制轴范围一致性。

**分类数据排序**：按意图排序，不按字母。关注最不满意就按"完全不满意"降序排。多个故事要讲时保持一个基础顺序不变。

## 模块四：迭代流程

**验光师迭代法**：每次只改一个元素，左右对比选优。防止多个变化混在一起，确保每个决策有直接对比支撑。

**纸笔草图**：用空白纸/白板探索方案，重结构不重细节。草图易丢弃，减少情感依附。

**外部反馈**：找代表受众的同事，不解释你的意图，让他们自己看。只听不辩。

## 模块五：伦理与适应

**数据伦理**：数据论证必须包含反对证据和上下文。把反对点不是当弱点，而是当深度分析的证据。

**载体适配**：

| 载体 | 规则 |
|------|------|
| 演讲 | 页面精简，视觉强化口头观点 |
| 报告/邮件 | 包含更多细节，确保自包含 |
| Slidument | 用附录满足文字细节需求 |

## 核心原则总结

1. **条形图零基线强制**：截断 Y 轴 = 视觉欺骗
2. **10 秒测试**：非专业人士能否 10 秒内理解要点
3. **功能优先于形式**：永远不要让美学覆盖功能清晰度
4. **分析完成后再花同等时间打磨沟通**：受众只看到沟通阶段，看不到你清洗数据的过程
