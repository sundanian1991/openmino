# 极简信息图设计规范

> Sources: Mino, 2026-04-27
> Raw:[极简信息图复刻指南](../../raw/toolbox/图表复刻-极简信息图复刻指南.md); [可视化原则](../../raw/toolbox/可视化-原则-原则.md)

## 概述

极简信息图设计规范定义了一种独特的 SVG 信息图风格——暖调极简主义。它的视觉参考是学术书籍中的印刷插图，不是数据可视化，不是 SaaS dashboard。核心特征是米色纸张质感背景、暖灰色层次容器、Georgia 衬线标题、单一暖棕强调色。这套规范与 Mino 体系的 Flat widget 风格互补：widget 风格用于数据仪表盘和对话内嵌展示，极简信息图风格用于概念解释、流程说明和对比展示。掌握这套规范，可以生成风格统一、信息层次清晰、学术感强的 SVG 信息图。

## 风格一句话定义

暖调极简主义信息图：米色纸张质感背景，暖灰色层次容器，Georgia 衬线标题，单一暖棕强调色。像学术书籍中的印刷插图，不是数据可视化不是 SaaS dashboard。

这个定义包含四个关键要素。材质感——不是纯白背景，而是米色纸张质感，传递温暖、学术、可信赖的感觉。层次系统——通过暖灰色容器建立信息层级，不是通过字号大小或颜色变化。字体配对——标题用 Georgia 衬线体（书卷气），正文用 system-ui 无衬线体（现代清晰）。色彩克制——单一暖棕（#c98a6a）作为唯一彩色，其他全是结构色（黑、灰、米）。

## 色彩系统

### 结构色（所有图通用，不可更改）

| 语义角色 | 色值 | 用途 | 为什么是这个值 |
|---------|------|------|--------------|
| **画布背景** | #faf9f7 | SVG 最外层 fill，覆盖全部区域 | 接近纸张的自然米色，不是刺眼的纯白 |
| **容器描边** | #1a1a1a | 大概念边界框 stroke（1.5px） | 接近纯黑但柔和，建立强边界 |
| **卡片填充** | #f5f4f1 | 卡片 rect fill | 暖灰米色，比背景深一级，建立层次感 |
| **卡片描边** | #d4d1c7 | 卡片 rect stroke（1px） | 中等暖灰，区分卡片但不抢眼 |
| **次要文字** | #6b6b6b | 标注、副标题、说明文字 | 中灰，比正文弱但不难读 |
| **主要文字** | #1a1a1a | 标题、卡片内正文 | 与容器描边一致，建立文字层级统一感 |
| **强调色** | #c98a6a | 唯一彩色——连接线、刻度线、标注点 | 暖棕，与整体暖调一致 |
| **虚线容器** | #e8e4dc | "free / 被丢弃"区域描边 | 淡于卡片描边，暗示可选性 |

### 语义色（按需使用，不超过 2 种/图）

| 色值 | 语义 | 示例 |
|------|------|------|
| #8f9b7f | 成功 / 已完成 / 正向 | 进度条、正向变化 |
| #6a8fb5 | 用户输入 / 信息 / 冷调对比 | 输入区域、标题栏 |
| #d4845a | 警告 / 注意 / 重要提示 | 高亮块、风险提示 |
| #a3867a | 辅助强调 / 灰色对比 | 淡色背景块 |

### 色彩铁律

第一，一张图最多 2 种语义色，不包括结构色。超过 2 种语义色会破坏极简主义的克制感。

第二，优先用灰度层次区分内容，而不是彩色。灰色层次已经能建立清晰的信息层级，彩色只在需要额外语义区分时使用。

第三，彩色只在"需要语义区分"时使用，比如正确 vs 错误方案。如果一张图只展示一个方案，不需要彩色。

第四，永远不用纯红（#ff0000）、纯蓝（#0000ff）、荧光色、高饱和色。这些颜色与学术书籍的印刷风格完全不符。

## 字体系统

### 字体配对

标题使用 Georgia, serif（衬线字体）。Georgia 的设计初衷是在屏幕上清晰可读，同时保留衬线体的书卷气质。在信息图中，Georgia 标题传递"这是一份严肃文档"的信号。

正文使用 system-ui, -apple-system, sans-serif（无衬线字体）。无衬线字体在中文环境下清晰易读，适合说明文字和标注。

### 字号层级

| 层级 | 字号 | 字重 | 字体 | 用途 |
|------|------|------|------|------|
| 主标题 | 18px | bold | Georgia | 图标题 |
| 卡片标题 | 11-12px | 500 | system-ui | 小卡片标题 |
| 正文 | 10-10.5px | 400 | system-ui | 卡片内说明文字 |
| 标注 | 8.5-9.5px | 400 | system-ui | 灰色小字、刻度标签 |
| 斜体标注 | 9.5-10px | italic | Georgia | 底部补充说明 |

### 文案规则

标题不超过 5 个词，使用名词短语。说明不超过 2 行，主谓宾完整句。标注不超过 1 行，灰色小字，可以省略主语。这些规则确保信息密度与视觉空间平衡，不会出现文字溢出或过度留白的情况。

## 容器系统

容器不仅是视觉装饰，更是信息组织工具。容器即信息。

### 三种容器

| 类型 | 圆角 | 描边 | 填充 | 语义 |
|------|------|------|------|------|
| 大容器 | rx=10-12 | 1.5px #1a1a1a | none | 概念边界，包一组内容 |
| 卡片 | rx=8 | 1px #d4d1c7 | #f5f4f1 | 独立信息单元 |
| 标签 | rx=5-6 | 0.5px #d4d1c7 | 跟随父容器 | 元数据、状态标记 |

### 容器的语义含义

大容器包住的内容等于一个概念组。容器嵌套表示包含关系。虚线容器表示可选的、被丢弃的、不存在的内容。容器大小差异表示信息量差异——大容器表示完整内容，小容器表示压缩内容。

理解容器的语义含义，可以在不增加文字说明的情况下，通过容器本身传递信息。这是极简信息图的核心能力。

## 线条系统

| 线条类型 | 语法 | 语义 |
|---------|------|------|
| 实线箭头 | `stroke-width="1" marker-end` | 确定的传递 / 流动方向 |
| 虚线箭头 | `stroke-dasharray="4 3"` | 间接关系 / 可选传递 |
| 粗竖线 | `stroke-width="3" stroke-linecap="round"` | 状态标记（绿=好，橙=注意） |
| 回环曲线 | `path d="M x1 y1 Q cx cy x2 y2"` | 回退 / 循环 / 重置 |
| 刻度线 | 短竖线 6px 高 | 度量标记点 |

线条必须有语义，不是纯装饰。每一条线都传递信息：箭头传递方向，虚线传递不确定性，粗竖线传递状态。这是极简主义的核心原则——每个视觉元素都有功能。

## 间距系统

| 间距类型 | 数值 | 说明 |
|---------|------|------|
| 卡片间距 | 6-8px | 同组卡片之间的水平/垂直间距 |
| 容器内边距 | 10-12px | 内容不贴容器边缘 |
| 组间距 | 20-30px | 不同概念组之间的留白 |
| 文字行距 | 13-14px | 多行文字的 line-height 效果 |
| 上下留白 | 顶部 25-30px，底部 15-20px | 标题距顶部、最后元素距底部 |

间距的均匀性是极简信息图品质的关键。不规则间距会让读者潜意识感到"不舒服"，但说不出原因。所有同层级的间距必须一致。

## 视觉隐喻模板库

当拿到一段文字内容时，先判断它属于哪种逻辑关系，然后选择对应的视觉隐喻模板。这是内容到视觉转换的核心流程。

### 模板 1：时间线 / 容量轴

触发条件：展示"组成部分按顺序排列"或"容量从左到右"。视觉结构：大容器框住所有卡片，底部水平轴加刻度标记。案例：Context Window 组成部分的展示。

### 模板 2：渐变衰减

触发条件：展示"从清晰到模糊"或"质量递减"。视觉结构：顶部渐变色条（深到浅）加下方解释卡片。案例：Context Gradient 展示信息质量递减。

### 模板 3：光谱排列

触发条件：展示"策略从简单到复杂"或"选项排列"。视觉结构：水平轴加箭头指向多个等宽卡片，右侧竖线标记。案例：五策略对比。

### 模板 4：左右对比（包含 vs 筛选）

触发条件：展示"全部内容 vs 实际看到"。视觉结构：两个等大容器并排，中间箭头标注。案例：Session vs Seen 对比。

### 模板 5：方案对比（A vs B）

触发条件：展示"两种方法的效果差异"。视觉结构：左右两栏卡片，各自的颜色标识，底部建议。案例：Correcting vs Rewinding 两种方法的对比。

### 模板 6：父子容器

触发条件：展示"什么传过去，什么留下来"。视觉结构：大容器加内部小容器，曲线箭头标注 spawn/return。案例：Parent/Subagent 关系展示。

### 模板 7：信息过滤

触发条件：展示"完整记录 vs 模型实际接收"。视觉结构：左侧密集列表，右侧精简卡片，中间过滤箭头。案例：Session History Filtering 展示信息压缩过程。

## SVG 生成标准流程

拿到一段文字内容后，按以下步骤生成极简信息图。

第一步：识别核心关系。判断是时间线、对比、流程、层级还是程度，选择对应的视觉隐喻模板。

第二步：切割文本块。标题控制在 5 词以内，说明控制在 2 行以内，标注为灰色小字。如果原文内容超出限制，需要提炼和压缩。

第三步：选择配色。结构色固定不变，如需语义区分选择最多 2 种语义色，确定哪些卡片需要特殊颜色。

第四步：布局计算。确定 SVG viewBox 高度（宽度固定 680px），计算每个卡片的 x/y/width/height 坐标，确保间距统一（6-8px），确保文字垂直居中在卡片内。

第五步：生成 SVG。按照顺序：背景到标题到容器到卡片到文字到线条到标注。运行自检清单。

第六步：验证。与原图或需求对比，检查对齐、间距、颜色一致性。

## 生成自检清单

生成完一张图后，逐项检查以下内容。背景色是 #faf9f7，不是纯白 #ffffff。卡片填充是 #f5f4f1，不是纯白。卡片描边是 #d4d1c7，不是 #cccccc。标题用 Georgia 衬线字体，正文用 system-ui 无衬线字体。圆角：容器 rx=10-12，卡片 rx=8。卡片间距 6-8px 均匀。文字垂直居中在卡片内。颜色不超过 2 种语义色加结构色。每条线都有语义，不是纯装饰。信息分层：标题到卡片到标注。留白足够，不拥挤不空洞。

## 极简信息图 vs Widget 风格的对比

极简信息图风格和 Mino widget 风格服务于不同的使用场景。

| 维度 | 极简信息图风格 | Widget Flat 风格 |
|------|--------------|-----------------|
| 背景 | 米色 #faf9f7 | 暖色渐变或纯白 |
| 标题字体 | Georgia 衬线体 | system-ui 无衬线体 |
| 强调色 | 暖棕 #c98a6a | Warm 色阶系统 |
| 容器 | 1.5px 黑色描边 | 淡边框或无边框 |
| 参考对象 | 学术书籍插图 | Bloomberg 终端 / 咨询报告 |
| 使用场景 | 概念解释、流程说明、对比展示 | 数据仪表盘、指标卡、趋势图 |
| 信息密度 | 中等，偏解释性 | 高，偏数据性 |
| 运行环境 | 独立 SVG 或 HTML | 对话内嵌 widget |

两种风格不冲突，而是互补。需要展示数据时用 widget 风格，需要解释概念时用极简信息图风格。
