# 管理场景与应对策略

## 概述

管理场景体系将年老师的日常工作按触发信号分类，每个场景对应一套结构化应对策略。核心是 22 个 Plan 模板（A/B/C/D/E 系列），覆盖从被动响应到主动评估的全场景。

## 场景分类矩阵

| 类别 | 触发信号 | 响应时间 | 代表场景 |
|------|---------|---------|---------|
| **A. 判断型** | 看到异常信号 | 10-15分钟 | 业绩异常、合规定级、清退评估 |
| **B. 规划型** | 拿到明确任务 | 15-30分钟 | 新供应商准入、出差巡检、月度复盘 |
| **C. 拆解型** | 遇到模糊问题 | 5-20分钟 | 问题拆解、数据整理、制度梳理 |
| **D. 响应型** | 被动被@/被点名 | 30秒-5分钟 | 同事问数据、领导点名、供应商来电 |
| **E. 评估型** | 新供应商接触 | 10-15分钟 | 初筛判断、准入评审 |

## A 类场景：异常判断

### 核心方法：6 步异常判断法

1. **确认信号真实性** — 排除数据质量/统计口径导致的"假异常"
2. **判断异常级别** — 按排名触发规则分黄（首次预警）/橙（连续预警）/红（清退评估/A级骤降）
3. **定位原因维度** — 产能/人效/质量/业务/人事五维度交叉分析
4. **产能利用率定性** — 双维度矩阵（自身峰值比 vs 头部差距）分为四类：标杆型/尽力型/问题型/暂时波动型
5. **检查周度突变信号** — 出勤率<85%/GMV降幅>20%/单周离职>5人，决定干预紧迫度
6. **决定下一步动作** — 对照决策授权矩阵：黄色自主执行，橙色事前报备，红色建议权

### 决策输出格式

```
供应商：[名称]
异常级别：[黄/橙/红]
原因定位：[维度+具体表现]
供应商类型：[标杆型/尽力型/问题型/暂时波动]
突变信号：[有/无，具体]
下一步动作：[动作+时间要求]
授权情况：[自主执行/事前报备/建议权]
```

## B 类场景：任务规划

关键场景含新供应商准入（8阶段T+15）、新项目对接（5W2H需求确认）、出差巡检准备（12项标准+FCI速记卡）、月度复盘（格局分析→趋势识别→下月计划）、向上汇报准备（分层+四大维度+问答预判）、降价谈判准备（成本分析+痛点+策略+底线）。

## C 类场景：问题拆解

遇到模糊问题时的通用拆解流程：MECE 拆解问题 → 识别信息缺口 → 拆解到叶子节点 → 每个叶子节点对应一个具体行动。制度流程梳理用 RACI 权责矩阵，确保每个环节有唯一责任人。

## D 类场景：被动响应

### 30 秒快速判断框架

**第一秒 — 识别来者身份和意图：**

| 来者 | 通常要什么 | 敏感度 |
|------|-----------|--------|
| 刘伟佳（策略组） | 排名/分量/产能数据 | 中 — 未确认排名不能外泄 |
| 刘乾坤（巡检） | 巡检发现/整改清单 | 低 — 按流程该公开 |
| 郭鑫（项目） | 项目里程碑/上线排期 | 中 — 涉及供应商承诺 |

**第二秒 — 判断紧急度：**

群里@+连续追问 → 5分钟内必须回复；群里@一次 → 30分钟内；私聊 → 当天内；邮件 → 24小时内。

**第三秒 — 能不能直接回答：**

数据在手且无敏感性 → 直接给；涉及未公开排名 → 确认后同步；不在手 → 告知预计时间；不确定能不能给 → 先问王易人。

### 5 分钟深入版

30 秒回复后对方追问时，按 5 步深入：

1. **确认需求边界** — 问的是哪个维度？问题模糊时先追问，别猜着给
2. **调取和核实数据** — 数据是否滞后？有无异常值需标注？口径是否一致？
3. **组织回复口径** — 给结论+给依据，预判追问方向，主动解释异常
4. **识别隐含需求** — 对方问这个问题背后在做什么决策？
5. **确认后续动作** — 是否需要正式文档？是否需要同步他人？

### 常见误区

| 误区 | 后果 | 正确做法 |
|------|------|---------|
| 问什么答什么，不追问 | 给了错误口径 | 先确认需求边界 |
| 把其他供应商数据一起给了 | 泄露不该泄露的信息 | 只给被问的那家 |
| 群里直接回复敏感数据 | 全群可见，信息不可控 | 敏感数据私聊发 |
| 给完数据不管了 | 对方拿数据做了错误判断 | 主动标注口径和注意事项 |

## E 类场景：新供应商评估

E1 初筛：基础门槛（注册资金、成立年限、无不良记录）→ 快速对标（同类型供应商横向比较）→ 类别判断（头部/中部/尾部）。

E2 准入评审：里程碑达标检查 → 三要素综合评估（业绩+合规+稳定）→ 转正/延长/清退。

## 日常管理：赛马机制

日常评估得分制度解决"小问题不断上报"的痛点：

- **总分 10 分**，按月累计扣分
- **数据回复延迟**：晚≤30分钟扣 0.5 分，30-60分钟扣 1 分，>60分钟扣 2 分
- **周末无人值守**：-1 分/次
- **运作**：记录在案，月底统一算账，得分影响排名和分量
- **核心价值**：从"消防员"变成"裁判"，用制度管不用人盯

## 分量调整场景

四大场景：业务收缩型（保优汰劣）、绩效惩罚型（优胜劣汰）、绩效激励型（激励优秀）、风险管控型（风险优先）。

混合维度减量是推荐方案：职场分层（按绩效排名分头/腰/尾）→ 供应商兜底（防集中度突破）→ 坐席执行（按坐席绩效从后往前减）。

集中度管控强制检查：减量不得低于下限，增量不得突破上限。
