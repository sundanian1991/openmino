# Plan 模板体系

> Sources: Mino, 2026-04-19
> Raw:[toolbox](../../raw/toolbox/plan-templates-_README.md)

## 概述

Plan 模板体系是年老师日常决策的思考检查清单。遇到场景时读对应模板，输出判断建议，再衔接 official-doc 生成公文。v2.0 体系包含 22 个模板，分为 A/B/C/D/E 五大类，覆盖从被动响应到主动评估的全场景。

## 体系架构

| 类别 | 编号 | 数量 | 定位 | 用时 | 核心问题 |
|------|------|------|------|------|---------|
| **A. 判断型** | A1-A6 | 6 | 主动发现问题后的系统判断 | 10-15分钟 | 看到信号 → 怎么判断 → 怎么行动 |
| **B. 规划型** | B1-B6 | 6 | 有明确目标后的任务拆解 | 15-30分钟 | 拿到任务 → 怎么拆解 → 怎么执行 |
| **C. 拆解型** | C1-C4 | 4 | 模糊问题的结构化梳理 | 5-20分钟 | 遇到模糊问题 → 怎么理清 |
| **D. 响应型** | D1-D4 | 4 | 被动被@/被点名的快速响应 | 30秒-5分钟 | 被找 → 怎么快速接住 → 怎么回复 |
| **E. 评估型** | E1-E2 | 2 | 新供应商初次接触评估 | 10-15分钟 | 新供应商 → 怎么判断行不行 |

## A 类：判断型模板

主动发现异常信号后，系统化定位原因并决定干预级别。

| 编号 | 模板名 | 触发信号 | 核心方法 |
|------|--------|---------|---------|
| A1 | 业绩异常判断 | 排名下降/周度断崖/峰值比低 | 6步法：确认信号 → 判断级别 → 定位原因 → 产能定性 → 突变信号 → 决定动作 |
| A2 | 约谈准备 | 首次预警/连续预警/A级骤降 | 数据呈现 → 引导提问 → 要求产出 → 记录存档 |
| A3 | 分量调整决策 | 月度分量调整/临时增量申请 | 5维打分 + 资源匹配度验证 + 集中度检查 |
| A4 | 转正延期决策 | 新供应商第6个月末试运行到期 | 三要素评估（业绩+合规+稳定） |
| A5 | 合规问题定级 | 客诉/违规/红线行为 | 三级分类（红线/严重/一般）+ 处理矩阵 |
| A6 | 清退评估 | 连续3个月后30%/诚信红线 | 触发确认 → 业务影响分析 → 决策建议 |

A1 的 6 步法是核心判断框架：先确认数据真实性（排除统计口径问题），再按排名触发规则判定异常级别（黄/橙/红），通过双维度矩阵（自身峰值比 vs 头部差距）将供应商分为标杆型/尽力型/问题型/暂时波动型，最后对照决策授权矩阵确定行动权限。

## B 类：规划型模板

拿到明确任务后的结构化拆解和执行规划。

| 编号 | 模板名 | 核心方法 |
|------|--------|---------|
| B1 | 新供应商准入 | 8阶段SOP（T+15）+ 9维度深度评估 |
| B2 | 新项目对接 | 5W2H需求确认 → 现有供应商扫描 → 决策路径 |
| B3 | 出差巡检准备 | 12项巡检标准 + FCI速记卡 + 重点预判 |
| B4 | 月度复盘 | 格局分析 → 重点标记 → 趋势识别 → 下月计划 |
| B5 | 向上汇报准备 | 分层（军哥/易人哥）+ 四大维度 + 问答预判 |
| B6 | 降价谈判准备 | 成本分析 + 供应商痛点 + 谈判策略 + 底线设定 |

## C 类：拆解型模板

遇到模糊问题时的结构化梳理工具。

| 编号 | 模板名 | 核心方法 |
|------|--------|---------|
| C1 | 问题拆解 | MECE拆解 → 信息缺口分析 → 叶子节点行动 |
| C2 | 邮件起草 | 类型选择 → 语气把握 → 模板套用 → 抄送规则 |
| C3 | 数据整理分析 | 数据体检 → 分析目标 → 洞察提取 → 报告输出 |
| C4 | 制度流程梳理 | RACI权责 → 阶段拆解 → 决策点 → 闭环验证 |

## D 类：响应型模板

被动被找时的快速响应框架，全部含 30 秒快速判断 + 5 分钟深入版两档。

| 编号 | 模板名 | 触发场景 | 核心特点 |
|------|--------|---------|---------|
| D1 | 被同事@数据进度 | 刘伟佳/刘乾坤/郭鑫要数据或进度 | 30秒识别来者意图 + 判断紧急度 + 一句话口径 |
| D2 | 被领导点名 | 军哥/王易人问"XX供应商怎么回事" | 分层回应（战略层/执行层）+ escalation 判断 |
| D3 | 供应商来电来消息 | 要量/要支持/反映问题/投诉 | 真问题 vs 撒娇判断矩阵 + 回应策略 |
| D4 | 供应商群异常信号 | 24h未回复/态度变化/推诿 | 异常类型识别 + 决策树判断 + 介入策略 |

## E 类：评估型模板

新供应商接触的评估框架。

| 编号 | 模板名 | 核心方法 |
|------|--------|---------|
| E1 | 新供应商初筛 | 基础门槛 + 快速对标 + 类别判断（头/中/尾） |
| E2 | 准入评审决策 | 里程碑达标检查 → 三要素综合评估 → 转正/延长/清退 |

## 调用决策树

```
遇到问题 → 判断场景类型
  ├─ 同事@/领导问/供应商来电 → D类模板（30秒-5分钟快速响应）
  ├─ 看到异常信号/需要决策 → A类模板（10-15分钟系统分析）
  ├─ 拿到任务/新项目/模糊问题 → B类/C类模板（拆解规划）
  └─ 新供应商接触 → E类模板（初筛/转正评估）
```

## 设计原则

- 每个模板不超过 200 行，可打印一页纸
- D 类全部含"30 秒快速判断"模块（两档响应）
- 思考步骤引用工具箱具体章节（§X 格式，可追溯）
- 衔接 official-doc 具体文件名，无缝流转
- 使用年老师的实际场景（毅航/毛毛虫/赛维斯/岐力/翰锐/华啸）

## 与旧体系的关系

旧体系（Q/N/D 系列 18 个模板）已归档。新体系相比旧版的核心改进：

- 从 18 个扩展到 22 个，覆盖场景更全
- 引入 A/B/C/D/E 分类法，替代原有的 Q/N/D 分类，逻辑更清晰
- 每个模板都含"下一步衔接"字段，与 official-doc 模板无缝衔接
- D 类新增 30 秒快速判断，解决"被@ 后秒回"的场景需求
