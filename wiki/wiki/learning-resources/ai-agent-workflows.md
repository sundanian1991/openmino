## 概述

本文档总结 Claude Code 的核心组件体系、AI 协作模式和 Agent 工作流模式，基于对 Claude Code 七大组件的深入学习以及 178 个 agency-agent 角色的系统性分析。

## Claude Code 七大组件

| 组件 | 作用 | 配置位置 |
|------|------|----------|
| **CLAUDE.md** | 项目记忆体，定义 AI 如何协作 | 项目根目录 |
| **Commands** | 快捷指令（`/xxx`） | `.claude/commands/` |
| **Hooks** | 自动化事件触发器 | `.claude/hooks.json` |
| **SubAgents** | 任务分解与并行执行 | Task 工具 |
| **Skills** | 工作流模板，按需加载 | `.claude/skills/` |
| **Plugins** | 功能扩展包 | 插件市场 |
| **MCP** | 连接外部工具和服务 | `~/.mcp.json` |

核心原则：从 CLAUDE.md 开始，逐步扩展其他组件，避免一次性启用所有组件导致上下文膨胀。

## AI 协作模式

### 文档写作

- 使用结构化框架（SCQA、STAR）
- 用数据支撑观点，具体案例说明
- 风格直白、接地气

### 问题分析三视角

1. **生态视角**：不是单个供应商，而是整个生态系统
2. **合规视角**：有规可依、合理、充分共识
3. **业务价值视角**：对业务 KPI 的贡献

### 建议原则

- 短期效率与长期风险平衡
- 考虑生态系统健康度（多样性、抗风险、进化能力）
- 考虑可复制性（个人经验转组织能力）

## Agent 工作流模式

### 角色分层（基于 178 个 agent 角色分析）

**高频核心角色**（保留）：

| 类别 | 核心角色 |
|------|----------|
| 工程 | 前端开发、后端架构、安全工程、代码审查、数据库优化、DevOps |
| 设计 | UI 设计、UX 研究、UX 架构、品牌策略 |
| 管理 | 项目协调、反馈分析、执行摘要 |
| 特殊 | MCP Server 构建、文档生成 |

**删除类别**：游戏开发（15+）、空间计算（8）、区块链（2）、付费媒体（7）、销售（8）—— 与日常工作无关的领域。

### 工作流最佳实践

- 避免一次性启用所有组件，从核心开始逐步扩展
- Hooks 中涉及数据修改的操作需要手动确认
- MCP/Plugins 优先使用官方和可信来源
- CLAUDE.md 保持简洁，避免过于冗长

## 技能加载优化

### 问题

启动时全量加载所有 SKILL.md 文件，导致 token 消耗过高（约 140-165K tokens）。

### 渐进披露方案

```
1. Metadata (name + description) — 始终加载 (~100 words)
2. SKILL.md body — 触发时才加载 (<500 行)
3. Bundled resources — 按需加载 (无限制)
```

预期效果：启动 token 降低约 70%（从 140K 降至 40K）。

### 技能库改进优先级

| 问题 | 影响数量 | 优先级 |
|------|----------|--------|
| 描述过短（触发不足） | ~15 个 | P0 |
| SKILL.md 超长（>500 行） | 6 个 | P1 |
| INDEX.md 过期 | 1 个 | P0 |
| 结构不完整 | ~20 个 | P1 |
| 缺少评估机制 | 全部 | P2 |
