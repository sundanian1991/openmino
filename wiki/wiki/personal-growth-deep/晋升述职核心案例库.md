## Overview

晋升述职的核心是案例。孙大年的V13版本最终确定了三个案例，每个案例250-260字，用SCQA结构讲述。这三个案例不是"我做了什么"的流水账，而是"我如何解决关键问题"的故事。本文章深度拆解三个案例的叙事结构、数据设计和钩子设置，形成可复用的案例库模板。

## 一、案例库全景

### 三个案例的定位

| 案例 | 核心主题 | SCQA结构 | 证明能力 |
|------|----------|----------|----------|
| 案例一 | 激活竞争 | 老商躺平→新商入场→赛马→倒逼优化 | 生态经营能力 |
| 案例二 | 合规风控 | 风险累积→制度建立→巡检覆盖→零纠纷 | 风险管控能力 |
| 案例三 | 线路应急 | 线路压降→主动补位→15天恢复→预案标准化 | 危机处理能力 |

### 案例的"一事一证"原则

每个案例只证明一个核心能力，不贪多。三个案例合在一起，证明一个完整的P7能力画像：

```
案例一（生态经营）→ 证明"能建体系"
案例二（合规风控）→ 证明"能守底线"
案例三（危机处理）→ 证明"能扛大事"
```

## 二、案例一：激活竞争——用"选汰机制"打破供应商舒适区

### SCQA结构拆解

**Situation（情境）**：
> 四团队合并后，我接手了金科全部33家供应商、3239人电销团队。

**Complication（冲突）**：
> 老供应商依赖严重，头部几家占据主要产能，缺乏竞争。部分供应商进入舒适区，不再主动优化。

**Question（问题）**：
> 如何在没有行政命令权的情况下，激活整个渠道的竞争活力？

**Answer（答案）**：
> 建立"选"和"汰"的完整机制——引入新商搅动竞争，清退劣商倒逼优化。

### 核心数据设计

| 数据点 | 数值 | 设计意图 |
|--------|------|----------|
| 引入新商 | 8家 | 证明"选"的突破 |
| 新商进TOP3 | 6家（75%） | 证明新商质量高 |
| 老商降价 | 3家主动降价 | 证明竞争倒逼效果 |
| 老商整改 | 2家主动整改 | 证明老商被激活 |
| 清退数量 | 4家零纠纷 | 证明"汰"的决断 |
| 引入周期 | 30天→15天 | 证明效率提升 |

### 案例叙事（V13精简版）

> "四团队合并后，老供应商依赖严重。头部几家占据主要产能，缺乏竞争——部分供应商进入舒适区。
>
> 我做了一件事：建立'选'和'汰'的机制。
>
> **选**：统一四个团队的准入标准，流程线上化，引入周期从30天缩短到15天。引入8家新供应商。
>
> **汰**：清退4家违规供应商，每家都有AB替代方案，产能零中断，全年零纠纷。
>
> 结果：新供应商75%进入季度TOP3，倒逼3家老商主动降价、2家主动整改。渠道活力被激活了。"

### 钩子设计

**结尾悬念**：
> "但这不只是引入新商的问题——如果没有一个'尺子'，你怎么知道新商好不好、该给多少量？"

**引导方向**：引出案例二的"自建标杆"主题。

### 为什么这个案例有效？

| 维度 | 分析 |
|------|------|
| 冲突明确 | "老商躺平"是共性问题，评委能理解 |
| 动作清晰 | "选"和"汰"两个动作，不复杂 |
| 数据有力 | "75%进TOP3"比"引入了8家"有冲击力 |
| 结果可量化 | 降价、整改都是可验证的成果 |
| 可复制 | 这套机制可以复制到其他场景 |

## 三、案例二：合规风控——用精细化管控守住底线

### SCQA结构拆解

**Situation（情境）**：
> 33家供应商、3000多人的渠道池，业务快速扩张。

**Complication（冲突）**：
> 合规风险累积——缺乏统一标准和常态化监控机制，一旦出事就是大事故。

**Question（问题）**：
> 如何在快速扩张的同时，守住合规底线？

**Answer（答案）**：
> 从事后救火转向过程监控——建立制度、执行巡检、递进式处置。

### 核心数据设计

| 数据点 | 数值 | 设计意图 |
|--------|------|----------|
| 制度数量 | 5项 | 证明体系建设成果 |
| 巡检次数 | Q4巡检7次，覆盖15家 | 证明执行力度 |
| 发现问题 | 每家6-7个 | 证明巡检有效性 |
| 整改闭环率 | 100% | 证明执行力 |
| 红灯预警 | 4次全部化解 | 证明预警有效 |
| 清退纠纷 | 零 | 证明处理得当 |

### 案例叙事

> "业务扩张带来合规风险累积。33家供应商、3000多人，如果没有统一标准，一出事就是大事故。
>
> 我从'事后救火'转向'过程监控'。
>
> **立制度**：制定了5项制度——职场现场规范、违规处罚条例、巡检标准、分量标准、现场处罚标准。全部上了集团制度中心。
>
> **抓过程**：Q4巡检7次，覆盖15家供应商，每家发现6-7个问题。红灯预警4次，全部化解。整改闭环率100%。
>
> 结果：全年零纠纷。清退4家违规供应商，零产能损失。"

### 钩子设计

**结尾悬念**：
> "制度建好了，巡检做完了。但更大的考验来了——当行业突发风险时，这些制度能不能扛住？"

**引导方向**：引出案例三的"线路应急"主题。

## 四、案例三：线路应急——危机中的主动补位

### SCQA结构拆解

**Situation（情境）**：
> 运营商突然压降客诉量，线路并发降到日常的30%。

**Complication（冲突）**：
> 日影响1300万呼叫、37亿周GMV。团队无专职应急负责人。

**Question（问题）**：
> 如何在没有专人负责的情况下，快速恢复线路、保障业务不中断？

**Answer（答案）**：
> 主动补位——发动资源、快速识别、高频调度、沉淀标准。

### 核心数据设计

| 数据点 | 数值 | 设计意图 |
|--------|------|----------|
| 恢复时间 | 15天 | 行业对比（行业30天） |
| 业务中断 | 零中断 | 证明应急能力 |
| GMV损失避免 | 约80亿 | 证明价值贡献 |
| 找到供应商 | 近10家带指标 | 证明资源调动能力 |
| 储备超配 | 150% | 证明风险管理意识 |
| 高频调度 | 25-30天每天复盘 | 证明持续执行力 |

### 案例叙事

> "年中运营商突然压降客诉量，线路并发降到日常的30%。日影响1300万呼叫、37亿周GMV。
>
> 团队没有专职应急负责人。我主动补位，做了六件事：
>
> 发动资源，15天内找到近10家带指标的供应商。快速识别，建立资源价值判断标准。聚焦锁定核心资源，储备超配150%。高频调度，25-30天每天复盘客诉数据。
>
> 结果：15天恢复（行业30天），零业务中断。事后推动预案成为部门标准流程。"

### 钩子设计

**结尾悬念**：
> "这三件事做完，体系运转了。但我发现一个矛盾——持续汰换，会不会让供应商池越来越小？"

**引导方向**：引出对生态终局的思考。

## 五、案例写作的五条铁律

### 铁律一：每个案例只证明一个能力

| 错误 | 正确 |
|------|------|
| 一个案例讲"选、用、育、留、汰" | 一个案例只讲"选和汰" |
| 面面俱到 | 聚焦一个核心故事线 |

### 铁律二：数据必须支撑观点，不是罗列

| 错误 | 正确 |
|------|------|
| "我引入了8家新供应商" | "引入8家新供应商，75%进TOP3" |
| "巡检了15家" | "巡检15家，每家发现6-7个问题" |

### 铁律三：SCQA必须完整

**缺失任何一个环节，案例就没有说服力**：

| 缺失环节 | 后果 |
|----------|------|
| 没有S | 不知道在说什么场景 |
| 没有C | 没有紧迫感，评委不关心 |
| 没有Q | 没有核心问题，故事散掉 |
| 没有A | 没有解决方案，空洞 |

### 铁律四：字数控制在250-260字

| 阶段 | 字数 | 原因 |
|------|------|------|
| V1 | ~4000 | 太长，评委记不住 |
| V5 | ~2000 | 还是偏长 |
| V13 | 250-260/案例 | 精炼到"每句话都有信息量" |

**为什么250字？** 这是口头表达约1分钟的内容量。8分钟述职，三个案例各1分钟，加上开场和总结，刚好8分钟。

### 铁律五：每个案例有钩子

钩子不是"设悬念"，而是**引导评委问你准备好的问题**。

| 钩子类型 | 效果 |
|----------|------|
| 问题式 | 引导评委追问你准备好的深度分析 |
| 价值式 | 引导评委关注你的核心贡献 |
| 矛盾式 | 引导评委思考你的战略视野 |

## 六、案例库的扩展使用

### 场景一：晋升述职

三个案例按顺序排列，用SCQA结构，250字/案例。

### 场景二：年终总结

提取核心数据，用STAR结构（情境-任务-行动-结果）。

### 场景三：面试/跳槽

用PREP结构（观点-理由-例证-观点），每个案例压缩到100字。

### 场景四：日常汇报

用SCQA的简化版：问题-方案-结果。

## 七、案例素材收集方法论

### 平时如何积累案例素材？

| 维度 | 方法 |
|------|------|
| 数据 | 每月更新核心指标（GMV、产能、合规率） |
| 事件 | 记录重大事件（线路应急、清退、降价谈判） |
| 对比 | 保存"之前 vs 之后"的对比数据 |
| 反馈 | 收集上级/供应商/同事的反馈和评价 |
| 教训 | 记录失败案例和反思 |

### 案例素材模板

```
事件名称：
时间：
背景（Situation）：
冲突（Complication）：
核心问题（Question）：
我的动作（Answer）：
关键数据：
结果：
反思：
可复用经验：
```

## 总结

晋升述职核心案例库的关键不是"有多少案例"，而是**"每个案例能证明什么能力"**。三个案例、三个能力（生态经营、风险管控、危机处理），合在一起构成一个完整的P7能力画像。SCQA结构保证叙事完整，250字控制保证精炼，钩子设计保证互动可控。案例写作不是"讲故事"，而是"用事实证明能力"。
