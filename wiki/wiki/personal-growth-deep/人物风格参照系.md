## Overview

在供应商管理的日常中，孙大年需要与不同角色高频沟通——对上级（军哥）要穿透式追问，对下级（业务管理组）要直接追问，对供应商要专业推动，对同事（刘伟佳、秦宇凝）要学习借鉴。通过观察和分析周围人的沟通风格，他建立了一套人物风格参照系：**不是模仿某个人，而是理解每个人的决策逻辑和信息需求，选择最合适的沟通策略。**

## 一、表达层级速判

在分析任何人（包括自己）的沟通质量时，用四层表达层级快速判断：

| 层级 | 特征 | 示例 | 判断标准 |
|------|------|------|----------|
| L1 | 只有感受 | "我觉得不太好" | 能验证吗？不能 |
| L2 | 有定性无定量 | "问题出在人员流失" | 有数据吗？没有 |
| L3 | 有定量无定性 | "达标率降了15%" | 为什么？说不清 |
| L4 | 定性+定量融合 | "达标率降15%，因人员流失，下周补充培训" | 有数据、有原因、有下一步 |

**核心标准**：好的表达必须是L4——定性+定量融合。

## 二、人物风格全景

### 军哥（卞海军）——穿透追问型

**身份**：上级，服务组直属领导

**特点**：
- 会穿透式追问，要数据+根因+时间线
- 不会停留在表面数据
- 追问到根因才停止

**原话参考**：

| 场景 | 原话 | 分析 |
|------|------|------|
| 穿透追问 | "为什么老人流失严重，还上那么多人？" | 追问根因，不接受表面回答 |
| 时间线施压 | "最大接受4-6周，追不上就砍掉。" | 给出明确时间窗口 |
| 问题定性 | "这属于严重事故，你们知道吗？" | 直接定性，不绕弯 |
| 行动指令 | "约老板来，把人排一遍，刷掉不行的。" | 具体到行动 |

**汇报策略**：

| 要点 | 做法 |
|------|------|
| 预判追问 | 在汇报前先准备好根因、时间线、方案 |
| 数据先行 | 先给数字，再给分析 |
| 带方案汇报 | 不是"您说怎么办"，是"我建议这样做" |

**可学之处**：穿透追问的思维框架——每次都问"为什么"，追到根因。

### 刘乾坤——高压管理型

**身份**：同事，供应商管理

**特点**：
- 立规矩非常明确
- 追责转嫁，把压力传导给供应商
- 加压链式管理

**原话参考**：

| 场景 | 原话 | 分析 |
|------|------|------|
| 立规矩 | "群内消息30分钟内回复，未能及时回复请提前报备原因" | 规则非常具体，有明确时限和后果 |
| 追责转嫁 | "大年兄，博岳截止10点43都没有按照要求回复数据" | 把压力传导给对方 |
| 加压链 | "直接电话他吧，已经2小时了。" | 逐步加压 |
| 制度拷问 | "周末的排班计划是什么？和总部的沟通机制是什么？" | 用制度拷问对方 |
| 三要素法 | 原因分析 + 问题定位 + 改进举措 | 要求对方给出完整回复 |

**可学之处（学形不学神）**：

| 可学 | 不可学 |
|------|--------|
| 立规矩的明确性 | 过于高压的语气 |
| 三要素法的追问逻辑 | 当众点名批评 |
| 追责转嫁的策略 | 过度施压导致对立 |

### 刘伟佳——战略服务型

**身份**：同事，策略组

**特点**：
- 成果前置，先说结果再说过程
- 用标签标记重点
- 服务姿态，不是命令姿态

**原话参考**：

| 场景 | 原话 | 分析 |
|------|------|------|
| 成果前置 | "25年增量GMV 580亿、核卡11万，转化率+34%" | 先给数字 |
| 制度标签 | "黑名单&频控机制升级，预计4月1日实施" | 用标签标记重点 |
| 服务姿态 | "支撑转化效率提升，帮助业管精准定位问题" | 不是命令，是服务 |
| 节奏控制 | "下周组织专题会，详细交流具体落地方案" | 控制节奏 |

**可学之处**：成果前置+服务姿态——汇报时先说结果，协作时用服务姿态。

### 王易人——精细化管控型

**身份**：上级，服务组负责人

**特点**：
- 目标量化到具体数字
- 责任压实到具体人和场景
- 问题点名，不回避

**原话参考**：

| 场景 | 原话 | 分析 |
|------|------|------|
| 目标量化 | "Q1达成率从93%冲刺到95%，日均5.5亿，峰值5.97亿" | 具体到每个数字 |
| 责任压实 | "压实到12家职场、三条产能线，按达成率分档复盘" | 具体到每个场景 |
| 问题点名 | "毅航、汇讯、博岳出勤<90%，通时<2.5h" | 直接点名 |
| 向上争取 | "100万存量激励费用申请再分配" | 有数据支撑的资源争取 |

**可学之处**：精细化管控——把目标和责任压实到具体数字和具体场景。

### 秦宇凝——专业型汇报

**身份**：供应商代表

**特点**：
- 结构化表达
- 数据支撑每个观点
- 有边界意识

**原话参考**：

| 场景 | 原话 | 分析 |
|------|------|------|
| 结构化+数据 | "人员结构化面试：通过率30%，选择率<50%" | 每个观点都有数据 |
| 背景解释 | "首贷2月赛马第二，拿到名额多，新人占比40%，带不动" | 有上下文 |
| 有边界 | "加数据变成负担就不好了" | 知道自己能力的边界 |

**可学之处**：结构化+数据+边界——每个观点都有数据支撑，知道自己的边界。

### 谭敏——进度型汇报

**身份**：供应商代表

**特点**：
- 分类清晰
- 进度汇报有条理

**原话参考**：

| 场景 | 原话 | 分析 |
|------|------|------|
| 分类汇报 | "马上用起来：结构化面试、人员准入...还没完成转培：计划下周完成...月底启动：师徒制" | 分类清晰 |

**可学之处**：分类清晰——把事项按进度分类。

**需要改进**：补充数据支撑。

## 三、场景化沟通策略选择

### 向上级汇报

**策略**：预判追问型

| 要素 | 做法 |
|------|------|
| 开场 | 结论先行，直接说结果 |
| 正文 | 数据+根因+方案 |
| 结尾 | 明确需要支持的内容 |
| 预判 | 提前准备好评委可能追问的3-5个问题 |

**模板**：
> "[结论]。数据是[X]。原因是[Y]。我建议[Z]。需要您支持[W]。"

### 对下级追问

**策略**：直接追问型

| 要素 | 做法 |
|------|------|
| 开场 | 直接给数据 |
| 追问 | 给选项（引导思考方向） |
| 结尾 | 给期限 |

**模板**：
> "[供应商]达标率X%，掉了Y个百分点。具体是[A]还是[B]？[时间]给我反馈。"

### 对供应商沟通

**策略**：专业推动型

| 要素 | 做法 |
|------|------|
| 开场 | 给数据，指出差距方向 |
| 要求 | 让对方分析（不是直接给答案） |
| 结尾 | 给期限 |

**模板**：
> "你们达成率X%，低了Y个百分点。差距主要在[时段]。请分析原因，[时间]前给我整改计划。"

## 四、生活化类比的使用

**核心原则**：用生活化的类比让复杂的管理概念变得容易理解。

### 经典案例：餐厅-外包服务员类比

| 管理概念 | 生活化类比 |
|----------|-----------|
| 供应商 | 外包服务员 |
| 分量 | 排班 |
| 赛马 | 服务员排名 |
| 清退 | 不再合作 |
| 自建团队 | 店里自己的服务员 |
| 京东适应体系 | 新员工入职培训 |

**类比的价值**：不是"准确"，而是**"快速建立理解框架"**。让不熟悉供应商管理的人也能瞬间理解核心逻辑。

## 五、沟通风格的自我定位

### 孙大年的沟通风格

| 维度 | 风格 |
|------|------|
| 整体 | 逻辑清晰、结构化、接地气 |
| 对上级 | 预判追问、带方案汇报 |
| 对下级 | 直接、给期限 |
| 对供应商 | 专业、有边界 |
| 晋升述职 | SCQA三幕叙事、数据驱动 |

### 与各人物风格的差异

| 对比 | 差异 | 调整方向 |
|------|------|----------|
| vs 军哥 | 不够穿透式追问 | 学习"追问到根因" |
| vs 刘乾坤 | 过于温和 | 学习"立规矩的明确性" |
| vs 刘伟佳 | 不够成果前置 | 学习"先说结果" |
| vs 王易人 | 不够细化到具体数字 | 学习"压实到每个数字" |
| vs 秦宇凝 | 边界意识不够强 | 学习"有边界" |

## 六、沟通风格的进化路径

### 从"一视同仁"到"因人制宜"

| 阶段 | 特征 | 问题 |
|------|------|------|
| 阶段一 | 对所有人用同样的方式 | 效率低，效果差 |
| 阶段二 | 知道要区分对象，但不知道怎么区分 | 有时对有时错 |
| 阶段三 | 建立了人物参照系 | 能主动调整沟通策略 |

### 阶段三的具体做法

| 动作 | 做法 |
|------|------|
| 观察 | 记录身边人的沟通风格和原话 |
| 分析 | 拆解每个人的决策逻辑和信息需求 |
| 适配 | 对不同对象用不同的话，但底层逻辑一致 |
| 迭代 | 根据反馈调整沟通方式 |

## 总结

人物风格参照系的核心不是"模仿某个人"，而是**"理解每个人的决策逻辑和信息需求"**。对上级要"全"（预判追问），对下级要"直接"（给期限），对供应商要"专业"（有边界），对同事要"学习"（取其精华）。沟通风格的进化路径是从"一视同仁"到"因人制宜"，最终实现"对不同的人用不同的话，但底层逻辑是一致的"。
