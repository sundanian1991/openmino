## 概述

供应商生态操作系统是孙大年在管理33家外包供应商、3239人电销团队的实践中，逐步建立的一整套能够自我运转、自我优化、价值创造、可复制的管理系统。这个系统由五大机制和四大体系构成，核心目标是从"人治"转向"机制"。

## 一、系统全景架构

### 三层架构模型

> "我管理的不是33家供应商、3239人、17个产品、4大业务线。我建立的是一个能够自我运转的供应商管理生态系统。"

| 层级 | 内容 | 功能 |
|------|------|------|
| 第一层：价值层 | 多元竞争+自主可控 | 成本优化+效率提升+风险可控 |
| 第二层：机制层 | 准入+使用+培育+淘汰+协调 | 自动运转、按标准执行 |
| 第三层：执行层 | 制度+标准+流程+工具 | 落地执行、有据可依 |

### 系统的四个核心能力

| 能力 | 说明 | 证据 |
|------|------|------|
| 自我运转 | 没有我也能运转，因为有标准、流程、制度、工具 | 任何人按系统做，能达到80%效果 |
| 自我优化 | 每次发现问题→分析问题→解决问题→固化经验 | 巡检标准从不统一到统一 |
| 价值创造 | 管理好了，供应商就能产出更好 | 降本100万/月，人效+120% |
| 可复制 | 这是方法论，不是具体操作 | 可复制到任何供应商管理场景 |

## 二、五大机制详解

### 机制一：准入机制（选）

**解决的核心问题**：谁能进来？怎么进来？多快能进来？

**9维输入标准**：

| 标准 | 考察内容 | 权重倾向 |
|------|---------|---------|
| 过往业绩 | 以前做过什么？业绩怎么样？ | 历史验证 |
| 同类型业务对标 | 有没有做过类似业务？ | 适配度 |
| 管理方式 | 怎么管理团队？有没有SOP？ | 管理能力 |
| 人员数量 | 有多少人？能不能快速上人？ | 产能弹性 |
| 公司背景 | 是什么公司？有没有实力？ | 稳定性 |
| 历史交付风格 | 以前配合度怎么样？ | 合作倾向 |
| 配合的商务 | 商务对接人怎么样？ | 沟通效率 |
| 内部团队架构 | 团队架构是否合理？ | 组织健康度 |
| 资质和合规 | 有没有资质？有没有违规记录？ | 合规底线 |

**流程**：寻源储备 → 标准评估 → 现场验厂 → 审批上线

**效率指标**：审批1-2天（之前4-5天），引入15天（之前30天）

**机制的价值**：不是我一个人说了算，是标准说了算。

### 机制二：使用机制（用）

**解决的核心问题**：给谁？给多少？怎么给？什么时候给？

**业务分类体系**：

| 业务类型 | 特点 | 适合的供应商 | 关注指标 |
|---------|------|------------|---------|
| 拉新型 | 需要大量新客，转化率高 | 擅长拉新、有名单资源 | 拉新数量、新客质量 |
| 复购型 | 需要老客复购，服务质量高 | 擅长复购、服务意识好 | 复购率、客诉率 |
| 创新型 | 新模式、新流程、不确定多 | 学习能力强、配合度高 | 适应速度、反馈质量 |
| 大额型 | 高客单价、长周期 | 擅长大额、稳定型 | 单客价值、周期管理 |

**赛马机制**：每月排名，好的多给量，差的少给量并约谈警告。

**动态调整原则**：不都给，也不一直不给，让供应商保持期待。

### 机制三：培育机制（育）

**解决的核心问题**：供应商不会做怎么办？做不好怎么办？想优化怎么办？

**培训体系（京东适应体系）**：

| 培训内容 | 说明 |
|---------|------|
| 京东怎么看事情 | 决策逻辑、价值观、优先级 |
| 京东在乎什么 | 合规底线、数据质量、品牌形象 |
| 京东怎么工作 | 沟通方式、汇报节奏、协作流程 |
| 以前踩过什么坑 | 历史教训、常见错误、避坑指南 |
| 怎么快速出产能 | 最佳实践、标杆方法、加速器 |

**资源倾斜**：表现好的给好名单和支持，表现差的给差名单和压力。

**核心认知**：不是只看结果，是帮供应商达成结果。

### 机制四：淘汰机制（汰）

**解决的核心问题**：不合适的供应商怎么退出？怎么让供应商心甘情愿退出？怎么避免退出时的纠纷？

**递进式清退规则（4级）**：

| 级别 | 动作 | 时间窗口 | 升级条件 |
|------|------|---------|---------|
| 1级：警告 | 口头警告，要求整改 | 3-7天 | 未改善 |
| 2级：约谈 | 约谈商务，要求整改方案 | 3-7天 | 未改善 |
| 3级：限制新进 | 限制新业务，只能做存量 | 持续 | 未改善 |
| 4级：清退 | 让供应商自己申请退出 | 即时 | 最终手段 |

**两难选择策略**：

> "闹 → 被京东拉黑 → 行业里有影响。走 → 日后好相见 → 不加黑名单。"

最终效果：清退4家，全年零纠纷。

### 机制五：协调机制（跨部门协作）

**解决的核心问题**：业务方要上新项目怎么办？供应商说名单质量不好怎么办？供应商说涨价怎么办？

**需求协调流程**：问清需求 → 评估需求 → 匹配供应商 → 决策分配 → 跟进结果

**问题协调流程**：发现问题 → 分析问题（是个案还是普遍？表层还是深层？）→ 解决问题 → 跟进结果

**价值协调流程**：评估合理性（市场行情、供应商表现、业务数据）→ 做决策 → 谈条件

## 三、四大体系详解

### 体系一：制度体系

| 制度 | 解决的问题 | 核心内容 |
|------|-----------|---------|
| 《职场现场规范》 | 供应商现场怎么管？ | 空间布置、人员状态、品牌使用 |
| 《违规处罚条例》 | 违规了怎么办？ | 处罚等级、处罚标准、处罚流程 |
| 《巡检标准》 | 检查什么？ | 12项巡检标准 |
| 《分量标准》 | 给多少量？ | 打分维度、排名规则、分配逻辑 |
| 《现场要求及处罚标准》 | 现场问题怎么处理？ | 现场要求、处罚标准、整改流程 |

### 体系二：标准体系

**巡检标准（12项）**：

| 标准项 | 检查内容 | 关键指标 |
|--------|---------|---------|
| 空间布置 | 龙虎榜、公开业绩、话术管理 | 是否有激励展示 |
| 人员状态 | 坐席在打电话？玩手机？睡觉？ | 工作状态 |
| 薪资情况 | 给供应商100元，员工拿多少？ | ≥60元 |
| 招聘情况 | 招了什么人？从哪里招？ | 是否用京东名义 |
| 品牌使用 | 有没有滥用logo？有没有违规宣传？ | 合规率 |
| 管理情况 | 主管有没有在管？有没有培训？ | 管理覆盖率 |
| 活动激励 | 供应商有没有激励？坐席知不知道？ | 激励触达率 |
| 作息时间 | 每天工作多久？加班怎么算？ | 合规性 |
| 培训方式 | 有没有培训？培训什么？ | 培训覆盖率 |
| 招聘方式 | 怎么招的？渠道是什么？ | 渠道健康度 |
| 品牌滥用 | 有没有滥用京东名义？ | 违规次数 |
| 质检合规 | 有没有质检？质检合格率多少？ | 质检合格率 |

### 体系三：流程体系

| 流程 | 步骤 | 关键节点 |
|------|------|---------|
| 供应商引入 | 寻源→评估→验厂→上线 | 9维标准、现场验厂 |
| 供应商管理 | 分类→标签→打分→排名→调整 | 5维打分、月度排名 |
| 供应商清退 | 发现→警告→约谈→限制→清退 | 递进式、零纠纷 |
| 巡检 | 计划→检查→发现→邮件→整改→反馈 | 12项标准 |
| 问题协调 | 发现→分析→解决→跟进 | 个案vs普遍 |

### 体系四：工具体系

| 工具 | 用途 | 核心价值 |
|------|------|---------|
| 《京东适应指南》 | 新供应商培训 | 减少磨合时间 |
| 《巡检清单》 | 现场检查标准化 | 每次检查都有针对性 |
| 《问题反馈表》 | 供应商反馈渠道 | 结构化问题收集 |

## 四、系统的进化路径

### 从"人治"到"机制"的四步法

| 步骤 | 动作 | 成果 |
|------|------|------|
| 第一步：隐性知识显性化 | 把脑子里的想法写成文档 | 文档化 |
| 第二步：个人经验制度化 | 把做法写成制度 | 制度上墙 |
| 第三步：临时措施常态化 | 把应急措施变成预案 | 预案入库 |
| 第四步：个人能力组织化 | 让团队/供应商/业务方也能做 | 能力扩散 |

|------|---|---|

## 总结

供应商生态操作系统的核心价值不在于它有多复杂，而在于它**解决了"人走了系统还在运转"的问题**。五大机制保证了输入和输出的自动流转，四大体系保证了执行过程的标准化和可复制。这个系统的终局是：日常管理动作被系统化，管理者聚焦在异常处理、系统优化和生态进化上。
