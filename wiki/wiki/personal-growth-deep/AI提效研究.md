## Overview

AI提效研究涵盖了两个核心方向：一是AI产品定价模式的商业案例研究（2026年国内大模型厂商从Coding Plan到Token Plan的定价演变），二是AI工具（如Quiver API）的高质量使用策略。这些研究反映了年老师对AI作为生产力工具的深度思考——不仅关注"怎么用AI提效"，更关注"AI生态的定价规律和成本结构"。

## 一、AI产品定价模式演变：从Coding Plan到Token Plan

### 核心结论

AI大模型定价正从"卖工具订阅"演进为"卖算力资源包"，这个过程精确重演了电信行业从话费套餐到流量套餐的20年历史。**Coding Plan是过渡产品，Token Plan是终态。**

能从中学到的不是具体的定价数字，而是三件事：
1. **计量单位即定价权**
2. **补贴-收紧-分层的三段式节奏**
3. **"入口产品"到"生态产品"的升级路径**

|------|------|----------|

### 两条路线并行

**模型厂商（专卖店）**：智谱GLM、Kimi、MiniMax、小米MiMo —— 只卖自家模型，深度优化

**云平台（超市）**：阿里云百炼、火山方舟、腾讯云 —— 聚合多模型，一键切换

### 五种计量单位并存（90%开发者踩坑）

| 计量方式 | 代表平台 | 问题 |
|----------|---------|------|
| API请求次数 | 阿里百炼、火山方舟、腾讯云 | 1次提问 = 5-30次API调用 |
| Prompt次数 | 智谱GLM、MiniMax早期 | 1 Prompt ≈ 15-20次模型调用 = 1200-1600 API请求 |
| Token计量 | Kimi（2026.1切换后）、MiniMax Token Plan | 仅统计未命中缓存的Token |
| Credit计量 | Cursor、小米MiMo、阿里百炼Token Plan | 统一Credit池，不同模型不同倍率 |
| 时间窗口配额 | 智谱GLM 5h窗口、Claude Code 5h+7d双层 | 限制节奏而非总量 |

### OpenClaw：Token消耗的催化剂

OpenClaw作为开源AI Agent框架（GitHub 355K stars），直接催热了Coding Plan需求：
- MiniMax M2 token用量：2月vs12月增长6倍，Coding Plan消耗增长10倍+
- 阿里云算力告急，限量供应 → Lite档4-13停售的直接原因
- Anthropic 2026-04从标准订阅移除OpenClaw访问 —— 成本撑不住了

**Agent自动循环调用让"不限量订阅"不可持续，倒逼厂商转向Token精细计量。**

### 芯片供给即定价天花板

**英伟达在中国的退缩**：
- 2024年市占率66% → 2025年54% → 2025年底跌破60%
- 2025-04全面禁止所有AI GPU对华出口

**华为昇腾的崛起**：
- 2025年累计交付81.2万片，市占率~20%，国产第一
- 950PR：FP4算力1.56 PFLOPS（H20的2.87倍），定价~7万元（H200的1/3）
- 2026年目标：AI芯片市占率50%，出货75-100万片，营收超800亿

**DeepSeek V4的定价冲击（2026-04-24发布）**：
| 模型 | 输入（缓存） | 输入（未缓存） | 输出 |
|------|-------------|---------------|------|
| V4-Pro | 1元/百万 | 12元/百万 | 24元/百万 |
| V4-Flash | 0.2元/百万 | 1元/百万 | 2元/百万 |
| 对比GPT-5 | $5/百万 | - | $15/百万 |

DeepSeek官方明确说"受限于高端算力，预计下半年昇腾950批量上市后价格大幅下调"。**芯片供给直接绑定定价天花板。**

### 核心洞察：三段式定价节奏

**阶段一：补贴获客（Price War）**
- DeepSeek带头降价
- 各平台首购优惠、超低价入局
- 目标：抢用户规模

**阶段二：收紧分层（Tighten & Tier）**
- 首购优惠取消
- 低档停售
- 限时抢购（阿里百炼Lite档4-13停售）
- 目标：利润修复

**阶段三：Token Plan终态（Resource Pricing）**
- 从"工具订阅"到"算力资源包"
- Credits统一计量
- 目标：精细计费，可持续盈利

## 二、Quiver API高质量使用策略

### 核心约束

- 每周20次请求（硬约束）
- 每次1 credit（包括n>1的情况）
- Rate limit: 20 requests/60 seconds

### 质量目标

参考标准：东京猫（精细线条~50个path元素）、李诞虾（双层技法+陶土色点缀）

### Prompt结构

```
prompt = [主体描述] + [风格描述] + [构图说明] + [细节要求]
```

**东京猫特征**：
- ~50个path元素（高精细度）
- `stroke-linecap="round"` `stroke-linejoin="round"`（手绘感）
- 双层：填充层+线条层
- 色彩：#BA3420（陶土）、#0B0800（墨黑）、#F7F3ED（米白）

**李诞虾特征**：
- 双层技法更明显（fill layer + ink line layer）
- 纸张纹理（feTurbulence filter）
- 渐变色彩深度（不同透明度）

### 关键假设待验证

| 假设 | 验证方法 | 风险 |
|------|---------|------|
| references接受URL还是base64？ | 测试两种格式 | 可能只接受一种 |
| n>1时是重复还是变体？ | 生成后对比 | 可能浪费配额 |
| instructions对中文prompt支持如何？ | 测试中英混合 | 可能英文更好 |
| context_length 131072是什么单位？ | 长prompt测试 | 可能token限制 |

## 三、AI提效的核心认知

### 1. 计量单位即定价权

无论是API定价还是内部资源分配，**谁定义计量单位，谁就掌握定价权**。Prompt计量→Token计量的切换，本质是智谱、Kimi等平台从"模糊计量"转向"精确计量"，让用户无法直接比价。

### 2. Agent模式改变成本结构

OpenClaw的出现证明：Agent自动循环调用会让Token消耗指数级增长（6-10倍）。这意味着：
- 个人使用AI的成本正在从"可控"变为"不可控"
- 企业使用AI需要建立Token预算管理机制
- "不限量订阅"在Agent时代不可持续

### 3. 芯片供给决定定价天花板

DeepSeek V4的定价逻辑明确展示了"芯片供给→推理成本→Token定价"的传导链条。国产芯片（华为昇腾950PR）的性价比提升，为Token价格下行打开空间。

### 4. 补贴-收紧-分层是标准节奏

所有平台的定价策略都遵循"补贴获客→收紧分层→精细计量"的三段式。这是互联网产品的经典定价套路，从美团外卖到滴滴打车，从电信流量到AI Token，无一例外。

### 5. AI工具使用效率最大化

在配额有限（每周20次API调用）的情况下，高质量Prompt的结构化设计至关重要：
- 主体描述要具体（不是"画一只猫"，是"日本组织架构图风格的猫"）
- 风格描述要精确（双层技法、色彩参数）
- 构图说明要完整（空间布局、元素关系）
- 细节要求要到位（stroke-linecap、feTurbulence等技术参数）
