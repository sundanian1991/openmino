## Overview

数据可视化是数据讲故事的载体。设计得好，数据自己跳出来；设计不好，再多数据也看不出名堂。本文章基于《Storytelling with Data》全书30个技能中的视觉设计部分，结合供应商管理的实际场景，提炼出一套可操作的数据可视化设计原则。核心哲学是**克制**——克制装饰、克制颜色、克制信息量，让关键信息自己跳出来。

## 一、设计的核心哲学：克制

### 克制的三层含义

| 层次 | 含义 | 做法 |
|------|------|------|
| 视觉克制 | 装饰性元素越少越好 | 去掉所有不影响信息传达的元素 |
| 色彩克制 | 颜色用得越少，效果越好 | 一页最多一个强调色，其余灰化 |
| 信息克制 | 信息量越少，重点越清晰 | 一页一个核心观点，不要贪多 |

### 克制的对立面

| 克制 | 不克制 | 结果 |
|------|--------|------|
| 一页一个观点 | 一页塞5个观点 | 受众不知道该看什么 |
| 一个强调色 | 五彩斑斓 | 没有视觉焦点 |
| 去掉装饰 | 保留所有默认元素 | 认知负荷过载 |

## 二、核心设计原则

### 原则一：功能决定形式

先定义受众该做什么/理解什么，再选图表。

| 功能目标 | 图表形式 | 反例 |
|----------|----------|------|
| 时间趋势 | 折线图 | 用饼图展示趋势 |
| 分类对比 | 条形图 | 用柱状图展示20个分类 |
| 部分与整体 | 堆积图 | 用饼图展示占比 |
| 一个关键数字 | 大字，不是图 | 把"1200万"做成饼图 |

### 原则二：条形图零基线强制

条形图代表的是长度，人脑默认从0开始比较。截断Y轴等于视觉欺骗。

| 规则 | 说明 |
|------|------|
| 条形图值轴必须从0开始 | 横/竖/列都适用 |
| 不要为了夸大差异裁剪坐标轴 | 损害可信度 |
| 折线图例外 | 编码的是位置，不是长度 |

### 原则三：禁止饼图/环形图

人脑不擅长比较角度和面积，饼图是认知负担。

| 场景 | 饼图 | 替代方案 |
|------|------|----------|
| 占比对比 | 不好 | 水平条形图 |
| 部分与整体 | 不好 | 100%堆叠条形图 |
| 关键数字 | 不好 | 大字展示 |
| 精确比较 | 不好 | 条形图 |

### 原则四：禁止双Y轴和3D

| 禁止项 | 问题 | 替代方案 |
|--------|------|----------|
| 双Y轴 | 暗示虚假关联 | 上下分图或直接标注 |
| 3D图表 | 扭曲感知 | 永远不用 |

### 原则五：≤10%高亮法则

按效果排序的前注意属性：

| 属性 | 用途 | 注意 |
|------|------|------|
| 加粗 | 文字标题、标签 | 最安全 |
| 颜色 | 核心数据、趋势 | 搭配另一种属性 |
| 大小 | 表示重要性 | 不要滥用 |
| 位置 | 排在序列首位 | 配合其他属性 |

铁律：永远不要同时高亮多个不相关的点。

## 三、去噪音清单

每个视觉元素都要问：它的信息价值值得它的认知成本吗？

### 必须去掉的

| 元素 | 原因 |
|------|------|
| 图表背景填充 | 分散注意力 |
| 不必要的网格线 | 视觉干扰 |
| 3D效果 | 扭曲感知 |
| 多余的颜色 | 没有焦点 |
| 冗余标签 | 信息重复 |
| 边框 | 增加认知负荷 |
| 斜体/下划线 | 干扰阅读 |

### 必须保留的

| 元素 | 原因 |
|------|------|
| 洞察导向标题 | 告诉受众看什么 |
| 核心数据标签 | 直接标注在元素上 |
| 来源脚注 | 增加可信度 |
| 上下文注释 | 解释异常值 |

### 10秒测试

非专业人士能否在10秒内理解你的图表要点？如果不行，说明：
- 标题不够洞察导向
- 噪音太多
- 没有视觉焦点

## 四、Gestalt与对齐原则

### 用感知分组替代显式分隔线

| Gestalt原则 | 做法 | 示例 |
|-------------|------|------|
| 接近性 | 相关项目靠近 | 标题和数据靠近 |
| 相似性 | 同色/同形/同大表示关联 | 同一类别的条形同色 |
| 闭合 | 去掉外框，人脑自动感知完整 | 不用外框包围内容 |
| 连续性 | 沿平滑路径对齐元素 | 数据流向自然引导视线 |
| 连接 | 用线条强关联元素 | 折线连接数据点 |

### 对齐规则

| 规则 | 说明 |
|------|------|
| 文字左对齐 | 自然阅读方向 |
| 图例和标签左上对齐 | 匹配自然视线 |
| 禁止对角文字 | 严重拖慢阅读 |
| 精确对齐 | 用软件网格，不用肉眼估算 |

### 留白规则

| 规则 | 说明 |
|------|------|
| 边距清 | 内容不要贴边 |
| 关键数字用周围空间隔离 | 让重要数字"呼吸" |
| 不要为了填空而填空 | 留白不是浪费 |

## 五、分类数据排序规则

### 按意图排序，不按字母

无内在顺序的分类数据，按分析意图排序：

| 关注点 | 排序方式 |
|--------|----------|
| 最满意 | 按"非常满意"降序 |
| 最不满意 | 按"完全不满意"降序 |
| 未使用 | 按"未使用过"降序 |

### 多个故事要讲怎么办？

保持一个基础顺序不变，用颜色/标注突出次要洞察。不要重新排序。

### 内在有序的分类

年龄段、学历等，保持自然顺序。

## 六、时间序列图表规则

### 轴标签间隔策略

| 数据粒度 | 标签间隔 | 说明 |
|----------|----------|------|
| 日线数据 | 每5或7天标一次 | 受众关心趋势，不关心精确日期 |
| 可加超级分类 | "第1周""第2周" | 帮助理解时间节奏 |
| 不要每个点都标 | 视觉混乱 | 信息密度过大 |

### 预测与实际数据区分

| 元素 | 实际数据 | 预测数据 |
|------|----------|----------|
| 线条 | 粗实线 | 细虚线/点线 |
| 背景 | 正常 | 淡色阴影区隔离 |
| 数值标签 | 只在关键点标 | 每点都标 |
| X轴标签 | ACTUAL | FORECAST |

## 七、多线图（意大利面图）解构

### 三步法

**第一步：评估数据必要性**

全数据集是否必需？去掉不相关类别或时间段。

**第二步：选择解构策略**

| 策略 | 做法 | 适用场景 |
|------|------|----------|
| 逐个高亮 | 其他灰化，目标加粗 | 演讲 |
| 垂直小多图 | 上下排列，共享X轴 | 关心各自趋势 |
| 水平小多图 | 左右排列，共享Y轴 | 特定时间点对比 |
| 组合 | 小多图+每面板高亮 | 报告 |

**第三步：强制一致性**

所有面板轴范围必须相同。

## 八、供应商管理常用图表模板

### 供应商排名——水平条形图

| 要点 | 做法 |
|------|------|
| 排序 | 按产能降序 |
| 高亮 | Top3用强调色，其余灰色 |
| 标注 | 直接在条形末端标数值 |
| 标题 | 洞察导向："新商75%进TOP3" |

### 月度趋势——折线图

| 要点 | 做法 |
|------|------|
| 核心线条 | 总GMV用强调色实线 |
| 辅助线条 | 各产线用灰色细线 |
| 轴标签 | 每季度标一次 |
| 标题 | "GMV持续增长，Q4增速放缓" |

### 正负变化——正负堆叠条形图

| 要点 | 做法 |
|------|------|
| 零基线 | 水平零基线=当前状态 |
| 正贡献 | 堆在上方（产能增长） |
| 负贡献 | 堆在下方（产能下降） |
| 缺口 | 用空心条形标注 |

### 巡检发现问题——热力图

| 要点 | 做法 |
|------|------|
| 行 | 供应商 |
| 列 | 巡检标准项 |
| 颜色 | 红（问题多）→绿（无问题） |
| 数字 | 保留单元格内的数字 |

### 供应商画像矩阵——对比矩阵

| 要点 | 做法 |
|------|------|
| 行 | 供应商 |
| 列 | 六维标签 |
| 单元格 | 实心圆（优势）/空心圆（劣势） |
| 颜色 | 强调色（优势）/灰色（劣势） |

## 九、迭代方法论

### 验光师迭代法

每次只改一个元素，左右对比选优：

| 轮次 | 改什么 | 对比 |
|------|--------|------|
| 1 | 标题 | 描述性 vs 洞察性 |
| 2 | 颜色 | 全彩 vs ≤10%强调色 |
| 3 | 图表类型 | 饼图 vs 水平条形图 |
| 4 | 数据标签 | 轴上 vs 直接在元素上 |
| 5 | 对齐 | 肉眼 vs 精确网格 |

### 外部反馈

找代表受众的同事，不解释你的意图，让他们自己看：

| 标准问题 | 目的 |
|----------|------|
| "你首先注意到什么？" | 验证视觉焦点 |
| "你得出了什么结论？" | 验证信息传达 |
| "怎样能让重点更清晰？" | 找到改进方向 |

## 十、常见错误与避坑

| 错误 | 问题 | 修正 |
|------|------|------|
| 标题是描述性的 | 没有洞察 | 改成结论句 |
| 一页放太多内容 | 认知负荷过载 | 拆分或移附录 |
| 图表颜色太多 | 没有焦点 | ≤10%强调色 |
| 没有数据来源 | 缺乏可信度 | 加页脚标注 |
| 用饼图展示占比 | 认知负担 | 改用条形图 |
| 双Y轴对比 | 虚假关联 | 上下分图 |
| 截断条形图Y轴 | 视觉欺骗 | 从零开始 |

## 总结

数据可视化设计的核心是**克制**。克制装饰、克制颜色、克制信息量，让关键信息自己跳出来。功能决定形式、零基线强制、禁止饼图双Y轴3D、≤10%高亮、Gestalt分组、逻辑排序、持续迭代——这些不是"规则"，而是**"让数据说人话的工具"**。最终目标不是"好看"，而是"让受众在10秒内理解你要说什么，然后做出行动"。
