## 概述

本文整合了供应商管理中八大核心场景的操作指南和沟通框架，涵盖KPI不达标整改、供应商绑架应对、份额调整、新供应商准入等关键场景。同时整合了场景问题清单、看板设计和Dashboard布局等辅助文档。这些场景是供应商管理者日常面对的最高频、最复杂的决策场景，每个场景都包含业务判断和沟通协调两个维度。

## 一、场景一：KPI不达标整改

### 判断标准

| 等级 | 条件 | 动作 |
|------|------|------|
| 黄灯 | 连续1个月不达标 | 提醒关注 |
| 橙灯 | 连续2个月不达标 | 正式整改 |
| 红灯 | 连续3个月不达标 | 熔断/清退启动 |

### 处置流程

1. **数据确认** — KPI数据无争议
2. **原因诊断** — 外部因素 vs 内部因素
3. **整改要求** — 三要素法：原因+目标+举措
4. **跟踪验证** — 周度跟进、月度复盘

### 沟通框架

| 沟通类型 | 对象 | 内容 | 风险点 |
|---------|------|------|--------|
| 内部同步 | 策略组 | KPI数据、整改计划 | 漏同步导致名单分配出问题 |
| 内部问询 | 策略组 | 名单质量是否有变化 | 漏问可能误判供应商责任 |
| 内部决策 | 上级 | 是否启动熔断机制 | 权限不清自己背锅 |
| 外部问询 | 供应商运营 | 真实困难是什么 | 只听表面原因，没深挖 |
| 外部决策 | 供应商老板 | 整改承诺：目标+举措 | 没有具体承诺，无法追踪 |

### 核心话术：三要素法

> "这个月KPIXX，原因是什么？下个月目标是多少？具体举措是什么？"

**追问技巧**：
- 原因太笼统 → "能具体说说是哪个环节的问题吗？"
- 目标模糊 → "具体数字是多少？"
- 举措空泛 → "第一周做到什么程度？谁负责？"

## 二、场景二：供应商"绑架"应对

### 判断标准

| 维度 | 信号 |
|------|------|
| 市场份额大 | 有外部客户可去 |
| 技术/人员壁垒 | 替换成本高 |
| 信息不对称 | 掌握关键客户/数据 |

### 处置策略

1. **判断真假** — 是否真有退路？真实诉求是什么？
2. **准备筹码** — 替代方案、中腰部培育情况
3. **谈判底线** — 什么能让？什么不能让？
4. **渐进替换** — 如需替换，先减份额再退出

### 判断威胁真假

| 真退出信号 | 假退出信号 |
|-----------|-----------|
| 已经在接外部客户 | 只是要价手段 |
| 人员开始转移 | 没有实际行动 |
| 合同快到期 | 还在投入 |

### 谈判筹码

- 有替代方案 → 立场更稳
- 无替代方案 → 缓兵之计，同时加速培育替代
- 自建团队成本（费效1.2）→ 压价锚点
- 替代供应商报价 → 对比筹码
- 长期合作承诺 → 以量换价

## 三、场景三：份额调整

### 判断标准

| 排名 | 动作 |
|------|------|
| Top3 | 增份额、优先给新业务 |
| 中游 | 份额稳定 |
| 后10% | 减份额、预警 |

### 处置流程

1. 数据确认（赛马排名、KPI数据）
2. 方案制定（调多少？怎么调？）
3. 内部对齐（策略组、上级）
4. 供应商沟通（预期管理）
5. 正式执行

### 预期管理话术

**增份额**：
> "这次赛马排名前三，下个月份额增加XX%，继续加油"

**减份额**：
> "这次赛马排名靠后，下个月份额调整XX%，原因是XX。如果想恢复，需要XX"

## 四、场景四：新供应商准入

### 判断标准

| 类型 | 标准 |
|------|------|
| 硬性标准 | 资质、规模、经验 |
| 软性标准 | 配合度、服务质量 |
| 一票否决 | 合规问题、虚假资料 |

### 一票否决项

- 资质造假
- 有重大合规违规记录
- 财务状况严重恶化
- 核心团队不稳定（流失率>50%）

### 处置流程

1. 资料审核（资质、案例、团队）
2. 现场考察（如需要）
3. 试单评估
4. 合同谈判
5. 正式准入

## 五、通用沟通原则

### 内部沟通三原则

1. **先同步后行动** — 重大事项先让相关方知道，避免被动
2. **明确决策权** — 谁能拍板？请示谁？不能自己越权
3. **留痕可追溯** — 关键沟通书面化，有据可查

### 外部沟通三原则

1. **先问后说** — 了解供应商真实情况再表态
2. **让供应商承诺** — 不给答案，让对方说怎么办
3. **说清后果** — 不达标会怎样，提前告知

### 沟通遗漏自查清单

**内部**：
- [ ] 策略组知道吗？
- [ ] 上级知道吗？需要审批吗？
- [ ] 财务/法务需要同步吗？

**外部**：
- [ ] 供应商老板知道吗？
- [ ] 供应商运营知道吗？
- [ ] 后果说清楚了吗？

## 六、看板设计与Dashboard布局

### 供应商管理看板的核心要素

供应商管理看板需要呈现的数据维度：
- 排名与趋势
- 产能利用率（自身峰值比+头部差距）
- 合规状态（红黄绿）
- FCI能力指数
- 主管胜任评估
- 异常信号清单

### Dashboard布局设计原则

- 一页手机屏幕，3秒扫完
- 红黄绿信号灯 + 每家一行 + 一句话结论
- 业绩达成（策略组数据）+ FCI能力（服务组数据）并排呈现

## 七、场景问题清单

### 待解决的问题

| 问题 | 描述 | 优先级 |
|------|------|--------|
| 内部角色不完整 | 缺少培训、业务管理、质检运营在各场景中的职责 | 高 |
| 角色视角单一 | 每个场景只写了年老师做什么，没写其他角色的主动动作 | 高 |
| 缺少角色交集矩阵 | 年老师"居中角色"连接上下左右 | 中 |
| 判断边界模糊 | 黄灯→橙灯加速条件、增减具体比例 | 高 |
| 自查清单缺横向协作 | 缺培训、业务管理、质检配合检查项 | 中 |
| 场景升级路径不清晰 | 什么情况下直接跳级？整改验收标准是什么？ | 高 |
