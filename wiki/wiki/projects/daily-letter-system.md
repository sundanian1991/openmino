# 每日书信系统

> Sources: Mino, 2026-04-12 ~ 2026-04-26
> Raw: [daily-letter SKILL.md](../../raw/skills/daily-letter-SKILL.md)

## 概述

每日书信系统从对话记录中提取"想法种子"，以李娟的人文叙事加芒格的智慧洞察视角，生成书信体记忆。不是每天一个快照，而是一部连续剧——每封信既属于今天，又属于整条线。

## 数据源架构

书信系统读取双数据源，合并后去重：

| 数据源 | 路径 | 说明 |
|--------|------|------|
| JSONL 会话 | `~/.myagents/sessions/*.jsonl` | MyAgents 全部对话，按文件修改时间筛选 |
| CodePilot 数据库 | `~/.codepilot/codepilot.db` | SQLite，按日期筛选所有项目会话 |
| 项目日志 | `memory/daily/YYYY-MM-DD.md` | 当日对话摘要（辅助） |
| MemOS 记忆 | search_memory | 当日新增记忆 |
| 文件变更 | git diff | 今日改动的文件 |

两份核心数据源必须都读取，不比较数量大小，不以谁为主，合并后按会话 ID 去重。

## 三遍过筛流程

### 第一遍：滤噪声

过滤掉纯操作记录、重复性事务、纯技术细节、系统消息。留下有判断、有犹豫、有追问、有反思、有情绪流露的对话。

### 第二遍：打标签

内容维度：认知、关系、工作、自己、世界

性质标签：判断（高权重）、困惑（中）、洞察（高）、决定（高）、情绪（中）、追问（反复出现升为极高）

### 第三遍：找模式

横向比对：今天不同事件间的共同倾向

纵向追问：同一主题是否反复出现

反常捕捉：与平时表现不同的信号

AI 自检：每个模式必须有 >= 2 个独立对话证据，否则降级为"微弱信号"

## 主题对齐机制

读近 7 天信件和近 30 天索引，建立时间上下文。对每个今天的模式问三个问题：

1. 过去 7 天出现过吗？→ 深化、偏离还是回到原点
2. 过去 30 天出现过但最近断了线吗？→ 标记为"回归"
3. 多个独立模式指向同一长期主题吗？→ 合并

时间叙事写法：

| 匹配结果 | 写法 | 示例句式 |
|----------|------|----------|
| 新种子 | 如实记录 | "今天第一次注意到..." |
| 深化 | 明确引用历史 | "上周写过他...今天..." |
| 转折 | 标注转折信号 | "但今天有一个变化..." |
| 回归 | 标记回归 | "这个想法已沉默两周，今天又出现了" |
| 合并 | 展现汇聚效应 | "三件不同的事，说的其实是同一件事" |

## 写作风格

李娟人文叙事：温暖、克制、有画面感，从细节切入，不宏大叙事。

芒格智慧洞察：穿透表象看本质，追问"为什么重要"、"背后是什么规律"。

三种写作形态：
- **形态A**：一封信一个主题（贯穿全天的核心主题）
- **形态B**：2-3个独立主题（多件有份量的事）
- **形态C**：日常切片（平淡但有意义的细节）

字数 600-1200 字，宁短勿长。空日写 200-300 字短信。

## 校验规则

双数据源都读取、模式有证据（>= 2个独立对话）、不迎合（必须包含困惑/犹豫）、时间对齐真实（不编造历史引用）、非流水账（按主题组织）、有实质内容（每段传递新信息）。

归档位置：`memory/daily-letter/YYYY-MM-DD.md`
