## 概述

电影叙事项目基于 cinematic-ui 技能，将电影语言翻译为网页设计——用导演的视觉系统、分镜驱动的布局规划和电影感动效，而非通用落地页模板。核心机制：选导演 + 选电影 → 提取情感语言 → 翻译为网页结构和实现规范。

## 四阶段工作流

项目严格遵循四个阶段，不可跳步：

| 阶段 | 输出文件 | 核心任务 |
|------|----------|----------|
| Phase 1: Decisions | `decisions.md` | 选导演、选电影、独特性审计、主构图族 |
| Phase 2: Storyboard | `storyboard.md` | 电影语法定义、场景论点、签名构图 |
| Phase 3: Compiled Spec | `compiled-spec.md` | 提取 CSS/JS、交互预算、实现规范 |
| Phase 4: Build | HTML 输出 | 按规范构建、验证、筛选室调整 |

在每个阶段内遵循严格顺序：先定义全站电影语法 → 为每个页面角色写独立场景论点 → 锁定每个页面不可替代的签名构图 → 最后才推导共享系统。

## 核心设计原则

### 导演+电影驱动

每个项目必须选定一位导演和一部具体电影，作为情感来源和视觉语言的根基。不做通用"高端品牌"推断。

### 独特性协议

如果用户已有过往作品，必须进行唯一性审计：

- 记录重复出现的特征（如：左文右图英雄区、顶部导航+堆叠面板、圆角卡片等）
- 建立"禁止列表"（Shell-ban list），新作品的主构图族必须与之前不同
- 判断标准：去掉颜色、字体和装饰效果后，线框图仍应不同

### 构图优先

网格是基础设施，不是构图本身。如果一个区块作为默认 2x2 或三列网格仍然成立，说明构图太弱。

### 克制即设计

高端作品往往去掉 20% 的显而易见的手法，而不是增加 20% 的细节。

## 交互预算

严格控制交互复杂度：

| 规则 | 限制 |
|------|------|
| 重度交互 | 每页最多 1 个 |
| 注意力吸引模式 | 每页最多 2 个 |
| fadeUp / opacity + translateY | 每页最多出现 2 次 |
| 入场类型 | 每页至少使用 4 种不同的入场类型 |
| 其余动效 | 必须服从页面的视觉论点 |

## 反模式

| 反模式 | 说明 |
|--------|------|
| 通用渐变英雄区 | 除非源电影真正支持，否则禁止 |
| 水印导演名/电影名 | 不在导航或英雄区暴露工作流标签 |
| 统一悬停/揭示模式 | 每个区块不能用同一种模式 |
| 动效演示场 | 页面不是效果采样器 |
| 参考图做模板 | 借质量维度，不复制完整构图 |
| 共享组件抹平差异 | 可复用卡片不能消除页面场景身份 |
| 先定组件再定构图 | 共享组件系统必须在页面级构图锁定后才决定 |
| 内部页面缩水 | 内部页面必须是同一部电影中的新场景，不是主页的简化版 |
