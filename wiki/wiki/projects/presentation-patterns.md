## 概述

演讲稿项目遵循统一的文件结构、变体命名和输出规范。每个项目包含 README、design_spec、逐页notes和sources等标准组件，支持同内容多风格变体并行产出。

## 项目结构

标准 PPT 项目包含以下文件：

| 文件类型 | 命名格式 | 用途 |
|----------|----------|------|
| README | `projects-{主题}_ppt169_{日期}-README.md` | 项目基本信息、目录结构说明 |
| 设计规格 | `...-design_spec.md` | 视觉风格、配色、字体、布局定义 |
| 逐页备注 | `...-notes-{序号}_{章节}.md` | 演讲者备注、要点、时长 |
| 汇总备注 | `...-notes-total.md` | 全部备注合并版 |
| 素材 | `...-sources-{序号}-{阶段}.md` | 头脑风暴、研究等原始素材 |

画布格式统一为 `ppt169`（16:9 比例），日期格式为 `YYYYMMDD`。

## 变体命名规律

同一主题通过后缀区分不同风格变体：

| 变体后缀 | 风格特征 | 代表项目 |
|----------|----------|----------|
| `editorial-v2` | 杂志编辑风，人文感 | 刘乾坤人物分析 |
| `exhibit` | 展览陈列风，客观呈现 | 刘乾坤人物分析 |
| `freestyle` | 自由演绎风，创意发挥 | 刘乾坤人物分析 |
| `infographic-v2` | 信息图风，数据密集 | 刘乾坤人物分析 |
| `mckinsey-v2` | 咨询报告风，结构化 | 刘乾坤人物分析 |
| `ppt169` | 默认标准版 | 所有项目 |

同一内容生成 5-6 个变体，覆盖从正式到创意的完整风格谱系，方便对比选择。

## 逐页备注规范

每个 notes 文件包含：

- **过渡语**（[Transition]）：承上启下的衔接语句
- **核心内容**：该页要表达的关键信息
- **要点列表**：3条以内核心要点（① ② ③）
- **时长**：预计讲解时间（0.5-3分钟）
- **停顿标记**（[Pause]）：需要留白的时刻
- **数据标记**（[数据]）：需要强调的具体数字

## 叙事结构模式

### 商业分析类（事实→框架→洞察→预判）

典型结构：封面 → 目录 → 事实层（时间线/对比）→ 框架层（五力/驱动/节奏）→ 洞察层（规律提炼）→ 预判层（趋势）→ 结尾

### 人物分析类（定位→拆解→应对）

典型结构：封面 → 一句话定位 → 权力内核 → 权力来源 → 行为模式 → 关系网络 → 应对策略 → 可学习内容 → 争议事件 → 结尾

## 输出产物

每个项目产出以下目录结构：

| 目录 | 内容 |
|------|------|
| `svg_output/` | 原始 SVG 输出 |
| `svg_final/` | 最终版 SVG |
| `images/` | 项目资源图片 |
| `notes/` | 演讲者备注 |
| `templates/` | 项目模板 |
| `sources/` | 素材和标准化 Markdown |
