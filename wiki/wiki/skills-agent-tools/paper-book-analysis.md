# Paper & Book Analysis + Content Casting Skills

> Sources: Epiplexity (via MyAgents), 2026-04-27
> Raw: [ljg-xray-paper](../../raw/skills/ljg-xray-paper-SKILL.md); [ljg-xray-book](../../raw/skills/ljg-xray-book-SKILL.md); [ljg-card](../../raw/skills/ljg-card-SKILL.md) + modes + taste

## 概述

这组技能由李继刚（Epiplexity）设计，构成一个完整的知识处理管线：先通过 `ljg-xray-paper` 和 `ljg-xray-book` 对学术论文和书籍进行深度结构提取与认知碰撞，再通过 `ljg-card` 将内容铸造成七种视觉形态的 PNG 图片。前半段解决"理解什么"，后半段解决"怎么让人看见"。

## 论文解读（ljg-xray-paper）

论文解读只做两件事：论文说了什么（问题 → 视角 → 结果），以及对我意味着什么（认知碰撞卡片）。

**核心约束**：

- **零术语规则**：每个技术概念必须落在读者见过的生活场景上，不给例子就不准用那个词。承重概念需要至少三级渐进展开，让读者在熟悉的事情上"感受到"概念变化。
- **认知卡片 = ASCII art**：卡片不是带框的文字列表，而是用空间布局表达结构关系（分叉、汇聚、层级、对比、拉扯）。好卡片的标准是遮住文字只看线条仍能感受到结构。
- **诚实原则**：delta ≈ 0 是正常结果，不硬凑。没有认知碰撞就不造卡片。

**执行流程**：预处理输入（arxiv 一律转 `/html/` 格式）→ 加载认知参照系（`soul.md` + `memory.md`）→ 论文三句话（问题/视角/结果，人话不要八股）→ 认知碰撞卡片（对照已有认知基线提取改变点）→ 生成 Org 报告。

**卡片类型示例**：分叉型（决策路径分化）、阈值型（非线性转折点）、张力型、缺口型、翻转型。碰撞有多种形状，选跟碰撞本身匹配的空间结构。

## 深度拆书机（ljg-xray-book）

基于《From Entropy to Epiplexity》论文的核心洞见：信息是相对的，取决于观察者的"认知算力"。同一本书 = 可学习的结构 + 不可学习的噪声，而 S/N 切割线的位置取决于投入多少算力。你觉得"读不懂"不是书太难，是算力不够。

**三轮认知压缩**：

1. **骨架扫描**：作者试图回答什么问题？核心答案一句话。章节骨架（每章不超 10 字）。论证结构类型（演绎/归纳/案例/对比）。
2. **血肉解剖**：论证链条如何支撑结论。最有说服力的 3 个证据/案例。作者没说但必须成立的隐形假设。结论在什么情况下失效。
3. **灵魂提取**：作者的盲点是什么。这个思想在其他领域叫什么。与读者已有知识的交叉点。读完应该做什么不同的事。

**餐巾纸压缩**：如果一本书只能留下一个公式、一张图、一句话，是什么。最终生成 Org 模式报告，包含三轮压缩的完整内容加最终的餐巾纸输出。

## ljg-card：铸

内容进去，PNG 出来。模具决定形状。七个模具覆盖从阅读卡片到漫画的完整光谱：

### 模具一览

| 参数 | 模具 | 尺寸 | 适用场景 |
|------|------|------|----------|
| `-l`（默认） | 长图 | 1080 x auto | 单张阅读卡，内容撑高 |
| `-i` | 信息图 | 1080 x auto | 内容驱动的自适应视觉布局 |
| `-m` | 多卡 | 1080 x 1440 | 自动切分为多张卡片（小红书 3:4） |
| `-v` | 视觉笔记 | 1080 x auto | 手绘风格 sketchnote |
| `-c` | 漫画 | 1080 x auto | 日式黑白漫画风格 |
| `-w` | 白板 | 1080 x auto | 白板马克笔风格，结构化框图 |
| `-b` | 大字 | 1080 x 1440 | 碑刻大字 + 和紙，单句/短段 |

所有模具共享同一个截图工具链：`node assets/capture.js`，通过 Playwright 渲染 HTML 后截图输出 PNG。

### 品味准则（全模具共享）

反 AI 生成痕迹是核心底线：禁止 Inter 字体、禁止纯黑 `#000000`（用 Off-Black 替代）、禁止三等分等宽卡片、禁止居中 Hero（除非极稀密度场景）、禁止 AI 文案腔（"赋能""无缝""释放"）、禁止假数据（`99.99%` 式）。最多 1 个强调色且饱和度低于 80%，同一张图严格统一冷暖调。每张图至少有一个让人第一眼就被抓住的锚点元素。

## 长图模具（-l）

单张阅读卡，不做切分，所有内容放在一张卡内。根据内容气质自动选择色调：思辨/哲学用暖米色+棕色、技术/工程用冷灰蓝、文学/叙事用暖黄+深棕、科学/研究用浅绿+深绿。

格式化时识别金句（独立成段、25 字以内、承载核心洞察）用 `.highlight` 类渲染，首个正文段落加 `dropcap` 首字下沉效果。章节之间用 `.divider` 分隔。

## 信息图模具（-i）

不存在"默认布局"——每一张信息图的视觉形式从思想的形状中生长出来。模板只提供画布材质（字体、颜色、噪点、署名），构图/排版/布局全部从零设计。

**三个设计维度**：密度（稀/中/密决定呼吸节奏）、结构（单点/对比/层级/流程/辐射/并列决定几何）、情绪（沉思/锐利/温暖/技术决定温度）。根据内容关键词匹配八种色调之一，用 90/8/2 法则控制色彩：90% 中性色、8% 结构色、2% 弹点色。

反死亡清单：发现自己在写 `.header { padding: 56px }` 时说明在用旧模板思维；三列等宽绝对禁止；正文字号小于 36px 手机上会不可读；弹点色用了 3 处以上要回到 2 处；最大元素和最小元素比例须大于 10:1。

## 多卡模具（-m）

1080x1440 固定画布，按视觉重量贪心切分。每卡约 380 字符等价视觉重量，普通段落权重 1.4、标题 6.0、金句 3.0、条目组 1.8、分割线固定 60。切分规则：绝不在句子中间切，优先在段落/条目/章节边界切，标题不落单。与长图共享色调系统。

## 大字模具（-b）

一句话砸在纸上——碑刻于和紙，悬空如附件。审美三柱：「重」（每行 2-10 字，字号 98-220px）、「旧」（云晕、毛边，SVG 湍流位移做旧）、「悬」（暗底外框、深层阴影，像悬在空气中）。

输入限制 60 字以内，超过则建议改用 `-l` 或 `-m`。字号根据字符数查表选择。手动 `<br>` 断行（不依赖浏览器自动换行），按语义单元切分。朱砂高亮每图最多 1 处。

## 视觉笔记模具（-v）

"像人手画出来的，不是排版出来的。"灵魂是"一边想一边画"的痕迹感——概念之间用箭头连起来、关键词被圈出来、旁边有小简笔画。

四种风格路线：火柴人叙事（Tim Urban 路线，适合有故事线的内容）、概念地图（Mike Rohde 路线，多概念并行）、餐巾纸草图（Dan Roam 路线，一个核心图解可说清）、概念拼贴（Christoph Niemann 路线，强视觉隐喻）。布局原则：概念节点散布在画面上，位置由关系决定不由网格决定，大小代表重要性。

## 漫画模具（-c）

"黑与白之间的张力，就是全部。"用日式黑白漫画的视觉语言讲故事——分格节奏、黑白对比、集中线和留白制造戏剧性。

五位漫画家风格可选：大友克洋（精密废墟，技术/系统类）、井上雄彦（水墨留白，哲学/沉思类，默认）、三浦建太郎（暗黑压迫，冲突/困境类）、松本大洋（生猛粗线，运动/能量类）、谷口治郎（静谧精描，日常/观察类）。同源多卡时统一风格，不逐张选。

分格系统用 CSS Grid 实现不规则分格，支持出血格、斜切分格。视觉元素包括集中线（`repeating-conic-gradient`）、速度线、对话泡/思想泡/吼叫泡、网点灰、拟声词、墨色浓淡渐变。文字竖排可选，字号对比极端（标题 120px+ 对正文 32px，比例大于 6:1）。

## 白板模具（-w）

"推理过程的可见化。"审美三柱：「余白」（空间不是剩余，是主角）、「枯」（色彩极度克制，暖灰调一抹微朱点睛）、「素」（无装饰，干净到只剩内容和空气）。

四种风格路线：逻辑链（默认，横向推理链加纵向层级）、脑暴墙（核心词居中放射状分支）、时间线（纵向时间轴）、矩阵分析（2x2 象限）。

色调为和紙奶白加朱砂：`--board` 为 `#F7F3EC` 和紙色，`--ink` 为 `#2C2826` 墨色，`--yellow` 为 `#7A6B4E` 焙茶色（高亮），`--red` 为 `#A09888` 石灰（箭头），`--shu` 为 `#C03C28` 朱砂（点睛，全图不超 3 处）。根据内容主题支持微妙的色温偏移（技术/人文/商业），只调 `--board` 和 `--yellow`，其余不动。

间距分层用间距说话不用线说话：零距（同链续行 0-4px）、小距（同话题不同链 14-20px）、中距（不同话题之间 32-44px）、大距（大转折/新章节 52-72px）。禁止 `border-top` 直线分隔。标题必须是一句完整的判断/结论而非一个概念词。

## 技术架构

所有模具共享同一基础设施：HTML 模板通过变量插槽（`{{CUSTOM_CSS}}`、`{{CONTENT_HTML}}`、`{{SOURCE_LINE}}`）注入内容，写入 `/tmp/` 目录后通过 Playwright 截图输出 PNG 到 `~/Downloads/`。Footer 统一为左侧 logo + 李继刚署名，右侧可选内容来源。截图时非固定画布模具使用 `fullpage` 模式，固定画布模具（-b、-m）使用精确宽高。
