## 概述

技能（Skills）是 Claude Code 生态中可复用的工作流模板。通过学习 Cowork 插件手册、技能改进计划、集成计划和上下文优化审计，建立了一套完整的技能管理和集成体系。

## Cowork 插件生态

Cowork 插件为 Claude Code 扩展了专业领域能力，已安装5个插件：

### 插件功能清单

| 插件 | Commands | Skills | 核心场景 |
|------|---------|--------|---------|
| **data** | 6个 | 7个 | 数据分析、SQL生成、可视化、数据验证 |
| **productivity** | 2个 | 3个 | 任务管理、工作记忆、优先级排序 |
| **sales** | 3个 | 6个 | 客户沟通准备、销售策略分析 |
| **legal** | 6个 | 6个 | 合同审查、NDA处理、合规检查 |
| **finance** | 5个 | 6个 | 财务对账、成本分析 |

### 使用方式

> **方式1（Slash Commands）**：直接输入命令如 `/legal:review-contract <文件>`
> **方式2（自然语言）**：描述需求，AI自动判断使用哪个插件

对于供应商管理工作，最有价值的是：
- **legal:vendor-check**：供应商合规检查
- **data:explore-data**：供应商KPI数据分析
- **finance:reconciliation**：成本对账

## 技能改进计划

技能改进计划文档（skill-improvement-plan.md）识别了现有技能的问题和改进方向：

### 常见问题

1. **技能加载过载**：会话启动时加载过多SKILL.md，占用上下文窗口
2. **技能边界模糊**：多个技能功能重叠，AI不知道该用哪个
3. **渐进式披露不足**：一次性加载全部技能文档，而非按需加载
4. **缺少验证机制**：技能执行完没有自检流程

### 改进方案

> **渐进式披露（Progressive Disclosure）**：技能文档分三层加载
> - 第一层（元数据）：技能名、用途、触发条件（<50字）
> - 第二层（核心流程）：主要步骤和关键规则（200-500字）
> - 第三层（完整文档）：仅在需要时加载全文

## 技能集成计划

技能集成计划定义了技能之间的协作关系：

### 技能协作模式

```
需求输入
    ↓
技能路由（判断用哪个技能）
    ↓
主技能执行
    ↓
辅助技能（如需要）
    ↓
验证检查（是否达标）
    ↓
输出
```

### 集成原则

1. **单一职责**：每个技能只做一件事，做精
2. **正交设计**：技能之间不重叠功能
3. **可组合性**：多个技能可以串联使用
4. **降级路径**：技能不可用时，有fallback方案

## 上下文优化审计

上下文优化审计（context-optimization-audit.md）分析了技能加载对上下文的影响：

### 上下文预算

| 组件 | 占用 Token | 优化策略 |
|------|-----------|---------|
| CLAUDE.md（主文件） | ~2000 | 保持精简，索引到其他文件 |
| rules/（5个核心文件） | ~8000 | 只加载核心5个 |
| skills/（全部加载） | ~20000+ | 按需加载，不全部加载 |
| 对话历史 | 持续增长 | 定期压缩，保留关键信息 |

### 优化建议

> **1. 核心规则不超过10KB**：5个核心rules文件总大小控制在10KB以内
> **2. 技能按需触发**：不预加载，通过TRIGGER条件动态加载
> **3. 索引代替全文**：用README索引各子文件，不直接加载所有内容

## 技能验证清单

每个技能交付前需要检查：

- [ ] 触发条件明确（不会误触发也不会漏触发）
- [ ] 输出格式标准化（遵循统一的输出规范）
- [ ] 错误处理完善（异常情况有明确的处理方式）
- [ ] 上下文影响可控（加载技能不会挤占核心上下文）
- [ ] 可以独立运行（不依赖其他未加载的技能）
- [ ] 有版本管理（SKILL.md中包含版本号）

## jiyi（记忆）技能

jiyi 技能聚焦于会话记忆管理：

> **核心能力**：跨会话记住关键信息，减少重复输入
> **与 MEMORY-L1 的关系**：jiyi 是技能实现，MEMORY-L1.md 是内容定义
> **使用场景**：年老师的偏好、供应商信息、项目状态等需要跨会话记住的信息

## OBSIDIAN 深度阅读技能

这个技能将 Claude Code 与 Obsidian 知识库连接：

> **核心流程**：Obsidian 笔记 → Claude 分析 → 结构化输出
> **适用场景**：从个人笔记中提取结构化知识，生成报告或分析

## CAPTURE 技能

CAPTURE 技能关注信息的即时捕获和保存：

> **核心能力**：在对话过程中即时捕获重要信息到文件系统
> **与 WAL 协议的关系**：CAPTURE 是 WAL（Write-Ahead Logging）协议的技能实现 — 关键信息先写后答
