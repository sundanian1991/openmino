# 工作表达与思考工具（Work + Work-Expression）

> Sources: MyAgents, 2026-04-27
> Raw:[work](../../raw/skills/work-SKILL.md); [work-expression](../../raw/skills/work-expression-SKILL.md); [templates](../../raw/skills/work-expression-templates.md); [accumulated](../../raw/skills/work-expression-accumulated.md)

## 概述

Work 和 Work-Expression 是年老师供应商管理日常工作的两套互补技能：Work 负责"想清楚"——根据场景自动匹配22个思考框架模板，系统化拆解问题；Work-Expression 负责"说明白"——将口述稿升级为定性+定量融合的L4级高分辨率表达。两者链式调用，形成"想清楚→说明白→生成公文"的完整工作闭环。

## Work 技能：22个场景思考框架

Work 技能面向年老师管理的32+供应商、3200+人团队、4大产线的日常场景。年老师描述一个工作场景或直接指定模板编号，系统自动匹配对应的思考检查清单（Plan模板），输出结构化的"第一步想什么、第二步想什么、关键变量有哪些"。

### 调用方式

- **直接指定编号**：`/work A1`（最准）
- **场景描述自动匹配**：`/work 毅航排名后30%了`（最方便，自动匹配A1）
- **对话中唤起**：`@Mino 用 work 分析一下毅航业绩`

### 模板体系（五大类22个）

**A类—日常管控**（6个）：A1业绩异常判断、A2约谈准备、A3分量调整决策、A4转正延期决策、A5合规问题定级、A6清退评估。核心是"看到信号→系统化判断→决策"。

**B类—项目与流程**（6个）：B1新供应商准入（8阶段SOP，T+15）、B2新项目对接（5W2H确认）、B3出差巡检准备（12项标准+FCI速记卡）、B4月度复盘、B5向上汇报准备、B6降价谈判准备。

**C类—通用能力**（4个）：C1问题拆解（MECE拆解到可执行）、C2邮件起草、C3数据整理分析（体检+洞察提取）、C4制度流程梳理。

**D类—快速响应**（4个）：D1被同事@数据进度（30秒快速判断）、D2被领导点名（快速组织口径）、D3供应商来电来消息（判断真问题vs撒娇）、D4供应商群异常信号（24h未回复、态度变化、推诿）。

**E类—新供应商**（2个）：E1新供应商初筛、E2准入评审决策（第6个月末转正/延期/清退）。

### 匹配逻辑

精确匹配（场景含明确关键词直接命中）→ 多候选（同时命中多个模板列出选项）→ 无匹配（建议C1问题拆解或手动指定编号）。

### 输出结构

统一输出：模板编号与用途、触发信号、思考步骤（含数据需求与判断标准）、关键变量、常见误区、决策输出格式、下一步衔接。D类模板额外输出30秒快速判断和5分钟深入版两个层级。

### 与 Official-doc 的链式调用

```
/work A1 分析毅航业绩 → 输出思考框架 + 数据需求 → 年老师提供数据
→ Work 给出判断结论 → /official-doc operations-missing 生成约谈通知
→ Official-doc 技能生成正式公文
```

模板文件全部位于 `docs/plan-templates/` 目录，思考步骤引用供应商管理工具箱、AI提效方案拆解（196小时）、100+公文模板等硬知识。

## Work-Expression 技能：定性+定量融合表达

将口述稿升级为高分辨率表达，让年老师在工作中说话有数据、有逻辑、有说服力。适用于汇报、同步、追问、预警、谈判、整改要求等工作场景。

### 核心理念：两步闭环

**诊断环**：用L1-L4四级模型判断当前表达质量——L1纯感受无可验证信息、L2有定性无定量、L3有定量无定性、L4定性+定量融合。目标是帮助年老师从L1/L2升级到L3/L4。

**升级环**：用三步公式重构信息呈现顺序——先说对方最关心的（利益/情绪/目标，利用首因效应）、再说客观事实或核心问题、最后给解决方案或行动建议。信息呈现顺序影响沟通效果10倍差异，本质是从"自我视角"到"对方视角"的转换。

### 场景适配

对上级用方案式（目标→问题→建议）、对平级用协作式（中立→倾听→共识）、高冲突场景用情绪优先（情绪→事实→方案）。

### 工作流程

Step 0：记忆调用（润色前必须先读取相关记忆，包括供应商画像、人物档案、历史判断记录，否则输出就是模板化的废话）→ Step 1：理解场景（对谁说、说什么、要什么结果）→ Step 2：判断当前表达层级 → Step 3：润色升级（选择模板，输出口语化润色稿+层级变化+参考案例）→ Step 4：积累学习（年老师认可后保存案例到accumulated.md）。

### 向上汇报的额外检查

仅对军哥/专项汇报触发：假设暴露（军哥第一个质疑假设）、数据标准（关键数据格式为"[指标]（口径）[周期][变化]，来源：[渠道]"）、口径自检（有效性、口径一致、周期合理、来源可信）、受众适配（高层看结论+假设+拍板项，不堆细节）。

## 模板库核心内容

### 向上级汇报（对军哥）

军哥特点是穿透式追问，要数据+根因+时间线。三个典型场景模板：业绩出问题（KPI掉了X个百分点，主要卡在具体指标，原因是根因，方案是行动，预计时间追回来）、需要资源支持（业务量涨X%，产能到Y%，缺口Z%，建议方案，需要支持内容）、清退决策（连续N个月KPI在X%以下，给了整改机会效果不明显，建议清退，转移需要时间）。

### 周会汇报三维度框架

本质区别在于：时间线罗列让人看到"你干了什么活"，维度打包让人看到"你覆盖了什么战场"。三个价值支柱——日常管理（供应商正常运转吗）、风险把控（有没有隐患提前看到）、前瞻建设（在想未来的事吗）。其中前瞻建设是区分度最大的维度，多数人只写前两项不写第三项。

每个维度下的公式：动作+结果/进展+意义（可选）。确定性高的事写全三层，确定性中等的写动作+结果，刚启动的事只写动作。关键细节是"本周无重大风险"比"发现了XX风险"更有价值——前者说明持续监控但一切正常，长期坚持会建立"年老师这块我不需要操心"的信任资产。

写法要点：数字跟上上周/目标的对比而非裸数字，每个维度1-3件事最佳（1件太单薄，4+件太密记不住），写完自己念一遍超3分钟就删（3分钟是领导注意力极限）。

### 案例原话学习

模板库收录了刘乾坤（高压管理型，学形不学神）、军哥（穿透追问型）、刘伟佳（战略服务型）、王易人（精细化管控型）、秦宇凝（专业型汇报）、谭敏（进度型汇报）的原话片段，涵盖立规矩、追责转嫁、加压链、制度拷问、穿透追问、时间线施压、成果前置、目标量化、责任压实、结构化+数据+边界等典型表达模式。年老师可按场景选择适配的表达风格。

## 越用越好机制

Work-Expression 的积累文件持续收录年老师认可的润色案例，每次润色后若年老师认可则追加到 `references/accumulated.md`，格式包含原口述、润色后版本、关键改进说明。已积累的案例包括：预算申请邮件结构（三段式+五维数据对比+条件句收尾）、供应商结算优化同步（先承认问题再解释后给方案）、周会发言4条汇总（结构统一+数据整合+责任到人+时间线明确+军哥适配）、业管群沟通（核心思路→关键设计→特别说明→待讨论）、言犀商务政策解读（老板视角一句话+梯队划分）。使用次数越多，模板库越丰富，对年老师需求的理解越自然。
