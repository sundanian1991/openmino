## 概述

规划流程采用四阶段模型：头脑风暴 → 研究分析 → 工作产出 → 复盘审查。所有计划文件遵循统一编号规范和目录结构，确保可追溯、可复用。

## 四阶段工作流

### 00 — 头脑风暴

自由探索阶段，核心输出：

- **核心问题**：一句话定义要解决什么
- **事实梳理**：时间线、参与者、数据基线
- **Themes**：反复出现的主题
- **Tensions**：张力与权衡（两端对照）
- **Gaps**：信息缺口清单
- **开放性问题**：待确认的决策点

头脑风暴不追求答案，追求"问题够准、事实够全"。典型特征：表格密集、分类清晰、留白充分。

### 01 — 研究分析

基于 00 提出的问题，针对性地搜索、验证、补充。输出结构化研究报告，为后续决策提供依据。

### 02 — 工作产出

实际交付物存放于 `02-workspace/` 子目录。文件命名遵循 `序号-主题.后缀` 格式，每个文件对应一个独立交付物。

常见产出类型：

| 类型 | 示例 |
|------|------|
| 需求文档 | 最终版、草稿版 |
| 分析报告 | 动机分析、影响评估 |
| 策略清单 | 应对策略、边界识别 |
| 管理工具 | 打分模板、评估框架 |

### 03 — 复盘审查

审查维度：战略对齐、数据准确性、可证伪性、范围适当性。问题分级：P1（阻塞）、P2（需修复）、P3（可优化）。

审查结论格式：X 个 P1 + Y 个 P2 + Z 个 P3，整体判断是否进入执行阶段。

## 计划类型

| 类型 | 特征 | 代表案例 |
|------|------|----------|
| **商业研究** | 事实+框架+洞察，多层分析 | AI定价模式演变 |
| **管理设计** | 评估体系+流程+模板 | 供应商分层分级、站点看板V7 |
| **技能评估** | 候选对比+测试设计+统一场景 | 设计技能大比武 |
| **职业分析** | 人物画像+行为模式+应对策略 | 权责不对等、应对带节奏 |
| **设计系统** | 参数体系+情绪矩阵+原则提炼 | mino-frontend参数系统 |

## 决策模式

**补贴→收紧→分层**：互联网定价通用三段式（从AI定价案例提炼，可迁移至其他场景）

**从管理问题出发，不从图表出发**：可视化项目的铁律，V1→V6 演进教训（站点看板项目）

**绝对分数 vs 相对排名**：评估体系设计时的核心张力，推荐相对排名以自动适应规模变化（供应商分层案例）

**统一场景公平对比**：技能评估时同一份Brief让不同技能各自演绎，对比视觉质量和信息传达效率（设计技能比武）

## 文件命名规范

文件夹：`YYYY-MM-DD-主题描述`（日期前置，主题具体）

内部文件：
- `00-brainstorm.md` — 头脑风暴
- `01-research.md` — 研究分析
- `02-workspace/` — 工作产出
- `03-review.md` — 复盘审查

生命周期：进行中保留在 `plans/`，已完结归档到 `workspace/archive/projects/`，无后续直接删除。
