## AI 定价模式演变

**核心判断**：Coding Plan → Token Plan 的转变本质是从"卖工具使用权"到"卖算力资源包"的商业模式升级。

### 关键决策点

| 决策 | 结论 | 依据 |
|------|------|------|
| 计量单位即定价权 | 谁定义计量单位谁掌握定价权 | MiniMax/Kimi从Prompt转Token |
| 订阅费是地板 | 实际账单是标价的2-5倍 | Token超额+premium加价+agent循环 |
| 不限量不可持续 | Agent时代必须转向精细计量 | OpenClaw使Token消耗增长6-10倍 |
| 国产芯片改变格局 | 国内厂商定价可更激进 | 华为昇腾+寒武纪+平头哥三角 |

### 可复用洞察

- **计量透明度演变**：API次数 → Prompt次数 → Token/Credit → 统一Credits，方向越来越透明
- **五阶段模型**：价格战启动 → Coding Plan涌现 → 全面铺开 → 收紧分层 → Token精细计量
- **成本结构真相**：85-95%的Token是上下文开销（文件读取、工具输出、重发），代码生成仅5-15%

## 供应商分层分级体系

**核心判断**：推荐相对排名切分而非绝对分数切分，与现有赛马排名逻辑一致，自动适应供应商总数变化。

### 体系设计

| 维度 | 决策 | 理由 |
|------|------|------|
| 分级方式 | 相对排名（A前30%/B 30-70%/C后30%） | 自动适应规模变化 |
| 评分权重 | 业务督导60% + 供应商管理30% + 质检10% | 定量为主，定性为辅 |
| C级处理 | 进入PIP流程 | 与现有淘汰规则衔接 |
| 打分频率 | 待确认（月度/季度） | 开放性问题 |

### 可复用资产

5个现有资产可直接复用：5维度评分框架、资源分配评估维度、整改通知机制、清退处置流程、10大风险维度。

## 供应商站点数据汇报体系（V7）

**核心判断**：从管理问题出发构建决策工具，不是从图表出发做数据展示。

### V1→V6 教训

| 教训 | V7做法 |
|------|--------|
| 数据展示 ≠ 管理决策 | 从管理问题出发：今天关注什么、找谁聊、有什么风险 |
| 无风险信号 | 10大风险维度实时计算引擎 |
| 无交叉分析 | 职场×业务线矩阵、供应商×产品线矩阵 |
| 无地理分析 | 区域/省份/城市多维度地理集中检测 |
| 性能问题（142KB数据无法显示） | 控制文件大小，分段加载 |

### 架构设计

6大模块、15-20个视觉单元、1个风险引擎。核心新增：风险检测引擎（实时计算10个风险维度）、聚合引擎（站点×业务线合并为物理职场）、4维度联动筛选器。

### 测试方法
