## 概述

工作标准目录汇集了年老师在日常工作中沉淀的核心方法论，涵盖供应商管理的培养悖论与诊断驱动、适应穿透式管理的向上汇报规范、定性定量融合的高分辨率沟通法，以及问题诊断三层次与失败复盘机制。这些方法论是年老师作为供应商管理岗位的核心决策框架。

## 供应商管理方法论

### 培养悖论定律

**核心洞察**：培养供应商的必然路径是小 → 给资源 → 做大 → 向外扩张 → 有退路 → 议价/流失。这不是背叛，是模式的必然结果。

**应对策略**：

| 策略 | 做法 |
|------|------|
| 梯队备份 | 始终培育第二梯队，保持替代选项 |
| BPO 联盟 | 把自营培训体系交给中腰部供应商 |
| 新业务优先 | 新业务分配优先给中腰部，不让头部一家独大 |
| 差异化定价 | 头部用市场价，中腰部给成长空间 |

### 诊断驱动决策

用结构化问题替代直觉判断，逼迫性提问获取具体可验证信息。

**准入诊断 6 问**：为什么必须是这家 → 没有这家会怎样 → 谁最需要说名字 → 最小试单是什么 → 有成功案例吗 → 3 年后还需要吗

**业绩诊断 6 问**：核心指标到底怎样 → 根本原因是什么 → 是能力问题还是结构问题 → 有没有替代方案 → 短期能做什么 → 长期需要什么改变

## 向上汇报规范

### 核心结构

```
结果（含数据）→ 根因/问题 → 下一步行动 → 时间线
```

适应军哥穿透式管理的汇报结构。

### 四大要素

| 要素 | 位置 | 说明 |
|------|------|------|
| 结果 | 最先 | 数据在前，说清楚达成情况 |
| 根因/问题 | 中间 | 问题不突出 = 没问题 |
| 下一步行动 | 后面 | 具体要做什么 |
| 时间线 | 最后 | 什么时候完成 |

### 核心原则

结论先行、数据前置、问题明确、避免信息罗列。不是"做了什么"，是"结果怎样、问题在哪、下一步是什么"。

**案例**：原版（进度罗列型）"消金白条取现&借钱电销试点项目：翰锐、伽玛 325 可上线 20 人，剩余 5 人 4 月 1 日上线"。润色版（穿透式结构）"消金试点进度：上线 20 人（达成），剩余 5 人因 XXX 原因延期至 4/1（问题→解决中），伽玛同步扩容（行动）"。

## 核心工作理念

### 三问原则

每个决策前问自己：

| 问题 | 目的 |
|------|------|
| 有规可依吗？ | 不凭感觉，找依据 |
| 合理吗？ | 符合逻辑和常识 |
| 充分共识吗？ | 不是一个人说了算 |

### 边界清晰原则

先想清楚边界和职责，再填充细节。边界清晰 > 功能复杂，职责明确 > 灵活多变。

### 职责明确原则

| 维度 | 说明 |
|------|------|
| 主责 | 供应商全量管理（合规、准入、清退、分工） |
| 非负责人 | 服务组 — 供应商管理岗，上面有王易人 |
| 重点跟进 | 毅航、毛毛虫、伽玛、赛维斯、岐力、翰锐 |

## 沟通方法论

### 四个表达层级

| 层级 | 定义 | 示例 | 问题 |
|------|------|------|------|
| Level 1 | 纯感受表达 | "我觉得不太行" | 高度主观，信息密度极低 |
| Level 2 | 结构化定性表达 | "问题出在用户路径过长" | 有分析结构，但不可验证 |
| Level 3 | 定量表达 | "转化率下降了 12%" | 可验证，但缺乏解释力 |
| Level 4 | 定性+定量融合表达 | "转化率下降 12%，因路径从 5 步增至 8 步" | 有数据、有解释、有因果结构 |

**目标**：用 Level 4 表达。向上汇报、跨部门协作、供应商谈判都需要 Level 4。日常闲聊用 Level 1-2 即可。

### 表达模板

**问题汇报**：问题 + 数据 + 原因 + 建议

**复盘时补定量**：日常来不及没关系，周报/月报时强制补数据。

## 问题处理方法论

### 问题诊断三层次

```
现象 → 结构 → 根源
```

找到系统性原因，而非表面问题。核心是区分能力问题 vs 结构问题，避免简单归因。

**示例**：新人流失率高 → 结算低 → 培训投入不足 → 新人体验差 → 根源：名单结构不合理，不是人力能力问题。

### 失败复盘机制

**停止反思铁律**：修补失败 2 次 → 立刻停下来反思方向。架构问题不能靠修补解决，必须重构。

**复盘三步**：记录教训 → 提炼铁律 → 形成方法论。
