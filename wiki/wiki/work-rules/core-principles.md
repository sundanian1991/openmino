# 核心工作原则

> Sources: 年老师, 2026-02 ~ 2026-04, 2026-04-28
> Raw:[核心工作理念](../../raw/work-standards/核心工作理念.md); [其他工作铁律](../../raw/work-standards/其他工作铁律.md); [MEMORY](../../raw/core-memory/MEMORY.md)

## 概述

年老师的三套核心决策原则：三问原则确保每个决策有据可依，边界清晰原则防止职责越界，职责明确原则保证每个机制有人负责、不重叠。这三条原则共同构成了日常工作决策的底层框架。

## 三问原则

每个决策前依次自问：

| 问题 | 目的 |
|------|------|
| 有规可依吗？ | 不凭感觉，找制度和先例依据 |
| 合理吗？ | 符合逻辑和常识 |
| 充分共识吗？ | 不是一个人说了算 |

**应用示例**：供应商准入决策时，先看《供应商准入指引》（有规），再评估试错成本是否可控（合理），最后确认上级和同事是否知情同意（共识）。

## 边界清晰原则

核心：先划定边界和职责，再填充具体方案。边界清晰度优先于功能复杂度。

| 原则 | 说明 |
|------|------|
| 不是先加功能 | 是先明确范围 |
| 边界清晰 > 功能复杂 | 职责明确 > 灵活多变 |

**具体应用**：
- 设计新机制时，先问"这部分应该归谁？"
- 遇到问题时，先问"这是我的职责范围吗？"

**边界意识的现实案例**：定价权在策略组，年老师不能单边调整；名单权在策略组，不能自行分配；负责供应商稳定，但非自己决策造成流失不能背锅。

## 职责明确原则

每个机制必须有明确职责归属，不重叠、不模糊。

| 维度 | 说明 |
|------|------|
| 主责 | 供应商全量管理（合规、准入、清退、分工） |
| 非负责人 | 服务组 — 供应商管理岗，上有王易人 |
| 重点跟进 | 毅航、毛毛虫、伽玛、赛维斯、岐力、翰锐（金条头部6家，3亿+职场） |
| 团队负责 | 日常业务运营 |

## 模板驱动：让原则自动生效

原则写在纸上，模板放在手上。年老师强调"用模板约束行为，不是原则指导行为"，因为原则不执行等于没写，模板能自动约束。

**常见模板**：
- 汇报模板：四段式（结果 → 根因 → 行动 → 时间线）
- 公文模板：三段式（事实 → 分析判断 → 行动请求）
- 诊断模板：准入6问 + 业绩6问

## 机制持久化判断

判断一个机制是否合格的标准："这个机制能持续多久？"

| 机制类型 | 持续性 |
|----------|--------|
| 流程规范 | 长期有效 |
| 临时方案 | 仅本次有效 |

临时方案不写入规范，只解决当前问题。

## 拦截性规则优于描述性

规则要能自动拦截行为，而非事后建议。

| 反模式 | 问题 | 正确做法 |
|--------|------|----------|
| "尽量用可视化" | 建议式，无法执行 | "含流程符号 → 强制替换为 widget" |
| "注意格式" | 提醒式，依赖人工 | "MD 内容用引用块，禁止代码块" |
