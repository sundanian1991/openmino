# 日常工作标准

> Sources: 年老师, 2026-03 ~ 2026-04
> Raw: [日常规范](../../raw/work-standards/日常规范.md); [向上汇报规范](../../raw/work-standards/向上汇报规范.md); [沟通方法论](../../raw/work-standards/沟通方法论.md)

## 概述

覆盖日常工作中的三类核心输出：向上汇报（周报/月报）、会议产出（会议纪要）、对外沟通（供应商）。核心标准是定量+定性融合表达（Level 4），用数据+因果结构替代模糊判断。

## 沟通表达四级

沟通质量分四层，目标是用 Level 4：

| 层级 | 定义 | 示例 | 问题 |
|------|------|------|------|
| Level 1 | 纯感受 | "我觉得不太行" | 高度主观，信息密度极低 |
| Level 2 | 定性结构化 | "问题出在用户路径过长" | 有结构但不可验证 |
| Level 3 | 纯定量 | "转化率下降12%" | 可验证但无解释力 |
| Level 4 | 定性+定量融合 | "转化率降12%，因路径从5步增至8步" | 有数据、有解释、有因果 |

**场景适配**：向上汇报、跨部门协作、供应商谈判 → Level 4；日常闲聊 → Level 1-2 即可。

## 向上汇报标准

### 周报：四段式穿透结构

适配军哥的穿透式管理风格：

```
结果（含数据）→ 根因/问题 → 下一步行动 → 时间线
```

**四项核心原则**：
1. 结论先行：不罗列过程，先说结果
2. 数据前置：有数据先说数据，没数据说进度
3. 问题明确：有问题必须说清楚，藏着更危险
4. 避免信息罗列：不是"做了什么"，是"结果怎样、问题在哪、下一步"

**案例对比**：

原版：翰锐、伽玛325可上线20人，剩余5人4月1日上线

穿透式：消金试点进度：上线20人（达成），剩余5人因XXX原因延期至4/1（问题→解决中），伽玛同步扩容（行动）

### 月报：六段式

```
整体概况 → 核心指标达成 → 供应商分级分布 → 本月重点事件 → 重点工作进展 → 下月计划
```

**关键要求**：用数据证明工作价值。例如：月度提效空间约83.5小时，提效比例43%。

### 适用场景

周会汇报、月度述职、项目进度同步、问题请示。

## 会议纪要标准

### 五段式结构

```
核心思路 → 关键设计 → 特别说明 → 待讨论 → 时间节点
```

行动导向：会议结论必须转化为具体行动项，明确执行人和时间。

群沟通原则：核心先行、问题明确，发到群里让所有人看清调整项。

## 公文写作标准

### 三段式结构

```
事实部分（列表/表格）→ 分析判断 → 行动请求
```

语气按对象调整：

| 对象 | 语气 |
|------|------|
| 上级 | 正式、简洁 |
| 下级 | 柔和、指导 |
| 平级 | 协商、协作 |
| 供应商 | 告知、质询 |

每个段落开头用**关键词：**标识，支持快速浏览。

## 供应商日常沟通

**调研确认流程**：供应商调研数据 → 年老师亲自沟通后确认。确认内容包括日GMV、激励目标、核心原因、影响判断。

**现场复盘要求**：信用卡和企金供应商现场复盘，策略侧按需参与。不同业务线采用不同沟通深度，避免资源浪费。

## 复盘补定量

日常沟通来不及用 Level 4 没关系，但周报和月报时强制补数据。
