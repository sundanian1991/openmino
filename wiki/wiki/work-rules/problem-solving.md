# 问题处理方法论

> Sources: 年老师, 2026-03 ~ 2026-04, 2026-04-28
> Raw: [问题处理方法论](../../raw/work-standards/问题处理方法论.md); [供应商管理方法论](../../raw/work-standards/供应商管理方法论.md); [其他工作铁律](../../raw/work-standards/其他工作铁律.md)

## 概述

年老师处理问题的系统化方法：从诊断到根因、从复盘到预防，强调找到系统性原因而非表面问题。核心是诊断三层次、失败复盘机制、风险预防制度化。

## 一、问题诊断三层次

所有问题按三层递进分析：

```
现象 → 结构 → 根源
```

| 层次 | 说明 | 示例（新人流失） |
|------|------|----------------|
| 现象 | 看到了什么 | 新人流失率高 |
| 结构 | 背后的机制是什么 | 结算低 → 培训投入不足 → 新人体验差 |
| 根源 | 系统性原因是什么 | 名单结构不合理，不是人力能力问题 |

**关键判断**：区分能力问题 vs 结构问题，避免简单归因。

## 二、诊断驱动决策

用结构化问题替代直觉判断，获取具体可验证信息。

### 准入诊断6问

| 问题 | 目的 |
|------|------|
| 为什么必须是这家？ | 验证必要性 |
| 没有这家会怎样？ | 验证依赖度 |
| 谁最需要说名字？ | 验证需求来源 |
| 最小试单是什么？ | 验证试错成本 |
| 有成功案例吗？ | 验证可行性 |
| 3年后还需要吗？ | 验证长期价值 |

### 业绩诊断6问

| 问题 | 目的 |
|------|------|
| 核心指标到底怎样？ | 采集具体信号 |
| 根本原因是什么？ | 追根溯源 |
| 谁受影响最大？ | 识别利益相关方 |
| 能改进吗要什么？ | 评估干预可行性 |
| 去现场看过吗？ | 验证信息来源 |
| 3个月后会怎样？ | 时序思考 |

## 三、失败复盘机制

**停止反思铁律**：修补失败2次 → 立刻停下来反思方向。架构问题不能靠修补解决，必须重构。

复盘三步：记录教训 → 提炼铁律 → 形成方法论。

## 四、风险预防制度化

提前识别高风险场景：

| 场景 | 风险点 |
|------|------|
| 供应商引入 | 准入标准不明确 |
| 定价调整 | 定价权不在自己 |
| 问题追溯 | 事后规则追溯挖坑 |

**保护策略**：边界清单 + 留痕模板 + 决策速查表。关键决策必须留痕，避免事后追溯无法自证。

## 五、策略性应对

不只解决当前问题，而是重新设计机制预防复发。

> 示例：供应商议价困境 → 不只是一对一谈判，而是培育第二梯队、建立BPO联盟，从结构上降低单点依赖。

## 六、时序思考

每个决策都考虑未来趋势，而非仅当下：

| 问题 | 时序思考 |
|------|----------|
| 准入评估 | 3年后还需要吗？ |
| 业绩诊断 | 3个月后会怎样？ |
| 问题处理 | 会复发吗？如何预防？ |

## 七、借力框架沟通

直接要权/要资源往往无效。用对方KPI框架沟通：

不说"我需要定价权"，说"共同优化费效比需要决策前置"。

因果链思维：理解传导路径。结算太低 → 供应商不投入 → 产出下降 → 费效比恶化 → 陷入恶性循环。
