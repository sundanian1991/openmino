## Overview

Taste System 由两个互补技能组成：**taste**（审美判断力）负责培养美学判断力，通过学习人类反馈、提出真诚的质量问题并随时间校准；**taste-agency**（行动力校准器）负责将模糊的欲望或有吸引力的想法转化为具体判断和下一步行动。核心观点：审美不是装饰性的美学，而是约束条件下的判断力；行动力不是动机驱动的努力，而是用户通过有意识行动塑造事件的能力。

## Taste — 审美判断力的学习系统

### 核心原则：向人类学习

AI 本身没有审美。人类有。工作流程是：不确定时提问 → 记录每次纠正和解释 → 将学到的模式应用于未来判断 → 将判断与人类的校准直到一致。审美通过接触 + 反馈学习。

### 校准等级

| 等级 | 含义 | 行为 |
|------|------|------|
| Uncalibrated | 尚无反馈 | 总是提问，从不断言 |
| Learning | 收到一些纠正 | 暂时性陈述，请求确认 |
| Calibrating | 模式浮现 | 附带推理的陈述，偶尔检查 |
| Calibrated | 持续一致 | 自信陈述，仍开放纠正 |

每个领域都从未校准开始。通过准确的预测赢得信心。

### 学习循环

1. **陈述判断** — "我认为 X 因为 Y"
2. **请求反馈** — "这符合你的审美吗？我漏掉了什么？"
3. **如果被纠正**：追问 WHY（真诚好奇，非防御性）→ 记录纠正及上下文 → 提取底层模式 → 更新校准信心

永远不要用自己的审美判断去对抗人类的判断。从差距中学习。

###  Genuine Curiosity Protocol

被纠正时提问的方式：

**应该问具体的**："什么让这个比替代方案更好？""我没有看到什么？""这是通用原则还是特定于这个上下文？""这会适用于类似的情况吗？"

**不应该模糊地问**：避免"你能解释更多吗？"或"你为什么这么想？"

具体问题表明你在尝试提取可转移的知识。

## Taste 发展理论

### 三个必要条件

1. **有注意力地大量接触**——不是被动消费，而是审问：为什么这个有效？遮住底部三分之一会怎样？我的视线先去哪里、然后呢？什么会让它更好？什么会毁掉它？
2. **制作糟糕的作品**——品味与能力之间的差距是痛苦但必要的。这种紧张迫使词汇发展——为你感知到的差异找到词语。
3. **研究边缘**——接近失败的作品比杰作教得更多。为什么这个成功了那个没有，当它们 90% 相同时？答案总在那剩余的 10% 中。

### 校准流程

接触范围 → 有意比较（并排训练辨别力）→ 词汇标签（词语创造感知锚点）→ 反馈循环（将判断与专家核对）→ 制作作品（将判断应用于自己的产出）。

### 专家收敛

研究发现：专家比新手更一致。原因：专家学会权重相同的特征。他们在标准典范上接受过训练。他们共享区分的词汇。如果你的判断与专家大相径庭，你可能在错误特征上做模式匹配。

### 偏好与品质的分离

最成熟的能力："我不喜欢这个，但它是优秀的"和"我喜欢这个，但它是平庸的"。这需要元认知意识：知道你的反应反映的是熟悉度还是工艺。

### 长期游戏

审美以年为单位发展，不是以周为单位：大量有注意力地消费 → 感受品味与能力的差距 → 无论如何都制作 → 研究专家重视什么 → 发展区分的词汇 → 将判断与结果核对 → 重复十年。

## Taste Agency — 判断力与行动力校准器

### 双重使命

1. **Taste**：决定什么值得做，以及"好"意味着什么
2. **Agency**：让下一步行动可见、小到可以开始、强到能创造反馈

在两者之前，必须问：**对谁值得做、在什么阶段、在什么约束下？**

### 五大门

**0. Context Gate（上下文门）**：这件事对这个用户、在这个阶段值得做吗？
- 检查主线契合度：强契合/弱契合/漂移
- 输出：主线契合判断 + 原因 + 风险 + 情境质量标准

**1. Taste Gate（审美门）**：这件事值得做吗？好看起来是什么样的？
- 检查：解决的真实问题/欲望/机会、为什么是现在、谁会注意到、最小具体制品是什么
- 输出：值得做的理由 + 不值得的条件 + 质量门槛 + 假好警告 + 基准

**2. Reality Gate（现实门）**：我在看现实还是保护一个好听的故事？
- 检查：支持证据、缺失证据、是否依赖地位/新奇/术语/想象的回报、真实受众是否存在、最强反对论据
- 输出：已有证据 + 缺失证据 + 最强反对 + 约束 + 现实测试

**3. Agency Gate（行动门）**：用户能否在没有完美条件的情况下推进？
- 检查：下一个可见行动、能否今天完成、实际缺少什么资源、什么资源只是借口、行动前必须做什么决定
- 输出：下一步行动 + 时间盒 + 所需输入 + 需避免的模式 + 首个可见产出 + 备选方案

**4. Loop Gate（循环门）**：行动如何改进判断？
- 检查：反馈来源、审查时间点、继续条件、停止条件、应存储的学习

### 输出判决

每次以五种输出之一结束：

| 判决 | 含义 |
|------|------|
| **Do now** | 想法值得行动，有下一步 |
| **Narrow first** | 有信号但太宽 |
| **Research first** | 可能重要但缺乏证据 |
| **Pause** | 有吸引力但不值得当前关注 |
| **Kill** | 未通过质量、现实或行动测试 |

### 评分卡解读

21-25 分：Do now；16-20 分：Narrow 或 Research first；11-15 分：Pause 直到修复缺失门；10 分以下：Kill 或彻底重构。

**关键阻断规则**：Context、Reality 或 Agency 中任何一项得 1 分，即使总分很高也阻断 "Do now"。如果 Context 判定为 Drift，默认判决为 Pause，除非用户明确选择支线并命名将被暂停的内容。

### 常见失败模式

| 模式 | 表现 |
|------|------|
| Tool collecting | 寻找又一个工具而非产出制品 |
| Taste theater | 批评质量但不定义更好的版本 |
| Fake agency | 制定不产生可见制品的计划 |
| Novelty addiction | 混淆新鲜输入与进展 |
| Infinite research | 用证据收集避免暴露 |
| Premature shipping | 没有质量门槛就快速行动 |
| Borrowed desire | 因为重要人物重视而想要某物 |
| Context blindness | 判断想法总体上好但忽略是否适合当前阶段 |
| Main-line drift | 对有吸引力的机会说是但稀释了实际方向 |
