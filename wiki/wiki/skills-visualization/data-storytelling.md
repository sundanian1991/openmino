# Data Storytelling — 数据沟通策略与编排

> Sources: data-storytelling SKILL.md (2026-04-26); 《Storytelling with Data》全书 30 skill 整合, 2026-04-28
> Raw: [SKILL.md](../../raw/skills/data-storytelling-SKILL.md)

## 概述

data-storytelling 是 Mino 技能体系中的"策略与编排层"——它不画图、不做 PPT，唯一产出是一份 Markdown 大纲，定义受众、Big Idea、图表清单、每张图的呈现策略和叙事顺序。大纲完成后，交给渲染技能（chart-visualization / echarts-visualization / html-ppt-skill / presentation-skill 等）完成图表和演示文稿。

该技能整合了《Storytelling with Data》全书 30 个 skill 的方法论，形成了一套从数据理解到最终交付的完整数据沟通流程。

---

## 一、定位与核心价值

**data-storytelling 不画图、不做 PPT。** 它的核心定位是"导演"而非"演员"：

- 定义受众是谁、要他做什么（Big Idea）
- 需要几张图、每张图什么类型、为什么选它
- 每张图怎么呈现（高亮什么、排序规则、标注什么）
- 图的叙事顺序（先讲什么后讲什么）

大纲完成后，**交给渲染技能去画和组装**。这种"策略与执行分离"的设计，确保数据沟通的决策过程是可审查、可迭代的，而非一次性黑箱输出。

---

## 二、六阶段工作流

### Phase 1：数据理解与策略

**1.1 理解数据**：这份数据有什么（指标、维度、时间范围、样本量），质量如何（缺失值、异常值、口径一致性），缺什么。

**1.2 定义受众**：禁止用"所有利益相关者"、"领导"等模糊定义。必须具体到人和角色，评估信任度（权威/中立/怀疑），如果受众多元则分开做。

**1.3 Big Idea**：一句话，包含洞察 + 建议。例如不是"供应商绩效数据分析"，而是"伽玛本季度履约率下滑 15%，建议启动备选供应商引入流程"。所有后续设计选择是否服务于这个 Big Idea？不是就去掉。

**1.4 探索性 vs 解释性**：默认走解释性路线（精简 3-7 张图，只展示关键洞察），除非明确说是探索性分析。

### Phase 2：图表清单

**功能决定形式**：先回答"受众需要做什么或理解什么"，再选图。

| 功能目标 | 图表形式 |
|----------|----------|
| 看时间趋势 | 折线图 |
| 分类对比 | 条形图（水平优先） |
| 看相关性 | 散点图 |
| 看构成/占比 | 堆叠条形图（禁止饼图） |
| 1~3 个关键数字 | 大字展示，不是图 |
| 表格数据看模式 | 热力图 |
| 看流入流出+缺口 | 正负堆叠条形图 |
| 调研排名数据 | 水平堆叠条（排名分段） |

**禁止项**：饼图/环形图、3D 效果、双 Y 轴、截断条形图轴、默认多色轮播、渐变/阴影/发光。

### Phase 3：每张图的呈现策略

**高亮规则（≤10% 法则）**：只高亮 ≤10% 的元素（关键数据点/线），其余灰化 `#CDCECD`。关键数据用 `#C13531`（红色）。永远不要同时高亮多个不相关的点。

**排序规则**：分类数据按意图排序（最优/最差/未使用），不按字母。多张图讲同一分类时保持顺序不变。

**预测与实际区分**：实际用粗实线，预测用细虚线 50% 透明度，最后实际点用白色填充 + 红边框 + 粗体数字锚点。不能混成一条无区分的线。

### Phase 4：复杂图表模板

覆盖多线图解构（>4 条且重叠）、正负堆叠条形图、调研排名堆叠条等复杂场景的标准化处理方案。

### Phase 5：叙事结构与大纲输出

**三幕叙事结构**：
- 第一幕（铺垫）：设定场景、指出失衡、预览方案
- 第二幕（冲突）：用证据展开问题、对比案例、不行动的后果
- 第三幕（决议）：明确行动号召、回顾紧迫性

**排序规则**：先全局后细节、先问题后方案、先事实后推断、每页/每张图只讲一件事。

**数据伦理**：论证类沟通不得只呈现支持结论的数据，必须提及已知反对证据或局限性。

### Phase 6：独立审查

大纲写完后，派一个子代理，只给 Markdown 大纲文档，不给任何背景上下文，以"第一次看到这份汇报的受众"视角评估。

审查维度包括：Big Idea 清晰度、图表必要性、高亮合规、排序一致性、叙事连贯性、数据真实性、标题行动导向、反对证据、格式规范。

输出分级问题列表：P0（必须修复）、P1（强烈建议修复）、P2（可选优化）。

## 三、渲染技能路由

data-storytelling 产出大纲后，按每张图的"渲染技能"字段，路由到对应技能。

### 图表渲染层

| 图表类型 | 首选技能 | 为什么 |
|----------|----------|--------|
| 标准数据图 | `echarts-visualization` | 支持交互（tooltip、缩放、筛选） |
| 特殊图表 | `echarts-visualization` | chart-visualization 不支持 |
| 信息图 | `infographic-creator` | AntV Infographic，适合知识普及 |
| 技术图 | `diagram-design` | 专业技术图表，inline SVG |

### 演示组装层

| 场景 | 首选技能 | 适合 |
|------|----------|------|
| 重要演示 | `cinematic-ui` | 独特视觉风格、强叙事感 |
| 品牌级汇报 | `huashu-design` | 品牌识别度、高保真、可导出 PDF/PPTX |
| 杂志风 | `magazine-web-ppt` | 高级设计感、WebGL 流体背景 |
| 通用汇报 | `html-ppt-skill` | 多风格可选、快速生成 |
| 重动画 | `frontend-slides` | 创意视觉、每页严格 100vh |
| 演讲稿 | `presentation-skill` | 快速生成、62 种风格 |
| 仪表盘 | `mino-frontend` | 高保真、复杂交互 |

### 统一传递参数

```
颜色：['#C13531', '#293C54', '#CDCECD']
高亮：主数据 #C13531，其余 #CDCECD
标题：行动导向
排序：按意图，不按字母
标注：差距用 markLine，关键点用实心圆
背景：transparent
动画：false（演讲动画版除外）
```

## 四、快速检查清单

大纲完成后逐项检查：

- [ ] Big Idea 一句话能说清？
- [ ] 每张图的类型都有理由支撑？
- [ ] 没用禁止项？
- [ ] 每张图的高亮 ≤10%？
- [ ] 分类数据有意图排序？
- [ ] 叙事顺序符合"先全局→细节、先问题→方案"？
- [ ] 包含了反对证据？
- [ ] 每张图都指定了渲染技能和指令？
- [ ] 已走 Phase 6 独立审查？无 P0 阻塞？
