# 风险预警机制与分级响应

## 概述

风险预警机制是供应商管理的"雷达系统"。从日常监控到排名预警，从数据异常到合规风险，通过多层信号检测和分级响应，实现"早发现、早干预、早解决"。本文梳理完整的预警信号体系、触发条件、响应流程和升级路径。

## 预警信号全景图

### 九种预警状态

| 状态 | 触发条件 | 紧急程度 | 响应方式 |
|------|---------|---------|---------|
| **首次预警** | 本月落入后30%，上月不在 | 常规 | 私下沟通，记录原因 |
| **连续预警** | 连续2个月在后30% | 正式 | 正式约谈，要求整改计划 |
| **清退评估** | 连续3个月在后30% | 紧急 | 启动清退评估流程 |
| **A级骤降** | 上月前30%本月跌到后30% | **立即** | 当日联系，48h内正式约谈 |
| **回正** | 连续2个月回到前70% | — | 取消预警，归档记录 |
| **诚信预警** | 数据质量分≤70或1项不实 | 正式 | 警告+3个月加倍核查 |
| **诚信红线** | ≥2项不实或同一项连续2次不实 | **立即** | 约谈高层+记入诚信档案 |
| **FCI预警** | FCI<75 | 事件触发 | 列入观察名单 |
| **巡检危险** | FCI<60或红色项≥2 | 紧急 | 立即整改，考虑替换 |

### 不触发预警的场景

| 场景 | 说明 |
|------|------|
| 正常波动 | 排名小幅变化但未入后30% |
| 首次预警 | 仅私下沟通，不走正式约谈流程 |
| 新供应商保护期 | 准入后3个月内排名不纳入预警 |

## SLA 三级阈值体系

SLA定义"能不能接受"的底线，与评估方案独立运作。

### 核心SLA指标

| 指标 | 达标线（绿色） | 预警线（黄色） | 红线（红色） |
|------|--------------|--------------|------------|
| 日均出勤率 | ≥95% | <90% | <85% |
| 产能达标率 | ≥90% | <80% | <70% |
| 数据报送及时率 | ≥98% | <95% | <90% |
| 数据准确率 | 100% | <98% | <95% |
| 人员持证率 | 100% | <95% | <90% |
| 录音留存率 | 100% | <98% | <95% |
| 话术合规率 | ≥95% | <90% | <85% |
| 信息安全 | 0违规 | — | ≥1次严重违规 |

### SLA触发响应流程

```
指标触及预警线
    │
    ├─ 黄色预警 → 书面预警通知 → 3个工作日内提交整改计划 → 整改期30天
    │
    └─ 红色红线 → 立即约谈 → 5个工作日内决议（清退/降级/整改）
```

## 排名预警体系

### 首次预警（私下沟通）

首次预警不正式约谈，保持私下沟通调性。

**操作步骤**：

| 步骤 | 操作 | 产出 | 时限 |
|------|------|------|------|
| 1 | 确认排名数据（本月后30%，上月不在） | 数据确认 | — |
| 2 | 私下联系供应商对接人/负责人 | 沟通记录 | 排名公布后24h内 |
| 3 | 引导式提问了解原因 | 原因记录 | — |
| 4 | 记录沟通内容存档 | 沟通台账 | 当日 |

**关键原则**：
- 不替供应商想解决方案 → 用问题引导
- 不上升到正式层面 → 保持私下沟通调性
- 必须记录原因 → 存档备查（连续预警时需要对比）

### 连续预警（正式约谈）

连续2个月后30%，必须正式约谈。

**约谈框架（约25分钟）**：

| 部分 | 时长 | 内容 | 要点 |
|------|------|------|------|
| 数据呈现 | 5min | 排名、GMV达成率、核心失分项 | 用数据说话，不加情绪 |
| 引导式提问 | 15min | 用问题引导对方分析根因 | "你觉得最难的是哪一步？" |
| 要求产出 | 5min | 要求提交整改计划 | 明确截止时间 |
| 记录存档 | — | 约谈纪要双方确认 | 当日完成 |

### A级骤降（立即约谈）

上月前30%本月跌到后30%，不等连续，立即处理。

**骤降常见原因**：

| 原因类型 | 典型表现 | 责任归属 |
|----------|----------|----------|
| 名单问题 | 甲方调整名单导致产能下降 | 甲方责任 |
| 系统问题 | 系统故障影响拨号/接通 | 甲方责任 |
| 人员流失 | 大面积离职导致产能缺口 | 供应商责任 |
| 管理松弛 | 主管松懈、排班不合理 | 供应商责任 |
| 外部因素 | 节假日、政策变化 | 需具体分析 |

**关键原则**：
- 不等三个月 → 立即约谈，比连续预警更紧急
- 区分甲方责任 vs 供应商责任 → 影响后续处理方式
- 骤降原因必须当日记录 → 避免记忆偏差

## 数据真实性预警

### 数据质量分检测（5项信号）

| 信号 | 检测方式 | 风险等级 | 触发动作 |
|------|----------|----------|----------|
| 出勤率连续4周≥98% | 人力表过于平滑 | 预警 | 下次巡检重点核实 |
| 离职率连续3月=0 | 30人以上团队极少见 | 预警 | 要求提供社保缴纳证明 |
| GMV日度波动<5% | 过于平稳可疑 | 关注 | 对比系统自动数据 |
| 熟手人数突增无新入职记录 | 可能虚标熟手 | 预警 | 要求入职日期凭证 |
| 考核节点前后突变 | 月末/季末数据异常优化 | 危险 | 启动专项核查 |

### 数据质量分计算

| 触发项数 | 得分 | 状态 | 后续动作 |
|----------|------|------|----------|
| 0项触发 | 100分 | 可信 | 正常管理 |
| 1项触发 | 70分 | 存疑 | 标注具体项，下次巡检重点核实 |
| ≥2项触发 | 40分 | 不可信 | 排名加注星号，不作为奖惩依据 |

## 关键异常信号

### 自动触发的异常信号

| 信号 | 异常标准 | 动作 |
|------|----------|------|
| 出勤异常 | 有效出勤率<85% | 问供应商出勤情况 |
| 业绩断崖 | 周GMV比近4周均值下降>20% | 按A级骤降逻辑立即约谈 |
| 流失加速 | 单周离职>5% | 了解离职原因 |
| 沟通频次下降 | 群内互动减少50%+ | 主动联系了解情况 |
| 响应延迟 | 群消息回复超过30分钟 | 提醒响应要求 |

### 表演型预警识别

供应商"表面整改"的行为模式：

| 表现 | 识别方法 | 根因 |
|------|---------|------|
| 整改计划完美但执行为零 | 计划通过但每周五进度为零 | 供应商不重视或缺乏能力 |
| 短期冲刺后回落 | 整改当月数据改善，次月回落 | 临时措施而非系统性改进 |
| 动作与问题不匹配 | 整改计划动作与失分项无对应关系 | 供应商不理解根因或敷衍 |
| 数据异常优化 | 考核节点前后数据突然变好 | 可能数据造假 |

## 预警升级路径

### 完整升级链路

```
正常管理
    │
    ├─ 首次预警 → 私下沟通 → 记录原因
    │
    ├─ 连续预警 → 正式约谈 → 整改计划 → 进度跟踪 → 验收
    │                                      │
    │                    验收通过 ─────→ 取消预警 ✓
    │                    验收部分通过 → 延长观察1月 → 复验
    │                    验收未通过 → 重新提交整改计划
    │                                      │
    │                    连续2次未通过 → 清退评估
    │
    ├─ A级骤降 → 立即约谈 → 区分责任 → 针对性处理
    │
    └─ 诚信预警 → 警告+加倍核查 → 信任重建或升级清退
```

### 升级决策矩阵

| 当前状态 | 持续月数 | 升级动作 | 审批人 |
|---------|---------|---------|--------|
| 首次预警 | 1月 | 私下沟通 | YL自行处理 |
| 连续预警 | 2月 | 正式约谈+整改 | YL执行，YR知会 |
| 整改未通过 | 2次 | 重新提交整改计划 | YL执行 |
| 连续2次未通过 | — | 清退评估 | YR审批，HJ终审 |
| 诚信预警 | 3月无改善 | 约谈高层 | YL+YR |
| 诚信红线 | — | 直接清退 | HJ终审 |

## 预警管理的设计原则

### 早发现，早干预

预警机制的核心价值不是"事后追责"，而是"事前预防"。首次预警的私下沟通设计，就是为了给供应商"悄悄改正"的机会。

### 分级响应，不一刀切

不同预警状态对应不同的响应方式。首次预警私下沟通、连续预警正式约谈、A级骤降立即处理——根据紧急程度匹配响应力度。

### 引导式干预，不替供应商想方案

预警约谈的核心手法是引导式设问。不是"你应该怎么做"，而是"你觉得问题在哪"、"你打算怎么解决"。让供应商自己承诺、自己制定计划、自己执行。

### 预警有记录，全程可追溯

每次预警、沟通、约谈、整改都必须记录存档。这不仅是管理需要，也是清退评估时的证据链。
