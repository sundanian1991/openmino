## 概述

供应商定价调整委员会是京东电销BPO供应商管理的核心决策机制。从"事后救火"转向"事前防火"，通过委员会制衡、多层决策、触发式复审，确保定价调整有约束、有温度、有复盘。本文系统梳理定价委员会的组织架构、决策流程、参考原则、风险防控和供应商反馈通道。

## 定价委员会的演进背景

### 为什么需要定价委员会

在定价委员会建立之前，定价调整存在三大问题：

| 问题 | 表现 | 后果 |
|------|------|------|
| **单点决策风险** | 策略组单方决策，缺少多角度评估 | 被供应商挑战、被业务督导质疑 |
| **事后救火模式** | 定价调整引发坐席离职、供应商投诉后才应对 | 大量时间花在"灭火"而非"防火" |
| **缺乏复盘机制** | 调整后没有系统性的效果追踪 | 无法判断决策质量，同样的错误重复发生 |

### 从V1.0到V1.1的核心变更

| 章节 | V1.0 | V1.1 |
|------|------|------|
| 决策机制 | 5人委员会表决（3/5或4/5通过） | C2/C3决策（不再表决） |
| 适用范围 | 所有业务 | 成熟赛道成熟业务 |
| 基本原则 | 红线 | 参考原则 |
| 复审机制 | 无 | 新增触发式复审 |
| 会议时间 | 每月第二周周三 | 每月最后一周（27号） |
| 会议形式 | 全员1小时 | 核心人员参会，按需邀请 |
| 预警信号 | 4项通用指标 | 优化为4项（更聚焦） |

V1.1的核心转变是从"民主表决"到"分层决策"，将决策权集中在C2/C3层面，其他委员提供专业意见但不参与表决。这一变更基于策略组沟通会的反馈，提高了决策效率。

## 委员会组织架构

### 委员会构成

| 角色 | 人员 | 主要职责 |
|------|------|----------|
| 秘书长 | 供应商管理岗（1人） | 组织会议、收集材料、初审提案、追踪落地、归档记录；收集供应商反馈 |
| 委员（专业意见） | 业务督导代表（1人） | 坐席层面信息（收入增减对业务的影响） |
| | 量化代表（1人） | 数据测算、ROI分析 |
| | 服务组负责人 C3（1人） | 业务运营、职场管理 |
| | 策略组负责人 C3（1人） | 定价策略、规则设计 |
| 终审 | 数据科技业务部负责人 C2 | 每项调整的最终审批 |

### 各角色职责详解

**秘书长（供应商管理岗）**：
- 会前：收集提案材料，初审材料完整性，协调业务督导评估承受力，协调量化复核测算逻辑
- 会中：主持会议，汇报初审意见，汇总争议焦点
- 会后：发送会议纪要（24小时内），追踪落地效果，收集供应商反馈，归档记录

**业务督导**：
- 代表坐席声音，提供坐席层面影响评估
- 评估定价调整对坐席收入、离职率的影响
- 参与复盘，提供一线反馈

**量化代表**：
- 提供数据测算、ROI分析、影响评估
- 测算费效比偏离度
- 在复盘时提供实际值 vs 预测值对比

**服务组C3**：
- 评估业务运营影响，协调职场安排
- 与策略组C3协商达成共识
- 在C2无法直接决策时，参与C3协商

**策略组C3**：
- 提供定价策略建议，设计调整方案
- 发起定价调整提案
- 在C2无法直接决策时，参与C3协商

**C2（终审）**：
- 每项调整的最终审批，掌握最终决策权
- 可以"直接决策"或"转C3协商"
- C3未达成共识时，单独裁决

### 回避原则

委员与调整涉及供应商有关联的，应主动回避。这是确保决策公正性的基础要求。

## 成熟业务定义与适用范围

### 纳入机制的业务

定价委员会机制仅适用于成熟赛道的成熟类业务，不纳入的场景另行处理。

**成熟赛道**：运营6个月以上、有稳定历史数据的业务线

**成熟类业务**（须同时满足）：

| 条件 | 标准 |
|------|------|
| 供应商数量 | ≥3家 |
| 坐席规模 | ≥500人 |
| 月营收 | ≥500万元 |
| 历史调整次数 | ≥2次 |

### 不纳入机制的场景

以下场景按"特殊调整"流程处理，简化评审（仅秘书长+C2审批）：

| 场景 | 说明 |
|------|------|
| 首次定价 | 新供应商、新业务线的首次定价 |
| 测试型项目 | 试点期、验证期的业务 |
| 创新型业务 | 无历史数据参考的业务 |

## 参考原则

参考原则是定价调整的"软约束"，违反会触发复审，但特殊情况可以在提案中说明理由，经C2/C3批准后执行。

| 原则 | 说明 | 触发复审条件 |
|------|------|---------|
| **费效比偏离度** | 调整前需测算费效比，复盘时对比实际值 | 连续3次实际值超出预测值±15% |
| **职场生存线** | 坐席平均薪酬不得低于当地最低工资2.5倍 | 连续2次低于 |
| **调整频率** | 同一业务线，12个月内调整不超过3次 | 超过即触发 |
| **调整幅度** | 单次调整幅度原则上不超过15% | 超过20%即触发 |

### 参考原则 vs 红线的区别

V1.0中这些是"红线"（不可违反），V1.1改为"参考原则"（违反触发复审）。这一变更基于策略组的反馈——过于僵化的红线限制了灵活性。

**为什么改为参考原则**：
- 市场变化可能需要在短期内做出较大调整
- 不同业务线的"底线"不同
- 特殊情况下，经过充分论证的调整应当被允许
- 复审机制提供了事后校验的保障

## 定价调整流程

### 六阶段流程概览

```
阶段1：提案（T-5天）
  ↓
阶段2：初审（T-2天）
  ↓
阶段3：C2/C3决策（T-1天）
  ↓
阶段4：生效后宣贯
  ↓
阶段5：复盘（+30天）
  ↓
阶段6：复审（触发式）
```

### 阶段1：提案（T-5天）

**提案人**：策略组或量化代表

**提案材料结构**：

| 章节 | 内容 |
|------|------|
| 调整概述 | 调整类型、幅度、时间、影响范围 |
| 调整原因 | 市场变化、成本变动、业务需求等 |
| 数据支撑 | 量化分析、对比数据、趋势分析 |
| 影响测算 | 对供应商、坐席、业务的影响预测 |
| 配套措施 | 如有，说明利好绑定、过渡方案等 |

**材料要求**：
- 简洁清晰，不超过5页
- 核心数据加粗标红
- 必须有"调整前后对比"表

### 阶段2：初审（T-2天）

**审查人**：秘书长（供应商管理岗）

**审查要点**：

| 审查项 | 标准 |
|--------|------|
| 材料完整性 | 提案材料5个章节齐全 |
| 参考原则符合性 | 调整幅度、频率是否符合参考原则 |
| 收入底线 | 调整后坐席收入是否不低于职场生存线 |
| 配套措施 | 是否有缓解负面影响的配套措施 |
| 供应商反馈 | 是否收集了供应商的初步意见 |

**协同审查**：
- 业务督导评估承受力
- 量化复核测算逻辑

**初审结果**：
- 材料齐全、符合参考原则 → 报C2决策
- 材料不全或明显违反参考原则 → 退回提案方补充

### 阶段3：C2/C3决策（T-1天）

**决策流程图**：

```
秘书长提交初审材料
    ↓
C2审阅
    ↓
┌─────────────────┐
│ C2能否直接决策？ │
└─────────────────┘
    ├─ 能 → C2直接决策（通过/驳回）
    └─ 否 → 转C3协商
              ↓
        服务组C3 + 策略组C3协商
              ↓
        ┌─────────────────┐
        │ C3能否达成共识？ │
        └─────────────────┘
            ├─ 能 → 回传C2确认（通常通过）
            └─ 否 → 回传C2裁决（附双方意见）
```

**C3协商**：
- 服务组C3和策略组C3需要就调整方案达成共识
- 未达成共识时，双方意见和争议焦点一并提交C2
- C2单独裁决：通过/驳回/要求补充材料

### 阶段4：生效后宣贯

定价调整生效后，秘书长需要跟踪以下信息：

| 宣贯内容 | 目的 |
|----------|------|
| 供应商对其员工的同步情况 | 了解信息传递是否到位 |
| 员工层面的反馈和意见 | 识别负面情绪和反弹 |
| 是否存在较大反弹 | 判断是否需要应急预案 |
| 供应商的应对措施 | 评估供应商管理能力 |

### 阶段5：复盘（生效后30天）

**输出**：《定价调整复盘报告》

**复盘重点**：

| 复盘项 | 内容 |
|--------|------|
| 费效比实际值 vs 预测值 | 判断测算准确性 |
| 坐席平均薪酬是否低于底线 | 检验职场生存线 |
| 供应商反馈汇总 | 收集执行层面的意见 |
| 离职率变化 | 评估对团队稳定性的影响 |

### 阶段6：复审机制（触发式）

**触发条件**（满足任一即触发）：

| # | 触发条件 | 说明 |
|---|---------|------|
| 1 | 连续3次调整不符合参考原则 | 费效比偏离度、职场生存线、调整频率 |
| 2 | 单次调整幅度超过20% | 超出常规调整范围 |
| 3 | 供应商提出书面异议且被2名以上委员支持 | 供应商反馈通道被实质性触发 |
| 4 | 早期预警信号触发 | 见风险防控章节 |

**复审结果**：

| 结果 | 说明 |
|------|------|
| 暂停调整 | 恢复至调整前价格，直至复审通过 |
| 撤销调整 | 本次调整无效 |
| 通过 | 予以追认 |

## 会议机制

### 会议安排

| 项目 | 标准 |
|------|------|
| 固定会议日 | 每月最后一周（通常27号） |
| 会议时间 | 14:00-15:00（1小时） |
| 必须参会 | 秘书长、服务组C3、策略组C3 |
| 按需参会 | 业务督导代表（涉及坐席收入时）、量化代表（涉及数据测算时） |

### 会议议程（60分钟）

| 环节 | 时间 | 内容 |
|------|------|------|
| 提案陈述 | 10分钟 | 策略组/量化代表陈述 |
| 委员询问 | 20分钟 | 委员提问、提案方回答 |
| C2/C3决策 | 30分钟 | C2决策或C3协商 |

### 会议纪要

**发送时间**：会后24小时内

**发送范围**：全体委员+C2

**纪要内容**：
- 提案概要
- 决策结果（C2直接决策/C3协商结果）
- 待办事项及责任人

## 风险防控

### 早期预警信号

| 信号 | 预警线 | 响应 |
|------|-------|------|
| **供应商集中度反馈** | >50%供应商集中反馈定价问题 | 秘书长组织专项调研 |
| **坐席收入偏离度** | >30%坐席收入低于底线价格 | 暂停调整，重新测算 |
| **离职率异常** | >行业平均2倍 | 秘书长组织调研 |
| **投诉率上升** | >20%环比 | 暂停，重新评估 |

### 预警数据来源

| 数据 | 提供方 |
|------|--------|
| 供应商月度反馈 | 秘书长收集 |
| 坐席收入报表 | 量化提供 |
| 离职率报表 | 业务督导提供 |

### 应急预案

**触发条件**：群体性事件、投诉率>50%上升、离职率>20%

**响应**：24小时内紧急会议→决策暂停/调整/继续

## 供应商反馈通道

### 异议申请

定价调整生效后，供应商可以提交异议申请，包含：

| 内容 | 说明 |
|------|------|
| 影响论证 | 定价调整对供应商经营的具体影响 |
| 数据佐证 | 成本测算、利润变化、人员影响等量化数据 |
| 替代建议 | 供应商提出的替代方案 |

### 处理流程

| 步骤 | 时限 | 内容 |
|------|------|------|
| 接收申请 | 生效后7天内 | 供应商提交书面申请 |
| 初审 | T+2 | 秘书长初审材料完整性 |
| 委员会评估 | T+4 | 评估异议合理性 |
| 反馈结果 | T+4内 | 书面反馈处理结果 |

## 电销BPO定价模式参考

### 市场定价模式对比

| 收费模式 | 参考价格 | 适用场景 | 优势 | 注意事项 |
|----------|---------|---------|------|---------|
| **按坐席数量** | 每席每月6000-7000元 | 业务量稳定、产出均匀 | 成本固定，易于预算管理 | 坐席利用率低会导致单价成本上升 |
| **按通话时长** | 按实际通话时长计费 | 咨询、客服类 | 按实际使用付费 | 需监控通话效率和质量 |
| **按业绩提成** | 放款金额的1%-2% | 销售结果导向型 | 风险共担，激励性强 | 需平衡高利润产品与整体产品线 |
| **混合模式** | 基础坐席费+提成比例 | 最常用 | 保障基本成本+激励 | 需明确约定服务标准和提成计算 |

### 供应商成本构成（单坐席/月）

| 成本大项 | 成本细项 | 参考估算（月/席） |
|----------|---------|-------------------|
| **直接人员成本** | 坐席基本工资 | 2,500-4,000元 |
| | 坐席绩效提成 | 按实际业绩 |
| | 社保公积金（单位） | 约为基数的30%-40% |
| | 福利费 | 300-800元 |
| **基础设施与运营** | 场地租赁费分摊 | 300-800元 |
| | 水电及物业费 | 100-200元 |
| | 办公设备折旧 | 150-300元 |
| | 通讯及网络费 | 500-1,500元 |
| | IT系统及软件费 | 200-600元 |
| **管理与支持** | 管理人员薪资分摊 | 600-1,200元 |
| | 招聘与培训费 | 200-500元 |
| | 员工流失成本 | 100-400元 |
| **利润** | 毛利润 | 总成本的10%-20% |

### 社平工资倍数法

单个坐席月成本 ≈ （当地社平工资 × 倍数A）+ 绩效提成

| 倍数A | 含义 |
|-------|------|
| 1.5-1.8倍 | 仅覆盖基本工资、社保和少量管理成本，利润较薄 |
| 2.0-2.5倍 | 较为常见且合理，覆盖综合成本+合理利润 |
| >2.5倍 | 高难度项目、多语言支持、品牌溢价较高的外包服务商 |

## 定价委员会的设计哲学

### 从"单点决策"到"有支撑的决策"

定价委员会的核心价值不是"民主表决"，而是"信息完整"。在C2做最终决策之前，已经获得了业务督导（坐席声音）、量化（数据测算）、服务组（运营影响）、策略组（策略视角）的多维信息。

### 分层决策 vs 全员表决

V1.0采用5人委员会表决（3/5或4/5通过），V1.1改为C2/C3分层决策。这一变更基于实际运行反馈：

- 全员表决导致"每个人都有一票否决的错觉"，讨论冗长
- 分层决策明确了"谁说了算"，提高了决策效率
- C2终审权不变，委员会只是前置评估

### 有温度：保障坐席收入底线

"职场生存线"（坐席平均薪酬不低于当地最低工资2.5倍）是定价调整的底线思维。定价不只是甲方和供应商之间的博弈，更关系到一线坐席的切身利益。

### 有复盘：从"事后救火"到"事前防火"

每次调整后30天复盘，对比预测值与实际值。连续3次偏离±15%触发复审。这是"学习型组织"的体现——通过复盘不断提升决策质量。

## 定价委员会的演进方向

- **收入底线标准正式化**：从临时标准转为基于行业对标、供应商访谈、历史数据的正式标准
- **系统固化**：将提案、初审、决策、复盘流程嵌入系统，减少人工操作
- **供应商反馈数据分析**：对供应商异议申请做趋势分析，识别系统性问题
- **AI辅助测算**：用AI辅助费效比测算、影响评估，提高测算效率和准确性
