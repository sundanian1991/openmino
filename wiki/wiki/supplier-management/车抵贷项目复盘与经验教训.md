# 车抵贷项目复盘与经验教训

> Sources: mino, 2026-04-28
> Raw:../../raw/docs-supplier/历史项目-车抵贷复盘.md

## 概述

本文基于车抵贷项目复盘会议的完整录音转写文本，提取了项目中的关键问题、决策过程和经验教训。车抵贷是京东金融电销业务的新产品线，由两家供应商（车晓/阅文）承接，在上线初期遇到人力流失、业绩下滑、沟通机制等问题。军哥（卞海军）主持了复盘会议，对项目管理中的问题进行了深入剖析。

## 一、项目背景

### 1.1 业务概况

车抵贷是京东金融电销业务中的车抵押消费贷款产品，通过BPO供应商进行电话营销触达目标客户。项目初期由两家供应商承接：

| 供应商 | 职场 | 上线人力 | 特点 |
|--------|------|---------|------|
| 车晓 | 合肥职场 | ~25人 | 原有供应商，节后流失严重 |
| 阅文 | 新引入 | ~25人 | 新供应商，有中联产品3年合作经验 |

**目标**：50人规模，两家各25人，平分数据。

### 1.2 问题触发

- 业绩从第8周（年后）开始显著下滑
- 老人节后流失严重，新人占比越来越高
- 产线（中联）对业绩不满，考虑将业务切回给中乾
- 沟通机制不畅：产线直接决定上人数量，未与供应商管理方充分沟通

---

## 二、核心问题分析

### 2.1 人力管理问题

**老人流失**：
- 节后老人流失严重，职场每次流失都说"过几天就回来"
- 实际上人已流失但未及时透传给京东方
- 内部私下联系老人，确认"压力太大不想回来"
- 职场没有做好人员保留工作

**新人扩张过快**：
- 产线（零犀）直接拍目标要上50人
- 供应商管理方提出压力大的担忧，但被业务压力压过
- 先上30人，又说不够，又引入阅文到50人
- 新人占比越来越高，整体业绩被稀释

**军哥原话**：
> "你业绩保持不住，你干嘛扩那么多人啊？"
> "该上多少人，我们自己一点主见都没有吗？"

### 2.2 沟通机制问题

**三层沟通断裂**：

| 层级 | 问题 | 后果 |
|------|------|------|
| 产线↔供应商管理 | 产线直接决定上人数量 | 供应商管理方被动执行 |
| 供应商管理↔供应商 | 职场隐瞒人员流失信息 | 错误的决策依据 |
| 供应商↔坐席 | 老人流失未及时挽留 | 人才流失不可逆 |

**关键发现**：车晓供应商知道老人已流失，但"这个消息没有透传给我们"，导致京东方基于错误信息做决策。

### 2.3 业务逻辑问题

**名单与人力不匹配**：
- 商务约定最多拿中联20%的数据
- 上到50人后实际占比要30%多
- 中联因此压数据，只给20%但要支撑50人
- 人均名单被摊薄，转化效率受影响

**系统复杂性**：
- 中联系统较复杂，熟手录件需10-15分钟
- 新人可能需要2小时才能完成一个驾照全部录入
- 新人不敢边录件边打电话，怕给客户录错

### 2.4 产能爬坡问题

**爬坡周期**：
- 车晓新人爬坡周期：预计2个月（8周）
- 阅文新人爬坡周期：预计1个月（因有中联3年经验）
- 军哥最大可接受周期：4-6周

**军哥质疑**：
> "你怎么还需要两个月呢？你之前上线的是多久啊？"
> "你沉淀这个东西有什么用？那不就是昨天问的吗，你到底有没有沉淀？"
> "如果说你之前两个月，你现在还是两个月，那你有什么沉淀？"

## 三、决策与行动计划

### 3.1 立即行动（本周内）

| 行动 | 责任人 | 时限 |
|------|--------|------|
| 全部坐席盘点 | 浩条 | 周一完成 |
| 约供应商老板来京 | 浩条 | 下周一二 |
| 业绩目标重拆 | 浩条+大年 | 本周内 |

### 3.2 坐席盘点要求

对两家供应商所有上线坐席进行全面盘点，建立ABCD分级：

| 评估维度 | 说明 |
|---------|------|
| 同业经验 | 是否做过类似车抵贷业务 |
| 历史产能 | 过去每月产能是多少（50万/100万/600万） |
| 历史薪酬 | 过去收入水平，验证定价合理性 |
| 当前表现 | 上线后的实际转化数据 |

**处置原则**：
- D/C级：先淘汰
- A/B级：保留培养
- 不合适的人直接砍掉，不是换，是砍

### 3.3 供应商管理策略调整

| 调整项 | 内容 |
|--------|------|
| 名单调配 | 从车晓向阅文/中乾倾斜 |
| 赛马机制 | 建立名单调配的赛马机制 |
| 供应商储备 | 做好3家备选供应商储备 |
| 收规模 | 先把业绩打上来，后续再扩人 |

### 3.4 长期改进

| 改进项 | 内容 |
|--------|------|
| 方法论沉淀验证 | 用新人快速成长验证沉淀的方法论是否有效 |
| 人员管理手段 | 建立穿透管理，不依赖供应商主管汇报 |
| 业务储备 | 市场上不缺车抵贷供应商，做好入池储备 |
| 扩项管理 | 业绩稳定后再扩人，不在波动期扩规模 |

## 四、经验教训

### 4.1 人力管理教训

| 教训 | 说明 |
|------|------|
| **不能盲目扩人** | 业绩不稳定时扩人只会稀释产能 |
| **基本盘要稳** | 先有稳定的老人基本盘，再按1带2比例扩新人 |
| **人员信息必须透明** | 供应商隐瞒流失信息是致命问题 |
| **新人爬坡周期要可控** | 最多4-6周，2个月说明方法论沉淀不到位 |

### 4.2 沟通机制教训

| 教训 | 说明 |
|------|------|
| **不能被产线牵着走** | 供应商管理方要有自己的判断和主见 |
| **信息必须双向透传** | 职场流失信息必须及时上报 |
| **决策要有数据支撑** | 上多少人要基于数据和转化效率，不是拍脑袋 |

### 4.3 供应商管理教训

| 教训 | 说明 |
|------|------|
| **不能太信任供应商** | "你说的话你真的也得打着听，不能太信任他" |
| **人员质量必须验证** | 要查工资流水、历史业绩、简历 |
| **赛马机制要早建立** | 多家供应商必须有赛马和名单调配机制 |
| **供应商储备要持续** | 市场上不缺供应商，随时有备选 |

### 4.4 方法论沉淀教训

| 教训 | 说明 |
|------|------|
| **沉淀必须可验证** | 方法论不是文档，是能缩短新人爬坡周期的能力 |
| **自建职场可参考** | 自建职场一个月就能追上来，供应商也应该能做到 |
| **培训体系要有效** | 两个月爬坡说明培训体系有问题 |

---

## 五、关键洞察

1. **核心矛盾**：产线要规模 vs 供应商管理方要稳定，沟通机制断裂导致决策失误
2. **根本原因**：供应商隐瞒人员流失信息 + 京东方缺乏穿透管理能力
3. **最大风险**：产线可能将车抵贷业务切回给中乾，前期努力白费
4. **改进方向**：收规模、砍尾部、建赛马、做储备、验沉淀
5. **军哥定调**：最多接受4-6周爬坡，追不上来就砍人收数据
