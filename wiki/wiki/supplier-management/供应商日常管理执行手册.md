## 概述

本文整合了供应商日常管理执行手册的12份文件：总览、监控扫描、预警约谈、整改闭环、供应商沟通、分量决策、巡检管理、向上汇报、新供应商跟进、数据真实性防控、HRBP式扶持以及协作矩阵。这是一套面向供应商管理岗的日常操作手册，将已有制度文件转化为"拿来就用"的操作指南。

## 一、手册定位与整体架构

### 1.1 定位

**不是什么**：不是重写制度文件（管理办法、运营要求V5.0、全生命周期指引V2.1保持不动）。

**是什么**：把已有积累变成"拿来就用"的操作手册。每个模块统一结构：触发条件 → 操作步骤 → 工具/模板 → 协作角色 → 输出物。

### 1.2 使用场景

| 场景 | 做法 |
|------|------|
| 每天打开 | 看"今日扫描"检查清单，确认有无异常信号 |
| 周度例行 | 看监控扫描，按3信号扫描 |
| 月度排名后 | 看预警约谈，按排名结果触发对应流程 |
| 遇到具体问题 | 按模块索引直接跳转到对应文件 |

### 1.3 整体架构

```
触发信号（数据/排名/异常）
    │
    ├─ 周度扫描 ──→ 01-监控扫描
    │                    │
    │              发现异常？
    │              是 ↓        否
    │         02-预警约谈     记录正常
    │              │
    │         03-整改闭环（如需）
    │              │
    │         04-供应商沟通（贯穿全程）
    │
    ├─ 月度周期 ──→ 05-分量决策 → 06-巡检管理 → 07-向上汇报
    │
    └─ 事件触发 ──→ 08-新供应商跟进 / 09-数据真实性防控 / 10-HRBP式扶持
```

## 二、10 模块详解

### 2.1 模块01：监控扫描（周度·高优先级）

**触发条件**：每周一自动执行

**三信号扫描**：
1. **数据信号**：产能、出勤、质检分数环比变化
2. **排名信号**：月度排名变化（上升/下降幅度）
3. **异常信号**：人员流失、客诉升级、违规事件

**输出物**：《周度扫描报告》（记录正常或标记异常）

### 2.2 模块02：预警约谈（月度/事件·高优先级）

**触发条件**：
- 月度评估D级（60-69分）
- 连续2个月C级（70-79分）
- 质检月均分70-79分
- 人员离职率超标（≥3人或≥10%）

**约谈流程**：
1. 下发预警通知（评估结果公布当日）
2. 供应商提交整改计划（3个工作日内）
3. 约谈会议（供应商商务负责人+站点负责人）
4. 记录约谈结论和整改承诺

**输出物**：《预警约谈记录》

### 2.3 模块03：整改闭环（事件·高优先级）

**触发条件**：预警约谈后需要整改

**闭环流程**：
1. 下发整改通知书（明确整改项、标准、时限）
2. 供应商制定整改计划（3个工作日内）
3. 整改实施（整改期30天）
4. 整改期间周报（每周）
5. 整改期满复评（第30天）
6. 验证通过/不通过

**验证标准**：整改项逐项对照验收，不笼统。

**输出物**：《整改闭环报告》

### 2.4 模块04：供应商沟通（日常·高优先级）

**沟通场景**：
- 日常数据对接
- 异常事件通报
- 排名结果反馈
- 政策/制度变更通知
- 专项问题讨论

**沟通原则**：
- 先定性后定量：先一句话总结核心信息，再给数据
- 有态度：不是冷冰冰的通知，有鼓励有关心
- 有行动：明确下一步要做什么、什么时间完成
- 有记录：所有重要沟通必须留痕

**输出物**：沟通记录存档

### 2.5 模块05：分量决策（月度·中优先级）

**触发条件**：月度排名后

**决策依据**：
- 月度排名结果
- 产能利用率
- 集中度管控要求
- 业务需求量变化

**决策流程**：
1. 汇总月度排名和产能数据
2. 计算分量调整建议
3. 按审批流审批（<50%: C3审批; ≥50%: C3→C2）
4. 执行分量调整

**输出物**：《分量调整通知》

### 2.6 模块06：巡检管理（月度/事件·中优先级）

**触发条件**：
- 月度例行巡检
- 异常事件触发专项巡检
- 新站点设立验收

**巡检内容**：
- 职场物理合规（标识/门禁/监控）
- 人员持证率
- 数据安全管理
- 质量管理制度执行情况

**输出物**：《巡检报告》

### 2.7 模块07：向上汇报（周/月/事件·中优先级）

**汇报类型**：
- **周报**：本周异常信号和处理进展
- **月报**：月度排名、分量调整、整改情况
- **事件报告**：重大违规/风险事件即时报告

**汇报对象**：服务组负责人（C3）→ 数据科技业务部负责人（C2）

**输出物**：周报/月报/事件报告

### 2.8 模块08：新供应商跟进（6个月里程碑·低优先级）

**触发条件**：新供应商上线

**里程碑跟踪**：
| 阶段 | 时间 | 核心目标 | 量化检查点 |
|------|------|---------|-----------|
| 引入期 | M1-M2 | 出勤稳定、合规达标 | 出勤率≥85%；无重大合规事件 |
| 成长期 | M3-M4 | 产能爬坡 | GMV达中位数60% |
| 稳定期 | M5-M6 | 排名进入前70% | 月度排名前70% |
| 评判点 | M6 | 留或退 | 未达前70%→评估延长或清退 |

**输出物**：《新供应商里程碑跟踪表》

### 2.9 模块09：数据真实性防控（持续·低优先级）

**防控重点**：
- 虚报出勤
- 虚报产能
- 篡改数据
- 录音造假

**防控手段**：
- 系统数据与手工报表交叉验证
- 随机抽查通话录音
- 异常数据波动自动告警
- 数据报送责任人签字确认

**红线**：数据造假 = 立即停岗+清退评估

### 2.10 模块10：HRBP式扶持（事件·低优先级）

**触发条件**：供应商出现持续性能力问题

**扶持方式**：
- 派驻辅导：京东方人员驻场指导
- 标杆配对：优秀供应商与落后供应商结对
- 专项培训：针对性培训（管理/质检/培训）
- 资源对接：引入外部资源（招聘渠道/培训体系）

**退出条件**：供应商能力恢复，连续2个月达标

## 五、关键洞察

1. **拿来就用**：每个模块统一"触发条件→操作步骤→工具/模板→协作角色→输出物"结构，打开就知道该做什么
2. **三信号扫描**：数据信号+排名信号+异常信号，周度自动扫描
3. **整改有标准**：整改项逐项对照验收，不笼统
4. **协作有边界**：RACI矩阵明确谁执行/谁审批/谁协作/谁知会
5. **分层管理**：周度扫描（监控）→月度排名（决策）→事件触发（应对），三个层级覆盖全部场景
