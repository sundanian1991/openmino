## 概述

赛马竞争机制是供应商管理的核心驱动力。排名不是目的，目的是通过相对竞争驱动供应商持续投入。但排名本身只能告诉管理者"谁好谁差"，不能告诉"为什么好/差"和"怎么改善"。排名诊断体系通过产能双维度、FCI能力指数、主管胜任评估等工具，从"结果排名"深入到"能力诊断"。本文系统梳理赛马机制、排名文化和诊断工具体系。

## 赛马竞争机制

### 核心逻辑

供应商管理的本质是通过相对竞争驱动持续投入。排名是竞争的结果，是分量分配、新业务优先权、清退评估的核心依据。

### 排名切分规则

| 等级 | 排名区间 | 管理策略 |
|------|----------|---------|
| 头部（A级） | 前70% | 增份额、优先新业务、公开表扬 |
| 中腰部（B级） | 30-70% | 份额稳定、设定更高目标 |
| 尾部（C级） | 后30% | 关注、预警、整改触发 |

### 长期应用规则

| 情形 | 后果 |
|------|------|
| 连续3个月A级 | 续约优先权、新业务优先权 |
| 连续2个月D级 | 升级为E级处理 |
| 连续3个月E级 | 启动清退流程 |
| 全年平均分末位 | 合约到期不续约 |

### 赛马机制通知模板

**核心要素**：
- 排名结果：列出各供应商排名、得分、关键指标
- 分量调整：明确增减幅度、生效时间
- 调整依据：引用排名数据、四维度评分
- 鼓励性语言："排名是公开透明的，规则大家都清楚"

### 赛马机制的设计原则

**排名优先于分数**：88分和89分没有本质区别，关键在于88分可能是第二名，也可能是倒数第二名。

**末位优先调整**：原则上谁排名最后，就去做相关调整。制度设计层面可以直接退掉业务，但实际操作中温和执行。

**尾部快速汰换**：合作>12个月+连续2季度后30%+日均出勤<30人，1个月观察期后汰换。

## 排名诊断体系

### 排名诊断的三层架构

```
第一层：结果排名（谁好谁差）
    ↓
第二层：产能双维度（为什么排名变了）
    ↓
第三层：FCI+主管胜任（怎么改善）
```

### 第一层：结果排名

月度综合评估100分制，四大维度：

| 维度 | 满分 | 核心指标 |
|------|------|---------|
| 交付质量 | 40分 | 质检月均分 |
| 业务达成 | 30分 | 出勤率(10)+产能(10)+报送及时(5)+准确率(5) |
| 合规表现 | 20分 | 信息安全(8)+持证率(6)+报备合规(3)+录音留存(3) |
| 管理配合 | 10分 | 异常报备(4)+整改响应(3)+会议参与(3) |

### 第二层：产能双维度

**自身峰值比**：当前GMV ÷ 该供应商历史最高GMV

**头部差距**：该供应商GMV ÷ 同产线第一名GMV

**双维度交叉矩阵**：

| 自身峰值比 | 头部差距 | 类型 | 根因 | 策略 |
|-----------|---------|------|------|------|
| ≥85% | ≥80% | 双高型 | 发挥正常且跟得上头部 | 保持，适当激励 |
| ≥85% | <80% | 外部限制型 | 自己发挥好但名单/系统限制了上限 | 优化资源分配 |
| <85% | ≥80% | 暂时波动型 | 跟得上头部但自己发挥下降 | 短期干预即可 |
| <85% | <80% | 双低型 | 自身问题和外部限制并存 | 深度整改或替换 |

### 第三层：FCI+主管胜任

**FCI能力指数**：七大维度覆盖资质、绩效、质检、管理、技术、财务、负面清单。

**主管胜任5项指标**：

| 指标 | 达标标准 |
|------|---------|
| 团队稳定性 | 月流失率≤5% |
| 培训执行力 | 新人7天达标率≥80% |
| 数据敏感度 | 主动发现并汇报异常 |
| 排班合理性 | 出勤率≥95% |
| 整改执行力 | 整改计划按期完成率≥70% |

## 排名文化与供应商心理

### 供应商对排名的心理反应

| 排名位置 | 心理反应 | 管理策略 |
|----------|---------|---------|
| 头部 | 满足感→可能松懈 | 设定更高目标，防止松懈 |
| 中腰部 | 焦虑感→有动力 | 给出明确的改善路径 |
| 尾部 | 挫败感→可能放弃 | 帮助识别根因，给出改善希望 |

### 排名的"表演型"风险

供应商可能为了排名而"表演"——表面数据好看，实际能力没有提升。

**表演型预警信号**：

| 信号 | 说明 |
|------|------|
| 数据好看但FCI低 | 结果好但过程差 |
| 排名上升但人均下降 | 靠堆人头提升总量 |
| 整改计划漂亮但执行不到位 | "说得好做不好" |
| 质检均分提升但A类违规增加 | 整体改善但底线问题未解决 |

### 排名的公平性保障

| 保障措施 | 说明 |
|----------|------|
| 统一标准 | 所有供应商使用相同的评估标准 |
| 数据透明 | 排名数据公开，供应商可以查看 |
| 申诉通道 | 对排名有异议可以申诉 |
| 季度校准 | 定期校准评估标准，确保公平 |

## 排名与分量分配的联动

### 分量调整的五因素原则

分量调整不是单纯看排名，需要综合考虑：

| 因素 | 说明 |
|------|------|
| 1. 业务业绩情况 | 供应商当前业绩表现和趋势 |
| 2. 增长速度情况 | 产能增长潜力和消化能力 |
| 3. 工程配合度 | 日常配合度和响应速度 |
| 4. 是否具有战略意义 | 是否是战略性供应商 |
| 5. 是否有战略扶持 | 是否有扶持计划的新供应商 |

### 分量调整后的跟踪

| 跟踪项 | 频率 | 关注点 |
|--------|------|--------|
| 总量GMV | 每周 | 是否达到预期增量 |
| 熟手人均GMV | 每周 | 是否因稀释而下降 |
| 新人拉低幅度 | 每周 | 新人占比是否>30% |
| 头部差距变化 | 每周 | 加量后差距是否扩大 |

## 排名诊断的设计原则

### 排名是工具，不是目的

排名是为了驱动供应商持续投入，不是为了"分三六九等"。排名是手段，产能是目的。

### 诊断比排名更重要

排名给出"谁好谁差"，诊断给出"为什么"和"怎么改"。前者用于决策，后者用于干预。

### 关注趋势，不只看当下

排名的短期波动可能是正常现象，长期趋势才是关键。连续3个月的排名趋势比单月排名更有参考价值。

### 帮助尾部，不只看头部

头部供应商需要"防止松懈"，尾部供应商需要"给出希望"。两者都需要关注，但策略不同。

## 排名诊断的演进方向

- **AI排名预测**：基于历史数据预测未来排名趋势
- **实时排名看板**：供应商可以实时查看自己的排名和各项得分
- **个性化诊断报告**：为每家供应商生成个性化的诊断报告和改善建议
- **排名游戏化**：在合规前提下，增加排名的"游戏化"元素，提升供应商参与度
