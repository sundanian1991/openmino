## 一、定价委员会机制V1.0核心内容

### 1.1 核心目标

| 目标 | 说明 |
|------|------|
| 从事后救火到事前防火 | 建立有约束的决策机制，不再临时拍脑袋 |
| 保障供应商/坐席合理收益 | 确保调整后收入不低于行业底线 |
| 维护电销生态健康稳定 | 避免频繁调整导致的不稳定 |

### 1.2 基本原则（红线）

| 原则 | 标准 | 说明 |
|------|------|------|
| 单次降幅上限 | ≤30% | 超过需分2次执行 |
| 生效延迟 | 次月1日生效 | 当月审批，避免突发冲击 |
| 收入底线 | 按业务线分别制定 | 需与行业拉齐+供应商调研 |
| 调整周期 | 3个月调整一次 | 避免频繁调整，特殊调整单独申请 |
| 双向公平 | 涨价/降价统一审批 | 涨价需配套利好，降价需严格审批 |

### 1.3 组织架构

| 角色 | 人员 | 主要职责 |
|------|------|----------|
| 秘书长 | 供应商管理岗（1人） | 组织会议、收集材料、追踪落地、归档记录 |
| 业务督导代表 | 1人 | 坐席层面信息（收入增减对业务的影响） |
| 量化代表 | 1人 | 数据测算、ROI分析 |
| 服务组负责人C3 | 1人 | 业务运营、职场管理 |
| 策略组负责人C3 | 1人 | 定价策略、规则设计 |
| 终审 | 数据科技业务部负责人C2 | 每项调整的最终审批 |

### 1.4 六阶段流程

```
阶段1：提案（T-5天）→ 阶段2：初审+评审（T-2天）→ 阶段3：终审（T-1天）
→ 阶段4：调整后宣贯（生效后）→ 阶段5：生效（次月1日）→ 阶段6：复盘（+30天）
```

## 二、策略组讨论要点

### 2.1 策略组角色职责

策略组在定价委员会中的核心职责：
- 提供定价策略建议，设计调整方案
- 分析市场趋势和竞品定价
- 评估调价对业务生态的影响
- 设计配套措施（利好绑定、过渡方案）

### 2.2 委员会整体职责

定价委员会作为一个整体的职责：
- 统一审批所有定价调整（无论大小）
- 确保调整符合五项基本原则
- 评估调整对供应商、坐席、业务的综合影响
- 建立预警机制和应急预案
- 定期复盘调整效果

### 2.3 QA详版核心问答

| 问题 | 回答 |
|------|------|
| 为什么需要定价委员会？ | 从"事后救火"转向"事前防火"，建立有约束的决策机制 |
| 降幅为什么设30%上限？ | 避免一次性冲击过大，超过需分2次执行 |
| 为什么次月生效？ | 给供应商和坐席足够的准备时间 |
| 如何保障坐席收入？ | 按业务线制定收入底线，测算时确保不低于底线 |
| 供应商如何表达异议？ | 设立正式申请通道，委员会评估后反馈 |

### 2.4 投影大纲（5页汇报结构）

| 页码 | 内容 |
|------|------|
| P1 | 背景与痛点：为什么需要定价委员会 |
| P2 | 机制设计：组织架构+流程 |
| P3 | 核心规则：五项基本原则 |
| P4 | 风险防控：预警信号+应急预案 |
| P5 | 实施计划：时间表+预期效果 |

## 三、V1.1修订要点

### 3.1 V1.1主要变更

| 变更项 | V1.0 | V1.1 | 原因 |
|--------|------|------|------|
| 供应商反馈通道 | 无 | 增设正式申请通道 | 保障供应商表达权 |
| 定期评估日 | 每月1次 | 每月第二个周三 | 固定时间提高效率 |
| 涨价流程 | 与降价一致 | 明确3/5通过 | 避免混淆 |
| 宣贯环节 | 简单提及 | 详细规定宣贯内容 | 确保执行效果 |

### 3.2 供应商反馈申请通道

| 要素 | 内容 |
|------|------|
| 申请时机 | 定价调整方案确定后、正式生效前 |
| 申请方式 | 电子邮箱 |
| 处理时限 | T+0接收→T+1初审→T+2评估→T+3讨论→T+4反馈 |
| 处理结果 | 采纳/部分采纳/维持原案/延期执行 |

### 3.3 修订说明核心逻辑

V1.1修订的核心逻辑是**增加透明度和供应商参与度**：
1. 设立正式反馈通道，让供应商有表达机会
2. 固定评估时间，提高决策效率
3. 明确涨价流程，避免与降价混淆
4. 细化宣贯环节，确保调整落地效果

## 四、C2汇报材料要点

### 4.1 汇报思路

向C2汇报的核心逻辑：
1. **痛点**：当前定价调整无约束，临时拍脑袋，影响生态
2. **方案**：成立定价委员会，建立有约束的决策机制
3. **保障**：五项原则+六阶段流程+预警机制
4. **效果**：从事后救火到事前防火

### 4.2 汇报摘要

> 定价委员会机制V1.1已于2026年3月完成修订，核心变更包括：增设供应商反馈申请通道、固定每月第二个周三为评估日、明确涨价流程与降价流程一致。机制已具备上线条件，建议尽快启动试运行。

### 4.3 汇报材料核心数据

| 指标 | 现状 | 目标 |
|------|------|------|
| 定价调整响应时间 | 临时决定（0-7天） | 固定流程（T-5到T+30） |
| 供应商满意度 | 低（无表达渠道） | 中（有正式反馈通道） |
| 调整频率 | 不规律 | 3个月一次 |
| 坐席收入保障 | 无明确底线 | 按业务线制定 |

## 五、参考定价模式

### 5.1 定价模式对比

| 模式 | 适用场景 | 优缺点 |
|------|---------|--------|
| **CPS（按效果付费）** | 成熟业务线 | 优点：风险共担；缺点：坐席收入波动大 |
| **固定单价** | 新业务线 | 优点：收入稳定；缺点：缺乏激励 |
| **阶梯定价** | 成长期业务线 | 优点：激励高产；缺点：计算复杂 |
| **混合模式** | 多业务线供应商 | 优点：灵活适配；缺点：管理成本高 |

### 5.2 定价建议

- **金条**：CPS为主，参考行业水平
- **企金**：阶梯定价，鼓励大额转化
- **信用卡**：固定单价+激活奖励
- **财富**：CPS+客户留存奖励

## 六、向军哥汇报框架

### 6.1 汇报结构

1. **背景**：当前定价管理存在的问题
2. **方案**：定价委员会机制设计
3. **规则**：五项基本原则和六阶段流程
4. **保障**：风险预警和应急预案
5. **计划**：实施时间表和预期效果

### 6.2 关键决策点

| 决策点 | 选项 | 建议 |
|--------|------|------|
| 降幅上限 | 20%/30%/50% | 30%（平衡风险控制与灵活性） |
| 生效时间 | 当月/次月 | 次月1日（给供应商准备时间） |
| 审批层级 | C3/C3+C2 | C2终审（重要决策需要最高层把关） |
| 评估频率 | 每月/每季 | 每月（及时响应市场变化） |

## 七、关键洞察

1. **机制建设从V1.0到V1.1**：核心是增加供应商参与度和流程透明度
2. **汇报策略**：痛点→方案→保障→效果，四步说服法
3. **定价模式多样化**：不同业务线适用不同模式，不能一刀切
4. **供应商反馈通道是亮点**：从"我定你接"到"有商有量"
5. **C2终审是底线**：所有定价调整必须经过最高层审批
