## Overview

供应商管理岗在过去半年推进了多个重要项目：315 合规月专项活动、定价委员会机制建设、BPO 供应商联盟运营、企微运营风险管控制度、供应商知识库建设等。这些项目从制度建设和风险防范两个维度夯实了管理体系基础。

## 315 合规月（2026年3月）

### 管控目标

零曝光（无媒体/监管点名）、零违规（无品牌侵权/数据泄露）、零事故（无黑衣人暗访得手/无负面舆情）。

### 活动阶段

| 阶段 | 时间 | 动作 | 覆盖范围 |
|------|------|------|---------|
| 通知下发 | 2月25日 | 发布合规通知，明确三条红线 | 全体供应商 |
| 自查自纠 | 3月1-10日 | 供应商对照14项清单自查，提交报告 | 100%覆盖 |
| 整改验收 | 3月11-20日 | 问题整改+照片验收 | 问题项100%闭环 |
| 飞行检查 | 3月21-31日 | 不定期现场/远程抽检 | 重点供应商100%覆盖 |

### 巡检清单（49项检查）

| 大类 | 检查项数 | 检查重点 |
|------|---------|---------|
| 品牌与标识 | 10项 | 无京东Logo/元素出现在职场各处 |
| 访客管理 | 8项 | 访客登记、禁止拍照标识、门禁 |
| 作业区管理 | 10项 | 手机保管、USB禁用、监控覆盖 |
| 数据安全 | 8项 | 数据导出限制、权限最小化、保密协议 |
| 职场合规 | 8项 | 物理隔离、监控分辨率、消防 |
| 315专项 | 5项 | 负责人知晓度、坐席合规意识 |

### 处罚标准（315期间）

| 违规行为 | 处罚 |
|----------|------|
| 未经授权使用京东Logo | 罚款2万+暂停新项目1个月 |
| 访客未登记进入作业区 | 罚款1万/次 |
| 私人手机带入作业区 | 罚款5000元/人次 |
| USB设备连接作业电脑 | 罚款2万+清退责任人 |
| 数据泄露/拍照传播 | 立即清退+法律追责 |

连坐机制：因供应商违规导致京东被曝光，供应商承担全部罚款+商誉损失费（最低10万）。

### 关键产出

- 5条管控措施（通知、自查、整改、飞行检查、处罚）
- 巡检检查员清单（单表版+统计内嵌版+检查员视角版）
- 主通知邮件模板+4个附件模板（合规通知、自查报告、访客登记表、禁止拍照标识）

## 定价委员会（2026年3月）

### 建设背景

从"事后救火"转向"事前防火"，建立有约束的定价决策机制，保障供应商/坐席合理收益，维护电销生态健康稳定。

### 组织架构

| 角色 | 人数 | 职责 |
|------|------|------|
| 秘书长（供管岗） | 1 | 组织会议、收集材料、追踪落地、归档 |
| 业务督导代表 | 1 | 坐席层面信息反馈 |
| 量化代表 | 1 | 数据测算、ROI分析 |
| 服务组负责人C3 | 1 | 业务运营、职场管理评估 |
| 策略组负责人C3 | 1 | 定价策略、规则设计 |
| 终审C2 | 1 | 每项调整最终审批 |

### 核心规则

| 规则 | V1.0 标准 | V1.1 修订 |
|------|-----------|-----------|
| 单次降幅上限 | ≤30% | 保持 |
| 调整周期 | 3个月一次 | 保持 |
| 生效时间 | 次月1日 | 保持 |
| 常规调整（≤15%） | 5人中3/5通过 | 保持 |
| 重大调整（>15%） | 5人中4/5通过+C2终审 | 保持 |
| 涨价调整 | 5人中3/5通过 | 与降价流程一致 |
| 定期评估日 | 每月第二周周三 | 保持 |

### 定价调整流程

```
提案(T-5天) → 初审+评审(T-2天) → 终审(T-1天) → 次月1日生效 → 宣贯 → 复盘(+30天)
```

### 供应商反馈通道

生效后前可提交异议申请，包含：影响论证、数据佐证、替代建议。处理流程：接收登记→初审→评估分析→委员会讨论→决策反馈（T+4内）。

### C2 汇报要点

- 核心问题：当前定价缺乏约束机制，"想降就降"
- 解决方案：成立5人委员会，C2终审，3/5或4/5表决
- 风险防控：早期预警信号（投诉率/利润率/坐席收入/离职率）
- 应急预案：群体性事件/投诉率>50%/离职率>20% → 24小时紧急会议

| 文件 | 说明 |
|------|------|
| 定价委员会机制 V1.0 | 初版机制文档 |
| 定价委员会机制 V1.1 | 202603修订版 |
| 定价调整申请表 | 提案用模板 |
| 供应商定价调整异议申请表 | 供应商反馈用 |
| 定价调整复盘报告模板 | 生效后30天复盘用 |

## BPO 供应商联盟（2026年Q1）

### 联盟定位

将供应商从"被动管理对象"转变为"主动合作伙"，通过联盟月报机制建立透明沟通和持续改进的文化。

### 月报模块（V9最终版）

月报核心模块包括：

| 模块 | 内容 |
|------|------|
| 业务业绩 | 各供应商GMV、产能、排名 |
| 质量合规 | 质检评分、合规事件、整改进展 |
| 人力运营 | 出勤率、人员变动、培训完成 |
| 优秀案例 | 最佳实践分享 |
| 改进要求 | 下月重点改进方向 |
| 联盟动态 | 政策变化、制度更新 |

### 汇报参考要点

- 用数据说话，避免主观评价
- 排名变化趋势比绝对值更重要
- 优秀案例要可复制、可推广
- 改进要求要具体、可量化、有时限

## 企微运营风险管控（2026年2-3月）

### 两层保障机制

第一层：内部管控制度（账号管理、数据安全、服务品质、应急响应）
第二层：供应商管理手段（合同约束、审计监督、违规处罚、优胜劣汰）

### 核心制度

| 领域 | 关键要求 |
|------|---------|
| 账号安全 | 一人一号、设备备案、异地登录预警、离职24小时冻结 |
| 数据安全 | 禁止导出/复制/截屏、数据不落地、操作全程留痕≥180天 |
| 服务品质 | 标准话术库、响应时效（VIP 5分钟/普通30分钟）、三级质检 |
| 应急响应 | 三级分级（严重2小时/一般24小时/轻微72小时） |

### 违规处罚矩阵

| 等级 | 典型行为 | 处罚 |
|------|---------|------|
| 严重 | 数据泄露/私下收款/恶意删客户 | 立即解除合作+全额赔偿+法律追责 |
| 重度 | 私自导出数据/未授权换设备 | 罚款5000-20000+警告+项目暂停 |
| 中度 | 话术违规/响应超时 | 罚款1000-5000+通报+增加质检频次 |
| 轻度 | 首次轻微违规/操作失误 | 警告+强制培训+观察1个月 |

### 月度审计

每月执行：账号台账核对、设备指纹比对、操作日志抽查、数据安全检查、质检报告复核。

## 供应商知识库建设（2026年3-4月）

### 建设背景

供应商管理涉及大量制度文件、历史数据、沟通记录，缺乏系统化的知识管理导致：
- 新人上手慢，需口口相传
- 历史决策依据难以追溯
- 制度版本混乱，不知以哪个为准

### 发现与建议

- 知识库应按"制度-执行-数据"三层组织
- 每份制度须有唯一编号和版本管理
- 历史项目须归档决策过程和关键产出
- 日常沟通须留痕，形成可检索档案

## 车抵贷项目复盘

### 项目背景

车抵贷作为创新业务试点，验证新供应商引入和快速上线能力。

### 关键教训

- 线上数标排名靠前的供应商，实际合作可能"水土不服"
- 建议多家同时试单PK，用实际效果筛选
- 宽进严出策略有效：降低准入门槛，严格过程管理

## 扩项申请项目

### 泉州速腾扩项

采购融资业务线扩项申请，验证了扩项审批流程和集中度管控机制的有效性。

### 关键经验

- 扩项前须检查集中度是否超标
- 供应商当前评估等级须不低于B级
- 扩项后须持续监控该业务线的集中度变化
