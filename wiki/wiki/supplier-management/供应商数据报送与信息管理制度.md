# 供应商数据报送与信息管理制度

## 概述

数据是供应商管理的"血液"。从每日出勤报送到月度产能汇总，从花名册变动到质检数据核对，数据的及时性、准确性和完整性直接决定了管理决策的质量。数据报送制度与数据真实性防控、空间模板化共同构成了供应商数据管理的"三驾马车"。本文系统梳理数据报送要求、数据汇总表设计、数据真实性验证和供应商工作空间管理。

## 数据报送体系

### 报送项目与频率

| 报送项 | 频率 | 报送时间 | 内容 | 渠道 |
|--------|------|---------|------|------|
| 出勤数据 | 每日 | 9:30前 | 应出勤人数、实际出勤人数、缺勤原因 | 企微群/系统 |
| 业绩数据 | 每日 | 18:00前 | 当日GMV、通话量、接通率、转化率 | 系统自动 |
| 花名册变动 | 即时 | 变动当日 | 入职、离职、调岗人员信息 | 企微群 |
| 异常事项 | 即时 | 发现后立即 | 异常描述、影响评估、应对措施 | 电话+书面 |
| 周度数据 | 每周 | 周五18:00 | 本周产能汇总、排名变化、趋势分析 | 系统 |
| 月度总结 | 每月 | 次月3日前 | 月度产能、质量、合规、管理汇总 | 邮件/系统 |

### SLA对数据报送的要求

| 指标 | 达标线 | 预警线 | 红线 |
|------|--------|--------|------|
| 数据报送及时率 | ≥98% | <95% | <90% |
| 数据准确率 | 100% | <98% | <95% |

### 数据报送的常见问题

| 问题 | 表现 | 应对 |
|------|------|------|
| 迟报 | 未在指定时间内报送 | 第一次提醒，第二次约谈，第三次扣分 |
| 漏报 | 遗漏部分数据项 | 要求补报，标注"补报" |
| 错报 | 数据明显不合理 | 退回重报，要求核实 |
| 瞒报 | 故意隐瞒异常数据 | 查实后按数据造假处理 |

## 数据汇总表设计

### 汇总表结构

数据汇总表是供应商数据标准化管理的核心工具，包含以下维度：

| 维度 | 字段 | 说明 |
|------|------|------|
| 基本信息 | 供应商名称、站点名称、业务线 | 标识数据来源 |
| 人力数据 | 在岗人数、出勤人数、缺勤人数、入职人数、离职人数 | 人力变动追踪 |
| 产能数据 | 日GMV、周GMV、月GMV、目标产能、产能达标率 | 产能追踪 |
| 质量数据 | 质检月均分、A类违规次数、话术合规率 | 质量追踪 |
| 合规数据 | 持证率、录音留存率、信息安全违规次数 | 合规追踪 |
| 排名数据 | 当前排名、上月排名、排名变化、自身峰值比、头部差距 | 排名分析 |

### 汇总表使用规则

| 规则 | 说明 |
|------|------|
| 数据源唯一 | 汇总表是唯一数据源，其他报表从汇总表生成 |
| 版本管理 | 每次更新生成新版本号，保留历史版本 |
| 数据校验 | 系统自动校验数据合理性（如出勤人数不能超过在岗人数） |
| 权限控制 | 供应商只能查看和填报自己的数据，管理员可查看所有数据 |

## 供应商工作空间

### 工作空间定位

供应商工作空间是每个供应商的独立数据管理区域，用于：
- 存放供应商的专属数据（人力、产能、合规）
- 执行数据填报和更新
- 接收管理通知和整改要求
- 查看排名和评估结果

### 工作空间结构

| 目录 | 内容 |
|------|------|
| 基本信息 | 供应商档案、联系人、站点信息 |
| 人力管理 | 花名册、人员准入、持证记录 |
| 数据报送 | 日报、周报、月报 |
| 评估记录 | 月度评估结果、FCI巡检报告 |
| 整改记录 | 整改计划、整改进展、验收报告 |
| 通知记录 | 管理通知、分量调整、预警通知 |

### 填报指南

| 填报项 | 说明 | 注意事项 |
|--------|------|---------|
| 花名册 | 所有在岗人员信息 | 人员离职后24小时内更新 |
| 持证记录 | 证书编号、有效期 | 证书过期前30天提醒续期 |
| 出勤数据 | 每日实际出勤人数 | 合法请假不计入缺勤 |
| 产能数据 | 每日/周/月GMV | 以业务系统数据为准 |
| 异常报备 | 异常描述和处理措施 | 重要异常须书面报备 |

## 数据真实性防控

### 造假类型

| 造假类型 | 典型行为 | 检测方法 |
|----------|---------|---------|
| 虚报出勤 | 将缺勤人员计入出勤 | 交叉验证：系统打卡 vs 报送数据 |
| 篡改业绩 | 虚报GMV、通话量 | 业务系统数据核对 |
| 隐瞒离职 | 不报备离职人员 | 花名册交叉核对 |
| 熟手虚标 | 将新人标注为熟手 | 产能与经验匹配分析 |
| 分流业绩 | 将A业务线业绩计入B业务线 | 业务系统业绩归因 |

### 防控手段

| 手段 | 说明 |
|------|------|
| 交叉验证 | 报送数据 vs 业务系统数据 vs 质检数据 |
| 异常检测 | 偏离均值>2个标准差触发复核 |
| 现场抽检 | 飞行检查时现场核对原始数据 |
| 历史趋势 | 与历史数据对比，识别异常波动 |
| 诚信档案 | 造假行为记入诚信档案，影响后续合作 |

### 数据造假的后果

| 严重程度 | 后果 |
|----------|------|
| 轻微（首次、非故意） | 警告+重新报送 |
| 一般（多次、有意但影响小） | 扣分+约谈 |
| 严重（系统性造假） | 直接清退+扣保证金+3年禁入 |

## 数据管理制度

### 数据报送的时效要求

| 场景 | 要求 |
|------|------|
| 日常数据报送 | 按规定时间报送 |
| 人员变动报备 | 变动前3个工作日 |
| 大面积人员变动（≥3人） | 即时电话+书面报备 |
| 异常事项报备 | 一般：当日；重要：2小时内；紧急：立即 |
| 月度总结 | 次月3日前 |

### 数据报送的准确性要求

| 要求 | 说明 |
|------|------|
| 数据一致性 | 同一数据在不同报表中的数值必须一致 |
| 数据可追溯 | 每条数据都能追溯到原始来源 |
| 数据可验证 | 报送数据能被系统数据或现场数据验证 |
| 数据完整性 | 所有要求的字段都必须填写 |

### 数据管理责任

| 角色 | 责任 |
|------|------|
| 站点对接人 | 数据填报的第一责任人 |
| 站点负责人 | 数据准确性的最终责任人 |
| 供应商管理岗 | 数据校验和异常处理 |
| 业务系统 | 自动采集产能数据，减少人工报送 |

## 数据管理的演进方向

- **系统自动化**：将手工报送的数据逐步转为系统自动采集
- **数据可视化**：为供应商提供自助查询的数据看板
- **AI异常检测**：用AI识别数据造假模式，提高检测效率
- **数据质量评分**：将数据报送质量和准确率纳入月度评估
