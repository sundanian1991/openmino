## 二、供应商管理办法（总纲）

### 2.1 适用范围

适用于所有承接京东2B电销业务（含金条、信用卡、财富等产线）的BPO供应商。

### 2.2 准入条件

**基本资质**：
| 条件 | 要求 |
|------|------|
| 企业资质 | 有效营业执照，经营范围包含呼叫中心或相关业务 |
| 财务状况 | 近一年无重大经营异常 |
| 行业经验 | 具有电销或客服外包运营经验 |
| 团队规模 | 具备承接业务所需的人员储备能力 |
| 场地设备 | 具备合规的办公场地和终端设备 |

**业务能力**：
| 条件 | 要求 |
|------|------|
| 管理团队 | 有专职项目管理团队，能对接京东管理要求 |
| 培训体系 | 具备人员培训和考核能力 |
| 信息系统 | 具备接入京东业务系统的技术条件 |
| 合规能力 | 理解并承诺遵守信息安全和业务合规要求 |

### 2.3 准入流程

```
供应商提交申请 → 资质初审（5工作日）→ 业务能力评审（10工作日）→ 试运营期（90天）→ 正式签约合作
```

### 2.4 评估体系

**四维度100分制**：
| 维度 | 分值 | 内容 |
|------|------|------|
| 交付质量 | 40分 | 质检分数、客户满意度 |
| 业务达成 | 30分 | 产能目标达成率 |
| 合规表现 | 20分 | 合规事件、数据真实性 |
| 管理配合 | 10分 | 信息报送及时性、整改响应 |

**评级与应用**：
| 评级 | 分数 | 应用 |
|------|------|------|
| A（优秀） | ≥90 | 续约优先权 |
| B（良好） | 80-89 | 正常合作 |
| C（达标） | 70-79 | 关注+约谈 |
| D（预警） | 60-69 | 整改30天 |
| E（红线） | <60 | 清退评估 |

### 2.5 预警机制

**预警触发条件**：
| 条件 | 说明 |
|------|------|
| 月度评估D级 | 60-69分 |
| 连续2个月C级 | 70-79分连续2个月 |
| 质检月均分70-79分 | 交付质量明显下降 |
| 人员离职率超标 | 离职≥3人或≥10% |

**红线触发条件**：
| 条件 | 说明 |
|------|------|
| 月度评估E级 | <60分 |
| 质检月均分<70分 | 交付质量严重不达标 |
| 信息安全严重违规 | 客户信息泄露 |
| 人员持证率<90% | 大量无证上岗 |
| 数据造假 | 虚报出勤/产能/篡改数据 |
| 人员离职率严重超标 | 离职≥8人或≥20% |

### 2.6 离职率阶梯管理

| 阶梯 | 触发条件 | 管理动作 |
|------|----------|----------|
| 关注 | 离职≥3人或≥10% | 站点负责人书面说明原因 |
| 预警 | 离职≥5人或≥15% | 提交整改计划+人员补充方案 |
| 红线 | 离职≥8人或≥20% | 启动清退评估 |

### 2.7 清退机制

**一票否决条件**：
- 信息安全严重违规
- 人员持证率<90%
- 质检月均分<70分
- 数据造假
- 连续3个月E级评估

**清退流程**：
```
触发清退条件 → 供应商管理岗提出清退建议 → 业务对接人确认 → 供应商通知（书面）→ 过渡期（30天）→ 正式清退 → 结算归档
```

**客户信息处理**（清退必备动作）：
1. 数据导出 → 京东侧确认完整性
2. 交接确认 → 双方签字
3. 数据删除 → 签署《数据删除确认书》
4. 保密延续 → 签署《保密延续协议》，延续2年

## 三、SLA协议

### 3.1 服务水平底线

| 指标 | 标准 | 考核周期 |
|------|------|---------|
| 出勤率 | ≥85% | 月度 |
| 人员持证率 | 100% | 持续 |
| 数据报送及时性 | 按时率≥95% | 月度 |
| 质检合格率 | ≥85% | 月度 |
| 响应时效 | 2小时内响应 | 持续 |
| 整改按期完成率 | ≥90% | 月度 |

### 3.2 SLA违约处理

| 违约程度 | 处理 |
|---------|------|
| 单项不达标 | 月度评估扣分 |
| 连续2个月不达标 | 预警约谈 |
| 连续3个月不达标 | 清退评估 |

## 四、质量管理规范

### 4.1 质检体系

| 要素 | 要求 |
|------|------|
| 质检频率 | 每月抽检≥10%通话录音 |
| 质检标准 | 统一质检评分表 |
| 质检团队 | 独立质检团队或第三方质检协议 |
| 质检反馈 | 质检结果3个工作日内反馈供应商 |

### 4.2 质检评分

| 评分项 | 分值 | 说明 |
|--------|------|------|
| 话术合规 | 30分 | 是否使用标准话术，有无违规 |
| 服务态度 | 20分 | 是否礼貌、耐心、专业 |
| 业务能力 | 25分 | 产品介绍是否准确完整 |
| 流程执行 | 15分 | 是否按标准流程操作 |
| 数据安全 | 10分 | 是否遵守数据安全管理要求 |

### 4.3 质检红线

- 话术违规（夸大/诱导/威胁）= 单项0分
- 服务态度恶劣 = 单项0分
- 数据安全管理不到位 = 单项0分

## 五、向上汇报体系

### 5.1 月报模板（矩阵视角）

月报采用矩阵视角，从业务线×供应商两个维度呈现数据：

| 维度 | 内容 |
|------|------|
| 业务线维度 | 各业务线产能、排名、环比变化 |
| 供应商维度 | 各供应商评分、排名、分项得分 |
| 矩阵交叉 | 各供应商在各业务线的表现 |
| 趋势分析 | 3个月趋势线 |
| 行动建议 | 下月重点关注事项 |

### 5.2 汇报框架

| 汇报类型 | 频率 | 对象 | 核心内容 |
|---------|------|------|---------|
| 周报 | 每周 | C3 | 异常信号+处理进展 |
| 月报 | 每月 | C3/C2 | 排名+分量调整+整改情况 |
| 专项汇报 | 按需 | C2 | 重大事件/决策建议 |

## 六、全生命周期管理指引

### 6.1 生命周期阶段

| 阶段 | 核心动作 | 关键文件 |
|------|---------|---------|
| 寻源引入 | 需求→寻源→评估→审批 | 引入管理规范 |
| 试运营 | 90天试运营+评估 | 试运营评估报告 |
| 正式合作 | 日常管理+月度评估 | 日常手册+评估方案 |
| 预警整改 | 预警→约谈→整改→复评 | 整改闭环报告 |
| 退出清退 | 通知→过渡→交接→结算 | 交接清单 |

### 6.2 各阶段核心管理动作

- **引入阶段**：资质审核、现场考察、综合评估、审批决策
- **试运营阶段**：月度评估、能力验证、转正评估
- **正式合作阶段**：日常监控、月度排名、分量调整
- **预警整改阶段**：预警通知、约谈、整改计划、复评
- **退出清退阶段**：清退通知、过渡期管理、客户信息处理、结算

## 八、关键洞察

1. **制度分层**：L1总纲（原则）→L2方案（标准）→L3手册（操作），层次分明
2. **评估四维度**：交付质量40+业务达成30+合规表现20+管理配合10=100分
3. **离职率阶梯**：关注（3人/10%）→预警（5人/15%）→红线（8人/20%），阶梯管理
4. **清退有底线**：信息安全/持证率<90%/质检<70/数据造假/连续3月E级，五类一票否决
5. **月报矩阵视角**：业务线×供应商二维矩阵，全面呈现供应商管理全貌
