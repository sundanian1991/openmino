## 一、供应商引入管理规范（V2.0）

### 1.1 业务分类

| 类型 | 定义 | 人力阈值 | 审批流 | 测试期 | 比选要求 |
|------|------|---------|--------|--------|---------|
| **核心业务** | 金条/企金/联名卡/财富 | >10人 | C3审核→C2终审 | 3-6个月 | 强制多方比选 |
| **创新业务** | 试点探索阶段业务 | ≤10人 | C3终审 | ≤12周 | 可直接引入 |

### 1.2 基础资质（一票否决）

以下任一条件不满足，直接否决：

| 序号 | 资质要求 | 验证方式 |
|------|---------|---------|
| 1 | 注册资本≥1000万元 | 营业执照+工商信息 |
| 2 | 《增值电信业务经营许可证》（全国性呼叫中心） | 许可证原件核验 |
| 3 | 营业执照覆盖拟合作业务范围 | 经营范围核对 |
| 4 | 资产负债率≤60% | 近一年财务报表 |
| 5 | 综合评分≥85分 | 9维度评估打分 |

**特殊引入规则**：综合评分<85分但确需引入的，必须C3+C2共同审批，并记录特殊引入理由存档。

### 1.3 9维度评估框架（10分制）

| 维度 | 权重 | 评分要点 |
|------|------|---------|
| 1. 资质合规 | 15% | 注册资本、许可证、经营范围、资产负债率 |
| 2. 项目经验 | 15% | 电销/呼叫/营销业务经验年限、甲方背景 |
| 3. 招聘能力 | 12% | 招聘渠道、到岗速度、人才储备 |
| 4. 培训体系 | 10% | 培训课程、达产周期、复训机制 |
| 5. 质量管理 | 12% | 质检体系、合规自查机制、改善闭环 |
| 6. 配合意愿 | 8% | 沟通响应速度、合作态度、管理配合度 |
| 7. 报价合理性 | 10% | 成本结构、市场对标、性价比 |
| 8. 技术能力 | 10% | 系统对接能力、数据安全保障 |
| 9. 组织管理 | 8% | 管理团队稳定性、现场管理能力 |

### 1.4 准入条件汇总（须同时满足）

1. 具备有效营业执照及相关业务经营资质
2. 坐席规模不低于50人
3. 人员持证率（金融相关资质）达到100%
4. 具备独立质检团队或第三方质检协议
5. 数据安全防护能力通过我方评估
6. 无重大商业违规或法律诉讼记录
7. 基础资质全部通过（一票否决项）
8. 综合评分≥85分

### 1.5 引入SOP（8阶段·15工作日）

| 阶段 | 名称 | 责任人 | 时效 | 输出物 |
|------|------|--------|------|--------|
| 1 | 需求发起 | C2 | T+0 | 《需求清单》 |
| 2 | 需求拆解 | 业管/供管 | T+1 | 《寻源需求表》 |
| 3 | 寻源触达 | 业管 | T+3 | 供应商回复汇总+意向名单 |
| 4 | 方案审批 | 供管 | T+5 | xbp审批通过方案 |
| 5 | 结果同步 | 供管 | T+5 | 《结果同步邮件》 |
| 6 | 供应商准备 | 供应商 | T+10 | CID信息表/站点报备/合同确认 |
| 7 | 内部操作 | 业管/供管 | T+12 | 京音供应商档案/账号开通 |
| 8 | 培训上线 | 供应商+业管 | T+15 | 培训记录/上线记录/首日质量报告 |

### 1.6 集中度管控

**n-1业务参与上限原则**：单一供应商最多参与n-1个核心业务（4个核心业务→最多参与3个）。

**AI场景集中度上限**：
| 供应商数量 | 单一供应商上限 |
|-----------|--------------|
| ≤3家 | ≤70% |
| 4家 | ≤60% |
| ≥5家 | ≤50% |

**人工场景集中度上限**：
| 业务线 | 人数上限 | 占比上限 |
|--------|---------|---------|
| 金条人工 | 360人 | 30% |
| 金条人机 | 120人 | 40% |
| 信用卡 | 220人 | 40% |
| 财富 | 100人 | 40% |
| 企业金融-主贷 | 100人 | 50% |
| 企业金融-小b | 60人 | 60% |
| 商家白条 | 12人 | 60% |

### 1.7 审批流速查

| 场景 | 审批流 | 时限 |
|------|--------|------|
| >10人（核心业务） | C3审核→C2终审 | C3≤3天，C2≤5天 |
| ≤10人（创新业务） | C3终审 | C3≤3天 |
| 特殊引入（<85分） | C3+C2共同审批 | C3≤3天，C2≤5天 |
| 快速通道（≤10人调整） | C3京ME备案 | 5个工作日内 |
| 分量调整<50% | C3审批 | 常规 |
| 分量调整≥50% | C3→C2 | 大幅调整 |

## 二、供应商日常管理规范

### 2.1 日常管控框架

| 管控维度 | 核心动作 | 频率 |
|---------|---------|------|
| 数据监控 | 产能/出勤/质检分数跟踪 | 每日 |
| 排名管理 | 月度评分排名 | 每月 |
| 预警约谈 | D级/连续C级触发约谈 | 月度/事件 |
| 整改闭环 | 整改项逐项验证 | 30天 |
| 供应商沟通 | 日常对接+异常通报 | 每日 |
| 分量决策 | 基于排名调整分配 | 每月 |

### 2.2 对接人机制

| 角色 | 所属方 | 职责 |
|------|--------|------|
| 站点负责人 | 供应商 | 站点最高管理者，统筹全部业务 |
| 站点对接人 | 供应商 | 唯一日常对口人，对接京东侧全部接口 |
| 业务对接人（业管） | 京东 | 业务交付管理：产能目标、业务调度 |
| 供应商管理岗 | 京东 | 制度管理：准入评审、评估考核、合规监督 |

### 2.3 站点管理

- 每家供应商公司最多设立4个站点
- 站点为最小考核单元：独立出勤、独立产能、独立评估排名
- 同一供应商下不同站点独立考核，互不影响
- 新增站点须经供应商管理岗审批

## 三、供应商清退管理规范

### 3.1 清退触发条件（五类一票否决）

| 条件 | 说明 |
|------|------|
| 信息安全严重违规 | 客户信息泄露、违规使用、外传 |
| 人员持证率<90% | 大量无证上岗 |
| 质检月均分<70分 | 交付质量严重不达标 |
| 数据造假 | 虚报出勤、虚报产能、篡改数据 |
| 连续3个月E级评估 | 月度评估<60分连续3个月 |

### 3.2 清退流程

```
触发清退条件 → 供应商管理岗提出清退建议 → 业务对接人确认 → 书面通知供应商
→ 过渡期（30天）：业务交接+人员安置+客户信息处理
→ 正式清退 → 结算+归档
```

### 3.3 异议处理

- 业务对接人对清退建议有异议的，须在2个工作日内提出书面异议及理由
- 由部门负责人仲裁，仲裁结果为最终结果
- 仲裁期间不影响清退建议的效力

### 3.4 供应商主动退出

| 步骤 | 内容 |
|------|------|
| 1. 书面通知 | 提前30个自然日提交书面终止申请 |
| 2. 影响评估 | 评估对业务的影响范围 |
| 3. 业务交接 | 确定交接方案 |
| 4. 客户信息处理 | 数据导出→交接确认→删除确认→保密协议 |
| 5. 正式终止 | 过渡期满后正式终止 |

## 四、基于HRBP方法论的工作手册

### 4.1 HRBP式核心理念

HRBP（Human Resource Business Partner）模式的核心是将供应商管理从"监督管控"转变为"赋能扶持"：

| 传统模式 | HRBP模式 |
|---------|---------|
| 发现问题→处罚 | 发现问题→帮扶 |
| 单向要求 | 双向共建 |
| 考核排名 | 能力共建 |
| 事后整改 | 事前预防 |

### 4.2 帮扶方式

| 方式 | 适用场景 | 说明 |
|------|---------|------|
| 派驻辅导 | 供应商管理能力不足 | 京东方人员驻场指导 |
| 标杆配对 | 落后供应商 | 优秀供应商与落后供应商结对 |
| 专项培训 | 特定能力短板 | 管理/质检/培训专项培训 |
| 资源对接 | 资源瓶颈 | 引入招聘渠道/培训体系 |

### 4.3 退出条件

供应商能力恢复，连续2个月达标后退出帮扶模式。

## 七、关键洞察

1. **引入规范是最完整的制度**：8阶段SOP+9维度评估+集中度管控+审批流速查，覆盖引入全流程
2. **基础资质一票否决**：注册资本/许可证/经营范围/资产负债率/综合评分，五项不通过直接否决
3. **集中度管控精细化**：AI场景按供应商数量动态上限，人工场景按业务线分别设置人数+占比双上限
4. **HRBP式转型**：从"监督管控"到"赋能扶持"，是供应商管理理念的升级
5. **可视化体系完备**：22模板+28图表，供应商管理的数据呈现方式全覆盖
