## Overview

六步工作法是供应商管理的日常操作框架，将"监控→预警→整改→分量→巡检→汇报"串联成闭环。每一步都有明确的触发条件、操作要点和产出物。六步工作法不是线性的流程，而是一个循环——从监控开始，到汇报结束，再回到监控。本文系统梳理每一步的操作细节和联动关系。

## 六步工作法全景

```
┌─────────────────────────────────────┐
│ 1. 监控扫描 → 2. 预警约谈            │
│      ↓            ↓                  │
│ 6. 向上汇报    3. 整改闭环            │
│      ↓            ↓                  │
│ 5. 巡检管理 ← 4. 分量决策            │
└─────────────────────────────────────┘
```

## 第一步：监控扫描

### 监控范围

| 监控维度 | 监控内容 | 数据来源 |
|----------|---------|---------|
| 出勤监控 | 每日出勤率、缺勤原因 | 供应商报送+系统打卡 |
| 产能监控 | 日/周/月GMV、排名变化 | 业务系统 |
| 质量监控 | 质检月均分、违规次数 | 质检团队 |
| 合规监控 | 持证率、信息安全、录音留存 | 供应商管理台账 |
| 人力监控 | 入职/离职/在岗人数 | 花名册 |

### 监控频率

| 频率 | 内容 |
|------|------|
| 每日 | 出勤数据、业绩数据、异常报备 |
| 每周 | 周度排名、产能趋势、离职人数 |
| 每月 | 月度评估、质检报告、合规检查 |

### 异常识别规则

| 异常类型 | 触发条件 | 优先级 |
|----------|---------|--------|
| A级骤降 | 上月前30%本月跌到后30% | 高 |
| 出勤异常 | 连续2周出勤率<90% | 高 |
| 业绩断崖 | 单周GMV环比下降>30% | 高 |
| 流失加速 | 单周离职人数>5人或月流失率>10% | 中 |
| 合规预警 | 持证率<95% | 中 |
| 数据异常 | 偏离均值>2个标准差 | 低 |

### 监控产出

每日监控扫描后，供应商管理岗需要：
- 识别异常并分类（高/中/低优先级）
- 将异常信息传递给相关供应商
- 记录到监控日志中

## 第二步：预警约谈

### 预警触发条件

| 触发条件 | 说明 | 响应时限 |
|----------|------|---------|
| 单次触及预警线 | 任一指标低于SLA预警线 | 3个工作日内约谈 |
| 连续2周黄色预警 | 同一指标连续2周低于达标线 | 24小时内约谈 |
| 月度评估D级 | 月度综合评估D级 | 5个工作日内约谈 |
| A级骤降 | 上月前30%本月跌到后30% | 立即约谈 |

### 约谈流程

| 步骤 | 内容 | 说明 |
|------|------|------|
| 1. 开场 | 直接亮出数据 | "你的XX指标是X，标准是X" |
| 2. 供应商陈述 | 让供应商说明情况 | 了解原因、困难、计划 |
| 3. 共同诊断 | 一起分析根因 | 区分内部/外部因素 |
| 4. 制定计划 | 明确整改目标 | 量化目标+时间节点 |
| 5. 确认纪要 | 书面确认 | 双方签字 |

### 约谈记录

每次约谈后须形成书面纪要，包含：
- 约谈时间、参与人
- 预警指标和数据
- 供应商陈述
- 共同诊断结论
- 整改计划（目标、措施、时间节点）
- 双方签字

## 第三步：整改闭环

### 整改触发条件

| 触发条件 | 说明 |
|----------|------|
| 月度评估D级 | 启动30天整改 |
| 月度评估E级 | 启动整改+清退评估 |
| 预警约谈后 | 如问题未解决，启动整改 |
| 巡检发现问题 | 红色项≥2，启动整改 |

### 整改流程

| 阶段 | 时限 | 内容 |
|------|------|------|
| 整改通知 | 触发后当日 | 书面通知，说明问题和要求 |
| 整改计划 | 3-5个工作日内 | 供应商提交整改计划 |
| 整改计划审核 | 2个工作日内 | 供应商管理岗审核计划合理性 |
| 整改执行 | 30-60天 | 供应商执行整改 |
| 整改验收 | 整改期满 | 四维度验收 |

### 整改期间管控

| 等级 | 产能管控 | 新业务 | 评估频率 |
|------|----------|--------|----------|
| D级整改 | 冻结新增 | 暂停新业务分配 | 每半月评估 |
| E级整改 | 逐步缩减 | 暂停所有新业务 | 每周评估 |

### 整改验收四维度

| 维度 | 验证方法 | 通过标准 |
|------|---------|---------|
| 目标达成 | 对比承诺的量化目标 | ≥80% |
| 动作执行 | 逐条核对承诺动作 | ≥70%按期完成 |
| 持续性 | 观察整改后2个月数据 | 指标未回落 |
| 根因消除 | 对比整改前后核心失分项 | 原失分项提升≥1个等级 |

### 验收结果处理

| 结果 | 处理 |
|------|------|
| 全部通过 | 取消预警，归档为成功案例 |
| 部分通过 | 延长观察期1个月 |
| 未通过 | 升级：连续2次未通过→进入清退评估 |
| 造假嫌疑 | 直接进入清退评估，记入诚信档案 |

## 第四步：分量决策

### 分量调整触发条件

| 触发条件 | 说明 |
|----------|------|
| 月度评估完成 | 根据排名和评分调整 |
| 新业务上线 | 分配初始分量 |
| 供应商产能变化 | 产能增减影响分量 |
| 集中度管控 | 需要重新平衡份额 |

### 四维度评分

| 维度 | 权重 | 评分要素 |
|------|------|----------|
| 业绩达成 | 40分 | 排名趋势、GMV达成率、近3个月走势 |
| 合作意愿 | 30分 | 响应速度、问题处理、配合度记录 |
| 资源可用 | 20分 | 产能可用性、质量稳定性、人员储备 |
| 策略匹配 | 10分 | 集中度控制、业务线协同、区域布局 |

### 加量后跟踪

加量后2周是关键窗口，每周跟踪：

| 跟踪项 | 关注点 |
|--------|--------|
| 总量GMV | 是否达到预期增量 |
| 熟手人均GMV | 是否因稀释而下降 |
| 新人拉低幅度 | 新人占比是否>30% |
| 头部差距变化 | 加量后差距是否扩大 |

## 第五步：巡检管理

### 巡检类型

| 类型 | 频率 | 触发条件 |
|------|------|---------|
| 月度巡检 | 每月 | 常规巡检 |
| 事件触发 | 按需 | 异常事件、投诉、举报 |
| 飞行检查 | 不定期 | 315合规月、专项整治 |

### 巡检内容

| 维度 | 检查项 | 权重 |
|------|--------|------|
| A 资质合规 | 持证率、培训记录、准入文件 | 15% |
| B 绩效表现 | 产能达成、GMV业绩、排名趋势 | 25% |
| C 质检合规 | 质检均分、A类违规、话术合规率 | 20% |
| D 管理能力 | 主管胜任指标、制度执行情况 | 15% |
| E 技术支持 | 系统对接、设备状态、工具使用 | 10% |
| F 财务健康 | 薪资发放及时性、结算合规性 | 5% |
| G 负面清单 | 投诉量、违规记录、舆情风险 | 10% |

### 巡检产出

| 产出物 | 说明 |
|--------|------|
| FCI能力指数 | 巡检的量化输出 |
| 巡检报告 | 现场检查记录 |
| 整改建议 | 红色项对应的整改要求 |
| 现场照片 | 职场环境、标识等 |

## 第六步：向上汇报

### 汇报类型

| 类型 | 频率 | 内容 | 汇报对象 |
|------|------|------|---------|
| 月度报告 | 每月 | 排名、分量、合规、整改 | C3 |
| 专项汇报 | 按需 | 定价调整、清退决议、重大事件 | C3/C2 |
| 季度对标 | 每季度 | 供应商对标分析 | C3 |
| 年度总结 | 每年 | 全年供应商管理回顾 | C2 |

### 月度报告结构

| 模块 | 内容 |
|------|------|
| 排名看板 | 各业务线供应商排名及变化 |
| 分量看板 | 分量调整执行情况及效果 |
| 合规看板 | 合规指标达成、违规事件、整改进展 |
| 风险预警 | 异常信号、预警供应商、应对措施 |
| 下月计划 | 下月重点工作安排 |

## 六步工作法的联动关系

六步工作法不是六个独立的步骤，而是相互关联的闭环：

- **监控扫描→预警约谈**：监控发现异常，触发约谈
- **预警约谈→整改闭环**：约谈后问题未解决，触发整改
- **整改闭环→分量决策**：整改结果影响分量调整
- **分量决策→巡检管理**：分量变化后通过巡检验证承接能力
- **巡检管理→监控扫描**：巡检发现新问题，纳入日常监控
- **向上汇报→所有步骤**：所有步骤的结果都需要向上汇报

## 六步工作法的设计原则

### 闭环管理

每一步都有明确的输入和输出，形成闭环。没有闭环的管理是"断头路"。

### 数据驱动

每一步都以数据为基础。没有数据的预警是"主观判断"，没有数据的整改是"空谈"。

### 分级响应

不同严重程度的问题有不同的响应级别。不是所有问题都需要"最高级别响应"。

### 记录可追溯

每一步的操作都需要记录。这是审计和追溯的依据，也是后续优化方法论的基础。

## 六步工作法的演进方向

- **系统自动化**：将六步工作法嵌入系统，实现自动触发、自动流转
- **AI辅助决策**：用AI辅助异常识别、根因诊断、策略推荐
- **供应商自助**：供应商可以在线查看自己的预警状态、整改进度
- **方法论迭代**：基于实际运行数据，持续优化六步工作法
