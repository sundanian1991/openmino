## 概述

本文整合了供应商工作手册模板体系的13份文件：使用说明、供应商基本信息、三个准入承接模板（人员准入/制度报送/站点报备）、五个日常运营模板（分量确认/排班考勤/数据报送/结算争议/质量合规）、两个持续提升模板（季度对标/月度复盘）和一个退出清退模板（交接清单）。这是供应商全生命周期管理的标准化操作模板集合。

## 一、手册体系概览

### 1.1 核心逻辑

```
京东定标准 → 供应商填写执行 → 京东检查 → 不合格则整改 → 整改不合格则确认合作意愿
```

### 1.2 手册结构

```
【供应商名称】工作手册/
├── 00-供应商基本信息.md          ← 供应商基础信息
├── 01-供应商运营管理要求.md       ← 京东管理要求（只读，由京东提供）
│
├── 00-准入承接/                   ← 准入阶段
│   ├── 站点报备/
│   ├── 人员准入/
│   └── 制度报送/
│
├── 01-日常运营/                   ← 运营阶段
│   ├── 分量确认/
│   ├── 排班考勤/
│   ├── 质量合规/
│   ├── 结算争议/
│   └── 数据报送/
│
├── 02-持续提升/                   ← 提升阶段
│   ├── 月度复盘/
│   └── 季度对标/
│
└── 03-退出清退/                   ← 退出阶段
    └── 交接清单/
```

### 1.3 填写规则

| 项目 | 说明 |
|------|------|
| 填写人 | 供应商指定对接人或各模块责任人 |
| 文件格式 | Markdown（.md）或 Excel/Word 按模板要求 |
| 命名规则 | `YYYYMMDD-内容主题.md` |
| 提交方式 | 按京东指定方式提交（邮件/系统/共享文件夹） |
| 更新频率 | 按各子模块要求的频率更新 |

## 二、准入承接阶段模板

### 2.1 供应商基本信息

记录供应商的基础信息，包括：
- 公司全称、简称、统一社会信用代码
- 注册资本、法人代表、成立日期
- 经营范围、许可证信息
- 联系人、联系方式、邮箱
- 合作业务线、站点数量

### 2.2 人员准入模板

| 填报项 | 说明 |
|--------|------|
| 人员姓名 | 入职人员真实姓名 |
| 身份证号 | 用于资质核验 |
| 岗位 | 坐席/主管/质检/培训 |
| 资质证书 | 金融相关资质名称+编号 |
| 入职日期 | 实际入职日期 |
| 培训记录 | 岗前培训完成情况 |
| 考核成绩 | 业务考核分数 |
| 合规承诺 | 是否签署合规承诺书 |

**要求**：入职24小时内在京音平台完成人员备案，持证上岗率100%。

### 2.3 制度报送模板

供应商须在承接业务后15个工作日内提交七个管理模块的内部管理规范：

| 编号 | 模块 | 须明确的制度内容 |
|------|------|-----------------|
| M1 | 人力配置管理 | 管理架构、质检与培训岗位配置、关键岗位备岗 |
| M2 | 培训管理 | 培训体系、达产周期、复训机制、辅导带教 |
| M3 | 排班与考勤 | 排班逻辑、出勤达标、异常缺勤跟进 |
| M4 | 质量管理 | 内部质检体系、质检→改善闭环、合规自查 |
| M5 | 绩效管理 | 考核体系、激励机制、改进机制、绩效面谈 |
| M6 | 现场管理 | 日常管理动作、会议制度、主管巡查 |
| M7 | 数据报送 | 报送责任人、数据核对、信息安全 |

### 2.4 站点报备模板

| 填报项 | 说明 |
|--------|------|
| 站点名称 | 职场名称 |
| 站点地址 | 省/市/区/详细地址 |
| 场地面积 | 平方米，人均≥3㎡ |
| 消防验收 | 是否合格 |
| 站点负责人 | 姓名+联系方式 |
| 站点对接人 | 姓名+联系方式 |
| 上线人力 | 当前上线人数 |
| 业务线 | 承接的业务线 |

**审批流程**：填写《职场变更申请表》→ 提交租赁合同+消防证明文件+人力考勤记录 → 供管岗审批（3工作日）→ 更新京音平台。

| 序号 | 交接项 | 内容 | 完成状态 | 交接人 | 接收人 | 日期 |
|------|--------|------|---------|--------|--------|------|
| 1 | 客户数据 | 全部客户数据导出 | [ ] | | | |
| 2 | 系统权限 | 京音系统账号注销 | [ ] | | | |
| 3 | 设备资产 | 甲方提供设备归还 | [ ] | | | |
| 4 | 文档资料 | 管理文档/制度文件交接 | [ ] | | | |
| 5 | 数据删除 | 删除所有客户数据副本 | [ ] | | | |
| 6 | 保密协议 | 签署保密延续协议 | [ ] | | | |

**必备动作**：客户信息处理是清退流程的必备环节，须在过渡期内完成：
1. 数据导出 → 京东侧确认完整性
2. 交接确认 → 双方签字
3. 数据删除 → 签署《数据删除确认书》
4. 保密延续 → 签署《保密延续协议》，合作终止后延续2年

## 六、审核机制

| 阶段 | 审核方 | 审核时限 | 结果 |
|------|--------|---------|------|
| 首次提交 | 供应商管理岗 | 5个工作日 | 通过/限期整改 |
| 日常更新 | 供应商管理岗 | 定期检查 | 合格/整改 |
| 整改复核 | 供应商管理岗 | 3个工作日 | 通过/合作意愿确认 |

### 不合格处理

1. **首次不合格**：出具整改通知书，限期5-10个工作日整改
2. **整改不合格**：约谈供应商商务负责人，确认合作意愿
3. **拒不配合**：视为放弃合作，启动强制退出流程

---

## 七、关键洞察

1. **标准化模板降低沟通成本**：13个模板覆盖全生命周期，供应商按模板填写，减少反复沟通
2. **四阶段递进**：准入承接→日常运营→持续提升→退出清退，每个阶段有明确的模板和输出
3. **24小时备案**：人员入职24小时内必须完成备案，确保人员管理时效
4. **15个工作日制度报送**：供应商承接后15个工作日内必须提交7个管理模块的制度
5. **清退必备动作**：客户信息处理是清退流程的必备环节，未完成不得进入正式结算
