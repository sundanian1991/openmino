## Overview

定价委员会机制的落地离不开完整的工具集支撑。本文整合了定价委员会相关的会前检查清单、会议议程、提案材料模板、定价调整申请表、异议申请表、复盘报告模板、C2汇报脚本等工具文档。这些工具将定价委员会从"概念机制"转化为"可执行的操作系统"，是供应商管理中定价权规范化的核心基础设施。

## 一、定价委员会的会议管理体系

### 会前检查清单

策略组讨论会议前的完整准备清单，确保万无一失：

**材料准备**：
| 材料 | 份数 | 发放对象 | 用途 |
|------|------|----------|------|
| 会议议程 | 1份 | 自己用 | 掌握会议节奏 |
| QA详版 | 1份 | 自己用 | 应对质疑 |
| 策略组角色职责整理 | 2份 | 策略组C3+团队 | 让策略组知道要做什么 |
| 委员会整体职责整理 | 5份 | 所有委员 | 让所有人知道各自职责 |
| 提案材料模板 | 5份 | 所有委员 | 展示"填完也就15分钟" |

**会前准备四步法**：
1. 与策略组C3预热 — 提前了解策略组的关注点和反对意见
2. 准备历史数据 — 过去一年定价调整次数、紧急调整案例及后果
3. 准备开场白 — "不是通知你们'机制已定'，是想同步我做的事，听听你们的真实想法"
4. 准备投影/PPT — 定价委员会组织架构图、核心流程图、三条红线表格

### 会议中记录三要素

| 记录内容 | 用途 |
|---------|------|
| 他们说的"绝对不能接受"的事 | 回去必须调整 |
| 他们愿意妥协的点 | 可以推进 |
| 他们愿意共同参与的部分 | 建立合作关系 |

### 五大风险预案

**风险1：策略组强烈反对，会议陷入僵局**
应对："今天不是要当场定下来，是听听你们的意见。如果有些问题今天解决不了，我们可以会后单独讨论。"

**风险2：策略组质疑"你没和C2对齐过"**
应对："我已经和C2汇报过，他认可这个方向。但细节部分还需要C2最终确认。"

**风险3：策略组质疑"产线/金分都没同步"**
应对："你说得对，这是风险点。会后我会安排与产线/金分对齐会议。"

**风险4：策略组说"我们没时间配合"**
应对："这个机制大概需要你们投入1.5-5.5小时/月。大部分是会议时间，不是额外工作量。"

**风险5：会议时间不够，讨论不完**
应对："今天先对齐大方向，没讨论完的问题会后单独沟通。"

### 核心姿态

1. **主动暴露短板**："我知道有短板"
2. **邀请共同参与**："你们愿意共同参与哪些部分？"
3. **强调试运行心态**："先建框架，在跑的过程中迭代"

## 二、定价调整申请表

### 申请表结构

定价调整申请表是定价委员会的核心工作文档，包含10个完整模块：

| 模块 | 内容 |
|------|------|
| 1.基本信息 | 申请编号、类型（单价下调/上调/结算规则变更）、生效日期 |
| 2.调整原因 | 主要原因、次要原因、背景说明 |
| 3.数据支撑 | 当前单价/结算额/利润率 vs 调整后预期 |
| 4.影响测算 | 对供应商、对坐席、对业务的三重影响 |
| 5.配套措施 | 名单质量提升、授信额度增加、补贴政策等 |
| 6.供应商反馈 | 前期沟通情况、主要反馈、情绪评估、应对策略 |
| 7.业管反馈 | 坐席层面影响评估、坐席反馈收集情况 |
| 8.终审建议 | 建议幅度、时间、风险等级 |
| 9.委员会评审 | 五委员投票记录、C2终审结果 |
| 10.执行与复盘 | 实际效果对比、供应商/坐席反应、改进建议 |

### 影响测算维度

**对供应商的影响**：
- 月度收入变化（金额或百分比）
- 利润率变化（百分点）
- 预估影响供应商数量
- 供应商承受能力评估（强/中/弱）

**对坐席的影响**：
- 月度收入变化
- 预估影响坐席数量
- 预估对离职率影响
- 预估对工作积极性影响

**对业务的影响**：
- 预估对产能影响
- 预估对质量影响
- 预估对客户满意度影响
- 预估对整体成本影响

## 三、供应商定价调整异议申请表

当供应商对定价调整有异议时，通过正式渠道提交异议申请：

**核心字段**：
- 异议编号：OBJ-[年份]-[序号]
- 供应商名称、业务线、调整项目
- 异议类型：单价调整/结算规则/考核门槛/奖惩系数
- 异议理由：数据支撑、影响分析、替代建议

**处理流程**：
```
供应商提交异议 → 委员会秘书处登记 → 数据复核 → 委员会讨论 → C2终审 → 结果通知
```

## 四、定价调整复盘报告模板

每次定价调整执行后的复盘文档：

**复盘内容**：
1. 调整效果评估（与预估的对比）
2. 供应商反应（实际反馈情况）
3. 坐席反应（收入变化、离职率变化）
4. 业务影响（产能、质量、客户满意度）
5. 风险控制评估
6. 供应商关系影响（积极/中性/消极）
7. 改进建议

## 五、C2汇报体系

### 汇报思路

向C2汇报定价委员会机制的核心思路：

1. **先说问题** — 当前定价调整的混乱情况（紧急调整、缺乏机制、没有复盘）
2. **再说方案** — 定价委员会的组织架构和运行机制
3. **强调可控** — 流程透明、有委员会把关、C2有最终决策权
4. **请求支持** — 需要C2站台，推动策略组和产线配合

### 口述汇报脚本核心段落

> "现在定价调整主要靠紧急沟通，没有固定机制。我想建一个定价委员会，由策略组牵头，量化提供数据支撑，我们做执行和监督。每个月定期讨论，有调整需求走正式流程。这样既能保证调整的透明和公平，也能避免'拍脑袋'决策。"

### 汇报要点

- 委员会架构：策略组牵头、量化支撑、执行监督分离
- 核心流程：T-5提案 → T-2讨论 → T-1公示 → 次月生效 → +30天复盘
- 三条红线：坐席收入底线、供应商利润率底线、客户体验底线
- 试运行计划：先在一个业务线试点，验证后推广

## 六、投影大纲（5页PPT结构）

| 页码 | 内容 | 目的 |
|------|------|------|
| 第1页 | 当前问题与痛点 | 引起共鸣，说明为什么需要机制 |
| 第2页 | 定价委员会组织架构 | 让大家知道"谁来做什么" |
| 第3页 | 核心流程图（T-5到+30天） | 让大家知道"怎么走流程" |
| 第4页 | 三条红线表格 | 让大家知道"什么不能碰" |
| 第5页 | 试运行计划与下一步 | 让大家知道"什么时候开始" |
