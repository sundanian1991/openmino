## 概述

供应商引入是供应商全生命周期的起点，决定了供应商池的质量基线。从寻源需求到准入评估、从试运行到正式签约，每个环节都需要标准化操作。扩项管理则是在现有供应商基础上的业务线扩展，需要平衡产能需求与集中度风险。本文系统梳理引入流程、准入标准、扩项策略和供应商补充材料机制。

## 供应商引入的八阶段SOP

### 引入流程全景

| 阶段 | 名称 | 核心动作 | 时限 | 责任人 |
|------|------|---------|------|--------|
| 1 | **需求确认** | 明确引入原因、业务线、坐席规模 | — | 供应商管理岗 |
| 2 | **寻源** | 市场调研、目标供应商筛选、初步接触 | 5-10个工作日 | 供应商管理岗 |
| 3 | **资质审核** | 营业执照、资质证书、财务状况、合规记录 | 10个工作日 | 供应商管理岗 |
| 4 | **现场考察** | 职场环境、硬件设施、管理制度、人员状态 | 5个工作日 | 供应商管理岗+业务方 |
| 5 | **试运行评审** | 小规模上线、30天试运行、数据评估 | 30天 | 供应商管理岗+质检+业务方 |
| 6 | **转正评估** | 综合评分、准入评分表打分 | 5个工作日 | 供应商管理岗 |
| 7 | **合同签约** | 合同条款确认、SLA签署、保密协议 | 10个工作日 | 法务+供应商管理岗 |
| 8 | **正式上线** | 人员准入、制度报送、站点报备、分量分配 | — | 供应商管理岗+业务方 |

### 阶段详解

#### 阶段1：需求确认

引入需求来源：

| 需求类型 | 场景 | 说明 |
|----------|------|------|
| **产能不足** | 现有供应商无法承接新增业务量 | 需扩项或引入新供应商 |
| **新业务线** | 启动新的金融产品电销 | 需要具有相关经验的供应商 |
| **区域覆盖** | 需要拓展新的地理区域 | 考虑当地成本和人才供给 |
| **风险分散** | 某业务线供应商过于集中 | 引入新供应商降低集中度 |
| **替换淘汰** | 现有供应商被清退 | 需要快速引入替代者 |

**需求确认需明确**：
- 引入原因（为什么需要新供应商）
- 业务线（承接什么业务）
- 坐席规模（需要多少人）
- 时间要求（何时上线）
- 特殊要求（竞品经验、区域偏好等）

#### 阶段2：寻源

**寻源策略**：

| 策略 | 适用场景 | 方法 |
|------|---------|------|
| **定向寻源** | 需要特定行业经验 | 从竞品合作方中筛选 |
| **公开招标** | 大规模引入、多个候选 | 发布招标需求、收集方案 |
| **推荐引入** | 已有供应商推荐 | 通过行业关系引荐 |
| **主动开拓** | 新区域、新领域 | 市场调研、主动联系 |

**寻源评估维度**：
- 企业资质（营业执照、业务范围）
- 行业经验（是否有电销BPO经验）
- 坐席规模（能否满足人力需求）
- 地理区域（职场所在城市）
- 合作意愿（是否有合作意向）

#### 阶段3：资质审核

**准入条件**（须同时满足）：

| # | 条件 | 标准 |
|---|------|------|
| 1 | 营业执照 | 有效营业执照及相关业务经营资质 |
| 2 | 坐席规模 | 不低于50人 |
| 3 | 人员持证率 | 金融相关资质达到100% |
| 4 | 质检能力 | 具备独立质检团队或第三方质检协议 |
| 5 | 数据安全 | 数据安全防护能力通过我方评估 |
| 6 | 合规记录 | 无重大商业违规或法律诉讼记录 |

**资质审核需收集的资料**：

| 资料类型 | 具体内容 |
|----------|---------|
| 企业资质 | 营业执照、税务登记证、组织机构代码证 |
| 财务资料 | 近2年财务报表、纳税证明 |
| 人员资料 | 核心管理团队简历、持证人员清单 |
| 合规资料 | 信息安全管理体系、数据安全保障方案 |
| 场地资料 | 职场租赁合同、消防安全证明 |
| 参考案例 | 同类项目合作经验、客户推荐信 |

#### 阶段4：现场考察

现场考察是判断供应商"真实状态"的关键环节。

**考察维度**：

| 维度 | 考察内容 | 关注点 |
|------|---------|--------|
| **职场环境** | 办公面积、工位布局、消防设施 | 是否符合安全标准 |
| **硬件设施** | 电脑、耳机、网络、监控系统 | 设备是否齐全、网络是否稳定 |
| **管理制度** | 考勤制度、质检制度、培训体系 | 是否有成文制度、执行情况 |
| **人员状态** | 坐席工作状态、团队氛围 | 是否专业、是否有纪律性 |
| **数据安全** | 门禁、手机管理、USB管控 | 物理隔离是否到位 |

**现场考察打分**：使用《供应商准入评分表》，每个维度满分100分，加权计算总分。

#### 阶段5：试运行评审

**试运行周期**：30天

**试运行评估维度**：

| 维度 | 评估内容 | 达标标准 |
|------|---------|---------|
| 产能达成 | 实际产出 vs 目标产出 | ≥80% |
| 质检合格 | 质检月均分 | ≥80分 |
| 出勤稳定 | 日均出勤率 | ≥95% |
| 合规表现 | 信息安全、持证率 | 0违规、100%持证 |
| 配合度 | 信息报送、异常响应 | 及时、准确 |

**试运行评审结果**：

| 结果 | 说明 |
|------|------|
| 通过 | 全部达标，进入转正评估 |
| 有条件通过 | 部分指标略低于标准，给予延长试运行（≤15天） |
| 不通过 | 多项指标不达标，终止合作 |

#### 阶段6：转正评估

**转正评估流程**：

| 步骤 | 内容 | 时限 |
|------|------|------|
| 1 | 整理试运行数据 | 2个工作日 |
| 2 | 按《供应商准入评分表》打分 | 2个工作日 |
| 3 | 撰写转正评估报告 | 1个工作日 |
| 4 | C3审批 | 1个工作日 |
| 5 | 通知供应商结果 | 当日 |

#### 阶段7：合同签约

**签约前须完成**：

| 事项 | 说明 |
|------|------|
| 合同条款确认 | 商务条款、服务条款、违约条款 |
| SLA签署 | 供应商确认服务等级协议 |
| 保密协议 | 签署数据保密协议 |
| 保证金缴纳 | 按合同约定缴纳履约保证金 |

#### 阶段8：正式上线

**上线前须完成**：

| 事项 | 说明 |
|------|------|
| 人员准入 | 完成人员信息录入、资质证书审核 |
| 制度报送 | 供应商报送内部管理制度 |
| 站点报备 | 报备职场地址、联系方式、对接人 |
| 分量分配 | 按初始分量规则分配产能 |
| 系统对接 | 完成京音、企微等系统账号开通 |

## 准入评分体系

### 评分维度与权重

| 维度 | 权重 | 满分 | 评分要素 |
|------|------|------|---------|
| **企业资质** | 15% | 100分 | 营业执照、资质证书、财务状况、合规记录 |
| **行业经验** | 20% | 100分 | 电销经验、金融经验、项目规模、客户评价 |
| **坐席规模** | 15% | 100分 | 现有坐席数、招聘能力、培训体系 |
| **职场条件** | 15% | 100分 | 职场面积、硬件设施、安全条件 |
| **数据安全** | 15% | 100分 | 安全管理体系、技术保障、物理隔离 |
| **合作意愿** | 10% | 100分 | 配合度、响应速度、投入承诺 |
| **成本优势** | 10% | 100分 | 报价合理性、成本结构、利润空间 |

### 准入判定标准

| 总分 | 判定 | 说明 |
|------|------|------|
| ≥85分 | 优秀 | 直接准入，优先分配产能 |
| 70-84分 | 合格 | 可以准入，正常分配 |
| 60-69分 | 待定 | 需C3审批，考虑是否有其他候选 |
| <60分 | 不通过 | 不符合准入标准 |

## 扩项管理

### 扩项类型

| 类型 | 说明 | 审批流 |
|------|------|--------|
| **业务线扩项** | 现有供应商增加新业务线 | C3审批 |
| **坐席扩项** | 现有供应商增加坐席规模 | C3审批（≤50人）/ C2审批（>50人） |
| **站点扩项** | 现有供应商新增站点 | C3审批 |
| **区域扩项** | 现有供应商进入新区域 | C3审批 |

### 扩项申请条件

| # | 条件 | 标准 |
|---|------|------|
| 1 | 当前评估等级 | 不低于B级 |
| 2 | 预警状态 | 无橙色及以上预警 |
| 3 | 集中度管控 | 扩项后仍符合集中度要求 |
| 4 | 专项评审 | 通过扩项专项评审 |

### 扩项申请实例：泉州速腾采购融资项目

**背景**：引入泉州速腾参与采购融资测试项目

**寻源策略**：从存量大额职场中寻源具备供应链金融经验的供应商

**选择依据**：泉州速腾服务网商银行，具备大额项目、供应链金融项目外呼经验

**人力规划**：

| 指标 | 目标值 |
|------|--------|
| 月均放款 | 60万 |
| 测试周期 | 3月1日-4月30日 |
| 新增人力 | 3人 |

**结算规则**：CPS=1.5%，结算金额=GMV×1.5%

**合规说明**：扩项人数在10人以内，符合小额零星采购标准

### 扩项与集中度管控的平衡

扩项必须检查是否违反集中度管控规则：

| 管控维度 | 上限 | 检查方法 |
|----------|------|---------|
| n-1原则 | 最多参与n-1个核心业务 | 检查扩项后是否超标 |
| 人工场景上限 | 金条≤360人/30% | 检查扩项后总人数 |
| AI场景上限 | ≤3家时≤70% | 检查扩项后份额占比 |
| 单一供应商上限 | 同一业务线≤40% | 检查扩项后总份额 |

## 供应商补充材料机制

### 触发场景

| 场景 | 说明 |
|------|------|
| 新供应商准入 | 需补充完整资质文件、场地证明、人员清单 |
| 供应商变更 | 法人变更、地址变更、名称变更 |
| 资质到期 | 证书续期、保险更新 |
| 站点变更 | 新增站点、搬迁站点、关停站点 |
| 人员变动 | 对接人变更、站点负责人变更 |

### 补充材料清单

| 材料类型 | 内容 | 提交时限 |
|----------|------|---------|
| 资质类 | 营业执照、资质证书、法人身份证明 | 变更发生后5个工作日内 |
| 场地类 | 租赁合同、消防安全证明、平面图 | 新增站点前15个工作日 |
| 人员类 | 花名册、持证清单、对接人信息 | 变动前3个工作日 |
| 合规类 | 信息安全承诺书、保密协议 | 签约前完成 |

### 补充材料通知模板

> **通知内容**：
> 1. 需要补充的材料清单
> 2. 提交方式和截止时间
> 3. 逾期不提交的后果（影响准入/扩项审批）
> 4. 联系人和联系方式

## 供应商引入的设计原则

### 质量优于速度

宁可慢一点，也要确保引入的供应商质量达标。试运行30天是底线，不能为了赶进度而压缩。

### 信息透明

供应商从引入第一天起就应当清楚了解管理要求、考核标准、合规红线。信息不对称是后续管理问题的根源。

### 分散风险

引入新供应商时，要考虑业务线分布、区域分布、规模分布，避免过度集中。

### 留有记录

每个阶段的评审结果、打分依据、决策记录都要存档。这是后续审计和追溯的依据。

## 供应商引入的演进方向

- **标准化评分工具**：将准入评分表数字化，实现系统打分
- **供应商画像**：基于历史数据构建供应商画像，辅助引入决策
- **自动预警**：系统自动检查集中度管控，超标时预警
- **AI辅助评估**：用AI辅助资质审核、风险评估，提高效率
