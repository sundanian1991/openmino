## Overview

本文整合了供应商管理制度体系中的基础信息、管理体系、校验报告、空间模板、制度速查卡、方法论关联矩阵、检查清单集、流程图集等内容。这些文档构成了供应商管理的"基础设施层"——从制度设计到落地执行的完整路径。

## 一、管理制度基础信息

### 管理体系架构

供应商管理制度体系由多个层级构成：

| 层级 | 文档类型 | 用途 |
|------|---------|------|
| 总纲 | 管理办法/总纲 | 供应商管理的总原则和框架 |
| 规范 | 管理规范 | 具体的管理要求和标准 |
| 手册 | 日常管理手册 | 操作层面的执行指南 |
| SLA | 服务等级协议 | 量化承诺和考核标准 |
| 模板 | 工作手册模板 | 填写和执行的具体格式 |

### 制度校验机制

**双重复核综合报告**：对制度体系进行完整性和一致性校验，确保各层级文档之间没有冲突、没有遗漏。

**校验确认要点**：
- 制度之间的逻辑关系是否清晰
- 模板是否与制度对应
- 流程是否闭环
- 角色职责是否明确

### 模板清单

| 模板 | 所属阶段 | 用途 |

| 交接清单填写模板 | 退出清退 | 清退交接流程 |

### 使用说明

工作手册模板的使用原则：
1. **先建框架** — 制度明确了再建模板
2. **最小化** — 能用一个字段解决的不用两个
3. **可追溯** — 每个填写项都有对应的制度依据
4. **可迭代** — 模板随制度优化而更新

## 三、空间模板与数据汇总表

### 工作空间使用指南

供应商独立工作空间的设计原则：

**先设计好结构再提要求**：
- 第一期只给3个表（人力明细、主管信息、整改计划）
- 跑通后再增加其他模板
- 供应商的配合度是逐步培养的

### 数据汇总表设计

汇总表的核心字段：
- 供应商基本信息（名称、业务线、对接人）
- 人力数据（总人数、熟手、新人、流失率）
- 业绩数据（GMV、达成率、排名）
- 合规数据（质检评分、A类错误、客诉）
- 整改数据（整改项、完成率、逾期数）

### 方法论关联矩阵

| 方法论 | 对应制度 | 落地工具 |
|--------|---------|---------|

| 引导式设问 | 日常管理手册 | 约谈框架 |

### 方法论速查卡

| 方法论 | 一句话定义 | 使用场景 |
|--------|-----------|---------|
| 排名思维 | 看相对排名，不看绝对分数 | 月度评估 |
| 引导式设问 | 你提问，他回答，你不替他想办法 | 约谈沟通 |
| 三要素法 | 原因+目标+举措，让供应商自己承诺 | 整改要求 |
| 双维度分析 | 自身峰值比+头部差距，区分内外因素 | 产能分析 |
| 标杆配对 | 同产线同规模精准对标 | 能力提升 |

## 五、流程图集与检查清单

### 核心流程图

供应商管理的主要流程：
1. **准入流程**：需求确认 → 资质审核 → 现场考察 → 试单评估 → 合同签署 → 正式准入
2. **日常管理流程**：数据监控 → 异常识别 → 约谈沟通 → 整改要求 → 效果验证
3. **清退流程**：清退决策 → 替代方案 → 供应商沟通 → 交接执行 → 结算收尾 → 档案归档

### 检查清单集

**准入检查清单**：
- [ ] 资质文件完整有效
- [ ] 场地符合要求（面积、设施、安全）
- [ ] 团队规模达标
- [ ] 管理团队稳定
- [ ] 无重大合规违规记录

**月度评估检查清单**：
- [ ] 数据已导出并核对
- [ ] 排名已计算并标记
- [ ] 触发规则已检查
- [ ] 异常供应商已约谈
- [ ] 整改计划已评估

**清退检查清单**：
- [ ] 清退依据已确认（合同/制度）
- [ ] 替代方案已准备
- [ ] 供应商已沟通
- [ ] 交接清单已签署
- [ ] 结算已完毕

## 六、审计与合规

### 审计要点

| 审计维度 | 检查内容 | 判定标准 |
|---------|---------|---------|
| 制度完整性 | 是否覆盖全生命周期 | 准入/日常/清退三阶段都有制度 |
| 制度一致性 | 各层级文档是否冲突 | 总纲→规范→手册逻辑一致 |
| 执行合规性 | 是否按制度执行 | 抽查约谈/整改记录 |
| 数据真实性 | 填报数据是否可信 | 随机核查+交叉验证 |

### 供应商补充材料通知

当供应商提交的材料不完整时，通过正式通知要求补充：
- 明确缺失项
- 给出补充期限
- 说明不补充的后果
