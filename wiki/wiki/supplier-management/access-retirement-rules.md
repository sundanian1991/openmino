## Overview

供应商准入与清退是全生命周期管理的两个端点。准入严控"入口关"（基础资质一票否决+9维度评分），清退分类处置（即时/预警/整改/尾部汰换），确保供应商池保持良性竞争、优胜劣汰。

## 准入规则

### 业务分类

| 类型 | 人力阈值 | 审批流 | 测试期 |
|------|---------|--------|--------|
| **核心业务** | >10人 | C3审核→C2终审 | 3-6个月 |
| **创新业务** | ≤10人 | C3终审 | ≤12周 |

### 基础资质（一票否决）

任一不满足，直接否决，不得进入后续流程：

| 序号 | 资质要求 | 验证方式 |
|------|---------|---------|
| 1 | 注册资本 ≥1000万元 | 营业执照+工商信息 |
| 2 | 《增值电信业务经营许可证》（全国性呼叫中心） | 许可证原件核验 |
| 3 | 营业执照覆盖拟合作业务范围 | 经营范围核对 |
| 4 | 资产负债率 ≤60% | 近一年财务报表 |
| 5 | 综合评分 ≥85分 | 9维度评估 |

### 业务能力要求

| 要求 | 标准 |
|------|------|
| 坐席规模 | ≥50人 |
| 业务经验 | 有营销型业务经验（电销/呼叫/在线营销） |
| 质量管控 | 独立质检团队或第三方质检协议 |
| 人员资质 | 金融相关资质持证率 100% |
| 数据安全 | 防护能力通过甲方评估 |
| 合规记录 | 无重大商业违规或法律诉讼 |

### 准入评分框架（9维度·10分制）

| 维度 | 权重 | 评分要点 |
|------|------|---------|
| 资质合规 | 15% | 注册资本、许可证、经营范围、资产负债率 |
| 项目经验 | 15% | 电销/呼叫/营销经验年限、甲方背景 |
| 招聘能力 | 12% | 招聘渠道、到岗速度、人才储备 |
| 培训体系 | 10% | 培训课程、达产周期、复训机制 |
| 质量管理 | 12% | 质检体系、合规自查、改善闭环 |
| 配合意愿 | 8% | 沟通响应速度、合作态度 |
| 报价合理性 | 10% | 成本结构、市场对标、性价比 |
| 技术能力 | 10% | 系统对接、数据安全保障 |
| 组织管理 | 8% | 管理团队稳定性、现场管理 |

综合评分 = Σ（各维度得分 × 权重），满分90分折算为百分制。入库门槛 ≥85分。

### 引入 SOP（8阶段·15工作日）

```
阶段1        阶段2        阶段3        阶段4        阶段5
需求发起 →  需求拆解 →  寻源触达 →  方案审批 →  结果同步
  C2         业管/供管    业管        供管        供管
  T+0        T+1         T+3         T+5         T+5

阶段6        阶段7        阶段8
供应商准备 → 内部操作 →  培训上线
  供应商     业管/供管    供应商+业管
  T+10       T+12        T+15
```

| 阶段 | 输出物 | 验收标准 |
|------|--------|---------|
| 1.需求发起 | 需求清单 | 需求清晰、可量化 |
| 2.需求拆解 | 寻源需求表 | 需求具体化为供应商画像 |
| 3.寻源触达 | 意向名单 | ≥3家有效回复 |
| 4.方案审批 | xbp审批通过方案 | 集中度检查通过 |
| 5.结果同步 | 结果同步邮件 | 各方均收到通知 |
| 6.供应商准备 | CID+站点+合同确认 | 三项准备完成 |
| 7.内部操作 | 京音系统建档 | 账号可用、权限正确 |
| 8.培训上线 | 上线外呼+持证率100% | 正式上线 |

### 新供应商 6 个月里程碑

| 阶段 | 时间 | 核心目标 | 量化检查点 |
|------|------|---------|-----------|
| 引入期 | 第1-2月 | 出勤稳定、合规达标 | 出勤率≥85%；管理制度7模块提交 |
| 成长期 | 第3-4月 | 产能爬坡 | GMV达同产线中位数60% |
| 稳定期 | 第5-6月 | 排名进入前70% | 月度排名前70%；主管胜任≥3项达标 |
| 评判点 | 第6月末 | 留或退 | 未达前70% → 延长或清退 |

### 站点设立要求

| 项目 | 要求 |
|------|------|
| 选址标准 | 商业写字楼；消防合格；人均≥3㎡ |
| 新增门槛 | 核心业务≥20人/站；创新业务≥5人/站 |
| 审批时限 | 3工作日 |

## 清退规则

### 清退分类

| 类型 | 触发层级 | 处置力度 |
|------|---------|---------|
| 即时清退 | 合规红线 | 立即终止全部业务，永久禁入 |
| 预警清退 | 连续2月E级 | 5个工作日内决议 |
| 整改清退 | 整改期满未达标 | 最长60天整改期 |
| 尾部快速汰换 | 长期低效+低规模 | 1月观察期后汰换 |
| 单业务线退出 | 单一产线不达标 | 退出该业务线，保留其他 |
| 站点缩减 | 人力持续低于缩点线 | 合并或关停站点 |

### 清退触发条件矩阵

| 清退类型 | 触发条件 | 时效 |
|----------|---------|------|
| 即时清退 | 信息安全泄露/数据造假/录音篡改/假证借证 | 触碰后48小时内决议 |
| 预警清退 | 连续2个月评估等级为E级 | 5个工作日内决议 |
| 整改清退 | 整改期满（最长60天）未达标 | 期满即判定 |
| 尾部快速汰换 | 合作>12月+连续2季后30%+日均出勤<30人 | 自动识别→1月观察期 |
| 站点缩减 | 连续2月低于缩点线+1月缓冲期未达标 | 缓冲期满次日确认 |
| 全业务线强制退出 | 重大服务事故/信息泄露/违规转分包 | 即时 |

### 缩点触发线

| 业务线 | 缩点触发线（连续2月） | 限留1个站点线 |
|--------|---------------------|--------------|
| 金条人工 | <60席 | <40席 |
| 金条人机 | <40席 | <30席 |
| 菁卡拉新 | <30席 | <20席 |
| 企金/固收 | <30席 | <20席 |

### 标准退出流程

| 步骤 | 动作 | 时限 | 责任人 |
|------|------|------|--------|
| 1 | 提交退出/清退申请 | — | 供应商/供管岗 |
| 2 | 评估影响范围，起草交接方案 | — | 供管岗 |
| 3 | 审批：C3→C2终审 | C3≤1天，C2≤2天 | C3/C2 |
| 4 | 发出退出通知，进入过渡期 | — | 供管岗 |
| 5 | 业务交接、人员安置、数据清理 | 30天 | 供应商+供管岗 |
| 6 | 正式终止合作 | — | 供管岗 |

### 退出交接

| 阶段 | 核心任务 |
|------|---------|
| 准备阶段 | 确认停呼时间、停发数据日期 |
| 执行阶段 | 数据盘点、数据回收、企微转移 |
| 收尾阶段 | 账号关闭、结算处理、客诉1月专项跟踪 |

### 清退结算

- 最后合作业务月结算款 **压3个月再结**
- 标准结算周期：T+2月

### 退出后管理

| 退出类型 | 退出后管理 |
|----------|-----------|
| 单业务线退出 | 进入备选池；3个月禁入该业务线 |
| 全业务线强制退出 | 永久列入禁止合作名单 |
| 尾部快速汰换 | 同单业务线退出规则 |
| 站点缩减 | 供应商保留其他站点合作 |

所有退出供应商保密义务延续 **2年**。

## 特殊场景

### 紧急引入流程

| 条件 | 审批 | 时效 |
|------|------|------|
| 重大业务缺口，48小时内须上线 | C2审批 | 压缩至7个工作日，事后补录 |

### 快速通道（分量调整 ≤10人）

向 C3 京ME 备案，无需事前审批，5个工作日内完成。

### 供应商主动退出

须提前 **30个自然日** 书面通知，过渡期30天（可协商缩短）。
