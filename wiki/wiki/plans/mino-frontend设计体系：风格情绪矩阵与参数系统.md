## 概述

mino-frontend 是年老师专属的前端设计技能，经历了从14种经验分类风格到「风格乘以情绪」二维决策矩阵的架构升级，并建立了三层参数系统来解决输出不稳定的问题。核心洞察是：方法论是骨架，参数系统是血肉。骨架相同但血肉不同，输出就不稳定。本文记录了风格矩阵的演变过程、参数系统的完整设计以及跨技能设计原则体系的构建。

## 问题诊断：为什么方法论升级后仍然不满意

年老师在 mino-frontend 的迭代过程中经历了多次方法论升级，但每次都不够满意：

| 阶段 | 方法论 | 反馈 |
|------|--------|------|
| V1 | 跳过流程 | 形似神不似 |
| V2 | 三区模型（标题/内容/结论） | 空间上的呈现不是很有方法论的感觉 |
| V3-V4 | 均匀压缩间距 | 效果更差了 |
| V5 | 视觉层级法（层级数+权重） | 不够专业 |
| 最新 | 内容-位置-权重三步法 | 一直没有特别满意 |

**核心问题定位：** 方法论是骨架，但骨架需要附着在具体的身体上。缺失的是视觉血肉——具体的设计语言参数。边框用多细？字体层级差多少？信息密度多高？专业到底是什么感觉？

**核心张力：** 不同场景的专业是不同的视觉语言。年老师没有明确过他想要的专业是什么风格。每次调方法论，都是在调骨架，没有调具体的视觉参数。

## 风格乘以情绪决策矩阵

### 核心模型

将14种经验分类风格简化为二维决策矩阵：

```
            情绪（内容气质）
         紧迫    中性    庆祝    沉思
       +-------+-------+-------+-------+
  数据 | 数据模式| 数据模式| 金融时报| 数据模式|
       | (红)  | (灰)  | (青绿) | (深灰)|
       +-------+-------+-------+-------+
  品牌 | 自信宣言| 品牌模式| 品牌模式| 纸墨文学|
       | (黑金)| (陶土)| (陶土)| (墨色)|
       +-------+-------+-------+-------+
  咨询 | 咨询模式| 咨询模式| 咨询模式| 咨询模式|
       | (红)  | (灰)  | (蓝)  | (灰)  |
       +-------+-------+-------+-------+

风格类型（设计系统）
```

**关键洞察：** 风格类型等于设计系统（数据、品牌、咨询三种基础），情绪等于Header颜色变体（红、灰、陶土、青绿、金六种颜色），组合等于3乘以4等于12种，比14种更简洁，且有逻辑。

### 情绪颜色映射

| 情绪标签 | Header颜色 | 使用场景 |
|---------|-----------|---------|
| 紧迫/警示 | 深红 #8B0000 | 处罚、风险、异常 |
| 中性/专业 | 炭黑 #111 | 数据通报、常规汇报 |
| 庆祝/激励 | 陶土 #E2725B | 业绩达成、激励海报 |
| 权威/正式 | 青绿 #0d7680 | 金融数据、战略汇报 |
| 奢华/高端 | 金 #D4AF37 | 颁奖、年度盛典 |

### 决策流程

内容到提取情绪关键词到匹配情绪标签到确定Header颜色到选择基础风格。

### 14种风格简化方案

| 基础风格 | 核心特征 | 情绪变体 |
|---------|---------|---------|
| 数据模式 | 表格、数字、黑白灰 | 紧迫/中性/庆祝/权威 |
| 品牌模式 | 陶土色、手绘图标、人文感 | 紧迫/中性/庆祝/权威 |
| 咨询模式 | So What结论、结构化 | 紧迫/中性/庆祝/权威 |
| 金融时报 | 青绿、KPI网格、信赖感 | 紧迫/中性/庆祝/权威 |
| 极简宣言 | 黑金/黑陶土、大字号、冲击力 | 紧迫/中性/庆祝/权威 |

删除的风格：现代工坊、分类标签、柔和几何、趣味拼接、复古报刊、纸墨文学、Anthropic品牌。原因：与上述5种重叠或过于细分。

## 统一灰阶系统

所有风格共用一套 6 级灰阶，只区分强调色。解决了 14 种风格各自定义灰度导致的视觉不一致问题。

| 灰阶级别 | 色值范围 | 用途 |
|----------|----------|------|
| 最深 | #1a1a1a-#2a2a2a | 主标题、关键数据 |
| 深灰 | #4a4a4a-#5a5a5a | 副标题、正文 |
| 中灰 | #7a7a7a-#8a8a8a | 辅助文字、图例 |
| 浅灰 | #b0b0b0-#c0c0c0 | 分隔线、边框 |
| 极浅 | #e0e0e0-#e8e8e8 | 背景、填充 |
| 最浅 | #f5f5f5-#fafafa | 底色、留白区域 |

## 三层参数系统

### 参数分层

**P0 参数（必填）**：风格类型、情绪标签、页面数量
**P1 参数（推荐）**：Header 颜色、强调色、图表类型推荐
**P2 参数（可选）**：动画复杂度、图标密度、数据密度

参数系统战略对齐审查全部通过：目标清晰度（执行不稳定→建立参数系统）、假设可证伪性（三个核心假设均可验证）、范围比例性（P0 60%、P1 30%、P2 10%）。

### 架构

```
Layer 1：全局参数（所有风格共享的基准参数）
  - 色彩系统（主色、辅助色、陶土色）
  - 间距基准（4pt 系统）
  - 响应式断点

Layer 2：风格参数（每种设计风格独有的参数）
  - 字体层级（T1-T6 px 值）
  - 边框系统（粗细、圆角）
  - 间距比例（页面 padding、卡片 gap）
  - 陶土色使用规则

Layer 3：内容参数（由内容特征决定的参数）
  - 信息密度（高/标准/低）到 间距调整
  - 情绪气质（紧迫/中性/庆祝/沉思）到 陶土色用量
  - 阅读方式（扫视/阅读/凝视）到 字号层级差
```

**参数优先级：** 内容参数大于风格参数大于全局参数。

### 色彩系统（克制专业主义）

```css
:root {
  --text-primary: #111111;
  --text-secondary: #666666;
  --text-tertiary: #999999;
  --bg: #FFFFFF;
  --surface: #FAFAFA;
  --border: #E5E5E5;
  --accent: #E2725B;
  --accent-light: rgba(226, 114, 91, 0.08);
}
```

陶土色使用规则：一页一处陶土色强调（紧迫情绪可放宽到2-3处），不用于装饰性元素，不用于正文文字。用途优先级：异常/风险大于排名/等级大于行动建议。

### 字体层级系统

| 层级 | 用途 | 基准值 | clamp 值 | 层级差 |
|------|------|--------|----------|--------|
| T1 | 页面标题 | 28px | clamp(1.75rem, 4vw, 2rem) | 基准 |
| T2 | 模块标题 | 18px | clamp(1.125rem, 2vw, 1.25rem) | -10px |
| T3 | 卡片标题 | 14px | clamp(0.875rem, 1.5vw, 1rem) | -4px |
| T4 | 正文 | 13px | clamp(0.8125rem, 1.3vw, 0.9rem) | -1px |
| T5 | 辅助文字 | 11px | clamp(0.6875rem, 1vw, 0.75rem) | -2px |
| T6 | 页码/标签 | 10px | clamp(0.625rem, 0.8vw, 0.7rem) | -1px |

层级差原则：T1到T2大跳跃（-10px）区分页面标题和模块标题，T2到T3中跳跃（-4px）区分模块和卡片，T3到T4小跳跃（-1px）卡片内层级，T4到T5中跳跃（-2px）正文和辅助，T5到T6小跳跃（-1px）辅助和标签。

### 边框系统

| 元素 | 边框粗细 | 边框颜色 | 圆角 |
|------|---------|---------|------|
| 卡片 | 1px | #E5E5E5 | 4px |
| 表头分割线 | 2px | #111111 | 0 |
| 表格行分割线 | 1px | #E5E5E5 | 0 |
| 顶部装饰线 | 2.4px | #E2725B | 0 |
| 强调卡片 | 1px | #E2725B | 4px |

### 间距系统

| 密度 | 页面 padding | 卡片内 padding | 卡片 gap | 区块 gap |
|------|-------------|---------------|----------|---------|
| 高密度 | 24px | 12px | 12px | 16px |
| 标准密度 | 32px | 16px | 16px | 24px |
| 低密度 | 48px | 24px | 24px | 32px |

密度映射由情绪和阅读方式决定：紧迫乘以扫视选择高密度，中性乘以阅读选择标准密度，中性乘以凝视选择低密度，庆祝乘以凝视选择低密度，沉思乘以凝视选择低密度。

### 陶土色使用场景

| 场景 | 使用方式 | 面积限制 |
|------|---------|---------|
| 顶部装饰线 | 2.4px 实线 | 不超过5px高 |
| 页码徽章 | 24px圆形填充 | 不超过24乘以24px |
| 强调卡片边框 | 1px 实线 | 卡片周长 |
| 强调卡片背景 | rgba(226,114,91,0.08) | 卡片面积 |
| 列表项圆点 | 4px 圆点 | 不超过4px |
| 关键词强调 | 文字颜色 | 仅关键词 |

## better-icons 整合方案

整合流程：内容分析到风格选择到图标需求识别到调用 better-icons search 关键词到嵌入 SVG inline 到生成 HTML。

图标使用原则（克制专业主义）：无图标大于有图标，简单几何图标大于复杂插画，手绘风格大于矢量线条，单色大于多色。

## 跨技能设计原则体系

跨技能设计原则体系整合了 mino-frontend、better-icons、chart-visualization 等多个设计技能的通用原则：

### 第一性原理（3条）
1. **克制 > 装饰**：每个元素必须有目的，不为装饰而装饰
2. **锐度 > 模糊**：边界清晰、焦点明确，不模棱两可
3. **诚实 > 套路**：反 AI 味，不滥用阴影、圆角、玻璃拟态

### 信息设计铁律（5条）
1. 一页一核心信息
2. 空白是设计的一部分
3. 强调色 ≤ 10% 画面
4. 无装饰性元素
5. 50% 视觉率

### 反 AI 味检查清单（6条）
1. 无阴影效果
2. 无圆角 premium cards
3. 无玻璃拟态
4. 无粒子背景
5. 无渐变 hero
6. 无打字机效果

### 默认决策流程（6步）
1. 确定内容和目标 → 2. 选择风格和情绪 → 3. 确定信息密度 → 4. 选择构图类型 → 5. 选择入场动画 → 6. 质量检查（反AI味清单）

**克制优先**：少即是多，不装饰不必要的内容。

## 重构后的架构

```
mino-frontend v5.0
+-- 基础系统
|   +-- 统一灰阶（6级）
|   +-- 情绪颜色（5种）
|   +-- 基础风格（5种）
+-- 决策矩阵
|   +-- 内容到情绪识别到基础风格选择到Header颜色到生成
+-- 图标系统
|   +-- 识别需求到 better-icons search 到 SVG嵌入
+-- 扩展风格（可选）
    +-- 复古报刊、纸墨文学等（不推荐但保留）
```

## 核心洞察

**从经验分类到决策逻辑**：14 种风格是经验的产物（"我做过这些"），3×4 矩阵是逻辑的产物（"这些组合覆盖所有场景"）。前者不可扩展，后者可无限扩展。

**参数系统解决确定性问题**：执行不稳定的本质是确定性不足。P0 必填参数确保每次生成都有确定起点，不确定性留给 P2 而非 P0。
