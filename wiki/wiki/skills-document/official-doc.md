# 公文编写

## 概述

撰写专业商务公文，基于场景选择合适模板，确保内容规范、逻辑清晰、专业得体。

## When to Use

```
需要撰写公文？
    ├─ 向上汇报？ → 使用 formal-report 模板
    ├─ 发通知？ → 使用 notification 模板
    ├─ 跨部门协作？ → 使用 collaboration 模板
    ├─ 内部沟通？ → 使用 internal 模板
    └─ 供应商管理？ → 选择对应生命周期模板
```

**Use when:**
- 撰写正式公文（报告、通知、协作函）
- 供应商管理文档（准入、运营、评估、清退、结算）
- 内部协调文档（决策请示、会议邀请、进度同步）
- 合规风险文档（培训通知、案例通报、风险预警）

**Don't use for:** 个人闲聊文档、非正式沟通

## Remix 模式（快速变体生成）

> **核心理念**：基础模板 → 4-6个语气变体 → 快速适配不同场景

### 语气变体矩阵

| 变体 | 特点 | 适用场景 | 关键词示例 |
|------|------|---------|-----------|
| **正式** | 严谨、规范、有据可依 | 向上审批、决策请示、供应商正式通知 | "根据规定"、"综合评估"、"特此通知" |
| **柔和** | 委婉、协商、留有余地 | 跨部门协作、供应商协调、问题沟通 | "建议考虑"、"是否可以"、"麻烦确认" |
| **紧急** | 直接、明确、限时要求 | 故障通报、风险预警、时限任务 | "紧急"、"请于X时前"、"立即处理" |
| **协商** | 平等、双赢、共同目标 | 商务谈判、激励方案、合同讨论 | "共同探讨"、"双赢方案"、"建议调整为" |
| **告知** | 客观、信息传递、无需反馈 | 信息同步、结果公布、数据通报 | "特此告知"、"供参考"、"无需回复" |
| **质询** | 严肃、要求解释、追责 | 违约问询、异常情况、整改要求 | "请说明"、"请解释"、"将采取" |

### Remix 工作流

```
1. 确定场景（向上/向下/平级/供应商）
2. 选择基础模板
3. 生成 4-6 个语气变体
4. 根据具体情况选择最合适的变体
5. 二次微调（补充具体数据、风险说明）
```

### 示例：结算通知公文的6种变体

| 变体 | 开头示例 | 使用时机 |
|------|---------|---------|
| 正式 | "根据合同约定，现就X月结算事宜通知如下..." | 标准月度结算 |
| 柔和 | "关于X月结算，有个小问题需要和您确认一下..." | 结算金额有差异需要协商 |
| 紧急 | "【紧急】X月结算务必于今日17点前确认，否则将影响付款..." | 逾期未确认、影响付款时限 |
| 协商 | "关于X月结算金额，我们建议调整为...，您看是否可以？" | 结算争议、需要协商 |
| 告知 | "X月结算已完成，明细如下，供参考..." | 结算已完成、无需对方确认 |
| 质询 | "X月结算数据存在异常，请于X时前说明原因..." | 数据异常、需要解释 |

## 写作标准

### 公文写作规范（专业简洁，空间舒适）

**核心原则：段落逻辑优先，视觉装饰为零，适度留白舒适**

| 正确做法 | 错误做法 |
|----------|----------|
| 段落开始用**加粗关键词**区分 | 用分割线/边框等装饰 |
| 表格用于数据展示 | 用表格做视觉分隔 |
| 段落靠内在逻辑推进 | 依赖分段/标题堆叠 |
| 适度留白保持阅读节奏 | 段落过密或过疏 |

**标准格式示例：**

```markdown
尊敬的XXX：

一句话概括公文内容。

**事实部分：**

具体事实描述，用列表或表格呈现关键数据（时间、人员、数据）。

**分析判断：**

基于事实的分析，使用"综合评估认为"、"这说明"等专业表述。

**建议/结论：**

明确的建议或结论，附上理由和依据。

**风险提示/后续安排：**

必要的风险说明或后续行动步骤。

结尾总结句和行动请求（请审阅/请批示/如有问题请反馈）。
```

**6条具体规则**：

1. **开头称呼**：称呼单行，空一行后接主体
2. **关键词标识**：每个段落开始用`**关键词：**`标识
3. **禁止装饰**：不使用分割线、边框等视觉装饰
4. **数据结构化**：时间、人员、数据等用列表/表格清晰呈现
5. **留白适度**：段落之间空一行，保持阅读节奏
6. **结尾行动**：明确的行动请求或请示

### 标题规范

- 以"关于..."开头，明确主题
- 简洁具体，避免模糊
- 时效性内容需包含时间范围

### 内容结构

1. **开头**：称呼+目的
2. **事实部分**：列表/表格呈现（时间、人员、数据等）
3. **过渡句**：段落间用"基于上述..."、"综合评估..."等连接
4. **分析判断**："我们认为..."、"这说明..."等加入观点
5. **结尾**：明确请求（"请审阅"、"如有问题请反馈"）

### 格式技巧

- 结构化数据用列表/表格
- 密集数据用表格呈现
- 叙述部分用段落+过渡句
- **重点内容用加粗**
- 明确的行动请求（Call to Action）
- 段落间使用过渡句
- 包含分析判断，不只是事实陈述

### 语调要求

- 专业正式，不随意
- 清晰直接，不机械
- 逻辑严密，结构清晰

## Common Mistakes

| ❌ 错误 | ✅ 修正 |
|-----------|-------|
| 模糊标题如"汇报" | 具体："关于供应商Q1评估汇报" |
| 没有行动请求 | 添加："请于周五前反馈" |
| 全文加粗 | 仅加粗关键项 |
| 缺少数据支撑 | 补充具体数据、案例 |
| 观点模糊 | 给出明确判断和依据 |

一是投诉率为0，二是无重大违规记录，三是月度考核为B级以上。

## 各阶段审计检查点

### 阶段 1：需求发起
- 《需求清单》是否存档
- 需求是否有数据支撑
- 是否有策略岗负责人签字/确认

### 阶段 2：需求拆解
- 《寻源需求表》是否存档
- 是否有现有供应商评估记录
- 外部寻源决策是否有依据

### 阶段 3：寻源触达
- 寻源邮件是否存档（简单版/详细版）
- 回复供应商名单是否记录
- 筛选标准是否客观、可追溯
- 验收标准：≥3 家有效回复，有明确意向

### 阶段 4：方案审批
- xbp 方案是否存档
- 审批流程是否完整
- 领导审批意见是否记录
- 补充材料是否完整

### 阶段 5：结果同步
- 《结果同步邮件》是否存档
- 发送名单是否完整（策略岗、业管、供应商）
- 供应商通知是否规范（入选/未入选分别发）

### 阶段 6：供应商准备
- CID 信息是否存档
- 站点报备是否有记录
- 合同模板是否确认并存档
- 资质文件是否完整

### 阶段 7：内部操作
- 京音系统是否有供应商档案
- 账号开通记录是否存档
- 权限配置是否符合规范
- 是否有账号测试记录

### 阶段 8：培训上线
- 培训记录是否存档
- 培训考核结果是否记录
- 上线外呼时间是否记录
- 首日质量报告是否存档

## 集中度检查

引入新供应商时必须检查：
- 当前 TOP3 供应商占比
- 引入后 TOP3 供应商占比
- 是否符合集中度要求

## CID 信息字段

公司全称、简称、统一社会信用代码、注册地址、实际经营地址、法人代表、法人身份证号、联系人（姓名/职务/电话/邮箱）、公司官网、成立时间、注册资金、员工规模、主营业务、现有合作甲方。

# 供应商选择决策框架

> 用途：规范供应商选择流程，确保决策可追溯、可解释
> 更新日期：2026-03-21

---
## 一、核心原则
### 1.1 决策可追溯原则
- 每次供应商选择必须有书面决策记录
- 记录必须包含：选择背景、评估维度、决策依据、审批意见
### 1.2 评估维度多元化原则
- **头部供应商**（排名前30%）：综合排名 + 专项能力 + 合作意愿
- **中部供应商**（排名30%-70%）：业绩趋势 + 改进意愿 + 成本优势 + 资源可用性
- **尾部供应商**（排名后30%）：特殊情况说明 + 风险预案 + 成本效益
### 1.3 集中度管理原则
- 同一业务线避免过度集中（单一供应商≤40%分量）
- 优先选择跨业务线供应商（提升协同效应）
- 区域分布合理性（风险分散）

## 三、不同场景的选择策略
### 3.1 头部供应商选择（排名前30%，约前10名）
**选择依据优先级：**
1. 综合排名（权重40%）：Q[季度]评估排名
2. 专项能力（权重30%）：与任务相关的历史表现
3. 合作意愿（权重20%）：响应速度、主动配合度
4. 资源可用性（权重10%）：是否有充足人力
**审计解释要点：**
- "XXX供应商Q[季度]排名第X，综合表现优秀"
- "该供应商在XXX业务上有历史成功案例"
- "供应商主动承接，配合度高"
### 3.2 中部供应商选择（排名30%-70%，约第11-22名）
**选择依据优先级：**
1. **业绩趋势**（权重30%）：近3个月趋势（↗优先）
2. **改进意愿**（权重25%）：是否积极响应整改、主动优化
3. **成本优势**（权重20%）：单价是否具有竞争力
4. **资源可用性**（权重15%）：是否立即可用
5. **合作历史**（权重10%）：过往任务完成情况
**审计解释要点：**
- "XXX供应商近3个月业绩持续上升，排名第X→X→X"
- "该供应商积极配合XXX整改，改进意愿强"
- "单价较市场平均低X%，成本优势明显"
- "当前有XX席空闲产能，可立即承接"
### 3.3 尾部供应商选择（排名后30%，约第23-32名）
**选择依据优先级：**
1. **特殊情况说明**（权重40%）：必须说明为何选择尾部供应商
2. **成本效益**（权重30%）：是否为唯一可行方案
3. **风险预案**（权重30%）：是否有备选方案
**特殊情况包括：**
- 头部/中部供应商均无资源
- 任务需要特殊资质（仅尾部供应商具备）
- 成本压力下的预算约束
- 区域覆盖要求（特定地区仅有尾部供应商）
**审计解释要点：**
- "因XXX原因，头部/中部供应商均无可用资源"
- "XXX供应商具备XXX特殊资质，是唯一可行方案"
- "单价仅为XX元/通，符合预算要求"
- "已制定风险预案：XXX供应商为备选，可随时切换"

**标准回答：**
"资源分配基于以下评估框架：
1. **量化评分**：业绩40分+合作30分+资源20分+策略10分

## 写法要点
### 一、本周重点进展（必填）
**适合描述的场景**：
- 商务谈判进展（价格、合同、激励方案）
- 供应商沟通（准入、清退、争议处理）
- 评估考核完成（季度排名、月度评估）
- 分量调整完成（增量、减量、调配）

**写法示例**：
```

● 具体进展：已完成X家供应商评估结果通知
● 关键数据：A级X家、B级X家、C级X家
● 当前状态：供应商确认中
```
### 二、需支持/卡点事项（按需填写）
**适合描述的场景**：
- 需要上级决策的事项（清退、重大处罚、合同变更）
- 需要跨部门协调的事项（法务、财务、合规）
- 供应商配合度低、进展缓慢
- 资源冲突、时间紧张
**写法示例**：
```
1. 博彦清退决策
● 卡点情况：该供应商连续2个月C级，但现有100席需承接
● 需要的支持：是否批准清退？备选方案为新动力（80席）

● 卡点情况：汇讯补充协议已提交X天，仍未收到评审意见
● 需要的支持：协调法务部门X月X日前完成评审

```
### 三、下周重点计划（必填）

**填写流程**：

2. 再写需支持事项（有就写，没有就写"无"）

4. 风险提示按需填写

## 注意事项
### 周报撰写原则
- **简洁直接**：每项3-5行，1-2分钟可读完
- **数据说话**：用具体数字和事实，不用模糊描述
- **突出卡点**：需支持事项要清晰具体，明确诉求
- **风险意识**：及时提示需要关注的风险
### 常见错误
- ❌ 写成流水账（做了很多琐事）
- ✅ 只写重点进展（核心成果）
- ❌ 没有数据支撑（"进展顺利"、"沟通良好"）
- ✅ 有具体数据（"完成X家沟通"、"达成XX协议"）
- ❌ 卡点描述模糊（"需要协调"）
- ✅ 卡点描述具体（"需要法务X日前评审"）
### 时间控制
- 1-2分钟读完 = 约200-300字
- 最多3项重点进展 + 2项需支持事项
