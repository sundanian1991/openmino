## 概述

公文编写技能覆盖正式公文、供应商管理文档、内部协调、合规风险等 40+ 场景模板，100+ 具体场景。核心理念：**段落逻辑优先，视觉装饰为零，适度留白舒适**。通过场景选择基础模板 → 6 种语气变体适配 → 微调数据完成写作。

## Remix 模式：语气变体适配

每个基础模板可生成 6 种语气变体，快速适配不同场景：

| 语气 | 特点 | 关键词 | 典型场景 |
|------|------|--------|----------|
| **正式** | 严谨、规范、有据可依 | "根据规定"、"综合评估"、"特此通知" | 向上审批、供应商正式通知 |
| **柔和** | 委婉、协商、留有余地 | "建议考虑"、"是否可以"、"麻烦确认" | 跨部门协作、问题沟通 |
| **紧急** | 直接、明确、限时要求 | "紧急"、"请于X时前"、"立即处理" | 故障通报、风险预警、时限任务 |
| **协商** | 平等、双赢、共同目标 | "共同探讨"、"双赢方案" | 商务谈判、激励方案 |
| **告知** | 客观、信息传递、无需反馈 | "特此告知"、"供参考"、"无需回复" | 信息同步、结果公布 |
| **质询** | 严肃、要求解释、追责 | "请说明"、"请解释"、"将采取" | 违约问询、异常情况、整改要求 |

### Remix 工作流

```
1. 确定场景（向上/向下/平级/供应商）
2. 选择基础模板
3. 生成 6 种语气变体
4. 根据具体情况选择最合适变体
5. 二次微调（补充具体数据、风险说明）
```

## 写作标准

### 公文结构

标准公文遵循五段式结构：

1. **开头**：称呼 + 一句话概括目的
2. **事实部分**：用列表/表格呈现关键数据（时间、人员、数据）
3. **分析判断**：基于事实的分析，使用"综合评估认为"、"这说明"等专业表述
4. **建议/结论**：明确建议或结论，附理由和依据
5. **结尾**：明确行动请求（"请审阅"/"请批示"/"如有问题请反馈"）

### 格式规则

| 规则 | 说明 |
|------|------|
| 标题 | 以"关于..."开头，明确主题，时效性内容包含时间范围 |
| 段落标识 | 段落开头用 `**关键词：**` 标识（如 **事实：**、**风险：**） |
| 禁止装饰 | 不使用分割线、边框等视觉装饰元素 |
| 数据结构化 | 时间、人员、数据用列表/表格呈现 |
| 留白 | 段落之间空一行，保持阅读节奏 |
| 重点标注 | 仅关键内容加粗，不全文加粗 |
| 结尾行动 | 必须有明确的行动请求或请示 |

### 常见错误

| 错误 | 修正 |
|------|------|
| 模糊标题如"汇报" | 具体："关于供应商Q1评估汇报" |
| 没有行动请求 | 添加："请于周五前反馈" |
| 全文加粗 | 仅加粗关键项 |
| 缺少数据支撑 | 补充具体数据、案例 |
| 观点模糊 | 给出明确判断和依据 |

### 语调

专业正式、清晰直接、逻辑严密。不说"看情况"，给明确判断。事实与判断分开陈述——先摆事实，后给分析。
