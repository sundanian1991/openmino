## 概述

本文档深入剖析供应商管理中最核心的两项能力：**分量策略**（把合适的供应商放在合适的地方）和**拿捏能力**（让供应商在有限奖励下依然愿意投入）。这两项能力是供应商管理岗位最不可替代的核心竞争力，也是从"管人"到"经营生态"的关键跃迁。

---

## 一、分量策略：最不可替代的能力

### 1.1 本质

分量策略的本质不是"分蛋糕"，而是**把合适的供应商放在合适的地方，让每家都能产出最大价值**。

### 1.2 供应商画像维度

对每家供应商建立多维画像：

| 维度 | 分类 | 用途 |
|------|------|------|
| 配合度 | 高 / 中 / 低 | 决定信任度和信息开放程度 |
| 业务结构 | 头部 / 中部 / 尾部 | 决定分量权重和期望值 |
| 学习能力 | 强 / 中 / 弱 | 决定新项目试水意愿 |
| 业务经验 | 拉新 / 企微 / 大额 / 创新型 | 决定业务匹配度 |
| 上人速度 | 快 / 中 / 慢 | 决定应急响应能力 |
| 历史表现 | 稳定 / 波动 | 决定风险预判 |

### 1.3 分量打分模型

**5 维打分模型**（日常分量决策使用）：

| 维度 | 权重 | 说明 |
|------|------|------|
| 业绩 | 30% | 最核心的指标，业绩决定一切 |
| 质检客诉 | 25% | 合规底线，触碰红线一票否决 |
| 上人数字 | 15% | 产能保障，能否按时上人 |
| 配合度 | 15% | 合作意愿，是否积极响应 |
| 战略打分 | 15% | 扶持潜力供应商、学习能力、额外技能 |

### 1.4 决策五要素

每次分量决策需要综合考量：

1. **业务类型**：拉新 vs 复购，不同业务需要不同类型的供应商
2. **业绩情况**：近期表现如何，趋势是上升还是下降
3. **上人速度**：多快能上人，紧急需求需要快速响应能力
4. **历史配合**：过往配合度，是否值得信赖
5. **战略分配**：调整供应商结构的需求，是否需要扶持新商、压制老商

### 1.5 动态管理原则

- **不都给，也不一直不给**——让供应商保持期待
- **头部和尾部重点管理**——头部保产能，尾部造竞争
- **中间供应商可以激励成头部**——标杆打造，给中游看到上升通道
- **季度内至少一个月进入综合排名前三**——保持竞争张力

### 1.6 网状结构思维

供应商不是线性排列，而是多维分类的网状结构。同一供应商在不同业务线有不同的位置：

```
         拉新型          企微型
            |              |
     头部：A、B        头部：E、F
     中部：C、D        中部：G、H
     尾部：I、J        尾部：K、L

         大额型          创新型
            |              |
     头部：M、N        头部：Q、R
     中部：O、P        中部：S、T
     尾部：U、V        尾部：W、X
```

分量策略需要综合考虑供应商在多个维度上的表现，而不是简单看一个总分。

## 二、拿捏能力：让供应商在有限奖励下依然愿意投入

### 2.1 核心逻辑

**胡萝卜 + 大棒，业绩论英雄。**

### 2.2 胡萝卜（正向激励）

| 激励手段 | 具体做法 | 效果 |
|----------|---------|------|
| 分量奖励 | 业绩好就有更多量 | 最直接、最有效的激励 |
| 新项目优先 | 新项目优先考虑表现好的供应商 | 给未来预期，锁定长期投入 |
| 信息同步 | 告知怎么决策的、为什么 | 增加透明度和信任感 |
| 内部规划透露 | 透露内部规划方向（但不详细） | 让供应商有准备、有预期 |

### 2.3 大棒（反向约束）

| 约束手段 | 触发条件 | 效果 |
|----------|---------|------|
| 频繁复盘问询 | 不听话、不配合 | 施加管理压力 |
| 新项目不找 | 挑肥拣瘦、只做"好做的业务" | 切断增长预期 |
| 分量削减 | 业绩差 | 直接影响收入 |
| 限制新进 | 整改期间不允许接新业务、不允许新增人力 | 让供应商"只能流失不能补充" |
| 清退 + 拉黑 | 严重违规 | 终极威慑 |

### 2.4 独特之处

- **以业绩论英雄**：不管背景、不管历史关系，只看数据
- **规则运行在"阳光"下**：相对透明，供应商知道规则、知道标准
- **口碑效应**：做好就有，躺平就没——供应商之间的口碑传播比我们的说教更有效

### 2.5 对冲策略（降价时的处理）

降价是供应商最敏感的话题，处理不好会导致供应商消极应对或直接退出。

**对冲方法**：
- 承诺"不是永远降，后面可能回调"——给供应商希望
- 用自建团队数据作为客观基准："自建能做到 110% 增速，你们只有 45%"
- 所有供应商一视同仁降价，不接受就不给业务
- 承诺"看好生命线"——不会降到供应商活不下去

## 三、对标能力：自建团队作为效能基准

### 3.1 为什么需要自建团队

- 供应商说"做不到"，是真的做不到还是不想做？需要一个客观基准来验证
- 自建团队是**供应商管理的对标配尺**——通过建立效能基准，倒逼整个生态优化
- 没有对标，供应商管理就是"供应商说什么就是什么"

### 3.2 自建团队的建设过程

**0→1 落地**：
- 亲自面试近 200 人，百里挑一选 20 人
- 看了 300 份简历，在宿迁待一个月
- 团队设置、绩效、结构设计全程参与
- 3 个月搭建金企融合 + 企贷大额能力

**选人画像**：
- 坐席：想赚钱、接受周六加班、年轻有冲劲、无干扰、纯粹
- 主管：有充分的带团队经验、有从 0 到 1 搭建团队的能力

### 3.3 对标效果

| 指标 | 自建团队 | BPO 供应商 |
|------|---------|-----------|
| 产能增长 | +110% | +45% |
| 费效比 | 健康区间（1.2 倍以内） | 高于行业 15% |

**反向推动**：用自建数据去谈判——"同样数据你们做得不如人家，还想拿更多钱？"

## 四、六大管理场景的拿捏应用

| 场景 | 拿捏方式 | 关键动作 |
|------|---------|---------|
| 新供应商引入 | 准入标准 + 储备池 | 9 维度评估，15 天完成引入 |
| 新项目上线 | 分量竞标 | 3-5 家竞标，选最优 |
| 供应商排名下滑 | 分量调整 + 培育 | 先分析原因，再决定给机会还是减分量 |
| 供应商违规 | 递进式清退 | 警告→约谈→限制→清退 |
| 供应商要求涨价 | 对标压价 | 用自建数据和其他供应商报价压制 |
| 线路中断 | 主动补位 + 资源调度 | 找有指标但没用的供应商，快速恢复 |

## 五、递进式清退规则详解

清退是拿捏能力的终极体现，也是最需要技巧的环节。

### 5.1 四阶段递进

```
警告 → 约谈 → 限制新进 → 清退
```

| 阶段 | 触发条件 | 具体动作 | 供应商感受 |
|------|---------|---------|-----------|
| 警告 | 首次轻微违规 | 口头/邮件警告，要求整改 | 还有机会 |
| 约谈 | 再次违规或整改不到位 | 正式约谈，记录在案 | 开始紧张 |
| 限制新进 | 持续不改善 | 不允许接新业务、不允许新增人力、只能持续流失 | 真正痛了 |
| 清退 | 严重违规或长期不达标 | 启动清退流程 | 最后通牒 |

### 5.2 "限制新进"的杀伤力

这是清退前最有效的施压手段：

- 不允许接其他业务 → 收入增长被锁死
- 不允许新增人力 → 产能只能持续流失
- 不主动清退，让供应商"慢慢流血"
- 给供应商两难选择：闹→被京东拉黑 vs 走→日后好相见

### 5.3 无损退出技巧

让供应商"自愿"申请退出：

- 告知公司制度，说明行业影响
- 承诺如果自己申请，不加黑名单
- 利用黑名单的行业威慑力（上了黑名单，其他家也不敢用）
- 给台阶下："日后好相见"

## 六、风险管控的事前-事中-事后框架

### 6.1 六大典型风险

| 风险类型 | 产生原因 |
|----------|---------|
| 话术红线违规 | 坐席不懂规矩、缺乏培训、侥幸心理 |
| 分包转包 | 供应商想赚取差价、监管不到位 |
| 现场管理不达标 | 配不齐（组长/管理/质检）、投入不足 |
| 人员不合格 | 招人标准低、培训跟不上 |
| 规模失控 | 站点多管不过来、随意新增站点 |
| 清退舆情 | 清退过程不透明、要求不明确 |

### 6.2 管控矩阵

| 阶段 | 管控动作 | 管控的风险 | 具体做法 |
|------|---------|-----------|---------|
| **事前** | 合同约定 | 分包转包 | 合同明确不许，违约严重处罚 |
| **事前** | 站点设立审批 | 规模失控 | 不允许随意更换或新增站点 |
| **事前** | 人员标准 | 人员不合格 | 上岗前/中/后标准、培训要求 |
| **事中** | 巡检机制 | 各类过程风险 | 年均 17 次，落地到坐席、看流水、访谈 |
| **事中** | 摄像头监控 | 分包转包 | 远程监控 |
| **事后** | 递进式清退 | 清退舆情 | 过程透明、避免纠纷 |

## 七、核心方法论总结

> 供应商管理的本质不是"管"，而是"经营"。

- **分量策略** = 经营资源分配，让每家供应商都有动力
- **拿捏能力** = 经营关系平衡，让供应商在有限奖励下依然愿意投入
- **对标能力** = 经营效能基准，让"做不到"有据可查
- **培育能力** = 经营能力提升，让新供应商快速出产能
- **进化能力** = 经营知识沉淀，让经验变成可复制的制度

五大能力共同构成一个自我运转的供应商管理生态系统。
