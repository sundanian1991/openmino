## 概述

Mino 的可视化体系以 SVG 手绘风格为核心，围绕 infograph-svg 设计规范建立。2026-04-20 交付 30 个供应商管理核心单元（60 文件）是体系的里程碑，沉淀出"金字塔验证法"——3 秒法则 + 3 步决策链 + 14 点质量清单。

## 设计哲学

> 技术是骨架，设计哲学是血肉。只有骨架没有血肉，输出不稳定。

**核心目标**：建立可识别的"年老师风格"，不是复刻特定作品。年老师原话："核心目标：建立可识别的风格系统，不是学会画虾"。

**风格定位**：Clean > Minimalist > Modern（Shadcn UI 风格）。

## SVG 设计规范

### 色彩系统

```
主色调（黑白灰）：
  Canvas背景  #faf9f7  (米白)
  容器描边    #1a1a1a  (深灰 1.5px)
  卡片描边    #d4d1c7  (暖灰 1px)
  卡片填充    #f5f4f1  (暖灰)
  文字主色    #1a1a1a
  文字次色    #6b6b6b

强调色（唯一彩色）：
  暖棕强调    #c98a6a  (仅用于数值/重点，≤10%面积)

语义色（每个图最多2个）：
  成功/正向   #8f9b7f  (橄榄绿)
  警告/注意   #d4845a  (焦橙)
  用户输入    #6a8fb5  (灰蓝)
```

### 字体系统

| 层级 | 字体 | 大小 | 粗细 | 颜色 |
|------|------|------|------|------|
| H1 标题 | Georgia | 18px | bold | #1a1a1a |
| H2 副标题 | system-ui | 12px | 500 | #1a1a1a |
| H3 标签 | system-ui | 10.5px | 500 | #1a1a1a |
| Body 正文 | system-ui | 10px | 400 | #1a1a1a |
| Value 数值 | monospace | 11px | 600 | #c98a6a |
| Caption 说明 | system-ui | 9px | italic | #6b6b6b |

### 容器与画布

```
画布：680px 固定宽度，高度 auto
边距：左右 40px，上下 25px
内容区：600px 宽
间距基线：4pt (4/8/12/16/20/24/28/32px)
大容器：rx=10-12 stroke=1.5px
卡片：rx=8 stroke=1px fill=#f5f4f1
```

### 手绘风格规则

- **零容忍 emoji**：全部用 SVG 替代
- **手工感**：Q 贝塞尔曲线优于直线/矩形，禁止几何直线
- **颜色统一**：同一物体颜色统一，不同物体按视觉对照分配
- **风格系统 > 单一作品复刻**：每种风格有自己的一套图标集

## 金字塔验证法

### 第一层：3 秒法则

核心信息 3 秒内可理解：
- 标题即结论（如"FCI速记卡"而非"FCI相关卡片"）
- 焦点元素一眼可见（中心位置、暖棕强调色）
- 次要信息自动退后（caption 9px 斜体 #6b6b6b）

### 第二层：3 步决策链

| 步骤 | 时间 | 问题 | 答案来源 |
|------|------|------|----------|
| 第 1 步 | 1 秒 | 这是什么？ | 标题 + 核心视觉 |
| 第 2 步 | 2 秒 | 说明了什么？ | 逻辑主链（3 个递进节点） |
| 第 3 步 | 3 秒 | 对我意味着什么？ | 行动提示 + 使用场景 |

### 第三层：14 点质量清单

| 类别 | 检查项 | 标准 |
|------|--------|------|
| 字体 | 标题 | Georgia 18px bold |
| 字体 | 正文 | system-ui 10.5px 500 |
| 字体 | 数值 | monospace 11px 600 #c98a6a |
| 字体 | 说明 | 9px 400 italic #6b6b6b |
| 容器 | 外框 | rx=10-12 stroke=1.5px #1a1a1a |
| 容器 | 卡片 | rx=8 stroke=1px #d4d1c7 fill=#f5f4f1 |
| 容器 | 背景 | #faf9f7 |
| 色彩 | 主色 | 黑白灰三色 |
| 色彩 | 强调色 | #c98a6a（唯一彩色，≤10%面积） |
| 色彩 | 语义色 | 橄榄绿/焦橙，最多 2 个 |
| 布局 | 画布宽 | 680px 固定 |
| 布局 | 边距 | 左右 40px |
| 布局 | 间距 | 4pt 基线网格 |
| 逻辑 | 信息流 | 符合阅读动线（Z 字/F 型），层级清晰 |

## 可视化产出工具

### generative-ui-widget

流程/关系/对比用 generative-ui-widget，禁止用文本符号（→ ↓ │）模拟可视化。

### chart-visualization

通用数据可视化技能，支持 26 种图表。2026-04-20 跑通 25/26（横向柱状图后端未注册）。

### echarts-visualization

复杂交互式图表（桑基图、K 线图、地图、热力图），生成独立 HTML。

### SVG 手绘

暖调学术系：米白背景 + 深灰文字 + 暖棕强调。Q 贝塞尔曲线，有机感。

## 质量检查流程

1. **用浏览器实际打开**看渲染效果，不能只检查代码语法（LRN-20260420-001）
2. **像素级反馈**：年老师的标准是具体的（"文字穿过"、"箭头没有指向对象"），不是"差不多"
3. **内容密度均衡**：各板块内容要均衡，避免视觉不协调
4. **大屏适配**：大屏幕不能撑满，必须有 max-width 限制

## 模板化批量生产

标准文档结构（8 章节）保证效率，主题化视觉隐喻保证独特性：

| 单元类型 | 视觉隐喻 | 焦点位置 |
|----------|----------|----------|
| 对比/分类 | 四象限矩阵 | 中心十字交点 |
| 因果分析 | 逻辑树 | 根节点居中 |
| 多维评估 | 雷达图 | 中心聚合点 |
| 流程/时序 | 横向管道 | 从左到右 |
| 状态监控 | 仪表盘 | 中心指标 |
| 结构展示 | 文件夹树 | 左侧层级 |
