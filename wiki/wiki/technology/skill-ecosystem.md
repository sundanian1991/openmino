## 概述

Mino 的技能生态围绕"体系化组合拳"思维构建——一个技能解决一类问题，技能之间有清晰边界。从安装、使用到清理，遵循全生命周期管理。

## 技能管理原则

### 核心原则

- **技能使用先按原样安装**：不要过度分析，开箱即用 > 自定义方案（LRN-20260405-001）
- **脚本内容比 SKILL.md 更重要**：评估技能时先看脚本和材料是否充分（LRN-20260329-001）
- **组合拳思维**：年老师要体系化技能，不是单点工具（LRN-20260323-001）
- **按风格设计独立图标集**：不是一套图标多种颜色（LRN-20260404-002）

### 安装与清理

| 阶段 | 规则 |
|------|------|
| 安装前 | 检查脚本和材料是否充分 |
| 安装时 | 按原样安装，不过度分析 |
| 使用中 | 多技能协同，边界清晰 |
| 清理时 | 技能试用三问：解决什么问题？和已有的区别？用完要不要删？ |

**清理教训**：上午做加法（30+5 单元交付），晚上做减法（卸载 6 个冗余技能）。先理解真实需求，再做减法。

## 技能发现

### 搜索渠道

| 优先级 | 平台 | 说明 |
|--------|------|------|
| 1 | clawhub.ai | 中国镜像，国内访问优先 |
| 2 | claw123.ai | OpenClaw 国际站（备用） |
| 3 | Tencent SkillHub | 腾讯 SkillHub，1.3 万 Skills |

**命令**：`curl -s https://clawhub.ai/api/skills.zh.json` 搜索中文技能列表。

### 安装流程

1. curl 搜索技能列表 → 推荐最匹配 1-3 个
2. 确认后再安装 → fetch url 获取 SKILL.md
3. 复制到 `.claude/skills/`
4. **按需安装，不批量装**

## 技能协作模式

### 多技能协同

年老师希望多个技能协同工作，共同解决复杂问题（FEAT-20260404-006）：

| 协同组 | 技能 | 协作模式 |
|--------|------|----------|
| 决策支持 | lenny-advisor + dbskill + gstack | 决策 → 诊断 → 验证 |
| 知识工作流 | kw-brainstorm → kw-plan → kw-work → kw-review → kw-compound | 6 步流水线 |
| 可视化体系 | chart-visualization + SVG + widget | 数据图表 + 手绘 + 交互式 |

### 技能融合教训

- **skill 融合需谨慎**：codebase-to-course 技能安装 → 增强 → 测试 → 反馈深度不够 → 恢复原版 + 保留中文语言（2026-03-27）
- **避免功能重叠**：合并 frontend-slides 到 mino-frontend，不保留两个做类似事的技能
- **去原项目文化元素**：技能移植时要去掉原项目的特定文化元素（ERR-20260404-004）
