## 概述

在 Mino 知识库的演示项目中，同一内容以多种叙事风格呈现是一个反复出现的模式。本文整理六种叙事风格变体的设计差异、视觉特征和适用场景，以刘乾坤人物画像项目（6 种变体）和 AI 定价项目（2 种风格混合）为主要案例。

六种叙事风格分别是：default（默认）、editorial（编辑风格）、exhibit（展览风格）、freestyle（自由风格）、infographic（信息图风格）、mckinsey（麦肯锡风格）。

## Default 风格

**定义**：基础呈现风格，不做特殊风格声明时的默认选择。

**视觉特征**：
- 浅色主题，白底 + 单一强调色
- 标准 16:9 格式（1280×720）
- 正文基线 18px
- 布局以三列卡片、左右 split、单列居中为主

**语气特征**：
- 结构化陈述（"要点：①②③"）
- 对话式但偏分析性
- 每页标注"这一页说明什么" + "关键识别信号"

**适用场景**：内部分析文档、知识沉淀、个人参考

**典型案例**：刘乾坤 default 版采用暖白背景（`#F8F5F2`）+ 红色系（`#C41E3A`）强调，布局模式包括四象限矩阵、金字塔图、流程图等咨询常用图表。

## Editorial 编辑风格

**定义**：将 PPT 内容转化为类似人物报道/深度特稿的叙事方式。

**视觉特征**：
- 与 default 版视觉体系相近
- 在页面标题和过渡文字中加入更多"故事钩子"
- 强调过渡语的连贯性（"接下来拆解这三个诉求"、"看他对不同人的策略完全不同"）

**语气特征**：
- 从"分析报告"转向"人物报道"
- 更多情感化语言："这份档案不是简单的观察记录，而是经过一个多月持续追踪、24 条案例验证的系统性分析"
- 争议事件中直接引用感受："挺可怕的，真的不可交"
- 应对策略部分采用更细腻的叙述："示弱加边界，刘乾坤会退让"

**适用场景**：需要打动受众的情感共鸣、人物分析汇报、需要建立"我在认真观察"的可信度

**与 default 的核心差异**：同样的内容，editorial 版通过语言包装让"冷分析"变成"热故事"。数据不变，叙事温度不同。

## Exhibit 展览风格

**定义**：咨询公司常用的"Exhibit"式呈现，强调结论先行和数据驱动。

**视觉特征**：
- 深色主题：`#0D1117` 深黑背景（封面/章节/结尾）
- 蓝紫渐变装饰条（`#1E40AF` → `#7C3AED`）
- 金色强调（`#D4AF37`）用于分隔线和高亮
- 所有页面底部居中显示 "CONFIDENTIAL" 保密标签
- 低透明度网格装饰背景
- 页面结构更加紧凑：Header 60px + 内容 520px + Footer 60px

**布局模式**：
- 左-右 split 5:5（数据对比）
- 左-右 split 3:7（图表+文字）
- 矩阵网格（多维度分析）
- 瀑布图（财务分析）
- 表格（数据汇总）

**语气特征**：
- 结论先行，每页顶部有 Key Message Bar（深色背景 + 金色左侧装饰）
- 演讲备注偏向董事会汇报风格
- 强调数据来源和保密性

**适用场景**：董事会汇报、投资分析、战略规划、需要"权威感"和"专业感"的场合

**与其他风格的差异**：Exhibit 风格是唯一使用深色主题的风格，其"CONFIDENTIAL" 标签和网格背景营造了一种"机密报告"的氛围，是六种风格中正式感最强的。

## Freestyle 自由风格

**定义**：最少结构约束的风格，偏向演讲者的自由发挥。

**视觉特征**：
- 结构上与 default 版相同（12 页骨架）
- 在图表选择和布局上更加灵活，不受固定模板限制

**语气特征**：
- 口语化程度最高
- 减少结构化标记（"要点：①②③" 出现频率降低）
- 更多即兴表达和演讲者个人风格

**适用场景**：即兴演讲、非正式分享、需要灵活调整内容的场合

## Infographic 信息图风格

**定义**：以数据可视化密度和信息压缩率为核心目标的风格。

**视觉特征**：
- 手绘 SVG 图标和图表大量使用
- 信息密度高于其他风格
- 关系图、流程图、矩阵图等信息图表作为核心内容载体
- 从文件名可见存在 v2 版本，说明经过迭代优化

**语气特征**：
- 文字精简，让图表说话
- 演讲备注更短，更多依赖视觉传递信息

**适用场景**：数据密集型报告、需要快速传递大量信息的场合、信息图分享

## Mckinsey 麦肯锡风格

**定义**：以麦肯锡咨询方法论为核心的呈现风格。

**视觉特征**：
- 与 default 版同为麦肯锡风，但 mckinsey-v2 做了进一步迭代
- 强调 MECE（Mutually Exclusive, Collectively Exhaustive）原则
- 金字塔原理：结论先行，以上统下

**语气特征**：
- 在描述争议事件时加入更多"关键识别信号"的描述
- 分析更加具体，每个判断都有支撑数据
- 应对策略部分采用更系统化的框架

**与 default 版的差异**：Default 版是"麦肯锡风"的初稿实现，mckinsey-v2 版在分析深度和结构化程度上做了进一步打磨，更接近真正的麦肯锡报告标准。

## 六种风格的对比矩阵

| 维度 | Default | Editorial | Exhibit | Freestyle | Infographic | Mckinsey |
|------|---------|-----------|---------|-----------|-------------|----------|
| **主题** | 浅色暖白 | 浅色暖白 | 深色深黑 | 浅色 | 浅色 | 浅色 |
| **正式感** | 中 | 中 | 极高 | 低 | 中 | 高 |
| **情感温度** | 冷分析 | 热故事 | 冷分析 | 热表达 | 冷分析 | 冷分析 |
| **信息密度** | 中 | 中 | 高 | 中 | 极高 | 高 |
| **图表依赖** | 中 | 低 | 高 | 低 | 极高 | 高 |
| **保密感** | 无 | 无 | 强(CONFIDENTIAL) | 无 | 无 | 无 |
| **适用场景** | 内部参考 | 人物汇报 | 董事会 | 即兴演讲 | 数据报告 | 战略规划 |

## 选择指南

**需要数据冲击力** → Exhibit 或 Infographic
**需要情感共鸣** → Editorial 或 Freestyle
**需要专业权威** → Mckinsey 或 Exhibit
**需要快速参考** → Default

## AI 定价项目的风格选择

AI 定价项目选择了"anthropic + General Consulting"混合风格，这是六种风格之外的第七种变体：
- 封面/章节页使用深色背景（`#1A1A2E`），内容页使用白色背景
- 橙色（`#D97757`）+ 蓝色（`#4A90D9`）+ 绿色（`#10B981`）+ 红色（`#EF4444`）四色系统
- 数据高亮使用 45px 大字（2.5x 基线）
- 引用了 8 种图表模板：timeline、horizontal_bar_chart、grouped_bar_chart、matrix_2x2、process_flow、porter_five_forces、waterfall_chart、dumbbell_chart

这说明在实际项目中，风格不是非此即彼的，而是可以根据内容类型自由组合的。
