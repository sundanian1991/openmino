## 概述

AI 定价方法论深度是对 AI 定价商业案例中提炼出的可迁移商业分析方法的系统性整理。该项目的核心价值不在于 AI 定价本身的数据和事实，而在于它所证明的一套商业分析框架——如何从事实层进入框架层、从框架层进入洞察层、从洞察层进入预判层。这套方法论适用于任何"算力即服务"的商业模式分析，包括云存储、API 服务、SaaS 产品等。

## 四层分析结构

项目采用的四层分析结构是一套通用的商业研究框架：

**第一层：事实层——发生了什么**。收集时间线、参与者、数据对比、计量体系。不做分析，只做记录。关键原则是"先说事实，不说结论"。AI 定价的事实层覆盖了 11 个关键时间节点、7 家参与者、5 种计量体系、7 款模型进步时间线、芯片供给线。

**第二层：框架层——为什么这么演变**。用经典分析框架（波特五力、电信类比、三层驱动）来解释事实。框架层的价值是提供"解释力"——不仅知道发生了什么，还知道为什么发生。AI 定价用了波特五力分析、电信套餐 20 年演进类比、成本-竞争-价值三层驱动模型。

**第三层：洞察层——能提炼什么**。从具体案例中提炼可复用的模式。洞察层的核心标准是"跨场景可复用"。AI 定价提炼了五条规律：计量单位即定价权、补贴-收紧-分层三段式、入口到生态升级、国内外差异、Agent 成本不可逆。

**第四层：预判层——未来会怎样**。基于事实、框架和洞察，对未来趋势做预判。预判不是猜测，而是基于已有分析的合理外推。AI 定价做了四个趋势预判：Token 成本持续下降、国产芯片替代加速、开源重塑定价权、Agent 成本指数增长。

## 计量单位即定价权

这是整个项目最核心的洞察。

**核心命题**：谁定义计量单位，谁就掌握定价权。模糊计量有利于早期获客，透明计量有利于长期信任。

**证据链**：
- MiniMax 从 Prompt 计量转为 Token 计量——从"模糊打包"到"透明计费"
- Kimi 从 Prompt 计量转为 Token 计量——同样的转变逻辑
- 云厂商用"API 请求次数"——数字看起来很大，但实际消耗不透明，信息不对称有利于厂商
- Cursor 从 request 计量转为 credit 计量——引发退款风波

**演变方向**：API 次数 → Prompt 次数 → Token/Credit → 统一 Credits。方向是越来越透明、越来越可预测。

**可迁移价值**：当你推出新产品时，计量单位的选择决定了用户感知和你的利润空间。SaaS 产品按"席位"计费还是按"使用量"计费？API 服务按"请求次数"计费还是按"数据处理量"计费？计量单位不是技术问题，是商业策略问题。

## 补贴-收紧-分层的三段式节奏

这是互联网定价的通用模式，所有赛道都走这三步：

| 阶段 | 目的 | AI 案例 | 可迁移模式 |
|------|------|---------|-----------|
| 补贴 | 抢用户规模，建立习惯 | 百万 Token 几毛钱，首月 7.9 元 | 烧钱圈用户 |
| 收紧 | 测试价格弹性，筛选付费用户 | 限购、停售低档、限时抢购 | 市场份额稳定后收紧 |
| 分层 | 利润最大化，匹配不同支付意愿 | MiniMax 极速版 ¥98-899 | 精细分层收割 |

**核心启示**：进入新市场不要一开始就想好定价。先补贴跑量，再收紧测试弹性，最后精细分层。这个模式适用于任何从零开始定价的新产品、新服务、新市场。

**反模式**：Cursor 2025 年 6 月突然从 request 改 credit-based，重度用户一周超支 350 美元，引发退款风波。教训是：收紧定价需要过渡期，突然改变计费模式会伤害用户信任。

## 从入口到生态的升级路径

**核心命题**：用高频刚需场景（编程）获客，用低频高价值场景（视频/音乐）提高 ARPU。

MiniMax 把 Coding Plan 改名为 Token Plan，因为它不只卖编程能力，打包了图片、语音、音乐、视频。从"编程工具"升级为"AI 工作室"。

**可迁移价值**：
- 任何平台型产品都应该思考"入口产品"和"生态产品"的关系
- 入口产品负责获客（高频、刚需、低价）
- 生态产品负责盈利（低频、高价值、溢价）
- 从入口到生态的升级路径越自然，用户留存率越高

## Agent 时代的成本结构剧变

这是 AI 定价项目中最具技术深度的洞察。

**核心数据**：
- 一个开发者 8 个月 Claude Code 消耗 100 亿 Token
- 按 API 价格计算超过 15000 美元，但 Max plan 只花了 800 美元，节省 93%
- Token 构成：文件读取 35-45%、工具输出 15-25%、上下文重发 15-20%、推理 10-15%、代码生成仅 5-15%
- 85-95% 的 Token 是 Agent 的上下文开销，不是实际计算

**核心结论**：订阅费是地板不是天花板。实际账单是标价的 2-5 倍。

**对定价的含义**：在 Agent 时代，"不限量"模式不可持续。一个 Agent 任务等于几十次模型调用，包月模式在 Agent 时代必然导致亏损。所有厂商最终都会转向精细计量——谁先转谁主动，谁后转谁被动。

**对年老师工作的含义**：在供应商管理中引入 AI Agent 时，需要考虑 Token 成本的可预测性问题。不是"用不用 AI"的问题，是"怎么计量 AI 成本"的问题。

## 芯片供给即定价天花板

DeepSeek V4 的定价策略揭示了一个深层规律：芯片供给直接绑定定价天花板。

**证据**：DeepSeek 官方明确说"受限于高端算力，预计下半年昇腾 950 批量上市后价格大幅下调"。

**数据支撑**：
- 华为昇腾 950PR：FP4 算力是 H20 的 2.87 倍，定价是 H200 的三分之一
- 英伟达市占率从 66% 降到 54%，2026 年可能跌破 50%
- 伯恩斯坦预测：2027 年国产芯片自给率从 2023 年 17% 到 55%

**可迁移价值**：底层供给约束直接决定了上层服务的定价空间。在供应商管理中，理解供应商的底层约束（人力成本、技术能力、资金状况），比看表面的报价更有预测力。

## IPO 与定价策略的关联

智谱和 MiniMax 在 2026 年 1 月港交所上市，这一事件与定价策略有直接关联：

- 上市公司必须证明商业可持续性，定价不能太激进也不能太保守
- 智谱 Coding Plan 两个月 ARR 破亿，证明固定月费模式有市场需求
- MiniMax 海外收入 73%，说明定价逻辑需要考虑全球化差异

**可迁移价值**：当竞争对手的资本状态发生变化（融资、上市、并购），其定价策略很可能随之调整。追踪竞争对手的资本动向，是预测其定价行为的重要信号。
