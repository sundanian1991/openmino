## 概述

2026 年 4 月 24 日发布的 DeepSeek V4 是 AI 定价演变中的最新冲击波。总参数 1.6T 但只激活 49B、输入缓存 Token 仅 1 元/百万、明确适配华为昇腾和寒武纪——这些数字和策略共同构成了一个全新的定价模式：开源 + 极致低价 + 不卖订阅。DeepSeek 不做 Coding Plan 也不做 Token Plan，只卖 API。它正在用不同于所有竞争对手的方式，重塑行业定价规则。

## DeepSeek V4 的定价表

| 模型 | 总参数 | 激活参数 | 输入（缓存） | 输入（未缓存） | 输出 |
|------|--------|----------|-------------|---------------|------|
| V4-Pro | 1.6T | 49B | 1 元/百万 | 12 元/百万 | 24 元/百万 |
| V4-Flash | 284B | 13B | 0.2 元/百万 | 1 元/百万 | 2 元/百万 |
| 对比 GPT-5 | - | - | $5/百万 | - | $15/百万 |

**换算对比**：V4-Pro 的输入缓存价格（1 元/百万）约为 GPT-5（$5/百万，约 36 元/百万）的 1/36。V4-Flash 的输入缓存价格（0.2 元/百万）约为 GPT-5 的 1/180。

这不是"便宜一点"，是"便宜一个数量级"。

## MoE 架构的效率革命

DeepSeek V4 的核心技术突破是 MoE（Mixture of Experts）架构：

**参数解耦**：V4-Pro 总参数 1.6T，但每次推理只激活 49B（约 3%）。V4-Flash 总参数 284B，只激活 13B（约 4.5%）。总参数决定模型能力，激活参数决定推理成本。MoE 让两者解耦——模型可以很大但推理成本很低。

**长上下文效率**：V4-Pro 在长上下文场景下，单 token 推理计算量仅为 V3.2 的 27%，KV Cache 占用降至 10%。这意味着同样的算力可以处理更长的上下文——100 万 token 上下文成为可能。

**国产芯片适配**：V4 同时适配华为昇腾和寒武纪，Day 0 支持。这意味着 V4 可以在国产芯片上运行，不受英伟达限制。

## DeepSeek 的独特定价模式

### 不卖订阅

DeepSeek 不做 Coding Plan，不做 Token Plan，只卖 API。这意味着：

- 没有月费，用多少付多少
- 没有额度限制，想怎么用就怎么用
- 没有套餐复杂度，价格一目了然

### 开源

DeepSeek V4 是开源的。这意味着：

- 任何人都可以下载并在自己的硬件上运行
- 免费部署能力持续压制定价上限
- 其他厂商的订阅模式面临"为什么不自己跑"的质疑

### 极致低价

V4-Flash 的输入缓存 0.2 元/百万 Token——这个数字低到什么程度？百万 Token 约等于 50-80 万字（中文），0.2 元处理 50 万字。

## 开源模型的定价权悖论

DeepSeek 的模式揭示了一个定价权悖论：

**传统定价权**：来自模型独占性。只有我的模型能做这个，所以你付我钱。

**DeepSeek 定价权**：来自服务便利性。你可以自己跑，但用我的 API 更方便。

这个悖论的含义是：当模型开源后，定价权不再来自"我能做你不能做"，而是来自"我做起来比你方便"。

这是一种更脆弱但更民主的定价权——用户选择你不是因为别无选择，而是因为"方便"。这意味着厂商必须持续提供便利，而不是依赖独占性。

## 对行业定价的影响

### 定价锚的重置

DeepSeek V4 的定价成为了新的行业锚。所有其他厂商的定价都要和 V4-Flash 的 0.2 元/百万 Token 对比。

**对订阅模式的冲击**：如果 API 价格已经低到这个程度，为什么还要付月费？订阅模式的合理性受到挑战。

**对 Token Plan 的冲击**：MiniMax 的 Token Plan 打包了图片、语音、音乐、视频，但 DeepSeek 的 API 价格低到让"打包"变得不那么有吸引力。

### 降价压力

DeepSeek V4 官方声明："受限于高端算力，预计下半年昇腾 950 批量上市后价格大幅下调。"

这意味着当前价格还不是底线。昇腾 950 量产后，价格还会下降。所有其他厂商都面临"你比 DeepSeek 贵多少"的压力。

### 差异化压力

当价格差距到一个数量级时，高价厂商必须回答一个问题："你为什么贵？"

答案可能是：
- 模型能力更强
- 服务质量更好（SLA、合规）
- 生态系统更丰富（多模型切换、集成工具）

但如果这些差异化不足以证明价格差距，用户就会流向低价方。

## 对供应商管理的迁移

### 价格锚的作用

在供应商管理中，价格锚同样重要：

- 知道市场上最低价格，才能在分量谈判中设定合理的分量条件
- 不是"最低价就是最好的"，但"最低价是基准线"
- 供应商的价格如果显著高于基准线，需要解释"为什么贵"

### 开源模式的对应

供应商管理中的"开源模式"是什么？是供应商愿意共享方法论、培训体系、数据工具。这种"开源"降低了其他供应商的进入门槛，但也提升了整体行业水平。

### 便利性的价值

DeepSeek 的定价权来自"服务便利性"。在供应商管理中，便利性同样有价值——一个响应快、流程简单、配合度高的供应商，即使价格略高，也可能被优先选择。便利性是差异化的重要维度。
