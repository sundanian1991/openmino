# 反模式识别：AI定价中的三个真实踩坑案例

## 概述

反模式是"看起来合理但实际会失败"的做法。在 AI 定价研究中，识别了三个真实的踩坑案例——Cursor 退款风波、计量单位混乱、饥饿营销反噬。这些案例的价值不在于 AI 行业本身，而在于它们揭示了一个通用规律：短期看起来有效的策略，长期可能伤害信任、品牌和用户关系。本文分析这三个反模式的细节、教训，以及它们对供应商管理的迁移价值。

## 反模式一：Cursor 退款风波

### 事件

2025 年 6 月，Cursor 突然从 request-based 计费改为 credit-based 计费。重度用户一周超支 350 美元，引发了大规模的退款请求和用户不满。

### 为什么"看起来合理"

从厂商角度看，改计量方式看起来是合理的：
- credit-based 更灵活，可以支持多模型
- 不同模型可以有不同的倍率
- 统一管理成本

### 为什么会失败

**突然改变**：用户没有过渡期。从"我知道我花多少"变成"我算不清楚了"，用户失去了成本可控性。

**隐性涨价**：credit-based 对重度用户来说实际上是涨价。原来 $20 的 pool 能做的事，现在要 $350。

**信任破坏**：用户觉得"规则被单方面改了"，而不是"我们一起商量了新方案"。

### 教训

改变计费模式需要过渡期。突然改变伤害用户信任。正确的做法是：
1. 提前通知（至少 1-2 个月）
2. 提供换算工具（旧的 request 数 = 新的 credit 数）
3. 给旧用户过渡优惠（前 3 个月按旧规则）

## 反模式二：计量单位混乱

### 事件

国内七家 AI 平台使用五种完全不同的计量单位，90% 的用户踩坑——直接比数字，被不同计量单位误导。

### 为什么"看起来合理"

从单个厂商角度看，使用自己的计量单位看起来是合理的：
- API 请求次数：数字大，用户觉得"好多"
- Prompt 次数：比 API 次数直观一些
- Token 计量：最透明，但用户需要学习

### 为什么会失败

**信息不对称最终伤害信任**：当用户发现"原来 18000 次 API 请求只等于 600-3600 次提问"，他们会觉得被欺骗。

**横向比较困难**：用户无法比较各平台的真实价格，这降低了市场效率，也增加了用户的决策成本。

**行业标准混乱**：没有统一的计量标准，用户每次换平台都要重新学习。

### 教训

短期的信息不对称优势会转化为长期的信任赤字。正确的做法是：
1. 提供换算工具（我们的 18000 次 API 请求 ≈ 其他平台的 X 次提问）
2. 标注实际等效（"约等于每天 600 次 API 请求，约 20-120 次提问"）
3. 主动透明化，而不是等着被用户发现

## 反模式三：饥饿营销反噬

### 事件

智谱 GLM 的低价套餐需要每天 10:30 抢购。短期有效（制造稀缺感），长期伤害品牌信任。

### 为什么"看起来合理"

从运营角度看，饥饿营销看起来是合理的：
- 制造稀缺感，提升用户渴望
- 控制成本，限制同时在线用户
- 制造话题，增加社交媒体讨论

### 为什么会失败

**人为制造的稀缺性最终被用户识破**：当用户发现"每天都能抢"，就知道这不是真正的稀缺，而是营销手段。

**伤害品牌信任**：用户觉得"你在玩弄我"，而不是"你在帮助我"。

**效率低下**：每天 10:30 抢购消耗用户时间，这种"时间成本"会降低用户的整体满意度。

### 教训

人为制造的稀缺性最终会被用户识破。正确的做法是：
1. 真实限量（有客观原因，不是人为制造）
2. 透明规则（为什么限量，什么时候不限量）
3. 替代方案（抢不到怎么办，有没有其他选择）

## 反模式的共同规律

三个反模式共享一个规律：**短期有效，长期有害**。

这个规律的背后是一个更深层的原则：**信任是长期资产，不是短期工具**。

- Cursor 用突然改变计费换取了短期利润，但损失了用户信任
- 厂商用计量单位混乱换取了定价自由度，但损失了市场信任
- 智谱用饥饿营销换取了话题热度，但损失了品牌信任

信任损失是不可逆的。一次信任损失可能不会立即反映在用户流失上，但会在用户心中积累，最终爆发。

## 对供应商管理的迁移

### 反模式一迁移：突然改变合作规则

**场景**：突然改变供应商分量分配规则、考核标准、响应时间要求。

**后果**：供应商觉得"规则被单方面改了"，失去信任，消极应对。

**正确做法**：提前通知（至少 1 个月），提供过渡期，给供应商适应时间。

### 反模式二迁移：信息不透明

**场景**：供应商评分标准不透明、分量分配逻辑不透明、考核数据不公开。

**后果**：供应商觉得"你不公平"，信任下降，配合度降低。

**正确做法**：主动透明化。评分标准、分量规则、考核数据全部公开。短期的信息优势不值得用长期信任来换。

### 反模式三迁移：人为制造紧迫感

**场景**："这个月不改进，下个月就削减分量"——但实际上没有真正削减。

**后果**：供应商发现"你说归说，不会真的做"，紧迫感失效，信任下降。

**正确做法**：说了就做。要么不威胁，要么威胁了真的执行。一致性比严厉性更重要。
