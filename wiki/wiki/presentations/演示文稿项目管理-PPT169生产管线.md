## Overview

PPT169演示文稿项目遵循标准化的生产管线，从头脑风暴到研究、从设计规格到多版本生成。通过分析AI定价、梁俊瑞、刘乾坤三个项目的原始文件结构，可以提炼出一套可复用的演示文稿生产方法论：brainstorm→research→design_spec→generate→review→output。这套管线将演讲制作从"艺术创作"变成了"工业生产"，保证了质量和效率的同时，也保证了不同项目之间的可比较性和可复用性。

## 生产管线的六个阶段

### Phase 1：头脑风暴（Brainstorm）

输入：一个主题或问题
输出：初步的想法、框架、关键论点

头脑风暴阶段的目标是快速生成初始的想法，不做评判，只求数量。AI定价项目的头脑风暴文件记录了从Coding Plan到Token Plan的定价模式演变的最初想法。

关键产出：
- 核心叙事线（故事的主线是什么）
- 关键论点（3-5个主要观点）
- 数据来源（用什么支撑这些观点）

### Phase 2：研究（Research）

输入：头脑风暴产生的想法
输出：经过验证的事实、数据、案例

研究阶段的目标是验证头脑风暴中的想法是否有数据支撑。AI定价项目的research文件记录了市场定价数据、竞争对手分析、定价模型对比等信息。

关键原则：
- 只记录有来源的信息（不能编造数据）
- 区分事实和推断
- 标注信息来源和时间

### Phase 3：设计规格（Design Spec）

输入：经过验证的内容框架
输出：visual design specification

设计规格定义了演示文稿的视觉呈现方式：配色、字体、布局、风格。每个项目都有一个design_spec.md文件，是内容层和视觉层之间的桥梁。

规格文件包含：
- 视觉风格定义（如mckinsey、editorial、infographic）
- 页面规格（16:9、边距、字号）
- 颜色方案
- 图表类型选择

### Phase 4：生成（Generate）

输入：内容大纲 + 设计规格
输出：多风格版本的演示文稿

生成阶段将内容和视觉组合，产出最终的演示文稿。PPT169系统支持一键生成多个风格版本。

### Phase 5：审查（Review）

输入：生成的演示文稿
输出：修改建议和修正版本

审查阶段检查内容的准确性、视觉的一致性和演讲的流畅性。

### Phase 6：输出（Output）

输入：最终版本
输出：可交付的演示文稿 + wiki归档

## 项目文件结构规范

每个PPT169项目都遵循统一的文件命名和结构规范：

```
projects-{主题}_ppt169_{日期}/
├── README.md                    # 项目概述
├── design_spec.md              # 设计规格
├── notes-total.md              # 所有演讲备注汇总
├── notes-01_*.md               # 第1页备注
├── notes-02_*.md               # 第2页备注
├── ...
├── notes-12_*.md               # 第12页备注
├── sources-00-brainstorm.md    # 头脑风暴
├── sources-01-research.md      # 研究资料
└── templates-design_spec.md    # 模板规格
```

这种结构的价值在于：任何项目都可以用同样的方式浏览和理解。新成员可以快速定位到需要的信息。

## 生产效率的提升

### 从单版本到多版本

传统的演讲制作是一次做一个版本。PPT169管线通过内容-视觉分离，实现了一次内容、多版本视觉。

### 从项目到模板

经过多个项目的积累，可以沉淀出可复用的模板：
- 12页标准结构模板
- 各风格的design_spec模板
- 头脑风暴模板
- 研究资料模板

### 从模板到管线

当模板积累到一定程度，整个生产过程就变成了一条标准化的管线：输入主题→运行管线→输出多版本演示。这就是从"手工创作"到"工业化生产"的转变。
