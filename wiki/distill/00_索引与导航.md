## 一、知识库架构总览

本知识库采用**卡帕西式三层结构**，确保AI调用时有据可查、有源可溯：

```
┌─────────────────────────────────────────────────────┐
│  distill/  提炼层  ← AI直接调用，结构化知识        │
│  （10个分册，约4000+行）                            │
├─────────────────────────────────────────────────────┤
│  raw/      原文层  ← 来源追溯，完整原文归档         │
│  （43个核心原文文件）                               │
├─────────────────────────────────────────────────────┤
│  notes/    健康判断层 ← 冲突记录、版本变更、待验证   │
│  （3个文件：冲突记录/待验证假设/已过时内容）         │
└─────────────────────────────────────────────────────┘
```

### 编码体系

> [来源：知识库编码体系设计 / 各分册文件命名规范]

| 编码前缀 | 含义 | 示例 |

> [来源：A1-A6/B1-B6/C1-C4/D1-D4/E1-E2场景文档汇总（见raw/）]

- A4 转正延期决策 → 6月里程碑未达标的处理
- A5 合规问题定级 → A类/B类/C类的判定和升级路径
- A6 清退评估 → 触发→评估→审批→交接

**项目执行类（B1-B6）**：[07 场景SOP-项目执行](07_场景SOP-项目执行.md)

- B5 向上汇报准备 → 结论→数据→建议
- B6 降价谈判准备 → 底价→筹码→话术→预案

**通用与应急（C1-C4 + D1-D4 + E1-E2）**：[08 场景SOP-通用与应急](08_场景SOP-通用与应急.md)

