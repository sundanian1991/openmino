## 一、调研核心原则

**"这次去不是谈'减量'，是去'取证'。三种证，三种药方。取到正确的证，才能开正确的方。"**

调研目标：穿透数据层→归因层→结构层，取产品/供应商/市场三方面的证。

---

## 二、八模块调研框架

### 模块一：产能漏斗数据表（现场必须拿到）

> [来源：raw/TM-2026-900 §模块一]

| 数据项 | 异常信号 | 行业参考值 |
|--------|---------|-----------|
| 总外呼量（月） | 下降>20%为异常 | — |
| 接通率 | <15%为线路/号码问题 | 20-30% |
| 意向率 | 外呼稳但意向掉=话术/产品问题 | — |
| 成交率 | 意向高但成交掉=门槛/资质问题 | — |
| 人均日外呼量 | <60通=管理问题 | 80-120通 |
| 人均月结算 | 核心指标，必须逐月数 | — |

**怎么问**："系统能导出近6个月的外呼量和结算走势图吗？"

### 模块二：时间分配与竞品分流

> [来源：raw/TM-2026-900 §模块二]

| 数据项 | 洞察 |
|--------|------|
| 日工作时长占比 | 竞品>50%=已用脚投票 |
| 结算单价 | 竞品高30%+是核心差距 |
| 结算周期 | 周结 vs 月结影响现金流 |

**怎么问**："你们团队一天8小时，我们的产品占多久？另一个产品占多久？"

### 模块三：人员结构与产能分布

> [来源：raw/TM-2026-900 §模块三]

| 数据项 | 健康信号 | 风险信号 |
|--------|---------|---------|
| 20-25岁占比 | <40% | >60%（太年轻=不稳定） |
| 在职<3个月占比 | <30% | >50%（培训没跟上） |
| 在职>6个月占比 | >40% | <20%（留不住人） |
| 前20%人均结算 | 稳住10000+ | 连头部都掉=产品问题 |
| 后20%人均结算 | 不为0 | 接近0=招了充数的人 |
| 近3个月离职率 | <15% | >30%（收入预期崩了） |

### 模块四：原因分层诊断表（核心——搞清楚"谁的锅"）

> [来源：raw/TM-2026-900 §模块四]

**三层归因**：

1. **产品原因排查**：客户被拒率、额度竞争力、审批速度、产品知名度、政策变动
2. **供应商原因排查**：入职培训时长、招聘标准、早会/晚会内容、提成差距、数据分析能力
3. **市场/竞品原因排查**：市场容量变化、竞品卖点、竞品近期动作、监管变化

**关键判断逻辑**：
- 客户被拒率>50%且因资质 = 门槛问题
- 业务主动提额度低 = 竞品教育了市场
- 我们3天 vs 竞品1天 = 致命差距
- <3天或没产品培训 = 供应商问题
- 没数据复盘 = 管理粗放

### 模块五：商业模式稳定性评估

> [来源：raw/TM-2026-900 §模块五]

| 评估维度 | 稳定 | 风险 |
|---------|------|------|
| 盈亏平衡线 | 清晰算账 | 算不清或严重亏损 |
| 现金流 | 不拖欠工资房租 | 拖欠工资/房租 |
| 对我方依赖度 | 有多元收入 | 100%依赖+竞品分流 |
| 竞品投入深度 | 混着做可调整 | 专属=已"叛变" |
| 场地租约 | 长租约 | 短租约或随时搬 |
| 核心人员稳定性 | 核心团队>6个月 | 近期核心离职 |
| 系统合规 | 系统完善 | 缺失=监管风险 |

### 模块六：数据洞察交叉验证矩阵

> [来源：raw/TM-2026-900 §模块六]

| 现象 | 再追问 | 真实原因可能是 |
|------|--------|--------------|
| 人均结算降30% | "所有人都降了还是尾部拖累？" | 全降=产品/市场；尾部拖累=供应商管理 |
| 竞品占时60% | "因为单价高还是更好成交？" | 单价高=结算政策；更好成交=产品竞争力 |
| 新员工产能低 | "培训多久？有人带吗？" | 培训不足=供应商；产品复杂=产品问题 |
| 管理层抱怨产品 | "具体抱怨什么？提改进建议了吗？" | 只抱怨不提=推卸责任 |
| 业务代表乐观 | "你们收入最近有变化吗？" | 管理层悲观但基层乐观=信息断层 |

### 模块七：每日现场速记表（每晚花15分钟填）

> [来源：raw/TM-2026-900 §模块七]

**速记结构**：
1. 今日关键数据（人均结算、外呼量、接通/意向/成交率、竞品占比、头部/尾部）
2. 原因判断（产品门槛、额度竞争力、审批速度、培训、招聘、管理、竞品、市场、监管）
3. 商业模式稳定性（盈亏平衡、现金流、依赖度、核心人员）
4. 今日关键洞察（一句话）
5. 明天需要补的数据

### 模块八：回去汇报骨架（5章结构）

> [来源：raw/TM-2026-900 §模块八]

```
一、基本情况（1页）
   - 站点规模、人员结构、合规状态
   - 近6个月业绩趋势图

二、核心发现（2页）
   ├─ 根因诊断：4个月下滑的触发点是______，深层原因是______
   ├─ 数据验证：人均产能差30%，拆解后发现是______
   └─ 竞品影响：同时在做的竞品是______，分流比例约______

三、供应商诉求（1页）
   ├─ 表面：想减量
   ├─ 深层：可能是______（逼涨价/要资源/转项目/真扛不住）
   └─ 验证：基于访谈中的______细节

四、风险评估（1页）
   ├─ 同意减量：对其他10家的信号、内部转项目口子、后续管理难度
   └─ 不同意：供应商可能的反制、我们的底线方案

五、建议方案（1页）
   ├─ 不答应减量的替代方案：______
   ├─ 需公司推动的调整：______
   └─ 时间线：______，责任人：______
```
