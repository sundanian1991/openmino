# 02 引入与清退规范

> 编码：SM-2026-010/030
> 来源：SM-2026-010_供应商引入管理规范.md、SM-2026-030_供应商清退管理规范.md、MT-2026-110_全生命周期管理框架.docx
> 核心定位：供应商"入口"和"出口"的制度规则 — 准入8阶段、试单期、清退触发矩阵、交接方案

---

## 总览

供应商管理不是单一动作，而是贯穿四个阶段的闭环体系。

```
准入期（选） → 成长期（育） → 稳定期（留） → 退出期（汰）
    ↑                                              │
    └────────────── 备选池 ────────────────────────┘
```

**核心管理思维转变**：
- 从"管人"转向"管规则和管预期"
- 把供应商当作微型组织理解，不是简单的服务提供方
- 每个阶段都有预设的管理动作和判断标准（不是等出问题才应对）

## 阶段一：准入期（选）

> [来源：MT-2026-110_全生命周期管理框架.docx §相关章节 / SM-2026-050_清洁版操作指引.md §相关章节 / SM-2026-010_供应商引入管理规范.md §相关章节 / SM-2026-030_供应商清退管理规范.md §相关章节 / SM-2026-035_供应商分级清退管理制度.md §相关章节]

**管理定位**：宽进严出，用试单验证代替完美评估

### 硬性门槛（不满足直接否决）

> [来源：MT-2026-110_全生命周期管理框架.docx §准入门槛 / SM-2026-010_供应商引入管理规范.md §资质要求]

| 维度 | 标准 |
|------|------|
| 注册资本 | ≥1000万元 |
| 资质许可 | 呼叫中心经营许可证，外包服务资质齐全 |
| 经营年限 | ≥3年 |
| 人员规模 | ≥100人 |
| 合规记录 | 近3年无重大违规 |
| 财务状况 | 无恶化迹象、无重大诉讼 |

### 一票否决红线

> [来源：SM-2026-010_供应商引入管理规范.md §一票否决 / SM-2026-050_清洁版操作指引.md §一票否决]

- 资质造假
- 重大违规（数据泄露、录音篡改等）
- 财务恶化（资金链断裂、失信、重大诉讼败诉）
- 老板/实控人的人品问题（行贿、诈骗等）

### 准入诊断6问

> [来源：MT-2026-110_全生命周期管理框架.docx §准入诊断6问]

| 问题 | 核心判断 |
|------|---------|
| 1. 为什么必须是这家？ | 市场有没有更好选择？不可替代优势是什么？ |
| 2. 他的核心能力是什么？ | 人力规模/管理经验/技术系统/成本优势？ |
| 3. 他的老板靠谱吗？ | 决策能力、风险意识、诚信度、行业经验？ |
| 4. 他愿不愿意投入？ | 重视程度、资源投入意愿、内部激励机制？ |
| 5. 3年后还需要他吗？ | 短期补充还是长期合作？依赖度如何？ |
| 6. 如果他不行，替代方案是什么？ | 有没有B计划？切换成本多高？ |

> **两个灵魂问题贯穿始终**："为什么必须是这家？" + "3年后还需要吗？"

### 现场考察3维度

> [来源：MT-2026-110_全生命周期管理框架.docx §现场考察 / SM-2026-010_供应商引入管理规范.md §现场考察]

1. **看办公环境**：场地规模、工位数量、设备配置、员工状态（精神面貌/忙碌程度）
2. **聊管理团队**：对接人决策权、老板态度、管理层稳定性、团队氛围
3. **查实际能力**：现有客户案例、人员流失率、培训体系、质控流程、系统工具

### 核心策略

> [来源：MT-2026-110_全生命周期管理框架.docx §准入策略]

- **宽进严出**：门槛合理但不过高，进入后用试单验证
- **多家试单PK**：同时引入3-5家，用实际业绩说话
- **不依赖单一指标**：综合评估资质、能力、态度、投入意愿
- **试单期管理**：清晰KPI + 1-3个月观察期

## 六盒诊断模型（跨阶段通用）

> [来源：MT-2026-110_全生命周期管理框架.docx §六盒模型 / 管理思路.md §六盒]

> 源自韦斯伯德(Weisbord)组织诊断框架，适配到供应商管理。

| 维度 | 诊断内容 | 典型问题 |
|------|---------|---------|
| **目的** | 供应商是否清楚合作目标？目标是否可衡量？ | "他们不知道我们要什么" |
| **结构** | 组织架构是否合理？职责是否清晰？ | "对接人换了3次，每次都要重新讲" |
| **关系** | 供应商内部协作是否顺畅？与京东对接是否高效？ | "他们内部信息不互通" |
| **奖励** | 激励机制是否有效？员工是否有动力？ | "员工抱怨提成低，积极性差" |
| **领导** | 管理层能力如何？决策是否及时？ | "老板不管事，主管说了算但又定不了" |
| **支持机制** | 培训、系统、工具是否到位？ | "系统老坏，培训靠口头" |

> **使用时机**：供应商出现多维度问题但看不清根因时，用六盒逐一排查。

## 阶段二：成长期（育）

> [来源：MT-2026-110_全生命周期管理框架.docx §相关章节 / SM-2026-050_清洁版操作指引.md §相关章节 / SM-2026-010_供应商引入管理规范.md §相关章节 / SM-2026-030_供应商清退管理规范.md §相关章节 / SM-2026-035_供应商分级清退管理制度.md §相关章节]

**管理定位**：帮助供应商从"能干活"到"干得好"

这是投入最多、问题最多、也最有价值的阶段。

### 关键动作

> [来源：MT-2026-110_全生命周期管理框架.docx §成长期关键动作]

| 动作 | 说明 |
|------|------|
| 系统对接 | 完成业务系统、数据报表、沟通机制的对接 |
| 培训赋能 | 业务知识、操作流程、合规要求培训 |
| KPI设定 | 明确短期（1-3月）和中期（3-6月）目标 |
| 密集监控 | 每周数据跟踪、每月复盘、及时纠偏 |
| 问题诊断 | 出现偏差时，快速定位根因 |
| 资源支持 | 必要时提供额外培训、人员支持、流程优化建议 |

### 成长期核心三问

> [来源：MT-2026-110_全生命周期管理框架.docx §成长期]

> **根本原因是什么？能改进吗？需要什么支持？**

- 先诊断根因 → 再判断可改进性 → 最后明确支持需求
- 不诊断就支持是浪费，不判断就开药是冒险

### 主管胜任5项指标（成长期关键评估）

> [来源：MT-2026-100_供应商管理思路梳理.md §主管胜任 / MT-2026-110_全生命周期管理框架.docx §主管评估]

| 指标 | 达标线 | 说明 |
|------|--------|------|
| 流失率 | <8%/月 | 团队稳定度 |
| 出勤率 | >90% | 出勤稳定度 |
| 人效走势 | 不连续2个月下降 | 业绩趋势 |
| 预警后48h回复率 | >80% | 响应质量 |
| 整改承诺按期完成率 | >70% | 整改执行力 |

- 5项全过 → 主管合格，问题在其他地方
- 3-4项过 → 主管有短板，看短板在哪（稳定度和出勤是硬伤）
- ≤2项过 → 主管不行，建议更换（需王易人确认）

## 阶段三：稳定期（留）

> [来源：MT-2026-110_全生命周期管理框架.docx §相关章节 / SM-2026-050_清洁版操作指引.md §相关章节 / SM-2026-010_供应商引入管理规范.md §相关章节 / SM-2026-030_供应商清退管理规范.md §相关章节 / SM-2026-035_供应商分级清退管理制度.md §相关章节]

**管理定位**：维持并提升供应商的长期价值

### 核心风险：绑架能力

> [来源：MT-2026-110_全生命周期管理框架.docx §稳定期风险 / SM-2026-050_清洁版操作指引.md §风险管控]

稳定期最危险的不是"业绩下滑"，而是"业绩好但有人能绑架你"。

| 绑架信号 | 说明 |
|---------|------|
| 份额占比过高 | 某供应商单业务线占比>50% |
| 业绩排名独占 | 某供应商长期霸榜，无有效竞争 |
| 人力规模唯一 | 某区域/业务只有一家能承接 |
| 切换成本极高 | 历史数据/客户关系/系统深度绑定 |

### 稳定期10大风险维度

> [来源：MT-2026-110_全生命周期管理框架.docx §稳定期风险]

| 维度 | 风险信号 |
|------|---------|
| 1. 人力集中度 | 单一供应商人力占比>15% |
| 2. 团队稳定性 | 对接人/主管/核心员工流失趋势恶化 |
| 3. 合规风险 | 质检合格率下滑、踩红线苗头 |
| 4. 财务状况 | 资金链风险、经营困难 |
| 5. 客户结构变化 | 接了更大客户、对京东优先级下降 |
| 6. 管理松懈迹象 | 日报/周报质量下降、例会缺席、响应变慢 |
| 7. 第二梯队成熟度 | 备选供应商是否具备快速承接能力 |
| 8. 行业竞争态势 | 市场是否有更好替代选项 |
| 9. 合作意愿变化 | 消极怠工、敷衍应对、选择性执行 |
| 10. 产能集中度 | 单一供应商产能占比>20% |

> **使用时机**：每月固定做一次10维风险评估，不是等出问题才看。

### 应对策略

> [来源：MT-2026-110_全生命周期管理框架.docx §相关章节 / SM-2026-050_清洁版操作指引.md §相关章节 / SM-2026-010_供应商引入管理规范.md §相关章节 / SM-2026-030_供应商清退管理规范.md §相关章节 / SM-2026-035_供应商分级清退管理制度.md §相关章节]

> **谁有绑架能力？第二梯队是否就位？**

- **梯队备份**：每个业务都要有第二梯队
- **培育方法**：新业务优先给中腰部供应商、定期给第二梯队试单、保持信息同步
- **梯队备份不是"不信任"，是风险管理基本功**

### 日常管理动作

> [来源：SM-2026-020_供应商日常管理规范.md §日常管理 / SM-2026-050_清洁版操作指引.md §日常管理]

- 定期复盘（月度/季度）
- 持续优化流程和效率
- 激励优秀供应商（更多业务、更好价格、荣誉认可）
- 关注人员稳定性和管理连续性

> **真的触发清退了吗？替代方案是什么？**

- 不是一时冲动 → 确认触发条件
- 不会业务中断 → 替代方案就绪

### 退出管理原则

> [来源：MT-2026-110_全生命周期管理框架.docx §相关章节 / SM-2026-050_清洁版操作指引.md §相关章节 / SM-2026-010_供应商引入管理规范.md §相关章节 / SM-2026-030_供应商清退管理规范.md §相关章节 / SM-2026-035_供应商分级清退管理制度.md §相关章节]

- **有据可依**：触发条件明确、证据充分
- **风险隔离**：单业务线退出不影响其他线
- **平稳过渡**：有过渡计划、数据回收、费用清算

## 六步法闭环（跨阶段通用）

> [来源：MT-2026-110_全生命周期管理框架.docx §六步法闭环]

```
发现问题 → 诊断根因 → 制定方案 → 执行跟踪 → 效果评估 → 持续优化
```

**关键**：很多管理者做到了前三步，漏了后三步。

| 步骤 | 常见问题 | 正确做法 |
|------|---------|---------|
| 发现问题 | 忽视预警信号 | 建立多维度监控体系 |
| 诊断根因 | 头痛医头 | 每个症状背后至少两种根因可能 |
| 制定方案 | 一刀切 | 定制化、可执行、有验收标准 |
| 执行跟踪 | 发了方案就不管 | 明确责任人、时间节点、检查点 |
| 效果评估 | 没有量化 | 对比前后数据，评估ROI |
| 持续优化 | 同样问题反复出现 | 沉淀为规则/SOP |

## 管理心法（四句话）
> [来源：MT-2026-110_全生命周期管理框架.docx §管理心法 / SM-2026-050_清洁版操作指引.md §管理原则]


| 阶段 | 心法 | 核心问题 |
|------|------|---------|
| **选** | 为什么必须是这家？3年后还需要吗？ | 避免"为了准入而准入" |
| **育** | 根本原因是什么？能改进吗？需要什么支持？ | 找到卡住的齿轮 |
| **留** | 谁有绑架能力？第二梯队是否就位？ | 防止被绑架 |
| **汰** | 真的触发清退了吗？替代方案是什么？ | 避免情绪化清退 |

> **管理的本质不是管人，是管规则和管预期。**
> 
> 好的供应商管理不是"把每个供应商都管到完美"，而是"建立一套让供应商自己变好的规则"。

## 季度评估触发机制

> [来源：SM-2026-035 §第三章 / §第十条]  
> 核心定位：季度九宫格评估后，C档且趋势下降明显的供应商**触发评估**，A档和B档均为可用状态，不等待连续月度预警

### 评估触发条件

> [来源：SM-2026-035_供应商分级清退管理制度.md §评估触发]

| 条件组合 | 判定 | 动作 | 时效 |
|---------|------|------|------|
| **C档 + 趋势下降↓** | 低贡献且恶化 | **触发评估**（不一定立即清退） | 季度评估后立即 |
| C档 + 趋势持平→ | 低贡献但未恶化 | 30天整改期，提交改善计划 | 季度评估后1周内 |
| C档 + 趋势上升↑ | 低贡献但有改善 | 30天整改期，给予资源观察 | 季度评估后1周内 |
| 红线违规 | 安全/合规事件 | 直接评级C，立即启动清退评估 | 发现后72h内 |

> **重要**：C档+趋势下降≠自动清退。进入评估流程后，根据评估结果决定：整改/延长观察期/启动清退。

> **与月度触发规则的衔接**：月度"连续3月后30%"（[05 §1](05_方法论体系.md) 触发规则）和季度"C档+趋势下降"是互补关系。月度用于及时发现异常并干预，季度用于综合判定生态位。实践中，月度连续预警的供应商大概率会在季度评估中落入C档。

### 评估后处理流程

> [来源：SM-2026-035_供应商分级清退管理制度.md §评估流程]

```
季度评估结果C档+趋势下降
    │
    ▼
触发评估
    │
    ├─ 评估结果：整改可行 → 30天整改期 + 月度跟踪
    ├─ 评估结果：无改善可能 → 启动清退流程
    └─ 评估结果：特殊情况 → 延长观察期
    │
    ▼
如需清退：
    72h内发出清退预警通知
    5个工作日内提交清退方案
    C3审批（王易人）→ 如涉及全业务线则C2审批（军哥）
    执行交接（标准周期参照常规清退8阶段流程）
```

### 观察区（新供应商保护期）

> [来源：SM-2026-035 §第九条]

| 属性 | 说明 |
|------|------|
| **保护期长度** | 入项不足2个季度（约6个月） |
| **评估方式** | 正常记录评分和趋势，但不映射九宫格 |
| **管理动作** | 按冷启动SOP执行30/60/90天跟进培育 |
| **保护内容** | 不触发清退、不缩减份额 |
| **退出条件** | 保护期结束后自动纳入九宫格评估；如提前达标可申请提前退出 |

> **与6月里程碑的衔接**：观察区（2季度保护）和6月里程碑（[05 §8](05_方法论体系.md)）时间窗口基本一致。观察区强调"不触发清退的保护"，6月里程碑强调"量化的评判标准"。两者并用：前6个月在观察区内按里程碑跟踪，第6月末同时完成里程碑评判和首次九宫格映射。

## 供应商业务测试管理

> [来源：SM-2026-050_清洁版操作指引.md §第五条]  
> 核心定位：通过标准化测试流程验证新供应商/新业务的承接能力，从源头控制业务风险

### 测试适用场景

> [来源：SM-2026-050_清洁版操作指引.md §第五条(1)]

**业务测试触发条件**：

| 场景 | 说明 |
|------|------|
| 新供应商引入 | 验证新引入供应商的实际业务承接能力和服务品质 |
| 现有供应商扩项 | 验证现有供应商承接新业务线的专业能力和适应性 |
| 业务模式验证 | 供应商POC测试，验证跑通业务模式 |
| 复测验证 | 供应商绩效改善后的重新测试 |

**测试类型分类**：

| 类型 | 定义 |
|------|------|
| 引入测试 | 针对新供应商准入前的业务能力验证性测试 |
| 扩项测试 | 针对现有供应商新增业务线的能力适配性测试 |
| 创新测试 | 针对创新业务模式的单供应商POC测试 |
| 复测验证 | 针对供应商的恢复性验证测试 |

### 差异化测试标准

> [来源：SM-2026-050_清洁版操作指引.md §第五条(2)]

| 维度 | 核心业务测试标准 | 创新测试标准 |
|------|----------------|-------------|
| 测试周期 | 3-6个月 | 最长不超过12周（约3个月） |
| 测试规模 | 10-100人 | 5-10人 |
| 考核方式 | 以季度为周期进行综合考核 | 测试期末进行综合评估 |
| 通过标准 | 综合绩效评分达到或超过预设的策略目标值 | 核心绩效排名指标达到预设目标，且运营流程顺畅 |

### 测试审批流程

> [来源：SM-2026-050_清洁版操作指引.md §第五条(3)]

**第一阶段：测试方案制定（3个工作日）**

1. 供应商管理牵头制定《供应商业务测试方案》
2. 方案内容需明确：
   - 测试目标、测试范围、参与供应商、测试周期
   - **测试期人力需求**（如核心业务**5人测试**、创新业务5-10人）
   - **转正后预期规模**：明确转正后3-6个月的预期人力规模、月均业务量、预期产值（立项评估核心依据）
   - 核心考核指标（KPI）、资源投入及风险预案
3. 与业务运营岗、策略运营岗共同评审并确认方案的可行性与考核标准的合理性

**第二阶段：审批流程**

| 测试类型 | 审批路径 | 决策时效 |
|---------|---------|---------|
| 核心业务测试 | 业务运营岗发起 → **C3审批 → C2终审** | 5个工作日 |
| 创新测试 | 业务运营岗发起 → **C3审批（终审）** | 1个工作日（备案） |

**第三阶段：测试执行**

| 角色 | 职责 |
|------|------|
| 供应商管理岗 | 整体测试过程的协调、监控和问题升级 |
| 业务运营岗 | 日常运营督导、数据追踪和现场管理 |
| 质检培训岗 | 测试期间的录音质检、合规检查及专项培训支持 |

**第四阶段：测试结果评估与过渡期**

> [来源：SM-2026-050_清洁版操作指引.md §第五条(3)]

1. **评估过渡期**：测试期结束后自动进入**5个工作日**的评估过渡期，期间业务数据正常交付
2. **过渡期结算规则**：
   - 若供应商最终**转正成功**：此5天计入正式合作期，按标准结算
   - 若**未通过**：按事先约定的测试期规则进行结算
3. 过渡期内，由业务运营岗牵头，联合质检培训岗出具《供应商测试期综合评估报告》
4. 供应商管理岗基于评估报告，判断供应商是否达到转正标准，并启动转正或清退流程
5. 所有测试相关文件（方案、报告、审批记录）均需在京音平台/XBP留档

## 尾部供应商快速汰换机制

> [来源：SM-2026-050_清洁版操作指引.md §第九条]  
> 核心定位：通过客观量化标准识别尾部供应商，执行高效有序的替换流程，实现供应商生态良性循环

### 汰换触发标准

> [来源：SM-2026-050_清洁版操作指引.md §第九条(2)]  
> **同时满足以下所有三个条件**的供应商，自动进入尾部供应商汰换候选名单：

| 条件 | 标准 |
|------|------|
| **合作时长** | 与京东金融电销业务合作时间**连续超过12个月** |
| **绩效表现** | **最近两个连续的日历季度**，在合作一年以上的供应商里综合绩效排名中均处于**后30%** |
| **运营规模** | 近三个月，承接京东业务的**日均出勤人力规模低于30人**或该项目总人力的**5%** |

### 快速汰换决策流程

> [来源：SM-2026-050_清洁版操作指引.md §第九条(3)]

**1. 自动识别与预警**

- **触发**：每季度末，供应商管理岗根据系统数据，筛选出完全符合上述三个标准的供应商
- **预警**：向被标记的供应商发出《绩效改进与汰换风险预警函》，给予**1个月的观察改进期**

**2. 汰换策略制定与审批**

- **触发**：观察期结束后，若供应商绩效仍无明显改善，立即启动汰换程序
- **策略制定**：供应商管理岗联合业务运营岗，在**3个工作日**内制定《尾部供应商汰换与替换方案》，方案必须包含：
  - **汰换原因**：明确引用触发条件
  - **替换策略**（二选一）：
    - **策略A：引入新供应商** — 适用于现有中小供应商无承接能力，或需引入新竞争者的情况
    - **策略B：份额再分配** — 适用于现有中小供应商有富余产能，可快速承接业务的情况
  - **执行计划**：详细的业务交接时间表、客户保障方案和风险控制措施
- **审批路径**：供应商管理岗发起 → **C3审批 → C2终审**
- **决策时效**：C3审批不超过**2个工作日**，C2审批不超过**3个工作日**

**3. 执行与交接**

- 审批通过后，立即向目标供应商发出《清退通知函》，并启动《供应商退出交接表》流程
- 交接缓冲期原则上不超过**2周**，以确保业务快速切换

### 替换资源执行策略

> [来源：SM-2026-050_清洁版操作指引.md §第九条(4)]

| 策略 | 执行要点 |
|------|---------|
| **策略A：引入新供应商** | 立即启动供应商引入流程，标记为"高优先级/紧急"，缩短各环节处理时间。目标：在目标供应商退出前，完成新供应商的测试方案，实现无缝衔接 |
| **策略B：份额再分配** | 业务运营岗根据中小供应商的历史绩效、当前产能和承接意愿，制定《业务量转移分配表》。需获得**C2最终批准**。向获得增量的供应商发出《业务量调整通知函》，确保其在规定时间内完成人力和系统的准备 |

### 留痕机制

> [来源：SM-2026-050_清洁版操作指引.md §第九条(5)]

- 《绩效改进与汰换风险预警函》及供应商回执
- 《尾部供应商汰换与替换方案》及其审批流记录
- 《清退通知函》及《供应商退出交接表》
- 《业务量转移分配表》（如适用）

## 供应商续签管理规则

> [来源：SM-2026-050_清洁版操作指引.md §第十条]  
> 核心定位：建立科学公正的续签评估体系，确保合作期满的供应商持续满足业务发展要求

### 续签评估触发

> [来源：SM-2026-050_清洁版操作指引.md §第十条(2)]

| 节点 | 动作 | 时效 |
|------|------|------|
| 合同到期前**90天** | 供应商管理岗启动续签评估程序 | — |
| 合同到期前90天 | 向供应商发送《合作到期及续签评估告知函》，明确评估流程和时间安排 | — |

### 评估流程与标准

> [来源：SM-2026-050_清洁版操作指引.md §第十条(3)]

#### a）硬性门槛（一票否决）

> [来源：SM-2026-050_清洁版操作指引.md §第十条(3)a]

| 门槛 | 标准 | 触发后处理 |
|------|------|-----------|
| 人力规模门槛 | 当前合同期内，专职坐席团队**日均出勤规模持续低于30人** | 次年原则上不再续签，直接进入"不续签"决策流程，无需启动后续评分 |

**执行**：由结算岗和业务运营岗提供合同期内的人力数据，供应商管理岗进行核验。

#### b）续签评估评分卡

> [来源：SM-2026-050_清洁版操作指引.md §第十条(3)b]  
> 通过硬性门槛后，使用百分制评分卡进行综合评估。**综合得分≥80分**方可获得续签资格。

| 评估维度 | 分值 | 评估标准与数据来源 |
|---------|------|-------------------|
| **历史业绩** | 70分 | 合同期内的GMV、转化率、客诉率、赛马排名等核心KPI的达成趋势及稳定性 |
| **合规表现** | 15分 | 是否发生重大合规、信息安全、数据泄露事件，以及日常巡检合规表现 |
| **协作配合度** | 15分 | 日常沟通效率、问题响应速度、项目配合主动性、结算协同顺畅度 |
| **总计** | 100分 | |

### 续签决策与审批

> [来源：SM-2026-050_清洁版操作指引.md §第十条(4)]

| 步骤 | 动作 | 时效 |
|------|------|------|
| 评估执行 | 合同到期前**60天**内，完成《供应商续签评估评分卡》填写，附上所有支撑报告 | — |
| 决策建议 | 基于评分结果，形成"建议续签"或"建议不续签"的初步结论 | — |
| 审批流程 | 发起《供应商续签评估评分卡》及支撑材料审批 | C3审批≤5工作日，C2审批≤7工作日 |

**审批路径**：供应商管理岗发起 → **C3审批 → C2终审**

### 结果应用与通知

> [来源：SM-2026-050_清洁版操作指引.md §第十条(5)]

| 结果 | 动作 | 时效 |
|------|------|------|
| 续签 | 合同到期前**60天**启动合同续签谈判与签署流程 | — |
| 不续签 | 合同到期前**60天内**向供应商发出《不予续签通知函》，并启动供应商退出管理流程，确保业务平稳过渡 | — |

### 中小供应商成长机制

> [来源：SM-2026-050_清洁版操作指引.md §第十条(6)]

| 属性 | 说明 |
|------|------|
| 适用对象 | 合作未满一年的新引入供应商 |
| 机制说明 | 首个合作年度内，拥有**最多3次"业务扩项"申请机会** |
| 执行流程 | 供应商根据自身能力主动申请承接现有合作范围之外的新业务线 → 供应商管理岗按照5.3节"现有供应商业务线扩增规则"的流程进行审批 |

## 供应商申请退项

> [来源：SM-2026-050_清洁版操作指引.md §第六章第五条]  
> 核心定位：供应商主动申请退出部分业务线的处理流程

| 属性 | 内容 |
|------|------|
| **场景** | 供应商主动申请退出部分业务线 |
| **审批发起条件** | 收到供应商正式的《退项申请函》 |
| **审批材料** | ①供应商《退项申请函》②《退项交接表》 |
| **审批路径** | 供应商管理岗在京音平台/XBP发起 → **C3审批 → 通知C2** |
| **决策时效** | C3审批不超过3个工作日，C2审批不超过5个工作日 |

## 分量/扩项调整审批

> [来源：SM-2026-050_清洁版操作指引.md §第六章第六条]  
> 核心定位：对现有供应商进行业务量调整或新增业务线的审批流程

| 属性 | 内容 |
|------|------|
| **场景** | 对现有供应商进行业务量调整或新增业务线 |
| **审批发起条件** | 收到业务运营岗《需求申请表》或主动发起调整 |
| **审批材料** | ①《需求申请表》②《分量/扩项方案》③供应商能力评估 |
| **审批路径（人力>10人）** | 供应商管理岗发起 → **C3审批 → C2终审** |
| **审批路径（人力≤10人）** | 供应商管理岗发起 → **C3审批（终审）** |
| **决策时效** | C3审批不超过2个工作日，C2审批不超过3个工作日 |
