## 一、五维考核评分模型

### 1.1 维度权重

| 维度 | 权重 | 说明 |
|------|------|------|
| **产能** | 30% | GMV/人力规模/增长速度 |
| **效率** | 25% | 人均产出/响应速度/达标率 |
| **品质** | 15% | 质检合格率/客户满意度/投诉率 |
| **配合度** | 15% | 响应速度/信息透明度/整改配合 |
| **合规性** | 15% | 违规次数/红线触碰/整改闭环 |

### 1.2 ABCD评级标准

| 评级 | 综合得分 | 产能得分 | 各维度最低线 | 管理动作 |
|------|---------|---------|------------|---------|
| **A（优秀）** | ≥85 | ≥90 | ≥75 | 优先新业务、价格激励、标杆宣传 |
| **B（良好）** | 75-84 | ≥80 | ≥65 | 维持合作、针对性提升 |
| **C（合格）** | 65-74 | ≥70 | ≥55 | 问题诊断、限期整改、限制业务 |
| **D（不合格）** | <65 | <70 | <55 | 业务收缩、退出评估、替代方案 |

### 1.3 红线一票降级

| 红线行为 | 处理 |
|---------|------|
| 数据泄露/泄露客户信息 | 直接降至D级 |
| 录音篡改/伪造 | 直接降至D级 |
| 重大合规事件 | 直接降至D级 |
| 连续两期D级 | 强制退出 |

## 二、红黄牌机制

### 2.1 触发条件

| 类型 | 触发条件 | 处理措施 |
|------|---------|---------|
| **黄牌** | 单项指标连续2期低于警戒线 | 口头警告、限期整改（30天） |
| **红牌** | 单项指标连续3期低于警戒线 / 整改不达标 / 触碰红线 | 暂停新业务、启动退出评估 |

### 2.2 效果评估

| 等级 | 标准 | 结果 |
|------|------|------|
| 整改优秀 | 下期考核提升2级 | 恢复正常、加分 |
| 整改合格 | 下期考核提升1级 | 恢复正常 |
| 整改不合格 | 未提升或下降 | 升级处理 |

## 三、赛马机制

### 3.1 目的

通过数据排名公开化，刺激供应商自我竞争。

### 3.2 核心规则

| 规则 | 说明 |
|------|------|
| **月度排名** | 按五维总分排名 |
| **公开透明** | 排名数据对供应商公开 |
| **优胜劣汰** | 末位淘汰机制 |
| **动态调整** | 新供应商1-3个月观察期，不参与排名 |

### 3.3 实施要点

| 要点 | 说明 |
|------|------|
| **排名维度** | 单维度排名 + 综合排名，多维度对比 |
| **对比基准** | 不只看绝对值，看相对大盘/同类供应商表现 |
| **反馈机制** | 排名变化时主动告知供应商原因 |
| **激励联动** | 排名与分量/价格/新业务挂钩 |
| **避免内卷** | 设定合理目标，防止恶性竞争 |

## 四、分量触发条件

### 4.1 分量增加触发条件

| 触发条件 | 说明 |
|---------|------|
| 业务增长 | 整体业务增长，产能不足 |
| 新增业务线 | 开辟新业务线 |
| 战略调整 | 业务策略调整导致部分供应商业务量增加 |
| 供应商表现优异 | 连续考核优秀、产能提升 |

### 4.2 分量减少触发条件

| 触发条件 | 说明 |
|---------|------|
| 业务量收缩 | 整体业务量下降 |
| 绩效惩罚 | 连续考核不达标或排名末位 |
| 风险管控 | 出现一般性违规或潜在风险 |
| 策略调整 | 业务策略调整 |

### 4.3 分量调整审批

| 调整幅度 | 审批要求 |
|---------|---------|
| <50% | 供应商管理岗发起 → C3审批 |
| ≥50% | 标准流程 + C2终审 |
| ≥100%（翻倍） | 专项汇报 → C2 |
| ≤10人（快速调整） | 5工作日内完成，向C3京ME备案，无需事前审批 |

## 五、清退规则

### 5.1 触发条件

| 条件 | 说明 |
|------|------|
| 连续考核不达标 | 连续两期D级 |
| 触碰红线 | 数据泄露、录音篡改等 |
| 整改不合格 | 黄牌整改后仍未达标 |
| 主动退出 | 供应商主动提出 |
| 业务策略调整 | 业务线取消或调整 |

### 5.2 退出流程

```
启动评估 → 替代方案 → 数据回收 → 费用清算 → 系统关闭 → 归档
```

| 环节 | 要求 |
|------|------|
| 启动评估 | 72小时内触发 |
| 替代方案 | 5个工作日内完成 |
| 数据回收 | 客户资料/业务数据/录音等 |
| 费用清算 | 结算至退出当日 |
| 系统关闭 | 账号/权限/系统接入关闭 |
| 归档 | 退出报告、数据归档 |

### 5.3 退出原则

- **有据可依**：触发条件明确、证据充分
- **风险隔离**：单业务线退出不影响其他线
- **平稳过渡**：有过渡计划、最小化业务中断

## 六、两套体系的关系说明

| 维度 | 五维考核 | 赛马机制 | SLA |
|------|---------|---------|-----|
| **定位** | "得多少分"——量化评估与排名 | "资源怎么分"——竞争分配 | "能不能接受"——服务质量底线 |
| **触发** | 每月固定评估 | 新增资源/定期评估/策略变更/申诉 | 日常监控 |
| **逻辑** | 按档打分，不考虑连续性 | 排名竞争，优胜劣汰 | 连续不达标才触发管理动作 |
| **输出** | ABCD评级+红黄牌 | 分量调整+末位淘汰 | 预警/红线/清退 |
| **使用者** | 供应商管理岗（你） | 策略组+供应商管理岗 | 供应商管理岗+供应商 |

> **关键区分示例**：产能达标率82%
> - SLA视角：<90%即不达标，82%触发**预警**（"需要整改"）
> - 评估方案视角：80-89%得**8分**（"扣2分，但仍有一定产出"）
> - 赛马视角：如果同产线其他供应商平均85%，82%排名靠后→**减量**

## 七、考核执行SOP

### 6.1 月度考核流程

```
月初：数据提取 → 自动评分 → 人工复核 → 结果确认 → 反馈供应商
```

### 6.2 关键时点

| 时点 | 动作 |
|------|------|
| 每月1日 | 提取上月数据 |
| 每月5日 | 完成自动评分 |
| 每月8日 | 人工复核完成 |
| 每月10日 | 结果确认、通知供应商 |
| 每季度 | 综合评级、红黄牌判定 |
