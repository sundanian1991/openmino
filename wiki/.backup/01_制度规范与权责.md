## 一、角色与RACI矩阵

| 角色 | 职责 | 在供应商管理中的定位 |
|------|------|-------------------|
| **供应商管理岗** | 供应商全生命周期管理、5维考核、分量建议、日常巡检 | **唯一负责**（建议权+流程管理权） |
| **业务运营岗** | 承接供应商管理岗分量方案、日常管理与培训、业务推动 | 执行层 |
| **业务质量岗** | 品质管理与检查、满意度调查、品质提升、退供执行 | 监督层 |
| **质检培训岗** | 服务质量与人员管理、质量检查、员工培训、红线管理 | 监督+赋能层 |
| **策略运营岗** | 策略制定、模型迭代、策略应用推动、定价与分量 | **定价权**（刘伟佳团队） |
| **结算岗** | 结算核算、薪酬数据、对账与争议处理 | 支持层 |
| **资源规划岗** | 需求预测、产能规划、资源协调 | 支持层 |

**关键权责不对等**：
- 供应商流失风险由年老师（资源规划岗？）承担
- 定价权在策略组（刘伟佳）
- 引入/清退建议权+流程管理权在你（供应商管理岗）
- **结构性困境**：权责分离，任何涉及清退/定价的方案需设计**间接杠杆路径**

## 二、业务分类与定价体系

### 2.1 业务分类

| 类别 | 定义 | 适用业务 | 供应商要求 |
|------|------|---------|-----------|
| **核心业务** | 团队承接业务总量≥70% | 金条、企业主贷大额、信用卡 | 资金/人员/职场充裕，风险承受强 |
| **创新业务** | 团队承接业务总量<70%（可含存量） | 财富等 | 较低门槛，注重成长潜力 |

### 2.2 定价要素

| 要素 | 说明 |
|------|------|
| **业务类别** | 核心/创新不同定价策略 |
| **供应商等级** | 根据历史表现和综合能力分级，高等级获更优价格 |
| **业绩指标** | 达成率、满意度、转化率等直接挂钩价格 |
| **合作期限** | 长期合作享价格优惠 |
| **市场竞争** | 参考市场行情和竞争对手定价 |

### 2.3 结算规则（关键）

- **结算依据**：以京东科技确认的**有效营销成果**为准
- **对账周期**：每月一次，次月15日前完成
- **付款方式**：银行转账，确认无误后30个工作日内支付
- **预留金**：预留结算总额5%作为预留金，合作终止且无遗留问题后结算

## 三、供应商分类分级

### 3.1 分类（按业务线）

| 分类 | 定义 | 管理策略 |
|------|------|---------|
| **核心供应商** | 业务占比≥40%或排名前三 | 深度绑定、优先资源、战略投资 |
| **常规供应商** | 业务占比20%-40% | 标准化管理、定期评估 |
| **补充供应商** | 业务占比<20% | 灵活管理、机会性合作 |
| **备选供应商** | 暂无业务但已评估 | 持续观察、储备激活 |

### 3.2 分级（按综合能力）

| 等级 | 综合能力评分 | 品质评分 | 业绩达标率 | 管理策略 |
|------|------------|---------|-----------|---------|
| **S级** | ≥90分 | ≥85分 | ≥90% | 资源倾斜、战略合作、优先新业务 |
| **A级** | 80-89分 | ≥75分 | ≥85% | 维持合作、重点提升、激励措施 |
| **B级** | 70-79分 | ≥65分 | ≥75% | 问题改进、限制业务、风险监控 |
| **C级** | <70分 | <65分 | <75% | 业务收缩、整改要求、退出评估 |

## 四、立项评估（准入前）

### 4.1 评估维度

1. **企业资质**：注册资本、经营年限、专业资质、财务状况
2. **业务能力**：过往案例、技术能力、人员规模、服务范围
3. **管理水平**：组织架构、管理制度、培训体系、质控流程
4. **合规记录**：历史违规、法律诉讼、行政处罚、信用记录
5. **合作意愿**：合作态度、沟通效率、响应速度、长期规划

### 4.2 一票否决项

- 提供虚假资质或财务信息
- 过去3年内有重大合规违规记录
- 存在资金链断裂风险或重大诉讼纠纷
- 实际控制人/高管有严重不良记录

### 4.3 准入流程

```
立项申请 → 资质审核 → 现场考察 → 综合评估 → 审批决策 → 合同签订 → 系统配置 → 试单启动
```

**审批权限**：
- 常规立项：C3审批
- 涉及特殊资源/大额投入：C2终审

## 六、巡检标准

### 6.1 巡检频次

| 供应商等级 | 巡检频次 | 方式 |
|-----------|---------|------|
| S级 | 每半年一次 | 现场+远程结合 |
| A级 | 每季度一次 | 现场为主 |
| B级 | 每月一次 | 现场为主 |
| C级 | 每两周一次 | 现场为主，重点关注 |

### 6.2 巡检12项核心内容

1. 人员规模与配置
2. 职场环境与安全
3. 系统设备与网络
4. 培训体系与执行
5. 质量管控流程
6. 合规管理制度
7. 数据安全与保密
8. 员工权益保障
9. 业务运营数据
10. 客户投诉处理
11. 应急响应机制
12. 合作配合度

## 八、SLA量化指标体系

> SLA与《供应商月度综合评估方案》是两套不同维度的体系。
> - SLA定位："能不能接受"——定义服务质量底线
> - 评估方案定位："得多少分"——量化评估与排名

### 8.1 核心交付指标

| 指标 | 定义 | 达标线 | 预警线 | 红线 |
|------|------|--------|--------|------|
| 日均出勤率 | Σ每日实际出勤÷Σ每日应出勤×100% | **≥95%** | **<90%** | **<85%** |
| 产能达标率 | 月度实际产能÷目标产能×100% | **≥90%** | **<80%** | **<70%** |
| 数据报送及时率 | 按时报送次数÷总应报送次数×100% | **≥98%** | **<95%** | **<90%** |
| 数据准确率 | 经核实准确数据条数÷总报送条数×100% | **100%** | **<98%** | **<95%** |

### 8.2 合规指标

| 指标 | 达标要求 | 预警线 | 红线 |
|------|---------|--------|------|
| 信息安全 | **0违规** | — | ≥1次严重违规→触发清退评估 |
| 人员持证率 | **100%** | <95% | <90% |
| 录音留存率 | **100%** | <98% | <95% |
| 话术合规率 | **≥95%** | <90% | <85% |

### 8.3 服务响应指标

| 场景 | 响应要求 |
|------|----------|
| 群消息 | **30分钟内回复** |
| 异常事项（一般） | 当日群内报备 |
| 异常事项（重要） | **2小时内**书面+电话报备 |
| 异常事项（紧急） | **立即**电话报备 |
| 整改通知响应 | **24小时内**提交整改方案 |
| 月度总结提交 | 次月**3日前** |
| 人员变动报备 | 变动前**3个工作日** |
| 大面积人员变动（≥3人） | **即时**电话+书面报备 |

### 8.4 达标/预警/红线响应流程

```
指标触及预警线
  → 供应商管理岗发出预警通知
  → 供应商提交整改方案
  → 整改期满复评
      ├─ 达标 → 解除预警
      └─ 未达标 → 升级处理

指标触及红线
  → 供应商管理岗启动清退评估
  → 业务对接人确认
  → 正式清退流程
```

## 九、赛马机制完整规则

### 9.1 触发条件

| 类型 | 触发场景 |
|------|---------|
| 常规 | 连续3日转化率低于平均20%、新增资源、定期评估 |
| 专项 | 策略变更、供应商申诉 |

### 9.2 考核维度（100分）

| 维度 | 权重范围 |
|------|---------|
| 转化效率 | 30%-50% |
| 运营成本 | 20%-40% |
| 过程质量 | 15%-25% |
| 客户体验 | 5%-15% |

### 9.3 结果应用

- **份额调整**：新份额 = 基础份额 × (1 + 绩效系数)
- **末位淘汰**：连续两次末位启动淘汰
- **系数审批**：需C3审批，每季度回顾更新
- **豁免机制**：预计月度结算<5万元可豁免（业务运营岗发起、供应商管理岗审批）

> **重要**：赛马机制与五维考核是**独立体系**——赛马管"资源分配竞争"，五维管"综合评估排名"。

## 十、集中度管控

### 10.1 n-1业务参与上限原则

- 单一供应商最多参与 n-1 个核心业务
- 当前4个核心业务，最多参与3个
- 豁免需C2终审（特例，不具普适性）

### 10.2 单一业务量集中度上限

**AI场景**（按供应商数量动态）：
- ≤3家：单一供应商≤70%
- 4家：≤60%
- 5家：≤55%
- 6家：≤50%
- 7-8家：≤45%
- ≥9家：≤40%

### 10.3 跨业务线人力集中度

- 单一供应商总人力占该业务线整体人力≤40%
- 前三大供应商合计占比≤60%

## 十二、核心原则汇总

| 原则 | 出处 | 含义 |
|------|------|------|
| 有据可依 | 清退规则 | 所有管理动作必须有制度依据和数据支撑 |
| 风险隔离 | 清退规则 | 单业务线退出不影响其他业务线 |
| 平稳过渡 | 清退规则 | 清退必须有替代方案和过渡计划 |
| 排名思维 | 实操指南 | 看相对排名不看绝对分数 |
| 三要素法 | 实操指南 | 每次沟通必问：原因？目标？举措？ |
| 诊断驱动 | 实操指南 | 头痛不医头，先诊断再开药 |
| 培养悖论 | 实操指南 | 越给方案越依赖，越让他想越有执行力 |
| 梯队备份 | 实操指南 | 永远有B计划 |
| 六步法闭环 | 实操指南 | 发现→诊断→方案→跟踪→评估→优化 |
