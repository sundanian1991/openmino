# 02 全生命周期流程

> 来源：供应商全生命周期管理框架-团队实操指南.docx
> 核心框架：选 · 育 · 留 · 汰

---

## 总览

供应商管理不是单一动作，而是贯穿四个阶段的闭环体系。

```
准入期（选） → 成长期（育） → 稳定期（留） → 退出期（汰）
    ↑                                              │
    └────────────── 备选池 ────────────────────────┘
```

**核心管理思维转变**：
- 从"管人"转向"管规则和管预期"
- 把供应商当作微型组织理解，不是简单的服务提供方
- 每个阶段都有预设的管理动作和判断标准（不是等出问题才应对）

---

## 阶段一：准入期（选）

**管理定位**：宽进严出，用试单验证代替完美评估

### 硬性门槛（不满足直接否决）

| 维度 | 标准 |
|------|------|
| 注册资本 | ≥1000万元 |
| 资质许可 | 呼叫中心经营许可证，外包服务资质齐全 |
| 经营年限 | ≥3年 |
| 人员规模 | ≥100人 |
| 合规记录 | 近3年无重大违规 |
| 财务状况 | 无恶化迹象、无重大诉讼 |

### 一票否决红线

- 资质造假
- 重大违规（数据泄露、录音篡改等）
- 财务恶化（资金链断裂、失信、重大诉讼败诉）
- 老板/实控人的人品问题（行贿、诈骗等）

### 准入诊断6问

| 问题 | 核心判断 |
|------|---------|
| 1. 为什么必须是这家？ | 市场有没有更好选择？不可替代优势是什么？ |
| 2. 他的核心能力是什么？ | 人力规模/管理经验/技术系统/成本优势？ |
| 3. 他的老板靠谱吗？ | 决策能力、风险意识、诚信度、行业经验？ |
| 4. 他愿不愿意投入？ | 重视程度、资源投入意愿、内部激励机制？ |
| 5. 3年后还需要他吗？ | 短期补充还是长期合作？依赖度如何？ |
| 6. 如果他不行，替代方案是什么？ | 有没有B计划？切换成本多高？ |

> **两个灵魂问题贯穿始终**："为什么必须是这家？" + "3年后还需要吗？"

### 现场考察3维度

1. **看办公环境**：场地规模、工位数量、设备配置、员工状态（精神面貌/忙碌程度）
2. **聊管理团队**：对接人决策权、老板态度、管理层稳定性、团队氛围
3. **查实际能力**：现有客户案例、人员流失率、培训体系、质控流程、系统工具

### 核心策略

- **宽进严出**：门槛合理但不过高，进入后用试单验证
- **多家试单PK**：同时引入3-5家，用实际业绩说话
- **不依赖单一指标**：综合评估资质、能力、态度、投入意愿
- **试单期管理**：清晰KPI + 1-3个月观察期

## 六盒诊断模型（跨阶段通用）

> 源自韦斯伯德(Weisbord)组织诊断框架，适配到供应商管理。

| 维度 | 诊断内容 | 典型问题 |
|------|---------|---------|
| **目的** | 供应商是否清楚合作目标？目标是否可衡量？ | "他们不知道我们要什么" |
| **结构** | 组织架构是否合理？职责是否清晰？ | "对接人换了3次，每次都要重新讲" |
| **关系** | 供应商内部协作是否顺畅？与京东对接是否高效？ | "他们内部信息不互通" |
| **奖励** | 激励机制是否有效？员工是否有动力？ | "员工抱怨提成低，积极性差" |
| **领导** | 管理层能力如何？决策是否及时？ | "老板不管事，主管说了算但又定不了" |
| **支持机制** | 培训、系统、工具是否到位？ | "系统老坏，培训靠口头" |

> **使用时机**：供应商出现多维度问题但看不清根因时，用六盒逐一排查。

## 阶段二：成长期（育）

**管理定位**：帮助供应商从"能干活"到"干得好"

这是投入最多、问题最多、也最有价值的阶段。

### 关键动作

| 动作 | 说明 |
|------|------|
| 系统对接 | 完成业务系统、数据报表、沟通机制的对接 |
| 培训赋能 | 业务知识、操作流程、合规要求培训 |
| KPI设定 | 明确短期（1-3月）和中期（3-6月）目标 |
| 密集监控 | 每周数据跟踪、每月复盘、及时纠偏 |
| 问题诊断 | 出现偏差时，快速定位根因 |
| 资源支持 | 必要时提供额外培训、人员支持、流程优化建议 |

### 成长期核心三问

> **根本原因是什么？能改进吗？需要什么支持？**

- 先诊断根因 → 再判断可改进性 → 最后明确支持需求
- 不诊断就支持是浪费，不判断就开药是冒险

### 主管胜任5项指标（成长期关键评估）

| 指标 | 达标线 | 说明 |
|------|--------|------|
| 流失率 | <8%/月 | 团队稳定度 |
| 出勤率 | >90% | 出勤稳定度 |
| 人效走势 | 不连续2个月下降 | 业绩趋势 |
| 预警后48h回复率 | >80% | 响应质量 |
| 整改承诺按期完成率 | >70% | 整改执行力 |

- 5项全过 → 主管合格，问题在其他地方
- 3-4项过 → 主管有短板，看短板在哪（稳定度和出勤是硬伤）
- ≤2项过 → 主管不行，建议更换（需王易人确认）

## 阶段三：稳定期（留）

**管理定位**：维持并提升供应商的长期价值

### 核心风险：绑架能力

稳定期最危险的不是"业绩下滑"，而是"业绩好但有人能绑架你"。

| 绑架信号 | 说明 |
|---------|------|
| 份额占比过高 | 某供应商单业务线占比>50% |
| 业绩排名独占 | 某供应商长期霸榜，无有效竞争 |
| 人力规模唯一 | 某区域/业务只有一家能承接 |
| 切换成本极高 | 历史数据/客户关系/系统深度绑定 |

### 稳定期10大风险维度

| 维度 | 风险信号 |
|------|---------|
| 1. 人力集中度 | 单一供应商人力占比>15% |
| 2. 团队稳定性 | 对接人/主管/核心员工流失趋势恶化 |
| 3. 合规风险 | 质检合格率下滑、踩红线苗头 |
| 4. 财务状况 | 资金链风险、经营困难 |
| 5. 客户结构变化 | 接了更大客户、对京东优先级下降 |
| 6. 管理松懈迹象 | 日报/周报质量下降、例会缺席、响应变慢 |
| 7. 第二梯队成熟度 | 备选供应商是否具备快速承接能力 |
| 8. 行业竞争态势 | 市场是否有更好替代选项 |
| 9. 合作意愿变化 | 消极怠工、敷衍应对、选择性执行 |
| 10. 产能集中度 | 单一供应商产能占比>20% |

> **使用时机**：每月固定做一次10维风险评估，不是等出问题才看。

### 应对策略

> **谁有绑架能力？第二梯队是否就位？**

- **梯队备份**：每个业务都要有第二梯队
- **培育方法**：新业务优先给中腰部供应商、定期给第二梯队试单、保持信息同步
- **梯队备份不是"不信任"，是风险管理基本功**

### 日常管理动作

- 定期复盘（月度/季度）
- 持续优化流程和效率
- 激励优秀供应商（更多业务、更好价格、荣誉认可）
- 关注人员稳定性和管理连续性

## 阶段四：退出期（汰）

**管理定位**：有序终止合作，最小化损失

### 退出触发条件

见 [03 考核评分与赛马机制](03_考核评分与赛马机制.md) 清退规则。

### 退出前两个灵魂问题

> **真的触发清退了吗？替代方案是什么？**

- 不是一时冲动 → 确认触发条件
- 不会业务中断 → 替代方案就绪

### 退出管理原则

- **有据可依**：触发条件明确、证据充分
- **风险隔离**：单业务线退出不影响其他线
- **平稳过渡**：有过渡计划、数据回收、费用清算

## 六步法闭环（跨阶段通用）

```
发现问题 → 诊断根因 → 制定方案 → 执行跟踪 → 效果评估 → 持续优化
```

**关键**：很多管理者做到了前三步，漏了后三步。

| 步骤 | 常见问题 | 正确做法 |
|------|---------|---------|
| 发现问题 | 忽视预警信号 | 建立多维度监控体系 |
| 诊断根因 | 头痛医头 | 每个症状背后至少两种根因可能 |
| 制定方案 | 一刀切 | 定制化、可执行、有验收标准 |
| 执行跟踪 | 发了方案就不管 | 明确责任人、时间节点、检查点 |
| 效果评估 | 没有量化 | 对比前后数据，评估ROI |
| 持续优化 | 同样问题反复出现 | 沉淀为规则/SOP |

---

## 管理心法（四句话）

| 阶段 | 心法 | 核心问题 |
|------|------|---------|
| **选** | 为什么必须是这家？3年后还需要吗？ | 避免"为了准入而准入" |
| **育** | 根本原因是什么？能改进吗？需要什么支持？ | 找到卡住的齿轮 |
| **留** | 谁有绑架能力？第二梯队是否就位？ | 防止被绑架 |
| **汰** | 真的触发清退了吗？替代方案是什么？ | 避免情绪化清退 |

> **管理的本质不是管人，是管规则和管预期。**
> 
> 好的供应商管理不是"把每个供应商都管到完美"，而是"建立一套让供应商自己变好的规则"。
