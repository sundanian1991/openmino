# AI转型推进评估：P0/P1/P2技能开发优先级

> 评估日期：2026-06-13
> 评估基准：Wiki知识库 5279行（清洁版全覆盖）+ 用户本地Agent能力（Claude Desktop + MCP）+ CapyGet云端能力

---

## 一、VKK可开发性判断框架

| 能力层级 | 定义 | 可用工具 |
|---------|------|---------|
| **VKK-L1 纯规则型** | 基于Wiki中的制度/流程/判断逻辑，纯文本推理即可 | CapyGet Skill、Claude Desktop本地Skill |
| **VKK-L2 结构化数据型** | 需要读取Excel/JSON/CSV等结构化数据，但不需要实时系统对接 | CapyGet批量处理、本地Python脚本 |
| **VKK-L3 内网自动化型** | 需要操作京东内网系统（山海、京音、XBP、joyspace等） | Claude Desktop + MCP + 浏览器自动化（用户自主） |
| **VKK-L4 实时数据型** | 需要API接入实时数据（GMV、转化率、出勤率等） | 需IT支持开放接口，当前不可行 |

### 2.1 供应商综合评估专家（九宫格评级+清退决策）

| 维度 | 评估 |
|------|------|
| **VKK覆盖度** | **95%** — 04评估体系（五维考核+季度ABC+九宫格）+ 05方法论体系（状态机+触发规则+九宫格判定）+ 09执行工具集（季度评估计算表+九宫格映射表+稳定性检查表+供管自评表）已全部到位 |

## 六、关键瓶颈与建议

### 瓶颈1：赛马排名计算逻辑
- **问题**：Wiki中描述赛马是"100分制"，但具体如何计算（加权平均？标准化排名？百分位？）不明确
- **解决**：问王易人或策略组要一份赛马排名的计算示例

### 瓶颈2：实际数据样本
- **问题**：所有L2/L3/L4技能都需要实际数据验证
- **解决**：先用脱敏/模拟数据开发逻辑框架，再逐步接入真实数据

### 瓶颈3：内网系统对接
- **问题**：山海、京音、XBP等系统需要内网访问
- **解决**：用户已有Claude Desktop + MCP + 浏览器自动化能力，可自主处理。CapyGet做规则型Skill，Claude Desktop做内网自动化型Skill

## 七、最小可行路径（MVP）

如果只能选1个先开发，**推荐：分量调整决策助手**

**理由**：
1. 纯规则型，不需要任何外部数据
2. Wiki中规则100%完整（n-1原则+8个业务线的上限表+审批路径）
3. 输入简单："某供应商当前在X、Y、Z业务线，申请新增A业务线"
4. 输出明确："✅通过 / ❌触限：已达n-1上限 / ❌触限：金条人工将超360人上限"
5. 可直接封装为CapyGet Skill或本地Claude Skill
6. 实际价值高：每次分量调整都需人工核对，AI可秒级完成
