# 决策树-供应商清退

> 供应商清退的完整决策流程

## 决策流程

```
触发清退条件
    │
    ▼
Step 1: 确认触发类型
    │
    ├─ 连续3月排名后30% → 清退评估E状态
    │
    ├─ C档+趋势下降 → 季度快车道清退
    │
    ├─ 红线违规 → 立即清退
    │
    └─ 整改2次未通过 → 启动清退
        │
        ▼
Step 2: 评估业务影响
    │
    ├─ 当前业务量
    │
    ├─ 替代供应商情况
    │
    └─ 切换成本
        │
        ▼
Step 3: 制定退出方案
    │
    ├─ 切量计划
    │
    ├─ 时间表
    │
    └─ 人员安置
        │
        ▼
Step 4: 内部审批
    │
    ├─ 供应商管理岗建议
    │
    ├─ 王易人确认
    │
    └─ C2终审（如需）
        │
        ▼
Step 5: 执行退出
    │
    ├─ 书面通知
    │
    ├─ 数据回收
    │
    ├─ 费用清算
    │
    └─ 系统关闭
```
