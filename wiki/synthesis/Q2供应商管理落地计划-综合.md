# Q2供应商管理落地计划-综合

> 核心目标：增效 + 引入新商
> 副目标：过程管理规则建立

## Q2三条主线

```
增效（核心）          引入新商（核心）       过程管理（基础）
├─ 4月中旬：诊断     ├─ 4月中旬： pipeline  ├─ 4月中旬：规则起草
├─ 4月下旬：试点     ├─ 4月下旬：首批评估   ├─ 4月下旬：边界确认
├─ 5月：推广         ├─ 5月：试单落地       ├─ 5月：首次检查
└─ 6月：固化         └─ 6月：梯队更新       └─ 6月：机制固化
```

## 主线一：增效

### 增效的本质

增效不是"让供应商更努力"，而是**帮供应商找到效率漏损点，用过程管理和技术手段补上**。

增效的杠杆在你手里有三个：
1. **过程规则** — 你定标准、要复盘、做检查
2. **数据诊断** — 你比供应商更清楚他们哪里漏了效率
3. **最佳实践共享** — 把做得好的供应商的方法论提炼出来，推广到其他家

### 增效切入点矩阵

| 切入点 | 增效逻辑 | 你能做什么 | 预期效果 |
|--------|---------|-----------|---------|
| **转化率提升** | 同样的人力，更多成单 | 要求供应商做转化率根因分析，你提供数据对比基准 | 直接提升GMV |
| **培训周期缩短** | 新员工更快达产 | 要求供应商提交培训计划，你检查培训质量和达产周期 | 减少"人力虚挂" |
| **通时通次提升** | 有效通话时间占比 | 要求供应商做排班优化、无效时间压缩 | 同等人力更多产出 |
| **流失率降低** | 减少招聘培训成本 | 要求供应商做离职归因，你识别共性问题 | 降低隐性成本 |
| **管理精细化** | 主管从"喊口号"到"做辅导" | 你通过检查会议质量、复盘质量来倒逼 | 管理效率提升 |

### 增效的关键原则

- **不替供应商做方案** — 你给方向、给基准、给案例，方案他们自己出
- **数据说话** — 所有改善都有基线和目标，不是"努力提升"
- **先试点再推广** — 先选2-3家配合度高的做试点，跑通了再推到其他家
- **共享不强制** — 最佳实践共享是"你可以参考"，不是"你必须照做"

## 主线二：引入新商

### 引入新商的战略目的

不是为"多而多"，而是：
1. **打破头部依赖** — 任何一家独大都意味着风险
2. **制造竞争压力** — 新供应商入场，老供应商才有动力改善
3. **储备新能力** — 有些新业务老供应商可能接不了
4. **价格发现** — 新供应商帮你了解市场真实价格

## 主线三：过程管理

### 过程管理的本质

过程管理不是"管供应商"，而是**建立一套规则，让供应商知道什么是对的、什么是错的、错了会怎样**。

### 过程管理框架

| 维度 | 管理内容 | 管理方式 |
|------|---------|---------|
| **数据报送** | 日报、周报、月报 | 定时+异常即时 |
| **会议机制** | 周会、月度复盘、季度评审 | 固定+按需 |
| **检查巡检** | 日常巡检、专项检查 | 计划+突击 |
| **整改闭环** | 问题发现→整改→验证→归档 | 流程化 |
