# 决策树-业绩异常处理

> 业绩异常的完整处理流程

## 决策流程

```
看到业绩异常信号
    │
    ▼
Step 1: 确认信号真实性
    │
    ├─ 数据有问题 → 修正数据，重新分析
    │
    └─ 数据真实 → 继续
        │
        ▼
Step 2: 判断异常级别（[[触发规则表（状态机）]]）
    │
    ├─ 黄色（首次预警）→ 私下沟通
    │
    ├─ 橙色（连续预警）→ 正式约谈
    │
    ├─ 红色（清退评估）→ 启动清退
    │
    └─ A级骤降 → 立即约谈
        │
        ▼
Step 3: 定位原因（[[产能利用率双维度判定]]）
    │
    ├─ 标杆型 → 标杆分享，优先新业务
    │
    ├─ 尽力型 → 评估产线适配，调预期
    │
    ├─ 问题型 → 立即约谈，启动整改
    │
    └─ 暂时波动 → 观察2周，不下结论
        │
        ▼
Step 4: 检查团队能力（[[主管胜任评估5项指标]]）
    │
    ├─ 胜任 → 维持合作
    │
    ├─ 需关注 → 约谈时针对性提问
    │
    └─ 不胜任 → 谈管理团队调整
        │
        ▼
Step 5: 交叉验证（[[异常信号31个]]）
    │
    └─ 输出动作（[[场景SOP-决策判断]]）
```
