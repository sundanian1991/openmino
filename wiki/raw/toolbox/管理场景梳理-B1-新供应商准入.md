# B1 新供应商准入

> 从寻源需求到正式准入的完整决策链路——用9维度框架选人、用6月里程碑管人、用授权矩阵定权。

## 触发信号

- 产线扩容需求（新增XX席，现有供应商吃不下）
- 现有供应商连续2个月业绩下滑，需要备选
- 策略组（刘伟佳）提出新业务线寻源需求
- 供应商池月度更新时发现可引入的候选
- 上级（王易人）点名要求引入新供应商

## 核心问题

> 这家供应商能不能在6个月内达到中位水平？投入的管理资源值不值？如果不确定，宁可暂缓。

## 思考步骤

### 第1步：确认寻源需求是否成立

**想什么**：真的需要引入新供应商吗？还是可以通过现有供应商扩容解决？

**检查什么数据**：
- 当前各产线产能利用率（总产能 vs 实际需求）、现有供应商扩容意愿和能力
- 近3个月是否有供应商退出导致产能缺口、业务侧（刘伟佳/郭鑫）的需求确认

**判断什么**：
- 产能利用率>90% + 现有供应商无法扩容 → 寻源需求成立
- 产能利用率80-90% → 优先让现有供应商扩容，寻源作为备选
- 产能利用率<80% → 不需要引入，调整现有分配即可
- 紧急需求（3天内要产能）→ 不适合引入新供应商，走临时扩容

**举例**：金条线产能利用率95%，毅航"人招满了"，毛毛虫"最多加30人" → 吃不下增量，寻源成立。

### 第2步：从供应商池筛选候选

**想什么**：池子里有哪些候选？谁最匹配这个需求？

**检查什么数据**：
- 潜在供应商池清单（`supplier-reserve.md`）、各候选标签（业务类型/规模/地域/成熟度）
- 上次淘汰或暂缓的候选——当时的原因是否已解决

**判断什么**：
- 有"重点跟进"状态的候选 → 优先考虑，省去初步接触成本
- 有"待考察"状态的候选 → 需要加速考察流程
- 池内无合适候选 → 启动外部寻源（行业协会、竞品挖掘、合作伙伴介绍）
- 外部寻源周期通常2-4周，紧急需求不适用

**举例**：池内3家"重点跟进"做信用卡/企金，需求是金条线 → 不匹配，需新寻源。

### 第3步：9维度初步评估（桌面调查）

**想什么**：在去现场之前，先把能查的都查了，避免浪费出差时间。

**检查什么数据**：

> 准入9维度桌面调查清单

| 维度 | 桌面能查的 | 需现场验证的 |
|------|-----------|------------|
| **过往业绩** | 行业口碑、客户名单、自述数据 | 实际录音验证 |
| **同类型对标** | 与毅航/伽玛/赛维斯可比性 | 运营细节 |
| **管理方式** | 组织架构图、有无质检/培训/排班体系 | 管理实效 |
| **人员数量** | 供应商声称规模 | 实际出勤核实 |
| **公司背景** | 天眼查/企查查（注册资本、股东、司法风险） | 实际经营状态 |
| **历史交付风格** | 老客户评价、续约情况 | 现场感受 |
| **配合商务** | 合同条款接受度、响应速度 | 正式谈判 |
| **团队架构** | PM/质检/培训三岗是否齐全（名片/介绍） | 管理团队实际能力 |
| **资质合规** | 营业执照、增值电信许可证、ISO扫描件 | 合规执行 |

**判断什么**：
- 9项桌面调查有≥6项数据 → 可以安排现场考察
- 桌面调查发现重大风险（司法纠纷、资质造假）→ 直接淘汰，不用出差
- 公司背景（天眼查）有异常记录 → 要求供应商解释，解释不清则淘汰
- 桌面调查数据不足 → 要求供应商补充材料，不急着安排考察

**举例**：天眼查显示"经营异常"（未按时年报）→ 要求说明，解释不清直接淘汰。

### 第4步：现场考察 + 9维度验证

**想什么**：桌面调查看到的都是供应商想让你看到的，现场才是真相。

**检查什么数据**：
- 实际职场规模：工位数、在岗人数、空位率
- 管理看板：数据是否实时更新、是否贴了违规案例警示
- 培训教室：有没有专门场地、培训记录是否完整
- 质检团队：质检员人数占比、抽检比例
- 员工状态：现场精神面貌、是否有人玩手机/睡觉
- 合规宣传：是否有合规标语、处罚案例公示

**判断什么**：
- 供应商声称300人、实际在岗200人 → 诚信问题，扣分
- 管理看板数据与供应商自述不一致 → 数据造假嫌疑
- 无专门质检团队 → 引入后管理成本极高
- 员工状态散漫 → 管理能力弱，引入期风险大

**举例**：某供应商声称500人、实际在岗320人、空位率36% → 诚信存疑，"管理方式"和"人员数量"均扣分。

### 第5步：决策授权 + 启动准入

**想什么**：评估做完了，结论我能不能定？需要谁点头？

**检查什么数据**：

> 决策授权矩阵

| 动作 | 年老师权限 | 需王易人确认 | 需军哥确认 |
|------|-----------|-------------|-----------|
| 准入启动（常规） | 建议权 | 审批 | 事前沟通 |
| 准入启动（紧急） | 建议权 | 审批+陪同面谈 | 事前报备 |
| 暂缓/拒绝 | 自主决定 | 同步 | 无需 |
| 特殊商务条款 | 建议权 | 审批 | 视条款内容 |

**判断什么**：
- 9维度均分≥4 → 建议准入，报王易人审批
- 9维度均分3.3-3.9 + 无单项1分 → 有条件准入，列明改进条件
- 9维度均分<3.3 或有单项1分 → 拒绝，年老师可直接定
- 涉及保底量承诺或特殊定价 → 需军哥层面知晓

**举例**：候选均分3.8但"管理方式"2分 → 有条件准入，要求先建质检体系。报王易人审批。

### 第6步：制定6月里程碑跟踪计划

**想什么**：准入只是开始，后面6个月才是真正的考验。

**检查什么数据**：

> 新供应商6月里程碑

| 阶段 | 时间 | 目标 | 量化检查点 | 检查频率 |
|------|------|------|-----------|---------|
| 引入期 | 第1-2月 | 站稳脚跟 | 出勤率≥85%、无重大合规问题 | 每周 |
| 成长期 | 第3-4月 | 业绩爬坡 | GMV≥中位供应商60% | 每周 |
| 稳定期 | 第5-6月 | 达到中位 | 月度排名进入前70% | 双周 |
| 评判点 | 第6月末 | 留或退 | 排名稳定前70%→正式纳入 | 月末 |

**判断什么**：
- 内部有没有人手盯这家供应商的引入期 → 没有 = 错开排期
- 质检资源是否够用 → 同时引入不超过2家
- 初始分量给多少 → 从总量的5-10%起步，不贪多
- 第1个月出勤率<80% → 立即预警，第2个月未恢复则启动评估

**举例**：已有1家新供应商在引入期，质检被占用 → 新供应商排下月启动，错开引入期。

## 关键变量

| 变量 | 判断标准 | 影响权重 |
|------|---------|---------|
| **9维度总分** | ≥36准入、30-35有条件、<30拒绝 | 核心依据 |
| **产能缺口紧迫度** | 3天内=紧急（不适合新引入）、1个月=正常 | 决定寻源节奏 |
| **管理资源可用性** | 能盯引入期=可行、无人手=排期 | 决定准入时机 |
| **现有供应商扩容能力** | 能吃下增量=不需要引入 | 决定是否启动寻源 |
| **决策授权层级** | 常规=王易人，特殊=军哥 | 决定审批路径 |

## 常见误区

1. **被供应商PPT打动**：只看可交叉验证的数据，不看承诺
2. **跳过桌面调查直接出差**：先查天眼查再买票，避免到现场才发现资质造假
3. **同时引入超过2家**：前2个月管理投入是老供应商的3-5倍，资源有限
4. **9维度走形式**：每项都打3-4分无区分度——必须有硬数据支撑
5. **6月里程碑定了不查**：每周检查引入期指标，第1周出问题第2周就要介入
6. **忽视对现有供应商的冲击**：提前沟通，用增量而非存量分配
