# Plan 模板体系 — 思考工具箱

> 遇到问题后，帮年老师想清楚、判断好、说出来。

---

## 定位

工具箱解决"日常观察的语言" — 我在看什么、怎么看的。
Plan 解决"遇到问题后的思考语言" — 我怎么想、怎么判断、怎么决策、怎么说出来。

**不是通用检查清单，是议题设定工具。**

---

## 体系架构

| 层级 | 代号 | 数量 | 用时 | 用途 |
|------|------|------|------|------|
| 快速判断卡 | Q1-Q7 | 7个 | 30秒 | 被触发后立即反应 |
| 决策思考框架 | D1-D8 | 8个 | 15分钟 | 灰色地带的深度判断 |
| 周期性叙事框架 | N1-N3 | 3个 | 月/季度 | 主动构建自己的议题 |

---

## 第一层：快速判断卡（30秒级）

遇到信号/被人找，30秒内判断：谁在找？要什么？紧急吗？口径是什么？

| 代号 | 场景 | 核心判断 |
|------|------|---------|
| Q1 | 被同事@数据 | 他到底要什么？我能给什么？ |
| Q2 | 被领导点名 | 他为什么问这个？我有什么信息？ |
| Q3 | 供应商来电/来消息 | 他的真实意图是什么？ |
| Q4 | 供应商群异常信号 | 是真问题还是撒娇？ |
| Q5 | 周度突变信号 | 三种信号分别对应什么动作？ |
| Q6 | 月度排名出炉 | 哪几家需要关注？关注什么？ |
| Q7 | 客诉/合规事件 | 多严重？我需要多快反应？ |

---

## 第二层：决策思考框架（15分钟级）

工具箱给了规则但没给完整决策路径的灰色地带。坐下来想15分钟，走一遍归因路径。

| 代号 | 场景 | 为什么是灰色地带 |
|------|------|-----------------|
| D1 | A级骤降归因 | 原因可能有5种以上，处理完全不同 |
| D2 | 整改定级 | 找到"重视但不翻脸"的平衡点 |
| D3 | 分量调整博弈 | 看起来是通知，实际是博弈 |
| D4 | 清退最终决策 | 不是数据问题，是判断问题 |
| D5 | 价格谈判准备 | 底线在哪？他说的是真的吗？ |
| D6 | 向上汇报叙事 | 把观察变成能打动军哥的叙事 |
| D7 | 新供应商评估 | 9维度怎么评？怎么对标？ |
| D8 | 转正/延期决策 | 数据在"差不多"区间怎么判？ |

每个框架包含：问题拆解 → 归因路径（决策树）→ 权衡因素 → 叙事草稿（向上/对外/内部）→ 衔接 official-doc 模板

---

## 第三层：周期性叙事框架（月度/季度级）

主动构建自己的议题，不等策略组先说。

| 代号 | 场景 | 核心目的 |
|------|------|---------|
| N1 | 月度复盘叙事 | 每月自己的观察判断，比策略组早2-4周 |
| N2 | 季度评估叙事 | 给军哥的季度判断，有自己的立场 |
| N3 | 出差巡检叙事 | 现场观察的叙事，策略组看不到的信息 |

---

## 使用方法

**日常**：看到信号 → 翻对应的 Q 卡 → 30秒判断 → 需要深入 → 翻对应的 D 框架

**月度**：排名出炉 → Q6 快速扫描 → 对异常供应商用 D1-D5 深入 → 月末用 N1 做复盘叙事

**季度**：用 N2 做季度叙事 → 对标策略组 → 给军哥有立场的汇报

**巡检**：巡检前用 D7/D8 评估新供应商 → 巡检后用 N3 整理现场观察

---

## 衔接 official-doc

Plan 的输出是"决策备忘录 + 叙事草稿"，直接填进 official-doc 模板：

> Plan（想清楚）→ 决策备忘录 → official-doc（写出来）→ 交付

每个 D 框架和 N 框架底部都标注了对应的 official-doc 模板文件名。
