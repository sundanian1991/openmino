# E1-新供应商初筛

> 有人推荐或自己寻源到一家新供应商时，快速判断是否值得深入评估

## 触发信号
- 同事推荐："有个新供应商翰锐，你要不要看看？"
- 自己寻源：参加行业会议/看到招聘信息/供应商主动联系
- 业务需求：新项目需要找供应商承接

## 核心问题
- 这家供应商的基础资质是否达标？
- 和我们现有业务匹配度如何？
- 是否值得启动9维度深度评估？
- 如果是尾部供应商，有什么特殊情况？

## 思考步骤

**Step 1：基础门槛筛查（5分钟）**
```
硬性门槛（任一不满足直接淘汰）：
□ 成立时间 > 2年（新公司风险高）
□ 团队规模 > 50人（有基本规模）
□ 无诚信记录（裁判文书网/行政处罚可查）
□ 资质合规（营业执照/人力资源许可证等齐全）
```
- 不满足任意一项 → 记录淘汰原因，不再跟进

**Step 2：快速对标（10分钟）**
- 同类供应商对比：该供应商的团队规模、成立时间、专注领域，与现有供应商（如毅航、毛毛虫、赛维斯、岐力）相比如何？
- 团队规模差距：与对标供应商差距是否 <30%？（工具箱§十：标杆配对逻辑）
- 专注领域匹配：该供应商擅长领域是否匹配我们的业务需求？（金条/企金/信用卡/财富）

**Step 3：判断供应商类别（工具箱§十九）**
```
头部供应商策略（排名前40%）：
✓ 看排名：目前业务规模是否已进入头部？
✓ 看专项：是否有我们急需的专项能力？
✓ 看意愿：配合度如何？是否愿意接我们项目？
→ 这类供应商是我们主动争取的对象

中部供应商策略（排名40%-70%）：
✓ 看趋势：近3个月排名是上升还是下降？
✓ 看改进：是否有明确的改进动作？
✓ 看成本：成本是否在合理区间？
→ 这类供应商是重点培养对象

尾部供应商策略（排名后30%）：
✓ 必须写明特殊情况：为什么选尾部？（价格极低/特殊技能/战略协同）
✓ 必须有风险预案：如果做不好怎么办？
✓ 必须经上级审批：需要军哥/易人哥特批
→ 这类供应商需要更强的理由和更高的审批
```

**Step 4：资源可用性评估**
- 该供应商当前业务饱和度：是否已有充足业务？是否有空余名额接新项目？
- 和我们合作意愿：是否主动接触？还是需要我们反复邀约？
- 历史交付风格：是否有同类项目经验？交付质量如何？（需查历史记录或同行口碑）

**Step 5：初筛决策**
| 判断 | 行动 | 输出 |
|------|------|------|
| 基础门槛不满足 | 淘汰，记录原因 | — |
| 尾部供应商且无特殊理由 | 不建议深入评估 | 写清淘汰理由 |
| 尾部供应商但有特殊理由 | 进入深度评估，但标注"尾部特殊" | `supplier-selection-framework.md` |
| 中部/头部供应商 | 进入9维度深度评估 | 启动准入流程 |

## 关键变量
- **供应商类别**：头部/中部/尾部，决定评估深度和审批级别
- **团队规模差距**：<30%可对标，>30%不可简单对比
- **专注领域匹配度**：业务线是否一致？（金条/企金/信用卡/财富）
- **资源可用性**：是否有空余名额承接新项目？
- **合作意愿**：主动接触还是被动响应？

## 常见误区
- ❌ 只看公司规模不看趋势：中部供应商重点看近3个月排名趋势
- ❌ 忽略尾部供应商的特殊审批：尾部供应商必须写明特殊原因+风险预案+上级审批
- ❌ 不查基础门槛：资质不全/有诚信记录的直接淘汰，不浪费时间
- ❌ 混淆业务线：信用卡的供应商不能直接用在金条项目，需评估适配性
- ❌ 过度关注价格：价格不是唯一因素，需综合9维度评估

## 决策输出

**初筛结果模板**：
```
供应商名称：[XXX]
初筛结论：[推荐深入评估 / 淘汰]
判断依据：
1. 基础门槛：[满足/不满足，具体说明]
2. 供应商类别：[头部/中部/尾部]
3. 团队规模：现有XX人，与[对标供应商]差距[XX%]
4. 业务匹配：[匹配/部分匹配/不匹配]
5. 资源可用性：[有空余名额/已满/需确认]

下一步：
□ 进入9维度深度评估（提效方案周期工作1）
□ 启动准入SOP流程（8阶段T+15）
□ 淘汰，原因：[具体原因]
```
**尾部供应商特殊标注**：
```
特殊情况：[写明为何选尾部，如：价格极低/特殊技能/战略协同]
风险预案：[如果交付不力的备选方案]
审批要求：需[军哥/易人哥]审批
```

## 下一步衔接
- 若通过初筛 → 调用【B1-新供应商准入】→ 启动8阶段SOP流程
- 若需9维度评估 → 调用【B1-新供应商准入】的评估环节 → `supplier-onboard.md`
- 若淘汰 → 记录到寻源备选库，定期复盘
