# N3 出差巡检叙事

> 现场观察的整理和叙事输出，把"看了什么"变成"发现了什么"。策略组看不了现场，这是只有年老师能提供的信息。

---

## 叙事主题

> 巡检叙事不是巡检报告——报告有模板，走流程，归档用的。叙事是**"我从现场带回来的判断"**。
>
> 巡检报告回答"现场合不合规"，巡检叙事回答"这家供应商现在到底怎么样、未来会怎样"。
>
> 好的巡检叙事像一个战地记者的观察笔记——不是 checklist 打勾，是"我在现场看到了什么、想到了什么、建议做什么"。例：
> - "巡检了3家报告都合格。但我的判断：只有1家真的在自运转，另外2家是在'表演合规'"
> - "岐力FCI评分68不算差。但现场比数字严重：主管开晨会时坐席在玩手机，培训记录工整但坐席都说不记得"

---

## 三层叙事

### 第一层：数据层 — 看到了什么

> 巡检叙事的数据层分两块：**工具箱的标准数据** + **现场观察记录**。

#### 硬数据（每次巡检必须记录）

| 数据项 | 来源 | 记录方式 |
|--------|------|----------|
| FCI速记卡 | 巡检后2分钟填写 | 工具箱12.1节格式 |
| 红黄绿巡检结果 | 巡检表单 | X红 Y黄 Z绿，每项编号+描述 |
| 现场人数 vs 填报人数 | 实际点人头 vs 人力表 | 差异超过10%标记异常 |
| 外网/手机/监控覆盖率 | 现场实测 | 管控率百分比 |
| 培训记录抽查 | 随机问2-3个坐席 | 坐席描述 vs 记录内容是否吻合 |
| 主管在场情况 | 观察 | 晨会/日常管理是否在岗 |

#### 现场观察记录（核心差异化信息）

> 这是策略组永远看不到的、只有你到现场才能捕捉的信息。

| 观察维度 | 看什么 | 怎么记录 |
|---------|--------|---------|
| **工作氛围** | 坐席状态——认真还是混时间？整体紧张还是松弛？ | 1-2句话印象 |
| **主管管理风格** | 晨会怎么开？严格还是放羊？ | "晨会15分钟只念数据不拆解，坐席听完各自回位" |
| **供应商老板状态** | 热情接待还是应付？对业务关注度？ | "老板主动拿出了完整的入职手册——不是临时准备的" |
| **硬件/环境** | 设备老化？工位利用率？环境维护？ | "50个工位实际用42个，人力表报47人——5人去哪了？" |
| **坐席随机对话** | 随机聊工作感受、培训内容、目标完成 | "3个坐席只有1个能说出本月目标——目标没拆解到人" |
| **培训效果验证** | 问坐席"上次培训学了什么" | "记录写'话术优化培训'，坐席说'就是看了个视频'" |
| **数据一致性** | 现场 vs 填报 vs 系统导出 | "系统日均80通，现场半小时人均2通——日均不到40通" |

**记录原则**：不记流水账，记**反差和异常**。只记三种：**和预期不一致的** / **和填报数据矛盾的** / **让你意外惊讶的**。每个观察一句话：[现象] + [判断]。

### 第二层：判断层 — 想到了什么

> 区分"看到的"和"想到的"——看到的是现象，想到的是判断。巡检叙事的核心价值在判断层。

#### 四类现场判断

| 判断类型 | 识别方式 | 判断模板 |
|---------|---------|---------|
| **合规真伪** | 现场和纸面记录不一致 | "报告显示[合规状态]，但现场看到[实际情况]。这不是[表面原因]，是[深层问题]。" |
| **管理质量** | 主管行为和坐席状态 | "FCI评分[分数]，不算差。但现场看到[问题]，说明[判断]。核心问题是[一句话]。" |
| **经营健康** | 老板态度、硬件状态、人员稳定性 | "表面看起来[状况]，但我判断[真实情况]。信号是[具体信号]。" |
| **数据真实性** | 现场数据和填报/系统数据交叉验证 | "填报数据和现场数据在[维度]上有[X]%的差异。可能是[原因A]，也可能是[原因B]。我倾向[判断]。" |

#### 现场对比判断

> 巡检最有价值的判断来自**对比**——不一致的地方往往是最有价值的。

| 对比维度 | 一致说明什么 | 不一致说明什么 |
|---------|------------|--------------|
| FC评分 vs 现场感受 | 数据可靠，体系有效 | FCI指标有问题或现场在表演 → 信现场 |
| 排名 vs 管理质量 | 排名反映真实能力 | 虚高（靠加人）或虚低（有潜力未释放） → 虚高预警虚低给机会 |
| 填报 vs 现场核实 | 供应商诚实可靠 | 数据做了手脚 → 启动核查 |
| 整改承诺 vs 现场进展 | 整改在真实推进 | 停在纸面上 → 升级约谈 |
| 上次 vs 这次巡检 | 持续改善 | 问题反复/新问题出现 → 找根因 |

#### 信心度标注

| 信心度 | 适用场景 | 标注方式 |
|--------|---------|---------|
| **高** | 数据+现场双重验证 / 多个信号指向同一结论 | "我判断[结论]。信心度：高。" |
| **中** | 现场观察强烈但缺乏数据支撑 / 单次观察可能不具代表性 | "我初步判断[结论]，需进一步验证[什么]。信心度：中。" |
| **低** | 直觉强烈但证据不足 / 可能受当天特殊因素影响 | "现场有[信号]，但不确定是否[偶发/常态]。需要[验证方式]。信心度：低。" |

### 第三层：建议层 — 该怎么做

#### 流程层面（走巡检报告）

| 巡检结果 | 触发动作 |
|---------|---------|
| 合格（红=0，黄≤2） | 无需动作 |
| 预警（红=1或黄≥3） | 整改通知 + 下次巡检重点复查 |
| 危险（红≥2或G项有红） | 立即约谈 + 系统性整改要求 |

#### 管理层面（走巡检叙事）

> 这是巡检叙事独有的价值——不是整改通知，是"基于现场观察的管理建议"。

| 建议类型 | 触发条件 | 格式 |
|---------|---------|------|
| **主管调整** | 主管管理能力不足（非FCI分数低） | 建议考虑[调整]，因为[现场问题]。不调整预计[后果] |
| **资源调整** | 产能被浪费或有弹性 | 现场[现象]+数据[数据]，建议[调整]，预计[效果] |
| **风险预警** | 数据看不到的风险信号 | 信号：[信号]。建议[验证方式] |
| **数据核实** | 现场和填报不一致 | [维度]差[X]%，建议[核实方式] |
| **标杆复制** | 标杆做法值得推广 | 确认[标杆][做法]有效，建议复制到[对标方] |

## 对标策略组
> 巡检叙事是年老师最独特的价值——策略组看不了现场。

### 策略组的盲区 vs 你的填补

| 维度 | 策略组的盲区 | 你的巡检叙事填补什么 |
|------|------------|-------------------|
| 合规执行 | 只有质检分数 | 合规分数 ≠ 合规执行（岐力自查全勾，坐席不知信息安全规范） |
| 管理质量 | 没有管理能力评估维度 | 排名 ≠ 管理水平（赛维斯排名还行但管理只是传声筒） |
| 供应商态度 | 只能从合作意愿推断 | 老板态度变化是比排名更早的信号（毛毛虫可能减少投入） |
| 数据真实性 | 只能看系统数据 | 现场拨打量和系统数据差一倍 |
| 前瞻风险 | 只能做数据趋势预判 | 竞品挖人、主管更替、老板态度变化——数据捕捉不到的前瞻信号 |

### 怎么说让军哥记住

每次叙事开头用一句话说清核心价值：

> - "核心发现：3家里有2家的合规是'表演式'的——报告合格，实际执行打折。"
> - "最重要观察：毅航晨会管理真实有效，建议标准化复制。"
> - "带回来的判断：毛毛虫可能在考虑退出，信号比数据早。"

### 给王易人的巡检摘要

> 王易人关心"需要我出面的事"和"进展卡在哪"。摘要要短、要具体。

| 要素 | 做法 | 禁忌 |
|------|------|------|
| 进展 | 哪些顺利、哪些卡住 | 不要流水账 |
| 卡点 | 卡在哪里、需要他做什么 | 不要"需要支持" |
| 时间线 | 具体到某天 | 不说"这周"、"尽快" |
| 后续 | 下一步 + 下次同步时间 | 不说"到时候再说" |

## 巡检叙事完整流程与输出物

| 阶段 | 动作 | 耗时 | 产出 |
|------|------|------|------|
| **巡检前** | 明确目的 + 调取近2月数据 + 带上次巡检记录 | 15min | 准备清单 |
| **巡检中** | FCI打分 + 红黄绿巡检 + 现场观察 + 坐席对话 + 主管观察 | 2-4h/家 | FCI速记卡 + 现场笔记 |
| **Day 1** | 硬数据 + 巡检报告（走流程）+ 整改通知（预警/危险时） | 30min | 巡检报告 + 整改通知 |
| **Day 1-2** | 判断层（四类判断）+ 信心度标注 + 王易人摘要 | 30min | 判断层 + 摘要 |
| **Day 2-3** | 写建议层 + 管理调整建议 | 20min | 建议层 |
| **Day 3-5** | 合成巡检叙事 + 发出 | 20min | **巡检叙事输出** |
| **合计** | | **约3-4小时**（不含现场时间） | |

**输出物受众**：巡检报告/整改通知 → 归档+供应商；摘要 → 王易人；巡检叙事/管理建议 → 军哥。

## 多家巡检的特殊处理
> 出差一趟通常不止看一家。多家巡检叙事要做**横向对比**，不只是逐家汇报。

| 部分 | 内容 | 目的 |
|------|------|------|
| **一句话总结** | 核心发现 | 30秒让军哥抓住重点 |
| **横向对比** | 同一维度下各家的差异 | 比逐家汇报更有洞察力 |
| **逐家判断** | 每家一个核心判断（不超过3句话） | 具体信息 |
| **管理建议** | 整体建议 + 逐家建议 | 可执行 |

> 横向对比示例："同一天看3家晨会：毅航主管拆解目标到人10分钟；赛维斯主管念数据15分钟；岐力没开晨会说'太忙了'。同样管理动作执行质量差三个层次。毅航转化率高3pp——不是坐席能力强，是管理质量在拉开差距。"

## 质量自检

| # | 检查项 | 通过标准 |
|---|--------|---------|
| 1 | 区分"看到的"和"想到的" | 数据层是观察，判断层是分析，没有混在一起 |
| 2 | 现场观察有"数据看不到的东西" | 有氛围/主管风格/老板态度/坐席对话等至少2项 |
| 3 | 判断层有信心度标注 | 每条判断标注了高/中/低 |
| 4 | 做了现场对比判断 | FC评分vs实际感受 / 填报vs现场 / 上次vs这次，至少1项 |
| 5 | 建议层超过巡检报告范围 | 有管理层面建议（主管调整/资源调整/风险预警等） |
| 6 | 给王易人的摘要有具体时间线和卡点 | 有。不是"需要支持"这种模糊表述 |
| 7 | 多家巡检有横向对比 | 不是逐家汇报，有跨供应商的对比洞察 |

## 和 N1/N2 的关系

| 维度 | N1 月度复盘 | N2 季度评估 | N3 出差巡检 |
|------|-----------|-----------|-----------|
| 触发方式 | 定时（每月） | 定时（每季度） | 事件触发 |
| 数据来源 | 系统数据+排名+填报 | 月度数据×3+趋势 | **现场一手信息** |
| 独特价值 | 一线管理者的月度判断 | 趋势和结构性分析 | **只有到现场才能看到的** |
| 对标策略组 | 补充数据盲区 | 建立叙事框架 | 他们完全没有的信息 |
| 核心受众 | 军哥/王易人 | 军哥 | 军哥/王易人/供应商 |

**衔接规则**：巡检中的管理判断 → 当月N1判断层；风险信号 → N2趋势判断；数据不一致 → 触发数据真实性核查（工具箱19节）；标杆做法 → 标杆配对（工具箱第九节）。
