# Premium Calibration

Use this after the director brief and before the compiled spec.
The goal is not to replace the film language. The goal is to stop the design from collapsing into generic "creative template" output.

## What Premium Means Here

Premium means the page feels directed, edited, and intentional.
It does not mean adding more gradients, more glass, or more effects.

## Required Outputs

First write the site-wide cinematic grammar. Then write these page decisions into the storyboard.

- `Site cinematic grammar`
  - The shared film language of the whole site: shell logic, navigation posture, framing discipline, density cadence, recurring materials, and what is allowed to repeat.

- `One big idea`
  - The single dominant visual concept for the page.
- `Page scene thesis`
  - The page's own scene inside the film of the site.
- `Hero dominance statement`
  - Why the hero feels commanding or expensive in one sentence.
- `Restraint statement`
  - What you will deliberately not do.
- `Material thesis`
  - What makes surfaces feel tactile or atmospheric.
- `Typography thesis`
  - What gives the type authority or elegance.
- `Page-role scene`
  - What kind of scene this page is inside the film of the site.
- `Signature composition`
  - The one layout move this page owns that should not collapse into a generic default grid.
- `Grid fallback test`
  - State what would be lost if this page were reduced to a generic card grid. If the answer is "not much", the composition is not strong enough yet.
- `Shared system holdback`
  - Which repeated UI rules must wait until all page compositions are locked.
- `Uniqueness guardrail`
  - State what this project must not inherit from the same user's previous sites or demos at the wireframe level.

## Premium Signals

Prefer signals like these:

- confident scale
- disciplined whitespace
- sharp hierarchy
- controlled contrast
- purposeful framing
- material depth
- restrained motion
- edited color usage

## Premium Failure Modes

Reject the design if it feels like any of these:

- a Framer template with a movie color palette
- a motion showcase with no hierarchy
- a SaaS page wearing cinematic makeup
- a pile of beautiful parts with no dominant idea
- a reference collage
- a page that leaks internal process language like chapter labels, film names, director names, or calibration tags into the public UI

## Review Questions

Ask these questions before moving to implementation:

1. What is the one thing the viewer remembers after 3 seconds?
2. What is intentionally absent?
3. Which detail makes the page feel expensive rather than just decorative?
4. If you removed 30 percent of the effects, would the page become stronger?
5. Does this page feel like its own scene in the same film, or only like a weaker copy of the homepage?
6. Would the page still feel directed if palette and typography were stripped back to neutral values?
7. If this page were turned into a generic two-by-two or three-column grid, what essential idea would break?
8. Does the page read like a standalone poster before the shared design system is added back?
9. If styling were stripped away, would this still look too much like the user's previous demo?
