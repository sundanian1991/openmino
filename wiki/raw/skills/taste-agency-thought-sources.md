# Thought Sources

This reference grounds the `taste-agency` skill. Use it when the user wants the deeper intellectual basis, wants citations, or is writing an article about the skill.

## Paul Graham: Taste As Maker Judgment

Source: `Taste for Makers` by Paul Graham
URL: https://www.paulgraham.com/taste.html

Useful idea: Taste is not only personal preference. For makers, it is the ability to recognize good work, notice where current work falls short, and keep refining until the artifact meets a higher standard.

How to use it in the skill:

- Ask what "good" visibly looks like.
- Ask what benchmark the user is comparing against.
- Ask what should be removed, simplified, or redone.
- Treat taste as a production constraint, not a personality trait.

## Ira Glass: The Taste Gap

Source: Ira Glass interview discussion summarized by Poetry Foundation
URL: https://www.poetryfoundation.org/poetry-news/65504/ira-glass-on-his-practice-catching-up-with-his-taste

Useful idea: Early creative frustration often comes from having good enough taste to see that one's work is not yet good. The gap is painful, but it is also a signal that the user has a quality bar.

How to use it in the skill:

- Diagnose whether the user lacks taste or lacks reps.
- Do not let "this is not good enough" become a reason to stop.
- Turn quality dissatisfaction into a practice loop.

## Albert Bandura: Human Agency

Source: `Toward a Psychology of Human Agency` by Albert Bandura
URL: https://journals.sagepub.com/doi/10.1111/j.1745-6916.2006.00011.x

Useful idea: Agency means people can influence their own functioning and life circumstances through intentionality, forethought, self-reactiveness, and self-reflectiveness.

How to use it in the skill:

- Ask what the user intends, not only what they feel.
- Ask what future outcome the next action is meant to influence.
- Ask how the user will regulate action when motivation drops.
- Ask how the user will reflect and adjust after feedback.

## Richard Hamming: Important Problems And Attack Paths

Source: `You and Your Research` by Richard Hamming
URL: https://www.cs.virginia.edu/~robins/YouAndYourResearch.html

Useful idea: Important work comes from spending time on important problems, but importance alone is not enough. The user also needs a plausible way to attack the problem.

How to use it in the skill:

- Ask whether this problem is important enough to deserve attention.
- Ask whether the user has a reasonable first attack.
- Avoid grand projects with no entry point.

## John Boyd: OODA Loop

Source: Robert E. Enck, `The OODA Loop`
URL: https://journals.sagepub.com/doi/10.1177/1084822312439314

Useful idea: Agency improves through action loops, not one-time decisions. Orientation matters: the same facts lead to different actions depending on the user's model of reality.

How to use it in the skill:

- Turn decisions into feedback loops.
- Ask what the next action will teach.
- Re-orient after feedback instead of defending the original plan.

## Brian Eno: Scenius

Source: David Perell, `Find Your Scenius`
URL: https://perell.com/note/find-your-scenius/

Useful idea: Taste is trained by environment. The user's judgment improves when they spend time near high-quality work, high-signal peers, and honest feedback.

How to use it in the skill:

- Ask what the user's input environment is training them to value.
- Ask who can give high-quality feedback.
- Ask whether the benchmark set is strong enough.

## Working Synthesis

Use this compact synthesis when writing or explaining the skill:

> Context decides whether this matters for the user's current main line. Taste chooses the right problem and defines the quality bar. Reality prevents self-deception. Agency creates the next artifact. Loop turns action into better judgment.

The skill is successful when the user stops floating in attractive ideas and produces one concrete next move with a clear quality standard.

## v0.2 Principle: Context Before Taste

Taste is not abstract. A decision can be excellent in general and still wrong for the user's current stage.

Use a Context Snapshot before judging:

- Current main line
- Current stage
- Existing assets
- Current shortfall
- Real constraints
- Recurring trap

This prevents two common errors:

- Calling an opportunity "good" because it is impressive, even when it drifts away from the user's main line.
- Calling an opportunity "bad" because the user feels unprepared, even when it directly addresses a current shortfall.
