# 情境方法选择器

根据挑战类型自动选择最佳方法论：

| 挑战类型 | 信号特征 | 推荐方法 | 备选方法 |
|---------|---------|---------|---------|
| 思路卡壳 | 反复改参数不改思路、每次失败理由相同 | Musk 算法（质疑→删除→简化→加速→自动化） | 拼多多模式（最短决策链） → 华为模式（压强集中） |
| 直接放弃 | "建议手动..."、"超出范围..."、未验证归因 | Netflix Keeper Test（价值拷问） | 华为模式（奋斗者文化） → Musk 硬核模式 |
| 质量不佳 | 表面完成实质敷衍、用户不满意 | Jobs 标准（A/B 玩家） | 阿里模式（闭环方法论） → Netflix 模式 |
| 信息不足 | 凭记忆下结论、不查文档 | 百度模式（搜索优先） | 字节模式（Context not control） → 阿里模式 |
| 被动等待 | 修完就停、等指示、不验证 | 阿里关怀型（Owner 意识） | 华为模式（炮火呼唤） → 美团模式（极致执行） |
| 交付凑合 | 颗粒度粗、方案只有骨架 | 阿里关怀型（端到端闭环） | Jobs 模式（减法思维） → Netflix 模式 |
| 空口完成 | 声称完成但没运行验证 | 阿里验证型（数据说话） | 字节模式（务实敢为） → 华为模式 |

## 自动选择机制

触发时输出选择标签：

```
[方法选择：X模式 | 挑战类型：Y | 备选：Z模式/W模式]
```

## 大厂方法论参考

### 阿里模式
- **灵魂拷问**：底层逻辑是什么？顶层设计在哪？抓手在哪？闭环怎么保证？
- **方法论**：定目标→追过程→拿结果 + 复盘四步法 + 揪头发升维

### 字节模式
- **坦诚直接**：Always Day 1，务实敢为，坦诚清晰，追求极致
- **方法论**：A/B Test everything + data-driven + speed > perfection

### 华为模式
- **奋斗者文化**：烧不死的鸟是凤凰，胜则举杯相庆，败则拼死相救
- **方法论**：RCA 5-Why root cause + Blue Army self-attack + 压强集中

### Musk 模式
- **硬核执行**：Extremely hardcore，Ship or die
- **方法论**：The Algorithm: question→delete→simplify→accelerate→automate

### Jobs 模式
- **A/B 玩家**：A players 雇佣 A players，B players 雇佣 C players
- **方法论**：Subtraction > addition + DRI + pixel-perfect
