# 供应商选择决策框架

> 用途：规范供应商选择流程，确保决策可追溯、可解释
> 更新日期：2026-03-21

---
## 一、核心原则
### 1.1 决策可追溯原则
- 每次供应商选择必须有书面决策记录
- 记录必须包含：选择背景、评估维度、决策依据、审批意见
### 1.2 评估维度多元化原则
- **头部供应商**（排名前30%）：综合排名 + 专项能力 + 合作意愿
- **中部供应商**（排名30%-70%）：业绩趋势 + 改进意愿 + 成本优势 + 资源可用性
- **尾部供应商**（排名后30%）：特殊情况说明 + 风险预案 + 成本效益
### 1.3 集中度管理原则
- 同一业务线避免过度集中（单一供应商≤40%分量）
- 优先选择跨业务线供应商（提升协同效应）
- 区域分布合理性（风险分散）

## 三、不同场景的选择策略
### 3.1 头部供应商选择（排名前30%，约前10名）
**选择依据优先级：**
1. 综合排名（权重40%）：Q[季度]评估排名
2. 专项能力（权重30%）：与任务相关的历史表现
3. 合作意愿（权重20%）：响应速度、主动配合度
4. 资源可用性（权重10%）：是否有充足人力
**审计解释要点：**
- "XXX供应商Q[季度]排名第X，综合表现优秀"
- "该供应商在XXX业务上有历史成功案例"
- "供应商主动承接，配合度高"
### 3.2 中部供应商选择（排名30%-70%，约第11-22名）
**选择依据优先级：**
1. **业绩趋势**（权重30%）：近3个月趋势（↗优先）
2. **改进意愿**（权重25%）：是否积极响应整改、主动优化
3. **成本优势**（权重20%）：单价是否具有竞争力
4. **资源可用性**（权重15%）：是否立即可用
5. **合作历史**（权重10%）：过往任务完成情况
**审计解释要点：**
- "XXX供应商近3个月业绩持续上升，排名第X→X→X"
- "该供应商积极配合XXX整改，改进意愿强"
- "单价较市场平均低X%，成本优势明显"
- "当前有XX席空闲产能，可立即承接"
### 3.3 尾部供应商选择（排名后30%，约第23-32名）
**选择依据优先级：**
1. **特殊情况说明**（权重40%）：必须说明为何选择尾部供应商
2. **成本效益**（权重30%）：是否为唯一可行方案
3. **风险预案**（权重30%）：是否有备选方案
**特殊情况包括：**
- 头部/中部供应商均无资源
- 任务需要特殊资质（仅尾部供应商具备）
- 成本压力下的预算约束
- 区域覆盖要求（特定地区仅有尾部供应商）
**审计解释要点：**
- "因XXX原因，头部/中部供应商均无可用资源"
- "XXX供应商具备XXX特殊资质，是唯一可行方案"
- "单价仅为XX元/通，符合预算要求"
- "已制定风险预案：XXX供应商为备选，可随时切换"

## 四、资源分配决策框架
### 4.1 资源分配评估维度
#### 4.1.1 业绩维度（40分）
| 指标 | 评分标准 |
|------|---------|
| 综合排名 | 前10名：10分；11-22名：7-3分；23-32名：3-0分 |
| 近期趋势 | 持续上升：10分；稳定：7分；波动：4分；下降：0分 |
| 目标达成 | 历史达成率≥90%：10分；80-90%：7分；70-80%：4分；<70%：0分 |
#### 4.1.2 合作维度（30分）
| 指标 | 评分标准 |
|------|---------|
| 响应速度 | 2小时内响应：10分；4小时内：7分；8小时内：4分；>8小时：0分 |
| 配合意愿 | 主动承接：10分；需要动员：5分；勉强接受：0分 |
| 问题处理 | 积极解决：10分；配合整改：7分；拖延：0分 |
#### 4.1.3 资源维度（20分）
| 指标 | 评分标准 |
|------|---------|
| 产能可用 | 立即可用：10分；3天内：7分；7天内：4分；>7天：0分 |
| 资源质量 | 有经验团队：10分；需培训：5分；新手：0分 |
| 职场稳定性 | 稳定运营：10分；近期有变动：5分；不稳定：0分 |
#### 4.1.4 策略维度（10分）
| 指标 | 评分标准 |
|------|---------|
| 集中度管理 | 符合分散要求：10分；需要调整：5分；不符合：0分 |
| 业务协同 | 有协同效应：5分；无影响：0分 |
| 区域覆盖 | 符合区域要求：5分；无影响：0分 |

## 五、常用审计问答话术
### 5.1 为什么选择这个供应商？
**标准回答：**
"基于Q[季度]评估结果及近期表现，综合考虑以下因素：
1. 综合排名：第X/32名，处于[头部/中部/尾部]
2. 业绩趋势：近3个月[持续上升/保持稳定]
3. 合作意愿：[积极响应/主动承接]，配合度高
4. 资源可用：[立即可用/X天内可准备]
5. 集中度管理：选择后该业务线分量分布为XXX，符合分散要求"
### 5.2 为什么不选排名更高的供应商？
**标准回答：**
"更高排名的供应商存在以下限制：
1. 资源限制：当前无空闲产能，最早X月X日可用
2. 业务匹配：该供应商专注于XXX业务，与本次任务匹配度不高
3. 成本考虑：单价超出预算X%
4. 风险分散：当前XXX业务线已集中XX%，需分散风险"
### 5.3 中尾部供应商的评估依据是什么？
**标准回答：**
"中尾部供应商除业绩外，重点评估：
1. **改进意愿**：是否积极响应整改、主动优化（近期案例XXX）
2. **业绩趋势**：近3个月是否持续上升（如X→X→X）
3. **资源可用性**：是否有立即可用的产能
4. **成本优势**：单价是否具有竞争力
5. **风险预案**：已准备备选方案XXX，可随时切换
### 5.4 如何确保资源分配公平？
**标准回答：**
"资源分配基于以下评估框架：
1. **量化评分**：业绩40分+合作30分+资源20分+策略10分
2. **多维度评估**：不只看排名，还看趋势、意愿、可用性
3. **全程记录**：每次分配都有书面决策记录（编号：XXX）
4. **持续跟踪**：分配后定期跟踪，及时调整
5. **审计可追溯**：所有决策记录可查阅XXX文件"

## 六、决策记录归档
### 6.1 归档要求
- 所有供应商选择决策必须有书面记录
- 决策记录保存期限：至少3年
- 按时间顺序归档，便于审计查阅
### 6.2 归档位置
- 临时任务决策：`workspace/供应商管理/决策记录/临时任务/`
- 资源分配决策：`workspace/供应商管理/决策记录/资源分配/`
