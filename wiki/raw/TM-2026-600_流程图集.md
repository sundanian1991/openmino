## 1. 供应商引入流程图（8阶段）

```mermaid
graph TD

    S2 -->|T+1| Decision1{能否内部消化?}
    Decision1 -->|能| Internal[内部扩项<br/>不走外部寻源]
    Internal --> End([流程结束])
    Decision1 -->|不能| S3[阶段3: 寻源触达]
```

## 2. 供应商清退流程图（8阶段）

> 来源：SM-2026-040 §5，标准周期约50-60天，即时清退约15天

```mermaid
graph TD
    TriggerSignal([触发信号识别]) --> ClassifyType{清退类型判定}

    %% 即时清退
    ClassifyType -->|合规红线 I-1~I-7| Immediate[即时清退]
    Immediate --> IConfirm[阶段1: 触发确认 即时]
    IConfirm --> IApprove[阶段2: 评估审批 T+1]
    IApprove --> INotice[阶段3: 清退通知 即时]

    %% 评估清退
    ClassifyType -->|连续E级 E-1~E-5| Assessment[评估清退]
    Assessment --> AConfirm[阶段1: 触发确认 T+1]
    AConfirm --> AApprove[阶段2: 评估审批 T+3]
    AApprove --> ANotice[阶段3: 清退通知 T+2]

    %% 整改清退
    ClassifyType -->|整改失败 R-1~R-3| Remediation[整改清退]
    Remediation --> RConfirm[阶段1: 触发确认 T+1]
    RConfirm --> RApprove[阶段2: 评估审批 T+3]
    RApprove --> RNotice[阶段3: 清退通知 T+2]

    %% 主动退出
    ClassifyType -->|供应商主动申请 V-1~V-3| Voluntary[主动退出]
    Voluntary --> VConfirm[阶段1: 触发确认 T+1]
    VConfirm --> VApprove[阶段2: 评估审批 T+3]
    VApprove --> VNotice[阶段3: 清退通知 T+2]

    %% 审批不通过分支
    IApprove --> ApprovalCheck{审批通过?}
    AApprove --> ApprovalCheck
    RApprove --> ApprovalCheck
    VApprove --> ApprovalCheck

    ApprovalCheck -->|不通过| Downgrade[降级处置/重新整改/延长观察]
    Downgrade --> EndNormal([流程结束])
    ApprovalCheck -->|通过| MergeNotice

    INotice --> MergeNotice[阶段4: 交接准备]
    ANotice --> MergeNotice
    RNotice --> MergeNotice
    VNotice --> MergeNotice

    MergeNotice -->|即时: T+3 标准: T+5| S5Phase[阶段5: 系统与数据回收]
    S5Phase -->|即时: T+2 标准: T+3| S6Phase[阶段6: 业务交接与过渡]
    S6Phase -->|即时: 15天 标准: 30天| S7Phase[阶段7: 结算与最终处理]
    S7Phase -->|即时: T+7 标准: T+15| S8Phase[阶段8: 清退复盘与归档]
    S8Phase -->|即时: T+3 标准: T+5| MarkBan[标注禁入期]
    MarkBan --> UpdateLedger[更新台账状态]
    UpdateLedger --> Archive[全流程归档]
    Archive --> EndClear([清退完成])

    classDef stage fill:#e3f2fd,stroke:#1565c0,stroke-width:2px
    classDef decision fill:#fff3e0,stroke:#e65100,stroke-width:2px
    classDef immediate fill:#fce4ec,stroke:#c62828,stroke-width:2px
    classDef normal fill:#e8f5e9,stroke:#2e7d32,stroke-width:2px
    class IConfirm,IApprove,INotice,AConfirm,AApprove,ANotice,RConfirm,RApprove,RNotice,VConfirm,VApprove,VNotice,MergeNotice,S5Phase,S6Phase,S7Phase,S8Phase stage
    class ClassifyType,ApprovalCheck decision
    class Immediate,IMergeNotice immediate
```

## 5. 评估决策流程图（月度评估→评级→分量调整→状态判定）

> 来源：SM-2026-030 §2-5，SM-2026-091 §3

```mermaid
graph TD
    Start([次月1-3日 数据收集]) --> Collect1[质检月报]
    Start --> Collect2[站点月度总结]
    Start --> Collect3[合规台账]
    Start --> Collect4[管理配合记录]

    Collect1 --> Score[次月3-5日 评分]
    Collect2 --> Score
    Collect3 --> Score
    Collect4 --> Score

    Score --> Dim1[交付质量 40分]
    Score --> Dim2[业务达成 30分]
    Score --> Dim3[合规表现 20分]
    Score --> Dim4[管理配合 10分]

    Dim1 --> TotalScore[总分 100分]
    Dim2 --> TotalScore
    Dim3 --> TotalScore
    Dim4 --> TotalScore

    TotalScore --> RedLineCheck{红线检查?<br/>信息安全严重/持证<90%<br/>质检<70/数据造假}
    RedLineCheck -->|是| EGrade[直接降为E级]
    RedLineCheck -->|否| GradeCalc

    GradeCalc -->|80-89分| BGrade[B级 良好]

    DGrade --> Rank

    Action2 --> End
    Action3 --> End
    Action4 --> End
```
