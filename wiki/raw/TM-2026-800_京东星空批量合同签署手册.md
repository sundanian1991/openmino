**通用填写策略**：

```python
def fill_field_by_label(page, label_text, value, field_type="input"):
    """
    通过label文本定位并填写字段
    field_type: input / textarea / select / date / radio
    """
    try:
        # 找到label，向上回溯到表单项容器，再找到对应的输入元素
        label = page.locator(f"text='{label_text}'").first
        container = label.locator("xpath=../../..")  # 尝试回溯3层父元素
        
        if field_type == "input":
            inp = container.locator("input").first
            inp.fill(str(value))
        elif field_type == "textarea":
            inp = container.locator("textarea").first
            inp.fill(str(value))
        elif field_type == "select":
            # 点击下拉触发器
            trigger = container.locator(".ant-select, [class*='select']").first
            trigger.click()
            time.sleep(0.5)
            # 选择选项
            page.locator(f"text='{value}'").first.click()
        elif field_type == "date":
            trigger = container.locator("input, .ant-picker").first
            trigger.click()
            time.sleep(0.3)
            page.keyboard.insert_text(value)
            page.keyboard.press("Enter")
        elif field_type == "radio":
            # 找到包含选项文本的radio
            page.locator(f"text='{value}'").first.click()
        
        return True
    except Exception as e:
        print(f"[WARN] 填写失败 {label_text}: {e}")
        return False
```

```python
# 对于每个模板变量字段，通过label中的变量名定位
fields_to_fill = {
    "${party_b_name}": record["party_b_name"],
    "${party_b_address}": record["party_b_address"],
    "${party_b_phone}": record["party_b_phone"],
    "${party_b_contact_email}": record["party_b_contact_email"],
    "${party_b_contact_person}": record["party_b_contact_person"],
    "${trial_period}": "/",
    "${trial_start_year}": "/", "${trial_start_month}": "/", "${trial_start_day}": "/",
    "${trial_end_year}": "/", "${trial_end_month}": "/", "${trial_end_day}": "/",
    "${renewal_start_year}": "2026", "${renewal_start_month}": "6", "${renewal_start_day}": "1",
    "${renewal_end_year}": "2027", "${renewal_end_month}": "5", "${renewal_end_day}": "31",
    "${party_b_email}": record["party_b_email"],
    "${account_name}": record["account_name"],
    "${account_number}": record["account_number"],
    "${bank_name}": record["bank_name"],
    "${uniform_credit_code}": record["uniform_credit_code"],
}

for field_label, value in fields_to_fill.items():
    try:
        # 通过label文本定位，label显示为变量名如 "${party_b_name}"
        label = page.locator(f"text='{field_label}'").first
        if label.count() == 0:
            continue
        container = label.locator("xpath=../..")
        inp = container.locator("input, textarea").first
        inp.fill(str(value))
        print(f"  已填 {field_label}")
    except Exception as e:
        print(f"  [WARN] 跳过 {field_label}: {e}")
```

## 五、登录态处理

**方案A（推荐）: 复用Chrome用户数据** 

如果用户已经在Chrome中登录，脚本启动的Chromium是独立进程，不共享Cookie。

解决方案：让用户在脚本启动的浏览器中手动登录一次，或手动复制Cookie。

更简单的做法：脚本打开页面后，**暂停等待用户手动登录**，然后按回车继续。

```python
print("请在打开的浏览器窗口中手动登录平台，登录完成后按回车继续...")
input()
```

**方案B: 使用已登录的Chrome实例**

Playwright可以通过CDP协议连接已运行的Chrome，但这需要启动Chrome时开启远程调试端口，操作复杂。不推荐。

## 六、错误处理与调试

### 通用排查流程

1. **元素找不到**：
   - 用 Playwright 的 `page.pause()` 打开调试器
   - 在浏览器中按 F12，用元素选择器获取准确的 selector
   - 更新脚本中的定位策略

2. **下拉选择失败**：
   - 某些下拉框是模拟的（非原生select），需要先点击触发下拉，再点击选项
   - 尝试先点击输入框，等待0.5秒，再点击选项

3. **日期选择失败**：
   - 尝试直接输入文本而不是点击日历
   - 或尝试 `page.keyboard.insert_text("2026-06-01")`

4. **页面加载超时**：
   - 增加 `time.sleep()` 等待时间
   - 某些操作后页面会重新加载，需要等待DOM稳定

### 调试模式

在脚本关键位置插入 `page.pause()` 可以打开Playwright Inspector，可视化调试：

```python
page.pause()  # 会暂停并打开调试窗口
```

## 七、数据校验规则

填写前对每条记录做基础校验：

```python
def validate_record(record):
    errors = []
    if not record.get("party_b_name"): errors.append("供应商名称不能为空")
    if not record.get("party_b_phone"): errors.append("电话不能为空")
    if not record.get("party_b_contact_email") or "@" not in record["party_b_contact_email"]:
        errors.append("联系人邮箱格式错误")
    if not record.get("account_number"): errors.append("银行账号不能为空")
    if not record.get("bank_name"): errors.append("开户行不能为空")
    if not record.get("uniform_credit_code"): errors.append("统一信用代码不能为空")
    return errors
```
