# Strategic Visual Emphasis Using Preattentive Attributes

Apply this skill when your visualization contains multiple competing elements but only one key insight must be immediately perceived by the audience.

## When to Use
- You’ve identified a single “hawk” (key data point or message) among many “pigeons” (background elements)
- The visual supports explanatory (not exploratory) analysis
- You’ve already removed clutter and distractions
- The audience should grasp the main point without referring to legends or doing mental cross-referencing

## How to Execute
1. **Identify the primary insight**—this should represent ≤10% of total visual area or elements.
2. **Emphasize using preattentive attributes**, prioritized by effectiveness:
   - **Bold**: Best for text (titles, labels)
   - **Color**: Use sparingly; combine with another attribute if possible
   - **Size**: Increase to signal importance
   - **Position**: Place first in sequence or at focal points
3. **De-emphasize all other elements**:
   - Render in light grey or desaturated tones
   - Reduce opacity, line thickness, or label prominence
4. **Avoid visual competition**:
   - Never highlight multiple unrelated insights
   - Do not use blinking/flashing (reserved for system alerts)
5. **Align emphasis with narrative**:
   - Color both data points and related annotations (e.g., headline, markers)
   - Add direct labels at critical points (e.g., “Launch: Q2 2023”)
6. **Validate**: Viewers’ eyes should land first on the intended focal point.

> **Note**: Layer attributes (e.g., large + bold + color) only for extremely critical elements.
