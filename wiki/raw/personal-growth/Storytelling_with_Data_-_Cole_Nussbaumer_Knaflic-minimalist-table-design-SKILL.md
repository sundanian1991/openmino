# Minimalist Table Design

Use this skill to format data tables so structural elements recede and values become immediately legible. Follow these steps:

1. **Remove heavy borders**—eliminate them entirely or replace with light grey lines to avoid visual competition.
2. **Use white space strategically** to separate rows and columns instead of relying on borders.
3. **Align data consistently**: right-align numeric values and left-align text to aid scanning.
4. **Avoid background colors or shading** that draw attention away from the data itself.
5. **Adapt for context**:
   - In *reports* and *printed materials*, include full tables with minimalist formatting.
   - In *dashboards*, ensure tables remain scannable at a glance.
   - In *live presentations*, avoid full tables; extract key insights visually or move detailed tables to an appendix.
6. **Retain table format when units of measure vary** across columns—it’s more effective than graphs for mixed units.
7. **Optimize for verbal processing**: audiences read tables row-by-row (often tracking with a finger), so prioritize readability over decoration.

## When to Use
- Presenting reference data for lookup
- Sharing tables with mixed units
- Creating reports or dashboards where precision matters

## When Not to Use
- Showing trends, patterns, or relationships (use charts instead)
- Live presentations with divided audience attention
- Situations where quick visual interpretation is more important than exact values
