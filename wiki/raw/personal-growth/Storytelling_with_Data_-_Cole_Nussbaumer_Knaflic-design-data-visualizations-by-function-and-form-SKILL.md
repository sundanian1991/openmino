# Design Data Visualizations by Function and Form

Use this skill whenever you are communicating findings to an audience and need to ensure your visual form directly enables their intended insight or action.

## When to Use
- You have a specific audience action or understanding goal (e.g., “Approve budget,” “Identify top performers”)
- You are moving from exploratory analysis (discovery) to explanatory communication (storytelling)
- You must avoid presenting raw analytical artifacts (“opening 100 oysters”) to audiences

## How to Execute
1. **Define the functional goal first**: Explicitly state what the audience should DO or UNDERSTAND (e.g., “Compare Q3 performance across regions”).
2. **Distinguish analysis types**:
   - **Exploratory**: For internal discovery—test many views, keep all outputs
   - **Explanatory**: For external communication—show only the “pearls” (key insights)
3. **Select form based on function**:
   - Trends over time → line chart
   - Category comparisons → bar chart
   - Relationships → scatterplot
   - Single/few numbers → large text (not a chart)
4. **Ensure every design choice supports the function**:
   - Remove decorative elements
   - Order categories by importance or performance
   - Suppress non-essential details (e.g., hide exact axis values to emphasize relative differences)
5. **Validate**: Does this form make the intended function effortless for the audience?

> **Note**: Never let chart aesthetics override functional clarity. If only one or two numbers matter, use prominent text—not a graph.
