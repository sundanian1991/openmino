# Pie Chart Alternatives for Proportional Change

Avoid dual pie charts—they force viewers to compare angles or areas across separate charts, which is cognitively difficult. Instead, choose one of four validated alternatives based on your message focus and data constraints.

## Step 1: Determine Your Primary Message Focus

- **Single key metric**: If only one result matters (e.g., “72% interested after program”), **show numbers directly**:
  - Display the headline percentage as large, prominent text
  - Add supporting context (e.g., baseline comparison, sample size)
  - Include response rate and data source

- **Full distribution comparison**: If you must show how all categories changed, proceed to Step 2.

## Step 2: Choose a Graphical Alternative Based on Constraints

### Option A: Simple Bar Graph
Use when category order must be preserved (e.g., ordinal scales like sentiment: Very Negative → Very Positive).
- Group bars by state (Before / After) with a shared baseline
- Use minimal color: neutral (e.g., gray) for context, accent (e.g., blue) for emphasis
- Place labels adjacent to relevant bars for clarity

### Option B: 100% Stacked Horizontal Bar Graph
Use when both ends of a spectrum matter (e.g., Strongly Disagree vs. Strongly Agree).
- Align negative responses to the left baseline, positive to the right
- Retain a consistent 0–100% x-axis scale for bidirectional readability
- Clearly shows part-to-whole relationships within each state

### Option C: Slope Graph
Use when change magnitude and relative ranking are key.
- Connect Before and After values with lines; steepness = magnitude of change
- Categories are automatically ordered by value (highest at top)
- **Warning**: May reorder categories unnaturally—avoid if sequence is fixed

## General Guidelines
- Never use dual pie charts for two-state comparisons
- Always report sample size, response rate, and data source
- Match your design choice to your analytical priority: simplicity, part-to-whole clarity, ordered comparison, or change magnitude
