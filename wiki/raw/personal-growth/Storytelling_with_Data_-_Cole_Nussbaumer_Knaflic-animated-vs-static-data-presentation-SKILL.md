# Animated vs Static Data Presentation

Use this skill when you need to present time-series data (e.g., line graphs) in a live setting **and** provide a circulated report afterward. The goal is to maximize engagement during delivery while ensuring clarity and completeness in the distributed version.

## When to Use
- You are presenting live (in person or virtual) **and** must distribute a static version afterward
- Your data tells a chronological story with distinct phases (e.g., launch, growth, plateau)
- Your visuals include line graphs or other time-series charts
- You can narrate confidently without relying on on-screen visuals

## How to Execute

1. **Prepare the animated version for live delivery:**
   - Start with a blank graph showing only axes and labels
   - Reveal data incrementally in sync with your spoken narrative:
     - First, show the starting point (e.g., “September 2013, 5,000 active users”)
     - Then reveal early growth
     - Highlight key inflection points (e.g., marketing campaigns, external events)
     - Show plateaus or declines with context
     - End with the most recent trend
   - Use built-in animation tools (e.g., PowerPoint Morph, Keynote Builds, Google Slides animations) to control element appearance

2. **Create the static version for circulation:**
   - Produce a single, fully annotated graph that includes all narrative milestones
   - Add callouts, labels, arrows, or shaded regions to explain each phase
   - Ensure it stands alone without verbal explanation

3. **Choose an implementation approach:**
   - **Option A:** Build layered animations on one slide; the final frame becomes the static version
   - **Option B:** Use multiple sequential slides for the presentation; circulate only the final annotated slide

4. **Critical reminder:** The presenter must know the story well enough to deliver it without depending on the visuals. Animation enhances storytelling—it does not replace narration.

> **Trigger phrase**: “The same visual must serve both live presentation and circulated report purposes.”
