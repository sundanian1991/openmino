## 一、数据沟通的完整流程

这本书 200 多页不是"每章一个 skill"。实际上它先讲了一个**完整的数据沟通框架**，然后把框架中的关键步骤和常见坑拆解成了 30 个独立 skill。核心骨架是：

```
理解受众 → 明确目标 → 选图表 → 去噪音 → 引注意力 → 讲故事 → 迭代打磨
   ↓           ↓         ↓        ↓        ↓         ↓        ↓
 context    big idea   选图    declutter   preattentive  story   iterate
```

30 个 skill 中，有些是**框架步骤**，有些是**具体场景的处理手法**，有些是**组织和流程建议**。它们不是平级的——我建议合并为 **5 个核心模块**（后文详述）。

## 模块一：沟通策略层（5 个 skill → 合并为 1 个模块）

> **回答你的问题**：这 5 个讲的是"做数据沟通之前该想什么"，属于元层面的指导，不是具体的图表技法。

### 1.1 define-audience-specific-communication-context — 定义受众

**核心**：拒绝"所有利益相关者"这种模糊定义。

- 找出**真正的决策者**（谁有权根据你的数据做决定）
- 评估**信任度**：你是被看作权威、中立还是需要更多证明
- 多类受众有分歧时，做**分开的沟通**，不要拼成一个谁都不满意的版本
- 根据受众画像决定语气（正式/对话）、深度（摘要/附录）、侧重（ROI/体验）

### 1.2 articulate-action-oriented-objectives — 行动导向目标

**核心**：每次数据沟通必须有明确的"你希望受众做什么"。

- 用强动词：**批准、调整、优先、启动**，而不是"了解、注意、关注"
- 即使暂时不能直接推荐，也要设一个能催化行动的开放式问题（"Q3 要不要试点？"）
- 表明你的信心级别（高/中/探索性）
- **不完美的建议也比被动的总结好**——至少能推动讨论

### 1.3 context-driven-data-storytelling — 上下文驱动的数据故事（总框架）

**核心**：全书的总纲，分两阶段六步骤。

**阶段一：建立上下文**
1. WHO：谁会受你的数据影响
2. WHAT：受众需要知道什么、做什么
3. HOW：用什么指标支撑你的信息
4. **Big Idea**：一句话，包含洞察 + 建议
5. 验证：所有设计选择是否服务于 Big Idea

**阶段二：六个核心步骤**
1. 理解上下文
2. 选合适的视觉形式
3. 消除噪音（Gestalt 原则）
4. 引导注意力（前注意属性）
5. 功能优先于形式
6. 用三幕结构讲故事

### 1.4 three-act-data-storytelling — 三幕叙事结构

**核心**：把数据论证变成能推动行动的叙事。

| 幕 | 内容 | 关键动作 |
|----|------|----------|
| **第一幕（铺垫）** | 设定场景、受众是主角、指出失衡、预览方案 | 回答"我为什么要关心" |
| **第二幕（冲突）** | 用证据展开问题：背景、对比案例、数据、不行动的后果 | 评估选项、建立你的推荐案的论据 |
| **第三幕（决议）** | 明确的行动号召、回顾紧迫性、Bing-Bang-Bongo 技术 | 具体到"做什么、谁做、何时" |

> Bing-Bang-Bongo：先告诉他们会说什么 → 再说 → 最后告诉他们你说了什么

### 1.5 time-budgeting-for-data-storytelling — 给讲故事留时间

**核心**：分析完成后，**留出和分析阶段同等甚至更多的时间做沟通打磨**。

- 受众只看到沟通阶段，看不到你清洗数据、建模的过程
- 默认图表设置缺乏叙事意图，几乎不能直接用
- 时间紧张时的优先动作：加洞察导向标题、去非数据墨、高亮关键结论
- 不要"画图就走"——沟通不是画完图就结束

## 模块二：图表选择与形态（6 个 skill）

> 解决"用什么图表、为什么用"的问题。

### 2.1 design-data-visualizations-by-function-and-form — 功能决定形式

**核心**：先定义受众该做什么/理解什么，再选图。

| 功能目标 | 图表形式 |
|----------|----------|
| 时间趋势 | 折线图 |
| 分类对比 | 条形图 |
| 相关性 | 散点图 |
| 一个/几个关键数字 | **大字，不是图** |

- **探索性分析**（内部发现）→ 多试，全保留
- **解释性沟通**（对外传达）→ 只展示"珍珠"（关键洞察）
- 永远不要让图表美学覆盖功能清晰度

### 2.2 avoid-pie-and-donut-charts — 禁止饼图/环形图

**核心**：人脑不擅长比较角度和面积，饼图是认知负担。

替代方案：
- **占比对比** → 水平条形图
- **部分与整体的关系** → 100% 堆叠条形图
- **关键数字只有一个** → 大字展示
- **需要精确比较** → 条形图（长度比角度容易判断）

### 2.3 pie-chart-alternatives-for-proportional-change — 比例变化的替代方案

**核心**：对比"之前 vs 之后"的占比，**绝对不要用两个饼图**。

根据关注点选择：

| 场景 | 方案 |
|------|------|
| 只关心一个核心指标变化 | **大数字展示** + 样本量 + 基线对比 |
| 必须展示全分布对比 | 分组条形图（保留类别顺序时） |
| 关心两端（非常同意 vs 非常不同意） | 100% 水平堆叠条 |
| 关心变化幅度和排名变化 | **坡度图**（Before→After 连线） |

### 2.4 enforce-zero-baseline-bar-charts — 条形图零基线

**核心**：条形图代表的是**长度**，人脑默认从 0 开始比较。截断 Y 轴 = 视觉欺骗。

- 条形图（横/竖/列）的值轴必须从 0 开始
- 永远不要为了夸大差异而裁剪坐标轴
- **例外**：折线图（编码的是位置，不是长度），但仍需透明处理
- 操纵基线来"强调观点"会损害可信度

### 2.5 avoid-secondary-y-axes-and-3d-visuals — 禁止双 Y 轴和 3D

**核心**：3D 扭曲感知，双 Y 轴暗示虚假关联。

**3D 的替代**：永远不用，除非三维连续变量（极少场景）

**双 Y 轴的三个替代**：

| 替代方案 | 适用场景 |
|----------|----------|
| **直接标注**：在线条/条形上标值，去掉副轴 | 需要精确数值时 |
| **上下分图**：共享 X 轴，各自独立左 Y 轴 | 对比趋势时 |
| **颜色关联轴标题**（不推荐） | 不得已时 |

### 2.6 distinguish-forecast-from-actual-data — 区分预测与实际数据

**核心**：受众不能把预测误认为已确认的结果。

| 元素 | 实际数据 | 预测数据 |
|------|----------|----------|
| 线条 | 粗实线 | 细虚线/点线 |
| 背景 | 正常 | 淡色阴影区隔离 |
| 数值标签 | 只在关键点标 | 每点都标（明确期望值） |
| 锚点（最后一个实际点） | 白色填充 + 彩色边框 + 粗体数字 | — |
| X 轴标签 | ACTUAL | FORECAST（大写） |
| 非关键元素 | 浅灰色 | 浅灰色 |

## 模块三：视觉清晰度和认知原则（8 个 skill）

> 解决"画出来之后怎么让受众一眼看懂"的问题。

### 3.1 minimize-cognitive-load-in-visual-communications — 最小化认知负荷

**核心**：每个视觉元素都要问："它的信息价值值得它的认知成本吗？"

操作：
- 去掉装饰边框、多余颜色、冗余标签、非必需样式
- 禁止斜体、下划线、多种字体
- 标题要有行动导向（不是"销售数据"，是"Q3 销售低于目标 12%"）
- 加来源脚注和上下文注释（"峰值因春节导致"）
- **10 秒测试**：非专业人士能否 10 秒内理解要点？

### 3.2 strategic-visual-emphasis-using-preattentive-attributes — 前注意属性引导注意力

**核心**：≤10% 的元素要突出，其余灰化。

按效果排序的前注意属性：

| 属性 | 用途 |
|------|------|
| **加粗** | 文字标题、标签 |
| **颜色** | 少用，最好搭配另一种属性 |
| **大小** | 表示重要性 |
| **位置** | 排在序列首位或焦点位置 |

- 永远不要同时高亮多个不相关的点
- 数据和注释用同色关联（关键数据点红 → 注释框也红）
- **层叠属性**（大+粗+色）仅用于极度关键的元素

### 3.3 apply-gestalt-and-alignment-principles-for-clarity — Gestalt 与对齐

**核心**：用感知分组替代显式分隔线。

| Gestalt 原则 | 做法 |
|-------------|------|
| **接近性** | 相关项目靠近，不相关的拉开间距 |
| **相似性** | 同色/同形/同大表示关联 |
| **闭合** | 去掉外框，人脑会自动感知完整性 |
| **连续性** | 沿平滑路径对齐元素，去不必要的轴线 |
| **连接** | 用线条（如折线）强关联元素 |
| **包围** | 用淡背景色替代边框 |

**对齐规则**：
- 文字左对齐
- 图例和标签左上对齐（匹配自然视线）
- 禁止对角文字（严重拖慢阅读）
- 软件网格精确对齐

**留白**：
- 边距清
- 关键数字用周围空间隔离
- **不要为了填空而填空**

### 3.4 spaghetti-graph-deconstruction — 多线图（意大利面图）解构

**核心**：线条太多重叠时，用结构化方法去混乱。

**三步法**：

1. **评估数据必要性**：全数据集是否必需？去掉不相关类别或时间段

2. **选择解构策略**：

| 策略 | 做法 | 适用场景 |
|------|------|----------|
| **逐个高亮** | 其他线条灰化，目标加粗+色+直接标注 | 演讲（可以口头切换焦点） |
| **垂直小多图** | 上下排列面板，共享 X 轴，相同 Y 轴范围 | 关心各自趋势，不关心跨类对比 |
| **水平小多图** | 左右排列，共享 Y 轴 | 需要在特定时间点对比量级 |
| **组合** | 小多图 + 每面板内高亮一条线 | 报告（信息密度高） |

3. **强制一致性**：所有面板轴范围必须相同

### 3.5 logical-ordering-categorical-data — 分类数据逻辑排序

**核心**：无内在顺序的分类数据（产品、服务类型），**按意图排序，不按字母**。

- **关注最满意** → 按"非常满意"降序排
- **关注最不满意** → 按"完全不满意"降序排
- **关注未使用** → 按"未使用过"降序排
- **多个故事要讲** → 保持一个基础顺序不变，用颜色/标注突出次要洞察，**不要重新排序**
- **内在有序的分类**（年龄段、学历）→ 保持自然顺序

### 3.6 effective-axis-labeling-for-time-series — 时间序列轴标签

**核心**：受众关心趋势，不关心精确的日期查询。

- 日线数据 → 每 5 或 7 天标一次
- 可选加"第 1 周""第 2 周"超级分类
- **不要每个点都标**
- Y 轴金额：千万级保留完整数字，亿级改用"（百万元）"为单位
- **没有唯一正确答案**——根据受众直觉做有意识的选择

### 3.7 minimalist-table-design — 极简表格

**核心**：表格的结构要退居幕后，让数据成为焦点。

- 去掉重边框，用浅灰细线替代
- 用**留白**分隔行和列，不靠边框
- **数字右对齐，文本左对齐**
- 避免背景色/阴影
- 单位不同时保留表格形式（比图表更有效）
- **报告**：保留完整表格；**演讲**：提取关键洞察，表格放附录

### 3.8 construct-heatmaps-for-tabular-pattern-recognition — 热力图

**核心**：数字表格转热力图，让受众一眼看出高低和模式。

- 确保同尺度和单位
- 选择单调渐变（浅→深或深→浅）
- 加图例或副标题解释颜色映射
- **建议保留单元格内的数字**（模式识别 + 精确读数双用）
- 用色盲友好色板（viridis、ColorBrewer）

## 模块四：迭代与流程（4 个 skill）

> 解决"怎么持续改进数据沟通质量"的问题。

### 4.1 iterate-visual-design-using-optometrist-approach — 验光师迭代法

**核心**：每次只改一个元素，左右对比选优。

1. 版本 A（当前最佳草案）
2. 复制为版本 B
3. 版本 B **只改一个元素**（改颜色/去轴线/调标题位置）
4. 并排对比："哪个版本更好支持我的信息、更易理解？"
5. 选优的作为新基线，重复
6. **保留所有旧版本**以便回退

> 这个方法防止多个变化混在一起，确保每个决策有直接对比支撑。

### 4.2 paper-sketching-for-data-ideation — 纸笔草图

**核心**：用不受工具限制的快速草图探索方案。

- 空白纸/白板，不用数字工具
- 多画几个方案并排（条形图 vs 坡度图 vs 标注表格）
- **重结构不重细节**：布局、信息层次、叙事流
- 草图易丢弃，减少情感依附
- 验证可行后再转数字

### 4.3 seeking-and-applying-external-feedback — 外部反馈

**核心**：找代表受众的同事，不解释你的意图，让他们自己看。

标准问题：
- "你首先注意到什么？"
- "你得出了什么观察或结论？"
- "这引发了什么问题？"
- "怎样能让重点更清晰？"

> **只听不辩**。如果对方关注对了、结论对了 → 有效。如果注意力偏了或困惑 → 找到问题元素。

### 4.4 animated-vs-static-data-presentation — 动画 vs 静态

**核心**：同一份数据要服务演讲和分发两种场景，做两个版本。

| 版本 | 场景 | 做法 |
|------|------|------|
| **动画版** | 演讲 | 从空白坐标轴开始，按叙事节奏逐步展示数据 |
| **静态版** | 分发/传阅 | 单张全标注图，含所有叙事节点、callout、箭头 |

- 实现方式：一层动画（最终帧=静态版）或多张顺序幻灯片
- **演讲者必须能脱离视觉讲故事**——动画是增强，不是替代

## 模块五：伦理与适应（2 个 skill）

### 5.1 integrate-supporting-and-opposing-data-ethically — 数据伦理

**核心**：数据论证必须包含反对证据和上下文，否则损害可信度。

- 分类所有证据：支撑、反对、上下文
- 高风险决策 → 更多透明度
- 专家受众 → 期望对局限性的细致处理
- 把反对点**不是当弱点**，而是当深度分析的证据

### 5.2 adapt-visual-design-to-context-and-tone — 根据载体和基调调整

**核心**：核心清晰度不妥协，但格式和语气要适配。

| 载体 | 规则 |
|------|------|
| **演讲** | 页面精简，视觉强化口头观点 |
| **报告/邮件** | 包含更多细节，确保自包含 |
| **Slidument（避免不了时）** | 用附录满足文字细节需求 |

- **暗背景**：反转突出逻辑（白/黄突出），用高对比饱和色
- **基调一致**：严肃话题不能用花哨图形

## 模块六：特定图表模板（3 个 skill）

### 6.1 positive-negative-stacked-bar-design — 正负堆叠条形图

**核心**：展示流入和流出如何导致最终的缺口或盈余。

- 水平零基线 = 当前状态
- 正贡献堆在上方，负贡献堆在下方
- **缺口用空心条形标注**（最关键信息）
- 视觉路径：标题 → 缺口数字 → 描述文字 → 支撑数据

### 6.2 horizontal-stacked-bar-survey-priority — 调研优先级堆叠条

**核心**：受访者排名数据，展现总重要性和排名分布。

- 总 % 降序排列（最高在上）
- 水平放置，类别名在左
- 排名从左到右：第一名最深、第二名中、第三名最浅
- 单色系色调变化，可选高亮 Top N
- 去掉 X 轴（直接标注），加样本量脚注

### 6.3 strategic-ordering-in-stacked-bar-charts — 堆叠条形图排序

**核心**：堆叠段的顺序必须服务于核心关注点。

- 关注 "Miss" → 放在底部
- 关注 "Exceed" → 放在顶部
- 有自然顺序的分类（绩效等级）→ 按自然顺序
- 核心关注段始终在同一个位置，跨图表保持一致
