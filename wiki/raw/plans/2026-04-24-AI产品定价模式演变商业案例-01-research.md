# AI 产品定价模式演变：从 Coding Plan 到 Token Plan

**类型:** Research
**状态:** Draft
**创建:** 2026-04-24

---

## 核心结论（先说答案）

AI 大模型定价正从"卖工具订阅"演进为"卖算力资源包"，这个过程精确重演了电信行业从话费套餐到流量套餐的 20 年历史。**Coding Plan 是过渡产品，Token Plan 是终态。** 能从中学到的不是具体的定价数字，而是三件事：计量单位即定价权、补贴-收紧-分层的三段式节奏、以及"入口产品"到"生态产品"的升级路径。

## 研究框架

**1.2 参与者格局**

两条路线并行：
- **模型厂商（专卖店）**：智谱 GLM、Kimi、MiniMax、小米 MiMo —— 只卖自家模型，深度优化
- **云平台（超市）**：阿里云百炼、火山方舟、腾讯云 —— 聚合多模型，一键切换
- **新入局者信号**：小米 MiMo 以极低定价入局（$1/$3 vs Claude $5/$25），但官方同时批评"低价是陷阱"，暗示后续收紧

**1.3 计量体系混乱**

五种计量单位并存是最大的信息壁垒：
- API 请求次数（阿里百炼、火山方舟、腾讯云）：1 次提问 = 5-30 次 API 调用
- Prompt 次数（智谱 GLM、MiniMax 早期）：1 Prompt ≈ 15-20 次模型调用 = 1200-1600 API 请求
- Token 计量（Kimi 2026.1.28 切换后、MiniMax Token Plan）：直接按输入输出 Token 计费，仅统计未命中缓存的 Token
- Credit 计量（Cursor、小米 MiMo、阿里百炼 Token Plan）：统一 Credit 池，不同模型不同倍率
- 时间窗口配额（智谱 GLM 5h 窗口、Claude Code 5h+7d 双层）：限制节奏而非总量

**直接比数字毫无意义，90% 开发者踩坑。**

**1.3.1 套餐使用量差异（入门档 ¥40-50 区间）**

| 平台 | 月费 | 计量 | 额度 | 实际等效 |
|------|------|------|------|----------|
| 智谱 GLM Lite | ¥49 | Prompt | ~80 prompts/5h, 2400/月 | 每天 80 次提问 |
| MiniMax Starter | ¥29 | Prompt | 40 prompts/5h，无周月限 | 连续每 5h 40 次 |
| 阿里百炼 Lite | ¥40 | API 请求 | 18000/月 | ~20-120 次提问/天 |
| 腾讯云 Token Lite | ¥39 | Token | 3500 万 Token | ~70 轮深度问答 |
| Cursor Pro | $20 | Credit | $20 池 ≈ 225 Sonnet 请求 | Auto 模式无限 |

关键发现：同样 ¥40-50/月，不同计量方式下实际可用量差异巨大。Agent 场景消耗爆炸——8 个月 Claude Code 消耗 100 亿 Token（API 价 $15,000+），Max plan 只花 $800。Token 构成：文件读取 35-45%，代码生成仅 5-15%。

**1.4 OpenClaw：Token 消耗的催化剂**

OpenClaw（GitHub 355K stars）作为开源 AI Agent 框架，直接催热了 Coding Plan 需求：
- MiniMax M2 token 用量：2 月 vs 12 月增长 6 倍，Coding Plan 消耗增长 10 倍+
- 阿里云算力告急，限量供应 → Lite 档 4-13 停售的直接原因
- Anthropic 2026-04 从标准订阅移除 OpenClaw 访问 —— 成本撑不住了

Agent 自动循环调用让"不限量订阅"不可持续，倒逼厂商转向 Token 精细计量。

**1.5 模型进步时间线（2026 年）**

| 时间 | 模型 | 参数 | 意义 |
|------|------|------|------|
| 2026-01 | Kimi K2.5 | 1T | 万亿参数级，编程能力跃升 |
| 2026-02 | 智谱 GLM-5 | 744B | 国产大模型进入第一梯队 |
| 2026-02 | MiniMax M2.5 | 229B | 小参数高效路线 |
| 2026-04 | 智谱 GLM-5.1 | - | 持续迭代 |

| 2026-04-24 | DeepSeek V4 | 1.6T(49B)/284B(13B) | 开源旗舰，成本暴降 73%，适配华为/寒武纪，100 万 token 上下文 |

模型能力跃升 → Agent 可用性提高 → Token 消耗指数级增长 → Coding Plan 承压 → Token Plan 成为必然

**1.6 芯片供给线：算力成本的底层变量**

英伟达在中国退缩：市占率 66%(2024) → 54%(2025)，2025-04 全面禁止 AI GPU 对华出口

华为昇腾崛起：
- 2025 年交付 81.2 万片，市占率 ~20%，国产第一
- 950PR：FP4 算力 1.56 PFLOPS（H20 的 2.87 倍），定价 ~7 万元（H200 的 1/3），4 月量产
- 路线图：950PR(26Q1) → 950DT(26Q4) → 960(27Q4) → 970(28Q4)
- 2026 目标：市占率 50%，出货 75-100 万片，营收超 800 亿

国产格局：2025 年自给率 41%，伯恩斯坦预测 2027 年达 55%。华为自研 HBM 绕开供应链限制。DeepSeek V4 同时适配华为+寒武纪。

**1.7 DeepSeek V4 定价冲击（2026-04-24 发布）**

| 模型 | 总参数 | 激活参数 | 输入（缓存） | 输入（未缓存） | 输出 |
|------|--------|----------|-------------|---------------|------|
| V4-Pro | 1.6T | 49B | 1 元/百万 | 12 元/百万 | 24 元/百万 |
| V4-Flash | 284B | 13B | 0.2 元/百万 | 1 元/百万 | 2 元/百万 |
| 对比 GPT-5 | - | - | $5/百万 | - | $15/百万 |

长上下文效率：单 token 推理计算量仅为 V3.2 的 27%，KV Cache 占用降至 10%。
关键信号："受限于高端算力，预计下半年昇腾 950 批量上市后价格大幅下调" → 芯片供给直接绑定定价天花板。

**1.8 IPO/资本化进程与定价策略的关联**

| 公司 | 上市时间 | 市值/募资 | 关键财务数据 | 定价信号 |
|------|----------|-----------|-------------|----------|
| 智谱 GLM（2513.HK） | 2026-01-08 | 市值 511 亿港元，募资 43 亿 | ARR 17 亿元 +60 倍，Coding Plan 两月 15 万付费开发者 | 固定月费模式有市场需求，ARR 快速破亿 |
| MiniMax（MMX） | 2026-01 | 募资 40 亿+港元 | 营收 7904 万美元 +159%，海外 73%，毛利率 25.4% | Token Plan 全球化定价需要考虑海外市场 |

IPO 对定价策略的影响：
- 上市公司必须证明商业可持续性 → Coding Plan/Token Plan 是可见的收入增长引擎
- 招股书披露的财务数据直接影响市场预期 → 定价不能太激进也不能太保守
- 智谱 Coding Plan 两个月 ARR 破亿 → 证明固定月费模式有市场需求 → 其他厂商跟进

**1.9 跨越鸿沟：AI 从实验到生产的信任挑战**

数据：
- Gartner：90% 企业推进 AI 实验，仅 41% 成功落地生产环境
- 预计 2027 年 40% Agentic AI 项目因无法证明 ROI 被终止
- 斯坦福 2026 AI 指数报告："能力-准备度鸿沟"，AI 事故年增 55%

行业判断：
- 真格基金戴雨森：Agent 产品仍处 early market，跨越鸿沟需要模型能力和产品形态继续进步，"依然需要几年的时间"
- 港股 AI 上市潮（Q1 IPO 募资 79 天破 1000 亿港元），但资本市场热情与实际落地存在落差

对定价的含义：
- 企业在实验阶段愿意尝试低价 Coding Plan → 进入生产环境后需要 SLA、稳定性、合规性 → Token Plan 企业版溢价的基础
- 跨越鸿沟的关键不是模型能力，而是"可信赖度" → 定价策略从"低价获客"转向"价值证明"

### Part 2: 框架层 —— 为什么这么演变

**2.1 类比框架：电信套餐演进史**

| 电信阶段 | AI 对应 | 核心逻辑 |
|----------|---------|----------|
| 2009 话费套餐（充 100 用不完） | 2024 API 按量计费（百万 Token 几十元） | 供给过剩，用户感知不到成本 |
| 2013 流量套餐（9.9 几 G） | 2025 Coding Plan（¥29-200/月） | 用量激增，需要包月降低感知 |
| 2017 "不限量"达量降速 | 2026 分层套餐（极速版/普通版） | 精细化分层，按速度/质量定价 |
| 2020 按 GB 精打细算 | 2026+ Token 精细计量 | 用户开始算账，厂商利润回归 |

**2.2 五力分析**

| 力量 | 分析 |
|------|------|
| **供应商议价力** | 算力（GPU）是刚性成本。NVIDIA 占 85-90% 市场，算力成本传导到 Token 定价 |
| **买方议价力** | 开发者选择多（7+ 平台），切换成本低（API Key 一换就行），议价力强 |
| **替代品威胁** | 开源模型（DeepSeek、Qwen）+ 本地部署 = 潜在替代。压制定价上限 |
| **新进入者威胁** | 低（需要百亿级融资 + 算力），但小米 MiMo 等新玩家仍在进入 |
| **行业内竞争** | 极高。7 家在一个池子里打价格战，首月优惠从 7.9 元起 |

**2.3 定价模式演进的三层逻辑**

```
第一层：成本驱动
  算力成本 → Token 成本 → 套餐定价

第二层：竞争驱动
  价格战 → 补贴圈用户 → 收紧利润

第三层：价值驱动
  从"卖 Token"到"卖工作流"到"卖生态"
```

### Part 3: 洞察层 —— 能提炼什么

**3.1 计量单位即定价权**

> 谁定义计量单位，谁就掌握定价权。

- MiniMax 从 Prompt → Token：从"模糊打包"到"透明计费"，争夺用户信任
- Kimi 从 Prompt → Token：同理
- 云厂商用"API 请求次数"：数字看起来大，但实际消耗多，信息不对称有利于厂商

**可复用模式**：当你推出新产品时，计量单位的选择决定了用户感知和你的利润空间。模糊计量有利于早期获客，透明计量有利于长期信任。

**3.2 补贴-收紧-分层的三段式节奏**

> 所有互联网赛道的定价都走这三步，AI 也不例外。

| 阶段 | 目的 | AI 案例 |
|------|------|---------|
| 补贴 | 抢用户规模，建立习惯 | 2024-2025：百万 Token 几毛钱，首月 7.9 元 |
| 收紧 | 测试价格弹性，筛选付费用户 | 2025 H2：限购、停售低档、限时抢购 |
| 分层 | 利润最大化，匹配不同支付意愿 | 2026：MiniMax 极速版 ¥98-899，Kimi Allegro ¥559 |

**可复用模式**：进入新市场时，不要一开始就想好定价。先补贴跑量，再收紧测试弹性，最后精细分层。

**3.3 从"入口"到"生态"的升级路径**

> Coding 是获客入口，Token 是生态载体。

- MiniMax 的 Token Plan 不只卖编程，打包了图片、语音、音乐、视频
- 从"编程工具"升级为"AI 多媒体工作室"
- ¥29 买到的不只是代码补全，是全模态 AI 能力

**可复用模式**：用高频刚需场景（编程）获客，用低频高价值场景（视频/音乐）提高 ARPU。

**3.4 国内外定价逻辑的根本差异**

| 维度 | 国内 | 海外 |
|------|------|------|
| 竞争环境 | 7 家在同一池子打价格战 | 面对 Claude/GPT/Gemini 三巨头 |
| 定价策略 | 低价抢量，比谁更便宜 | 证明"我不比 Claude 差"，比谁更强 |
| 用户预期 | "你在国内买到的便宜，是因为有人在替你卷" | 为质量和生态付费 |

**3.5 Agent 时代的成本结构剧变**

- 70% coding agent token 是浪费（DEV Community）
- 一个开发者 8 个月消耗 100 亿 Token
- Cursor 2025-06 转 credit-based，重度用户一周超支 $350
- 订阅费是地板不是天花板，实际账单 2-5 倍

**含义**：Agent 时代，用户成本会指数级增长。定价模式必须适应这个现实。

### Part 4: 提炼实战清单

**4.1 定价策略决策清单**

- [ ] 定义计量单位：用什么计费？（Token/请求/次数/credit）
- [ ] 选择获客策略：补贴还是差异化？
- [ ] 设计分层结构：几档？价差多少？
- [ ] 确定透明度：额度描述要多透明？
- [ ] 规划演进路径：从什么到什么？

**4.2 商业模式转型检查点**

- [ ] 用户成本可预测性：用户能不能算清楚自己要花多少？
- [ ] 利润弹性：收紧定价后用户留存如何？
- [ ] 生态粘性：从单一场景到多场景的升级是否自然？
- [ ] 竞争护城河：计量单位/定价结构是否容易被复制？

**4.3 反模式（踩过的坑）**

- Cursor 2025-06 退款风波：突然改变计费模式，用户反弹
- 计量单位混乱：90% 用户踩坑，信息不对称最终伤害信任
- 饥饿营销反噬：每天 10:30 抢购，短期有效，长期伤品牌

### Part 5: 未来预判

**5.1 Token 成本将持续下降**

驱动力：国产芯片放量（华为 950 年内 75 万片）+ DeepSeek V4 开源 MoE 效率革命 + 推理专用芯片竞争（英伟达 Rubin CPX / 谷歌 TPU v8 / 华为 950PR）

结果：Token Plan 价格战继续，利润空间来自"服务溢价"而非"算力差价"。

**5.2 国产芯片替代加速改变供给格局**

英伟达市占率持续下降（66% → 54% → 2026 年可能低于 50%）。华为昇腾 + 寒武纪 + 阿里平头哥形成国产算力三角。软件生态从 CUDA 迁移到 CANN，效率已超 90%。

含义：算力供给不再被英伟达卡脖子 → 国内厂商成本结构和海外产生分化 → 定价策略可以更激进。

**5.3 开源模型重塑定价权**

DeepSeek 模式：开源 + 极致低价 + 不卖订阅。V3.2 在 OpenRouter 月调用 5.35 万亿 token，全球第三。但 DeepSeek 没有 Coding Plan/Token Plan，只卖 API。

冲击：开源模型的免费部署能力持续压制定价上限。厂商定价权来自"服务便利性"而非"模型独占性"。

**5.4 Agent 时代的成本指数级增长不可逆**

推理需求 2026 年已达训练的 4-5 倍，IDC 预测 2028 年推理占比 73%。Coding Plan "不限量"模式不可持续。所有厂商最终转向 Token 精细计量——谁先转谁主动，谁后转谁被动。
