# 开源模型的定价权悖论

DeepSeek 月调用 5.35 万亿 token（全球第三），但不卖 Coding Plan/Token Plan，只卖 API。开源模型的免费部署能力持续压制定价上限——厂商定价权来自"服务便利性"而非"模型独占性"。

## Context

DeepSeek V4（2026-04-24 发布）开源后：
- V3.2 在 OpenRouter 月调用 5.35 万亿 token，全球第三
- 开源模型占所有开源 Token 的一半以上
- MIT 开源许可，无任何版权约束，可商业使用/修改/再分发
- 但 DeepSeek 没有 Coding Plan/Token Plan，也没有 App 订阅付费
- 主流云厂商（AWS、阿里云、火山引擎、腾讯云）都部署了 DeepSeek，不付费不分成

## Implication

开源模型对定价权的冲击是结构性的：
- 当用户可以免费部署同级别模型时，Coding Plan/Token Plan 的价值只能来自"便利性溢价"
- DeepSeek 的商业模式是"卖 API 服务"而非"卖模型独占权"
- 这迫使所有厂商的定价必须考虑开源模型的替代成本

规则：如果你的定价权依赖于"模型独占"，开源模型会持续侵蚀你的护城河。定价权应该建立在"服务生态"（多模态打包、Agent 编排、行业解决方案）上，而非"模型能力"上。
