# /kw-workflow — Compound Knowledge 完整工作流

> 一键启动从头脑风暴到知识沉淀的完整流程

---

## 功能

当需要对复杂问题进行**系统化深度分析**时使用。

自动执行完整工作流：
```
brainstorm → plan → review → work → compound
```

---

## 使用场景

| 场景 | 说明 |
|------|------|
| **组织问题分析** | 权责不对等、流程优化 |
| **战略决策** | 职业规划、业务方向 |
| **复杂调研** | 技术选型、竞品分析 |
| **问题梳理** | 会议纪要整理、经验总结 |

---

## 触发方式

```bash
# 完整触发词
/kw-workflow 供应商权责不对等问题

# 简化触发
/kw-workflow 如何在一年内升到P7

# 自然语言
完整分析一下这个问题
帮我梳理一下组织架构
从零开始分析供应商流失
深度分析这个战略决策
```

## 工作流步骤

### 1. Brainstorm（头脑风暴）
- 提取核心要素：决策点、开放问题、约束条件
- 搜索相关知识库
- 识别主题、张力、缺口

**输出**：`plans/问题名-日期/00-brainstorm.md`

### 2. Plan（制定计划）
- 研究问题优先级排序
- 研究方法和数据来源
- 时间框架（短期/中期/长期）

**输出**：`plans/问题名-日期/01-research.md`

### 3. Review（双重审查）
- 战略对齐审查（目标清晰度、可证伪性）
- 数据准确性审查（来源验证、缺口标记）

**输出**：`plans/问题名-日期/03-review.md`

### 4. Work（执行计划）
- 按优先级执行研究任务
- 产出结构化交付物

**输出**：`plans/问题名-日期/02-workspace/*.md`

### 5. Compound（知识沉淀）
- 提炼可复用洞察
- 保存到 `docs/knowledge/`
- 更新知识库索引

**输出**：`docs/knowledge/洞察名.md`

```
plans/问题名-日期/

├── 03-review.md           # 双重审查
└── 02-workspace/          # 执行产出
    ├── 交付物1.md
    ├── 交付物2.md
    └── ...
```
