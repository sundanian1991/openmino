# 观察者

> 触发方式：手动触发 — 用户说"观察"、"总结观察"或"/observer"

---

## 工作流程

### Step 1: 记录到 insights.md

在 `memory/insights.md` 中追加：

**两个部分**：
1. **关键事件** — 今天做了什么、产生了什么结果
2. **洞察** — 需求洞察、模式信号、我的复盘

**记录格式**：
```markdown
### YYYY-MM-DD 事件标题

**事实**：一句话描述

**洞察**：
- 需求洞察：表面问的是A，实际可能在解决B？
- 模式信号：最近反复出现的信号？
- 我的复盘：学到了什么？
```

---

### Step 2: 更新 MEMORY.md

如果有重要变化（决策、待办、项目），更新 `memory/MEMORY.md`：
- 当前待办变化 → 更新"当前状态"
- 新决策 → 更新"关键决策"
- 新项目 → 更新"项目索引"

---

### Step 3: 提交

```bash
git add -A && git commit -m "docs: 更新记忆"
git push
```

---

## 质量原则

- **简洁** — 一句话描述事实
- **深入** — 洞察不只记录发生了什么，要理解为什么
- **不强求** — 没有值得保留的洞察，就不写

---

## 文件结构

```
memory/
├── MEMORY.md     # 详细记忆索引（启动加载）
├── insights.md   # 洞察记录
├── projects/     # 项目背景
└── archive/      # 历史归档
```
