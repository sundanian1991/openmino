# HTML Slide 设计规格

> 1920x1080 固定尺寸 + scale-wrap 自适应缩放
> 用于 AI 定价演变及后续同类 PPT 项目

---

## 色彩系统

| 角色 | 色值 | 用途 |
|------|------|------|
| 背景主色 | `#F5F0E8` | 页面底色（暖米色） |
| 卡片/块背景 | `#EDE8DD` | 所有卡片、区块、容器的背景 |
| 文字主色 | `#2C2825` | 标题、正文 |
| 文字次色 | `#6B6864` | 描述、辅助文字 |
| 文字弱化色 | `#9A9590` | 标签、页码、装饰性文字 |
| 强调色（红） | `#C13531` | 高亮数据、关键结论、序号 |
| 强调色（蓝） | `#293C54` | 对比数据、次要强调 |
| 分割线 | `rgba(44,40,37,0.12)` | 内部结构分隔（非卡片边框） |

### 卡片颜色铁律

- 卡片背景永远用 `#EDE8DD`，**禁止用 `#fff` 白色**
- 卡片背景色与页面背景色同色系不同深浅，形成柔和层次
- 半透明强调卡片用 `rgba(色值, 0.08)` 或 `rgba(色值, 0.10)`

### 禁止

- 渐变 `linear-gradient`、`radial-gradient`（封面光晕除外，需极淡）
- 任何 `#fff` 卡片背景在暖色底页面上
- 彩色边框替代纯色背景（badge 用实心色块，不用透明+边框）

---

## 边框规则

### 禁止

- 卡片外边框（`border: 1px solid ...`）
- IPO badge、价格框、市场条等区块的彩色边框
- 卡片内部的颜色分隔线（如 `border-top: 1px solid rgba(193,53,49,0.15)`）

### 允许

- 表格/列表的内部结构分隔线（极淡 `rgba(154,149,144,0.25)` 以下）
- 价格对比表的行分隔（`border-bottom: 1px solid rgba(44,40,37,0.06)`）
- timeline 轴心线（`#CDCECD`）
- 封面 tag 的强调性边框（有意设计）

### Badge 规范

```css
/* 正确：实心色块 */
.badge.red { color: #F5F0E8; background: #C13531; }

/* 错误：透明背景+彩色边框 */
.badge.wrong { color: #C13531; background: transparent; border: 1px solid #C13531; }
```

## 版式与布局

### 页面结构

所有页面统一使用 scale-wrap 模式：

```html
<div class="scale-wrap">
  <!-- 页码/标签 -->
  <!-- 主内容 -->
  <script>fit()</script>
</div>
```

### 居中

- 所有内容必须**强制水平+垂直居中**
- scale-wrap 已有 `translate(-50%, -50%)` 确保居中
- 内容容器用 `display: flex; flex-direction: column; align-items: center; justify-content: center`
- **禁止**使用 `writing-mode: vertical-rl` 做侧边标注（容易 collapse 成单字）

### 间距系统

| 元素间距 | 值 | 用途 |
|----------|----|------|
| 小间距 | 8-12px | 图标+文字、badge 之间 |
| 中间距 | 20-36px | 卡片之间、列表项之间 |
| 大间距 | 48-60px | 内容区块之间 |
| 超大间距 | 70-120px | 封面/结论页的大块留白 |

### 网格布局

- 4 个卡片 → 2x2 网格，不用 4 列横排
- 3 个卡片 → 3 列横排（卡片宽度适中时）或 1 列垂直堆叠（内容多时）
- 2 个卡片 → 左右并排
- 对比型 → 左右两列 + 中间箭头/鸿沟

### 对称检查

- 五力图等放射布局：中心圆必须在卡片矩阵的几何中心
- 卡片数量不对称时，考虑用网格而非 flex

## 字体层级

| 层级 | 字号 | 字重 | 字体 | 用途 |
|------|------|------|------|------|
| 超大标题 | 80-120px | 700 | Noto Serif SC | 结论页大字 |
| 页面标题 | 48-52px | 700-900 | Noto Serif SC | 每页标题 |
| 卡片标题 | 22-32px | 700-900 | Noto Serif SC | 卡片/区块标题 |
| 正文 | 16-22px | 400 | 默认 | 描述文字 |
| 强调正文 | 16-22px | 600-700 | 默认 | 需要加粗的描述 |
| 标签/标签 | 12-16px | 600 | JetBrains Mono | 页码、badge、计量单位 |
| 数据大字 | 28-72px | 700 | JetBrains Mono | 关键数字 |

### 字号比例检查

- 页面标题与卡片标题的差距至少 16px
- 数据大字与描述文字的差距至少 12px
- 标签文字不得大于 16px，不得喧宾夺主

---

## 图标与 SVG

### 规范

- 线条画（line-art），`fill="none"`
- `stroke-width="2.5"`，`stroke-linecap="round"`，`stroke-linejoin="round"`
- 颜色跟随主题色：红 `#C13531`、蓝 `#293C54`、灰 `#CDCECD`
- 尺寸：行内图标 36-48px，小图标（列表前）6-12px

### 禁止

- 填充型图标、emoji、text-based logos
- 过于简单的圆/方框冒充图标

---

## 开发流程

### 强制步骤

1. **先写设计规格**：确认色彩、版式、组件样式，再动手
2. **做 3 页后立刻合并预览**：不要 25 页全做完才合并
3. **每页完成后自检**：
   - 卡片背景色是否为 `#EDE8DD`（不是 `#fff`）
   - 是否有多余的 `border`
   - 内容是否居中
   - 字号层级是否符合上表
   - SVG 图标是否统一风格
4. **最终全量合并**后检查：
   - grep `background: #fff` → 0 结果
   - grep `border: 1px solid rgba` → 0 结果
   - grep `linear-gradient` → 0 结果（除非封面有意）
