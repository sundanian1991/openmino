# 供应商管理 AI 转型方案 · 介绍

> 电销 AI 转型全员执行方案 · 供应商管理板块
> 范围：32+ 家供应商，3200+ 人，4 大产线（金条/企金/财富/信用卡）、17 条细分业务
> 初期金条 12 家试跑

---

## 一、定位与目标

本方案是"电销 AI 转型全员执行方案"中供应商管理板块的完整框架。

**核心目标**：构建一个可被动咨询、可主动监控、可群聊协作的**供应商管理数字人（Agent）**，将日常运营操作→异常判断→管理决策的全链路系统化。

**关键效益**：

| 指标 | 数值 |
|------|------|
| 月度工作总量 | ~196 小时 |
| AI 可提效 | ~83.5 小时（**43%**） |
| 人工保留 | ~112.5 小时 |
| 释放时间投向 | 供应商寻源、站点布局、BPO 联盟、深度关系建设 |

**角色转变**：供应商管理岗从"一线操作工"升级为"指挥官"——数字人处理 80% 的日常运营，你处理 20% 的异常 + 持续优化系统。

## 二、整体架构

方案采用四层架构设计，从数据到决策逐层支撑：

```
                        ┌─────────────────────────────────────┐
                        │        供应商管理数字人（Agent）        │
                        │  人设：供应商管理专家 · 能对话 · 能推送  │
                        └─────────────────────────────────────┘
                        │              │              │
                  ┌─────┴─────┐ ┌─────┴─────┐ ┌─────┴─────┐
                  │  被动响应   │ │  主动监控   │ │  群聊协作   │
                  │ 即时咨询   │ │ 定时巡检   │ │ 信息抓取    │
                  │ 查询档案   │ │ 异常推送   │ │ 主动交互    │
                  └───────────┘ └───────────┘ └───────────┘
                        │              │              │
                        └──────────────┼──────────────┘
                                       │
    ┌──────────────────────────────────┼──────────────────────────────────┐
    │      L3 Skills 能力层（28 个）     │    L2 可信来源（5 个知识库）        │
    │  引入·管理·清退·战略·通用           │    SOP / 档案 / 案例 / 规则 / 商务 │
    └──────────────────────────────────┴──────────────────────────────────┘
                                       │
                  ┌────────────────────┼────────────────────┐
                  │   L1a 结果数据      │    L1b 过程数据      │
                  │  数仓有表·维度丰富   │  群聊/人工反馈/商务   │
                  │  → data-agent 搭建  │  → joyclaw 接入     │
                  └────────────────────┴────────────────────┘
                                       │
                                ┌──────┴──────┐
                                │   口径统一    │
                                │ 结算/出勤/排班 │
                                │ /产能口径拉齐  │
                                └─────────────┘
                                       │
                                ┌──────┴──────┐
                                │  L4 验证层    │
                                │ 离线评估·衰变监控 │
                                │ 维护机制·纠偏   │
                                └─────────────┘
```

### 各层说明

| 层级 | 名称 | 核心职责 |
|------|------|---------|
| **L1** | 数据底座 | 结果数据（data-agent 自动看板）+ 过程数据（joyclaw 群聊采集），前提是口径统一 |
| **L2** | 可信知识库 | 5 个独立知识库，供数字人检索——不依赖模型自身知识，只查你沉淀的内容 |
| **L3** | Skills 能力层 | 28 个可调用的标准化工作流程，覆盖从准入到清退的全生命周期——这是准确率从 21%→95%+ 的关键杠杆 |
| **L4** | 验证层 | 离线评估 + 衰变监控 + 维护机制，防止系统准确率随时间下滑 |

## 三、工作全景

### 3.1 四类工作总览

| 类别 | 月度耗时 | AI 可提效 | 人工保留 | 提效比例 |
|------|---------|---------|---------|---------|
| A. 日常工作（每天循环） | ~92h | ~44h | ~48h | **48%** |
| B. 周期性工作 | ~38h | ~19.5h | ~18.5h | **51%** |
| C. 战略性工作 | ~42h | ~11h | ~31h | **26%** |
| D. 支撑性工作 | ~24h | ~9h | ~15h | **38%** |
| **合计** | **~196h/月** | **~83.5h/月** | **~112.5h/月** | **43%** |

### 3.2 AI 能做的 vs 不能做的

| 维度 | AI 能做的 | AI 不能做的 |
|------|---------|-----------|
| 数据 | 采集和汇总（排名、KPI、站点分布） | 解读数据背后的原因 |
| 文档 | 初稿生成（邮件、报告、巡检记录） | 审核修改、判断分寸和语气 |
| 规则 | 执行（5 维打分、分量范围、风险信号） | 战略打分、动态调整策略 |
| 信息 | 监控和分类（群消息、异常推送） | 判断哪些是真问题、怎么解决 |
| 商务 | 准备材料（画像、数据包、谈判建议） | 现场谈判、拿捏力度、关系感知 |
| 决策 | 提供数据和建议 | 最终决策、战略方向 |

> **核心思路**：AI 做采集、汇总、初稿、执行规则；人做解读、判断、谈判、最终决策。人机分工清晰，不是替代而是升级。

## 四、全生命周期

供应商管理覆盖**引入 → 管理 → 清退**三个阶段的完整闭环，分量管理贯穿始终。

### 4.1 引入阶段

```
供应商提交资料 → 资料初评 → 商务交叉验证 → 准入评估报告 → 确认 → 建档入库 → 分初始量
```

**准入评估 9 维度**：过往业绩、同类型对标、管理方式、人员数量、公司背景、历史交付风格、配合商务、团队架构、资质合规。

**审批周期变化**：30 天 → 15 天（当前） → **5-7 天（AI 辅助后目标）**

**关键 Skill（5 个）**：

| Skill | 功能 |
|-------|------|
| `/供应商资料初评` | 9 维度自动打分，输出缺漏清单 + 建议补件 |
| `/商务交叉验证` | 查询资质/涉诉/关联方/异常名录 |
| `/评分体系完整性检查` | 检核评估维度覆盖度 |
| `/准入评估报告` | 整合初评+验证 → 准入/待考察/拒绝结论 |
| `/寻源匹配` | 基于需求+标签自动推荐候选 |

> 资料初评项目已有前期成果，评估标准和评分逻辑可直接复用。

### 4.2 管理阶段

核心是**日常监控 + 周期性考核 + 动态调整 + 合规管理**。

**日常监控维度**：

| 维度 | 数据来源 | 频率 | 对应 Skill |
|------|---------|------|-----------|
| 业绩表现 | 数仓（data-agent） | 日 | `/业绩异常诊断` `/日简报` |
| 配合表现 | 群聊 + 人工反馈 | 周 | 规则判定 |
| 人员变动 | 数据表 + 群聊 | 周 | `/人员异动跟踪` |
| 品质合规 | 质检系统 | 月 | 对接后自动化 |
| 月度考核 | 综合评分 | 月 | `/月度考核评分` |
| 合规问题 | 客诉 + 内部反馈 + 巡检 | 持续 | `/合规问题处理` |

**月度考核评分模型（5 维打分）**：

| 维度 | 权重 | 指标 |
|------|------|------|
| 业绩 | **30%** | 达标率、完成率、增长率 |
| 质检客诉 | **25%** | 质检通过率、客诉次数/率 |
| 上人 | **15%** | 人员到位率、出勤率、稳定性 |
| 配合 | **15%** | 响应时效、整改效率、商务支持 |
| 战略 | **15%** | 与业务方向匹配度、发展潜力 |

加权 → 月度总分 → ABCD 评级 → 红黄牌判定

**巡检管理**：出差巡检 12 项标准（空间/人员/薪资/招聘/品牌/管理/激励/作息/培训/招聘方式/品牌滥用/质检合规），覆盖出发前准备→现场检查→记录→报告→整改跟踪全流程。

### 4.3 清退阶段

```
红黄牌累积 / 触发清退条件 → 清退评估 → 切量方案 → 退出执行 → 档案归档 + 复盘
```

**关键 Skill（3 个）**：

| Skill | 功能 |
|-------|------|
| `/清退评估` | 综合考核/配合/品质等多维数据 → 清退建议 + 风险提示 |
| `/切量方案` | 计算清退后量怎么分、转移到哪些供应商 |
| `/清退复盘` | 原因分析 + 经验归档 + 规则优化建议 |

### 4.4 分量管理（贯穿全生命周期的核心动作）

**三种量类型**：

| 类型 | 场景 | 频率 |
|------|------|------|
| **增量** | 有新需求/新业务线，分配给谁 | 按需 |
| **减量** | 某供应商表现下滑，减少业务量 | 月度/按需 |
| **切量** | 清退/替换时，量从 A 转移到 B | 清退时 |

**分量算法**：5 维评分 → 按权重排序 → 参考产能饱和度 → 分量区间建议 → 你确认

## 五、核心亮点

### 亮点 1：分量算法（决策模型化）

将供应商分量从"凭感觉分"变为**5 维评分 + 权重排序 + 产能饱和度校准 + 输出建议区间**的标准化决策模型。既保证公平可追溯，又保留你最终拍板的灵活度。

### 亮点 2：规则引擎——用业务语言写规则

不需要代码，直接用日常用语定义触发条件和动作。例如：

> *"如果毅航连续 3 天达标率 < 80%，黄牌预警，推送到金条管理群 @ 所有人"*
> *"如果某供应商月度考核评分 < 70 分，自动进入观察期，下月分量减半"*

规则写在后台自动执行，不依赖手动调用；你遇到新情况直接加一条规则即可。

### 亮点 3：28 个 Skill 全覆盖

| 生命周期 | Skill 数量 | P1（核心） |
|---------|-----------|-----------|
| 引入阶段 | 5 | 3 |
| 管理-日常 | 5 | 3 |
| 管理-周期 | 6 | 3 |
| 管理-商务 | 2 | 0 |
| 清退阶段 | 3 | 0 |
| 战略 | 3 | 0 |
| 通用 | 4 | 1 |
| **合计** | **28** | **10** |

每个 Skill 是一个标准化工作流程，调用即可输出 —— 这是准确率从 21% 飙升到 95%+ 的核心杠杆。

### 亮点 4：数字人三种交互模式

| 模式 | 说明 | 场景示例 |
|------|------|---------|
| **被动响应** | 你随时提问，数字人检索知识库+实时数据回答 | "XX 供应商最近配合怎么样？" |
| **主动监控** | 定时巡检+异常告警+月报，推送到你和相关群 | 每日简报推送至管理群 |
| **群聊协作** | 监听群消息，提取关键信息入库；群内可 @ 查询 | "@数字人 毅航上月评分多少" |

## 六、角色转变

### 从操作员到规则定义者

| 维度 | 现状（你做的事） | AI 转型后（目标） |
|------|------------|---------------|
| 日常工作 | 47%（盯数据做表、追异常、手算考核） | **30%（-17%）** |
| 周期性工作 | 19%（准入/分量/诊断/巡检） | **15%（-4%）** |
| 战略性工作 | 22%（寻源/布局/联盟/深绑） | **38%（+16%）** |
| 支撑性工作 | 12%（制度维护/汇报/会议） | **17%（+5%）** |

### 释放的时间投向

| 投向 | 每周增加 | 价值 |
|------|---------|------|
| 供应商寻源与储备 | +5h | 人才储备是生态健康基础 |
| 站点合理布局 | +4h | 生态韧性、竞争优势 |
| BPO 联盟机制 | +4h | 从价格压降到价值共创 |
| 深度关系建设 | +4h | 跳过中间层获取真实信息 |
| 向上管理与资源争取 | +2.5h | 讲洞察 + 给方案 |
| 制度与体系化 | +1.5h | 从人治到机制 |

### 关键原则

1. **结构优先于数据量**：正确的结构（分层知识库 + 模块化 Skill）比海量数据有效得多
2. **Skill 是最大杠杆**：不加 Skills 准确率 21%，加了超过 95%
3. **系统会衰变**：一个月内从 95% 掉到 65%，维护是严肃的工程任务
4. **95% 够了**：追完美性价比太低
5. **最难的是口径统一**：口径问题必须先解决

## 七、实施路径（3 个 Phase）

### Phase 1 — 数据底座 + 固定档案 + 核心 Skill（0-1 个月）

**目标**：跑通金条 12 家的数据采集 + 固定档案 + 日简报 + 核心日常 Skill

| 关键产出 | 说明 |
|---------|------|
| 供应商档案库（库2） | 12 家金条供应商基础信息 + 老板画像 |
| 每日业绩自动化 | data-agent 搭建业绩数据看板 |
| 群消息采集通道 | joyclaw 接入关键群（内部 + 供应商） |
| 口径统一文档（库4） | 各相关方对齐口径定义 |
| 4 个核心 Skill 上线 | `/日简报` `/业绩异常诊断` `/邮件报告生成` `/合规问题处理` |

### Phase 2 — 规则引擎 + 管理动作 + 周期性 Skill（1-2 个月）

**目标**：规则引擎上线，月度考核 + 分量建议自动化，周期性工作 Skill 化

| 关键产出 | 说明 |
|---------|------|
| 规则引擎运行 | 告警/异常自动触发 |
| 月度考核自动 | 5 维评分模型上线 |
| 分量建议 | 分量算法部署 |
| 3 个周期性 Skill | `/整改跟踪` `/巡检记录` `/业绩诊断` |
| 知识库补全 | SOP 库 + 规则库 + 商务沟通库（库1+4+5） |

### Phase 3 — 数字人成型 + 引入/清退 + 战略 Skill（2-3 个月）

**目标**：全技能就绪，数字人全模式运行，生命周期闭环

| 关键产出 | 说明 |
|---------|------|
| 引入流程上线 | 4 个引入 Skill |
| 清退流程上线 | 3 个清退 Skill |
| 站点布局分析 | `/站点布局分析` Skill |
| 经验案例库完成（库3） | 历史案例 + 判断逻辑沉淀 |
| 数字人全模式运行 | 被动 + 主动 + 群聊全开 |
| 验证机制上线 | L4 验证层：离线评估 + 衰变监控 |
