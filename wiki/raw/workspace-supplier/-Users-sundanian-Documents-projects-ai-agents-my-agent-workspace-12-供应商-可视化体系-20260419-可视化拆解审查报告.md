## 二、核心问题详解

### 问题1: 容器自适应规范缺失 ❌ P0

**问题描述**:
V3规范明确要求"禁止边框穿过文字",但拆解文档中**80%的单元**没有明确说明如何处理容器自适应。

**影响单元示例**:

- **2.2 供应商对比矩阵**: 气泡大小、位置如何自适应?
- **2.3 排名×人效交叉矩阵**: 象限标签过长怎么办?
- **3.6 供应商分级标准**: 供应商名列表很长,如何处理溢出?
- **5.1 异常信号总表**: 28个信号怎么布局不重叠?

**V3规范要求**:
```css
/* 卡片容器必须包住内容 */
.card {
  width: max-content;           /* 宽度适应内容 */
  min-width: 100px;             /* 最小宽度 */
  padding: space-3;             /* 内边距 */
  box-sizing: border-box;       /* 包含边框 */
}

/* 防止文字溢出 */
.text {
  max-width: 200px;
  overflow: hidden;
  text-overflow: ellipsis;
  white-space: nowrap;
}
```

**改进建议**:
在每个可视化单元的"配色"章节后,增加"容器自适应"说明,明确:
1. 哪些容器需要自适应宽度/高度
2. 文字溢出处理方案(省略号/换行/滚动)
3. 最大尺寸约束

### 问题2: 配色过度使用,违反"少用绿色"原则 ⚠️ P1

**问题描述**:
V3规范要求"少用绿色",但拆解文档中**大量使用红绿黄**三色体系(🟢🟡🔴 emoji),导致视觉过于花哨。

**问题单元统计**:

| 分类 | 单元数 | 使用红绿黄配色 | 占比 |
|------|--------|---------------|------|
| 四大看板 | 20个 | 8个 | 40% |
| 战略维度 | 15个 | 6个 | 40% |
| 异常信号 | 12个 | 10个 | 83% |
| **合计** | | **24个** | **30%** |

**典型问题单元**:

**3.4 红绿灯月报** - 直接使用🟢🟡🔴:
```
🟢 绿灯（5家）：毅航、毛毛虫...
🟡 黄灯（2家）：翰锐、赛维斯...
🔴 红灯（2家）：华啸、岐力...
```
→ **违反**: "少用绿色"+"禁止超过3种语义色"

**5.4 严重程度分级** - 使用🔴🟡🟢:
```
🔴 严重（P0）
🟡 警告（P1）
🟢 关注（P2）
```
→ **违反**: 同上

**3.10 风险预警等级** - 使用🟢🟡🔴:
```
🟢 低风险（绿色）
🟡 中风险（黄色）
🔴 高风险（红色）
```
→ **违反**: 同上

**V3规范要求**:
> 配色优先级:
> 1. 优先语义色 - 需要表达状态时(问题/标杆/进行中)
> 2. 其次中性色 - 不需要强调的普通内容
> 3. 背景用次背景色 - 区分不同区域
>
> 禁止事项:
> - 禁止超过3种语义色
> - 能不用绿色就不用

**改进建议**:
**方案A - 简化双色体系**:
- 问题型: 琥珀橙 `#cd4f00`
- 标杆型: 深蓝 `#061b31` (不用绿)
- 普通型: 灰蓝 `#425466`

**方案B - 单色+强度体系**:
- 用**琥珀橙一种色**,通过透明度表达严重程度:
  - P0严重: `#cd4f00` (100%)
  - P1警告: `#cd4f00` (60%透明度)
  - P2关注: `#cd4f00` (30%透明度)

**推荐方案B**,更符合"金融级克制"原则。

### 问题3: 信息层级标注不完整 ⚠️ P1

**问题描述**:
V3规范定义了**9级信息层级**,但拆解文档中**约40%的单元**只写了"信息层级:1/2/3",没有明确标注具体的字号、字重、颜色。

**V3规范的9级信息层级**:

| 层级 | 字号 | 字重 | 颜色 | 用途 |
|------|------|------|------|------|
| H1 | 18px | 600 | `#061b31` | 页面主标题 |
| H2 | 16px | 600 | `#061b31` | 区域标题、象限名 |
| H3 | 14px | 500 | `#1a1f36` | 卡片标题、供应商名 |
| Body | 13px | 400 | `#1a1f36` | 正文内容 |
| Caption | 11px | 400 | `#425466` | 次要信息、注释 |
| Small | 10px | 400 | `#425466` | 图例、标签 |
| **数值** | 13px | 400/600 | 按语义 | monospace |
| **链接** | 13px | 400 | `#533afd` | 可点击 |
| **禁用** | 13px | 400 | `#94a3b8` | 不可用 |

**问题单元示例**:

**1.1 产能利用率双维度公式**:
```
**信息层级**:
1. 标题（H2, 16px, 600）
2. 公式（Body, 13px, monospace）
3. 评估标准（Caption, 11px）
```
→ ✅ **正确**: 明确标注了字号、字重

**2.2 供应商对比矩阵**:
```
**配色**:
- 标杆象限：淡绿 `#defff4` 透明度0.4
- 问题象限：淡橙 `#fff8ed` 透明度0.4
```
→ ❌ **缺失**: 没有说明信息层级(象限名用H2?供应商名用H3?数值用Body?)

**2.6 周度产能达成率对比**:
```
**配色**:
- 达标柱：深绿 `#0e8345`
- 未达标柱：琥珀橙 `#cd4f00`
```
→ ❌ **缺失**: 没有说明坐标轴、图例、标注的层级

**3.2 双指标卡片**:
```
**配色**:
- 优秀（≥85%）：绿色边框 `#10b981`
- 合格（70-85%）：蓝色边框 `#533afd`
```
→ ❌ **缺失**: 没有说明数值、标签、说明的层级

**改进建议**:
在每个单元的"信息层级"章节,必须明确标注:
1. **所有文字元素**的层级(H1/H2/H3/Body/Caption/Small/数值)
2. 每个层级对应的**字号、字重、颜色**
3. **视觉优先级**(哪个最重要,应该第一眼看到)

**标准格式**:
```
**信息层级**:
- H1标题: 18px, 600, #061b31 - 页面标题"供应商管理看板"
- H2标题: 16px, 600, #061b31 - 象限名"潜力型/标杆型"
- H3标签: 14px, 500, #1a1f36 - 供应商名"毅航"
- Body数值: 13px, 600, monospace, 按语义色 - "92%"
- Body正文: 13px, 400, #1a1f36 - 说明文字
- Caption说明: 11px, 400, #425466 - 辅助说明
- Small图例: 10px, 400, #425466 - 图例

**视觉优先级**: P0问题象限(橙) → P1标杆象限(绿) → P2普通象限(灰)
```

### 问题4: 视觉引导不清晰 ⚠️ P1

**问题描述**:
V3规范强调"用户扫描路径:**问题区域 → 标杆区域 → 普通区域**",但拆解文档中**约50%的单元**没有明确说明视觉引导逻辑。

**V3规范的视觉引导原则**:

| 优先级 | 视觉处理 | 目的 |
|--------|----------|------|
| **P0-问题** | 琥珀橙背景+边框 | 第一眼看到,需行动 |
| **P1-标杆** | 淡绿背景+绿框 | 正向确认,安心 |
| **P2-观察** | 灰蓝背景+灰框 | 背景信息 |

**问题单元示例**:

**2.2 供应商对比矩阵**:
```
**配色**:
- 标杆象限: 淡绿 `#defff4`
- 问题象限: 淡橙 `#fff8ed`
- 其他象限: 浅灰 `#fafbfc`
```
→ ✅ **有配色**,但❌ **缺少说明**: 用户第一眼应该看哪里?

**2.9 排名趋势图**:
```
**配色**:
- 标杆线(Top3): 深绿 `#0e8345`
- 中游线: 蓝色 `#533afd`
- 下滑线: 琥珀橙 `#cd4f00`
```
→ ✅ **正确**: 通过颜色突出异常(下滑)

**2.15 供应商健康度评分**:
```
**配色**:
(缺失)
```
→ ❌ **完全缺失**: 没有说明如何引导用户视线

**5.7 风险预警模板**:
```
**配色**:
- P0预警: 橙色背景 `#fff8ed`
- P1预警: 黄色背景 `#fef3c7`
- P2预警: 灰蓝背景 `#f7fafc`
```
→ ✅ **正确**: 用颜色区分严重程度

**改进建议**:
在每个单元增加"**视觉引导**"章节,明确说明:

1. **扫描路径**: 用户应该按什么顺序看?
2. **视觉焦点**: 哪个元素最重要?如何让它跳出来?
3. **对比关系**: 不同元素之间如何对比?(颜色/大小/位置)

**标准格式**:
```
**视觉引导**:
- 扫描路径: 问题象限(左下,橙色) → 标杆象限(右上,绿色) → 其他象限
- 视觉焦点: 华啸、岐力(橙色边框气泡,第一眼看到)
- 对比关系:
  - 象限对比: 背景色差异(橙>绿>灰)
  - 气泡对比: 大小=产能规模(大泡泡=大供应商)
  - 数值对比: monospace等宽数字,便于对比
```

### 问题5: 组件细节不一致 ℹ️ P2

**问题描述**:
部分单元的**组件尺寸、间距、圆角**等细节不一致,或没有明确说明。

**问题示例**:

**气泡大小**:
- 2.2矩阵: 气泡大小=产能规模,但没说具体尺寸
- 2.3矩阵: 没说气泡尺寸
- V3规范: 小48px、中60px、大72px

**圆角大小**:
- 大部分单元: 没说圆角大小
- V3规范: sm 4px、md 6px、lg 8px

**间距系统**:
- 部分单元: 没说用哪个间距
- V3规范: space-1(4px)到space-6(24px)

**改进建议**:
统一组件库,所有单元必须明确:
1. 容器圆角: 4px/6px/8px
2. 边框粗细: 1px(标准)/2px(强调)
3. 内边距: 12px(紧凑)/16px(标准)
4. 元素间距: 8px/12px/16px
