# 视觉原子规范 V3（Visual Atoms Specification）

> 克制、几何、高对比度的视觉语言
> 目标：消除噪音，凸显结构
>
> **核心改进**（V3 vs V1）：
> - 莫兰迪色系精简至3个核心色
> - 新增容器自适应规范（解决边框穿过文字问题）
> - 强化信息层级规范（7级层级系统）

---

## 一、设计哲学（不变）

### 核心原则

| 原则 | 含义 | 执行标准 |
|------|------|----------|
| **克制** | 拒绝装饰，每个元素必须有功能性 | 无阴影、无渐变（除语义渐变）、无圆角滥用 |
| **几何** | 统一使用圆角矩形作为基本容器 | Radius 4px-8px，保持稳重感 |
| **高对比** | 信息层级通过明度和饱和度区分 | 关键信息实心填充，背景信息低饱和 |
| **语义优先** | 颜色承载信息，而非装饰 | 每个色值对应明确的逻辑状态 |

### 视觉→逻辑的映射

```
视觉特征          →    逻辑含义
─────────────────────────────────────────
圆角矩形          →    容器/状态/数据块
细实线/虚线       →    流向/因果/顺序
细边框            →    边界/区分
填充色            →    数据性质/状态
渐变（特定场景）   →    衰减/截止/信息丢失
箭头              →    动作/方向/因果
菱形              →    判断/决策点
```

### 2.2 莫兰迪色系（精简至3个）—— 低饱和区分

> **V3精简理由**：原6个莫兰迪色太多，容易造成视觉混乱。精简为3个核心场景色。

| 名称 | 色值 | 用途 | CSS变量 |
|------|------|------|---------|
| **M-Gray** | `#8B9094` | 历史数据、已归档、过去时间 | `--m-gray` |
| **M-Beige** | `#C4BDB0` | 辅助信息、备注、次要说明 | `--m-beige` |
| **M-Sage** | `#A8B5A0` | 正向趋势、完成、通过状态 | `--m-sage` |

**使用原则**：
- 莫兰迪色仅用于"非核心但需区分"的信息
- 核心状态优先使用语义色（见2.3）
- 避免莫兰迪色与语义色同时出现在同一视觉层级

### 2.5 色彩使用优先级（新增）

当需要表达状态时，按以下优先级选择：

| 优先级 | 色系 | 适用场景 |
|--------|------|----------|
| 1 | 语义色（S-*） | 核心状态（成功/警告/危险/信息） |
| 2 | 中性色（N-*） | 无状态信息、容器背景 |
| 3 | 莫兰迪色（M-*） | 历史数据、辅助信息、次要状态 |
| 4 | 功能色（F-*） | 特定交互场景 |

**禁止**：同一视觉层级同时使用语义色和莫兰迪色

#### T-2 信息层级系统（V3新增）

> **核心原则**：通过字号、字重、颜色的三重差异，建立清晰的信息层级。

**层级对比示例**：

```
┌─────────────────────────────┐
│ 供应商管理看板               │  ← H1, 24px, N-08, 600
├─────────────────────────────┤
│ 产能利用率                   │  ← H3, 16px, N-07, 600
│ 本月熟手人均GMV ÷ 近6个月峰值  │  ← Body, 14px, N-06, 400
│                             │
│ 88%                         │  ← Data, 28px, N-08, 600
│ ↑ 较上月+5%                 │  ← Status, 12px, S-Success, 500
│                             │
│ 注：新人不计入计算           │  ← Caption, 12px, N-05, 400
└─────────────────────────────┘
```

**层级使用规则**：

1. 同一容器内不超过4个层级
2. 数据值（Data层级）必须是最显眼的
3. 辅助说明（Caption层级）必须是最弱的
4. 状态文字使用语义色，但字号保持Caption层级

## 四、容器自适应规范（V3新增）

> **问题**：容器边框穿过文字、内容溢出、布局错位
> **解决方案**：定义容器的"内容感知"和"自适应行为"

### 4.1 容器自适应原则

| 原则 | 说明 | 违反示例 |
|------|------|----------|
| **内边距优先** | 容器必须有足够的padding，内容不能紧贴边框 | 边框紧贴文字 |
| **宽度弹性** | 容器宽度应为`min-width`而非固定宽度，内容过长时自动扩展 | 内容被截断 |
| **高度自适应** | 容器高度由内容决定，不设固定高度 | 内容溢出容器 |
| **文字不换行限制** | 长文本（如数据字段）应在容器内换行，不溢出 | 文字穿出边框 |
| **最小可读宽度** | 容器最小宽度不小于240px，确保内容可读 | 过窄导致换行混乱 |

### 4.2 容器内容感知规则

#### 规则1：边框与内容的距离

```
❌ 错误：
┌─────────────────┐
│数据: 128        │  ← 文字紧贴左边框
└─────────────────┘

✅ 正确：
┌─────────────────┐
│ 数据: 128       │  ← padding-left: 16px
└─────────────────┘
```

#### 规则2：标签-值对齐

```
┌─────────────────────┐
│ 标签        │ 128   │  ← 标签左对齐，值右对齐
├─────────────────────┤     中间用分隔线或空格
│ 峰值比      │ 88%   │
│ 头部差距    │ 25%   │
└─────────────────────┘
```

#### 规则3：内容换行处理

```
┌─────────────────────────────────┐
│ 这是一个很长的标签名称，应该    │  ← 自动换行
│ 在容器内正确换行显示            │
├─────────────────────────────────┤
│ 长数据值：12345678901234567890  │  ← 等宽字体，可横向滚动
└─────────────────────────────────┘
```

#### 规则4：嵌套容器的边距

```
┌─────────────────────────────────┐
│ 外层容器                         │
│ ┌───────────────────────────┐   │  ← 内层padding与外层padding叠加
│ │  内层容器                 │   │
│ │  ┌─────────────────┐      │   │  ← 最内层padding确保内容不贴边
│ │  │  数据: 128      │      │   │
│ │  └─────────────────┘      │   │
│ └───────────────────────────┘   │
└─────────────────────────────────┘
```

### 4.3 自适应容器类型（V3新增）

#### AC-1 弹性宽度容器（Flexible Container）

| 属性 | 值 | 说明 |
|------|-----|------|
| min-width | 240px | 最小可读宽度 |
| max-width | 640px | 最大宽度，超过则换行 |
| width | auto | 由内容决定 |
| padding | 16px | 标准内边距 |

**适用**：数据卡片、信息面板

#### AC-2 固定比例容器（Aspect Ratio Container）

| 属性 | 值 | 说明 |
|------|-----|------|
| aspect-ratio | 16/9 或 4/3 | 保持固定宽高比 |
| width | 100% | 响应父容器 |
| object-fit | contain | 内容完整显示 |

**适用**：矩阵图、散点图

#### AC-3 滚动容器（Scrollable Container）

| 属性 | 值 | 说明 |
|------|-----|------|
| max-height | 400px | 最大高度 |
| overflow-y | auto | 超出时滚动 |
| padding | 16px | 标准内边距 |

**适用**：长列表、表格

### 4.4 文字与容器冲突检测（V3新增）

生成可视化时，自动检查：

| 检查项 | 检测方法 | 修复方法 |
|--------|---------|----------|
| 边框穿过文字 | 计算文字边界框与容器边框距离 | 增加padding |
| 文字溢出 | 文字宽度 > 容器宽度 | 设置`word-break: break-all` |
| 行高不足 | 文字行高 < 字号的1.4倍 | 设置`line-height: 1.4` |
| 标签值重叠 | 标签和值的宽度之和 > 容器宽度 | 使用分隔线或上下布局 |

## 六、语义渐变（Semantic Gradients）

### 6.1 信息衰减渐变

│████████░░░░░░░░░░░░░░░░░░░░░░░│  ← 从实到虚

background: linear-gradient(

  rgba(26, 115, 232, 0) 100%

background: linear-gradient(

## 七、组合模式（Patterns）

### 7.1 容量平衡模式（Capacity Pattern）

表示"在有限空间内管理无限输入"：

```
┌─────────────────────────────────┐
│ 总容量: 1M tokens               │
│ ┌─────────────────────────────┐ │
│ │████████████░░░░░░░░░░░░░░░░░│ │  ← 已占用 40%
│ │ 已用: 400K  │  剩余: 600K    │ │
│ └─────────────────────────────┘ │
│ ⚠ 建议压缩阈值: 800K (80%)      │
└─────────────────────────────────┘
```

### 7.2 生命周期模式（Lifecycle Pattern）

表示"输入→处理→结果"的流程：

```
输入数据      处理        结果
┌──────┐    ┌──────┐    ┌──────┐
│ 原始 │ →  │ 过滤 │ →  │ 输出 │
│ 数据 │    │ 清洗 │    │ 结果 │
└──────┘    └──────┘    └──────┘
                ↓
           ┌──────┐
           │ 丢弃 │
           │ 数据 │
           └──────┘
```

### 7.3 层级嵌套模式（Hierarchy Pattern）

表示"大容器套小容器"的抽象关系：

```
┌─────────────────────────────────┐
│ 主上下文                        │
│ ┌─────────────────────────────┐ │
│ │ 子代理 A                     │ │
│ │ ┌─────────┐  ┌─────────┐   │ │
│ │ │ 任务 A1 │  │ 任务 A2 │   │ │
│ │ └─────────┘  └─────────┘   │ │
│ └─────────────────────────────┘ │
│ ┌─────────────────────────────┐ │
│ │ 子代理 B                     │ │
│ └─────────────────────────────┘ │
└─────────────────────────────────┘
```

### 7.4 对比决策模式（Comparison Pattern）

并排对比两个选项：

```
        方案 A              vs           方案 B
┌─────────────────┐        ┌─────────────────┐
│ 优势            │        │ 优势            │
│ • 性能高        │        │ • 成本低        │
│ • 可扩展        │        │ • 部署快        │
├─────────────────┤        ├─────────────────┤
│ 劣势            │        │ 劣势            │
│ • 成本高        │        │ • 性能低        │
└─────────────────┘        └─────────────────┘
```

## 八、禁止事项（Anti-patterns）

| 禁止 | 原因 | 替代方案 |
|------|------|----------|
| 使用 emoji | 风格不统一，可读性差 | 使用统一图标组件 |
| 超过3种主色 | 视觉混乱，信息层级不清 | 使用语义色板 |
| 阴影效果 | 增加视觉噪音 | 使用边框和填充色 |
| 圆角超过8px | 过于柔和，失去稳重感 | 4-6px 圆角 |
| 虚线表示主流程 | 虚线表示可选/弱化 | 主流程用实线 |
| 渐变用于装饰 | 渐变有特定语义 | 仅用于衰减/截止场景 |
| 文字使用居中对齐（正文） | 阅读困难 | 正文左对齐 |
| 无意义的动画 | 分散注意力 | 静态展示，必要时用状态变化 |
| **容器边框穿过文字** | 信息无法阅读 | 设置足够的padding |
| **同一层级语义色+莫兰迪色混用** | 视觉混乱 | 遵循色彩优先级 |
