# 供应商管理可视化体系 — 内容清单与视觉规范

> 版本：v1.0
> 创建：2026-04-19
> 目标：52个可视化内容点，5层递进，逐项标注类型、层次、依赖、视觉规范
> 总原则：用最干净的几何语言，讲述关于"容量、精度、路径"的博弈故事

---

## 视觉原子规范

### 基础元素

| 元素 | 视觉特征 | 逻辑含义 |
|------|---------|---------|
| 容器 | 圆角矩形（rx=8px），细边框（1.5px），无阴影或极轻微 | 上下文、记忆、数据块、环境 |
| 填充 | 极低饱和度莫兰迪色系（灰、米、浅褐）或纯白 | 区分数据性质（历史 vs 当前） |
| 线条 | 细实线/细虚线（1.5px），箭头转角平滑 | 数据流向、因果关系、时间顺序 |
| 文本 | 无衬线字体，字号分层明确（13/11/10px三级） | 核心信息在容器内，注解在外部 |
| 遮罩/渐变 | 线性渐变（实色→透明） | 衰减、过期、截止、信息丢失 |

### 色彩系统

| 色彩 | 用途 | 50（填充） | 500（描边） | 800（标题） |
|------|------|-----------|-----------|-----------|
| Warm | 主色、核心数据 | #faf0e6 | #c26d3a | #6b3410 |
| Teal | 正面/增长/达标 | #e6f5f0 | #2e8b6e | #0f5040 |
| Coral | 负面/下降/预警 | #faeae5 | #c25030 | #6b2010 |
| Sage | 中性/参考 | #f0f2ec | #6f8660 | #3a4a30 |
| Stone | 背景/分割 | #f2f0eb | #857d74 | #454240 |
| Sky | 信息/标注 | #e8f1fa | #3a7ab8 | #0e4070 |
| Amber | 警告/注意 | #faf0dc | #b88018 | #6a4005 |

**规则**：单次可视化不超过3种色彩坡道。

### 四种逻辑模式

| 模式 | 表现形式 | 应用场景 |
|------|---------|---------|
| 容量逻辑 | 长矩形容器 + 内部填充块 | 满载、溢出、窗口限制 |
| 生命周期逻辑 | 从左到右水平排列 + 箭头连接 | 输入→处理→结果、失败→修正→成功 |
| 抽象/具象逻辑 | 大容器套小容器 / 细节 vs 摘要对比 | 压缩、汇总、层级包含 |
| 关系/依赖逻辑 | 平行排列 / 分支结构 | 分支任务、并行处理、父子关系 |

### 空间即时间

- **水平向右** → 时间线 / 处理先后
- **上下/层级** → 包含关系 / 抽象层级
- **单向箭头 →** → 任务分发 / 数据流向 / 因果触发
- **环形/回溯箭头 ↷** → 反馈 / 重置 / 回滚 / 循环
- **实线形状 → 虚线形状** → 从实数据到抽象总结/丢弃信息

## 第一层：概念框架（7项）

> 入门级，回答"看什么、为什么看"。所有后续层都建立在此之上。

### L1-01：产能利用率双维度公式

- **可视化类型**：公式卡片（并列对比）
- **依赖**：无（全局起点）
- **下游**：L3-02 四象限矩阵、L3-04 排名×人效趋势交叉、L4-04 人力结构修正
- **视觉规范**：
  - 左右两张公式卡片，圆角矩形容器
  - 左卡：自身峰值比 = 本月熟手人均GMV ÷ 近6个月熟手峰值
  - 右卡：头部差距 = 本月熟手人均GMV ÷ 同产线第1名熟手人均
  - 中间用细箭头连接，标注"同一分子，两个分母"
  - 卡片下方各附一个"回答的问题"注解（小字号，外部）

### L1-02：四大战略维度总览

- **可视化类型**：思维导图 / 总览图
- **依赖**：无（全局认知）
- **下游**：L1-03 指标落地矩阵、L5-10 数据源映射
- **视觉规范**：
  - 中心容器"供应商管理"，四个分支向外辐射
  - 合规红线（生存）→ Coral 强调
  - 运营效率（利润）→ Warm 强调
  - 组织治理（稳定性）→ Teal 强调
  - 风险控制（连续性）→ Amber 强调
  - 每个分支挂3个二级节点（一级指标名称）
  - 外部注解：不可替代性一句话

### L1-03：战略维度 × 指标落地矩阵

- **可视化类型**：矩阵表（4×N 网格）
- **依赖**：L1-02 四大战略维度
- **下游**：L2-02 频率受众矩阵、L4-14 异常信号总表
- **视觉规范**：
  - 四行（四大维度），每行内嵌多个小圆角矩形（一级+二级指标）
  - 列头：战略维度 | 一级指标 | 二级指标 | 数据源 | 判定标准
  - 判定标准用三色小标签：Teal=合格、Amber=预警、Coral=危险

### L1-04：指标提炼路径（三层翻译）

- **可视化类型**：流程图（三层递进）
- **依赖**：L1-02 四大战略维度
- **下游**：L4-01 单体深度看板
- **视觉规范**：
  - 三个容器从左到右排列，箭头连接
  - 层1（原始数据）→ 层2（结构化指标）→ 层3（叙事结论）
  - 每层内嵌一个示例小框（如"外网管控率60%→危险→叙事一句话"）
  - 使用渐变表达"信息浓缩"（从左到右容器渐小但内容渐精）

### L1-05：五大看板关系总览图

- **可视化类型**：流程图 / 架构图
- **依赖**：L1-01 双维度公式
- **下游**：L2-01 管理节律、L3-01 全景看板
- **视觉规范**：
  - 五个大容器（看板一至看板四 + 现场巡检），按使用频率排列
  - 看板一（月度）→ 发现异常 → 看板三（深度）
  - 看板二（周度）→ 发现异常 → 看板三（深度）
  - 看板四（新供应商专用）
  - 现场巡检（事件触发）→ 叙事输出
  - 箭头标注触发条件（"异常时"、"每月初"、"每周一"）
  - 右侧输出四个叙事模板卡片

### L1-06：FCI速记卡

- **可视化类型**：数据卡片
- **依赖**：L2-04 红黄绿巡检
- **下游**：L2-03 一页纸红绿灯月报、L5-03 数据质量分检测
- **视觉规范**：
  - 单卡片，圆角矩形，内分7行（站点/日期/总分/趋势/最低分项/红色项/根因一句话）
  - 示例数据填充（华啸站点示例）
  - 顶部标注"2分钟内填写"

### L1-07：双指标卡片

- **可视化类型**：对比卡片列表
- **依赖**：L1-01 双维度公式、L1-06 FCI速记卡
- **下游**：L2-03 一页纸红绿灯月报
- **视觉规范**：
  - 表格式卡片列表，列：供应商 | 业绩达成（🟢🟡🔴） | 趋势 | FCI能力（🟢🟡🔴） | 趋势 | 状态判断
  - 状态判断用8种类型的标签卡片（真标杆/透支型/高风险/潜力型/观察型/预警型/委屈型/危机型）
  - 底部附8种类型的判断规则矩阵

## 第二层：日常运营（8项）

> 上手级，回答"什么时候看、怎么看、填什么表"。依赖第一层的概念理解。

### L2-01：管理节律（周度+月度时间表）

- **可视化类型**：时间线 / 日历视图
- **依赖**：L1-05 五看板关系总览
- **下游**：L3-01 全景看板、L3-05 周度对标看板
- **视觉规范**：
  - 两条并行时间线（周度线 + 月度线）
  - 周度：周一（对标15min）→ 周二三（约谈30-60min）→ 周四五（整改评估20-30min）→ 周末（0min）
  - 月度：第1周（90min排名+通报）→ 第2周（60min约谈+配对）→ 月中-末（持续巡检+扶持）→ 月末（30min总结）
  - 每个时间节点是一个圆角矩形，内部标注动作+耗时
  - 两条线用虚线连接对应关系（周一对标 ←→ 第1周排名）

### L2-02：频率与受众矩阵

- **可视化类型**：矩阵表（5×5 网格）
- **依赖**：L1-03 指标落地矩阵
- **下游**：L3-01 全景看板、L3-05 周度对标看板
- **视觉规范**：
  - 行：日度/周度/月度/事件触发/季度
  - 列：叙事类型 | 产出物 | 受众 | 对标策略组
  - 每格用圆角小标签填充

### L2-03：一页纸红绿灯月报

- **可视化类型**：仪表盘 / 信号灯卡片
- **依赖**：L1-06 FCI速记卡、L1-07 双指标卡片
- **下游**：L4-14 异常信号总表（作为输出形式）
- **视觉规范**：
  - 手机屏幕比例的长卡片
  - 分三段：🔴 需要关注（N家）/ 🟡 需要跟踪（N家）/ 🟢 正常运行（N家）
  - 每家一行：供应商名 + FCI分数 + 趋势箭头 + 一句话根因 + 当前状态
  - 底部：一句话结论 + 建议动作 + 指标说明
  - 信号灯用 CSS 圆形色块（12px直径）

### L2-04：红黄绿巡检叙事联动规则

- **可视化类型**：状态流程图 / 信号灯
- **依赖**：L1-05 五看板关系总览
- **下游**：L4-14 异常信号总表、L4-15 问题→后果→应对闭环
- **视觉规范**：
  - 三个大容器垂直排列：合格（绿）/ 预警（黄）/ 危险（红）
  - 每个容器内嵌判定条件（圆角小标签）
  - 合格→无需动作（终点）
  - 预警→周度关注+下次复查（循环箭头）
  - 危险→立即约谈+整改（单向箭头指向约谈）
  - 右侧附巡检叙事模板卡片

### L2-05：周度监控扫描（3个突变信号）

- **可视化类型**：告警卡片
- **依赖**：L1-05 五看板关系总览
- **下游**：L4-14 异常信号总表、L4-15 问题→后果→应对闭环
- **视觉规范**：
  - 三张并列卡片：出勤异常 / 业绩断崖 / 流失加速
  - 每张卡片分三行：信号名称 | 异常标准 | 触发动作
  - 异常标准用 Coral 色标注（<85% / >20%降 / >5%离职）
  - 底部附"不需要做的事"清单（灰色，划掉样式）

### L2-06：供应商独立空间文件夹结构

- **可视化类型**：树形图 / 目录结构
- **依赖**：无（基础设施）
- **下游**：L2-07 填报模板、L5-10 数据源映射
- **视觉规范**：
  - 树状层级结构，根节点"[供应商名称]-工作空间"
  - 8个子节点（01-人力管理 → 08-通知与公告）
  - 必须填报的节点边框加粗（Warm色），自主使用的用虚线（Stone色）
  - 系统自动的节点填充Teal 50（表示无需人工）
  - 叶节点标注填报频率（周报/月报/事件触发）

### L2-07：填报模板（6个模板）

- **可视化类型**：卡片列表
- **依赖**：L2-06 文件夹结构
- **下游**：L5-10 数据源映射
- **视觉规范**：
  - 6张卡片垂直排列，每张代表一个模板
  - 卡片头部：模板编号 + 名称 + 填报频率（彩色标签）
  - 卡片内容：核心字段列表（小字号，圆角标签）
  - 底部标注截止时间（Amber 标签）+ 不填报后果（Coral 标签）

### L2-08：分步实施建议（3期路线图）

- **可视化类型**：甘特图 / 路线图
- **依赖**：L2-06 文件夹结构、L2-07 填报模板
- **下游**：无（终点交付物）
- **视觉规范**：
  - 三条水平泳道（第一期/第二期/第三期）
  - 每个泳道内嵌2-4个圆角矩形（要做的事项）
  - 泳道之间用箭头连接，标注"触发条件"
  - 第一期：骨架搭建（Warm 强调）
  - 第二期：合规+巡检（Sky 强调）
  - 第三期：完整版（Teal 强调）
  - 底部附核心原则卡片："不要一次性给10个表"

## 第三层：分析评估（10项）

> 进阶级，回答"怎么排名、怎么对标、怎么发现异常"。依赖前两层的概念和运营基础。

### L3-01：供应商全景看板（7个核心信息）

- **可视化类型**：仪表盘 / 卡片组
- **依赖**：L1-01 双维度公式、L1-05 五看板关系总览、L2-01 管理节律
- **下游**：L3-02 四象限矩阵
- **视觉规范**：
  - 7张信息卡片网格排列（2×4 布局，最后一张居中）
  - 每张卡片：信息名称 | 看什么 | 怎么判断 | 对应动作
  - 综合排名卡最大（主卡），其余围绕
  - 异常判断标准用颜色标签（<65% Coral / <85% Amber / >85% Teal）

### L3-02：供应商对比矩阵（四象限：总量×人均GMV）

- **可视化类型**：散点矩阵 / 四象限图
- **依赖**：L1-01 双维度公式、L3-01 全景看板
- **下游**：L3-03 矩阵使用流程、L4-01 单体深度看板
- **视觉规范**：
  - 经典四象限：X轴=总GMV（左低右高），Y轴=熟手人均GMV（下低上高）
  - 四个象限填充不同底色：右上 Teal 50（标杆型）、右下 Amber 50（臃肿型）、左上 Sky 50（潜力型）、左下 Coral 50（问题型）
  - 象限内嵌圆角标签：类型名称 + 含义 + 动作
  - 供应商用圆点标注，标注名称
  - 外部注解：为什么用总量×人均（三个问题）

### L3-03：矩阵使用流程（5步）

- **可视化类型**：流程图
- **依赖**：L3-02 四象限矩阵
- **下游**：L4-01 单体深度看板
- **视觉规范**：
  - 五个容器从左到右排列，箭头连接
  - 每步容器内：步骤编号 + 动作描述 + 判断条件
  - 第2步标出"异常落位"的4种情形（掉落到不同象限）
  - 第4步有分支箭头指向看板三（深度）
  - 第5步有4个分支输出（约谈/扶持/观察/清退）

### L3-04：排名×人效趋势交叉分析（四种类型）

- **可视化类型**：矩阵图（2×2）
- **依赖**：L1-01 双维度公式、L3-01 全景看板
- **下游**：L4-14 异常信号总表（隐性退步/隐性进步信号）
- **视觉规范**：
  - 2×2 矩阵：X轴=排名变化（升/降），Y轴=人均GMV趋势（升/降）
  - 四格：真进步（Teal）/ 真退步（Coral）/ 隐性退步（Amber，最危险）/ 隐性进步（Sky，最值得关注）
  - 每格内嵌：类型标签 + 含义 + 动作
  - 外部注解：为什么"隐性退步"最危险、"隐性进步"最值得奖励

### L3-05：周度对标看板（4个核心对比）

- **可视化类型**：对比仪表盘
- **依赖**：L2-01 管理节律、L2-05 周度监控扫描
- **下游**：L3-06 周度产能达成率示例
- **视觉规范**：
  - 四张对比卡片：周度GMV排名 / 周度产能达成率 / 周度转化率对比 / 周度出勤对比
  - 每张卡片分两栏：看什么 | 怎么判断 | 对应动作
  - 产能达成率卡最大，内嵌示例表格（供应商列表+达成率+状态）
  - 异常供应商用 Coral 标签标出

### L3-06：周度产能达成率示例表

- **可视化类型**：表格 + 柱状图
- **依赖**：L3-05 周度对标看板
- **下游**：无（具体数据展示）
- **视觉规范**：
  - 上方：横向柱状图，10家供应商，按达成率排列
  - 下方：数据表格，列：供应商 | 周目标 | 本周实际 | 达成率 | 状态
  - 达成率<75%的行用 Coral 背景高亮
  - 右侧附判断规则（<75%标出 / 连续2周<75%立即介入）

### L3-07：排名追踪5维度权重表

- **可视化类型**：雷达图 / 权重卡片
- **依赖**：L1-03 指标落地矩阵
- **下游**：L3-08 触发规则检查
- **视觉规范**：
  - 左侧雷达图：5个维度（业绩达成35%、合规20%、配合度20%、组织健康15%、成长性10%）
  - 右侧5张权重卡片：维度 | 数据项 | 数据来源 | 权重
  - 权重用百分比标签，占比越大卡片越宽（容量逻辑）

### L3-08：触发规则检查（8种状态流转）

- **可视化类型**：状态机 / 流程图
- **依赖**：L3-07 排名追踪权重表
- **下游**：L4-07 约谈框架、L4-08 整改评估、L4-15 问题→后果→应对闭环
- **视觉规范**：
  - 状态机形式，从左到右排列8个状态容器
  - 月度正常（Teal，中心）→ 首次预警（Amber）→ 连续预警（Amber）→ 清退评估（Coral）
  - 回正（Teal，从任意状态回流）
  - A级骤降（Coral，从月度正常直接跳转）
  - 诚信预警/红线/信任重建（独立分支）

- [ ] 内容完整，无遗漏
- [ ] 视觉规范统一（色彩/线条/容器风格一致）
- [ ] 逻辑流向正确（箭头方向/层级关系/因果关系）
- [ ] 与前后层的依赖关系正确
- [ ] 注解完整（外部说明文字不缺漏）

确认一层，进入下一层。
