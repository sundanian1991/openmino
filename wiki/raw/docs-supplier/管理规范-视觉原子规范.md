# 视觉原子规范（Visual Atoms Specification）

> 克制、几何、高对比度的视觉语言
> 目标：消除噪音，凸显结构

---

## 一、设计哲学

### 核心原则

| 原则 | 含义 | 执行标准 |
|------|------|----------|
| **克制** | 拒绝装饰，每个元素必须有功能性 | 无阴影、无渐变（除语义渐变）、无圆角滥用 |
| **几何** | 统一使用圆角矩形作为基本容器 | Radius 4px-8px，保持稳重感 |
| **高对比** | 信息层级通过明度和饱和度区分 | 关键信息实心填充，背景信息低饱和 |
| **语义优先** | 颜色承载信息，而非装饰 | 每个色值对应明确的逻辑状态 |

### 视觉→逻辑的映射

```
视觉特征          →    逻辑含义
─────────────────────────────────────────
圆角矩形          →    容器/状态/数据块
细实线/虚线       →    流向/因果/顺序
细边框            →    边界/区分
填充色            →    数据性质/状态
渐变（特定场景）   →    衰减/截止/信息丢失
箭头              →    动作/方向/因果
菱形              →    判断/决策点
```

```
│                         │  ← 圆角 6px
```

## 五、语义渐变（Semantic Gradients）

### 5.1 信息衰减渐变

表示信息从完整到丢失的过程：

```
┌────────────────────────────────┐
│████████░░░░░░░░░░░░░░░░░░░░░░░│  ← 从实到虚
└────────────────────────────────┘
```

CSS:
```css
background: linear-gradient(
  to right,
  rgba(26, 115, 232, 0.8) 0%,
  rgba(26, 115, 232, 0.4) 50%,
  rgba(26, 115, 232, 0) 100%
);
```

### 5.2 时间截止渐变

表示时间窗口或有效期：

```
现在                    截止
├────────────────────────┤

```

CSS:
```css
background: linear-gradient(
  to right,
  rgba(52, 168, 83, 0.6) 0%,
  rgba(52, 168, 83, 0.2) 80%,
  rgba(249, 171, 0, 0.3) 100%

```

## 六、组合模式（Patterns）

### 6.1 容量平衡模式（Capacity Pattern）

表示"在有限空间内管理无限输入"：

```
┌─────────────────────────────────┐
│ 总容量: 1M tokens               │
│ ┌─────────────────────────────┐ │
│ │████████████░░░░░░░░░░░░░░░░░│ │  ← 已占用 40%
│ │ 已用: 400K  │  剩余: 600K    │ │
│ └─────────────────────────────┘ │
│ ⚠ 建议压缩阈值: 800K (80%)      │
└─────────────────────────────────┘
```

### 6.2 生命周期模式（Lifecycle Pattern）

表示"输入→处理→结果"的流程：

```
输入数据      处理        结果
┌──────┐    ┌──────┐    ┌──────┐
│ 原始 │ →  │ 过滤 │ →  │ 输出 │
│ 数据 │    │ 清洗 │    │ 结果 │
└──────┘    └──────┘    └──────┘
                ↓
           ┌──────┐
           │ 丢弃 │
           │ 数据 │
           └──────┘
```

### 6.3 层级嵌套模式（Hierarchy Pattern）

表示"大容器套小容器"的抽象关系：

```
┌─────────────────────────────────┐
│ 主上下文                        │
│ ┌─────────────────────────────┐ │
│ │ 子代理 A                     │ │
│ │ ┌─────────┐  ┌─────────┐   │ │
│ │ │ 任务 A1 │  │ 任务 A2 │   │ │
│ │ └─────────┘  └─────────┘   │ │
│ └─────────────────────────────┘ │
│ ┌─────────────────────────────┐ │
│ │ 子代理 B                     │ │
│ └─────────────────────────────┘ │
└─────────────────────────────────┘
```

### 6.4 对比决策模式（Comparison Pattern）

并排对比两个选项：

```
        方案 A              vs           方案 B
┌─────────────────┐        ┌─────────────────┐
│ 优势            │        │ 优势            │
│ • 性能高        │        │ • 成本低        │
│ • 可扩展        │        │ • 部署快        │
├─────────────────┤        ├─────────────────┤
```

## 八、禁止事项（Anti-patterns）

| 禁止 | 原因 | 替代方案 |
|------|------|----------|
| 使用 emoji | 风格不统一，可读性差 | 使用统一图标组件 |
| 超过3种主色 | 视觉混乱，信息层级不清 | 使用语义色板 |
| 阴影效果 | 增加视觉噪音 | 使用边框和填充色 |
| 圆角超过8px | 过于柔和，失去稳重感 | 4-6px 圆角 |
| 虚线表示主流程 | 虚线表示可选/弱化 | 主流程用实线 |
| 渐变用于装饰 | 渐变有特定语义 | 仅用于衰减/截止场景 |
| 文字使用居中对齐（正文） | 阅读困难 | 正文左对齐 |
| 无意义的动画 | 分散注意力 | 静态展示，必要时用状态变化 |
