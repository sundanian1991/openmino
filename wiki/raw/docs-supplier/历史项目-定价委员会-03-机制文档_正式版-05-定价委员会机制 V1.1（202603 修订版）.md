# 供应商定价调整委员会机制

**版本号**：V1.1（2026-03 修订版）
**生效日期**：2026-04-01
**解释权**：定价委员会秘书处
**起草人**：供应商管理岗

---

## 第一章 总则

### 1.1 目的与目标

为规范供应商定价调整流程，建立有约束的决策机制，特成立供应商定价调整委员会（以下简称"委员会"）。

**核心目标**：
- 从"事后救火"转向"事前防火"
- 保障供应商/坐席合理收益
- 维护电销生态健康稳定

### 1.2 适用范围

**本机制适用于成熟赛道的成熟类业务**。

**纳入机制的定价调整类型**：
- 单价下调（降本）
- 单价上调（激励）
- 结算规则变更（考核门槛、奖惩系数等）

**不纳入机制的场景**（另行处理，简化流程）：
- 首次定价（新供应商、新业务线）
- 测试型项目（试点期、验证期）
- 创新型业务（无历史数据参考）

### 1.3 成熟业务定义

**成熟赛道**：运营 6 个月以上、有稳定历史数据的业务线

**成熟类业务**：同时满足以下条件
| 条件 | 标准 |
|------|------|
| **供应商数量** | ≥3 家 |
| **坐席规模** | ≥500 人 |
| **月营收** | ≥500 万元 |
| **历史调整次数** | ≥2 次 |

**不满足条件的业务**：按"特殊调整"流程处理，简化评审（仅秘书长+C2 审批）。

### 1.4 参考原则

| 原则 | 说明 | 触发复审条件 |
|------|------|---------|
| **费效比偏离度** | 调整前需测算费效比，复盘时对比实际值 | 连续 3 次实际值超出预测值±15% |
| **职场生存线** | 坐席平均薪酬不得低于当地最低工资 2.5 倍 | 连续 2 次低于 |
| **调整频率** | 同一业务线，12 个月内调整不超过 3 次 | 超过即触发 |
| **调整幅度** | 单次调整幅度原则上不超过 15% | 超过 20% 即触发 |

**说明**：
- 参考原则非刚性约束，但违反会触发复审
- 特殊情况可在提案中说明理由，经 C2/C3 批准后执行

## 第二章 组织架构

### 2.1 委员会构成

| 角色 | 人员 | 主要职责 |
|------|------|----------|
| 秘书长 | 供应商管理岗（1 人） | 组织会议、收集材料、追踪落地、归档记录 |
| 委员 | 业务督导代表（1 人） | 坐席层面信息（收入增减对业务的影响） |
| | 量化代表（1 人） | 数据测算、ROI 分析 |
| | 服务组负责人 C3（1 人） | 业务运营、职场管理 |
| | 策略组负责人 C3（1 人） | 定价策略、规则设计 |
| 终审 | 数据科技业务部负责人 C2 | 每项调整的最终审批 |

### 2.2 职责分工

| 角色 | 职责 |
|------|------|
| 供应商管理岗（秘书长） | 组织会议、收集材料、初审提案、追踪落地、归档记录；收集供应商反馈 |
| 业务督导 | 代表坐席声音，提供坐席层面影响评估 |
| 量化代表 | 提供数据测算、ROI 分析、影响评估 |
| 服务组 C3 | 评估业务运营影响，协调职场安排 |
| 策略组 C3 | 提供定价策略建议，设计调整方案 |
| 终审（C2） | 每项调整的最终审批，掌握最终决策权 |

### 2.3 决策流程（核心变更）

**不再采用"5 人委员会表决"机制**，决策权集中在 C2/C3 层面。

**决策流程**：
```
秘书长提交初审材料
    ↓
C2 审阅
    ↓
┌─────────────────┐
│ C2 能否直接决策？│
└─────────────────┘
    ├─ 能 → C2 直接决策（通过/驳回）
    └─ 否 → 转 C3 协商
              ↓
        服务组 C3 + 策略组 C3 协商
              ↓
        ┌─────────────────┐
        │ C3 能否达成共识？│
        └─────────────────┘
            ├─ 能 → 回传 C2 确认（通常通过）
            └─ 否 → 回传 C2 裁决（附双方意见）
```

**说明**：
- C2 直接决策：常规调整、无明显争议
- C3 协商：C2 认为需要策略组和服务组先达成共识
- C3 达成共识：回传 C2 确认，通常为程序性审批
- C3 未达成共识：回传 C2 裁决，C2 单独决策（通过/驳回/要求补充材料）

**委员职责**：
- 业务督导、量化代表提供专业意见
- **不参与表决**，仅作为决策参考

**回避原则**：
- 委员与调整涉及供应商有关联的，应主动回避

## 第三章 定价调整流程

### 3.1 流程概览

```
阶段 1：提案（T-5 天）
  ↓
阶段 2：初审（T-2 天）
  ↓
阶段 3：C2/C3 决策（T-1 天）
  ↓
阶段 4：生效后宣贯
  ↓
阶段 5：复盘（+30 天）
  ↓
阶段 6：复审（触发式）
```

### 3.2 阶段 1：提案（T-5 天）

**提案人**：策略组或量化代表

**必需材料**：《定价调整提案》（包含调整原因、数据支撑、影响测算、配套措施汇总）

**提案材料结构**：
1. **调整概述**：调整类型、幅度、时间、影响范围
2. **调整原因**：市场变化、成本变动、业务需求等
3. **数据支撑**：量化分析、对比数据、趋势分析
4. **影响测算**：对供应商、坐席、业务的影响预测
5. **配套措施**：如有，说明利好绑定、过渡方案等

### 3.3 阶段 2：初审（T-2 天）

**审查人**：秘书长（初审）

**审查要点**：
- 材料完整性
- 参考原则符合性
- 收入底线
- 配套措施
- 供应商反馈

**协同审查**：
- 业务督导评估承受力
- 量化复核测算逻辑

**初审结果**：
- 材料齐全、符合参考原则 → 报 C2 决策
- 材料不全或明显违反参考原则 → 退回提案方补充

### 3.4 阶段 3：C2/C3 决策（T-1 天）

**决策流程**：
```
秘书长提交初审材料
    ↓
C2 审阅
    ↓
┌─────────────────┐
│ C2 能否直接决策？│
└─────────────────┘
    ├─ 能 → C2 直接决策（通过/驳回）
    └─ 否 → 转 C3 协商
              ↓
        服务组 C3 + 策略组 C3 协商
              ↓
        ┌─────────────────┐
        │ C3 能否达成共识？│
        └─────────────────┘
            ├─ 能 → 回传 C2 确认
            └─ 否 → 回传 C2 裁决（附双方意见）
```

**C2 直接决策**：
- 常规调整、无明显争议
- C2 直接审批：通过/驳回

**C3 协商**：
- C2 认为需要策略组和服务组先达成共识
- 转 C3 协商：服务组 C3 + 策略组 C3

**C3 达成共识**：
- 回传 C2 确认，通常为程序性审批

**C3 未达成共识**：
- 回传 C2 裁决，需附：
  - 服务组 C3 意见及理由
  - 策略组 C3 意见及理由
  - 秘书长汇总的争议焦点
- C2 单独裁决：通过/驳回/要求补充材料

### 3.5 阶段 4：生效后宣贯

**宣贯对象**：供应商管理层

**内容重点**：
- 供应商对其员工的同步情况
- 员工层面的反馈和意见
- 是否存在较大反弹或负面情绪
- 供应商的应对措施

**目的**：了解执行效果，为下次调整提供参考

### 3.6 阶段 5：复盘（生效后 30 天）

**输出**：《定价调整复盘报告》（包含执行效果、反馈汇总、改进建议）

**复盘重点**：
- 费效比实际值 vs 预测值
- 坐席平均薪酬是否低于底线
- 供应商反馈汇总

### 3.7 阶段 6：复审机制（触发式）

**触发条件**（满足任一即触发）：
1. 连续 3 次调整不符合参考原则（费效比偏离度、职场生存线、调整频率）
2. 单次调整幅度超过 20%
3. 供应商提出书面异议且被 2 名以上委员支持
4. 早期预警信号触发（见第五章）

**复审流程**：
1. 秘书长发起复审通知（1 个工作日内）
2. 7 个工作日内召开临时会议
3. 调整方需提供书面说明
4. C2/C3 决策（流程同 3.4）

**复审结果**：
- 暂停调整：恢复至调整前价格，直至复审通过
- 撤销调整：本次调整无效
- 通过：予以追认

## 第四章 会议机制

### 4.1 会议时间

**固定会议日**：每月最后一周（通常 27 号）

**会议时间**：14:00-15:00（1 小时）

### 4.2 参会人员

**必须参会**：
- 秘书长（供应商管理岗）
- 服务组 C3
- 策略组 C3

**按需参会**：
- 业务督导代表（涉及坐席收入时）
- 量化代表（涉及数据测算时）

### 4.3 会议材料

**会前 3 天提供**：
- 定价调整提案（完整版）
- 数据测算表
- 秘书长初审意见

**材料要求**：
- 简洁清晰，不超过 5 页
- 核心数据加粗标红
- 必须有"调整前后对比"表

### 4.4 会议议程（60 分钟）

| 环节 | 时间 | 内容 |
|------|------|------|
| 提案陈述 | 10 分钟 | 策略组/量化代表陈述 |
| 委员询问 | 20 分钟 | 委员提问、提案方回答 |
| C2/C3 决策 | 30 分钟 | C2 决策或 C3 协商 |

### 4.5 会议纪要

**纪要内容**：
- 提案概要
- 决策结果（C2 直接决策/C3 协商结果）
- 待办事项及责任人

**发送时间**：会后 24 小时内

**发送范围**：全体委员+C2

## 第五章 风险防控

### 5.1 早期预警信号

| 信号 | 预警线 | 响应 |
|------|-------|------|
| **供应商集中度反馈** | >50% 供应商集中反馈定价问题 | 秘书长组织专项调研 |
| **坐席收入偏离度** | >30% 坐席收入低于底线价格 | 暂停调整，重新测算 |
| **离职率异常** | >行业平均 2 倍 | 秘书长组织调研 |
| **投诉率上升** | >20% 环比 | 暂停，重新评估 |

**预警数据来源**：
- 供应商月度反馈（秘书长收集）
- 坐席收入报表（量化提供）
- 离职率报表（业务督导提供）

### 5.2 应急预案

**触发条件**：群体性事件、投诉率>50% 上升、离职率>20%

**响应**：24 小时内紧急会议→决策暂停/调整/继续

## 第六章 附则

**生效时间**：2026-04-01

**解释权**：定价委员会秘书处

**与 V1.0 的主要差异**：
| 章节 | V1.0 | V1.1 |
|------|------|------|
| 决策机制 | 5 人委员会表决（3/5 或 4/5 通过） | C2/C3 决策（不再表决） |
| 适用范围 | 所有业务 | 成熟赛道成熟业务 |
| 基本原则 | 红线 | 参考原则 |
| 复审机制 | 无 | 新增触发式复审 |
| 会议时间 | 每月第二周周三 | 每月最后一周（27 号） |
| 会议形式 | 全员 1 小时 | 核心人员参会，按需邀请 |
| 预警信号 | 4 项通用指标 | 优化为 4 项（更聚焦） |
