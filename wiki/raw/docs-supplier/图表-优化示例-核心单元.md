## 示例1: 2.2 供应商对比矩阵(四象限)

### 优化前的问题

❌ **配色过度**: 使用红绿黄三色(🟢🟡🔴)
❌ **容器自适应缺失**: 气泡位置、大小、文字溢出未说明
❌ **信息层级不明确**: 没有标注所有元素的层级
❌ **视觉引导不清晰**: 用户第一眼看哪里?

### 优化后

**可视化类型**: 四象限散点图

**内容结构**:
```
┌──────────────────────────────────────────┐
│ 供应商对比矩阵                            │
│ ──────────────────────────────────────   │
│                                          │
│  自身峰值比高                             │
│       ↑                                  │
│  ┌────┴────┐                             │
│  │潜力型   │标杆型│                      │
│  │        │      │                      │
│  │[广达]  │[毅航]│                      │
│  └─────────┘      │                      │
│  │问题型   │波动型│                      │
│  │        │      │                      │
│  │[华啸]  │[赛维斯]│                     │
│  └─────────┘                             │
│       ↓          ←     →                 │
│  自身峰值比低    头部差距大    头部差距小 │
│                                          │
│ 气泡大小 = 产能规模                       │
│ 气泡颜色 = 严重程度(琥珀橙强度)            │
└──────────────────────────────────────────┘
```

**配色**(方案B - 单色强度):
```
象限背景(按优先级):
- P0问题象限(左下): rgba(255,248,237,0.4)  - 琥珀橙40%
- P1标杆象限(右上): rgba(255,248,237,0.15) - 琥珀橙15%
- P2波动象限(右下): #f7fafc                 - 中性灰
- P2潜力象限(左上): #f7fafc                 - 中性灰

气泡(白底+边框):
- P0严重型: 白底 + #f0a940边框(2px)        - 强烈橙
- P1标杆型: 白底 + rgba(240,169,64,0.6)(1px) - 中度橙
- P2普通型: 白底 + #e3e8ee边框(1px)         - 灰蓝

文字: 按信息层级标注(见下)
```

**信息层级**(9级详细标注):
```
H1页面标题: 18px, 600, #061b31 - "供应商对比矩阵"
H2象限名: 16px, 600, #061b31 - "潜力型/标杆型/问题型/波动型"
H3供应商名: 14px, 500, #1a1f36 - "毅航/华啸/广达/赛维斯"
Caption象限说明: 11px, 400, #425466 - "自身好,距标杆有差距"
Body数值: 13px, 600, monospace, #cd4f00 - "92%/58%"
Small轴标签: 10px, 400, #425466 - "自身峰值比低/头部差距大"

视觉优先级:
1. P0问题象限(橙40%) → 第一眼看到
2. P1标杆象限(橙15%) → 正向确认
3. P2其他象限(灰) → 背景信息
```

**视觉引导**:
```
扫描路径: 问题象限(左下,橙40%) → 标杆象限(右上,橙15%) → 其他象限
视觉焦点: 华啸(问题型,橙色边框2px,第一眼看到)
对比关系:
  - 象限对比: 背景透明度差异(40% > 15% > 0%)
  - 气泡对比: 边框粗细(2px > 1px) + 大小(大72px/中60px/小48px)
  - 数值对比: monospace等宽数字,便于对比
```

**容器自适应**:
```
象限容器:
  - 宽度: 100%, max-width: 600px
  - 高度: aspect-ratio: 1/1
  - 内边距: space-4(16px)
  - 溢出: hidden(气泡超出象限时不显示)

气泡容器:
  - 宽度: max-content(自适应供应商名长度)
  - 最小宽度: 48px(小气泡)
  - 最大宽度: 100px(防止过长)
  - 文字溢出: text-overflow: ellipsis(显示省略号)

供应商名:
  - 最大宽度: 80px
  - 溢出处理: ellipsis(超过显示"毅...")
  - 完整名称: hover时显示tooltip

数值标签:
  - 宽度: max-content
  - 字体: monospace 13px
  - 字重: 600(加粗突出)
```

**组件细节**:
```
圆角:
  - 象限: 8px(lg)
  - 气泡: 50%(圆形)
  - 图例: 4px(sm)

边框:
  - 象限分割线: 1px, #e3e8ee
  - 气泡边框: 1px(普通) / 2px(P0强调)
  - 轴线: 1px, dashed, #e3e8ee(虚线)

间距:
  - 象限间距: 1px(网格线)
  - 气泡与象限边框: min(12px, 5%)(最小12px或象限宽度的5%)
  - 标题与象限: space-4(16px)
  - 图例与矩阵: space-3(12px)
```

**交互设计**(可选):
```
hover气泡:
  - 阴影: 0 4px 8px rgba(0,0,0,0.15)
  - transform: scale(1.05)(放大5%)
  - transition: all 0.2s ease

点击气泡:
  - 显示详细数据卡片
  - 高亮该供应商的其他数据点
```

## 示例2: 3.4 月度状态分类(原"红绿灯月报")

### 优化前的问题

❌ **配色过度**: 直接使用🟢🟡🔴 emoji
❌ **违反规范**: "少用绿色"+"禁止超过3种语义色"

### 优化后

**可视化类型**: 状态分类卡片

**内容结构**:
```
┌──────────────────────────────────────────┐
│ 月度状态分类                              │
│ ──────────────────────────────────────   │
│                                          │
│  ┌────────────────────────────────┐     │
│  │ 需行动（P0）                    │     │
│  │ ─────────────────────────      │     │
│  │ 华啸、岐力                      │     │
│  │ → 立即约谈,制定整改计划         │     │
│  └────────────────────────────────┘     │
│                                          │
│  ┌────────────────────────────────┐     │
│  │ 维持观察（P1-P2）              │     │
│  │ ─────────────────────────      │     │
│  │ 毅航、毛毛虫、广达、伽玛、赛维斯 │     │
│  │ → 常规关注,分享标杆经验         │     │
│  └────────────────────────────────┘     │
│                                          │
│  行动数: 2个                              │
└──────────────────────────────────────────┘
```

**配色**(方案B - 单色强度):
```
P0需行动卡片:
  - 背景: rgba(255,248,237,0.4)  - 琥珀橙40%
  - 边框: #f0a940(2px)           - 橙色强调
  - 标题: #cd4f00                - 橙色文字
  - 说明: #1a1f36                - 深蓝正文

P1-P2维持观察卡片:
  - 背景: #f7fafc                 - 中性灰
  - 边框: #e3e8ee(1px)           - 灰蓝
  - 标题: #061b31                - 深蓝
  - 说明: #425466                - 灰蓝

汇总标签:
  - 背景: transparent
  - 文字: #425466(11px, Caption)
```

**信息层级**:
```
H1标题: 18px, 600, #061b31 - "月度状态分类"
H2分类名: 16px, 600, 按语义色 - "需行动"(橙) / "维持观察"(蓝)
H3供应商: 14px, 500, #1a1f36 - "华啸、岐力"
Caption说明: 11px, 400, #425466 - "→ 立即约谈,制定整改计划"
Small汇总: 10px, 400, #425466 - "行动数: 2个"
```

**视觉引导**:
```
扫描路径: 需行动卡片(橙40%) → 维持观察卡片(灰)
视觉焦点: 华啸、岐力(P0,橙色边框2px)
对比关系:
  - 分类对比: 背景透明度(40% vs 0%)
  - 边框对比: 2px(P0强调) vs 1px(普通)
  - 文字对比: 橙色(P0) vs 深蓝(普通)
```

**容器自适应**:
```
分类卡片容器:
  - 宽度: 100%, max-width: 800px
  - 内边距: space-4(16px)
  - 圆角: 6px(md)
  - 边框: 按语义色

供应商列表:
  - 布局: flex, flex-wrap: wrap
  - 每行最多: 3个供应商
  - 间距: space-2(8px)
  - 超出行数: 自动换行

供应商名:
  - 宽度: max-content
  - 间距: space-2(8px)
  - 标点: 中文顿号"、"分隔
```

## 示例3: 5.7 风险预警模板

### 优化前的问题

❌ **配色过度**: 使用🔴🟡🟢三色
❌ **视觉引导不清晰**: 用户第一眼应该看到什么?

### 优化后

**可视化类型**: 预警卡片

**内容结构**:
```
┌──────────────────────────────────────────┐
│ 风险预警通知                              │
│ ──────────────────────────────────────   │
│                                          │
│  预警编号: W-2026-04-001                  │
│  预警级别: P0(严重) ████ 100%             │
│  生成时间: 2026-04-15 09:30               │
│                                          │
│  【预警对象】                              │
│  供应商: 华啸                             │
│  触发原因: 连续4月排名下滑,进入Bottom3     │
│                                          │
│  【关键数据】                              │
│  • 本月产能: 58%(↓8%)                    │
│  • 当前排名: #15(↓3)                      │
│  • 峰值比: 52%(历史最低)                  │
│                                          │
│  【风险分析】                              │
│  • 主观原因: 主管离职+核心员工流失         │
│  • 客观原因: 行业人力竞争加剧              │
│  • 趋势判断: 若无干预,可能持续下滑         │
│                                          │
│  【建议行动】                              │
│  1. 立即约谈(本周内)                      │
│  2. 制定整改计划(人力+培训)                │
│  3. 评估启动清退可能性(保留选项)           │
│                                          │
│  【责任人】年老师                          │
└──────────────────────────────────────────┘
```

**配色**(方案B - 单色强度):
```
P0预警卡片整体:
  - 背景: rgba(255,248,237,0.4)  - 琥珀橙40%
  - 边框: #f0a940(2px)           - 橙色强调
  - 标题: #cd4f00                - 橙色

P1预警(示例):
  - 背景: rgba(255,248,237,0.2)  - 琥珀橙20%
  - 边框: rgba(240,169,64,0.6)(1px)
  - 标题: rgba(205,79,0,0.7)

P2预警(示例):
  - 背景: rgba(255,248,237,0.1)  - 琥珀橙10%
  - 边框: rgba(240,169,64,0.3)(1px)
  - 标题: rgba(205,79,0,0.5)

严重程度指示条:
  - P0: ████ 100% (#f0a940)
  - P1: ██░░ 60%  (rgba(240,169,64,0.6))
  - P2: █░░░ 30%  (rgba(240,169,64,0.3))

内部卡片(数据/分析/行动):
  - 背景: rgba(255,255,255,0.6) - 半透明白
  - 边框: #e3e8ee(1px)
  - 标题: #061b31(H3)
  - 内容: #1a1f36(Body)
```

**信息层级**:
```
H1标题: 18px, 600, #cd4f00 - "风险预警通知"
H2预警级别: 16px, 600, #cd4f00 - "P0(严重)"
H3小标题: 14px, 600, #061b31 - "【预警对象】"等
Body正文: 13px, 400, #1a1f36 - "供应商: 华啸"
Body数值: 13px, 600, monospace, #cd4f00 - "58%/#15/52%"
Caption元数据: 11px, 400, #425466 - "预警编号: W-2026-04-001"
Small编号: 10px, 400, #425466 - "1. 立即约谈"
```

**视觉引导**:
```
扫描路径:
  1. 预警级别标签(P0,橙色) → 第一眼判断严重程度
  2. 关键数据(数值,橙色) → 理解具体问题
  3. 建议行动(编号列表) → 知道该做什么

视觉焦点:
  - 最强: 预警级别标签(P0背景+橙色文字)
  - 次强: 关键数据(monospace数值,橙色)
  - 辅助: 元数据编号(灰色,小字)

对比关系:
  - 严重程度: 指示条长度(100% / 60% / 30%)
  - 数据对比: ↓符号(红色)表示下降
  - 优先级: 编号列表(1/2/3)表示行动顺序
```

**容器自适应**:
```
预警卡片容器:
  - 宽度: 100%, max-width: 600px
  - 内边距: space-4(16px)
  - 圆角: 8px(lg)
  - 边框: 2px(P0强调)

内部卡片:
  - 宽度: 100%
  - 内边距: space-3(12px)
  - 圆角: 4px(sm)
  - 边框: 1px
  - 背景: rgba(255,255,255,0.6)

数值列:
  - 宽度: max-content
  - 对齐: text-align: right
  - 字体: monospace 13px

行动列表:
  - counter-increment: 自动编号
  - 间距: space-2(8px)
  - 缩进: space-3(12px)
```

## 优化模板(通用)

基于以上3个示例,所有可视化单元都应按以下结构优化:

### 必须包含的章节

```markdown
### X.X 单元名称

**可视化类型**: [类型]

**内容结构**:
[ASCII布局图]

**配色**(方案B - 单色强度):
[详细配色说明,包括背景/边框/文字]

**信息层级**(9级详细标注):
- H1: 18px, 600, #061b31 - "xxx"
- H2: 16px, 600, #061b31 - "xxx"
- H3: 14px, 500, #1a1f36 - "xxx"
- Body: 13px, 400, #1a1f36 - "xxx"
- Body数值: 13px, 600, monospace, 语义色 - "xxx"
- Caption: 11px, 400, #425466 - "xxx"
- Small: 10px, 400, #425466 - "xxx"

**视觉引导**:
- 扫描路径: xxx → xxx → xxx
- 视觉焦点: xxx(第一眼看到)
- 对比关系: xxx

**容器自适应**:
[详细说明宽度/溢出/自适应规则]

**组件细节**:
- 圆角: 4px/6px/8px
- 边框: 1px/2px
- 间距: space-2/3/4
```

### 配色规范速查

```markdown
P0严重(100%强度):
  - 背景: rgba(255,248,237,0.4)
  - 边框: #f0a940(2px)
  - 文字: #cd4f00

P1警告(60%强度):
  - 背景: rgba(255,248,237,0.2)
  - 边框: rgba(240,169,64,0.6)(1px)
  - 文字: rgba(205,79,0,0.7)

P2关注(30%强度):
  - 背景: rgba(255,248,237,0.1)
  - 边框: rgba(240,169,64,0.3)(1px)
  - 文字: rgba(205,79,0,0.5)

正常(0%强度):
  - 背景: #f7fafc
  - 边框: #e3e8ee(1px)
  - 文字: #425466
```
