# 供应商管理工作手册

> **基于 HRBP 方法论的系统化供应商管理框架**
> 版本：v1.0 | 日期：2026-04-09
> 方法论来源：HRBP（人力资源业务伙伴）+ 微软 HR 改革实践

---

## 总纲：为什么是 HRBP？

供应商管理的本质，和 HRBP 的本质是同构的：

| HRBP | 供应商管理 |
|------|-----------|
| 管理对象是"人"（员工/管理者） | 管理对象是"组织"（供应商团队） |
| 核心目标：让人才服务于业务战略 | 核心目标：让供应商产能服务于业务目标 |
| 手段：盘点、辅导、激励、淘汰 | 手段：评估、整改、分配、清退 |
| 挑战：懂业务但不越界、管人但不替人管 | 挑战：懂产线但不替产线管、管供应商但不替供应商管 |

**六模块全景**：

```
战略对齐 → 组织诊断 → 赋能提升 → 动态调度 → 梯队建设 → 数据驱动
  (懂业务)   (看真相)   (帮成长)   (调资源)   (建梯队)   (做决策)
```

## 模块一：战略对齐 — 从"管供应商"到"懂业务的供应商管理者"

### 1.1 HRBP 原理

HRBP 的第一铁律：**不懂业务，就别做 HRBP**。

HRBP 不是坐在办公室里做表格的人，而是要吃透业务的"打仗逻辑"——前台怎么打单、中台怎么支持、后台怎么交付。微软改革的第一刀就是"合并所有工程 HR 团队"，原因很直白：不懂工程师的 HR 做不出对工程师有用的决策。

### 1.2 映射到供应商管理

你现在面临的同样问题：如果不懂金条的外呼策略是什么、企金的转化链路怎么走、信用卡的合规红线在哪、财富的客户画像什么样，你怎么判断一个供应商"做得好不好"？只能看 KPI 数字——但数字只能告诉你"不好"，不能告诉你"为什么不好"。

### 1.3 业务理解框架

**四维业务理解矩阵**：

| 维度 | 要搞懂的问题 | 信息从哪里来 |
|------|-------------|------------|
| **产线策略** | 这个月产线目标是什么？主打什么产品？转化率要求多少？ | 产线负责人周会、策略组调度规则 |
| **产能需求** | 需要多少人？什么技能？什么时段？ | 历史数据、业务预测 |
| **质量红线** | 哪些行为绝对不能碰？投诉容忍度？ | 合规制度、质检标准 |
| **成本约束** | 单价多少？预算够不够？性价比最优解？ | 财务数据、合同条款 |

### 1.4 供应商定位矩阵

每个供应商在你的体系里应该有清晰的定位：

```
                战略价值（高）
                    │
        核心伙伴    │    战略储备
        （稳产出）  │   （有潜力但待验证）
    ────────────────┼────────────────
        普通产能    │    观察对象
        （可替代）  │   （高风险低价值）
                    │
                战略价值（低）
          ←───────┼───────→
         产能贡献低  产能贡献高
```

- **核心伙伴**（如毅航#1）：稳定产出，维护关系，给足安全感
- **战略储备**（如岐力从#12升到#4）：有潜力，给机会验证，但设观察期
- **普通产能**：完成任务即可，不额外投入管理精力
- **观察对象**：重点关注，连续不达标则启动退出

### 1.5 与产线负责人的协同机制

**关键原则**：你不是产线的"传声筒"，是"翻译器"。

> 产线说："这个月转化率不行。"
> 你翻译成供应商语言："A 供应商转化率低于均值 15%，根因是新员工占比 40%，需要 2 周培训窗口。"

**协同 checklist**：

- [ ] 每周和策略组对齐一次：本月调度规则有没有变化？
- [ ] 每月和产线负责人对齐一次：供应商表现的业务反馈？
- [ ] 重大调整（缩量/清退）前，先和军哥/易人确认方向

## 模块二：组织诊断 — 供应商九宫格与多维评估

### 2.1 HRBP 原理

人才盘点九宫格是 HRBP 的核心工具。逻辑很简单：一个人不能只看"绩效好不好"，还要看"潜力大不大"。高绩效 + 低潜力 = 老黄牛，要稳住；高绩效 + 高潜力 = 明星，要重用；低绩效 + 高潜力 = 璞玉，要培养。

### 2.2 供应商九宫格

把"人"换成"供应商"，完全同构：

```
              潜力/健康度（高）
                    │
         明星供应商  │   璞玉供应商
         (高产出高潜力) │  (低产出高潜力，待培养)
    ──────────────────┼────────────────────
         老黄牛供应商  │   问题供应商
         (高产出低潜力) │  (低产出低潜力，待退出)
                    │
              潜力/健康度（低）
            ←───────┼───────→
          产出不佳    产出优秀
```

**两个维度的具体指标**：

**横轴：产出贡献（定量）**
- 成单量 / 转化率（核心指标）
- 产能利用率
- 合规率（一票否决项）

**纵轴：组织健康度（定性 + 定量）**
- 人员稳定性（流失率）
- 管理层能力（现场管理水平、会议质量）
- 投入意愿（是否主动改善、响应速度）
- 战略配合度（是否配合新业务试点）

### 2.3 KPI 不达标诊断树

当供应商 KPI 不达标时，用这棵树追根因：

```
KPI 不达标
  ├── 是共性问题还是个性问题？
  │    ├── 共性（所有供应商都差）→ 产线策略/线索质量问题 → 找策略组
  │    └── 个性（只有这家差）→ 继续向下诊断
  │
  ├── 个性问题：哪个维度出了问题？
  │    ├── 人的问题 → 新人占比高？核心员工流失？培训不到位？
  │    ├── 管理的问题 → 现场管理混乱？目标不清晰？激励无效？
  │    ├── 资源的问题 → 场地不够？设备不足？系统不稳定？
  │    └── 意愿的问题 → 投入收缩？重心转移？博弈行为？
  │
  └── 根因确认后：进入模块三（GROW 整改沟通）
```

**关键判断标准**（来自你的实践经验）：

| 根因类型 | 判断信号 | 应对方向 |
|---------|---------|---------|
| 共性问题 | 多家供应商同时出现相似问题 | 不能归责供应商，需从上游解决 |
| 个性问题-能力 | 其他供应商能做好，这家做不好 | 辅导 + 整改窗口 |
| 个性问题-意愿 | 有能力但投入收缩、重心转移 | 缩量警告 + 培养替代 |
| 个性问题-管理 | 现场混乱、会议只是施压无辅导 | 管理赋能 + 现场督导 |

### 2.4 异常根因分析五问法

追根因不要只问"为什么"，要追问五层：

1. **What** — 数据上发生了什么？（转化率从 8% 降到 5%）
2. **When** — 从什么时候开始？（从第 3 周开始）
3. **Who** — 谁的问题？（新员工组 vs 老员工组）
4. **Why** — 为什么这个群体出了问题？（新员工培训没跟上，老员工心态波动）
5. **How** — 为什么管理动作没起作用？（主管只施压没辅导，新员工流失加剧）

**原则**：供应商告诉你的"原因"往往是表层归因（"市场不好"、"线索质量差"），你要做的是穿透到真正的根因。

## 模块三：赋能提升 — GROW 教练模型在供应商整改中的应用

### 3.1 HRBP 原理

GROW 是 HRBP 教练式辅导的标准模型：

- **G**oal（目标）— 你想要什么结果？
- **R**eality（现状）— 现在什么情况？
- **O**ptions（方案）— 有哪些路可以走？
- **W**ill（意愿）— 你确定要走哪条路？

**五个 KP 心法**（HRBP 的内功）：
1. KP1：维护自尊，加强自信
2. KP2：仔细聆听，善意回应
3. KP3：寻求帮助，鼓励参与
4. KP4：分享观点，传情达理
5. KP5：给予支持，鼓励承担

### 3.2 改造版 S-GROW 模型（供应商场景）

HRBP 的 GROW 是给"自己人"用的，供应商场景需要加一个 **S（Signal，信号）** 阶段——先亮牌，再对话。

```
S → G → R → O → W
信号   目标   现状   方案   意愿
```

**为什么加 S？** 对供应商不能一上来就问"你想要什么目标"，必须先明确：这是一个正式的整改沟通，不是闲聊。信号阶段的作用是定调子、给压力、但留出口。

### 3.3 S-GROW 全流程实操

#### S — Signal（信号定调）

**话术模板**：

> "今天找你来，是因为你们连续两个月排名靠后。业务那边也反馈了这个问题。我想听听你们的想法，一起看看下个月怎么改善。"

**四个要点**：
1. 陈述事实（连续两个月排名靠后）— 不是你的主观判断，是客观数据
2. 传递压力（业务那边也反馈了）— 不是你在为难他，是业务端的共识
3. 给出出口（一起看看怎么改善）— 不是来宣判的，是来解决问题的
4. 引导参与（想听你们的想法）— 把话语权先给他

**禁忌**：
- 不要上来就威胁（"再做不好就缩量"）— 先给信号，威胁是后手
- 不要替他归因（"你们就是管理不行"）— 让他自己说
- 不要一次性说完所有底牌（留缩量/清退作为后手）

#### G — Goal（目标设定）

**话术模板**：

> "下个月你们的目标是什么？具体数字是多少？"

**三个要点**（来自你的三要素方法）：
1. **目标必须具体到数字** — 不是"提升"，是"从第 8 名升到第 5 名"
2. **目标必须有共识** — 不是你说多少就多少，是他认可且承诺的数字
3. **目标必须 SMART** — 具体的、可衡量的、可达成的、相关的、有时限的

**供应商常见的模糊目标 vs 升级后**：

| 模糊目标 | 升级后的 SMART 目标 |
|---------|-------------------|
| "我们下个月努力提高转化率" | "下个月转化率从 5.2% 提升到 6.5%，排名进入前 6" |
| "我们会加强管理" | "每周一次全员质量复盘，主管每日巡检不少于 2 次" |
| "我们会减少投诉" | "投诉率从 0.8% 降到 0.5% 以下，零重大合规事件" |

**如果供应商不敢承诺具体数字？**

> "你不确定具体数字没关系，但你得告诉我一个方向——你觉得做到什么程度算改善？哪怕说个范围。"

#### R — Reality（现状诊断）

**话术模板**（你的引导式设问法）：

> "你业绩差，什么时候差的？谁差的？做了哪些改进？为什么没效果？是时间问题、力度不够、还是其他？"

**五个追问层次**：

| 层次 | 问题 | 目的 |
|------|------|------|
| 1. 事实层 | "具体哪个环节出了问题？" | 把笼统归因变成具体事实 |
| 2. 时间层 | "从什么时候开始？之前呢？" | 判断是偶发还是趋势 |
| 3. 人员层 | "新员工和老员工分别什么情况？" | 判断是能力问题还是管理问题 |
| 4. 行动层 | "你们已经做了哪些改进？" | 判断是意愿问题还是方法问题 |
| 5. 根因层 | "为什么这些改进没效果？" | 逼他深入到根因，不停留在表面 |

**KP 心法在 Reality 阶段的应用**：

- **KP2（仔细聆听）**：他说"线索质量不好"，不要立刻反驳。追问："能具体说说哪个质量的维度不好吗？接通率？意向度？还是其他？"
- **KP3（鼓励参与）**：不要自己下结论。"你觉得问题出在哪？"比"我觉得你们的问题是..."有效十倍
- **KP1（维护自尊）**：先肯定再质疑。"我看到你们上个月也做了调整，方向是对的。但效果没出来，你觉得卡在哪？"

#### O — Options（方案探索）

**话术模板**：

> "你觉得有哪些办法？我能提供什么支持？"

**引导供应商自己想方案，而不是你给方案**：

| 错误做法 | 正确做法 |
|---------|---------|
| "你们应该加强培训" | "你们觉得培训方面需要怎么调整？" |
| "你们得换掉这个主管" | "你觉得目前的管理配置够不够支撑目标？" |
| "你们要优化排班" | "如果要提升产能利用率，你们觉得排班上有什么空间？" |

**但有一种情况你需要给方案**：供应商确实没有思路。这时用"先给再问"（你的引导式设问）：

> "我看了你们的数据，有几个方向你可以考虑：第一，新员工占比 40%，如果能把培训周期从 2 周压缩到 1 周，影响会很大；第二，老员工的转化率其实不差，但接起率偏低，是不是激励有问题？你觉得这两个方向哪个更可行？或者你还有别的想法？"

**先给方向（2-3个），再问他的判断** — 不是替他做决定，是帮他打开思路。

#### W — Will（意愿确认）

**话术模板**（你的三要素收尾法）：

> "好，我们总结一下。这个月 KPI 是 XX，原因你也说了。下个月目标是 XX，具体举措是 XX。第一周做到什么程度？谁负责？"

**四个确认要素**：
1. **行动计划** — 具体做什么、谁做、什么时候做
2. **时间节点** — 第一周、第二周分别检查什么
3. **问责机制** — 谁对结果负责、怎么跟踪
4. **支持承诺** — 你能提供什么支持、他需要什么帮助

**收尾话术**：

> "行，目标、举措、时间节点都清楚了。我这边会帮你协调 XX（如有）。下周三我们过一次进度，如果有偏差及时调整。我相信你们能做出来。"

最后一句"我相信你们能做出来" — 这就是 KP5（给予支持，鼓励承担）。压力给到位了，信心也要给到位。

### 3.4 整改时间窗设计

GROW 沟通完了，不是结束，是开始。整改需要有时间窗：

| 阶段 | 时间 | 动作 | 检查点 |
|------|------|------|--------|
| **启动期** | 第 1 周 | 供应商提交整改方案，你确认 | 方案是否具体到人和时间？ |
| **执行期** | 第 2-3 周 | 供应商执行，你周度跟踪 | 进度是否符合方案节奏？ |
| **验证期** | 第 4 周 | 月末 KPI 验证 | 是否达成承诺目标？ |
| **决断期** | 次月初 | 根据结果决定下一步 | 达标 → 解除预警；不达标 → 进入黄/橙/红灯流程 |

### 3.5 五种常见供应商类型 × GROW 策略调整

不是所有供应商都用同一套 GROW，要根据类型调整：

| 供应商类型 | 特征 | G 调整 | R 调整 | O 调整 | W 调整 |
|-----------|------|--------|--------|--------|--------|
| **态度好能力差** | 配合度高，但确实做不好 | 目标要低一些 | 重点挖能力瓶颈 | 你多给方向 | 确认学习意愿 |
| **能力强态度差** | 做得好但不配合、博弈 | 目标不能低 | 重点挖真实原因 | 让他自己出方案 | 强调规则，减少协商 |
| **态度能力双差** | 既做不好也不配合 | 明确最后期限 | 快速判断是否值得救 | 给出最后选项 | 确认是否接受，不接受就启动退出 |
| **态度能力双好** | 偶尔失误的优质供应商 | 目标保持 | 帮他找偶发因素 | 一起优化 | 表达信任，不需要太多问责 |
| **表演型** | 说得好听、做得难看 | 目标必须量化 | 不看他说什么，看他做了什么 | 要求书面方案 | 所有承诺白纸黑字 |

## 模块四：动态调度 — 赛马机制与产能再分配

### 4.1 HRBP 原理

微软新设的 Workforce Acceleration 部门，核心逻辑是：**组织内部的人力需要动态再分配，而不是静态配置**。AI 替代了部分岗位，新业务产生了新需求，人要从旧岗位转到新岗位。

### 4.2 映射到供应商管理

你面临的局面一模一样：
- AI 外呼在替代人工 → 供应商的人力需求在变化
- 新产线/新产品在起来 → 需要匹配有能力的供应商
- 旧产线在缩量 → 需要有序引导产能转移

你的角色从"考核者"升级为"产能调度者"。

### 4.3 排名-分配联动模型

**核心原则**：排名决定分配，分配决定收入，收入决定投入，投入决定排名 — 形成闭环。

```
排名结果 → 份额调整 → 收入变化 → 投入意愿 → 下期排名
   ↑                                            │
   └────────────────────────────────────────────┘
```

**具体规则**：

| 排名区间 | 份额调整 | 新业务优先权 | 管理动作 |
|---------|---------|------------|---------|
| **前 20%** | +10%~20% | 优先分配 | 维护关系，了解战略诉求 |
| **中间 60%** | ±5% | 正常排队 | 关注波动，及时辅导 |
| **后 20%** | -10%~20% | 暂停新业务 | 启动整改沟通 |
| **连续 3 月底 20%** | 启动清退流程 | 暂停 | 有序替换 |

### 4.4 份额调整的五因素原则

排名是主维度，但不是唯一维度。调整份额时综合考虑五因素：

1. **业务业绩情况** — 绝对产出 vs 相对排名
2. **增长速度情况** — 趋势是向上还是向下
3. **工程配合度** — 是否配合新系统、新流程
4. **是否具有战略意义** — 是否有不可替代的能力
5. **是否有战略扶持** — 是否是重点培育的新供应商

**集中度控制**：不能让一家独大（单家份额不超过 30%-50%），保持竞争张力。

### 4.5 AI 替代下的产能迁移

**场景**：AI 外呼替代率从 30% 提升到 60%，供应商需要的人力减少。

**调度策略**：

| 步骤 | 动作 | 责任方 |
|------|------|--------|
| 1. 预判 | 根据 AI 替代计划，预判各产线人力需求变化 | 策略组 + 你 |
| 2. 沟通 | 提前 1-2 个月和供应商通气，给准备时间 | 你 |
| 3. 迁移 | 引导供应商将人力转移到 AI 不覆盖的产线/环节 | 你协调 |
| 4. 优化 | 对无法迁移的产能，有序缩量 | 你决策 |
| 5. 引入 | 新产线/新能力优先匹配有迁移意愿的供应商 | 你评估 |

**关键**：不是"突然通知你减量"，而是"提前沟通 + 给迁移路径 + 有序调整"。这既保护了供应商的利益，也保证了你的产能弹性。

### 4.6 缩量规则

缩量是最敏感的动作，必须有规则支撑，不是临时拍脑袋：

| 缩量类型 | 触发条件 | 缩量幅度 | 前置动作 |
|---------|---------|---------|---------|
| **排名缩量** | 连续 2 月排名后 20% | 10%-20% | 整改沟通 → 无改善 → 缩量 |
| **业务缩量** | 产线整体收缩 | 按比例分配 | 公平分摊，保优汰劣 |
| **风险缩量** | 合规风险、重大投诉 | 立即缩量或暂停 | 应急处理 |
| **战略缩量** | 培养新供应商，调整集中度 | 5%-10% | 提前沟通 |

**缩量沟通三步**：
1. 电话铺垫 — "最近排名情况你也看到了，业务压力比较大"
2. 告知决策依据 — "根据规则，连续两个月靠后需要调整份额"
3. 邮件通知 — 正式文件，留档

## 模块五：梯队建设 — 供应商三级梯队与继任计划

### 5.1 HRBP 原理

HRBP 的继任计划分三级：
- **READY-NOW**（即时可用）— 明天就能上岗
- **ONE-JOB-AWAY**（1-2 年培养）— 有基础，需要发展计划
- **TWO-JOB-AWAY**（2-3 年潜力）— 有潜质，需要职业路径规划

### 5.2 供应商三级梯队

完全同构：

```
第一梯队：在用供应商（READY-NOW）
  ├── 头部：稳定产出，不可替代性强
  ├── 腰部：稳定产出，有一定可替代性
  └── 尾部：产出波动，需要重点关注

第二梯队：在培供应商（ONE-JOB-AWAY）
  ├── 新准入但已试单，正在验证能力
  └── 腰部供应商中重点培养的"种子选手"

第三梯队：在选供应商（TWO-JOB-AWAY）
  ├── 储备池中的 22 家待验证供应商
  └── 市场上正在接触的潜在供应商
```

### 5.3 新供应商准入评估框架

**六维评估模型**：

| 维度 | 评估内容 | 权重 | 评估方式 |
|------|---------|------|---------|
| **产能能力** | 场地规模、人员招聘能力、设备条件 | 25% | 现场考察 |
| **管理能力** | 管理团队经验、现场管理水平、会议质量 | 20% | 现场旁听会议 |
| **合规能力** | 合规制度、质检体系、历史合规记录 | 20% | 文档审查 + 背调 |
| **技术能力** | 系统对接能力、数据分析能力 | 15% | 技术验证 |
| **财务状况** | 资金实力、经营稳定性 | 10% | 财务报表 |
| **战略匹配** | 业务方向匹配度、合作意愿 | 10% | 深度沟通 |

**现场考察重点**（你的经验）：

| 考察项 | 好信号 | 坏信号 |
|--------|--------|--------|
| 机房环境 | 恒温恒湿、独立监控、消防完备 | 环境差、安全隐患 |
| 会议质量 | 员工参与、问题定位准确、有辅导 | 单向施压、只喊口号 |
| 团队氛围 | 有士气、有互动 | 沉闷、恐惧驱动 |
| 管理精细度 | 有数据看板、有流程文档 | 凭感觉、凭经验 |

### 5.4 保护期规则

新供应商进入后有保护期，但保护期不是"免死金牌"：

| 保护期阶段 | 时长 | 规则 |
|-----------|------|------|
| **试单期** | 1-3 个月 | 给少量单量验证能力，不纳入正式排名 |
| **成长期** | 3-6 个月 | 逐步放量，纳入排名但给予一定容错 |
| **成熟期** | 6 个月后 | 正式纳入排名，与其他供应商公平竞争 |

### 5.5 淘汰标准与退出机制

**淘汰触发条件**：

| 级别 | 条件 | 动作 |
|------|------|------|
| 黄灯 | 1 个月不达标 | 预警 + 关注 |
| 橙灯 | 连续 2 个月末位 | 正式整改 + 缩量 |
| 红灯 | 连续 3 个月不达标 | 启动清退流程 |
| 一票否决 | 重大合规事件、数据安全事件 | 立即终止 |

**有序退出四步**（你的成功经验：4 家清退零纠纷）：
1. **缩量** — 逐步减少份额，不是一次性砍掉
2. **替代** — 同步引入替代供应商，保证产能不断
3. **过渡** — 给 1-2 个月过渡期，完成在途业务
4. **结算** — 按合同结算，不留尾巴

## 模块六：数据驱动 — 从报表到决策闭环

### 6.1 HRBP 原理

微软把 People Analytics 并入 Employee Experience，核心逻辑是：**数据本身不产生价值，数据驱动的行动才产生价值**。"实时洞察 → 快速行动"的闭环，避免数据和业务脱节。

### 6.2 映射到供应商管理

你每天/每周/每月看供应商的 KPI 报表，但如果报表只是"看完就完了"，那它就不是决策工具，是记录工具。真正的数据驱动是：

```
数据采集 → 异常识别 → 根因分析 → 沟通验证 → 整改跟踪 → 效果评估
   ↑                                              │
   └──────────────────────────────────────────────┘
```

### 6.3 监控指标体系

**三层指标架构**：

| 层级 | 指标 | 频率 | 阈值 | 触发动作 |
|------|------|------|------|---------|
| **结果层** | 成单量、转化率、投诉率 | 日/周/月 | 偏离均值 ±15% | 启动诊断 |
| **过程层** | 接起率、有效通话率、平均通话时长 | 日/周 | 偏离目标 ±10% | 现场验证 |
| **健康层** | 人员流失率、新员工占比、培训完成率 | 周/月 | 流失率 >15%/月 | 管理访谈 |

### 6.4 异常预警机制

**三级预警**：

| 级别 | 触发条件 | 响应时间 | 响应动作 |
|------|---------|---------|---------|
| **蓝色预警** | 单日指标偏离 >15% | 次日 | 数据确认，不急于行动（可能是偶发） |
| **黄色预警** | 连续 3 天或单周指标偏离 >10% | 3 天内 | 电话沟通供应商，了解情况 |
| **橙色预警** | 连续 2 周或单月指标偏离 >10% | 1 周内 | 启动 GROW 整改沟通 |
| **红色预警** | 连续 2 月不达标或出现重大合规事件 | 立即 | 启动缩量/清退流程 |

### 6.5 数据驱动沟通的四种场景

| 场景 | 数据信号 | 沟通方式 | 沟通重点 |
|------|---------|---------|---------|
| **日常跟踪** | 指标正常 | 周度简报 | "保持，有问题随时说" |
| **异常关注** | 单次波动 | 电话沟通 | "这个数据有点异常，什么情况？" |
| **趋势预警** | 连续下滑 | 正式会议 | "连续三周下滑，我们一起看一下" |
| **整改问责** | 承诺未达成 | 正式整改沟通 | "上个月承诺的目标是 XX，实际是 XX，差距在哪？" |

### 6.6 避免"数据陷阱"

**三个常见陷阱**：

1. **唯数据论** — 只看 KPI 数字，不看背后的故事。数据告诉你"不好"，但故事告诉你"为什么"。
2. **横向比较陷阱** — 不同供应商的基础不同（新人占比、场地条件），直接横向比较不公平。要在比较的同时考虑基线。
3. **短期主义** — 单月波动正常，要看趋势。但如果连续下滑趋势，必须果断介入。

## 模块七：矩阵视角 — 32家 × 17条业务线的产能管理

> 这个模块是你和单业务线管理者的核心区别。你不是管"一家供应商做得好不好"，而是管"32家供应商在17条业务线上怎么排兵布阵"。

### 7.1 为什么矩阵视角完全不同

单业务线管理者看的是：供应商A 排名多少、产能多少、达标没有。

你看的是：
- **金条这条线**，有6家在跑，产能够不够？集中度是不是太高（某家占了40%）？
- **某供应商**，同时跑了5条线，他是"全能型"还是"精力分散"？
- **新业务线起来了**，现有供应商里谁能接？需要新准入吗？
- **某条线要缩量**，那家供应商的其他线能不能补上？会不会整体流失？

**本质上，你在做的是"产能的二维优化"：横轴是供应商，纵轴是业务线，目标是让每个交叉格子里的产能匹配业务需求。**

### 7.2 业务线健康度评估

每条业务线应该有一个"健康分"，由三个维度组成：

| 维度 | 评估内容 | 健康信号 | 风险信号 |
|------|---------|---------|---------|
| **产能充足度** | 供应商数量 + 总产能 vs 业务需求 | ≥3家供应商，产能利用率70-85% | 只有1-2家，或产能利用率>95% |
| **集中度** | 最大供应商份额占比 | 最大一家 ≤35% | 最大一家 >50% |
| **备份深度** | 如果头部供应商退出，有没有替代者 | 储备池有2家以上可接 | 储备池没有匹配能力的供应商 |

**业务线分级**：

| 级别 | 条件 | 管理动作 |
|------|------|---------|
| **绿色（健康）** | 产能充足 + 集中度合理 + 有备份 | 常规管理，月度跟踪 |
| **黄色（关注）** | 上述任一维度有隐患 | 制定改善计划，引入新供应商或分散份额 |
| **红色（风险）** | 两个以上维度有问题 | 紧急引入替代供应商，制定应急预案 |

### 7.3 供应商跨线分析

每个供应商的"跨线特征"决定了你对他的管理策略：

| 跨线类型 | 特征 | 管理要点 | 风险 |
|---------|------|---------|------|
| **专精型**（1-2条线） | 深耕特定业务，专业度高 | 维护好关系，给稳定预期 | 该业务线如果缩量，他可能整体流失 |
| **多元型**（3-5条线） | 跨多条线，综合能力较强 | 关注精力分散问题，防止顾此失彼 | 管理复杂度上升，需要更强的现场管理 |
| **全能型**（6条线以上） | 跨线广，体量大 | 定期评估各线表现，防止"大而不能管" | 绑架风险、退出成本高、单点故障影响大 |

**"大而不能管"预警**：

当一个供应商同时满足以下条件时，启动风险预案：
- 跨 ≥5 条业务线
- 在任一业务线份额 >30%
- 总人数占你的供应商池 >15%

### 7.4 产能调度矩阵

你的核心调度能力体现在"跨线迁移"：

```
          业务线A（增量）    业务线B（稳定）    业务线C（缩量）
供应商甲     ★ 加派人力        维持             考虑迁移人力到A
供应商乙     新准入试单        维持             维持
供应商丙     观望             考虑加派          ★ 缩量，预警
```

**调度决策框架**：

| 场景 | 决策逻辑 | 动作 |
|------|---------|------|
| 新业务线启动 | 现有供应商能力匹配度评估 → 最优匹配 + 新准入备选 | 1-2家试单，3个月后评估放量 |
| 业务线缩量 | 影响供应商的总收入占比 → >30% 需要引导迁移 | 提前1-2个月沟通，给迁移路径 |
| 业务线增量 | 现有供应商产能天花板 → 不够则引入新供应商 | 先给现有供应商放量空间，不够再引新 |
| 供应商退出 | 影响业务线清单 → 逐线找替代 | 有序替换，不断产 |

### 7.5 矩阵月报 vs 单线月报的差异

| 维度 | 单线月报 | 矩阵月报 |
|------|---------|---------|
| **核心视角** | 供应商排名 | 产能矩阵健康度 |
| **数据重点** | 单线KPI | 跨线分布 + 集中度 |
| **风险关注** | 单个供应商不达标 | 业务线产能不足 / 供应商过度集中 |
| **调度动作** | 调份额 | 调份额 + 调业务线匹配 + 跨线迁移 |
| **梯队建设** | 供应商三级梯队 | 供应商梯队 × 业务线需求交叉匹配 |
| **给军哥的决策** | 某供应商怎么处理 | 整体产能格局怎么调、新业务线怎么布局 |

## 附录

### A. 内部协同沟通清单

| 协同对象 | 沟通内容 | 频率 | 目的 |
|---------|---------|------|------|
| **策略组** | 线索质量、价格基线、调度规则变化 | 每周 | 确保供应商分配规则合理 |
| **产线负责人** | 供应商业务表现反馈、新需求 | 每月 | 对齐业务期望 |
| **军哥（上级）** | 重大决策（清退/新准入）、风险预警 | 按需 | 获取授权和支持 |
| **质检培训** | 合规数据、培训需求 | 每月 | 供应商能力提升 |
| **财务/法务** | 合同审核、结算周期 | 按需 | 合规保障 |

### B. 外部供应商沟通矩阵

| 沟通对象 | 沟通内容 | 频率 | 沟通方式 |
|---------|---------|------|---------|
| **供应商老板** | 战略合作、重大问题、关系维护 | 季度 | 面谈/电话 |
| **供应商运营负责人** | KPI 沟通、整改跟踪、日常管理 | 周/月 | 电话/会议 |
| **供应商现场管理** | 具体问题、现场督导 | 按需 | 现场走访 |

### C. 沟通内外部六类场景

**内部沟通**：
1. 策略对齐（找策略组）
2. 业务反馈（找产线负责人）
3. 风险上报（找军哥/易人）
4. 合规协调（找质检）
5. 合同结算（找财务/法务）
6. 培训赋能（找培训运营）

**外部沟通**：
1. 战略对话（供应商老板）
2. 月度复盘（运营负责人）
3. 整改沟通（运营负责人）
4. 现场督导（现场管理）
5. 预警通知（运营负责人）
6. 关系维护（供应商老板）

### D. 三问决策原则

做任何供应商相关决策前，先问三个问题：

1. **有规可依吗？** — 是否已有制度/规则覆盖这种情况？有规依规。
2. **合理吗？** — 规则是否适用于当前情境？不合理则需要特批 + 记录。
3. **充分共识吗？** — 利益相关方是否达成一致？没有共识的决策执行成本极高。
