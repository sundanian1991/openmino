# 供应商管理可视化 - 核心单元精选（质量优先）

> 基于 infographic-svg 规范，精选核心单元，每个做到极致清晰
>
> 原则：不以量取胜，以质取胜

---

## 精选策略

### 为什么要精选？

**问题诊断**：
- ❌ 80个单元太多，**信息噪音大**
- ❌ 多个单元**功能重叠**（排名趋势×人效趋势×同比环比）
- ❌ 部分单元**使用频率低**（年度评估模板、季度评估模板、月度评估模板）
- ❌ 用户**记不住、找不到**需要的可视化

**精选原则**：
- ✅ 聚焦**高频使用场景**
- ✅ 聚焦**核心决策点**
- ✅ 每个单元**独特价值清晰**
- ✅ 逻辑链条**极其清晰**

## 精选后的30个核心单元

### 一、核心决策工具（11个）- 高频必用

#### 1.1 供应商对比矩阵（四象限）⭐⭐⭐⭐⭐
**核心价值**：发现异常的第一入口

**独特性**：
- 双维度评估，一眼看出哪些供应商有问题
- 气泡大小=产能规模，信息密度高
- 四象限分类，行动指南清晰

**逻辑链条**：
```
步骤1：查看矩阵 → 找到问题象限（左下）
步骤2：定位具体供应商 → 气泡标注（华啸、岐力）
步骤3：决定行动 → 约谈/分享标杆/观察
```

**视觉呈现**：
- 问题象限用焦橙突出（P0）
- 标杆象限用橄榄绿确认（P1）
- 其他象限暖灰背景（P2）
- 视觉引导：左下 → 右上 → 其他

---

#### 1.2 排名×人效交叉矩阵 ⭐⭐⭐⭐⭐
**核心价值**：识别隐性退步/进步

**独特性**：
- 发现"人效降但排名未变"的隐性退步
- 发现"人效升但排名未升"的隐性进步
- 避免被排名表象迷惑

**逻辑链条**：
```
维度1：排名变化 → 上升/下降/持平
维度2：人效变化 → 上升/下降/持平
交叉分析 → 四种状态：
  - 显性进步（人效↑+排名↑）→ 表扬
  - 隐性进步（人效↑+排名→）→ 关注
  - 隐性退步（人效↓+排名→）→ ⚠️ 警告
  - 显性退步（人效↓+排名↓）→ 立即行动
```

**视觉呈现**：
- 隐性退步用焦橙边框突出
- 显性进步用橄榄绿确认
- 箭头标注状态变化方向

---

#### 1.3 归因逻辑树 ⭐⭐⭐⭐⭐
**核心价值**：问题定位核心流程

**独特性**：
- 从"业绩异常"到"具体行动"的完整路径
- 逐层拆解，避免盲目行动
- 可复用的诊断框架

**逻辑链条**：
```
第一层：是否异常？
  → 否 → 正常通报
  
  → 是 → 第二层：哪个维度差？
      → 产能不足 → 加人/培训
      → 人效低 → 调整结构
      → 质量差 → 流程优化
      
第三层：制定行动计划
  → 约谈/整改/验证
```

**视觉呈现**：
- 问题分支用焦橙突出
- 解决方案用橄榄绿
- 箭头清晰，路径明确

#### 1.4 单体深度看板（7指标雷达）⭐⭐⭐⭐⭐
**核心价值**：单个供应商的全方位诊断

**独特性**：
- 7个维度同时展示（产能/人效/质量/成本/流失/增长/合规）
- 雷达图一眼看出优劣势
- 综合得分+薄弱项标注

**逻辑链条**：
```
步骤1：选择供应商 → 打开深度看板
步骤2：查看雷达图 → 识别形状（饱满=好，凹陷=差）
步骤3：定位薄弱项 → 成长78分（关注点）
步骤4：查看近6个月趋势 → 验证是否持续问题
步骤5：决定行动 → 分享标杆/整改/观察
```

**视觉呈现**：
- 雷达图填充色用暖棕#c98a6a（10%透明度）
- 薄弱项用焦橙标注
- 趋势图辅助判断

#### 1.5 FCI速记卡 + 双指标卡片 ⭐⭐⭐⭐⭐
**核心价值**：月度通报的核心工具

**独特性**：
- FCI是综合财务指标，比单一业绩更全面
- 双指标（业绩+FCI）避免"唯业绩论"
- 速记卡方便快速查阅

**逻辑链条**：
```
月度数据 → 计算FCI → 分级：
  - ≥90%：优秀（标杆型）
  - 80-90%：合格（正常型）
  - <80%：预警（问题型）
  
双指标展示：
  - 业绩好+FCI好 → 维持现状
  - 业绩好+FCI差 → 关注利润
  - 业绩差+FCI好 → 关注增长可持续性
  - 业绩差+FCI差 → 立即干预
```

**视觉呈现**：
- 两个指标卡片并排
- 进度条颜色表示状态
- FCI速记卡放在侧边，随时查阅

**独特性**：
- 对比上月vs本月，看出趋势

- 柱状图直观对比每周表现

**逻辑链条**：
```
每周查看：
步骤1：看本月柱 → 是否达标（绿色/焦橙）
步骤2：对比上月 → 上升/下降趋势？
步骤3：看累计进度 → 按当前速度，Q1能否达成？
步骤4：决定行动 → 加速/维持/调整
```

**视觉呈现**：
- 达标柱：橄榄绿
- 未达标柱：焦橙
- 目标线：虚线，100%位置
- 累计进度条在底部

#### 1.8 主管胜任评估仪表盘 ⭐⭐⭐⭐
**核心价值**：评估供应商管理能力

**独特性**：

#### 2.5 份额调整规则 + 流程 ⭐⭐⭐⭐
**核心价值**：动态优化资源配置

**独特性**：
- 份额不是固定的，而是动态调整
- 基于表现+产能的双重判断
- 公平性（做得好加份额，做得差减份额）

**逻辑链条**：
```
季度评估：
步骤1：评估表现 → 毅航优秀（↑12%），岐力下滑（↓15%）
步骤2：评估产能 → 毅航产能充足，岐力产能不足
步骤3：决策矩阵 → 毅航+份额↑3%，岐力-份额↓2%
步骤4：内部审批 → 军哥审批
步骤5：通知供应商 → 说明原因/生效时间
步骤6：执行调整 → 下月生效
```

**视觉呈现**：
- 决策矩阵（表现×产能）
- 调整前后对比
- 流程清晰标注

## 单元优化示例（极致清晰版）

### 示例：供应商对比矩阵

#### 设计目标
让用户**3秒内**看懂：
1. 哪些供应商有问题？
2. 问题严重程度如何？
3. 需要采取什么行动？

#### 逻辑梳理
```
第一层：快速扫描（1秒）
→ 视觉焦点：左下象限（焦橙背景，最醒目）
→ 第一眼看到：华啸、岐力有问题

第二层：深度理解（2秒）
→ 查看气泡位置：华啸在左下（问题型），毅航在右上（标杆型）
→ 查看气泡大小：华啸小气泡（产能小），毅航大气泡（产能大）
→ 得出结论：华啸问题严重+产能小，优先处理

第三层：行动决策（3秒）
→ 查看象限标注："问题型" → "立即约谈"
→ 确定行动：本周约谈华啸、岐力
```

#### 视觉呈现策略

**色彩层次**（3层清晰度）：
```
最醒目：焦橙20%背景（P0问题象限）
  → 第一眼就看到，无法忽视

次醒目：橄榄绿15%背景（P1标杆象限）
  → 正向确认，让人安心

最平淡：暖灰0%背景（P2普通象限）
  → 背景信息，不抢占注意力
```

**信息层级**（4层清晰度）：
```
第一层：H1标题（18px, bold, Georgia）
  → "供应商对比矩阵"

第二层：H2象限名（12px, 500, system-ui）
  → "问题型"、"标杆型"

第三层：H3供应商名（10.5px, 500, system-ui）
  → "华啸"、"毅航"

第四层：Caption数值（10.5px, 600, monospace, 暖棕）
  → "58%"、"92%"
```

**容器自适应**（防溢出）：
```
矩形容器：width: 100%, max-width: 500px, aspect-ratio: 1/1
象限：相对定位，各占50%
气泡：position: absolute
  - 文字容器：max-width: 80px, ellipsis溢出
  - 数值标签：monospace, 居中
```

#### 独特性强化

**与竞品对比**：
- ❌ 传统看板：用红绿黄三色，视觉噪音大
- ✅ 本设计：焦橙单色，克制有力

**与过去版本对比**：
- ❌ V1版本：用emoji🟢🟡🔴，不专业
- ❌ V2版本：用红绿黄配色，过于花哨
- ✅ 本版本：暖棕单色，学术+专业
