## 二、周报

> 周会1-2分钟汇报，聚焦商务进展+卡点+下周计划。

### 2.1 周报结构

| 部分 | 内容 | 时长 |
|------|------|------|
| **整体概况** | 本周S/A/B/C/D各级数量+环比变化 | 20秒 |
| **最值得关注的变化** | 上升最快/下降最多+根因 | 30秒 |
| **风险项** | 连续预警供应商+主管胜任≤2项 | 20秒 |
| **建议动作** | 具体到"对谁做什么+什么时间完成" | 20秒 |

### 2.2 叙事模板

**整体概况**：

> "本周S级X家、A级X家、B级X家、C级X家、D级X家。比上周：S+A增加X家，C+D减少X家。"

**最值得关注的变化**：

> "XX供应商排名上升X名，原因是XX。"
> "XX供应商排名下降X名，主要原因是XX。"

**风险项**：

> "连续预警：XX供应商（连续2月后30%），已要求提交整改计划。"
> "主管胜任≤2项：XX供应商，建议关注管理团队。"

**建议动作**：

> "下周建议：约谈XX供应商（X月X日前），跟进XX整改进度。"

### 2.3 常见认知误区

- 只列数据不写根因 → 需引用看板诊断的一句话说原因
- 建议笼统 → 需具体到动作、对象、时间
- 忽略层级失衡 → C级太多说明淘汰不够快，S级太少说明标杆不够

## 三、月报

> 月度运营汇报，包含四大战略维度分析。

### 3.1 月报结构

| 部分 | 内容 | 要点 |
|------|------|------|
| **整体格局** | S/A/B/C/D各级数量+占比+环比 | 层级是否失衡 |
| **四大战略维度** | 合规红线/运营效率/组织治理/风险控制 | 每维度核心指标+趋势 |
| **最值得关注的变化** | 上升最快+下降最多+根因 | 引用双维度/主管胜任数据 |
| **风险项** | 连续预警+主管胜任≤2项+A级骤降 | 具体到供应商+原因 |
| **新供应商** | 引入期/成长期是否达里程碑 | 6个月跟踪 |
| **建议动作** | 下月重点2-3家 | 具体到"对谁做什么+什么时间" |
| **整改闭环率** | 本月应完成X项，实际完成Y项，闭环率Z% | ≥70%合格 |

### 3.2 四大战略维度

| 维度 | 核心指标 | 数据来源 | 判定标准 |
|------|----------|----------|----------|
| **合规红线** | 质检A类次数、信息安全、持证率 | 质检月报 | A类=0达标，≥1红线 |
| **运营效率** | 熟手人均GMV、自身峰值比、头部差距 | 系统数据 | 峰值比>85%正常 |
| **组织治理** | 主管胜任5项、流失率、出勤稳定度、配合度 | 人力+沟通记录 | 胜任≥3项合格 |
| **风险控制** | FCI能力指数、巡检红黄绿、整改闭环率、商业健康 | 巡检+整改 | FCI≥75合格 |

### 3.3 指标三层翻译

> 不能只列数字，要翻译成叙事结论。

| 层级 | 内容 | 示例 |
|------|------|------|
| **原始数据** | 系统导出的数字 | "XX供应商出勤率87%" |
| **结构化指标** | 对标标准后的判断 | "出勤率低于SLA达标线(95%)，触及预警线(90%)" |
| **叙事结论** | 一句话说清楚 | "XX供应商出勤持续低迷，已影响产能交付" |

### 3.4 受众差异化

| 受众 | 关注重点 | 汇报频率 | 模板 |
|------|----------|----------|------|
| **YR（易人）** | 运营效率、日常异常、执行进展 | 周报+月报 | weekly-report + monthly-report |
| **HJ（军哥）** | 战略维度、风险控制、重大事件 | 月报+专项 | formal-report |

### 3.5 常见认知误区

- 只汇报业绩数据（策略组也能看）→ 需说能力数据（服务组独有价值）
- 指标不翻译成叙事 → 不能只列数字，要写"这说明..."
- 不分受众 → 周报给YR看效率，月报给HJ看战略

## 四、专项汇报

> 重大事件时的正式汇报。

### 4.1 触发条件

| 触发条件 | 说明 |
|----------|------|
| 清退评估 | 连续3月后30%或重大违规 |
| 诚信红线 | 数据造假≥2项 |
| 重大合规事件 | 信息安全、安全事故 |
| 重大业务调整 | 产线切换、名单大调整 |
| 新供应商转正/退出 | 6个月评估结论 |

### 4.2 专项汇报结构

| 部分 | 内容 |
|------|------|
| **事件概述** | 什么事件、涉及谁、影响范围 |
| **事实依据** | 数据+时间线+证据 |
| **分析判断** | 根因分析+影响评估 |
| **处理方案** | 建议方案+风险预案+审批需求 |
| **后续计划** | 跟踪安排+闭环时间 |

### 4.3 模板

| 模板用途 | 文件路径 |
|----------|----------|
| 正式汇报 | `official-doc/formal-report.md` — 向上专项汇报 |
| 内部请示 | `official-doc/internal-missing.md` — 决策请示 |

## 五、排名公告

> 月度排名结果的正式发布。

### 5.1 公告结构

| 部分 | 内容 |
|------|------|
| **公告目的** | 月度评估结果公示 |
| **评估维度** | 5维度权重说明（简要） |
| **排名结果** | S/A/B/C/D分布+各供应商排名 |
| **重点事件** | 预警/回正/新供应商等 |
| **下月重点** | 需关注的事项 |

### 5.2 审批链路

| 步骤 | 操作 | 审批人 |
|------|------|--------|
| 1 | 起草排名公告 | YL |
| 2 | 审核数据准确性 | YL |
| 3 | 审批公告内容 | YR → HJ |
| 4 | 发布给全体供应商 | YL |
