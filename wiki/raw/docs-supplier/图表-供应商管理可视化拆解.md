# 供应商管理可视化拆解

> 来源：供应商管理看板.md + 供应商管理工具箱.md
> 目标：识别所有可可视化内容，按视觉原子规范分类
> 视觉原子：容器、填充、线条、文本、遮罩/渐变

## 第一部分：核心计算公式可视化（8个单元）

### 1.1 产能利用率双维度计算流程

**类型**：生命周期逻辑（State → Action → State）

```
原始数据 → 熟手筛选 → 双维度计算 → 交叉判断 → 类型输出
```

| 元素 | 视觉处理 |
|------|---------|
| 输入容器 | 圆角矩形，浅灰填充 |
| 计算节点 | 圆角矩形，白色填充，细边框 |
| 判断菱形 | 菱形，浅蓝填充 |
| 输出类型 | 四种颜色区分（标杆/尽力/问题/波动） |
| 箭头 | 细实线，标注动作 |

**数据**：
- 维度1：自身峰值比 = 本月熟手人均GMV ÷ 近6个月最高熟手人均GMV
- 维度2：头部差距 = 本月熟手人均GMV ÷ 同产线第1名熟手人均GMV
- 交叉矩阵：2×2 = 4种类型

### 1.2 人力结构修正决策树

**类型**：层级逻辑（条件分支）

```
                  [新人占比>30%?]
                 /             \
               是               否
               ↓                ↓
    [总人数变化>20%?]      [正常计算]
       /        \
     是          否
     ↓            ↓
[双指标看]    [熟手人均GMV]
```

**可视化要点**：
- 条件判断用菱形
- 结果用圆角矩形
- 路径用箭头连接
- 异常路径用橙色高亮

### 1.3 自身峰值比 × 头部差距 交叉矩阵

**类型**：关系逻辑（2×2矩阵）

|  | 头部差距>80% | 头部差距<60% |
|--|-------------|-------------|
| **自身峰值比>85%** | 标杆型（绿） | 尽力型（蓝） |

**视觉处理**：

- 每个象限不同填充色

### 1.4 主管胜任评估仪表盘

**类型**：容量逻辑（5个容量条）

| 指标 | 当前值 | 达标线 | 状态 |
|------|--------|--------|------|
| 团队稳定度（流失率） | 12% | <8% | 红色 |
| 出勤稳定度 | 88% | >90% | 黄色 |
| 人效趋势 | 下降 | 不连续2月降 | 红色 |
| 响应质量 | 75% | >80% | 黄色 |
| 整改执行力 | 60% | >70% | 黄色 |

**视觉处理**：
- 5个水平进度条
- 达标线用虚线标记
- 当前值用填充块表示
- 颜色：绿/黄/红表示状态

**综合判定**：
- 5项达标 = 胜任（绿）
- 3-4项达标 = 需关注（黄）
- ≤2项达标 = 不胜任（红）

### 1.5 标杆分享配对流程

**类型**：生命周期逻辑（5步流程）

```
选标杆 → 找对标 → 配对条件筛选 → 提取做法 → 跟踪差距
  ↓        ↓           ↓              ↓           ↓
峰值比>85%  峰值比<70%  规模差距<30%   具体可量化   头部差距变化
头部差距>80% 同产线                可复制
```

**可视化**：水平时间线，5个节点，箭头连接

### 1.6 新人稀释场景计算示例

**类型**：容量逻辑（数值分解）

**赛维斯示例**：
```
总GMV 5200万
├── 熟手140人 × 37.1万 = 5194万（实际产能）
└── 新人50人 × 0.1万 = 5万（暂不计入）

对比计算：
├── 按总人数算：27.4万（误判为65%）
└── 按熟手算：37.1万（正确88%）

新人拉低幅度 = (37.1-27.4)/37.1 = 26%
```

**视觉**：堆叠柱状图，展示两种计算方式差异

### 1.7 甲方责任场景对照表

**类型**：关系逻辑（责任归属矩阵）

| 场景 | 你的动作 | 责任归属 | 追踪指标 |
|------|---------|---------|---------|
| 你要求加人 | 下达指令 | 甲方责任 | 新人拉低幅度 |
| 你加数据量 | 资源分配 | 甲方责任 | 总量vs人均 |
| 你调整产线 | 产线变更 | 甲方责任 | 头部差距变化 |
| 供应商自主扩编 | 无动作 | 供应商责任 | 正常评估 |

**视觉**：流程图，分支判断"是否甲方要求"

### 1.8 人力拆分结构图

**类型**：层级逻辑（树状分解）

```
总人数
├── 熟手（入职>30天）→ 产能计算分母
│   └── 熟手人均GMV = 总GMV ÷ 熟手人数
├── 新人（入职≤30天）→ 辅助参考
│   └── 新人拉低幅度 = (熟手人均-总人均) ÷ 熟手人均
├── 本月新入职 → 跟踪扩编
└── 本月离职 → 计算流失率
```

#### 2.3.5 归因判断逻辑树（核心）

**类型**：层级逻辑（决策树）

```

│   └── 主管≥4项达标 → 外部因素 → 看7指标
│       ├── 出勤降+人均不变 → 人力缺口 → 补充人力
│       ├── 出勤不变+人均降 → 管理松弛 → 约谈管理节奏
│       ├── 接通降+转化不变 → 名单问题 → 反馈策略组
│       ├── 接通不变+转化降 → 销售能力 → 培训/话术
│       └── 件均降 → 客户结构 → 大额能力
├── 自身峰值比正常 + 同产线降 → 产线问题 → 反馈策略组
└── 自身峰值比<65% + 同产线降 → 混合问题 → 综合处理
```

## 第三部分：战略维度可视化（15个单元）

### 3.1 四大战略维度总览

**类型**：关系逻辑（四象限价值矩阵）

| 维度 | 回答的问题 | 不可替代性 | 对标策略组 |
|------|-----------|-----------|-----------|

**视觉**：2×2矩阵，每个维度一个卡片，标注核心价值主张

### 3.3 指标提炼三层翻译

**类型**：生命周期逻辑（三层流程）

```
原始数据采集 → 结构化指标 → 叙事结论
     │              │             │
   外网20个      管控率60%      危险
   工位未管控    → 判定危险    → 技术基建缺失
```

**示例**：
1. 华啸现场巡检，20个工位外网未管控，总工位50个
2. 外网管控率 = (50-20)/50 = 60% → 危险（<70%）
3. "华啸合规风险：外网管控率60%，技术基础设施缺失"

## 第四部分：操作流程可视化（20个单元）

### 4.1 月度操作流程（月初第1周）

**类型**：生命周期逻辑（时序流程）

```
导出数据 → 更新排名 → 产能双维度 → 主管胜任
                                         ↓
四象限矩阵 ← 排名×人效交叉 ← 人力结构修正
     ↓
发现异常 → 打开单体看板 → 约谈/整改 → 输出叙事
```

**时间**：90分钟
**产出**：排名表 + 月度通报

---

### 4.2 月度触发规则检查表

**类型**：关系逻辑（条件→动作矩阵）

| 状态 | 触发条件 | 动作 | 完成标志 |
|------|---------|------|---------|
| 月度正常 | 排名非后30% | 无需动作 | — |
| 首次预警 | 本月后30%，上月不在 | 私下沟通 | 记录存档 |
| 连续预警 | 连续2月后30% | 正式约谈+整改计划 | 计划已收 |
| 清退评估 | 连续3月后30% | 评估留或退 | 结论已记 |
| 回正 | 连续2月后回到前70% | 取消预警 | 预警取消 |
| A级骤降 | 上月前30%本月后30% | 立即约谈 | 原因查明 |
| 诚信预警 | 数据质量≤70或1项不实 | 警告+加倍核查 | 警告已送 |
| 诚信红线 | ≥2项不实或同一项连续2次不实 | 约谈高层 | 记录存档 |

### 4.3 周度操作流程（每周一）

**类型**：生命周期逻辑（周度循环）

```

       ↓
周四-五：整改评估、HRBP沟通（20-30min）→ 跟踪进度
       ↓
```

### 4.4 约谈流程框架

**类型**：生命周期逻辑（4步流程）

```

     │                  │                    │              │
  排名变化            你觉得原因？          整改计划        日期
  GMV达成率           什么时候开始？        目标+动作        原因
  失分维度            内部有没有分析？      时间节点        整改时间
                     最难的是哪步？
```

### 4.5 整改计划评估标准

**类型**：关系逻辑（评估维度矩阵）

| 维度 | 合格标准 | 不合格示例 |
|------|---------|-----------|
| 目标具体 | "下月排名前50%" | "会努力的" |
| 动作具体 | 列出3-5个具体动作 | 只有笼统方向 |
| 时间节点 | 每个动作有完成时间 | 没有时间线 |
| 根因对应 | 动作针对提出的问题 | 动作和问题不对应 |

---

### 4.6 整改结果验证维度

**类型**：容量逻辑（4个验证条）

| 维度 | 验证方法 | 通过标准 |
|------|---------|---------|
| 目标达成 | 对比承诺目标 | ≥80%承诺值 |
| 动作执行 | 逐条核对 | ≥70%按期完成 |
| 持续性 | 观察整改后2月 | 指标未回落 |
| 根因消除 | 对比前后失分项 | 提升≥1等级 |

**结果处理**：
- 全部通过 → 取消预警，归档成功案例
- 部分通过 → 延长观察期1月
- 未通过 → 升级处理，连续2次→清退评估
- 造假嫌疑 → 直接清退评估

### 4.7 新供应商培育阶段流程

**类型**：生命周期逻辑（4阶段时间线）

| 阶段 | 时间 | 目标 | 动作 | 量化检查点 |
|------|------|------|------|-----------|
| 引入期 | 1-2月 | 站稳脚跟 | 开通权限、指定对接、确认名单 | 出勤≥85%、业务跑通 |
| 成长期 | 3-4月 | 业绩爬坡 | 双周复盘、对标中位 | GMV达中位60% |
| 稳定期 | 5-6月 | 达到中位 | 月度复盘、减少特殊照顾 | 排名前70% |
| 评判点 | 6月末 | 留或退 | 综合评估 | 前70%→正式纳入 |

## 第五部分：异常信号与应对可视化（12个单元）
