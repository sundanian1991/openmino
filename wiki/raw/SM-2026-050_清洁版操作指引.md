# **第一章 总则**

**第一条**为规范京东科技控股股份有限公司（以下简称"京东科技"）金融电销业务授权自采第三方供应商的全周期管理，确保在引入、合作、考核、退出等各环节的合规性、效率与风险可控性，特制定本细则。旨在落地为可执行、可追溯的标准化操作流程。

**第二条**
本规范适用于金融科技事业群-数据科技业务部所辖所有授权自采第三方金融电销业务供应商的合作管理，涵盖供应商全生命周期的所有关键节点。

**第三条 核心原则**

- **合规先行，风险严控**：所有操作必须符合内外部法律法规及公司制度，将风险防控置于首位。

- **过程留痕，权责清晰**：所有决策、沟通、审批环节必须在指定系统（京音平台/星海CRM/JoySpace/outlook邮箱/星空-渠道管理平台/XBP）中留下完整记录，确保可追溯。

- **决策拆分，分级授权**：重大决策需经多级审批，明确各岗位职责与权限，避免权力集中。

- **差异管理，效率优先**：根据业务规模（**核心业务** vs.
  **创新业务**）实施差异化管理，在合规前提下，为创新业务开辟快速通道。

- **数据驱动，闭环管理**：所有管理决策需基于业务数据、考核结果和风险评估，形成"计划-执行-检查-处理"的闭环。

# **第三章 业务分类与管理原则**

**第一条 业务分类**

- **核心业务**：指业务模式成熟、体量稳定、贡献主要的业务线。包括：

  - 京东金条

  - 企业金融

  - 京东联名卡

  - 财富管理

- **创新业务**：指处于试点、探索阶段，单次人力需求在10人（含）以内的快速上线测试业务。

**第二条 管理原则**

1.  **分类管控原则：**
    根据业务属性（核心业务/创新业务）与需求规模，采取差异化的管理策略与资源匹配方式。

2.  **效率与合规平衡原则：**
    在确保合规与风险可控的前提下，通过简化流程、快速匹配等方式，优先提升创新业务及小型需求的运营效率。

3.  **阈值触发原则：**
    以关键量化指标（如人力需求规模）为界，自动触发并适用不同的管理流程与审批权限。

4.  **充分竞争原则：**
    为保障供应商生态健康，核心业务及大规模需求引入时，必须执行强制性的多方比选机制。

5.  **全程留痕与可追溯原则：**
    即使在简化或豁免流程中，有关键决策的依据与过程，确保管理行为可追溯。

# **第四章 业务立项评估**

**第一条 目标**
为确保所有新金融电销业务在启动前经过充分的可行性评估，明确业务目标、资源需求及关键预期，避免承接信息不全、资源不匹配的业务，特设立本评估流程。旨在实现决策前置、信息对等，确保业务策略与运营执行能力的高效协同。

**第二条 适用范围**
本规范适用于所有**新增**的金融电销业务，包括但不限于由策略运营岗、产品部或其他业务部门发起的**电销业务**。所有此类业务在进入供应商引入流程前，必须完成立项评估并获得审批。

**第三条 管理导向与常规实践**

为保障业务健康、有序发展，电销服务组在业务承接与资源配置中，通常遵循以下管理导向：

- 业务启动前，须评估现有人力、管理能力及供应商资源的匹配度，确保资源支撑力与业务需求相适应；

- 业务发起方应于立项阶段提供完整、清晰的业务预期，特别是规模与目标，以便精准评估与合理规划；

- 业务目标一经立项明确，将作为后续资源调配、供应商管理及效果评估的重要基准，推动目标落地与过程可控。

特殊情况或紧急业务，可在履行必要审批程序后，结合实际情况灵活处理。

**第四条 立项评审与审批流程**

**执行步骤**：

**第一步：立项发起**

1.  **发起方**：策略运营岗。

2.  **触发条件**：有明确的新业务意向，需通过电销模式开展。

3.  **提交材料**：发起方需填写并提交《金融电销业务立项申请表》*，内容须包含不限于以下关键信息*：

    - **业务基本信息**：业务名称、业务类型（核心/创新）、产品类型、目标客群。

    - **业务目标与策略**：业务背景、预期商业价值、核心转化指标（如名单转化率、客单价等）。

    - **测试期规划**：明确测试期人力需求（如：5人测试）、测试周期、核心考核目标。

    - **转正后预期规模**：【必填项】明确转正后3-6个月的预期人力规模、月均业务量、预期产值。此为立项评估的核心依据。

    - **时间规划**：期望测试启动时间、期望转正时间。

    - **特殊资源需求**：是否需要新的系统支持、特殊的合规流程、定制化培训等。

**第二步：立项评审会**

1.  **会议机制**：为快速决策并充分对齐，设立立项评审会机制。

2.  **评审委员会**：由**电销服务组负责人**与**电销策略组负责人**共同组成，负责对项目是否立项进行最终裁定。

3.  **会议输入**：策略运营岗提交的《金融电销业务立项申请表》。

4.  **会议讨论核心**：

    - **电销服务组负责人**基于申请表，重点评估**人力匹配度、供应商匹配度、运营可行性及风险合规性**。

    - **电销策略组负责人**重点阐述**业务价值、市场机会及战略契合度**。

5.  **会议输出**：会议记录并形成《立项会议纪要》，明确评审结论：**"同意立项"**、**"有条件同意立项"**（需附带明确条件）或
    **"驳回立项"** 。

**第三步：立项审批**

1.  **发起审批**：评审会结论为"同意立项"或"有条件同意立项"后，由**策略运营岗**将《金融电销业务立项申请表》*及*《立项会议纪要》一同在京音平台/XBP发起审批。

2.  **审批路径**：

    - **策略运营岗发起 → 策略运营岗直属管理岗（C3）审批 →
      上一级管理岗（C2）终审。**

3.  **决策节点**：

    - **审批通过**：项目正式立项，《立项会议纪要》中确认的"转正后预期规模"等信息将作为后续供应商引入、分量扩项的硬性参考标准。

    - **审批驳回**：流程终止，由策略运营岗根据驳回意见调整业务方案或终止项目。

**第四步：结果执行与归档**

1.  立项审批通过后，所有相关文件（申请表、会议纪要）必须在京音平台/XBP归档。

2.  电销服务组根据立项结论和会议纪要中明确的条件，正式启动后续的供应商引入或匹配流程。

# **第五章 供应商全生命周期管理流程**

**第一条 供应商引入**

**所有新业务的供应商引入，必须以已通过审批的《业务立项评估审批》为前提。引入过程中的供应商招募、测试期考核等环节，均需对照立项时确定的关键信息（特别是转正后预期规模）进行管理，需求发起到寻源引入供应商上线，至少预留
2周时间，新增供应商则需预留 4周时间。**

**目标**：合规、高效地引入符合业务需求的优质供应商。

**核心原则**：公平透明、高效筛选、资质审核。

**（1） 供应商准入需求** 可在以下任一情形发生时发起：

1.  **业务拓展**：开展新业务类型，且现有供应商库中无具备相应服务经验的供应商。

2.  **体量扩容**：现有供应商库的承接能力无法满足业务量增长需求。

3.  **策略优化**：基于运营策略，需引入具备竞争优势或服务品质更优的新供应商。

4.  **现供异动**：现有供应商发生经营范围变更、服务质量下滑、重大违规或风险事件等情况。

5.  **指定合作**：在重要业务项目中，合作方或协同方明确指定特定供应商参与（非公开竞争性场景）。

6.  **汰换更替**：基于供应商赛马考核机制或常规优化需求进行的供应商淘汰与替换。

7.  **技术升级**：业务创新所需技术能力超出当前供应商库的技术储备。

8.  **合规调整**：新颁布的监管政策要求供应商具备特定新增资质。

9.  **成本优化**：需引入新供应商以实现运营成本的合理化降低。

**（2） 供应商招募与资质审核**

- **招募方式**：供应商管理岗应通过主动邀约（行业头部或优质供应商）、接收公开投标、组织招标会、内外部推荐等多元渠道寻找意向合作方，确保招募过程公平透明。

- **京东员工推荐**：

  1.  员工需将供应商详情通过邮件（邮箱：zhaoshang-bpo@jd.com）发送至供应商管理岗及业务运营岗，并抄送其直属管理岗及供应商管理岗直属管理岗。

  2.  若员工与推荐供应商存在利害关系，必须依据《京东集团反腐败条例》进行"潜在利益冲突申报"，邮件主题明确标注，正文说明情况，并发送至
      jiancha@jd.com，同时抄送其直属、间接管理层及CCO体系-监察部。

  3.  推荐员工须确保推荐、考察、评标过程公平公正，严禁泄露任何项目敏感信息及评选标准。

- **资质审核**：供应商管理岗严格审核入围供应商资质，确保合规性。审核资料详见《金融电销业务供应商入库审核资料清单》，资料须通过公司正式邮箱提交至邮箱：zhaoshang-bpo@jd.com。

**（3） 储备库活跃度要求**

- **业务能力要求**：

  - 公司总人力规模 ≥50人。

  - 具备健全的应急预案、灾备机制及合规的消防安全设施。

- **需披露与京东侧关联供应商关系，引入时签署承诺函。**

**第二条 试案与转正**

**目标**：通过实际业务测试，验证新增/扩项/复试供应商承接能力，确保转正供应商质量。

**核心原则**：目标导向、量化评估、择优转正。

    - **多家供应商测试**：依据统一标准量化排名，符合测试目标获得转正资格。

    - 供应商管理岗基于评估报告，若达标，则准备转正材料。

4.  **转正审批**：

    - **审批材料**：*《供应商转正申请》*、*《测试期考核指标确认函》*、*《供应商测试期综合评估报告》*、质检报告。

      - **新增业务**：供应商管理岗发起 → C3审批 → C2终审。

**第三条 分量与扩项**

**目标**：科学、合理地调整供应商业务分配，满足业务增长需求，同时维持生态平衡。

**管理原则**：绩效导向、按能分配、生态平衡。

**（1） 现有供应商业务线扩增规则**

- **适用前提**：现有供应商拟在其服务范围内跨核心产品线新增金融电销业务线。

- **准入条件**：寻源同时满足以下任意三项条件的存量供应商，有意愿参与新项目，可进行业务线扩增。

  1.  在现有合作金融电销项目中，该供应商投入的总运营人力规模持续不低于20人。

  2.  该供应商在金融电销业务综合评级中，最近1个月排名均位于前50%。

  3.  具有从事相关金融电销或相关关联业务的服务经验。

  4.  在金融电销业务中，最近一次巡检无红线问题。

- **审批流程**：

  1.  **审批流**：

      - **人力\>10人**：供应商管理岗在京音平台/XBP发起 → C3审批 →
        C2终审。

      - **人力≤10人**：供应商管理岗在京音平台/XBP发起 → C3审批（终审）。

  2.  若未能满足三项条件但确需特殊引入，必须提交申请，并获得其直属上级领导及上一级间接领导双重审批同意后，方可执行。邮件需充分说明特殊引入理由。

**（2） 供应商增减员分量管理**

**目标**：建立动态、高效的分量调整机制，确保业务分配与业务需求、供应商绩效及风险状况紧密联动，实现运营效率最大化与供应商生态的持续健康。

**供应商业务量管理应遵循以下基本原则：**

- **绩效优先**：在同等条件下，优先向绩效优异、产能稳定的供应商倾斜资源；

- **灵活应变**：根据业务发展与外部环境变化，动态调整资源分配策略；

- **风险可控**：所有调整须评估对业务稳定性与服务质量的影响，确保整体风险可控；

- **过程闭环**：调整决策应有理有据、审批完整、执行可追溯，确保管理过程完整闭环。

**a） 分量调整触发条件**

**分量增加触发条件**：

- **业务量增长**：现有供应商承接能力达到上限（集中度/短期招聘压力），需增加人力或引入新供应商以应对业务增量。

- **绩效激励**：供应商在赛马考核中表现优异，为激励其持续贡献，需增加其业务分配。

- **业务承接**：因其他供应商退出、清退或申请退项，需将其原承接业务量重新分配给现有供应商。

- **策略调整**：基于业务策略优化（如名单策略调整/测试、激励政策变更），需对现有业务量进行重新分配。

**分量减少触发条件**：

- **业务量收缩**：整体业务量下降，为避免资源浪费，需相应减少供应商分配的业务量。

- **绩效惩罚**：供应商连续考核不达标或排名末位，需依据考核规则进行减量处理。

- **风险管控**：供应商出现一般性违规或潜在风险，为控制风险蔓延，需暂时削减其业务量进行观察。

- **策略调整**：同上，业务策略调整可能导致部分供应商业务量减少。

**b） 分量审批流**

**分量调整阈值**：

- **单次分量调整幅度 ＜ 50%**：供应商管理岗提交分量方案发起 审批→ C3审批
  。

- **单次分量调整幅度 ≥ 50%**：在标准流程基础上，审批流需上升至C2终审。

- **单次分量调整幅度 ≥
  100%**（即业务量翻倍）：视为重大调整，启动专项汇报，说明分量计划，审批至
  C2。

**快速分量调整（≤10人）**：

- **适用范围**：所有业务类别（核心/创新）的单次分量调整涉及人力≤10人的场景。

- **处理时效**：由供应商管理岗主导，5个工作日内完成调整。

- **审批要求**：需向C3进行京ME备案，无需事前审批。

**执行与通知**：

向供应商发出《业务量调整通知函》，并抄送结算岗、业务运营岗、策略运营岗、质检培训岗。

**第四条 供应商清退**

**目标**：及时、有序地终止与不合格或高风险供应商的合作，最大限度降低业务损失和品牌风险。

**核心原则**：有据可依、风险隔离、平稳过渡。

**单业务线退出规则**

- **适用场景**：供应商同时承接多个业务线时，单条业务线触发退出不影响其他业务线合作。

- **退出标准（满足任一即触发）**：

  - **考核不达标**：该业务线连续考核周期内综合绩效未达基准线，或达到末位淘汰/减量标准。

  - **客诉违规红线**：违反金融电销客诉或品质管理细则中的强制性退出条款（如重大销售误导、系统性服务缺陷、和客户对骂造成影响）。

  - **业务调整终止**：因甲方战略调整或外部因素导致该业务线整体终止运营。

  - **合约到期未续**：合作协议履行期满且双方未达成续约意向。

  - **合规整改失效**：出现经营不合规行为（如：员工账号冒用、大规模欠薪纠纷、实际在岗人力持续低于约定下限等），且未在规定时限内完成有效整改。

  - **主动退出**：供应商主动申请退出业务，不再继续合作。

- **退出后管理**：

  - 供应商从该业务线正式退出后，自动进入备选供应商池。

  - 退出之日起三个月内不得重新参与该业务线合作。

  - 三个月后可重新参与该业务线准入评估（需满足现行准入标准）。

**全业务线强制退出规则**

- **适用场景**：供应商出现重大系统性风险或严重违规行为，触发全业务线永久终止合作。

- **强制退出标准（满足任一即触发）**：

  - **重大服务声誉事故**：

    - 发生因供应商责任导致的A/B/C类重大客户投诉事件（如：欺诈销售、大规模服务瘫痪、虚假承诺、伪造沟通记录、客户对骂等）。

    - 事件经公开渠道扩散发酵，对京东品牌声誉、监管评级或客户信任度造成实质性损害。

    - **处置措施**：立即终止全部业务合作，并永久列入禁止合作名单。

  - **信息安全及数据泄露事故**：

    - 因供应商责任导致信息系统出现重大故障/瘫痪持续≥2小时，严重影响业务连续性或造成重大财务损失。

    - **泄露保密信息**：供应商或其员工（含离职人员）非法获取、复制、传播客户隐私数据、通话录音、商业机密等保密信息。

    - **处置措施**：立即终止全部业务合作，追究法律责任，永久列入禁止合作名单。

  - **违规业务转分包**：

    - 未经京东科技书面许可，擅自将承接业务转包或分包给未签约第三方。

    - 通过关联公司、隐形职场等形式变相转移业务主体。

    - **处置措施**：立即冻结合作，限期整改无效后清退全部业务线，视情节决定是否列入禁止合作名单。

**执行步骤**：

1.  **清退触发**：

    - 单业务线退出：触发任一标准（考核不达标、客诉红线、业务调整等）。

    - 全业务线强制退出：触发任一标准（重大声誉事故、信息泄露、违规转包等）。

2.  **退出决策与方案制定**：

    - 供应商管理岗在确认触发条件后72小时内，联合业务运营岗、质检培训岗、结算岗，在5个工作日内制定《金融电销业务供应商退出交接表》。

    - 方案需包含：业务承接过渡计划、存量客户处理、数据回收、未结费用清算等。

3.  **清退审批**：

    - **审批材料**：*《退出触发事实说明》*、*《供应商清退申请》*、*《金融电销业务供应商退出交接表》*、相关证据（如客诉记录、质检报告）。

    - 供应商管理岗在京音平台/XBP[发起审批。](https://jingyin.jd.xn--com)-ec1gx72boulvz7f./)

    - **清退审批流**：供应商管理岗发起 → C3审批 → C2终审。

    - **撤项审批流**：供应商管理岗发起 → C3审批→ 通知C2。

4.  **执行与交接**：

    - 审批通过后，供应商管理岗向供应商发出书面《退出通知函》。

    - 双方按《退出交接表》执行，供应商管理岗全程监督，确保数据安全、客户平稳过渡。

    - 交接完成后，供应商签署《交接确认书》。

5.  **闭环处理**：季度内，供应商管理岗在京音平台更新供应商状态为"禁止合作"或"备选池"，归档所有文件。

**第五条 供应商业务测试管理**

**目标**：通过标准化的业务测试流程，科学、客观地验证新供应商或新业务的承接能力，确保正式合作的供应商具备稳定、高效、合规的服务水平，从源头控制业务风险。

**在测试及管理过程中，遵循以下管理原则：**

- **公平性与一致性**：测试标准与流程应在合理范围内保持统一，以客观数据和事实为基础开展评价。

- **目标一致性**：各项测试活动应以支持业务目标实现为核心，聚焦关键考核指标，避免资源分散或偏离重点。

- **风险识别与前置管控**：在测试设计及实施阶段，应主动识别可能存在的运营或合规风险，并提前规划应对方案。

- **过程与结果闭环**：确保测试从计划、执行到评估的全流程完整可追溯，结果能有效支撑后续决策与优化。

- **动态适配与持续改进**：根据实际执行效果、环境变化及反馈意见，适时优化管理方式，提升管理精准度与有效性。

**（1） 测试适用场景**

**业务测试触发条件**：

- **新供应商引入**：验证新引入供应商的实际业务承接能力和服务品质。

- **现有供应商扩项**：验证现有供应商承接新业务线的专业能力和适应性。

- **业务模式验证**：供应商POC测试，验证跑通业务模式。

- **复测验证**：供应商绩效改善后的重新测试

**测试类型分类**：

- **引入测试**：针对新供应商准入前的业务能力验证性测试。

- **扩项测试**：针对现有供应商新增业务线的能力适配性测试。

- **创新测试**：针对创新业务模式的单供应商POC测试。

- **复测验证**：针对供应商的恢复性验证测试。

**（2） 差异化测试标准**

**核心业务测试标准**：

- **测试周期**：3-6个月。

- **测试规模**：10-100人。

- **考核方式**：以季度为周期进行综合考核。

- **通过标准**：综合绩效评分达到或超过预设的策略目标值。

**创新测试标准**：

- **测试周期**：最长不超过12周（约3个月）。

- **测试规模**：5-10人。

- **考核方式**：测试期末进行综合评估。

- **通过标准**：核心绩效排名指标达到预设目标，且运营流程顺畅。

**（3） 测试审批流程**

**第一阶段：测试方案制定（3个工作日）**

1.  供应商管理牵头制定《供应商业务测试方案》。

2.  方案内容需明确：测试目标、测试范围、参与供应商、测试周期、核心考核指标（KPI）、资源投入及风险预案。

3.  与业务运营岗、策略运营岗共同评审并确认方案的可行性与考核标准的合理性。

**第二阶段：审批流程**

- **核心业务测试审批**：

  1.  业务运营岗在京音平台/XBP发起测试申请，并附上《供应商业务测试方案》。

  2.  **审批路径**：

      - **核心业务**：业务运营岗发起 → C3审批 → C2终审。

  3.  **决策时效**：整个审批流程需在5个工作日内完成。

- **创新测试审批**：

  1.  业务运营岗在京音平台/XBP发起测试申请，并附上《供应商业务测试方案》。

  2.  **审批路径**：

      - **创新业务**：业务运营岗发起 → C3审批（终审）

  3.  **决策时效**：备案流程需在1个工作日内完成。

**第三阶段：测试执行**

1.  **供应商管理岗**：负责整体测试过程的协调、监控和问题升级。

2.  **业务运营岗**：负责供应商的日常运营督导、数据追踪和现场管理。

3.  **质检培训岗**：负责测试期间的录音质检、合规检查及专项培训支持。

**第四阶段：测试结果评估**

1.  测试期结束后3个工作日内，由业务运营岗牵头，联合质检培训岗出具《供应商测试期综合评估报告》。

2.  供应商管理岗基于评估报告，判断供应商是否达到转正标准，并启动转正或清退流程。

3.  所有测试相关文件（方案、报告、审批记录）均需在京音平台/XBP留档。

**第六条 供应商生态管理**

**目标**：构建一个结构平衡、共同发展、长期共赢的供应商生态系统。通过主动的生态管理，确保供应商体系的健康、稳定与活力，支撑业务可持续增长。

**工作开展的基本遵循**：

- **结构平衡**：避免对单一供应商的过度依赖，促进核心与中尾部供应商的协同发展。

- **共同成长**：为供应商提供能力提升与业务发展的支持，实现价值共创。

- **主动管理**：建立常态化的沟通与关系维护机制，前置性地解决问题，提升合作紧密度。

- **互利共赢**：在确保京东利益最大化的同时，保障供应商的合理利润与发展空间。

**（1） 供应商结构优化**

- **核心供应商管理**：

  - 维持核心供应商的稳定合作，建立战略合作伙伴关系。

  - 在业务分配、新机会获取等方面给予核心供应商优先权益。

  - 支持核心供应商在人员、技术、管理等方面持续发展壮大。

- **中尾部供应商管理**：

  - 建立中尾部供应商的扶持与培养机制，帮助其提升运营能力。

  - 通过测试项目、小额分量等方式，给予中尾部供应商更多业务机会。

  - 定期评估中尾部供应商的成长潜力，将其作为核心供应商的储备力量。

- **生态平衡目标**：

  - 避免核心供应商业务量过度集中，降低业务连续性风险。

  - 防止中尾部供应商因业务量过少而被边缘化，保持生态活力。

  - 建立一个竞争有序、梯队分明、健康发展的供应商生态。

**（2） 业务分配平衡**

**业务量分配原则：**

- 基于供应商的绩效排名和综合能力进行分配，体现优胜劣汰。

- 充分考虑各供应商的实际承载能力，避免超负荷运营或资源闲置。

- 在分配过程中兼顾生态平衡目标，促进整体体系的健康发展。

- **控制业务集中度**：为分散核心业务风险，避免对单一供应商产生过度依赖，并维持供应商间的良性竞争，特实施
  **"n-1"业务参与上限原则**。即，单一供应商最多只能同时参与 **n-1**
  个核心业务（当前核心业务共4个，即最多参与3个）。

<!-- -->

- **执行场景**：

  - **新供应商引入**：在引入新供应商时，需评估其计划参与的核心业务数量，不得违反此上限。

  - **现有供应商扩项**：当供应商申请新增核心业务线时，供应商管理岗需核验其当前参与的核心业务数量。若已达到n-1上限，则原则上不得批准其扩项至最后一个核心业务。

  - **定期生态评估**：供应商管理岗需每季度审视所有核心供应商的业务组合，确保持续符合此规则。

  - **过渡期安排**：本规则生效时，若已存在供应商参与了全部n个核心业务，供应商管理岗应制定专项调整计划，在下一个合作周期（如续约时）内，通过业务分量等方式，引导其调整至符合n-1原则。

  - **豁免机制**：在极端特殊情况下（如供应商具备不可替代的独家技术或资源），可申请豁免。豁免申请必须由供应商管理岗发起，提交详尽的业务依赖性及风险评估报告，并经由
    **C2（间接领导）终审** 方可生效。豁免为特例，不具普适性。

**（3） 单一供应商业务量集中度上限**

**a） 规则目的**

为有效管理集中度风险，确保在单一供应商出现重大波动（如服务质量断崖式下跌、主动退出、合规风险等）时，业务连续性不受致命影响，特设定本规则。

**b） 上限标准**

为防止在单一业务线内形成垄断，对供应商承接的业务量级进行双重管控，具体标准如下：

- **AI场景业务**：
  针对主要依赖AI外呼的业务，根据参与供应商的数量，动态调整单一供应商可分配的业务量级上限：

  - **情况1**：当该业务3家供应商以内参与，单一供应商分配的业务量级不得超过该项目总外呼量级的
    **70%**。

  - **情况2**：当供应商增加至4家时，单一供应商分配的业务量级上限调整为
    **60%**。

  - **情况3**：当供应商增加至5家或以上时，单一供应商分配的业务量级上限调整为
    **50%**。

- **人工场景业务**： 针对主要依赖人工坐席的业务，采用 **"人数上限"** 与
  **"占比上限"** 双维度管控，最终分配量取两者中的 **孰低值**。

  -----------------------------------------------------------------------------
  **业务线/产品**         **单一供应商人数上限**   **单一供应商人数占比上限**
  ----------------------- ------------------------ ----------------------------
  金条人工                360人                    30%

  金条人机                120人                    40%

  信用卡                  220人                    40%

  财富                    100人                    40%

  企业金融 - 主贷         100人                    50%

  企业金融 - 小b业务      60人                     60%

  商家白条                12人                     60%

  创新业务                一事一议                 一事一议
  -----------------------------------------------------------------------------

**c) 执行与监控**

- **执行校验**：在每次进行业务分量决策时（如季度赛马后、新增资源分配时），供应商管理岗必须依据此规则对分配方案进行校验，确保不触及上限。

- **定期监控**：供应商管理岗需按月度监控各业务线的供应商集中度情况，确保持续合规。

**d) 上限调整机制**

- **定期回顾**：以上上限标准并非一成不变。供应商管理岗需 **每半年**
  组织一次业务回顾会议，联合业务运营岗、策略运营岗，根据市场变化、业务策略调整及供应商生态健康度，对上述标准进行评估。

- **调整审批**：任何对上限标准的调整，均需由供应商管理岗发起，提交详细的《业务分配上限调整建议报告》，阐明调整理由、数据支撑及预期影响。

- **审批流程**：该报告需提交至 **供应商管理岗直属领导（C3）**
  进行审批。审批通过后，更新本管理规范，并邮件通知所有相关方。

**e) 上限突破审批机制**

- **触发条件**：仅在供应商表现远超同行、具备不可替代的核心优势，或因其他供应商突发性退出导致紧急承接等极端特殊情况下，可申请阶段性突破上限。

- **审批流程**：由供应商管理岗发起申请，提交详细的《集中度风险突破申请报告》，阐明突破理由、风险应对措施及预计持续时间。审批路径为：供应商管理岗发起
  → **C3审批 → C2终审**。

- **留痕要求**：申请报告及全部审批流程记录需上传至京音平台归档。

**第七条 供应商站点管理**

**（1） 目的**

为规范供应商职场站点的设立、扩容、缩减与清退管理，确保所有站点符合选址标准、人力门槛及合规要求，保障业务连续性与运营安全。

**（2） 适用范围**

本SOP适用于所有与京东金融电销业务合作的供应商，对其名下承接京东业务的所有职场站点进行全生命周期管理。

**（3） 工作开展遵循原则**

- **数据驱动**：所有站点变更决策必须基于人力规模、赛马排名等客观数据。

- **标准先行**：站点选址、设立必须严格遵守既定标准，不得降低要求。

- **流程透明**：变更申请、审批、执行全过程留痕，确保可追溯。

- **缓冲与强制**：给予供应商合理的整改缓冲期，对逾期未达标者执行强制措施。

**选址标准**

1.  **地域限制**：职场站点须位于京东科技确认的产业带及周边半径100公里范围内的城市。

2.  **场所要求**：须设立于商业用途写字楼，并按要求提供房屋产租权证明及消防验收合格文件。

3.  **禁设区域**：严禁在居民住宅、商住混合社区及治安高风险区域周边500米内设立站点。

**（4） 站点管理场景与执行流程**

**场景一：站点新增/扩容**

**触发条件**：

1.  供应商首次承接业务，需设立首个站点。

2.  现有供应商某业务线人力规模达到《职场站点管理规范》"新设第二站点条件"。

**执行步骤 (SOP)**：

1.  **供应商自查与选址**：供应商根据标准确认自身符合新增条件，并按照"选址标准"完成新站点的选址，备齐房屋产租权证明及消防验收合格文件。

2.  **供应商发起申请**：供应商填写《职场变更申请表》，选择变更类型为"站点新增"，附上新站点的租赁合同、消防证明文件、以及证明其符合新增门槛的人力考勤记录和赛马排名截图。

3.  **供应商管理岗审批**：

    - **材料审核**：供应商管理岗在**3个工作日**内，对申请材料的完整性、真实性进行审核。

    - **标准核验**：严格核验新站点是否符合选址标准，及供应商是否满足新增门槛。

    - **决策**：

      - **审批通过**：在京音平台更新供应商站点信息，并邮件回复供应商，抄送C3、业务运营岗。

      - **审批驳回**：邮件中明确说明驳回理由及整改要求。

4.  **留痕机制**：

    - 供应商通过邮件将签字盖章的《职场变更申请表》及所有附件发送至供应商管理岗。

    - 供应商管理岗将完整的申请包（邮件、申请表、附件）保存至京音平台。

**场景二：站点变更**

**触发条件**：
供应商因租赁到期、成本优化、职场环境升级等原因，需对现有站点进行位置变更，且新址与旧址位于**同一城市、符合选址标准**的范围内（如：同一栋楼的不同楼层、隔壁楼宇、同一产业园区内等），不涉及站点总数量的增减。

**执行步骤 (SOP)**：

1.  **供应商自查与选址**：供应商根据"选址标准"对新址进行预审，确保新址为商业用途写字楼、非禁设区域，并能提供合规的房屋产租权证明及消防验收文件。

2.  **供应商发起申请**：

    - 供应商通过邮件，向供应商管理岗提交站点变更申请。

    - **邮件主题**：【站点变更申请】供应商名称 - 原址变更为新址

    - **邮件正文**：须清晰说明以下内容：

      - 变更原因（如：租约到期、降本增效等）。

      - 原站点详细地址。

      - 新站点详细地址。

      - 计划搬迁生效日期（建议安排在周末，确保业务连续性）。

      - 变更期间的业务保障方案（简述）。

    - **邮件附件**：必须包含新址的《房屋租赁合同》或《产权证明》及《消防验收合格文件》的扫描件。

3.  **供应商管理岗审批**：

    - **材料审核**：供应商管理岗在**2个工作日**内，确认邮件信息完整，附件齐全。

    - **标准核验**：严格核验新址是否符合第四条"选址标准"的所有要求。

    - **决策**：

      - **审批通过**：回复邮件"同意变更"，并抄送C3、业务运营岗。

      - **审批驳回**：回复邮件，明确说明驳回理由（如：新址位于禁设区域、消防文件不合格等）。

4.  **系统更新与留痕**：

    - 审批通过后，供应商管理岗须在京音平台中更新该供应商的站点地址信息。

**场景三：站点缩减**

**触发条件**：

1.  **供应商主动发起**：因自身经营策略调整，主动申请缩减站点。

2.  **绩效触发预警**：供应商某业务线人力规模连续2个月触及《职场站点管理规范》"站点缩减触发线"。

**执行步骤 (SOP)**：

1.  **供应商发起申请（主动场景）**：

    - 供应商填写《职场变更申请表》，选择变更类型为"站点缩减"，详细说明缩减原因、拟缩减的站点、人力转移方案及生效日期。

    - 供应商管理岗在**3个工作日**内审批，重点评估缩减方案是否会影响业务连续性和客户服务质量。决策流程与留痕同"场景一"。

2.  **京东侧预警与缓冲（绩效触发场景）**：

    - **系统预警**：每月初，供应商管理岗根据上月数据，识别出连续2月触及缩减触发线的供应商及站点。

    - **发出预警**：供应商管理岗向该供应商发出《站点绩效预警通知函》，明确告知其已触及缩减线，并给予**1个自然月**的整改缓冲期。

    - **缓冲期管理**：供应商可选择在缓冲期内通过增加人力达标，或主动发起站点缩减申请（按步骤1执行）。

3.  **留痕机制**：

    - **材料审核**：供应商管理岗在**3个工作日**内，对申请材料的完整性、真实性进行审核。

    - **标准核验**：严格核验新站点是否符合第四条选址标准，及供应商是否满足第五条新增门槛。

    - **决策**：

      - **审批通过**：在京音平台更新供应商站点信息，并邮件回复供应商，抄送C3、业务运营岗。

      - **审批驳回**：邮件中明确说明驳回理由及整改要求。

    - 京东侧发出的《站点绩效预警通知函》及供应商回执，需上传至京音平台。

**场景四：站点清退**

**触发条件**：

1.  **供应商主动发起**：供应商决定完全关闭某个站点。

2.  **强制清退**：供应商在收到《站点绩效预警通知函》后，缓冲期结束仍未达标，且未主动发起缩减申请。

3.  **重大风险**：站点发生重大合规、安全或消防问题。

**执行步骤 (SOP)**：

1.  **供应商发起申请（主动场景）**：

    - 供应商填写《职场变更申请表》，选择变更类型为"站点清退"，详细说明清退原因、生效日期、人员安置计划及客户交接方案。

    - 供应商管理岗在**3个工作日**内审批，重点评估清退方案的完整性与风险。决策流程与留痕同"场景一"。

2.  **京东侧强制执行（绩效触发场景）**：

    - **触发**：缓冲期结束次日，供应商管理岗确认该站点仍未达标。

    - **内部审批**：供应商管理岗起草《站点清退通知书》，在京音平台发起强制清退审批。

      - **审批流**：供应商管理岗发起 → **C3审批** → 抄送
        C2、业务运营岗。

    - **发出通知**：审批通过后，向供应商发出正式的《站点清退通知书》，要求其在规定日期内完成关停。

    - **执行监督**：供应商管理岗监督站点关停过程，确保数据安全、客户平稳过渡、资产妥善处理。

3.  **留痕机制**：

    - **材料审核**：供应商管理岗在**3个工作日**内，对申请材料的完整性、真实性进行审核。

    - **标准核验**：严格核验新站点是否符合第四条选址标准，及供应商是否满足第五条新增门槛。

    - **决策**：

      - **审批通过**：在京音平台更新供应商站点信息，并邮件回复供应商，抄送C3、业务运营岗。

      - **审批驳回**：邮件中明确说明驳回理由及整改要求。

    - 强制清退场景下，《站点清退通知书》、内部审批记录、与供应商的沟通邮件、站点关停确认文件等，均需上传至京音平台，作为强制执行的流程依据。

**第八条 对于未明确规定事宜的审批**
为确保所有工作事项均能得到有效监管，同时兼顾审批效率，特制定本原则：

1.  **初始审批**：对于现行工作规范与审批流中尚未覆盖的新增或临时性事项，为控制风险，其审批权限默认上溯至C2层级。

2.  **流程优化**：此类事项在经过不少于三个月的试运行后，应就其审批必要性与效率进行评估。

3.  **授权下沉**：经评估确认，若该事项风险可控、流程稳定，可将其审批流程规范化，并正式授权至C3层级进行审批。

**第九条 尾部供应商快速汰换机制**

**（1） 目的与原则**

为优化供应商结构，提升整体运营效能，避免长期低效能供应商占用资源，制定快速汰换机制。旨在通过客观、量化的标准识别尾部供应商，并执行高效、有序的替换流程，实现供应商生态的良性循环。

**（2） 汰换触发标准**

同时满足以下**所有三个条件**的供应商，将自动进入尾部供应商汰换候选名单：

1.  **合作时长**：与京东金融电销业务合作时间**连续超过12个月**。

2.  **绩效表现**：**在最近两个连续的日历季度**（如Q1与Q2，Q2与Q3），在合作一年以上的供应商里综合绩效排名中均处于**后30%**。

3.  **运营规模**：近三个月，承接京东业务的**日均出勤人力规模低于30人或该项目总人力的
    5%**。

**（3） 快速汰换决策流程**

1.  **自动识别与预警**：

    - **触发**：每季度末，供应商管理岗根据系统数据，筛选出完全符合5.9.2中所有标准的供应商。

    - **预警**：供应商管理岗向被标记的供应商发出《绩效改进与汰换风险预警函》，给予其**1个月的观察改进期**。

2.  **汰换策略制定与审批**：

    - **触发**：观察期结束后，若供应商绩效仍无明显改善，供应商管理岗需立即启动汰换程序。

    - **策略制定**：供应商管理岗联合业务运营岗，在**3个工作日**内制定《尾部供应商汰换与替换方案》。方案必须包含以下核心内容：

      - **汰换原因**：明确引用5.9.2中的触发条件。

      - **替换策略**：明确选择以下两种策略之一，并阐述理由。

        - **策略A：引入新供应商**：适用于现有中小供应商无承接能力，或需引入新竞争者的情况。

        - **策略B：份额再分配**：适用于现有中小供应商有富余产能，可快速承接业务的情况。

      - **执行计划**：包含详细的业务交接时间表、客户保障方案和风险控制措施。

    - **审批**：供应商管理岗将《尾部供应商汰换与替换方案》发起审批。

      - **审批路径**：供应商管理岗发起 → **C3审批 → C2终审**。

      - **决策时效**：C3审批不超过**2个工作日**，C2审批不超过**3个工作日**。

3.  **执行与交接**：

    - 审批通过后，立即向目标供应商发出《清退通知函》，并启动附件六《供应商退出交接表》流程。

    - 交接缓冲期原则上不超过**2周**，以确保业务快速切换。

**（4） 替换资源执行策略**

1.  **策略A：引入新供应商**

    - **执行**：供应商管理岗立即启动第4章"供应商引入"流程，并在所有流程中标记为"高优先级/紧急"，以缩短各环节处理时间。

    - **目标**：在目标供应商退出前，完成新供应商的测试方案，实现无缝衔接。

2.  **策略B：份额再分配**

    - **执行**：业务运营岗根据中小供应商的历史绩效、当前产能和承接意愿，制定详细的《业务量转移分配表》。

    - **审批**：该分配表需获得C2的最终批准。

    - **执行**：供应商管理岗向获得增量的供应商发出《业务量调整通知函》，并确保其在规定时间内完成人力和系统的准备。

**（5） 留痕机制**

- 《绩效改进与汰换风险预警函》及供应商回执。

- 《尾部供应商汰换与替换方案》及其审批流记录。

- 《清退通知函》及《供应商退出交接表》。

- 《业务量转移分配表》（如适用）。

**第十条 供应商续签管理规则**

**（1） 目的与原则**

为建立科学、公正的供应商续签评估体系，确保合作期满的供应商能够持续满足业务发展要求，并激励供应商不断提升服务质量与运营效能，制定本规则。本规则遵循客观评估、公平公正、择优续签、动态管理的原则。

**（2） 续签评估触发**

- **评估启动**：所有正式供应商的《客户外包服务协议》到期前**90天**，由供应商管理岗启动续签评估程序。

- **供应商告知**：供应商管理岗应在此时间点，向供应商发送《合作到期及续签评估告知函》，明确评估流程和时间安排。

**（3） 评估流程与标准**

**a） 硬性门槛（一票否决）**

- **人力规模门槛**：在当前合同期内，供应商为京东金融电销业务服务的**专职坐席团队日均出勤规模持续低于30人**，则次年原则上不再续签。

- **执行**：由结算岗和业务运营岗提供合同期内的人力数据，供应商管理岗进行核验。一旦触发，直接进入"不续签"决策流程，无需启动后续评分。

**b） 续签评估评分卡**
通过硬性门槛后，供应商管理岗需协同业务运营岗、质检培训岗，使用以下百分制评分卡对各业务涉及供应商进行综合评估。**综合得分≥80分**方可获得续签资格。

  ----------------------------------------------------------------------------------------------------------------------------------
  **评估维度**     **分值**    **评估标准与数据来源**                                                                     **得分**
  ---------------- ----------- ------------------------------------------------------------------------------------------ ----------
  **历史业绩**     **70分**    依据合同期内的**GMV、转化率、客诉率、赛马排名**等核心KPI的达成趋势及稳定性进行综合评分。   

  **合规表现**     **15分**    评估合同期内是否发生重大合规、信息安全、数据泄露事件，以及日常巡检合规表现。

**（4） 续签决策与审批**

1.  **评估执行**：供应商管理岗在合同到期前**60天**内，完成《供应商续签评估评分卡》的填写，并附上所有支撑报告。

2.  **决策建议**：基于评分结果，供应商管理岗形成明确的"建议续签"或"建议不续签"的初步结论。

3.  **审批流程**：供应商管理岗将《供应商续签评估评分卡》及所有支撑材料发起审批。

    - **审批路径**：供应商管理岗发起 → **C3审批 → C2终审**。

    - **决策时效**：C3审批不超过**5个工作日**，C2审批不超过**7个工作日**。

**（5） 结果应用与通知**

- **续签执行**：审批通过后，供应商管理岗在合同到期前**60天**启动合同续签谈判与签署流程。

- **不续签执行**：审批未通过后，供应商管理岗在合同到期前**60天内**向供应商发出《不予续签通知函》，并启动第7章"供应商退出管理"流程，确保业务平稳过渡。

**（6） 中小供应商成长机制**

为激励中小供应商成长，避免因单一业务线人力规模不达标而被淘汰，设立此机制。

- **适用对象**：合作未满一年的新引入供应商。

- **机制说明**：在首个合作年度内，此类供应商拥有**最多3次"业务扩项"申请机会**。

- **执行流程**：

  1.  供应商可根据自身能力，主动申请承接现有合作范围之外的新业务线。

  2.  供应商管理岗按照 **5.3节"现有供应商业务线扩增规则"**
      的流程进行审批。

# **第六章 专项场景审批规范**

**第一条 引入审批**

- **场景**：所有新供应商的正式引入。

- **审批发起条件**：完成资质审核、述标评分，且综合评分达标或获得特殊引入批准。

- **审批材料清单（以系统提交资料为准）**：

  - *《金融电销业务新增供应商需求表》*

  - 《供应商入库审核资料清单》及所有证明文件

  - 《供应商准入评分表》

  - 供应商背景调查及风险评估

- **审批路径**：

  - **人力\>10人**：供应商管理岗在京音平台/xbp发起 → C3审批 → C2终审。

  - **人力≤10人**：供应商管理岗在京音平台/xbp发起 → C3审批（终审）。

- **决策时效**：C3审批不超过3个工作日，C2审批不超过5个工作日。

**第二条 转正申请**

- **场景**：测试期供应商申请转为正式合作。

- **审批发起条件**：测试期满，且综合评估达标。

- **审批材料清单（以系统提交资料为准）**：

  - *《供应商转正申请表》*

  - *《测试期考核指标确认》*

  - *《供应商测试期综合评估报告》*

- **审批路径**：

  - **新增业务**：供应商管理岗在京音平台/xbp发起 → C3审批 → C2终审。

  - **扩项业务**：供应商管理岗在京音平台/xbp发起 → C3审批（终审）。

- **决策时效**：C3审批不超过2个工作日，C2审批不超过3个工作日。

**第三条 延期申请**

- **场景**：合作协议/测试即将到期，需续约合作。

- **审批发起条件**：合同到期前45天/测试到期前 15天。

- **审批材料清单（以系统提交资料为准）**：

  - *《供应商合作延期申请表》*

  - *《供应商年度/季度绩效评估报告》*

- **审批路径**：

  - **新增业务**：供应商管理岗在京音平台/xbp发起 → C3审批 → C2终审。

  - **扩项业务**：供应商管理岗在京音平台/xbp发起 → C3审批（终审）。

- **决策时效**：C3审批不超过3个工作日，C2审批不超过5个工作日。

**第四条 供应商清退**

- **场景**：触发退出标准，需终止合作。

- **审批发起条件**：确认（满足任一即触发）退出标准：

  - **重大服务声誉事故**：

    - 发生因供应商责任导致的A/B/C类重大客户投诉事件（如：欺诈销售、大规模服务瘫痪、虚假承诺、伪造沟通记录、和客户对骂等）。

    - 事件经公开渠道扩散发酵，对京东品牌声誉、监管评级或客户信任度造成实质性损害。

    - **处置措施**：立即终止全部业务合作，并永久列入禁止合作名单。

  - **信息安全及数据泄露事故**：

    - 因供应商责任导致信息系统出现重大故障/瘫痪持续≥2小时，严重影响业务连续性或造成重大财务损失。

    - **泄露保密信息**：供应商或其员工（含离职人员）非法获取、复制、传播客户隐私数据、通话录音、商业机密等保密信息。

    - **处置措施**：立即终止全部业务合作，追究法律责任，永久列入禁止合作名单。

  - **违规业务转分包**：

    - 未经京东科技书面许可，擅自将承接业务转包或分包给未签约第三方。

    - 通过关联公司、隐形职场等形式变相转移业务主体。

    - **处置措施**：立即冻结合作，限期整改无效后清退全部业务线，视情节决定是否列入禁止合作名单。

- **审批材料清单（以系统提交资料为准）**：*《退出触发事实说明》*、*《供应商清退申请》*、*《金融电销业务供应商退出交接表》*、相关证据（如客诉记录、质检报告）。

- **审批路径**：供应商管理岗在京音平台/xbp发起 → C3审批 → C2终审。

- **决策时效**：C3审批不超过1个工作日，C2审批不超过2个工作日（紧急情况需咚咚沟通后补流程）。

**第五条 供应商申请退项**

- **场景**：供应商主动申请退出部分业务线。

- **审批发起条件**：收到供应商正式的《退项申请函》。

- **审批材料清单（以系统提交资料为准）**：

  - 供应商《退项申请函》

  - *《退项交接表》*

- **审批路径**：

  - **撤项审批流**：供应商管理岗在京音平台/xbp发起 → C3审批→ 通知C2。

- **决策时效**：C3审批不超过3个工作日，C2审批不超过5个工作日。

**第六条 分量/扩项调整**

- **场景**：对现有供应商进行业务量调整或新增业务线。

- **审批发起条件**：收到业务运营岗《需求申请表》或主动发起调整。

- **审批材料清单（以系统提交资料为准）**：*《需求申请表》*、*《分量/扩项方案》*、供应商能力评估。

- 供应商管理岗在京音平台/XBP[发起审批。](https://jingyin.jd.xn--com)-ec1gx72boulvz7f./)

  - **审批流**：

    - **人力\>10人**：供应商管理岗发起 → C3审批 → C2终审。

    - **人力≤10人**：供应商管理岗发起 → C3审批（终审）。

- **决策时效**：C3审批不超过2个工作日，C2审批不超过3个工作日。

# **第七章 供应商巡检管理**

**第一条 巡检计划与内容**

- **计划制定**：供应商管理岗需制定年度飞行巡检计划，明确巡检范围、频率、重点内容。

- **巡检频率**：

  - 对重点供应商实施每季度或每半年一次的巡检。

  - 具体频率可依据供应商合作风险等级、历史表现动态调整。

- **巡检范围**：供应商的办公场地、核心团队、信息系统、设备设施、运营管理机制、核心业务流程。

- **重点内容**：

  - **合规运营情况**：确保作业流程、数据使用、话术规范等严格遵循内外部合规要求。

  - **服务质量保障**：考察其质量监控体系、服务水平协议达成情况、客诉处理能力。

  - **人员培训与管理**：包括员工资质核验、培训记录、薪酬福利规范性、人员稳定性等。

  - **运营风险管理**：评估其在信息安全、灾备应急、现场安全、舆情应对等方面的管控能力。

  - **现场管理规范性**：如工位管理、工作纪律、合规宣导落实等。

  - **服务流程执行一致性**：检查实际操作是否符合约定流程与标准。

  - **合同关键条款履行情况**（如数据安全、结算合规、产能保障等）。

- **执行依据**：详细检查项参照《供应商巡检清单》。

**第二条 巡检人员组织**

- **人员构成**：组建由供应商管理岗牵头、跨职能人员参与的飞行巡检小组。

- **成员来源**：可从业务运营、技术/产品、财务/结算、合规/法务、风控/内控、策略等相关部门抽调。

- **成员要求**：具备丰富的业务/风控经验、专业领域知识及敏锐的风险洞察力。

- **隔离原则**：

  - 原则上小组需1名以上成员组成，确保现场互相补充、监督制衡。

  - **严格实施保密机制**：巡检小组名单、行程、内容等信息属内部敏感信息，**严禁任何人员向供应商及其关联方泄露**。巡检小组与供应商接触期间，**不得私下接受馈赠或进行非工作目的交流**。

**第三条 巡检流程与方式**

- **检查方式**：采用不预先通知的现场突击检查（即"飞行检查"）为主，远程调取运营日志、录音/屏录抽样、数据分析为辅。

- **标准流程**：

  1.  **进场与首次会**：小组直接抵达供应商经营场所。向供应商现场负责人阐明目的、范围、流程。要求供应商简述近期运营及风控概况。

  2.  **文件与记录审查**：系统调阅/查阅关键文件（如：合同履行记录、培训签到考核表、员工人事档案、系统权限清单、客诉处理台账、巡检整改报告等）。

  3.  **现场观察与操作验证**：

      - 实地查看场地环境、物理安防、消防设施、设备状态、工位管理、网络IP。

      - 按《供应商巡检清单》逐项核验关键流程（如身份核验、投诉处理等环节）的操作合规性与系统控制有效性。

      - 抽样测试信息安全控制（如：工位屏保、终端USB禁用、文件存储与销毁等）。

      - 验证应急预案有效性（如灾备系统切换演练）。

  4.  **员工随机访谈**：匿名随机抽取一线及管理人员访谈，验证其对操作规程、合规要求、风险点、内部举报渠道的了解程度。

  5.  **问题汇总与初步沟通**：小组内部闭门会议，整理问题证据链、评估风险等级，形成初步结论及整改建议。与供应商负责人进行单点沟通，听取解释，要求其确认事实。

  6.  **末次会通报**：向供应商管理团队正式通报检查发现的问题点、风险等级、整改要求及限定期限（现场提交的立即整改项除外）。

**第四条 巡检结果处理**

- **整改指令**：供应商管理岗于巡检结束后5个工作日内正式下发包含问题描述、整改要求、明确期限的《供应商巡检整改通知书》。

- **整改跟踪**：

  - 供应商需在整改截止日前提交详实的《巡检问题整改报告》及佐证材料。

  - 供应商管理岗应组织对供应商整改情况采取现场复查或远程复核（关键问题必赴现场）。

  - 确保所有问题闭环处理，验证整改有效性。

- **信息归档**：

  - 以下文件建议上传至京音平台/XBP统一归档：

    - *《供应商巡检整改通知书》*

    - 供应商提交的《整改报告》及证明材料

  - 电子归档应确保信息齐全、分类清晰、可追溯性强。

# **第八章 供应商赛马机制**

**第一条 总则**

**（1） 目的与原则**
为优化电销外呼渠道效能，激发供应商竞争活力，建立"能者上、平者让、庸者下"的动态管理机制，特制定本规范。本机制遵循公平、公正、公开、激励与约束并重的原则。

**（2） 适用范围**
本规范适用于所有承接我司贷款产品电销外呼业务的合作供应商。

**第二条 赛马触发条件**

**（1） 常规触发**

- **业绩预警**：单个供应商连续3个工作日核心转化率，低于团队平均基准值20%以上。

- **新增资源**：当有新的外呼数据包或新产品线启动时，原则上需通过赛马机制测试各供应商承接能力。

- **定期评估**：每双周/月/季度末，对所有在营供应商进行一次综合性赛马，以评定下一双周/月/季度资源分配优先级。

**（2） 专项触发**

- **策略变更**：当外呼话术、定价策略或目标客群发生重大调整时，启动专项赛马以评估适应效果。

- **供应商申诉**：当供应商因数据质量等客观因素提出业绩异议，经核实后，可启动小范围赛马进行验证。

**（3） 赛马豁免机制**

- **豁免条件**：当新增资源或专项测试的业务量过小，不具备统计学意义时，可豁免赛马。具体标准为：**预计月度业务结算低于5万元**。

- **豁免决策**：满足豁免条件时，由 **业务运营岗**
  发起豁免申请，**供应商管理岗**
  审批。由双方共同商议决定，决策邮件需抄送C3。

- **留痕要求**：豁免申请邮件及审批回复，须上传至京音平台归档。

**第三条 赛马规模与周期**

**（1） 规模标准**

- **小规模赛马**：针对单一供应商的业绩预警或策略验证，参与方为该供应商及1-2家对标供应商。

- **大规模赛马**：针对新增资源或双周/月/季度评估，原则上所有除未转正的新供应商外，在项供应商均需参与。

**（2） 周期与资源**

- **考核周期**：常规赛马周期为15-30个自然日；专项赛马可视情况缩短至3-5个自然日。

- **测试资源**：赛马期间，应为所有参与方分配同质化、等量的数据包，确保起跑线公平。数据量需满足统计学意义（如：不少于5000条有效号码）。

**第四条 考核机制与指标**

**（1） 考核维度与权重**
采用综合评分制（满分100分），应参考规定考核维度及其 **权重范围**

- **转化效率（权重范围：30% - 50%）**

- **运营成本（权重范围：20% - 40%）**

- **过程质量（权重范围：15% - 25%）**

- **客户体验（权重范围：5% - 15%）**

**（2） 数据监控**
赛马期间，业务运营岗每日监控各供应商关键指标，形成日报，并在赛马结束后出具综合分析报告。

**第五条 目标设定**

**（1） 基准设定**
以历史数据均值、行业标杆及业务预期为基础，设定赛马各项指标的"基准值"、"目标值"和"挑战值"。

- **基准值**：供应商必须达到的及格线，未达则进入观察期。

- **目标值**：符合我司预期的标准水平，是资源分配的主要依据。

- **挑战值**：超越预期的优秀水平，将获得额外激励。

**第六条 分量依据与结果应用**

**（1） 分量计算逻辑**
赛马结果直接与未来一个月（或下一季度）的外呼数据份额挂钩。份额调整基本公式：新份额
= 基础份额 × (1 + 绩效系数)

**（2） 动态系数配置**

- **系数方案**：为体现不同业务线的战略价值和利润空间，绩效系数的具体数值与业务线绑定。

- **系数范围**：各业务线（金条、企业金融等）在不同排名下的具体绩效系数范围。

- **审批与更新**：系数需由 **C3（业务运营岗直接领导）**
  审批生效。每季度可回顾更新一次。

**（3） 结果应用**

- **优胜激励**：排名前列的供应商，除增加数据份额外，还可优先获得优质数据包、新业务测试资格。

- **末位管理：**连赛马排名末位续两次的供应商，将启动淘汰程序，终止合作。

- **动态调整**：赛马结果将作为供应商评级、合同续签及商务谈判的核心依据。

**第七条 生态管理规则**

**（1） 公平性保障**
严禁任何形式的数据泄露、或恶意竞争。一旦发现，立即取消赛马资格并追究违约责任。

**（2） 透明化沟通**
赛马规则、数据口径、考核结果对所有参与方公开透明。设立正式的申诉渠道，供应商可在结果公布后24小时内提出异议。

**（3） 合作共赢**
定期组织供应商复盘会议，分享优秀实践，共同探讨提升策略，构建健康、可持续的合作生态。

# **第九章 定价管理机制**

**第一条 定价策略制定**

- **定价依据**：策略运营岗负责主导定价方案制定，需综合以下核心要素：

  - **成本分析**：包括人力成本、技术成本、管理成本等刚性支出。

  - **市场对标**：同类型金融电销服务的市场价格水平及波动趋势。

  - **业务策略**：业务发展阶段、目标客群、预期利润空间等核心策略。

  - **供应商反馈**：供应商管理岗提供的供应商成本结构、产能瓶颈及合理溢价区间建议。

- **动态调整机制**：策略运营岗须建立透明的价格调整触发机制，明确：

  - **启动条件（任一满足即可）**：

    - 同类型服务市场均价波动幅度 ≥±5%（或行业公认阈值）。

    - 关键成本项（如基础人力成本、合规认证费）同比变动 ≥10%。

    - 业务模式、服务标准或合规要求发生重大变更。

    - 供应商评级规则调整影响成本结构。

  - **调整周期**：常规价格评审每半年一次；触发条件满足时即时启动特别评审。

- **管理要求**：供应商管理岗同步《价格调整通知》，规范调整通知期（建议≥15天）、供应商异议协商流程及新旧价格过渡期安排，确保信息对称、程序合规。

**第二条 定价审批与决策流程**

1.  **方案拟定**：策略运营岗基于定价依据拟定初始价格方案。

2.  **跨部门协商**：策略运营岗需与供应商管理岗、业务运营岗就以下关键点达成一致：

    - 成本分摊的合理性与完整性。

    - 价格方案相较市场水平的竞争力分析。

    - 该定价策略对各等级供应商合作意愿的潜在影响。

3.  **成本效益评估**：经协商一致的方案提交至经营分析部及产品部进行财务测算，重点评估：

    - 方案对公司利润率、现金流的影响。

    - 价格敏感性分析（量-价平衡模型）。

    - 与年度预算目标的契合度。

4.  **管理层审批**：

    - 完成财务评估后，策略运营岗将定价方案、协商记录、财务意见整合为《定价调整邮件》。

    - 按权限提交逐级审批：策略运营岗直属管理层 →
      金融科技事业群-数据科技业务部相关间接管理层。

    - 决策层基于业务可持续性、市场竞争格局、供应商生态健康度等因素进行终审，批复意见为：同意执行、否决或要求修订后重审。

**第三条 价格条款落地执行**

- **合同约束**：审批通过的定价方案转化为具有法律效力的合同条款，需写入《外包服务协议》或其补充邮件通知。应明确：

  - **计价基准**：清晰定义服务单价（如：元/通次、元/成功单）或阶梯式计价规则。

  - **结算机制**：支付周期（月结/半月结）、结算依据（系统统计标准、质检达标量、结算口径）、开票要求。

  - **调价程序**：触发条件、通知流程、协商机制、生效时点。

  - **争议处理**：价格分歧的复核方式与仲裁路径。

- **变更管理**：任何价格调整均需补充邮件/XBP审批，禁止口头约定。供应商管理岗负责确保新老价格在切换期平稳过渡，并将审批通过的定价文件上传京音平台/XBP备查（供财务、审计追溯）。

# **第十章 金融电销业务供应商佣金结算管理**

（注：未实现线上计费及走线上流程的业务线(目前固收、企金类业务)除外 ）

**第一条 结算数据生成与校验**

1.  **系统自动计算**：

    - 每月结算周期截止后，山海系统依据签署的《外包服务协议》约定规则（如GMV转化率、有效用户数、阶梯单价等），自动生成供应商原始佣金计算清单。

2.  **调增项处理**：

    - 因特殊场景需调整结算金额的，按以下规则分类处理：

      - **业务激励款**：由业务管理岗发起激励方案，经电销服务组负责人及产品侧管理层审批通过后执行。业管需向结算岗提供审批的《业务激励明细表》，明确各供应商激励金额及依据。

      - **跨业务线倒挂扣减**：因订单佣金小于扣减项，导致需结算金额为负数。需通过交叉/本业务线优先扣减出负数金额，再进行结算处理。

3.  **结算凭证生成**：

    - 结算岗将调增项数据导入山海系统，系统自动整合生成含以下要素的《结算对账确认函》：

      - 原始佣金计算清单

      - 调增项总金额及部分费用扣减项

      - 最终应付佣金总额

**第二条 结算内部复核机制**

**（1） 复核目的**
为确保结算数据的准确性、合规性，防范操作风险与资金风险，设立本内部复核机制。通过
**"策略配置、结算制作、业务复核"**
的职责分离，在结算数据对外推送前进行二次核验，确保流程完备、规则准确、金额无误。

**（2） 复核角色与职责**

- **系统配置岗**：由 **策略运营岗**
  担任。负责在山海系统中配置和维护所有结算相关的定价规则、阶梯单价和激励方案。每次配置变更都必须有完整的审批记录。

- **数据制作岗**：由 **结算岗**
  担任。负责根据系统中已配置的规则，生成月度《结算对账确认函》。

- **复核岗**：由 **电销服务组业务运营岗**
  担任。负责对系统配置和结算草稿进行全面、独立的交叉复核。

**（3） 复核触发与流程**

1.  **触发**：结算岗完成《结算对账单》草稿生成后，立即触发复核流程。

2.  **提交复核**：结算岗通过山海系统提交OA审批流，将结算对账单及所有支撑文件（如业务激励审批表、特殊剔除项等），提交给电销服务组及产线复核人(OA审批节点)复核审批。

3.  **执行复核**：复核岗需在 **24个工作小时** 内，依据以下标准完成复核：

    - **系统配置准确性**：核验系统中当前生效的定价规则、阶梯单价是否与最新审批通过的《定价调整邮件》或合同条款完全一致。

    - **流程完备性**：检查所有调增项是否均有完整的、已生效的审批记录。

    - **结算规则应用**：核验结算草稿是否正确应用了系统中的配置规则。

    - **结算金额**：进行合理性校验，如环比/同比波动分析，对异常波动进行溯源；可抽样进行手工重算，验证系统计算逻辑。

4.  **复核结果处理**：

    - **复核通过**：复核岗在OA系统或邮件中审批确认，邮件审批未提出异议。

    - **复核不通过**：复核岗需详细列明发现的问题点、风险等级及修改建议，退回给结算岗（若为数据问题）或策略运营岗（若为配置问题）。

5.  **修正与再复核**：相关责任岗根据复核意见完成修正后，必须重新提交复核，直至复核通过。

**（4） 留痕机制**

- 所有复核请求、复核意见、修正记录及最终的"复核通过"确认，必须在结算系统或邮件中形成完整记录。

- 《结算对账单》的最终版本，必须是通过产销线复核确认无误的账单数据并保存审批通过的记录，方可进入下一步"10.3
  结算核对与确认"流程。所有相关文件需作为结算包的一部分归档。

**第三条 结算核对与确认**

1.  **供应商核对**：

    - 结算岗通过山海系统/邮件向供应商推送《结算对账确认函》。供应商需在72小时内完成核对：

      - **确认无误**：在线签署或邮件回复确认。

      - **存在异议**：在系统中提交异议说明并附佐证材料，同步邮件至结算岗备案。

2.  **异议处理流程**：

    - **结算岗受理**：2个工作日内进行受理问题：

      - **系统逻辑/规则争议**：结算岗出具《结算差异原因核实记录》并同步产品侧备案。

      - **业务数据差异**：联合业管岗复核底层数据源，5个工作日内由责任人出具邮件说明。

    - **处理结果**：

      - 确认差异属实的，纳入次月结算周期进行补退（备注：严禁跨季度调整）。

      - 异议不成立的，向供应商回复差异原因。

**第四条 结算单审批与付款**

1.  **结算账单审批**：

    - 结算岗对计算好的结算账单发起审批：

      - 结算岗发起 → 业管岗审核 → 业管岗直属领导（C3）审批 →
        业管岗间接上级（C2）审批 → 产品侧运营审核 →
        产品侧负责人（（C3））审批→ 产品侧运营间接上级（C2）审批

    - 注：个别业务线审批节点微调以实际系统配置为准。

2.  **供应商开票**：

    - 结算单终审通过后，供应商需在5个工作日内开具合法增值税发票：

      - **普票**：扫描件上传至山海系统

      - **专票**：原件寄送至金融电销服务组指定地址/数电发票系统上传

3.  **付款执行**：

    - 结算岗在收到合规发票后45个工作日内，于财务系统发起付款申请。

    - 对接科技财务部完成付款指令审核，跟进款项到账状态。

**第五条 归档与内控**

- **全流程留痕**：

  - 以下文件建议上传至结算系统归档：

    - 《结算对账确认函》及供应商签章回传件

    - 发票扫描件及寄送凭证或数电发票

  - 以下线下归档：

    - 调增项审批邮件（业务激励、倒挂扣减）

    - 异议处理文档（如差异处理记录）（复核报告、驳回通知等）

    - 《月度结算台账》及审批记录
