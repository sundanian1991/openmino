# 提醒和问责系统 (WAL)

## 问责体系框架

### 1. 自我问责机制
- **目标承诺声明** - 公开宣布目标，增加心理约束
- **进展日志** - 记录每日进展，建立行为连续性
- **定期自省** - 设定反思时间，审视承诺履行情况

### 2. 外部问责机制
- **伙伴系统** - 寻找目标伙伴，互相监督
- **教练制度** - 寻求导师或教练指导
- **社群支持** - 加入目标实现社群

## 提醒系统设计

### 1. 日历提醒系统
- **硬性时间点** - 设置固定的回顾和检查时间
- **浮动提醒** - 基于任务状态的动态提醒
- **预警通知** - 距离截止日期前的预警

### 2. 多渠道提醒
- **设备提醒** - 手机、电脑、手表同步提醒
- **应用推送** - 目标管理应用的通知
- **物理提醒** - 桌面看板、便签纸等

### 3. 提醒层级设计

## 问责流程

### 1. 目标设定阶段问责
```
目标确认清单:
□ 目标符合SMART-WAL原则
□ 已分解为可执行任务
□ 设定了明确时间节点
□ 分配了充足资源
□ 识别了潜在风险
□ 确定了衡量标准
```

### 2. 执行阶段问责
- **周度问责** - 每周回顾完成情况
- **月度问责** - 月度目标达成度评估
- **里程碑问责** - 关键节点完成情况

### 3. 结果阶段问责
- **目标达成评估** - 完成质量与数量评估
- **过程学习总结** - 从过程中的学习点
- **能力提升评估** - 实现目标中的成长

## 提醒触发器

### 1. 时间触发器
- **周期性提醒** - 固定间隔的重复提醒
- **截止提醒** - 特定日期的时间提醒
- **里程碑提醒** - 达到特定阶段的提醒

### 2. 事件触发器
- **完成事件** - 前置任务完成后的提醒
- **阻塞事件** - 遇到障碍时的提醒
- **风险事件** - 风险指标达到阈值时

### 3. 行为触发器
- **习惯触发** - 每日习惯养成提醒
- **状态变化** - 目标状态变化提醒
- **依赖关系** - 影响其他任务时的提醒

## 提醒系统技术实现

### 1. 基于日历的提醒
```bash
# 设置日历事件脚本示例
create_reminder() {
    local title="$1"
    local datetime="$2"
    local description="$3"

    # 创建日历事件 (macOS示例)
    osascript -e "tell application \"Calendar\" to ¬
        tell calendar \"Goals\" to ¬
            make new event with properties {title:\"${title}\", ¬
            description:\"${description}\", ¬
            start date:date \"${datetime}\"}"
}

# 使用示例
create_reminder "月度回顾" "2026-03-01 20:00:00" "进行月度目标回顾"
```

### 2. 基于文件的提醒系统
创建一个提醒配置文件 `reminders.yml`:
```yaml
goals_reminders:
  - name: "周度回顾"
    schedule: "weekly on Sunday at 20:00"
    description: "回顾本周目标进展，规划下周重点"
    lead_time: "1 day"

  - name: "月度评估"
    schedule: "monthly on 1st at 19:00"
    description: "进行月度目标评估和调整"
    lead_time: "3 days"

  - name: "季度复盘"
    schedule: "quarterly at March/Jun/Sep/Dec 1st 20:00"
    description: "进行季度全面复盘和策略调整"
    lead_time: "1 week"
```

### 3. 预警阈值设置
- **进度滞后预警** - 连续2周进度低于计划
- **时间不足预警** - 距离截止日期少于预期剩余工作量
- **质量下降预警** - 连续3次质量评估低于标准

## 问责结果处理

### 1. 成功庆祝机制
- 记录成功原因
- 强化成功行为
- 分享成功经验
- 设定更高目标

### 2. 失败处理机制
- 分析失败原因
- 调整策略方法
- 申请额外支持
- 重新承诺目标

### 3. 中途调整机制
- 识别调整必要性
- 评估调整影响
- 重新设置目标
- 通知相关方
