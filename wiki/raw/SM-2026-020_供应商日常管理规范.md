## 第一章 总则

### 1.1 目的

规范京东2B电销BPO供应商的日常管理动作，覆盖信息报送、人员管理、质量合规、分量调整、集中度管控、续签评估、定价结算、巡检、站点管理等全场景，确保业务连续、交付可控、信息通畅、合规有底。

### 1.2 适用对象

- 所有承接京东2B电销业务的BPO供应商站点
- 站点负责人、对接人、一线坐席
- 供应商管理岗、业务运营岗、策略运营岗、结算岗、质检培训岗

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| **站点** | 供应商承接业务的职场单位，为最小考核单元，独立出勤、独立产能、独立评估 |
| **熟手** | 入职>30天的有效坐席（产能计算的分母） |
| **新人** | 入职≤30天的坐席（辅助参考） |
| **核心业务** | 金条、企业金融、联名卡、财富管理（>10人需C2终审） |
| **创新业务** | 试点探索阶段（≤10人，C3终审即可） |

### 1.4 管理原则

| 原则 | 含义 |
|------|------|
| **标准化** | 所有站点执行同一套标准，不因人而异 |
| **及时性** | 信息报送有明确时间要求，超时即为违规 |
| **闭环性** | 每一项管理动作都有发起、执行、反馈、归档四个环节 |
| **可追溯** | 所有重要事项留存书面记录，事后可查 |
| **合规底线** | 信息安全、持证上岗、数据真实性为不可触碰的红线 |

### 1.5 文件定位

本规范为**日常操作层**规范，与以下文件共同构成供应商管理制度体系：

| 文件 | 定位 | 与本规范关系 |
|------|------|-------------|
| 供应商管理办法（总纲） | 总纲 | 本规范是配套操作文件 |
| 供应商管理指引-精华版 | 精华速查 | 本规范以精华版为最高依据展开 |
| 供应商运营管理要求（V5.0） | 过程管理 | 本规范引用其七模块管理要求 |
| 供应商管理日常手册 | 操作手册 | 本规范合并并升级其全部内容 |
| 供应商管理SLA | 量化指标 | 本规范所有指标阈值与SLA对齐 |
| 供应商月度综合评估方案 | 评估标准 | 本规范结果作为评估数据输入 |

## 第二章 角色与职责

### 2.1 RACI 角色定义

> 来源：精华版·二 | 总纲·第八至十一条

| 角色 | 职责（精华版） | 日常职责（总纲） |
|------|---------------|-----------------|
| **供应商管理岗（R）** | 全生命周期执行、关系管理、生态管理、商务管理 | 制度制定、准入审核、评估考核、清退执行 |
| **业务运营岗（C）** | 业务策略、KPI追踪、需求发起 | 业务指导、交付管理、数据提供、问题反馈 |
| **策略运营岗（C）** | 定价主导、策略输出、立项发起 | 定价拟定、系统配置、跨部门协商 |
| **结算岗（I）** | 佣金计算、系统对账、付款推进 | 数据制作（生成结算对账单） |
| **质检培训岗（I）** | 质检标准、培训要求、现场检查 | 质检执行、数据产出、问题通报、校准维护 |

### 2.2 站点侧角色定义

| 角色 | 所属方 | 定义 | 职责 |
|------|--------|------|------|
| 站点负责人 | 供应商 | 供应商派驻各职场的最高管理者 | 统筹本站点全部业务，对供应商负责 |
| 站点对接人 | 供应商 | 各站点指定的唯一日常对口人 | 信息报送、沟通协调、异常报备 |
| 业务对接人（业管） | 京东 | 京东侧负责业务交付管理的岗位 | 产能管理、业务目标、日常交付监督 |

## 第三章 信息报送管理

> 来源：日常手册·第二章 | 运营管理要求·第六节 | SLA·第四章 | 精华版·十六

### 3.1 报送总则

- 所有信息通过指定渠道报送，不得使用个人微信或口头传达代替书面记录
- 报送格式按本规范附录模板执行，不得自行变更格式
- 站点对接人为报送责任人，对报送数据的及时性和准确性负责

### 3.2 日常报送

#### 3.2.1 每日出勤报送

| 项目 | 要求 |
|------|------|
| 报送内容 | 当日实际出勤人数、请假人数、异常缺勤情况 |
| 报送时间 | 每日 **10:00 前** |
| 报送人 | 站点对接人 |
| 报送方式 | 群内指定文档/共享表格"到岗情况"页 |
| 分类要求 | 须区分正常出勤、病假、事假、旷工四类 |
| 异常说明 | 异常缺勤须在备注栏说明原因 |
| 数据一致性 | 须与考勤系统一致，不得瞒报虚报 |

#### 3.2.2 每日排班报送

| 项目 | 要求 |
|------|------|
| 报送内容 | 次日各班次人员安排 |
| 报送时间 | 每日 **下班前** |
| 报送人 | 站点对接人 |
| 报送方式 | 共享表格"排班表"页 |
| 排班逻辑 | 须说明排班依据（业务量预测、人力匹配） |
| 人力利用率 | 说明"在岗人数"和"实际上线人数"的差异及原因 |

#### 3.2.3 每日业绩复盘

| 项目 | 要求 |
|------|------|
| 报送内容 | 当日核心业务数据（按产品线分列） |
| 报送时间 | 每日 **下线前** |
| 报送人 | 站点对接人 |
| 报送方式 | 群内发送 |
| 复盘内容 | 产能达成情况（实际值 vs 目标值）、关键指标波动说明、次日重点安排 |

#### 3.2.4 群昵称规范

| 项目 | 要求 |
|------|------|
| 格式 | `职场-职务-姓名` |
| 示例 | `金条-站点负责人-张三` |
| 时限 | **首次要求后24小时内**完成修改 |
| 未完成处理 | 清除出群，由站点负责人重新申请入群 |

### 3.3 周期报送

#### 3.3.1 周报

| 项目 | 要求 |
|------|------|
| 报送内容 | 本周出勤汇总、产能完成情况、异常事项、下周计划 |
| 报送时间 | 每周 **周五下班前** |
| 报送人 | 站点对接人 |
| 报送方式 | 邮件或共享文档 |

**周报内容结构**：
1. 本周出勤概况（日均出勤率、请假率、异常缺勤情况）
2. 产能完成情况（日均产能、达成率、与目标差距）
3. 质量情况（如已有质检数据，简要说明）
4. 异常事项清单（已处理/待处理）
5. 下周重点事项及排班预告

#### 3.3.2 月度总结

| 项目 | 要求 |
|------|------|
| 报送内容 | 月度业务数据、人员变动、合规自查、下月计划 |
| 报送时间 | 次月 **3日前** |
| 报送人 | 站点负责人 |
| 报送方式 | 邮件 |

**月度总结内容结构**：
1. 月度核心数据汇总（出勤率、产能达标率、数据准确率）
2. 人员变动汇总（入职、离职、调岗）
3. 质检→改善专节（本月质检发现、改善动作、效果验证）
4. 合规自查结果
5. 培训完成情况
6. 异常事项回顾与改进
7. 下月重点事项

#### 3.3.3 人员花名册

| 项目 | 要求 |
|------|------|
| 报送内容 | 在岗人员完整信息（姓名、工号、入职日期、岗位、资质状态） |
| 报送方式 | 共享表格，**实时更新** |
| 更新要求 | 人员变动后 **3个工作日内** 更新 |
| 报送人 | 站点对接人 |

**花名册必填字段**：姓名、身份证号（仅供应商管理岗可查阅）、入职日期、所属站点、岗位、资质证书编号及有效期、培训记录、当前状态（在岗/试用/离职）。

**身份证号保密实现方式**：站点对接人在花名册表格的"身份证号"列填写时，将号码装入密封信封提交，或通过加密邮件直接发送供应商管理岗。共享表格中该列权限设为"仅供应商管理岗可编辑"，站点对接人仅可见"已填写/未填写"状态。

#### 3.3.4 合规自查报告

| 项目 | 要求 |
|------|------|
| 报送时间 | 每月 **25日前** |
| 报送人 | 站点负责人 |
| 自查内容 | 信息安全、话术规范、操作合规 |

#### 3.3.5 培训记录

| 项目 | 要求 |
|------|------|
| 报送时间 | 每月随月度总结提交 |
| 内容 | 培训日期、主题、讲师、参与人员、考核结果、签到表扫描件 |

### 3.4 报送模板速查

| 报送项 | 频率/时限 | 模板编号 | 说明 |
|--------|----------|---------|------|
| 花名册 | 实时更新 | SM-TPL-005 | 人员变动 3 个工作日内更新 |
| 每日出勤 | 每日 10:00 前 | SM-TPL-001 | 当日实际出勤、请假、异常 |
| 每日业绩 | 每日下线前 | SM-TPL-002 | 核心业务数据 |
| 排班报送 | 每日下班前 | — | 当日/次日排班计划 |
| 周报 | 每周五下班前 | SM-TPL-003 | 出勤汇总、产能、异常、计划 |
| 月度总结 | 次月 3 日前 | SM-TPL-004 | 月度数据、人员、合规、计划 |
| 合规自查报告 | 每月 25 日前 | 新增模板 | 自查结果、问题、整改 |
| 培训记录 | 每月随月度总结 | 新增模板 | 本月培训完成情况 |

### 3.5 数据报送 SLA 指标

| 指标 | 达标线 | 预警线 | 红线 |
|------|--------|--------|------|
| 数据报送及时率 | ≥98% | <95% | <90% |
| 数据准确率 | 100% | <98% | <95% |

### 3.6 报送违规处理

| 违规情形 | 处理方式 |
|---------|---------|
| 数据报送逾期 ≤3 天 | 群内提醒，记入月度考核 |
| 数据报送逾期 >3 天 | 出具整改意见，限期补报 |
| 连续 2 个月未执行合规自查 | 限期整改，纳入月度评估扣分 |

## 第四章 人员日常管理

> 来源：日常手册·第三章 | 运营管理要求·第二节 | 精华版·四

### 4.1 人员准入

#### 4.1.1 上岗前要求

| 事项 | 要求 | 完成时限 |
|------|------|----------|
| 入职培训 | 完成公司统一入职培训课程 | 上岗前 |
| 合规培训 | 完成信息安全、话术规范等合规培训 | 上岗前 |
| 岗位考核 | 通过岗位技能考核（笔试+模拟通话） | 上岗前 |
| 持证上岗 | 取得岗位所需资质证书 | 上岗前 |
| 花名册入档 | 完成花名册信息登记 | 上岗后3日内 |

#### 4.1.2 试用期管理

- 新入职人员设 **30天试用期**
- 试用期内：质检抽检比例提高至常规标准的 **2倍**
- 试用期不达标：站点须提交改进计划或调整安排
- 试用期满：站点出具考核意见，供应商管理备案

### 4.2 在岗管理

#### 4.2.1 定期复训

| 培训类型 | 频率 | 说明 |
|----------|------|------|
| 话术更新培训 | 按需（话术变更时） | 话术变更后规定工作日内完成全员培训 |
| 合规复训 | 每季度1次 | 信息安全、录音合规等 |
| 业务技能复训 | 每月1次 | 由业务方或站点自行组织 |

#### 4.2.2 岗位权限管理

- 坐席权限由站点负责人统一管理
- 人员离职或调岗后 **当日** 回收系统权限
- 权限变更须同步报备供应商管理岗

#### 4.2.3 终端设备管理

- 所有坐席终端须安装公司指定的安全软件
- 禁止使用个人设备接入业务系统
- 终端设备定期安全检查（每季度1次）

### 4.3 人员变动管理

#### 4.3.1 变动类型与报备要求

| 变动类型 | 报备时限 | 报备内容 | 报备方式 |
|----------|----------|----------|----------|
| 新入职 | 上岗前3个工作日 | 人员信息、培训完成情况、替代安排 | 书面 |
| 主动离职 | 提前3个工作日 | 离职原因、工作交接方案、到岗替代人员 | 书面 |
| 被动辞退 | 即时 | 辞退原因、交接方案、合规审查结果 | 书面+电话 |
| 岗位调换 | 变动前3个工作日 | 调换原因、新岗位培训安排 | 书面 |
| 大面积离职（≥3人/月） | 第一人离职时 | 情况说明、影响评估、补员计划 | 书面+电话 |

#### 4.3.2 离职交接清单

| 序号 | 交接事项 | 责任人 |
|------|----------|--------|
| 1 | 客户信息/工单交接 | 离职人员 → 接手人 |
| 2 | 系统权限回收 | 站点负责人 |
| 3 | 终端设备归还 | 离职人员 |
| 4 | 花名册状态更新 | 站点对接人 |
| 5 | 三方确认（离职人员、站点、供应商管理） | 站点负责人 |

**离职归因要求**：每人次离职须有归因记录。大面积离职（≥3人/月）须有专项复盘报告。

### 4.4 异常人员管理

| 情形 | 处理方式 |
|------|----------|
| 连续2个月质检评分低于站点平均分20% | 站点出具改进计划，供应商管理备案 |
| 单月投诉≥3次 | 停岗检查，提交情况说明 |
| 资质证书过期未续 | 立即停岗，补齐资质后方可复岗 |
| 严重合规违规（信息安全红线） | 立即停岗，启动调查 |

## 第五章 质量与合规管理

> 来源：日常手册·第四章 | 运营管理要求·第三节 | SLA·第三章 | 总纲·第四至十五条

### 5.1 内部质检

| 要求 | 说明 |
|------|------|
| 质检体系 | 明确质检覆盖率、抽检比例、质检标准、质检团队配置 |
| 质检→改善闭环 | 质检发现的问题须反馈到业务改善（培训调整、话术优化、人员调整） |
| 月度专节 | 月度总结中须增加"质检→改善"专节，说明本月发现、改善动作、效果验证 |

### 5.2 合规自查

| 要求 | 说明 |
|------|------|
| 自查频率 | 每月至少一次 |
| 报告提交 | 每月 25 日前提交合规自查报告 |
| 自查内容 | 涵盖信息安全、话术规范、操作合规 |
| 连续违规 | 连续 2 个月未执行合规自查 → 限期整改，纳入月度评估扣分 |

### 5.3 SLA 合规指标

| 指标 | 达标要求 | 预警线 | 红线 |
|------|---------|--------|------|
| 信息安全 | **0违规** | — | ≥1次严重违规 → 触发清退评估 |
| 人员持证率 | **100%** | <95% | <90% |
| 录音留存率 | **100%** | <98% | <95% |
| 话术合规率 | **≥95%** | <90% | <85% |

### 5.4 合规红线（一票否决）

> 来源：总纲·第四至十五条

| 红线类别 | 严禁行为 | 触发后果 |
|---------|---------|---------|
| **信息安全** | 泄露客户个人信息、未经授权数据下载导出、存储于非授权系统、数据传输未加密 | 立即启动清退流程，情节严重者追究法律责任 |
| **持证上岗** | 无证人员上岗、证书过期未续、持假证或借用他人证书 | 无证上岗：每人扣5分，暂停新产能分配；假证/借证：直接清退 |
| **数据造假** | 篡改伪造业绩/出勤数据、伪造质检/培训记录、篡改删除通话录音 | 直接清退，扣除全部履约保证金，3年内不得重新准入 |
| **录音篡改** | 剪辑拼接通话录音、删除销毁录音、留存时长<6个月、存储于非授权系统 | 直接清退，情节严重者追究法律责任 |
| **违规转分包** | 擅自转包、关联公司变相转移 | 立即终止全部业务，永久列入禁止合作名单 |

### 5.5 异常处理分级

| 级别 | 定义 | 典型场景 | 响应时间 | 报备方式 | 升级路径 |
|------|------|---------|---------|---------|---------|
| **一般异常** | 对业务影响有限，可自行处理 | 个别人迟到、系统偶发卡顿 | 当日处理 | 群内报备 | 站点自行处理 |
| **重要异常** | 对业务有明显影响，需升级处理 | 大面积请假、系统故障、批量客户投诉 | **2小时内** | 书面+电话 | 升级至供应商管理岗 |
| **紧急异常** | 对业务或安全造成严重威胁 | 信息安全事件、安全事故、群体性事件 | **立即** | 电话先行，书面补报 | 升级至供应商管理岗+业务对接人 |

### 5.6 供应商自建制度要求

供应商须在承接业务后 **15个工作日内**，提交覆盖以下 **7个管理模块** 的内部管理规范，报送审核后执行：

| 编号 | 模块 | 须明确的制度内容 |
|------|------|-----------------|
| M1 | 人力配置管理 | 管理架构、质检与培训岗位配置、关键岗位备岗 |
| M2 | 培训管理 | 培训体系、达产周期、复训机制、辅导带教 |
| M3 | 排班与考勤 | 排班逻辑、出勤达标、异常缺勤跟进 |
| M4 | 质量管理 | 内部质检体系、质检→改善闭环、合规自查 |
| M5 | 绩效管理 | 考核体系、激励机制、改进机制、绩效面谈 |
| M6 | 现场管理 | 日常管理动作、会议制度、主管巡查 |
| M7 | 数据报送 | 报送责任人、数据核对、信息安全 |

**核心逻辑**：供应商自建制度 → 报送审核 → 确认后执行 → 以自身制度为标尺接受检查。

### 5.7 数据真实性防控

> 来源：思路梳理·§十九

| 信号 | 检测方式 | 风险等级 | 触发动作 |
|------|---------|---------|---------|
| 出勤率连续4周≥98% | 人力表过于平滑 | 预警 | 下次巡检重点核实 |
| 离职率连续3月=0 | 30人以上团队极少见 | 预警 | 要求提供社保缴纳证明 |
| GMV日度波动<5% | 过于平稳可疑 | 关注 | 对比系统自动数据 |
| 熟手人数突增无新入职记录 | 可能虚标熟手 | 预警 | 要求入职日期凭证 |
| 考核节点前后突变 | 月末/季末数据异常优化 | 危险 | 启动专项核查 |

**数据质量分判定**：
- 0项触发 = 可信（100分）
- 1项触发 = 存疑（70分），标注具体项
- ≥2项触发 = 不可信（40分），排名加注星号，不作为奖惩依据

## 第六章 分量调整管理

> 来源：精华版·六、八 | 运营管理要求·第四节 | 总纲·第六、二十一条

### 6.1 分量调整触发条件

#### 6.1.1 增加触发

- 业务量增长
- 绩效激励（赛马排名靠前）
- 其他供应商退出
- 策略调整

#### 6.1.2 减少触发

- 业务量收缩
- 绩效惩罚（赛马排名末位）
- 风险管控

### 6.2 审批阈值

| 调整幅度 | 审批流 | 决策时效 |
|---------|--------|---------|
| <50% | C3 | C3≤2天 |
| ≥50% | C3→C2 | C3≤2天，C2≤3天 |
| ≥100%（翻倍） | C3→C2（专项汇报） | C3≤2天，C2≤3天 |

### 6.3 快速通道

- **适用条件**：所有业务，调整人数 ≤10人
- **审批方式**：向C3京ME备案，无需事前审批
- **时效**：5个工作日内完成

### 6.4 赛马机制

#### 6.4.1 触发条件

- 常规：连续3日转化率低于平均20%、新增资源、定期评估
- 专项：策略变更、供应商申诉

#### 6.4.2 考核维度（100分）

| 维度 | 权重范围 |
|------|---------|
| 转化效率 | 30%-50% |
| 运营成本 | 20%-40% |
| 过程质量 | 15%-25% |
| 客户体验 | 5%-15% |

#### 6.4.3 结果应用

- **份额调整公式**：新份额 = 基础份额 × (1 + 绩效系数)
- **末位淘汰**：连续两次末位启动淘汰
- 系数需C3审批，每季度回顾更新

#### 6.4.4 考核频率

- 月度评估、季度排名
- 定期向所有供应商公布排名结果
- Top 供应商增份额、优先获取新业务；末位供应商减份额，连续末位启动淘汰程序

### 6.5 豁免机制

- **条件**：预计月度结算 <5万元
- **决策**：业务运营岗发起，供应商管理岗审批
- **留痕**：上传京音平台

### 6.6 常规分量规则

| 项目 | 说明 |
|------|------|
| 分量依据 | 赛马排名结果 |
| 分量测算 | 根据目标产能与当前产能差值计算所需人数调整量 |
| 切换规则 | 按职场规模分批切换，确保业务平稳过渡 |
| 审批流程 | 供应商管理岗拟定方案 → C3审批 → C2终审（≥50%调整幅度须C2审批） |

### 6.7 突发增量需求分量规则

当出现突增需求时，分量不单纯依据赛马排名，而是综合以下因素决策：

| 考量因素 | 说明 |
|---------|------|
| 业绩趋势 | 近阶段业绩的走势，是上升还是下滑 |
| 业绩排名 | 在同类供应商中的相对排名 |
| 相关经验 | 是否做过类似业务线 |
| 上人能力 | 能否在规定时间内补充足够人力 |
| 意愿度 | 供应商是否愿意承接增量 |
| 配合度 | 历史配合情况，响应速度和执行效果 |

### 6.8 产能监控与诊断

- 产能达标率：≥90% 达标，<80% 预警，<70% 红线（详见 SLA §2.2）
- **产能异常排查**：当产能不达标时，使用[产能双维度]§七 四象限判定矩阵（自身峰值比 + 头部差距），区分"供应商自身问题"还是"产线/甲方问题"
- **产能不足处置路径**：
  - 问题型（双低）→ 立即约谈，定位内部管理问题
  - 尽力型（自身高/头部低）→ 评估产线适配，考虑调整
  - 暂时波动（自身低/头部高）→ 观察2周
  - 标杆型（双高）→ 标杆分享，推广经验
- **关键规则**：产能计算必须使用**熟手人均**（入职>30天），排除新人稀释干扰。新人占比>30%时须标注新人拉低幅度。

## 第七章 集中度管控

> 来源：精华版·九 | 运营管理要求·第四节 | 总纲·第十九条

### 7.1 n-1 业务参与上限原则

- 单一供应商最多参与 **n-1** 个核心业务
- 当前 4 个核心业务（金条、企金、联名卡、财富管理），最多参与 **3** 个
- 豁免需C2终审（特例，不具普适性）

### 7.2 AI 场景集中度上限

| 供应商数量 | 单一供应商上限 |
|-----------|---------------|
| ≤3家 | ≤70% |
| 4家 | ≤60% |
| ≥5家 | ≤50% |

### 7.3 人工场景集中度上限

> 按业务线人数上限和占比上限双重控制，取孰低

| 业务线 | 人数上限 | 占比上限 |
|--------|---------|---------|
| 金条人工 | 360人 | 30% |
| 金条人机 | 120人 | 40% |
| 信用卡 | 220人 | 40% |
| 财富 | 100人 | 40% |
| 企业金融-主贷 | 100人 | 50% |
| 企业金融-小b | 60人 | 60% |
| 商家白条 | 12人 | 60% |

### 7.4 监控与调整

| 项目 | 说明 |
|------|------|
| 监控频率 | 每月监控集中度情况 |
| 回顾频率 | 每半年组织业务回顾会议 |
| 调整审批 | 调整需C3审批 |
| 突破上限 | 需C3→C2审批 |

### 7.5 集中度预警（总纲补充）

| 预警条件 | 处置措施 |
|---------|---------|
| 单供应商占比接近上限（80%） | 暂停新产能分配，启动扩项评审 |
| 已达上限 | 不再增加产能，推动业务分流 |

### 7.6 扩项审批条件（总纲补充）

供应商扩项（增加业务产线或增加坐席规模）须满足：
1. 当前评估等级不低于B级
2. 无橙色及以上预警
3. 扩项后仍符合集中度管控要求
4. 通过扩项专项评审

## 第八章 续签评估管理

> 来源：精华版·十一、十二 | 总纲·第七条

### 8.1 触发时间

- 合同到期前 **90天** 启动评估
- 合同到期前 **60天** 完成评估

### 8.2 硬性门槛（一票否决）

- 合同期内日均出勤规模持续 **<30人** → 不续签

### 8.3 续签评分卡

| 维度 | 分值 | 评估标准 |
|------|------|---------|
| 历史业绩 | 70分 | GMV、转化率、客诉率、赛马排名 |
| 合规表现 | 15分 | 重大事件、巡检合规 |
| 协作配合度 | 15分 | 沟通效率、响应速度、项目配合 |

**续签条件**：总分 **≥80分** 续签

### 8.4 中小供应商成长机制

- **适用对象**：合作未满一年的新供应商
- **权益**：首个合作年度最多 **3次**"业务扩项"申请机会

### 8.5 尾部供应商快速汰换

> 来源：精华版·十二

#### 触发条件（同时满足）

1. 合作时长连续 **>12个月**
2. 最近两个连续季度在 **后30%**
3. 近三个月日均出勤 **<30人** 或该项目的 **5%**

#### 汰换流程

1. 自动识别预警 → **1个月** 观察改进期
2. 制定汰换方案 → C3→C2审批（≤5天）
3. 执行交接 → 原则上 **≤2周**

#### 替换策略

- **策略A**：引入新供应商
- **策略B**：份额再分配

### 8.6 单业务线退出条件（满足任一）

| 情形 | 说明 |
|------|------|
| 连续考核周期综合绩效未达基准 | 按《供应商月度综合评估方案》判定 |
| 触发客诉违规红线 | 重大销售误导、和客户对骂等 |
| 业务调整终止 | 业务方向调整 |
| 合约到期未续 | 合同到期不再续签 |
| 合规整改失效 | 整改后仍未达标 |
| 供应商主动退出 | 供应商主动提出终止合作 |
| 连续赛马末位 | 经分量调整仍未改善 |

### 8.7 全业务线强制退出条件（满足任一）

| 情形 | 说明 |
|------|------|
| 重大服务声誉事故 | A/B/C类重大投诉、公开扩散、品牌损害 |
| 信息泄露事故 | 系统故障≥2小时、泄露保密信息 |
| 违规转分包 | 擅自转包、关联公司变相转移 |
| 数据造假 | 报送数据、质检记录、培训记录造假 |

### 8.8 退出流程

| 步骤 | 动作 | 时限 |
|------|------|------|
| 1 | 供应商提交退出申请 / 供应商管理岗发起清退申请 | — |
| 2 | 评估影响范围，起草退出交接方案 | — |
| 3 | 审批：供应商管理岗 → C3审批（≤1天）→ C2终审（≤2天） | 3个工作日 |
| 4 | 发出退出通知，进入过渡期 | — |
| 5 | 过渡期内完成业务交接、人员安置、数据清理 | 30天 |
| 6 | 过渡期满，正式终止合作 | — |

### 8.9 退出交接

#### 客户信息处理

| 步骤 | 动作 |
|------|------|
| 1. 数据导出 | 供应商将客户数据以指定格式导出 |
| 2. 交接确认 | 客户数据交接至指定接替供应商或业务系统 |
| 3. 数据删除 | 供应商删除所有持有副本，提交删除确认函 |
| 4. 保密延续 | 签署《保密延续协议》，合作终止后保密义务延续2年 |

#### 退出交接核心内容

- **准备阶段**：停呼时间确认、与供应商确定最后停呼日期和停发数据日期
- **执行阶段**：数据盘点（确认所有数据归属）→ 数据回收（从京音系统回收数据）→ 企业微信转移（确认接收账号并发起转移）
- **收尾阶段**：账号关闭（确认客户已全部转移后禁用京音/企微/质检账号）→ 结算处理（核对未结款项、扣除违约金、走财务审批流程）→ 客诉处理（设立1个月专项跟踪期，防止客诉无人处理）

### 8.10 退出后管理

| 要求 | 说明 |
|------|------|
| 进入备选池 | 退出后自动进入备选供应商池 |
| 禁入期 | 退出之日起 **3个月** 内不得重新参与该业务线合作 |
| 重新准入 | 3个月后可重新参与准入评估（须满足现行标准） |

## 第九章 定价与结算管理

> 来源：精华版·十三、十四 | 运营管理要求·第五节

### 9.1 定价管理

#### 9.1.1 定价依据

- 成本分析（人力、技术、管理）
- 市场对标
- 业务策略
- 供应商反馈

#### 9.1.2 调整触发（满足任一）

- 市场均价波动 **≥±5%**
- 关键成本变动 **≥10%**
- 业务模式/标准重大变更
- 评级规则调整影响成本

#### 9.1.3 审批流

策略运营岗拟定 → 跨部门协商 → 财务测算 → 管理层审批（C3→C2）

#### 9.1.4 调整周期

- **常规**：每半年评审一次
- **触发**：即时启动特别评审

### 9.2 结算管理

#### 9.2.1 复核机制（职责分离）

| 角色 | 职责 |
|------|------|
| 策略运营岗 | 系统配置（定价规则、阶梯单价） |
| 结算岗 | 数据制作（生成结算对账单） |
| 业务运营岗 | 交叉复核（**24小时内**） |

#### 9.2.2 结算流程

| 步骤 | 动作 | 时限 |
|------|------|------|
| 1 | 系统自动计算 → 生成原始清单 | — |
| 2 | 调增项处理（业务激励、倒挂扣减） | — |
| 3 | 复核流程 | **24小时** |
| 4 | 供应商核对 | **72小时** |
| 5 | 异议处理 | 系统逻辑争议：**2天**；业务数据差异：**5天** |
| 6 | 审批付款 | 收票后 **45个工作日内** |

#### 9.2.3 结算周期

- 标准结算周期：**T+2月**
- 合同到期结算：合同到期前 **60天** 启动结算清算
- 清退结算：最后合作业务月的结算款压 **3个月** 再结

#### 9.2.4 关键规则

- 跨季度调整：**严禁**
- 留痕：所有文件上传结算系统归档
- 内审：季度抽样审计

### 9.3 争议与申诉

#### 9.3.1 结算重大问题处理

| 情形 | 处理流程 |
|------|---------|
| 结算金额存在重大差异 | 供应商书面提出异议 → 供应商管理岗会同结算岗复核 → 差异说明 → 重新出具结算单 |
| 结算数据缺失或错漏 | 供应商提供补充材料 → 结算岗核实 → 补充结算或调账 |
| 结算争议无法协商解决 | 供应商管理岗发起 → C3协调 → C2裁定 |

#### 9.3.2 评估申诉

| 项目 | 说明 |
|------|------|
| 申诉条件 | 对评估得分有异议，可在评估结果公布后 **3个工作日内** 发起 |
| 申诉方式 | 书面申诉（说明异议项+理由） |
| 受理时效 | 供应商管理岗 **1个工作日内** 受理 |
| 复核时效 | 指定独立复核人 **2个工作日内** 完成复核 |
| 最终结果 | **3个工作日内** 反馈，复核结果为最终结果，不再二次申诉 |

#### 9.3.3 复核人规则

| 规则 | 说明 |
|------|------|
| 独立性 | 复核人须与被评估站点无直接利益关联 |
| 范围 | 复核内容限于申诉所列异议项 |
| 监控 | 某站点连续3个月申诉率>50%，供应商管理岗介入了解原因 |

> 来源：思路梳理·§五

| 编号 | 检查项 | 说明 |
|------|--------|------|
| A | 资质证照 | 营业执照、许可证有效性 |

**FCI判定标准**：≥75合格，60-75预警，<60危险

## 第十四章 预警状态机与异常响应

> 来源：SLA·第五章 | 总纲·第十六至十八条 | 思路梳理·§一、§二

### 14.1 6级预警状态机

| 状态 | 名称 | 条件 | 处置措施 |
|------|------|------|---------|
| **S0** | 正常 | 所有指标均在达标线以上 | 正常合作 |
| **S1** | 关注 | 单项指标触及预警线或连续2周轻度异常 | 书面提醒，要求提交改进计划 |
| **S2** | 预警 | SLA指标触及预警线或单项指标持续不达标 | 约谈供应商负责人，限制新产能 |
| **S3** | 整改 | 月度评估D级/E级或S2未改善 | 下发《整改通知书》，冻结新增产能，30-60天整改期 |
| **S4** | 观察退出 | 连续2个月E级或整改期满未达标 | 逐步缩减产能，暂停所有新业务，启动清退评估 |
| **S5** | 清退 | 触碰合规红线或连续3个月E级 | 立即终止合作，进入清退流程 |

### 14.2 月度评估评级与状态映射

> 来源：评估方案·第五章

| 评级 | 总分 | 对应状态 | 处置措施 |
|------|------|---------|---------|
| A（优秀） | ≥90分 | S0 | 群内公示表扬，续约优先权 |
| B（良好） | 80-89分 | S0 | 群内公示 |
| C（达标） | 70-79分 | S1 | 约谈站点负责人，了解原因，提出改进建议 |
| D（预警） | 60-69分 | S3 | 约谈站点负责人，要求提交整改计划，整改周期30天 |
| E（红线） | <60分 | S5 | 启动清退评估流程 |

**一票否决**：即使总分≥60分，如触发以下任一红线，直接降为E级：
- 信息安全发生严重违规
- 人员持证率<90%
- 质检月均分<70分
- 数据造假

### 14.3 触发规则（9种状态）

> 来源：思路梳理·§一、§二

| 供应商状态 | 触发条件 | 动作 | 完成标志 |
|-----------|---------|------|---------|
| 月度正常 | 排名不在后30% | 无需动作 | — |
| 首次预警 | 本月落入后30%，上月不在 | 私下沟通：问原因，记录原因 | 沟通记录已存档 |
| 连续预警 | 连续2个月在后30% | 正式约谈：要求提交整改计划 | 整改计划已收到 |
| 清退评估 | 连续3个月在后30% | 评估：延长观察期/启动清退 | 评估结论已记录 |
| 回正 | 连续2个月回到前70% | 取消预警，归档记录 | 预警取消 |
| A级骤降 | 上月前30%本月跌到后30% | **立即约谈**（不等连续） | 原因已查明 |
| 诚信预警 | 数据质量分≤70或1项不实 | 警告+标记：3个月加倍核查 | 警告已送达 |
| 诚信红线 | ≥2项不实或同一项连续2次不实 | 约谈高层，告知造假后果 | 约谈记录已存档 |
| 信任重建 | 连续3个月数据质量分=100 | 取消加倍核查，恢复正常 | 重建完成 |

### 14.4 SLA与评估方案的关系

> SLA与《供应商月度综合评估方案》是两套不同维度的体系，独立运作

| 维度 | SLA | 评估方案 |
|------|-----|---------|
| **定位** | "能不能接受"——定义服务质量底线 | "得多少分"——量化评估与排名 |
| **触发** | 管理动作（预警、清退） | 得分与评级 |
| **逻辑** | 连续不达标才触发管理动作 | 按档打分，不考虑连续性 |

两套体系**同时生效**，SLA的预警不直接等同于评估方案的扣分。

### 14.5 整改闭环验证

> 来源：思路梳理·§六

| 维度 | 验证方法 | 通过标准 |
|------|---------|---------|
| 目标达成 | 对比承诺的量化目标 | ≥80% |
| 动作执行 | 逐条核对承诺动作 | ≥70%按期完成 |
| 持续性 | 观察整改后2个月数据 | 指标未回落 |
| 根因消除 | 对比整改前后的核心失分项 | 原失分项提升≥1个等级 |

**验证结果处理**：
- 全部通过 → 取消预警，归档为成功案例
- 部分通过 → 延长观察期1个月
- 未通过 → 升级：连续2次未通过进入清退评估
- 造假嫌疑 → 直接进入清退评估，记入诚信档案

### 14.6 状态流转规则

- S0 → S1：当月评估降至D级或触及预警线，下月起加强监控
- S1 → S2：D级连续2月或红线指标1次，发出书面预警
- S2 → S3：E级1次或整改失败，30日限期整改
- S3 → S4：整改期满未达标或E级连续2月，启动清退评估
- S4 → S5：清退审批通过或红线一票否决，进入清退流程
- S3 → S1：整改成功，跳过S2直接回S1观察1月
- S2 → S1：预警解除，观察1月后回S0

## 第十五章 方法论关联与诊断框架

> 来源：思路梳理·§七至十九

本章建立管理制度与年老师方法论的关联映射，使日常管理动作有方法支撑。

### 15.1 触发规则关联

> [触发规则]§二 → 月度排名后的状态判定

月度排名公布后，按9种状态判定表（第十四章·14.3）逐家匹配供应商状态，决定后续动作。

### 15.2 产能双维度诊断

> [产能双维度]§七 → 业绩诊断根因

**人力拆分规则**：
- 熟手 = 入职>30天的有效坐席（产能计算的分母）
- 新人 = 入职≤30天的坐席（辅助参考）
- 新人占比>30% → 必须用熟手人均GMV，标注新人拉低幅度
- 总人数变化>20% → 同时看总量GMV和熟手人均GMV

**维度1：自身峰值比**

自身峰值比 = 本月熟手人均GMV ÷ 该供应商近6个月最高熟手人均GMV

| 区间 | 含义 | 动作 |
|------|------|------|
| >85% | 正常发挥 | 无需动作 |
| 65%-85% | 有下滑空间 | 观察趋势 |
| <65% | 严重未发挥 | 约谈定位原因 |

**维度2：头部差距**

头部差距 = 本月熟手人均GMV ÷ 同产线当前第1名的熟手人均GMV

| 区间 | 含义 | 动作 |
|------|------|------|
| >80% | 接近头部水平 | 标杆候选 |
| 60%-80% | 有差距但不大 | 对标学习 |
| <60% | 跟头部差距显著 | 评估产线适配 |

**双维度交叉矩阵**：

| 自身峰值比 | 头部差距 | 类型 | 解读 | 动作 |
|-----------|---------|------|------|------|
| 高 | 高 | 标杆型 | 自己发挥好，也跟得上头部 | 标杆分享 |
| 高 | 低 | 尽力型 | 自己已尽力，但产线/产品人效偏低 | 评估产线适配 |
| 低 | 低 | 问题型 | 自己没发挥好，差距也在拉大 | 立即约谈 |
| 低 | 高 | 暂时波动 | 短期因素导致，底子不差 | 观察2周 |

### 15.3 主管胜任评估

> [主管胜任]§八 → 管理团队评估

| # | 指标 | 计算方式 | 达标线 |
|---|------|---------|--------|
| 1 | 团队稳定度 | 月度流失率 = 离职人数/月初总人数 | <8%/月 |
| 2 | 出勤稳定度 | 月度日均有效出勤率 | >90% |
| 3 | 人效趋势 | 近3个月人均GMV趋势 | 不连续2个月下降 |
| 4 | 响应质量 | 预警后48小时内提交原因分析的比例 | >80% |
| 5 | 整改执行力 | 承诺的整改动作按期完成率 | >70% |

**判定**：
- 5项达标 → 胜任，无需动作
- 3-4项达标 → 需关注，约谈时针对性提问
- ≤2项达标 → 不胜任信号，约谈时谈管理团队调整

### 15.4 异常信号扫描

> [异常信号]§十六 → 周度/月度扫描雷达

#### 周度扫描（只看3个突变信号）

| 信号 | 异常标准 | 触发动作 |
|------|---------|---------|
| 出勤异常 | 有效出勤率<85% | 问供应商出勤情况 |
| 业绩断崖 | 周GMV比近4周均值下降>20% | 按A级骤降逻辑立即约谈 |
| 流失加速 | 单周离职>5% | 了解离职原因 |

#### 月度计算信号

| 信号 | 判定 | 动作 |
|------|------|------|
| 自身峰值比<65% | 严重未发挥 | 约谈定位原因 |
| 头部差距>40% | 跟头部差距拉大 | 评估产线适配 |
| 新人拉低>30% | 新人稀释严重 | 要求新人培养计划 |
| 隐性退步 | 排名升≥3名且人效降≥10% | 提前预警（假好） |
| 隐性进步 | 排名降≥3名且人效升≥10% | 评估是否加资源（真进步） |
