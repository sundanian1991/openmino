# 四层架构

> 参考Anthropic agentic analytics stack设计的供应商管理数字人架构

## 架构

```
                   供应商管理数字人（Agent）
                  ┌──────────────────────────┐
                  │ 被动响应 · 主动监控 · 群聊协作 │
                  └──────────┬───────────────┘
                             │
          ┌──────────────────┼──────────────────┐
          │  L3 Skills能力层  │  L2 可信来源（5库） │
          │   28个标准化流程  │  SOP/档案/案例/规则/商务 │
          └──────────────────┴──────────────────┘
                             │
          ┌──────────────────┼──────────────────┐
          │  L1a 结果数据     │  L1b 过程数据      │
          │  data-agent 看板  │  joyclaw 群聊接入  │
          └──────────────────┴──────────────────┘
                             │
                      ┌──────┴──────┐
                      │  L4 验证层   │
                      │ 离线评估·衰变监控 │
                      └─────────────┘
```

## 各层说明

| 层级 | 名称 | 核心职责 |
|------|------|---------|
| L1 | 数据底座 | 结果数据（data-agent）+ 过程数据（joyclaw），前提是口径统一 |
| L2 | 可信知识库 | 5个独立知识库，供数字人检索 |
| L3 | Skills能力层 | 28个可调用标准化流程，准确率21%→95%+的关键杠杆 |
| L4 | 验证层 | 离线评估+衰变监控+维护机制，防止准确率随时间下滑 |

## 关键原则

- 结构优先于数据量
- Skill是最大杠杆
- 系统会衰变，维护是工程任务
- 95%就够了，追完美性价比低
- 口径统一是前提
