# 方法论体系

> 供应商管理的"判断引擎" — 看到信号后，知道调哪个工具、按什么顺序、输出什么格式

## 总览：方法论调用链

```
看到信号 → [[触发规则表（状态机）]] 判定状态 → [[产能利用率双维度判定]] 定位根因
                                               ↓
              [[主管胜任评估5项指标]] 评估团队 → [[异常信号31个]] 交叉验证
                                               ↓
              [[整改闭环验证标准]] 验收 → [[场景SOP]] 输出动作
```

## 7个核心方法论工具

1. [[触发规则表（状态机）]] — 业绩异常后，判定严重程度
2. [[产能利用率双维度判定]] — 定位业绩异常的根因
3. [[主管胜任评估5项指标]] — 判断是"人的问题"还是"管理的问题"
4. [[整改闭环验证标准]] — 验收整改是否真正到位
5. [[异常信号31个]] — 提前发现问题的"雷达"
6. [[数据真实性防控]] — 识别数据造假
7. [[标杆配对方法论]] — 找到合适的对标学习对象
