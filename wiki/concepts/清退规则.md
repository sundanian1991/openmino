# 清退规则

> 供应商清退的触发条件和退出流程

## 触发条件

| 条件 | 说明 |
|------|------|
| 连续考核不达标 | 连续两期D级 |
| 触碰红线 | 数据泄露、录音篡改等 |
| 整改不合格 | 黄牌整改后仍未达标 |
| 主动退出 | 供应商主动提出 |
| 业务策略调整 | 业务线取消或调整 |

## 退出流程

```
启动评估 → 替代方案 → 数据回收 → 费用清算 → 系统关闭 → 归档
```

| 环节 | 要求 |
|------|------|
| 启动评估 | 72小时内触发 |
| 替代方案 | 5个工作日内完成 |
| 数据回收 | 客户资料/业务数据/录音等 |
| 费用清算 | 结算至退出当日 |
| 系统关闭 | 账号/权限/系统接入关闭 |
| 归档 | 退出报告、数据归档 |

## 退出原则

- **有据可依**：触发条件明确、证据充分
- **风险隔离**：单业务线退出不影响其他线
- **平稳过渡**：有过渡计划、最小化业务中断
