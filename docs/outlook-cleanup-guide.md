## 🧹 清理步骤

### 第一步：在 Outlook 中重建数据库（推荐）

**作用**：压缩数据库，删除已标记但未清理的数据

**操作步骤**：

1. **完全退出 Outlook**（Cmd+Q）

2. **打开终端**，运行：
   ```bash
   /Applications/Microsoft\ Outlook.app/Contents/MacOS/Microsoft\ Outlook -resetdb
   ```

3. 如果上述命令无效，手动重建：
   - 打开 Outlook
   - 顶部菜单 → 工具 → 数据库 → 重建

**预期效果**：可能释放 5-15G（取决于有多少已删除但未清理的数据）

---

### 第二步：清理大附件邮件

**操作步骤**：

1. 打开 Outlook
2. 在搜索框输入：`has:attachments size:>10MB`
3. 按大小排序（视图 → 排序方式 → 大小）
4. 逐个检查，删除不需要的邮件
5. **重要**：清空"已删除邮件"文件夹（右键 → 清空）

**快速定位最大附件**：
```
搜索条件：has:attachments size:>50MB
```

**预期效果**：取决于你的邮件积累情况，可能释放 5-20G

---

### 第三步：调整缓存设置（防止继续膨胀）

**操作步骤**：

1. Outlook → 偏好设置 → 账户
2. 选择你的邮箱账户
3. 点击"高级"
4. 找到"下载邮件"选项
5. 改为"最近 12 个月"（默认可能是"全部"）

**预期效果**：防止未来继续膨胀

---

### 第四步：清理已注销账户的数据

如果你曾经登录过多个 Outlook 账户，已注销的账户数据可能还在。

**检查方法**：
1. 打开 Finder
2. 前往：`~/Library/Group Containers/UBF8T346G9.Office/Outlook/Outlook 15 Profiles/`
3. 查看是否有多个 Profile 文件夹

---

### 第五步：清理 Time Machine 本地快照（如果适用）

Time Machine 可能会保留大量本地快照。

**检查命令**：
```bash
tmutil listlocalsnapshots /
```

**删除快照**：
```bash
# 删除所有本地快照
for d in $(tmutil listlocalsnapshots / | awk -F'.' '{print $5}'); do sudo tmutil deletelocalsnapshots $d; done
```

## ⚠️ 注意事项

1. **备份重要邮件**：清理前建议导出重要邮件
2. **已删除邮件需要清空**：删除后还在"已删除邮件"文件夹里
3. **Exchange 用户**：如果是 Exchange 账户，服务器上有完整备份
4. **IMAP 用户**：清理前确保重要邮件已同步到服务器

## 🔄 维护建议

1. **每月清理一次**：搜索 `size:>10MB` 清理大附件
2. **启用"只缓存 12 个月"**：防止无限增长
3. **定期清空已删除邮件**：删除后记得清空
4. **使用云端附件**：用 OneDrive/云盘链接代替大附件
