## 文件命名规范

| 场景 | 格式 | 示例 |
|------|------|------|
| 业务文档 | `{YYYYMMDD}-{业务域}-{内容}.{ext}` | `20260401-供应商-定价委员会机制.md` |
| 模板文件 | `{类别}-TPL-{编号}.{ext}` | `供应商-TPL-001-申请表.md` |
| 项目归档 | `{YYYY-MM}-{项目名}.{ext}` | `2026-03-成本调研.md` |

**禁止**：
- 用 `_` 作为主题分隔符（统一用 `-`）
- 空格（统一用 `-` 连接）

## AI 访问指南

当 AI 需要在 docs/ 创建文件时：

### 第一步：判断文件类型

| 文件类型 | 存放位置 | 命名规则 |
|---------|---------|---------|
| 供应商相关文档 | docs/供应商/{子类}/ | `{业务}-{内容}.md` |
| 个人成长相关 | docs/个人成长/{子类}/ | `{主题}-{内容}.md` |
| 学习资料 | docs/学习资料/{主题}/ | `{主题}-{内容}.md` |
| 通用知识 | docs/知识/{类别}/ | `{类别}-{内容}.md` |
| workspace 模板 | docs/workspace-knowledge/ | `{类型}-{名称}.md` |

### 第二步：检查目录是否存在

```bash
# 检查目录
ls -la docs/供应商/

# 创建目录（如需要）
mkdir -p docs/供应商/{子类}
```

### 第三步：创建文件

- 确认命名符合规范（用 `-` 连接）
- 确认文件放在正确的一级目录下
