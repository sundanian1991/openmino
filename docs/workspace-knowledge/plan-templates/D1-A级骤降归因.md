# D1 A级骤降归因

> 上月前30%本月后30%，只有一个月窗口期。先排雷（是不是自己的锅），再排供应商。

---

## 问题拆解

收到A级骤降信号时，把模糊的"怎么掉这么多"拆成3个可判断的具体问题：

1. **这个骤降是不是我的决策导致的？** — 近期有没有调名单、缩量、切产线、改策略
2. **如果是供应商的问题，问题出在哪个环节？** — 人（主管/团队）、系统（名单/拨号）、管理（流程/制度）
3. **供应商有没有在隐瞒或铺垫什么？** — "人不好招"可能是减量铺垫，沉默比抱怨更危险

**时间压力**：只有一个月窗口。如果归因错了，浪费的是唯一的干预窗口期。

---

## 归因路径（决策树）

### 第一步：排除甲方因素（先检查自己）

> 核心原则：先检查甲方责任，再检查供应商。制度盯人，不是人盯人。

**检查清单**（任何一个为"是"→ 进入甲方归因分支）：

| 检查项 | 具体看什么 |
|--------|-----------|
| 名单调整 | 近30天是否有名单策略变更？该供应商的名单质量是否整体下降？ |
| 分量调整 | 近30天是否缩量？缩量比例是否超出正常波动？ |
| 产线切换 | 是否切了产线？新产线难度是否显著高于旧产线？ |
| 系统变更 | 策略组是否调整了拨号策略、分配逻辑、系统参数？ |
| 新制度发布 | 是否有新合规要求/新质检标准导致不适应期？ |

**如果是甲方决策导致**：
- 修正归因，不处罚供应商
- 启动考核豁免期（通常1个月）
- 向王易人汇报："这个骤降是我的X决策引起的，已启动豁免，预计Y月回正"
- 同时评估：这个决策本身对不对？如果决策是对的，供应商的适应期是否合理？

**如果排除甲方因素** → 进入第二步。

### 第二步：定位供应商侧原因

**关键分支**（不是线性步骤，是"看到什么信号→走哪条路"）：

**分支A：主管/团队问题**

| 信号 | 观察 |
|------|------|
| 主管突然离职或更换 | 查HR记录，确认交接情况 |
| 多个核心坐席同时流失 | 查离职率，对比其他供应商 |
| 团队氛围变化 | 质检报告中的通话质量、态度评分是否同步下滑 |

处理方式：
- 评估主管胜任5项（业务理解/团队管理/数据分析/抗压能力/合规意识）
- 约谈供应商负责人，话术："最近数据有波动，想了解一下团队情况。你有没有考虑过管理团队的调整？"
- 注意：不说"你主管不行"，而是引导对方自己评估

**分支B：名单/系统问题**

| 信号 | 观察 |
|------|------|
| 接通率骤降但转化率正常 | 名单质量出了问题 |
| 接通率正常但转化率骤降 | 产品/话术/流程出了问题 |
| 拨号量骤降 | 系统故障或名单量不足 |

处理方式：
- 与策略组沟通，带上数据："毅航本月接通率从35%降到22%，请帮忙看下名单质量分布"
- 要求策略组出分析报告，7天内回复
- 如果确认是名单问题，追溯名单来源，防止其他供应商也被影响

**分支C：大客户/大项目流失**

| 信号 | 观察 |
|------|------|
| 某个产线突然为零 | 查业务分配记录 |
| 总量正常但产线结构剧变 | 查产线维度数据 |

处理方式：
- 确认是主动退出还是被动调整
- 如果是被动调整，评估产能重新分配方案
- 如果是主动退出（供应商挑活），这是态度问题，走正式约谈

**分支D：供应商自身管理问题**

| 信号 | 观察 |
|------|------|
| 无外部因素，持续下滑 | 对比同产线其他供应商，是否普遍下滑 |
| 培训投入减少 | 查培训记录 |
| 质检分数同步下降 | 查质检报告 |

处理方式：
- 正式约谈，走 D2 整改定级框架
- 要求供应商7天内提交改善方案
- 设定观察期（通常4-6周）

**分支E：隐性博弈信号**

> 供应商说"人不好招"可能是在铺垫减量；不争资源的供应商比发牢骚的更危险。

| 信号 | 解读 |
|------|------|
| "人不好招" | 可能在铺垫减量要求，先查同地区其他供应商是否也有此问题 |
| 突然不争资源、不提要求 | 可能在评估退出，或在找其他甲方 |
| 突然异常配合、主动表态 | 可能在掩盖问题，或者拿到竞对offer在做选择 |
| 投诉其他供应商 | 转移注意力策略 |

处理方式：
- 不点破，但提高监控频率
- 通过行业关系了解该供应商是否有新动向
- 准备应急预案（替代供应商是谁）

## 权衡因素

### 正方（快速归因、快速干预）

| 因素 | 权重 | 说明 |
|------|------|------|
| 窗口期只有一个月 | **高** | 等不起，一个月后如果没干预就成趋势了 |
| 信号明确时不犹豫 | **高** | 如果信号清晰（主管离职+数据同步下滑），直接判断 |
| 先排雷避免误判 | **高** | 如果是甲方因素导致，处罚供应商会严重损害信任 |

### 反方（不要急着下结论）

| 因素 | 权重 | 说明 |
|------|------|------|
| 信息不对称 | **高** | 供应商可能隐瞒，第一轮沟通拿到的不一定是真因 |
| 多因叠加 | **中** | 可能同时存在主管离职+名单质量下降，不是单一原因 |
| 季节性波动 | **中** | 春节后/节假日前本身就有波动，要排除正常波动区间 |
| 供应商试探 | **低** | 可能在试探甲方反应，过度反应会暴露管理底线 |

**核心权衡**：速度 vs 准确性。一个月窗口要求速度，但信息不对称要求准确性。解法是**分层推进**：第一周排雷+初步归因，第二周验证+深度诊断，第三周决策+启动干预。

## 叙事草稿

### 向上汇报（给王易人/军哥）

> "毅航上月A级排名第3，本月降到第28。我已经做了初步归因：
> 第一步排除了甲方因素——近期没有名单调整和分量变更。
> 第二步定位到主管层——毅航金条团队主管小王上月中旬离职，交接期间团队核心坐席流失了4个，占团队的15%。
> 我已经约了毅航负责人本周三沟通，重点是确认接任主管的人选和团队重建计划。
> 预计需要4-6周恢复，我建议本月考核给豁免期，下月中旬看恢复情况再定。"

**关键**：说清楚"什么原因→我在做什么→预计什么时候回正→我的建议是什么"。带方案去，不是带问题去。

### 对供应商

> "XX，最近一个月的数据有些波动，我想了解一下情况。你们团队最近是不是有些变化？（听对方说）
> 我理解团队调整期的压力。这样，我们定个恢复计划——新主管什么时候到位？核心坐席补充计划是什么？需要我这边支持什么？
> 我这边会跟进名单质量，确保不是名单的问题。咱们一起把数据拉回来。"

**关键**：先问再判断，不预设原因。给对方说话的空间，但要拿到具体承诺（时间、人数、计划）。

### 内部同步（给策略组）

> "毅航本月数据骤降，已定位到主管离职导致。请帮忙看下毅航本月的名单质量分布是否正常，排除名单因素。周三前给我个分析。"

**关键**：简洁、有具体请求、有时间节点。

### 需要填的数据字段

- 供应商名称 + 产线
- 骤降前排名 / 骤降后排名 / 排名变动
- 同产线其他供应商本月表现（对比用）
- 近30天甲方决策变更记录
- 供应商团队变动记录（主管/核心坐席离职）
- 名单质量数据（接通率、有效接通率）
