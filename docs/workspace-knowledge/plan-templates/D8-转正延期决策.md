# D8 转正/延期决策

> 6月里程碑到了，数据在"差不多"区间。转正、延期还是清退？给明确判断，不给模糊地带。

## 触发信号

- 新供应商引入满6个月 / 里程碑节点到期 / 供应商主动询问转正 / 持续不达标 / 表现出色可能提前转正

## 问题拆解

1. **达标判断**：数据在边界附近时，怎么判断"达标"？
2. **趋势判断**：即使当前不达标，趋势是改善还是恶化？趋势比绝对值更重要。
3. **适配判断**：整体不达标但在某产线有亮点 — 清退还是调整分工？

## 6月里程碑标准

> 明确基线，所有判断都以此为锚。

| 阶段 | 时间 | 核心指标 | 达标标准 | 判断方法 |
|------|------|---------|---------|---------|
| **引入期** | 第1-2月 | 出勤率 | >=85% | 月均值，不取单日极值 |
| **成长期** | 第3-4月 | GMV | >=同产线中位数的60% | 和同期引入+现有供应商对比 |
| **稳定期** | 第5-6月 | 排名 | 进入前70% | 同产线所有供应商排名 |

**补充指标**（影响综合判断）：转化率（与均值差<10%正常）、质检得分（低于合格线=一票否决）、人员稳定性（月流失>15%关注）、管理配合度。

## 归因路径（决策树）

### 第一层：硬性指标判断

| 检查项 | 达标 | 边界区间 | 不达标 |
|--------|------|---------|--------|
| 质检合格线 | 通过 | — | 一票否决→清退评估 |
| 合规红线 | 通过 | — | 一票否决→清退评估 |
| 出勤率（引入期） | >=85% | 80%-85%，看趋势 | <80%→延期或清退 |
| GMV（成长期） | >=中位60% | 50%-60%，看趋势 | <50%→延期或清退 |
| 排名（稳定期） | 前70% | 70%-80%，综合判断 | 后20%→延期或调整 |

### 第二层：边界区间判断（"差不多"怎么办）

**边界判断四维模型**：

| 维度 | 看什么 | 积极→给机会 | 消极→不给机会 |
|------|--------|-----------|-------------|
| **趋势方向** | 最近4周涨还是跌 | 上涨 | 下跌 |
| **趋势速度** | 每周改善多少 | >=1个百分点/周 | <1个百分点/周 |
| **归因清晰度** | 不达标原因能否说清 | 能说清且有对策 | 说不清 |
| **管理配合度** | 供应商配合表现 | 积极整改、主动沟通 | 推诿拖拉 |

| 四维积极数 | 结论 | 动作 |
|-----------|------|------|
| 3-4个 | "差不多"可以算达标 | 转正，设观察期指标 |
| 2个 | 有亮点但不稳定 | 延期2个月，设具体复查节点 |
| 0-1个 | 基本不达标 | 不转正，评估是否清退 |

**示例**：出勤率83%（标准85%），4周从79%涨到83%（趋势积极），前期扩编新人多现已过培训期（归因清晰），供应商主动报数据积极配合（配合度积极）→ 3维积极 → 转正，设2月观察期要求>=87%。

### 第三层：决策输出

**路径A — 全部达标 → 转正**：出具评估报告 → 向上请示（附报告+建议转正）→ 通知供应商 → 调整分量 → 纳入正式管理（月度排名/季度评估/年度考核）。

**路径B — 部分达标有改善趋势 → 延期**：出具延期报告 → 设具体改善要求（"8月底前出勤率>=87%"而非"提升出勤率"）→ 设复查日期（"9月1日复查"而非"8月底前"）→ 通知供应商 → 向上报备 → 每周跟踪。

**路径C — 关键指标不达标且无改善 → 不转正**：出具评估报告 → 评估是否清退或调整 → 向上请示 → 通知供应商后续安排。

**路径D — 有亮点但整体不达标 → 调整分工**：按产线拆分数据 → 判断适合其他产线 → 设计调整方案 → 向上请示 → 通知供应商 → 重新设里程碑。

## 延期管理铁律

| 铁律 | 怎么做 |
|------|--------|
| 延期不超过2个月 | 到期必须给结论 |
| 必须设具体改善要求 | "出勤率>=87%"而非"提升出勤率" |
| 必须设复查日期 | "9月1日复查"，不是"8月底前" |
| 每周跟踪进度 | 不能等复查日才发现没改善 |
| 提前预判结果 | 第3周趋势不好就提前准备不转正方案 |
| 不二次延期 | 延期一次，到期必须给结论 |

## 权衡因素

| 正方（宽松/转正） | 反方（严格/不转正） |
|-------------------|-------------------|
| 培养需要时间，太严会让供应商没信心 | 标准太松以后管理没有威信 |
| 有改善趋势说明方向对了 | 趋势不等于结果，要看绝对值 |
| 清退成本高，重新引入更贵 | 不清退不合格供应商是对达标者的不公平 |
| 供应商有亮点可调整产线 | 调整产线是额外管理成本 |

**决策原则**：有改善趋势+归因清晰+管理配合→倾向宽松；无改善+归因不清+管理推诿→倾向严格；介于两者→延期2个月。

## 叙事草稿

### 向上汇报

**转正建议**：

> "XX供应商6月里程碑评估完成。出勤率X%（标准85%），GMV达到中位X%（标准60%），排名第X（标准前70%）。全部达标。管理配合度[具体表现]。建议转正，初始正式分量X%。"

**延期建议**：

> "XX供应商6月里程碑评估完成。出勤率达标，GMV达到中位X%略低于标准，排名处于边界区间。最近4周GMV每周提升X个百分点，趋势向上。不达标原因[具体原因]，已有改善对策。建议延期2个月至[日期]，改善要求：GMV达到中位X%。"

**不转正建议**：

> "XX供应商6月里程碑评估完成。出勤率X%、GMV中位X%、排名第X，均不达标，且最近4周无改善趋势。归因：[具体原因]。建议不转正，[清退/调整产线]。"

### 对供应商

**转正**："恭喜，6个月考核通过了。数据和日常管理都比较满意。下周出正式通知，分量会做调整。继续保持。"

**延期**："整体方向对，但[具体指标]还差一点。给2个月延期，[具体要求]。[复查日期]再看，达标就转正。这两个月我每周和你同步数据。"

**不转正**："评估结果不太理想。[具体指标]没达到标准，最近也没有明显改善趋势。[后续安排]。"

## 常见误区

| 误区 | 正确做法 |
|------|---------|
| "差不多就行了"给转正 | 边界可灵活，但要有明确判断逻辑 |
| 延期不给具体要求 | 延期必须绑定改善要求和复查日期 |
| 不转正但不给后续安排 | 不转正=清退/调整/缩减，必须给方向 |
| 只看最后阶段数据 | 三个阶段都要看，前面达标后面不达标=趋势恶化 |
| 忽略管理配合度 | 配合度是隐性权重，转正后管理成本极高 |
| 复查日才看数据 | 每周跟踪，第3周预判结果 |
| 二次延期 | 延期一次，到期必须给结论 |
