# D6 向上汇报叙事

> 把运营一线的观察变成能打动军哥的叙事，不输给策略组的数据叙事。

## 触发信号

- 周会/月会需要汇报供应商运营情况
- 军哥/王易人主动问供应商状态
- 发现需要上级决策的供应商问题
- 季度/年度述职
- 供应商重大事件（引入/清退/转正）需要上级拍板

## 问题拆解

> 策略组汇报的是转化率、核准率、通过率 — 全是数据。你汇报的不能也是数据，因为那是他们的主场。你的主场是"我从运营一线看到的"。

三个核心问题：

1. **叙事差异化**：军哥凭什么听你说而不只是看策略组的数据？你有什么独特视角？
2. **信息密度**：军哥穿透式管理，追问根因。你的叙事能不能经得住连环追问？
3. **决策价值**：你的汇报能不能帮军哥做判断？还是只增加信息噪音？

## 叙事差异化定位

> 策略组看到的是"数字的变化"，你看到的是"数字背后的人在发生什么变化"。

| 维度 | 策略组视角 | 你的视角 | 你的独特价值 |
|------|-----------|---------|-------------|
| 核心指标 | 转化率、通过率、核准率 | 人效、出勤率、人员稳定性 | 人是一切产出的基础 |
| 时间颗粒 | 周度/月度数据 | 日常沟通中的即时信号 | 你比数据早2-4周发现问题 |
| 分析单位 | 产线级、供应商级 | 供应商老板/主管的言行 | 你知道"人"的状态 |
| 预测能力 | 基于历史数据趋势 | 基于一线观察预判 | 你能说"我预判这家3个月后会出问题" |
| 解决方案 | 策略调整（调名单、调模型） | 管理干预（约谈、分量调整、赋能） | 你有落地手段 |

**一句话定位**：策略组告诉你"发生了什么"，你告诉军哥"为什么发生、会怎样、我准备怎么办"。

## 叙事框架

### 核心结构（四层叙事）

> 军哥的汇报偏好：结果 + 根因 + 行动 + 时间线。你的叙事在这个基础上加一层"一线观察"。

| 层次 | 内容 | 目的 | 军哥的关注点 |
|------|------|------|-------------|
| **第一层：结论先行** | 一句话说清楚结果和判断 | 让军哥30秒内抓住重点 | "所以呢？" |
| **第二层：一线观察** | 你看到的数据之外的信号 | 展示独特信息源 | "你怎么知道的？" |
| **第三层：根因分析** | 数据 + 观察综合判断 | 经得住追问 | "为什么？" |
| **第四层：建议 + 预判** | 你的方案和预期效果 | 帮军哥做决策 | "你准备怎么办？" |

### 叙事模板

**模板一：供应商状态汇报（月度常规）**

> **结论**：X月供应商整体运行[稳定/有压力/需关注]。[排名前3家]表现[情况]，[排名后3家]我判断[预判]。
>
> **观察**：我日常和供应商沟通中发现几个信号——[具体观察]。这些信号在数据上还没完全体现，但我预判[Y周/月]后会反映到[具体指标]上。
>
> **根因**：[数据变化]的根本原因是[原因]，不是表面上的[表面原因]。
>
> **建议**：我建议[具体动作]，预计[效果]。如果[风险]，备选方案是[方案]。

**模板二：问题升级汇报（突发/紧急）**

> **结论**：[供应商名称]出现[问题类型]，目前影响范围[范围]，我的判断是[判断]。
>
> **事实**：[具体发生了什么，含时间、人数、金额]。数据层面[数据]，我一线了解的情况是[观察]。
>
> **根因**：初步判断原因是[原因]，是[能力问题/结构问题/管理问题]。不是[错误归因]。
>
> **行动**：我已经做了[已采取的行动]，下一步[计划]，预计[时间线]解决。需要上级支持的是[具体事项]。

**模板三：决策建议（引入/清退/转正/调价）**

> **结论**：建议[决策内容]。理由是[核心理由]。
>
> **数据支撑**：[排名/出勤/转化率/人效等具体数据]。
>
> **一线判断**：数据之外，我观察到的[信号/风险/机会]。如果只看数据可能[误判]，但结合一线情况我认为[判断]。
>
> **方案**：[方案A] vs [方案B]，我推荐[A]，因为[理由]。预计效果[效果]，风险[风险]。

## 不同对象的叙事差异

### 对军哥（战略视角）

> 军哥关心：趋势、风险、投入产出。不喜欢细节罗列，喜欢穿透到根因。

| 要素 | 做法 | 禁忌 |
|------|------|------|
| 开场 | 一句结论 + 核心数据 | 不要铺垫、不要过程 |
| 展开 | 趋势变化 + 根因判断 | 不要罗列做了什么 |
| 风险 | 主动说风险 + 你的应对 | 不要藏着掖着 |
| 建议 | 给明确建议，不给选项堆砌 | 不要说"请领导指示" |
| 预判 | 说"我预计X周后会..." | 不要只说当下 |
| 数据 | 关键数据2-3个，不堆砌 | 不要超过5个数字 |

**军哥追问预判清单：**

| 追问方向 | 军哥典型话术 | 你的准备 |
|---------|-------------|---------|
| 追根因 | "为什么会这样？" | 至少准备2层归因 |
| 追责任 | "这个问题存在多久了？为什么现在才说？" | 说清楚发现时间和已采取行动 |
| 追方案 | "你打算怎么办？" | 准备1个主方案 + 1个备选 |
| 追时间线 | "多久能解决？" | 给具体周数，不说"尽快" |
| 追数据 | "数据呢？" | 关键数据张口就来 |

### 对王易人（执行视角）

> 王易人关心：进展、卡点、需要他出面的事。更实操，更细节。

| 要素 | 做法 |
|------|------|
| 开场 | 进展概述 + 关键节点完成情况 |
| 展开 | 哪些顺利、哪些卡住、卡在哪里 |
| 请求 | 明确说"需要你做什么"——约谈陪同、审批、资源协调 |
| 时间线 | 具体到某天，不说"这周" |
| 后续 | 下一步动作 + 下次同步时间 |

## 叙事弹药准备

> 汇报前15分钟准备清单。

**必带数据：**

| 数据项 | 来源 | 用途 |
|--------|------|------|
| 月度排名变化（环比） | 排名表 | 趋势 — 谁在升谁在降 |
| 核心指标异动（出勤/转化/人效） | 运营数据 | 根因 — 异动说明什么 |
| 供应商沟通摘要（最近2周） | 你的沟通记录 | 观察 — 数据之外的信号 |
| 待决策事项清单 | 待办列表 | 行动 — 需要上级做什么 |
| 上次汇报承诺的进展 | 上次汇报记录 | 闭环 — 答应的事做到哪了 |

**观察信号整理：**

> 你的独特优势是日常沟通积累的观察。汇报前整理这些：

| 信号类型 | 例子 | 怎么转化成叙事 |
|---------|------|---------------|
| 供应商老板态度变化 | 原来主动找你，最近很安静 | "我注意到XX最近没有主动联系我，结合他们排名下降趋势，我预判可能在考虑退出或减少投入" |
| 人员流动信号 | 听说某供应商核心主管在招人被拒 | "XX供应商核心主管上个月提出离职但被挽留了，结合他们出勤率连续2月下降，管理层稳定性需要关注" |
| 行业信息 | 听说某竞品在大量挖人 | "最近行业内XX公司在大量招聘电销人员，可能导致我们供应商人员流失加剧，建议提前做梯队备份" |
| 供应商之间的对比 | A家和B家同时扩编但效果差很多 | "同样扩编50人，岐力2周就到位了，翰锐还在招，管理能力的差距在扩编期暴露得最明显" |

## 权衡因素

| 正方（多说） | 反方（不说/少说） |
|-------------|-------------------|
| 主动汇报显示你在掌控 | 说太多显得管理不稳 |
| 一线观察能帮你建立议题设定权 | 观察未经验证就上报可能翻车 |
| 建议让军哥觉得你能做决策 | 建议太多显得越位 |
| 预判对了加分巨大 | 预判错了扣分更大 |
| 问题早上报比晚上报安全 | 小事也报显得没判断力 |

**核心平衡点**：

- 有数据支撑的观察 → 大胆说
- 纯直觉但信号强烈 → 说但要标注"我初步判断，需要进一步验证"
- 不确定的事 → 不在正式汇报说，在非正式场合提

## 常见误区

| 误区 | 后果 | 正确做法 |
|------|------|---------|
| 照着数据念 | 和策略组没区别，你的价值被忽略 | 数据是配角，你的判断是主角 |
| 罗列做了什么 | 军哥不关心过程，关心结果 | 先说结果，过程只在追问时展开 |
| 不给建议 | 军哥觉得你没有判断力 | 给明确建议，哪怕说"我倾向于A" |
| 模糊时间线 | "尽快""下周" | 给具体日期，"4月25日前完成" |
| 藏问题 | 问题爆发后比提前说的后果严重10倍 | 有问题早说，带上你的应对方案 |
| 只报好消息 | 军哥会质疑你的信息完整度 | 好的坏的都说，但要说清楚你在处理 |
| 和策略组抢数据叙事 | 你抢不过他们 | 补充他们的盲区，而不是重复他们的数据 |

| 场景 | 对应模板 | 需要填的数据字段 |
|------|---------|-----------------|
| 月度供应商运营汇报 | `supplier-evaluation-full.md` → 月度评估通报 | 排名变化、核心指标、观察信号、建议 |
| 问题升级汇报 | `internal-coordination.md` → 决策请示 | 问题描述、影响范围、根因分析、建议方案 |

| 日常非正式同步 | 无需模板，口头/消息 | 关键1-2个信息点 |
