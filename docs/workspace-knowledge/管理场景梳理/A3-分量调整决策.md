# A3 分量调整决策

> 用5维打分决定给谁加量、给谁减量，不凭感觉凭数据。

## 触发信号

以下任一信号出现即启动本模板：

| 信号类型 | 具体表现 |
|---------|---------|
| **产能富裕** | 供应商产能利用率持续>85%且排名前30%，有余力接更多量 |
| **产能吃紧** | 供应商产能利用率<70%或连续排名后30%，量给多了消化不了 |
| **新产线/新产品上线** | 需要重新分配各供应商的产线占比 |
| **供应商准入/退出** | 新供应商引入或老供应商清退，需要重新平衡分量 |
| **季度分量规划** | 每季度例行评估，根据综合表现调整 |
| **业务量变化** | 某产线总量增加或减少，需要按比例调整 |

## 核心问题

> 把量给谁才能实现整体产出最大化、风险最小化？

## 思考步骤

### 第1步：明确调整背景

**想什么**：为什么现在要调分量？是常规规划还是应急调整？

**检查什么数据**：
- 触发信号属于哪一类
- 涉及哪条产线（金条/企金/信用卡/财富）
- 涉及多少量（具体GMV或座席数）
- 时间要求（立即/本周/本月/下季度）

**判断什么**：

| 调整类型 | 时间要求 | 决策权限 |
|---------|---------|---------|
| 常规季度调整 | 按季度节奏 | 自主执行，抄送王易人 |
| 应急增减量 | 1-2周内 | 自主执行 |
| 新产线分配 | 按项目节奏 | 需王易人确认 |
| 重大调整（涉及>20%供应商） | 视情况 | 需王易人确认 |

### 第2步：跑5维打分

**想什么**：用统一的评分框架横向对比所有候选供应商，不凭印象。

**5维打分标准**：

| 维度 | 权重 | 打分依据 | 数据来源 |
|------|------|---------|---------|
| **业绩** | 30% | 月度排名位置、GMV达成率、人均产出 | 月度排名表 |
| **质检客诉** | 25% | 质检得分、客诉率、合规合格率 | 质检报告 |
| **上人** | 15% | 招聘到位速度、座席到位率、培训通过率 | HR数据 |
| **配合** | 15% | 响应速度、问题解决效率、数据提交及时性 | 日常记录 |
| **战略** | 15% | 产线覆盖能力、扩项潜力、抗风险能力 | 综合评估 |

**检查什么数据**：
- 每个候选供应商在5个维度的具体数据
- 近3个月趋势（不是只看一个月）

**打分方式**：
- 每个维度1-5分（1=很差，3=达标，5=优秀）
- 加权总分 = 各维度分数 × 权重之和
- 总分范围：1.0 - 5.0

### 第3步：看产能利用率

**想什么**：打分高不一定能接住更多的量。产能是硬约束。

**检查什么数据**：
- 各供应商当前座席数 + 实际使用数
- 峰值利用率（当前产出 / 自身历史峰值）
- 招聘中人数 + 预计到位时间

**判断什么**（产能利用率双维度）：

| 自身峰值比 | 头部差距 | 类型 | 分量决策建议 |
|-----------|---------|------|-------------|
| 高 | 高 | 标杆型 | 优先加量，能消化 |
| 高 | 低 | 尽力型 | 维持或小幅加量，人效天花板在产线层面 |
| 低 | 低 | 问题型 | 减量，把量释放给更高效的供应商 |
| 低 | 高 | 暂时波动 | 维持，观察1-2个月再决定 |

### 第4步：评估风险平衡

**想什么**：不能把量全压给一家。集中度风险怎么控？

**检查什么数据**：
- 当前各供应商分量占比
- 调整后的占比预测
- 每条产线的供应商数量

**判断什么**：

| 风险维度 | 红线标准 | 处理方式 |
|---------|---------|---------|
| **单家集中度** | 单家供应商占该产线总量>40% | 分散到第二、第三家 |
| **产线集中度** | 某供应商>80%产能集中在单一产线 | 适度分散到其他产线 |
| **区域集中度** | 某区域供应商总占比>50% | 引入其他区域供应商平衡 |
| **依赖度** | 某供应商退出后影响>15%总产出 | 必须有备用供应商 |

### 第5步：设计调整方案

**想什么**：加量给谁、减量从谁减、过渡期怎么安排？

**检查什么数据**：
- 5维打分排名
- 产能利用率类型
- 风险平衡约束
- 涉及供应商的当前分量和调整后分量

**方案结构**：

| 项目 | 内容 |
|------|------|
| 调整范围 | 哪条产线、哪些供应商 |
| 加量方 | 供应商名 + 增加量 + 理由（5维得分+产能类型） |
| 减量方 | 供应商名 + 减少量 + 理由 |
| 过渡期 | 分几步到位、每步时间节点 |
| 风险评估 | 集中度变化、对总产出的影响 |
| 回退条件 | 如果加量方接不住，什么信号触发回退 |

**判断什么**：
- 调整后是否满足风险红线
- 过渡期是否给足供应商准备时间（加量至少给2周招人，减量提前1个月通知）
- 有没有"试用期"机制（先加一部分，2周后看数据再加剩余）

### 第6步：验证和确认

**想什么**：方案是否经得起推敲？上级是否需要审批？

**检查什么数据**：
- 调整前后各供应商排名预测
- 对整体GMV的影响测算
- 历史类似调整的执行效果

**判断什么**：
- 是否需要模拟测算（用历史数据推演调整效果）
- 权限确认：
  - 常规调整 → 自主执行，抄送王易人
  - 涉及>20%供应商或产线重新分配 → 王易人确认
  - 清退相关调整 → 王易人确认 + 事前沟通军哥

### 第7步：制定通知和跟踪计划

**想什么**：调整决定后怎么通知、怎么跟踪？

**设计什么**：

| 动作 | 时间 | 形式 |
|------|------|------|
| 内部确认 | 方案确定后 | 报备王易人（如需） |
| 通知增量供应商 | 调整前2周 | 正式通知 + 电话 |
| 通知减量供应商 | 调整前1个月 | 正式通知 + 约谈 |
| 执行调整 | 按计划节点 | 系统配置变更 |
| 数据验证 | 调整后2周 + 1个月 | 对比产能/人效/排名 |

## 关键变量

| # | 变量 | 判断标准 | 用途 |
|---|------|---------|------|
| 1 | **5维加权总分** | ≥4.0优先加量 / ≤2.5考虑减量 | 核心排序依据 |
| 2 | **产能利用率** | ≥85%有余力 / <70%消化不了 | 硬约束条件 |
| 3 | **单家集中度** | ≤40%为安全线 | 风险红线 |
| 4 | **调整幅度** | 单次≤该供应商当前量的30% | 控制冲击 |
| 5 | **过渡期** | 加量≥2周 / 减量≥1个月 | 操作安全 |

## 常见误区

| 误区 | 正确做法 |
|------|---------|
| 凭印象给量 | 必须跑5维打分，用数据说话 |
| 只看业绩不管产能 | 毅航业绩好但产能到顶，加不了多少 |
| 减量不提前通知 | 至少提前1个月通知 |
| 一把到位不分步 | 大调整分2-3步，每步验证后再继续 |
| 忽略集中度 | 毛毛虫占某产线45%很危险，需分散 |
| 只加不减 | 从低效供应商收回量才能给高效供应商 |
| 不设回退条件 | 必须"2周数据不行就回退"的止损线 |

## 决策输出

> **调整背景**：金条产线Q2总量增加15%，需要分配增量
> **候选供应商**：毅航、毛毛虫、伽玛、赛维斯、岐力、翰锐、华啸
> **5维打分排名**：1.毅航4.2 2.伽玛3.9 3.赛维斯3.7 ... 7.华啸2.3
> **产能判断**：毅航标杆型（高/高）可加量；伽玛尽力型小幅加；华啸问题型减量
> **调整方案**：毅航+15%（5维第1+标杆型）| 伽玛+10%（产线覆盖广）| 华啸-20%（末位+问题型）
> **过渡期**：分两步4月底+5月中 | 集中度：毅航28%→32%安全线内
> **权限**：涉及3家（<20%），自主执行，抄送王易人
> **回退条件**：2周后毅航产能利用率>95%且人效下降，回退50%增量
