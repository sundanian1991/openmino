# A6 清退评估

> 供应商触达清退触发条件时，系统评估是否启动清退、如何执行、谁来决策。

## 触发信号

- 连续3个月排名后30%
- 触碰诚信红线（伪造数据、私下收费、泄露隐私、虚报人员）
- A5合规定级为高危且整改验收不合格
- G项评估指标出现红色（季度评估）
- 王易人或军哥指示评估某供应商

## 核心问题

> 清退的必要性是否成立？退出方案是否可行？风险是否可控？

## 思考步骤

### 步骤1：确认触发条件是否充分

**想什么**：清退是重大决策，触发条件必须硬。

**检查什么数据**：
- 排名数据：连续3个月在后30%的具体排名和月份
- 合规记录：红线行为的证据链（完整、可追溯）
- 评估记录：G项红色的具体指标和评分
- 整改历史：之前是否给过整改机会、整改效果如何

**判断什么**：
- 排名触发 → 确认是否"连续"3个月（中间有1个月跳出后30%就不算连续）
- 红线触发 → 证据是否确凿（单一来源证据不够，需要交叉验证）
- 整改不合格 → 之前的整改要求是否合理、是否给了足够时间
- 结论：触发条件充分 → 继续；不充分 → 补充数据或调整策略

### 步骤2：评估退出可行性

**想什么**：能退是一回事，退得掉是另一回事。先看退出有没有硬障碍。

**检查什么数据**：
- **业务承接**：该供应商当前负责哪几条产线？总量多大？其他供应商能否承接？
- **人员规模**：该供应商有多少人在场？这些人怎么安置？
- **合同条款**：合同中关于终止的条款（提前期、违约金、交接要求）
- **结算状态**：有没有未完成的结算？有没有预付款？
- **客户影响**：退出是否会导致客户服务中断？

**判断什么**：
- **可以直接退**：业务量小、其他供应商有富余产能、人员在50人以内 → 常规清退
- **需要过渡期**：业务量大（>500万/月）、涉及多条产线、人员在200人以上 → 渐进式退出
- **退不了（暂时）**：唯一供应商产线、没有替代方案、客户绑定期内 → 先管控（减量+冻结扩项），同步培育替代供应商

**具体到年老师管理的供应商**：
- 毅航/毛毛虫/伽玛：头部6家，业务量大，清退需要较长过渡期和替代方案
- 赛维斯/岐力/翰锐：中等规模，退出相对容易但需要确认承接方
- 华啸：看具体产线和业务量判断

### 步骤3：计算清退成本与保留成本

**想什么**：清退有成本，但不清退也有成本。两本账都要算。

**清退成本**：
| 项目 | 计算方法 |
|------|---------|
| 业务迁移成本 | 迁移周期 × 迁移期间的产能损失 |
| 人员安置成本 | 交接验收标准中的人员相关费用 |
| 结算尾款 | 合同约定的终止结算金额 |
| 违约金（如有） | 提前终止的违约金（看谁违约在先） |
| 客户影响成本 | 服务中断期间的客户流失风险 |

**保留成本**：
| 项目 | 计算方法 |
|------|---------|
| 排名拖累 | 后30%供应商对整体数据的拖累幅度 |
| 管理精力 | 额外投入的管理时间（约谈、整改、跟进） |
| 合规风险 | 问题供应商再次违规的概率 × 单次违规损失 |
| 团队士气 | 其他供应商看到"差的也没被清退"的负面影响 |

**判断什么**：
- 清退成本远大于保留成本 → 先管控，条件成熟再退
- 清退成本约等于保留成本 → 看长期趋势，趋势恶化则退
- 保留成本远大于清退成本 → 果断启动清退

### 步骤4：制定退出方案

**想什么**：清退方案要具体到"谁在什么时候做什么"，不留灰色地带。

**方案要素**：
1. **退出时间表**：什么时候通知 → 什么时候停止新增 → 什么时候完成交接 → 什么时候正式退出
2. **业务分配方案**：退出的量分给谁？分几步完成？需要提前和承接方沟通吗？
3. **人员过渡方案**：现有人员是否愿意转签到新供应商？是否有竞业限制？
4. **结算方案**：尾款怎么结？押金怎么退？
5. **风险预案**：供应商不配合怎么办？交接期间出问题怎么办？

**决策授权**：
| 动作 | 年老师权限 | 需王易人确认 | 需军哥确认 |
|------|-----------|-------------|-----------|
| 清退评估启动 | 建议 | 确认 | 事前沟通 |
| 清退方案制定 | 主导 | 审核 | — |
| 约谈供应商高层 | 建议 | 必须陪同 | 事前报备 |
| 正式启动清退 | — | 确认 | 确认 |

### 步骤5：准备汇报材料

**想什么**：清退决策需要层层审批，材料必须完整、有说服力。

**材料清单**：
1. 触发条件证据（排名数据/红线证据/评估结果）
2. 整改历史记录（给过几次机会、每次整改效果）
3. 可行性评估（退出方案 + 成本对比）
4. 风险预案（最坏情况 + 应对措施）
5. 业务承接方确认（承接方同意书或产能确认）

**判断什么**：
- 材料完整 → 向王易人汇报
- 材料有缺口 → 先补齐再汇报，不拿半成品去审批

### 步骤6：执行交接与闭环

**想什么**：审批通过后，执行阶段同样需要严密跟进。

**检查什么数据**：
- 交接进度：按时间表逐项核对
- 业务承接：新供应商是否正常接手、数据是否平稳过渡
- 结算完成：尾款是否结清
- 人员安置：人员是否妥善处理

**判断什么**：
- 交接顺利 → 出清退完成报告，归档
- 交接出现争议 → 启动争议升级流程（`punishment-dispute.md`）
- 供应商不配合 → 记录证据，必要时法律手段

## 关键变量

| # | 变量 | 判断标准 | 数据来源 |
|---|------|---------|---------|
| 1 | 触发条件充分性 | 排名连续性/红线证据链/整改记录 | 月度排名+合规档案 |
| 2 | 退出可行性 | 替代产能是否充足、人员规模、合同约束 | 产能数据+合同文本 |
| 3 | 清退vs保留成本 | 量化的迁移成本对比管理拖累成本 | 财务数据+管理记录 |
| 4 | 业务承接方 | 是否有供应商愿意且能够承接 | 产能摸底 |
| 5 | 决策链路 | 按授权矩阵确认需要谁审批 | 决策授权矩阵 |

## 常见误区

1. **只看排名不看原因** — 连续3个月后30%可能是分量不够、产线不对口、系统问题。清退前必须排除这些因素
2. **低估人员安置难度** — 200人的供应商退出了，这些人怎么处理？不考虑清楚会引发劳动纠纷
3. **没有替代方案就启动清退** — 清退了但没人接，业务断了，客户投诉，最后还得请回来。更丢人
4. **审批材料只写问题不写方案** — 跟王易人说"这家供应商不行要清退"是废话，必须带着方案去
5. **忽略供应商反制** — 供应商收到清退通知后可能消极怠工、截留客户、煽动人员闹事。方案里必须有预案
6. **清退完不归档** — 清退的完整记录是未来类似决策的参考，也是可能的争议证据

## 决策输出

> **示例：岐力连续3个月排名后30%清退评估**
>
> **触发条件**：岐力1-3月排名分别为第27、29、28名（32家中的后30%为第23名及之后），连续3个月满足条件
>
> **排除检查**：
> - 分量：金条产线分量与翰锐相当，非分量不足
> - 产线匹配：岐力主营金条，无产线错配
> - 整改记录：2月已发整改通知，3月整改验收3项未完成（承诺完成率55%<70%达标线）
>
> **可行性评估**：
> - 业务量：月均GMV约300万，占金条产线总量约4%
> - 人员：85人，华啸表示愿意承接其中60人
> - 合同：提前30天书面通知即可终止，无违约金
> - 结算：无预付款，尾款约15万
> - 结论：可直接退，过渡期1个月
>
> **成本对比**：清退成本（迁移1个月+尾款15万）< 保留成本（持续管理拖累+合规风险+团队士气影响）
>
> **退出方案**：
> 1. 第1周：向王易人汇报清退方案 → 军哥事前沟通
> 2. 第2周：约谈岐力高层（王易人陪同）
> 3. 第3-4周：业务交接，华啸承接
> 4. 第5周：结算完成，出清退完成报告
>
> **风险预案**：岐力不配合交接 → 停止结算+争议升级至军哥
>
> **审批**：年老师建议 → 王易人确认 → 军哥事前沟通
