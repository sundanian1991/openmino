# D3 供应商来电/来消息

> 30秒判断对方要什么、该给什么回应、要不要升级

## 触发信号

- 供应商老板/主管电话打来
- 微信/企微消息主动找你
- 供应商在工作群里@你
- 供应商通过邮件发来请求

## 核心问题

> 对方此刻的**真实意图**是什么？是例行沟通、试探底线、还是真有事？你能当场回复还是需要收集信息再答？

## 30秒快速判断

**谁在找？**

| 来电人 | 权重 | 备注 |
|--------|------|------|
| 供应商老板 | 高 | 日均1h沟通对象，通常聊业务方向 |
| 项目主管 | 中 | 多数是具体问题，能直接判断 |
| 一线员工 | 低 | 一般不应直接找你，先问"这事你主管知道吗" |

**要什么？** 对照以下6类快速归类：

| 类型 | 识别信号 | 典型话术 |
|------|---------|---------|
| 要量 | 提"产能""饱和""能不能加点" | "最近量不太够啊" |
| 要支持 | 提"系统""流程""能不能帮忙" | "这个审批卡住了" |
| 反映问题 | 提"客户投诉""质检""异常" | "今天出了个差错" |
| 试探/撒娇 | 提"别人家""听说""不太公平" | "听说XX那边量更多" |
| 投诉/施压 | 提"不合理""你们""要反映" | "这个考核标准有问题" |
| 走近关系 | 闲聊、约饭、问近况 | "最近忙不忙，一起吃个饭" |

**紧急吗？**

| 信号 | 紧急度 | 回复时效 |
|------|--------|---------|
| 涉及在途客户/在催案件 | 紧急 | 立即处理 |
| 系统故障影响作业 | 紧急 | 立即处理 |
| 数据安全/合规风险 | 极紧急 | 立即处理+同步王易人 |
| 日常业务咨询 | 常规 | 当天回复 |
| 分量/资源请求 | 常规 | 1-3个工作日 |
| 关系维护类 | 低优 | 当周内安排 |

**能回答吗？** 对着授权矩阵判断：

| 情况 | 回应方式 |
|------|---------|
| 权限内（日常沟通、首次预警） | 当场回复 |
| 需确认（约谈结果、分量调整） | "我确认一下，回头给你准信" |
| 需上级（更换主管、清退评估） | "这个需要内部讨论，我反馈上去" |
| 不确定 | "我先了解情况，XX时间前回复你" |

**一句话回复口径**（30秒必须给出去）：

> "收到，我看看/我确认一下，[具体时间]前给你回复。"

绝不当场答应具体数字或承诺，除非你100%确定。

## 5分钟深入版

如果判断需要进一步思考，按以下步骤快速展开：

**Step 1：还原背景**
- 该供应商最近排名？前30%/中段/后30%？
- 最近一次沟通聊了什么？有什么承诺未兑现？
- 当前分量占比？是否有扩缩计划？

**Step 2：判断真实意图**
- 表面说A，实际要B的情况很常见
- 毅航说"最近人不好招" → 实际可能在铺垫要减量或降指标
- 毛毛虫说"别的渠道量更大" → 试探你能不能给更多分量
- 赛维斯说"客户太刁" → 可能在为后续质检扣分找借口

**Step 3：准备回复弹药**
- 要量 → 查当前排名、产能利用率、同类供应商分量对比
- 要支持 → 确认是系统问题还是流程问题，找对应接口人
- 反映问题 → 问清时间/人数/金额/已采取措施，记录下来
- 投诉 → 听完不反驳，记录要点，不当场定性

**Step 4：确定回复策略**

| 意图类型 | 回复策略 |
|---------|---------|
| 要量（合理） | 看排名和产能，有空间就给，没空间就说清楚原因 |
| 要量（无理） | 拿数据说话——排名、转化率、质检分，不凭感觉 |
| 要支持 | 能帮就帮，但记下来防止变成常态性依赖 |
| 反映问题 | 先肯定"及时反馈是对的"，再核实，再处理 |
| 投诉/施压 | 不硬顶，不退让，记录+升级 |
| 试探 | 不接茬，用数据回应，不给模糊承诺 |
| 走近关系 | 正常社交即可，不聊业务敏感信息 |

## 思考步骤

### Step 1：识别来电性质（10秒）

> 想：对方为什么此刻找你？
> 检查：最近有没有和该供应商相关的事件（排名变化、分量调整、质检问题）
> 判断：例行沟通 / 有事相求 / 情绪宣泄 / 紧急求助

### Step 2：评估信息充分度（10秒）

> 想：你能当场回答吗？
> 检查：是否需要查数据（排名、分量、结算）、是否需要确认上级意见
> 判断：能当场回 / 需要XX时间收集信息 / 需要上级确认

### Step 3：确定回应底线（10秒）

> 想：这个事情你能答应的底线在哪？
> 检查：授权矩阵——你的权限边界
> 判断：给明确答复 / 给倾向性意见+承诺确认时间 / 升级处理

### Step 4：组织回复内容（30秒）

> 想：怎么回复既不承诺又给对方信心？
> 检查：有没有可引用的数据/政策/先例
> 判断：回复话术+跟进时间节点

### Step 5：记录+跟进

> 想：这次沟通需要留痕吗？后续动作是什么？
> 检查：是否需要写入沟通记录、是否触发下一步动作
> 判断：随手记 / 需要出正式文档 / 需要安排后续会议

## 关键变量

| 变量 | 判断标准 | 影响 |
|------|---------|------|
| **供应商当前排名** | 前30% / 中段 / 后30% | 决定你在沟通中的底气 |
| **来电人级别** | 老板 / 主管 / 一线 | 决定沟通深度和信息密度 |
| **历史承诺状态** | 有未兑现承诺 / 无 | 有未兑现时不能做新承诺 |
| **影响范围** | 单供应商 / 涉及其他供应商 / 涉及产线 | 涉及面越广越不能当场拍板 |
| **可量化程度** | 有明确数据 / 主观感受 | 能量化就量化，不能量化就先记录 |

## 常见误区

| 误区 | 后果 | 正确做法 |
|------|------|---------|
| 当场答应具体数字 | 后面反悔更被动 | "我确认一下，XX时间前回复" |
| 被情绪带跑跟着急 | 判断失准 | 先听，记录，不急着表态 |
| 只听一面之词就定性 | 决策偏差 | 记录要点，核实后再判断 |
| 觉得小事不记录 | 后续无据可查 | 通话/消息后30秒内记要点 |
| 拖着不回复 | 供应商觉得你不重视 | 30秒内给一句"收到"，再跟进 |

## 决策输出

**场景：毛毛虫老板来电说量不够**

> 30秒判断：
> - 谁在找：毛毛虫老板（重点供应商，日均1h沟通对象）
> - 要什么：要量（说量不够）
> - 紧急吗：常规（不是紧急事项）
> - 能回答吗：需要查数据——毛毛虫当前排名、产能利用率、金条产线分量分配
> - 回复口径："收到，我看一下你们最近的数据和排名情况，今天下班前给你回复。"
>
> 挂电话后：
> 1. 查毛毛虫最近月度排名（前30%？中段？）
> 2. 查毛毛虫当前分量占比 vs 产能利用率
> 3. 对比同类供应商（翰锐、岐力）的分量
> 4. 判断：有空间就加，没空间就说明原因+给时间表
> 5. 如涉及产线分量调整 → 用 `supplier-allocation.md` 出增量通知

**场景：赛维斯主管反映质检扣分不公平**

> 30秒判断：
> - 谁在找：赛维斯主管（重点供应商）
> - 要什么：投诉（质检扣分不公平）
> - 紧急吗：不紧急但敏感（涉及考核）
> - 能回答吗：不能当场定性，需要核实
> - 回复口径："我记下来了，回头核实一下质检记录，这两天给你一个说法。"
>
> 挂电话后：
> 1. 查赛维斯当月质检记录和扣分明细
> 2. 对比质检标准，判断扣分是否合理
> 3. 如确有不公 → 内部纠正+给赛维斯回复
> 4. 如扣分合理 → 拿着数据和赛维斯解释，不接受申诉的话走 `punishment-dispute.md` 流程
