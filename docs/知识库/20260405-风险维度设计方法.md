# 风险维度设计方法

> **来源**：供应商站点看板 V6 需求梳理（2026-04-04）
> **类型**：playbook
> **标签**：#供应商管理 #数据可视化 #风险监控

---

## 核心洞察

风险维度必须从**管理问题**出发，不是技术堆砌。

错误做法：先想"有哪些技术可以实现"，再凑风险维度
正确做法：先问"管理者在担心什么"，再找数据支撑

---

## 设计流程

```
四大管理角度 → 管理问题 → 风险维度 → 数据可行性 → 去重合并 → 最终维度
```

---

## 第一步：列出四大管理角度

| 角度 | 管理目标 | 示例问题 |
|------|---------|---------|
| **站点管理** | 确保每个站点健康、稳定、可持续运营 | 站点太小不经济？太大有风险？不稳定？ |
| **人员管理** | 确保人力配置合理、流失可控、效率达标 | 流失率？人均产出？冗余？ |
| **趋势管理** | 及时发现异常趋势，提前预警 | 持续下降？爆发增长？震荡波动？ |
| **风险管理** | 识别集中度风险、供应链脆弱性、过度依赖 | 过度依赖单一项目？供应链太脆弱？ |

---

## 第二步：将管理问题转化为风险维度

**示例**（站点管理角度）：

| 管理问题 | 风险维度 | 定义 | 阈值 |
|---------|---------|------|------|
| 站点太小不经济 | 微小站点 | 站点人力 <20人 | 0 |
| 站点太大有风险 | 超大站点 | 站点人力 >100人 | 0 |
| 站点不稳定 | 站点波动 | 月度环比波动 >30% | 连续2个月 |

---

## 第三步：检查数据可行性

| 数据项 | 检查方法 | 结果 |
|--------|---------|------|
| RAW数据字段是否齐全 | grep 源文件 | ✅/❌ |
| 聚合引擎是否支持 | grep "function merge" | ✅/❌ |
| 计算逻辑是否可实现 | 检查所需字段是否存在 | ✅/❌ |

**原则**：有数据 → 直接实现；无数据 → 暂不实现（标注"无数据源"）

---

## 第四步：去重合并

**常见重复模式**：

| 重复类型 | 示例 | 解决方案 |
|---------|------|----------|
| 定义相同 | #3"单产品供应"和#5"供应链单薄"都是"某产品线供应商数<3" | 合并为"供应链单薄" |
| 语义重复 | #4"单产品线供应商"和#8"项目单一风险"原定义相同 | 重新定义#8为"最大单一项目占比>70%" |
| 定义方式重复 | 地理集中度有两种定义（站点数/占比） | 选择更直观的一种（站点数） |

---

## 第五步：确定阈值

**阈值设定原则**：

| 类型 | 示例 | 说明 |
|------|------|------|
| 绝对值 | headcount < 20 | 0表示绝对值，直接比较 |
| 相对值 | 占比 > 30% | 百分比，需计算 |
| 时间序列 | 连续2个月下降 | 需要历史数据 |

**验证方法**：
- 用实际数据跑一遍，看阈值是否能触发
- 如果阈值太高/太低导致无法触发，需调整

## 可复用模板

### 风险维度定义表

| # | 风险维度 | 对象 | 定义 | 阈值 | 颜色 |
|---|---------|------|------|------|------|
| 1 | [名称] | 站点/供应商/项目 | [计算逻辑] | [阈值] | 🔴/🟡/🟢 |

### 计算函数模板

```javascript
function calc[Name]Risk(data) {
  const riskyItems = [];
  data.forEach(item => {
    // 计算逻辑
    if (condition) {
      riskyItems.push({
        name: item.name,
        value: calculatedValue,
        threshold: THRESHOLD
      });
    }
  });
  return {
    count: riskyItems.length,
    items: riskyItems,
    threshold: THRESHOLD
  };
}
```

## 避免的陷阱

| 陷阱 | 问题 | 解决方案 |
|------|------|----------|
| 风险维度重复 | 定义混乱、计算重复 | 系统性梳理（四大管理角度） |
| 阈值不合理 | 无法触发或误报 | 用实际数据验证 |
| 只换颜色不换内容 | 看起来有差异，实际一样 | 风险维度必须从管理问题出发 |
| 数据源不支持 | 计算不出来 | 明确标注"无数据源，暂不实现" |
