# 教学材料：任务卡片式布局比信息展示式更有效

当教学内容多且复杂时，不要试图在一个页面里塞下所有信息。改为"问题驱动"的卡片式布局：用户点击自己遇到的问题，直接看到对应的解决方案。

## Context

为AI长周期任务协作方法论制作教学HTML。尝试了两个版本：
1. 旧版：7章理论框架（概述→实验→Agent Loop→5习惯→对比→可迁移框架→工具箱→范式升级）→ 用户反馈"不好理解，不好落地"
2. 新版：5个操作+代码块（建GOALS.md→读文件→更新文件→拆里程碑→三句话汇报）→ 用户反馈"不够清晰"

两次迭代都失败，用户最终指出问题是"呈现方式不对"。

## Implication

设计教学材料时：
1. **先问"用户遇到什么问题"，而不是"我要教什么内容"** — 用户是来解决问题的，不是来听课的
2. **每个问题对应一个解决方案卡片** — 卡片包含：问题描述→原因分析→操作步骤→验证方法
3. **操作步骤要可直接复制** — 用户不想理解原理，只想复制prompt开始用
4. **验证方法很重要** — 用户需要知道"怎么知道做对了"

适用于：任何内容较多的教学材料、操作手册、工具指南。
