# 边界识别三分法：可控/灰色/不可控

## 核心方法

在权责不对等的环境中，将所有事项分为三类：**不可控**、**灰色**、**可控**。聚焦可控，接受不可控。

## 三类边界

### 不可控（不要试图改变）

| 领域 | 具体事项 | 原因 |
|------|----------|------|
| 组织架构 | 策略组vs服务组分工 | 既定结构 |
| 权力分配 | 定价权、名单权、数据分配权 | 在策略组手中 |
| KPI设定 | 策略组费效比、服务组供应商稳定 | 来自上级 |
| 刘伟佳行为模式 | 责任边界管理、分析外化 | 生存策略 |
| 供应商定价 | 具体供应商结算价格 | 策略组决策 |

**应对策略**:
- 接受这些是不可控的
- 不在沟通中试图改变
- 不暴露"我想改变定价权"的意图

### 灰色（可影响但不能决定）

| 领域 | 具体事项 | 影响方式 | 限度 |
|------|----------|----------|------|
| 供应商关系 | 日常沟通、问题反馈 | 直接管理 | 无法改变结算价格 |
| 供应商能力提升 | 培训、数据使用指导 | 可以提供 | 供应商意愿是变量 |
| 数据呈现 | 如何呈现数据、分析角度 | 可以选择 | 无法改变原始数据 |
| 向上汇报 | 如何向军哥呈现问题 | 可以控制 | 军哥是否重视是变量 |
| 与王易人的协作 | 能否形成共识 | 可以试探 | 她有自己立场 |

**应对策略**:
- 这些领域可以投入精力
- 但要有预期：可能失败
- 失败时不自责

### 可控（完全由你决定）

| 领域 | 具体事项 | 价值 |
|------|----------|------|
| 自己的判断 | 是否接受现实、边界在哪里 | 心理健康 |
| 自己的表达 | 周报怎么写、问题怎么说 | 建立记录 |
| 供应商沟通 | 传达什么信息、管理什么预期 | 减少误解 |
| 数据记录 | 留存什么文档、记录什么决策 | 未来追责有证据 |
| 响应策略 | 遇到问题怎么应对 | 避免被动背锅 |
| 选择战场 | 哪些事争取、哪些事放手 | 精力分配 |
| 与王易人的关系 | 是否建立协作 | 潜在盟友 |

**应对策略**:
- 在这些领域做到极致
- 这是你真正能控制的
- 把精力花在这里

## 关键判断标准

当遇到问题时：

```
1. 决策权在谁？
   策略组/上级 → 不可控
   供应商意愿 → 灰色
   我自己 → 可控

2. 改变需要什么？
   组织架构调整 → 不可控
   权力重新分配 → 不可控
   说服某人 → 灰色
   我自己的行为 → 可控

3. 失败的后果谁承担？
   策略组KPI → 我无能为力
   供应商流失 → 记录原因，不是我一人之责
   我的表达不当 → 我负责
```

## 实际应用

### 场景1: 供应商流失率高

| 分析 | 结论 |
|------|------|
| 不可控: 结算价格（策略组决定） | 不要试图改变定价 |
| 灰色: 供应商关系维护（你能影响） | 继续沟通，但接受结果 |
| 可控: 记录流失原因、建议策略组 | 做好记录，明确边界 |

### 场景2: 被上级追问"为什么供应商流失？"

| 分析 | 回应策略 |
|------|----------|
| 不可控: 定价决策 | 说明根本原因是策略组定价 |
| 灰色: 你已采取的行动 | 列出你已做的努力 |
| 可控: 你的回应方式 | 用REACT框架（见另一学习成果） |

## 常见错误

❌ **在不可控领域挣扎**:
- 试图说服策略组让权
- 期待"多次反馈后会改变"
- 把精力浪费在"推动调整"上

❌ **把灰色当可控**:
- 假设供应商一定会配合你的培训
- 假设军哥一定会重视你提出的问题
- 失败后过度自责

✅ **正确做法**:
- 接受不可控，聚焦可控
- 在灰色领域有预期地尝试
- 在可控领域做到极致
