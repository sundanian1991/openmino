# 手绘 SVG 设计哲学 v2.0

**创建时间**：2026-04-04
**运动名称**：Informed Imperfection（知情的不完美）
**版本**：2.0 — 融入贝塞尔模式库

---

## 宣言

### 核心论点

**"不完美是精心设计的，不是随机的。"**

真正的手绘感不是让 AI 随机生成"看起来像手绘"的线条，而是深入理解手绘的视觉原理，然后用精确的代码实现这种"知情的不完美"。

**v2.0 新增核心**：我们建立了完整的贝塞尔曲线模式库，将"不完美"从抽象概念转化为可执行的参数系统。

---

## 一、空间与形式

### 1.1 双层结构（基础架构）

**填充层 + 描边层 = 手绘感的 DNA**

填充层负责体积和重量，描边层负责手绘的"笔触感"。两层叠加创造深度，就像传统绘画中的底色 + 线条。

```xml
<g id="fill-layer">
  <path d="..." fill="#D6654B"/>  <!-- 体积 -->
</g>
<g id="stroke-layer" fill="none" stroke="#1A1612" stroke-linecap="round">
  <path d="..." stroke-width="5.206"/>  <!-- 笔触 -->
</g>
```

---

### 1.2 有机几何化（v2.0 核心）

**问题**：几何形状（矩形、圆形）是完美的，但手绘不是。

**解决**：所有几何形状必须有机化

#### 贝塞尔微抖动系统

完美的直线和圆角是矢量软件的特征，不是手绘。真正的手绘有轻微的不规则：

```
机械直线（我的画法）：
M 50 50 L 150 50 L 150 150 L 50 50 Z

手绘直线（有机化）：
m52 48 98 4 2 96-96-2-2-95z
    ↑   ↑  ↑  ↑  ↑
    每个控制点都有 ±0.5-1.5px 的随机偏移
```

**参数表**：

| 线条类型 | 控制点数量 | 偏移范围 | 段数 |
|---------|-----------|----------|------|
| 短线（<20px） | 1-2 | ±0.5-1px | 2 |
| 中线（20-50px） | 2-3 | ±0.5-1.5px | 3 |
| 长线（>50px） | 3-4 | ±1-2px | 4-5 |
| 圆角 | 2 | ±0.5-1px | 2-3 |

#### 形状有机化参数

矩形不应该完美，要有"生命力"：

```
机械矩形：
┌─────────────┐
│             │
└─────────────┘

手绘矩形（有机）：
  ┌───────────┐
 ╱             ╲    ← 顶部微微内收（0.92-0.96）
│               │
│               │
 ╲             ╱    ← 底部微微外扩
  └───────────┘
  ↑             ↑
  左右不对称：±1-2px
```

**参数表**：

| 形状类型 | 宽度比 | 高度比 | 控制点偏移 |
|---------|--------|--------|----------|
| 靠背 | 顶/底 = 0.92-0.96 | 左/右 = 0.98-1.02 | ±1-2px |
| 桌面 | 顶/底 = 0.94-0.98 | - | ±1-2px |
| 镜框 | 顶/底 = 0.95-0.99 | 左/右 = 0.99-1.01 | ±0.5-1px |

### 1.3 贝塞尔模式库（v2.0 核心）

我们提取了 5 种可复用的贝塞尔模式：

#### 模式 1：有机直线

```xml
<!-- 短线（<20px）：2 段贝塞尔 -->
<path d="m{x1} {y1} q{x1+0.5} {y1} {x2} {y2}" />

<!-- 中线（20-50px）：3 段贝塞尔 -->
<path d="m{x1} {y1}
     q{x1+0.3*w} {y1-0.5} {x1+0.5*w} {y1+0.5}
     q{x1+0.7*w} {y1+1.5} {x2} {y2}" />
```

#### 模式 2：有机圆角

```xml
<!-- 不是 arc，而是 2-3 段贝塞尔拼接 -->
<path d="m{x2} {y1}
     q{x2+1.5} {y1} {x2+2} {y1+0.5}
     q{x2+2} {y1+1.5} {x2+1.5} {y2}" />
```

#### 模式 3：有机 S 形曲线

```xml
<!-- 不是简单圆弧，而是 3 段贝塞尔的 S 形 -->
<path d="m{x1} {y1}
     q{x1+0.3*w} {y1-2} {x1+0.5*w} {y1}
     q{x1+0.7*w} {y1+2} {x2} {y2}" />
```

#### 模式 4：有机填充

```xml
<!-- 不是椭圆，而是 8-12 段贝塞尔的"土豆形状" -->
<path d="m{x} {y}
     c9-8 12-15 15-15       <!-- 不规则凸起 -->
     c3 0 6 2 8 5          <!-- 微微鼓起 -->
     c-3 3-5 7-8 10         <!-- 不规则凹陷 -->
     ..."/>
```

#### 模式 5：有机对称

```xml
<!-- 不是完美对称，而是"有误差的一致" -->
<!-- 左侧间距：18px, 22px, 19px -->
<!-- 右侧间距：21px, 19px, 20px -->
<!-- 不是镜像，而是独立随机 -->
```

---

### 1.4 stroke-linecap="round"（灵魂）

圆头端点让每条线的终点看起来像毛笔或马克笔的自然收束，而不是矢量软件的机械切断。

```xml
<svg>
  <g fill="none" stroke="#1A1612" stroke-linecap="round" stroke-linejoin="round">
    <path d="..." />
  </g>
</svg>
```

这是区分"手绘风格"和"矢量风格"的唯一最重要的视觉特征。

## 二、色彩与情绪

### 2.1 陶土色系（温暖与专业的平衡）

**东京猫**：#BA3420（深陶土红）+ #FEFFFA（米白）+ #0B0800（墨黑）
**李诞虾**：#D6654B（亮橙陶土）+ #1A1612（深墨黑）

深色更稳重、亮色更活泼。墨黑描边提供对比，就像传统中国画的墨线勾勒。

---

### 2.2 一页一焦点（克制的美学）

陶土色是手绘风格中最稀缺的资源。一页只用一处（最多两三处在紧迫情绪下），用在小图标、背景色块是对这种稀缺的浪费。

**焦点标记要有选择性**：只标记最重要的异常、风险、或行动建议。

---

### 2.3 灰色层次（可读性）

主标题深灰（#111111），正文中灰（#666666），注释浅灰（#999999）。三层灰度让信息有层级。

---

## 三、规模与节奏

### 3.1 线条粗细分层（视觉节奏）

**粗线条（4-5px）**：定义轮廓，塑造体积
**中等线条（2-4px）**：刻画结构，连接各部分
**细线条（1.7-2.3px）**：绘制纹理，添加细节

三层粗细就像书法中的"提按"，让画面有呼吸感。不是所有线条都一样粗。

---

### 3.2 线条呼吸感（v2.0 新增）

**问题**：我的线条统一 1.8px，没有变化。

**解决**：同一条线分段，粗细有呼吸感。

```
李诞虾的身体轮廓：
- 第一段：5.206px（粗，起点）
- 第二段：4.631px（中，过渡）
- 第三段：4.055px（细，终点）

转折处加粗：
- 直线段：2.9px
- 转折处（圆角）：3.5px
- 再回到直线段：2.9px
```

**参数表**：

| 线条类型 | 粗细范围 | 用途 | 变化方式 |
|---------|----------|------|----------|
| 外轮廓 | 4.5-5.5px | 主体边界 | 分段：5.2 → 4.6 → 4.0px |
| 结构线 | 3.5-4.5px | 内部结构 | 分段：4.0 → 3.5px |
| 纹理线 | 2.5-3.5px | 细节纹理 | 均匀 2.9px 或轻微变化 |
| 微细节 | 1.5-2.5px | 极细细节 | 均匀 1.8px |

---

### 3.3 复杂度谱系（不是 binary）

从超简单（1 个 path）到简单（9-29 个 path）到复杂（61-88 个 path），每个层级都有价值。不要用复杂的方法解决简单的问题：一个箭头图标不需要 API 调用，两行代码足够。

---

### 3.4 结构模式 > path 数量

即使是 88 个 path 的猫咪图，也能分解为 5-6 个组件（头、身、四肢、尾巴、装饰）。每个组件 10-20 个 path，不是 88 个独立 path。

## 四、工艺与执行

### 4.1 手绘感参数系统（v2.0 核心）

#### 微抖动参数表

| 元素类型 | 控制点数量 | 偏移范围 | 段数 |
|---------|-----------|----------|------|
| 短直线（<20px） | 1-2 | ±0.5-1px | 2 |
| 中直线（20-50px） | 2-3 | ±0.5-1.5px | 3 |
| 长直线（>50px） | 3-4 | ±1-2px | 4-5 |
| 圆角 | 2 | ±0.5-1px | 2-3 |
| 矩形边 | 2-3 | ±1-2px | 2-4 |
| 曲线 | 2-3 | ±2-5px | 3-4 |
| 填充形状 | 3-4 | ±2-4px | 8-12 |

#### 形状有机化参数表

| 形状类型 | 宽度比 | 高度比 | 控制点偏移 |
|---------|--------|--------|----------|
| 靠背 | 顶/底 = 0.92-0.96 | 左/右 = 0.98-1.02 | ±1-2px |
| 桌面 | 顶/底 = 0.94-0.98 | - | ±1-2px |
| 镜框 | 顶/底 = 0.95-0.99 | 左/右 = 0.99-1.01 | ±0.5-1px |
| 抱枕 | 宽/高 = 0.85-0.95 | - | ±2-4px |

#### 负空间呼吸参数表

| 元素类型 | 基础间距 | 变化范围 | 随机性 |
|---------|----------|----------|--------|
| 竖条纹 | 20px | ±2px (±10%) | 每个间距独立 |
| 横条纹 | 15px | ±1.5px (±10%) | 每个间距独立 |
| 散点 | - | ±30% 位置偏移 | 每个点独立 |

**关键**：避免完美对称，左右独立随机。

---

### 4.2 参数固化（执行稳定）

圆角 2px（锐利）、分割线 2px（可见）、进度条 3px（刚好）、封面标题 35-40px（存在感）。这些数字不是随意选择，是无数次迭代验证的结果。

**v2.0 新增固化参数**：

- 贝塞尔微抖动：±0.5-2px（根据线条长度）
- 形状有机化：宽度比 0.92-0.98，高度比 0.98-1.02
- 负空间呼吸：间距 ±10% 随机变化
- 线条呼吸感：分段粗细，转折加粗 20%

---

### 4.3 质量评估（Binary 清单）

```
□ 双层分离（填充层 + 描边层）
□ stroke-linecap="round" 全局应用
□ 色彩系统完整（陶土色 + 墨黑 + 灰度层次）
□ path 数量合理（简单 10-30，复杂 60-80）
□ 结构清晰（头部/身体/四肢分组）
□ 中文字符清晰可读（如果需要）
□ 视觉平衡（左右对称、重心稳定）

通过率 7/7 = 100 分
每少 1 项扣 15 分
```

## 五、视觉层级

### 5.1 前注意特征编码（可读性）

大小编码层级、位置编码结构、色调编码焦点、强度编码确定性。一个视觉变量只编码一个信息属性，这是铁律。

---

### 5.2 格式塔原理（阅读路径）

围合律（卡片是一组）、连续律（时间线是连续的）、相近律（间距决定分组）、相似律（同色同类）。读者第一秒看到的分组，就是格式塔在起作用。

---

### 5.3 负空间呼吸（v2.0 新增）

**问题**：我的椅子靠背竖条纹均匀分布，机械死板。

**解决**：间距应该有呼吸感。

```
机械均匀：
│  │  │  │  │
间距全部 20px

手绘呼吸：
│  │   │  │   │
  18px 22px 19px 21px
  ↑    ↑   ↑    ↑
  间距在 ±10% 范围内随机变化
```

**关键**：负空间是"设计的"，不是"剩余的"。

## 从哲学到实践（三阶段路径）

### 阶段一：临摹学习（当前阶段）

**目标**：深度临摹 5-10 个传神作品，提取贝塞尔模式库

**执行**：
- 选择 5 个不同复杂度的传神作品
- 逐个拆解每个 path，标注控制点坐标
- 提取贝塞尔曲线模式（5 种可复用模式）
- 建立手绘感参数系统（4 张参数表）

**时间投入**：1-2 周
**产出**：贝塞尔曲线模式库 + 参数配置器

---

### 阶段二：模板体系（E3）

**目标**：建立可复用的组件库

- Layer 1：代码片段库（超简单图标）
- Layer 2：参数化模板（简单-中等复杂度）
- Layer 3：结构模板（复杂场景的组件化）

**触发条件**：临摹作品能达到 80% 以上相似度
**时间投入**：2-3 周
**产出**：完整的模板生成工具

---

### 阶段三：API 增强（E4）

**目标**：API 生成框架，手工填充细节

- 用 API 生成基础结构和路径布局
- 手工调整贝塞尔曲线的"微抖动"
- 手工添加双层分离和 stroke-linecap
- 手工应用色彩系统和参数规范

**触发条件**：纯 API 质量无法达到 80 分
**时间投入**：持续优化
**产出**：半自动化工作流

## 核心原则总结（v2.0）

1. **不完美是精心设计的** — 手绘感不是随机，是理解后的精确实现
2. **双层结构是基础** — 填充层 + 描边层，缺一不可
3. **stroke-linecap="round" 是灵魂** — 圆头端点区分手绘和矢量
4. **有机几何化是核心**（v2.0 新增）— 贝塞尔微抖动系统
5. **线条有呼吸感**（v2.0 新增）— 分段粗细 + 转折加粗
6. **一页一焦点是克制** — 陶土色是稀缺资源，用对位置
7. **参数固化保证稳定** — 圆角 2px、微抖动 ±0.5-2px
8. **质量由清单定义** — 7 项 binary 检查，通过率决定分数
