CL

CLAUDE.md — 项目记忆

让 Claude 自动理解你的项目背景

▼

💔 痛点：每次都要重复交代项目背景

你是不是经历过这样的对话：

  • 你：帮我在用户服务里加个接口
  • Claude：好的，请问用什么框架？
  • 你：NestJS
  • Claude：测试怎么跑？
  • 你：npm run test
  • Claude：代码风格有什么要求？
  • 你：用 ESLint...

每次开新会话，都要重复这些背景信息，很浪费时间。

✅ 解决方案：CLAUDE.md

CLAUDE.md 是一个特殊的 Markdown 文件，Claude Code 会在每次会话开始时自动读取它。

# 项目命令
- npm run dev: 启动开发服务器
- npm run test: 运行测试
- npm run lint: 检查代码风格

# 代码规范
- 使用 TypeScript strict 模式
- 优先使用 ES 模块语法 (import/export)
- 组件用函数式，不用类组件

# 工作流
- 提交前必须跑测试和 lint
- 优先写单元测试，覆盖率要求 80%+

你可以把它放在项目根目录（推荐），这样整个团队都能共享。也可以放在 ~/.claude/ 目录，让它应用到所有会话。

HK

Hooks — 自动执行

在特定事件触发时自动执行 shell 命令

▼

💔 痛点：重复性手工操作太多了

每次代码修改后，你都要手动运行测试、检查 lint、格式化代码。这些操作很机械，但又必须做。

✅ 解决方案：Hooks

Hooks 让你在 Claude Code 的特定事件发生时，自动执行 shell 命令。

{
  "userPromptSubmit": {
    "command": "npm run lint -- --fix",
    "description": "Auto-fix lint issues"
  }
}

可用的事件类型包括：

  • tool_use：工具调用后
  • user_prompt_submit：用户提交 prompt 后
  • session_start：会话开始时
  • 每个工具调用：如 Bash、Edit 等

SA

SubAgents — 分解复杂任务

专门的 AI 子代理，处理特定任务

▼

💔 痛点：单个 Agent 处理复杂任务容易出问题

当你让 Claude 处理一个复杂任务时，比如重构整个认证系统，它可能会：

  • 遗漏边界情况
  • 在不同部分之间不一致
  • 中途跑偏

为什么会这样？因为上下文太长，单一 Agent 难以保持专注。

✅ 解决方案：SubAgents

SubAgents 是专门的 AI 子代理，每个负责一个子任务。它们有独立的上下文，可以并行工作，最后合并结果。

主 Agent：规划整体架构
├── SubAgent 1：重构登录逻辑
├── SubAgent 2：重构 Token 管理
├── SubAgent 3：重构权限验证
└── SubAgent 4：更新单元测试

每个 SubAgent 独立工作，专注自己的部分。完成后，主 Agent 负责整合和验证。

CM

Commands — 斜杠命令

快速触发预定义的复杂任务

▼

💔 痛点：重复输入相同的指令很麻烦

比如代码审查，你每次都要说"请检查代码质量、安全性、性能..."。重复且容易遗漏。

✅ 解决方案：Commands

Commands 是可自定义的斜杠命令，让你用简短的命令快速触发复杂的 prompt。

# .claude/commands/review.md
请对当前分支的改动进行代码审查，重点检查：

1. 代码质量和可维护性
2. 潜在的安全漏洞
3. 性能问题
4. 测试覆盖率
5. 文档完整性

请使用 GitHub CLI 查看 PR 详情，并提供改进建议。

现在你只需要输入 /review，Claude 就会自动执行完整的审查流程。

🎮 交互练习

问题：Commands 文件应该放在哪里？

SK

Skills — 可复用工作流

把复杂操作封装成可复用的技能包

▼

💔 痛点：有些复杂操作，每次都要一步步教

比如，经常要做"添加新 API 端点"这个操作，包括：创建路由、添加控制器、添加验证、添加测试、更新文档...

每次都要跟 Claude 解释一遍，很累。

✅ 解决方案：Skills

Skills 是你自定义的可复用技能包。把一个完整工作流封装起来，下次直接调用。

---
name: add-api-endpoint
description: 添加新的 API 端点，包括路由、控制器、验证、测试和文档更新
---

# Add API Endpoint

添加一个新的 API 端点，包括以下步骤:

1. 创建路由文件 (如果不存在)
2. 添加控制器方法
3. 添加请求验证 (DTO)
4. 添加单元测试
5. 更新 API 文档 (README.md)

以后只要说"请帮我添加一个 GET /api/users 端点"，Claude 会自动识别这个请求匹配 add-api-endpoint 技能。

MP

MCP — 连接外部世界

让 Claude 访问 Figma、Google Drive、Slack 等

▼

💔 痛点：Claude Code 只能访问本地文件和命令

有时候你需要 Claude 访问外部资源，比如：从 Jira 读取需求、从 Figma 获取设计稿、从 Google Docs 读取规范...

✅ 解决方案：MCP Servers

MCP（Model Context Protocol）是 Anthropic 推出的开放标准，让 AI 工具能连接外部数据源。

# 安装 Figma MCP 后，你可以说：
请读取这个 Figma 文件的设计稿，然后用 React 实现它：
https://www.figma.com/file/xxx

# Claude 会：
1. 通过 MCP 连接 Figma
2. 读取设计稿的图层、样式、布局
3. 生成对应的 React 代码
4. 保持设计规范一致

🎮 交互练习

PL

Plugins — 打包完整工作流

一键安装 Commands、Skills、Hooks、MCP 的组合

▼

💔 痛点：每个项目都要重新配置 Commands、Skills、Hooks

你花了很多时间配置好一套开发工作流，结果新项目又要重新来一遍。复制粘贴配置文件，还容易漏掉东西。

✅ 解决方案：Plugins

Plugins 是打包好的功能组合，可以包含 Commands、SubAgents、Skills、Hooks、MCP 配置等任何组件。

# 安装 PR 审查工作流插件
/plugin marketplace add anthropics/claude-code-plugins
/plugin install pr-review-toolkit

# 一个命令完成全面审查
/pr-review-toolkit:review-pr all

# 这个插件包含：
# - 6 个专业审查代理并行工作
# - 从不同角度分析 (注释、测试、错误处理等)
# - 汇总所有发现，给出综合建议

三者对比：

  • Commands = 一个快捷键
  • Skills = 一本操作手册
  • Plugins = 一整套工具箱

问题：哪个组件最适合打包多个 Commands + Skills + Hooks？

💥 组合威力：1+1>2

单个组件很好用，但组合起来才是真正的威力所在

CLAUDE.md + Hooks

双重质量保障

CLAUDE.md 提供项目上下文，Hooks 自动执行质量检查。新成员加入时，CLAUDE.md 确保 Claude 理解项目规范，Hooks 确保每次修改都符合规范。

SubAgents + MCP

多源数据并行分析

主 Agent 规划分析框架，SubAgent 1 从 GitHub 读取竞品代码，SubAgent 2 从文档网站读取竞品文档，SubAgent 3 从社交媒体读取用户反馈，最后汇总分析报告。

Commands + Skills

快捷入口 + 自定义能力

Commands 提供快捷入口，Skills 提供自定义能力。/feature-dev 触发功能开发流程 → Claude 自动加载相关 Skills → /review 触发代码审查 → /commit 生成提交。

Plugins 一键配置

团队标准化

一个命令完成团队工作流标准化。新项目启动：/plugin install team-coding-standards → 10 个 Commands + 5 个 Skills + 3 个 Hooks + 2 个 MCP + CLAUDE.md 模板全部就绪。

🚀 立即上手指南

4 个立即可做的任务，5 分钟内开始使用

1

创建你的第一个 CLAUDE.md

在项目根目录运行 /init，Claude 会自动生成一个初始 CLAUDE.md，你可以根据需要调整。

预计耗时：2 分钟

2

配置一个简单的 Hook

在 .claude/settings.json 或 ~/.claude.json 中添加 Edit 事件的 lint 自动修复。

预计耗时：1 分钟

3

创建一个简单的 Command

在 .claude/commands/review.md 中创建代码审查命令，然后用 /review 快速触发。

预计耗时：2 分钟

4

安装一个官方 Plugin

运行 /plugin marketplace add anthropics/claude-code-plugins 然后 /plugin install code-review 体验打包好的专业工具。

预计耗时：1 分钟
