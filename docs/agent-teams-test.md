## 典型测试场景

### 场景：新供应商引入风险评估

**背景**：
- 公司：XX数据科技公司
- 业务：金融营销外呼
- 规模：100人坐席
- 价格：比现有高20%，但承诺转化率提升30%

---

## 如何测试

### 方法1：直接在Claude Code中输入

```
请用Agent Teams方式评估以下供应商引入风险：

供应商信息：
- 公司：XX数据科技公司
- 业务：金融营销外呼
- 规模：100人坐席
- 价格：比现有高20%
- 承诺：转化率提升30%

请创建三个teammate：
1. 安全风险分析agent
2. 合规风险分析agent
3. 业务价值分析agent

让三个agent并行分析，最后汇总给出综合建议。
```

### 方法2：简化版测试

```
创建一个Agent Team，包含三个成员：
- 成员A：分析安全风险
- 成员B：分析合规风险
- 成员C：分析业务价值

任务：评估是否引入XX数据科技公司作为新供应商
```

---

## 预期效果

**如果功能正常**：
- 三个agent会并行工作
- 每个agent有独立的上下文
- 可以互相发送消息
- 最后汇总综合建议

**观察点**：
- 并行执行速度
- agent间通信
- 结果汇总质量

---

## 与当前方式的对比

| 特性 | 当前方式 | Agent Teams |
|------|---------|-------------|
| agent数量 | 单一主agent | 多个teammate |
| 执行方式 | 串行/手动并行 | 自动协调 |
| 通信 | 无直接通信 | 可以互相发消息 |
| 上下文 | 共享 | 各自独立 |
| token消耗 | 较低 | 较高 |
