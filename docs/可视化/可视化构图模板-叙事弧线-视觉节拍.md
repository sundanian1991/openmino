# 可视化构图模板 · 叙事弧线 · 视觉节拍 — 结构化数据

> 来源：Edward Tufte（The Visual Display of Quantitative Information / Envisioning Information）、Alberto Cairo（The Truthful Art / How Charts Lie）、McKinsey Visualizing Insights、FT Visual Journalism、NYT Upshot Graphics Team

---

## A. 构图模板（13 种）

### A-01 | 单中心聚焦（Single-Center Focus）

| 维度 | 规格 |
|------|------|
| **画布比例** | 16:9（1280×720）或 4:3（1024×768） |
| **区域划分** | 标题区 12% 顶部 · 主图区 72% 居中 · 注释区 10% 底部 · 脚注区 6% 最底 |
| **视觉重心** | 画布几何中心偏移 5% 向上（即重心在 y=45% 处） |
| **间距规则** | 主图距左右边距各 8% · 上下边距各 5% · 标题与主图间距 3% |
| **适用图表** | 单个关键指标大字报、KPI 仪表盘、单一趋势线、单一柱状图 |
| **使用条件** | 只有一个核心信息需要传递 |
| **Tufte 原则对应** | Data-ink ratio 最大化 → 去装饰，数据墨水比 ≥ 0.7 |

### A-02 | 左右对比（Left-Right Contrast）

| 维度 | 规格 |
|------|------|
| **画布比例** | 16:9（1280×720） |
| **区域划分** | 标题区 10% · 左图区 42% · 分割线/留白 4% · 右图区 42% · 脚注 2% |
| **视觉重心** | 双重心：左重心 x=26% y=50%，右重心 x=74% y=50% |
| **间距规则** | 两图之间间距 ≥ 4% 画布宽度 · 各图内边距 3% |
| **适用图表** | A vs B 柱状图、Before/After 散点图、两地区趋势对比、实际 vs 目标 |
| **关键设计** | 两图 Y 轴刻度必须一致 · 颜色语义一致（同指标同色） |

### A-03 | 上下分层（Top-Bottom Split）

| 维度 | 规格 |
|------|------|
| **画布比例** | 16:9 或 3:2（1200×800） |
| **区域划分** | 标题 8% · 上图 40% · 分隔线 2% · 下图 40% · 脚注 10% |
| **视觉重心** | 上半区重心 y=30%，下半区重心 y=70%（X 居中） |
| **间距规则** | 上下图共用 X 轴时对齐刻度线 · 分隔线高度 1px 颜色 #E0E0E0 |
| **适用图表** | 上图总量 + 下图构成/占比、上图趋势 + 下图增速、地图 + 图例 |

### A-04 | 小 multiples 网格（Small Multiples Grid）

| 维度 | 规格 |
|------|------|
| **画布比例** | 按网格自适应：3×3 → 1:1（900×900），4×3 → 4:3（1200×900） |
| **区域划分** | 标题 8% · 网格区 84% · 脚注 8% |
| **网格规格** | 每格尺寸 =（画布宽 − 左右边距 6%）/ 列数 · 格间距 = 画布宽 × 1.5% |
| **视觉重心** | 整体重心居中，单格重心在各自单元格中心 |
| **阅读顺序** | 左→右、上→下（F 型扫描） |
| **适用图表** | 多地区同指标趋势、多品类同维度对比、时间序列分面 |
| **Tufte 原则对应** | "Small multiples 是最高效的信息密度工具" · 每格共享坐标轴 |

### A-05 | 主次嵌套（Nested Hierarchy）

| 维度 | 规格 |
|------|------|
| **画布比例** | 16:9 或 4:3 |
| **区域划分** | 主图占 60%（左侧或上方） · 次图占 32%（右侧或下方） · 间距 4% · 标题 4% |
| **视觉重心** | 主重心 x=35% y=50% · 次重心 x=78% y=50% |
| **适用图表** | 主：堆叠面积/桑基图 · 次：局部放大/细分柱状图/构成饼图 |
| **关键设计** | 次图必须有"zoom-in"视觉暗示（缩略框线或放大图标） |

### A-06 | 非对称平衡（Asymmetric Balance）

| 维度 | 规格 |
|------|------|
| **画布比例** | 16:9 或 2:1（宽幅 1600×800） |
| **区域划分** | 左侧大图区 55% · 右侧信息区 37%（含 3-5 个关键指标卡片 + 文字） · 间距 4% · 标题 4% |
| **视觉重心** | 左重心 x=30%（面积大） · 右重心 x=78%（密度高，用文字/数字密度平衡） |
| **间距规则** | 右侧卡片内间距 12px · 卡片间间距 16px · 卡片圆角 4px |
| **适用图表** | 主图 + 关键发现列表、地图 + 城市数据卡片、趋势图 + 洞察要点 |

### A-07 | 径向中心（Radial Center）

| 维度 | 规格 |
|------|------|
| **画布比例** | 1:1（800×800）或 4:3 留白 |
| **区域划分** | 径向图占 70% 居中 · 标题在顶部或底部 · 图例环绕或底部 · 四周留白 ≥ 12% |
| **视觉重心** | 绝对几何中心 |
| **适用图表** | 雷达图、极坐标柱状图、圆形进度环、环形树图、玫瑰图 |
| **关键设计** | 径向图数据墨水比低 → 必须放大字号和线宽补偿 |

### A-08 | 时间轴脊线（Timeline Spine）

| 维度 | 规格 |
|------|------|
| **画布比例** | 宽幅 21:9（1680×720）或横向滚动 |
| **区域划分** | 中央脊线 3% 宽度贯穿 · 上事件区 35% · 下事件区 35% · 标题 10% · 脚注 17% |
| **视觉重心** | 沿脊线均匀分布 · 每个节点 x = (序号 / 总节点数) × 画布宽 |
| **节点规格** | 节点直径 12px · 节点间距 ≥ 画布宽的 8% · 节点线宽 2px |
| **适用图表** | 里程碑时间线、产品迭代历程、政策演变、公司发展史 |
| **关键设计** | 重要节点放大 1.5 倍 + 颜色标记 · 节点交替上下排列防重叠 |

### A-09 | 三栏仪表板（Three-Column Dashboard）

| 维度 | 规格 |
|------|------|
| **画布比例** | 16:9（1920×1080 最佳） |
| **区域划分** | 标题栏 6% · 左栏 28% · 中栏 40% · 右栏 28% · 脚注 4% · 栏间距 2% |
| **视觉重心** | 中栏重心 x=50%（最大面积） · 左右为辅助 KPI 和明细 |
| **适用图表** | 中：主趋势/主地图 · 左：KPI 卡片列表 · 右：排行/构成/筛选器 |
| **间距规则** | 卡片内 padding 16px · 卡片圆角 6px · 卡片间距 12px |

### A-10 | 瀑布流/阶梯（Waterfall / Ladder）

| 维度 | 规格 |
|------|------|
| **画布比例** | 16:9 或 3:2 |
| **区域划分** | 标题 8% · 瀑布图 78% · 底部汇总行 6% · 脚注 8% |
| **视觉重心** | 从左到右流动 · 起点和终点柱子加粗/加深色 |
| **适用图表** | 利润桥（Revenue → EBITDA → Net Income）、预算差异分析、指标贡献分解 |
| **关键设计** | 正增量用绿色/蓝色，负增量用红色/橙色 · 总计柱子与增量柱子宽度比 = 1.5:1 |

### A-11 | 地图 + 附件图（Map + Insets）

| 维度 | 规格 |
|------|------|
| **画布比例** | 16:9 或 4:3 |
| **区域划分** | 主地图 55%（左侧） · 附件区 37%（右侧） · 间距 4% · 标题 4% |
| **视觉重心** | 地图重心 x=30% · 附件重心 x=78% |
| **适用图表** | 地理分布热力图 + 区域排名、地图 + 气泡图、地图 + 折线趋势 |
| **关键设计** | 地图颜色梯度不超过 5 级 · 图例高度 = 地图高度的 15% |

### A-12 | 全幅沉浸式（Full-Bleed Immersive）

| 维度 | 规格 |
|------|------|
| **画布比例** | 自适应，通常 16:9 或 21:9 |
| **区域划分** | 图表占满 100% 画布 · 标题叠加在图表上（左上角，背景半透明黑 60%） · 无独立脚注区 |
| **视觉重心** | 由图表本身的视觉热点决定 |
| **适用图表** | 大型散点云、高分辨率热力图、复杂桑基图、网络关系图 |
| **关键设计** | 标题区域背景 rgba(0,0,0,0.6) · 文字白色 · padding 20px 32px |

### A-13 | 卡片矩阵（Card Matrix）

| 维度 | 规格 |
|------|------|
| **画布比例** | 自适应 · 2×2 用 1:1 · 3×2 用 3:2 |
| **区域划分** | 标题 6% · 卡片网格 88% · 脚注 6% |
| **卡片规格** | 每卡片：圆角 8px · 内边距 20px · 卡片间距 16px · 阴影 0 2px 8px rgba(0,0,0,0.08) |
| **卡片内容结构** | 顶部小标题（12px 灰色） → 核心数字（36px 粗体） → 趋势标识（±X%） → 迷你图（可选） |
| **适用场景** | KPI 概览页、多指标并行展示、供应商/地区评分卡 |

## B. 叙事弧线（7 种）

### B-01 | 调查式（Investigative）

| 维度 | 规格 |
|------|------|
| **结构** | 震撼开场 → 全景扫描 → 逐层深挖 → 核心发现 → 行动结论 |
| **节奏** | 图1 大字报/冲击力（3s 读完）→ 图2 全景/背景（10s）→ 图3 分解/分面（15s）→ 图4 深挖/细节（20s）→ 图5 结论/建议（5s） |
| **总时长** | 约 53 秒 |
| **何时使用** | 揭示问题、审计发现、异常数据解读 |
| **典型案例** | NYT"新冠疫情死亡人数超越二战" → FT"英国薪资实际缩水十年" |
| **视觉节拍序列** | anchor_number → density_shift → contrast_reveal → annotation_arrow → anchor_number |

### B-02 | 对比式（Comparative）

| 维度 | 规格 |
|------|------|
| **结构** | A 现状 → B 现状 → 直接对比 → 差异解读 → 结论 |
| **节奏** | 图1 A 单图（5s）→ 图2 B 单图（5s）→ 图3 同轴对比（10s）→ 图4 差异高亮（8s）→ 图5 结论卡片（3s） |
| **总时长** | 约 31 秒 |
| **何时使用** | 两家供应商对比、两年同期对比、方案 A vs B |
| **视觉节拍序列** | anchor_number → color_pop → baseline_crossing → annotation_arrow → anchor_number |

### B-03 | 演进式（Evolutionary）

| 维度 | 规格 |
|------|------|
| **结构** | 起点状态 → 过程变化 → 关键拐点 → 当前终态 → 未来预判 |
| **节奏** | 图1 起点快照（5s）→ 图2 趋势线（10s）→ 图3 拐点放大（12s）→ 图4 终态对比（8s）→ 图5 预测区间（8s） |
| **总时长** | 约 43 秒 |
| **何时使用** | 供应商历史表现、业务增长轨迹、政策演变影响 |
| **典型案例** | FT"全球利率四十年变迁" · NYT"美国房价周期" |
| **视觉节拍序列** | anchor_number → trend_inflection → annotation_arrow → color_pop → density_shift |

### B-04 | 诊断式（Diagnostic）

| 维度 | 规格 |
|------|------|
| **结构** | 问题信号 → 维度分解 → 根因定位 → 解决方案 |
| **节奏** | 图1 问题大字报（3s）→ 图2 树状分解/瀑布（15s）→ 图3 根因散点/热力（12s）→ 图4 方案对比矩阵（10s） |
| **总时长** | 约 40 秒 |
| **何时使用** | 业绩下滑归因、异常指标排查、客户流失分析 |
| **McKinsey 对应** | MECE 分解 → Issue Tree → Root Cause → Recommendation |
| **视觉节拍序列** | anchor_number → isolation → grouping → contrast_reveal → annotation_arrow |

### B-05 | 提案式（Proposal）

| 维度 | 规格 |
|------|------|
| **结构** | 痛点共鸣 → 方案框架 → 证据支撑 → 行动呼吁 |
| **节奏** | 图1 痛点冲击（3s）→ 图2 方案结构图（12s）→ 图3 案例/证据（10s）→ 图4 ROI/预期收益（8s）→ 图5 行动清单（5s） |
| **总时长** | 约 38 秒 |
| **何时使用** | 预算申请、策略提案、供应商准入/清退建议 |
| **视觉节拍序列** | anchor_number → grouping → contrast_reveal → color_pop → annotation_arrow |

### B-06 | 层级穿透式（Drill-Down）

| 维度 | 规格 |
|------|------|
| **结构** | 总览 → 一级分解 → 二级分解 → 个例详情 → 行动项 |
| **节奏** | 图1 总量仪表盘（5s）→ 图2 按维度分解（10s）→ 图3 按子维度分解（12s）→ 图4 Top3 明细卡片（10s）→ 图5 建议列表（5s） |
| **总时长** | 约 42 秒 |
| **何时使用** | 供应商季度评审、多层级 KPI 拆解、组织架构分析 |
| **视觉节拍序列** | anchor_number → density_shift → isolation → grouping → annotation_arrow |

### B-07 | 故事式（Narrative Journey）

| 维度 | 规格 |
|------|------|
| **结构** | 人物/场景引入 → 冲突/挑战 → 努力/过程 → 结果/反转 → 余韵/启示 |
| **节奏** | 图1 人物卡片/引用（5s）→ 图2 背景数据（8s）→ 图3 冲突可视化（10s）→ 图4 转折点（8s）→ 图5 结局数据（5s）→ 图6 金句/引用（3s） |
| **总时长** | 约 39 秒 |
| **何时使用** | 人物画像叙事、供应商合作故事、客户案例 |
| **Alberto Cairo 对应** | "Charts should tell stories, not just show data" — The Truthful Art |
| **视觉节拍序列** | isolation → anchor_number → trend_inflection → color_pop → contrast_reveal → anchor_number |

## C. 视觉节拍（20 种）

### C-01 | 锚定数字（Anchor Number）

| 维度 | 规格 |
|------|------|
| **视线轨迹** | 第一眼：超大数字 → 第二眼：数字标签/单位 → 最终：上下文解释文字 |
| **技术** | 字号 48-72px · 粗体 · 颜色使用品牌主色或数据驱动色 · 周围留白 ≥ 数字宽度的 1.5 倍 |
| **场景** | KPI 大字报、关键增长率、总数冲击 |
| **Tufte 原则** | "最大的数据 = 最大的视觉权重" |

### C-02 | 色彩弹出（Color Pop）

| 维度 | 规格 |
|------|------|
| **视线轨迹** | 第一眼：唯一的彩色元素 → 第二眼：彩色元素的标签 → 最终：与灰色/浅色背景元素的对比关系 |
| **技术** | 全局仅 1 个元素使用饱和色（HSL 色相 220° 蓝或 0° 红），其余元素灰度 30-50% |
| **场景** | 突出某个供应商/地区/品类 |
| **关键值** | 强调色饱和度 ≥ 80% · 背景色饱和度 ≤ 15% |

### C-03 | 标注箭头（Annotation Arrow）

| 维度 | 规格 |
|------|------|
| **视线轨迹** | 第一眼：箭头指向的数据点 → 第二眼：箭头尾部的注释文字 → 最终：理解因果关系 |
| **技术** | 箭头线宽 1.5px · 箭头头部 8px · 注释文字 11px · 注释框背景半透明或无框 |
| **场景** | 标注政策变化点、异常值解释、拐点原因 |
| **关键值** | 箭头长度 20-60px · 注释文字 ≤ 25 字 |

### C-04 | 基准线穿越（Baseline Crossing）

| 维度 | 规格 |
|------|------|
| **视线轨迹** | 第一眼：基准线（0 线/目标线/去年同期线）→ 第二眼：哪些在上方、哪些在下方 → 最终：幅度大小 |
| **技术** | 基准线 1.5px 虚线（#999）或 实线（#666）· 正负区域用双色填充（透明度 15%） |
| **场景** | 业绩 vs 目标、增速 vs 0、实际 vs 预算 |
| **Tufte 原则** | Reference lines reduce cognitive load |

### C-05 | 趋势拐点（Trend Inflection）

| 维度 | 规格 |
|------|------|
| **视线轨迹** | 第一眼：趋势线整体走向 → 第二眼：拐点标记 → 最终：拐点前后的斜率变化 |
| **技术** | 拐点处画圆点（直径 8px，白色描边 2px） · 拐点前后分段用不同颜色或线型 · 辅助阴影区域标记阶段 |
| **场景** | 增长率转折、政策干预前后、季节性拐点 |

### C-06 | 对比揭示（Contrast Reveal）

| 维度 | 规格 |
|------|------|
| **视线轨迹** | 第一眼：整体相似性 → 第二眼：关键差异区域 → 最终：差异量化值 |
| **技术** | 双系列重叠图 · 差异区域用阴影或颜色填充 · 差异值直接标注在差异区域中心 |
| **场景** | 今年 vs 去年、A 供应商 vs B 供应商、实际 vs 预测 |
| **关键值** | 差异标注字号 ≥ 14px · 差异区域填充透明度 20-30% |

### C-07 | 密度变换（Density Shift）

| 维度 | 规格 |
|------|------|
| **视线轨迹** | 第一眼：整体密度模式 → 第二眼：高密度聚集区 → 最终：低密度空白区（同样重要） |
| **技术** | 散点图 / 热力图 · 使用 hexbin 或 kernel density · 颜色梯度 5 级（浅色→深色） |
| **场景** | 客户分布、供应商能力雷达、地域覆盖度 |
| **Tufte 原则** | "Empty space is data too" — 空白区域传递信息 |

### C-08 | 隔离突出（Isolation）

| 维度 | 规格 |
|------|------|
| **视线轨迹** | 第一眼：被隔离的元素（独立卡片/放大区域）→ 第二眼：隔离元素的数据 → 最终：为什么它被隔离 |
| **技术** | 物理隔离（独立卡片、放大框、callout 框） · 隔离区背景色差异化（#F8F9FA） · 隔离区有微妙阴影 |
| **场景** | Top 1 / Bottom 1 单独展示、异常值单独分析、冠军/倒数 |

### C-09 | 分组归类（Grouping）

| 维度 | 规格 |
|------|------|
| **视线轨迹** | 第一眼：组间间距/分隔线 → 第二眼：组内元素相似性 → 最终：分组逻辑标签 |
| **技术** | 组间距 ≥ 组内间距的 2 倍（Gest proximity 原则） · 每组用浅底色背景框 · 组标签在顶部 11px 灰色 |
| **场景** | 供应商按等级分组、指标按维度分组、时间按季度分组 |
| **Gestalt 原则** | Proximity + Similarity + Common Region |

### C-10 | 近邻关联（Proximity Pairing）

| 维度 | 规格 |
|------|------|
| **视线轨迹** | 第一眼：靠近的两个元素被视为一组 → 第二眼：两者之间的关系 → 最终：这种关系的含义 |
| **技术** | 两元素间距 < 元素宽度的 30% · 用细线或箭头连接 · 共享同色系但明度区分 |
| **场景** | 原因→结果并置、投入→产出并排、问题→方案配对 |

### C-11 | 排序阶梯（Sorted Ladder）

| 维度 | 规格 |
|------|------|
| **视线轨迹** | 第一眼：从左到右/从上到下的渐变色条 → 第二眼：最高/最低值 → 最终：排名和差距 |
| **技术** | 严格排序（降序或升序） · 条形图颜色渐变（最大 = 饱和色，最小 = 浅色） · 标签在左侧/上方 |
| **场景** | 供应商排名、地区业绩排序、品类贡献排行 |
| **Cairo 原则** | "Sorting is the simplest and most powerful data transformation" |

### C-12 | 缩放聚焦（Zoom Spotlight）

| 维度 | 规格 |
|------|------|
| **视线轨迹** | 第一眼：全局缩略图 → 第二眼：高亮框选的局部 → 最终：放大后的细节 |
| **技术** | 左侧小图（30% 宽度）带矩形选择框 · 右侧大图（65% 宽度）为选区放大 · 选择框颜色与高亮区一致 |
| **场景** | 长趋势线中的关键区间、大范围数据的局部细节 |

### C-13 | 叠加融合（Overlay Fusion）

| 维度 | 规格 |
|------|------|
| **视线轨迹** | 第一眼：底层图表 → 第二眼：叠加层（趋势线/目标线/预测带）→ 最终：两层之间的关系 |
| **技术** | 底层：柱状图/面积图（不透明度 100%）· 叠加层：线型（2px 实线或虚线）· 预测带：半透明区域（20%） |
| **场景** | 柱状图 + 目标线、散点 + 回归线、面积图 + 预测区间 |

### C-14 | 面积编码（Area Encoding）

| 维度 | 规格 |
|------|------|
| **视线轨迹** | 第一眼：最大的面积块 → 第二眼：面积大小排序 → 最终：占比关系 |
| **技术** | 面积 ∝ 数值（严格比例） · 使用矩形树图或气泡图 · 最大气泡直径 ≤ 画布宽的 40% |
| **场景** | 市场份额、预算构成、资源分配 |
| **警告** | 人类对面积的感知误差约 15% · 需配合数字标签 |

### C-15 | 流动引导（Flow Guidance）

| 维度 | 规格 |
|------|------|
| **视线轨迹** | 第一眼：流动方向（箭头/连线）→ 第二眼：流量大小（线宽/颜色）→ 最终：源→目标路径 |
| **技术** | 桑基图/流线图 · 线宽 ∝ 流量 · 颜色区分流向 · 节点标签在节点中心或旁边 |
| **场景** | 资金流向、用户路径、供应商→客户映射 |

### C-16 | 时序脉冲（Temporal Pulse）

| 维度 | 规格 |
|------|------|
| **视线轨迹** | 第一眼：时间轴方向 → 第二眼：脉冲波峰/波谷 → 最终：周期性和异常点 |
| **技术** | 面积图或柱状图 · 时间轴等距刻度 · 季节性用背景色带标记 · 异常点用红色标记 + 注释 |
| **场景** | 月度/季度业绩脉冲、呼叫量波动、季节性分析 |

### C-17 | 维度旋转（Dimension Rotation）

| 维度 | 规格 |
|------|------|
| **视线轨迹** | 第一眼：当前维度的全貌 → 第二眼：切换维度后的变化 → 最终：哪个维度最关键 |
| **技术** | 同一数据集的多视图切换（通过 Tab 或翻页） · 每个视图使用相同的坐标轴范围 |
| **场景** | 同一指标按地区→按产品→按时间的多维分析 |

### C-18 | 层级折叠（Hierarchy Fold）

| 维度 | 规格 |
|------|------|
| **视线轨迹** | 第一眼：顶层汇总 → 第二眼：展开/折叠的交互暗示 → 最终：下钻后的细节 |
| **技术** | 树状缩进 + 展开图标（▶） · 每层缩进 24px · 层级线 1px #E0E0E0 · 展开状态用动画或分页 |
| **场景** | 组织架构 + 业绩、供应商→团队→个人的逐层分析 |

### C-19 | 异常闪烁（Anomaly Flag）

| 维度 | 规格 |
|------|------|
| **视线轨迹** | 第一眼：异常标记（红色/图标/边框）→ 第二眼：异常值的具体数字 → 最终：偏离正常范围的程度 |
| **技术** | 正常范围用灰色阴影带 · 超出范围的元素用红色边框或红色填充 · 添加 ↑↓ 箭头 · 偏差百分比直接标注 |
| **场景** | KPI 超标、合规异常、SLA 违约标记 |
| **关键值** | 红色 #E74C3C · 正常范围背景 #F0F0F0 透明度 50% |

### C-20 | 叙事锚点（Narrative Anchor）

| 维度 | 规格 |
|------|------|
| **视线轨迹** | 第一眼：人物/引用/故事元素 → 第二眼：支撑数据 → 最终：数据赋予故事的意义 |
| **技术** | 左侧/顶部：引用框（左边框 3px 品牌色，背景 #F8F9FA，padding 16px）· 右侧/底部：数据图表 · 引用文字斜体 13px |
| **场景** | 客户故事 + 数据、供应商反馈 + 评分、管理者引言 + 业绩 |
| **Cairo 原则** | "Data without context is noise; context without data is opinion" |

### 构图 × 叙事弧线 推荐搭配

| 叙事弧线 | 推荐构图 | 核心节拍 |
|----------|----------|----------|

| 对比式 | A-02 左右对比 → A-02 左右对比 → A-01 单中心 | C-02 → C-02 → C-06 → C-04 → C-01 |
| 演进式 | A-08 时间轴 → A-08 时间轴 → A-01 单中心 | C-05 → C-05 → C-03 → C-02 → C-07 |
| 诊断式 | A-10 瀑布 → A-05 主次嵌套 → A-01 单中心 | C-01 → C-09 → C-08 → C-06 → C-03 |
| 提案式 | A-06 非对称 → A-09 三栏 → A-01 单中心 | C-01 → C-09 → C-06 → C-02 → C-03 |
| 层级穿透 | A-13 卡片矩阵 → A-04 小 multiples → A-06 非对称 | C-01 → C-07 → C-08 → C-09 → C-03 |
| 故事式 | A-12 全幅 → A-08 时间轴 → A-01 单中心 | C-20 → C-01 → C-05 → C-02 → C-06 → C-20 |

### 节拍组合禁忌

| 禁止组合 | 原因 |
|----------|------|
| C-02 色彩弹出 + C-02 色彩弹出（同图多个） | 多个"重点" = 无重点 |
| C-01 锚定数字 + C-01 锚定数字（>3 个同区） | 视觉打架，无主次 |
| C-19 异常闪烁 + C-02 色彩弹出 | 冲突：哪个是异常？哪个是重点？ |
| C-08 隔离 + C-12 缩放聚焦（同元素） | 隔离已足够，缩放冗余 |
| C-11 排序阶梯 + C-15 流动引导 | 排序暗示序列，流动也暗示序列，冲突 |
