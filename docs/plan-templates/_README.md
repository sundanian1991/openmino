# Plan 模板体系 — AI思考指令集

> 年老师遇到问题 → 场景和数据给AI → AI读对应Plan → 输出判断建议 → 年老师确认 → official-doc输出公文

---

## 定位

Plan 模板是**给AI的思考指令**（system prompt级别），不是给人看的文档。

**完整链路**：
1. 年老师遇到判断场景
2. 把场景描述+相关数据丢给AI（可指定模型）
3. AI读对应Plan模板 → 按模板逻辑分析
4. AI输出结构化判断建议
5. 年老师确认或微调
6. 调用 official-doc 技能生成公文

**成本拆分**：
- Q卡（≤80行）→ 小模型（便宜快速，30秒判断）
- D/N框架（150-200行）→ 大模型（深度分析，15分钟思考）

## 使用方法

### 日常快速判断
```bash
# 场景：月度排名出炉，需要快速判断
年老师：@Mino 帮我看看这个月排名，哪几家需要重点关注

# Mino 的做法：
# 1. 读取 Q6 模板
# 2. 输入：本月排名数据
# 3. 输出：快速判断清单（3-5家需要关注的供应商+一句话现状+建议动作）
```

### 深度决策分析
```bash
# 场景：A供应商A级骤降，需要深入分析
年老师：@Mino 用 D1 分析 A级骤降。数据：[A供应商近6个月排名、人效、出勤、主管变动]

# Mino 的做法：
# 1. 读取 D1 模板
# 2. 按归因路径分析5种可能原因
# 3. 输出：决策备忘录 + 向上汇报/对供应商/内部同步三个版本口径
# 4. 年老师确认后，调用 official-doc 生成整改通知/约谈邀约
```

### 主动构建叙事
```bash
# 场景：月度复盘，想有自己的观察
年老师：@Mino 帮我做月度复盘。数据：[本月排名、人效、出勤、异常信号]

# Mino 的做法：
# 1. 读取 N1 模板
# 2. 提炼本月最重要的一件事
# 3. 输出：三层叙事（数据/判断/建议）+ 对标策略组的差异化
# 4. 年老师确认后，生成月度管理报告
```

## 与 official-doc 的衔接

每个 D 框架和 N 框架底部都标注了对应的 official-doc 模板文件名。

**Plan（想清楚） → Decision Memo（决策备忘录） → Doc（写出来）**

Plan 输出结构化结论 → 直接映射到 Doc 模板的占位符 → official-doc 生成公文。

## 模板特点

- **内嵌硬知识**：每个模板包含该场景所需的全部硬知识（阈值、规则、授权），AI无需翻其他文件
- **if-then决策树**：不是线性步骤，是条件判断路径
- **结构化输出**：输出格式固定，方便年老师快速确认或程序化处理
- **叙事草稿**：每个决策框架都包含向上/对外/内部三个版本的口径
- **边界清晰**：明确标注 AI 什么时候该停下来让年老师自己判断
