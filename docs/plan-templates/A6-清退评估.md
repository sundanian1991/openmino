# A6-清退评估

> 供应商达到清退条件（连续3个月后30%/诚信红线/G项红色）时，系统化评估是否启动清退

## 触发信号
- 连续3个月排名后30%（工具箱§一：清退评估触发条件）
- 诚信红线：数据造假≥2项或同一项连续2次不实
- G项红色：负面清单触发（重大合规/诚信/经营风险）
- 主管胜任≤2项且整改无效（连续2次整改未通过）

## 核心问题
- 该供应商是否确实达到清退条件？
- 清退的影响是什么？（业务影响/备选供应商/切换成本）
- 需要走什么流程？（评估→决策→执行）
- 谁需要参与决策？（决策授权矩阵）
- 清退后如何防止业务中断？

## 思考步骤

**Step 1：确认清退触发条件（工具箱§一 + §二 + §十六）**
```
清退条件（满足任一即可触发）：
□ 连续3个月后30%：
  第1月后30% → 首次预警（私下沟通）
  第2月后30% → 连续预警（正式约谈+整改）
  第3月后30% → 清退评估启动

□ 诚信红线：
  数据造假≥2项 或 同一项连续2次不实
  → 约谈高层告知后果 → 记入诚信档案 → 启动清退评估

□ G项红色：
  负面清单触发（如：欺诈、重大泄密、群体事件）
  → 立即启动清退

□ 主管不胜任且整改无效：
  主管胜任≤2项 + 连续2次整改未通过
  → 建议换人未果 → 启动清退评估
```
- 每条条件需有数据支撑（排名表/诚信记录/整改记录）
- 记录触发时间线：第几月何时触发

**Step 2：评估清退影响（业务影响分析）**
```
影响评估三要素：

1. 业务影响：
   - 该供应商当前份额：XX%（是否超过20%？）
   - 月均GMV：XXX万元（是否头部供应商？）
   - 业务线分布：金条/企金/信用卡/财富（是否关键产线？）
   → 若份额>20%或为头部供应商，需谨慎评估切换风险

2. 备选供应商：
   - 是否有现成备选？（中部/尾部中有意愿接手的）
   - 备选的产能是否可用？（排名、出勤、资源饱和度）
   - 备选的切换周期：T+？天（工具箱§三引入SOP）
   → 若无备选，需启动寻源

3. 切换成本：
   - 人员转移成本：离职补偿、重新入职
   - 业务中断风险：切换期业务停摆多久？
   - 客户影响：是否影响客户体验？
   → 成本 > 继续合作成本？需量化对比
```

**Step 3：决策选项与建议**
```
根据影响评估，给出三选一建议：

选项1：立即清退
条件：
✓ 清退条件明确（如诚信红线）
✓ 有备选且备选可立即承接
✓ 业务影响可控（份额<15%）
建议：按清退流程执行（工具箱§十九）

选项2：评估后清退
条件：
✓ 连续3个月后30%（已达清退条件）
✓ 但业务影响较大（份额15-25%）
✓ 有备选但需准备期（2-4周）
建议：启动清退评估流程，给60-90天过渡期

选项3：延长观察再评估
条件：
✓ 连续3个月后30%但有特殊原因（名单问题/产线问题）
✓ 无合适备选（强行清退业务中断）
✓ 供应商有明确改进计划且刚开始执行
建议：延长观察3个月，设定明确提升目标，若未达标再清退
```

**Step 4：决策授权与流程（工具箱§十九）**
```
决策权限矩阵：
动作                │ 年老师 │ 王易人 │ 军哥
───────────────────┼────────┼────────┼──────
清退评估启动        │ 建议权 │ 确认   │ 事前沟通
约谈供应商高层      │ 建议权 │ 必须陪同 │ 事前报备
更换主管建议        │ 建议权 │ 确认后执行 │ —
正式清退通知        │ —      │ 审批   │ 事前沟通

流程：
1. 年老师发起清退评估 → 附6个月数据+影响分析
2. 王易人确认 → 组织评估会
3. 军哥事前沟通 → 重大清退需军哥知情
4. 评估结论：立即清退 / 延长观察 / 终止评估
5. 执行：发清退通知 → 业务切换 → 结算 → 归档
```

**Step 5：输出清退评估报告**
```
清退评估报告结构：

一、供应商基本情况
   名称：[XXX]  业务线：[金条/企金/...]  份额：[XX%]
   合作时长：[X年X月]  月均GMV：[XXX万元]

二、清退触发条件
   □ 连续3个月后30%：[第X月XX% 第Y月YY% 第Z月ZZ%]
   □ 诚信红线：[具体问题，数据造假X项]
   □ G项红色：[具体问题]
   □ 主管不胜任：[5项中X项不达标，整改X次未通过]

三、业务影响分析
   - 影响业务量：月均XXX万元，占该产线XX%
   - 切换周期预估：T+XX天
   - 备选供应商：[是否有/谁/产能是否充足]
   - 切换成本预估：[人员转移/业务中断/客户影响]

四、决策建议
   □ 立即清退（理由）
   □ 评估后清退（延长X个月，目标XXX）
   □ 延长观察（再观察X个月，目标XXX）

五、审批记录
   提议人：[年老师]
   审核人：[王易人]
   审批人：[军哥]
   日期：YYYY-MM-DD
```

## 关键变量
- **连续月份数**：3个月是硬标准，2个月不触发
- **业务影响**：份额>20%的供应商清退需更谨慎
- **备选可用性**：无备选时清退会导致业务中断，需先寻源
- **诚信等级**：诚信红线是最高优先级，必须清退
- **上级态度**：军哥/易人哥对该供应商的历史评价

## 常见误区
- ❌ 连续2个月就启动清退：必须连续3个月
- ❌ 只看排名不看诚信：诚信红线优先级高于排名
- ❌ 清退不评估业务影响：可能导致业务中断
- ❌ 无备选就清退：需先有备选再清退，或同步寻源
- ❌ 不按决策授权：清退需易人哥确认、军哥事前沟通
- ❌ 不记录：清退评估需保存完整记录（审计要求）
