# D4-供应商群异常信号

> 在供应商群里观察到异常（24h未回复、态度变化、推诿）时，快速判断是真问题还是"撒娇"

## 触发信号
- 供应商在群里24小时未回复消息（其他供应商都回复了）
- 供应商突然态度变好/变差（过于热情或冷淡）
- 供应商在群里推诿责任、找借口、不直面问题
- 供应商频繁@你但不说具体事（刷存在感）
- 供应商在群里公开质疑规则或决策

## 核心问题
- 这个信号意味着什么？真有问题还是情绪表达？
- 我需要立即介入吗？还是观察？
- 如果需要回应，我的口径是什么？（公开回应还是私聊？）
- 后续需要跟进什么动作？

## 30秒快速判断

**Step 1：识别异常类型**
| 异常信号 | 可能含义 | 严重等级 | 应对方式 |
|---------|---------|---------|---------|
| **24h未回复** | ① 真问题（出勤/离职/系统问题）<br>② 撒娇（想引起关注）<br>③ 故意对抗 | 中/低 | 私聊问原因 |
| **态度突变** | ① 有事求你（要量/要支持）<br>② 做错事心虚<br>③ 单纯情绪波动 | 低/中 | 结合上下文判断 |
| **推诿找借口** | ① 确实有问题但不想担责<br>② 习惯性甩锅 | 中 | 当场指出，要求具体说明 |
| **刷存在感** | ① 量少想增加曝光<br>② 缺乏安全感 | 低 | 明确规则，不接招 |
| **公开质疑** | ① 对规则不满<br>② 对分配结果不服 | 高 | 公开回应规则，私聊深入 |

**Step 2：快速查证数据（工具箱§十六）**
- 该供应商当前状态：正常/首次预警/连续预警/A级骤降？
- 近7天群消息回复率：是否真的24h未回复？
- 业绩趋势：近4周GMV趋势（上升/下降/波动）？
- 是否有触发状态：触发预警的供应商异常概率更高

**Step 3：判断"真问题"vs"撒娇"（决策树）**
```
该供应商有触发状态吗？（首次预警/连续预警/A级骤降）
├─ 有 → 大概率真问题 → 立即私聊了解情况
└─ 无 ↓
  近4周业绩趋势是下降吗？（↓>20%）
   ├─ 是 → 可能真问题 → 私聊问原因
   └─ 否 ↓
  该供应商排名在70%以上且稳定吗？
    ├─ 是 → 大概率撒娇 → 公开回应规则，不单独安抚
    └─ 否 ↓
  最近有约谈/整改要求吗？
    ├─ 有 → 可能是对抗 → 指出问题，要求按整改计划执行
    └─ 无 → 观察，不立即介入
```

**Step 4：回应策略**
- **真问题**：私聊 → 了解原因 → 要求整改/支持
- **撒娇**：公开回应规则 → 不单独安抚 → 避免鼓励这种行为
- **对抗**：公开重申规则 + 私聊严肃沟通

## 5分钟深入版（如有时间）

**Step 1：根因定位（工具箱§十六）**
- 24h未回复：出勤问题？离职潮？系统故障？故意对抗？
- 态度突变：有求于你？做错事心虚？看到排名变化？
- 推诿责任：整改不力找借口？系统性能力不足？
- 公开质疑：对分量不满？对规则理解偏差？

**Step 2：关联触发状态**
```
若供应商处于以下状态，异常信号可信度更高：
✓ 首次预警：私下沟通后仍不回复 → 可能问题严重
✓ 连续预警：已有整改计划未完成 → 推诿可能是逃避
✓ A级骤降：排名大幅下滑 → 情绪波动正常，需安抚+约谈
✓ 诚信预警：数据质量问题 → 对抗可能是心虚
```

**Step 3：判断是否需要升级**
- 涉及合规红线（G项红色、信息安全）→ 立即升级，`compliance-missing.md`
- 涉及诚信问题（数据造假嫌疑）→ 记录，纳入下次诚信核查
- 涉及整改对抗（承诺动作未完成还找借口）→ 升级约谈层级
- 纯情绪撒娇 → 公开回应规则，不单独处理

**Step 4：设定跟进计划**
- 立即动作：私聊了解 / 公开回应 / 记录留痕
- 短期动作：要求整改 / 约谈 / 上报
- 长期观察：持续跟踪该供应商群消息回复率/态度变化

## 关键变量
- **供应商历史行为**：该供应商是否经常"撒娇"？是否有对抗历史？
- **当前触发状态**：预警状态下的异常信号更需重视
- **问题性质**：合规/业绩/配合度，决定响应优先级
- **群内影响**：其他供应商是否在围观？是否会影响规则公信力？
- **我的角色定位**：我是规则执行者，不是情绪安抚者

## 常见误区
- ❌ 被情绪绑架：供应商一撒娇就心软增量，破坏规则公平性
- ❌ 过度反应：每个异常都私聊介入，消耗大量时间
- ❌ 公开对线：在群里和供应商争辩，失去权威性
- ❌ 不记录：群内异常不记录，后续无法追溯模式
- ❌ 忽视模式：同一供应商反复异常，未纳入诚信或胜任评估

## 决策输出

**真问题处理流程**：
```
私聊了解 → 核实原因 → 要求整改 → 记录留痕 → 持续跟踪
```

**撒娇处理流程**：
```
公开回应规则 → 不单独安抚 → 观察是否持续 → 若反复出现纳入配合度评分
```

**对抗处理流程**：
```
公开重申规则 + 私聊严肃沟通 → 要求书面说明 → 升级约谈 → 记录到诚信档案
```

**回应话术示例**：

公开回应（撒娇型）：
```
@供应商 规则是统一的，所有供应商按同一套标准评估。你的情况我已记录，有具体问题可以私聊我。
```

私聊介入（真问题型）：
```
[供应商]，看到你在群里没回复。我看了下数据[具体问题]，需要你[具体动作]，X月X日前给我。有问题随时沟通。
```

严肃警告（对抗型）：
```
[供应商]，你在群内的说法与事实不符。我已记录，请你今天12点前提交书面说明。如果继续推诿，将升级处理。
```
