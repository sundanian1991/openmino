# E2-准入评审决策

> 新供应商完成6个月试运行后，决定是正式纳入还是延长观察/清退

## 触发信号
- 新供应商第6个月末，试运行期将满
- 该供应商近6个月排名：是否稳定在前70%？
- 第6个月表现：排名是否达标？是否有合规问题？

## 核心问题
- 该供应商是否达到正式纳入标准？
- 如果未达标，是延长观察期还是启动清退？
- 延长观察的时长和条件是什么？
- 如果清退，流程怎么走？（工具箱§十九决策授权矩阵）

## 思考步骤

**Step 1：对照6个月里程碑达标线（工具箱§七）**
```
引入期（第1-2月）：
✓ 出勤率 ≥85%
✓ 业务跑通，无重大合规问题

成长期（第3-4月）：
✓ GMV达到中位供应商的60%以上

稳定期（第5-6月）：
✓ 月度排名进入前70%（不在后30%）
✓ 排名稳定（不出现大幅波动）
```
- 任一阶段不达标 → 标记风险项，需说明原因
- 第6月末未达前70% → 触发"留或退"决策点

**Step 2：综合评估三要素**
| 评估维度 | 数据来源 | 达标标准 | 权重 |
|---------|---------|---------|------|
| **业绩排名** | 月度排名表 | 第6月末排名前70% | 40% |
| **合规记录** | 质检/客诉记录 | 无A类客诉/无诚信问题 | 30% |
| **稳定性** | 出勤/流失数据 | 出勤率≥85%、流失率<15% | 30% |

**计算方式**：
- 每项达标 = 100分，未达标 = 0分
- 加权总分 = 业绩×40% + 合规×30% + 稳定×30%
- ≥80分 → 正式纳入
- 60-80分 → 延长观察3个月
- <60分 → 启动清退评估

**Step 3：延长观察 vs 清退决策**
```
延长观察条件（工具箱§七）：
✓ 排名接近前70%（如第65-70%）
✓ 有明确改进动作且有效果
✓ 无合规红线问题
✓ 团队稳定，无大规模流失
→ 延长3个月，设定明确提升目标

清退评估条件（工具箱§一、§二）：
✓ 连续3个月后30%（已满足）
✓ 或出现诚信红线（数据造假≥2项）
✓ 或G项红色（负面清单）
→ 启动清退评估流程（工具箱§十九决策授权矩阵）
```

**Step 4：决策授权矩阵确认（工具箱§十九）**
| 动作 | 年老师权限 | 需王易人确认 | 需军哥确认 |
|------|-----------|-------------|-----------|
| 试运行转正 | 建议权 | 确认 | — |
| 延长观察期 | 建议权 | 确认 | — |
| 清退评估启动 | 建议权 | 确认 | 事前沟通 |

**Step 5：输出决策记录**
- 决策编号：TRN-YYYYMMDD-序号（转正）或 EXT-YYYYMMDD-序号（延长）或 CLR-YYYYMMDD-序号（清退）
- 决策依据：6个月数据总结 + 三要素得分
- 决策结论：转正/延长/清退
- 后续动作：转正后纳入常规管理 / 延长后的提升目标 / 清退流程启动

## 关键变量
- **第6个月排名**：是否稳定在前70%？（不在后30%是底线）
- **合规记录**：是否有A类客诉？是否有诚信问题（数据质量分≤70）？
- **改进趋势**：近3个月是上升还是下降？改进动作是否有效？
- **团队稳定性**：出勤率、流失率是否健康？
- **资源需求**：该供应商是否仍需资源扶持？还是已可独立运行？

## 常见误区
- ❌ 只看第6个月单月数据：需看6个月整体趋势和稳定性
- ❌ 忽略合规一票否决：有A类客诉或诚信问题即使业绩达标也应延长或清退
- ❌ 延长观察无限期：最长延长3个月，需设定明确提升目标
- ❌ 清退不按流程：需按决策授权矩阵，经易人哥确认、军哥事前沟通
- ❌ 不记录决策依据：需保存6个月数据总结作为审计依据

## 决策输出

**转正决策（≥80分）**：
```
决策编号：TRN-20260419-001
供应商：[供应商名]
试运行结论：正式纳入
依据：6个月排名均在前70%（最高第XX名，最低第XX名），无合规问题，出勤稳定≥88%。
下一步：纳入常规月度排名管理，按正式供应商执行所有流程。
```
→ 衔接：`supplier-onboard.md`（试运行转正通知书）

**延长观察（60-79分）**：
```
决策编号：EXT-20260419-001
供应商：[供应商名]
试运行结论：延长观察3个月
提升目标：[具体量化目标，如：第7月末排名进入前65%]
依据：排名接近前70%（第68%），但有[具体问题，如：出勤率波动大]。
下一步：每月复盘进展，第9月末再次评估。
```
→ 衔接：`onboarding-missing.md`（试运行延期通知）

**启动清退（<60分或诚信红线）**：
```
决策编号：CLR-20260419-001
供应商：[供应商名]
试运行结论：启动清退评估
依据：连续3个月后30%（第4月72% → 第5月78% → 第6月82%），或出现[诚信问题具体说明]。
流程：按清退评估流程执行（工具箱§一、§二、§十九）。
审批：需易人哥确认、军哥事前沟通。
```
→ 衔接：`clearance-missing.md`（清退决策）+ `compliance-missing.md`（如涉及诚信）

## 下一步衔接
- 转正 → `supplier-onboard.md`（试运行转正通知书）→ 纳入常规管理
- 延长 → `onboarding-missing.md`（试运行延期通知）→ 设定提升目标，每月复盘
- 清退 → 调用【A6-清退评估】→ `clearance-missing.md`（清退决策流程）
