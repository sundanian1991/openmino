# D1-被同事@数据进度

> 同事在内部群@你要数据或进度时，30秒内判断并快速回复

## 触发信号
- 刘伟佳问："XX供应商上周排名多少？"
- 刘乾坤问："今天巡检结果出来了吗？"
- 郭鑫催："那个新项目对接进度怎么样了？"
- 其他同事在群内@你要数据或进度更新

## 核心问题
- 这个信息我现在能回答吗？
- 如果需要查数据，最快什么方式拿到？
- 回复口径是什么（一句话模板）？

## 30秒快速判断

**Step 1：这是谁？要什么？**
| 提问人 | 典型问题 | 我的定位 |
|--------|---------|---------|
| 刘伟佳（策略组负责人） | 排名、趋势、异常信号 | 数据准确性优先，附简要分析 |
| 刘乾坤（同事） | 巡检结果、当日进度 | 事实陈述优先，简洁直接 |
| 郭鑫（同事） | 项目进度、待办事项 | 进度状态 + 下一步动作 |
| 其他同事 | 信息同步类 | 数据 + 状态，不展开分析 |

**Step 2：我现在能回答吗？**
- ✅ 能直接回答：排名、状态、进度百分比 → 立即回复
- ⚠️ 需查数据：具体数值、详细数据 → 告知"X分钟内发你"
- ❌ 需确认/向上请示：涉及决策、敏感操作 → "我先确认下，稍后回复"

**Step 3：回复口径（一句话模板）**
```
[同事名]，[事实陈述]。[[如果需要行动] 我[做什么]，[时间点]给你。]
```
示例：
- "伟佳，毅航上周排名第12（上升3名），数据已私发你。"
- "乾坤，今天巡检已完成，FCI 78分（绿区），报告已上传共享。"
- "郭鑫，新项目对接目前卡在供应商资质审核，我今天下班前给你明确时间表。"

## 5分钟深入版（如有时间）

**Step 1：明确需求本质**
- 是要原始数据？还是要我的判断？
- 是要单次快照？还是要趋势分析？

**Step 2：快速组织信息结构**
- 数据类：数值 + 对比（环比/同比/对标）
- 进度类：当前状态 + 完成度 + 阻塞点
- 判断类：结论 + 依据（1-2个关键指标）

**Step 3：预判追问**
- 对方可能接着问什么？
- 是否需要附上数据来源或详细报告？

## 关键变量
- **提问人身份**：策略组/同事/上级，决定回复详细程度
- **信息敏感度**：是否涉及决策、供应商评价
- **数据可得性**：系统自动数据 vs 需要人工整理
- **时间要求**："马上" vs "今天内" vs "不急"

## 常见误区
- ❌ 过度解释：同事问排名，不需要讲双维度分析
- ❌ 陷入细节：问进度时不需要展开每个卡点
- ❌ 不设预期：需要时间整理时不告知交付时间
- ❌ 被动等待：被追问第二次才回复，应主动告知进展

## 决策输出
**快复格式**：
```
[人名]，[核心事实]。[如需行动：我[动作]，[时间点]发你/给你。]
```

**示例**：
```
伟佳，赛维斯上周排名第8（下降2名），数据已私发你。
郭鑫，新项目寻源已完成3家候选，我今天下班前给你评估建议。
```
