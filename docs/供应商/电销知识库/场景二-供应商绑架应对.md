## 一、绑架场景分类

| 类型 | 表现 | 危害程度 |
|------|------|---------|
| **跳级绑架** | 不找你，找上级/业管，从上至下施压 | 中等 |
| **规模绑架** | 份额太大（50-60%），大而不能倒 | 高（最难处理） |
| 资源绑架 | 说人力不足、投入不够，威胁无法合作 | 低（可替代） |
| 外部退路绑架 | 有外部客户，威胁退出 | 低（可替代） |

**核心判断**：只要不是大到不能倒，都有替代方案。

## 三、规模绑架

### 场景描述

**特点**：
- 供应商在某业务占比超过50-60%
- 双向依赖：你离不开他，他也离不开你
- 大而不能倒

**现状**：
- 让他吃份额 → 很容易
- 让他往下掉 → 你也很痛苦
- 无法有效制衡

### 痛点

| 你能做的 | 后果 |
|---------|------|
| 强压他 | 他反压你（说不做就不做） |
| 让他掉份额 | 你也痛苦（业绩缺口） |
| 不动他 | 他越来越强，你越来越弱 |

## 五、管理方法论（可复用）

### 底层原则

**原则1：不妥协原则**
> 绑架类问题，妥协一次 = 后续不断妥协

**原则2：问题定义权**
> 跳级绑架的核心是"谁定义问题"，不被动接问题，自己定义边界和解决方案

**原则3：验证优先**
> 判断真假用交叉验证：同行→同业→上下游→内部交叉验证

---

### 判断框架

**"有绑架能力"标准**：
- 占比 ≥30%
- 业绩前三
- 人力 ≥100人

**验证链条**：
```
同行（其他公司情况）→ 同业（行业成本）→ 上下游（供应链）→ 内部（多家供应商交叉验证）
```

---

### 筹码盘点模型

| 能给的 | 不能给的 | 底线 |
|-------|---------|------|
| 稳定（少调整少波动） | 利润（给不了） | 用规则限制涨幅上限 |
| 规模（以规模换价格） | 原则性让步 | 培育替代供应商 |

---

### 反制跳级绑架公式

```
他不给方案 + 只抛问题
→ 你不被动回答
→ 自己定位具体问题 + 让供应商提建议 + 快速决策
```

---

### 差异化管理策略

| 供应商诉求优先级 | 你的激励方式 |
|-----------------|-------------|
| 规模优先 | 给规模 |
| 利润优先 | 给稳定（少调整） |
| 稳定优先 | 承诺少波动 |
