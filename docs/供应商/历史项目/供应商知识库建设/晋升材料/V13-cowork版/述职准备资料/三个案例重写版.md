# 三个案例重写版

> 基于深挖信息，融合同事的结构优势

---

## 案例一：激活竞争——用"选汰机制+自建标杆"打破渠道舒适区

**贡献1：通过引入竞争+自建标杆，打破渠道垄断，实现成本和效率双优化**

**结果：**
- 供应商单价**下降15%**，每月**降本100万**（671万→571万）
- 新供应商**75%进入季度TOP3**，倒逼老供应商主动优化
- 自建团队产能**+110%**（vs BPO +45%），成为对标配尺

**挑战：**
接手时渠道管理存在"三高一低"——老供应商**占比70%+**且价格**高于行业15%**，新商**6个月成活率不足20%**，缺乏成本对标，供应商挑肥拣瘦只做"好做的业务"，难的业务没人接。

**核心举措：**

**1、破垄断：引入8家新商，建立赛马机制**

**【引入提速】**
- 统一准入标准（9维度评估：过往业绩、同类型业务对标、管理方式、人员数量、公司背景、历史交付风格、配合的商务、内部团队架构等）
- 流程线上化（审批从4-5天→1天）
- 引入周期从30天→15天

**【分量策略】**
建立5维打分模型：
- 业绩：30%
- 质检客诉：25%
- 上人数字：15%
- 配合度：15%
- 战略打分：15%（扶持潜力、学习能力、额外技能）

**【动态激活】**
- 头部和尾部重点管理，中间供应商可激励成头部
- 不都给，也不一直不给，让供应商保持期待
- 季度内至少一个月进入综合排名前三（名单产能、名单转化、赛马、质检质量）

**2、建标杆：自建宿迁职场掌握核心产能**

**【0→1落地】**
- 亲自面试近200人，百里挑一选20人
- 看了300份简历，在宿迁待一个月
- 团队设置、绩效、结构设计全程参与
- 3个月搭建金企融合+企贷大额能力

**【选人画像】**
坐席：想赚钱、接受周六加班、年轻有冲劲、无干扰、纯粹
主管：有充分的带团队经验、有从0到1搭建团队的能力

**【压价锚点】**
- 自建产能+110% vs BPO +45%
- 费效进入健康区间（1.2倍以内）
- 人效达外包110%，反向推动供应商降价

**3、促降价：用数据倒逼供应商优化**

拿着自建团队数据去谈：
- "自建能做到110%增速，你们只有45%"
- "同样数据你们做得不如人家，还想拿更多钱？"
- 所有供应商一视同仁降价，不接受就不给业务
- 对冲：承诺"不是永远降，看好生命线"

**价值：** 从"单一依赖"到"多元竞争+自主可控"，形成可持续的成本优化机制。自建团队不仅是产能补充，更是**供应商管理的对标配尺**——通过建立效能基准，倒逼整个生态优化。

## 案例二：合规风控——在千人增长下建立"全过程防控"体系

**贡献2：通过建立全过程防控体系，在新增1000+人情况下实现全年零重大风险**

**结果：**
- **全年零纠纷**
- 清退4家违规供应商，**零产能中断**
- 巡检7次覆盖15家，红灯4次**全部化解**
- 6家问题供应商**全部整改闭环**

**挑战：**
业务新增1000+人，分布在15家供应商，缺乏统一标准和过程监控，合规风险累积。新人100%培训、巡检全覆盖、红线熔断，但怎么保证质量不降？怎么让供应商真正执行而不是走过场？

**核心举措：**

**1、事前：立规矩，让供应商"知道底线"**

**【五项制度】**
制定《职场现场规范》《违规处罚条例》等五项制度，统一准入门槛

**【薪资保障检查】**
- 给供应商100元，员工最少拿60元（检查薪资流水）
- 不允许供应商压榨员工，保证团队稳定性

**【品牌规范】**
- 禁止借用京东名义招聘
- 禁止使用未授权的logo、商标
- 招聘必须用自己主体，不得用子公司

**2、事中：抓过程，让风险"早发现早化解"**

**【巡检全覆盖】**
- 7次巡检，每次看2-3家供应商，每次3-5天
- 覆盖地区：承德、石家庄、合肥、宿迁等高频区
- 现场检查：空间布置、龙虎榜、公开业绩、话术管理、活动激励、作息薪资、员工满意度、工作时间、培训方式、招聘方式、品牌滥用

**【红灯化解机制】**
- 当场说问题+解决方法
- 24小时内发邮件
- 要求3-7天整改，后续提供整改情况
- 有张贴的撕掉，有实用的改掉

**【高频管理】**
- 每天复盘客诉数据
- 快速迭代调整
- 调整晚了会从好跌到差

**3、事后：严清退，让违规"有代价"**

**【递进式清退规则】**
- 警告 → 约谈 → **限制新进**（一段时间内不允许接其他业务，只能持续流失，不允许新增人力） → 清退

**【最难整改案例】**
现场硬件设施/工位不达标，100多名员工需要渐进式调整，花了约1个月时间

**【无损退出】**
让供应商心甘情愿自己申请：
- 两难选择：闹→被京东拉黑 vs 走→日后好相见
- 告知公司制度、说明行业影响、承诺不加黑名单（如果自己申请）
- 利用黑名单的行业威慑力

**价值：** 从"事后救火"到"过程防控"——让合规投入成为业务增长的护城河，而非成本负担。五项制度+管控机制成为**部门标准**，可复用到新业务。

## 案例三：线路应急——用"主动补位"化解核心业务中断危机

**贡献3：通过主动补位+高频调度，15天完成线路恢复（行业30天），避免约80亿GMV损失**

**结果：**
- **15天完成恢复**，领先行业15天
- 避免约**80亿GMV损失**（日影响1300万呼叫、37亿周GMV）
- 预案成为**部门标准流程**

**挑战：**
核心业务线路突发中断风险，日影响1300万呼叫、37亿周GMV。行业恢复周期约30天，团队无专职应急负责人（只有一个偏运营的业务运营，不是偏商务角色），这是边缘角色，没有应急角色，业务紧迫。

**核心举措：**

**1、找资源：快速识别"有指标但没用"的供应商**

**【结构性差异洞察】**
- 全国范围内突发政策监管针对三大运营商
- 每家供应商有自己的指标
- 成熟供应商指标用光了
- 部分供应商有指标但没用

**【快速匹配】**
- 快速找到有指标但没用的供应商
- 让他具备做业务的能力
- 这是我们能快速恢复的核心

**2、控客诉：高频管理确保指标不崩**

**【客诉比控制】**
- 剔除不适合的人
- 管理外呼频次
- 过程监控
- 每天沟通调整，快速迭代

**【关键：高频管理】**
- 供应商指标是次日前一天反映客诉结果
- 如果调整晚了，供应商的表现会从相对好直接跌到极差
- 每天复盘会：分析GMV指标、客诉比、客诉分数

**【数据和过程指标监控】**
- 实时跟踪客诉数据
- 快速调度资源
- 做好资源管控相关管理

**3、建预案：沉淀应急管理标准**

**【价值评估模型】**
- 成本（30%）：线路费（找涨幅相对少的，市场价涨200%-300%）
- 稳定性（40%）：过往做过类似案例、直联、有落地机房
- 覆盖率（30%）：资源丰富程度、重点覆盖的城市/运营商、管控严格程度

**【预案流程】**
- 预警→应急等级→对应动作
- 动作：降名单、降频控、降人力
- 梳理流程，从预警到应急的等级，到对应的动作

**【推动成为标准】**
复盘会 → 制度发文 → 培训落地

**价值：** 不仅解决突发危机，更通过主动补位与机制建设——从"被动救火"到"主动防控"。预案成为**部门标准流程**，未来类似危机可快速响应。

## 核心差异对比

| 维度 | 原版本 | 重写版本 |
|------|--------|----------|
| 结果位置 | 埋在最后 | **放在贡献的最前面** |
| 挑战描述 | 抽象形容词（"散僵盲弱乱"） | **具体业务痛点（"老供应商占比70%+且价格高于行业15%"）** |
| 动作逻辑 | 平行罗列（选、用、育、汰） | **因果链条清晰（破垄断→建标杆→促降价）** |
| 数据使用 | 只有结果有数据 | **每个动作都有数据支撑** |
| 价值提炼 | 概念化（"从依赖外包到自主可控"） | **量化+可复用（"自建团队成为供应商管理的对标配尺"）** |
| 方法论 | 隐含 | **显性化（五维打分模型、递进式清退规则、价值评估模型）** |

## 关键方法论提炼

### 1. 分量打分模型（案例一）
```
业绩（30%）+ 质检客诉（25%）+ 上人数字（15%）+ 配合度（15%）+ 战略打分（15%）
```

### 2. 递进式清退规则（案例二）
```
警告 → 约谈 → 限制新进（不允许接其他业务、只能持续流失、不允许新增人力） → 清退
```

### 3. 价值评估模型（案例三）
```
成本（30%）+ 稳定性（40%）+ 覆盖率（30%）
```
