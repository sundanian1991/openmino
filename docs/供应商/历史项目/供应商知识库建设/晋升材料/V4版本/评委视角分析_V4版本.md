## 四、"钩子"设计策略

### 4.1 评委提问的心理过程

评委看到材料时，会经历：

```
第一遍快速浏览 → 抓住关键数字和亮点
第二遍深度思考 → 寻找可以深入挖掘的点
第三遍准备问题 → 找"可以聊但没说透"的部分
```

**你的目标**：在第二遍、第三遍时，**引导评委关注你准备好的话题**

### 4.2 现有材料中的潜在钩子（未刻意设计）

| 位置 | 原文 | 评委可能问 | 你的准备度 |
|------|------|-----------|----------|
| 案例一-育 | "自建团队并搭建金企融合、企贷大额0-1能力" | "具体怎么搭建的？遇到了什么困难？" | ⚠️ 准备不充分 |
| 案例一-选 | "周期30天→15天" | "这15天是怎么省下来的？IT资源怎么协调的？" | ⚠️ 准备不充分 |
| 案例三-建 | "沉淀《线路应急管理预案》" | "预案的具体内容是什么？能讲一个具体应用场景吗？" | ⚠️ 准备不充分 |
| 未来目标 | "AI Agents应用探索" | "具体怎么落地？有什么初步想法？" | ⚠️ 准备不充分 |

### 4.3 建议的钩子设计方案

#### 设计原则

1. **高亮个人创新点** → 评委容易注意到
2. **留下具体数字但不说透** → 引导评委追问细节
3. **提及挑战但不说解决方案** → 展现你解决复杂问题的能力
4. **标注"可展开"的关键词** → 明确告诉评委"这里可以深挖"

#### 具体设计方案

**方式一：用【】标注关键钩子**

