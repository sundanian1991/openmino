# 收尾期 · 效果追踪与 Dashboard

> **定位**：培训效果管理的核心工具——培训不是结束，30天后才是验收点。
> **核心约束（萃取经验）**：年老师"想追踪但要轻量"——拿不到供应商内部数据，不能搞重型追踪。所以效果追踪的核心是**轻量但闭环**：不报数据，只问动作。

---

## ⚠️ 关键修正（来自经验萃取⑨）

原设计把 T+30 定为"系统追踪/回收落地效果数据"，但**供应商内部数据（完件率/授信率/流失率）实际拿不到**，T+30 重型追踪会落空。

**修正方向**：
- T+30 = **轻量回访**（只问动作，不报数据）
- T+90 = **重型数据核算**（联盟联合复盘，能拿到多少算多少）

---

## 一、效果测量时间线（修正版）

```
培训前   ← 供应商填写预习表（预期问题）
   │
培训中   ← 供应商填写对照记录表（可抄的动作）
   │
培训后 7天 ← 供应商提交 30天行动回收表（选定 1-2 个动作）
   │
培训后 30天 ← 【轻量回访】运营官1v1：动作启动了吗？有什么阻碍？（不报数据）
   │
培训后 90天 ← 【联合复盘】能拿到的数据核算（达标周期/流失率/首贷/复贷变化）
```

---

## 二、T+30 轻量回访（核心环节·修正重点）

> 这是效果追踪的真正抓手。不要求供应商报数据，只问 3 个问题，5 分钟搞定。

### T+30 回访 3 问

对每个参加培训的供应商，微信问这 3 个问题：

> "老 X，上次培训到现在一个月了，问你三个事：
>
> 1. **动起来了吗？**——上次选定的落地动作，开始做了没有？
> 2. **卡在哪了？**——如果没动，或者动了一半，主要卡在什么？
> 3. **需要什么支持？**——京东这边能帮上什么？"

### 回访目标

| 情况 | 占比目标 | 你的动作 |
|------|---------|---------|
| 已启动且有进展 | ≥40% | 记录亮点，下次分享案例 |
| 启动了但卡住 | ≥30% | 聊阻碍，京东侧能帮的帮 |
| 完全没动 | ≤30% | 了解原因（没空？不适用？没资源？），不强求 |

**核心原则**：回访不是考核，是**帮供应商扫除落地的障碍**。没动的不要批评，要问"为什么没动"——答案往往是培训内容不适用，这本身是有价值的反馈。

### 回访记录模板

```
供应商：[公司名]  回访日期：[YYYY-MM-DD]
动作启动：是/否
进展：[一句话]
卡点：[一句话，如有]
需要的支持：[一句话，如有]
分类：已启动有进展/启动卡住/完全没动
```

## 三、效果指标（两类·保留供 T+90 用）

### A. 人员层面（培训对供应商坐席的影响）

| 指标 | 衡量公式 | 评估时点 | 数据可得性 |
|------|---------|---------|-----------|
| 达标周期 | 新坐席从入职到稳定出单时间 | T+90 | 需供应商提供 |
| 流失率 | 季度离职率变化 | T+90 | 需供应商提供 |
| 首次放款转化率 | 首贷转化率变化 | T+90 | 需供应商提供 |
| 复贷率 | 复贷率变化 | T+90 | 需供应商提供 |

### B. 管理层面（培训对供应商自身的影响·这部分你能拿到）

| 指标 | 衡量公式 | 评估节点 |
|------|---------|---------|
| 动作启动率 | T+30回访中"已启动"的占比 | T+30 |
| 动作完成率 | 启动的动作按期完成率 | T+90 |
| 供应商满意度 | 培训 NPS（反馈评分表） | T+7 |
| 供应商续约率 | 参与下一期培训的比例 | 下一期 |

## 四、T+90 联合复盘（重型·能拿多少算多少）

每季度最后一期培训后做。这是唯一正式回收数据的节点。

- 向供应商收集：达标周期/流失率/首贷/复贷的变化趋势（能拿到多少算多少）
- 识别共性：哪些动作多家都在用？哪些卡点多家都遇到？
- 决策：继续？调整形式？暂停？

**降低预期**：T+90 的数据回收率可能只有 30-50%，这是正常的。重点是 T+30 的轻量回访要 100% 覆盖——那才是你能掌控的效果信号。

