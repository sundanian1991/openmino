> **定位**：⑥讲师评价技能的数据沉淀。累计每位讲师的打分+排名+荣誉轨迹，让荣誉授予有据可依。
> **特点**：框架已建好，数据随每期培训后的问卷回收积累。跑几期后形成"讲师能力排行榜"。
> **配合技能**：instructor-evaluator（⑥讲师评价）

## 五、数据来源与更新机制

| 数据 | 来源 | 更新时机 |
|------|------|---------|
| 单场打分 | ⑥讲师评价（问卷算分后） | 每期T+7 |
| 累计排名 | ⑥自动计算 | 每期T+7 |
| 荣誉触发 | ⑥自动判断 | 每期T+7 |
| 改进反馈 | 运营官记录 | 给讲师反馈后 |
| 梯队调整 | 季度复盘时 | 每季度 |

### 更新触发
以下情况**必须**更新本档案：
- 某讲师完成一次分享（补充上台记录+打分）
- 问卷回收算分完成（⑥输出打分卡）
- 荣誉触发条件满足（标记荣誉状态）
- 季度复盘（调整梯队）

## 六、年度评选流程（12月年度峰会前）

1. ⑥汇总全年数据 → 自动生成排名候选
2. 运营官审核候选名单 → 确认/调整
3. 输出年度荣誉清单：
   - 年度金牌讲师（个人，前3）
   - 年度金牌供应商（公司，前1）
   - 年度活跃供应商（公司，2-3）
   - 年度贡献供应商（公司，承办专场）
4. 年度大会颁发
