# 筹备期 · 课程设置编排表

> **定位**：培训课程编排的核心工具。用这张表，每次培训 5 分钟就能搭出骨架。
> **原则**：先定骨架（主题×时间×讲者×模块），再填血肉（每个模块的追问要点/预习问题/对照模板）。

---

## 一、课程设置核心原则

1. **供应商讲为主，京东讲为辅**：供应商实战分享占 60% 以上，京东同事不超过 20%
2. **讲已做到的，不讲想做的**：每模块要求主讲人给 2 个以上真实数据点
3. **每个模块留追问时间**：主讲人讲完不退场，留 5-10 分钟给组织者追问
4. **模块之间不切割**：前一模块的讨论可以自然延续到下一模块，不打断思路
5. **不贪多**：一天最多拆解 4-5 个模块，让每个模块讲透

### 底层方法论：亮点驱动（萃取经验）

> ⚠️ 这条经验在原资产里没有显式记录，但它是年老师课程设计的底层逻辑。

**核心逻辑：先找亮点供应商，围绕他的亮点设计整场。**

不是"先定主题再找能讲的人"，而是"先发现哪家有亮点，围绕这个亮点搭整场"。二期毅航宿州专场就是典型——毅航的复贷 S 率 50%、首贷转化高是亮点，所以全天围绕毅航拆解。

**操作步骤**：
1. 发现某家供应商有可量化的亮点（数据信号/行为信号/口碑信号，详见[讲师发展机制](./讲师发展机制.md)）
2. 围绕这个亮点设计核心模块（"他做对了什么"）
3. 补充延伸模块（"这个方法怎么迁移到其他家"）
4. 安排圆桌讨论（"各家怎么落地"）

**反例**：不要"先定主题'降本增效'再到处找谁能讲"——这样找到的人往往跟主题贴合度不够。要"发现某家降本做得特别好，再以他的案例为锚定主题"。

## 三、课程编排 Checklist

开工前逐项勾对：

### 定骨

- [ ] 确认培训形式：线上圆桌 / 线下大额培训
- [ ] 确认目标时长：60 分钟 / 半天 / 全天
- [ ] 确认核心主题：从年度选题库中选或新定
- [ ] 确认主讲方：[供应商名称]
- [ ] 确认主讲人：[姓名]，[角色]
- [ ] 确认京东同事参与（如有）：[姓名]，[角色]

### 填肉

- [ ] 给主讲人发分享模板（5 页 PPT），限定准备时间 2 小时以内
- [ ] 主讲人提交提纲后，运营官花 15 分钟审：
  - 有没有 2 个以上具体数据点？
  - 有没有真实的踩坑案例？
  - 有没有可复制的动作？
- [ ] 给主讲人反馈 2-3 条改进建议

### 定细节

- [ ] 确认主持人（线上：轮值供应商 / 线下：自己）
- [ ] 确认录制方案（线上：腾讯会议录制 / 线下：录屏+录音）
- [ ] 确认分组规则（线下：按层级分——老板组/运营组/培训组，每组 3-4 家）
- [ ] 确认茶歇/午饭安排（线下）
- [ ] 确认设备（投影/网络/白板/计时器）

## 四、模块选择指南

根据培训目标选模块，不每次都用全套。

| 培训目标 | 建议模块 | 建议形式 |
|---------|---------|---------|
| 新供应商入门（了解京东规则） | 数据策略+合规+激励 | 线上圆桌 60 分钟 |
| 供应商互学（提升运营水平） | 人力管理+话术结构+企微运营 | 线上圆桌 60 分钟 |
| 深度赋能（解决特定问题） | 一个模块深挖 4 小时 | 线下大额培训半天 |
| 年度复盘+下年规划 | 年度总结+下年方向讨论 | 线下大额培训全天 |

---

## 五、主讲方备课引导

给主讲方的核心提示：

- **不要讲"我们做得好"**：要讲"我们怎么做的"——拆解步骤、列数据、说踩过的坑
- **不要讲理论**：要讲真实案例，有具体数字和具体动作
- **不要留一手**：分享的案例会被收录进联盟案例库，署名，供其他供应商学习
- **准备时间不超过 2 小时**：5 页 PPT 模板，填空就好，不需要设计

### 选题五原则

选题是运营官最重要的决策。选题不对，整期报废。

| 原则 | 解释 | 反面案例 |
|------|------|---------|
| **讲已做的，不讲想做的** | 必须是有结果、有数据的实战，不是方法论 | "我们认为应该 XX" ✗ |
| **讲具体的，不讲宏大的** | 一篇只聚焦一个问题，不要"全方位管理提升" | "供应商精细化管理体系建设" ✗ |
| **讲对同行有用的** | 其他供应商听完能直接借鉴，不是公司宣传 | "我们公司发展历程介绍" ✗ |
| **有数据颗粒度** | 必须有具体数字，不能用"显著""大幅""明显" | "效率大幅提升" ✗ |
| **有失败的环节** | 诚实讲踩过的坑比全是成功经验更有价值 | "一切都很顺利" ✗ |

### 紧急替补话题（Plan B）

当原定主讲人临时不能讲时：

| 话题 | 不需要准备的人 | 形式 |
|------|-------------|------|
| "我最近遇到的一个麻烦" | 任何供应商 | 每人 5 分钟讲一个当前最头疼的问题，其他人给建议 |
| 行业热点快评 | 运营官 | 最近一条行业新闻，大家讨论对各自业务的影响 |
| "如果重来一次" | 任何供应商 | 每人分享一个"如果重来一次我会怎么做"的决策 |
| 工具互换 | 任何供应商 | 每人推荐一个最近在用的提效工具/方法 |

### 选题池扩充机制

从第三季度开始，选题不再由运营官一个人定：

1. **9 月复盘会**：供应商投票"今年最好的三期是哪些"，答案即是最好的选题方向
2. **供应商自荐**：复盘会上开放"明年你想讲什么"的征集，当场认领月份
3. **运营官补充**：供应商提的选题偏重操作层面，运营官补充 2-3 个战略/趋势类话题

**控制比例**：供应商提出+认领 8-9 期；京东同事场 2 期；运营官主讲 1-2 期（启动+年度）。
