# 需求调研工具包

> **定位**：⓪需求调研技能的可参考物料。调研模板+分类表+优先级矩阵。
> **特点**：框架已建好，调研数据随每期培训积累。每期调研结果归档，形成"供应商真实痛点"的纵向记录。
> **配合技能**：research-collector（⓪需求调研）

---

## 一、调研模板（1v1用，运营官直接复制）

> 每期培训前3-4周，对3-5家重点供应商1v1调研。每家15分钟。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
供应商：[公司名]  被访人：[姓名/职务]
调研日期：[YYYY-MM-DD]  调研人：[运营官]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【1. 最近最头疼的事】（场景化追问）
问："你最近一个月在[具体业务]上，具体卡在哪了？"
记录：___________________________________________

【2. 如果只能解决一个】（逼优先级）
问："如果这次培训只能帮你解决一个问题，你选哪个？"
记录：___________________________________________

【3. 现在的数据 vs 想达到的数据】（数字锚定）
问："这个问题你们现在的数字是多少？想做到多少？"
  痛点1：现在[___] → 想达到[___]
  痛点2：现在[___] → 想达到[___]

问："你觉得这个问题，其他供应商也会遇到吗？"

【5. 讲师线索】
问："你知道哪家在这个方面做得比较好吗？"
记录：___________________________________________

【6. 补充】（对方主动提到的、不在预设内的）
记录：___________________________________________
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 二、问题分类表（调研后整理用）

> 把所有1v1调研记录的问题，按三个维度分类填入。

| # | 问题（原话提炼） | 提出商家数 | 主题归类 | 对应元主题 | 痛点深度 | 普遍性 |
|---|---------------|-----------|---------|-----------|---------|--------|
| 1 | [问题1] | [X]家 | 人员/业务/管理/合规/方法/协作 | [元主题#] | 深/浅 | 高频/低频 |
| 2 | [问题2] | | | | | |

### 分类维度说明

**主题归类**（对照[元主题库](../04-持续工具层/元主题库.md)6大类）：
- 👥 人员：留存/排班/激励/培训
- 🎯 业务：冷启动/客诉/外耗
- 🛡️ 管理：降本/质检/危机
- 🔧 方法：数据/工具
- 🤝 协作：甲方沟通/跨供应商
- 🪞 复盘：年度复盘

**痛点深度判断**：
- 深：不解决会影响业绩/成本/合规（值不值得花一期讲）
- 浅：只是麻烦，不影响核心指标

**普遍性判断**：
- 高频：≥3家提到（讲了覆盖面广）
- 低频：1-2家提到（个别处理）

## 三、优先级矩阵（核心决策工具）

> 分类后填入矩阵，决定本期讲什么。

| | 深痛（影响业绩/成本/合规） | 浅痛（只是麻烦） |
|---|---|---|
| **高频**（≥3家） | 🔴 **必讲**（本期主题候选） | 🟡 值得讲（备选/穿插） |
| **低频**（1-2家） | 🟠 个案处理（1v1辅导，不占培训） | ⚪ 不讲 |

### 决策规则
- 🔴 **必讲**中选 **1个** 作为本期主题（一期聚焦一个，不贪多）
- 🟡 值得讲的作为备选，或穿插在圆桌讨论环节
- 🟠 个案的转 1v1 辅导（不浪费大家时间）
- ⚪ 不讲的果断放弃

## 四、选题建议模板（输出给运营官拍板）

```markdown
# 第[X]期 · 选题建议

## 本期主题推荐
推荐：[高频深痛主题]（[X]家提到，对应元主题[#]）
  候选讲师方向：[供应商A]（数据：[X]）、[供应商B]（数据：[X]）
备选：[高频深痛主题2]（[X]家提到）

## 问题分布
  🔴 必讲：[X]个 → [列出]
  🟡 备选：[X]个 → [列出]
  🟠 个案：[X]个 → [转1v1]
  ⚪ 不讲：[X]个

## 遗留问题（进入下期调研池）
  [本期排不上的高频问题]
```

## 五、调研结果纵向归档（跨期积累）

> 每期调研结果归档，跑几期后能看出"持续高频深痛"vs"一次性问题"。

| 期数 | 调研日期 | 调研家数 | 本期选定主题 | 遗留到下期的问题 |
|------|---------|---------|------------|----------------|
| 第3期 | [日期] | [X]家 | [主题] | [问题1,问题2] |
| 第4期 | | | | |

**纵向分析价值**：
- 某问题连续3期被高频提到 → 持续深痛点，值得反复讲或做专题
- 某问题只出现1次就消失 → 一次性问题，已解决或不重要
- 某主题反复被选 → 元主题库的热门方向，深化优先级高

## 六、使用说明

| 时机 | 用什么 |
|------|--------|
| 培训前3-4周 | 用【调研模板】1v1深访3-5家 |
| 调研完 | 用【分类表】整理问题 |
| 整理完 | 用【优先级矩阵】定优先级 |
| 决策时 | 用【选题建议模板】输出给运营官拍板 |
| 每期结束 | 用【纵向归档】记录，跨期分析 |
