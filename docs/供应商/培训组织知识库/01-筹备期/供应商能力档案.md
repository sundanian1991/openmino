## 四、数据来源与更新机制

### 数据从哪来

| 来源 | 怎么获取 | 更新频率 |
|------|---------|---------|
| 业绩排名 | 每月业绩数据 | 月度 |
| 日常观察 | 运营官对接过程中的观察 | 随时 |
| 培训分享 | 每期讲师讲的内容（④总结转化产出） | 每期 |
| T+30回访 | ⑦效果追踪中"已启动有进展"的供应商 | 每期后30天 |
| 同行口碑 | 其他供应商提到的"XX做得不错" | 随时 |

### 更新触发

以下情况**必须**更新本档案：
- 某供应商有新的业绩亮点（月度排名异常）
- 某供应商完成一次培训分享（补充已讲场次+评价）
- 调研中有人提到"XX做得好"（补充口碑信号）
- T+30回访发现某供应商落地效果好（补充实战验证）

## 五、怎么用这份档案

### ②a亮点挖掘时
1. ⓪确认本期主题（对应某元主题）
2. 查本档案"能力矩阵"→ 谁在这个元主题上有✓
3. 查对应供应商"详情卡"→ 亮点数据+适合台阶
4. 输出候选讲师建议

### 讲师梯队建设时
- 查总览→谁活跃、谁沉默
- 查矩阵→谁亮点多（L3候选）、谁单一（L2稳定）
- 查详情→谁的适合台阶还没到位（需培养）

### 年度荣誉评选时
- 供应商轨道：查总览的"上台次数+接待专场"
- 结合⑥讲师档案库的打分→算综合分
