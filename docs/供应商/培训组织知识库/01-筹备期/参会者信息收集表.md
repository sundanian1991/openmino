# 参会者信息收集表

> **用途**：线下大额培训前，统一收集所有参会者的信息（人员/行程/住宿/特殊需求），避免信息散在聊天记录里、临场变卦难追踪。
> **适用场景**：线下大额培训（线上圆桌不需要）
> **推荐工具**：飞书多维表格 / 腾讯文档协同表（多人同时填、改、查）

---

## 为什么需要这张表

萃取经验：年老师实际用表格收集，但**只收了人员基本信息**，行程/住宿/特殊需求散在微信里。直接导致两个痛点——

1. **信息乱**：谁到谁不到、几点到、特殊需求，散在多个聊天记录里理不清
2. **临场变卦多**：确认了又改、临时换人，通知要反复发

解法：**一张表收敛所有维度**，所有变卦都在表里改，通知依据表来发。

## 完整字段定义

### 维度一：人员基本信息

| 字段 | 说明 | 示例 | 必填 |
|------|------|------|------|
| 公司名称 | 参会者所属供应商 | 毅航 | ✅ |
| 姓名 | 本人姓名 | 曹总 | ✅ |
| 职务 | 实际职务（影响分组） | 总经理 / 运营主管 / 组长 | ✅ |
| 手机号 | 现场联络用 | 138xxxx | ✅ |
| 微信 | 备用联络 | 同手机号 | ⬜ |
| 是否主讲人 | 标识角色 | 是 / 否 | ✅ |
| 任职级层 | 用于按层级分组（核心经验） | 老板层 / 管理层 / 一线层 | ✅ |

> **级层字段说明**（萃取经验）：圆桌讨论按层级分组——老板聊经营战略、主管聊管理执行、一线聊带教话术。同层级更敢说真话。这一条来自二期引导策略。

### 维度二：行程信息

| 字段 | 说明 | 示例 | 必填 |
|------|------|------|------|
| 是否参会 | 最终确认 | 是 / 否 / 待定 | ✅ |
| 到达方式 | 高铁/飞机/自驾 | 高铁 | ✅ |
| 出发城市 | 接站/接机判断 | 宿州本地 / 合肥 / 北京 | ✅ |
| 到达时间 | 几月几日几点到哪 | 5/7 下午 Gxxxx 宿州东站 | ✅ |
| 是否需接站 | 站点/机场接送 | 是（宿州东站）/ 否 / 自理 | ✅ |
| 返程时间 | 几号离开 | 5/8 晚上 | ⬜ |

### 维度三：住宿信息

| 字段 | 说明 | 示例 | 必填 |
|------|------|------|------|
| 是否需安排住宿 | 统一订房还是自理 | 是 / 否 | ✅ |
| 入住日期 | 几号入住 | 5/7 | ✅(若需住宿) |
| 离店日期 | 几号退房 | 5/8 | ✅(若需住宿) |
| 房型要求 | 单间/标间/大床 | 单间 | ✅(若需住宿) |
| 同住人 | 标间拼房安排 | 无 / 指定 XXX | ⬜ |
| 酒店特殊要求 | 高层/安静/无烟 | 无 | ⬜ |

### 维度四：特殊需求

| 字段 | 说明 | 示例 | 必填 |
|------|------|------|------|
| 饮食禁忌 | 忌口/过敏/宗教 | 不吃辣 / 海鲜过敏 / 清真 | ✅ |
| 其他备注 | 任何需要提前知道的 | 行动不便需一楼 / 提前离场 | ⬜ |

## 协同使用规范

### 谁来填

**推荐方式**：发给每家供应商的对接人，由对接人统一填报本公司所有参会者。不要让每个人单独填——会乱、会漏、会重复。

> 话术参考：「老X，麻烦你帮我把你们这边参会的人都填进这个表，包括行程和住宿。后面接送站、订房、分组都按这个表来，有变动随时在表里改，我每天看一次。」

### 变更管理

**核心规则：所有变卦都在表里改，不在微信里说。**

| 场景 | 错误做法 | 正确做法 |
|------|---------|---------|
| 临时换人 | 微信说"换成张三" | 表里改"李四→张三"，同步通知 |
| 时间变动 | 多条微信反复确认 | 表里改到达时间，标红 |
| 取消参会 | 默认不来了不说 | 表里改"参会=否"，触发后续调整 |

### 你的使用节奏

| 时间点 | 你要做的 |
|--------|---------|
| 发出邀请函后 | 把表链接发给各对接人，要求 T-10 前填完 |
| T-7 | 第一次核对，催未填的 |
| T-3 | 锁定，统计接送站/住宿/餐饮总数 |
| T-1 | 最终确认，打印一份备用 |
| 现场签到 | 对照表核实到场情况 |

## 派生用途（一张表多处用）

填完这张表后，可以自动派生出：

1. **接送站清单**：按"到达方式+到达时间"排序，安排接站车辆
2. **住宿安排表**：按"酒店+房型+入住日期"汇总，提交给酒店
3. **餐饮统计**：按"饮食禁忌"汇总，提前告知餐厅
4. **分组名单**：按"任职级层"分组，现场直接宣布
5. **签到表**：按"公司+姓名"生成现场签到用

## 飞书多维表格建议视图

如果用飞书多维表格，建议建这几个视图：

- **全部信息**：默认视图，所有字段
- **接送站**：只看 行程信息 相关字段，按到达时间排序
- **住宿**：只看 住宿信息 相关字段，按酒店/房型分组
- **分组名单**：只看 公司/姓名/级层，按级层分组
- **待跟进**：筛选"是否参会=待定"或关键字段为空的行
