## 第一章 总则

### 1.1 目的

规范京东数据科技业务部电销服务组新供应商引入全流程，确保引入质量、控制集中度风险、保障业务连续性。所有规则以《供应商全生命周期管理指引（精华版）》为最高依据。

### 1.2 适用范围

适用于所有拟承接 C2 电销 BPO 业务的新供应商引入，覆盖四大核心业务线（金条、企业金融、联名卡、财富管理）及创新业务试点。含人工外呼供应商和人机协同供应商。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 引入 | 从需求发起到供应商培训上线的全流程 |
| 寻源 | 对外触达、筛选、评估潜在供应商的过程 |
| xbp | 内部审批系统 |
| 京音 | 京东电销业务系统，用于供应商管理、账号开通、数据对接 |
| CID | 供应商唯一身份标识信息 |
| 标准周期 | 从需求发起到上线外呼共15个工作日 |

### 1.4 角色定义

| 角色 | 简称 | 职责 |
|------|------|------|
| 策略运营岗 | C2 | 需求发起、终审（>10人）、分量最终决策 |
| 服务组负责人 | C3 | 审批（≤10人）、日常审核、备案 |
| 供应商管理岗 | 供管岗 | 引入全流程执行、评估、考察、建档 |
| 业务管理岗 | 业管 | 协同需求拆解、系统操作、日常对接 |
| 质检团队 | 质检 | 质量评估、合规审查、试运行评审 |

### 1.5 管理原则

| 原则 | 含义 | 来源 |
|------|------|------|
| **业务分类** | 按人力阈值区分核心/创新，走不同审批链路 | [精华版]§一 |
| **多方比选** | 核心业务引入须至少2家供应商对比评估 | [精华版]§一 |
| **量化评估** | 所有评价基于客观数据，评分有据可查 | [总纲]§三 |
| **合规底线** | 基础资质一票否决，无例外 | [精华版]§四 |
| **集中度管控** | 引入前须检查集中度，超标不予引入 | [精华版]§九 |
| **全流程留痕** | 每阶段输出物须存档，审计检查点可追溯 | [SOP] |

## 第二章 业务分类与管理原则

> 来源：[精华版]§一、§三

### 2.1 业务分类

| 类型 | 定义 | 人力阈值 | 管理差异 |
|------|------|---------|---------|
| **核心业务** | 金条、企业金融、联名卡、财富管理 | >10人 | C2终审，强制多方比选 |
| **创新业务** | 试点探索阶段业务 | ≤10人 | C3终审，快速通道 |

### 2.2 分类判定规则

| 维度 | 核心业务 | 创新业务 |
|------|---------|---------|
| 引入人数 | >10人 | ≤10人 |
| 审批流 | C3审核 → C2终审 | C3终审 |
| 审批时限 | C3≤3天，C2≤5天 | C3≤3天 |
| 测试期 | 3-6个月 | ≤12周 |
| 测试规模 | 10-100人 | 5-10人 |
| 比选要求 | 强制多方比选 | 可直接引入 |

## 第三章 准入标准

> 来源：[精华版]§四、[总纲]§四

### 3.1 基础资质（一票否决）

以下任一条件不满足，**直接否决，不得进入后续流程**：

| 序号 | 资质要求 | 验证方式 |
|------|---------|---------|
| 1 | 注册资本 **≥1000万元** | 营业执照 + 工商信息 |
| 2 | 《增值电信业务经营许可证》（全国性呼叫中心） | 许可证原件核验 |
| 3 | 营业执照覆盖**拟合作业务**范围 | 经营范围核对 |
| 4 | 资产负债率 **≤60%** | 近一年财务报表 |
| 5 | 综合评分 **≥85分**方可入库 | 9维度评估打分（见3.3） |

**特殊引入规则**：综合评分 <85分但确需引入的，必须 **C3 + C2 共同审批**，并记录特殊引入理由存档。

### 3.2 业务能力要求

| 序号 | 能力要求 | 标准 | 来源 |
|------|---------|------|------|
| 1 | 总人力规模 | **≥50人**坐席 | [总纲]§四 |
| 2 | 业务经验 | 有**营销型业务经验**（电销/呼叫/在线营销） | [精华版]§四 |
| 3 | 质量管控 | 具备**成熟质量管控体系**（独立质检团队或第三方质检协议） | [精华版]§四 |
| 4 | 人员资质 | 金融相关资质持证率 **100%** | [总纲]§四 |
| 5 | 数据安全 | 数据安全防护能力通过我方评估 | [总纲]§四 |
| 6 | 合规记录 | 无重大商业违规或法律诉讼记录 | [总纲]§四 |

### 3.3 准入评分框架（9维度 · 10分制）

> 综合评分采用 **9维度评估**，每项0-10分，满分90分。换算为百分制判定入库门槛。

| 维度 | 权重 | 评分要点 | 数据来源 |
|------|------|---------|---------|
| 1. 资质合规 | 15% | 注册资本、许可证、经营范围、资产负债率 | 资质材料 |
| 2. 项目经验 | 15% | 电销/呼叫/营销业务经验年限、甲方背景 | 案例证明 |
| 3. 招聘能力 | 12% | 招聘渠道、到岗速度、人才储备 | 招聘方案 |
| 4. 培训体系 | 10% | 培训课程、达产周期、复训机制 | 培训方案 |
| 5. 质量管理 | 12% | 质检体系、合规自查机制、改善闭环 | 质检方案 |
| 6. 配合意愿 | 8% | 沟通响应速度、合作态度、管理配合度 | 沟通过程 |
| 7. 报价合理性 | 10% | 成本结构、市场对标、性价比 | 报价方案 |
| 8. 技术能力 | 10% | 系统对接能力、数据安全保障 | 技术评估 |
| 9. 组织管理 | 8% | 管理团队稳定性、现场管理能力 | 现场考察 |

**综合评分 = Σ（各维度得分 × 权重）**（满分90分，按满分90折算为百分制）

**入库门槛**：百分制综合评分 **≥85分**

**评估方式**：

| 方式 | 阶段 | 说明 |
|------|------|------|
| 桌面调查 | 资质审核阶段 | 9维度初步打分 |
| 现场考察 | 现场考察阶段 | 9维度实地验证 |

### 3.4 准入条件汇总（须同时满足）

> [总纲]§四

1. 具备有效营业执照及相关业务经营资质
2. 坐席规模 **不低于50人**
3. 人员持证率（金融相关资质）达到 **100%**
4. 具备**独立质检团队**或第三方质检协议
5. 数据安全防护能力通过我方评估
6. 无重大商业违规或法律诉讼记录
7. 基础资质全部通过（一票否决项）
8. 综合评分 **≥85分**

## 第四章 引入SOP（8阶段）

> 来源：[SOP]阶段1~8、[精华版]§十七、[运营管理要求]第三章

### 4.1 流程总览

```
阶段1        阶段2        阶段3        阶段4        阶段5
需求发起 →  需求拆解 →  寻源触达 →  方案审批 →  结果同步
  C2         业管/供管    业管        供管        供管
  T+0        T+1         T+3         T+5         T+5

阶段6        阶段7        阶段8
供应商准备 → 内部操作 →  培训上线
  供应商     业管/供管    供应商+业管
  T+10       T+12        T+15

整体标准周期：15个工作日
```

### 4.2 阶段1：需求发起

| 项目 | 内容 |
|------|------|
| 触发条件 | 策略岗识别到业务缺口，需要新供应商承接 |
| 责任人 | 策略运营岗（C2） |
| 输入物 | 业务量预测数据、现有供应商产能分析、缺口说明 |
| 执行清单 | 填写《需求清单》；明确需求量（人力/产能）；明确业务类型（拉新/复购）；明确时效要求 |
| 输出物 | 《需求清单》（邮件/文档） |
| 验收标准 | 需求清晰、可量化、可拆解、可执行 |
| 时效 | 标准 T+0 |
| 审计检查点 | 《需求清单》已存档；需求有数据支撑；有策略岗负责人签字/确认 |

### 4.3 阶段2：需求拆解

| 项目 | 内容 |
|------|------|
| 触发条件 | 收到策略岗《需求清单》 |
| 责任人 | 业管 / 供应商管理岗 |
| 输入物 | 《需求清单》 |
| 执行清单 | 解读需求（业务类型、人力规模、技能要求）；拆解为可执行需求（供应商画像）；评估现有供应商承接能力；确定是否需要外部寻源；填写《寻源需求表》 |
| 输出物 | 《寻源需求表》 |
| 验收标准 | 需求具体化为供应商画像，可量化、可筛选 |
| 时效 | 标准 T+1 |
| 审计检查点 | 《寻源需求表》已存档；有现有供应商评估记录；外部寻源决策有依据 |

### 4.4 阶段3：寻源触达

| 项目 | 内容 |
|------|------|
| 触发条件 | 《寻源需求表》确认需要外部寻源 |
| 责任人 | 业管 |
| 输入物 | 《寻源需求表》 |
| 执行清单 | 发送寻源邮件；设定回复截止时间；跟进未回复供应商；收集供应商回复；初步筛选意向供应商 |
| 输出物 | 供应商回复汇总 + 意向名单 |
| 验收标准 | ≥3家有效回复，有明确意向 |
| 时效 | 标准 T+3 |
| 审计检查点 | 寻源邮件已存档；回复供应商名单已记录；筛选标准客观、可追溯 |

### 4.5 阶段4：方案审批

| 项目 | 内容 |
|------|------|
| 触发条件 | 完成供应商初步筛选 |
| 责任人 | 供应商管理岗 |
| 输入物 | 意向供应商名单、供应商评估表 |
| 执行清单 | 整理供应商资质信息；填写xbp方案（含供应商对比评估、集中度检查、分量安排、定价建议）；提交领导审批；跟进审批进度 |
| 输出物 | xbp审批通过方案 |
| 验收标准 | 领导审批通过，无 pending 问题 |
| 时效 | 标准 T+5 |
| 审计检查点 | xbp方案已存档；审批流程完整；领导审批意见已记录；集中度检查已通过 |

**xbp方案须包含**：

> 方案摘要 → 需求背景（业务缺口 + 需求量测算）→ 寻源过程（触达名单 + 筛选标准）→ 推荐供应商评估（基本信息 + 核心能力评估 + 风险点及应对）→ 合作方案（分量安排 + 定价建议 + 集中度检查）→ 审批请求

**集中度检查**（强制）：引入前须检查引入后各业务线集中度是否超标，超标不予引入或同步缩减其他供应商分量。详见第九章。

### 4.6 阶段5：结果同步

| 项目 | 内容 |
|------|------|
| 触发条件 | 领导审批通过 |
| 责任人 | 供应商管理岗 |
| 输入物 | 审批通过的方案 |
| 执行清单 | 起草《结果同步邮件》；发送给策略岗和业管；发送入选/未入选通知给参选供应商；存档同步记录 |
| 输出物 | 《结果同步邮件》+ 发送记录 |
| 验收标准 | 各方都收到通知，无信息差 |
| 时效 | 标准 T+5 |
| 审计检查点 | 同步邮件已存档；发送名单完整；参选供应商均收到通知 |

### 4.7 阶段6：供应商准备

| 项目 | 内容 |
|------|------|
| 触发条件 | 结果同步完成 |
| 责任人 | 供应商（执行），业管跟进 |
| 输入物 | 准入通知 |
| 执行清单 | 供应商同步CID信息；完成站点报备；确认合同模板；提交所需资质文件 |
| 输出物 | CID信息表、站点报备确认单、合同模板确认函 |
| 验收标准 | 三项准备全部完成，无遗漏 |
| 时效 | 标准 T+10 |
| 审计检查点 | CID信息已存档；站点报备有记录；合同模板已确认并存档；资质文件完整 |

### 4.8 阶段7：内部操作

| 项目 | 内容 |
|------|------|
| 触发条件 | 供应商完成阶段6准备 |
| 责任人 | 业管 / 供应商管理岗 |
| 输入物 | CID信息、站点报备确认、合同模板确认 |
| 执行清单 | 在京音系统建供应商；建立供应商架构；开通账号；配置权限；测试账号可用性 |
| 输出物 | 京音供应商档案、账号开通确认、权限配置记录 |
| 验收标准 | 账号可用，权限配置正确 |
| 时效 | 标准 T+12 |
| 审计检查点 | 京音系统有供应商档案；账号开通记录已存档；权限配置符合规范；有账号测试记录 |

### 4.9 阶段8：培训上线

| 项目 | 内容 |
|------|------|
| 触发条件 | 账号开通完成 |
| 责任人 | 供应商（执行培训）+ 业管（组织/跟进） |
| 输入物 | 开通的账号、培训材料 |
| 执行清单 | 供应商完成培训；培训考核/验收；安排上线外呼；监控首日外呼质量；记录上线情况 |
| 输出物 | 培训完成记录、上线外呼记录、首日质量报告 |
| 验收标准 | 供应商正式上线，开始外呼；人员持证率100% |
| 时效 | 标准 T+15 |
| 审计检查点 | 培训记录已存档；培训考核结果已记录；上线外呼时间已记录；首日质量报告已存档；人员持证率达标 |

| M7 | 数据报送 | 报送责任人、数据核对、信息安全 |

## 第五章 测试期管理

> 来源：[精华版]§五

### 5.1 测试期标准

| 类型 | 测试周期 | 测试规模 | 通过标准 |
|------|---------|---------|---------|
| **核心业务** | 3-6个月 | 10-100人 | 达到策略目标值 |
| **创新业务** | ≤12周 | 5-10人 | 核心指标达标 |

### 5.2 关键规则

- **自动过渡期**：测试期结束后自动进入 **5个工作日** 评估过渡期（业务正常交付）
- **结算规则**：过渡期计入正式合作期并按标准结算
- **多家测试**：统一标准量化排名，按排名决定引入顺序和初始分量
- **评估主体**：供管岗 + 质检 + 业务方共同评估

### 5.3 测试期与准入流程的关系

| 测试类型 | 关系 | 说明 |
|---------|------|------|
| 核心业务测试 | 准入流程8阶段完成后进入 | T+15上线后开始3-6个月测试期 |
| 创新业务测试 | 准入流程8阶段完成后进入 | T+15上线后开始≤12周测试期 |
| 测试期通过 | 自动进入5个工作日评估过渡期 → 正式合作 | |

## 第六章 新供应商6个月里程碑

> 来源：[产能双维度]§七（新供应商培育4阶段）、[管理思路梳理]模板8

### 6.1 里程碑阶段划分

| 阶段 | 时间 | 阶段名称 | 核心目标 | 量化检查点 |
|------|------|---------|---------|-----------|
| **M1-M2** | 第1-2月 | 引入期 | 出勤稳定、合规达标 | 出勤率 ≥85%；无重大合规事件；管理制度7模块全部提交 |
| **M3-M4** | 第3-4月 | 成长期 | 产能爬坡 | GMV达到同产线中位数的 **60%**；人员稳定 |
| **M5-M6** | 第5-6月 | 稳定期 | 排名进入前70% | 月度排名进入同产线 **前70%**；主管胜任评估≥3项达标 |
| **M6** | 第6月末 | 评判点 | 留或退 | 综合评估：未达前70% → 评估延长或清退 |

### 6.2 每阶段复盘机制

每阶段**双周复盘一次**，回答三个问题：

1. **达标了吗？** → 对照量化检查点
2. **没达标的原因？** → 区分供应商自身问题 vs 甲方问题（名单/产品/系统）
3. **需要调整什么？** → 加名单/换主管/延长阶段

### 6.3 合规红线（一票否决）

> [产能双维度]§七

- 合规红线 **一票否决**，不看业绩
- 触发红线直接启动整改或清退流程

### 6.4 甲方责任追溯

若引入期/成长期未达标，须判断是否为甲方责任：

| 甲方责任场景 | 处理方式 |
|-------------|---------|
| 甲方要求加人 | 不考核新人产能，给予2个月豁免期 |
| 甲方给名单质量问题 | 调整名单，不纳入供应商考核 |
| 甲方调整产线策略 | 调整目标值，不纳入供应商考核 |
| 供应商自主扩编 | 不享受豁免，正常考核 |

## 第九章 集中度管控

> 来源：[精华版]§九

### 9.1 n-1业务参与上限原则

| 项目 | 内容 |
|------|------|
| 规则 | 单一供应商最多参与 **n-1** 个核心业务 |
| 当前 | 4个核心业务，最多参与 **3个** |
| 豁免 | 须C2终审（特例，不具普适性） |

### 9.2 单一业务量集中度上限

#### AI场景（按供应商数量动态）

| 供应商数量 | 单一供应商上限 |
|-----------|--------------|
| ≤3家 | ≤70% |
| 4家 | ≤60% |
| ≥5家 | ≤50% |

#### 人工场景（人数上限 + 占比上限，取孰低）

| 业务线 | 人数上限 | 占比上限 |
|--------|---------|---------|
| 金条人工 | 360人 | 30% |
| 金条人机 | 120人 | 40% |
| 信用卡 | 220人 | 40% |
| 财富 | 100人 | 40% |
| 企业金融-主贷 | 100人 | 50% |
| 企业金融-小b | 60人 | 60% |
| 商家白条 | 12人 | 60% |

### 9.3 引入前集中度检查（强制）

> [精华版]§九、[SOP]阶段4

引入新供应商前必须检查：

- 引入后各业务线集中度是否超标
- TOP3供应商占比变化
- 是否触发 n-1 原则

**超标处理**：集中度超标 → 不予引入或同步缩减其他供应商分量

### 9.4 监控与调整

- 月度监控集中度指标
- 超标时启动分量调整
- 调整须遵守审批流速查（第七章）

## 第十章 异常处理

> 来源：[SOP]、[总纲]§五、[日常手册]第四章

### 10.1 引入过程异常

| 异常类型 | 处置方式 |
|----------|---------|
| 供应商反悔 | 从意向名单中顺位递补，或重新启动寻源触达（阶段3） |
| 审批不通过 | 根据审批意见补充材料或更换供应商，重新提交方案（阶段4） |
| 资质造假 | 立即终止引入，纳入黑名单，1年内禁入 |
| 供应商准备超时 | 催办后仍不配合，评估是否更换供应商 |
| 首日质量不达标 | 延期转正，加强培训，纳入6月里程碑跟踪 |
| 基础资质不满足 | 一票否决，终止流程 |

### 10.2 应急措施

**紧急引入流程**（缩短时效）：

| 条件 | 审批 | 时效 |
|------|------|------|
| 重大业务缺口，现有供应商无法承接，且48小时内须上线 | C2审批 | 标准周期压缩至7个工作日，跳过非关键审计检查点，事后补录 |
