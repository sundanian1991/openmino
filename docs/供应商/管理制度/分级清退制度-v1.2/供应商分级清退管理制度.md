# 供应商分级清退管理制度

> 版本：v1.2（终稿）| 生效日期：2026-06-01 | 适用范围：全产线供应商 | 修订日期：2026-05-13

---

## 第一章 总则

**第一条 目的**

建立供应商分级评估和清退机制，实现"优者留、劣者退"，确保产能质量可控、资源分配合理、供应商生态健康。

**第二条 适用范围**

本制度适用于所有在合作供应商（含金条、企金、信用卡、财富四条产线），以及储备库中的潜在供应商。

**第三条 评估频率与体系关系**

| 评估类型 | 频率 | 定位 | 输出 |
|---------|------|------|------|
| **月度评估** | 每月 | 站点级日常考核（100分制，ABCDE五级） | 站点排名、预警/整改触发 |
| **季度评估** | 每季度末 | 供应商公司级综合评级（九宫格，ABC三档） | 三张清单（保留/培养/优化） |
| **半年度盘点** | 6月、12月 | 深度盘点：战略意义、长期潜力、风险敞口 | 最终决策校准 |

**体系关系**：月度评估是季度评估的数据基础。季度评估不重复采集数据，而是聚合月度评估结果并追加稳定性评定和供管自评。两套体系独立运作，但数据同源。月度评估按《供应商月度综合评估方案》执行，季度评估按本制度执行。

## 第二章 评估体系

**第四条 季度评估维度**

季度评估采用"三维评分 + 加分项"体系：

| 维度 | 说明 | 数据来源 |
|------|------|---------|
| **赛马基础分** | 季度内三个月月度评估总分的算术平均值（满分100） | 月度评估结果汇总 |
| **合规扣分** | 季度内SLA红线/黄牌事件累计扣分 | SLA记录+巡检台账 |
| **稳定性得分** | 季度内交付稳定性量化评分（+5/-10分区间） | 供管按检查表勾选 |
| **供管自评加分** | 战略协作、响应质量、高效执行、共同成长、坦诚透明（每项0或1分） | 供管主观评估 |

**第五条 评分公式**

> 最终得分 = 赛马基础分 - 合规扣分 + 稳定性得分 + 供管自评加分

最低分为0，不因扣分出现负分。最高分可达110分（赛马100 + 稳定性+5 + 自评+5），不设上限。由于九宫格按排名分档而非绝对分数线，高分不影响档位判定。

**合规扣分与月度评估的关系**：月度评估中的"合规表现"维度（20分）已对站点级合规事件进行了评分。季度评估中的合规扣分针对的是公司级重大事件（黄牌整改函、红线违规、严重客户投诉等），与月度站点合规不重复。具体判定：黄牌整改函以巡检正式发文为准，红线违规以SLA触发为准，投诉/一般违规以质检月报异常清单为准。

**多站点供应商聚合规则**：一家供应商公司在京东可能设有多个站点。月度评估按站点独立评分，季度评估按供应商公司聚合。聚合方式：
- 赛马基础分 = 该供应商旗下所有站点当季度月度评估总分的算术平均值（跨站点+跨月份）。例：某供应商2个站点，Q1各月分数为 85/78/90 和 80/82/88 → 赛马基础分 = (85+78+90+80+82+88)/6 = 83.8分
- 稳定性评定和供管自评以供应商公司为单位，不按站点拆分
- 合规扣分以公司为单位累计所有站点事件
- 清退以公司为单位执行，不因单站点问题清退整家公司，除非该站点是唯一站点或公司级管理失职

**单站点供应商计算示例**：某供应商仅1个站点，1月85分、2月78分、3月90分 → 赛马基础分 = (85+78+90)/3 = 84.3分。此分数直接来自月度评估，不需要重新计算。

**稳定性得分采用加减分制**（替代乘法系数），避免极端惩罚导致分数失真：

| 稳定性档位 | 得分调整 | 触发条件 |
|-----------|----------|----------|
| **稳定 A** | +5分 | 人员无大幅变动、产能无异常波动、SLA全达成 |
| **波动 B** | 0分（不加分不减分） | 任一：人员流失>10%、产能波动>20%、SLA一次未达成 |
| **失控 C** | -10分 | 任一：核心人员离职、产能断崖、SLA两次以上未达成、严重影响交付 |

**设计说明**：加减分制下，稳定性最多影响±10分，不会导致高分供应商因单一稳定性事件直接掉档。供应商可通过其他维度弥补稳定性短板，体现"允许犯错但持续扣分"的管理逻辑。

**第六条 合规扣分标准**

| 事件类型 | 扣分 | 说明 |
|---------|------|------|
| 黄牌整改 | -10分/次 | 巡检发现问题并下发整改函 |
| 投诉/一般违规 | -5分/次 | 质检通报、客户投诉 |
| 红线违规 | 直接评级C | 安全事件、数据泄露、严重合规违规 |

**第七条 稳定性评定标准**

月底评估采用四项量化检查表，逐项打分，避免"满足任一即降级"的主观判断：

| 检查项 | A档（+1.25分） | B档（0分） | C档（-2.5分） | 数据来源 |
|--------|---------------|-----------|--------------|----------|
| 人员变动 | 无大幅变动（流失≤10%，无核心人员变动） | 流失>10%或单月新增>15% | 核心人员离职（负责人/主管级） | 供应商自报+供管核实 |
| 产能波动 | 月度产能波动<10% | 月度产能波动10%-20% | 月度产能波动>20%或断崖式下降 | 业务系统数据 |
| SLA达成 | 季度SLA全部达成 | 季度SLA 1次未达成 | 季度SLA 2次以上未达成 | 供管日常记录 |
| 突发情况 | 无突发情况 | 轻微影响交付（可24小时内恢复） | 严重影响交付（>24小时无法恢复） | 事件报告+供管评估 |

**评定规则**：四项得分相加即为稳定性得分。四项均A=+5分，四项均B=0分，四项均C=-10分。混合档位按各档项数乘以对应分值相加。供管只需逐项勾选档位，系统自动计算。

**计算示例**：某供应商人员变动A(+1.25)、产能波动B(0)、SLA达成A(+1.25)、突发情况B(0) → 稳定性得分 = 1.25 + 0 + 1.25 + 0 = +2.5分。

**第八条 供管自评加分标准**

五项各评"高/中/低"，高=1分，中低=0分。每项必须填写具体事实依据，无事实依据的不给"高"：

| 维度 | 高=1分的触发条件 |
|------|-----------------|
| **战略协作** | 主动提出建设性建议、参与规划讨论、与业务方向一致 |
| **响应质量** | 24小时内响应、反馈有实质内容 |
| **高效执行** | 临时任务按时按质完成，无返工 |
| **共同成长** | 会议积极贡献、分享经验、对整体产线有额外贡献 |
| **坦诚透明** | 主动暴露风险和问题、不隐瞒客诉和人员变动、出问题第一时间自报而非被动发现 |

供管自评应客观公正，连续两个季度全部高分的需进行复核。"坦诚透明"维度为加分项而非必选项——不隐瞒问题的供应商应获得认可，而非只有"做得好"的供应商才能得分。

## 第三章 分级与九宫格

**第九条 九宫格映射**

按最终得分将所有供应商分为三档（X轴），按环比趋势分为三档（Y轴）：

| X轴档位 | 定义 | 占比 |
|---------|------|------|
| 高贡献（前30%） | 得分排名在前30% | 四舍五入取整 |
| 中贡献（中40%） | 得分排名在30%-70% | 四舍五入取整 |
| 低贡献（后30%） | 得分排名在后30% | 四舍五入取整 |

占比计算采用四舍五入。供应商数量变动时（如清退后不足30家），按实际数量 × 比例四舍五入取整，不足1家按1家计。

**示例**：当前30家供应商 → 高贡献9家(30×0.3)、中贡献12家(30×0.4)、低贡献9家(30×0.3)。若清退后剩28家 → 高贡献8家(28×0.3=8.4→8)、中贡献11家(28×0.4=11.2→11)、低贡献8家(28×0.3=8.4→8)，合计27家，余1家。余数分配规则：优先分配给小数部分最大的档位（本例中高中低小数部分均为0.4，则优先给中贡献档）→ 最终：高8、中12、低8，合计28家。

| Y轴趋势 | 定义 |
|---------|------|
| 趋势上升↑ | 环比得分增长≥5% |
| 趋势持平→ | 环比变化在-5%~+5%之间 |
| 趋势下降↓ | 环比得分下降≥5% |

**第十条 ABC评级映射**

九宫格9个格子对应ABC三档：

| | 趋势下降↓ | 趋势持平→ | 趋势上升↑ |
|---|---|---|---|
| **高贡献** | A（需稳住） | A（优秀） | A（优秀） |
| **中贡献** | B（需关注） | B（合格） | B（合格） |
| **低贡献** | C（淘汰） | C（不合格） | C（不合格） |

X轴决定档位，Y轴决定后续管理动作。

**新供应商保护期豁免**：入项不足两个季度的供应商，不参与九宫格映射，单独列为"观察区"。观察期内只记录评分和趋势，不触发清退。保护期结束后自动纳入九宫格评估。

| 档位 + 趋势 | 管理动作 |
|------------|---------|
| A + 趋势下降 | 启动保留动作：一对一沟通，识别风险点，必要时调整资源倾斜 |
| A + 趋势持平 | 维持现有合作深度，纳入保留清单 |
| A + 趋势上升 | 资源优先倾斜，纳入深度绑定计划 |
| B + 趋势下降 | 明确提升方向，月度跟踪改进进度 |
| B + 趋势持平 | 维持现状，纳入培养清单 |
| B + 趋势上升 | 资源适度倾斜，纳入重点培养 |
| C + 趋势下降 | 触发快车道清退（见第十一条第2款） |
| C + 趋势持平/上升 | 启动整改流程，限期30天 |

## 第四章 清退机制

**第十一条 清退触发条件**

| 触发条件 | 处理方式 |
|---------|---------|
| 连续两个季度评级为C | 自动启动解约流程 |
| 当季评级为C且趋势下降（九宫格"淘汰"格） | 当季直接启动清退，无需等待下一季度 |
| 当季触发红线违规 | 直接评级C，并启动清退评估 |

**与月度评估的关系**：月度评估中连续3个月E级或连续2个月D级升级为E级的站点，其所属供应商在季度评估中自动评为C档，不再等待季度末评估。即月度E级触发季度C级的快速通道。

**第十二条 清退流程**

清退分三种路径，按触发条件选择对应流程：

**路径一：常规清退（首次C，趋势持平/上升）**

| 步骤 | 动作 | 时限 |
|------|------|------|
| 1. 整改期 | 向供应商发出整改通知，明确整改项和要求 | 评级公布后10天内 |
| 2. 观察期 | 持续跟踪下一个季度表现，月度跟进整改进度 | 整改期30天 |
| 3. 复评 | 下一季度评估，如仍为C则触发解约 | 下一季度末 |
| 4. 解约 | 连续两次C后，正式启动解约流程 | 复评确认后30天内完成交接 |

**路径二：快车道清退（当季C且趋势下降，或月度连续E触发快速通道）**

| 步骤 | 动作 | 时限 |
|------|------|------|
| 1. 通知 | 向供应商发出清退通知 | 评级确认后3个工作日内 |
| 2. 交接 | 启动产能、数据、客户信息交接 | 通知后30天内完成 |

**路径三：红线清退（触发红线违规）**

| 步骤 | 动作 | 时限 |
|------|------|------|
| 1. 评估 | 启动清退评估，联合质检/法务/业务方 | 红线触发后5个工作日内 |
| 2. 决策 | 管理层确认是否立即清退或限期整改 | 评估完成后3个工作日内 |
| 3. 执行 | 确认后按路径二流程执行交接 | 决策后30天内完成 |

**产能分配**：被清退供应商的产能份额按赛马排名分配给上一档供应商。若分配结果导致A级合计产能占比超过60%上限（第十六条），则优先保障核心产线的分配，超出部分顺延至下一档供应商，确保集中度管控不受突破。

**第十三条 新供应商入项**

- 清退前1个月从储备库中确定1-2家替代供应商
- 新供应商入项后有两个季度保护期，保护期内不参与九宫格映射和清退评估，仅记录评分趋势
- 保护期结束后按正常评估体系参与季度评级和九宫格映射

## 第五章 半年度深度盘点

**第十四条 盘点内容**

半年度（6月、12月）在季度评估基础上追加三个维度：

| 维度 | 说明 | 评估方式 |
|------|------|---------|
| **战略意义** | 是否在关键产线/关键项目有不可替代性 | 管理层评估 |
| **长期潜力** | 规模扩张意愿、技术升级能力、团队稳定性 | 访谈+观察 |
| **风险敞口** | 过度依赖单一大客户、财务状况、舆情 | 公开信息+调研 |

**第十五条 半年度结果应用**

| 季度评级 | 半年度校准 | 最终决策 |
|---------|-----------|---------|
| A | 良好 | 保留，深度绑定 |
| A | 差 | 标记"高贡献但脆弱"，准备替代方案 |
| B | 良好 | 培养，给资源倾斜 |
| B | 差 | 观察，明确提升方向 |
| C | 良好 | 标记"低贡献但值得救"，延长整改期 |
| C | 差 | 执行清退 |

## 第六章 管控与监督

**第十六条 集中度管控**

- 头部（A级）供应商合计产能占比不超过60%
- 单一供应商产能占比不超过总产能30%
- 各产线具体限额按《全生命周期管理指引》执行（如金条人工≤360人/≤30%）。本制度的百分比为全局上限，产线限额为具体执行标准。全局上限与产线限额同时生效，任一突破即触发管控。

**第十七条 申诉机制**

- 供应商对评级结果有异议的，可在结果公布后5个工作日内提出申诉
- 申诉由服务组组织联合复核，10个工作日内给出答复
- 申诉期间不改变评级结果

**第十八条 数据保密**

- 本制度评估结果仅作为内部管理工具
- 不对外公布具体排名，只告知供应商"提升方向"和"整改项"
- 供应商间不得互相比较评级结果

## 第七章 附则

**第十九条** 本制度由服务组负责解释和修订。

**第二十条** 本制度自2026年6月1日起生效。原《供应商ABC分层分级管理方案》（2026-04-13版）中关于供应商分级的条款以本制度为准，其余条款（如具体评分维度定义）继续有效，直至后续统一修订。

> **计算示例**：人员变动A(+1.25)、产能波动B(0)、SLA达成A(+1.25)、突发情况C(-2.5) → 稳定性得分 = 1.25 + 0 + 1.25 + (-2.5) = 0分。供管只需勾选，无需手动计算。
