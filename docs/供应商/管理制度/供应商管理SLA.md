## 第一章 总则

### 1.1 目的

明确供应商在承接京东2B电销业务中的服务水平承诺，以量化指标定义"好"与"不好"，为日常管理和月度评估提供客观依据。

### 1.2 适用范围

本SLA适用于所有承接京东2B电销业务的BPO供应商，按站点单独考核。

### 1.3 与评估体系的关系

本SLA定义的指标是《供应商月度综合评估方案》中"业务达成"和"合规表现"两个维度的核心数据来源。

### 1.4 SLA确认

供应商签约时须确认本SLA，确认后即视为承诺达到以下指标要求。

## 第二章 核心交付指标

### 2.1 日均出勤率

| 项目 | 说明 |
|------|------|
| 定义 | 月度内每日实际出勤人数 ÷ 应出勤人数 的平均值 |
| 计算方式 | Σ(每日实际出勤) ÷ Σ(每日应出勤) × 100% |
| 达标线 | **≥95%** |
| 预警线 | **<90%** |
| 红线 | **<85%** |

> **出勤率计算说明**：应出勤人数 = 在岗总人数 - 合法请假人数（病假、年假等按公司规定批准的假期）。旷工计入缺勤，计划内请假不计入缺勤。

### 2.2 产能达标率

| 项目 | 说明 |
|------|------|
| 定义 | 月度实际完成产能 ÷ 目标产能 × 100% |
| 计算方式 | 月度实际产能 ÷ 月度目标产能 × 100% |
| 达标线 | **≥90%** |
| 预警线 | **<80%** |
| 红线 | **<70%** |

### 2.3 数据报送及时率

| 项目 | 说明 |
|------|------|
| 定义 | 按时报送次数 ÷ 总应报送次数 × 100% |
| 达标线 | **≥98%** |
| 预警线 | **<95%** |
| 红线 | **<90%** |

### 2.4 数据准确率

| 项目 | 说明 |
|------|------|
| 定义 | 经核实的准确数据条数 ÷ 总报送数据条数 × 100% |
| 达标线 | **100%** |
| 预警线 | **<98%** |
| 红线 | **<95%** |

| 项目 | 说明 |
|------|------|

| 达标要求 | **100%** |
| 预警线 | **<95%** |
| 红线 | **<90%** |

| 项目 | 说明 |
|------|------|
| 定义 | 实际留存通话录音数 ÷ 应留存通话录音数 × 100% |
| 达标要求 | **100%** |
| 预警线 | **<98%** |
| 红线 | **<95%** |

|------|------|
| 定义 | 以质检结果中话术合规项的达标比例为准 |

## 第四章 服务响应指标

### 4.1 日常响应

| 场景 | 响应要求 |
|------|----------|
| 群消息 | **30分钟内回复** |
| 异常事项（一般） | 当日群内报备 |
| 异常事项（重要） | **2小时内** 书面+电话报备 |
| 异常事项（紧急） | **立即** 电话报备 |

### 4.2 管理响应

| 场景 | 响应要求 |
|------|----------|
| 整改通知响应 | **24小时内** 提交整改方案 |
| 月度总结提交 | 次月 **3日前** |
| 人员变动报备 | 变动前 **3个工作日** |
| 大面积人员变动（≥3人） | **即时** 电话+书面报备 |

### 4.3 会议响应

| 场景 | 响应要求 |
|------|----------|
| 周度复盘会 | 全勤，有事提前请假 |
| 月度评估会 | 站点负责人必须参加 |
| 专项沟通会 | 按通知要求参加 |

## 第五章 达标/预警/红线机制

### 5.0 SLA与评估方案的关系说明

> **重要澄清**：SLA与《供应商月度综合评估方案》是两套不同维度的体系，请勿将两套阈值直接对比。

| 维度 | SLA | 评估方案 |
|------|-----|---------|
| **定位** | "能不能接受"——定义供应商服务质量底线 | "得多少分"——量化评估与排名 |
| **触发** | 管理动作（预警、清退） | 得分与评级 |
| **逻辑** | 连续不达标才触发管理动作 | 按档打分，不考虑连续性 |

**示例**：产能达标率82%
- SLA视角：<90%即不达标，82%触发**预警**（"不达标，需要整改"）
- 评估方案视角：80%-89%得**8分**（"扣2分，但仍有一定产出"）

两套体系**独立运作**，SLA的预警不直接等同于评估方案的扣分，反之亦然。

### 5.1 状态定义

| 状态 | 条件 | 说明 |
|------|------|------|
| **达标** | 所有指标均在达标线以上 | 正常合作 |
| **预警** | 任一指标触及预警线 | 需要整改 |
| **红线** | 任一指标触及红线 | 启动清退评估 |

### 5.1.1 SLA阈值与评估方案得分档对照表

> **使用说明**：本表展示SLA定义的"底线"与评估方案"评分档"的对应关系，帮助执行者理解两套体系的差异。

| 指标 | SLA达标 | SLA预警 | SLA红线 | 评估方案得分档（满分） |
|------|---------|---------|---------|----------------------|
| **日均出勤率** | ≥95% | <90% | <85% | ≥95%=10分，90-94%=8分，85-89%=6分，80-84%=4分，<80%=0分 |
| **产能达标率** | ≥90% | <80% | <70% | ≥90%=10分，80-89%=8分，70-79%=6分，60-69%=4分，<60%=0分 |
| **数据报送及时率** | ≥98% | <95% | <90% | 100%=5分，≥95%=4分，90-94%=3分，<90%=0分 |
| **数据准确率** | 100% | <98% | <95% | 100%=5分，≥98%=4分，95-97%=3分，<95%=0分 |
| **人员持证率** | 100% | <95% | <90% | 100%=6分，≥95%=4分，90-94%=2分，<90%=0分（红线） |
| **录音留存率** | 100% | <98% | <95% | 100%=3分，≥98%=2分，<98%=0分 |
| **话术合规率** | ≥95% | <90% | <85% | 纳入质检月均分体系（见评估方案3.1.1） |

**理解要点**：
- SLA的"预警"触发管理动作（如下发整改通知），但不等同于评估方案扣分
- 评估方案的每一档得分是独立计算的，不因SLA预警而额外扣分
- 两套体系**同时生效**，SLA预警期间评估方案照常计分

### 5.2 预警响应流程

```
指标触及预警线
  │
  ↓
供应商管理岗下发预警通知（书面）
  │
  ↓
供应商 **3个工作日内** 提交整改计划
  │
  ↓
整改期 **30天**
  │
  │ 每周提交整改进展
  │
  ↓
整改期满复评
  │
  ├─ 达标 → 解除预警
  └─ 未达标 → 升级处理
```

### 5.3 红线响应流程

```
指标触及红线
  │
  ↓
供应商管理岗启动清退评估
  │
  ↓
业务对接人确认
  │
  ↓
正式清退流程（见《供应商管理办法》第五章）
```

### 5.4 各产线指标差异说明

不同产线的业务特性不同，部分指标可能需要按产线设定差异化标准。具体差异在SLA签署时由供应商管理岗与供应商协商确定，并写入SLA补充条款。

### 5.5 供应商主动终止合作

如供应商因自身原因（如经营困难、战略调整等）主动提出终止合作，按以下流程处理：

```
供应商提出书面终止申请
  │
  │ （提前30个自然日）
  ↓
供应商管理岗受理，评估影响范围
  │
  ↓
业务对接人确认业务交接方案
  │
  ↓
过渡期（30天）：业务交接、客户信息处理
  │
  ↓
正式终止合作
  │
  ↓
结算、归档
```

**与京东主动清退的区别**：

| 维度 | 京东主动清退 | 供应商主动退出 |
|------|-------------|---------------|
| 通知时限 | 京东自行决定 | 供应商须提前**30个自然日**书面通知 |
| 过渡期 | 30天 | 30天（可协商缩短） |
| 客户信息处理 | 必须完成 | 必须完成 |
| 违约责任 | 按合同条款执行 | 按合同条款执行 |

## 第六章 附则

- SLA指标每 **季度** review一次
- 指标调整须提前 **1个月** 书面通知供应商
- 当月调整不追溯

本SLA由京东2B电销供应商管理岗负责解释。
