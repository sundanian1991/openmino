# 供应商运营管理要求

> **文件编号**：SM-2026-006
> **版本**：V5.0
> **起草人**：供应商管理岗
> **日期**：2026-04-10
>
> **定位**：京东服务组对供应商全生命周期的运营管理要求。本规范明确京东对供应商的管理细则，供应商须在工作手册中逐项填报并执行。
>
> **适用范围**：本要求适用于所有承接业务的供应商，自承接之日起生效，至合作终止之日止。
>
> **执行依据**：供应商须按《供应商工作手册模板》建立专属工作手册，逐项填写并提交审核。未按本要求执行者，按第七章整改规则处理。

---

## 第一章 总则

### 第一条 目的

建立供应商全生命周期的过程管理机制，实现：

- 对供应商内部管理过程的**可见、可查、可比**
- 通过过程管理发现优秀做法，**跨供应商推广**
- 通过过程管理发现管理缺陷，作为**评估与清退决策依据**
- 推动供应商整体管理水平持续提升

### 第二条 管理原则

供应商须在承接业务后，建立覆盖本规范所列管理模块的内部制度，报送审核后执行。审核通过后，将以供应商自己提交的制度为依据进行过程检查和评估。

核心逻辑：**供应商自建制度 → 报送审核 → 确认后执行 → 以自身制度为标尺接受检查**。

---

## 第二章 全生命周期管理框架

供应商从承接业务到退出，经历四个阶段。

```
准入承接 → 日常运营 → 持续提升 → 退出清退
  建体系    抓落实      促改善      做交接
```

### 阶段总览

| 阶段 | 核心任务 | 交付物 |
|------|---------|--------|
| **一、准入承接** | 建制度、配人力、设站点 | 《内部管理规范》《花名册》《职场信息表》 |
| **二、日常运营** | 抓考勤、保质量、赛分量、清结算 | 每日出勤/业绩报送、赛马结果、结算单、合规自查 |
| **三、持续提升** | 月复盘、季对标、高管对谈、推广最佳实践 | 月度复盘、季度对标报告、高管对谈纪要 |
| **四、退出清退** | 做评估、保交接、清数据 | 退出交接表、数据删除确认函 |

## 第三章 阶段一：准入承接

### 第一节 管理制度报送

供应商须在承接业务后 **15 个工作日** 内，提交覆盖以下 **七个管理模块** 的内部管理规范。

| 编号 | 模块 | 须明确的制度内容 |
|------|------|-----------------|
| M1 | 人力配置管理 | 管理架构、质检与培训岗位配置、关键岗位备岗 |
| M2 | 培训管理 | 培训体系、达产周期、复训机制、辅导带教 |
| M3 | 排班与考勤 | 排班逻辑、出勤达标、异常缺勤跟进 |
| M4 | 质量管理 | 内部质检体系、质检→改善闭环、合规自查 |
| M5 | 绩效管理 | 考核体系、激励机制、改进机制、绩效面谈 |
| M6 | 现场管理 | 日常管理动作、会议制度、主管巡查 |
| M7 | 数据报送 | 报送责任人、数据核对、信息安全 |

#### 首次报送清单与参考格式

> 以下为参考框架，供应商按此结构提交即可，具体内容由供应商结合自身情况填写。

| 序号 | 材料 | 参考框架 |
|------|------|---------|
| 1 | 《供应商内部管理规范》 | 覆盖 M1-M7 七个模块，每模块包含：制度名称、适用范围、责任人、具体要求、执行频率、记录方式 |
| 2 | 《供应商管理对接人信息表》 | 供应商名称、对接人姓名/职务/联系方式、站点负责人/姓名/联系方式、各模块责任人 |
| 3 | 《管理制度汇总清单》 | 列明 7 个模块的文档名称、版本号、更新日期、责任人 |

**《供应商内部管理规范》参考结构**（每模块一份）：

> 1. 制度名称与适用范围
> 2. 管理目标
> 3. 组织架构与责任人
> 4. 具体管理要求（流程、标准、频次）
> 5. 记录与台账管理
> 6. 异常处理机制
> 7. 制度修订记录

#### 站点设立参考格式

供应商提交《职场变更申请表》时须包含：

> - 供应商名称、申请日期、变更类型（新增/扩容/缩减/清退/地址变更）
> - 当前站点信息（各站点地址、人力规模）
> - 变更后规划（新站点地址、变更后人力规划、生效日期）
> - 申请原因与数据依据
> - 附件：租赁合同、消防验收文件、人力考勤记录

#### 人员准入参考格式

供应商报送人员上岗信息时须包含：

> - 姓名、岗位、所属站点、上岗日期
> - 培训完成情况（入职培训、合规培训、技能考核结果）
> - 资质证书编号及有效期
> - 试用期起止日期

| 要求 | 说明 |

| 加强质检 | 试用期内质检抽检比例提高至常规标准的 2 倍 |

| 情形 | 报备时限 | 方式 |
|------|---------|------|

| 大面积离职（≥3人/月） | 第一人离职时立即报备 | 书面+电话 |

#### 权限管理

| 要求 | 说明 |
|------|------|
| 坐席权限由站点负责人统一管理 | 权限变更须同步报备供应商管理岗 |
| 人员离职或调岗后当日回收系统权限 | 防止权限遗漏 |

## 第四章 阶段二：日常运营

### 第一节 排班与考勤管理

#### 出勤报送

| 要求 | 说明 |
|------|------|
| 出勤达标 | 出勤率不低于 SLA 要求（≥95%） |
| 报送时间 | 每日 10:00 前 |
| 分类报送 | 须区分正常出勤、病假、事假、旷工 |
| 异常缺勤 | 须在备注栏说明原因 |
| 数据一致性 | 须与考勤系统一致，不得瞒报虚报 |

#### 排班管理

| 要求 | 说明 |
|------|------|
| 排班报备 | 每日下班前报送排班计划 |
| 排班逻辑 | 须说明排班依据（业务量预测、人力匹配） |
| 人力利用率 | 说明"在岗人数"和"实际上线人数"的差异及原因 |
| 排班变更 | 重大调整须提前说明原因 |

### 第二节 人员日常管理

#### 在岗管理

| 要求 | 说明 |
|------|------|
| 合规复训 | 每季度 1 次 |
| 业务技能复训 | 每月 1 次 |
| 话术更新培训 | 按需开展 |
| 终端设备 | 安装指定安全软件，禁止个人设备，每季度安全检查 1 次 |

#### 离职管理

| 要求 | 说明 |
|------|------|
| 离职交接 | 客户信息交接 → 权限回收 → 设备归还 → 花名册更新 → 三方确认 |

#### 异常人员管理

| 触发情形 | 处理方式 |
|---------|---------|
| 连续 2 个月质检评分低于站点平均分 20% | 出具改进计划，备案 |
| 单月投诉 ≥3 次 | 停岗检查，提交情况说明 |
| 资质证书过期 | 立即停岗，补齐后复岗 |
| 严重合规违规 | 立即停岗，启动调查 |

### 第三节 质量与合规管理

#### 内部质检

| 要求 | 说明 |
|------|------|
| 质检体系 | 明确质检覆盖率、抽检比例、质检标准、质检团队配置 |
| 质检→改善闭环 | 质检发现的问题须反馈到业务改善（培训调整、话术优化、人员调整） |
| 月度专节 | 月度总结中须增加"质检→改善"专节，说明本月发现、改善动作、效果验证 |

#### 合规自查

| 要求 | 说明 |
|------|------|
| 自查频率 | 每月至少一次 |
| 报告提交 | 每月 25 日前提交合规自查报告 |
| 自查内容 | 涵盖信息安全、话术规范、操作合规 |

#### 异常处理

| 级别 | 响应要求 |
|------|---------|
| 一般异常 | 当日处理，群内报备 |
| 重要异常 | 2 小时内报备（书面+电话） |
| 紧急异常 | 立即电话报备，书面补报 |

### 第四节 赛马与分量调度

#### 赛马机制

| 项目 | 说明 |
|------|------|
| 考核频率 | 月度评估、季度排名 |
| 排名公布 | 定期向所有供应商公布排名结果 |
| 结果应用 | Top 供应商增份额、优先获取新业务；末位供应商减份额，连续末位启动淘汰程序 |

#### 常规分量规则

| 项目 | 说明 |
|------|------|
| 分量依据 | 赛马排名结果 |
| 分量测算 | 根据目标产能与当前产能差值计算所需人数调整量 |
| 切换规则 | 按职场规模分批切换，确保业务平稳过渡 |
| 审批流程 | 供应商管理岗拟定方案 → C3 审批 → C2 终审（≥50% 调整幅度须 C2 审批） |

#### 突发增量需求分量规则

当出现突增需求时，分量不单纯依据赛马排名，而是综合以下因素决策：

| 考量因素 | 说明 |
|---------|------|
| 业绩趋势 | 近阶段业绩的走势，是上升还是下滑 |
| 业绩排名 | 在同类供应商中的相对排名 |
| 相关经验 | 是否做过类似业务线 |
| 上人能力 | 能否在规定时间内补充足够人力 |
| 意愿度 | 供应商是否愿意承接增量 |
| 配合度 | 历史配合情况，响应速度和执行效果 |

> **以上分量规则透明公开**，供应商可通过排名和数据自行判断分量趋势。分量调整前会与供应商提前沟通，确保双方认知一致。

#### 集中度管控

| 原则 | 说明 |
|------|------|
| 新增人力增幅上限 | 一般不超过 30% 或 50% |
| 防一家独大 | 任何单一供应商份额不得超过集中度上限 |
| 梯队健康 | 优先保障中腰部供应商成长空间 |

### 第五节 结算与争议处理

#### 结算周期

| 项目 | 说明 |
|------|------|
| 标准结算周期 | T+2 月 |
| 合同到期结算 | 合同到期前 60 天启动结算清算 |
| 清退结算 | 最后合作业务月的结算款压 3 个月再结 |

#### 结算流程

| 步骤 | 动作 | 责任方 |
|------|------|--------|
| 1 | 供应商管理岗发起结算审批 | 供应商管理岗 |
| 2 | C3 审批（≤2 个工作日） | 审批人 |
| 3 | C2 终审（≤3 个工作日） | 审批人 |
| 4 | 结算单下发供应商确认 | 供应商管理岗 |
| 5 | 供应商确认后推进财务付款 | 结算岗 |

#### 争议与申诉

| 项目 | 说明 |
|------|------|
| 申诉条件 | 对评估得分有异议，可在评估结果公布后 **3 个工作日** 内发起 |
| 申诉方式 | 书面申诉（说明异议项 + 理由） |
| 受理时效 | 供应商管理岗 1 个工作日内受理 |
| 复核时效 | 指定独立复核人 2 个工作日内完成复核 |
| 最终结果 | 3 个工作日内反馈，复核结果为最终结果，不再二次申诉 |

#### 复核人规则

| 规则 | 说明 |
|------|------|
| 独立性 | 复核人须与被评估站点无直接利益关联 |
| 范围 | 复核内容限于申诉所列异议项 |
| 监控 | 某站点连续 3 个月申诉率 >50%，供应商管理岗介入了解原因 |

#### 结算重大问题处理

| 情形 | 处理流程 |
|------|---------|
| 结算金额存在重大差异 | 供应商书面提出异议 → 供应商管理岗会同结算岗复核 → 差异说明 → 重新出具结算单 |
| 结算数据缺失或错漏 | 供应商提供补充材料 → 结算岗核实 → 补充结算或调账 |
| 结算争议无法协商解决 | 供应商管理岗发起 → C3 协调 → C2 裁定 |

### 第六节 数据报送

#### 日常报送

| 报送项 | 频率/时限 | 模板 | 说明 |
|--------|----------|------|------|
| 花名册 | 实时更新 | SM-TPL-005 | 人员变动 3 个工作日内更新 |
| 每日出勤 | 每日 10:00 前 | SM-TPL-001 | 当日实际出勤、请假、异常 |
| 每日业绩 | 每日下线前 | SM-TPL-002 | 核心业务数据 |
| 排班报送 | 每日下班前 | — | 当日/次日排班计划 |
| 周报 | 每周五下班前 | SM-TPL-003 | 出勤汇总、产能、异常、计划 |
| 月度总结 | 次月 3 日前 | SM-TPL-004 | 月度数据、人员、合规、计划 |
| 合规自查报告 | 每月 25 日前 | 新增模板 | 自查结果、问题、整改 |
| 培训记录 | 每月随月度总结提交 | 新增模板 | 本月培训完成情况 |

#### 日常报送参考格式

> 以下为各类报送材料的参考结构，供应商按此格式提交即可。

**每日出勤报送**（SM-TPL-001）参考结构：

> - 报送日期、供应商名称、站点名称
> - 应出勤人数、实际出勤人数、出勤率
> - 分类明细：正常出勤 / 病假 / 事假 / 旷工（各列明人数及姓名）
> - 异常缺勤说明（如有）

**每日业绩报送**（SM-TPL-002）参考结构：

> - 报送日期、供应商名称、站点名称
> - 核心业务指标（通次、通时、接通率、转化率等，按业务线要求）
> - 与目标值对比

**周报**（SM-TPL-003）参考结构：

> - 周报期间、供应商名称
> - 出勤汇总（本周各日出勤情况）
> - 产能汇总（本周核心指标完成情况）
> - 异常情况与处理结果
> - 下周工作计划

**月度总结**（SM-TPL-004）参考结构：

> 1. 月度经营数据汇总（出勤、产能、转化率等核心指标）
> 2. 人员变动情况（入职、离职、在岗人数）
> 3. 质检→改善专节（本月质检发现、改善动作、效果验证）
> 4. 合规自查情况（自查结果、问题发现、整改措施）
> 5. 培训完成情况（本月培训主题、参与人数、考核结果）
> 6. 存在问题与下月改善计划

**合规自查报告**参考结构：

> 1. 自查期间、自查人
> 2. 自查范围（信息安全、话术规范、操作合规）
> 3. 自查结果（达标项 / 问题项）
> 4. 问题描述与整改措施（含整改责任人和时限）
> 5. 签字确认

**培训记录**参考结构：

> - 培训日期、培训主题、培训讲师、参与人员名单
> - 培训内容概要
> - 考核结果（成绩/通过率）
> - 签到表扫描件

## 第五章 阶段三：持续提升

### 第一节 月度经营沟通与复盘

| 项目 | 说明 |
|------|------|
| 参与方 | 供应商管理岗、业务对接人、供应商站点负责人及运营团队 |
| 频率 | 每月一次，覆盖全部合作供应商 |
| 复盘内容 | 本月过程检查结果、SLA 指标达成、问题整改进度、下月改善重点 |
| 赛马结果通报 | 公布月度赛马排名及分量调整方向 |
| 结算情况通报 | 本月结算进度、争议处理情况 |
| 产出 | 月度经营沟通纪要 |

---

### 第二节 月度复盘报告参考格式

供应商参与月度经营沟通与复盘时，建议准备以下材料框架：

> 1. 本月经营数据（出勤、产能、转化率、客诉等核心指标达成）
> 2. 过程管理自查结果（七模块执行情况）
> 3. 上月问题整改完成情况
> 4. 本月最佳实践分享（如有）
> 5. 下月改善计划与资源需求

---

### 第三节 季度对标与高管对谈

| 项目 | 说明 |
|------|------|
| 参与方 | 供应商双方高管（供应商老板/商务负责人 + 京东服务组管理层） |
| 频率 | 每季度一次，覆盖全部合作供应商 |
| 对标内容 | 七模块横向对比、最佳实践提炼、共性问题分析 |
| 高管对谈内容 | 季度经营总结、战略方向对齐、重大议题讨论、下季度规划 |
| 产出 | 季度对标报告 + 高管对谈纪要 |

---

### 第四节 最佳实践推广

| 步骤 | 动作 | 说明 |
|------|------|------|
| 1. 发现 | 检查中发现某供应商在某模块做得好 | 如：培训体系有效缩短达产周期 |
| 2. 提炼 | 把具体做法提炼为可复制的经验 | 拆解为：步骤、条件、效果 |
| 3. 验证 | 确认做法确实有效 | 用数据验证 |
| 4. 推广 | 通过分享会或书面指引同步给其他供应商 | "可以参考"，非"必须照做" |
| 5. 跟踪 | 跟踪采纳情况和效果 | 形成闭环 |

## 第六章 阶段四：退出清退

### 第一节 站点缩减与清退

#### 缩减触发条件

| 业务线 | 缩点触发线 | 限留站点线 |
|--------|-----------|-----------|
| 金条人工 | 连续 2 月 <60 席 | <40 席限留 1 个站点 |
| 金条人机 | 连续 2 月 <40 席 | <30 席限留 1 个站点 |
| 菁卡拉新 | 连续 2 月 <30 席 | <20 席限留 1 个站点 |
| 企金/固收 | 连续 2 月 <30 席 | <20 席限留 1 个站点 |

触及缩减线后给予 **1 个月整改缓冲期**。

#### 站点清退触发条件

| 情形 | 说明 |
|------|------|
| 供应商主动申请 | 供应商主动提出站点关停 |
| 强制清退 | 缓冲期结束仍未达标 |
| 重大风险 | 站点发生重大合规、安全或消防问题 |

#### 强制清退流程

| 步骤 | 动作 |
|------|------|
| 1 | 缓冲期结束次日确认未达标 |
| 2 | 起草《站点清退通知书》 |
| 3 | 发起审批：供应商管理岗 → C3 审批 → 抄送 C2、业务运营岗 |
| 4 | 发出正式通知，要求在规定日期内完成关停 |
| 5 | 监督站点关停过程 |

---

### 第二节 供应商退出与清退

#### 单业务线退出条件（满足任一）

| 情形 | 说明 |
|------|------|
| 连续考核周期综合绩效未达基准 | 按《供应商月度综合评估方案》判定 |
| 触发客诉违规红线 | 重大销售误导、和客户对骂等 |
| 业务调整终止 | 业务方向调整 |
| 合约到期未续 | 合同到期不再续签 |
| 合规整改失效 | 整改后仍未达标 |
| 供应商主动退出 | 供应商主动提出终止合作 |
| 连续赛马末位 | 经分量调整仍未改善 |

#### 全业务线强制退出条件（满足任一）

| 情形 | 说明 |
|------|------|
| 重大服务声誉事故 | A/B/C 类重大投诉、公开扩散、品牌损害 |
| 信息泄露事故 | 系统故障 ≥2 小时、泄露保密信息 |
| 违规转分包 | 擅自转包、关联公司变相转移 |
| 数据造假 | 报送数据、质检记录、培训记录造假 |

#### 退出流程

| 步骤 | 动作 | 时限 |
|------|------|------|
| 1 | 供应商提交退出申请 / 供应商管理岗发起清退申请 | — |
| 2 | 评估影响范围，起草退出交接方案 | — |
| 3 | 审批：供应商管理岗 → C3 审批（≤1天） → C2 终审（≤2天） | 3 个工作日 |
| 4 | 发出退出通知，进入过渡期 | — |
| 5 | 过渡期内完成业务交接、人员安置、数据清理 | 30 天 |
| 6 | 过渡期满，正式终止合作 | — |

### 第三节 退出交接

#### 客户信息处理

| 步骤 | 动作 |
|------|------|
| 1. 数据导出 | 供应商将客户数据以指定格式导出 |
| 2. 交接确认 | 客户数据交接至指定接替供应商或业务系统 |
| 3. 数据删除 | 供应商删除所有持有副本，提交删除确认函 |
| 4. 保密延续 | 签署《保密延续协议》，合作终止后保密义务延续 2 年 |

#### 退出交接参考格式

《退出交接表》应包含以下核心内容：

> **准备阶段**：停呼时间确认、与供应商确定最后停呼日期和停发数据日期
>
> **执行阶段**：
> - 数据盘点：确认所有数据归属
> - 数据回收：从京音系统回收数据
> - 企业微信转移：确认接收账号并发起转移
>
> **收尾阶段**：
> - 账号关闭：确认客户已全部转移后禁用京音/企微/质检账号
> - 结算处理：核对未结款项、扣除违约金（如有）、走财务审批流程
> - 客诉处理：设立 1 个月专项跟踪期，防止客诉无人处理

#### 退出后管理

| 要求 | 说明 |
|------|------|
| 进入备选池 | 退出后自动进入备选供应商池 |
| 禁入期 | 退出之日起 3 个月内不得重新参与该业务线合作 |
| 重新准入 | 3 个月后可重新参与准入评估（须满足现行标准） |

## 第七章 整改与不合格处理

### 第一节 工作手册审核

| 项目 | 说明 |
|------|------|
| 审核方 | 供应商管理岗 |
| 审核范围 | 工作手册各阶段子模块填报完整性、制度内容合规性、数据真实性 |
| 审核结果 | 通过 / 限期整改 / 不通过 |

### 第二节 整改规则

#### 整改触发条件

| 情形 | 说明 |
|------|------|
| 未按期提交工作手册 | 准入承接阶段 15 个工作日内未提交 |
| 填报内容缺失 | 任一子模块未填写或填写不完整 |
| 制度内容不合规 | 报送的内部管理规范不符合本运营管理要求 |
| 数据不真实 | 报送数据与实际不符，或前后矛盾 |
| 日常运营未按规范执行 | 数据报送、排班考勤、质量合规等未按期执行 |

#### 整改流程

| 步骤 | 动作 | 时限 |
|------|------|------|
| 1 | 供应商管理岗出具《整改通知书》，列明整改项 | 发现问题后 2 个工作日 |
| 2 | 供应商收到通知后提交整改计划 | 收到通知后 3 个工作日 |
| 3 | 供应商完成整改并提交复核 | 整改期限 5-10 个工作日（视问题严重程度） |
| 4 | 供应商管理岗复核 | 收到整改材料后 3 个工作日 |
| 5 | 复核通过 → 恢复正常运营；复核不通过 → 启动合作意愿确认 | — |

#### 合作意愿确认

整改期满仍不合格，供应商管理岗将发起合作意愿确认：

| 步骤 | 动作 |
|------|------|
| 1 | 供应商管理岗约谈供应商商务负责人，确认是否继续合作 |
| 2 | 确认继续合作 → 重新设定整改期限；确认不再合作 → 启动退出流程 |
| 3 | 拒不配合或无回应 → 视为放弃合作，按强制退出处理 |

### 第三节 日常管理整改

日常运营期间未按本要求执行的，按以下规则处理：

| 违规情形 | 处理方式 |
|---------|---------|
| 数据报送逾期 ≤3 天 | 群内提醒，记入月度考核 |
| 数据报送逾期 >3 天 | 出具整改意见，限期补报 |
| 连续 2 个月未执行合规自查 | 限期整改，纳入月度评估扣分 |
| 月度复盘缺席 ≥2 次 | 约谈站点负责人 |
| 季度对标缺席 | 约谈供应商高管 |

## 第八章 附则

### 第一条 解释权

本规范由供应商管理岗负责解释。

### 第二条 生效与修订

本规范自发布之日起生效，修订须提前通知所有供应商。

### 第三条 工作手册

供应商须在准入承接阶段，按《供应商工作手册模板》建立专属工作手册文件夹，包含：
- `01-供应商运营管理要求.md`（本文档，京东侧管理要求）
- `00-准入承接/`（站点报备、人员准入、制度报送）
- `01-日常运营/`（分量确认、排班考勤、质量合规、结算争议、数据报送）
- `02-持续提升/`（月度复盘、季度对标）
- `03-退出清退/`（交接清单）

各子文件夹须按对应模板要求填写完整，提交供应商管理岗审核确认。

### 第四条 与现有制度的关系

| 文件 | 定位 | 与本规范的关系 |
|------|------|---------------|
| 《供应商管理办法》 | 总纲 | 本规范是配套文件 |
| 《供应商管理日常手册》 | 操作层 | 本规范引用日常手册的报送规范 |
| 《供应商管理 SLA》 | 量化层 | 本规范补充 SLA 的过程管理要求 |
| 《供应商月度综合评估方案》 | 评估层 | 本规范的检查结果作为评估的"管理配合"维度输入 |
