## 第一章 总则

### 1.1 目的

为规范京东2B电销BPO供应商的全生命周期管理，保障业务连续性与交付质量，建立公平、透明、可量化的供应商管理体系，特制定本办法。

### 1.2 适用范围

本办法适用于所有承接京东2B电销业务（含金条、信用卡、财富等产线）的BPO供应商。

### 1.3 管理依据

- 公司供应商管理相关制度
- 公司信息安全管理制度
- 公司质量管理相关规定
- 本办法及相关操作手册、SLA协议

### 1.4 管理原则

| 原则 | 说明 |
|------|------|
| 规则先行 | 所有管理动作有据可依，不因人而异 |
| 数据驱动 | 以量化数据为评估和决策依据 |
| 公平透明 | 规则公开、标准统一、结果公示 |
| 闭环管理 | 每项要求有发起、执行、反馈、归档 |

### 1.5 文件体系

本办法为供应商管理的顶层文件，以下文件为本办法的操作细则：

| 文件 | 定位 |
|------|------|
| 供应商管理办法（本文件） | 总纲——制度与规则 |
| 供应商管理日常手册 | 操作——每日动作规范 |
| 供应商管理SLA | 量化——服务水平承诺 |
| 供应商月度综合评估方案 | 评估——统一评分标准 |
| 质量管理规范 | 质量——质检要求与考核 |

## 第二章 供应商准入

### 2.1 准入条件

#### 2.1.1 基本资质

| 条件 | 要求 |
|------|------|
| 企业资质 | 有效营业执照，经营范围包含呼叫中心或相关业务 |
| 财务状况 | 近一年无重大经营异常 |
| 行业经验 | 具有电销或客服外包运营经验 |
| 团队规模 | 具备承接业务所需的人员储备能力 |
| 场地设备 | 具备合规的办公场地和终端设备 |

#### 2.1.2 业务能力

| 条件 | 要求 |
|------|------|
| 管理团队 | 有专职项目管理团队，能对接京东管理要求 |
| 培训体系 | 具备人员培训和考核能力 |
| 信息系统 | 具备接入京东业务系统的技术条件 |
| 合规能力 | 理解并承诺遵守信息安全和业务合规要求 |

### 2.2 准入流程

```
供应商提交申请
  │
  │ （5个工作日）
  ↓
资质初审（供应商管理岗）
  │
  │ 通过 ↓
  │
业务能力评审（业务方+供应商管理联合评审）
  │
  │ （10个工作日） ↓
  │
试运营期（60天）
  │
  │ 试运营期达标 ↓
  │
正式签约合作
```

### 2.3 试运营期

- 试运营期为 **90天**（3个月，可根据业务需要调整）
- 试运营期内适用全套管理规范和SLA要求
- 试运营期评估标准：月度综合评估达到 **B级及以上**
- 试运营期不达标：终止合作
- **注："试用期"与"试运营期"的区分：试运营期适用于新供应商准入阶段；试用期适用于新员工入职管理，按日常手册规定执行。**

### 2.4 决策机制

| 决策事项 | 决策人 | 决策依据 |
|----------|--------|----------|
| 准入评审通过 | 供应商管理岗 + 业务对接人联合审批 | 资质审查结果+业务能力评审 |
| 试运营通过 | 供应商管理岗 | 月度综合评估结果 |
| 终止准入 | 供应商管理岗 | 资质/能力不达标 |

## 第三章 日常管理

### 3.1 对接人机制

**角色对照表**：

| 角色 | 所属方 | 职责 |
|------|--------|------|
| 站点负责人 | 供应商 | 站点最高管理者，统筹全部业务 |
| 站点对接人 | 供应商 | 站点指定的唯一日常对口人，对接京东侧全部接口 |
| 业务对接人（业管） | 京东 | 负责业务交付管理：产能目标、业务调度、日常交付监督 |
| 供应商管理岗 | 京东 | 负责制度管理：准入评审、评估考核、合规监督 |

- 每个站点须指定 **一名站点对接人**，作为站点与京东侧的唯一日常接口
- 站点对接人同时对接京东侧业务对接人（业管）和供应商管理岗
- 对接人变更须提前 **5个工作日** 书面通知

### 3.2 信息报送

- 所有报送按《供应商管理日常手册》执行
- 报送时间节点、格式、责任人均有明确规定
- 超时未报或格式不符按评估方案扣分

### 3.3 合规红线

以下为不可触碰的合规底线：

| 红线 | 说明 | 后果 |
|------|------|------|
| 信息安全 | 客户信息泄露、违规使用、外传 | 立即停岗+清退评估 |
| 人员资质 | 无证上岗、资质造假 | 立即停岗+清退评估 |
| 数据造假 | 虚报出勤、虚报产能、篡改数据 | 立即停岗+清退评估 |
| 录音篡改 | 删除、修改通话录音 | 立即停岗+清退评估 |

### 3.4 站点结构

- 每家供应商公司最多设立 **4个站点**
- **站点为最小考核单元**：独立出勤、独立产能、独立评估排名
- 同一供应商下的不同站点独立考核，互不影响
- 一个站点可承接多条产品线，日报按产品线分列，但评估按站点总量考核
- 新增站点须经供应商管理岗审批

### 3.5 集中度管控

- 各产线供应商份额须符合公司集中度管控要求
- 具体标准按公司相关规定执行
- 供应商管理岗负责监控和预警

## 第四章 评估与考核

### 4.1 评估体系

- 采用 **月度综合评估** 机制，100分制
- 四维度：交付质量（40分）、业务达成（30分）、合规表现（20分）、管理配合（10分）
- 详细规则见《供应商月度综合评估方案》

### 4.2 评估结果应用

| 评级 | 分数 | 应用 |
|------|------|------|
| A（优秀） | ≥90 | 续约优先权 |
| B（良好） | 80-89 | 正常合作 |
| C（达标） | 70-79 | 关注+约谈 |
| D（预警） | 60-69 | 整改30天 |
| E（红线） | <60 | 清退评估 |

### 4.3 评估申诉

- 对评估结果有异议的，可在结果公布后 **3个工作日内** 提起申诉
- 申诉由供应商管理岗受理，指定独立复核人复核
- 详细流程见《供应商月度综合评估方案》第六章

## 第五章 预警与退出

### 5.1 预警机制

#### 5.1.1 预警触发条件

| 条件 | 说明 |
|------|------|
| 月度评估D级 | 60-69分 |
| 连续2个月C级 | 70-79分连续2个月 |
| 质检月均分 70-79分 | 交付质量明显下降，预警区间 |
| **人员离职率超标** | 离职≥3人 **或** ≥10%（两者满足其一即触发） |

#### 5.1.2 红线触发条件

| 条件 | 说明 |
|------|------|
| 月度评估E级 | <60分 |
| 质检月均分<70分 | 交付质量严重不达标 |
| 信息安全严重违规 | 客户信息泄露、违规使用、外传 |
| 人员持证率<90% | 大量无证上岗 |
| 数据造假 | 虚报出勤、虚报产能、篡改数据 |
| **人员离职率严重超标** | 离职≥8人 **或** ≥20%（两者满足其一即触发） |

#### 5.1.3 离职率阶梯说明

> **计算方式**：以30天为计算周期
> `月度离职率 = 月度离职人数 ÷ 月度平均在岗人数 × 100%`
> **触发逻辑**：人数门槛与比例门槛**哪个先触发行使**，以先到为准

| 阶梯 | 触发条件 | 管理动作 |
|------|----------|----------|
| 🟡 关注 | 离职≥3人 **或** ≥10% | 站点负责人书面说明原因，供应商管理岗存档 |
| 🟠 预警 | 离职≥5人 **或** ≥15% | 站点提交整改计划，30天内提交人员补充方案 |
| 🔴 红线 | 离职≥8人 **或** ≥20% | 启动清退评估，业务交接准备 |

#### 5.1.4 预警响应

| 步骤 | 时限 | 责任人 |
|------|------|--------|
| 下发预警通知 | 评估结果公布当日 | 供应商管理岗 |
| 供应商提交整改计划 | 预警后 **3个工作日** | 站点负责人 |
| 整改实施 | 整改期 **30天** | 供应商 |
| 整改期间周报 | 每周 | 站点负责人 |
| 整改期满复评 | 第30天 | 供应商管理岗 |

### 5.2 清退机制

#### 5.2.1 清退触发条件（一票否决）

- 信息安全严重违规
- 人员持证率<90%
- 质检月均分<70分
- 数据造假
- 连续3个月E级评估

#### 5.2.2 清退流程

```
触发清退条件
  │
  ↓
供应商管理岗提出清退建议
  │
  ↓
业务对接人确认
  │
  ├─ 同意 ↓
  │
  供应商通知（书面）
  │
  ↓
过渡期（30天）：业务交接、人员安置、客户信息处理
  │
  ↓
正式清退
  │
  ↓
结算、归档

  │
  └─ 有异议 → 提交上级（部门负责人）仲裁
                  │
                  ├─ 维持清退 → 供应商通知（书面）
                  └─ 撤销清退 → 终止流程，记录归档
```

> **异议处理规则**：业务对接人对清退建议有异议的，须在 **2个工作日内** 提出书面异议及理由，由部门负责人仲裁。仲裁结果为最终结果，不再二次申诉。仲裁期间不影响清退建议的效力。

#### 5.2.3 客户信息处理（清退必备动作）

> 客户信息处理是清退流程的必备环节，须在过渡期内完成，否则不得进入正式结算。

| 步骤 | 内容 | 完成标准 |
|------|------|----------|
| 1. 数据导出 | 供应商将所有客户数据以京东指定格式导出 | 京东侧确认数据完整性 |
| 2. 交接确认 | 客户数据交接至京东指定接替供应商或京东业务系统 | 交接清单双方签字 |
| 3. 数据删除 | 供应商删除所有持有副本，并提交删除确认函 | 供应商签章《数据删除确认书》 |
| 4. 保密延续 | 供应商签署《保密延续协议》，合作终止后保密义务延续 **2年** | 协议原件留存 |

**违约责任**：若清退后发生客户信息泄露且查明源于供应商，供应商承担相应法律责任。

#### 5.2.4 过渡期安排

- 清退通知发出后，留出 **30天** 过渡期
- 过渡期内供应商须完成业务交接和人员安置
- 过渡期内仍需执行日常管理规范
- 过渡期满后正式终止合作

#### 5.2.5 供应商主动终止合作

如供应商因自身原因主动提出终止合作，按以下流程处理：

| 步骤 | 内容 | 说明 |
|------|------|------|
| 1. 书面通知 | 供应商提前 **30个自然日** 提交书面终止申请 | 说明终止原因 |
| 2. 影响评估 | 供应商管理岗评估对业务的影响范围 | 确定过渡期安排 |
| 3. 业务交接 | 与业务对接人共同确定交接方案 | 确保业务连续性 |
| 4. 客户信息处理 | 同5.2.3节要求 | 数据导出→交接确认→删除确认→保密协议 |
| 5. 正式终止 | 过渡期满后正式终止合作 | 完成结算和归档 |

**与京东主动清退的差异**：

| 维度 | 京东主动清退 | 供应商主动退出 |
|------|-------------|---------------|
| 通知方式 | 京东书面通知供应商 | 供应商书面通知京东 |
| 通知时限 | 京东自行决定 | 供应商须提前30日 |
| 过渡期 | 30天 | 30天（可协商缩短） |
| 违约责任 | 按合同执行 | 按合同执行 |

## 第六章 权责划分

### 6.1 角色与职责

| 角色 | 所属方 | 职责 |
|------|--------|------|
| 供应商管理岗 | 京东 | 制度制定、准入评审、评估考核、合规监督、预警清退 |
| 业务对接人（业管） | 京东 | 业务目标设定、交付质量监控、业务沟通 |
| 质检团队 | 京东 | 质检标准制定、质检执行、质检数据输出 |
| 站点负责人 | 供应商 | 统筹本站点全部业务、对管理要求的响应 |
| 站点对接人 | 供应商 | 日常信息报送、异常报备、台账维护，对接京东侧全部接口 |

### 6.2 协作边界

| 事项 | 主导方 | 配合方 |
|------|--------|--------|
| 准入评审 | 供应商管理岗 | 业务对接人（业管） |
| 日常数据报送 | 站点对接人 | 供应商管理岗 |
| 业务目标达成 | 业务对接人（业管） | 站点负责人 |
| 质量管理 | 质检团队 | 供应商管理岗（引用数据） |
| 月度评估 | 供应商管理岗 | 业务对接人（业管）（审核） |
| 预警清退 | 供应商管理岗 | 业务对接人（业管）（确认） |

### 6.3 沟通机制

| 场景 | 参与方 | 频率 |
|------|--------|------|
| 日常沟通 | 站点对接人 ↔ 供应商管理岗 ↔ 业务对接人 | 每日 |
| 周度复盘 | 三方（业务+管理+站点） | 每周 |
| 月度评估会 | 供应商管理岗 + 各站点 | 每月 |
| 专项沟通 | 按需发起 | 按需 |

## 第七章 附则

### 7.1 解释权

本办法由京东2B电销供应商管理岗负责解释。

### 7.2 修订机制

- 本办法每 **半年** review一次
- 重大修订须提前 **1个月** 通知所有供应商
- 修订记录留存备查

### 7.3 确认书

供应商签约时须签署《供应商管理办法确认书》，确认已阅读并同意遵守本办法及相关文件。合作终止时须另行签署《数据删除确认书》和《保密延续协议》。
