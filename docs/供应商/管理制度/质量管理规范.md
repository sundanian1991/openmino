## 第一章 总则

### 1.1 目的

明确供应商管理岗在质量管理中的角色定位——**不替代质检、不绕过质检，而是管理质检数据的产出和应用**，确保质检体系服务于供应商评估和业务提升。

### 1.2 与质检体系的关系

```
质检团队：制定标准 → 执行质检 → 输出数据
                        ↓
供应商管理岗：收数据 → 算分数 → 纳入评估 → 应用结果
                        ↓
供应商：接受质检 → 配合质检 → 按评估结果执行
```

**供应商管理岗的定位**：质检数据的使用者和评估者，不是质检的执行者。

### 1.3 管理对象

| 对象 | 管什么 |
|------|--------|
| 质检团队 | 数据有没有给、数据完不完整、标准是否透明 |
| 供应商 | 接不接受质检、配不配合、改不改进 |

## 第二章 质量管理角色与职责

### 2.1 角色划分

| 角色 | 职责 | 不做 |
|------|------|------|
| **质检团队** | 制定质检标准、执行质检、输出质检数据、处理质检申诉 | 不参与供应商评估决策 |
| **供应商管理岗** | 收集质检数据、按规则计算评估分、将结果纳入评估体系、监控质检数据完整性 | 不执行质检、不制定质检标准 |
| **供应商** | 接受质检、配合质检、根据质检结果整改 | 不拒绝质检、不干预质检 |

### 2.2 数据流向

```
质检月报（质检团队出）
  │
  │ 次月3日前
  ↓
供应商管理岗收数据
  │
  ↓
套入评估公式 → 得出"交付质量"维度得分（0-40分）
  │
  ↓
合并其他维度 → 得出总分 → 评级 → 排名 → 公示
```

## 第三章 对质检的要求

### 3.1 质检标准

| 要求 | 说明 |
|------|------|
| 成文标准 | 质检团队须有成文的质检评分标准和评分细则 |
| 备案 | 质检标准须向供应商管理岗备案一份 |
| 透明度 | 供应商有权了解质检评分标准 |

### 3.2 质检月报

质检团队每月须出具以下内容的质检月报：

| 内容 | 说明 |
|------|------|
| 各站点质检平均分 | 百分制 |
| 各维度分 | 按质检评分表维度分别统计 |
| 异常清单 | 严重违规事项清单（如有） |
| 覆盖率 | 实际抽检量/应抽检量 |
| 抽检人员分布 | 各坐席抽检量分布 |

**报送时间**：次月 **3日前**
**报送对象**：供应商管理岗

### 3.3 抽检覆盖率要求

| 要求 | 标准 |
|------|------|
| 常规质检覆盖率 | 每人每月 ≥X条（具体数量按业务实际确定） |
| 试用期人员 | 常规标准的 **2倍** |
| 覆盖率不足处理 | <80%时，按实际覆盖率折算质检分数 |

### 3.4 校准机制

质检团队须定期进行质检校准（质检标准一致性检验），确保不同质检员对同一通话的评分偏差在可接受范围内。

供应商管理岗有权要求质检团队提供校准记录。

## 第四章 对供应商的要求

### 4.1 接受质检

- 所有在岗坐席须接受质检抽检
- 不得拒绝、阻挠、延迟质检
- 不得提前通风报信规避质检

### 4.2 质检结果处理

- 质检结果以录音和数据为依据
- 对质检结果有异议的，按申诉流程处理（见《供应商月度综合评估方案》第六章）
- 质检不达标须在预警通知后 **30天内** 完成整改（参照《供应商管理办法》5.1.2）

### 4.3 内部质量机制

供应商须建立内部质量保障机制：

| 要求 | 说明 |
|------|------|
| 内部质检 | 供应商须有内部质检机制（自查自纠） |
| 新人培训 | 新上岗人员须完成话术和合规培训 |
| 定期复训 | 每月至少1次质量相关培训 |
| 问题追踪 | 质检问题须有跟进闭环机制 |

## 第五章 质量数据在评估中的应用

### 5.1 评分规则

质检月均分（百分制）折算为月度评估中"交付质量"维度得分（满分40分）：

| 质检月均分 | 交付质量得分 |
|-----------|-------------|
| ≥95 | 40 |
| 90-94 | 36 |
| 85-89 | 32 |
| 80-84 | 28 |
| 75-79 | 24 |
| 70-74 | 20 |
| <70 | 0（触发红线） |

### 5.2 红线检查

质检月报中的异常清单须逐条检查是否触发以下红线：

| 红线 | 说明 |
|------|------|
| 质检月均分<70分 | 交付质量严重不达标 |
| 信息安全违规 | 严重违规即触发清退评估 |
| 严重话术违规 | 承诺超范围、虚假承诺等 |

### 5.3 数据质量监控

供应商管理岗对质检数据保留核实权：

| 场景 | 处理方式 |
|------|----------|
| 质检月报未按时提供 | 书面催办，连续2月未提供则启动上级协调 |
| 数据明显异常 | 要求质检补充说明或启动复核 |
| 覆盖率不足 | 按实际覆盖率折算 |

## 第六章 兜底机制

### 6.1 质检数据缺位

| 场景 | 处理 |
|------|------|
| 质检未按时出月报 | 交付质量维度记0分，书面催办 |
| 质检月报内容不完整 | 要求补充，未补充部分按缺失处理 |
| 某站点无质检数据 | 该站点交付质量维度记0分 |

### 6.2 质检标准争议

| 场景 | 处理 |
|------|------|
| 供应商对质检标准有异议 | 供应商管理岗发起三方校准会（质检+供应商+供应商管理） |
| 校准结果 | 校准结果对后续评估生效，不追溯 |
| 质检内部标准不一致 | 要求质检团队完成校准后重新出具数据 |

### 6.3 质检与业务方的关系

质检数据独立于业务方的管理要求。供应商管理岗引用质检数据时，以质检月报为唯一依据，不接受业务方的单独质量评价作为替代。
