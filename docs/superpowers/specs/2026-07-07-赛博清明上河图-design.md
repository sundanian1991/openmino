## 0. 一句话目标

把北宋张择端《清明上河图》重译成**赛博朋克版新汴京**，做成一个**横向滚动、可交互、自包含的 HTML 长卷**，保留原作三幕式空间叙事（郊野→虹桥→市井），用 CSS 程序化元素填充密度、AI 高保真插画做三处"画眼"、Canvas 动效在画眼处增色。

## 1. 世界观设定

**时空**：公元 2087 年，北宋汴京遗址上的赛博朋克超级都市「新汴京 Neo-Bianjing」。

**核心原则——古今叠加**：每个画面元素都必须同时承载"古画原型"和"赛博重译"两层含义，缺一不可。

| 古风原型（清明上河图） | 赛博化重译 |
|---|---|
| 青瓦飞檐 | 全息投影的飞檐轮廓，霓虹描边 |
| 红灯笼 | 漂浮的等离子灯笼，脉冲发光 |
| 驴 / 驼队 | 机械驮兽（四足机甲，载发光集装箱） |
| 木船 | 反重力驳船，霓虹吃水线 |
| 市民服饰 | 汉服剪裁 + LED 纹样 + 义体植入 |
| 招牌幌子 | 中文霓虹招牌 + 全息广告 |
| 晨雾 | 工业雾霾 + 数据粒子 |
| 汴河 | 液冷渠道（数据中心冷却水道），发冷光 |
| 纤夫 | 岸边拉光纤缆束的维护工 |
| 茶楼 | 全息茶艺表演 + 脑机接口顾客 |
| 佛像 / 神龛 | 全息佛像，赛博宗教 |

## 2. 叙事结构（忠于原作三幕）

从右到左展开（符合古卷展开方向）。三幕占长卷比例：**25% / 40% / 35%**，第二幕（虹桥）最长，是高潮。

### 第一幕 · 郊野晨雾（25%）— 冷·孤寂
- 工业雾霾笼罩的城郊废墟带，半坍塌土楼
- 机械驴队驮发光集装箱沿土路前行
- 远处核聚变塔冷光，低空物流无人机编队
- 穿汉服戴 VR 面罩的"踏青者"在雾中漫游
- 稀疏等离子柳树（光丝垂落）
- **画眼 #1**：晨雾中回望新汴京天际线剪影

### 第二幕 · 汴河虹桥（40%）— 繁华·高潮
- 发冷光的液冷渠道（汴河），水面倒映霓虹
- 反重力驳船排队过桥洞，船身有中文霓虹船名
- **全息拱桥**：半透明力场结构 + 物理桥墩
- 桥上密集人群：全息小贩、扫码市民、无人机警
- 两岸层叠赛博牌楼（飞檐 + 霓虹招牌），招牌："汴京霓虹""全息茶馆""义体维修"
- 纤夫 = 拉光纤缆束的维护工
- **画眼 #2**：高桅反重力驳船艰难过桥洞，船工放倒桅杆的瞬间（致敬原作最经典场景）

### 第三幕 · 城内市井（35%）— 暖·喧嚣
- **巨型全息牌楼城门**：物理残骸 + 力场投影飞檐
- 机械驼队（四足机甲）过安检扫描门
- 垂直赛博市井：地面摊贩 + 悬浮连廊 + 顶层飞行器停泊
- 密集中文霓虹招牌森林（九龙城寨 × 银翼杀手气质）
- 茶楼全息茶艺师，顾客戴脑机接口
- 飞行轿车穿梭楼宇间，拉出光带
- 街角全息佛像 / 神龛
- **画眼 #3**：城门下机械驼队入城，霓虹光从门洞倾泻

## 3. 交付物与技术架构

### 3.1 交付物
- **单一 HTML 文件**，所有 CSS / JS 内联，AI 插画以 base64 内嵌
- 字体走 Google Fonts CDN（需联网；离线场景降级为系统字体）
- 可直接双击打开，或部署到任意静态托管
- 建议文件名：`cyber-qingming.html`

### 3.2 技术栈分层

```
┌─────────────────────────────────────────────────────────┐
│  Canvas 动画层   ｜ 霓虹脉冲/粒子/扫描线（仅 3 处画眼）   │
│  AI 插画层       ｜ 3 张高保真场景 PNG（base64 内嵌）     │
│  CSS 微型元素层  ｜ 人物/招牌/灯笼/楼宇（密度主力）       │
│  渐变背景层      ｜ 夜空/地平线/扫描线纹理                │
└─────────────────────────────────────────────────────────┘
```

### 3.3 关键技术决策

| 决策点 | 选择 | 理由 |
|---|---|---|
| 布局 | `overflow-x: scroll` + `scroll-snap-type: x mandatory` | 真正"展卷"体验 |
| 密度主力 | 纯 CSS + JS 程序化生成 | 轻量、可控、流畅、"浩荡感"强 |
| 画眼插画 | AI 生图（3 张 PNG） | 关键节点高保真，规避全 AI 长卷连贯性问题 |
| 画眼动画 | Canvas 局部叠加 + IntersectionObserver 控启停 | 动态迷人但不拖垮性能 |
| 自包含 | 单 HTML，base64 内嵌图 | 可离线分享，部署零配置 |

### 3.4 骨架参考（可选借鉴，非依赖）
如执行者所在项目有 `workspace/archive/projects/2026-04-25-电影版叙事/cinematic.html`，可借鉴其分幕结构、扫描线纹理、IntersectionObserver reveal 动画，把纵向（`scroll-snap-type: y`）改为横向（`x`）。**无此文件不影响执行**，本 spec 自包含。

## 4. 视觉规范

### 4.1 色板

```css
:root{
  /* 深空底色（占 ~70% 面积） */
  --void:      #06080f;   /* 主背景，近黑深紫 */
  --void-2:    #0a0e1a;   /* 次背景 */
  --void-3:    #120818;   /* 紫调过渡层 */

  /* 霓虹主色（~25%，三幕各有主导色） */
  --neon-cyan:   #00f0ff;  /* 第一幕主导：郊野·冷·孤寂 */
  --neon-pink:   #ff2d95;  /* 第二幕主导：虹桥·热·繁华 */
  --neon-yellow: #ffd23f;  /* 第三幕主导：市井·暖·喧嚣 */
  --neon-violet: #b14dff;  /* 辅色：全息/宗教/神秘 */

  /* 古画保留色（~5%，点睛） */
  --vermillion:  #d4391e;  /* 朱砂：印章、牌匾、神龛 */
  --gold:        #c8a96a;  /* 金：灯笼骨架、桥拱、卷轴裱边 */
  --ink-silk:    #e8dcc4;  /* 绢色：卷轴边缘、题字底 */

  /* 文字 */
  --text-hi:     #f0f4ff;
  --text-mid:    #9fb0cc;
  --text-dim:    #5a6a8a;
}
```

**三幕色温递进**（情绪弧线）：
1. 第一幕：冷青 + 紫雾 → 黎明前孤寂
2. 第二幕：粉 / 青 / 黄混战 → 繁华高潮
3. 第三幕：暖黄 + 朱砂升温 → 市井喧嚣烟火

每幕背景渐变应有差异，过渡区做色温平滑衔接。

### 4.2 字体

| 用途 | 字体 | weight | 场景 |
|---|---|---|---|
| 古画题字 / 印章 | **Noto Serif SC** | 900 | 卷首题款、幕标题、朱砂印章 |
| 赛博标题 / 招牌 | **Chakra Petch** | 600-700 | 霓虹招牌、幕间标签（如 `ACT II · 虹桥`） |
| 等宽数据 / 坐标 | **Space Mono** | 400/700 | HUD 坐标、画眼标注、时间戳 |

Google Fonts 引入：
```html
<link href="https://fonts.googleapis.com/css2?family=Chakra+Petch:wght@400;500;600;700&family=Space+Mono:wght@400;700&family=Noto+Serif+SC:wght@400;700;900&display=swap" rel="stylesheet">
```

**混搭规则**：大字用衬线（古画气质），小字 / 数据用等宽（赛博气质）。一个文本块只认一种身份，绝不混用。

- 画布总宽 **~12000px**（三屏半，每幕 ≥1 视口宽以保证 scroll-snap 沉浸感）
- 高度锁 **100vh**（建议 900px 体验最佳）

## 6. 三种密度的实现规范

### 6.1 层 2 · CSS 微型元素（密度主力，全卷）

程序化生成大量微型元素填充背景。每幕的目标元素清单：

| 元素 | 第一幕 | 第二幕 | 第三幕 | 实现 |
|---|---|---|---|---|
| 人物剪影 | 稀疏 ~40 | 密集 ~160（桥上+两岸） | 极密 ~200 | `<div>` 小条，深色 + 微霓虹光晕 |
| 霓虹招牌方块 | 远景窗灯 ~50 | 牌楼招牌群 ~80 | 招牌森林 ~120 | 彩色方块 + box-shadow 发光 |
| 灯笼 | ~10 等离子 | ~20 | ~30 | 圆形 radial + 强发光 |
| 飞行器 | 无人机编队 ~5 | 驳船桅顶光 ~8 | 飞行轿车 ~12 拉光带 | 椭圆 + 轨迹线 |
| 建筑 | 土楼剪影 | 牌楼 + 桥墩 | 垂直楼宇森林 | SVG path / div 块 |

JS 生成模板（**伪代码，示意逻辑**，`size`/`randomNeonGlow` 等变量由调用方传入或函数内部随机生成）：
```js
function spawnCrowd(container, count, {yMin, yMax, sizeMin, sizeMax, glowColors}){
  for(let i=0;i<count;i++){
    const size = sizeMin + Math.random()*(sizeMax-sizeMin);       // 像素尺寸
    const glow = glowColors[Math.floor(Math.random()*glowColors.length)];
    const el = document.createElement('div');
    el.style.cssText = `
      position:absolute;
      left:${Math.random()*100}%;
      top:${yMin + Math.random()*(yMax-yMin)}%;
      width:${size}px; height:${size*2.2}px;
      background:#0a0408;
      box-shadow:0 0 2px ${glow};
      border-radius:40% 40% 30% 30%;
    `;
    container.appendChild(el);
  }
}
// 调用示例：spawnCrowd(act2CrowdLayer, 160, {yMin:55,yMax:90,sizeMin:2,sizeMax:5,glowColors:['#00f0ff','#ff2d95','#ffd23f']});
```

**性能护栏（强制）**：
```js
const focals = document.querySelectorAll('.focal');
const io = new IntersectionObserver((entries)=>{
  entries.forEach(e=>{
    const canvas = e.target.querySelector('.focal-canvas');
    if(e.isIntersecting) startFocalAnim(canvas, e.target.dataset.focal);
    else stopFocalAnim(canvas);
  });
},{threshold:0.15});
focals.forEach(f=>io.observe(f));
```
画眼离开视口必须暂停 Canvas 动画循环，禁止全卷同时跑动画。

### 嵌入方式

每张图生成后，压到 ~150-300KB（PNG 优化或转 WebP），转 base64 内嵌：
```html
<div class="focal-img" style="background-image:url(data:image/png;base64,<BASE64_HERE>)"></div>
```
或存为独立文件，用相对路径（牺牲单文件自包含）：
```html
<div class="focal-img" style="background-image:url(./assets/focal-1-dawn.jpg)"></div>
```

### 生图风险与迭代策略（执行者必读）

| 风险 | 应对 |
|---|---|
| 风格不统一（三张图像不同世界） | 严格遵守共享风格锚；先生画眼 #2（高潮）定调，满意后再生成 #1、#3 |
| 中文招牌文字乱码 | AI 生图的中文常乱码，可接受抽象字形；或后期用 CSS 叠加真正的中文霓虹文字层覆盖 |
| 人物密度不足 | 在 prompt 强调 "extremely dense crowd, hundreds of tiny figures"；不足处用层 2 的 CSS 人物补充 |
| 色调与 CSS 层冲突 | 生成后检查，必要时给 `.focal-img` 加 `filter: brightness(0.9) hue-rotate(-5deg)` 微调融合 |
| 画眼与相邻 CSS 层衔接生硬 | 在 `.focal` 两侧加渐变 mask（`mask-image: linear-gradient`）羽化边缘 |

**推荐迭代节奏**：先生 1 张画眼 #2 试水 → 确认风格与色板 → 再批量生成 #1、#3。避免三张一起生成后风格全错。

## 9.5 移动端与降级

**本长卷为桌面体验优先的艺术作品**（横向卷轴在大屏展卷才有气势）。移动端做最小降级，不作为首要验收目标：

- **尺寸**：移动端（视口宽 <768px）总宽等比缩小至 ~6000px，高度保持 100vh，保留 scroll-snap
- **密度**：程序化元素数量减半，避免移动端渲染卡顿
- **画眼动画**：移动端可关闭 Canvas 动效层（`@media (pointer:coarse){ .focal-canvas{display:none} }`），仅保留静态 AI 插画
- **字体**：移动端 Google Fonts 降级为系统衬线 + 系统等宽，减少网络请求
- **横屏提示**：竖屏进入时给一句轻提示"建议横屏浏览"，不强制旋转

## 10. 验收标准

完成后需满足：

1. ✅ **单文件自包含**：双击 HTML 即可打开，无本地依赖（字体可走 CDN）
2. ✅ **横向滚动展卷**：scroll-snap 在每幕自然停顿
3. ✅ **三幕可辨**：郊野（冷青）→ 虹桥（粉黄混战）→ 市井（暖朱砂）色温递进清晰
4. ✅ **密度达标**：每幕有"看不过来"的浩荡感，非空旷（参考样张 A 栏密度量级）
5. ✅ **三画眼高保真**：AI 插画清晰、风格统一、嵌入位置正确
6. ✅ **画眼有动效**：Canvas 动画在画眼进入视口时启动，离开暂停
7. ✅ **HUD 正常**：进度条、幕标签、坐标随滚动实时更新
8. ✅ **古今叠加**：每个关键元素都能看出"古画原型 + 赛博重译"双层
9. ✅ **性能可接受**：滚动流畅 ≥30fps，无全卷动画同时跑
10. ✅ **卷轴物理感**：金线裱边、卷首尾印章、样式化滚动条

## 11. 实施顺序建议

1. **骨架先行**：搭 `.scroll-container` + 七个 section + scroll-snap + HUD，验证横向滚动与幕标签切换
2. **背景层**：三幕渐变 + 扫描线纹理 + 卷轴裱边
3. **CSS 密度层**：JS 程序化生成各幕微型元素，调密度到"浩荡"
4. **画眼 #2 试水生图**：生成一张虹桥图，确认风格与色板
5. **画眼 #1、#3 生图**：批量生成其余两张，base64 内嵌
6. **画眼 Canvas 动效**：三处动画 + IntersectionObserver 启停
7. **过渡区 + 卷首尾**：纤夫剪影、城墙浮现、印章、题款
8. **收尾**：HUD 坐标、进度条、滚动条样式、性能调优
