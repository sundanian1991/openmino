# B5 向上汇报准备

> 汇报前想清楚：对面的人关心什么、我该说什么、什么不该说。

## 触发信号

- 需要给王易人（直属上级）汇报供应商情况
- 需要给军哥（卞海军，二级部门负责人）汇报
- 月度/季度汇报节点
- 出现重大异常需要向上同步（A级骤降、连续预警、清退决策）

## 核心问题

> 这场汇报的目标是什么？对面的人听完之后应该做什么决定、或者获得什么判断？

## 思考步骤

### 第1步：确认汇报对象和场景

**想什么**：不同对象关心不同的事，口径不能一套打天下。

**检查什么数据**：
- 汇报对象是谁？（王易人 / 军哥 / 两人都在）
- 汇报场景是什么？（周会 / 月度汇报 / 专项问题 / 决策请示）

**判断什么**：

| 汇报对象 | 关心什么 | 不关心什么 | 口语特点 |
|---------|---------|-----------|---------|
| **王易人** | 具体供应商状态、需要他决策的事、风险预警 | 你怎么做到的、过程细节 | 直接说结论，他熟悉业务细节 |
| **军哥** | 大盘数据、趋势、部门间对比、投入产出 | 个别供应商的个案细节 | 数字说话，趋势为主，少说个案 |
| **两人都在** | 先说军哥关心的（大盘），再说王易人关心的（个案） | 不要在军哥面前展开过多细节 | 分层表达，可随时展开或收起 |

**举例**：王易人问你"毛毛虫最近怎么样"，他要的是结论（连续2个月后30%，问题型，建议约谈）而不是你分析的过程。军哥问你"供应商整体情况"，他要的是"32家供应商，上月GMV达成率95%，3家需要关注，已安排约谈"。

### 第2步：确定汇报目标

**想什么**：这场汇报结束后，对面的人应该做什么？

**判断什么**：

| 目标类型 | 具体表现 | 口径重心 |
|---------|---------|---------|
| **信息同步** | 让上级知道情况，不需要决策 | 现状 + 已采取措施 + 下一步计划 |
| **决策请示** | 需要上级拍板 | 现状 + 方案选项（≥2个）+ 我的建议 + 需要确认的点 |
| **风险预警** | 有事情可能变坏 | 现状 + 风险点 + 已做防护 + 可能需要上级介入的点 |
| **资源请求** | 需要上级协调资源 | 背景 + 为什么需要 + 具体要什么 + 预期效果 |

**举例**：毛毛虫连续2个月后30% → 目标是"信息同步+轻微决策请示"（同步问题，请示是否约谈）。翰锐A级骤降+疑似管理问题 → 目标是"风险预警"（让王易人知道风险，确认干预力度）。

### 第3步：组织核心论据

**想什么**：用叙事模板把数据串成有逻辑的故事，不是堆数字。

**检查什么数据**（按需选取）：

| 汇报内容 | 需要的数据 | 叙事模板 |
|---------|-----------|---------|
| 整体月度情况 | 总GMV达成率、产线分布、环比趋势 | 模板一：整体趋势→头部亮点→尾部警示→下月重点 |
| 单条产线分析 | 产线内供应商排名、峰值利用率对比 | 模板二：排名→对比分析→异常归因→建议 |
| 单个供应商问题 | 排名变化、人效数据、出勤率、原因分析 | 模板三：数据→故事→建议 |

**判断什么**：
- 每个论点必须有1-2个数据支撑，不能空说"不太好"
- 数据要翻译成人话："峰值利用率65%"不如"产能只用了三分之二"
- 负面信息必须有对应的行动方案，不能只报问题不报方案

**举例**：
- 不好的口径："毛毛虫最近排名不太好，连续两个月靠后。"
- 好的口径："毛毛虫连续2个月排名后30%，产能利用率从78%降到65%，主要原因是新员工占比从20%升到42%。建议正式约谈要求整改，约谈提纲已准备好。"

### 第4步：预判上级可能问的问题

**想什么**：上级会追问什么？提前准备好答案。

**判断什么**：

| 上级 | 常问问题 | 需要准备的答案 |
|------|---------|---------------|
| 王易人 | "你觉得是什么原因？" | 原因定位（参考A1第3步的5个维度） |
| 王易人 | "你打算怎么处理？" | 具体方案 + 时间节点 |
| 王易人 | "需要我出面吗？" | 明确说需要/不需要，需要的话具体说什么时候 |
| 军哥 | "整体趋势怎么样？" | 3个月环比数据 + 趋势判断 |
| 军哥 | "和其他部门比呢？" | 这个问题如果答不上来就直说"我回去查一下"，不要编 |
| 军哥 | "投入产出比怎么样？" | 单客成本、人均GMV、管理投入 vs 产出 |

**举例**：汇报毛毛虫问题时，王易人可能问"他们新员工占比高的原因是什么？" → 提前准备好答案（是他们自己招聘管理问题，还是我们分量调整导致扩招太快？这决定了责任归属和干预方式）。

### 第5步：控制汇报节奏和深度

**想什么**：汇报不是把所有信息倒出来，是让上级高效拿到他需要的东西。

**判断什么**：

| 场景 | 时间 | 深度控制 |
|------|------|---------|
| 周会简短同步 | 2-3分钟 | 结论+1个关键数据+下一步，不展开 |
| 月度正式汇报 | 10-15分钟 | 整体趋势+重点供应商（2-3家）+下月重点 |
| 专项问题汇报 | 5-10分钟 | 问题+原因+方案选项+建议，聚焦不发散 |
| 书面汇报材料 | 阅读时间3-5分钟 | 结构化文档，关键数据加粗，结论前置 |

**原则**：
- 结论先行（先说结论再说原因）
- 数据翻译成人话（百分比>绝对值，趋势>单点）
- 每个问题带方案（不空手来）
- 控制供应商数量（口头汇报最多展开3家，其余附表）

### 第6步：准备书面材料（如需要）

**想什么**：如果汇报需要书面材料，用什么格式？

**判断什么**：

| 场景 | 推荐格式 | 对应模板 |
|------|---------|---------|
| 月度汇报邮件 | 正式汇报邮件 | `formal-report.md`（正式汇报场景） |
| 月度供应商报告 | 月度运营报告 | `supplier-monthly-report.txt` |
| 专项问题决策请示 | 决策请示邮件 | `internal-coordination.md`（决策请示场景） |
| 数据附件 | 表格形式 | `supplier-evaluation-full.md`（数据收集/评估场景） |

**材料撰写要点**：
- 标题直接点明主题（"3月供应商管理月报"而不是"工作汇报"）
- 正文第一段是摘要（3句话以内说清重点）
- 数据用表格不用大段文字描述
- 结论和建议加粗

## 关键变量

| # | 变量 | 判断标准 | 说明 |
|---|------|---------|------|
| 1 | **汇报对象** | 王易人 / 军哥 / 两人 | 决定口径深度和重点 |
| 2 | **汇报目标** | 信息同步 / 决策请示 / 风险预警 / 资源请求 | 决定口径结构 |
| 3 | **可用时间** | 3分钟 / 10分钟 / 书面 | 决定展开深度 |
| 4 | **关键数据量** | 1-3个核心数据点 | 不是越多越好，挑最能说明问题的 |
| 5 | **需要确认的决策** | 0个 / 1个 / 多个 | 有决策项要明确列出，不能含糊 |

## 常见误区

| 误区 | 正确做法 |
|------|---------|
| 汇报从过程开始说 | 结论先行。先说"毛毛虫需要约谈"，再说为什么 |
| 把分析过程当汇报内容 | 上级不需要知道你怎么分析的，他要的是结论和建议 |
| 只报问题不报方案 | 每个问题必须带方案，空手来等于没准备 |
| 军哥面前展开个别供应商细节 | 军哥要的是大盘和趋势，个案一笔带过，他追问再展开 |
| 被追问时编数据 | 答不上来说"我回去确认一下"，不编不猜 |
| 书面材料写太长 | 书面汇报控制在1页A4以内，细节放附件 |

## 决策输出

> **B5 汇报准备清单**
>
> **汇报对象**：王易人
> **场景**：月度汇报
> **目标**：信息同步 + 请示毛毛虫约谈方案
> **时间**：10分钟
>
> **口径结构**：
> 1. 整体：上月GMV达成率95%，金条线超目标，信用卡线未达标
> 2. 亮点：毅航金条线连续3个月前3
> 3. 问题：毛毛虫连续2个月后30%（问题型），岐力首次预警
> 4. 请示：毛毛虫约谈方案——建议正式约谈，要求提交整改计划
> 5. 下月重点：约谈毛毛虫 + 信用卡线分量评估
>
> **预期追问**：
> - "毛毛虫问题原因？"→ 新员工占比42%+培训跟不上
> - "岐力需要干预吗？"→ 首次预警，先私下沟通，暂不约谈
>
> **书面材料**：`supplier-monthly-report.txt`（月报模板）
