# D1 被同事@数据/进度

> 同事突然@你问数据或进度，30秒内判断该给什么、怎么回复。

## 触发信号

- 刘伟佳（策略组）@你问排名/分量相关数据
- 刘乾坤@你问巡检结果/整改进度
- 郭鑫@你催项目进度/上线时间
- 群里被直接点名："@年老师 XX数据有吗"
- 私聊突然发来："XX供应商最近怎样"

## 核心问题

同事@你不是来聊天的，是要一个具体的数或结论。核心问题只有三个：

1. 他到底要什么？（排名？进度？异常原因？）
2. 这个数据我能不能直接给？（有没有敏感性）
3. 给了之后会不会引来追问？（口径是否闭环）

## 30秒快速判断

> 谁？要什么？紧急吗？能回答吗？一句话口径。

**第一秒 — 识别来者身份和意图：**

| 来者 | 通常要什么 | 敏感度 |
|------|-----------|--------|
| 刘伟佳 | 排名数据、分量配比、供应商产能 | 中 — 策略组需要数据做决策，但未确认的排名不能外泄 |
| 刘乾坤 | 巡检发现、整改清单、合规状态 | 低 — 巡检结果按流程该公开 |
| 郭鑫 | 项目里程碑、上线排期、交接进度 | 中 — 涉及供应商承诺的时间节点 |
| 其他同事 | 一般是具体供应商的某个指标 | 视情况 |

**第二秒 — 判断紧急度：**

| 信号 | 紧急度 | 动作 |
|------|--------|------|
| 群里@ + 连续追问 | 高 | 5分钟内必须回复，哪怕先给个方向 |
| 群里@ 一次 | 中 | 30分钟内回复 |
| 私聊问 | 中低 | 当天内回复 |
| 邮件问 | 低 | 24小时内回复 |

**第三秒 — 能不能直接回答：**

| 情况 | 处理 |
|------|------|
| 数据在手，无敏感性 | 直接给 |
| 数据在手，但涉及未公开排名 | "数据有，但排名还没最终确认，确认后同步你" |
| 数据不在手，需要调取 | "我查一下，X分钟后给你" |
| 不确定能不能给 | 先问王易人，不自己决定 |

**一句话回复口径模板：**

> "XX数据我手上有/需要调一下，[具体数字/预计时间]，[补充说明如果需要]。"

## 5分钟深入版

如果30秒回复后对方追问，或者问题本身需要展开说明，按以下步骤深入。

## 思考步骤

### 步骤1：确认需求边界

**想什么**：他问的到底是哪个维度的数据？

**检查什么数据**：
- 刘伟佳问排名 → 确认是哪个月、哪条产线、要综合排名还是单项
- 刘乾坤问巡检 → 确认是哪家供应商、哪次巡检、要清单还是要结论
- 郭鑫催进度 → 确认是哪个项目、哪个节点、卡在哪里

**判断什么**：
- 问题明确 → 直接给数据
- 问题模糊 → 先追问再回答，别猜着给
- 问题超出你的职责 → 指向正确的人，别揽过来

**举例**：刘伟佳群里说"@年老师 毅航最近表现怎么样"，这个问法很宽。追问："是看综合排名还是某个指标？我手上有3月数据。"

### 步骤2：调取和核实数据

**想什么**：这个数据的最新版本是什么？有没有异常需要提前说明？

**检查什么数据**：
- 排名类 → 查最新月度排名表，看毅航/毛毛虫/伽玛等头部家的位置变化
- 进度类 → 查项目跟踪表，看当前节点和延迟天数
- 巡检类 → 查巡检记录，看问题清单和整改状态

**判断什么**：
- 数据有没有滞后？比如现在4月中旬，3月数据是最新的但不是实时的
- 有没有异常值需要标注？比如赛维斯本月出勤率突然从92%掉到78%
- 数据口径和对方期望的口径是否一致？

### 步骤3：组织回复口径

**想什么**：怎么给数据让对方不再追问，一次把话说透？

**判断什么**：
- **给结论 + 给依据**："毛毛虫3月排名第2，比2月上升1位，主要因为金条产线转化率提升了12%。"
- **预判追问方向**：如果数据有异常，主动解释。"出勤率下降是因为春节后招了一批新人，正在培训期。"
- **控制信息边界**：只回答被问的，不主动扩展到其他供应商。

### 步骤4：识别隐含需求

**想什么**：他问这个问题，背后可能在做什么决策？

**判断什么**：
- 刘伟佳问分量相关 → 可能要调整分配方案，主动问"是要调分量吗？"
- 刘乾坤问巡检 → 可能有合规风险要上报，确认"需要我同步王易人吗？"
- 郭鑫催进度 → 可能是上面在催他，回复时带上时间节点显得你也在跟

### 步骤5：确认后续动作

**想什么**：回复完之后需要做什么？

**检查什么**：是否需要同步其他人、是否需要发正式文档。

**判断什么**：
- 如果给了口头数据，对方可能需要正式版本 → 提醒"需要我发正式版邮件吗？"
- 如果暴露了问题 → 确认"这个需要同步王易人吗？"
- 如果涉及供应商层面 → 确认"要我和供应商确认一下吗？"

## 关键变量

| 变量 | 判断标准 | 影响 |
|------|---------|------|
| **数据时效性** | 最新可用数据是哪个时间点？超过7天的数据要标注"截至X月X日" | 决定回复可信度 |
| **数据敏感性** | 涉及未确认排名、其他供应商对比、处罚信息 → 不能直接给 | 决定能否当场回复 |
| **追问压力** | 对方是简单要个数还是背后有决策压力？ | 决定回复深度和主动补充量 |
| **关系权重** | 刘伟佳（策略组负责人，平级协作）、刘乾坤（巡检，流程性沟通）、郭鑫（项目，催促型沟通） | 决定语气和配合程度 |
| **你的时间** | 手头有没有更紧急的事？这个回复能不能排后面？ | 决定是30秒版还是5分钟版 |

## 常见误区

| 误区 | 后果 | 正确做法 |
|------|------|---------|
| 问什么答什么，不追问 | 给了错误口径的数据，回头纠正更被动 | 先确认需求边界，再给数据 |
| 把其他供应商的数据一起给了 | 泄露不该泄露的信息 | 只给被问的那家 |
| 怕得罪人，不敢说"我不确定" | 给了错误数据，后续全盘被动 | 不确定就说"我确认一下"，比瞎说强 |
| 群里直接回复敏感数据 | 全群可见，信息不可控 | 敏感数据私聊发，群里只给结论 |
| 给完数据不管了 | 对方拿你的数据做了错误判断 | 给数据时主动标注口径和注意事项 |

## 决策输出

**回复格式示例：**

> 简单数据回复（群聊）：
> "@XX 毅航3月综合排名第3，金条产线排名第2。数据截至3月31日。"
>
> 带异常说明的回复：
> "@XX 毛毛虫3月出勤率78%，比2月低14个百分点。已和供应商确认，原因是3月扩编80人，新人培训期出勤不计入有效出勤。预计4月中旬恢复正常。"
>
> 需要追问的回复：
> "伟佳，你说的排名是综合排名还是单产线的？我手上有3月完整数据，确认后马上发你。"
