# C1 问题拆解

> 把脑子里一团浆糊的事情，拆成能动手干的具体步骤。

## 触发信号

- 领导说"你看看XX供应商怎么回事"，你一时不知道从哪下手
- 月度数据出来，某家供应商排名掉了，原因说不清
- 同时有好几件事催着你，不知道先做什么
- 供应商老板提了一个模糊诉求，你不确定他到底要什么
- 想推进某个制度改进，但不知道第一步干什么

## 核心问题

> 模糊问题之所以模糊，是因为混了多个层面。拆解的核心动作只有一个：**把"一团"变成"一列"**。

## 思考步骤

### 第1步：用一句话写下你面对的问题

**想什么**：你现在脑子里那个"一团"到底是什么？别追求精准，先用大白话写下来。

**检查什么**：写完后问自己一个问题——"如果这句话让同事看，他能不能知道我要解决什么？"如果看不懂，说明还没想清楚边界。

**判断什么**：

| 你写出来的样子 | 说明 | 怎么改 |
|--------------|------|--------|
| "毛毛虫最近不行" | 太模糊，"不行"指什么？ | 改成"毛毛虫3月排名从第2跌到第7，金条产线GMV降了22%" |
| "要搞定供应商管理制度更新" | 太大，"搞定"是个筐 | 改成"把供应商准入到清退的全流程写成制度文件" |
| "好多事要处理" | 没有聚焦 | 先列清单，再逐个改写 |
| "伽玛老板觉得分量不公平" | 比较具体，可以往下拆 | 直接进入第2步 |

**举例**：军哥问"赛维斯怎么回事"→ 你先写："赛维斯3月出勤率78%，低于85%预警线，需要找到原因并决定下一步动作。"

### 第2步：把问题按"事实-判断-行动"三层分开

**想什么**：很多问题之所以想不清楚，是因为事实和判断搅在一起了。先把它们分开。

**检查什么数据**：

| 层面 | 问题 | 示例（赛维斯出勤率低） |
|------|------|---------------------|
| **事实** | 发生了什么？数据是什么？ | 出勤率78%，低于85%阈值。新员工占比35% |
| **判断** | 这意味着什么？是好是坏？严重不严重？ | 出勤率低是因为扩编，不是管理松散。属于"可解释的异常" |
| **行动** | 需要做什么？谁来做？什么时候？ | 不需要约谈。本周和赛维斯主管确认培训周期，预计4月中旬恢复 |

**判断什么**：
- 事实层有没有缺数据？缺了先去查，别在缺数据的情况下做判断
- 判断层有没有多个可能？列出所有可能的原因，别只留一个
- 行动层有没有超出你的权限？有就标注出来

**举例**：毛毛虫老板说"最近量不够"。事实：毛毛虫当前产能利用率68%，排名第4。判断：利用率确实有空间，但排名第4不差也不突出。行动：查金条产线分量分配方案，看是否有调增空间，如果有给毛毛虫增量通知。

### 第3步：找到问题的"最小可行动作"

**想什么**：拆完之后别急着做完整方案，先找到第一步能做的事。

**检查什么**：
- 哪个动作能最快减少不确定性？（比如先查数据、先打电话确认一个前提）
- 哪个动作不做，后面的动作就没意义？（比如不确认排名数据就约谈，等于盲目行动）

**判断什么**：

| 情况 | 最小可行动作 | 不要做 |
|------|------------|--------|
| 不清楚原因 | 先查数据，列出所有可能原因 | 别急着约谈或写方案 |
| 不清楚对方意图 | 先问一句确认 | 别猜着行动 |
| 原因清楚但方案多 | 列出2-3个方案，每个写优缺点 | 别急着选一个 |
| 方案清楚但权限不够 | 先准备材料，再报领导 | 别越权执行 |

**举例**：歧力质检连续3个月下滑。最小可行动作：调取歧力近3个月质检扣分明细，按扣分类型归类（话术错误？流程违规？数据安全？），找到最大扣分项，再去和歧力主管聊。而不是一上来就写整改通知。

### 第4步：排列优先级（如果有多个子问题）

**想什么**：拆出来可能有5件事，不可能同时做。按什么排优先级？

**检查什么**：

用两个维度排序：**紧急程度**（时间压力）x **影响程度**（不做的后果）。

| | 影响大 | 影响小 |
|------|--------|--------|
| **紧急** | 先做 | 第二做 |
| **不紧急** | 第三做（安排时间） | 可以不做 |

**判断什么**：
- 领导问的事 → 默认紧急，优先处理
- 涉及合规/数据安全 → 影响大，优先处理
- 供应商主动找你 → 先判断紧急度再排
- 自己想推进的改进 → 不紧急但影响大，安排固定时间做

**举例**：本周有4件事：(1) 军哥问伽玛情况（紧急+影响大）(2) 翰锐增量请求（不紧急+影响中）(3) 月度排名报告（紧急+影响大）(4) 想优化分量分配公式（不紧急+影响大）。顺序：1→3→2→4。

### 第5步：把行动写成"谁在什么时候做什么"

**想什么**：拆完了如果还是"要跟进""要处理"，等于没拆。每个行动必须具体到人、时间、交付物。

**检查什么**：每个行动能不能回答这三个问题——谁做？什么时候完成？做完产出什么？

**判断什么**：

| 写法 | 合格吗 | 改成 |
|------|--------|------|
| "跟进毛毛虫排名" | 不合格 | "周三前查毛毛虫近4周排名变化，输出排名趋势表" |
| "和伽玛聊聊" | 不合格 | "本周五前和伽玛主管电话沟通，确认离职率升高的原因，记录沟通要点" |
| "更新管理制度" | 不合格 | "下周三前完成供应商准入流程章节的初稿，交付给王易人审阅" |

## 关键变量

| 变量 | 判断标准 | 作用 |
|------|---------|------|
| **事实完整度** | 关键数据是否已拿到？缺失几项？ | 决定能不能进入判断层 |
| **时间压力** | 有没有人在催？最晚什么时候要？ | 决定拆解深度——急的拆粗点，不急的拆细点 |
| **权限边界** | 最终行动在不在你的授权范围内？ | 决定是否需要报备或升级 |
| **影响范围** | 涉及1家供应商还是多条产线？ | 影响范围越大，拆解越要细致 |
| **信息来源可靠性** | 数据来自系统报表还是供应商自述？ | 决定行动前是否需要交叉验证 |

## 常见误区

| 误区 | 后果 | 正确做法 |
|------|------|---------|
| 拆完发现还是一团 | 说明还停留在"分类"没到"行动" | 每个子问题必须能转化成一个具体动作 |
| 想一步到位出完整方案 | 复杂问题不可能一步到位 | 先做最小可行动作，信息充足了再迭代 |
| 混淆事实和判断 | 把"我觉得XX供应商管理不行"当事实 | 先拿数据确认，再下判断 |
| 拆得太细失去重点 | 列了20个行动项，分不清主次 | 用紧急×影响矩阵排序，最多留5项 |
| 跳过拆解直接行动 | 看到问题就冲，方向可能错了 | 花5分钟拆解，省下几小时返工 |

## 决策输出

> **问题拆解表**

| 维度 | 内容 |
|------|------|
| 原始问题 | （一句话描述） |
| 事实层 | （数据、现状，标注"已确认"/"待查"） |
| 判断层 | （可能的原因，标注"可能性高"/"需验证"） |
| 行动清单 | （编号 + 具体动作 + 负责人 + 完成时间 + 交付物） |
| 权限确认 | （哪些自己做，哪些需要报备） |

> **示例：毛毛虫分量调整请求**

| # | 行动 | 负责人 | 时间 | 交付物 |
|---|------|--------|------|--------|
| 1 | 查毛毛虫当前产能利用率和近3月排名 | 年老师 | 今天 | 利用率数据+排名趋势 |
| 2 | 对比翰锐、岐力同类供应商的分量分配 | 年老师 | 今天 | 分量对比表 |
| 3 | 评估金条产线总量是否有调增空间 | 年老师 | 明天 | 调增可行性判断 |
| 4 | 如果有空间，出增量通知 | 年老师 | 周三 | 正式邮件 |
