# B2 新项目对接

> 同事提了新项目需求，怎么评估供应商匹配度、怎么分配、怎么跟踪——从需求到上线的一站式思考框架。

## 触发信号

- 刘伟佳（策略组）发消息："策略调整了，需要XX产线加XX席"
- 郭鑫催项目进度："那个XX项目什么时候能上？"
- 王易人转发消息："业务侧有个新需求，你看一下"
- 产线负责人口头提了新项目（金条/企金/信用卡/财富）
- 每日新项目寻源的1小时时间块到了

## 核心问题

> 这个新项目给谁做最合适？给的量能不能消化？上线时间能不能跟上？如果现有供应商都不匹配，需不需要引入新的？

## 思考步骤

### 第1步：需求澄清（5W2H）

**想什么**：同事说"有个新项目"，具体是什么？不问清楚就动手是浪费时间。

**检查什么**：
- 向需求方确认以下信息（不确认不动手）

> 5W2H需求澄清表

| 维度 | 问什么 | 为什么重要 |
|------|--------|-----------|
| **What** | 具体做什么业务？新产线/新话术/新产品？ | 决定匹配哪类供应商 |
| **Why** | 为什么有这个需求？业务扩张/策略调整/试水？ | 决定紧急程度和资源投入 |
| **Who** | 谁负责这个项目的业务侧对接？ | 后续有问题找谁 |
| **When** | 什么时候要上线？最晚什么时间？ | 决定排期和是否能引入新供应商 |
| **Where** | 有地域要求吗？必须本地/可远程？ | 决定供应商选择范围 |
| **How many** | 需要多少席？峰值多少？ | 决定一家还是多家分担 |
| **How much** | 预算/单价范围？ | 决定能匹配什么档次供应商 |

**判断什么**：
- 需求方说"尽快"但没有具体时间 → 追问"本周？本月？下月？"，逼出具体时间
- 需求方自己说不清楚 → 拉个15分钟电话聊清楚，不要来回发消息
- 需求量<50席 → 优先从现有供应商调配，不用引入新的
- 需求量>100席 + 紧急 → 现有供应商扩容+新供应商引入并行

**举例**：刘伟佳说"金条线策略调整，需要加200席"。追问：What=金条新话术、When=下月初、How many=200席。→ 这是大需求，需要评估现有供应商能不能消化。

### 第2步：现有供应商匹配度评估

**想什么**：先看现有供应商，能用熟的就用熟的。

**检查什么数据**：
- 最近一个月各供应商排名数据（每日0.5小时看的数据）
- 各供应商当前产能利用率（在岗人数/总工位）
- 各供应商的业务类型覆盖（做金条的、做企金的、做信用卡的）
- 各供应商近3个月的"配合度"得分（分量5维中的配合维度）

**判断什么**：

> 供应商匹配度评估矩阵

| 条件 | 匹配度 | 优先级 |
|------|--------|--------|
| 同产线经验 + 产能有余 + 排名前50% + 配合度高 | 高匹配 | 首选 |
| 同产线经验 + 产能有余 + 排名后50% | 中匹配 | 可以给，但需加强监控 |
| 不同产线但有意愿转型 + 管理能力OK | 待验证 | 需要培训期，适合小量试水 |
| 产能已满 或 近期合规问题频发 | 不匹配 | 不给 |

**举例**：金条线加200席需求。毅航（同产线、排名前3、老板说能加50人）→ 高匹配，首选给50席。毛毛虫（同产线、产能余量30人、配合度高）→ 高匹配，给30席。剩余120席现有供应商吃不下 → 需要引入新供应商或跨产线调配。

### 第3步：分量分配方案设计

**想什么**：怎么分才公平、有效、风险可控？

**检查什么数据**：
- 各供应商历史分量调整记录（`supplier-allocation.md` 中的分量调整记录）
- 分量5维打分表（业绩30%、质检客诉25%、上人15%、配合15%、战略15%）
- 各供应商老板的沟通反馈（谁想要、谁不想要）

**判断什么**：
- 分量分配原则：排名思维（看相对排名不看绝对分数），奖励头部、淘汰尾部
- 单家供应商不超过该产线总量的30% → 防止单点依赖
- 新给量不超过该供应商现有量的50% → 避免消化不良
- 至少2家供应商分担同一项目 → 风险分散

> 分量分配原则

| 供应商类型 | 分配策略 | 比例参考 |
|-----------|---------|---------|
| 头部（排名前30%） | 奖励性扩量 | 可给到新增量的40-50% |
| 中位（排名30-70%） | 正常扩量 | 可给到新增量的30-40% |
| 尾部（排名后30%） | 不给或收缩 | 不给新量，观察改善 |

**举例**：200席新增量分配——毅航（头部，加50席）、毛毛虫（头部，加30席）、伽玛（中位，加40席）、赛维斯（中位，加40席）、新引入供应商（40席试水）。5家分担，头部拿大头。

### 第4步：确认资源到位（人、系统、培训）

**想什么**：量分好了，供应商能不能真的接住？别上线了发现人没招到。

**检查什么**：
- 供应商确认：口头沟通OK → 需书面确认时间和人数；人员到位招聘进度和培训计划
- 系统接入：CRM账号数量、线路容量；培训安排：新话术+合规+上岗考核

**判断什么**：
- 供应商说"没问题"但没有具体计划 → 要求提交上人计划和时间表
- 人员到位时间<培训周期 → 需要压缩培训或分批上线
- 系统容量不够 → 提前协调技术团队扩容
- 供应商要求的培训支持超过现有资源 → 排期，错峰上线

**举例**：赛维斯说能加40人，但问具体上人计划说"正在招" → 要求2周内提交名册。如果2周后人没到位，这部分量转给毅航。

### 第5步：确定上线节奏和跟踪机制

**想什么**：不能一下子全部上线，要有节奏。

**检查什么数据**：
- 各供应商承诺的人员到位时间、培训周期（新话术通常3-5天）
- 系统接入周期（老供应商1-2天、新供应商5-7天）、历史类似项目上线经验

**判断什么**：

> 上线节奏参考

| 时间 | 动作 | 检查点 |
|------|------|--------|
| 第1周 | 头部供应商先上（毅航、毛毛虫） | 日均通话量、接通率 |
| 第2周 | 中位供应商跟上（伽玛、赛维斯） | 转化率、合规率 |
| 第3周 | 小量试水供应商上线 | 各项指标是否达到基线 |
| 第4周 | 全量运行 | 与需求方同步数据 |

- 跟踪频率：上线前2周每天看数据，之后每周看排名
- 异常阈值：转化率低于产线平均20%以上 → 立即介入
- 汇报对象：需求方（刘伟佳/郭鑫）+ 上级（王易人）

**举例**：金条新话术项目，第1周毅航50人先上，日均数据正常；第2周毛毛虫+伽玛跟进，整体数据稳定；第3周赛维斯上线后发现转化率偏低 → 单独约谈赛维斯项目负责人，安排话术强化培训。

### 第6步：需求方同步 + 内部协调

**想什么**：别闷头干活，需求方和上级要定期知道进展。

**检查什么**：
- 需求方（刘伟佳/郭鑫）期望的同步频率
- 王易人需要知道的节点（启动、上线、异常）
- 是否需要跨部门协调（法务合同、技术接入）

**判断什么**：
- 启动前：向需求方确认分配方案 → 用 `internal-coordination.md`（进度同步场景）
- 上线中：每周向需求方同步数据 → 用 `supplier-operations.md`（运营日报场景）
- 有异常：第一时间通知需求方和上级 → 用 `internal-coordination.md`（突发事件通报场景）
- 涉及新供应商引入 → 走 B1 准入流程

**举例**：分配方案出来后，给刘伟佳发进度同步邮件："金条新项目200席分配方案——毅航50席、毛毛虫30席、伽玛40席、赛维斯40席、新引入40席。第1周头部先上，预计第4周全量运行。"

## 关键变量

| 变量 | 判断标准 | 影响权重 |
|------|---------|---------|
| **需求紧急度** | 3天内=紧急用现有、1个月内=正常、不急=可引入新供应商 | 决定整体策略 |
| **现有供应商产能余量** | 总余量≥需求量=内部解决、<需求量=需引入新供应商 | 决定是否触发B1 |
| **需求量级** | <50席=单家解决、50-200席=多家分担、>200席=必须引入新供应商 | 决定分配复杂度 |
| **供应商配合度** | 高=优先给量、低=不给新量先改善 | 决定分配优先级 |
| **上线时间约束** | 紧急=老供应商优先、宽裕=可培训新供应商 | 决定上线节奏 |

## 常见误区

1. **不澄清需求就动手**："加200席"就马上分配——结果发现是换话术不是加人
2. **只给头部不给中位**：新增量全给毅航和毛毛虫，导致单点依赖
3. **忽视供应商消化能力**：给了量但人没到位，先确认上人计划再给量
4. **不设跟踪机制**：量分完就不管了——第1周就要开始看数据
5. **忘记同步需求方**：启动前、上线中、完成后三个节点必须同步
6. **跨产线调配不考虑经验差异**：不同产线的话术和合规要求差异大
