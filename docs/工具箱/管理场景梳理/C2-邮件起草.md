# C2 邮件起草

> 用最少的字把事情说清楚，让对方读完知道该做什么。

## 触发信号

- 需要向王易人或军哥汇报供应商情况
- 需要给供应商发正式通知（整改、约谈、增量等）
- 需要和跨部门同事协调某件事
- 口头沟通后需要书面留痕
- 定期报告（月报、周报）

## 核心问题

> 邮件的唯一目的：让收件人读完就知道"发生了什么 + 我该做什么 + 什么时候做"。除此之外的任何文字都是噪音。

## 思考步骤

### 第1步：确定邮件类型

**想什么**：这封邮件属于哪种类型？不同类型的结构完全不同。

**检查什么**：

| 类型 | 识别标准 | 核心诉求 |
|------|---------|---------|
| **向上汇报** | 收件人是王易人/军哥 | 让领导做决策或知情 |
| **对供应商通知** | 收件人是供应商老板/主管 | 告知事项+要求配合 |
| **跨部门协调** | 收件人是策略组/质检/其他部门 | 请求配合或同步信息 |
| **留痕确认** | 口头沟通后的书面确认 | 双方对齐事实 |
| **定期报告** | 月报/周报 | 呈现数据+发现+建议 |

**判断什么**：
- 如果一封邮件同时包含汇报+请求+通知 → 拆成多封，每封一个核心诉求
- 如果不确定该不该发邮件 → 问自己"不发邮件有什么风险"，没风险就别发

### 第2步：选对邮件结构模板

**想什么**：不同类型的邮件有不同的最佳结构，不要自由发挥。

**检查什么**：根据第1步确定的类型，选用对应结构。

> **类型A：向上汇报**

| 段落 | 内容 | 语气 |
|------|------|------|
| 标题 | [供应商名] + [事项] + [结论/状态] | 简洁，让领导扫一眼就知道主题 |
| 第1段：结论 | 一句话说清楚情况 + 你的判断 | 先说结论，别铺垫 |
| 第2段：关键数据 | 2-3个核心指标，带环比变化 | 用数字，不用"有所下降" |
| 第3段：原因分析 | 简要说明原因，标注哪些已确认哪些是推断 | 区分事实和推断 |
| 第4段：你的动作 | 你已经做了什么、正在做什么 | 展示你在跟 |
| 第5段：请求 | 需要领导做什么（审批/决策/知情） | 明确，别让领导猜 |

> 示例："标题：伽玛—3月排名降至第9位，建议启动约谈 → 结论先行（排名第9，首次后30%）→ 关键数据（GMV降22%）→ 原因（熟手流失，已确认）→ 你的动作（已做首次预警）→ 请求（知悉，如无改善建议月底约谈）"

> **类型B：对供应商通知**

| 段落 | 内容 | 语气 |
|------|------|------|
| 标题 | [明确事项] + [供应商名] | 不模糊，不客气 |
| 第1段：事项说明 | 发生了什么 / 我们决定什么 | 陈述事实 |
| 第2段：具体要求 | 需要对方做什么，写清楚标准 | 量化，不写"尽快改进" |
| 第3段：时间节点 | 截止日期 + 逾期后果 | 必须有 |
| 第4段：联系方式 | 有问题找谁 | 给一个明确的出口 |

> 示例："标题：整改通知—赛维斯3月质检扣分异常 → 事项说明（质检72分，低于85分）→ 具体要求（4月25日前提交整改方案）→ 时间节点+逾期后果（影响分量评估）→ 联系方式"

> **类型C：跨部门协调**

| 段落 | 内容 | 语气 |
|------|------|------|
| 标题 | [请求/同步] + [事项] | 直接说明来意 |
| 第1段：背景 | 为什么找你 | 一句话交代来龙去脉 |
| 第2段：请求 | 需要你做什么 | 具体，别含糊 |
| 第3段：时间 | 什么时候需要 | 给deadline |
| 第4段：附件/数据 | 相关参考资料 | 减少对方的信息负担 |

> 示例："标题：数据请求—毅航4月前两周出勤数据 → 背景（排名异常信号）→ 请求（按3条产线分别统计出勤率）→ 时间（本周五前）→ 附件（3月排名数据）"

> **类型D：口头沟通留痕**

| 段落 | 内容 | 语气 |
|------|------|------|
| 标题 | [沟通纪要] + [日期] + [对象] | 明确是纪要 |
| 第1段：背景 | 什么时候和谁沟通了什么 | 一句话 |
| 第2段：沟通要点 | 列出讨论的关键点 | 条目式，不长篇大论 |
| 第3段：双方约定 | 谁做什么、什么时候做 | 动作+责任人+时间 |
| 第4段：请确认 | "如有出入请回复指正" | 给对方一个纠错窗口 |

### 第3步：写完后做3项检查

**想什么**：邮件发出去就收不回来。花1分钟检查比发完追着解释强一百倍。

**检查什么**：

| 检查项 | 标准动作 | 常见错误 |
|--------|---------|---------|
| **数据准确性** | 核对关键数字是否有笔误 | "78%"写成了"87%"，意思完全反了 |
| **收件人/抄送** | 谁需要看到这封邮件？ | 忘了抄送王易人，或把不该看的人加进去了 |
| **语气匹配** | 语气是否和邮件类型匹配 | 向上汇报用"你看看行不行"（太随意）、对供应商用"我们觉得可能"（太弱） |

**判断什么**：
- 向上级邮件 → 能不能删掉30%的字？删完意思不变就删
- 对供应商邮件 → 有没有模糊措辞？"尽快""适当""可能"都删掉，换成具体数字或时间
- 跨部门邮件 → 对方看完知道该做什么吗？如果不确定，说明你的请求不够具体

### 第4步：确认附件和后续

**想什么**：邮件有没有漏附件？发完后要不要做后续动作？

**检查什么**：
- 提到"见附件"的，附件是否已添加
- 数据来源是否标注清楚（"数据截至X月X日"）
- 发完后是否需要微信/电话提醒对方看邮件

**判断什么**：
- 给领导的重要邮件 → 发完私聊一句"X总，邮件已发，请查阅"
- 给供应商的通知 → 如果紧急，发完电话确认"已发邮件，请查收"
- 跨部门请求 → 发完如果24小时没回复，催一次

## 关键变量

| 变量 | 判断标准 | 影响 |
|------|---------|------|
| **收件人层级** | 王易人（直属）/ 军哥（上上级）/ 供应商 / 同事 | 决定语气、详细度、是否需要王易人先审 |
| **紧急程度** | 紧急（48h内需反馈）/ 常规（1周内）/ 不急 | 决定邮件长度——紧急的写短点 |
| **敏感程度** | 涉及排名/处罚/清退 → 敏感 | 敏感邮件发前给王易人过目 |
| **是否需要留痕** | 口头已说清的 → 发邮件只是确认 | 留痕邮件要简洁，把口头结论固化即可 |
| **附件数量** | 超过3个附件 → 考虑是否可以合并 | 附件太多领导不看，精简到最必要的 |

## 常见误区

| 误区 | 后果 | 正确做法 |
|------|------|---------|
| 向上级邮件写成长篇报告 | 领导没时间看，重点被淹没 | 控制在300字以内，数据用表格 |
| 对供应商邮件语气太软 | 对方不当回事 | 通知类邮件用陈述句，要求和截止日期必须明确 |
| 一封邮件塞多个主题 | 收件人只处理了其中一个 | 一封邮件一个主题 |
| 忘记标注数据截止日期 | 数据被误以为是实时的 | 每次给数据都标注"截至X月X日" |
| 抄送太多人 | 信息扩散不可控 | 只抄送需要知情的人 |
| 发完不管 | 对方没看到或没处理 | 重要邮件发完后跟踪确认 |
