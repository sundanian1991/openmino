# D3-供应商来电来消息

> 供应商主动联系你要量、要支持、反映问题、投诉时，需要现场判断给出回应

## 触发信号
- 供应商老板微信："年老师，这个月量太少了，能不能加点？"
- 供应商主管电话："我们最近出勤有问题，想跟你汇报一下。"
- 供应商消息："XX项目名单有问题，很多无效。"
- 供应商投诉："客户投诉我们服务质量，要给我们解释。"

## 核心问题
- 供应商要什么？（增量/减量/支持/解释/诉苦）
- 这事我当场能决定吗？需要什么信息支撑？
- 这是真问题还是"撒娇"？（结合排名、异常信号判断）
- 我的回应口径是什么？（安抚/承诺/拒绝/要求补充材料）

## 30秒快速判断

**Step 1：识别诉求类型**
| 诉求类型 | 典型话术 | 我的应对策略 | 能否当场决定 |
|---------|---------|------------|------------|
| **要量** | "量太少了""别人量都很大" | 看5维打分 + 资源匹配度 | 通常需查数据 |
| **要支持** | "名单有问题""需要培训" | 区分甲方责任 vs 供应商自主 | 部分可当场承诺 |
| **反映问题** | "出勤下滑""离职多" | 听陈述 → 验证数据 → 要求整改 | 需了解具体情况 |
| **投诉/诉苦** | "客户针对我们""不公平" | 先安抚 → 核实 → 给出处理方案 | 需调查后回复 |
| **撒娇型** | 排名正常但抱怨量少 | 给予情绪价值 + 明确规则 | 可当场回应 |

**Step 2：查关键数据（2分钟内）**
- 该供应商当前排名？触发状态？（工具箱§二）
- 产能双维度：自身峰值比？头部差距？（工具箱§七）
- 近3个月趋势：上升/下降/波动？
- 本月分量是否已满？历史增减记录？

**Step 3：判断"真问题"还是"撒娇"**
```
真问题信号：
✓ 排名在后30% 或 首次/连续预警
✓ 自身峰值比<65% 或 头部差距<60%
✓ 周度突变：出勤<85% / 业绩断崖>20% / 流失>5%
✓ 主管胜任≤2项

撒娇信号：
✓ 排名在70%以上且稳定
✓ 自身峰值比>85% 且 头部差距>80%
✓ 无触发状态 + 无异常信号
✓ 历史已多次抱怨但数据正常
```

**Step 4：回应口径模板**
```
[供应商]，[共情/确认诉求]。我看了下数据：[关键数据呈现]。[判断结论]。我建议/要求：[具体动作]。
```
- **要量**："我理解你想增量。看数据你本月排名第XX（后30%），先整改达标，下月分量评估时优先考虑。"
- **要支持**："名单问题我核实下。如果是我们给的名单质量差，我们会调整；如果是你们转化能力问题，需要加强培训。"
- **反映问题**："你说的情况我记下了。我要求：[具体整改动作]，X月X日前反馈。"
- **投诉诉苦**："我理解你的感受。我会核实情况，X天内给你答复。"

## 5分钟深入版（如有时间）

**Step 1：定位问题根因（工具箱§十六）**
- 业绩问题 → 用产能双维度区分：自己问题（峰值比<65%） vs 产线问题（头部差距<60%）
- 出勤问题 → 周度扫描信号 + 主管胜任评估
- 客诉问题 → 合规定级（工具箱§五）
- 名单问题 → 判断是甲方责任（名单质量差）还是供应商能力问题

**Step 2：评估资源匹配度（工具箱§十九）**
```
资源分配5维打分：
业绩（30%）：当前排名/趋势
质检客诉（25%）：近3月客诉率/A类次数
上人（15%）：人员稳定性/出勤率
配合（15%）：响应质量/整改执行力
战略（15%）：标杆/战略协同价值
```
- 总分≥80分 → 可考虑增量
- 总分60-80分 → 先整改后评估
- 总分<60分 → 暂不考虑增量，要求整改

**Step 3：明确回应边界**
- 我能当场决定：安抚话术、明确规则、承诺核实
- 我需要查证：数据核实、历史记录、上级意见
- 我需要升级：涉及清退/换人/重大处罚，需军哥/易人哥确认（工具箱§十九决策授权矩阵）

**Step 4：设定预期与跟进**
- 告知下一步："我今天核实后X点前回复你"
- 明确动作："你需要在X月X日前提交整改计划"
- 记录留痕：在供应商空间（工具箱§十八）或沟通记录中存档

## 关键变量
- **供应商等级**：头部（S/A） vs 中游（B） vs 尾部（C/D） — 回应语气和资源倾斜不同
- **历史记录**：是否有诚信问题？是否连续预警？
- **当前触发状态**：首次预警/连续预警/A级骤降 → 回应需更严肃
- **资源饱和度**：该供应商是否已满量？是否有减量空间？
- **甲方责任比例**：名单/产品/系统问题 vs 供应商自主问题（工具箱§十九）

## 常见误区
- ❌ 当场承诺增量：未查5维打分和资源匹配度就答应
- ❌ 被情绪带偏：供应商诉苦就心软，忽略数据事实
- ❌ 模糊回应："会考虑""再看看" — 需明确"可以/不可以+原因+条件"
- ❌ 不记录：口头沟通后无留痕，后续无法追溯
- ❌ 混淆撒娇与真问题：排名正常的抱怨vs排名异常的求助

## 决策输出
**回应矩阵**：

| 供应商诉求 | 数据判断 | 回应口径 | 下一步动作 |
|-----------|---------|---------|-----------|
| **要量** | 排名前70% + 5维打分≥80 | "达标可以增量，下月分量评估时优先" | 记录到分量规划表 |
| | 排名后30% 或 <60分 | "先整改达标，达标后再评估增量" | 触发【A2-约谈准备】 |
| **要支持** | 甲方责任（名单/产品问题） | "问题我们认，会调整" | 内部同步问题给责任方 |
| | 供应商能力问题 | "需要你们自己提升，我可以推荐标杆" | 触发【A9-HRBP式扶持】 |
| **反映问题** | 属实 + 可整改 | "限时X天内整改，提交计划" | 触发【A7-整改计划评估】 |
| | 不属实 / 借口 | "数据不支持你的说法，请客观对待" | 记录到沟通档案 |
| **投诉诉苦** | 属实 | "问题我们认，[处理方案]" | 触发【A5-合规问题定级】 |
| | 不属实 | "数据不支持，请理性看待" | 记录到诚信档案 |

**现场回应话术示例**：
```
"我理解你的难处。看数据你本月排名第28（后30%），先整改达标，下月分量评估时我会优先考虑达标供应商。"
```
```
"名单问题我核实。如果是我们名单质量差，我们调整；如果是你们转化能力问题，需要你们自己提升。我周三前给你答复。"
```
```
"出勤问题我已记录。你需要在明天12点前提交书面说明和整改计划，我会评估后回复。"
```
