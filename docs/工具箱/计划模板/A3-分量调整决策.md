# A3-分量调整决策

> 月度分量调整或临时增量申请时，系统化评估与决策

## 触发信号
- 月度分量评估：每月初根据上月表现调整下月份额
- 新项目资源分配：新项目需要分配供应商和资源
- 临时增量申请：供应商临时申请增加人手或份额
- 减量需求：某供应商表现不佳需压缩份额

## 核心问题
- 调整的依据是什么？（5维打分 + 资源匹配度）
- 增/减量的幅度如何确定？
- 调整后对整体集中度的影响？（TOP3占比是否超标）
- 需要哪些数据支撑？决策记录怎么写？

## 思考步骤

**Step 1：明确调整类型与背景**
```
调整类型：
□ 月度常规调整：每月初根据上月表现分配下月份额
□ 新项目分配：新项目需要供应商承接
□ 临时增量：供应商临时申请加人/加量
□ 紧急减量：供应商出现重大问题需压缩份额

资源总量：本次可调整的总量是多少？
申请方数：几家供应商在申请？期望分配原则？
约束条件：单一供应商≤40%（工具箱§十九集中度检查）
```

**Step 2：5维打分评估（提效方案分量5维 + 工具箱§十九）**
```
维度与权重：
1. 业绩（30%）：近3月排名/趋势/GMV达成率
2. 质检客诉（25%）：近3月客诉率/A类次数/整改执行
3. 上人（15%）：出勤稳定度/流失率/人效
4. 配合（15%）：响应质量/整改执行力/日常配合度
5. 战略（15%）：标杆价值/战略协同/集中度平衡

打分规则（每维度0-100分）：
□ 业绩：排名前30%=90-100分，30-70%=70-89分，后30%=0-69分
□ 质检：无A类=100分，1次A类=70分，≥2次=40分
□ 上人：出勤≥90%+流失<8%=100分，一项不达标=70分，两项=40分
□ 配合：整改完成率≥70%=100分，50-70%=70分，<50%=40分
□ 战略：标杆型=100分，尽力型=70分，问题型=40分，尾部特殊=单独评估

加权总分 = 业绩×30% + 质检×25% + 上人×15% + 配合×15% + 战略×15%
```

**Step 3：资源匹配度验证（工具箱§十九）**
```
匹配度三问：
1. 头部差距变化：本月头部差距 vs 上月，是扩大还是缩小？
   - 扩大 → 可能资源不足，需警惕
   - 缩小 → 可考虑加量

2. 总量 vs 熟手人均：总人数变化>20%时
   - 总量涨但人均降 → 消化能力不足，暂停加量
   - 总量涨且人均涨 → 消化能力好，可继续加量

3. 集中度检查：引入后TOP3占比
   - 单一供应商≤40% ✓
   - 若超标 → 需调整分配策略，避免过度集中
```

**Step 4：增量/减量幅度决策**
```
增量规则：
总分≥80分：可增量，幅度10-20%
总分60-79分：保持或微调（±5%），需整改后复评
总分<60分：不增量，触发整改

减量规则：
连续预警/诚信问题：减量20-30%
A级骤降未恢复：减量15-20%
排名后30%且无改善：减量10-15%

特殊情形：
尾部供应商有特殊价值：增量但需标注"特殊审批"
头部供应商（S级）：保持或微增，避免过度集中
```

**Step 5：决策记录与输出**
```
决策记录模板（supplier-selection-framework.md）：
决策编号：RES-YYYYMMDD-序号
决策类型：[月度分量 / 新项目 / 临时增量 / 紧急减量]
分配背景：[资源总量 / 申请方数 / 时间要求]

评分结果（5维）：
- 业绩：[XX分，排名/趋势说明]
- 质检：[XX分，客诉情况]
- 上人：[XX分，出勤/流失数据]
- 配合：[XX分，整改执行率]
- 战略：[XX分，标杆价值]
- 总分：[XXX分]

分配结论：
□ 增量：[供应商名] +XX%，理由：[高分+资源匹配度好]
□ 减量：[供应商名] -XX%，理由：[触发预警/配合度低]
□ 保持：[供应商名] 不变，理由：[需观察/整改中]

集中度检查：调整后TOP3占比[YY%] ✓/✗

下一步：
□ 发出分量通知（allocation-adjustment-notice.md）
□ 要求整改（连续预警触发）
□ 跟踪调整后效果（每周看人效趋势）
□ 需上级确认：[是/否，如需需王易人确认]
```

## 关键变量
- **5维总分**：≥80分可增量，60-79分保持，<60分减量
- **资源匹配度**：头部差距是否缩小？总量涨是否伴随人均涨？
- **集中度约束**：单一供应商≤40%，调整后需验证
- **触发状态**：连续预警/诚信问题必须减量
- **战略价值**：标杆型/尾部特殊需单独权衡

## 常见误区
- ❌ 只看业绩排名：需综合5维，质检/配合权重更高（25%+15%）
- ❌ 忽略集中度：增量后TOP3占比可能超标
- ❌ 加量不跟踪：总量涨但人均降=消化能力不足，需暂停加量
- ❌ 尾部随意选：尾部供应商必须有特殊理由+风险预案+上级审批
- ❌ 不记录决策：每次分量调整必须留决策记录（审计要求）

## 决策输出

**分量调整通知（给供应商）**：
```
[供应商名]，你好。
月度分量评估结果：
上月份额：[XX%]
本月份额：[YY%]（[↑/↓/→]ZZ%）
调整理由：[基于5维打分，你的[业绩/配合/战略]维度表现[优秀/需改进]）
提升目标：[下月需达成的具体目标]
请确认接收。
```

**分量规划表（内部记录）**：
```
| 供应商 | 上月份额 | 本月份额 | 变化 | 5维总分 | 排名 | 触发状态 | 头部差距变化 | 决策依据 |
|--------|---------|---------|------|--------|------|---------|------------|---------|
| 毅航   | 25%     | 28%     | +3%  | 85     | 5    | 正常     | ↑5%        | 业绩稳+配合好 |
| 赛维斯 | 18%     | 15%     | -3%  | 58     | 28   | 连续预警 | ↓8%        | 连续后30%减量 |
| 翰锐   | 12%     | 12%     | →    | 72     | 15   | 正常     | →          | 保持观察 |
```

## 下一步衔接
- 发出分量通知 → `supplier-allocation.md`（增量通知/减量通知）
- 触发预警 → 调用【A1-业绩异常判断】→ 【A2-约谈准备】

- 调整后跟踪 → 每周查看该供应商人效趋势，验证消化能力
