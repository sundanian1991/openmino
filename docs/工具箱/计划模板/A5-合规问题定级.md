# A5-合规问题定级

> 收到客诉、违规、红线行为时，快速定级并启动对应处理流程

## 触发信号
- 客诉：客户投诉供应商服务质量
- 违规：招聘违规（用京东logo）、信息安全违规（外网管控）、质检A类
- 红线行为：G项红色（负面清单）
- 数据异常：数据质量分≤70或发现造假嫌疑

## 核心问题
- 这个问题的严重等级是什么？（一般/严重/红线）
- 涉及哪些合规维度？（服务质量/信息安全/招聘合规/数据真实）
- 应该启动什么处理流程？（整改/警告/约谈高层/清退）
- 需要哪些人参与？（供应商主管/老板/军哥）
- 后续如何跟踪验证？

## 思考步骤

**Step 1：问题分类与定级（工具箱§五 + §十六）**
```
合规问题三级分类：

🔴 红线级（立即升级）：
- G项红色：负面清单触发项（如：欺诈、重大泄密、群体事件）
- 诚信红线：数据造假≥2项 或 同一项连续2次不实
- 重大信息安全事件：外网管控<70%且涉及敏感数据
- 重大客诉：造成实际损失或负面舆情
→ 动作：立即约谈高层 + 军哥知情 + 启动清退评估

🟡 严重级（48h内处理）：
- 质检A类：合规违规（具体看A类哪一项）
- 诚信预警：数据质量分≤70或1项不实
- 外网管控<70%（未涉及敏感数据）
- 招聘违规（用京东logo等）
→ 动作：发整改通知 + 48h内反馈 + 加倍核查3个月

🟢 一般级（72h内处理）：
- 客诉（服务质量问题，未造成损失）
- 轻微违规（流程不规范但无实质影响）
- 数据异常但未核实
→ 动作：要求说明 + 限期整改 + 跟踪验证
```

**Step 2：定位问题根因（工具箱§十六）**
```
按问题类型匹配根因：

质检A类：
□ 服务质量不达标（响应慢/态度差）
□ 流程执行不规范（未按SOP操作）
□ 技能不足（培训不到位）
→ 动作：整改通知 + 培训要求

招聘违规：
□ 使用竞品logo（京东/银行等）
□ 虚假承诺（承诺未提供的福利）
□ 虚假宣传（夸大产品功能）
→ 动作：立即整改 + 警告 + 诚信记录

信息安全：
□ 外网管控不严（手机/私人电脑接入）
□ 监控缺失/遮挡
□ 数据泄露风险
→ 动作：立即整改 + 安全培训 + 加倍核查

数据异常：
□ 出勤平滑（连续4周≥98%可疑）
□ 离职为零（30人以上团队）
□ GMV平稳（日波动<5%）
□ 熟手突增无入职记录
→ 动作：启动专项核查 + 诚信评估
```

**Step 3：决定处理方式与参与人**
```
处理矩阵：

问题类型   │ 定级   │ 处理方式              │ 参与人         │ 时效
──────────┼────────┼─────────────────────┼──────────────┼─────
客诉       │ 一般   │ 要求书面说明 + 整改   │ 主管           │ 72h
质检A类    │ 严重   │ 整改通知 + 48h反馈    │ 主管+老板      │ 48h
招聘违规   │ 严重   │ 警告 + 立即整改       │ 主管+老板      │ 24h
信息安全   │ 严重   │ 立即整改 + 安全培训   │ 主管+老板      │ 24h
诚信预警   │ 严重   │ 警告 + 标记3个月加倍核查 │ 主管+老板    │ 24h
诚信红线   │ 红线   │ 约谈高层 + 告知后果   │ 高层+军哥      │ 立即
G项红色    │ 红线   │ 紧急处理 + 升级上报   │ 高层+军哥      │ 立即
```
- 红线级：需军哥知情或参与
- 严重级：48h内必须反馈整改计划
- 一般级：72h内完成说明与整改

**Step 4：验证标准与跟踪（工具箱§六）**
```
整改验证4维度：
1. 目标达成：整改目标量化，达成≥80%
2. 动作执行：承诺动作按期完成≥70%
3. 持续性：观察整改后2个月数据
4. 根因消除：原失分项提升≥1个等级

验证结果处理：
✓ 全部通过 → 取消预警，归档
✓ 部分通过 → 延长观察1个月
✗ 未通过 → 升级：连续2次未通过→进入清退评估
✗ 造假嫌疑 → 直接进入清退评估，记入诚信档案
```

**Step 5：数据质量分计算（工具箱§十九）**
```
月度检查5项信号：
1. 出勤率连续4周≥98%（过于平滑）→ 预警
2. 离职率连续3月=0（30人以上团队极少）→ 预警
3. GMV日度波动<5%（过于平稳）→ 关注
4. 熟手人数突增无新入职记录 → 预警
5. 考核节点前后突变（月末/季末数据异常优化）→ 危险

判定：
0项触发 = 可信（100分）
1项触发 = 存疑（70分），标注具体项
≥2项触发 = 不可信（40分），排名加注星号，不作为奖惩依据

处理：
100分 → 正常
70分 → 标记"存疑"，下次巡检重点核查
40分 → 标记"不可信"，排名加★，不参与奖惩评估
```

## 关键变量
- **问题性质**：信息安全/诚信问题 > 服务质量 > 流程规范
- **历史记录**：是否初犯？是否有多次违规记录？
- **整改态度**：是否主动承认？整改计划是否具体？
- **影响范围**：是否造成实际损失？是否涉及客户投诉？
- **上级要求**：军哥/易人哥是否有特别指示？

## 常见误区
- ❌ 定级过松：客诉不重视，导致重复发生
- ❌ 整改无标准：只说"加强管理"，不量化目标
- ❌ 不跟踪闭环：收整改计划后不验证效果
- ❌ 造假不升级：发现数据问题不记录诚信档案
- ❌ 不集中度检查：单一供应商问题频发未考虑分散风险

1. 48小时内提交书面说明和整改计划
