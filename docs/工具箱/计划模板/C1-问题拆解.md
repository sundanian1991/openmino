# C1-问题拆解

> 遇到任何想不清楚、说不明白的问题时，用结构化框架拆解到可执行

## 触发信号
- "这个问题很复杂，我不知道从哪入手"
- "有几个事混在一起，理不清"
- "需要做个决定但信息不全"
- "要写邮件/报告但思路混乱"

## 核心问题
- 这个问题的本质是什么？（现象还是根因？）
- 这个问题涉及哪些方面？（MECE 拆解）
- 哪些是已知？哪些是未知？
- 拆解到哪一层可以行动？（叶子节点）
- 需要谁参与决策？

## 思考步骤

**Step 1：问题陈述（一句话说清）**
```
错误示范："供应商管理有问题"
正确示范："毅航连续2个月后30%，我需要决定是约谈整改还是启动清退"

5W2H模板：
What：什么问题？（[具体现象]）
Why：为什么是问题？（[影响/后果]）
Who：涉及谁？（[供应商/同事/领导]）
When：何时发生/何时需解决？（[时间要求]）
Where：在哪个环节/业务线？（[业务线/站点]）
How much：量化影响？（[金额/排名/比例]）
How：怎么解决？（[我的初步想法]）
```
**产出**：1句话问题陈述 + 5W2H表格

**Step 2：MECE 拆解（相互独立，完全穷尽）**
```
拆解维度选择（根据问题类型）：

类型1：决策类（"要不要做？"）
├─ 目标：达成什么？（量化目标）
├─ 选项：有哪几种选择？（≥2个）
├─ 标准：按什么判断？（权重/评分）
├─ 数据：各选项的数据支撑？
└─ 风险：每个选项的风险？

类型2：归因类（"为什么？"）
├─ 内部因素：供应商自身（管理/人员/流程）
├─ 外部因素：甲方因素（名单/产品/系统）
├─ 市场因素：行业/季节/竞争
└─ 偶然因素：一次性事件（系统故障/人员离职）

类型3：规划类（"怎么做？"）
├─ 阶段：分几个阶段？（T+X时间轴）
├─ 动作：每个阶段做什么？（MECE拆解到可执行）
├─ 资源：需要什么资源？（人/钱/系统）
├─ 风险：每个阶段的风险与预案
└─ 验收：怎么算完成？（量化标准）

类型4：信息类（"是什么？"）
├─ 定义：这个概念/事件的准确定义
├─ 边界：包含什么/不包含什么
├─ 构成：组成部分/要素
├─ 标准：衡量标准/判断依据
└─ 案例：典型示例/历史案例
```

**Step 3：识别已知与未知（信息缺口分析）**
```
信息四象限：

已知已知（Know Knowns）：
✓ 我已掌握的信息：[列出]
✓ 数据来源：[系统/记录/沟通]

已知未知（Known Unknowns）：
? 我需要查的数据：[列出]
? 数据来源：[系统A/供应商填报/巡检记录]
? 获取方式：[导出/索要/现场核实]

未知已知（Unknown Knowns）：
⚠ 我可能知道但没意识到的：
  - 历史类似案例的处理方式
  - 相关方的潜在态度
  - 规则中隐含的限制条件

未知未知（Unknown Unknowns）：
❓ 完全没考虑到的风险：
  - 供应商可能的反制措施
  - 连锁反应（影响其他供应商）
  - 规则盲区

行动：优先填补"已知未知"，明确数据获取路径
```

**Step 4：拆解到可执行单元（叶子节点）**
```
可执行单元标准：
□ 单一动作（不是"加强管理"，是"约谈主管并要求提交计划"）
□ 可验证（有完成标志：文档/记录/系统状态）
□ 有负责人（谁做？）
□ 有时间点（何时完成？）

示例：拆解"毅航业绩下滑"
错误拆解：
- 人员问题（太模糊）
- 管理问题（太模糊）
- 名单问题（未验证）

正确拆解（MECE）：
1. 人员结构问题（工具箱§七：新人占比>30%？）
   → 执行：导出人力表，计算新人占比（负责人：年老师，截止：今天）
2. 出勤问题（工具箱§十六：周度出勤<85%？）
   → 执行：查周度出勤数据（负责人：年老师，截止：今天）
3. 名单问题（工具箱§十九：甲方责任场景？）
   → 执行：核对名单质量数据（负责人：年老师，截止：明天）
4. 主管能力（工具箱§八：胜任≤2项？）
   → 执行：计算主管胜任5项（负责人：年老师，截止：明天）

每个叶子节点都是独立可执行的动作
```

**Step 5：制定执行路线图**
```
优先级排序（ Eisenhower Matrix ）：
┌─────────────┬──────────────┬──────────────┐
│ 紧急重要    │ 立即执行     │ 本周P0       │
├─────────────┼──────────────┼──────────────┤
│ 重要不紧急  │ 计划执行     │ 本月P1       │
├─────────────┼──────────────┼──────────────┤
│ 紧急不重要  │ 委托/简化    │ 本周P2       │
├─────────────┼──────────────┼──────────────┤
│ 不重要不紧急│ 暂缓/放弃    │ 不做          │
└─────────────┴──────────────┴──────────────┘

路线图模板：
P0（本周必须）：
1. [动作A] → 负责人：年老师 → 完成时间：[日期] → 交付物：[XX报告]
2. [动作B] → 负责人：年老师 → 完成时间：[日期] → 交付物：[XX通知]

P1（本月重点）：
3. [动作C] → 负责人：年老师 → 完成时间：[日期] → 交付物：[XX决策]
4. [动作D] → 负责人：[同事] → 完成时间：[日期] → 交付物：[需配合事项]

P2（可委托）：
5. [动作E] → 负责人：[供应商/同事] → 完成时间：[日期] → 交付物：[自查报告]

不做：
6. [原计划F] → 原因：[信息不足/优先级低]
```

## 关键变量
- **问题类型**：决策/归因/规划/信息，决定拆解维度
- **信息完整度**：已知已知多则可直接决策，未知多则需先调研
- **时间压力**：紧急问题（24h）优先P0，不紧急可系统调研
- **决策层级**：年老师自主 / 需易人哥 / 需军哥
- **资源可用**：是否有数据/人力/系统支持

## 常见误区
- ❌ 停留在表面：只解决现象不挖根因
- ❌ 拆解不MECE：重叠或遗漏，逻辑混乱
- ❌ 过度拆解：拆到10层仍不可执行，应停在可行动节点
- ❌ 忽略信息缺口：不知道缺什么，乱做决策
- ❌ 无优先级：所有事都重要，实际都没做
- ❌ 不设定时间点：只有动作没有截止时间

## 决策输出

**问题拆解卡**：
```
问题：[一句话陈述]
类型：[决策/归因/规划/信息]
拆解维度：[选择的拆解框架，如：MECE三要素]
已知已知：[我已经知道什么]
已知未知：[我需要查什么（3-5项）]
叶子节点（可执行动作）：
1. [动作A]（负责人/截止时间/交付物）
2. [动作B]（负责人/截止时间/交付物）
3. [动作C]（负责人/截止时间/交付物）
优先级：P0[X个] / P1[X个] / P2[X个]
决策点：[在什么信息完备后做决策]
下一步：立即执行[动作A]，同步收集[已知未知]
```

**MECE 拆解示例**（问题：连续预警供应商不整改）
```
问题：连续2个月后30%的供应商，约谈后整改计划不合格，怎么办？
拆解维度：归因类（内部/外部/市场/偶然）

内部因素（供应商自身）：
1. 主管能力问题（工具箱§八：胜任≤2项？）
   → 动作：计算主管胜任5项（今天）
2. 人员结构问题（工具箱§七：新人占比>30%？）
   → 动作：导出人力表计算新人占比（今天）
3. 整改意愿问题（工具箱§六：整改执行<50%？）
   → 动作：查历史整改记录（明天）

外部因素（甲方责任）：
4. 名单质量问题（工具箱§十九：甲方责任场景？）
   → 动作：核对名单转化率数据（明天）
5. 产品/系统问题（业务方反馈？）
   → 动作：问业务方是否有产品变更（明天）

市场因素：
6. 行业整体下行？（同行都在下滑？）
   → 动作：查行业数据（如有）

偶然因素：
7. 一次性事件（系统故障/人员离职潮？）
   → 动作：核实具体原因（约谈时问）

叶子节点（前3项今天完成）→ 数据齐全后调用【A6-清退评估】
```

## 下一步衔接
- 决策类问题 → 拆解后调用对应决策模板（A1-A6 / B1-B6）
- 归因类问题 → 拆解后调用【A1-业绩异常判断】定位根因
- 规划类问题 → 拆解后按B类模板执行
- 信息类问题 → 拆解后按C2/C3整理输出
- 需要写公文 → 调用【C2-邮件起草】
