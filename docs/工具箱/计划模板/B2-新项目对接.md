# B2-新项目对接

> 同事提新项目需求时，快速评估可行性并启动寻源流程

## 触发信号
- 刘伟佳："有个新项目需要供应商，你帮忙看下。"
- 业务方："我们下个月有个新活动，需要XX人支持。"
- 临时需求："客户临时加单，需要快速找供应商承接。"

## 核心问题
- 需求真实吗？是临时活动还是长期项目？
- 需要多少人？何时上线？预算多少？
- 现有供应商谁能接？还是需要寻新？
- 如果寻新，走标准SOP还是紧急流程？
- 风险是什么？（集中度/时效/质量）

## 思考步骤

**Step 1：需求澄清（5W2H）**
```
必须问清楚的7个问题：
1. What（什么项目）：
   - 项目名称：[XXX]
   - 业务线：[金条/企金/信用卡/财富]
   - 项目类型：[长期/临时/试点]

2. Why（为什么做）：
   - 业务目标：[提升GMV/测试新产品/应对旺季]
   - 不可替代性：必须新供应商吗？现有能否扩容？

3. Who（谁负责）：
   - 需求方：[部门/同事名]
   - 业务负责人：[对接人]
   - 决策人：[军哥/易人哥]

4. When（时间要求）：
   - 最晚上线日：[YYYY-MM-DD]
   - 期望开始日：[YYYY-MM-DD]
   - 紧急程度：[高/中/低]

5. Where（场地/站点）：
   - 需要站点：[城市/具体地址]
   - 场地是否已就绪？

6. How much（资源需求）：
   - 需要人数：[XX人]
   - 人均成本上限：[XXX元/人/月]
   - 总预算：[XXX万元/月]

7. How（如何交付）：
   - 交付标准：[KPI要求]
   - 验收方式：[数据/客户反馈]
```
**产出**：需求确认邮件（发需求方+抄送上级）

**Step 2：现有供应商快速扫描**
```
3分钟排查：
□ 现有供应商是否有空余名额？
  - 头部供应商（S/A）：通常已满，可问能否扩编
  - 中部供应商（B）：常有空余名额，优先考虑
  - 尾部供应商（C/D）：有产能但质量需评估

□ 同类项目经验：
  - 哪家供应商做过类似项目？
  - 历史交付效果如何？（工具箱§十七历史记录）

□ 资源匹配度（调用【A3-分量调整决策】）：
  - 现有供应商5维打分：谁分数高且资源可用？
  - 头部差距变化：谁在持续改善？
```
- 有≥1家现有供应商可承接 → 进入Step 3（内部协调）
- 无现有供应商可承接 → 进入Step 4（新寻源）

**Step 3：内部协调方案（如现有供应商可承接）**
```
协调方案：
供应商：[毅航/毛毛虫/...]
资源：从[现有项目]调剂[XX]个名额
时间：T+[X]天调配
成本：[增加/减少]预算
风险：影响原项目[具体说明]

输出：
□ 协调方案邮件（需求方+供应商+易人哥）
□ 分量调整通知（调用【A3-分量调整决策】）
□ 内部系统配置变更（IT/运营）
```
**前提**：供应商资源可用 + 原项目同意调剂

**Step 4：新寻源启动（调用【B1-新供应商准入】）**
```
若需寻新供应商：
1. 需求说明书已确认（Step 1产出）
2. 启动9维度评估（B1-Step 3）
3. 候选≥3家（工具箱§十：避免单一依赖）
4. SOP流程：标准T+15 或 紧急流程T+10

若时效紧急（<10天）：
□ 加速阶段3寻源（并行触达）
□ 压缩阶段7内部操作（预配置）
□ 试运行可适当缩短（但≥2周）
□ 记录加速原因（审计要求）
```

**Step 5：风险识别与预案**
```
风险矩阵：
风险项           │ 概率 │ 影响 │ 应对方案
─────────────────┼──────┼──────┼─────────────
供应商不达标     │ 中   │ 高   │ 严格9维度评估 + 试运行
上线延期         │ 中   │ 中   │ 预留缓冲时间（T+15而非T+10）
成本超预算       │ 低   │ 中   │ 人均成本锁死+增量部分重新谈判
业务中断         │ 低   │ 高   │ 备选供应商预案 + 切换计划
集中度过高       │ 中   │ 中   │ 单一≤40%，至少2家供应商
```
**输出**：风险评估表（附件到需求确认邮件）

**Step 6：决策与审批**
```
决策路径：
□ 小额/常规项目（<50万/月）：年老师自主决策
□ 中等项目（50-200万/月）：需易人哥审批
□ 大额/战略项目（>200万/月）：需军哥审批

决策材料：
- 需求说明书
- 供应商评估报告（9维度打分）
- 风险评估
- 资源规划（分量调整/新增预算）
- SOP时间表（T+15计划）
```

## 关键变量
- **项目紧急度**：标准T+15，紧急可压缩但需记录原因
- **预算规模**：决定审批层级（年老师/易人哥/军哥）
- **现有资源**：优先内部协调，减少寻源成本
- **业务线匹配**：金条供应商不能直接用于信用卡项目
- **集中度约束**：单一供应商≤40%（工具箱§十九）

## 常见误区
- ❌ 需求不清晰就启动：必须5W2H确认后再评估
- ❌ 跳过现有供应商扫描：优先内部协调，再寻新源
- ❌ 候选<3家：对比不充分，决策风险高
- ❌ 忽略时效缓冲：T+15标准周期，紧急项目压缩需记录原因
- ❌ 不评估集中度：新供应商引入后TOP3占比可能超标
- ❌ 无风险预案：供应商不达标/上线延期无应对方案

**需求确认模板**：
```
项目需求确认
项目名称：[XXX]
业务线：[金条/企金/信用卡/财富]
需求方：[部门/同事名]
需求内容：[XX人，YY业务，ZZ月上线]
预算：XXX万元/月
时间要求：YYYY-MM-DD前上线
决策：□ 内部协调 [供应商名] 承接
       □ 启动新寻源（9维度评估）
       □ 暂缓，原因：[具体原因]
```

**新项目对接清单**：
```
□ Step 1：需求5W2H已确认（邮件发出）
□ Step 2：现有供应商扫描完成（X家可承接）
□ Step 3：决策路径确定（审批人：XXX）
□ Step 4：若需寻新 → 启动B1新供应商准入流程
□ Step 5：风险预案已制定（[风险项X个]）
□ Step 6：SOP时间表（T+15计划已输出）
```
