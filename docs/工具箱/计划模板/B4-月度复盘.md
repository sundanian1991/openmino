# B4-月度复盘

> 每月初系统化复盘上月排名，识别重点供应商和行动项

## 触发信号
- 每月初系统导出上月排名数据后
- 月度管理会议前（准备汇报材料）
- 月度分量决策前（评估各供应商表现）

## 核心问题
- 整体格局是什么？（S/A/B/C/D分布）
- 哪些供应商需要重点关注？（首次预警/连续预警/A级骤降）
- 趋势线有哪些？（连续上升/下降的供应商）
- 新供应商试运行是否达标？
- 整改闭环率是多少？
- 下月重点动作清单是什么？

## 思考步骤

**Step 1：整体格局分析（工具箱§三）**
```
分布统计：
S级（前20%）：X家  [列出名单]
A级（20-40%）：X家  [列出名单]
B级（40-70%）：X家  [列出名单]
C级（70-80%）：X家  [列出名单]
D级（后30%）：X家  [列出名单]

层级失衡判断：
□ C级太多（>30%）→ 淘汰不够快，需加快清退节奏
□ D级太多（>35%）→ 尾部供应商过多，需加速清理
□ S级太少（<15%）→ 标杆不够，需重点培养头部
□ A→S晋升率：[X%]  B→A晋升率：[X%]  （流动性是否健康？）

环比变化：
上月S级：[X家] → 本月S级：[Y家] （↑/↓Z家）
上月D级：[X家] → 本月D级：[Y家] （↑/↓Z家）
```
**输出**：分布饼图/柱状图 + 一句话结论（如："S级减少2家，D级增加3家，层级失衡加剧"）

**Step 2：重点供应商标记（工具箱§一 + §二）**
```
触发状态分类：

🔴 立即行动（A级骤降 + 诚信红线）：
1. [供应商A]：上月前30% → 本月后30%（A级骤降）
   根因：[快速查双维度+周度信号]
   动作：24-48h内约谈

🟡 本周行动（首次预警 + 连续预警）：
2. [供应商B]：首次预警（本月首次后30%）
   动作：3-5天内私下沟通
3. [供应商C]：连续预警（连续2个月后30%）
   动作：本周正式约谈+整改计划

🟢 观察清单（隐性退步 + 主管胜任≤2项）：
4. [供应商D]：排名升但人效降（隐性退步）
   动作：关注，不立即干预
5. [供应商E]：主管胜任≤2项
   动作：下次约谈时加入管理问题

重点标记不超过5家（聚焦Top问题）
```

**Step 3：趋势线识别（工具箱§三 + §七）**
```
上升最快（Top 3）：
1. [供应商A]：排名上升[XX名]，原因[具体说明]
2. [供应商B]：排名上升[XX名]，原因[具体说明]
3. [供应商C]：排名上升[XX名]，原因[具体说明]

下降最多（Top 3）：
1. [供应商X]：排名下降[XX名]，根因[双维度分析]
2. [供应商Y]：排名下降[XX名]，根因[双维度分析]
3. [供应商Z]：排名下降[XX名]，根因[双维度分析]

新供应商表现（引入<6个月）：
[供应商N]：第[X]月，排名[XX]，是否达里程碑？
[供应商M]：第[X]月，排名[XX]，是否需干预？
```

**Step 4：新供应商里程碑检查（工具箱§七）**
```
试运行供应商跟踪：
供应商：翰锐（第3月）
里程碑：成长期（3-4月）→ GMV应达中位60%
当前：GMV达中位58%（接近达标）
动作：下月重点跟踪，若无改善需预警

供应商：XXX（第6月）
里程碑：稳定期末 + 评判点
当前：排名前65%（未达前70%底线）
决策：调用【A4-转正延期决策】（延长/清退）
```

**Step 5：整改闭环率统计（工具箱§六）**
```
本月应完成整改：[X]项
实际完成：[Y]项
闭环率：[Y/X×100%]

判定：
≥70%：正常
50-70%：偏低，需加强跟踪
<50%：危险，需升级管理

未完成项清单：
1. [供应商A]：整改计划[XX问题]逾期
   原因：[供应商未提交/提交未达标]
   动作：[升级约谈/清退评估]

2. [供应商B]：整改计划[XX问题]进行中
   进度：[XX%]，预计完成时间[日期]
```

**Step 6：下月重点动作清单**
```
优先级排序：

🔴 P0（立即执行，本周）：
□ 约谈[供应商A]（A级骤降）
□ 发整改通知[供应商C]（连续预警）

🟡 P1（本周内）：
□ 私下沟通[供应商B]（首次预警）
□ 巡检[红色FCI供应商D]（FCI<60）
□ 发送[供应商E]转正/延期通知（试运行到期）

🟢 P2（本月内）：
□ 月度复盘会（向易人哥汇报）
□ 月度通报发布（全员）
□ 整改闭环跟踪（跟进逾期项）
```
每项需明确：对象、动作、完成时间

## 关键变量
- **D级数量**：>35%需加快清退节奏
- **A→S晋升率**：<10%说明标杆培养不足
- **闭环率**：<70%需升级管理强度
- **新供应商达标率**：6个月达标<50%说明引入质量需提升
- **A级骤降数量**：>2家需系统排查共性问题

## 常见误区
- ❌ 只列数据不分析：需写"这说明..."给出判断
- ❌ 标记重点超过5家：应聚焦Top 3-5家
- ❌ 不统计闭环率：月度必统计指标
- ❌ 忽略层级失衡：C/D级过多或S级过少都是问题
- ❌ 不复盘趋势：只看单月不看连续3个月趋势

## 决策输出

**月度复盘卡片**（一页纸）：
```
[月份]月度复盘
整体格局：S[X]家 / A[X]家 / B[X]家 / C[X]家 / D[X]家
层级判断：[S偏少/D偏多/正常流动]
重点标记（Top 5）：
1. [供应商A]：A级骤降 → 立即约谈
2. [供应商B]：连续预警 → 正式约谈+整改
3. [供应商C]：主管胜任≤2项 → 约谈管理问题
4. [供应商D]：隐性退步 → 关注
5. [供应商E]：试运行到期 → 转正/延期决策
上升最快：[供应商F]（↑XX名）
下降最多：[供应商G]（↓XX名）
整改闭环率：[XX%]（应完成X项/实际Y项）
下月P0：[具体动作+对象+时间]
```
**用途**：写在排名表旁边，或单独发给易人哥

**月度通报结构**（调用【B5-向上汇报准备】）：
```
一、整体格局（S/A/B/C/D分布 + 环比）
二、重点事件（首次预警X家/连续预警X家/A级骤降X家）
三、趋势分析（上升最快Top3 + 下降最多Top3）
四、新供应商进展（试运行X家，达标X家）
五、整改闭环（闭环率XX%，逾期X项）
六、下月计划（P0/P1/P2清单）
```
→ 衔接：`supplier-monthly-report.md`
