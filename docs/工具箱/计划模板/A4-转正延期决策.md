# A4-转正延期决策

> 新供应商试运行期（6个月）到期时，决定正式纳入还是延长观察/终止合作

## 触发信号
- 新供应商第6个月末，试运行期将满
- 该供应商已引入2-6个月，进入评估节点
- 供应商主管询问："我们转正有什么要求？"

## 核心问题
- 该供应商是否达到正式供应商标准？
- 如果未达标，是延长3个月还是直接终止？
- 延长期间的目标和考核标准是什么？
- 转正后纳入常规管理，需注意什么？

## 思考步骤

**Step 1：对照6个月里程碑（工具箱§七）**
```
阶段检查点：
引入期（1-2月）：
✓ 出勤率 ≥85%
✓ 无重大合规问题
→ 若不达标，第2月末就要预警

成长期（3-4月）：
✓ GMV达到中位供应商60%以上
→ 若不达标，第4月末评估是否延长

稳定期（5-6月）：
✓ 月度排名进入前70%（不在后30%）
✓ 排名稳定（无大幅波动）
→ 第6月末最终评判：留或退
```
- 每个阶段不达标 → 标记风险，下一阶段重点跟踪
- 第6月末未达前70% → 触发转正/延期/终止决策

**Step 2：三要素综合评估**
```
评估维度与权重：
1. 业绩排名（40%）：
   - 第6月末排名是否前70%？
   - 近3个月趋势：上升/下降/波动？
   - GMV是否稳定？

2. 合规记录（30%）：
   - 有无A类客诉？
   - 有无诚信问题（数据质量分≤70）？
   - 有无G项红色（负面清单）？

3. 稳定性（30%）：
   - 出勤率是否≥85%？
   - 流失率是否<15%？
   - 团队架构是否稳定（主管未更换）？

评分：
≥80分 → 转正
60-79分 → 延长3个月
<60分 → 终止合作（清退）
```

**Step 3：决策选项与条件**
```
选项1：正式转正（≥80分）
条件：
✓ 6个月排名稳定在前70%
✓ 无A类客诉/诚信问题
✓ 出勤率≥85%
动作：
□ 发转正通知书（supplier-onboard.md）
□ 纳入常规月度排名管理
□ 享受正式供应商全部权益

选项2：延长观察（60-79分 或 接近达标）
条件：
✓ 排名接近前70%（如65-70%）
✓ 有明确改进动作且部分有效
✓ 无合规红线
动作：
□ 发延期通知（onboarding-missing.md）
□ 延长3个月（第7-9月）
□ 设定明确提升目标（如：第7月末进入前65%）
□ 每月复盘进展

选项3：终止合作（<60分 或 诚信红线）
条件：
✓ 连续3个月后30%（已满足清退条件）
✓ 或出现诚信红线（数据造假≥2项）
✓ 或G项红色（重大合规问题）
动作：
□ 启动清退评估（调用【A6-清退评估】）
□ 发终止合作意向（按clearance-missing.md流程）
□ 做好业务切换准备
```

**Step 4：决策授权（工具箱§十九）**
```
权限矩阵：
决策事项          │ 年老师 │ 王易人 │ 军哥
─────────────────┼────────┼────────┼──────
试运行转正        │ 建议权 │ 确认   │ —
延长观察期        │ 建议权 │ 确认   │ —
清退评估启动      │ 建议权 │ 确认   │ 事前沟通
```
- 转正/延长：需易人哥确认
- 清退：需易人哥确认 + 军哥事前沟通

**Step 5：输出决策记录**
```
决策编号：TRN-YYYYMMDD-001（转正）
         EXT-YYYYMMDD-001（延长）
         CLR-YYYYMMDD-001（清退）
供应商：[供应商名]
试运行期：[第1月-第6月] 总结
综合评分：[XXX分]
决策结论：[转正/延长3个月/终止合作]
决策依据：[6个月数据总结 + 三要素得分]
审批流程：[已报易人哥确认 / 需军哥沟通]
下一步：[具体动作+时间点]
```

## 关键变量
- **第6个月排名**：是否稳定在前70%？（底线要求）
- **合规记录**：有无A类客诉或诚信问题（一票否决）
- **改进趋势**：近3个月是否持续改善？
- **团队稳定性**：出勤率、流失率、主管是否稳定？
- **资源需求**：该供应商是否需要继续扶持？

## 常见误区
- ❌ 只看第6个月单月数据：需看6个月整体趋势和稳定性
- ❌ 忽略合规一票否决：有诚信问题即使业绩达标也不应转正
- ❌ 延长观察无限期：最长3个月，需设定明确目标
- ❌ 不记录决策依据：需保存6个月数据作为审计依据
- ❌ 转正后放松管理：转正后纳入常规管理，继续月度排名追踪

## 决策输出

**转正通知书模板**：
```
[供应商名]，你好。
你司试运行期（[起始月]-[结束月]）已满，经评估：
✓ 6个月排名均在前70%（最高第X名，最低第Y名）
✓ 无重大合规问题
✓ 出勤稳定≥[ZZ%]
评估结论：正式纳入我司供应商体系，自[下月1日]起执行正式供应商全部管理流程。
下一步：纳入月度排名管理，请持续提升业绩。
```
→ 衔接：`supplier-onboard.md`（试运行转正通知书）

**延长观察通知模板**：
```
[供应商名]，你好。
你司试运行期第6个月评估结论：延长观察3个月（[第7月]-[第9月]）。
提升目标：[具体量化目标，如：第7月末排名进入前65%]
月度复盘：[每月X日复盘进展]
请于[日期]前提交第7月提升计划。
```
→ 衔接：`onboarding-missing.md`（试运行延期通知）

**终止合作通知模板**：
```
[供应商名]，你好。
经6个月试运行评估，你司连续3个月后30%[或其他原因]，达到清退条件。
我司决定启动清退评估流程，[具体安排]。
请做好业务切换准备。
```
→ 衔接：调用【A6-清退评估】→ `clearance-missing.md`

## 下一步衔接
- 转正 → `supplier-onboard.md` → 纳入常规管理
- 延长 → `onboarding-missing.md` → 每月复盘，调用【A1-业绩异常判断】跟踪
- 终止 → 调用【A6-清退评估】→ `clearance-missing.md` 流程
