# B1-新供应商准入

> 有新供应商寻源需求时，系统化执行8阶段SOP流程

## 触发信号
- 业务需求：新项目需要供应商承接（什么项目/多少人/何时上线/预计业绩）
- 寻源发现：自己或同事推荐了一家新供应商
- 战略布局：需引入特定领域供应商补足能力缺口

## 核心问题
- 这个需求是否真实？项目背景和规模是什么？
- 需要什么样能力的供应商？（9维度评估）
- 从寻源到上线标准周期是多久？（T+15）
- 每个阶段的审计检查点是什么？
- 谁负责哪个阶段？输入输出物是什么？

## 思考步骤

**Step 1：需求确认与拆解（提效方案周期工作1）**
```
需求三要素：
1. 项目信息：
   - 项目名称：[XXX]
   - 业务线：金条/企金/信用卡/财富
   - 需要人数：[XXX人]
   - 上线时间：[YYYY-MM-DD]
   - 预计月GMV：[XXX万元]

2. 能力要求：
   - 专注领域：[电销/催收/客服等]
   - 团队规模：需要[XX-XX]人团队
   - 特殊资质：[如：人力资源许可证、行业认证]
   - 地域要求：[城市/站点]

3. 约束条件：
   - 成本区间：[人均成本上限]
   - 时间要求：最晚上线时间
   - 集中度限制：单一供应商≤40%
```
**输出**：需求说明书（内部邮件或文档）

**Step 2：快速初筛（调用【E1-新供应商初筛】）**
```
基础门槛（任一不满足直接淘汰）：
□ 成立时间 > 2年
□ 团队规模 > 50人
□ 无诚信记录（裁判文书网/行政处罚）
□ 资质合规（营业执照/人力资源许可证等齐全）

快速对标：
□ 团队规模差距 <30%（与现有同类供应商对比）
□ 专注领域匹配度：高/中/低
□ 资源可用性：是否有空余名额承接？

初筛结论：
□ 推荐深度评估（进入Step 3）
□ 淘汰，原因：[具体原因]
```

**Step 3：9维度深度评估（提效方案周期工作1）**
```
9维度打分表（每维度0-10分，满分90分）：

维度1：过往业绩
- 历史月均GMV：[XXX万元]
- 排名稳定性：[连续X个月前70%]
- 同比/环比趋势：[上升/下降/波动]
得分：___/10

维度2：同类型对标
- 与[对标供应商]团队规模差距：[XX%]
- 专注领域匹配度：[高/中/低]
- 历史交付风格：[快速响应/稳扎稳打/价格优先]
得分：___/10

维度3：管理方式
- 主管稳定度：[X年经验，流失率X%]
- 管理架构：扁平/层级
- 信息化程度：系统工具使用情况
得分：___/10

维度4：人员数量
- 总人数：[XXX人]
- 熟手占比：[XX%]
- 人员流动性：[月流失率X%]
得分：___/10

维度5：公司背景
- 成立时间：[X年]
- 股东背景：[民营/国企/上市公司]
- 融资情况：[未融资/A轮/B轮]
得分：___/10

维度6：历史交付风格
- 同类项目经验：[X个类似项目]
- 客户评价：[优秀/良好/一般]
- 投诉记录：[A类X次/B类X次]
得分：___/10

维度7：配合商务
- 响应速度：[小时级/天级]
- 配合意愿：[主动/被动]
- 谈判态度：[灵活/强硬]
得分：___/10

维度8：团队架构
- 组织架构：是否完整（业务/质检/培训/HR）
- 核心岗位：主管/质检/培训是否齐备
- 梯队建设：是否有后备干部
得分：___/10

维度9：资质合规
- 营业执照：✓/✗
- 人力资源许可证：✓/✗
- 行业特殊资质：[如有]
- 安全生产/信息安全认证：✓/✗
得分：___/10

总分：___/90
≥70分 → 推荐引入
60-69分 → 谨慎引入，需加强某维度
<60分 → 不建议引入
```

**Step 4：资源分配决策（调用【A3-分量调整决策】）**
```
若为现有供应商增额：
→ 直接调用分量5维打分，决定增量幅度

若为新供应商首次引入：
□ 资源总量：[XXX%]（从何处腾挪？）
□ 单一供应商上限：≤40%
□ 集中度检查：引入后TOP3占比是否超标？
□ 决策记录：supplier-selection-framework.md
```

**Step 5：8阶段SOP流程（工具箱§三 + official-doc SOP参考）**
```
阶段1：需求发起（T+0）
责任人：年老师
输入：需求说明书
输出：寻源任务单
检查点：需求是否清晰？预算是否明确？

阶段2：需求拆解（T+1）
责任人：年老师
输入：寻源任务单
输出：供应商画像（9维度要求）
检查点：画像是否覆盖9维度？是否可量化？

阶段3：寻源触达（T+2 ~ T+5）
责任人：年老师/同事
输入：供应商画像
输出：候选供应商列表（3-5家）
检查点：是否主动触达？是否≥3家候选？

阶段4：方案审批（T+6 ~ T+8）
责任人：年老师 → 王易人
输入：候选评估报告
输出：审批意见
检查点：是否按9维度评估？是否经易人哥审批？

阶段5：结果同步（T+9）
责任人：年老师
输入：审批意见
输出：结果通知（入选/淘汰）
检查点：是否通知所有候选？淘汰理由是否清晰？

阶段6：供应商准备（T+10 ~ T+12）
责任人：供应商
输入：准入通知
输出：准备材料（人员/系统/场地）
检查点：供应商是否按时准备？是否有卡点？

阶段7：内部操作（T+13 ~ T+14）
责任人：年老师/IT/财务
输入：供应商材料
输出：系统配置/账号开通/合同签署
检查点：系统是否就绪？合同是否签署？

阶段8：培训上线（T+15）
责任人：年老师/供应商
输入：系统就绪
输出：试运行开始
检查点：培训是否完成？业务是否跑通？
```
**标准周期**：T+15天（从需求发起到上线）
**审计检查点**：每阶段必须有输入/输出/存档记录

## 关键变量
- **项目紧急度**：标准周期T+15，紧急项目可压缩但需说明原因
- **候选数量**：必须≥3家对比，避免单一依赖
- **9维度总分**：≥70分推荐引入，<60分直接淘汰
- **资源可用性**：是否有空余名额？是否需要调整现有供应商份额？
- **审批路径**：需易人哥审批（工具箱§十九决策授权矩阵）

## 常见误区
- ❌ 跳过初筛直接深度评估：基础门槛不满足的浪费评估时间
- ❌ 候选<3家：对比不充分，决策风险高
- ❌ 9维度打分不量化：凭感觉打分，无依据
- ❌ 忽略集中度检查：引入后TOP3占比可能超标
- ❌ 不按SOP阶段：跳过审计检查点，后续无法追溯
- ❌ T+15周期压缩：紧急项目可压缩但需记录原因

```
项目：[XXX项目]
需求：[XX人，YY月ZZ日上线]
能力要求：
- 专注领域：[电销/客服/催收]
- 团队规模：[50-100人]
- 特殊资质：[人力资源许可证]
- 地域：[杭州/北京/上海]

9维度最低要求：
- 过往业绩：月均GMV≥XXX万元，排名前70%
- 人员数量：总人数≥50，熟手占比≥60%
- 资质合规：三证齐全
- 其他：[根据项目特殊要求]
```

**候选评估对比表**：
```
| 供应商 | 9维总分 | 排名 | 团队规模 | 资源可用性 | 优势 | 风险 | 推荐度 |
|--------|---------|------|---------|-----------|------|------|-------|
| 毅航   | 82      | 5    | 120人   | 有空余名额 | 业绩稳、配合好 | 成本高 | ★★★★★ |
| 毛毛虫 | 75      | 12   | 80人    | 需等2周    | 成本低、响应快 | 新人多 | ★★★★☆ |
| 赛维斯 | 58      | 28   | 60人    | 已满        | — | 连续预警 | ★☆☆☆☆ |
```

```

阶段3寻源触达：进行中（已触达3家，2家回复）
阶段4方案审批：待易人哥审批（T+8）
阶段5结果同步：待定
阶段6供应商准备：待定
阶段7内部操作：待定
阶段8培训上线：目标T+15
```

## 下一步衔接
- 初筛通过 → 启动8阶段SOP流程
- 决策审批 → 调用【B5-向上汇报准备】（如需军哥审批）
- 供应商准备 → 调用【E2-准入评审决策】（试运行跟踪）
- 上线后 → 纳入月度排名，调用【A1-业绩异常判断】跟踪
- 输出公文 → `supplier-onboard.md`（准入通知/现场考察报告）
