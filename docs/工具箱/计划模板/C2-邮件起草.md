# C2-邮件起草

> 想清楚后需要写邮件时，快速生成结构清晰、要素齐全的商务邮件

## 触发信号
- 约谈邀约：需要发邮件约供应商沟通
- 整改通知：要求供应商限期整改
- 进度同步：向同事或领导汇报进展
- 内部协调：与其他部门协调资源或信息
- 正式公文：需调用 official-doc 技能生成

## 核心问题
- 这封邮件的目的是什么？（通知/请求/汇报/警示）
- 收件人是谁？语气如何把握？（供应商/同事/领导）
- 邮件必须包含哪些要素？（背景/事实/要求/期限）
- 需要哪些附件或数据支撑？
- 是否需要抄送？抄送给谁？

## 思考步骤

**Step 1：明确邮件类型与目的**
```
邮件类型矩阵：
┌─────────────┬──────────────┬──────────────┬──────────────┐
│ 类型         │ 目的         │ 收件人       │ 语气         │
├─────────────┼──────────────┼──────────────┼──────────────┤
│ 约谈邀约     │ 邀请面对面沟通 │ 供应商主管   │ 正式+尊重    │
│ 整改通知     │ 要求限期整改  │ 供应商负责人 │ 严肃+明确    │
│ 进度同步     │ 告知进展      │ 同事/领导    │ 简洁+事实    │
│ 内部协调     │ 请求支持      │ 跨部门同事   │ 协作+清晰    │
│ 警示通知     │ 警告后果      │ 供应商高层   │ 严厉+留痕    │
│ 正式公文     │ 合规留档      │ 多方收件人   │ 规范+严谨    │
└─────────────┴──────────────┴──────────────┴──────────────┘

目的单一原则：一封邮件只解决一个问题
```

**Step 2：确定收件人与语气**
```
收件人分析：
□ 供应商主管（执行层）：
  - 语气：尊重但明确
  - 内容：事实+要求+期限
  - 抄送：供应商老板（知悉）

□ 供应商老板（决策层）：
  - 语气：严肃+正式
  - 内容：问题性质+后果+高层介入原因
  - 抄送：军哥/易人哥（如涉及重大事项）

□ 同事（平级）：
  - 语气：协作+简洁
  - 内容：事实+进展+需配合事项
  - 抄送：相关同事（知悉）

□ 领导（上级）：
  - 语气：尊重+汇报式
  - 内容：结论先行+数据支撑+需决策事项
  - 抄送：相关同事（按需）
```

[供应商高层]，你好。

1. [异常1：具体说明，如：出勤率连续4周≥98%过于平滑]
2. [异常2：具体说明，如：离职率连续3月为0]

**Step 4：附件与数据支撑**
```
邮件附件清单：
□ 数据表：排名表/双维度表/主管胜任表
□ 截图：FCI检查项照片/系统数据截图
□ 历史记录：上次整改要求/本次对比
□ 公文草稿：official-doc模板预览（如需正式发文）

附件命名规范：
[日期]_[供应商]_[事项].[扩展名]
示例：20260419_毅航_月度排名数据.xlsx
```

**Step 5：抄送与时效**
```
抄送规则：
□ 重大事项（清退/诚信红线）：军哥+易人哥
□ 约谈/整改：供应商老板（知悉）+ 相关同事（刘伟佳/刘乾坤等）
□ 内部协调：相关方同事
□ 进度同步：直接上级 + 相关方

时效要求：
□ 约谈邀约：提前3-5天发出
□ 整改通知：立即发出，明确截止时间
□ 警示通知：24h内发出
□ 进度同步：每周固定时间（如周一上午）

跟进机制：
发出后24h内未确认 → 电话/微信跟进
截止日前1天 → 提醒
逾期未完成 → 升级处理
```

## 关键变量
- **邮件目的**：决定模板选择和语气
- **收件人身份**：主管（执行层）/老板（决策层）/领导（战略层）
- **问题严重度**：一般/严重/红线，决定抄送范围和时效
- **是否留痕**：合规/诚信问题必须留痕（抄送+存档）
- **后续动作**：邮件是手段不是目的，需明确后续流程

## 常见误区
- ❌ 主题模糊："关于XX的事" → 需明确"约谈邀警/整改通知/进度同步"
- ❌ 正文冗长：商务邮件≤200字，核心信息前置
- ❌ 语气不当：对供应商太软（丧失权威）或太硬（破坏关系）
- ❌ 无明确要求：只说"请改进"，不说"具体做什么+何时完成"
- ❌ 不抄送关键人：重大事项需抄送上级留痕
- ❌ 发后不跟进：邮件发出就当完成，需跟踪是否读阅/执行

## 决策输出

**邮件自检清单**（发出前核对）：
```
□ 主题清晰：[事由]-[供应商]-[日期]
□ 核心结论第一句：[一句话说明 purpose]
□ 背景数据：[1-3个关键数据支撑]
□ 要求明确：[做什么+何时完成]
□ 语气恰当：[收件人身份匹配]
□ 附件齐全：[如有附件已添加]
□ 抄送正确：[关键人是否抄送]
□ 时效合理：[截止时间是否留足缓冲]
```

**邮件模板速查表**：
```
问题类型          │ 模板号 │ 关键要素
─────────────────┼────────┼─────────────────────
约谈邀约         │ 模板1  │ 问题+数据+时间+准备要求
整改通知         │ 模板2  │ 问题清单+整改要求+截止时间
进度同步         │ 模板3  │ 已完成+进行中+下一步+需支持
警示通知         │ 模板4  │ 异常说明+依据+处理+后果
内部协调         │ 模板3变体 │ 背景+请求+时限
正式公文         │ official-doc │ 调用技能生成
```

**快速起草流程**：
1. 确定邮件类型 → 选择模板
2. 填充关键信息（[方括号]内容）
3. 补充背景数据（1-3个关键指标）
4. 检查语气与抄送
5. 发出 → 设置提醒跟进

## 下一步衔接
- 正式公文（合同/函件/报告）→ 调用 official-doc 技能生成
- 约谈/整改邮件 → 触发【A2-约谈准备】或【A7-整改计划评估】流程
- 警示邮件 → 触发【A5-合规问题定级】后续动作
- 进度同步 → 更新任务状态，调用【TodoWrite】跟踪
- 发出后 → 设置日历提醒跟进（24h确认/截止前1天提醒）
