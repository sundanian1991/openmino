# C3-数据整理分析

> 拿到一堆数据（排名/人力/业绩/巡检）后，系统化看懂并提取洞察

## 触发信号
- 每月初系统导出上月排名和业绩数据
- 收到供应商人力表/巡检记录/客诉数据
- 需要做月度分析/专项分析/异常核查
- 领导问："这个数据你怎么看？"

## 核心问题
- 这个数据集包含什么？（字段/时间范围/供应商范围）
- 数据质量如何？（可信度/完整性/一致性）
- 核心指标是什么？（排名/人效/达成率/异常率）
- 异常点在哪里？（Top 5 问题供应商）
- 我能得出什么结论？下一步做什么？

## 思考步骤

**Step 1：数据体检（工具箱§十九）**
```
数据质量三问：
1. 数据来源可靠吗？
   - 系统自动（A系统，32%权重）✓ 最可信
   - 供应商填报（B系统，21%权重）⚠ 需抽查
   - 质检数据（C系统，11%权重）✓ 较可信
   - 年老师记录（D系统，18%权重）✓ 可信
   - 巡检记录（E系统，18%权重）✓ 现场核实

2. 数据完整吗？
   - 时间跨度：[X个月] 是否完整？
   - 供应商覆盖：32家是否齐全？
   - 字段完整：5维度（业绩/合规/配合/组织/成长）是否都有？

3. 数据一致吗？
   - 人力表 vs 系统人数：是否一致？
   - 业绩数据 vs 财务结算：是否一致？
   - 本周数据 vs 上周：波动是否合理？

数据质量分计算（工具箱§十九）：
信号1：出勤平滑（连续4周≥98%）→ 预警
信号2：离职为零（30人以上团队）→ 预警
信号3：GMV平稳（日波动<5%）→ 关注
信号4：熟手突增无记录 → 预警
信号5：考核节点突变 → 危险

判定：
0项触发 → 100分（可信）
1项触发 → 70分（存疑，标注）
≥2项触发 → 40分（不可信，排名加★）
```

**Step 2：确定分析目标与维度**
```
分析类型选择：

类型1：月度排名分析（最常用）
目标：看整体格局 + 识别异常供应商
维度：
- 排名分布（S/A/B/C/D）
- 触发状态（首次预警/连续预警/A级骤降）
- 双维度（自身峰值比/头部差距）
- 主管胜任（5项达标情况）

类型2：人力分析
目标：看人员结构/稳定性/新人影响
维度：
- 总人数/熟手人数/新人占比
- 流失率（月度/周度）
- 出勤率（日均/周度）
- 新人拉低幅度计算

类型3：业绩分析
目标：看达成率/趋势/异常点
维度：
- GMV达成率（目标vs实际）
- 人均GMV（总量/熟手人均）
- 近4周趋势（上升/下降/波动）
- 峰值比（本月vs近6月峰值）

类型4：巡检/FCI分析
目标：看能力短板/整改效果
维度：
- FCI总分 + 七大项得分
- 红黄绿灯分布
- 红色项/黄色项数量
- 整改进度跟踪

类型5：专项分析（如：新供应商跟踪/头部差距拉大原因）
目标：深挖特定问题
维度：根据问题定制
```

**Step 3：数据清洗与准备**
```
清洗步骤：
1. 去重：同一供应商多条记录 → 保留最新/最全
2. 补缺：缺失字段 → 标注或按规则估算（如：新人占比高需标注）
3. 统一单位：金额/人数/百分比统一格式
4. 异常值识别：
   - 出勤率>100%？ → 数据错误
   - GMV为负？ → 数据错误
   - 流失率>50%？ → 核实是否真实

数据表结构（月度排名分析示例）：
| 供应商 | 排名 | 排名变化 | 触发状态 | 自身峰值比 | 头部差距 | 类型 | 主管胜任 | 备注 |
|--------|------|---------|---------|-----------|---------|------|---------|------|
| 毅航   | 28   | ↓5      | 首次预警 | 62%       | 55%     | 问题型 | 3/5     | 新人45% |
| 毛毛虫 | 5    | ↑3      | 正常     | 88%       | 75%     | 标杆型 | 5/5     | — |
| ...    | ...  | ...     | ...     | ...       | ...     | ...   | ...     | ... |

标注规则：
🔴 连续预警：连续2个月后30%
🟡 首次预警：本月首次后30%
🔵 A级骤降：上月前30%本月后30%
🟢 诚信问题：数据质量分≤70
```

**Step 4：数据分析与洞察提取**
```
分析流程：

1. 看整体（描述统计）：
   - 平均排名：[XX名]
   - 中位数排名：[XX名]
   - D级占比：[XX%]
   - 平均人效：[XXX元]

2. 看分布（直方图/饼图）：
   - S/A/B/C/D各多少家？
   - 与上月相比变化？

3. 看趋势（时间序列）：
   - 连续上升：[X]家（上升≥3名）
   - 连续下降：[X]家（下降≥3名）
   - 波动正常：[X]家

4. 看异常（Top N）：
   - 排名下降最多：[供应商A ↓XX名]
   - 人效下降最多：[供应商B 人效↓XX%]
   - 新人拉低最严重：[供应商C 新人占比XX% 拉低幅度XX%]

5. 看关联（交叉分析）：
   - D级供应商中，主管胜任≤2项的比例？
   - 新人占比>30%的供应商中，多少落在后30%？
   - FCI红灯的供应商，业绩排名如何？

洞察提取公式：
[数据现象] + [对比基准] + [根因指向] = 洞察
示例：
"毅航排名↓5名到28（后30%）+ 自身峰值比62%（<65%）+ 新人占比45% → 新人稀释导致人效下降，需约谈"
```

```
报告结构：

## 关键变量
- **数据来源**：系统自动 > 巡检记录 > 供应商填报（可信度递减）
- **数据质量分**：≤70需标注，≤40排名加★不作为奖惩依据
- **时间范围**：月度/季度/年度，分析维度不同
- **分析目标**：排名分析/人力分析/业绩分析/专项分析，决定看哪些维度
- **异常阈值**：后30%/新人>30%/出勤<85%/流失>5%等，工具箱已定义

## 常见误区
- ❌ 只列数据不分析："毅航排名28" → 需说"首次预警，新人稀释"
- ❌ 忽略数据质量：低质量数据直接用于决策会出错
- ❌ 不交叉验证：单维度看问题，需多维度交叉（排名+双维度+胜任）
- ❌ 过度解读：小样本波动当趋势，需看连续3个月
- ❌ 无行动导向：分析完不产出"下一步做什么"
- ❌ 不看历史：不对比上月/去年，失去参照系

```

**快速分析 checklist**：
```
□ 数据来源确认（系统/填报/巡检）
□ 数据质量分计算（5项信号检查）
□ 数据清洗（去重/补缺/统一格式）
□ 确定分析目标（排名/人力/业绩/专项）
□ 核心指标计算（排名/人效/达成率/FCI）
□ 异常识别（后30%/新人>30%/出勤<85%等）
□ 交叉验证（多维度看同一供应商）
□ 洞察提取（现象+根因+行动）
□ 输出报告（一页纸或邮件）
```

## 下一步衔接
- 月度排名分析 → 调用【B4-月度复盘】→ 标记重点供应商
- 发现异常供应商 → 调用【A1-业绩异常判断】→ 触发约谈/整改
