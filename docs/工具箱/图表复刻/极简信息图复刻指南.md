# 极简信息图复刻指南 — Minimalist Infographic Style Guide

> 适用于 SVG 生成 AI，可直接作为 system prompt 使用

---

## 一、风格一句话描述

**暖调极简主义信息图**：米色纸张质感背景，暖灰色层次容器，Georgia 衬线标题，单一暖棕强调色。像学术书籍中的印刷插图，不是数据可视化不是 SaaS dashboard。

---

## 二、色彩系统（严格固定）

### 2.1 结构色（所有图通用）

| 语义角色 | 色值 | 用途 |
|---------|------|------|
| **画布背景** | `#faf9f7` | SVG 最外层 fill，覆盖全部区域 |
| **容器描边** | `#1a1a1a` | 大概念边界框 stroke（1.5px） |
| **卡片填充** | `#f5f4f1` | 卡片 rect fill（暖灰米色，**不是纯白**） |
| **卡片描边** | `#d4d1c7` | 卡片 rect stroke（1px） |
| **次要文字** | `#6b6b6b` | 标注、副标题、说明文字 |
| **主要文字** | `#1a1a1a` | 标题、卡片内正文 |
| **强调色** | `#c98a6a` | 唯一彩色——连接线、刻度线、标注点 |
| **虚线容器** | `#e8e4dc` | "free / 被丢弃" 区域描边（dasharray） |

### 2.2 语义色（按需使用，不超过 2 种/图）

| 色值 | 语义 | 示例 |
|------|------|------|
| `#8f9b7f` | 成功 / 已完成 / 正向 | Correcting 的进度条 |
| `#6a8fb5` | 用户输入 / 信息 / 冷调对比 | Rewinding 标题栏 |
| `#d4845a` | 警告 / 注意 / 重要提示 | 橙色高亮块 |
| `#a3867a` | 辅助强调 / 灰色对比 | 淡橙色背景块 |

### 2.3 色彩铁律

1. **一张图最多 2 种语义色**（不包括结构色）
2. **优先用灰度层次区分内容**，而不是彩色
3. **彩色只在"需要语义区分"时使用**（比如正确 vs 错误方案）
4. **永远不用**：纯红 `#ff0000`、纯蓝 `#0000ff`、荧光色、高饱和色

## 三、字体系统

### 3.1 字体配对

```
标题 → Georgia, serif（衬线 = 书卷气）
正文 → system-ui, -apple-system, sans-serif（无衬线 = 现代清晰）
```

### 3.2 字号层级

| 层级 | 字号 | 字重 | 字体 | 用途 |
|------|------|------|------|------|
| **主标题** | 18px | bold | Georgia | 图标题 |
| **卡片标题** | 11-12px | 500 | system-ui | 小卡片标题 |
| **正文** | 10-10.5px | 400 | system-ui | 卡片内说明文字 |
| **标注** | 8.5-9.5px | 400 | system-ui | 灰色小字、刻度标签 |
| **斜体标注** | 9.5-10px | italic | Georgia | 底部补充说明 |

### 3.3 文案规则

```
标题 ≤ 5 词（名词短语）
说明 ≤ 2 行（主谓宾完整句）
标注 ≤ 1 行（灰色小字，可省略主语）
```

## 四、容器系统

### 4.1 三种容器

| 类型 | 圆角 | 描边 | 填充 | 语义 |
|------|------|------|------|------|
| **大容器** | rx=10-12 | 1.5px #1a1a1a | none | 概念边界，包一组内容 |
| **卡片** | rx=8 | 1px #d4d1c7 | #f5f4f1 | 独立信息单元 |
| **标签** | rx=5-6 | 0.5px #d4d1c7 | 跟随父容器 | 元数据、状态标记 |

### 4.2 容器即信息

- **大容器包住的内容 = 一个概念组**
- **容器嵌套 = 包含关系**
- **虚线容器 = 可选的 / 被丢弃的 / 不存在的**
- **容器大小差异 = 信息量差异**（大 = 完整，小 = 压缩）

---

## 五、线条系统

| 线条类型 | 语法 | 语义 |
|---------|------|------|
| **实线箭头** | `stroke-width="1" marker-end` | 确定的传递 / 流动方向 |
| **虚线箭头** | `stroke-dasharray="4 3"` | 间接关系 / 可选传递 |
| **粗竖线** | `stroke-width="3" stroke-linecap="round"` | 状态标记（绿=好，橙=注意） |
| **回环曲线** | `path d="M x1 y1 Q cx cy x2 y2"` | 回退 / 循环 / 重置 |
| **刻度线** | 短竖线 6px 高 | 度量标记点 |

---

## 六、间距系统

| 间距类型 | 数值 | 说明 |
|---------|------|------|
| **卡片间距** | 6-8px | 同组卡片之间的水平/垂直间距 |
| **容器内边距** | 10-12px | 内容不贴容器边缘 |
| **组间距** | 20-30px | 不同概念组之间的留白 |
| **文字行距** | 13-14px | 多行文字的 line-height 效果 |
| **上下留白** | 顶部 25-30px，底部 15-20px | 标题距顶部、最后元素距底部 |

## 七、视觉隐喻模板库

当拿到一段文字内容时，先判断它属于哪种逻辑关系，然后选择对应模板：

### 模板 1：时间线 / 容量轴
- **触发条件**：展示"组成部分按顺序排列"或"容量从左到右"
- **视觉结构**：大容器框住所有卡片 + 底部水平轴 + 刻度标记
- **案例**：图 1 Context Window

### 模板 2：渐变衰减
- **触发条件**：展示"从清晰到模糊"或"质量递减"
- **视觉结构**：顶部渐变色条（深→浅） + 下方解释卡片
- **案例**：图 2 Context Gradient

### 模板 3：光谱排列
- **触发条件**：展示"策略从简单到复杂"或"选项排列"
- **视觉结构**：水平轴 + 箭头 → 多个等宽卡片 + 右侧竖线标记
- **案例**：图 3 五策略对比

### 模板 4：左右对比（包含 vs 筛选）
- **触发条件**：展示"全部内容 vs 实际看到"
- **视觉结构**：两个等大容器并排 + 中间箭头标注
- **案例**：图 4 Session vs Seen

### 模板 5：方案对比（A vs B）
- **触发条件**：展示"两种方法的效果差异"
- **视觉结构**：左右两栏卡片 + 各自的颜色标识 + 底部建议
- **案例**：图 5 Correcting vs Rewinding

### 模板 6：父子容器
- **触发条件**：展示"什么传过去，什么留下来"
- **视觉结构**：大容器 + 内部小容器 + 曲线箭头标注 spawn/return
- **案例**：图 6 Parent/Subagent

### 模板 7：信息过滤
- **触发条件**：展示"完整记录 vs 模型实际接收"
- **视觉结构**：左侧密集列表 + 右侧精简卡片 + 中间过滤箭头
- **案例**：图 7 Session History Filtering

## 八、SVG 生成模板

```xml
<svg width="100%" viewBox="0 0 680 {高度自适应}" xmlns="http://www.w3.org/2000/svg">

  <!-- 1. 画布背景 -->
  <rect width="680" height="{高度}" fill="#faf9f7"/>

  <!-- 2. 主标题（Georgia 衬线） -->
  <text x="40" y="28" font-family="Georgia,serif" font-size="18" font-weight="bold" fill="#1a1a1a">
    标题文字
  </text>

  <!-- 3. 副标题（灰色小字） -->
  <text x="40" y="46" font-family="system-ui" font-size="11" fill="#6b6b6b">
    说明文字
  </text>

  <!-- 4. 大容器（概念边界） -->
  <rect x="40" y="62" width="600" height="..." rx="12"
        fill="none" stroke="#1a1a1a" stroke-width="1.5"/>

  <!-- 5. 卡片（信息单元） -->
  <rect x="..." y="..." width="..." height="..." rx="8"
        fill="#f5f4f1" stroke="#d4d1c7" stroke-width="1"/>

  <!-- 6. 卡片文字（居中对齐） -->
  <text x="中心点" y="..." text-anchor="middle"
        font-family="system-ui" font-size="10.5" fill="#1a1a1a">
    文字
  </text>

  <!-- 7. 连接线和箭头 -->
  <line x1="..." y1="..." x2="..." y2="..."
        stroke="#c98a6a" stroke-width="1"
        marker-end="url(#arrowhead)"/>

  <!-- 8. 底部标注 -->
  <text x="..." y="..." text-anchor="middle"
        font-family="Georgia" font-size="10" fill="#6b6b6b" font-style="italic">
    补充说明
  </text>

</svg>
```

## 九、生成自检清单

生成完一张图后，逐项检查：

- [ ] 背景色是 `#faf9f7`（不是纯白 `#ffffff`）
- [ ] 卡片填充是 `#f5f4f1`（不是纯白）
- [ ] 卡片描边是 `#d4d1c7`（不是 `#cccccc`）
- [ ] 标题用 Georgia 衬线字体
- [ ] 正文用 system-ui 无衬线字体
- [ ] 圆角：容器 rx=10-12，卡片 rx=8
- [ ] 卡片间距 6-8px 均匀
- [ ] 文字垂直居中在卡片内
- [ ] 颜色不超过 2 种语义色（+ 结构色）
- [ ] 每条线都有语义（不是纯装饰）
- [ ] 信息分层：标题 → 卡片 → 标注
- [ ] 留白足够（不拥挤不空洞）

## 十、内容→视觉转换流程

拿到一段文字内容后，按以下步骤生成：

```
Step 1：识别核心关系
  → 是时间线？对比？流程？层级？程度？
  → 选择对应视觉隐喻模板（第七节）

Step 2：切割文本块
  → 标题：≤5 词
  → 说明：≤2 行
  → 标注：灰色小字

Step 3：选择配色
  → 结构色固定
  → 如需语义区分，选 ≤2 种语义色
  → 确定哪些卡片需要特殊颜色

Step 4：布局计算
  → 确定 SVG viewBox 高度
  → 计算每个卡片的 x/y/width/height
  → 确保间距统一（6-8px）
  → 确保文字垂直居中

Step 5：生成 SVG
  → 背景 → 标题 → 容器 → 卡片 → 文字 → 线条 → 标注
  → 运行自检清单

Step 6：验证
  → 与原图/需求对比
  → 检查对齐、间距、颜色一致性
```
