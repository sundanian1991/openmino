# 自我认知：我是怎么管理的？

---

## 一、我管的是什么？

### 管理规模
- **30多条业务线**：金条、金企融合、企贷大额、白条、金农贷...
- **32家供应商**（人工）+ 1家自建
- **3000+人**
- **4大产线、17条细分业务**

### 这意味着什么？
- 每天有无数个问题需要解决
- 每天有无数个决策需要做
- 每天有无数个关系需要维护
- 每天有无数个风险需要防控

## 二、我的一天是怎么过的？

### 早晨（8:30-9:30）：看数据，找问题

**第一件事：看排名**
- 打开电脑，看所有供应商的排名数据
- 不是看今天，是看周趋势、月趋势
- 了解谁最好、谁最差、谁在进步、谁在退步

**我在想什么？**
- 为什么A供应商上周第3，这周掉到第8了？
- 为什么B供应商突然提升了？他做了什么？
- C供应商连续3周垫底，是不是该找他了？

**第二件事：刷内部群**
- 看看有没有业务方的@我
- 看看有没有新的项目需求
- 看看有没有紧急的合规问题

**第三件事：刷供应商群**
- 看看有没有供应商在问问题
- 看看有没有供应商在抱怨
- 看看有没有供应商在要资源

### 上午（9:30-12:00）：处理紧急问题

**场景1：供应商老板打电话来了**
- "孙经理，我们最近排名不太好，能不能多给点量？"
- "孙经理，我们最近业绩下滑了，能不能给点支持？"
- "孙经理，你们最近名单质量不太好，能不能调整一下？"

**我怎么回应？**
- 如果是他的问题，我会直接说："你们的问题在XX，我给你XX时间解决"
- 如果是我们的问题，我会说："这个问题我记下了，我去协调，回头给你答复"
- 如果是他在撒娇，我会直接怼："你看看你自己的排名，还好意思要资源？"

**场景2：内部同事来找我了**
- "大年，我们这个月要上一个新项目，你能帮我找几家供应商吗？"
- "大年，这个供应商最近表现不太好，能不能帮他优化一下？"
- "大年，这个月名单可能有点紧，能不能调整一下分量？"

**我怎么处理？**
- 先问清楚需求：是什么项目？需要多少人？什么时候上线？
- 再想供应商：谁适合这个业务？谁有这个能力？谁愿意接？
- 然后做决策：找3-5家合适的，让他们竞标，选最好的

**场景3：合规问题来了**
- "大年，这个供应商有客诉了，需要你处理一下"
- "大年，这个供应商在招聘时用了京东logo，需要你制止一下"
- "大年，这个供应商的坐席在骂人，需要你处罚一下"

**我怎么处理？**
- 先了解情况：是个案还是普遍问题？严重程度如何？
- 再做决策：如果是轻微的，口头警告；如果是严重的，发邮件处罚
- 最后跟进：要求对方整改，3-7天给我反馈

### 下午（14:00-18:00）：深入解决问题

**场景4：出差巡检**
- 如果这周有巡检计划，我可能在去供应商现场的路上
- 到了现场，我第一时间看：环境怎么样？人员精神怎么样？有没有违规？

**我检查什么？**
1. **空间布置**：有没有龙虎榜？有没有公开业绩？有没有话术管理？
2. **人员状态**：坐席有没有在打电话？有没有玩手机？有没有睡觉？
3. **薪资情况**：给供应商100元，员工拿多少？如果拿不到60元，我会要求调整
4. **招聘情况**：最近招了什么人？从哪里招的？有没有用京东名义招聘？
5. **品牌使用**：有没有滥用京东logo？有没有违规宣传？
6. **管理情况**：主管有没有在管？有没有培训？有没有复盘？

**发现问题怎么办？**
- 如果是轻微的，当场指出，要求整改
- 如果是严重的，发邮件，要求3-7天整改，后续跟进

**场景5：新项目对接**
- 如果有新项目，我可能要去对接供应商
- 我会问：你们能不能接？需要多少人？什么时候能上人？预计业绩多少？
- 然后做评估：他们适不适合？能不能做好？值不值得给？
- 最后做决策：给还是不给？给多少？什么时候给？

**场景6：降价谈判**
- 如果公司要求降价，我会去跟供应商谈
- 我会怎么说："现在公司要求降价，你们怎么看？"
- 如果他们说"不降"，我会说："你看，自建团队做得比你们好，如果不降，我就把量给自建团队了"
- 如果他们说"降多少"，我会说："先降15%，后续看情况"

### 晚上（18:00之后）：复盘总结

**场景7：写邮件**
- 如果今天有巡检，我会写巡检报告
- 如果今天有处罚，我会写处罚通知
- 如果今天有新项目，我会写项目需求

**场景8：准备明天**
- 看看明天有什么安排
- 看看有没有紧急问题需要处理
- 看看有没有供应商需要跟进

## 三、我是怎么管30多条业务线的？

### 第一步：给业务分类

**不是所有业务都一样管理，我会分三类：**

**类型1：拉新型业务**
- 特点：需要大量新客，转化率要求高
- 适合的供应商：擅长拉新、有名单资源、能快速上人
- 管理方式：关注拉新数量、新客质量、转化率

**类型2：复购型业务**
- 特点：需要老客复购，服务质量要求高
- 适合的供应商：擅长复购、服务意识好、能维护客户
- 管理方式：关注复购率、客诉率、满意度

**类型3：创新型业务**
- 特点：新模式、新流程、不确定因素多
- 适合的供应商：学习能力强、配合度高、愿意尝试
- 管理方式：关注适应速度、问题反馈、优化建议

### 第二步：给供应商贴标签

**每个供应商都有标签：**

**标签1：业务类型标签**
- 拉新型 / 复购型 / 创新型 / 大额型 / 企微型

**标签2：能力标签**
- 头部 / 中部 / 尾部
- 擅长拉新 / 擅长复购 / 擅长大额 / 擅长企微

**标签3：配合度标签**
- 高配合 / 中配合 / 低配合
- 主动响应 / 被动响应 / 需要催促

**标签4：稳定性标签**
- 稳定 / 波动 / 不稳定

**标签5：潜力标签**
- 高潜力 / 中潜力 / 低潜力

### 第三步：把合适的供应商放在合适的地方

**举例：**
- 拉新型业务 → 找擅长拉新的头部供应商
- 创新型业务 → 找学习能力强的高配合度供应商
- 大额型业务 → 扴擅长大额的稳定型供应商

**为什么这么做？**
- 因为供应商不是万能的，各有各的擅长
- 因为业务不是一样的，各有各的需求
- 把合适的供应商放在合适的地方，才能产出最大价值

### 第四步：动态调整分量

**我不会一成不变地给分量：**
- 如果这个供应商表现好，我会多给
- 如果这个供应商表现差，我会少给
- 如果这个供应商在进步，我会鼓励
- 如果这个供应商在退步，我会施压

**为什么这么做？**
- 让供应商保持期待
- 让供应商不敢躺平
- 让供应商持续优化

## 四、我是怎么管30多个供应商的？

### 第一步：准入（选）

**不是所有供应商都能进来，我有9个标准：**

1. **过往业绩**：以前做过什么？业绩怎么样？
2. **同类型业务对标**：有没有做过类似业务？
3. **管理方式**：怎么管理团队？有没有SOP？
4. **人员数量**：有多少人？能不能快速上人？
5. **公司背景**：是什么公司？有没有实力？
6. **历史交付风格**：以前配合度怎么样？
7. **配合的商务**：商务对接人怎么样？
8. **内部团队架构**：团队架构是否合理？
9. **资质和合规**：有没有资质？有没有违规记录？

**流程线上化后：**
- 之前：审批4-5天，引入周期30天
- 现在：审批1-2天，引入周期15天

### 第二步：使用（用）

**不是所有供应商都一样用，我有5个打分维度：**

1. **业绩（30%）**：GMV、转化率、名单产能
2. **质检客诉（25%）**：客诉率、质检合格率、合规率
3. **上人数字（15%）**：能上多少人、上人速度
4. **配合度（15%）**：响应速度、配合程度、沟通质量
5. **战略打分（15%）**：是不是我要扶持的？有没有学习能力？有没有额外技能？

**赛马机制：**
- 每月排名，看谁好谁差
- 好的有奖励：多给量、给新项目
- 差的有处罚：少给量、约谈、警告

### 第三步：培育（育）

**不是所有供应商都能自己成长，我会帮助他们：**

**怎么帮？**
1. **给培训**：教他们怎么适应京东、怎么做好业务
2. **给资源**：给好名单、给支持
3. **给机会**：给新项目、给增量
4. **给反馈**：定期沟通，告诉他们哪里好、哪里差

**文档化：京东适应体系**
- 我自己总结了一份文档，告诉供应商：
  - 京东怎么看事情
  - 京东在乎什么
  - 京东怎么工作
  - 以前踩过什么坑
  - 怎么快速出产能

### 第四步：淘汰（汰）

**不是所有供应商都适合留下来，我有递进式清退规则：**

**第一级：警告**
- 发现问题，口头警告
- 要求整改，给3-7天时间

**第二级：约谈**
- 警告后没改善，约谈商务
- 要求给出整改方案

**第三级：限制新进**
- 约谈后没改善，限制新业务
- 只能做存量，不能做增量

**第四级：清退**
- 限制后还没改善，清退
- 但要让供应商心甘情愿自己申请

**为什么让供应商自己申请？**
- 如果我强制清退，供应商会闹、会索赔、会传播负面信息
- 如果让供应商自己申请，他们不会闹，日后好相见
- 我会给两难选择：闹→被京东拉黑 vs 走→日后好相见

- 发现问题→发邮件→要求整改→跟进反馈

**第四层：给支持**
- 培训：教他们怎么适应京东

## 六、我是怎么协调资源的？

### 场景1：业务方要上新项目

**我的处理流程：**
1. **问清楚需求**：什么项目？需要多少人？什么时候上线？预计业绩多少？
2. **想供应商**：谁适合？谁有能力？谁愿意接？
3. **做决策**：找3-5家合适的，让他们竞标，选最好的
4. **跟进**：看他们上线情况，看他们业绩表现

### 场景2：供应商说名单质量不好

**我的处理流程：**
1. **先看数据**：是不是真的不好？不好到什么程度？
2. **再看对比**：其他供应商怎么样？行业水平怎么样？
3. **找原因**：是名单问题？是供应商能力问题？是配合问题？
4. **做决策**：如果是名单问题，我去协调业务方调整；如果是供应商问题，我去要求供应商优化

### 场景3：供应商说业绩下滑

**我的处理流程：**
1. **先看数据**：下滑了多少？什么时候开始的？
2. **再看对比**：是普遍下滑还是个别下滑？
3. **找原因**：是市场原因？是名单原因？是供应商原因？
4. **做决策**：如果是外部原因，我去协调资源；如果是供应商原因，我去要求优化

### 场景4：供应商说涨价

**我的处理流程：**
1. **先听他说**：为什么涨价？涨多少？
2. **再讲事实**：市场行情怎么样？你们业绩怎么样？其他供应商怎么样？
3. **做决策**：如果合理，可以考虑；如果不合理，直接拒绝
4. **谈条件**：如果涨价，能不能给更好的业绩？

## 七、我是怎么总结经验的？

### 第一步：发现问题

**我发现问题的来源：**
1. **日常巡检**：每次巡检发现6-7个问题
2. **供应商反馈**：供应商会告诉我他们在哪里遇到困难
3. **内部反馈**：业务方会告诉我哪里有问题
4. **数据监控**：我会看排名、看客诉、看质检，发现异常

### 第二步：分析问题

**我的分析逻辑：**
1. **是个案还是普遍问题？**
   - 如果是个案，单独处理
   - 如果是普遍问题，需要制度解决

2. **是表层问题还是深层问题？**
   - 如果是表层问题，直接解决
   - 如果是深层问题，需要深入分析

3. **是我能解决的还是需要协调的？**
   - 如果是我能解决的，直接解决
   - 如果是需要协调的，找相关方协调

### 第三步：解决问题

**我的解决方式：**
1. **如果是管理制度问题** → 修改制度、优化流程
2. **如果是供应商能力问题** → 培训、辅导、资源倾斜
3. **如果是名单质量问题** → 协调业务方调整
4. **如果是系统问题** → 找技术方协调

### 第四步：固化经验

**我的固化方式：**
1. **写成文档**：把经验写成文档，发给所有供应商
2. **做成制度**：把经验做成制度，成为部门标准
3. **做成培训**：把经验做成培训，教给新供应商
4. **做成SOP**：把经验做成SOP，让其他人也能用

## 八、我是怎么把经验变成制度的？

### 案例1：京东适应体系

**背景：**
- 新供应商不知道京东怎么运作
- 新供应商不知道京东看什么、在乎什么
- 新供应商不知道怎么快速出产能
- 磨合时间太长，影响业务

**我的做法：**
1. **总结经验**：把过去踩过的坑、走过的弯路总结出来
2. **写成文档**：写成《京东适应指南》，发给所有供应商
3. **做培训**：新供应商进来，我先培训一次，告诉他们京东怎么看事情
4. **做答疑**：供应商有问题，我随时解答

**结果：**
- 新供应商磨合时间缩短了
- 新供应商出产能速度加快了
- 新供应商投诉减少了

### 案例2：巡检标准

**背景：**
- 巡检没有统一标准，每次检查的东西不一样
- 有些问题没有及时发现
- 有些供应商不知道什么是好的

**我的做法：**
1. **制定标准**：制定12项巡检标准（空间布置、龙虎榜、话术、薪资...）
2. **做成清单**：做成巡检清单，每次按清单检查
3. **写进制度**：写进《巡检标准》，成为部门标准
4. **做培训**：培训所有供应商，让他们知道标准是什么

**结果：**
- 巡检更有针对性了
- 问题更早被发现了
- 供应商知道怎么做才是好的了

### 案例3：递进式清退规则

**背景：**
- 之前清退供应商，没有明确流程
- 有些供应商突然被清退，会闹
- 有些供应商不知道自己问题严重性

**我的做法：**
1. **制定规则**：制定4级清退规则（警告→约谈→限制新进→清退）
2. **写进制度**：写进《违规处罚条例》，成为部门标准
3. **公示规则**：告诉所有供应商，规则是什么
4. **执行规则**：按规则执行，让供应商知道后果

**结果：**
- 清退零纠纷
- 供应商知道底线在哪里
- 供应商知道如果不改进，会有什么后果

## 九、我怎么把"人治"变成"机制"？

### 我的思考

**我之前是怎么管的？**
- 靠我自己的人脉
- 靠我自己的沟通
- 靠我自己的判断
- 靠我自己的关系

**这样管的问题？**
- 如果我不在了，怎么办？
- 如果供应商换了对接人，怎么办？
- 如果业务复杂度增加了，我管得过来吗？

**所以我要把"人治"变成"机制"**

### 具体怎么做？

**第一步：把隐性知识显性化**
- 把我脑子里的想法写成文档
- 把我的沟通经验写成SOP
- 把我的决策逻辑写成标准

**第二步：把个人经验制度化**
- 把我的做法写成制度
- 把我的标准变成部门标准
- 让其他人也能用

**第三步：把临时措施常态化**
- 把应急措施变成预案
- 把临时协调变成流程
- 让其他人也能操作

**第四步：把个人能力组织化**
- 让团队里有其他人也能做
- 让供应商自己也能做
- 让业务方也能配合做

## 十一、我的核心能力是什么？

### 能力1：拿捏供应商

**本质：** 让供应商在有限奖励下依然愿意投入

**怎么做？**
- 胡萝卜（分量）+ 大棒（合规+分量削减）
- 以业绩论英雄，不管背景/历史
- 规则运行在"阳光"下（相对透明）
- 口碑效应：做好就有，躺平就没

### 能力2：分量策略

**本质：** 把合适的供应商放在合适的地方

**怎么做？**
- 给供应商贴标签（业务类型、能力、配合度、稳定性、潜力）
- 5维打分（业绩30%、质检25%、上人15%、配合15%、战略15%）
- 动态调整（不都给，也不一直不给，让供应商保持期待）

### 能力3：建立对标

**本质：** 通过自建团队建立效能基准，倒逼供应商优化

**怎么做？**
- 建自建团队，成为对标配尺
- 拿着数据去跟供应商谈
- 供应商接受降价

### 能力4：把经验变成制度

**本质：** 把个人能力组织化，让其他人也能用

**怎么做？**
- 总结经验 → 写成文档 → 做成制度 → 做成培训 → 做成SOP
