# 供应商管理风险管控逻辑梳理

> 用途：贡献2"制度体系建设"的风险管控部分
> 对象：年老师 - 供应商管理岗

---

## 一、电销外呼业务的典型风险

### 风险全景图

```
┌─────────────────────────────────────────────────┐
│              电销外呼供应商管理风险全景             │
└─────────────────────────────────────────────────┘
        │
        ├─ 【话术红线违规】坐席违规话术、过度承诺、误导用户
        ├─ 【分包转包】供应商违规分包、转包赚取差价
        ├─ 【现场管理不达标】软硬件差、人员配置缺、制度不完善
        ├─ 【人员不合格】招到不合适的人、培训不足
        ├─ 【规模失控】站点多、精力顾不过来、侥幸心理、随意新增站点
        └─ 【清退舆情】清退过程纠纷、投诉升级
```

---

## 二、风险产生的原因

| 风险类型 | 产生原因 |
|----------|---------|
| **话术红线违规** | 坐席不懂规矩、缺乏培训、侥幸心理 |
| **分包转包** | 供应商想赚取差价、监管不到位 |
| **现场管理不达标** | 配不齐（组长/管理/质检）、投入不足 |
| **人员不合格** | 招人标准低、培训跟不上、京东不关注 |
| **规模失控** | 站点多管不过来、随意新增站点、缺乏审批 |
| **清退舆情** | 清退过程不透明、要求不明确、供应商不满 |

## 三、我们的管控方式（事前-事中-事后）

### 事前：明确红线

| 管控动作 | 管控的风险 | 具体做法 |
|----------|-----------|---------|
| **合同约定** | 分包转包风险 | 合同明确不许分包转包、违约严重处罚 |
| **站点设立审批** | 规模失控风险 | 不允许随意更换或新增站点，现有站点才可作业 |
| **人员标准** | 人员不合格风险 | 上岗前/上岗中/上岗后标准、培训要求、京东关注点 |
| **准入审核** | 各类风险源头 | 审核供应商的现场、人员、制度、组织配置 |

### 事中：过程监控

| 管控动作 | 管控的风险 | 具体做法 |
|----------|-----------|---------|
| **巡检机制** | 各类过程风险 | 落地到坐席、看流水、访谈、现场检查（17次/年） |
| **摄像头监控** | 分包转包风险 | 远程监控分包转包行为 |
| **京东要求关注** | 管理松懈风险 | 倒逼供应商规范管理（京东关注→供应商就重视） |

### 事后：平稳退出

| 管控动作 | 管控的风险 | 具体做法 |
|----------|-----------|---------|
| **递进式清退** | 清退舆情风险 | 警告→约谈→限制→清退，过程透明、避免纠纷 |

## 四、评委关心的维度（HR/综合管理/法务）

### HR会关心
- 人员怎么管？（标准、培训、晋升）
- 规模这么大，怎么确保人不出问题？

### 综合管理会关心
- 供应商违规了怎么办？（巡检、处罚）
- 怎么避免纠纷？（合同约定、清退规则）

### 法务会关心
- 合同怎么约定风险？
- 出问题了怎么处理？（递进式清退）

## 五、我们的核心价值

### 从"人治"到"法治"

| 之前（人治） | 现在（法治） |
|------------|-----------|
| 标准不一 | 4条业务线管理标准统一 |
| 靠个人经验 | 11项制度明确规则 |
| 容易扯皮 | RACI责任地图职责清晰 |
| 离开一个人体系就转不了 | 管理可复制、可传承 |

### 风险"可控、可见、能处理"

| 状态 | 体现 |
|------|------|
| **可控** | 事前明确红线、供应商知道什么不能做 |
| **可见** | 巡检17次、红灯4家100%整改闭环 |
| **能处理** | 递进式清退、清退4家零纠纷 |
