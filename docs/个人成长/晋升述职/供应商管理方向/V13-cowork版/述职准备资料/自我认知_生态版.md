# 自我认知：我建立了一个什么样的供应商管理生态？

---

## 核心观点

我管理的不是33家供应商、3239人、17个产品、4大业务线。

**我建立的是一个能够自我运转的供应商管理生态系统。**

这个生态系统：
- **有输入**：新的供应商、新的业务、新的需求进来
- **有机制**：通过制度、标准、流程自动运转
- **有输出**：供应商产生业绩、控制风险、持续优化
- **有进化**：经验被总结、被固化、被复制

**如果没有我，这个系统也能运转。**

---

## 第一层：我建立的顶层架构是什么？

### 生态系统的三层架构

```
┌─────────────────────────────────────────────────────────┐
│                   第一层：价值层                          │
│    多元竞争 + 自主可控 → 成本优化 + 效率提升 + 风险可控   │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                   第二层：机制层                          │
│    准入机制 + 使用机制 + 培育机制 + 淘汰机制 + 协调机制     │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                   第三层：执行层                          │
│    制度体系 + 标准体系 + 流程体系 + 工具体系              │
└─────────────────────────────────────────────────────────┘
```

### 这个架构的运转逻辑

**输入端：**
- 新供应商想进来 → 进入"准入机制"
- 新业务要上线 → 进入"协调机制"
- 老供应商有问题 → 进入"培育机制"或"淘汰机制"

**机制层：**
- 每个机制都有明确的标准、流程、责任人
- 按标准执行、按流程操作、按结果奖惩

**输出端：**
- 供应商产生业绩
- 供应商控制风险
- 供应商持续优化

**进化端：**
- 经验被总结 → 写成文档 → 变成制度 → 成为标准 → 能够复制

## 第二层：我建立的五大机制是什么？

### 机制1：准入机制（选）

**这个机制解决什么问题？**
- 谁能进来？
- 怎么进来？
- 多快能进来？

**机制的构成：**

**输入标准（9维度）：**
1. 过往业绩
2. 同类型业务对标
3. 管理方式
4. 人员数量
5. 公司背景
6. 历史交付风格
7. 配合的商务
8. 内部团队架构
9. 资质和合规

**决策流程：**
- 寻源储备 → 标准评估 → 现场验厂 → 审批上线

**时间标准：**
- 审批：1-2天（之前4-5天）
- 引入：15天（之前30天）

**输出结果：**
- 符合标准的供应商进入"使用机制"
- 不符合标准的供应商淘汰

**机制的价值：**
- 不是我一个人说了算，是标准说了算
- 不是拍脑袋决策，是有流程可循
- 不是每个都要从头开始，有储备池

---

### 机制2：使用机制（用）

**这个机制解决什么问题？**
- 给谁？给多少？
- 怎么给？什么时候给？
- 怎么评价？怎么奖惩？

**机制的构成：**

**业务分类体系：**
- 拉新型业务 → 找擅长拉新的供应商
- 复购型业务 → 找擅长复购的供应商
- 创新型业务 → 找学习能力强的供应商
- 大额型业务 → 找擅长大额的供应商

**供应商标签体系：**
- 业务类型标签（拉新/复购/创新/大额/企微）
- 能力标签（头部/中部/尾部）
- 配合度标签（高/中/低）
- 稳定性标签（稳定/波动/不稳定）
- 潜力标签（高/中/低）

**分量打分体系（5维）：**
- 业绩（30%）
- 质检客诉（25%）
- 上人数字（15%）
- 配合度（15%）
- 战略打分（15%）

**赛马机制：**
- 每月排名，看谁好谁差
- 好的有奖励：多给量、给新项目
- 差的有处罚：少给量、约谈、警告

**输出结果：**
- 表现好的供应商 → 进入"培育机制"或保持"使用机制"
- 表现差的供应商 → 进入"淘汰机制"

**机制的价值：**
- 不是凭感觉分配，是凭数据分配
- 不是一成不变，是动态调整
- 不是我有偏心，是公平竞争

---

### 机制3：培育机制（育）

**这个机制解决什么问题？**
- 供应商不会做怎么办？
- 供应商做不好怎么办？
- 供应商想优化怎么办？

**机制的构成：**

**培训体系（京东适应体系）：**
- 京东怎么看事情
- 京东在乎什么
- 京东怎么工作
- 以前踩过什么坑
- 怎么快速出产能

**资源倾斜体系：**
- 表现好的供应商 → 给好名单、给支持
- 表现差的供应商 → 给差名单、给压力

**反馈体系：**
- 定期沟通：告诉供应商哪里好、哪里差
- 数据透明：让供应商看到自己的排名、问题、进步空间
- 问题诊断：帮助供应商分析问题、给出建议

**输出结果：**
- 进步的供应商 → 保持"使用机制"或进入"赛马机制"上位
- 没进步的供应商 → 进入"淘汰机制"

**机制的价值：**
- 不是只看结果，是帮供应商达成结果
- 不是只惩罚，是帮助和引导
- 不是一次性合作，是长期共赢

---

### 机制4：淘汰机制（汰）

**这个机制解决什么问题？**
- 不合适的供应商怎么退出？
- 怎么让供应商心甘情愿退出？
- 怎么避免退出时的纠纷？

**机制的构成：**

**递进式清退规则（4级）：**
1. 警告：发现问题，口头警告，要求整改，给3-7天时间
2. 约谈：警告后没改善，约谈商务，要求给出整改方案
3. 限制新进：约谈后没改善，限制新业务，只能做存量
4. 清退：限制后还没改善，清退

**两难选择策略：**
- 闹 → 被京东拉黑 → 行业里有影响
- 走 → 日后好相见 → 不加黑名单

**供应商的理性选择：**
- 与其闹，不如走
- 与其被动清退，不如自己申请

**输出结果：**
- 清退的供应商 → 被黑名单记录，不再合作
- 腾出的位置 → 进入"准入机制"，引入新供应商

**机制的价值：**
- 不是强制清退，是让供应商自己选择
- 不是有纠纷，是零纠纷
- 不是情绪化，是规范化

---

### 机制5：协调机制（跨部门协作）

**这个机制解决什么问题？**
- 业务方要上新项目怎么办？
- 供应商说名单质量不好怎么办？
- 供应商说业绩下滑怎么办？
- 供应商说涨价怎么办？

**机制的构成：**

**需求协调流程：**
1. 业务方提出需求（新项目、新业务、新资源）
2. 我评估需求（是什么业务？需要多少人？什么时候上线？）
3. 我匹配供应商（谁适合？谁有能力？谁愿意接？）
4. 我决策分配（给谁？给多少？什么时候给？）
5. 我跟进结果（上线情况、业绩表现）

**问题协调流程：**
1. 发现问题（供应商反馈、业务方反馈、数据监控）
2. 分析问题（是个案还是普遍？是表层还是深层？是我能解决还是需要协调？）
3. 解决问题（如果是名单问题→协调业务方；如果是供应商问题→要求优化；如果是系统问题→找技术方）
4. 跟进结果（问题解决了吗？效果如何？）

**价值协调流程：**
1. 公司要求降价/供应商要求涨价
2. 我评估合理性（市场行情、供应商表现、业务数据）
3. 我做决策（合理→考虑；不合理→拒绝）
4. 我谈条件（如果降价/涨价，能不能给更好的业绩？）

**输出结果：**
- 协调成功 → 问题解决，业务继续
- 协调失败 → 进入"淘汰机制"或寻找替代方案

**机制的价值：**
- 不是我一个人扛，是有流程可循
- 不是凭感觉决策，是有数据支撑
- 不是推诿扯皮，是有人负责

## 第三层：我建立的四大体系是什么？

### 体系1：制度体系

**解决什么问题？**
- 让供应商知道底线在哪里
- 让管理有据可依
- 让结果可预期

**包含的制度：**
1. 《职场现场规范》
2. 《违规处罚条例》
3. 《巡检标准》
4. 《分量标准》
5. 《现场要求及处罚标准》

**制度的价值：**
- 不是我说了算，是制度说了算
- 不是临时决策，是有据可依
- 不是情绪化，是规范化

---

### 体系2：标准体系

**解决什么问题？**
- 让检查有标准
- 让评价有依据
- 让改进有方向

**包含的标准：**

**巡检标准（12项）：**
1. 空间布置（龙虎榜、公开业绩、话术管理）
2. 人员状态（坐席有没有在打电话、玩手机、睡觉）
3. 薪资情况（给供应商100元，员工拿多少？必须≥60元）
4. 招聘情况（招了什么人？从哪里招？有没有用京东名义？）
5. 品牌使用（有没有滥用logo？有没有违规宣传？）
6. 管理情况（主管有没有在管？有没有培训？有没有复盘？）
7. 活动激励（供应商有没有激励？坐席知不知道？）
8. 作息时间（每天工作多久？加班怎么算？）
9. 培训方式（有没有培训？培训什么？）
10. 招聘方式（怎么招的？渠道是什么？）
11. 品牌滥用（有没有滥用京东名义？）
12. 质检合规（有没有质检？质检合格率多少？）

**评价标准（5维打分）：**
- 业绩（30%）+ 质检客诉（25%）+ 上人数字（15%）+ 配合度（15%）+ 战略打分（15%）

**清退标准（4级）：**
- 警告 → 约谈 → 限制新进 → 清退

**标准的价值：**
- 不是凭感觉检查，是按标准检查
- 不是凭印象评价，是按数据评价
- 不是凭喜好清退，是按规则清退

---

### 体系3：流程体系

**解决什么问题？**
- 让做事有步骤
- 让协作有顺序
- 让责任有划分

**包含的流程：**

**供应商引入流程：**
寻源储备 → 标准评估 → 现场验厂 → 审批上线

**供应商管理流程：**
业务分类 → 供应商标签 → 分量打分 → 赛马排名 → 动态调整

**供应商清退流程：**
发现问题 → 警告 → 约谈 → 限制新进 → 清退

**巡检流程：**
制定计划 → 现场检查（按12项标准）→ 发现问题 → 发邮件 → 要求整改 → 跟进反馈

**问题协调流程：**
发现问题 → 分析问题 → 解决问题 → 跟进结果

**流程的价值：**
- 不是临时抱佛脚，是有流程可循
- 不是责任不清，是有人负责
- 不是重复犯错，是持续优化

---

### 体系4：工具体系

**解决什么问题？**
- 让经验可复制
- 让培训可标准化
- 让新供应商快速适应

**包含的工具：**

**《京东适应指南》：**
- 京东怎么看事情
- 京东在乎什么
- 京东怎么工作
- 以前踩过什么坑
- 怎么快速出产能

**《巡检清单》：**
- 按照标准制定检查项
- 每次按清单检查
- 发现问题记录问题

**《问题反馈表》：**
- 供应商反馈问题
- 我分析问题
- 给出解决建议

**工具的价值：**
- 不是靠我一对一教，是靠工具批量教
- 不是每次都重新讲，是有工具可查
- 不是经验在我脑子里，是经验在文档里

## 第四层：这个生态系统的核心能力是什么？

### 核心能力1：自我运转能力

**这个系统没有我也能运转吗？**
- 能。因为有标准、有流程、有制度、有工具
- 任何人按照这个系统来做，都能达到80%的效果
- 我的作用是优化系统、处理异常、持续进化

**怎么证明？**
- 京东适应体系：新供应商来了，按照指南做，就能快速适应
- 巡检标准：任何人按照标准检查，都能发现问题
- 清退规则：任何人按照规则执行，都能零纠纷清退

---

### 核心能力2：自我优化能力

**这个系统会自己进化吗？**
- 会。因为每次发现问题 → 分析问题 → 解决问题 → 固化经验
- 固化的经验变成制度、标准、流程、工具
- 下次遇到同样问题，直接用新制度、新标准、新流程、新工具

**怎么证明？**
- 京东适应体系：从无到有，帮助新供应商快速适应
- 巡检标准：从不统一到统一，每次检查都更有针对性
- 清退规则：从没有到有，从有纠纷到零纠纷

---

### 核心能力3：价值创造能力

**这个系统只解决管理问题，还是也能创造价值？**
- 也能创造价值。因为管理好了，供应商就能产出更好

**怎么证明？**
- 成本优化：单价下降15%，每月降本100万
- 效率提升：人效提升120%，头尾差距缩小20%
- 风险控制：全年零纠纷，红灯全部化解
- 业务支撑：17个产品、4大业务线都能运转

---

### 核心能力4：可复制能力

**这个系统能复制到其他业务吗？**
- 能。因为这是方法论，不是具体操作

**怎么证明？**
- 京东适应体系：可以复制到任何新供应商
- 巡检标准：可以复制到任何业务线
- 清退规则：可以复制到任何管理场景
- 五大机制：可以复制到任何供应商管理场景

**这个基础设施的价值：**

- 不是不可复制的，是可复制的

## 第七层：我的不可替代性是什么？

### 如果没有我，会发生什么？

**会发生的：**
- 系统还能运转，因为有标准、有流程、有制度、有工具
- 业务不会停，至少两个人可以接手

**但是：**
- 系统不会自我进化了
- 遇到新问题，不知道怎么优化了
- 遇到异常情况，不知道怎么处理了
