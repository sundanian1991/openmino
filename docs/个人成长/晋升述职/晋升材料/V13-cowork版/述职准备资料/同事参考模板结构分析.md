## 一、整体框架结构

```
┌─────────────────────────────────────────┐
│  岗位职责和认知（一句话概括）           │
├─────────────────────────────────────────┤
│  贡献1                                  │
│  ├─ 结果1（量化数据）                   │
│  ├─ 挑战（具体问题）                    │
│  └─ 核心举措：                         │
│      ├─ 问题1：[关键词]                │
│      │   ├─ 【动作标签】动作描述 → 量化  │
│      │   └─ 【动作标签】动作描述 → 量化  │
│      └─ 问题2：[关键词]                │
│          ├─ 【动作标签】动作描述 → 量化  │
│          └─ 【动作标签】动作描述 → 量化  │
├─────────────────────────────────────────┤
│  贡献2（同样结构）                      │
│  └─ ...（可继续贡献3、4...）           │
└─────────────────────────────────────────┘
```

## 二、亮点提炼

| 维度 | 特点 | 示例 |
|------|------|------|
| **问题导向** | 先说挑战，再说如何解决 | "挑战：电销黑名单静态管控..." |
| **关键词提炼** | 四字概括核心问题 | "盘存量"、"获新客"、"客户洞察-打法重构" |
| **【】标签** | 突出核心动作类型 | 【黑名单释放】、【频控策略】 |
| **箭头指向** | 清晰展示因果关系 | "可经营名单+51万（↑30%）" |
| **量化全覆盖** | 每个动作都有数据支撑 | GMV↑106%、费效↓39%、转化率+2.5倍 |

## 三、格式规范

### 1. 层级关系
- 用缩进表示层级（2个空格一层）
- 使用 `└─` 表示下级

### 2. 符号使用

| 符号 | 用途 | 示例 |
|------|------|------|
| `【】` | 标注动作类型/核心动作 | 【黑名单释放】 |
| `→` | 表示因果关系 | 拨打深度+20% |
| `↑` `↓` | 表示增长/下降 | ↑30%、↓39% |
| `+` | 表示增加 | +51万、+2.5倍 |

### 3. 数据呈现
- **核心数据在前**，变化幅度在括号内
- 格式：`数据值（变化幅度%）`
- 例：`GMV同比↑106%`、`费效同比↓39%`

## 四、字数限制

| 部分 | 字数 |
|------|------|

| 单个贡献（含结果+挑战+举措） | ~280字 |
| **总计（2-3个贡献）** | **~600-900字** |

## 五、写作要点

### 1. 岗位职责和认知
- **一句话概括**：岗位通过什么手段，达成什么目标，沉淀什么价值
- 例：通过制定电销经营策略，推动企金电销业务规模化增长，并沉淀可复用的策略方法论

### 2. 贡献结构（固定三要素）
| 要素 | 内容要求 |
|------|----------|
| **结果** | 量化数据（GMV、转化率、效率等） |
| **挑战** | 具体问题（痛点、瓶颈） |
| **核心举措** | 分2-3个问题，每个问题下2个具体动作 |

### 3. 问题与举措
- 每个贡献下分 **2-3个问题**
- 每个问题下 **2个具体动作**
- 每个动作必须带 **量化结果**
- 用 **四字关键词** 概括问题（如"盘存量"、"获新客"）

### 4. 动作描述格式
```
【动作标签】具体动作描述 → 量化结果（变化幅度）
```

例：
```
【黑名单释放】B端专属黑名单动态分级和轮转机制，实现动态释放可经营名单+51万（↑30%）
```

## 六、原始参考模板（全文）

### 岗位职责和认知

通过制定电销经营策略，推动企金电销业务规模化增长，并沉淀可复用的策略方法论。

---

### 贡献1：通过策略创新和模式升级，实现成本、效率、体验优化

**结果1**：企业主贷GMV同比↑106%，费效同比↓39%，大盘占比30%+

**挑战**：电销黑名单静态管控存客无法触达、新客尾部名单经营低效

**核心举措**：

**1、盘存量**：创新"动态运营+专属频控"组合策略，累计贡献GMV93亿

- 【黑名单释放】B端专属黑名单动态分级和轮转机制，实现动态释放可经营名单+51万（↑30%）
- 【频控策略】洞察B端用户特征，分客群制定差异化频控策略，拨打深度+20%

**2、获新客**：通过经营模式变革，人机协同链路优化，有效提升尾部拉新效率，累计贡献新户11.5万，同比+5.8倍，GMV64亿

- 【经营模式升级】围绕B端产品和用户特性，将经营模式由"纯机"→"人机"深度经营，名单转化率+2.5倍
- 【人机能力提升】制定名单分层和差异化触达策略，提升人机协同效率，转人工率+11%、转人漏损率20%→6%

---

### 贡献2：通过策略沉淀和快速复用，0-1驱动金企融合新业务跨越式增长

**结果2**：金企融合GMV119亿，较金条经营余额+20%，荣获金科H1交叉智多星奖

**挑战**：名单量级有限，历史人机模式存在规模化效率不足，需在2个月内从0-1快速跑通新项目，并实现客群余额提升15亿

**核心举措**：

**1、客户洞察-打法重构**

- 【经营模式调整】针对痛点，快速将人机经营转向"人工+企微"深度经营，转化率较人机+33%
- 【策略复制】已验证策略快速复用，1个月快速打通电销流程闭环，H1余额增幅超10%，支撑H2项目开量

**2、体系构建-扩量稳定增长**

随着H2项目开量，跨部门高效协同资源实现项目规模化。依托稳定有效的策略底座，扩量期间仍能持续提升名单效率+30%，支持项目电销规模14亿→105亿，成功完成从"试点验证"到"规模增长"的跨越
