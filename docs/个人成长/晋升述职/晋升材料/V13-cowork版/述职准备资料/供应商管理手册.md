# 供应商管理手册

---

## 手册说明

**手册定位：** 本手册是金科电销供应商管理的指导性文件，定义了供应商管理的框架、标准、流程、工具。

**适用范围：** 4条业务线、17个小产品、33家供应商、3239人的电销外呼团队。

**手册结构：**
- **母题**：大的管理框架，相对稳定
- **子议题**：具体的管理事项，可与时俱进

**使用方式：**
- 作为日常管理的参考
- 作为新人培训的教材
- 作为制度优化的基础

## 第一部分：管理顶层设计

### 母题1.1：战略定位

**子议题1.1.1：生态定位**
- **关键词**：多元竞争 + 自主可控
- **核心逻辑**：不是单一依赖，而是建立"多家供应商竞争+自建团队对标"的管理体系
- **具体动作**：
  - 引入竞争：保持8-10家新供应商在储备池
  - 建立标杆：自建团队作为对配件尺
  - 动态调整：老供应商表现好就多用，表现差就少用

**子议题1.1.2：价值主张**
- **关键词**：成本优化 + 效率提升 + 风险可控
- **核心逻辑**：管理不只是管人，更是创造价值
- **具体动作**：
  - 成本优化：通过自建标杆倒逼供应商降价
  - 效率提升：把合适供应商放在合适的地方
  - 风险可控：全过程防控，从"事后救火"到"过程防控"

**子议题1.1.3：管理哲学**
- **关键词**：拿捏 + 对标 + 生态
- **核心逻辑**：让供应商在有限奖励下依然愿意投入
- **具体动作**：
  - 拿捏：胡萝卜（分量）+ 大棒（合规+分量削减）
  - 对标：自建团队作为效能基准
  - 生态：从甲乙方关系到合作伙伴关系

### 母题1.2：管理全景图

**子议题1.2.1：三层管理架构**

```
第一层：价值层
    多元竞争 + 自建标杆 → 成本优化 + 效率提升 + 风险可控

第二层：机制层
    准入机制 + 使用机制 + 培育机制 + 淘汰机制 + 协调机制

第三层：执行层
    制度体系 + 标准体系 + 流程体系 + 工具体系
```

**子议题1.2.2：六大管理场景**

| 场景 | 触发条件 | 处理机制 | 预期结果 |
|------|---------|---------|---------|
| **新供应商引入** | 业务扩容、供应商淘汰 | 准入机制 | 15天完成引入 |
| **新项目上线** | 业务方提出需求 | 协调机制 | 快速匹配供应商 |
| **供应商排名下滑** | 数据监控发现 | 使用机制+培育机制 | 帮助优化或清退 |
| **供应商违规** | 巡检发现、客诉 | 淘汰机制 | 递进式清退 |
| **供应商要求涨价** | 市场变化、成本上涨 | 协调机制 | 评估合理性 |
| **线路中断** | 突发情况 | 协调机制 | 快速恢复 |

**子议题1.2.3：五大管理能力**

| 能力 | 定义 | 证明方式 |
|------|------|---------|
| **拿捏能力** | 让供应商在有限奖励下依然愿意投入 | 老供应商主动优化 |
| **分量能力** | 把合适的供应商放在合适的地方 | 头尾差距缩小 |
| **对标能力** | 通过自建团队建立效能基准 | 供应商接受降价 |
| **培育能力** | 帮助供应商快速适应京东 | 新供应商快速出产能 |
| **进化能力** | 把经验变成制度，持续优化 | 系统可复制 |

## 第二部分：供应商全生命周期管理

### 母题2.1：准入管理

**子议题2.1.1：准入标准（9维度）**

| 维度 | 评估内容 | 权重 |
|------|---------|------|
| **过往业绩** | 以前做过什么？业绩怎么样？ | 高 |
| **同类型业务对标** | 有没有做过类似业务？ | 高 |
| **管理方式** | 怎么管理团队？有没有SOP？ | 中 |
| **人员数量** | 有多少人？能不能快速上人？ | 中 |
| **公司背景** | 是什么公司？有没有实力？ | 中 |
| **历史交付风格** | 以前配合度怎么样？ | 中 |
| **配合的商务** | 商务对接人怎么样？ | 低 |
| **内部团队架构** | 团队架构是否合理？ | 低 |
| **资质和合规** | 有没有资质？有没有违规记录？ | 一票否决 |

**子议题2.1.2：准入流程**

```
寻源储备 → 标准评估 → 现场验厂 → 审批上线
```

**关键动作：**
- **寻源储备**：日常沟通，了解最新案例，储备10-15家潜在供应商
- **标准评估**：按9维度打分，达到阈值进入下一环节
- **现场验厂**：检查场地、人员、管理，是否符合标准
- **审批上线**：线上化审批，1-2天完成

**子议题2.1.3：准入时间标准**

| 环节 | 之前 | 现在 | 提效 |
|------|------|------|------|
| 审批 | 4-5天 | 1-2天 | 50%+ |
| 引入 | 30天 | 15天 | 50% |

**子议题2.1.4：储备池管理**

**关键词**：寻源、分类、沟通、更新

**具体动作：**
- 每周/每月与潜在供应商沟通
- 了解最新案例更新
- 按业务类型分类（拉新型/复购型/创新型/大额型/企微型）
- 按能力分类（头部/中部/尾部）
- 按配合度分类（高/中/低）
- 保持10-15家储备池

### 母题2.2：使用管理

**子议题2.2.1：业务分类体系**

| 业务类型 | 特点 | 管理重点 | 适合的供应商 |
|---------|------|---------|-------------|
| **拉新型** | 需要大量新客、转化率要求高 | 拉新数量、新客质量、转化率 | 擅长拉新、有名单资源、能快速上人 |
| **复购型** | 需要老客复购、服务质量要求高 | 复购率、客诉率、满意度 | 擅长复购、服务意识好、能维护客户 |
| **创新型** | 新模式、新流程、不确定因素多 | 适应速度、问题反馈、优化建议 | 学习能力强、配合度高、愿意尝试 |
| **大额型** | 客单价高、转化周期长 | 转化率、风控能力、跟进能力 | 擅长大额、稳定、有经验 |
| **企微型** | 需要企微运营、客户关系维护 | 加微率、响应速度、维护质量 | 擅长企微、沟通能力强 |

**子议题2.2.2：供应商标签体系**

**标签类型：**
1. **业务类型标签**：拉新型/复购型/创新型/大额型/企微型
2. **能力标签**：头部/中部/尾部
3. **配合度标签**：高配合/中配合/低配合
4. **稳定性标签**：稳定/波动/不稳定
5. **潜力标签**：高潜力/中潜力/低潜力

**具体动作：**
- 每个供应商有5个标签
- 标签动态更新（每月/每季度）
- 标签作为分量决策的依据

**子议题2.2.3：分量打分体系（5维）**

| 维度 | 权重 | 评估内容 | 数据来源 |
|------|------|---------|---------|
| **业绩** | 30% | GMV、转化率、名单产能 | 数据报表 |
| **质检客诉** | 25% | 客诉率、质检合格率、合规率 | 质检系统 |
| **上人数字** | 15% | 能上多少人、上人速度 | 人事报表 |
| **配合度** | 15% | 响应速度、配合程度、沟通质量 | 主观评价+客观表现 |
| **战略打分** | 15% | 扶持潜力、学习能力、额外技能 | 主观评价 |

**子议题2.2.4：赛马机制**

**关键词**：每月排名、动态调整、奖惩分明

**具体动作：**
- 每月排名，看谁好谁差
- 好的奖励：多给量、给新项目
- 差的处罚：少给量、约谈、警告
- 头部和尾部重点管理
- 中间供应商可以激励成头部

**子议题2.2.5：动态调整策略**

**关键词**：不都给、也不一直不给、让供应商保持期待

**具体动作：**
- 表现好的供应商：多给量、给新项目、鼓励扩大
- 表现差的供应商：少给量、约谈、警告、限制新进
- 表现进步的供应商：鼓励、给机会、帮助上位
- 表现退步的供应商：施压、要求整改、进入淘汰流程

### 母题2.3：培育管理

**子议题2.3.1：京东适应体系**

**关键词**：培训、文档、答疑

**《京东适应指南》内容：**
- 京东怎么看事情
- 京东在乎什么
- 京东怎么工作
- 以前踩过什么坑
- 怎么快速出产能

**具体动作：**
- 新供应商进来，先培训一次
- 发送《京东适应指南》
- 供应商有问题，随时解答
- 定期更新指南内容

**子议题2.3.2：资源倾斜体系**

**关键词**：胡萝卜、大棒

**胡萝卜（给资源）：**
- 表现好的供应商 → 给好名单、给支持
- 表现差的供应商 → 给差名单、给压力

**大棒（减资源）：**
- 表现差的供应商 → 少给量、约谈、警告
- 严重问题的供应商 → 限制新进

**子议题2.3.3：反馈体系**

**关键词**：定期沟通、数据透明、问题诊断

**具体动作：**
- 定期沟通：每月/每季度跟头部、尾部供应商沟通
- 数据透明：让供应商看到自己的排名、问题、进步空间
- 问题诊断：帮助供应商分析问题、给出建议

### 母题2.4：淘汰管理

**子议题2.4.1：递进式清退规则（4级）**

| 级别 | 触发条件 | 处理方式 | 预期结果 |
|------|---------|---------|---------|
| **警告** | 发现问题 | 口头警告，要求整改，给3-7天时间 | 问题解决 |
| **约谈** | 警告后没改善 | 约谈商务，要求给出整改方案 | 给出方案 |
| **限制新进** | 约谈后没改善 | 限制新业务，只能做存量 | 压力增大 |
| **清退** | 限制后还没改善 | 清退，心甘情愿自己申请 | 零纠纷 |

**子议题2.4.2：两难选择策略**

**关键词**：闹 vs 走

**具体动作：**
- 如果供应商闹 → 被京东拉黑 → 行业里有影响
- 如果供应商走 → 日后好相见 → 不加黑名单
- 让供应商自己选择：与其闹，不如走；与其被动清退，不如自己申请

**子议题2.4.3：黑名单管理**

**关键词**：行业威慑力

**具体动作：**
- 被清退的供应商记录在黑名单
- 告知供应商：如果闹，会被行业拉黑
- 利用黑名单的行业威慑力

### 母题2.5：协调管理

**子议题2.5.1：需求协调流程**

```
业务方提出需求 → 我评估需求 → 我匹配供应商 → 我决策分配 → 我跟进结果
```

**关键动作：**
- 评估需求：是什么业务？需要多少人？什么时候上线？
- 匹配供应商：谁适合？谁有能力？谁愿意接？
- 决策分配：给谁？给多少？什么时候给？
- 跟进结果：上线情况、业绩表现

**子议题2.5.2：问题协调流程**

```
发现问题 → 分析问题 → 解决问题 → 跟进结果
```

**关键动作：**
- 分析问题：是个案还是普遍？是表层还是深层？是我能解决还是需要协调？
- 解决问题：名单问题→协调业务方；供应商问题→要求优化；系统问题→找技术方
- 跟进结果：问题解决了吗？效果如何？

**子议题2.5.3：价值协调流程**

```
公司要求降价/供应商要求涨价 → 评估合理性 → 做决策 → 谈条件
```

**关键动作：**
- 评估合理性：市场行情、供应商表现、业务数据
- 做决策：合理→考虑；不合理→拒绝
- 谈条件：如果降价/涨价，能不能给更好的业绩？

## 第三部分：四大执行体系

### 母题3.1：制度体系

**子议题3.1.1：五项制度**

| 制度 | 解决什么问题 | 核心内容 |
|------|-------------|---------|
| **《职场现场规范》** | 现场管什么 | 空间布置、龙虎榜、公开业绩、话术管理 |
| **《违规处罚条例》** | 违规怎么办 | 处罚标准、处罚流程、处罚等级 |
| **《巡检标准》** | 巡检看什么 | 巡检清单、巡检频率、巡检要求 |
| **《分量标准》** | 怎么分配分量 | 分量规则、分量流程、分量调整 |
| **《现场要求及处罚标准》** | 员工待遇有什么要求 | 薪资保障、员工权益、激励要求 |

**子议题3.1.2：薪资保障标准**

**关键词**：给供应商100元，员工必须拿≥60元

**具体动作：**
- 检查供应商的薪资流水
- 如果员工拿不到60元，要求调整薪酬方案
- 加大员工激励/绩效薪酬
- 保证团队稳定性

### 母题3.2：标准体系

**子议题3.2.1：巡检标准（12项）**

| 项目 | 检查内容 | 判断标准 |
|------|---------|---------|
| **1. 空间布置** | 龙虎榜、公开业绩、话术管理 | 有龙虎榜、有公开业绩、有话术管理 |
| **2. 人员状态** | 坐席有没有在打电话、玩手机、睡觉 | 坐席在打电话、没有玩手机、没有睡觉 |
| **3. 薪资情况** | 给供应商100元，员工拿多少 | 员工必须拿≥60元 |
| **4. 招聘情况** | 招了什么人？从哪里招？ | 招聘符合要求、没有用京东名义 |
| **5. 品牌使用** | 有没有滥用logo？有没有违规宣传？ | 没有滥用logo、没有违规宣传 |
| **6. 管理情况** | 主管有没有在管？有没有培训？ | 主管在管、有培训、有复盘 |
| **7. 活动激励** | 供应商有没有激励？坐席知不知道？ | 有激励、坐席知道 |
| **8. 作息时间** | 每天工作多久？加班怎么算？ | 符合劳动法、加班有补偿 |
| **9. 培训方式** | 有没有培训？培训什么？ | 有培训、培训内容有效 |
| **10. 招聘方式** | 怎么招的？渠道是什么？ | 渠道合规、方式合法 |
| **11. 品牌滥用** | 有没有滥用京东名义？ | 没有滥用京东名义 |
| **12. 质检合规** | 有没有质检？质检合格率多少？ | 有质检、合格率达标 |

| 维度 | 权重 | 评估方式 |

| **上人数字** | 15% | 人事报表 |
| **配合度** | 15% | 主观评价+客观表现 |
| **战略打分** | 15% | 主观评价 |

**子议题3.2.3：清退标准（4级）**

| **约谈** | 警告后没改善 | 约谈商务，要求给出整改方案 | 给出方案 |
| **限制新进** | 约谈后没改善 | 限制新业务，只能做存量 | 压力增大 |
| **清退** | 限制后还没改善 | 清退，心甘情愿自己申请 | 零纠纷 |

### 母题3.3：流程体系

**子议题3.3.1：供应商引入流程**

```
寻源储备 → 标准评估 → 现场验厂 → 审批上线
```

**关键动作：**
- 日常沟通潜在供应商，了解最新案例
- 按9维度评估，达到阈值进入下一环节
- 检查场地、人员、管理
- 线上化审批，1-2天完成

**子议题3.3.2：供应商管理流程**

```
业务分类 → 供应商标签 → 分量打分 → 赛马排名 → 动态调整
```

**关键动作：**
- 给业务分类（拉新/复购/创新/大额/企微）
- 给供应商贴标签（5个标签）
- 按5维打分，决定分量
- 每月排名，动态调整

**子议题3.3.3：供应商清退流程**

```
发现问题 → 警告 → 约谈 → 限制新进 → 清退
```

**关键动作：**
- 发现问题，口头警告，给3-7天整改
- 警告后没改善，约谈商务，要求整改方案
- 约谈后没改善，限制新业务
- 限制后还没改善，清退（让供应商自己申请）

**子议题3.3.4：巡检流程**

```
制定计划 → 现场检查（按12项标准）→ 发现问题 → 发邮件 → 要求整改 → 跟进反馈
```

**关键动作：**
- 每季度制定巡检计划
- 按12项标准检查
- 发现问题当场指出
- 24小时内发邮件
- 要求3-7天整改
- 跟进整改情况

**子议题3.3.5：问题协调流程**

```
发现问题 → 分析问题 → 解决问题 → 跟进结果
```

**关键动作：**
- 发现问题来源（供应商/业务方/数据）
- 分析问题类型（个案/普遍/表层/深层）

## 第五部分：六大管理场景

### 母题5.1：场景1 - 新供应商引入

**场景描述：** 业务扩容或供应商淘汰，需要引入新供应商

**处理流程：**
```
确认需求 → 寻源储备 → 标准评估 → 现场验厂 → 审批上线 → 培训适应
```

**关键动作：**
- 确认需求：是什么业务？需要多少人？什么时候上线？
- 寻源储备：从储备池选择3-5家合适的
- 标准评估：按9维度打分
- 现场验厂：检查场地、人员、管理
- 审批上线：线上化审批，1-2天
- 培训适应：发送《京东适应指南》，帮助快速适应

**预期结果：** 15天完成引入，快速出产能

---

### 母题5.2：场景2 - 新项目上线

**场景描述：** 业务方要上新项目，需要找供应商承接

**处理流程：**
```
业务方提出需求 → 我评估需求 → 我匹配供应商 → 我决策分配 → 我跟进结果
```

**关键动作：**
- 评估需求：是什么业务？需要多少人？什么时候上线？
- 匹配供应商：谁适合？谁有能力？谁愿意接？
- 决策分配：给谁？给多少？什么时候给？
- 跟进结果：上线情况、业绩表现

**预期结果：** 快速匹配，快速上线，快速出产能

---

### 母题5.3：场景3 - 供应商排名下滑

**场景描述：** 数据监控发现供应商排名下滑

**处理流程：**
```
发现下滑 → 分析原因 → 区分处理 → 跟进结果
```

**关键动作：**
- 分析原因：是市场原因？名单原因？供应商原因？
- 区分处理：
  - 如果是外部原因 → 协调资源、调整策略
  - 如果是供应商原因 → 要求优化、进入培育或淘汰流程
- 跟进结果：排名是否回升

**预期结果：** 要么供应商优化，要么供应商淘汰

---

### 母题5.4：场景4 - 供应商违规

**场景描述：** 巡检发现或客诉发现供应商违规

**处理流程：**
```
发现违规 → 分析严重程度 → 区分处理 → 跟进结果
```

**关键动作：**
- 分析严重程度：轻微？严重？
- 区分处理：
  - 如果是轻微 → 当场指出，要求整改
  - 如果是严重 → 发邮件，要求3-7天整改
  - 如果是特别严重 → 进入淘汰流程
- 跟进结果：整改情况、是否复发

**预期结果：** 问题解决，或供应商淘汰

---

### 母题5.5：场景5 - 供应商要求涨价

**场景描述：** 市场变化或成本上涨，供应商要求涨价

**处理流程：**
```
供应商提出涨价 → 评估合理性 → 做决策 → 谈条件
```

**关键动作：**
- 评估合理性：市场行情？供应商表现？业务数据？
- 做决策：合理→考虑；不合理→拒绝
- 谈条件：如果涨价，能不能给更好的业绩？

**预期结果：** 合理的涨价会被考虑，不合理的会被拒绝

---

### 母题5.6：场景6 - 线路中断

**场景描述：** 突发情况，线路中断，影响业务

**处理流程：**
```
发现中断 → 快速评估 → 找资源 → 控客诉 → 跟进结果
```

**关键动作：**
- 快速评估：影响多大？多久能恢复？
- 找资源：找有指标但没用完的供应商
- 控客诉：动态剔除易诉人群，每天复盘
- 跟进结果：什么时候恢复？损失多少？

**预期结果：** 15天完成恢复（行业30天）
