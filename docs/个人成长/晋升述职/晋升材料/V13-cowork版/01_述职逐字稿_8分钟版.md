# 述职讲解框架（8分钟版）

> ⚠️ **重要提醒**：这不是逐字稿！请勿照念！
> 本文档用于讲解结构和逻辑提示，帮助你自由发挥、自信讲解。

---

## 使用说明

**✅ 应该这样做**：
- 看框架，用自己的话解释
- 每次讲都略有不同（更自然）
- 评委问你时，能灵活调整

**❌ 不要这样做**：
- 照着念（评委能听出来）
- 背下来（容易忘词、显得机械）
- 每次都一模一样（显得像排练的）

---

## 一、开场（约1分钟）

### 讲解框架

| 要素 | 讲解要点 | 记忆提示 |
|------|---------|---------|
| **我是谁** | 姓名、背景、转岗时间 | 简单介绍 |
| **做什么** | 33家供应商、3239人、4条线、17条业务 | **数字要准** |
| **怎么做** | "选、用、育、留、汰" | **五个字** |
| **绩效** | Q3 A、Q4 A、年度 A、2次匠心独运奖 | **证明做得好** |
| **结构** | 三部分：量化指标、三个案例、未来规划 | **预告结构** |

### 讲解思路

1. **先建立认知**：让评委知道你管的是什么（最大的销售渠道）
2. **再讲怎么管**："选用育留汰"五个字（评委会问，你有准备）
3. **然后证明**：绩效A + 奖项（建立信任）
4. **最后预告**：今天讲什么（让评委有预期）

### 可能的评委问题

| 问题 | 回答要点 |
|------|---------|
| "33家供应商怎么管理的？" | 全生命周期管理，后面案例会详细讲 |
| "为什么转岗？" | 京东云的经验可以复用到金科 |

## 二、量化指标（约40秒）

### 讲解框架

| 层级 | 讲解要点 | 为什么这样分 |
|------|---------|-------------|
| **第一层：业务增量** | 新增3个项目（金企融合、企贷大额、金条），___ 人，___ 亿GMV | 证明我创造了新价值 |
| **第二层：降本增效** | 自建团队标杆 → 单价下降___ %；引入周期 30天→15天 | 证明我提升了效率 |
| **第三层：风险防控** | 清退4家零纠纷；线路15天恢复（行业30天），避免80亿损失 | 证明我控制了风险 |

### 讲解思路

1. **为什么分三层？** —— 从"创造价值"到"提升效率"再到"控制风险"，层层递进
2. **每层讲什么？** —— 只讲结果数字，过程留到案例里讲
3. **评委可能问什么？** —— 准备好每个数字的来源和计算逻辑

### 可能的评委问题

| 问题 | 回答要点 |
|------|---------|
| "新增的3个项目是哪几个？" | 金企融合、企贷大额、金条（案例一会详细讲） |
| "单价下降怎么算的？" | 自建团队人效120%，供应商认可并接受降价 |
| "80亿损失怎么算的？" | 37亿/周 × 2周 ≈ 80亿 |

## 三、案例一：激活竞争（约80秒）

### 讲解框架（STAR逻辑）

| 要素 | 讲解要点 | 评委想听什么 |
|------|---------|-------------|
| **S（问题）** | 年初接手时"散僵盲弱乱"——四团队标准不一、老商依赖固化、新商成活率低 | 你发现的问题是不是真的问题？ |
| **T（目标）** | 建立全生命周期管理机制，同时自建团队 | 你的目标清晰吗？ |
| **A（做法）** | 选（30→15天）、用（赛马）、育（宿迁0→1）、汰（清退零中断） | 你的做法是不是有效？ |
| **R（结果）** | 新商75%进TOP3；清退零纠纷；宿迁120%业绩 | 结果能证明目标达成吗？ |
| **价值** | 从"依赖外包"到"自主可控" | 这件事的更大价值是什么？ |

### 讲解思路

1. **先讲问题** —— 让评委理解这件事的必要性
2. **再讲思路** —— "选、用、育、汰"是环环相扣的，不是孤立动作
3. **然后讲结果** —— 用数字证明（75%、120%、零纠纷）
4. **最后升华** —— 这不是简单的供应商管理，是渠道生态的重构

### 关键钩子解释

| 钩子 | 如果评委问，怎么解释 |
|------|---------------------|
| **30天→15天** | 准入评估、背景调查、合同签署3环节线上化，各砍一半 |
| **75%进TOP3** | 筛选标准5维度；老供应商看到新商表现，3家主动降价、2家申请整改 |
| **宿迁120%** | 投入___人、___月、___成本；人效达外包110%，倒逼供应商降价 |

## 四、案例二：合规风控（约90秒）

### 讲解框架（STAR逻辑）

| 要素 | 讲解要点 | 评委想听什么 |
|------|---------|-------------|
| **S（问题）** | 业务扩张到1000人规模，33家供应商、3000+人渠道池，缺标准、缺监控，风险累积 | 规模增长带来什么问题？ |
| **T（目标）** | 建立全周期合规管控体系，千人增长下零重大风险 | 目标是不是合理？ |
| **A（做法）** | 事前（五项制度）、事中（巡检17次、红灯4次）、事后（递进式清退，6家全闭环） | 体系是不是完整？ |
| **R（结果）** | 全年零纠纷；清退4家零中断；制度成部门标准 | 结果能证明目标达成吗？ |
| **价值** | 从"事后救火"到"过程防控" | 这件事的更大价值是什么？ |

### 讲解思路

1. **先讲挑战** —— 千人增长不是简单地加人，而是质量和风险的平衡
2. **再讲体系** —— "事前、事中、事后"三道防线，缺一不可
3. **然后讲效果** —— 巡检17次不是走过场，红灯4次全部化解是真的在防控
4. **最后升华** —— 合规不是成本负担，是业务增长的护城河

### 关键钩子解释

| 钩子 | 如果评委问，怎么解释 |
|------|---------------------|
| **千人增长零风险** | 100%培训、巡检全覆盖、红线熔断，质量不降 |
| **巡检17次、红灯4次** | 8项巡检清单；红灯24小时整改+7天复查 |
| **递进式清退** | 警告→约谈→限制→清退，每阶段7-30天，给机会也给底线 |
| **6家全闭环** | 最难的是___供应商，___问题，历时___天解决 |

## 五、案例三：线路应急（约80秒）

### 讲解框架（STAR逻辑）

| 要素 | 讲解要点 | 评委想听什么 |
|------|---------|-------------|
| **S（问题）** | 核心业务线路中断风险，日影响1300万呼叫、37亿周GMV；行业恢复30天；团队无专职负责人 | 这个问题有多紧急？ |
| **T（目标）** | 主动补位，15天恢复，建立长效机制 | 目标是不是有挑战？ |
| **A（做法）** | 链（链接运营商）、判（评估模型）、建（预案+标准化） | 你的做法是不是关键？ |
| **R（结果）** | 15天恢复（领先15天），避免80亿损失；预案成部门标准 | 结果能证明目标达成吗？ |
| **价值** | 从"被动救火"到"主动防控" | 这件事的更大价值是什么？ |

### 讲解思路

1. **先讲紧急性** —— 37亿周GMV、行业30天，让评委理解这不是小事
2. **再讲为什么是你** —— 团队无专职负责人，业务等不了流程，我主动补位
3. **然后讲关键动作** —— 运营商直连、决策授权、双班并行，这三个是领先行业的关键
4. **最后升华** —— 不是解决了这一次危机，而是建立了机制，下次不会再慌

### 关键钩子解释

| 钩子 | 如果评委问，怎么解释 |
|------|---------------------|
| **领先行业15天** | 运营商直连关系（同行要重新走商务）、决策授权（不用层层审批）、双班并行（24小时不停） |
| **37亿怎么算** | 日均1300万呼叫 × 接通率___% × 转化率___% × 客单价___元 = 日GMV，× 7天 |
| **主动补位** | 原属___团队负责但无专职负责人；业务紧迫等不了；我有运营商资源可以快速链接 |
| **预案成标准** | 复盘会→制度发文→培训落地，已触发___次，均有效避免损失 |

## 六、未来规划（约70秒）

### 讲解框架

| 要素 | 讲解要点 | 评委想听什么 |
|------|---------|-------------|
| **战略定位** | 从"价格压降"到"价值共创" | 你对未来的思考是不是有深度？ |
| **核心举措** | 输出京东体系、供应商分级、学习联盟 | 举措是不是具体可行？ |
| **三个突破** | 商务谈判→生态经营；单一→网络；价格→体系 | 突破点是不是有洞察？ |
| **量化目标** | 2026年Q2运转，覆盖10家 | 目标是不是清晰可衡量？ |
| **个人规划** | AI Agents探索、传帮带 | 你个人有什么成长计划？ |

### 讲解思路

1. **先讲方向** —— 不是继续压价，而是共建生态（体现战略思考）
2. **再讲怎么做** —— 输出能力、分级管理、联盟学习（体现可落地）
3. **然后讲目标** —— Q2运转、10家覆盖（体现可衡量）
4. **最后讲个人** —— AI Agents是技术趋势，传帮带是组织贡献

### 可能的评委问题

| 问题 | 回答要点 |
|------|---------|
| "BPO联盟和赛马机制矛盾吗？" | 不矛盾，是分层：核心10家进联盟深度合作，其他23家赛马优胜劣汰 |
| "AI Agents具体能做什么？" | 智能巡检（自动分析录音）、风险预警（监控波动）、知识沉淀（自动整理） |

## 七、收尾（约50秒）

### 讲解框架

| 要素 | 讲解要点 | 记忆提示 |
|------|---------|---------|
| **回顾** | 这一年做了三件事 | **三件事** |
| **第一件事** | 找对人 —— 选汰机制激活竞争，依赖→自主 | **对应案例一** |
| **第二件事** | 立好规矩 —— 全周期合规管控，救火→防控 | **对应案例二** |
| **第三件事** | 能力长在组织上 —— 自建团队+机制沉淀 | **对应案例一+三** |

### 讲解思路

1. **停顿2秒** —— 全场安静，制造仪式感
2. **逐字清晰** —— 找对人、立好规矩、让能力长在组织上（三个金句）
3. **微笑收尾** —— 自信、从容

## 九、讲解技巧提醒

### ✅ 应该这样做

| 技巧 | 具体做法 |
|------|---------|
| **眼神交流** | 讲的时候看着评委，不是看PPT/稿子 |
| **用图示** | 有PPT的话，用手势引导评委看重点 |
| **讲逻辑** | "因为...所以..."、"首先...其次...最后..." |
| **停顿** | 讲完一个部分，停顿1秒，让评委消化 |
| **重音** | 关键数字和关键词加重语气 |

### ❌ 不要这样做

| 错误 | 后果 |
|------|------|
| 照着念 | 评委能听出来，显得没准备 |
| 背诵式 | 容易忘词，显得不自然 |
| 语速太快 | 评委听不清，显得紧张 |
| 声音太平 | 没有重点，显得无聊 |
| 没有停顿 | 评委跟不上，显得赶时间 |

## 十、最后提醒

**核心原则**：基于材料，但不等于材料

- 材料是你的**笔记**
- 讲解是你的**理解**
- 评委想听的是**你的思考**，不是你的背诵

**每次练习时**：
1. 看一眼框架要点
2. 用自己的话解释
3. 每次略有不同（更自然）
4. 录音自查（语速、停顿、重音）

**正式演讲时**：
1. 放松（深呼吸3次）
2. 看着评委（眼神交流）
3. 自信（你是最了解这些事的人）
4. 享受（这是你展示自己的机会）
