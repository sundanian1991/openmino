01 — 核心判断 2026-03-29

执行侧 vs 决策侧权责对比

决策侧 — 策略组
权力 定价权、名单权、数据分配权
KPI 费效比优化
达成路径 压降价格 = 最快达成目标
结果：达成费效比目标，供应商流失风险转移至服务组
VS
风险转移
执行侧 — 服务组
权力 建议权
KPI 供应商稳定 / 生态健康
承担结果 供应商流失风险
困境：两头挤压 — 服务组要求质量↑，策略组压降价格↓
核心判断
策略组达成目标 → 供应商流失 → 服务组承担风险。权责不对等是结构性问题，不是沟通问题，也不是个人能力问题。

02 — KPI 冲突 2026-03-29

激励机制下的恶性循环

策略组KPI：费效比优化
压降价格是达成目标的最快路径，每压低1元单价，费效比立即改善
供应商激励扭曲
增长的产能被抹杀 → 不愿投入培训/质量 → 产能不增长 → 质量下降 → 策略组更有理由压价
供应商流失
质量下降成为"供应商能力不行"的证据，压价获得合理性支持，形成恶性循环
服务组承担风险
策略组收紧并达成目标，供应商流失导致年老师KPI受损，但定价权不在服务组手中

03 — 行为模式 2026-03-29

刘伟佳的责任边界管理

模式 行为表现 潜台词
责任边界管理 分析外化、模糊化处理 问题我看到了，但责任不在我
战略型表达 宏观把控，不陷细节 我在战略层面有贡献
服务型姿态 "支撑""帮助"，不强调权力 策略组是服务业务的角色
节奏控制 不承诺结果，只说方向 有节奏但不贸然承诺，留退路
洞察
不是恶意，是理性自保。在现有KPI下，刘伟佳的行为模式是"不背锅、有贡献、让别人动"。理解这一点，才能有效互动。

05 — 自我保护 2026-03-29

在权责不对等下不背锅

预防层：日常记录
• 重要沟通留痕
• 决策追溯文档
• 周报明确责任归属
• 汇报口径vs实际数据差异记录
应对层：事件发生时
• 快速提供真实数据
• 说明根本原因（策略决策导致）
• 列出已采取的应对措施
• 建议解决方案而非只提问题
红线清单
• 不揽责：策略组决策导致的问题，不是你的锅
• 不掩盖：数据美化要记录差异，保留证据
• 不指责：让数据自己说话，不攻击个人

06 — 触发条件 2026-03-29

什么条件下可能推动调整

高可能性
重大业务事故
头部供应商集体流失、交付危机 → 军哥被迫介入 → 你提供数据说明根因
⭐⭐⭐
高可能性
VP关注
VP询问供应商问题 → 向上追溯根源 → 准备结构化汇报材料
⭐⭐⭐
高可能性
数据无法美化
汇报口径vs实际差距太大 → 持续提供真实数据 → 让军哥自己发现问题
⭐⭐⭐
中可能性
协调成本过高
策略组与服务组频繁冲突 → 军哥频繁调解 → 记录冲突展示成本
⭐⭐
中可能性
新业务模式
BPO联盟、中腰部扶持 → 需要新的权责安排 → 小范围试点证明可行
⭐⭐
低可能性
完美论证/主动让权
依赖他人主动或外部偶然。不要期待自然调整，不要把精力放在推动调整上

07 — 行动框架 2026-03-29

时间线 + 联盟策略

短期（1-3月）
观察 + 记录
• 记录触发信号：流失率、冲突次数、数据差距
• 准备材料：流失根因分析、权责不对等说明
• 识别机会：VP关注点、业务节点、试点机会
中期（3-6月）
主动创造小机会
• 推动小范围试点（BPO联盟、中腰部扶持）
• 让试点需要"新的权责安排"
• 用试点成功证明"另一种方式可行"
长期（6-12月）
等待大机会
• 重大事故发生时快速响应
• VP关注时准备好材料
• 组织重组时抓住机会
与王易人的联盟评估
中-低
最佳策略：信息同步联盟 + 选择性议题合作

✅ 可以：定期交流信息、在供应商成本等具体问题上各自表达相似观点

❌ 避免：要求她公开对抗策略组、在公开场合表现太"一致"、强求表态

前提：先确认她的立场，不要假设
