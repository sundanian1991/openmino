# Logical Ordering for Categorical Data

Apply this skill when visualizing categorical data that has no natural sequence (e.g., product features, service types) and your goal is to highlight a specific analytical angle like user satisfaction, dissatisfaction, or feature usage.

## When to Use
- You are creating horizontal bar charts, categorical survey summaries, or feature satisfaction reports.
- Your categories are **non-ordinal** (e.g., "Email Support," "Live Chat," "Phone")—not age ranges, Likert scales, or other intrinsically ordered groups.
- You have a clear analytical focus: showing what’s most/least satisfying or most/least used.

## How to Execute
1. **Identify your primary story focus**: Choose one of:
   - *Most satisfied*: Sort by descending “Very/Completely satisfied” responses.
   - *Least satisfied*: Sort by descending “Not satisfied at all” responses.
   - *Least used*: Sort by descending “Have not used” responses.
   - *Comprehensive*: Combine insights in a single view using consistent ordering and visual emphasis (see references).

2. **If presenting multiple stories**:
   a. Establish a base visual with a consistent category order (e.g., based on your primary story).
   b. Keep this order unchanged across all related visuals to reduce cognitive load.
   c. Use color coding, callout boxes, and annotations—not reordering—to highlight secondary insights.

3. **For written reports** (not live presentations):
   - Create one integrated visualization.
   - Group bars by insight type using distinct colors.
   - Add descriptive text boxes to explain each highlighted pattern.

4. **Never reorder arbitrarily**—every sequence must reflect intentional analytical logic.

5. **Exception**: If categories have intrinsic order (e.g., age brackets, education levels), preserve their natural sequence regardless of story focus.

6. **Reinforce visual alignment**:
   - Align negative or unused segments to a common baseline (e.g., left edge).
   - Align positive segments to the opposite baseline (e.g., right edge) to support accurate comparison.
