# 定价委员会各角色职责一览

> 这个文档写给所有委员看，说明每个人在定价委员会里要做什么、有什么权力、彼此怎么配合。

## 一、委员会架构总览

                    │   数据科技业务部  │

### 1.2 权力层级

| 层级 | 角色 | 权力 |
|------|------|------|
| **L1 终审** | C2 | 最终决策权，每项调整必须审批 |
| **L2 评审** | 5人委员会 | 投票权（3/5或4/5通过），前置评估 |
| **L3 提案** | 策略组、量化代表 | 发起定价调整提案 |
| **L4 执行** | 供应商管理岗 | 组织会议、收集材料、追踪落地 |

## 二、各角色职责详解

### 2.1 秘书长（供应商管理岗）

#### 核心定位
> **委员会的"大脑和手脚"** — 组织、协调、记录、追踪

#### 具体职责

| 阶段 | 职责 | 产出 |
|------|------|------|
| **会前** | 1. 收集提案材料<br>2. 初审材料完整性<br>3. 安排会议时间/地点<br>4. 发送会议通知 | 会议议程<br>会议材料 |
| **会中** | 1. 主持会议<br>2. 记录会议讨论<br>3. 统计表决结果<br>4. 整理决策意见 | 会议纪要<br>表决结果 |
| **会后** | 1. 提交C2终审<br>2. 通知供应商生效<br>3. 追踪执行情况<br>4. 收集供应商反馈<br>5. 组织复盘 | 终审申请<br>生效通知<br>复盘报告 |
| **日常** | 1. 归档所有提案和决议<br>2. 维护机制文档<br>3. 处理供应商反馈申请<br>4. 协调各方沟通 | 委员会档案<br>机制更新 |

#### 时间投入预估
| 活动 | 频次 | 每次 | 月度合计 |
|------|------|------|----------|
| 会议组织 | 1次/月 | 120分钟 | 120分钟 |
| 材料初审 | 1-2次/月 | 30分钟 | 30-60分钟 |
| C2终审协调 | 1-2次/月 | 30分钟 | 30-60分钟 |
| 执行追踪 | 按需 | 30分钟/月 | 30分钟 |
| 供应商反馈处理 | 按需 | 30分钟/次 | 0-60分钟 |
| 复盘组织 | 0-2次/月 | 60分钟 | 0-120分钟 |
| **合计** | - | - | **3.5-6.5小时/月** |

### 2.2 业务督导代表

#### 核心定位
> **坐席声音的代表** — 确保定价调整不会伤害坐席利益

#### 具体职责

| 职责类型 | 具体内容 |
|----------|----------|
| **提案评估** | 评估定价调整对坐师收入的影响 |
| **会议发言** | 从坐席角度提供意见：收入变化、离职风险、管理难度 |
| **投票表决** | 根据坐席利益投票 |
| **落地支持** | 定价调整生效后，协调坐师管理 |

#### 关键问题清单
- 这次调整会让坐师收入增加还是减少？多少？
- 会不会影响坐师稳定性？离职率会不会上升？
- 坐师会不会有情绪？怎么安抚？

#### 时间投入预估
| 活动 | 频次 | 每次 | 月度合计 |
|------|------|------|----------|
| 委员会会议 | 1次/月 | 60-90分钟 | 60-90分钟 |
| 提案材料审阅 | 1-2次/月 | 15-30分钟 | 15-60分钟 |
| 复盘会议 | 0-2次/月 | 30-60分钟 | 0-120分钟 |
| **合计** | - | - | **1.5-4.5小时/月** |

### 2.3 量化代表

#### 核心定位
> **数据支撑的提供者** — 用数据说话，确保决策有依据

#### 具体职责

| 职责类型 | 具体内容 |
|----------|----------|
| **提案发起** | 可以发起定价调整提案（需策略组支持） |
| **数据测算** | 提供调整前后对比、ROI分析、影响测算 |
| **会议发言** | 从数据角度提供意见：测算逻辑是否合理、数据是否准确 |
| **投票表决** | 根据数据结论投票 |
| **复盘支持** | 提供复盘所需的数据分析 |

#### 关键问题清单
- 这次调整的测算逻辑对吗？数据准确吗？
- 调整前后的对比是什么？ROI如何？
- 影响测算覆盖了哪些维度？有没有遗漏？

#### 时间投入预估
| 活动 | 频次 | 每次 | 月度合计 |
|------|------|------|----------|
| 委员会会议 | 1次/月 | 60-90分钟 | 60-90分钟 |
| 数据测算 | 1-2次/月 | 30-60分钟 | 30-120分钟 |
| 提案材料审阅 | 1-2次/月 | 15-30分钟 | 15-60分钟 |
| 复盘数据分析 | 0-2次/月 | 30-60分钟 | 0-120分钟 |
| **合计** | - | - | **2-6.5小时/月** |

### 2.6 终审（数据科技业务部负责人 C2）

#### 核心定位
> **最终决策者** — 掌握最终决策权，确保符合业务战略

#### 具体职责

| 职责类型 | 具体内容 |
|----------|----------|
| **终审审批** | 每项定价调整的最终审批 |
| **战略把控** | 确保调整符合业务整体战略 |
| **风险控制** | 通过前置机制规避潜在风险 |
| **例外决策** | 处理委员会无法达成一致的情况 |

#### 审批重点
- 战略合理性：是否符合业务战略？
- 整体影响：对业务、供应商、坐师的整体影响？
- 风险控制：有没有识别风险？应对方案是什么？

## 三、委员会运作流程

### 3.1 定价调整流程总览

```
┌─────────────────────────────────────────────────────────────────┐
│                     阶段1：提案（T-5天）                         │
│  提案人：策略组/量化代表                                        │
│  产出：《定价调整提案》                                         │
│  秘书处：收集材料、初审完整性                                   │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                  阶段2：初审+评审（T-2天）                       │
│  秘书长：初审材料完整性                                         │
│  5人委员会：评审提案、发表意见、投票表决                        │
│  表决规则：常规3/5通过、重大4/5通过                             │
│  产出：会议纪要、表决结果                                       │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                     阶段3：终审（T-1天）                         │
│  审批人：C2                                                     │
│  产出：终审决议                                                 │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                   阶段4：生效（次月1日）                         │
│  秘书处：通知供应商、结算                                       │
│  服务组：系统配置、职场协调                                     │
│  业务督导：坐师沟通                                             │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                   阶段5：复盘（+30天）                           │
│  秘书处：组织复盘、收集数据                                     │
│  量化代表：提供数据分析                                         │
│  5人委员会：讨论效果、提出改进                                 │
│  产出：《定价调整复盘报告》                                     │
└─────────────────────────────────────────────────────────────────┘
```

---

### 3.2 表决规则

| 调整类型 | 审批人 | 通过条件 | 说明 |
|----------|--------|----------|------|
| **常规调整**（降幅≤15%） | 5人委员会 | 3/5通过 | 简单多数 |
| **涨价调整** | 5人委员会 | 3/5通过 | 与常规调整一致 |
| **重大调整**（降幅>15%） | 委员会 + C2终审 | 4/5通过 + C2终审 | 超级多数 + 终审 |
| **每项调整** | C2 | C2审批 | 最终决策权 |

---

### 3.3 回避原则

**什么情况需要回避？**
- 委员与调整涉及供应商有关联的
- 委员与调整提案人有直接利益关系的
- 其他可能影响公正性的情况

**回避后怎么表决？**
- 回避的委员不参与表决
- 表决基数相应减少（如5人变4人，3/5通过改为3/4通过）

## 四、委员会协同机制

### 4.1 会议机制

| 会议类型 | 频率 | 时长 | 参会人 | 议题 |
|----------|------|------|--------|------|
| **常规评审会** | 每月1次（第二周周三） | 60-90分钟 | 5人委员会 | 审议当月提案 |

| **复盘会** | 每次+30天 | 30-60分钟 | 5人委员会 | 复盘调整效果 |

| 场景 | 沟通方式 | 责任人 |
|------|----------|--------|

| **材料分发** | 邮件附件 | 秘书长 |
| **会议纪要** | 邮件发送 | 秘书长 |
| **执行通知** | 邮件/电话 | 秘书长 |
| **供应商反馈** | 邮箱（pricing-committee@JD.com） | 秘书长 |

---

### 4.3 决策分歧处理

**如果委员会意见分歧怎么办？**

| 分歧程度 | 处理方式 |
|----------|----------|
| **轻微分歧**（4:1或3:2） | 按表决规则执行，少数意见记录在纪要 |
| **严重分歧**（2:2或3:1-1） | 准备两套方案，提交C2选择 |
| **无法达成一致** | 暂缓提案，补充材料后再次评审 |

## 六、委员会的权责边界

### 6.1 委员会可以做什么

| 权力 | 说明 |
|------|------|
| **提案评审** | 评估提案的合理性、风险、可行性 |
| **投票表决** | 按表决规则决定提案是否通过 |
| **提出建议** | 对提案提出优化建议 |
| **要求补充材料** | 如果材料不足，可以要求补充 |

---

### 6.2 委员会不能做什么

| 限制 | 说明 |
|------|------|
| **不能单方调整价格** | 所有调整必须经委员会+C2审批 |
| **不能绕过C2决策** | C2有最终决策权 |
| **不能修改提案内容** | 只能提出建议，修改权在提案人 |
| **不能代表供应商承诺** | 委员会是评审方，不是谈判方 |

---

### 6.3 各角色的权力边界

| 角色 | 能做什么 | 不能做什么 |
|------|----------|------------|
| **秘书长** | 组织、协调、记录、追踪 | 没有投票权 |
| **委员** | 发言、投票、建议 | 不能单方决策 |
| **提案人** | 发起提案、解释提案 | 不能要求委员会通过 |
| **C2** | 最终决策、战略把控 | 不能绕过委员会直接调整 |

## 七、总结

### 委员会的核心价值

> 定价委员会不是"增加流程"，是"降低风险"。
>
> 委员会的目标：
> 1. 让定价决策有依据（数据支撑）
> 2. 让定价决策有制衡（多方视角）
> 3. 让定价决策有闭环（复盘改进）

### 委员会的协同原则

| 原则 | 说明 |
|------|------|
| **专业互补** | 每个角色有专业视角，互补而不是对抗 |
| **权力制衡** | 5人委员会 + C2终审，避免单方决策 |
| **透明决策** | 所有决策有记录、有依据、可追溯 |
| **闭环改进** | 每次调整后复盘，持续优化 |
