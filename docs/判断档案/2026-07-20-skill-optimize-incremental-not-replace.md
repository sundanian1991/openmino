# 判断样本 · judgment-archive skill 用外部文章优化（6 问扩展）

## 元信息
- 日期：2026-07-20
- 场景类型：工作（skill 优化决策）
- 协作对象：用《职场能力跃升指南：从经验积累到构建可复用的判断模型》优化 judgment-archive SKILL
- 提取范围：当前对话全程（7 项改动 + 1 次抓取回退）
- 模板版本：6 问扩展后首次实战（自指验证）

## 判断样本

### Q1 · 否定了什么
- 否定"照搬文章 6 模块替换 5 问"（方案 A·大改）——推翻式重写，丢弃当前已经磨过的字段约束和 Failure Patterns 里的"自创章节禁令"
- 否定"把 skill 改成事前决策器"——文章"最小验证动作"是事前/事中概念，没让它入侵 think-taste-agency 的领地
- 否定"把文章 3 大支撑（保质期/杠杆率/认知带宽）塞进判别标准"——这些是说服性论证，不是操作性判别
- 用户否定档位选项里的"核心改/最小改"，选"全改"
- 用户否定 AnySearch 失败后的默认回退（webReader），选 Tavily——**这条否定被现实推翻**：Tavily 配额也耗尽，最终还是绕回 webReader

### Q2 · 为什么否定
- **照搬替换** → 机制层：AI 默认有"结构完整性倾向"——看到外部完整框架就想全盘采纳。但当前 5 问经过 Grant Sanderson 访谈迭代，字段约束是磨出来的。冲突：文章的结构完整性 vs 当前的字段约束稳定性
- **事前决策器化** → 机制层：AI 默认"功能扩张倾向"——看到相关能力就想纳入。但 skill 价值在聚焦和边界清晰，think-taste-agency 已占事前位置。冲突：功能扩张倾向 vs skill 边界纪律
- **3 大支撑塞判别** → 语境层：文章 3 大支撑是元层次（为什么判断模型重要），判别是操作层次（这次要不要调用）。混层次会让判别从 4 条聚焦变成 7 条臃肿
- **选 Tavily** → AGENTS.md 明确标注"Tavily 配额易耗尽"，但用户仍选它。这一项不是 AI 的偏好问题，是规则提醒未被采纳——下次遇到类似情况应主动引用 AGENTS.md 的标注

### Q3 · 保留了什么
**胜出方案的关键判别特征**：
- 增量不改存量：保留已磨出的字段约束，只在缺口处补
- 定位守边界：每个新增字段都问"这属于事前还是事后"
- 真盲区优先：4 个 gap 里识别出 2 个真盲区（关键变量/校准记录）独立成项，2 个半盲区（判断顺序/最小验证动作）借已有字段升级
- 操作化落地：每个新增给出可执行标准 + 好坏例 + AI 能直接照做的指引

**区别于被否定版本的核心差异**：不是"用文章 6 模块覆盖当前 5 问"，而是"用文章 6 模块做 gap 扫描，只补真盲区"

### Q4 · 真正关心的问题本质
- **表层需求**：用文章优化 judgment-archive SKILL
- **实际校验**：优化一个"提取判断"的 skill，本身就是一次高密度判断实践——双重价值在于既要产出更好的 SKILL，又要用本次协作做样本验证新 SKILL 是否跑得通（自指检验）
- **更深一层隐含校验**：AI 会不会为了证明优化有效而过度美化判断样本？Failure Patterns 里的"迎合性提取"正是这个陷阱。所以必须诚实写出"选 Tavily 被现实推翻"这类容易被跳过的瞬间

### Q5 · 下次类似任务的判断顺序 + 避免误判

**判断顺序**（下次优化任何 skill 时按此排查）：
1. 定位边界先划清 → 这个 skill 该做什么、不该做什么（事前/事中/事后）
2. 已有能力盘点 → 当前 SKILL 经过哪些迭代、哪些字段已固化、哪些 Failure Patterns 是磨出来的
3. 外部框架做 gap 扫描 → 不照搬，只找真盲区（独立成项）和半盲区（借力升级）
4. 每个新增字段操作化 → 给出可执行标准、好坏例、AI 能直接照做的指引
5. 自检验证 → 改完后用本次协作做样本跑一遍，看新模板是否真的跑得通

**避免误判**：
- 下次遇到"用外部框架优化内部 skill"，先问"当前框架磨过几轮？"——磨过的字段约束优先保留，外部框架只补缺口
- 下次给用户档位选项时，若 SKILL 本身强调"克制"，不要把"全改"列在推荐位——这会和 SKILL 内部倾向冲突
- 下次抓取失败需要回退工具时，主动引用 AGENTS.md 里对该工具的配额标注，避免用户选了又失败绕路

### Q6 · 关键变量
- **定位纪律度** → 这个 skill 改完后定位是否仍清晰（事后提取器 vs 事前决策器）。决定 skill 能否长期被正确调用——定位漂移的 skill 会被遗忘或误用
- **已有能力固化度** → 当前 SKILL 经过几轮迭代、哪些字段约束是磨出来的。决定是"增量补充"还是"推翻重写"
- **Gap 真伪度** → 外部框架 vs 当前框架的差异，是真盲区还是半盲区还是已覆盖。决定每个 gap 用哪种力度补（独立成项 / 借力升级 / 不动）
- **操作化程度** → 新增字段是否给出可执行标准、好坏例。决定 AI 能否正确填写——只声明字段不教怎么填，字段会空转
- **自检严格度** → 改完后是否用真实样本跑一遍验证。决定优化是否真的有效——没经过自检的优化只是纸面完整

## 价值标记
- 可复用度：高（"增量不改存量 + 定位守边界 + 真盲区优先 + 操作化落地 + 自检验证"可迁移到任何 skill/系统优化）
- 通用规则：优化已有系统时，先守已磨出的边界和约束，再用外部框架补真盲区，每个新增必须操作化到 AI 能照做，最后用真实场景自检
- 最小验证动作：下次优化任何 skill，先问"这个 skill 现在的 Failure Patterns 是什么"——能说清 Failure Patterns 说明已磨过，走增量路线；说不清说明是新 skill，可更自由
- 升级判断：暂不升级。理由：这是 skill 优化场景的首次样本，未跨场景验证；"增量不改存量"虽强但需观察是否在所有优化场景都成立（新 skill 优化可能不适用）。按层 3 标准，需同类判断在 INDEX 出现 2-3 次才升级
