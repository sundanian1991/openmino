# 判断档案 · 入口

> 个人判断力档案库。从人机协作中提取判断样本，沉淀可复用的判断逻辑。
>
> 技能定义：`.agents/skills/judgment-archive/SKILL.md`

---

## 这里是什么

15 条判断样本，来自 7 次人机协作（2026-06 到 2026-07）。每条样本用 5 问解压缩（否定了什么 / 为什么 / 保留了什么 / 真正关心什么 / 下次避免什么），把模糊的"感觉不对"拆解成可复用的判断规则。

**收集机制**：周度 cron 自动收集（每周日 21:00）+ 协作中手动触发。详见下方"自动化收集机制"。

**判断样本 ≠ 内容总结**——压缩的是判断方式，不是协作内容。

## 文件命名约定

```
YYYY-MM-DD-主题词.md
```

- 日期：原协作日期（不是提取日期）
- 主题词：协作对象或判断瞬间（如 `haglofs-basis-assumption-flip` / `42zhangjing-three-voices-fidelity`）
- 同一协作的多条样本：用协作品牌词作前缀（如 `42zhangjing-*` / `haglofs-*`），字母序自动聚合

## 自动化收集机制

**周度 cron 任务**（`judgment-weekly-collect`，每周日 21:00）自动执行：

1. 扫描本周协作记录（workspace/ 新本周新增/修改文件、cron 执行日志、memory 记录）
2. 按 4 维度筛选值得提取的协作
3. 跑 5 问解压缩
4. 写入样本文件 + 更新 INDEX.md 的 4 个视图
5. 升级判断（克制原则：单次经验不升级）

prompt 文件：`.cron/prompts/judgment-weekly-collect.md`

**为什么是周度而不是即时**：判断提取需要反思距离，当场做和隔一天做质量不一样。周回顾能批量覆盖整周协作，token 成本也更低。

## 何时手动新增样本

除了周度自动收集外，如果你在协作中明确感觉到"这个判断值得留"，可以直接说：

- "提取判断样本"
- "沉淀这次判断"
- "判断复盘"

参见 [SKILL.md 的判别标准](../../.agents/skills/judgment-archive/SKILL.md)——4 维度至少满足 2 条才提取，避免档案膨胀。

## 升级到 memory

样本累积到一定数量后，高频通用规则升级到项目 memory（feedback 类）。当前所有样本均为"暂不升级"——单次经验不够，等同类判断出现 2-3 次后再考虑。
