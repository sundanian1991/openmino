# 判断样本 · judgment-archive skill 的设计与首次实战

## 元信息
- 日期：2026-07-06
- 场景类型：工作
- 协作对象：将"个人判断档案方法论"落地为 AI agent skill，并完成首次实战调用
- 提取范围：当前对话全程（从形式选型到 skill 落地再到首次提取）

## 判断样本

### Q1 · 否定了什么

**形式层面的否定：**
- 否定"做成 Hook"——Hook 是事件触发的自动注入，本质是无感执行规则校验
- 否定"扩展已有 skill"（curatorial-log-method-recovery）——会模糊两个 skill 各自的定位
- 否定"全自动触发"——会消解方法论"主动反思"的内核

**内容层面的否定（在 SKILL.md 设计中）：**
- 否定"原版方法论直接当 skill 用"——原文是方法论文章，缺操作层（触发条件、输入定义、深度标准、输出结构、场景适配、累积复用 6 个缺口）
- 否定"每条样本都升级到 memory"——会污染记忆，只有高频复用的才升级

### Q2 · 为什么否定

**Hook 否定的深层原因：**
Hook 适合"复刻行为"，skill 适合"承载方法论"。这套方法论的内核是"主动选择值得提取的协作"——如果做成 Stop hook 自动跑，会出现两个问题：(1) 时机错配，对话结束时强行启动打扰收尾；(2) 价值消解，无人认真对待则退化为变相的内容总结。AI 默认偏好"自动化一切"，但这里自动化恰恰是反方法论的。

**扩展已有 skill 否定的深层原因：**
curatorial-log-method-recovery 的定位是"策展项目级决策记录"，焦点是项目；本 skill 是"跨场景人机协作判断样本"，焦点是个人的判断方式。看起来都讲"判断 vs 流水账"，但对象和粒度完全不同。AI 的默认偏好是"复用已有"（避免重复造轮子），但这里复用会牺牲定位清晰度。

**原版方法论直接当 skill 的深层原因：**
方法论文章和 skill 是两种文体。文章讲"为什么对"，skill 要讲"怎么操作"。原文 5 问只有问句没有操作深度，AI 直接用会产出浅层填充。补的"深 vs 浅对照示例"是关键——它把抽象方法论变成了可执行的质量标准。

**升级克制原则的深层原因：**
AI 默认偏好"尽量积累"，但记忆膨胀会稀释每条记忆的权重。"判断样本 ≠ 内容总结"这个判别标准本身就是防膨胀机制——四维度至少满足 2 条才提取。

### Q3 · 保留了什么

**胜出方案的关键判别特征：**

1. **手动调用 + 轻量提醒**的组合，而非纯手动或全自动——保留了"主动反思"内核，又解决了"用户可能忘记"的问题。AGENTS.md 里的提醒规则有明确触发条件（多轮 + 产出 + 取舍），避免无意义打扰。

2. **判别前置**的设计——核心判别表（4 维度至少满足 2 条）放在 skill 最前面，先回答"值得提取吗"再进入流程。这是原版方法论没有的，但实战中最关键的一道闸门。

3. **双层定位**（通用 5 问 + 三类场景卡片）——通用保证可迁移，场景卡片保证深度。区别于 curatorial-log 的单一场景绑定。

4. **Failure Patterns 章节**——原版没有，但实战中价值极高。把"会怎么提取失败"前置显性化，是 skill 区别于方法论文章的标志。

**核心差异**：从"如何自动化这套方法"切到"如何让这套方法在保护内核的前提下可操作"。

### Q4 · 真正关心的问题本质

- **表层需求**：这套内容适合做成 hook 还是 skill（一个形式技术问题）
- **实际校验**：AI 工具形式的选择，本质是在判断"这套方法论的内核是什么、什么形式能保护这个内核"。不是在比较技术选项，而是在保护一个方法论的灵魂——主动反思 vs 自动执行，哪个是它的命。

更深一层：用户问"内容是否足够清晰完整"，表面在问 skill 文档质量，**实际在校验"AI 能否识别方法论文章和 skill 操作文档的文体差异，并补全缺口"**——这是对 AI 协作能力的元判断。

### Q5 · 下次避免的误判

- **遇到"内容做成什么形式"的决策，先问"内核是什么"再选形式**——而不是先比较技术选项。形式服务于内核，技术适配度是第二位的。
- **遇到方法论类内容要落地为 skill，默认补操作层**——方法论文章讲"为什么对"，skill 要补"什么时候用 / 输入是什么 / 每步深度标准 / 输出什么 / 会怎么失败"。这 5 项是方法论→skill 的标准转换清单。
- **遇到"是否扩展已有 skill"的诱惑，先确认核心定位是否真的重叠**——看起来像（都讲"判断"）≠ 真的像（对象和粒度不同）。检查焦点和场景，不检查关键词。
- **遇到 AI 默认"自动化一切"的倾向，先问"自动化会不会消解内核"**——有些方法论的价值恰恰在于主动选择，自动化是反方法论的。
- **遇到"积累 vs 克制"的取舍，默认选克制**——判断样本、memory 升级都适用。质量永远优先于数量，膨胀的档案等于没有档案。

## 价值标记
- 可复用度：高
- 通用规则：**形式选择要先服务于内容内核，再考虑技术适配度。判断"内核是什么"→"什么形式能保护它"→"技术怎么实现"，这个顺序不能反。**
- 升级判断：暂不升级
  - 理由：虽然可复用度高且提炼出了通用规则，但仅单次经验，未跨多个场景验证。按 skill 默认倾向（克制），暂不升级到 memory，等同类判断出现 2-3 次后再考虑
