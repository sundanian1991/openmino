# 每日记忆整理任务 — 次日上午 10:00

> 本文件是每日整理的指令集。当 cron 触发时，AI 按此指令执行整理。

---

## 触发时机

- **时间**：每日上午 10:00（Asia/Shanghai）
- **范围**：整理前一日的 buffer.md 内容
- **原因**：年老师通常在 10 点开机开始工作，此时整理结果可直接看到

---

## 执行流程

### 第一步：读取 buffer.md

读取 `memory/thinking/buffer.md`，获取前一日的未处理条目。

**判断**：如果 buffer 只有头部说明（无实际条目），直接回复"📋 昨日无新内容需要整理"并结束。

---

### 第二步：逐条分类

对每条未处理条目，判断其归属：

| 条目类型 | 归属目录 | 文件命名 |
|:---|:---|:---|
| 决策（选了什么/为什么） | `memory/decisions/2026/` | `YYYY-MM-DD-主题.md` |
| 人员信息（偏好/习惯/关系变化） | `memory/projects/关键人画像/对应目录/` | 追加到对应人.md |
| 会议内容（讨论/决策/承诺） | `memory/meetings/2026/` | `YYYY-MM-DD-会议名.md` |
| 洞察/方法论/框架 | `memory/topics/工作方法/` 或相应知识目录 | 追加或新建 |
| 项目进展/变更 | `memory/projects/对应项目.md` | 追加更新 |
| 日常流水 | `memory/daily/YYYY-MM-DD.md` | 如果当日文件不存在则创建 |
| 无法分类 | 标记为 `⚠️ 需人工分类` 留在 buffer | — |

---

### 第三步：写入目标目录

按分类结果执行写入/追加操作。

**注意**：
- `decisions/` 中的每条决策使用独立文件（格式见 decisions/README.md）
- `meetings/` 中的记录使用 TEMPLATE.md 的4类模板
- 人员信息的追加要注意去重（如果已有类似信息，合并而非重复）

---

### 第四步：更新 MOC 索引

每写入/更新一个目录后，同步更新该目录的 `README.md` 索引：

- decisions/README.md → 追加新决策条目
- meetings/README.md → 追加新会议条目
- people 相关目录 → 更新最近更新时间
- topics 相关目录 → 追加新洞察条目

---

### 第五步：清理 buffer

已处理条目从 buffer.md 中移除（整条删除）。
如果某条目标记为 `⚠️ 需人工分类`，保留在 buffer 中等待人工处理。

---

### 第六步：生成整理摘要

输出格式：

```
📋 昨日整理完成（YYYY-MM-DD → YYYY-MM-DD）

归档 N 条：
- 决策 2 条 → decisions/2026/YYYY-MM-DD-xxx.md
- 人员 1 条 → projects/关键人画像/上级/王易人.md（更新）
- 洞察 3 条 → topics/工作方法/xxx.md（追加）
- 日常 1 条 → daily/YYYY-MM-DD.md（新建）

新建文件：N 个
更新文件：N 个
MOC 同步：✅ decisions ✅ meetings ✅ daily

⚠️ 需人工：X 条（保留在 buffer 中）
```

---

## 与每周记忆维护的关系

| 维度 | 每日整理 | 每周维护 |
|:---|:---|:---|
| **频率** | 每天 10:00 | 每周日 |
| **触发** | cron 自动 | 手动或 cron |
| **范围** | 前一日 buffer | 全周内容 |
| **深度** | 分类归档+索引 | 去重合并+insights升级 |
| **产出** | 分布式归档 | insights + MEMORY.md 更新 |
