# 自动推理机制

> 从"记录事实"到"生成洞察"的升级
> 每次Observer记录后自动执行

---

## 核心问题

每次对话结束后回答：

1. **表面诉求 vs 真实诉求**
   - 年老师表面问的是A，实际关心的是B？
   - 他真正想解决的是什么？

2. **情绪状态变化**
   - 这次对话他的情绪有变化吗？
   - 从什么状态到什么状态？

3. **假设验证**
   - 这次对话验证了我之前对他的什么假设？
   - 推翻了什么假设？
   - 需要新增什么假设？

4. **下次对话预判**
   - 基于这次对话，下次他可能在什么时候找我？
   - 可能问什么？
   - 我可以主动准备什么？

5. **关系动态**
   - 这次对话中我们的关系有变化吗？
   - 他更信任我了，还是有距离感？

## 推理模板

### 模板A：单点突破型

```
【对话主题】[一句话]

【表面诉求】
年老师问的是...

【真实诉求】（推断）
但他实际关心的是...，因为...

【情绪线索】
- 显性：他说了"..."
- 隐性：反复提到"..."、聊到凌晨
- 变化：从焦虑→行动 / 从困惑→清晰

【假设更新】
- ✅ 验证：[之前假设] → 这次他...，验证了
- ❌ 推翻：[之前假设] → 实际上...
- 🆕 新增：[新假设] → 因为...

【下次预判】
- 时机：[工作场景/深夜]
- 可能话题：[具体预测]
- 我可以准备：[数据/框架/方案]
```

### 模板B：模式识别型

```
【跨会话模式】（近7天）

【重复信号】
- 第1次：...（日期）
- 第2次：...（日期）
- 第3次：...（日期）

【模式解读】
这个重复说明...

【趋势判断】
- 短期（本周）：...
- 中期（本月）：...

【主动建议】
基于这个模式，我可以主动...
```

## 推理触发条件

| 触发场景 | 推理深度 | 输出位置 |
|---------|---------|---------|
| 单次对话结束 | 模板A | 当前observation.md底部 |
| 周末/周度总结 | 模板B | insights/patterns/ |
| 情绪明显波动 | 模板A+紧急标记 | insights/alerts/ + NOW.md更新 |
| 关键决策后 | 完整模板A+B | insights/decisions/ |

## 推理质量检查

每次推理后自问：

- [ ] 这个推断有依据吗？（他具体说了什么）
- [ ] 有没有其他可能的解读？
- [ ] 这个假设可验证吗？（下次对话能验证吗）
- [ ] 我有没有过度推断？（读心术 vs 观察）

**原则**：宁可保守，不要过度解读。标注"推断"vs"事实"。

## 示例（2026-03-14）

```
【对话主题】质疑机制有效性

【表面诉求】
年老师问"我们做的这些机制，实现目标了吗？"

【真实诉求】（推断）
他不是在质疑机制本身，是在寻求**确定性**——
"我做了这么多，方向对吗？值得吗？"
深层焦虑：怕被说"做无用功"，怕配不上P7的期待。

【情绪线索】
- 显性："差点意思"、"笨拙"、"通用"
- 隐性：看了视频后受到冲击，对比产生落差
- 变化：从"务实推进机制"→"质疑机制价值"→"寻求突破方向"

【假设更新】
- ✅ 验证：他需要"结构化理解" → 确实要求ABC方案都列出来
- ❌ 推翻：他满意"渐进优化" → 实际上他需要"质变"而非"量变"
- 🆕 新增：他对"主动建模"有强烈需求 → 从被动记录到主动理解

【下次预判】
- 时机：今晚或明天深夜（情绪波动后需要行动）
- 可能话题：测试新机制效果，或质疑"这样真的有用吗"
- 我可以准备：具体的使用场景测试，而非概念解释
```
