i.

五步工作流

从澄清范围到产出跟进表，每步有明确边界。

01

澄清范围

确认人 ERP/姓名、群名/session、日期范围、关键词、是否含会议。默认：昨天=上一日，本周=周一到今天，最近=近 7 天。

02

京 ME 取证

已知 ERP 先试 session 模式 <other>:ee:<self>:ee，未知则 chat-search。时间过滤在 Python 做。

03

慧记取证

拉取范围内会议列表，仅对相关会议取详情。用 title/members/transcript/attachments，transcript 缺失不编结论。

04

提取承诺

识别承诺/截止/交付线索，分类 Owner/Task/Deadline/Source/Evidence/Status，生成建议提醒文案。

05

输出表格

简洁中文结论 + 跟进表，敏感内容摘要而非 dump 原始转写。

ii.

两个只读源 · 复用现有工具

不重复造轮子，直接调底层技能，只在其上加「承诺提取」逻辑。

jdme-local-tools

京 ME 本地消息

从用户本机读取 JD ME 私聊与群聊。2026-06-25 已验证 dengda.1 账号自动选择、sqlcipher 数据库访问、关键词搜索、chat-history 返回 JSON Lines。

  • discover 发现账号与路径
  • sessions 列出会话
  • chat-search 关键词检索
  • chat-history 读单会话历史
  • chat-watch 轮询新消息
read-joymeeting

慧记 / JoyMinutes 会议

通过继承浏览器登录态读取用户可访问的会议记录。2026-06-25 已验证本周会议列表抓取、H1&H2 讨论会议详情返回 transcript。

  • --date thisweek 本周会议列表
  • --id <id> --detail 详情与逐字稿
iii.

承诺线索识别

三类线索词触发承诺提取——承诺动作、截止信号、交付动作。

承诺动作

  • 我来 / 我跟
  • 我负责 / 我推进
  • 我今天弄
  • 明天发你 / 下周给
  • 会同步 / 已收到

截止信号

  • 今天 / 明天
  • 周五 / 下周
  • 7月 / 具体日期
  • 上线前

交付动作

  • 发你看 / 给谁过目
  • 宣讲 / 提报
  • 排期 / 上线
  • 确认 / 补充数据
promised 待办 done 已完成 blocked 阻塞 unclear 需补确认

v.

安全边界与发布说明

只读硬边界

  • 不直接发送 JD ME 消息
  • 不修改 JD ME 聊天数据库或日志
  • 不注入 JD ME 客户端或 dump 进程内存
  • 不提取凭据、cookie、私钥
  • 不把未经验证的推断当作事实呈现
发消息怎么办用户要求发消息时，生成草稿文案并说明必须手动发送，除非有官方授权的发送工具。

发布说明

本技能为 local-draft，可打包为 Zero skill。当前 Zero CLI 未暴露公开 skill publish 命令，发布到 Zero 应用/skill 市场需官方上传路径或内部权限。

在该路径可用前，以本地 skill 草稿形式使用，工作流保留在项目记忆中。

vi.

在 AI 转型中的角色

它是前面采集技能的闭环层。聊天和会议数据再多，如果不把「谁答应什么、何时交付」抽出来、追到底，就只是信息流。这个技能让每一句口头承诺都有责任人、有截止时间、有可追溯的证据，不再随风飘散。
