iii.

四条核心规则

这些规则决定了技能如何对用户表现——结果优先、默认时间、强制确认、收件人解析。

结果优先，过程最小化

直接给邮件结果、结论、表格和建议动作。不主动展开脚本名、命令行、SOAP 字段、鉴权链路等内部实现。除非用户明确追问「你怎么查的」，否则不展示底层调用逻辑。

默认只查最近 1 天

所有「查询邮箱内容」的操作，如果用户没有明确给出时间范围，默认只查最近 1 天（按 DateTimeReceived）。用户明确说「今天/本周/某月」时严格按用户给的范围，不再套默认值。

修改类操作强制二次确认

send / reply / forward / batch-* / flag 等所有会修改邮箱状态的操作，必须先在对话层二次确认再执行。确认时展示收件人、主题、正文摘要、附件摘要，用户明确确认后才真正调用。

收件人必须先解析

用户给中文名或 ERP 而非邮箱地址时，不得直接发送。先调 lookup-recipient（queryByErp）解析，展示 ERP/姓名/部门/邮箱，让用户选定后再发送。优先命中最近真实往来联系人。

iv.

search 命令 · 常用筛选

最常用的查询命令，支持丰富的客户端过滤组合。

run.sh search
# 收件箱未读邮件
bash scripts/run.sh search --folder inbox --unread --limit 20

# 按发件人 + 主题查询
bash scripts/run.sh search --folder inbox --sender zhangsan@jd.com --subject 周报

# 自定义文件夹，带附件
bash scripts/run.sh search --folder-name 项目归档 --has-attachments

# 时间范围 + 重要级别
bash scripts/run.sh search --folder inbox \
  --after 2026-06-20 --before 2026-06-27 --importance High

# 结果转表格总结
bash scripts/run.sh search --folder inbox --unread --limit 20 \
  | bash scripts/run.sh summarize
查询推荐做法 自定义文件夹先 folders 查目录 · 用户只给显示名直接用 --folder-name · 查较早日期提高 --max-pages 不要只扫第一页 · 看全文先 search 拿 item_id 再 detail · 展示结果优先 Markdown 表格。
v.

写操作 · 回复 / 发送 / 转发

reply · 基于原邮件回复

自动用原发件人作收件人、生成 Re: 主题，通过 EWS CreateItem+ReplyToItem 发送。可选 --reply-all、--include-original 带原文、--attachments 追加附件。

send / forward · 新建与转发

send 新建邮件，支持 to/cc/bcc/subject/body/importance/attachments。forward 转发已有邮件，可加附言和附件。两者同样需二次确认。用户给 ERP/姓名而非邮箱时，先走 lookup-recipient，不要猜邮箱。

lookup-recipient · 收件人解析

run.sh lookup-recipient
# 按姓名 / ERP 查（底层同一接口 queryByErp）
bash scripts/run.sh lookup-recipient --condition 韩 --page-no 1 --page-size 10
bash scripts/run.sh lookup-recipient --name 韩松岩
bash scripts/run.sh lookup-recipient --erp hansongyan3

# 优先顺序：先匹配最近往来联系人，未命中再查 HR 接口
vi.

鉴权链路与边界

鉴权策略

  • 有 JOYMAIL_SOAP_AUTH_TOKEN 直接用；否则取 sso_token（SSO_TOKEN / ME_TOKEN / SO_TOKEN，或 ~/.joyclaw/openclaw.json）
  • 调 clawJwtLogin 拿 token，请求头带 Cookie: sso.jd.com=<token>
  • token 放 auth + Authorization: Bearer 头
  • 401 自动刷新 token，最多重试 3 次
  • endpoint 默认 mail-skill.jd.com，EWS 版本 Exchange2016

局限

  • 依赖 FindFolder / FindItem / GetItem 三件套
  • body-keyword 当前匹配 preview/text body，不保证等价服务端全文检索
  • 自定义「已处理」通过「是否有后续发件回复」近似判断
  • 缺依赖（如 requests）优先自动安装，不先问用户
  • 无命令执行权限的会话无法查邮件，需切到支持 shell 的会话
