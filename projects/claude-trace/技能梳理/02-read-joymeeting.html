把会议的逐字稿
原样取回来

从京东 JoyMinutes 提取用户参与的会议记录——复制本机浏览器 profile 启动 headless 浏览器，继承 ERP SSO 登录态，原生 CDP 协议直调 API，一个脚本拿到会议全文与逐字稿，而不是摘要。

只读 · 不动登录态 headless CDP SSO 继承 多浏览器 fallback v1.0.0

iii.

命令清单

单入口 scripts/fetch-meetings.js，Node.js 脚本，输出 JSON。

按日期范围查询会议列表

支持 today / yesterday / thisweek / lastweek / lastNdays / YYYY-MM-DD / --start+--end 区间。默认今天。

fetch-meetings.js
# 今天的会议
node "$SKILL_DIR/scripts/fetch-meetings.js"
# 本周（周一到今天）
node "$SKILL_DIR/scripts/fetch-meetings.js" --date thisweek
# 日期区间
node "$SKILL_DIR/scripts/fetch-meetings.js" --start 2026-06-03 --end 2026-06-09

查指定会议详情与逐字稿

meeting-id 来自 JoyMinutes URL（joyminutes.jd.com/video/{id}）。加 --detail 才会拉取 transcript。

fetch-meetings.js
node "$SKILL_DIR/scripts/fetch-meetings.js" --id 21ac80c26e507f76 --detail

# 列表 + 详情一次拿全
node "$SKILL_DIR/scripts/fetch-meetings.js" --date thisweek --detail
iv.

日期解析规则

用户的自然语言说法 → 脚本参数的映射，直接决定查询窗口。

"今天" / 未指定today（默认）· 仅今天
"昨天"yesterday · 仅昨天
"本周"thisweek / week · 本周一到今天
"上周"lastweek · 上周一到上周日
"近 N 天" / "最近 N 天"lastNdays（如 last7days）· 从 N 天前到今天
"YYYY-MM-DD"--date YYYY-MM-DD · 指定单日
"X月X日到X月X日"--start YYYY-MM-DD --end YYYY-MM-DD · 指定区间
v.

浏览器登录态继承

这是技能能「免登录」的关键机制。脚本按以下顺序检测并使用浏览器：

  1. i.
    当前活跃窗口的浏览器
    用户正在用的浏览器优先，登录态最新鲜
  2. ii.
    Chrome
    优先级最高的标准浏览器
  3. iii.
    Tabbit
    京东内部浏览器，Chrome 登录态失效时的 fallback
  4. iv.
    Edge
    额外备选方案

脚本自动完成：检测并启动 headless 浏览器（完全不可见）→ 复制 profile 继承登录态 → CDP 协议在浏览器上下文调 API → 关闭浏览器释放资源 → stdout 输出 JSON。支持 macOS（活跃窗口/进程/路径检测）与 Windows（进程/路径检测）。

端口自动避让 CDP 默认端口被占用时（"CDP not ready"），脚本自动在 9333–9350 范围内查找可用端口，无需手动指定。

vii.

总结输出与常见错误

总结规范

单日：总览（会议数、主要议题）→ 每场（标题、时间、链接、参会人、内容概述、关键讨论、决策结论、待办）→ 综合要点。

多日/区间：总览（区间、会议总数、主要主题）→ 按日分组 → 综合要点（跨天主题归纳、持续跟进事项）。

质量硬约束每条会议必含标题与链接可追溯 · 基于详细文字记录不编造 · 缺失信息明确标注 · 不输出 Cookie/Token/登录信息。

常见错误

错误
原因与处理
SSO_REDIRECT
ERP SSO cookie 过期。脚本自动 fallback 其他浏览器；全部失效时自动打开登录页，登录后重试
数据为空但 totalInList>0
日期过滤未命中。检查 --start/--end 或扩大查询范围
未找到浏览器
需安装 Chrome / Tabbit / Edge 之一并登录过 JoyMinutes
CDP not ready
端口占用，脚本自动在 9333–9350 找可用端口
viii.

边界与局限

硬边界（绝对不做） 修改浏览器 profile 原始数据 · 提取/输出 Cookie、Token、登录凭据 · 发送会议邀请或修改会议 · 访问用户无权限的会议。只读取用户已登录可访问的会议记录。

读不到什么

  • 未登录过任何支持浏览器的会议平台——SSO cookie 不存在则无法继承
  • 无转写记录的会议——transcript 为 null，可能是未录制或转写未生成
  • 用户无权限访问的会议——即使知道 ID 也拉不到
  • 会议视频/音频二进制本体——只拿文字转写，不下载音视频
风险
说明
登录态依赖
强依赖浏览器 SSO cookie，cookie 过期需重新登录；无独立鉴权路径
转写质量
转写文本由平台 ASR 生成，可能有识别错误，涉及专有名词/数字需核验原文
合规边界
会议内容涉及多方隐私，逐字稿喂云端 LLM 做总结时需注意数据外溢
profile 复制开销
每次复制浏览器 profile 有 IO 开销，大量历史会议批量拉取时较慢
★
ix.

在 AI 转型中的角色

把原本散落在 JoyMinutes 平台、只能靠人脑记忆和手抄纪要的会议讨论、决策、承诺、分歧，变成可全文检索、可按发言人/时间定位、可直接喂 LLM 做复盘和周报的结构化数据。逐字稿让「谁在会上答应了什么」有据可查。
