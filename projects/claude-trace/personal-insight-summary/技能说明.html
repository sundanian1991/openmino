personal-insight-summary

Version 1.0.0 Prompt 技能 多源融合

核心能力

作为个人工作信息助手，从京ME、邮件、慧记三个数据源收集信息，通过跨源关联分析，提取工作洞察，输出结构化的每日工作简报。

跨源关联分析 信息融合 洞察提取 结构化简报

跨源融合

并行收集三个数据源信息
时间过滤和会话筛选

输入格式

{ "sources": [ { "skillId": "jdme-local-tools", "prompt": "获取昨天的聊天记录" }, { "skillId": "read-email", "prompt": "获取昨天的邮件" }, { "skillId": "read-joymeeting", "prompt": "获取昨天的会议" } ] }

整合分析维度

时间过滤：按日期范围筛选
会话筛选：识别相关会话
人名映射：统一不同系统的人员标识
相关性评分：评估信息与用户的关联度
证据强度标记：区分强/弱证据
跨源关联：串联聊天、邮件、会议信息

输出结构（6个固定Section）

1

"昨天你主要参与了 {N} 个项目"

项目参与概览，展示工作重心分布

2

"今天建议优先处理 {N} 件事"

基于紧急度和重要度排序的待办建议

3

"重点关注"

需要持续关注的人或事项

4

"风险提醒"

潜在风险和需要预警的事项

5

"已定决策"

已确认的决策和结论

6

"待定问题"

尚未解决的问题和待跟进事项

输出约束

强制规则

  • 只输出"强证据"项，不推测
  • 不编造信息
  • 每 section 最多 7 项
  • 不输出日期行
  • 不添加额外 section 标题
  • 严格按 6 section 结构输出

降级处理

无有效信息时

当没有收集到有效信息时，输出特定的降级消息，说明原因（如：数据源不可用、时间范围内无相关活动等），而非编造内容。

使用场景

每日晨会准备：快速了解昨日工作
工作回顾：结构化的工作总结
优先级排序：基于数据的待办建议
风险预警：跨源信息关联发现风险

关键突破

突破信息孤岛

传统方式下，IM、邮件、会议是三个独立系统，需要分别查看。本技能将三个数据源融合成一个统一的工作视图：

聊天中的非正式沟通
邮件中的正式沟通
会议中的决策讨论
三者交叉验证和关联

Prompt 技能，无独立脚本，通过协调三个数据源技能实现跨源分析。
