# 年底赞 Skill 设计系统 — 验证包

## 用途
本 ZIP 用于让外部 AI 检查「年底赞」PPT Skill 的设计系统文档是否存在漏洞，导致按流程生成的 HTML 布局崩坏。

## 文件清单

### 1. 设计文档（核心资产）
- `tokens.md` — 19色 + 字号 + 间距 + 圆角 + 边框的完整 Token 体系
- `system.md` — 布局系统/三区系统/5种布局模式/排版规则/10条不可违背规则
- `components.md` — 29个组件按9大类聚合，含变体表、DOM、CSS、Do/Don't
- `templates.md` — 12页逐页模板定义 + 主题映射 + 交替节奏
- `skill-flow.md` — 6步生成流程（主题映射→布局模式→组件选择→填充内容→生成HTML→检查）

### 2. 输入材料
- `content.md` — 从原始12页PPT截图提取的结构化内容（标准输入材料）

### 3. 原始素材
- `Camera_*.jpg`（12张）— 原始 Digital Nomad PPT 截图，每页一张

### 4. 生成结果
- `reference.html` — 旧版（未按流程，自由发挥，被用户判定不协调）
- `reference_v2.html` — 新版（严格按 skill-flow 6步流程 + 设计文档生成）

## 问题描述
按 skill-flow 流程生成的 `reference_v2.html`，用户反馈"很多页布局崩坏"。需要外部 AI 检查：
1. 设计文档本身是否有矛盾/缺失/不可执行之处？
2. 流程步骤是否有漏洞？
3. 组件定义在实际 HTML 中是否有样式冲突？
4. 是否需要新增/修改组件或布局模式？

## 关键约束
- 单文件 HTML，零外部依赖
- 16:9 画布，每页 `min-height: 100vh`
- 仅使用 CSS 变量引用 tokens（如 `--orange`, `--text-primary`）
- 零阴影原则，层级通过颜色+间距建立
- 单情绪色：仅 `#E85D2B` 为全局强调色
