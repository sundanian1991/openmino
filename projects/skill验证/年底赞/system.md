# Digital Nomad Work Infrastructure — System Design Document

> 系统层设计文档：布局、网格、信息层级、排版、间距、颜色语义、图标规范。
> 与 `tokens.md` 配合使用：`tokens.md` 提供可量化的原子值，本文档提供系统规则和组合逻辑。

---

## 1. 页面结构与外壳（Page Architecture）

### 1.1 页面基础规格

- **画布比例**：16:9（推荐实现宽度 1920px，高度 1080px）
- **安全区域**：内容不超出左右 80px、上下 60px 的边距
- **出血元素**：封面球体（P01）、右侧大图（P12）可突破边距直达边缘
- **主题切换**：每页独立声明主题，三种主题不可混用
  - `theme-light`：白底 + 黑字
  - `theme-dark`：深黑底 + 白字
  - `theme-cream`：米色底 + 黑字（仅 P02 使用）

### 1.2 页面外壳 DOM 结构

```html
<section class="slide" data-theme="light|dark|cream">
  <!-- 可选：装饰元素（DecorCircle） -->
  <div class="decor-circle" style="position:absolute; top:60px; right:80px;"></div>
  
  <!-- 标题区（TitleBlock） -->
  <div class="zone-header">
    <h1 class="text-h1">...</h1>
    <p class="text-body">...</p>
  </div>
  
  <!-- 呼吸留白区（ZoneBreathing） -->
  <div class="zone-breathing"></div>
  
  <!-- 内容区（ZoneContent） -->
  <div class="zone-content">
    <!-- 具体页面内容 -->
  </div>
</section>
```

### 1.3 主题切换规则

| 规则 | 说明 |
|------|------|
| **单页单主题** | 每页只能使用一种主题，禁止混用 |
| **呼吸交替** | Light 页与 Dark 页交替出现，形成视觉呼吸（P02奶油色可视为 Light 变体） |
| **Dark 页文字透明度** | 正文使用 `rgba(255,255,255,0.6)`，Caption 使用 `rgba(255,255,255,0.4)`，**不可使用纯白**（除标题外） |
| **Light 页文字** | 标题 `#1A1A1A`，正文 `#4A4A4A`，Caption `#888888` |

## 2. 网格系统（Grid System）

### 2.1 基础网格参数

```
Columns:     12
Gutter:      24px
Page Margin: 80px (left & right)
```

内容区域可用宽度 = `100vw - 160px`（左右各 80px）。
在 1920px 画布下，单列宽度 ≈ `136px`，gutter = `24px`。

### 2.2 网格使用原则

- **从不使用全12列铺满**：最大内容宽度约占 10-11 列，保留边缘呼吸感
- **对齐到隐形基线**：标题、卡片顶部、图表顶部应对齐到同一水平线
- **gutter 恒定**：无论分栏比例如何，gutter 始终保持 24px

### 2.3 常用分栏模式

#### 模式 A：`layout-4-6`（40% / 60%）

```
+--------+--------------------------------+
| 40%    | 60%                            |
| 文字区 | 图形/图表区                    |
+--------+--------------------------------+
```

- **使用页面**：P03（工具系统）、P10（柱状图）
- **gap**：40px
- **文字区**：左侧，垂直居中对齐或顶部对齐
- **图形区**：右侧，内容可水平/垂直居中

#### 模式 B：`layout-5-5`（50% / 50%）

```
+--------------------+--------------------+
| 50%                | 50%                |
| 文字区             | 图形/图片区        |
+--------------------+--------------------+
```

- **使用页面**：P06（三角形层级）、P12（总结+建筑图）
- **gap**：40px
- **垂直对齐**：通常居中对齐，保持左右视觉平衡

#### 模式 C：`layout-3-3-3`（三等分）

```
+-----------+-----------+-----------+
| 33.3%     | 33.3%     | 33.3%     |
| 卡片列    | 卡片列    | 卡片列    |
+-----------+-----------+-----------+
```

- **使用页面**：P11（三列优势卡片）
- **gap**：24px
- **列高**：不要求等高，顶部对齐，底部自然错落

#### 模式 D：`layout-2-2-2-2`（四等分 / 2×2 网格）

```
+-----------+-----------+
| 50%       | 50%       |
| 卡片      | 卡片      |
+-----------+-----------+
| 50%       | 50%       |
| 卡片      | 卡片      |
+-----------+-----------+
```

- **使用页面**：P04（四宫格工具分类）
- **gap**：16px（列和行）
- **特殊规则**：卡片支持不同底色（白/深棕/暖米），打破单调

#### 模式 E：`layout-full`（全宽 / 上下分区）

```
+-------------------------------------+
| 标题区（约25%高度）                  |
+-------------------------------------+
| 呼吸留白（约15%高度）                |
+-------------------------------------+
| 内容区（约60%高度）                  |
| 图表 / 流程图 / 卡片 / 甘特图        |
+-------------------------------------+
```

- **使用页面**：P05（系统流程）、P07（实践流程）、P08（生态树）、P09（路线图）
- **标题区**：左对齐，大标题 + 描述段落（可左右分栏）
- **内容区**：水平居中，占据页面宽度的 80-90%

## 3. 信息层级区系统（Zone System）

每页（除封面外）遵循严格的三区划分：

### 3.1 Zone Header（标题区）

- **高度占比**：约 20-30%（通常约 200-280px）
- **内容**：
  - 页面大标题（`text-h1`，36px，粗体）
  - 描述段落（`text-body`，14px，最大宽度 480px）
  - 可选：标签（`text-caption`，12px）
- **对齐**：左对齐
- **多行标题**：标题允许换行，每行 3-6 个单词，形成块状

### 3.2 Zone Breathing（呼吸留白区）

- **高度占比**：约 10-20%（通常约 80-160px）
- **内容**：**纯空白，无任何元素**
- **功能**：
  - 强制分隔标题与内容，避免视觉拥挤
  - 让观者的心理焦点从"阅读文字"切换到"观看图表"
  - 是这套设计"编辑感"的核心来源

### 3.3 Zone Content（内容区）

- **高度占比**：约 50-70%（通常约 500-700px）
- **内容**：图表、卡片、流程图、图片、数据可视化
- **对齐**：通常水平居中或左对齐（视布局模式而定）
- **最大宽度**：不超过页面宽度的 90%

### 3.4 三区系统示例（P05）

```
┌────────────────────────────────────────┐
│ [标题区]                                │
│ "Create a remote work system..."       │
│ "A connected workflow ensures..."      │
├────────────────────────────────────────┤
│                                        │
│ [呼吸留白 — 约120px 纯空白]             │
│                                        │
├────────────────────────────────────────┤
│ [内容区]                                │
│      ┌────────┐    ┌────┐ ┌────┐      │
│      │Worker  │───▶│Chat│ │Docs│ ...  │
│      └────────┘    └────┘ └────┘      │
└────────────────────────────────────────┘
```

## 4. 排版系统（Typography System）

### 4.1 字体家族策略

| 位置 | 字体 | 理由 |
|------|------|------|
| 封面主标题 | Serif（Playfair Display） | 材质对比——内页全部无衬线，封面衬线形成"仪式性开场" |
| 内页标题 | Sans-serif（Inter Bold） | 理性、清晰、现代 |
| 正文 | Sans-serif（Inter Regular） | 长文可读性 |
| 数据 | Sans-serif（Inter Bold） | 数据需要视觉重量 |
| 装饰编号 | Sans-serif（Inter Light） | 大号但极细，退后为背景纹理 |

### 4.2 字号层级使用场景

| Token | 典型使用位置 |
|-------|-------------|
| `text-display` (64px serif) | P01封面："Digital Nomad Work Infrastructure" |
| `text-h1` (36px bold) | P02-P12每页顶部大标题 |
| `text-h2` (20px bold) | 卡片标题（P04、P11）、要点标题（P07右上） |
| `text-h3` (16px semibold) | 小节标题、阶梯列表标题（P02）、卡片副标题 |
| `text-body` (14px regular) | 所有描述段落、说明文字 |
| `text-caption` (12px regular) | 标签（P02右上角"Digital Nomad Deck"）、来源注释 |
| `text-number-xl` (56px light) | P11中右列的水印编号"02.""03." |
| `text-number-lg` (36px bold) | P02数据卡片的"+40%""+48%" |
| `text-card-label` (13px semibold) | 流程节点标签、卡片内标签 |

### 4.3 排版规则

#### 标题换行策略

标题较长时按语义换行，而非按字符数强制换行：

```
Good:
"Flexible Work Needs
More Than Freedom"

Bad:
"Flexible Work Needs More
Than Freedom"
```

原则：在介词、连词前换行，保持语义完整。

#### 段落宽度控制

- 正文段落最大宽度：`480px`（约 55-70 字符）
- 超过最大宽度时强制换行
- 多段落之间间距：`24px`

#### 字距微调

- Display 级别：`-0.02em`（紧凑，有冲击力）
- H1：`-0.01em`
- 正文：0（标准）
- Caption/Micro：`+0.01em` 到 `+0.02em`（略微放宽，提升小字号可读性）

## 5. 间距系统（Spacing System）

### 5.1 间距使用决策树

```
需要间距？
├── 页面边缘 → page-padding-x (80px) / page-padding-y (60px)
├── 大模块之间 → space-16 (64px) 到 space-24 (96px)
│   └── 标题区 ↔ 内容区 → space-24 (96px) 必须
├── 中等模块之间 → space-10 (40px) 到 space-12 (48px)
│   └── 4:6/5:5 分栏 gap → space-10 (40px)
├── 小组件之间 → space-6 (24px)
│   └── 卡片之间、三列 gap
│   └── 卡片内元素间距
├── 紧凑间距 → space-2 (8px) 到 space-4 (16px)
│   └── 金字塔层之间 → space-2 (8px)
│   └── 2×2 网格 gap → space-4 (16px)
└── 微调 → space-1 (4px)
    └── 图标与文字之间
```

### 5.2 关键间距规则

| 场景 | 间距值 | 说明 |
|------|--------|------|
| 页面边距 | 80px / 60px | 不可压缩，保证呼吸感 |
| 标题区与内容区 | 96px | 核心规则，不可妥协 |
| 标题与描述段落 | 48px | 同区块内的垂直间距 |
| 4:6/5:5 分栏 gap | 40px | 左右区之间的水平间距 |
| 三列卡片 gap | 24px | 等宽三列之间的间距 |
| 2×2 网格 gap | 16px | 四宫格行列间距 |
| 卡片内边距 | 24px | 标准卡片 padding |
| 卡片内元素间距 | 16-24px | 标题、图标、描述之间的间距 |
| 金字塔层间距 | 8px | 层与层之间的紧凑间距 |
| 流程节点间距 | 24px | 节点之间的水平/垂直间距 |
| 甘特条垂直间距 | 8px | 条与条之间 |

## 6. 颜色语义系统（Color Semantics）

### 6.1 单情绪色原则

整份PPT只允许**一种情绪色**——橙色 `#E85D2B`。所有需要强调、高亮、引导的位置都使用这一颜色。

#### 橙色的精确使用位置

| 页面 | 橙色元素 |
|------|----------|
| P01 | 球体渐变（中心 `#F0652B` → 边缘 `#C44A1A`） |
| P03 | 金字塔顶层（"Coordination"） |
| P05 | 左侧起始节点（"Worker/Team"）+ 右侧圆形节点（橙色实心） |
| P07 | 右上6个要点的箭头图标（↘） |
| P08 | 中层橙色长条（"Client / team touchpoints"） |
| P09 | Phase 2 和 Phase 3 的甘特条 |
| P10 | 最后一根柱子（$9.4B） |
| P12 | 2×2 要点的橙色箭头图标（↘） |

#### 灰色到橙色的叙事规则

P10 柱状图是"灰→橙"叙事的典范：
- $5.4B：浅灰 `#777777`
- $6.1B：中灰 `#666666`
- $7.5B：深灰 `#555555`
- $9.4B：**橙色 `#E85D2B`**

灰色柱子明度递减，形成"积累→高潮"的视觉节奏，橙色作为终点释放视觉张力。

### 6.2 双主题颜色映射

#### Light Theme → Dark Theme 转换规则

| Light | Dark |
|-------|------|
| 白底 `#FFFFFF` | 深黑底 `#111111` |
| 黑字 `#1A1A1A` | 白字 `#FFFFFF` |
| 灰字 `#4A4A4A` | 半透明白 `rgba(255,255,255,0.6)` |
| 浅灰底卡片 `#F0F0F0` | 深灰底卡片 `#2A2A2A` |
| 黑底双色圆 | 同（不变） |

> **注意**：Dark 主题中，正文不使用纯白，而是 `60%` 透明度白。这是刻意的设计选择——纯白在深色背景上过于刺眼，60% 透明度创造更舒适的阅读体验。

### 6.3 卡片底色变体规则

P04 四宫格展示了三种卡片底色变体：

| 变体 | 底色 | 文字色 | 边框 | 使用场景 |
|------|------|--------|------|----------|
| 标准白 | `#FFFFFF` | `#1A1A1A` | 1px `#E8E8E8` | 默认 |
| 深棕 | `#3A3228` | `#FFFFFF` | 无 | 强调某张卡片，打破单调 |
| 暖米 | `#F5F0E8` | `#1A1A1A` | 无 | 柔和变体，与深棕形成冷暖对比 |

规则：在一个 2×2 网格中，**至少使用两种不同底色**，避免四张全白卡片的单调感。

## 7. 图标系统（Iconography System）

### 7.1 图标风格定义

- **风格**：纯线框（Outline），无填充
- **描边宽度**：1.5px（标准），1px（极小图标）
- **线端样式**：round cap，round join
- **颜色**：继承当前主题主文字色（Light 黑 / Dark 白），**绝不使用彩色图标**
- **例外**：箭头图标（↘）使用橙色 `#E85D2B`，作为视觉引导

### 7.2 图标尺寸映射

| 尺寸 | px | 使用位置 |
|------|-----|----------|
| XS | 12-14px | 金字塔层内小图标（P03）、内联标记 |
| SM | 16-18px | 卡片内图标（P11方形边框内） |
| MD | 20-24px | 标准UI图标、流程节点图标、DecorCircle |
| LG | 32px | 卡片标题旁的大图标 |

### 7.3 特殊图标处理

#### P11 卡片图标容器

P11 的卡片图标不是裸露的 SVG，而是置于一个**细边框容器**内：

```
┌─────────┐
│  [icon] │  ← 16px 线框图标
│   16px  │
└─────────┘
   32px

容器：32×32px，圆角 6-8px，1px 边框（与卡片边框同色）
```

#### P05/P08 流程图节点图标

- 小图标（16-20px）置于节点内部或上方
- 图标颜色与节点文字色一致
- 图标配以简短标签（1-2个单词）

#### 封面地球图标

- 20-24px 线框风格
- 白色，置于右下角标签组内
- 与文字"Productivity Systems for Remote Teams"水平排列

### 7.4 箭头图标规范

P07 和 P12 使用橙色箭头作为要点前缀：

- **形状**：右下箭头（↘ 或 →）
- **颜色**：`#E85D2B`（唯一彩色图标）
- **尺寸**：14-16px
- **与文字间距**：8px

## 8. 图像与摄影规范

### 8.1 摄影图风格

| 属性 | 规范 |
|------|------|
| 色调 | 低饱和、偏冷、黑白或单色灰调 |
| 主题 | 现代建筑、几何抽象、极简空间 |
| 人物 | **无人物** |
| 处理 | 可轻微降低饱和度至 60-70%，提升对比度 |

### 8.2 图像使用位置

| 页面 | 位置 | 尺寸 | 说明 |
|------|------|------|------|
| P11 | 左列卡片顶部 | 约 280×180px | 建筑几何摄影，灰色调 |
| P12 | 右侧半区 | 约 500×600px | 现代建筑摄影，曲线白色建筑 |

### 8.3 图像占位处理

如无法获取真实摄影图，使用以下CSS占位方案：

```css
/* 几何抽象占位 */
background: linear-gradient(135deg, #d8d8d8 0%, #a8a8a8 50%, #c8c8c8 100%);

/* 或 */
background: repeating-linear-gradient(
  45deg,
  #e0e0e0,
  #e0e0e0 10px,
  #d0d0d0 10px,
  #d0d0d0 20px
);
```

> **注意**：占位样式仅用于开发阶段，最终输出应替换为真实摄影图。

## 9. 系统组合规则

### 9.1 组件组合公式

每页可视为以下公式的组合：

```
Page = PageShell(theme) + TitleBlock + [BreathingZone] + ContentArea

ContentArea = ComponentA + ComponentB + ...
```

#### P02 公式

```
P02 = PageShell(theme-cream) 
    + TitleBlock(H1 + Body, 左右分栏) 
    + DataCard×2(right) 
    + StepList(4 items, 编号在右, 垂直分割线)
```

#### P05 公式

```
P05 = PageShell(theme-light)
    + TitleBlock(H1 + Body, 左右分栏)
    + BreathingZone
    + FlowDiagram(
        left: FlowNode(accent, "Worker/Team")
        center: FlowNode×4 (2×2 grid: Chat/Docs/Tasks/Meetings)
        right: OrangeCircle×2 (File Storage, Clients)
        connectors: L-shaped lines with arrows
      )
```

#### P08 公式

```
P08 = PageShell(theme-dark)
    + TitleBlock(H1 + Body, 左右分栏)
    + BreathingZone
    + TreeArchitecture(
        top: Card×3 (with subtitles)
        mid: PillBar(orange, "Client / team touchpoints")
        bottom: Card×5 (dark bg)
        connectors: tree-branch lines
      )
```

### 9.2 不可违背的系统规则

| # | 规则 | 违反后果 |
|---|------|----------|
| 1 | **单页单主题** | 页面视觉混乱 |
| 2 | **标题与内容之间必须有≥80px留白** | 页面拥挤，失去编辑感 |
| 3 | **只允许一种情绪色（橙）** | 视觉焦点分散，失去克制感 |
| 4 | **Dark主题正文使用60%白，非100%** | 文字刺眼，阅读疲劳 |
| 5 | **不使用任何阴影** | 破坏平面感，显得过时 |
| 6 | **封面唯一使用衬线** | 内页混用衬线会破坏统一性 |
| 7 | **卡片至少使用两种底色**（2×2网格中） | 单调乏味 |
| 8 | **建筑摄影必须低饱和、无人物** | 风格不统一 |
| 9 | **编号水印必须使用极浅色（#E0E0E0）** | 编号抢夺注意力 |
| 10 | **流程图起点使用不同形状（圆vs矩）** | 观者无法识别起点 |

## 10. 快速检查清单（Checklist）

在实现任何页面之前，逐项确认：

- [ ] 页面主题已声明（light/dark/cream）
- [ ] 页面边距≥80px（左右）/≥60px（上下）
- [ ] 标题区与内容区之间留白≥80px
- [ ] 只使用了黑/白/灰/橙四种颜色（封面渐变除外）
