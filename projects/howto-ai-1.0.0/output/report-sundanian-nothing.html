数据分析 / 中频 / 7.8h / AI 潜力高 — 月度最高价值改造项
BUILD

AI 场景翻译

当前流程：数仓导出 → Excel 清洗 → 透视表 → 手工美化 → HTML 可视化。最耗时环节是"同样数据该分析哪些维度"的判断 + Excel 手工操作链。

改造方案：用 data-cleaner 自动清洗 CSV/Excel（去重、格式化、异常值标记），再用 kpi-viz 生成交互式图表（趋势线、分布图、排名表），最后用 report-gen 将数据 + 图表组装成结构化 HTML 报告。

预计效果：数据清洗从 2-3h 降至 < 5min，维度分析从试错变为 AI 推荐 Top5，HTML 生成从 1-2h 降至一键输出。月节省约 5h。

BUY

AI 场景翻译

内部群 + 供应商群每天大量消息，手动筛选紧急事项效率低。关键需求：自动识别 @消息、新需求、合规问题，按紧急度分级推送。

JoyClaw 已具备定时任务和消息处理能力，通过 msg-classifier 配置分类规则（紧急/普通/忽略），结合 JoyAgent 的消息监控接口，实现每日 18:00 自动推送摘要。无需额外安装，属于配置优化。

预计效果：每日群监控从全天刷群降至 2 次摘要查看（早+晚），月节省约 20h。

报告与文档自动生成
文档处理+会议沟通 / 中频 / 15.6h（#7+#11 合计）
BUY

AI 场景翻译

巡检报告、处罚文档、汇报材料等结构化文档反复撰写，格式重复但每次重写。用 doc-template 定义文档模板（输入字段 + 输出结构），AI 根据数据自动生成完整报告。

预计效果：单篇报告从 1-2h 降至 10-15min，月节省约 10h。

Build 路径指引

自建技能链：数据整理 → KPI 分析 → 报告生成
从数据管道搭建到自动化输出，三步走
  1. 搭建数据管道
    Python 连接数仓导出接口，pandas 自动清洗（去重、格式化、异常值标记），输出标准化 CSV。用 JoyCode 辅助编写脚本。
  2. 配置 KPI 可视化
    kpi-viz 技能绑定数据源，自动生成趋势线、分布图、排名表。支持交互式筛选（按业务线/时间/供应商）。
  3. 组装 HTML 报告
    report-gen 将清洗数据 + 图表 + 分析结论组装成结构化 HTML，支持 Jinja2 模板定制。
  4. 定时自动化
    JoyClaw cron 配置：每周一 09:00 自动执行完整流程（清洗 → 分析 → 报告 → 推送）。

定时任务配置

Cron Prompt（完整文本）

以下 prompt 可直接复制到定时任务平台，每周一早 9:00 自动执行 KPI 数据分析全流程。

Prompt / 每周 KPI 自动分析
你现在是供应商 KPI 分析助手。请执行以下任务： 1. 读取本周 KPI 数据文件（路径：~/kpi-data/本周/） 2. 使用 data-cleaner 清洗数据： - 去重、格式化、异常值标记 - 输出标准化 CSV 3. 使用 kpi-viz 生成可视化： - 趋势图：各供应商 KPI 月度变化 - 排名表：综合得分 Top10/Bottom10 - 分布图：各维度得分分布 4. 使用 report-gen 组装 HTML 报告： - 标题、日期、数据概览 - 图表 + 分析结论 - 异常项标注 + 建议动作 5. 输出：完整 HTML 报告 + 异常清单 输出格式：HTML 文件。语言：中文。
Platform Commands
JoyClaw（小龙虾）
在 JoyClaw 定时任务面板中创建新任务：
触发条件：0 9 * * 1（每周一早 9:00）
技能链：data-cleaner → kpi-viz → report-gen
将上方 Prompt 粘贴到任务指令框中，保存即可。
MyAgents
在 MyAgents 会话配置中设置定时触发：
Cron 表达式：0 9 * * 1
将上方 Prompt 作为会话首条消息模板，配置周一自动触发。
系统 Crontab
终端执行：
crontab -e
添加以下行：
0 9 * * 1 cd ~/projects/ai-agents/my-agent && claude --skill data-cleaner,kpi-viz,report-gen -p "$(cat ~/cron-prompts/kpi-analysis.txt)" >> ~/cron-logs/kpi-$(date +\%Y\%m\%d).log 2>&1
