## 三、高频追问模式（Session 数据特有）

### 3.1 模式 1：问题 → 计划 → 执行 → 验证

**典型案例**：Excalidraw 手绘字体问题

```
Session 1: "这个保存以后怎么用"
    ↓ (发现问题)
Session 2: "Implement this plan:" (首次 plan)
    ↓ (执行失败)
Session 3: "Implement this plan:" (再次尝试)
    ↓ (仍有问题)
Session 4: "你看这种的图，obsidian 可以通过 claude 做出来吗"
    ↓ (转换思路)
Session 5: "给转成 Excalidraw 吧"
    ↓ (最终解决)
```

**思维特征**：
- **不轻言放弃**：一个问题跟踪 5 个会话
- **允许转换思路**：从"配置字体"→"自动绘制"
- **结果导向**：不纠结于技术细节，要的是"能用的结果"

### 3.2 模式 2：发现 → 分享 → 关联

**典型案例**：Vibe Marketing 小红书发现

```
用户输入："做 Vibe Marketing 的人都该知道这个 - 小红书。帮我找到张咋啦这边开源的一个 Skills..."
```

**思维特征**：
- **即时分享**：看到有价值内容立即分享给 Mino
- **主动关联**： "帮我找到...Skills" — 关联到正在使用的工具
- **生态思维**：把小红书、Skills、AI 工具视为一个整体生态

### 3.3 模式 3：书籍导入 → 批量清理

**典型案例**：《沟通的方法 - 脱不花》电子书清理

```
用户指令：
```

**思维特征**：
- **批量思维**：不是一本一本处理，是"所有文件一起处理"
- **标准清晰**：9 项清理规则，每项都有具体示例
