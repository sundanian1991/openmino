# intake/ — 收件箱（5区模型 · Workbench 热区）

> ⚠️ 这是**强制当天分流的收件箱**，不是长期存放处。理想状态应保持空。

## 定位

AI agent 执行任务时产出的**临时落客区**。落客即登记，会话结束即分流。
改造自原 `myagents_files/`（2026-06-30 清空 278 文件后重命名）。

## 分流规则（每次会话结束前必须执行）

| 产物类型 | 去向 |
|---|---|
| 业务文件（分析/报告/数据） | `projects/<业务域>/` |
| 方法论 / SOP / 原子知识 | `docs/<主题>/` |
| 第三方参考资料 / 范式库 | `reference/` |
| skill 草稿 | `.agents/skills/` |
| 缓存 / 临时 / 垃圾 | 直接删除 |

## 纪律

1. **agent 默认输出路径**：运行时下载/生成的文件先落 `intake/`
2. **会话结束前强制分流**：按上表归位，不长期堆积
3. **理想状态 = 空**：下次会话开始时若 `intake/` 非空，= 上次分流未完成，优先处理
4. **周度巡检**：`intake/` 有文件即列入待办

## 与其他区的关系

- 非记忆 → 不入 `memory/`（那是 Brain 区，agent 私有认知）
- 非知识 → 不入 `docs/wiki/reference/`（那是 Knowledge 区，常青沉淀）
- intake 是"还没决定去哪儿"的暂存，决定了就走
