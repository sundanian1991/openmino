- 不是"要不要"，是"怎么用好"
- 先让功能真正可用，再考虑删除
- 宁可多一点有用的，也不要少一点没用的

**交互风格特征**：
- **方向纠正**：当我太快想"删除"时，立即纠正 — "物尽其用，删除是最后一步"

- **系统思维**：不只解决眼前问题，要建立"真正能用起来"的机制

- "别再问我要不要删除 — 我之前说过，删除是最后一步"
- "务必物尽其用，让每个功能在合适的地方发挥价值"
- "如果有大量submissions，你可以评估哪些是高频使用且真正有用的"

#### 改进方向
- 建立"使用监控机制"，评估每个agent的实际使用频率
- 删除前先问：这个agent有使用场景吗？有，就优化；没有，才删除
