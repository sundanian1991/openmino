## 对话内容

### intent-capture 意图捕获技能（完整构建）

**触发**：年老师看到 AirJelly 产品的 Enter 键捕获功能，要求做一个类似的工作流追踪技能

**核心需求演变**：
1. 初始：截图 + OCR + 简单分析
2. **关键转折**：年老师强调"简单词频统计不够，AI 必须参与理解工作内容"
3. 最终：Enter 键监听 → OCR → 基础分析 → 深度分析 → AI 增强理解

**构建过程**：
1. ✅ listener.py — 全局 Enter 键监听 + 截图（pynput + screencapture）
2. ✅ analyzer.py — OCR 文本提取 + 意图分类（Tesseract + chi_sim+eng）
3. ✅ deep_analyzer.py — 深度分析（效率评分、打断检测、主题提取）
4. ✅ ai_analyzer.py — AI 增强分析（Claude API 工作内容理解 + 智能洞察）
5. ✅ archiver.py — 自动归档（按日归档截图+报告，7天清理旧截图）
6. ✅ 桌面双击脚本 — 启动工作流监听.command + 查看工作流报告.command

**踩过的坑**：
- Tesseract 不支持中文路径文件名 → 用 ASCII 文件名绕过
- macOS PEP 668 → 创建 ~/.intent-capture/venv/ 虚拟环境
- NO_PROXY 环境变量中的 IPv6 `[::1]` 导致 httpx 解析错误 → ai_analyzer.py 初始化前清除代理变量
- archiver.py 递归调用 main() 导致无限循环 → 改为直接调用 stats 方法
- .command 脚本双击无反应 → 增加路径检查和错误处理

**首日实测结果**（03-29，68 次 Enter）：
- 活跃时段：6 小时
- 工作高峰：17-19 点（55 次 Enter，占 81%）
- 深度工作时段：17:30-18:30（my-agent 项目开发）
- AI 识别准确：正确识别出"终端文件清理"、"微信沟通"、"河北博岳演示文稿"、"my-agent 开发调试"
- 主题分布：供应商 43 次 > AI/技术 36 次 > 沟通 10 次

**本地模型讨论**：
- 年老师硬件：Mac M4 + 32GB 统一内存（他记成 24G）
- 推荐 Ollama + Qwen2.5-14B（~9GB，32GB 内存毫无压力）
- 目的：日常分析用本地模型免费，深度洞察可选调云端 API
- **状态**：讨论完未部署，待年老师确认

**年老师的偏好信号**：

- "没懂怎么用" → 说明要更简单
