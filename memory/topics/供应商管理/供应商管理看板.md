## 核心计算公式（先看这个，再看看板）

### 公式1：产能利用率（双维度+人力结构修正）

**问题**：用行业均值作分母，约50%供应商自然低于50%，没有区分度。

**修正**：用两个维度同时看，加入人力结构修正。

| 维度 | 公式 | 回答的问题 | 异常标准 |
|------|------|-----------|----------|
| **自身峰值比** | 本月**熟手**人均GMV ÷ 该供应商近6个月最高**熟手**人均GMV | "他自己跟自己比，发挥了几成功力？" | 从>80%降到<65% = 内部出现管理/团队问题 |
| **头部差距** | 本月**熟手**人均GMV ÷ 同产线当前第1名的**熟手**人均GMV | "他跟最好的比，差多少空间？" | 差距从20%拉大到40% = 别人在跑他在掉 |

**人力结构修正**：人均GMV必须基于"熟手"计算，不是总人数。

| 人力场景 | 修正方式 | 原因 |
|---------|---------|------|
| 新人占比>30%（入职<30天） | 分母改用"熟手人数"，同时标注"新人拉低幅度" | 新人不出产能，按总人数算会严重失真 |
| 总人数变化>20%（扩编或缩编） | 同时看"总量GMV"和"熟手人均GMV"两个指标 | 只看人均会忽略规模变化的影响 |
| 正常波动（新人<30%，总人数变化<20%） | 正常计算，不做修正 | 在正常范围内 |

**人力拆分要求**：
- 必须拆分：熟手人数、新人人数（入职<30天）、本月新入职、本月离职
- 熟手人均GMV = 总GMV ÷ 熟手人数
- 新人人均GMV = 总GMV ÷ 新人人数（辅助参考，通常很低）
- 新人拉低幅度 = (熟手人均GMV - 总人均GMV) ÷ 熟手人均GMV

**示例 — 新人稀释场景**：
- 赛维斯本月总GMV = 5200万，总人数190人（其中50个新人，140个熟手）
- 总人均GMV = 5200÷190 = 27.4万
- 熟手人均GMV = 5200÷140 = 37.1万
- 新人拉低幅度 = (37.1-27.4)÷37.1 = 26%
- 近6个月熟手峰值 = 42万
- 修正后自身峰值比 = 37.1÷42 = 88%（正常）

→ 如果按总人数算：27.4÷42 = 65%（会被误判为问题型）
→ 按熟手算：88%（正常）
→ **结论**：人均下降主要是新人稀释，不是管理问题。但需要跟踪新人什么时候能成长为熟手。

**两个维度组合判断**：

| 自身峰值比 | 头部差距 | 判断 | 动作 |
|-----------|---------|------|------|
| >85% | <20% | 标杆型，接近自己最好水平，跟头部差距小 | 保持，选为标杆分享 |
| >85% | >40% | 尽力型，自己发挥好了但跟头部差距大 | 评估是否产线/产品适配问题 |
| <65% | >40% | 问题型，自己没发挥好，跟头部差距也在拉大 | 约谈，定位内部问题 |
| <65% | <20% | 暂时波动型，短期因素发挥不好，但底子不差 | 观察1个月 |

### 甲方责任场景（你是甲方，以下场景你该管）

作为供应商管理者，你的资源调配决策会直接影响供应商的产能表现。以下场景需要你自己承担责任并追踪效果：

| 场景 | 你的动作 | 产能下降时谁的责任 | 追踪方式 |
|------|---------|-------------------|---------|
| **你要求供应商加人** | 你说"这个月需要加50人" | 你的责任：加的这批人什么时候能出产能？ | 跟踪"新人拉低幅度"，要求供应商提交新人培养计划 |
| **你给排名靠前的供应商加了数据量** | 他排名好，你多分配了名单 | 你的责任：名单加了，产能有没有跟上？消化能力够不够？ | 跟踪"总量GMV vs 熟手人均GMV"——总量涨了但人均降了=消化能力不足 |
| **你调整了产线分工** | 把他从A产线调到B产线 | 你的责任：新产线的产能爬坡是否达到预期？ | 跟踪"头部差距"在新产线的变化 |
| **供应商自己主动扩编** | 你没要求，他自己招人 | 供应商的责任：他自己决定扩的，产出了就该跟上 | 正常走产能利用率评估 |

**关键区别**：
- 你要求/你分配的 → 产能下降时先看**自己的决策是否合理**，不要直接归因为供应商问题
- 供应商自主决策的 → 产能下降直接走归因诊断流程

### 公式2：主管胜任评估

**背景**：作为供应商管理者，你无法直接评估对方主管的个人能力，但可以从外部观察其管理效果。以下5个指标都是可观察的。

| 指标 | 公式/数据 | 合格标准 | 反映什么 |
|------|----------|---------|---------|
| **团队稳定度** | 月度流失率 = 离职人数/月初总人数 | <8%/月 | 主管能不能留住人 |
| **出勤稳定度** | 月度日均有效出勤率 | >90% | 日常管理是否严格 |
| **人效趋势** | 近3个月人均GMV趋势（升/平/降） | 不连续2个月下降 | 主管能不能维持或提升产出 |
| **响应质量** | 预警后48小时内提交原因分析的比例 | >80% | 主管对问题的重视程度和管理成熟度 |
| **整改执行力** | 承诺的整改动作按期完成率 | >70% | 说到能不能做到 |

**主管胜任判断**：
- 5项全达标 = 胜任
- 3-4项达标 = 需关注，了解短板
- ≤2项达标 = 不胜任信号，约谈时重点谈管理能力

### 公式3：标杆分享配对

**逻辑**：不是让所有供应商互相学习，而是在产线相近、团队规模相近的供应商之间做精准配对。

| 步骤 | 动作 | 标准 |
|------|------|------|
| 1. 选标杆 | 每个产线选人均GMV最高的1-2家 | 连续2个月保持前2 |
| 2. 找对标 | 选出同产线中自身峰值比<70%的供应商 | 有提升空间 |
| 3. 配对条件 | 团队规模差距<30%（避免"他有200人我只有50人没法学"） | 可比较 |
| 4. 提取做法 | 从标杆供应商的具体数据中提取可复制的做法 | 必须是具体的、可量化的 |
| 5. 跟踪差距 | 配对后每月看头部差距是否缩小 | 差距缩小=有效 |

**可复制的做法示例**（不是泛泛的"加强管理"）：
- "A供应商每天上午10点做15分钟晨会，当日目标拆解到人，转化率比产线平均高3个百分点"
- "B供应商的复贷客户单独分组拨打，复贷转化率比混组拨打高8个百分点"
- "C供应商的新人前7天只听录音不打电话，第8天上岗后首周转化率比直接上岗高5个百分点"

## 看板一：供应商全景看板（月度打开）

**用途**：月初打开一次，看所有供应商上个月的整体表现和趋势方向。

**核心信息**：

| 信息 | 看什么 | 怎么判断 | 对应动作 |
|------|--------|----------|----------|
| **综合排名** | 1~N名排名 + 后30%标红 | 连续后30%的供应商需要处理 | 查触发规则（预警/约谈/清退） |
| **排名变动** | 本月排名 - 上月排名 = 变动值（+3表示上升3名，-5表示下降5名） | 下降超过5名的 = 骤降，需要关注 | 骤降的立即了解原因 |
| **自身峰值比** | 本月熟手人均GMV ÷ 近6个月熟手峰值 | <65% = 内部问题 | 进入矩阵定位 |
| **头部差距** | 本月熟手人均GMV ÷ 同产线第1名熟手人均 | >40% = 跟头部差距大 | 评估产线适配 |
| **人力结构** | 熟手人数 vs 新人人数 | 新人>30% = 产能被稀释 | 跟踪新人成长计划 |
| **出勤率** | 上月日均有效出勤率 | 低于85% = 人力不稳定 | 了解原因 |
| **流失率** | 上月离职人数/月初总人数 | 超过10%/月 = 高流失 | 了解是主管问题还是待遇问题 |

### 供应商对比矩阵（看板一的核心视图）

**矩阵维度**：X轴 = 总GMV（总量），Y轴 = 熟手人均GMV（人效）。

**为什么用总量×人均，而不是其他组合**：

总量回答"他产了多少"，人均回答"他每个人产了多少"。两者拆开看才能知道：
- 总量高是因为人多还是因为效率高？
- 人均高但总量低，能不能给更多资源？
- 总量高但人均低，是不是人海战术？

**四象限**：

| 象限 | 总量 | 人均 | 类型 | 含义 | 动作 |
|------|------|------|------|------|------|
| 右上 | 高 | 高 | 标杆型 | 规模大+效率高，真正的强者 | 选为标杆，分享做法 |
| 右下 | 高 | 低 | 臃肿型 | 靠人多撑着，效率被稀释 | 约谈：人均太低，管理是否松弛？ |
| 左上 | 低 | 高 | 潜力型 | 人少但效率高，给资源能放大 | 评估：要不要加数据量/加人？ |
| 左下 | 低 | 低 | 问题型 | 规模小效率也差 | 约谈或启动清退 |

**矩阵怎么用**（重点）：

矩阵不是画完就完了。它是**发现异常的入口**。使用流程：

```
第1步：打开矩阵，看每个供应商落在哪个象限
第2步：标出"异常落位"的供应商：
  - 从右上掉到右下（标杆→臃肿）= 效率恶化
  - 从左上掉到左下（潜力→问题）= 潜力消失
  - 从右下掉到左下（臃肿→问题）= 规模也在缩
  - 连续两月待在左下 = 长期问题
第3步：点击异常供应商，看他的"自身峰值比+头部差距"
  - 自身峰值比<65% = 他自己没发挥好
  - 头部差距>40% = 跟头部差距大
  - 两者结合 = 确定是内部问题还是外部差距
第4步：确认异常后，打开看板三（单体深度看板）
  - 7指标交叉验证，定位根因
  - 结合主管胜任评估，判断是管理问题还是外部因素
第5步：决定动作
  - 约谈 / 扶持 / 观察 / 清退
```

**总量越高，人均产出是高还是低？**

没有必然关系。这就是矩阵的意义——把两者拆开看：

- 有些供应商总量高、人均也高 → 真强
- 有些供应商总量高、人均低 → 靠人多，效率差
- 有些供应商总量低、人均高 → 小而精，有潜力
- 有些供应商总量低、人均低 → 真差

**人力是否需要拆分？**

需要。矩阵上的人均GMV必须用"熟手人数"做分母，不是总人数。同时，矩阵旁边要标注每家供应商的"新人占比"——如果一个供应商落在右下（臃肿型），但新人占比40%，那他的"人均低"可能是暂时的（新人稀释），不是管理问题。

**和刘乾坤的类比**：
- 他的时段趋势图看的是**今天vs昨天**的曲线形状
- 你的矩阵看的是**所有供应商**在总量×效率平面上的落位关系
- 他通过对比发现"大盘爆发你凹谷"，你通过矩阵发现"他到底是真强还是虚胖"

### 排名×人效趋势交叉（月度专属，不可周度使用）

**为什么只能月度查看**：排名在其他时间维度不够稳定，趋势也不完整。只有月度数据才能看出真实的趋势方向。

**交叉逻辑**：月度排名变化 × 熟手人均GMV趋势 = 四种类型。

| 排名变化 | 人均GMV趋势 | 类型 | 含义 | 动作 |
|---------|------------|------|------|------|
| 上升 | 上升 | 真进步 | 排名和效率都在变好 | 表扬，确认是否可持续 |
| 下降 | 下降 | 真退步 | 排名和效率都在恶化 | 约谈，定位根因 |
| 上升 | 下降 | 隐性退步 | 排名升了但人效降了（可能靠加人） | 警惕：不可持续，人效跟不上 |
| 下降 | 上升 | 隐性进步 | 排名降了但人效升了（可能别人加人更快） | 评估：是否名单分配不足导致排名下降 |

**关键洞察**：

- "真进步"和"真退步"一目了然
- "隐性退步"最危险——看起来排名升了，实际上人效在降，说明他靠的是加人不是提效。这种人效下降迟早会反映到排名上
- "隐性进步"最值得你关注——他自己的人效在提升，但排名反而降了，说明可能是你给的数据量不够，或者竞争对手在疯狂加人。这时候你应该考虑给他更多资源

## 看板二：供应商对标看板（周度打开）

**用途**：每周一打开，在所有供应商之间做横向对比，发现谁在掉队。

**核心信息**：

| 信息 | 看什么 | 怎么判断 | 对应动作 |
|------|--------|----------|----------|
| **周度GMV排名** | 本周每家供应商的GMV排名 | 和月度排名对比：如果周排名比月排名差很多 = 在加速恶化 | 可能需要提前介入，不等月底 |
| **周度产能达成率** | 本周GMV / 月度目标÷4（即周均应达成量） | 低于75% = 本周严重落后，整月目标可能完不成 | 了解本周异常原因 |
| **周度转化率对比** | 每家供应商的转化率放在一起排 | 转化率显著低于平均的 = 名单质量或坐席能力问题 | 转化率低的看是共性问题（所有人转化率都低=名单问题）还是个性问题（只有他家低=供应商问题） |
| **周度出勤对比** | 每家供应商本周出勤率放在一起排 | 出勤率最低的2-3家 = 人力风险 | 了解原因 |

**最关键的一个视图：周度产能达成率**

```
供应商    周目标    本周实际    达成率    状态
─────────────────────────────────────────
毅航      2000万    2100万     105%     正常
毛毛虫    1800万    1750万     97%      正常
伽玛      1500万    1400万     93%      正常
赛维斯    1200万     900万     75%      ⚠ 落后
汇讯       800万     550万     69%      ⚠ 严重落后
翰锐       700万     720万     103%     正常
岐力       650万     600万     92%      正常
广达       400万     480万     120%     超额
华啸       350万     200万     57%      ⚠ 严重落后
人和       300万     310万     103%     正常

→ 达成率<75%的标出来 = 本周需要关注的供应商
→ 连续2周<75% = 不用等月底，现在就该了解原因
```

**和刘乾坤的类比**：
- 他的轮盘巡检看的是**今天vs昨天**每家供应商的曲线压制关系
- 你的对标看板看的是**本周所有供应商**的达成率对比
- 他通过对比发现"大盘爆发你凹谷"，你通过对比发现"大家都正常就你掉队"

## 看板三：单体供应商深度看板（发现异常后打开）

**用途**：当全景看板或对标看板发现某家供应商异常时，打开这家供应商的深度视图，定位问题根因。

### 7个核心归因指标

| 指标 | 公式 | 反映什么 | 异常标准 |
|------|------|----------|----------|
| 出勤率 | 有效出勤/目标人数 | 人力稳定性 | <85% |
| 总拨打量 | 当日总外呼通次 | 整体产出规模 | 比均值降20% |
| 人均拨打量 | 总拨打量/出勤人数 | 人效/员工努力程度 | 比均值降15% |
| 接通率 | 接通通次/拨打通次 | 名单可打性 | 比均值降5个百分点 |
| 转化率 | 转化用户数/接通用户数 | 销售能力 | 比均值降5个百分点 |
| 件均 | GMV/转化件数 | 客户质量+大额能力 | 比均值降10% |
| 日均GMV | 当日GMV | 最终结果 | 比均值降20% |

**为什么不是3个指标而是7个**：

- **出勤率+人均拨打量**组合，区分"人少"和"人懒"
- **接通率**是名单问题的核心指标，没有它就无法区分"名单打不通"和"人没在打"
- **件均**区分"转化差"还是"金额小"
- 3个指标（出勤、转化、拨打量）只能看表面，7个指标能**交叉验证定位根因**

### 看板视图

| 视图 | 看什么 | 能回答什么 |
|------|--------|-----------|
| **日度GMV折线图（近30天）** | 每天的GMV连成一条线 | 哪天开始掉下来的？断崖还是缓降？ |
| **日度转化率+接通率双折线图（近30天）** | 两条线叠加 | 是名单变差了还是坐席变差了？ |
| **日度出勤+人均拨打量双折线图（近30天）** | 两条线叠加 | 是人少了还是在的人不努力了？ |
| **日度件均折线图（近30天）** | 件均趋势线 | 客户结构有没有变化？ |
| **近3个月月度关键指标对比** | 月度GMV、转化率、出勤率、流失率并列 | 结构性恶化还是暂时波动？ |

### 归因判断逻辑树

```
问题：某家供应商GMV在下滑

第一步：看自身峰值比 — 是他自己没发挥好，还是产线整体不行？
├── 自身峰值比<65% + 同产线其他供应商正常 → 内部问题，进入第二步
├── 自身峰值比正常 + 同产线其他供应商也在降 → 产线/名单问题，反馈策略组
└── 自身峰值比<65% + 同产线其他供应商也在降 → 混合问题，进入第二步

第二步：看主管胜任评估 — 是管理能力问题还是外部因素？
├── 主管5项评估中≥3项不达标 → 管理能力问题
│   → 约谈重点："团队流失率高+人效连续下降+整改未执行，管理层面需要系统性提升"
├── 主管5项评估中≥4项达标 → 不是管理问题，进入第三步
│   → 说明主管还在状态，可能是名单/策略/竞争等外部因素
└── 主管评估中等（3项达标） → 管理有短板但不是主因，结合第三步

第三步：定位具体原因（7指标交叉验证）
├── 出勤率下降 + 人均拨打量不变 → 人力缺口（缺人）
│   → "出勤从X%降到Y%，需要补充人力"
├── 出勤率不变 + 人均拨打量下降 → 管理松弛（在的人不努力）
│   → "出勤人数没变但人均拨打量降了27%，管理节奏出了问题"
├── 接通率下降 + 转化率不变 → 名单可打性下降
│   → "接通率降了5个百分点，需要评估名单质量"
├── 接通率不变 + 转化率下降 → 销售能力下降
│   → "转化率降了4个百分点，需要培训或调整话术"
└── 件均下降 → 客户结构变差
    → "转化量没少但件均降了，大额产出不足"

第四步：决定处理方式
├── 首次发现 + 主管态度积极 → 给2周观察期，看是否短期波动
├── 连续发现 + 主管配合 → 要求提交整改计划，设定具体目标和时间线
├── 连续发现 + 主管不配合 → 升级处理，约谈对方更高层级
└── 连续3个月未改善 → 启动清退评估（不是降分工，是留或退）
```

### 约谈话术模板

根据归因结果，生成约谈话术：

> "我看了你近30天的数据。你的GMV从X万降到Y万，下降幅度Z%。
> 从归因来看，主要问题是[归因结论]。
> - 具体数据：[关键指标变化]
> - 横向对比：[其他供应商同期数据]
>
> 你觉得你的问题出在哪？打算怎么调整？"

**示例 — 管理问题约谈**：

> "你的GMV从5200万降到3800万，下降27%。归因分析显示：出勤人数稳定在135-140人，但人均拨打量从80通/天降到58通/天（降幅27%），接通率和转化率保持稳定。所以你的GMV下降不是转化能力问题，是管理松弛导致人均拨打量大幅下降。同期其他供应商的人均拨打量保持在72-78通/天。你觉得管理上出了什么问题？打算怎么调整？"

### 区分个性问题与共性问题

| 现象 | 判断 | 处理方向 |
|------|------|----------|
| 只有一家供应商在降 | 个性问题 | 约谈该供应商 |
| 多家供应商同时转化率下降 | 共性问题，可能是名单/策略 | 反馈策略组/量化组 |
| 某个产线所有供应商都在降 | 产线问题 | 找业务管理协调 |
| 接通率全行业下降但只有你家转化率没降 | 名单问题但这家能力强 | 表扬+确认是否名单分配公平 |

**和刘乾坤的类比**：
- 他的时段趋势图 = 你的日度折线图（近30天）
- 他看三天叠加对比波峰波谷 = 你看30天趋势线判断是断崖还是缓降
- 他根据曲线形状判断"调整了拨打节奏不适应" = 你根据指标组合判断是转化率问题还是人力问题
- 区别：他的诊断靠经验（"纯心电图啊"），你的诊断靠指标交叉验证

## 看板四：新供应商成长看板（新供应商专用）

**用途**：跟踪新引入供应商前6个月的成长曲线。

**核心信息**：

| 信息 | 看什么 | 判断标准 |
|------|--------|----------|
| **月度排名轨迹** | 引入后每个月的排名位置 | 第2个月应在后50%，第4个月应到中位，第6个月应到前70% |
| **产能爬坡曲线** | 月度GMV折线图 | 应该逐月上升 |
| **转化率对比** | 与同产线老供应商的转化率对比 | 第3个月转化率应达到老供应商的80%以上 |
| **出勤率** | 月度出勤率 | 应保持在85%以上 |
| **里程碑达成** | 引入期→成长期→稳定期的阶段性目标 | 每个阶段结束前必须达标才能进入下一阶段 |

## 五个看板之间的关系

```
每月初：打开看板一（全景看板）
  → 先看供应商对比矩阵（总量×人均）
  → 标出"异常落位"的供应商
  → 看排名×人效趋势交叉（真进步/真退步/隐性退步/隐性进步）
  → 对自身峰值比<65%的供应商，打开看板三（单体深度看板）
    → 7指标交叉验证，定位根因
    → 主管胜任评估：是管理问题还是外部因素？
    → 决定动作：约谈/扶持/观察/清退
  → 更新综合排名，触发规则（预警/约谈/清退）
  → **输出叙事**：月度通报（见工具箱 叙事模板二）

每周一：打开看板二（对标看板）
  → 只看突变信号（出勤异常/业绩断崖/流失加速）
  → 出现异常 → 打开看板三定位根因
  → 决定是否提前介入（不等月底）
  → **输出叙事**：周度对标速报（见工具箱 叙事模板一）

事件触发：打开看板三（单体深度看板）
  → 确认异常根因后，触发约谈
  → **输出叙事**：预警约谈（见工具箱 叙事模板三）

新供应商：打开看板四（成长看板）
  → 跟踪里程碑
  → 判断是否按预期成长

现场巡检：巡检红黄绿 → 状态判定
  → 预警/危险 → **输出叙事**：巡检预警（见工具箱 红黄绿巡检叙事）

标杆分享（每月）：
  → 选标杆（自身峰值比>85% + 头部差距>80%）
  → 找对标（同产线自身峰值比<70%）
  → 配对跟踪：下个月看头部差距是否缩小
```

**看板与叙事的关系**：看板是"看"，叙事是"说"。看板帮你发现问题，叙事帮你把问题讲清楚、推得动人。

## 和刘乾坤的终极对比

| | 刘乾坤的看板 | 你的看板 |
|---|-------------|---------|
| **主看板** | 时段趋势图（每日） | 全景看板（每月）+ 对标看板（每周） |
| **看到什么** | 今天vs昨天的曲线形状 | 本周/本月的趋势和排名 |
| **发现异常后** | 直接@供应商问"啥情况" | 打开单体深度看板定位根因 |
| **诊断方式** | 经验判断（"纯心电图"） | 7指标交叉验证（出勤×人均拨打量×接通率×转化率×件均） |
| **动作触发** | 每天巡检 | 异常才触发 |
| **信息丰富度** | 1个维度（时段×金额） | 多个维度（排名+产能+转化率+出勤+流失） |

**你的看板比他的信息更丰富**，但频率更低。这是对的 — 你管的是月度趋势，不是小时波动。你不需要每天看，但你每次看的时候，看到的要比他多。
