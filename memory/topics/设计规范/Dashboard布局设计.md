## 核心经验

### 1. 内容密度不均衡时的处理

**问题**：当内容长短不一（如边界识别清单 vs 自我保护策略），单列列表会导致"瘦瘦长长"的视觉失衡。

**解法**：
- 改变版式：从单列列表 → 多列网格（3列卡片）
- 视觉填充：加图标、进度条、颜色区分、徽章
- 信息重组：短内容用卡片式，长内容保持列表或表格

### 2. 配色选择

**观察**：年老师不喜欢深色主题，偏好品牌模式（米色 #F5F1EE + 陶土 #E2725B）

**原则**：
- 先问偏好，不默认深色
- Dashboard 不必须深色，浅色同样专业

### 3. 布局居中问题

**问题**：主内容区 `max-width` 限制会导致内容偏右或挤压。

**解法**：
- 去掉 `max-width` 限制，让内容区占满右侧
- 或用 flex 居中 + 内部限制宽度
- 避免 `margin-left` 和 grid `1fr` 叠加导致的混乱

### 4. 组件复用策略

**做法**：
- 定义通用卡片样式（如 `.boundary-card`）
- 通过 modifier 区分状态（`.danger`/`.warning`/`.success`）
- 复用同一套交互逻辑（`toggleBoundaryItem`）

### 5. Dashboard 设计的核心判断

| 内容特征 | 推荐版式 |
|---------|---------|
| 内容密度高、条目多 | 列表或表格 |
| 内容密度低、分类清晰 | 卡片网格 |
| 需要快速扫描 | 图标+标签+进度 |
| 需要深度阅读 | 展开式详情 |

## 应用规则

后续做 Dashboard 时：
1. **先评估内容密度** — 长短是否均衡？
2. **不均衡时优先考虑网格布局** — 3列或2列卡片
3. **加视觉锚点** — 图标、颜色条、进度指示
4. **确认配色偏好** — 不问则默认品牌模式
