### ppt-master (PPT 生成)
- **首用日期**：2026-05-08
- **项目**：Aham PPT 介绍（11 页，公众号→咨询级 PPT）
- **流程**：source → project init → strategist (8 confirmations) → executor (SVG) → quality check → PPTX
- **效果**：原生可编辑（非图片），11/11 成功，约 2 小时
- **局限**：update_spec.py 有 delta 检测限制（直接 edit spec_lock 再运行会跳过）；radial gradient 在 svglib 下产生无害警告
- **教训**：
  - spec 上的配色批准 ≠ 视觉确认（#D94E3D 通过 confirmations 但看完完整 deck 后被拒）
  - 源材料清晰的公众号/报告 ✅，创意设计/复杂图表/严格品牌 decks ❌
  - 适合年老师"先跑→验收→逐步放权"的工作模式
- **外部记忆指针**：`memory/events/2026-05/2026-05-09.json`

## 前端设计经验

### Dashboard 布局
- 内容密度不均衡时：单列 → 多列网格
- 配色：年老师不喜欢深色，偏好品牌模式（米色+陶土）
- 主内容区 max-width 限制会导致视觉失衡

### SVG 手绘风格
- 设计哲学：线条+陶土色点缀，有机感
- Q贝塞尔曲线优于直线/矩形
- 禁止 emoji
- 风格系统 > 单一作品复刻

### 手绘 SVG 工具局限
- Quiver AI arrow-preview 模型：不支持精确颜色代码、不支持复杂结构指令、不支持特定文字生成
- 来源：`memory/topics/设计规范/手绘风格方案分析.md`

## Cron / 定时任务

### Cron / 定时任务治理（2026-05-09）

- **问题**：10 个 running 任务中 7 个长期空转（晚间复盘/晨间思考/轻量心跳/MemOS上传/书信补充检查/workspace整理/每日回顾）
- **根因**：prompt 设计像 SOP 不像会主动发现事的人，按步骤走完就结束
- **治理**：删除 7 个空转任务，保留 5 个有实际产出的
- **保留任务**：夜间待办提取（17次正常）、每周记忆维护（2次）、技能上游同步检查-调度（1次）、OPINIONS更新（1次）、每日书信-李娟风格（26次）
- **模型切换**：主 agent 从 qwen3.6-plus 切到 glm-5.1（zhipu provider），所有 cron/task 自动跟随
- **教训**：定时任务必须有实际产出验证，空转超过 3 次就应砍掉

### HEARTBEAT 机制
- **正确位置**：项目根目录 `HEARTBEAT.md`（非 `.claude/rules/`）
- **作用**：Agent 心跳苏醒时读取正文执行
- **推送四层架构**（2026-05-05）：会话检查 → 30min心跳 → 推送队列 → 紧急直推
- **写入铁律**：对话中实时写入 > 依赖心跳扫描
