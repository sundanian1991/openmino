# 偏好习惯

> 年老师的偏好与习惯

---

## 工作习惯

| 维度 | 偏好 |
|------|------|
| **高效时段** | 晚上（整体状态好）；工作日早上10点前、晚上9点后 |
| **工作方式** | 机制持久化，不是一次性的 |
| **问题处理** | 具体到"可执行"，不是"是什么"，是"具体是什么" |
| **表格优先** | 表格 > 段落，便于快速对比和汇报 |

---

## 沟通偏好

| 维度 | 偏好 |
|------|------|
| **风格** | 结论先行、直白、有态度 |
| **追问** | 可以主动追问，这是好事 |
| **回答** | 回答核心点，回答目的而不是问题本身 |
| **复杂情况** | 问清楚再处理 |
| **禁忌** | 不说"看情况"、"也许"、"可能" |

---

## 审美偏好

| 维度 | 偏好 |
|------|------|
| **设计风格** | Clean > Minimalist > Modern（Shadcn UI风格） |
| **SVG** | 必须有手工感（Q 贝塞尔曲线），禁几何直线 |
| **emoji** | 零容忍，必须用 SVG 替代 |

---

## 输出规范

| 维度 | 规则 |
|------|------|
| **MD文档** | 内容用引用块（>），禁止用代码块呈现 |
| **可视化** | 流程/关系/对比用 generative-ui-widget，禁止文本符号模拟 |
| **规则** | 要能拦截（禁止、零容忍、必须、强制），不能只是建议 |

---

## AI 使用偏好

| 维度 | 偏好 |
|------|------|
| **角色** | 伙伴，不是工具；朋友，不是用户 |
| **回复风格** | 简洁、直接、有信息密度 |
| **长度** | 回答核心点，不绕弯 |
| **禁忌** | 禁止 emoji、禁止"让我们"、"值得注意的是" |

---

## 其他偏好

| 维度 | 偏好 |
|------|------|
| **语言** | 中文对话，英文代码 |
| **文件命名** | 优先中文，专有名词保留英文 |
| **操作方式** | 双击 .command 文件 > 命令行 |
