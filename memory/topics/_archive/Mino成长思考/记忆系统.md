## 架构方案

### 当前方案：方案B（简化索引）

```
memory/
├── MEMORY.md        # 详细记忆索引（用户画像、决策、项目）
├── insights.md      # 洞察记录（合并daily/observations/weekly）
├── projects/        # 项目背景
├── topics/          # 主题文件（2026-03-18新增）
├── daily/           # 每日日志
└── archive/         # 历史归档
```

### 核心机制

**WAL协议**（Write-Ahead Logging）：
- 关键信息先写后答
- 触发条件：修正、专有名词、偏好、决策、草稿修改、具体值
- 执行方式：立即写入 MEMORY.md 或项目文件

**三层披露**：
1. MEMORY-L1.md：会话启动必读（核心记忆）
2. MEMORY.md：详细索引
3. topics/：项目深度文件

## 维护规则

**信息只存一处**：topic file 写详细了，核心记忆只放指针

**每条记忆带时间戳**：(YYYY-MM-DD)

**删比留更重要**：过时信息是噪音
