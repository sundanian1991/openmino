# project-workflow.md — 项目工作流模板

> **5 文件流融合版 — 让 AI 自主执行**

---

## 使用说明

**适用场景**：
- 复杂项目（>4 小时工作量）
- 需要 AI 自主执行多步骤任务
- 需要防止目标偏移/架构混乱

**核心文件**（必做）：
1. **Prompt** — 定义目标边界（要什么/不要什么）
2. **Plans** — 任务拆解 + 验收命令（1-2 小时里程碑）

**辅助文件**（按需）：
3. **Architecture** — 技术规范（防止架构频繁变更）
4. **Implement** — 操作手册（复杂流程才需要）
5. **Documentation** — 进度日志（daily/自动记录）

## 快速启动

**三步落地法**：

1. **准备阶段**：复制本模板到 `active/tasks/tracking/[项目名].md`
2. **填写 Prompt + Plans**（核心必做）
3. **启动指令**：
   ```
   先读 active/tasks/tracking/[项目名].md，按 Plans 顺序执行，每步跑验证命令
   ```

**监控方式**：定期查看 Documentation 进度，或等待 AI 汇报

---

## 核心洞察

- **80% 无效开发时间源于目标不明确和缺乏验证机制**
- **Plans 的"小里程碑 + 强验证"是核心解决方案**
- **排除项比目标更重要** — 防止 AI 过度发挥
