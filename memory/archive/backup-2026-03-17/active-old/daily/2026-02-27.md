## 下午 | API认证冲突问题排查

### 问题描述
- API报错：`API Error: 400 {"type":"error","error":{"type":"api_error","message":"模型不存在，请检查模型代码。"}}`
- 同时存在认证冲突警告：`Auth conflict: Both a token (ANTHROPIC_AUTH_TOKEN) and an API key (ANTHROPIC_API_KEY) are set`

### 排查过程
1. 检查环境变量发现同时设置了 `ANTHROPIC_AUTH_TOKEN` 和 `ANTHROPIC_API_KEY`
2. 确认根因：两个认证方式冲突导致API请求异常
3. 解决方案：执行 `unset ANTHROPIC_AUTH_TOKEN` 移除冲突认证
4. 验证测试：发送测试请求确认配置有效

### 涉及文件
- 更新：`memory/active/tasks/lessons.md` - 记录经验教训
- 更新：`.claude/rules/reference/05-self-review.md` - 记录到错题本

### 核心教训
- 环境变量操作需谨慎，先检查再修改
- 错误信息可能具有误导性，需要系统性排查
- 类似操作必须验证，不能假设问题已解决
