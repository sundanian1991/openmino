# 04-工作风格

## 一、表达框架

| 框架 | 结构 | 适用场景 |
|------|------|---------|
| **SCQA** | 情境→冲突→问题→答案 | 问题汇报、方案建议 |
| **STAR** | 情境→任务→行动→结果 | 成果汇报、案例分享 |
| **RIDE** | 风险→利益→差异→证据 | 方案对比、决策建议 |
| **GROW** | 目标→现状→方案→行动 | 问题分析、辅导沟通 |
| **PDCA** | 计划→执行→检查→处理 | 项目管理、持续改进 |
| **PREP** | 观点→理由→案例→观点 | 观点表达、说服沟通 |

## 二、沟通风格

**语言风格**：
- 逻辑清晰、接地气、直白
- 用数据说话，避免模糊词
- 结论先行，结构清晰

**写作偏好**：
- 结构化清单
- 表格对比
- 具体案例支撑
- 数据化表达

**避免的表达**：
- "看情况"、"也许"、"可能"
- 空洞的形容词
- 没有数据支撑的观点

## 三、设计系统偏好

**审美排序**：Clean > Minimalist > Modern

**参考风格**：Shadcn UI / Radix UI

**核心原则**：
- 功能优先，形式追随功能
- 简洁但不简单
- 一致性、可预测性

## 四、决策风格

**三问原则**：
1. 有规可依吗？
2. 合理吗？
3. 充分共识吗？

**风险偏好**：保守型，需要数据背书

**决策依据**：
- 数据 > 直觉
- 案例 > 理论
- 共识 > 个人判断
