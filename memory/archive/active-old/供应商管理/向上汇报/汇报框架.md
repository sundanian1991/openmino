# 汇报框架

## 一、核心故事线（四步法）

### 1. 定锚点
- 业务目标是什么？
- 领导关心什么？
- 我的价值定位？

### 2. 建体系
- 方法论是什么？
- 机制如何运转？
- 为什么可持续？

### 3. 亮成果
- 数据说话（GMV、满足率、淘汰率）
- 案例支撑（选汰、合规、应急）
- 价值可见（从"隐形管家"到"专业顾问"）

### 4. 展未来
- 下一步计划
- 需要什麼支持
- 长期愿景

## 二、50 个核心问题清单

**10 个模块**：
1. 业务定位与价值
2. 供应商储备与引入
3. 运营管理与分量调整
4. 评估与续签
5. 清退与汰换
6. 定价与结算
7. 合规与风控
8. 集中度与站点管理
9. 跨部门协同
10. 未来规划

## 三、价值可见性策略

**从"隐形管家"到"专业顾问"**：
- 问题发现 → 问题预防
- 执行汇报 → 方案建议
- 数据罗列 → 洞察输出

**汇报节奏**：
- 周报：常规运营数据
- 月报：深度分析 + 洞察
- 季报：成果总结 + 规划
- 年报：价值呈现 + 战略
