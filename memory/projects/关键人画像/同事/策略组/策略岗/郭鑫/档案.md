## 核心行为模式

### 1. 事实驱动的推动者 — 用数据说话，自己想好方案找人执行
**特征**：先定位根因，再判断可优化空间，最后提出具体技术方向找能执行的人。不包装、不表演、不抛问题。

**演变**：
- 04-01：焦虑型确认者，连环追问要确定性
- 04-13：冷静推动者，摆数据+指缺口+推行动
- 04-28：方案型推动者，自己想好技术方向，找人验证可行性

**判断**：健康的推动方式——问题导向、数据驱动、带着答案找执行者。与刘乾坤的"抛问题让别人答"本质不同。

### 2. 压力下的精准切割者 — 缩小范围、自己给方案、锁定交付物
**特征**：面对流程拖延时不等结论，自己给具体方案+缩小到最小切口+锁定具体交付物。不泛泛要结果，而是自己定义问题边界。

**演变**：
- 04-01：焦虑型确认者，连环追问要确定性
- 04-13：冷静推动者，摆数据+指缺口+推行动
- 04-28：方案型推动者，自己想好技术方向找人验证
- 04-30：精准切割者，不等马贵贤全量方案，自己给"30天≤9次"方案，切到"复贷人工场景"最小切口，最后"先搞定一个"锁定交付物

**判断**：从04-28到04-30的升级——不只是"带着答案找人"，而是**带着答案+缩小范围+锁定交付物**三步齐发。面对马贵贤的流程拉长（等分析+等节后），直接切最小可行场景破局。这是郭鑫最强的武器。

### 2026-04-30 — 外呼超频扣罚攻防战：压力下的精准切割

**事件**：邹嘉琪（郭鑫实习生）在molo端做临时上游拦截，排除29天内拨打≥4次的名单，名单量级减少43%。郭鑫借此推动质检岗（马贵贤）修改超频扣罚规则。

**行为链**：

| 阶段 | 行为 | 目的 |
|------|------|------|
| 接力跟进 | 邹嘉琪14:54抛拦截方案，郭鑫14:56立刻@马贵贤@王易人追问 | 把临时拦截变成根因解决的起点 |
| 追问节奏 | "他们要分析什么""5月初能有结论吗？"×2 | 不满意"等分析"的拖延，施压要时间节点 |
| 自己给方案 | "复贷在库名单，命中实时行为，30天内≤9次，不对职场超频扣罚" | 不等马贵贤，自己定义答案 |
| 缩小范围 | "复贷人工场景，先搞定一个吧" | 策略性退让：不要全量，要最小切口 |
| 锁交付物 | "辛苦了，先把任务大脑这个场景搞定" | 收束到具体交付，不接受发散 |

**与马贵贤的攻防**：

| 郭鑫（进攻） | 马贵贤（防守） |
|-------------|---------------|
| 追问时间节点 | 拉流程拉长节奏 |
| 自己给具体方案（≤9次） | "我已在推动""等林峰分析" |
| 缩小范围到最小切口 | "你确定只有复贷？"反将一军 |
| "先搞定一个"锁定交付 | 三层安抚：结论会有的+历史回溯+我负责 |

**关键判断**：
- 郭鑫的"5月同比又要20%"是核心驱动力——43%名单被拦截直接阻断业绩目标
- "先搞定一个"是刻意收敛：不给马贵贤"全量方案需要时间"的拖延空间
- 实习生邹嘉琪的43%拦截=谈判筹码，把业务损失摆在台面上，倒逼质检改规则
