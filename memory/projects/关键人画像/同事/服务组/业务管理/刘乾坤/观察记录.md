## 2026-03-18 — 立规矩：刚接手时的规则制定

### 事件
刘乾坤在03-18 20:16发立规矩消息

### 手法拆解
1. **身份宣布**："自今日起由我正式负责"
2. **沟通机制**：30分钟回复+指定对接经理+异常电话报备
3. **日常管理**：10:00前报出勤+每日复盘+数据准确+昵称格式
4. **明确后果**："明天12点前未修改的清除处理"

### 年老师评价
这是"立规矩"的经典案例——先定规则，后按规则办事。后续的管理都是"按规则办事"，不是"突然变严"。

### 效果分析
- **建立预期**：供应商知道"这个领导是这么工作的"
- **降低谈判空间**：不是"你们觉得怎么样"，是"就这么定了"
- **展示决心**：不是"说说而已"，是"会执行的"
- **后续有依据**：后续的管理是"按规则办事"

### 与后续案例的逻辑链条
1. **先立规矩**（03-18）：宣布规则
2. **后按规则办事**（03-20）：按规则要求反馈
3. **不合规就有后果**（潜在的）：按规则处理

### 模式识别

## 2026-03-20 — 任务下放案例（深度分析）

### 手法拆解
1. **任务具体化**：分配日报、录音学习、数据分析三项具体工作
2. **层层递进**：从简单（做日报）到复杂（深挖底层原因）
3. **CC上级展示**：同时抄送王易人，展示自己在培养新人
4. **责任明确+模糊并存**：具体任务明确，但包含"深挖底层原因"等具有挑战性的要求
5. **资源提供+期望设定**：提到可以从日报中找问题职场开始听录音，提供思路但不降低要求

### 年老师评价
这是典型的"任务下放+功劳上报"模式——给实习生可执行的任务，同时通过CC上级展示自己的管理能力

### 潜台词解读
- **向上展示管理能力**：CC王易人，让上级看到自己在重视人才培养
- **向下施压与培养并存**：任务看似简单，但包含深度要求，既考核实习生又能发现问题
- **责任下放**：如果实习生发现了有价值的见解，是"我带出来的"；如果没发现，是"实习生不够用心"

### 后续待观察
张世跃的执行情况、是否能提供有价值的输出，刘乾坤如何评价和利用这些输出

### 二次分析发现
- **任务设计艺术**：不是简单的"给任务"，而是"设计成长路径"
  - 第一层：管马老师要录音 → 学习标杆+建立跨组连接
  - 第二层：每天听5通录音 → 量化可执行+优先问题职场（闭环验证）
  - 第三层：一起共建方案 → 主导权在手+数据支撑
- **系统性思维**：日报发现问题 → 录音验证问题 → 数据分析 → 方案输出（完整闭环）
- **跨组影响力**：通过实习生任务连接马贵贤，扩大自己的影响范围
- **"我们一起"话术**：典型责任转移——主导权在我，执行你来做

## 2026-03-20 — "抛问题"手法的深化：连"如何答复"都要别人教

### 事件
刘乾坤转发供应商数据下降问题，@郭鑫说："鑫鑫帮我看下，我该如何解答，后续帮我整几个抓手给到职场"

### 背景
刘乾坤（金条业管）、郭鑫（金条策略）曾是同团队成员，关系密切

### 手法拆解
1. **亲昵称呼+客气请求**："鑫鑫"+"帮我看下"—— 用私交覆盖上下级界限
2. **装作不知道**："我该如何解答"—— 明明可以自己查证判断，却装作不会
3. **责任三重转移**：
   - "如何解答" → 让郭鑫设计回复内容
   - "给职场抓手" → 让郭鑫产出行动方案
   - 如果出问题 → "我让策略看了，他们给的方案"
4. **群里公开施压**：不私聊，@所有人都能看到，制造回复压力

### 潜台词解读
- **"用你的嘴，出我的力"**：让你背锅答责，我居功组织推动
- **"我不下判断"**：如果结论错了，不是我的锅；如果对了，我是"发现问题的人"
- **"别只解释，要方案"**：压郭鑫出成果，不是"解释一下"就完事
- **"让观众看到"**：让群（包括上级）看到"我在盯这件事、在推动解决"

### 郭鑫的实际反应
- 在群里解释了数据下降的原因
- 给出了解决方案
→ **被成功调动，而且是在公开场合被调动**

### 年老师评价
这是"抛问题"手法的**深化表现**—— 之前只是摆数据@策略，这次连"怎么回供应商"都要别人来教他，**责任转移更加彻底**

### 与之前的对比
- 之前案例（03-19）：摆数据+抛问题@策略，自己不下原因分析
- 这次案例（03-20）：**连"如何答复"都不自己想，完全让郭鑫来设计回复**
→ **极致之处**：零责任声明，所有判断、答复、方案都让别人来，自己只保留"组织者"身份

### 模式识别
- 模式1：抛问题（深化）

## 2026-03-20 — "抛问题"手法的完整闭环：从发起问题到别人执行

### 事件
人工首复贷数据分析引发的完整讨论链

### 完整对话链
1. 刘乾坤抛出3个问题（人工/人机目标、协同状态、协同机制）
2. 刘伟佳回应5个方向（目标、激励、过程指标、名单、复盘），@郭鑫 @刘乾坤 "你们再好好研究下"
3. 刘乾坤不接任务，而是细化成4个"明确"（目标、机制、奖惩、复盘）
4. 郭鑫接任务："好的 中午前过下"

### 手法拆解
1. **发起问题**：摆数据（"我拉了近一个月数据"），抛出3个问题
2. **不接任务**：刘伟佳说"你们研究"，他不说"我来研究"，而是说"建议..."
3. **细化问题**：把任务变成4个"明确"，看起来在推进，实际是不承诺执行
4. **任务落到别人**：郭鑫最终接了任务

### 潜台词解读
- **"我发现了问题"**：摆数据，展示工作量
- **"你们来解释"**：抛出3个问题，不下判断
- **"我不接任务"**：刘伟佳说"你们研究"，他不说"我来研究"
- **"我把问题细化"**：把任务变成4个"明确"，看起来在推进，实际是不承诺执行
- **"最终是你做"**：@刘伟佳 @郭鑫，但郭鑫是实际执行人

### 年老师评价
这是"抛问题"手法的**完整闭环**——从发起问题到别人执行。关键在于：刘乾坤**不接任务**，而是把任务细化成更多问题，最终让别人去执行。

### 闭环的关键
- 刘伟佳说"你们再好好研究下"——派回任务
- 刘乾坤不说"我来研究"，而是说"建议..."——不接任务
- 刘乾坤把任务细化成4个"明确"——看起来在推进，实际是不承诺执行
- 郭鑫接任务："好的 中午前过下"——实际执行人

### 模式识别
- 模式1：抛问题（完整闭环）

## 2026-03-20 — 对供应商的"抛问题"：公开质问人力管理

### 事件
在群里@9个供应商（毅航、伽玛、赛维斯、博岳、广大、华啸、翰锐、人和、岐力），质疑人力管理

### 手法拆解
1. **摆数据**：人力100%，但培训班和入职都是0——发现问题
2. **@9个供应商**：公开施压，让所有人都看到
3. **5个连续问题**：活水补充计划？尾部淘汰后有无储备？汰换机制？每周招聘计划？
4. **客气施压**："辛苦同步到群里"——表面客气，实际要求公开回复

### 潜台词解读
- **公开质问**：不是私聊沟通，而是在群里公开摆数据、抛问题
- **向上展示**：让上级（年老师）看到"我在盯供应商人力问题"
- **向下施压**：9个供应商都被@，不回显得不配合
- **不下判断**：摆数据（培训班和入职都是0），但不下结论，让供应商自己解释

### 年老师评价
这是"抛问题"手法的**供应商版本**——与对同事的"请教"不同，对供应商是"质问"。摆数据、公开@、连续问题，更强势。

### 与之前案例的对比
- 郭鑫案例（03-20）："帮我看下，我该如何解答"——客气请求
- 供应商案例（本次）：5个连续问题——公开质问
→ **差异**：对平级是"请教"，对下级/供应商是"质问"

### 模式识别
- 模式1+模式3：抛问题+公开锁定（组合应用）

## 2026-03-20 — 供应商管理：利益驱动型（业绩冲刺）

### 事件
月末冲刺，3.4亿才有奖金

### 手法拆解
1. **利益驱动**："全现场要做3.4亿才有奖金可以拿！！"——用奖金作为动力
2. **数据说话**："今天时间过半，我们日度达成率38%"——精确数据
3. **责任下放**：4个问题让供应商自己安排
4. **责任到人**："把目标拆解到每个站点、每个团队长"
5. **明确反馈时间**："下班前反馈我"

### 潜台词解读
- **利益驱动**：用"3.4亿才有奖金"作为动力，让供应商自己急
- **数据说话**："时间过半、达成38%"——精确数据，不模糊
- **责任下放**：4个问题都是让供应商自己安排，不是他来安排
- **责任到人**："每个站点、每个团队长"——责任到人
- **明确反馈时间**："下班前"——不模糊

### 年老师评价
这是"目标驱动型管理"——不是"我来安排"，是"你们来安排，下班前告诉我"。

### 与"质问式"的对比
- **质问式**："怎么回事？"——让供应商解释
- **驱动式**："没奖金你们自己看着办"——让供应商自己急

## 2026-03-20 — 自己拉数据+抛问题：责任转移的进阶版

### 事件
在群里发"人工首复贷数据分析"，@李刚 @李吉斌 @王易人

### 手法拆解
1. **展示工作量**："我拉了近一个月数据"
2. **下结论但立即转移**："40%非人工业绩"——自己下了判断，但"这里边几个问题"——把分析责任转移出去
3. **三个问题都是"是什么"**：不问"为什么"，只问"是什么"——让你们给事实，我不下判断
4. **@三个人**：李刚、李吉斌、王易人，制造公开压力

### 潜台词解读
- **"我在干活"**：展示自己拉数据、分析数据的工作量
- **"数据摆在这，你们来解释"**：即使下了结论，但把后续分析抛给别人
- **"不下判断"**：问题都是"是什么"，不是"为什么"——让你们给事实，我来汇总
- **"公开施压"**：@三个人，不回显得不配合

### 年老师评价

## 2026-03-21 — 周末@人习惯：责任边界模糊

### 事件描述

年老师反馈：**刘乾坤会在周末不断@他处理各种供应商问题**。年老师双休日不常上班，但刘乾坤不管，@就来了。

**典型情况**：
- 供应商晚10分钟回复数据
- 供应商晚半小时回复数据
- 供应商出行安排不到位
- 各种日常小问题

### 核心矛盾

**年老师的困惑**：
- 周末双休被@处理问题，烦人且不爽
- 责任边界不清：供应商日常出勤管理，到底是谁的职责？
  - 是年老师制定制度并监管？
  - 还是所有事都要年老师亲自完成？

**刘乾坤的逻辑链**：
```
供应商归你管 → 出问题你是第一责任人 → 周末也该你盯 → 我@你是天经地义
```

**偷换概念**：把"管理责任"偷换成了"随叫随到"。

### 行为模式

| 维度 | 平时 | 周末 |
|------|------|------|
| @对象 | 工作时间能找到人 | 双休日也能@到你 |
| 预期 | 你赶紧处理 | 你"应该"在处理 |
| 潜台词 | "你来办" | "你的供应商，你负责" |

### 年老师的不适

**深层不适**：刘乾坤用年老师当"执行器官"，自己当"大脑"。
- 口头上指示
- 其他方面施压
- 工作完成后，他的掌控力得以体现
- 仿佛这件事既能落实，又显示了控制力

**年老师的总结**：
> "无论如何，他自己没有完成的工作必须由其他人帮忙一起完成。工作完成后，他的掌控力似乎得以体现。虽然看起来是他自己在做，但实际上这不仅涉及口头上的指示，还涉及许多其他方面。这种做法让我感到不太舒服。"

### 模式识别
- 模式1（抛问题）的周末变体
- 新维度：责任边界模糊——"供应商归你管"变成"你随叫随到"
- 隐性期望：周末也要像工作日一样响应

### 责任边界的正确理解

| 职责 | 谁来做 |
|------|--------|
| 制定出勤制度框架 | 年老师（或公司统一规定） |
| 供应商日常执行 | 供应商自己 |
| 监督和考核 | 年老师（定期看数据，不是实时盯群） |
| 周末应急处理 | 供应商自己该有机制 |
| **周末@年老师处理** | **不是年老师的义务** |

### 互动案例：年老师的反击（2026-03-21）

#### 背景
刘乾坤周末不断@年老师处理供应商问题，年老师不舒服，决定用制度解决。

#### 年老师的操作
年老师在群里公开提议"供应商日常配合扣分项纳入赛马"方案：
- "坤哥，你看我这个方案是不是可行"
- "我后面也能有抓手更好的追踪"
- "坤哥你给点建议"

#### 刘乾坤的回复（11:16）
> "@孙大年 谢谢兄弟，有思考有深度~我也建议做个小额扣分项，周一咱一起看下"

#### 潜台词解读
| 表面回复 | 潜台词 |
|----------|--------|
| "谢谢兄弟" | 我不叫你"大年"，我主动拉近关系 |
| "有思考有深度" | 给你面子，认可你的方案 |
| "我也建议做个小额扣分项" | 我也这么想，咱俩想到一块了 |
| "周一咱一起看下" | 不现在讨论，延后——不给你施压机会 |

#### 模式分析
- **刘乾坤接住球**：他没反对，反而赞同，避免对立
- **"我也这么想"**：把你的提议变成"我们共同的提议"
- **"周一一起看下"**：不现在敲定，留缓冲——可能他觉得需要想想

#### 年老师的高明之处
| 做法 | 效果 |
|------|------|
| 在群里公开提方案 | 不是私聊，是公开提议——其他人也看到 |
| "你看这个方案是不是可行" | 不是"我决定了"，是"征求意见"——给足他面子 |
| "我后面也能有抓手更好的追踪" | 解释动机，不是针对他，是工作需要 |
| "坤哥你给点建议" | 主动示弱，让他来"建议"而不是"被动接受" |

#### 结果
- 刘乾坤没法反对——反对就是反对"更好的追踪供应商"
- 刘乾坤接住了球——"我也建议"
- 方案推进了——周一一起看
