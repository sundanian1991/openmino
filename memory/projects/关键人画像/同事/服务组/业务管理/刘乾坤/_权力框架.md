## 权力内核

**他想要什么？**

| 诉求 | 表现 | 深层动机 |
|------|------|----------|
| **控制权** | 立规矩、阻断信息、责任下放 | 刚接手新业务，需要建立"我在管"的信号 |
| **展示权** | 群内表演、打鸡血、CC上级 | 最在意"老板知不知道我在解决" |
| **安全感** | 信息管控、建立依赖 | 怕被替代，所以要让别人依赖自己 |

**破绽**：
- 表演型管理 → 真实能力可能不足
- 信息管控 → 信任缺失，长期合作困难
- 施压过度 → 容易引发抵触

## 权力来源

**权力从哪来？**

### 1. 规则权

| 维度 | 内容 |
|------|------|
| **为什么有效** | 后续管理是"执行既定规则"不是"个人施压" |
| **对应模式** | 立规矩（模式7）、机制先行 |
| **典型案例** | 03-18 接手金条，宣布 7 条规则 |
| **识别信号** | 刚接手就"先建机制再谈业务" |
| **应对方式** | 确认规则合理性，不盲目接受 |
| **可学习** | 规则先行，让管理有据可依 |

### 2. 信息权

| 维度 | 内容 |
|------|------|
| **为什么有效** | 对手只能依赖自己获取信息 |
| **对应模式** | 两头堵（模式13）、阻断信息 |
| **典型案例** | 03-23 "别找供应商"+"量化能抓的别找他们" |
| **识别信号** | 阻断了你和数据源的接触 |
| **应对方式** | 建立独立渠道，"我需要一线真实反馈" |
| **可学习** | 信息是权力，保持独立信息源 |

### 3. 关系权

| 维度 | 内容 |
|------|------|
| **为什么有效** | 人情债换取配合 |
| **对应模式** | 拉拢圈人（模式8）、人情债管理（模式12） |
| **典型案例** | 03-20 "兄弟帮你一块弄了" |
| **识别信号** | 私聊帮忙、亲昵称呼、主动让利 |
| **应对方式** | 接受但不欠人情，合作但不依赖 |
| **可学习** | 主动建立互惠关系 |

### 4. 注意权

| 维度 | 内容 |
|------|------|
| **为什么有效** | 在上级面前建立"我在解决"的形象 |
| **对应模式** | 群内表演（模式2）、打鸡血 |
| **典型案例** | 03-19 连发两条数据、03-23 领导力展示版 |
| **识别信号** | 消息多发在有上级的群、CC上级 |
| **应对方式** | 不抢戏，专注自己的责任边界 |
| **可学习** | 向上展示要有，但不过度 |

### 5. 施压权

| 维度 | 内容 |
|------|------|
| **为什么有效** | 公开施压让人不得不接 |
| **对应模式** | 公开锁定（模式3）、高压管理（模式4）、加压链（模式10） |
| **典型案例** | 03-19 @郭鑫+@马贵贤、03-21 三连击 |
| **识别信号** | 群里@具体人、连环追问 |
| **应对方式** | "你的想法是什么？我先听你的" — 不接任务，反问回去 |
| **可学习** | 三要素法让供应商自己承诺 |

## 识别信号

**怎么看出他在用哪招？**

### 早期信号（刚接手时）
- "先建机制再谈业务" → 规则权
- 宣布规则、要求格式 → 立规矩

### 日常信号（模式触发场景）
- 群里@具体人 + 所有人看着 → 公开锁定
- 私聊帮忙 + 亲昵称呼 → 拉拢圈人
- 连发消息 + CC上级 → 群内表演
- "别找XX，他们不如我们清楚" → 两头堵

### 危险信号（什么时候会被锁定）
- 被指定为"第一责任人"和"最终验收人" → 责任锁定
- 被追问"你的想法是什么" → 抛问题
- 被连环@追问 → 加压链

## 可学习内容

**能学什么？**

| 技巧 | 核心逻辑 | 适用场景 |
|------|----------|----------|
| **立规矩** | 先定规则后办事，后续是"执行既定规则" | 接手新业务、管理新团队 |
| **三要素法** | 问题 → 原因 + 提升目标 + 举措，让别人承诺 | 供应商管理、推动执行 |
| **数据说话** | 精确数据展示 + 公开施压 | 汇报、驱动、施压 |
| **机制先行** | 不急于谈业务，先要机制 | 建立长期管理体系 |
