# 交互速度偏好

**日期**：2026-03-10
**来源**：年老师反馈
**类型**：偏好/修正

---

## 核心问题

会话启动时加载文件过多，导致交互太慢。

---

## 具体反馈

> "刚刚交互忒慢了啊"

---

## 原因分析

当前会话启动流程（06-NOW.md）：
1. 读 SOUL.md
2. 读 USER.md
3. 读 session_status
4. 读 README.md
5. 读 CLAUDE.md
6. 读 06-NOW.md
7. 读 MEMORY-L1.md
8. 读 daily 文件列表
9. 读最近的 daily 文件

**问题**：串行读取，每个 read 工具调用都要等待，累积延迟。

---

## 优化方案

### 方案 A：最小化启动（推荐）
- **只读核心 3 文件**：06-NOW.md、MEMORY-L1.md、session_status
- **按需读取**：其他文件在需要时才读
- **预期收益**：减少 60-70% 启动时间

### 方案 B：并行加载
- 将不依赖的文件改为并行读取
- 但需要评估工具是否支持并行

### 方案 C：精简文件
- 将 06-NOW.md 中的非必要内容移到按需读取区
- 减少 MEMORY-L1.md 的篇幅

---

## 决策

**采纳方案 A**：最小化启动 + 按需读取

**新启动流程**：
1. 06-NOW.md（核心气质 + 最近讨论）
2. MEMORY-L1.md（核心记忆）
3. session_status（当前状态）
4. **其他文件按需读取**
