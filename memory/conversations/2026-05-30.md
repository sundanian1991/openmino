### 主题一：How to AI 技能开发

**上下文**：年老师在销售综合运营转型通道（100+人、1000+工作项），大部分同事没接触过 AI。核心需求：改造 howto-ai 技能，让同事通过对话式引导完成 AI 转型。

**讨论（多轮迭代）**：

1. **技能改造方向**：原版是个人用的 AI 落地顾问，改造后要支持团队——通过 Excel 员工工作数据 + 对话式引导，帮助同事梳理工作、匹配技能、创建自动化任务。

2. **交互流程设计**（5-6轮对话）：
   - 第一轮：身份识别（ERP姓名 → 匹配 Excel 数据 → 确认部门岗位）
   - 第二轮：工作优先级划分（3-5个优先改造项）
   - 第三轮：技术能力评估 + 痛点确认 + 时间预算
   - 第四轮：扫描匹配（buy/build 分流 → skill或自建）
   - 第五轮：引导创建 skill / 安装现成 skill + 定时任务脚本
   - 第六轮：生成 HTML 报告

3. **多风格报告模板**：年老师要求 7 种 HTML 风格模板（apple简洁、hallmark、loreal欧莱雅、muller瑞士网格、nothing、web-design、tufte），最终选定 nothing 为首选。

4. **进化机制**：技能需要持续迭代能力，不是一次性使用。通过 state.json 记录状态（当前周次、已装技能、待做事项），每次调用读取状态 → 推进到下一步。

5. **模板质量反馈**：
   - 欧莱雅风格"散的"，不像模板
   - 部分模板占位符数量差异大
   - 来源标签应横向放置（能放得下就不竖向）
   - 定时任务示例统一用 joyclaw（小龙虾）

6. **Excel 解析问题**：如果解析逻辑复杂，可改为 Markdown/CSV 格式。

**结论**：技能框架确定，7 种风格模板已生成，交互流程 5-6 轮对话定稿，state.json 状态管理机制确认。

**关键原话**：
> "我想把策展改成我个人在做可视化、分析、呈现时的工具"
> "我希望给多个版本根据不同的这个人的基础能力——偏小白还是偏有一点基础的——提问的措辞和深度要有所差异"
> "不是只有第一周选定一些技能聊一次，后续就无法使用，那也算不上 How to AI"
> "占位符模板才可以，我看现在的几种风格模板，它不是那个占位符模板呀"

### 主题二：策展技能改造

**上下文**：年老师希望将 curatorial（策展）技能从"可视化后的展示"改造为"可视化前的分析工具"。

**讨论**：
- 策展技能的定位：先想清楚有什么内容、从什么角度、如何呈现
- 数据讲故事一脉相承：策展在深度上推动分析过程
- 目标：任何材料交给策展 → 聚焦视角 → 串起链路 → 快速、专业、标准且有独特角度完成工作

**结论**：改造方向明确——策展作为分析前置工具，与可视化技能串联。

**关键原话**：
> "策展把分析思路变成真正的展览，提升深度、广度和聚焦，使其更专业"

### 主题四：Storytelling with Data 可视化网页

**上下文**：用 nothing 风格将 1.6 万行《Storytelling with Data》读书笔记做成可视化 HTML 网页。

**讨论**：
- 要求浅亮色、策展技能完整跑一遍分析流程
- 内容需要足够深度、信息密度、可读性综合考虑
- 对图的反馈：描述与内容未对应、位置重合、缺少延展
- 使用 nothing-design 风格

**结论**：需改进图质量——描述与内容对应、避免重合、增加延展内容。

**关键原话**：
> "策展的技能可以用来分析。内容还是要有足够的深度的，然后信息密度以及可读性也要综合考虑"
> "图一这边，感觉你的描述和图的内容没有完全对应，也没有做好突出"
