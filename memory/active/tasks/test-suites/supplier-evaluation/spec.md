# 供应商评分计算 Spec

> **版本**: v1.0
> **日期**: 2026-02-28
> **作者**: 年老师

---

## 功能目标

根据供应商的 KPI 得分、合规记录、交付及时率，自动计算综合评分、等级和风险标识，为供应商选汰提供数据支撑。

---

## 输入

| 字段 | 类型 | 必填 | 范围 | 说明 |
|------|------|------|------|------|
| kpi_score | number | 是 | 0-100 | KPI 得分 |
| is_compliant | boolean | 是 | - | 是否合规（true=合规） |
| ontime_rate | number | 是 | 0-1 | 交付及时率 |
| quality_rate | number | 否 | 0-1 | 质量合格率（默认 1） |

---

## 输出

| 字段 | 类型 | 说明 |
|------|------|------|
| grade | string | 等级（A/B/C/D） |
| risk | string | 风险标识（正常/预警/高危） |
| score | number | 综合得分（0-100） |
| reasons | string[] | 评级原因说明 |

---

## 业务规则

### 规则 1：合规一票否决

**条件**: `is_compliant = false`

**结果**:
- `grade = 'D'`
- `risk = '高危'`
- `score = 0`

**说明**: 合规是底线，一旦违规直接列入高危，无论其他指标多好。

---

### 规则 2：KPI 阈值判定

**条件**: `kpi_score < 60`

**结果**:
- `grade ≤ 'C'`（最高为 C）
- `risk = '预警'`（如果 risk 不是高危）

**说明**: KPI 低于 60 分说明表现不佳，需要预警。

---

### 规则 3：及时率惩罚

**条件**: `ontime_rate < 0.8`

**结果**:
- 等级降级处理（A→B, B→C, C→D）
- 如果已经是 D，保持 D

**说明**: 交付及时率低于 80% 需要惩罚。

---

### 规则 4：综合得分计算

**公式**:
```
score = kpi_score * ontime_rate * quality_rate * 权重系数
```

**权重系数**:
- A 级：1.0
- B 级：0.95
- C 级：0.9
- D 级：0

**说明**: 综合得分反映供应商的整体价值。

---

### 规则 5：等级判定

| KPI 范围 | 基础等级 |
|---------|---------|
| 90-100 | A |
| 80-89 | B |
| 60-79 | C |
| 0-59 | D |

**说明**: 先根据 KPI 确定基础等级，再应用惩罚规则。

---

## 边界条件

### 边界 1：输入为空

**条件**: 任一必填字段为 null/undefined

**处理**: 抛出错误 `"缺少必填字段：[字段名]"`

---

### 边界 2：KPI 超出范围

**条件**: `kpi_score < 0` 或 `kpi_score > 100`

**处理**: 抛出错误 `"KPI 得分必须在 0-100 之间"`

---

### 边界 3：及时率超出范围

**条件**: `ontime_rate < 0` 或 `ontime_rate > 1`

**处理**: 抛出错误 `"交付及时率必须在 0-1 之间"`

---

### 边界 4：质量合格率缺失

**条件**: `quality_rate` 为 undefined

**处理**: 使用默认值 `1.0`

---

## 性能要求

- 单次计算响应时间：< 10ms
- 批量计算（1000 条）：< 1s

---

## 验收标准

- [ ] 测试用例 1: 合规 + 高 KPI → A/正常
- [ ] 测试用例 2: 不合规 → D/高危
- [ ] 测试用例 3: KPI<60 → C 或以下
- [ ] 测试用例 4: 及时率<0.8 → 降级
- [ ] 边界测试 1: 输入为空 → 抛错
- [ ] 边界测试 2: KPI 超范围 → 抛错
- [ ] 边界测试 3: 及时率超范围 → 抛错
- [ ] 边界测试 4: 质量率缺失 → 默认 1.0
