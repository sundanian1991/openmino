## 分形文档架构 + 自指三行注释

### 背景
引入 OpenViking 记忆管理理念，学习 Vibe Coding 分形文档架构。

### 核心成果

**三级 CLAUDE.md 体系**：
```
memory/
├── CLAUDE.md              # L1: 全景地图
├── daily/CLAUDE.md        # L2: 事实记录模块
├── observations/CLAUDE.md # L2: 洞察模块
├── weekly/CLAUDE.md       # L2: 周文档模块
├── my-thoughts/CLAUDE.md  # L2: 随感模块
└── tasks/CLAUDE.md        # L2: 任务系统
    ├── system/CLAUDE.md   # L3: 系统机制
    ├── reminders/CLAUDE.md # L3: 定时提醒
    ├── scripts/CLAUDE.md  # L3: 管理脚本
    ├── cognition/CLAUDE.md # L3: 认知框架
    ├── tracking/CLAUDE.md # L3: 跟踪数据
    └── templates/CLAUDE.md # L3: 模板
```

**自动化生长规则**：
| 操作 | 自动执行 | 手动命令 |
|------|---------|---------|
| 写新文件 | observer/UPDATE_MEMORY 调用 | `--action update-all` |
| 修改文件 | - | 手动运行 `update-daily` |
| 新建目录 | - | `--action create-claude-md` |
| 删除文件 | lifecycle_manager 清理 | `--action cleanup` |

**管理脚本**：
- `index_manager.py` — 更新 daily/observations 索引 + 创建 CLAUDE.md
- `lifecycle_manager.py` — 检查/清理超期文件（P0/P1/P2）

### 自指三行注释（Vibe Coding）

每个文件开头包含三行注释：
```markdown
---
input: [依赖外部资源]
output: [对外提供功能]
pos: [系统局部地位]
# 文件更新需同步注释及所属文件夹 md
---
```

**已更新文件**：
- daily/CLAUDE.md — 加入三行模板
- scripts/CLAUDE.md — Python 文件头规范
- index_manager.py — 文件头加入 input/output/pos
- lifecycle_manager.py — 文件头加入 input/output/pos
- memory/CLAUDE.md — 加入自指三行规则章节
- observer.md — Step 1 加入文件头模板
- UPDATE_MEMORY.md — 生命周期标记章节加入三行说明

### 核心认知

**环境优先原则**："不是让 AI 更聪明，是给它一个配得上它能力的工作环境"

**分形文档**：每层级统一要素（Summary/Members/Rules），层级间形成"地图 - 入口"关系

**自指机制**：文件说明自己的 input/output/pos，实现"局部影响整体，整体影响局部"的自组织
