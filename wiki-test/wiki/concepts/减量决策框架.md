# 减量决策框架

> 供应商提出减量诉求时，年老师的核心判断点：要不要同意减量？

## 要搞清楚的真实问题

| 问题 | 为什么重要 |
|------|-----------|
| "人多亏得多"具体亏多少？ | 量化亏损，判断是真亏还是预期管理 |
| 亏的是固定成本还是变动成本？ | 固定成本（房租/管理）减人也解决不了 |
| 减到多少人能盈亏平衡？ | 判断合理区间 |
| 减下来的人去哪？ | 是转项目还是裁员？ |
| 减量后业绩会不会更差？ | 人少了产能可能更低（士气/管理成本摊薄） |

## 年老师的顾虑验证

| 顾虑 | 需要验证的 |
|------|-----------|
| 短期波动不应调整 | 近4个月是趋势性下滑还是波动？看6个月以上数据 |
| 开内部转项目口子 | 他们有没有提转项目？转去哪？ |
| 后续结算波动出大问题 | 如果同意减量，什么条件下可以加回来？ |

## 决策输出模板

```
建议：同意/不同意/有条件同意减量

理由：
1. 数据支撑：...
2. 现场观察：...
3. 一线反馈：...

条件（如果同意）：
1. 减到X人，盈亏平衡线
2. 月度review，达标后可加回
3. 不开放内部转项目，只做自然减员

替代方案（如果不同意）：
1. ...
2. ...
```
