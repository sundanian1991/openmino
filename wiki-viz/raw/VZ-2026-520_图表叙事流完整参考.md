## 一、体系架构概览

图表叙事流分为四层：

```
Layer 1: 意图识别 → 72+16 种视觉模式
    ↓
Layer 2: 叙事设计 → 11 种叙事弧线
    ↓
Layer 3: 单图设计 → 20 种构图模板 + 26 种视觉节拍
    ↓
Layer 4: 视觉执行 → 13 种开场骨架
```

### 2.3 小多图矩阵设计原则（Wilke）

> **关键原则：所有面板必须使用相同坐标轴范围和刻度**

| 参数 | 规范 |
|------|------|
| 网格排列 | 左→右、上→下（阅读顺序）|
| 轴范围 | 所有面板必须相同（关键！）|
| 轴刻度 | 所有面板必须一致 |
| 面板标签 | 由分面变量自动定义（非字母）|
| 排序逻辑 | 按数据维度有意义的顺序 |

**Wilke 警告**：不同 Y 轴范围会严重误导读者（见图 21.3 vs 21.4 对比）

### 2.4 复合图设计原则（Wilke）

| 参数 | 规范 |
|------|------|
| 面板标签 | 从左上角 "a" 开始，左到右、上到下 |
| 标签样式 | 低调如页号，不要过度突出 |
| 视觉语言 | 颜色、符号、字体跨面板统一 |
| 对齐 | 轴线和图形元素必须精确对齐 |

## 三、视觉节拍（26 种）

### 3.1 基础节拍（20 种）

| 编号 | 名称 | 阅读路径 | 关键参数 |
|------|------|---------|---------|
| C-01 | 锚定数字 | 大数字→单位→上下文 | 48-72px，周围留白 ≥ 数字宽度的 1.5 倍 |
| C-02 | 基准线穿越 | 基准线→上下分布→幅度 | 1.5px 虚线，正负区域填充透明度 15% |
| C-03 | 排序阶梯 | 渐变色条→最高最低→差距 | 严格降序，颜色渐变（最大=饱和色）|
| C-04 | 色彩弹出 | 唯一点彩色→标签→对比 | 全局仅 1 个饱和色（HSL ≥ 80%），其余灰度 30-50% |
| C-05 | 对比揭示 | 相似→差异区→量化值 | 双系列重叠，差异区域阴影/颜色填充（透明度 20-30%）|
| C-06 | 异常闪烁 | 红色标记→数值→偏离度 | 正常范围灰色阴影带 #F0F0F0 透明度 50%，超出范围红色 #E74C3C |
| C-07 | 标注箭头 | 数据点→注释→因果 | 箭头线宽 1.5px，箭头头部 8px，注释文字 ≤ 25 字 |
| C-08 | 趋势拐点 | 走向→拐点→斜率变化 | 拐点处画圆点（直径 8px，白色描边 2px），分段着色 |
| C-09 | 叠加融合 | 底层→叠加层→关系 | 底层不透明度 100%，叠加层 2px 实线/虚线，预测带半透明 20% |
| C-10 | 密度变换 | 密度模式→聚集区→空白 | 散点/热力图，hexbin/kernel density，颜色梯度 5 级 |
| C-11 | 隔离突出 | 隔离元素→数据→原因 | 独立卡片/放大框，背景色 #F8F9FA，微妙阴影 |
| C-12 | 分组归类 | 组间距→组内相似→标签 | 组间距 ≥ 组内间距的 2 倍（Gestalt proximity）|
| C-13 | 近邻关联 | 靠近→关系→含义 | 两元素间距 < 元素宽度的 30%，细线连接，共享色系 |
| C-14 | 面积编码 | 最大块→排序→占比 | 面积 ∝ 数值，最大气泡直径 ≤ 画布宽 40%，必须配数字标签 |
| C-15 | 流动引导 | 方向→流量→路径 | 桑基/流线，线宽 ∝ 流量，颜色区分流向 |
| C-16 | 时序脉冲 | 时间轴→波峰谷→周期 | 面积图或柱状图，时间轴等距刻度，季节性背景色带标记 |
| C-17 | 缩放聚焦 | 全局→选区→细节 | 左侧小图 30% 带选择框，右侧大图 65%，选择框颜色与高亮区一致 |
| C-18 | 维度旋转 | 当前维度→切换→关键 | 同一数据集多视图切换（Tab/翻页），相同坐标轴范围 |
| C-19 | 层级折叠 | 汇总→展开→细节 | 树状缩进+展开图标（▶），每层缩进 24px，层级线 1px #E0E0E0 |
| C-20 | 叙事锚点 | 人物/引用→数据→意义 | 引用框（左边框 3px 品牌色，背景 #F8F9FA，padding 16px）|

### 3.2 高级节拍（6 种新增，基于认知心理学）

| 编号 | 名称 | 理论依据 | 关键参数 |
|------|------|---------|---------|
| C-21 | 视觉动线引导 | Gestalt 连续律 + 眼动热力图 | 引导线 1-2px，曲率半径 ≥ 100px，终点锚定数字强化 |
| C-22 | 渐进揭示 | Progressive disclosure 认知负荷理论 | 信息分 3-4 层展开，每层延迟 300-500ms，动画时长 200-300ms |
| C-23 | 视觉停顿 | Fixation points 眼动研究 | 停顿点直径 8-12px，间距 ≥ 画布宽度 15%，微妙阴影或边框 |
| C-24 | 负形强调 | Negative space 设计原理 | 负形占比 ≥ 40%，主体周围留白 ≥ 主体面积的 50% |
| C-25 | 动态模糊引导 | Motion blur 注意力捕获 | 模糊半径 2-4px，方向向量 15-30°，速度感暗示 |
| C-26 | 层级缩进 | Visual hierarchy 层级感知 | 每层缩进 24px，字体递减 2-4pt，颜色明度递减 10% |

### 3.3 节拍组合禁忌

| 禁止组合 | 原因 |
|----------|------|
| C-04 × 2（同图多个） | 多个"重点" = 无重点 |
| C-01 × >3（同区） | 视觉打架，无主次 |
| C-06 + C-04 | 冲突：哪个是异常？哪个是重点？|
| C-11 + C-17（同元素） | 隔离已足够，缩放冗余 |
| C-03 + C-15 | 排序与流动都暗示序列，冲突 |

### 4.3 Wilke 叙事结构原则

| 结构类型 | 格式 | 适用场景 |
|---------|------|---------|
| Opening–Challenge–Action–Resolution | 开场→挑战→行动→解决 | 经典四幕故事 |
| Lead–Development | 结论前置→细节展开 | 新闻式汇报 |
| Lead–Development–Resolution | 结论→发展→解决 | 短篇新闻 |
| Action–Background–Development–Climax–Ending | 行动→背景→发展→高潮→结局 | 情感连接优先 |

### 4.4 多图叙事数量建议（Wilke）

| 媒介 | 建议图数 |
|------|---------|
| 博客/社交媒体 | 1 张图 |
| 科学论文 | 3-6 张图（超过入附录）|
| 小时级演讲 | 3 个独立故事，每故事 3-6 张图 |
| 书籍/论文章节 | 每章/节一个故事，3-6 张图 |

### 5.3 "为将军们做图"原则（Wilke）

> "将军们很忙，不会花 30 分钟破解一张神秘图表。他们需要立即看到项目如何成功、如何超越以往能力。"

**开场图设计要点**：
- 立即传达核心信息（3 秒规则）
- 避免过度复杂（多维度 ≠ 好图）
- 如果复杂不可避免，先展示简化版
- 标题必须是结论性的，不是描述性的

## 六、全局设计规范

### 6.1 间距系统（8px 网格）

| 用途 | 值 |
|------|-----|
| 元素内紧密间距 | 4px / 8px |
| 卡片内间距 | 12px / 16px |
| 卡片/图表间距 | 16px / 24px |
| 区块间距 | 24px / 32px / 48px |
| 页边距 | 32px / 40px / 48px |

### 6.2 字号层级

| 层级 | 字号 | 字重 | 用途 |
|------|------|------|------|
| Hero | 48-72px | 700 | 开场锚定数字 |
| H1 | 32-40px | 700 | 页面主标题 |
| H2 | 24-28px | 600 | 区块标题 |
| H3 | 18-20px | 600 | 图表标题 |
| Body | 14-16px | 400 | 正文/轴标签 |
| Small | 11-12px | 400 | 注释/图例 |
| Caption | 10px | 400 | 脚注/来源 |

### 6.3 颜色规范

| 用途 | 值 |
|------|-----|
| 数据墨水比 | ≥ 0.7（Tufte）|
| 色彩弹出饱和度 | ≥ 80% HSL |
| 背景灰度 | 30-50% |
| 正常范围阴影 | #F0F0F0，透明度 50% |
| 异常/警示 | #E74C3C |
| 正增长 | 绿色系 |
| 负减少 | 红色/橙色系 |

### 6.4 轴线规范

| 元素 | 规范 |
|------|------|
| 基准线 | 1.5px 虚线 #999 或实线 #666 |
| 轴线 | 1px #E0E0E0 |
| 网格线 | 0.5px，透明度 20-30% |
| 刻度标签 | 11-12px |

## 九、质量检查清单

### 9.1 构图检查

- [ ] 画布比例符合内容需求
- [ ] 视觉重心位置正确
- [ ] 间距符合 8px 网格系统
- [ ] 小多图矩阵坐标轴一致（Wilke 关键原则）
- [ ] 复合图视觉语言统一

### 9.2 节拍检查

- [ ] 节拍组合无冲突
- [ ] 高亮 ≤10%
- [ ] 标注写原因+幅度，不写数字
- [ ] 颜色是信号不是装饰

### 9.3 叙事检查

- [ ] 弧线结构清晰
- [ ] 开场骨架与弧线匹配
- [ ] 每张图一个核心洞察
- [ ] 标题是结论不是描述

### 9.4 Wilke "将军测试"

- [ ] 3 秒内能否看懂核心信息？
- [ ] 是否过度复杂？
- [ ] 是否为"将军们"简化？
- [ ] 是否先展示简化版再展示复杂版？

## 十、使用流程

```
Step 1: 意图识别
    ↓ 确定要表达什么（趋势/对比/占比/关系...）
Step 2: 选择叙事弧线
    ↓ 从 11 种弧线中选最适合的
Step 3: 设计开场骨架
    ↓ 从 13 种骨架中选第一张图
Step 4: 规划构图序列
    ↓ 每张图选择 20 种构图模板之一
Step 5: 配置视觉节拍
    ↓ 每张图选择 26 种节拍组合
Step 6: 应用设计规范
    ↓ 间距/字号/颜色/轴线规范
Step 7: Wilke 质量检查
    ↓ "将军测试" + 清单逐项核对
```
