## 搜索成本控制

**原则**：先定位，再读取，最小化Token消耗

```bash
# ❌ 错误：直接读全文件
Read 04-MEMORY.md  # 2000+ 行

# ✅ 正确：先grep定位
grep "供应商管理" 04-MEMORY.md  # 找到行号
Read 04-MEMORY.md offset=100 limit=50  # 只读相关部分
```

**成本对比**：
| 方式 | Token消耗 | 准确率 |
|------|----------|--------|
| 读全文件 | ~2000 tokens | 100% |
| grep+Read部分 | ~200 tokens | 95%+ |
| **节省** | **90%** | 损失<5% |
