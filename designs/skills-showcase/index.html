01

jdme-local-tools

京ME 本地聊天数据只读工具——完整数据库访问，突破「只能读摘要」限制

关键突破：SQLCipher 解密 → 原始消息 JSON
数据深度
⭐⭐⭐⭐⭐
数据源
chat.data + SDK日志
平台
macOS + Windows
命令数
10
🔍
路径自动发现
discover 命令自动扫描 macOS / Windows 所有默认安装位置，推荐最活跃账户。
🔐
SQLCipher 解密
通过 CLI 或 Windows DLL fallback 读取加密 chat.data，密钥为 md5(account_pin)。
📋
日志快照恢复
从 sdk-event.log / renderer.log 恢复消息，数据库不可用时的可靠后备。
👁️
实时消息监听
watch 命令 tail 日志，解析消息推送为稳定 JSON 输出。
推荐工作流
discover→ sessions --name-contains→ chat-history --session

先发现路径 → 解析会话ID → 读取完整历史。日志不可用时回退到 messages 命令。

核心命令
# 路径发现
python3 scripts/jd_me_tool.py discover

# 日志恢复（推荐首选）
python3 scripts/jd_me_tool.py messages --auto --contact fanbaolin3 --limit 100

# 关键词搜索
python3 scripts/jd_me_tool.py chat-search --auto --keyword 上线 --limit 20

# 数据库完整历史
python3 scripts/jd_me_tool.py chat-history --auto --session erp1:ee:erp2:ee --limit 50

# 实时监听
python3 scripts/jd_me_tool.py watch --auto --keyword 上线 --at-me

# 日报摘要
python3 scripts/jd_me_tool.py summary --auto --since 2026-05-14
macOS 默认路径
/Applications/ME.app
~/Library/Application Support/com.jd.me
~/Library/Logs/ME
Windows 默认路径
~/Documents/JD/ME（chat.data）
%APPDATA%/ME/logs（renderer.log）
%LOCALAPPDATA%/Programs/ME
过滤能力
参数作用
--contact匹配 sessionId / senderPin / receiverPin / 内容
--session精确会话 ID 过滤（可重复）
--keyword内容子串匹配（可重复）
--at-me仅 @ 我的消息
--limit保留最近 N 条
安全边界：本工具不得修改京ME应用、注入 Electron、转储进程内存、写入聊天数据库。所有 chat-* 数据库查询均为只读。
架构分层
文件职责
jd_me_tool.py稳定命令入口
jd_me_tool_core.py共享数据访问、解析、CLI 行为（平台无关）
jd_me_tool_platform.pyOS 特定路径候选和默认二进制位置

关键突破：Headless CDP → 逐字稿全文
数据深度
⭐⭐⭐⭐⭐
数据源
JoyMinutes API
前置条件
Chrome / Tabbit / Edge
依赖
零（无 Playwright）
🧬
SSO 自动继承
复制本机浏览器 profile，自动继承 ERP 登录态，无需手动登录。
📝
逐字稿全文
--detail 返回 transcript 数组：speaker + text + beginMs + endMs。
🔄
浏览器自动回退
Chrome → Tabbit → Edge → 自动打开登录页，确保总能拿到数据。
📊
发言人统计
返回每人发言占比和时长，快速判断谁是会议核心参与者。
日期范围
用户说法参数
今天 / 未指定today（默认）
昨天yesterday
本周thisweek
上周lastweek
近 N 天lastNdays（如 last7days）

日期区间--start / --end
命令示例

# 指定日期 + 逐字稿

# 按会议 ID 精确查

  "text":   "合并后的发言内容",

返回字段
字段来源说明

url构建joyminutes.jd.com/video/{id}
总结规范：单日查询含：总览 → 每场详情（标题/时间/链接/参会人/内容概述/决策/待办）→ 综合要点。多日查询按日分组，额外含跨天主题归纳。不编造、缺失标注。

read-email

JoyMail 统一入口——全文 + 附件 + 文件夹管理，一个 SOAP 入口覆盖所有邮件场景

关键突破：EWS SOAP 协议 → 邮件正文 + 附件元数据
数据深度
⭐⭐⭐⭐⭐
命令数
19
协议
EWS SOAP Exchange2016
鉴权
SSO → JWT → 自动刷新
🔍
11 维筛选
文件夹、发件人、收件人、主题、正文、时间、已读/未读、标签、附件、重要级别、翻页深度。
✉️
回复 / 发送 / 转发
基于原邮件回复（含 reply-all、带原文、附件）；新建邮件支持抄送/密送。
📊
智能总结
search → summarize 管道：核心统计 + 总览表格 + 优先处理 + 观察建议。
🔒
二次确认保护
所有修改操作（send/reply/forward/batch-*）必须先展示摘要，用户确认后才执行。
关键规则
默认时间窗口
用户未指定日期时默认只查最近 1 天（按 DateTimeReceived）。folders 和 detail 不受此限。
收件人解析
用户给中文名或 ERP 时必须先调 lookup-recipient 解析邮箱，不猜邮箱、不跳过确认。
输出风格
结果优先、过程最小化——直接给结论/表格/摘要，不展开内部调用细节。
Token 策略
本地缓存 + 过期自动刷新 + 401 自动重试（最多3次）。SSO Token 优先从环境变量读取。

bash scripts/run.sh search --folder inbox --unread --limit 20
bash scripts/run.sh detail --item-id "AAMkADFlNG..."

# 写（需二次确认）
bash scripts/run.sh reply --item-id "..." --body "收到，今天处理。"
bash scripts/run.sh send --to "a@jd.com" --subject "项目同步" --body "..."

# 分析

# 批量

personal-insight-summary

多源信息融合洞察——把聊天、会议、邮件串成完整故事线

关键突破：跨源关联分析 → 结构化简报
数据深度
⭐⭐⭐⭐
数据源
京ME + 邮件 + 慧记
分析步骤
3 步流水线
输出板块
固定 6 个
三步工作流
1. 并行采集三源数据→ 2. 整合分析（11 步流水线）→ 3. 固定 6 板块输出
数据源配置
用户通过 JSON 指定 sources（skillId + prompt）。未提供时默认采集今天的京ME + 邮件 + 慧记。时间范围从各 source 的 prompt 中识别。
整合分析管线
时间过滤 → 京ME指定群过滤 → 人名映射 → 身份解析 → 相关性分级（强/中/弱） → 项目聚类 → 优先级判断 → 决策提取 → 分歧识别 → 风险识别 → 待办生成
固定 6 板块输出模板
昨天你主要参与了 {N} 个项目：
项目名 + 摘要上下文（时间/人名/来源/内容/参与证据）。每项 ≤5 条上下文。
今天建议优先处理 {N} 件事：最多 5 条
具体行动 + 原因 + 当前状态。
重点关注：
值得持续关注的事项、进展或协作动态。
风险提醒：
风险描述 + 建议处理方式 + 影响范围。
已定决策：
决策内容 + 来源/提出者/最终结论。
待定问题：
未解决问题/分歧/待确认事项 + 各方观点 + 当前卡点。
核心约束
  • 不编造人名、项目名、事件。项目名用原始叫法。
  • 只输出「强」证据要点；中/弱仅作内部分析，不得进入最终输出。
  • 严格使用固定 6 个一级标题，不新增/删除/改名。
  • 无内容板块写「暂无」，不省略。
  • 摘要上下文保留原话关键词和语气，不过度书面化。

jingME

承诺追踪与待办提取——自动识别「谁承诺了什么、截止何时」

关键突破：责任闭环追踪——从聊天和转写中自动识别承诺
数据深度
⭐⭐⭐⭐
数据源
京ME + 慧记
线索词
3 类 × N 个
状态分类
4 种
🎯
承诺线索识别
三类线索词：承诺（我来/我负责/明天发你）、截止（今天/周五/上线前）、交付（发你看/提报/上线）。
📋
结构化跟进清单
逐条分类：负责人、事项、截止时间、来源（京ME/慧记）、证据引用、状态、建议提醒文案。
🔗
双源交叉验证
京ME 聊天 + 慧记转写联合分析，会议中口头承诺 + 群聊中文字确认互相印证。
🛡️
只读安全边界
不发送消息、不修改数据库、不提取凭据。用户要求发送时生成草稿文案。
五步工作流
1. 明确范围→ 2. 采集京ME证据→ 3. 采集慧记证据→ 4. 提取承诺→ 5. 输出清单
三类线索词
承诺线索
我来 · 我跟 · 我负责 · 我推进 · 我今天弄 · 明天发你 · 下周给 · 会同步 · 已收到
截止线索
今天 · 明天 · 周五 · 下周 · 7月 · 上线前
交付线索
发你看 · 给谁过目 · 宣讲 · 提报 · 排期 · 上线 · 确认 · 补充数据

unclear不明确承诺存在但时间/范围不清晰
