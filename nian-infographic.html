Chapter 01 · 策展诊断

你的 H 系列教我重新认识"大气"

读完三个文件后的系统性对比——每个差异都对应一个具体的设计选择错误。

128
Section 纵向间距
H062 用 --s-5xl: 128px
v1 用 --s-4xl: 96px
360
Ghost 大字尺寸
H062 用 360px Playfair
v1 无 ghost 层
8:1
Hero 字号比
H001 72px / body 16px = 4.5:1
目标 ≥ 8:1
Issue 01 · 间距过紧
v1 数据：section padding 96px，组件 gap 24px，hero padding 80px
H 系列数据：section padding 128px，hero padding 128px，组件 gap 48px
差 32px = 一层呼吸的差距。紧凑不等于精致，紧凑等于赶。
Fix 01 · 拉开
section → 128px 纵向（+33%）
组件 gap → 48px（+100%）
水平 padding → 96px（+200%）
Issue 02 · 衬线缺席
v1 数据：Playfair 仅 Hero 标题
H 系列数据：Playfair 出现在 Hero、section 标题、卡片编号、引号、ghost 大字、meta 数值
衬线 + italic 是 H 系列的"语气器官"，v1 把它限制在 Hero，丢了整个语感层。
Fix 02 · 扩展 Playfair 使用域
Hero → Playfair（保留）
section 标题 → Playfair italic
卡片编号 → Playfair（56px, sand opacity 0.4）
数值展示 → Playfair（如 36px KPI）
Issue 03 · 无背景层
v1 数据：所有元素在前景。页面只有一层深度。
H 系列数据：全部有背景层。H062 的 hero-letter 360px、H001 的 hero-year 180px、H061 的 hero-ghost 200px
背景层 = 空间感。没有它，页面是平的。
Fix 03 · 插入幽灵层
Hero → 360px Playfair ghost letter
Section → 每个模块背景 + 140px 透明编号
装饰点阵 → 从 H001 借 4px dot pattern
Issue 04 · 组件无定制
v1 数据：Bento Grid, Widget KPI, Asymmetric Table 直接从 COMPONENTS-MASTER 复制
H 系列数据：每个组件都是为所在页面定制的。Principles Scroller、Color Bar、Tension Strip 有独特视觉身份。
"组件库"应该提供灵感，不是终稿。
Fix 04 · 按表达需求选形态
并列 → Principles Scroller（非 Bento）
展示 → KPI Bar（非 Widget）
对比 → Tension Grid（非 Asymmetric Table）
色板 → Color Swatch（从 H001 继承）

Rule A
先问"这个内容要表达什么关系"——并列/对比/排名/占比/时间——再选形态。
Rule B
选完形态后读 H 系列——找风格相近的 section，提取间距/字体/ghost 用法。
Rule C
从 COMPONENTS-MASTER 取基础代码，但必须按 H 风格调样式（间距加倍、加 ghost、改 Playfair）。
Chapter 04 · 新流程

施工流水线 2.0 · 从复制到翻译

核心变化：不再从组件库"取"，而是从 H 系列"翻译"——判断标准变了。

Step 01 · 分析
读内容，定表达需求
这段内容是并列/对比/排名/占比/时间/叙事？写出明确的一句话。如："这 6 个设计原则是并列关系，不是主次关系"。
Step 02 · 查形态
COMPONENT-FORMS.md → 确定候选形态
匹配表达需求 × 可用形态。写出 2-3 个候选，并备注各形态的优劣。如："并列可选：Principles Scroller（推荐）vs Bento Grid（主次错位）"。
Step 03 · 回 H 系列
读 H 文件找风格参考
打开 H001/H061/H062，找到类似内容的 section。提取：间距值、字体角色、ghost 用法、颜色用法。把这些参数记下来。
Step 04 · 取代码 + 调样式
从 COMPONENTS-MASTER 取基础代码，按参考参数调整
复制基础 HTML/CSS → 替换文字 → 按 Step 03 的参数调整间距/字体/ghost → 确保 Playfair 用于装饰层。
Step 05 · 校验
过 9 条硬规则 + 3 条新自检
新自检：
① 眯眼看 — 是否有 1/3 是空白？
② 是否缺少背景层（ghost 字/装饰编号）？
③ Playfair 是否至少出现在了 3 个地方？
Chapter 05 · 新标准

"大气舒展"自检 3 条

基于 H 系列提取的补充规则——应该写入 CRAFT-RULES.md 的 Step 4。

S
Space
呼吸检查
每个 section 眯眼看——至少 1/3 是空白。如果感觉"满"，间距不够 128px 或组件 gap 小于 48px。
D
Depth
背景层检查
页面除了前景，至少有一个"背景层"——ghost 大字、装饰编号、点阵纹理。至少有 3 层垂直深度。
T
Tone
衬线语气检查
Playfair 至少出现在 3 处：Hero、section 标题、数据数值或引号。如果只有 Hero 用了衬线，页面会冷。
