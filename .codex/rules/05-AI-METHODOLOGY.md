# 05-AI-METHODOLOGY.md — AI协作方法论

> 核心理念：**AI越强，越怕你没想清**

---

## 核心认知

**"地图不等于疆域"**

你提供给AI的提示词、技能、上下文只是待完成工作的抽象地图，而非真实的工作本身。

多数职场人使用AI翻车的根源集中在开工前的前3分钟：没有把任务的背景、标准、限制、默认规则向AI表述清楚，高能力AI会自动用通用经验补全信息空白，最终产出不符合真实职场需求的结果。

---

## 任务信息四象限

| 分类 | 英文 | 定义 | 示例 |
|------|------|------|------|
| **已知已知** | Known knowns | 已经明确写进需求里的信息 | 截止时间、汇报对象、交付物形式 |
| **已知未知** | Known unknowns | 清楚自己还没确定的问题 | 项目预算、数据口径、老板偏好 |
| **隐性已知** | Unknown knowns | 心中默认但从未告知AI的潜规则 | 汇报风格不追求花哨、老板先看结论 |
| **彻底盲点** | Unknown unknowns | 完全没有预想到的潜在风险 | 需法务审核、数据禁止外发、指标口径冲突 |

---

## 工作流程

### 一、开工前：5问盲点扫描

不要直接让AI生成方案，先发送以下提示词完成前置信息对齐：

```
我正在做【任务】，目标是给【对象】看，最后要交付【结果】。
我目前知道的信息有：【材料1、材料2、材料3】。
我不熟的地方有：【行业/数据/流程/工具/老板真实意图】。
1. 请先帮我做一次盲点扫描：
2. 我可能漏掉了哪些关键信息？
3. 哪些问题会影响最终方案方向？
4. 你需要先问我哪些问题？
5. 哪些地方可以先假设，哪些地方需要我确认？
请先不要直接产出正文，先帮我把问题问清楚。
```

**价值**：避免AI用通用经验补全信息空白，产出不符合实际需求的结果

---

### 二、执行中：维护决策记录表

要求AI同步维护决策记录，覆盖4个核心维度：

| 维度 | 说明 |
|------|------|
| 信息来源 | 我根据什么信息做了这个判断 |
| 待确认 | 哪些数据还没确认 |
| 临时假设 | 哪些内容是临时假设 |
| 待拍板 | 哪些地方需要你最终拍板 |

**价值**：后续面对老板追问时，可以清晰说明每一个判断的信息来源，区分已确认内容和待补充数据

---

### 三、完工后：小测验验证掌握度

AI完成复杂任务后，让它生成说明文档和针对性测试题，确保你完全理解AI产出内容。

提示词：

```
请把上面的方案改成一页汇报说明，说明目标、关键判断、风险、待确认信息。然后出5个问题测试我是否真正看懂。题目要围绕老板可能追问的点。
```

**判断标准**：如果无法顺利答出测试题，说明这份AI产出还没有转化为你自己的可交付成果

---

## 反常识要点

| 要点 | 说明 |
|------|------|
| **提示词并非越长越好** | 写得太细AI会死守你的错误方向，写得太空AI会直接套用行业通用做法 |
| **先问清楚再动手** | 开工前5分钟的信息对齐，比返工5小时更高效 |
| **隐性已知是最大风险** | 你心中默认成立但从未告知AI的职场潜规则，往往是翻车的根源 |

---

## 适用场景

### 优先适用的4类高频任务

1. 老板临时布置的"一句话写方案"任务
2. 零散会议纪要整理为结构化行动清单
3. 大量原始资料梳理为标准汇报框架
4. 快速学习陌生工具或全新行业知识

### 懒人极简三步法

1. 把任务交给AI前，先明确告知AI：我知道什么，我不确定什么
2. 让AI执行盲点扫描，只筛选出会直接改变最终结果的核心问题
3. AI产出内容后，让它生成5道针对性测试题，确认你能完整讲清所有细节

## 与现有体系的整合

| 现有理念 | 对应方法论 |
|----------|------------|
| **三问原则**（有规可依？合理？充分共识？） | 开工前盲点扫描的简化版 |
| **先对齐再动手** | 5问盲点扫描的核心逻辑 |
| **拦截性规则** | 执行中决策记录表的自动化版本 |
| **乔哈里协作模型** | 四象限分类的沟通应用 |
