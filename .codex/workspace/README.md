# Codex 工作空间

> AI协作的专属目录，分工清晰互不干扰

---

## 目录结构

```
.codex/workspace/
├── README.md           # 本文件
├── Rules/              # AI交互规则文件
│   ├── about-me.md     # 职位身份、工作标准
│   ├── write-style.md  # 输出风格规范
│   ├── work-detail.md  # 核心工作内容
│   └── ai-methodology.md # AI协作方法论
├── Outputs/            # AI生成的交付文件
└── Files/              # 待处理的原始工作文件
```

---

## 目录说明

### Rules/

存储定义个人工作标准、输出风格、协作方法的配置文档。

| 文件 | 用途 | 读取时机 |
|------|------|----------|
| `about-me.md` | 个人职位身份、岗位对应的工作标准 | 启动时 |
| `write-style.md` | 期望AI输出内容的风格规范 | 启动时 |
| `work-detail.md` | 个人日常的核心工作内容边界 | 启动时 |
| `ai-methodology.md` | AI协作方法论（盲点扫描、决策记录、验证） | 复杂任务前 |

---

### Outputs/

存储AI生成的交付文件。

| 类型 | 示例 |
|------|------|
| 文档 | 制度规范、SOP、方法论 |
| 报告 | 周报、月报、分析报告 |
| 材料 | 培训材料、汇报PPT、公文 |
| 代码 | 脚本、工具、自动化流程 |

**命名规范**：`YYYY-MM-DD-项目名-文件名`

---

### Files/

存储待处理的原始工作文件。

| 类型 | 示例 |
|------|------|
| 数据源 | 待核对的表格、待分析的数据 |
| 文档 | 待翻译的文档、待整理的材料 |
| 素材 | 待加工的图片、待引用的资料 |

**处理后**：文件留在Files/，产出物放Outputs/

---

## 使用规则

1. **Rules/** — 只读，由用户维护，AI启动时自动加载
2. **Outputs/** — AI写入，用户查阅，定期归档
3. **Files/** — 用户放入待处理素材，AI读取后产出物放Outputs/

---

## 核心方法论

### "地图不等于疆域"

你提供给AI的提示词、技能、上下文只是待完成工作的抽象地图，而非真实的工作本身。

### AI越强，越怕你没想清

开工前的信息对齐比返工更高效。隐性已知（你心中默认但从未告知AI的潜规则）是翻车的最大风险源。

### 三步工作法

1. **开工前**：5问盲点扫描，把问题问清楚
2. **执行中**：维护决策记录表，区分已确认/待确认/临时假设/待拍板
3. **完工后**：小测验验证掌握度，确保产出转化为自己的可交付成果
