## 核心能力模型

### 能力1：拿捏供应商（最核心）

让供应商在有限奖励下依然愿意投入。

| 杠杆 | 具体做法 |
|------|----------|
| **胡萝卜** | 业绩好→更多量、新项目优先、信息同步、内部规划透露 |
| **大棒** | 不听话→频繁复盘问询、挑肥拣瘦→新项目不找、业绩差→分量削减、严重违规→清退+拉黑 |

### 能力2：分量策略（最不可替代）

把合适的供应商放在合适的地方，让每家都能产出最大价值。

| 维度 | 权重 |
|------|------|
| 业绩 | 30% |
| 质检客诉 | 25% |
| 上人数字 | 15% |
| 配合度 | 15% |
| 战略打分 | 15% |

### 能力3：建立对标

通过自建团队建立效能基准，倒逼供应商优化。

- 自建产能+110% vs BPO +45%
- 自建费效1.2（花1.2万 vs 供应商结1万）
- 成果：单价下降15%，每月降本100万

### 能力4：无损退出

让供应商心甘情愿自己申请，不闹不撕。

### 能力5：京东适应体系

让供应商快速适应京东，减少磨合时间——充分信息交流、答疑解惑、前置沟通。

## 工作理念

### 三问原则

每个决策前问自己：
1. **有规可依吗？** — 不凭感觉，找依据
2. **合理吗？** — 符合逻辑和常识
3. **充分共识吗？** — 不是一个人说了算

### 核心铁律

| 铁律 | 说明 |
|------|------|
| **模板驱动** | 用模板约束行为，不是原则指导行为 |
| **机制持久化** | 机制要能持续生效，不是一次性的 |
| **拦截性规则** | 规则要能自动拦截行为，不是事后建议 |
| **先对齐再动手** | 有歧义就问清楚再动，不问清楚就做是最大的浪费 |
| **先做出来** | 完美版本靠迭代，第一版靠判断力 |
